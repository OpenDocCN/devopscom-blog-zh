# FinOps 101:揭开 DevOps 云财务运营的神秘面纱

> 原文：<https://devops.com/finops-101-demystifying-cloud-financial-operations-for-devops/>

在理想情况下，所有组织都应该有一个专门的 FinOps 团队——或者至少有一个专家可以确保云以财务上明智的方式运行。当然，在现实中，大多数组织都缺乏技能，并且都在运营一个责任被广泛分配给员工的精益团队。在这种情况下，DevOps 专业人员很可能会被留下来负责 FinOps，并负责将 FinOps 最佳实践实施到他们的日常云操作中。如果这听起来很熟悉——这是你的入门指南。

## 考虑财务计划的文化转变

从很多方面来说，DevOps 开始考虑 FinOps 并不是一个很大的飞跃。毕竟，这两种战略的根本目标都是优化、改进和简化运营。不同的是，DevOps 侧重于软件开发和技术技能，而 FinOps 则着眼于在云上运营的财务成本。

最大的挑战是，有时这些目标可能相互矛盾。如果 DevOps 团队的任务是尽快获得新的软件版本和修复，这通常会以最具成本效益的解决方案为代价。例如，考虑一下云中的 AWS 随需应变实例。可以说，寻找预留实例、节省计划和承诺使用折扣，并在可能的情况下提供合适的机会，对 FinOps 更友好。然而，在 DevOps 团队保持敏捷的发布周期和提供快速修复的压力下，任何阻碍他们灵活性和自由的事情都是值得的。

因此，对于工程师和 DevOps 团队来说，首先考虑 FinOps 可能是一个真正的范式转变，特别是如果他们已经觉得自己的工作已经满了。这里有一些最佳实践，可以帮助你克服前进道路上的任何困难。

## 让工程师负起责任

不要简单地增加新的负责成本管理和财务运营的利益相关者，因为他们会很快成为你的工程师的对手。相反，要确保你的技术团队和开发人员意识到他们要对他们的云成本负责。他们需要知道他们在使用什么，他们需要整理和记录他们如何更有效地使用他们的预算。

当然，业务在这方面也有作用，向工程师展示 FinOps 的最佳实践，如关闭闲置资源、调整实例大小或做出明智的架构决策。

## 解释 FinOps 不是一个削减成本的计划

如果到目前为止 DevOps 团队已经得到了一张相当于空白支票的东西，那么开始收紧钱袋可能会很难。向 DevOps 团队解释您不打算削减他们的预算。事实上，你甚至可以增加它！

FinOps 完全是关于*更智能的*财务运营，所以如果工程师能够证明他们可以在一个方向上省钱，他们就可以将钱用于让他们满意的新功能、技术和工具。让 DevOps 团队控制 FinOps 决策提供了更多的自主权和控制权——这是一件好事。

## 用唾手可得的水果减轻精神负担

一旦 DevOps 团队获得了对其云使用的急需的可见性，并意识到他们正在获得对其行为的责任和自主权，您希望引导他们认识到他们可以在哪里做出改变。用相同的 KPI 挑战您所有的 DevOps 利益相关者，看看谁能想出创新的方法将成本节约融入到日常业务中。一些 KPI 可能是一个良好的开端，包括:

*   **覆盖:**使用预约实例、储蓄计划、现货实例或承诺使用折扣。
*   **调整规模:**缩减规模或终止闲置或未充分利用的实例的机会价值。
*   **孤立/空闲状态:**跟踪孤立 EBS 卷或快照的百分比；您可以深入到特定的时间段，例如一个月以上。
*   **预测指标:**考虑使用预测账单和实际成本之间的差异，并尽可能减少差异。
*   **不正确的分配:**这将突出显示过大或不正确的实例类型、错误层上的 S3 或 EBS 存储、未连接的弹性 IP 或标记/未标记的资源。
*   **高峰时段:**有多少百分比的开发资源在高峰时段运行，或者在可以关闭的时候运行；比如周末？

## 自动化以不干涉的方式结合了 FinOps 的最佳实践

如果这听起来像是一份全职工作，你没有错。如果您想知道 DevOps 团队什么时候真正开始开发和构建，这是可以理解的！为了保持工程师和技术团队的生产力，请确保您提供了正确的工具和技术，以尽可能减轻负担，特别是在自动化和人工智能可以比一屋子 DevOps *或* FinOps 专家做得更好的地方。

一个例子是实时自动购买和销售预订的实例，这样您就永远不会按需购买。另一个是分别自动删除闲置或未充分利用的资源或调整其大小。当你的 DevOps 团队看到你提供的技术使 FinOps 变得简单时，他们更有可能在其他地方做他们的工作。

## DevOps 和 FinOps

近年来，安全团队支持 DevOps 在安全最佳实践上“左移”的想法，以便在构建阶段将安全性包含在敏捷软件开发的过程中。

好吧，你首先听到的是:是时候为财务和成本优化进行类似的转变了。

正如工程师在工作中不能忽视安全性一样，财务优化需要成为 DevOps 团队日常工作的核心。为了实现这一点，为了让 DevOps 无缝整合 FinOps 目标，企业需要支持云使用的可见性、成本优化文化以及将最佳实践融入日常工作的强大自动化基础。
