# 开发人员想要什么:针对当今开发人员的安全培训

> 原文:[https://devo PS . com/what-a-dev-wants-security-training-for-todays-developer-work force/](https://devops.com/what-a-dev-wants-security-training-for-todays-developer-workforce/)

众所周知，发展强大的安全文化是生产安全产品的重要组成部分，但组织经常忽略这样一个事实，即他们的安全培训计划为这种文化奠定了基础。谈到[开发人员](https://www.securitycompass.com/blog/three-important-steps-to-launching-a-successful-security-training-program/)培训，我们的目标应该是让学习者的内在动机与公司的战略和目标保持一致。但是，如果这两者中的任何一个出现偏差，培训计划，最终是组织，都会受到影响，因为它要么是不相关的，要么被视为需要检查的强制框。

### 解决开发人员和组织的利益

不幸的是，在购买者角色和开发人员之间有一条鸿沟，即使开发人员倾向于对他们想要的培训方式有一个相当一致的反应。与其实施一个注定要被视为耗时的法规遵从性要求的计划，组织更应该问这个简单而相关的问题:开发人员真正想从他们的安全培训计划中得到什么？

当谈到今天的开发人员安全培训计划时，基于[法规遵从性的](https://devops.com/need-for-increased-training-strained-budgets-strain-enterprise-it-teams/)培训和以开发人员为中心的培训之间经常有所区别；换句话说，组织的利益并不总是与开发人员的利益一致。在大多数高度管控的组织中，包括开发团队在内的所有员工都必须每年参加一次强制性的安全培训课程。这导致了一个漫长而痛苦的过程，让员工尽可能快地点击模块。如果开发人员希望在他们的代码中实现安全主题，那么他们可能需要定期回去查看这些主题。

虽然开发人员肯定在努力遵守安全性要求，但这大大降低了以安全有效的方式将产品推向市场的能力。

### 开发者真正想要的培训是什么？

组织应该通过询问那些开发人员实际上在寻找什么来优先考虑专门针对他们的开发团队的项目。市场上有如此多的培训项目，组织可能很难确定哪一个最能引起员工的共鸣，特别是他们的开发团队。事实是，当涉及到安全培训时，大多数开发人员希望尽可能少地进行培训。出于合规性目的，他们愿意勾选复选框，但他们希望尽可能做到无缝。随着训练策略的复杂性不断发展，理想的课程是简单的，就像可以嵌入开发人员日常使用的工具中的程序。

最有效的培训项目是吸引人的和相关的。经理们倾向于从实践、互动、参与和与开发人员任务相关的培训中看到更好的结果。游戏化也是吸引开发者的好方法。例如，组织已经开始用排行榜来活跃他们的培训课程。事实上，最近的研究表明，开发者会选择共享公司排行榜，并根据消费的内容进行评分。这些程序不仅吸引了开发人员，而且非常有效，因为它们通过利用个人纯粹的竞争驱动力来激励团队查看更多的内容。

### 赢得安全冠军的计划

组织还可以实施安全冠军计划。这些计划因组织而异，但最简单的形式是，安全冠军计划从每个团队中指定一名“冠军”。然后，冠军的任务是在安全方面作为主题专家领导团队。这里要记住的一点是，如果操作不当，这个程序可能会是一个重大的失败。请记住，您不希望将任何人指定为团队安全冠军；您希望确保您选择的是一名敬业的员工，该员工将优先考虑安全实践，并主动在整个团队中传达最新信息。

不管一个组织选择什么途径，游戏的名字是确保开发团队积极地参与相关的和现实的培训计划。其结果将对下游产生重要的影响，因为组织正在努力建立一种在安全方面具有坚实基础的文化。毕竟，在开发一个成功且安全的解决方案时，“检查盒子”只能做到这一步。
