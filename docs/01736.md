# Red Hat 启动 Ansible-Native 容器工作流项目

> 原文:[https://devo PS . com/red-hat-launchs-ansi ble-native-container-workflow-project/](https://devops.com/red-hat-launches-ansible-native-container-workflow-project/)

RALEIGH, N.C.–(Business Wire)–Red Hat, Inc. (NYSE: RHT), the world’s leading provider of open source solutions, today announced the launch of Ansible Container under the Ansible project, which provides the leading simple, powerful, and agentless open source IT automation framework. Available now as a technology preview, Ansible Container allows for the complete creation of Docker-formatted Linux containers within Ansible Playbooks, eliminating the need to use external tools like Dockerfile or docker-compose.While standardization is emerging for container formats in the form of the Open Container Initiative, the tooling space is still fragmented, with developers often forced to work outside of the enterprise workstream to create and deploy containerized applications. Ansible Container changes this dynamic, by bringing Linux container development into the existing workstream through the easy-to-consume Ansible Playbook, adding cloud-native application development side-by-side with traditional application building.

除了 Ansible Container，Ansible 项目还推出了新的 Kubernetes 模块，允许直接从 Ansible 剧本生成 Kubernetes 模板。结合这两种新产品，IT 团队现在可以直接从剧本中构建 Linux 容器，并以更简化和高效的方式部署到基于 Kubernetes 的容器应用平台，如 Red Hat OpenShift。

Ansible 使开发人员和 IT 运营人员能够更轻松、更快速地部署 IT 应用和环境，使他们能够通过自动化网络配置、云部署和开发环境创建等日常活动来消除 IT 团队之间的障碍。Ansible 的模块化代码库，加上贡献的便利性，以及 GitHub 中的贡献者社区，使强大的 IT 自动化平台能够管理当今的基础设施，同时适应新的 IT 需求和 DevOps 工作流。

借助 Ansible Container 的功能，用户将能够使用 Ansible 行动手册自动化他们现有的基础设施和网络，同时使用额外的行动手册将他们的应用程序容器化并部署在容器应用程序平台上。这可以通过保护现有的 IT 投资和关键应用程序/系统来实现运营的连续性，而不会牺牲云原生应用程序和大规模容器编排形式的创新。

Ansible factor 提供的自动化容器创建和部署到 Red Hat 现有的容器基础设施堆栈中，现在包括:

*   Red Hat Enterprise Linux Atomic Host 中稳定的、以容器为中心的操作系统。
*   通过 **Red Hat OpenShift** 和最近宣布的下一代 [OpenShift 在线公共云服务](http://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fwww.redhat.com%2Fen%2Fabout%2Fblog%2Fnext-generation-openshift-online-here&esheet=51365602&newsitemid=20160620005670&lan=en-US&anchor=OpenShift+Online+public+cloud+service&index=1&md5=d3194e3ca5ef17f78d33ab42da6f8699)的企业级、Kubernetes-和 Docker-native 容器应用平台。
*   借助 **Red Hat CloudForms、Red Hat Insights、Red Hat Satellite** 和 Red Hat 的 **Ansible Tower，跨混合环境进行基础设施管理、自动化和监控。**
*   通过 **Red Hat OpenStack Platform** 和 **Red Hat Cloud Suite** 支持大规模容器部署的可大规模扩展的私有云和混合云架构，其中还包括 Red Hat OpenShift。

Red Hat 的容器堆栈提供了完全开放的企业级体验，从构建云原生应用的开发人员到管理和监控各种混合应用和环境的运营团队。Red Hat 的容器组合完全基于上游、社区驱动的项目构建，提供了驱动下一代企业 IT 的创新，同时还允许保留关键任务的现代和遗留系统。
