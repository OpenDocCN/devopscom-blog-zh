# Web 3.0 能解决 Linkrot 危机吗？

> 原文：<https://devops.com/can-web-3-0-solve-the-linkrot-crisis/>

网络上的数据比你意识到的更加短暂。哈佛法学院最近对 linkrot 和内容漂移进行了一项调查，发现随着时间的推移，基于网络的内容越来越少。通过分析《纽约时报》文章中的 50 万个链接，发现 2018 年的 6%深度链接无法访问，1998 年的 72%链接完全死亡。

腐烂的网络可能会产生严重的后果，影响社会的集体理解。考虑在法律过程中用来证实关键论点的链接。现在的研究表明，最高法院判决中 49%的超链接不再起作用。linkrot 的情况破坏了互联网曾经承诺的人类共享智能。

网络上的很多内容都是过时的、不准确的或者已经改变了它的 URL 目的地，这个问题被称为内容漂移。因此，像[way back Machine internet archive](https://archive.org/web/)这样的创可贴已经成为历史网络档案的管家。然而，该行业的一些支持者认为，我们需要一个更现代、分散、不变的历史数据参考。你可能已经猜到了这篇文章的主题——区块链。

我最近会见了协议实验室的 Jonathan Victor，看看 Web 3.0 和区块链如何被用来解决 linkrot 的流行。根据 Victor 的说法，引用规范 URL 的智能合同可以帮助创建一个更有弹性的网络。实际上，这需要一个对等模式，共享所有权和激励模式来维持内容的寿命。

## 林克罗特的状态

数据存放的时间越长，失去跟踪的可能性就越大。被遗忘的数据库和不断上升的数据存储成本很容易导致整个网站网络和用户数据被意外(或故意)破坏。维克托解释说，对私人公司无限期保存数据如此信任是不可行的。

公开数据并不像你想象的那样公开。旧的、用户生成的内容通常是第一个倒下的多米诺骨牌。例如， [MySpace](https://www.zdnet.com/article/myspace-lost-13-years-worth-of-user-data-after-botched-server-migration/) 上的大量媒体在 2015 年一次拙劣的服务器迁移中被删除。2021 年，[雅虎问答](https://www.businessinsider.com/deleting-yahoo-answers-disastrous-idea-preserve-our-digital-record-2021-4)在其关闭过程中同样删除了 16 年的用户生成内容。令人惊讶的是，用户对他们在论坛和社交媒体上的数据几乎没有所有权或控制权。例如，禁止 Twitter 可能会导致帖子中的非活动链接。

维克多说，当一篇文章指向一个网站时，这就隐含了对外部组织的依赖，以保持域名的活力。第二个依赖是域所有者保持内容不变。“如果我们可以谈论内容以及如何以不同的方式引用它，也许我们可以创建更具弹性的引用，”Victor 建议道。

## Web 3.0 如何提供帮助

解决 linkrot 问题的一个方法是为 web 资源创建一个“指纹”。这可能是对特定内容的规范引用。Victor 解释说，组织将根据合同本身的散列来存储和检索内容。这样一个系统可能需要一个基于激励的、非剥削的、分散的网络，网络用户拥有他们的数据并控制谁从中获利。

一些人创造了这个概念 [Web 3.0](https://www.forbes.com/sites/forbestechcouncil/2020/01/06/what-is-web-3-0/?sh=1ce06a7f58df) 。Victor 将 Web 3.0 描述为分布式技术和工具的广义分类，它实现了由[区块链](https://devops.com/?s=blockchain)支持的点对点互联网。

例如，在这方面领先的一个开源项目是星际文件系统 [IPFS](https://ipfs.io/) 。IPFS 是一种对等超媒体协议，它使用区块链来存储加密哈希。该项目“旨在通过使网络可升级、有弹性和更加开放来保存和增长人类的知识。”使用 IPFS，客户端使用哈希请求内容并检索内容。IPFS 是基于 [Filecoin](https://filecoin.io/) ，一个无许可的存储网络。

## 利益

去中心化的本质符合开放网络的最初目标，允许创建者横向拥有和共享内容。Victor 说，如果你能把数据存储和检索从一个特定的位置分离出来，它会给你一种令人印象深刻的能力来重新构建现代通信。这可能有许多好处:

*   **冗余**:首先，一个主要的好处是数据的冗余。分布式节点确保数据在全球范围内可用且可临时访问。资源位于多个位置，确保了更好的全局可用性和时间持久性。
*   **激励措施**:基于激励措施的分散式网络可以使用整体奖励来补贴成本，帮助为存储计划提供资金。建立在区块链的基础上还可以缓解支付接受方面的问题——这对基于捐赠的数据托管团体来说非常好。
*   **效率**:通过使用智能合约，您可以以编程方式处理存储请求。“区块链为你提供了更高级别的原语来自动运行这些东西，”维克托说。维克多补充说，这有助于将链上活动与现实世界的存储联系起来。
*   **低成本**:在分布式网络上存储内容比典型的云提供商更便宜。令人印象深刻的是，维克多估计它的成本是亚马逊的 1/1000。分布式环境也能带来速度的提高；Victor 说，类似于种子烘焙，从分布式来源播种可以提高几个数量级的速度。
*   公共所有权:这种模式可能会刺激一些内容的公共所有权。这可能有利于没有明确所有者的公益资产。根据 Victor 的说法，这一方面特别有利于大学和依赖公众支持运作的开源数据倡议。

## 迫在眉睫的问题

构建一个新的点对点互联网显然有很大的障碍。也就是说，一个分散的结构需要大量的团队努力，这种社区行动可能难以鼓励，更不用说协调了。它还需要围绕开放标准和协议进行整合，因为许多不同的分散项目会出现在现场。将先前存在的架构转移到这种新风格也可能会带来障碍。

“互联网上的许多内容都变成了公共产品，”维克托说。然而，永久访问这些内容是不稳定的，即使是对于成熟的在线社区。一些研究发现一个网站的平均寿命只有 100 天。托管费用或意外故障通常是数据丢失的根本原因。

网络的脆弱性不仅需要创新的开发努力，还需要全新的思维方式来解决。“这是一个非常引人注目的问题，”维克托补充道。使用基于区块链的智能合约来存储内容的规范引用是一个提议的解决方案。但是 Web 3.0 还处于初级阶段，要取得进步，还需要大量的社区行动。

以一种可持续的方式这样做将是一个诀窍。开放标准和基于区块链的协议可以让这种网络的技术手段蓬勃发展。加密货币支持可能有助于激励参与，并提供一种融资手段。这种分散的非短暂存储有助于实现网络最珍贵资源的永久存储。
