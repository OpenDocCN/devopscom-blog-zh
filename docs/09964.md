# 文化是发展的绊脚石

> 原文:[https://devo PS . com/culture-a-thimble-block-to-devo PS-devsecops/](https://devops.com/culture-a-stumbling-block-to-devops-devsecops/)

根据对 600 多名 IT、安全、应用程序开发和 DevOps 决策者的全球进展[](https://www.progress.com/papers/devsecops-simplifying-complexity-in-a-changing-world?utm_medium=pr&utm_campaign=kmp_survey_DevSecOps)调查，由于缺乏文化一致性和投资，各组织正在努力改进 DevSecOps 实践。

许多公司在实现 DevOps 和 DevSecOps 目标方面落后，73%的组织表示他们可以做得更多。

超过四分之三(76%)的受访者承认，他们需要在如何管理 DevSecOps 方面更具战略性，17%的受访者表示，他们仍然认为自己处于探索和概念验证阶段。

Progress 的产品管理副总裁 Prashanth Nanjundappa 表示，最令人惊讶的发现是，组织认为非常重要的事情与其实际的一致性、承诺或投资之间存在明显的矛盾。

“虽然安全性是大多数 DevOps 和 DevSecOps 实施背后的头号驱动因素，但只有 30%的人表示他们对安全性和开发之间的合作水平充满信心，”他补充道。

与此同时，86%的受访者在他们当前的安全方法中遇到了挑战，51%的受访者承认他们不完全理解安全如何融入 DevSecOps。

Nanjundappa 指出，事实上，公司文化是 DevOps 和 DevSecOps 成功的最大障碍。

“我们发现 71%的受访者同意文化是 DevSecOps 进步的最大障碍，但只有 16%的人将文化作为未来 12 至 18 个月优化的优先领域，”他解释道。

该研究揭示了推动开发运维在其组织内的采用和发展的主要业务因素，包括关注敏捷性以及降低质量、安全性和停机时间或性能问题的业务风险。

组织还认识到需要实施开发运维来支持云任务或他们向云的迁移。

此外，对公共云工作负载的担忧、不断增加的监管要求以及对财务影响的担忧也在推动开发运维安全的实现方式。

Nanjundappa 说:“组织在如何管理安全方面有许多优先事项，安全的关键重点领域是数字营销工作、员工应用程序和面向客户的应用程序。

安全性方面的挑战包括优先开发面向外部的功能而不是内部应用程序，使用不同的开发和交付方法保护不同工作负载的能力，满足交付期限以及满足审计要求的困难。

他指出:“通过调查，我们看到了成熟度较高的组织与其他组织之间的一些显著差异。”。

有些与文化方面有关，通过快速和频繁地交付应用程序功能，以及使用 DevOps 来更好地管理分布式和/或[远程工作](https://devops.com/?s=remote+work)环境，突出了对业务敏捷性的关注。

Nanjundappa 说，确保富有成效的合作的最佳实践包括利益相关者赞助与组织目标的战略方向一致的工具和成果的领导力。

“让团队在工具和流程方面在基层进行创新，以实现预期的结果，”他解释道。“确定已经取得成功的团队，庆祝这些成功，并在迭代模型中在其他团队中实施这些流程。”

随着组织进入 2023 年，紧张的 IT 劳动力市场持续存在，Nanjundappa 指出，一个关键的问题是找到他们需要的人才，以实现这些 [DevSecOps](https://devops.com/?s=DevSecOps) 计划。

他说:“尽管雇佣符合其组织需求的经过预先培训的人才是理想的，但由于多种原因，这要么成本高昂，要么有时近乎不可能。”。

他建议采用多管齐下的方法来弥补人才缺口，其中包括[招聘有经验的关键人才，并通过不同团队(包括安全、开发和运营)的经验交流来进行再培训。T3】](https://devops.com/devops-onramp/)

Nanjundappa 还建议用现代技术提升现有团队的技能，用云和云原生技术取代传统平台和技术，并使用适当级别的工具和流程(包括策略即代码和 CI/CD)进行补充。
