# 调查揭示开源软件不安全的深度

> 原文：<https://devops.com/survey-uncovers-depth-of-open-source-software-insecurity/>

Snyk 和 Linux 基金会今天发布的一项调查发现，不到一半的受访者(49%)就职于拥有使用或开发开源软件的安全政策的组织。

该调查对 550 名软件开发专业人员进行了调查，由 Snyk 和 Linux 基金会进行，Snyk 是一家软件安全工具提供商。不到三分之一(30%)没有制定任何政策的受访者也承认，他们团队中目前没有人直接解决开源安全问题。总共有 41%的人说他们对开源软件的安全性没有很高的信心。尽管如此，59%的受访者表示他们部署的开源软件在一定程度上或高度安全。

Snyk 的开发人员关系总监马特·贾维斯表示，许多组织都没有意识到开源软件现在被网络犯罪分子瞄准的程度，这些网络犯罪分子试图在广泛使用的软件项目中偷偷插入恶意软件。他补充道，同样是这些组织也不清楚如何构建开源软件。

Jarvis 指出，在许多方面，开源软件社区现在已经成为其自身成功的受害者。

总的来说，调查发现平均应用程序开发项目有 49 个漏洞和 80 个对开源软件的直接依赖。只有 18%的受访者表示，他们对自己针对可传递依赖关系的控制措施有信心。在所有发现的漏洞中，有整整 40%是在可传递依赖关系中发现的。

根据调查，只有三分之一的受访者(33%)在持续集成流程中使用静态应用安全测试(SAST)工具或软件组合分析(SCA)工具来发现这些漏洞。总的来说，44%的人报告说他们使用某种工具来分析源代码。

在附加功能方面，59%的受访者表示他们希望看到这些工具增加更多智能，而 52%的受访者表示需要更明确定义的网络安全最佳实践。近一半(49%)的受访者还希望提高自动化程度，并获得更多工具来进行安全审计。

调查还发现，修复开源项目中的漏洞比专有项目多花 19%的时间。在制定了安全策略的组织中，80%的组织将责任赋予了安全团队。调查显示，没有此类政策的组织中，只有 40%采取了同样的措施。

Linux 基金会的一个分支——开源安全基金会(OpenSSF)正在关注 10 个投资流(T1 ),总共需要超过 1 . 5 亿美元的资金来推动开源软件项目的维护者更多地采用 DevSecOps 最佳实践。今天的开发人员经常重复使用开源软件。问题是，这些项目中有许多是由少数程序员维护的，他们自愿贡献他们的时间和精力来构建其他人可以免费使用的组件。像任何其他开发人员一样，这些人拥有的安全专业知识是有限的。确保项目和软件安全的责任在于决定部署该软件的组织。

不幸的是，许多 IT 供应商和大型企业 IT 组织重用这些代码，而没有为项目做出任何有意义的贡献——无论是在融资方面还是帮助开源维护者发现和修复漏洞方面。希望在拜登政府发布行政命令后，开源安全的水平将在未来几个月稳步提高。
