# New Relic 推出了 New Relic One，使团队能够加速创新并更快地交付更完美的软件

> 原文：<https://devops.com/new-relic-introduces-new-relic-one-empowering-teams-to-accelerate-innovation-and-deliver-more-perfect-software-faster/>

*新特性包括 AWS Lambda 监控、新仪表板、可编程性、全局搜索和跨账户服务地图——所有这些都旨在帮助分布式团队集中管理和自动化现代软件的复杂性*

**旧金山–2019 年 5 月 14 日**–[New Relic](https://newrelic.com/)(纽约证券交易所代码:NEWR)，业界最大、最全面的基于云的仪器平台，旨在帮助客户创建更完美的软件，今天推出了 New Relic One。作为 New Relic 平台的扩展，New Relic One 帮助复杂环境中的开发运维团队快速查找、可视化和理解他们的数据，以便他们可以立即采取行动，为客户提供更好的数字体验，并增加收入。借助 New Relic One，企业现在能够将他们的团队与客户体验的实时健康状况联系起来，从而加快数字业务的发展。

对于现代软件团队来说，要成功地管理他们日益复杂的环境的性能，有许多挑战。许多大型企业有数百个孤立的团队，负责管理相互依赖的系统。这些系统越来越复杂，通常由容器或微服务组成，在内部和云中运行。当服务问题出现时，团队通常很难快速明确地找出问题，并在客户遇到中断之前解决问题。

“由于有关客户体验的数据驻留在多个系统、控制面板和组织中，DevOps 团队继续努力应对其复杂环境的碎片化视图。New Relic One 是业内首个以实体为中心的可观察性平台，统一了多个账户的所有客户数据，因此团队可以看到所有关系和依赖关系的泛企业视图，”New Relic 首席执行官兼创始人 Lew Cirne 说。“New Relic One 为我们今天的客户解决了一系列重要问题，它也是我们为客户提供未来十年创新的平台。我们刚刚开始我们的使命，即帮助公司通过 New Relic One 更快地交付更完美的软件。”

**介绍新遗迹一**

New Relic One 旨在改变 DevOps 团队管理其数字系统性能的方式。New Relic One 跨多个帐户统一了客户的所有数据，因此每个团队都可以看到一个泛企业的共享视图，了解他们使用的所有工具。除了在一个地方看到他们的所有数据，New Relic One 还使团队能够快速深入到他们负责管理的服务。

**定义实体**

New Relic One 将所有需要检测的东西都视为一个“实体”,并对实体进行索引，以跟踪它们之间的关系和依赖关系，因此团队除了度量标准之外还有上下文。New Relic 将实体定义为数字业务成功所需的任何东西，包括微服务、移动应用、Kubernetes 集群或 Lambda 函数等等。实体发出数据，该数据提供关于实体内部状态的上下文—包括上游和下游依赖关系。在关键时刻，对于团队来说，度量指标已经不够了。团队还需要快速了解实体之间的关系及其相互依赖关系，这样他们就可以更快地发现问题，并提供更好的客户体验。与公司今天使用的分散工具不同，New Relic One 及其以实体为中心的可观察性方法提供了单个客户环境中所有系统、团队和技术的清晰、关联视图。

新遗迹一号交付:

*   **复杂环境的统一泛企业视图**—**New Relic One 推出了新的跨客户服务地图，可以自动可视化实体的上下游依赖关系，从而在对事件进行故障排除时轻松快速地确定根本原因。**
*   **互联搜索和发现平台** — New Relic One 提供新的全局搜索和通用标签过滤，因此团队可以快速轻松地找到整个企业中的实体。
*   **易于创建、功能强大的仪表板** — New Relic One 引入了新的和改进的仪表板功能，使用户能够快速创建信息丰富的自定义仪表板，将技术工作和业务影响联系起来。一个名为 chart builder 的全新功能使得使用点击式管理工具创建强大的查询变得非常容易，这是对新遗留查询语言(NRQL)的补充。客户在 New Relic Insights 中的现有仪表板将自动出现在 New Relic One 中，以供查看和编辑。
*   **用于创建特定于业务的视图的可编程性** — New Relic One 引入了针对特定于业务和领域的需求快速扩展可视化的能力。例如，很容易将存储销售点系统数据与实时遥测数据结合起来，全部覆盖在地图上。运营团队可以鸟瞰整个地理区域的商店绩效，也可以深入查看发出警报的特定商店的绩效。凭借可编程性，New Relic One 可以嵌入到企业业务绩效管理中。
*   **统一的用户体验和全球主页** — New Relic One 提供了全新的主页体验，使客户能够在一个地方查看他们的所有性能数据。New Relic One 位于现有的 New Relic 产品平台之上，用户可以继续快速关注问题领域，进行更深入的探索。

New Relic One 是 New Relic 最新创新的大本营，包括:

*   **监视 AWS Lambda** —提供对 Lambda 功能进行监视、可视化、故障排除和警报的能力。Pro 客户现在可以总体监控 Lambda 性能，或者深入到单个 Lambda 调用，查看跟踪、错误、端到端跟踪和其他故障排除信息。
*   **Kubernetes cluster explorer**—提供 Kubernetes 集群的多维表示，允许团队在简化复杂环境的高保真、精选 UI 中深入 Kubernetes 数据和元数据。这一现有功能现在可以在新的 Relic One 界面中使用。
*   **分布式跟踪全局搜索** —提供一个简单而强大的工具，使用自定义属性或标签来查找踪迹。因此，团队现在可以更快地解决分布式软件环境中的性能问题。

New Relic 还宣布，它计划在本财年结束前在 New Relic One 中提供日志管理和 AIOps 解决方案。

**客户报价:**

“New Relic One 改进的仪表板功能提供了我们测量的所有实体之间的依赖关系的迫切需要的可见性。其他工具给我数据，New Relic One 给我背景，”考克斯汽车公司首席软件工程师马修·b·沃恩说。“能够看到跨职能数据在一个集中的平台中是如何关联的，这是一种巨大的力量和安慰。当我们发现失败的风险时，我们能够迅速查明根本原因是至关重要的，New Relic One 提供了一个全新的粒度级别，有助于我们提供更好的客户体验。”

**分析师报价:**

“对微服务和基于云的服务的需求继续加剧企业技术团队的复杂性。IDC I&O 项目副总裁史蒂芬·艾略特表示:“要解决这些复杂问题，需要一种深思熟虑的方法来简化观察和测量软件应用的过程。“这些要求从来都不是可有可无的，随着现代应用程序架构、开发运维以及敏捷方法的发展，这些要求变得比以往任何时候都更加重要。”

**定价和供货情况**

New Relic One 将于 2019 年 5 月 15 日(周三)上线，免费向专业用户开放。现在，专业客户可以通过新的 Relic One 界面在有限时间内根据产品文档免费获得 AWS Lambda 支持监控。Kubernetes cluster explorer 和分布式跟踪全局搜索通常面向专业客户提供。其他解决方案，包括日志管理和 AIOps 以及其他可编程功能，预计将在 New Relic 财年结束时交付。
