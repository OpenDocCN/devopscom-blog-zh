**空间态势感知**

成千上万的人造空间物体(ASOs)围绕地球运行。随着火箭发射的私有化，aso 进入轨道的速度将会急剧增加。

aso 的大小从直径几厘米的油漆斑点到国际空间站都有。一小块油漆听起来可能不危险，但当它以每秒约 8000 米的速度行进时，与卫星或其他载人车辆的碰撞有可能是灾难性的。

[空间态势感知](https://github.com/IBM/spacetech-ssa) (SSA)项目是 IBM 和德克萨斯大学奥斯汀分校[Moriba Jah](https://sites.utexas.edu/moriba/)博士的联合工作，旨在确定 aso 在哪里(轨道确定)以及它们在未来的位置(轨道预测)。

轨道预测的最新方法依赖于基于物理学的模型。为了成功，这些模型需要关于 ASO 及其环境的极其精确的数据。问题是，我们从地面传感器获得的关于 ASOs 的位置数据很少，而且有噪音——而且由于我们没有完全理解像空间天气和大气密度这样的现象如何影响 ASOs，这一事实使问题变得更加复杂。

对于 SSA，我们的方法是使用机器学习方法来改进轨道预测，不是预测轨道，而是创建模型来学习物理模型何时错误地预测 ASO 的未来位置。如果我们试图建立一个 ML 模型来预测 ASO 的所有轨道动力学，我们将需要大量的数据来学习轨道力学。物理模型知道相当多的轨道动力学，所以我们只需要知道它们是如何出错的。

**SSA 如何工作:利用可靠的数据建立机器学习模型**

我们使用了由美国战略司令部 (USSTRATCOM)通过 space-track.org[](https://www.space-track.org/documentation#/howto)网站和 API 提供的数据来构建我们的基线模型。数据以 [两行元素集](https://en.wikipedia.org/wiki/Two-line_element_set) 格式提供，这是一种固定宽度的文本格式，包含某一时间点 ASO 的 [开普勒轨道元素](https://en.wikipedia.org/wiki/Orbital_elements) 。对于低地球轨道(LEO)中的大多数 aso，该数据集每天仅更新一次，这对于移动物体来说并不常见。

我们已经建立了两种类型的基线模型:

*   基于物理学的模型预测近地天体的轨道。我们的基线物理模型是一个 [两体模型](https://en.wikipedia.org/wiki/Two-body_problem) ，它使用考威尔公式对地球引起的 ASO 轨道扰动进行建模。
*   被训练来预测物理模型的轨道预测中的误差的机器学习模型。我们的基线机器学习方法使用 [XGBoost](https://xgboost.readthedocs.io/en/latest/) 来构建 [梯度增强回归树](https://en.wikipedia.org/wiki/Gradient_boosting) 模型。然后根据我们的基线物理模型的轨道预测对该模型进行训练，以使用 USSTRATCOM 数据作为地面事实来预测物理模型的误差。

SSA 项目将两个模型结合起来，使用物理模型进行轨道预测，然后使用机器学习模型估计误差。最后，根据预测误差调整物理模型的预测。

我们采用了一台 IBM Cloud 裸机服务器，配有 16 个英特尔至强处理器；120 GB RAM 两个 Nvidia Tesla V100 GPUs(每个具有 16GB RAM)运行物理模型以预测 LEO 中所有 aso 的轨道，训练 ML 模型以学习物理模型误差，并使用 physics/ML 组合模型推断 LEO ASOs 的未来轨道。我们还使用云对象存储来存储来自 USSTRATCOM、物理模型预测和最终组合模型预测的数据。

**KubeSat，一个认知自主框架**

打算使用卫星群(合作完成特定任务的卫星群)作为更传统的大型卫星的模块化、低成本替代方案的初创公司和研究小组需要能够创建和控制卫星群的底层技术，以及支持实现特定目标的附加软件。

进入 IBM 的 KubeSat，一个认知自主框架。

KubeSat 是一个认知自主框架，专为立方体卫星群设计，允许模拟和优化多卫星通信。

该项目:

*   通过 [或](https://www.orekit.org/) 模拟每个物体的精确轨道力学
*   使用这些计算来限制卫星、地面站和地面传感器之间的通信
*   整合了 NATS 消息服务；通过强化学习优化他们的沟通
*   将这些通信发布到用户界面
*   使用 Kubernetes 作为微服务的核心平台

这个框架遵循群体智能的原则，如意识、自治、团结、可扩展性和弹性。

**为什么我们要开源 KubeSat 认知自主框架**

开源 KubeSat 认知自主框架使卫星群产业民主化，并允许较小的运营商利用空间技术，特别是在卫星群产业，这是以前无法实现的。

航天工业的创新可以受益于这一框架。特定用例的强化学习模型可以用这个框架作为许多卫星组的底层软件来构建。这些卫星群之间的协作和通信可以是自主的，允许卫星群根据需要集成或分离。这种模块化的群体创造是我们软件的一个关键组成部分，允许卫星利用我们的软件产生最大的影响。

我们还将我们团队与斯坦福大学本科生一起开发的应用程序代码开源到 Orekit，这是轨道力学的基础库。这些扩展用于模拟卫星对卫星、卫星对地面站和卫星对地面传感器在网状 NATS 信息平台上的通信。使用铯和碳编写的可视化层也是可用的

**推进太空创新的未来**

我们的团队致力于推进太空创新的未来。随着各种尺寸的卫星在空间近地轨道上的扩散，将产生与地球观测、空间交通管理和空间态势感知相关的大量数据。这些数据需要得到保护、分析、传输和关联。

这就是空间中的边缘计算可以提供推理和数据过滤能力的地方，基本上是卫星中的数据处理。接下来，云将进入画面，通过地面站接收来自卫星星座的聚合和原始数据，以进行进一步处理。
