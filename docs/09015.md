# 调查预测大规模向云迁移

> 原文：<https://devops.com/survey-predicts-massive-migration-to-the-cloud/>

一项针对 1600 名企业 IT 决策者的全球调查发现，超过一半的企业计划在未来两年内淘汰数据中心。

这项调查是由安全访问服务边缘(SASE)平台提供商 Aryaka 进行的，发现四分之一的受访者关闭了 25%至 50%的办公网站；75%的受访者表示，至少有四分之一的员工将在疫情事件后永久保持远程办公。不到三分之一(32%)的受访者预计至少有一半的员工将永久远程办公。

尽管向云的转变明显加快，但报告发现，整整四分之三的受访者(75%)预计 IT 支出至少会增长 10%，四分之一的受访者预计 IT 预算会增长 25%或更多。然而，超过三分之一(36%)的受访者表示他们担心 IT 成本。

Aryaka 的产品和解决方案副总裁 David Ginsburg 表示，新冠肺炎疫情之后向云的转变主要是由数字业务转型计划驱动的，这些计划要求在家工作的 IT 团队快速构建和管理应用程序。他指出，随着疫情进入第三年，定期在家和办公室之间转移的混合劳动力现在是永久性的。

因此，随着越来越多的人在家工作，近三分之二的受访者(64%)表示，他们将在未来一年转向基于 SASE 的方法，以通过软件定义的广域网(SD-WAN)提供对云应用的安全访问。超过三分之二的受访者表示，他们将选择托管 SASE 产品，而不是自己构建和维护这些平台。

SASE 最初是由 Gartner 创造的，用来描述一种将网络和安全产品结合在一起的云交付服务。这些产品应包括防火墙即服务(FWaaS)、安全网络网关(SWG)、云访问安全代理(CASB)和零信任网络访问(ZTNA)。实际上，它是一个安全的软件定义的广域网(SD-WAN)。随着组织采用基于 SASE 的平台和服务，不到三分之一的受访者(61%)认识到，随着员工在办公室和家庭之间转移，动态带宽重新分配将是一项非常重要的要求。

向 SASE 的过渡包括制定战略(35%)、逐步淘汰传统虚拟专用网络(32%)以及通过零信任整合云安全(29%)。最受欢迎的功能包括 SWG (47%)、SD-WAN (36%)和 FWaaS (28%)。整整 41%的受访者表示，网络和安全决策的职责现已整合到一个团队中。然而，只有略多于三分之一(34%)的受访者认为安全是重中之重。

以这种速度和规模水平向云迁移的影响可能会对开发运维团队产生深远的影响。虽然云基础架构设计为按需扩展，但通过 SD-WAN 访问应用程序会对应用程序性能产生重大影响。然而，调查发现，只有 29%的受访者担心应用程序是否能正常运行。尽管如此，超过三分之二(69%)的人表示将更加关注[可观察性](https://devops.com/?s=observability)和控制，而 42%的人指出识别远程和移动用户的缓慢性能将是一个关键问题，37%的人预计分支机构的缓慢性能也是一个问题。42%的受访者表示，应用程序性能不佳是一个时间陷阱。

自然，许多 IT 团队会添加新的工具和平台来部署和管理云应用程序。Aryaka 的调查发现，只有略多于四分之一的受访者担心他们可能采用的新技术。

接下来的一年将会对 IT 领域产生难以置信的破坏性影响。挑战在于确保在成本、灵活性和应用程序性能之间保持平衡。
