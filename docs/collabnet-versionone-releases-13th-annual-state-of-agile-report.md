第 13 次年度报告的特点是新的主题、优先事项的变化和重点领域的扩大。随着 DevOps 的重要性不断增加，DevOps 部分得到了扩展，在改进 DevOps 实践方面有了新的成果。价值流管理是一个新兴的类别，帮助组织实现连接敏捷和 DevOps 实践的价值，在第 13 次年度报告中首次出现。

今年报告的显著调查结果包括:

*   **敏捷越来越频繁地被采用来降低成本**–新的报告显示，项目成本降低增加了 71 %,这是采用敏捷的主要原因。它还将项目成本降低 27%作为敏捷的一个主要优势。根据这份报告，与前一年相比，敏捷的采用更加注重降低项目成本和提高团队士气，而不是提高生产率和降低风险。
*   **开发运维是更高的组织优先级—**了解和改进开发运维的需求继续上升，成为组织的优先事项——90%的受访者认为这很重要。该报告还显示，认为 DevOps 转型非常重要的受访者比去年增加了 27%。
*   **端到端可追溯性是改进 DevOps 实践的关键**–共有 38%的受访者认为从业务到开发、测试和部署的端到端可追溯性是 DevOps 改进的最有价值的能力。
*   **价值流管理是新的，但很重要**–尽管价值流管理今年首次亮相，但 67%的受访者表示，它对于将业务与其软件交付能力联系起来很重要。
*   **SAFe 主导扩展方法**–扩展的敏捷框架(SAFe)继续被采用，并被 30%的受访者认为是首选的扩展方法。

“敏捷状态报告是敏捷和软件开发社区的宝贵资源，因为它指出了敏捷的核心问题。CollabNet VersionOne 致力于每年提供这一资源，以支持社区的进步，”CollabNet VersionOne 首席执行官 Flint Brenton 说。“今年，我们扩大了调查的 DevOps 部分，并增加了价值流管理，因为每个部分都继续受到软件专业人员的更多采用。我们已经观察到，随着组织在交付高价值、高质量的软件方面不断努力改进，度量这些实践的成功的重要性增加了。”
