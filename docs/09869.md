# 红帽和 IBM 承诺将 NLP 带到 Ansible

> 原文：<https://devops.com/red-hat-and-ibm-pledge-to-bring-nlp-to-ansible/>

在今天的 [Ansiblefest 2022](https://www.ansible.com/ansiblefest) 大会上，红帽及其母公司 IBM 宣布了将自然语言处理(NLP)引入开源 Ansible IT 运营自动化平台的项目智慧倡议。

与此同时，Red Hat 宣布，来自 Red Hat 的 Ansible 的策划实例现在可以在亚马逊网络服务(AWS)和微软运营的市场上买到。

Red Hat ansi ble 产品管理高级经理 Richard Henshall 表示，目标是让更广泛的终端用户和 IT 专业人员更容易实现 IT 自动化，这些用户和专业人员缺乏使用 YAML 文件实现 IT 流程自动化的声明式编程专业知识。相反，Red Hat 和 IBM 研究团队承诺将他们的专业知识结合起来，创建一个人工智能(AI)模型，使任何人都可以使用自然语言来自动化 IT 流程。

Ansible 已经成为事实上的开源标准，用于自动化各种 IT 流程，但是可以自动化的流程数量仍然远远超过当前自动化的 IT 流程数量。近年来， [IT 自动化](https://devops.com/?s=IT+automation)工具已经变得更加容易获得，部分原因是低代码和无代码工具的兴起，但是仍然需要一定程度的编程专业知识。项目智慧正在启动，以消除在能够自动化流程之前获取这些技能的需要。

当然，仍然会有复杂的流程需要能够以编程方式操作 YAML 文件，但是能够自动化的琐碎 IT 流程的数量仍然是无法计算的。理论上，Ansible 的自然语言界面将极大地减少任何人自动化 IT 过程所需的时间和精力，不管他们的技能水平如何。

随着全球经济持续疲软，人们对 IT 自动化的兴趣再次上升。组织希望自动化尽可能多的流程，以尽可能减少 IT 员工人数。IT 团队在手动流程上花费的时间越多，他们管理日益复杂的 IT 环境的能力就越弱。

尚不清楚 It 流程可以自动化到什么程度，但随着企业越来越依赖 IT，IT 自动化和业务流程之间的界限继续模糊。自然语言处理能力可以使组织更容易大大加快数字业务转型计划。

不管 it 自动化水平如何，IT 管理方式显然将再次演变。随着 Ansible 等平台被更广泛地采用，IT 团队使用脆弱的定制脚本配置和供应 IT 资源的日子即将结束。唯一有待观察的问题是 IT 流程自动化的速度有多快，以及自动化的速度是否能跟上目前正在开发的应用程序的数量。
