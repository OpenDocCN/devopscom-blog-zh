# NS1 宣称为开发人员和运营人员提供了一个通用的交付平台

> 原文:[https://devo PS . com/NS1-toogles-a-common-delivery-platform-for-devs-and-ops/](https://devops.com/ns1-touts-a-common-delivery-platform-for-devs-and-ops/)

随着 [NS1 Connect](https://ns1.com/ns1-connect) 的发布，NS1 扩展了其 DNS 和网络管理平台的功能。该平台旨在作为一个交付平台，集成开发人员和网络运营团队的任务，用于在高度分布式的云本地和内部环境中创建和管理应用。

虽然开发运维团队通常依赖数十种工具来部署软件和管理软件运行的网络连接，但 NS1 的创建目的之一是通过提高这些流程的可见性、自动化和管理来减少开发运维团队所需的不同工具的数量。

这样，NS1 Connect 提供了一个统一的 API 框架来自动化跨异构网络的 DNS、流量优化和网络服务管理，NS1 产品副总裁 Sanjay Ramnath 告诉 DevOps.com。NS1 Connect 的覆盖范围包括通过内部部署、云或混合基础架构部署的内部和外部应用程序。Ramnath 说，这使 NetOps 和 DevOps 团队能够以较低的复杂性将 NS1 解决方案集成到他们的基础设施和 CI/CD 管道中，并使 NetOps 团队能够加快管理速度，最大限度地降低配置错误的风险。“在此之前，团队必须处理复杂的手动流程或不同的 API 框架，这降低了他们开发、部署和交付应用程序的能力。”

在他的演讲中，[您的业务依赖于消除应用程序、用户、基础架构和数据之间的界限](https://resources.ns1.com/your-business-depends-on-eliminating-boundaries-between-applications-users-infrastructure-and-data)在[NS1 的年度用户大会 2021](https://ins1ghts.ns1.com/event/45e80dca-c499-421b-bb0c-5ebd9e3ccff0) 期间，Ramnath 描述了许多 NS1 产品和服务如何集成到 NS1 Connect 保护伞下的单一平台中。其中包括面向 DNS、DHCP 和 IPAM (DDI)的云原生网络服务，面向 NS1 托管 DNS、专用 DNS、私有 DNS 和中国 DNS 的边缘到云网络，以及面向 VPN 流量导向、pulsar 主动流量导向和通用流量管理的应用流量自动化。Ramnath 说，还提供了其他常用的网络服务，包括可以在网络上集中管理的 IP 地址管理。

![](../Images/b54a5aba6bd89d3b91f4788a39e91cb0.png)

这个想法是为了消除 DevOps 团队在寻求部署高度分布式基础设施时面临的许多障碍。“我们的目标是消除基础设施方面的一些界限和障碍，”Ramnath 说。“[这有助于确保]我们的控制面板作为云解决方案集中提供和集中使用，我们的产品可以以零接触的方式、轻量级的方式部署在任何需要的地方，确保我们与云原生运营保持一致。”

跨分布式基础架构保持低延迟的需求也很关键。正如 David Coffey 在[应用流量智能和自动化的未来](https://resources.ns1.com/the-future-of-application-traffic-intelligence-and-automation)会议中所解释的，低延迟对于“为用户创造卓越体验”至关重要

Coffey 说:“低延迟不是该功能的属性，它是人们需要完成的首要任务，但考虑到现有的所有框架，这是一个非常复杂的方面，包括支持和部署这些现代应用程序所需的工具、DevOps 功能和组件。“这使得应用公司能够提供低延迟，并将其视为系统中的一个属性；不需要特意去创造这个。我们希望所有开发人员都能够将这一点嵌入到他们的系统中，我们可以为他们提供一定程度的成功，为他们的客户和网络提供出色的体验。”
