# 靠边站，独石:见见可组合企业

> 原文:[https://devo PS . com/step-side-monoliths-meet-the-composable-enterprise/](https://devops.com/step-aside-monoliths-meet-the-composable-enterprise/)

Gartner 将可组合企业定义为“交付业务成果并适应业务变化步伐的组织”，它依赖于可互换应用程序构建模块的组合。这种架构变革在很大程度上是由对更可配置的应用程序体验的需求，以及对现有应用程序组合的发展需求所驱动的，这些应用程序组合通常风险太大、替换成本太高。

新的业务机会需要应用程序组合的敏捷性；然而，许多企业的适应能力仍然有限。为什么？他们依赖于单一的 ERP 系统和笨重的遗留应用程序，这些应用程序具有静态流程和杂乱的结构。模块化设置可以使企业根据外部或内部因素(如消费者态度的转变或供应链的突然中断)进行必要的重组。组织现在正在经历这些转变，需要一种新的企业应用程序方法来适应。

### 以更高的可组合性弥合 CX 鸿沟

可组合性方法是获取现代企业软件所有优势的最佳方式。根据最近的一份 Boomi 报告 ，到 2023 年，采用可组合方法的组织将在新功能实现的速度上超过竞争对手 80%。这种转变要求企业重新思考他们如何设计他们的运营，利用打包功能和技术的组合，并成功地向他们的客户提供无缝的服务。

企业可以通过三种主要方式实现可组合性转变:

*   采用服务思维
*   扩展微服务的交付
*   使用应用程序编程接口(API)打包业务功能。

**1。为服务而设计:在结果上竞争，而不是在产品上竞争**

一个可组合的企业，其投资组合由来自一流解决方案调色板的异构应用程序组成，允许组织解决每个客户、产品或服务生命周期中的关键转折点。

随着越来越多的消费者要求资产在整个生命周期中具有持续的价值和可靠性，企业必须转向销售成果和体验，而不是产品，以满足这些对优质服务的新期望。这要求运营的每个部分都保持一致，不是围绕直接的销售或收入，而是围绕提供优质服务的时刻——这是一个转折点，在这个转折点上，所有事情都结合在一起，为客户创造更好的价值和结果。

转移到服务化模型需要一个可组合的堆栈来交付订单服务。在系统层面，这种转变要求组织动态地调整应用程序，并通过有效的质量管理、客户支持和访问有关服务产品的完整信息来提供积极的客户体验。与传统的企业软件不同，这涉及到将通常位于不同孤岛中的数据和应用程序连接起来。一个可组合的企业可以提供一个面向服务的架构，使业务变得基于结果，并准备好快速适应未来的中断。

### 2.使用微服务扩展基于组件的架构

如果应用程序是作为松散耦合的服务构建的，那么公司可以采用可组合的架构来利用围绕业务功能组织的独立可部署的模块。这使得组织可以交换模块，以适应紧急需求，并为其独特的业务构建结构良好的最佳解决方案。

与单一架构相比，企业可以轻松地向最需要的微服务添加资源，而不必随着应用需求的增加而扩展整个应用。在实践中，这使企业能够简化可定制的工作流程并优化业务流程，同时利用和应用机器人流程自动化、人工智能或当今可用的大量超自动化功能等工具。

### 3.采用开放式 API 来最大化数据共享能力

使用 API 组装的打包业务功能(PBC)是每个可组合企业的基础。企业使用它们来保护云服务、业务系统和移动应用程序中的数据。从历史上看，API 已经被用在单片应用程序中，以在整个应用程序和外部应用程序和服务之间交换数据。在可组合软件中，API 从单个模块到外部应用程序以及在应用程序内部交换数据——从模块到模块。这对于系统的设计和构建有着重要的意义。

[API](https://devops.com/intel-expands-scope-of-oneapi-toolkit-initiative/)通过创造适合个人需求的体验，为连接和共享消费者和企业数据提供一种可控的、可消费的方法。例如，以 API 为中心的模型可以保护和管理数据访问，以帮助企业更快地做出决策，并提供适应市场变化的相关新服务。在未来，随着机器学习模型推荐甚至主动进行流程更改以改善业务和最终客户的成果，将会有更多自动化的持续流程改进。

### 选择加入可组合性进化

为了克服单一应用程序的局限性，企业必须重新思考他们的企业应用程序方法——从业务架构和技术栈开始。可组合的软件架构使组织能够解决在整个价值链中传递冲击波的内部和外部压力。

作为一个可组合的企业，组织可以重新设计他们的业务，以确保客户接触点和阶段聚集在一起，获得更好的服务，但公司必须确保流程在每个转折点都得到优化，以减少问题并促进增长。建立在可组合性基础上的 IT 架构对于成功交付软件驱动的业务开发策略是必不可少的，它可以为客户和业务本身提供持续的价值。
