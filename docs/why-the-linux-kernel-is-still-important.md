# 为什么 Linux 内核仍然重要

> 原文：<https://devops.com/why-the-linux-kernel-is-still-important/>

去年，Linux 内核迎来了 30 岁生日。如果你像我一样从 Y2K 开始就沉浸在 Linux 世界中，那么从内核诞生到现在已经过去了这么长时间，这可能会让人觉得有点不可思议。

作为一名云专家(ACG)的培训架构师，我教授 关于 Linux 的所有课程，并专攻 的实验室实践学习。在加入 ACG 之前，我曾在 GE 和 IBM 担任 Unix 系统工程师，也曾是 Red Hat 的技术客户经理和客户支持者。我对 Linux 充满热情，就像许多其他工程师和爱好者一样，不仅仅是因为它对我们职业生涯的重要性，而是它的整体影响。

在其[存在](https://devops.com/what-30-years-of-linux-taught-the-software-industry/)的 30 年里，Linux 内核对现代计算领域产生了 [m 的负面影响](https://www.wired.com/2016/08/linux-took-web-now-taking-world/)——彻底改变了操作系统的可能性，并让无数的修补者在这个过程中弄脏了他们的手。由于其不断进化的本质，Linux 也成为了多年来范式转换 [创新](https://opensource.com/article/19/8/everyday-tech-runs-linux) 的基础。

以下是内核如此有价值的一些关键原因，以及它在未来几年仍将是计算领域的主要参与者的原因 。

### 我自己的 Linux J ourney

我第一次接触 Linux 是在 1999 年的大学里。我上了一堂 Unix Shell 脚本入门课，Fedora 被安装在实验室服务器上，因为它是免费的，而且易于扩展。我将 Linux 视为我进入可定制的 操作系统的切入点 。在 Linux 上学习了基本的 Unix 命令后，我最终继续从事支持多个商业 Unix 供应商的职业。

随着 Linux 进入更大的企业级公司，我很快回到了 T2，从事 Linux 发行版的工作。我一直喜欢 Linux 的一点是它能够微调您的系统，以支持最适合您的项目的应用程序和发行版。Linux 改进了你正在运行的任何应用程序的功能 。Linux 内核确实改变了世界处理信息的方式，这也是我如此热衷于这个软件的原因。

现在，我的工作是与其他技术人员分享我对 Linux 的知识和热情。在 ACG，我开发课程来帮助 aspiri ng Linux 专家学习如何优化他们的系统。此外，我在本月为[Linux](https://acloud.guru/series/linux-this-month/)撰稿，这是一个 ACG 主办的网络系列，提供来自全球 Linux 社区的每月更新。

我 能够 以教授 Linux 和了解最新的 Linux 新闻为职业，这一事实表明了内核的大量使用案例和应用。是什么让 Linux 如此与众不同和经久不衰，是因为它的开源特性——Linux 的创新仅仅受到使用和改造它的技术人员的创造力的限制。

### 开源效应

当 Linux 刚出现时，它主要是热情的工程师和计算机科学学生的爱好，他们可以通过开发代码来做出贡献。将 Linux 安装到你的机器上的陡峭的学习 曲线对于更多的程序员新手来说是一个障碍。

随着时间的推移，这种情况发生了很大的变化。在线论坛、研讨会和课程让普通互联网用户更容易接触到 Linux。思想的免费共享 已经成为开源社区的缩影，对于软件工程师来说， [Linux 是这个社区的核心](https://devops.com/the-benefits-of-open-source-technologies/) 。Linux 的这种民主化 对计算世界产生了难以置信的 积极影响。

现在，Linux 无处不在。企业级公司使用 Linux 发行版来处理世界上最大的生产工作负载。它以更好的稳定性和更少的停机时间取代了 T2 大公司的专有商业 Unix 操作系统。因为 Linux 系统可以根据你的需要变小或变大，所以它现在也在我们的家庭中用于智能和移动设备。

Linux 的开源性质对于这些企业级的公司来说是不可思议的[](https://www.techradar.com/news/how-open-source-communities-work)。Linux 发行商利用了整个开源社区的贡献。这种广泛的贡献者产生了具有更多功能的更稳定的产品，但也确保了操作系统保持增长并解决对广泛用户有益的现实世界问题。

### Linux 内核运行云端

内核最具影响力的潜在成果是它对现代云计算基础设施的重要性。Linux 的可伸缩性为超级计算机和服务器场高效运行铺平了道路，同时需要相对轻量级的计算资源 ces。事实上，Linux 支持大约 [90%的公有云工作负载。](https://www.rackspace.com/en-gb/blog/realising-the-value-of-cloud-computing-with-linux)

没有 Linux，我们所知的云就不会存在。这在某种程度上是因为 使用 [Linux 已经变得如此普遍](https://www.cbtnuggets.com/blog/certifications/open-source/why-linux-runs-90-percent-of-the-public-cloud-workload)——它的使用案例几乎是无限的。 因为它经过了时间的考验，许多工程师和 it 专业人员对 Linux 的基础知识有着坚实的掌握，这使得它成为处理云的企业公司的一个有吸引力的选择。

基于云的 软件 和产品越来越成为工程界的规范。不出所料，AWS、Azure 和 Google Cloud 等主要云提供商也都支持 Linux。Linux 是独一无二的，因为它是一个变形体，可以符合任何给定的工程环境的需要，而且它非常稳定，因为大量的贡献者巩固了软件中的弱点。

### 永远的 Linux】

Linux 的概念和 Linux 产品一样重要。这个概念允许一个自由的 开源的 操作系统在贡献者的无尽的 w eb 中被精炼、强化和复制。三十年对于软件来说是一段很长的时间，尤其是随着科技产品 周期的不断缩短。因为它的设计意图是由开源社区来改变和更新，所以 Linux 没有可预见的到期日。

如果 Linux 没有取得今天的成就，我们会看到更多的商业 Unix 供应商试图解决 Linux 解决的一些问题，但没有一个供应商能解决所有问题。此外，客户将不得不选择 投资哪种操作系统，基于哪种操作系统解决了他们的一些使用案例，但是没有哪种操作系统像 Linux 操作系统那样有益。

Linux 一直在发展，在未来几年内会变得更加流行。随着越来越多的人熟悉 Linux 和 l 开始使用它，我看到了 majo 在移动计算领域的增长潜力，包括个人电脑和各种大小公司。事实上，我们已经看到它渗透到家庭游戏系统和 Raspberry Pi 项目中。在 Linux 中，sk 是极限！
