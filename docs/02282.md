# 无所畏惧地向左移动:安全性在支持开发运维中的作用

> 原文:[https://devo PS . com/shift-left-now-the-role-of-security-in-enabling-devo PS/](https://devops.com/shift-left-without-fear-the-role-of-security-in-enabling-devops/)

这是 DevOps 的时代，我们都想左移，这是指在 CI/CD 周期中更早执行流程的想法。这包括安全检查和审计。通过在交付管道中更早地开始安全操作，发现问题变得更容易，并且团队在将代码投入生产之前有更多的时间来解决它们。

当然，挑战在于构建一个左移安全操作，允许您在不降低交付速度的情况下尽早执行安全操作。让我们来看看如何平方那个圆。

## **定义左移安全性**

DevOps 文化中的一个流行术语是“左移”。但是左移是什么意思呢？左移指的是 DevOps 团队在软件开发生命周期的早期实现措施以确保应用程序质量的努力。在应用程序安全的情况下，这意味着在开发应用程序时，而不是在过程结束时，实施措施以确保考虑到安全问题。

左移安全性的总体目标是在开发生命周期的早期识别和解决潜在的漏洞，此时修复这些问题的成本较低。这样做将有助于减轻潜在的安全问题对交付计划和最终用户造成的风险。

## **DevSecOps:将安全性集成到开发过程中**

实话实说:更快更频繁的开发周期不会消失。降低交付速度以在开发过程结束时分析和保护应用程序不再是实现安全性的可行格式。开发运维服务和 CI/CD 需要得到安全人员的认可和支持。

对于支持开发运维流程的安全团队来说，协作是关键。通过这种合作，DevOps 的过程转变成了所谓的 DevSecOps。在 [DevSecOps](https://devops.com/how-to-design-devsecops-compliance-processes-to-free-up-developer-resources/) 中，安全人员与 DevOps 组织的其他成员密切合作，帮助那些创建和交付应用程序的人更好地理解在开发生命周期的早期解决安全问题需要什么。

以下提示是实现这种协作的起点:

*   **让安全成为整个 DevOps 团队的责任**:让开发和运营人员记住，安全现在是整个组织的责任。简单地在开发后解决潜在的漏洞会消耗太多的时间，并增加遗漏某些东西的风险。接受这个过程是在安全团队和组织的其余部分之间实现成功协作的重要部分。这方面的一个例子是让开发人员记住安全的[代码](https://www.owasp.org/index.php/OWASP_Security_Knowledge_Framework)。
*   **在整个开发过程中促进关于应用程序安全性的定期讨论**:在开发新功能时，安全人员有责任将安全考虑放在开发人员的首要位置。利用敏捷过程的优势。Backlog 会议可以让安全人员参与进来，以确保用户故事的创建包括对安全问题的考虑。你的每日站立会议是另一个包括安全人员的地方。这将为安全人员提供一个机会，在开发人员描述的工作需要(或可能需要)与尚未解决的安全相关的考虑事项时，他们可以畅所欲言。这样做将有助于确保球不会掉下来，使潜在的问题得以纠正，甚至在它们存在之前。
*   **利用测试自动化和持续集成的优势**:当涉及到解决整个开发过程中的安全问题时，持续测试是你最好的朋友。并且，在有意义的情况下，DevOps 团队应该将用于检测安全性的自动化测试脚本集成到他们的持续集成过程中。这种测试可以包括开发人员编写的脚本，用于测试代码库中的漏洞，以及用于静态代码分析的工具。持续测试策略的另一部分可能涉及到基础设施监控工具的使用。

这些策略产生的数据可以由安全人员进行分析。根据这一分析，可以向 DevOps 团队提供关于前进道路的建议，以便在潜在的安全漏洞进入生产并被利用之前将其消除。

## **了解安全人员在开发流程中的角色**

安全人员通常不是编码人员，开发人员通常也不是安全专家。虽然有时可能会令人沮丧，但管理这种脱节的解决方案就在上述的协作中。开发人员需要使用各种技术来开发高质量的产品。有可能这些技术将有助于解决需要安全专业知识的安全问题。安全人员的工作不应该总是反击这些技术的使用。安全团队应该充当推动者的角色，了解开发和运营人员打算做什么，并预见创新将对安全产生的潜在影响，然后了解这些潜在影响，并深入了解需要采取哪些措施来降低风险。

## **结论**

在过去，安全性通常被认为是应用程序及其基础设施开发中的事后想法。更快的开发周期和更频繁的应用程序发布迫使开发组织以不同的方式考虑安全性。现在，一个应用程序在短时间内可以改变很多东西。这迫使 DevOps 组织将安全性集成到开发管道中，使安全性成为开发生命周期中的一个持续考虑因素。通过这种整合，安全人员的职责已经转移。安全团队现在需要成为 DevOps 文化的推动者，在考虑安全性的情况下增强 DevOps 流程。
