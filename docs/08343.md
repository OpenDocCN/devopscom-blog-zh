# F5 Networks 正在规划 NGINX 之外的开源项目

> 原文:[https://devo PS . com/F5-网络-规划-开源-项目-超越-nginx/](https://devops.com/f5-networks-planning-open-source-projects-beyond-nginx/)

F5 Networks 今天披露了在未来几个月推出多个项目的计划，这些项目将扩展其对开源的承诺，超越该公司在 2019 年收购的 NGINX 代理软件。

该声明是在 NGINX Sprint 的一次在线活动中宣布的；其中两个项目是 NGINX 的扩展，除了管理 NGINX 实例之外，还将解决基于微服务的应用程序之间的网络需求。第三个项目是一个平台无关的开源开发工具，用于构建不一定需要 NGINX 代理软件的现代应用程序。

与此同时，F5 Networks 还推出了开源 NGINX 现代应用参考架构，使 IT 团队能够访问完整、完全可操作的基于微服务的应用，该应用可在 Kubernetes 集群上几分钟内完成部署。基于由预连线和预集成的开源组件组成的模块化方法，F5 Networks 致力于与多家第三方厂商和组织合作扩展这一框架。

![F5](../Images/3f8ae76b84552022fdde388ca7e048a4.png)

F5 Networks NGINX 产品组总经理 Rob Whiteley 表示，该公司还致力于更新其 NGINX 代理软件，以超越竞争对手开源特使代理软件，该软件是在云计算原生计算基金会( [CNCF](https://www.cncf.io/) )的支持下开发的。NGINX 仍然比 Envoy 应用更广泛，主要是因为它更轻量级，更容易部署，但 Whiteley 承认 Envoy 具有更先进的功能。

当然，特使是 Istio 服务网络的核心。F5 Networks 既支持 Istio，也支持基于 NGINX 开发的服务网格。根据组织需要支持的应用程序编程接口([API](https://devops.com/?s=APIs))的数量，IT 团队可能会利用代理软件、入口控制器或服务网格的 API 网关来管理它们。

最后，F5 Networks 表示，它将参与 Kubernetes ingress 项目，并加入网关 API 社区，因为它实施了自己的基于 NGINX 的网关控制器，以确保其代理软件和 Kubernetes 在未来保持紧密一致。

总的来说，Whiteley 表示，F5 Networks 还承诺更加明确其产品的开源和商业版本之间的界限。通常，开源版本是为开发人员和小型团队设计的，而商业版本是为在企业 IT 环境中扩展而设计的。Whiteley 说，总体目标是促进创新，而不使云服务提供商能够通过创建基于开源软件的服务来获得 F5 研发的所有收益。

Whiteley 表示，作为这项工作的一部分，F5 Networks 正在扩大其支持的开源项目的数量，以使开发人员能够更轻松地贡献核心 NGNIX 代理软件之外的代码。

目前还不清楚 F5 Networks 能够在多大程度上团结开源社区来推进 NGINX 之外的其他项目。该公司承诺在未来某个未指明的日期提供更多细节。然而，Whiteley 表示，不仅在服务网格等新兴技术中，而且在许多企业 IT 组织中已经广泛使用的代理软件中，仍然有很大的创新空间。尤其是因为他们中的大多数都在采用基于微服务的应用，这将最终改变 IT 的方方面面。
