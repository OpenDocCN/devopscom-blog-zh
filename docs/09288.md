# 在云中运行分布式应用的 Encore 平台到来了

> 原文:[https://devo PS . com/encore-platform-for-running-distributed-apps-in-the-cloud-arrives/](https://devops.com/encore-platform-for-running-distributed-apps-in-the-cloud-arrives/)

Encore 提供了一个同名的后端开发引擎，用于跨多个云平台快速构建可扩展的分布式 IT 环境。

刚刚筹集了 300 万美元的种子资金，Encore 首席执行官安德烈·埃里克森(André Eriksson)表示，Encore 平台可以分析源代码，从而消除手动配置、连接和设置便携式云计算环境的需要。

除了能够根据静态分析工具提供的元数据自动设置和管理云基础架构之外，Encore 还提供对内置持续集成/持续交付(CI/CD)平台的访问、环境预览、云环境的实时重新加载、身份验证、机密管理和自动分布式跟踪功能。

Eriksson 表示，Encore 环境的开发在一定程度上受到了当今游戏开发人员如何使用引擎来构建应用环境的启发，而不是向开发人员提供一套基于文本的编辑工具，以使用开发人员必须为每个应用实例编写的低级样板代码来构建这些环境。

Eriksson 用 Go 编程语言编写，他说 Encore 平台基于开源的 Encore Go 框架，该框架可以防止 IT 组织被锁定在 Encore 平台上，如今可以部署在亚马逊网络服务、谷歌云平台和微软 Azure 云服务上。

分布式应用程序的构建和部署一直是一项重大挑战。Encore 旨在自动执行 DevOps 团队今天需要为他们支持的每个分布式应用程序手动执行的大多数例行任务。随着跨各种云计算平台部署的应用程序数量的增加，与管理这些环境相关的挑战也在增加。Eriksson 指出，Encore 旨在使开发人员能够通过 DevOps 团队集中管理的方式轻松构建和部署应用程序代码。

Eriksson 说，它还利用了 Go 语言的并发能力，以云无关的方式简化了分布式应用程序的构建和部署。他补充说，DevOps 团队也可以在某些限制的边界内使用 Go 来扩展 Encore 环境。

本质上，Encore 是在为固执己见的分布式计算环境做辩护，作为对灵活性降低的回报，它承诺使复杂应用程序的部署变得更简单。Eriksson 说，Encore 没有看到许多 DevOps 团队寻求部署跨多个云服务的分布式应用程序，但他们被要求在不同的云平台上部署更多的应用程序。他指出，DevOps 团队需要支持的每个云平台只会使整体 IT 环境变得更加复杂，难以管理。

尚不清楚 It 组织在多大程度上准备好接受一种更固执己见的方法来管理云计算环境。然而，依赖一个的概念并不新鲜。平台即服务(PaaS)环境已经使用多年。然而，随着固有分布式的基于微服务的应用程序的兴起，这个问题可能很快就会迫在眉睫，因为 IT 组织根本没有其他方法来跟上开发[应用程序](https://devops.com/the-future-of-application-development/)并在部署后不断扩展的速度。
