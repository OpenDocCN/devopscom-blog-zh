# Imperva 推出新产品，通过了解应用层和代码级漏洞来保护无服务器功能

> 原文：<https://devops.com/imperva-launches-new-product-to-secure-serverless-functions-with-visibility-into-the-application-layer-and-code-level-vulnerabilities-2/>

*Imperva 无服务器保护为 AWS Lambda 环境提供了额外的安全层*

**加利福尼亚州圣马特奥，2021 年 5 月 24 日—** [Imperva，Inc.](https://www.imperva.com/?utm_source=press-release&utm_medium=pr) ，( [@Imperva](https://twitter.com/Imperva) )网络安全领导者，其使命是保护数据和通往数据的所有路径，推出了 [Imperva 无服务器保护](https://www.imperva.com/products/serverless-security-protection/?utm_source=press-release&utm_medium=pr)，这是一款新产品，旨在保护组织免受无服务器计算环境中错误配置的应用程序和代码级安全风险造成的漏洞。设计时考虑到了开发人员和安全团队，新产品很容易部署为 AWS Lambda 层，无需更改代码即可保护功能。

Imperva 无服务器保护构建于亚马逊网络服务(AWS)之上，是 AWS Lambda 扩展中完全集成的工具。该集成使开发人员能够更快地访问新的 Imperva 产品，为他们的 AWS Lambda 环境提供额外的安全层。Imperva 的这一最新创新为该公司保护所有形式的应用程序的解决方案增添了新的内容:传统、API、微服务和无服务器功能。

AWS Lambda 总经理 Holly Mesrobian 表示:“我们看到成千上万各种规模的客户采用无服务器应用程序来快速为他们的客户提供价值。“Imperva 扩展使客户只需进行简单的配置更改，就可以轻松地在无服务器应用的开发运维流程中嵌入额外的安全性。”

开发人员越来越多地采用无服务器功能，这种功能成本更低，配置更少，部署更快。然而，“到 2022 年，对无服务器[平台即服务]平台即服务的 80%的成功攻击都是由于不成熟的工具和流程导致的错误配置或已知易受攻击代码的使用，”Gartner 副总裁、杰出分析师 Neil MacDonald 在 2020 年 3 月的报告“保护无服务器平台即服务的安全考虑因素和最佳实践”中写道。他补充说，“将需要新的方法和技术来保护无服务器，并且应该使用生命周期方法来设计，从开发开始，一直进行到运营。”

Imperva 无服务器保护提供市场差异化功能，帮助组织管理无服务器功能中出现的复杂安全风险。

*   **防范恶意活动:**imper va 专为无服务器计算而构建，独特地支持积极的安全模式，可防范功能内的恶意更改，如零日漏洞。作为 AWS Lambda 层部署，它可以部署一次并应用于多个 AWS Lambda 功能。
*   **内部和外部代码漏洞的可见性和保护:** Imperva 无服务器保护可保护无服务器功能免受嵌入第一方和第三方代码中的漏洞的影响，这是可能触发软件供应链攻击的潜在风险因素。它可以有效地监控和阻止漏洞，而无需复杂的或手动的步骤。
*   **OWASP 无服务器 10 大覆盖范围:** Imperva 无服务器防护可针对错误配置、代码级风险、注入和弱点提供保护。它阻止 HTTP 响应分割和方法篡改、代码注入以及其他复杂的威胁。它还监视不安全的 cookies 和传输、敏感信息的记录、未经授权的网络活动、弱身份验证以及其他潜在的漏洞。
*   **深入了解应用层的安全事件** : Imperva 无服务器保护运行时监控收集日志级信息以提供取证细节，因此安全团队可以全面了解每一次攻击的背景，而几乎不会影响延迟。它还标识和映射运行时使用的第三方依赖项。

“传统的安全技术不是为了了解无服务器功能等短暂的工作负载并为其提供保护而设计的。客户需要功能保护、环境感知和高性能的结合。此外，客户对修改工作负载或更改代码以支持安全功能不感兴趣。Imperva 的首席技术官 Kunal Anand 说:“Imperva 无服务器保护正是为了解决这些需求而创建的。借助 Imperva 无服务器保护，DevOps 和 SecOps 团队拥有了阻止无服务器攻击的强大新功能

prolog is IT 安全总监 Tyler Warren 表示:“我们很高兴与 Imperva 合作设计他们的 Imperva 无服务器保护解决方案，并相信他们通过在功能中嵌入保护，为无服务器安全提供了正确的方法。“Imperva 是一个真正的竞争优势，确保我们始终处于防范不断发展的安全威胁的前沿，包括那些在无服务器功能中出现的威胁。”
