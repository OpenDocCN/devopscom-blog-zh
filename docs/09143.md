# CircleCI 收购 Ponicode 获得 AI 测试工具

> 原文:[https://devo PS . com/circle ci-acquisites-poni code-to-gain-ai-testing-tool/](https://devops.com/circleci-acquires-ponicode-to-gain-ai-testing-tool/)

CircleCI 今天透露，它已经[收购了 Ponicode](https://www.businesswire.com/news/home/20220309005008/en/CircleCI-Acquires-Ponicode-Adding-AI-powered-Testing-to-Their-CICD-Offering-to-Ease-Daily-Life-for-Software-Developers) 以获得对注入人工智能(AI)的单元测试平台的控制，这使得在将应用程序代码上传到构建之前对其进行审查变得更加简单。交易的财务条款没有披露。

CircleCI 工程高级副总裁 Jean-Philippe (JP) Leblanc 表示，目标是将 Ponicode 的测试技术整合到 CircleCI 的持续集成/持续交付( [CI/CD](https://devops.com/?s=CI/CD) )平台中。他补充说，这种方法将有可能将算法应用于单个开发人员的代码，以及在一个版本中从多个开发人员聚合的代码，因为它在整个 DevOps 工作流中不断更新。

Ponicode 最初于 2020 年 6 月推出，作为微软 VS Code 的 JavaScript 单元测试扩展。此后，该公司开始支持 [TypeScript](https://devops.com/is-typescript-the-new-javascript/) ，Java 和 Python，包括变量嘲讽和推断嵌套对象复杂结构建议等复杂功能。

DevOps 团队经常遇到的问题是，即使单个开发人员在上传到构建之前测试他们的代码，集成不同开发人员的代码仍然会导致应用程序失败。Leblanc 指出，Ponicode AI 测试工具将使开发人员和 IT 运营团队对用于驱动应用程序的代码质量更有信心。

虽然没有多少开发人员愿意承认这一点，但 Leblanc 指出，仍然有许多草率的代码进入应用程序。

此外，Leblanc 表示，收购 Ponicode 将有助于代码审查过程变得更加高效。他说，大多数问题将通过人工智能测试工具浮出水面，而不是要求另一名开发人员或专门的测试人员来审查代码，人工智能测试工具提供了内置的同行审查。Leblanc 补充说，因此，人工智能测试工具将提高用于驱动应用程序的代码的整体质量。开发人员也可以将更多的时间花在驱动应用程序的业务逻辑上，而不是花在代码审查上。

然而，Leblanc 说，目标不是消除对专门测试团队的需要，而是增加他们和他们一起工作的开发人员。

目前还不清楚测试代码的责任在多大程度上转移到了开发人员身上。然而，许多开发人员更喜欢通过人工智能工具来解决问题，这样他们就可以自己解决问题，而不是让别人发现问题。Leblanc 说，事实上，已经有超过 20，000 名用户使用直接插入开发人员集成开发环境(IDE)的 Ponicode AI 工具。

总的来说，从来没有像现在这样关注开发人员的生产力。组织正试图同时构建和部署比以往更多的应用程序，这主要归功于新冠肺炎疫情之后推出的数字业务转型计划。他们面临的挑战是招募和留住开发人员。因此，组织正在寻找通过消除尽可能多的手动过程来提高开发人员效率的方法。
