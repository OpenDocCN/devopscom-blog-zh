# DevOps 如何从新冠肺炎手中拯救你的企业

> 原文：<https://devops.com/how-devops-can-save-your-business-from-covid-19/>

新冠肺炎彻底改变了美国劳动力的面貌。远程工作已经从一个小众领域变成了数百万人的一种生活方式。事实上，研究表明，由于新冠肺炎危机，大约一半的美国雇员开始在家工作。

当一些雇主努力适应这种新的生活方式时，另一些雇主很快就适应了。展望未来，以这种方式适应的能力可能意味着许多企业的生死存亡。

一旦发现针对这种疾病的疫苗，这种趋势似乎不会结束。包括蒙德勒兹 [和巴克莱](https://www.cnbc.com/2020/05/01/major-companies-talking-about-permanent-work-from-home-positions.html) 在内的许多大公司都表示，在家办公可能会成为他们商业模式的永久固定模式。原因显而易见。在这样一个技术和通讯发达的时代，人们可以通过电话或电脑相互交谈，就像走到某人的办公桌前一样容易。

很快，一家企业就可以利用 1，000 个家庭办公地点，而不仅仅是一个。但所有这些都归结于全国 IT 部门的能力。

一份 [来自 GitHub](https://github.blog/2020-05-06-octoverse-spotlight-an-analysis-of-developer-productivity-work-cadence-and-collaboration-in-the-early-days-of-covid-19/) 的最新报告出炉，聚焦新冠肺炎时代的新软件开发。虽然许多行业都不可避免地会出现中断，但 IT 专业人员已经应对了这一挑战。该报告的作者 Nicole Forsgren 甚至被引述说，“面对不确定性，开发人员继续做出贡献并表现出弹性。”展望未来，每个组织团队最好的朋友是什么？

如果你看看市场的走向，那么 DevOps 肯定会迎来一场大繁荣。

## **devo PS 的弹性**

随着越来越多的人拥有智能手机和其他必要的技术，对可靠软件的需求只会增加。五十亿人现在拥有手机，这将导致全球范围内的[](https://devops.com/devops-places-software-testing-in-the-crosshairs/)速度问题。这就是 DevOps 介入收拾残局的地方。

DevOps 能够同步、组织和自动化新软件发布的速度。根据设计，它是为远程操作而设计的。但是，对于在办公室或任何其他地点进行的操作，它仍然是可行的。DevOps 有能力适应和改变周围的世界。

在新冠肺炎时代，这是一种极其有利的模式，因为公司可能希望本周在办公楼工作，下周在家办公。它是你所生活的世界所需要的软件。

然而，DevOps 的好坏取决于使用它的人。对于公司的个人来说，适应 DevOps 模式可能是一个挑战，他们已经有太多的事情要做了。这就是为什么公司推出了[devo PS Institute](https://devops.com/devops-institute-upskilling-the-enterprise/)。在这一点上，它只是归结为公司找到合适的人，他们拥有必要的技术实力和软技能，将 DevOps 的好处带到他们的工作场所。

根据 DevOps 收集的 [数据，许多企业显然面临着信息技术和其他技能方面的人才短缺，这些技能可以帮助他们适应更远程的工作方式。](https://devops.com/need-for-increased-training-strained-budgets-strain-enterprise-it-teams/)

报告显示，48%的受访者没有具备管理现代 IT 环境所需的技能。在所有接受调查的人中，只有 28%的人认为新员工具备基本技能。

随着越来越多的企业开始在家办公，这些技能只会变得更加重要。例如，当人们不在同一地点时，快速沟通的能力是必不可少的。这也不一定是人为错误。有缺陷的技术可能导致人们不能及时看到信息或电子邮件，从而延误重要的项目。

完美的 DevOps 工程师应该是软硬技能完美平衡的人。他知道如何迅速纠正网络错误，但有必要的沟通技巧在问题出现时伸出援手。一家公司理想的开发工程师是这样的人:他 [了解 VPN 从里到外是如何工作的](https://surfshark.com/learn/what-is-vpn) 但也以 IT 部门以外的其他人可以理解的方式进行交流。

## **实施 DevOps 的其他考虑事项**

对于有意利用 DevOps 的公司来说，还需要考虑一些额外的因素。首先，公司里的人需要开始向云推进技术和开发。在像新冠肺炎危机这样的情况下，没有明确的时间表来说明事情何时会恢复正常。如果你的公司还没有利用云，那么现在是时候开始了。

接下来，公司需要开始尽可能实施自动化。沟通的延迟会使公司的时间表推迟几周甚至几个月。另外，当人们意识到的时候，你会意识到有多少时间实际上是花在执行各种可以通过软件完成的任务上的。

你可以让每个人在他们的电脑上安装恶意软件扫描器，而不是让一个人检查恶意软件，这样他们的 [安全就会在后台](https://privacycanada.net/best-malware-scanner/) 得到检查，因为他们可以专注于更重要的任务。检测恶意软件只是开始，因为企业还可以使用自动化来测试功能，并同时部署到多个环境中。

最后，当雇主们求助于 DevOps 来履行他们公司的义务时，关注潜在的倦怠是至关重要的。随着越来越多的人在家工作，他们需要在更近的角落里兼顾工作、家庭和其他义务。

DevOps 是一个巨大的救命稻草，但是你要确保给你的员工提供你的团队可以处理的可持续工作负载。这个时间也有利于在人们的个人生活和工作之间建立清晰的界限。为员工之间的交流设定明确的时间是个好主意。在这段时间之外，重要的是让人们放松下来，关注生活的其他方面。

## **结论**

新冠肺炎危机会过去，但美国的劳动力可能永远不会像以前一样了。许多组织可能最终决定在家工作比租用办公室有更多的好处。人们会度过这个难关， 德沃普斯会在那里 伸出援手。
