Runecast 在内部安全运行，能够洞察内部和云中发生的情况，自动对 Kubernetes 进行安全合规性检查。没有敏感的公司、员工或客户数据需要离开组织。根据 CISA 的 KEV 目录，Runecast 旨在使漏洞管理变得容易，在仪表板上列出 CVE 以及已知已被利用的漏洞。

## **扩展的 Runecast Kubernetes 覆盖范围“左移”**

部署 Runecast 后(在几分钟内)，IT 团队会看到一个关键问题列表和根据 Kubernetes 最佳实践进行补救的步骤。混合环境也可以看到 AWS、Azure、VMware、Windows 和 Linux 的问题，所有这些都来自同一个控制面板。

新的 Kubernetes 的智能和能力将在 Q2 的结尾出现，包括:

**图像扫描**

通过为部署的 Kubernetes 容器映像提供漏洞扫描，我们的 Kubernetes 漏洞管理将从集群向下扩展到节点。这将为客户提供自动化 Kubernetes 漏洞管理的能力，确保 Kubernetes 部署满足最佳实践、安全性和合规性要求。

**CI/CD 整合**

新版本的 Runecast 将通过直接与 CI/CD 管道集成，为开发和 DevOps 团队带来容器扫描功能，以确保“快速开发和部署”方法的安全性。CI/CD 集成将允许客户在拉动式请求等方面有限制标准。会被自动拒绝。Runecast CI/CD 集成将按需扫描模板或清单，以查找作为拉请求的一部分要部署的容器，如果在该请求的容器映像上存在预定数量的漏洞，它将自动拒绝拉请求并阻止该容器被部署。

**扩展(基于代理)收集**

Runecast 平台将允许团队更深入地了解 Kubernetes 集群上运行的实际节点，并将大大增加 Runecast 能够主动发现的漏洞数量。
