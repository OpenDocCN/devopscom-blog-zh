# 网上研讨会:你不能“做”DevOps。你可以“做”连续交货。

> 原文:[https://devo PS . com/webinar-you-cant-do-devo PS-you-can-do-continuous-delivery/](https://devops.com/webinar-you-cant-do-devops-you-can-do-continuous-delivery/)

“DevOps”是最新的时尚。像网飞、Etsy 和 Intuit(“独角兽”)这样的领先公司已经将其 it 开发和运营组织转变为一种称为 DevOps 的新文化:开发和运营协同工作，以加快软件和应用交付的步伐。但是他们没有实现 DevOps。DevOps 向他们文化的转变是他们实现软件持续交付的结果。连续交付是将自动化和编排技术应用于应用程序生命周期的过程。在本次网络研讨会中，您将了解到您的组织如何实施持续交付，以此作为实现您自己的 DevOps 转型的一个步骤。通过持续交付，您可以加快应用交付，这将使 it 更好地满足组织的需求。通过持续交付，it 成为业务的推动者。
