# 5G 将如何改变我们所知的公共云

> 原文：<https://devops.com/how-5g-will-change-the-public-cloud-as-we-know-it/>

说到实现 5G 的承诺，我们还有几年的时间。然而，在引擎盖下，5G 基础设施和 5G 将实现的新应用的推出——从 AR 和 VR 到[新的无人机功能](https://devops.com/indoor-robotics-raises-15-million-in-series-a-to-revolutionize-building-monitoring-and-security-practices/)等——将推动基础设施需求的巨大变化，首先是公共云。

在 5G 时代，许多将成为可能并利用这种网络基础设施的应用将需要移动和处理大量数据，无论是联网汽车、带实时分析的视频监控还是 AR/VR，以至于这些应用对集中式公共云的响应能力要求如此之高，以至于无法真正解决数据需求。随着 5G 创建一个新的生态系统，许多应用程序将以新的方式构建，并依赖于边缘云的属性:边缘计算作为类似云的服务交付。支撑它的将是云原生架构和其他高度自动化和灵活的基础设施，以帮助它成为现实。让我们来看看一些关键要素。

### 什么是“边缘云”？

如今，在公共云中构建应用的 SaaS 公司已经习惯了自助式弹性基础设施。你不必考虑硬件，你按使用付费，有 API 可以利用，最重要的是一切都是高度自动化和灵活的。更有可能的是，你只需要在建造时考虑几个集中的地点——一个在西海岸，一个在东海岸，一两个在欧洲或亚洲。

在 5G 未来，这种情况可能将不再存在。许多新的创收服务将依赖于低延迟、高带宽能力，使应用程序和数据处理更接近用户或设备。从工业应用、物联网、沉浸式信息娱乐(虚拟和增强现实)到远程医疗等一切都将能够利用 5G 网络和边缘计算处理来提供新的功能和体验。

诸如此类的服务不仅依赖于连接，还依赖于收集数据、处理数据的能力，更重要的是，以非常高的速度移动和处理数据。

在这个新世界中，您仍然希望从您的基础架构中获得同样的东西—它需要具有弹性，并且能够扩展和缩减。可靠性、自动化和敏捷性(快速迭代和部署的能力)变得更加重要。它需要在更多的地方可用和运行，以便接近您所服务的客户和设备。你不会想到全球的三个地方或美国的两三个地方，相反，你可能会想到仅在一个都会区就有几十个地方。必须围绕这一点构建一种全新的计算环境，以及云基础架构环境。

### 现在是多模式，下一步是什么

5G 环境将需要非常灵活的基础设施，这些基础设施可以高度自动化，可重复使用，并且可以以非常通用的方式使用。如果您正在全国各地建设城域、接入点数据中心，那么在每个站点为每个应用程序使用高度专用的基础设施是不划算的；它在灵活性方面太受限制，而且过于繁琐，难以大规模管理。

基础架构挑战的一个方面是，服务提供商仍然部分依赖于倾向于在物理或虚拟化环境中运行的单片传统应用程序，以使它们可部署、可控并能够自动化。但是新的应用程序是用基于容器的不同技术栈设计的。对于同时存在于这两个领域的应用程序，跨虚拟机和容器的灵活性是必不可少的。因此，5G 的推出将采用多模式基础设施，结合虚拟机和容器，以支持传统和现代应用。目标是:在任何地方实现简单的自动化。

虚拟化环境和容器环境将需要相互协调运行。它们都需要自动化，但是因为每一个都有自己的需求，所以选择一个技术堆栈和方法来结合并使用这两者是很重要的。

虽然虚拟机和虚拟化网络功能今天更加普遍，但这种情况不会持续太久。随着运营商转向推动持续改进和加速价值实现的微服务架构，CNFs 的采用率越来越高。

### 前方的路

由于 5G 用例仍处于早期阶段，架构灵活性将允许运营商转向任何出现的杀手级应用。能够同时提供性能、敏捷性和灵活性的以数据为中心的基础架构至关重要。

当我们展望 5G 的可能性时，回想一下 20 世纪 90 年代初是有帮助的。有了 AOL 拨号和一个 2400 波特的调制解调器，你可以发电子邮件、上网冲浪(尽管速度慢得令人痛苦)和玩几个游戏。宽带的出现带来了全新的应用和用例，如电子商务和媒体流，并推动了基础设施的变化以支持它们。5G 展现了同样类型的巨变潜力，这将为消费者和企业开辟一个全新的创新世界。
