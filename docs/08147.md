*Vercel 和 Google 向超过 50，000 名与会者展示跨平台创新。*

**加利福尼亚州三藩市，2021 年 6 月 15 日—** [Vercel](https://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fvercel.com%2F&esheet=52445236&newsitemid=20210615005365&lan=en-US&anchor=Vercel&index=1&md5=7f1259f8bc569e46b38fca8af78984fe) ，一个创建无与伦比的 web 用户体验的综合平台和 [Next.js](https://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fnextjs.org%2F&esheet=52445236&newsitemid=20210615005365&lan=en-US&anchor=Next.js&index=2&md5=ff7c198c00e292f20b5c0b9d6fd689fc) 的创建者，领先的 React 和 JavaScript 框架，用于前端 web 开发，今天宣布 Next.js 11，通过速度和协作加速前端开发人员构建下一个大东西的能力。新的创新包括显著更快的启动和更改、实时反馈、即时实时协作和显著的图像优化增强。此外，谷歌 Chrome 中的 Aurora 团队为 Next.js 开发了 Conformance，这是一个提供精心制作的解决方案和规则以支持最佳加载的系统；js 脚本优先级和字体增强自动化，减少了第一次和最大的内容绘制时间。总之，这些改进使开发者能够提高核心的网络生命指数得分，这些得分现在正被谷歌用来影响搜索排名。开发者可以通过访问 nextjs.org/11 来尝试这些新功能。

Next.js 是第一个大规模成功的混合框架，也是唯一一个同时满足静态和动态网站和应用需求的框架。利用 React 作为协作的基本单位，为包括抖音、Hulu、Ticketmaster、Staples 和漫威在内的公司的开发人员提供了从一个页面扩展到十亿个页面的灵活性。

自 2020 年 10 月第一次 Next.js 会议以来，Next.js 开源贡献者已超过 1600 人，推动 NPM 的下载量从 410 万增加到 620 万。Vercel edge 网络上所有网站和应用的流量增加了一倍，每周部署量增加了三倍，使用 Next.js 的 Alexa 前 10，000 个主页的数量增加了 50%。

Vercel 首席执行官 Guillermo Rauch 表示:“无论您是从一个静态页面开始，还是部署一个拥有数十亿页面的动态网站，Next.js 都是一个混合框架，可以满足您的需求，并与您一起成长。“我们相信，最快、最愉快的最终用户体验必须与最快、最愉快的开发人员体验携手并进。”

**next . js 11 中的新创新包括:**

*   **有史以来最快的前端平台—** Next.js 的创新将启动时间提高了 24%,将从编辑到显示的处理时间减少了 40%。
*   **Next.js Live(预览版)** — 将 Next.js 的全新开发服务器引擎与 Vercel 平台相结合，使团队能够在世界任何地方即时共享、聊天、绘图和编辑，开发人员能够在不到五秒钟的时间内生成一个 URL。由于对静态和动态 Next.js 应用程序的支持，共享和访问 URL 总是即时的，不管页码如何，因为不涉及构建过程。Next.js 将完全在 web 浏览器中运行。利用 ServiceWorker、WebAssembly 和 ES 模块，当您在本地机器上运行 Next.js 时，在远程协作的上下文中，一切都是可能的。Next.js Live 即使在离线时也能完美运行，无需运行或操作远程虚拟机。
*   **实时反馈—** Vercel 使开发人员能够立即看到对变化的反应。从第一天起，Next.js 就优先考虑这一新功能，它建立在对零配置 webpack HMR 的支持以及与脸书的密切合作基础上，这导致了 Next.js 中快速刷新创新的开发。现在，开发人员可以运行 next dev 命令，他们的前端项目已经准备就绪。文件保存后，浏览器会立即反映更改。这不仅适用于样式，而且由于快速刷新，即使进行了更改，所有状态也会保留。
*   **业界首创的图像增强功能** **—** Next.js 让开发人员可以快速地将 HTML 图像标签替换为 Next.js 图像组件，从而为图像传送到的每个设备视窗优化图像。Lighthouse 测试提供了在运行 Next Start 或通过其超高速 edge 网络部署到 Vercel 时，提高开发中开箱即用的加载性能的见解。与 Cloudinary、Cloudflare、Akamai 和 Imgix 的紧密社区驱动集成将这些优势扩展到了领先的第三方内容网络。新的增强功能包括对 src 的 import 关键字的支持，该关键字可用于引用作为存储库一部分的基本图像，并自动设置本地图像的宽度和高度参数，减少累积布局偏移，这是一个核心的 web 重要功能。它还支持导入图像上的图像占位符，简化了从空白到图像的过渡，此外还支持动态图像的模糊。
*   **符合 Next.js —** 这一新系统提供了精心制作的解决方案和规则，以支持最佳装载。它将开发人员从通过编译器加载性能的复杂规则中解放出来，该编译器执行最佳实践以确保可预测的结果。这使得团队富有成效，随着团队的成长和功能的增加，这变得至关重要。
*   **Next.js 脚本组件—** 这一基础优化使开发人员能够设置第三方脚本的加载优先级，从而显著提高加载性能。开发人员现在可以简单地定义策略属性，Next.js 将对其进行优先级排序。它支持三种不同的加载策略——在交互之前，对于需要在页面交互之前获取和执行的关键脚本，如安全性和身份验证；after-interactive，用于标签管理器和分析等脚本，这些脚本可以在页面交互后获取和执行；和 lazy-onload，用于聊天支持小部件等脚本，它们可以在空闲时间等待最后加载。
*   **字体增强自动化—** 超过 80%的网页正在使用定制的网络字体作为其品牌、设计的一部分，或者用于保持跨浏览器和设备的一致性。这种定制会造成性能损失。4 月，谷歌推出了字体优化，包括 Next.js 上的谷歌字体和 Adobe Typekit，以及谷歌字体的 Angular 字体优化。在 Next.js 中，自定义字体的 CSS 现在会在构建时自动内联。这消除了获取字体声明的往返行程，将第一次和最大内容绘制时间缩短了 5 秒。默认情况下，Next.js 中的 preconnect 标记可以更快地与底层字体文件建立连接。

“在谷歌 Chrome，我们相信我们与 Next.js 的合作是如何为加载性能设定可预测结果的一个例子，”谷歌开发者权益倡导者侯赛因·吉尔德(Houssein Djirdeh)说。“我们喜欢与伟大的框架一起工作，帮助开发人员使 web 更快，我们非常感谢那些使 web 蓬勃发展的框架的维护者和赞助者。特别感谢 Next.js 团队与我们一起测试，并分担了许多早期尝试的成本。”
