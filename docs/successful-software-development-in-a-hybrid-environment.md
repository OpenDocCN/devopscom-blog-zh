# 混合环境中成功的软件开发

> 原文：<https://devops.com/successful-software-development-in-a-hybrid-environment/>

随着新冠肺炎·疫情的出现，在家或在任何地方工作的愿望越来越强烈。员工们列举了很多不去办公室的原因，包括减少通勤时间、提高工作效率、照顾家人和探索新的地点。人们越来越适应远程交互和新技术工具，这些工具的出现支持了在任何地方工作的趋势。根据对 5，000 名美国人进行的[](https://wfhresearch.com/)月度调查，员工平均每周有 2.5 天想在家工作，而 73%的员工寻求混合工作环境 在这种环境中，他们可以灵活地选择工作地点和时间。

新冠肺炎德尔塔变体的迅速传播也阻止了雇主们强制要求全职员工重返办公室。因此，今天的工作人员的新常态充其量仍将是混合式的——几天在办公室，几天在家工作。我们是转型中的劳动力。对于许多公司来说，这给他们的数字化转型计划带来了不必要的压力。

一个问题仍然存在:当员工离开他们的团队和办公室时，公司如何继续创新和执行他们的软件开发计划？下面是我的一些建议，可以帮助组织在今天的混合工作环境中继续他们的软件开发项目:

## 包容性环境

混合劳动力需要高度包容的环境。 作为一家与数百家客户远程合作的近岸 IT 合作伙伴，我们确保我们的团队 有同样的机会接触经理、人力资源和技术，这些都是项目成功和与客户有效合作所必需的。例如，我们了解到 包容性的最佳方式是在客户会议期间采取虚拟或面对面的方式。培训你的经理为员工提供同样的晋升机会也很重要，无论他们是否在办公室。从高管团队开始，每个人都需要支持员工，不管他们在哪里。新员工和合伙人的入职也是如此——在过去的 18 个月里，这几乎是一个虚拟的过程。增加在线指导和入职平台，以便在员工混合的情况下联系员工和管理客户项目，这是我们采取的另一个重要方法，以确保每个人都感到被包容和参与。

## 协作和知识共享

员工们突然被迫离开办公室，因为他们习惯了频繁的临时会议和午餐聊天。对许多人来说，这种转变对[](https://www.wework.com/ideas/research-insights/research-studies/the-impact-of-working-from-home-on-collaboration)【头脑风暴】、协作和知识共享以及创新产生了负面影响。我们为我们的团队提供了使用不同技术工具的灵活性，以弥补他们在办公室环境中最缺失的东西——定期、随意的人际互动。我们的团队使用各种通信工具进行频繁登记，以便与客户及其同事联系。因此，我们的员工表示，他们比以往任何时候都更加关注团队成员和客户。他们与同事和客户进行更多的即兴聊天和“即时”会议，就像在办公室一样。我们还将视频会议作为我们的首选沟通渠道。视频会议工具提高了参与度，并允许团队成员从他们喜欢的设备在任何地方参加虚拟会议。团队在任何地方进行互动、分享和协作的能力，对于开发 [尖端团队和软件是至关重要的。](https://www.cinqtechnologies.com/how-to-develop-cutting-edge-software-teams/)

## 信任文化

根据哈佛商业评论的报道，34%的员工认为他们的主管“对他们的工作技能缺乏信心”，更多的员工认为他们需要随时证明自己。这样的情绪凸显了信任的缺乏，影响了发展 的成果。雇主需要相信他们的员工在工作日或任何他们能够成功完成手头任务的时候都在勤奋、持续地工作。例如，远程工作允许个人在他们感觉最有效率的时候工作。组织需要认识到个人的生产力周期，并理解不同的团队成员可能在一天中的不同时间(包括正常工作时间之外)具有生产力。

对期望保持透明、坦诚和开放是与员工建立信任的关键。只有拥有合适的团队和人员，成功的数字化转型和创新软件开发才有可能。从客户的角度来看，这些是他们想要的 [类型的 IT 合作伙伴](https://www.cinqtechnologies.com/cinq-is-a-great-place-to-work-in-brazil/) 。技术能力和一个让所有员工都感到被重视、被信任并有动力尽最大努力的环境，对于开发有效的数字解决方案至关重要。

员工们已经明确表示，他们不想全职回到办公室。对于许多组织来说，回归常态的核心是培养一支能够与数字开发伙伴有效合作的混合劳动力。创建合适的环境和加强员工关系是确保软件开发项目能够在今天的新常态下成功继续的关键。
