# **快速 Node.js 讲解人** 

在专业人士的圈子里，Node.js 和 best Node.js 框架被认为是促进 web 应用程序开发的一种方式。

**客户端应该知道的 Node.js** :这是一个特殊的运行时环境，开发者使用它来创建 web 应用。它的前端(客户端)和后端(服务器端)都可以用同一种语言编码:JavaScript。以前，这是两个不同的领域，分别由不同的开发人员用不同的编程语言来处理。

## **了解 Node.js 的哪些内容**

Node.js 特别适合开发用于多个用户同时连接的实时应用程序。但是，不建议用于复杂计算的应用程序。

你所知道的一些大公司——网飞、特雷罗、贝宝、LinkedIn、沃尔玛、优步、Groupon、易贝和 NASA——都在他们的应用程序中使用 Node.js。

Node.js 为公司带来了诸多好处，包括:

*   JavaScript 属于最受欢迎的编程语言之一——根据 [GitHub 统计数据](https://madnight.github.io/githut/)，截至 2017 年第三季度，JavaScript 是最受欢迎的编程语言，占拉请求的 22.5%。这意味着**很容易找到一个有经验的开发者**；
*   Node.js 的流行意味着**众多现成的解决方案**和贡献，可以被开发者用于他们的应用；
*   Node.js 给了开发者很大的自由和空间来创建一个特定的应用；
*   由于前端和后端用同一种语言编码，所以只需要**一个程序员**；
*   由于上述这些因素，开发和部署时间缩短， **app 可以在短时间内上市**；
*   更少的搜索时间、更少的开发者和更快的开发意味着更低的成本。

## Node.js 背后是谁？

Node.js 出现于 2009 年，现在得到了活跃且快速发展的社区的支持。截至今天，Node.js 有 800 万个实例，GitHub 上有超过 1500 个贡献者和 39672 颗星。Node.js 基金会的执行董事 Mark R. Hinkle 说，他的组织计划发展、参与和教育 T1。因此，未来的应用程序在可维护性和最终扩展方面将会有很好的前景。

# **值得考虑的 Node.js 框架**

### **express . js(GitHub 上 35310 颗星)**

Node.js 的经典版本出现在 2010 年，是一长串流行 Node.js 框架的基础。它被认为是极简和简单的。Express 并不固执己见，并为开发者提供了更多的自由，因为他们可以使用各种模块来构建应用程序。由于其灵活的本质，它最适合于计划定制和扩展并需要长期支持的大规模应用程序。

***关于 Express.js** **客户应该了解什么？*** 快速、简单、灵活

### **sails . js(GitHub 上 18088 颗星)**

这个框架使用 Express 作为基础，但它是一个完整的产品，可以通过可用的功能来应用。它适合快速发射。Sails 特别适合开发和设置需要快速回答的实时应用程序，并且兼容任何前端。

***客户端应该了解 Sails.js 的哪些内容？*** 完整，开发快捷，适合实时应用

### 流星(GitHub 上的 38，755 颗星星)

这是一个全包的流行 Node.js 框架，允许构建实时可伸缩的 web 应用程序。它可以处理任何设备(web、iOS、Android)的应用程序。有了 Meteor，开发人员可以用更少代码完成更多的工作。它适合作为在所有主要平台上启动项目的完美工具。

***关于 Meteor.js** **客户应该了解什么？*** 全栈，省时，通用

### **Loopback(GitHub 上的 8000 颗星星)**

这是一个高度可扩展的模块化框架。Loopback 专门用于快速轻松地创建 API。它允许在内部或云中运行应用程序。Loopback 非常适合编写用户的身份验证和授权。

***关于 Loopback.js，客户端应该了解什么？*** 开发简单快捷，因为提供了很多开箱即用的功能，适合复杂的集成。

### **哈比神(GitHub 上的 8701 颗星星)**

该框架出现在 2011 年，也是基于 Express。从那以后，它发展成为一个独立的框架，采用不同的方法:提供尽可能多的开箱即用的用例。它是为大型团队和大型项目设计的，对于一个简单的应用程序来说太复杂了。

***客户端应该了解 Hapi.js 的哪些内容？*** 配置优于代码，可靠省时由于专注于编写可重用的 app 逻辑，大型项目

### 奖金框架:**KOA . js(GitHub 上 18495 星)**

这是新一代的框架。Koa 是 Express 的一个更轻便、更灵活的版本。它适合大型和小型项目，并提供进一步的定制和扩展。

***客户端应该了解 Koa.js 的哪些内容？*** 快速开发，可靠 app 基础，灵活
