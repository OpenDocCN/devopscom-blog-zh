# 为什么你应该撕掉你的组织结构图，围绕价值流进行重组

> 原文:[https://devo PS . com/why-you-should-rip-up-your-org-chart-and-reorganize-around-value-streams/](https://devops.com/why-you-should-rip-up-your-org-chart-and-reorganize-around-value-streams/)

为了在数字经济中竞争，团队必须优化 的效率和效果——这是一项艰巨的任务。传统的支配者层级是一个问题，因为使用命令和控制的组织创建了抑制团队层面创新的筒仓和移交。协调自治释放了个人的创造精神——但领导者如何确保与他们的愿景保持一致？

消除与竖井相关的延迟和浪费并培养自主性的一种方法是围绕[价值流](https://www.vsmconsortium.org/)进行安排。价值流是拥有客户的任何东西，或者是交付产品或服务的任何东西。所有的价值流都始于创意，止于变革的实现。或者，更好的是，当我们珍爱的客户意识到预期的价值结果时。敏捷的初衷是更快地为我们的客户提供“价值”……但是我们很少谈论这个价值是什么，或者衡量它的存在。

传统的组织结构图根据专业知识来安排资源。这意味着随着工作在价值流中的进展，工作被移交给不同的团队。不管人们是否这样称呼价值流，也不管他们是否积极管理价值流，价值流都是存在的。每次切换发生时，都会产生延迟或错误的机会。这也意味着人们很难理解端到端系统。相反，他们按照要求去优化他们的拼图，却没有真正理解他们的行为如何影响上游或下游的工作。

更先进的工作方式，如敏捷和 DevOps，确实促进了跨职能、自治的团队，但它们通常会在计划和交付之间留下很大的差距，并且在将客户体验反馈连接到计划方面并不总是明智的。价值流导向通过将个人背景(跨职能)识别到交付给客户的价值中来促进团队多样性。

围绕价值流进行组织是培养以价值流为中心的思维和工作方式的基础，这种方式可以提高效率和效力。然后，客户体验得到改善，组织绩效达到更高水平。这是因为:

*   将客户体验放在首位会带来更高的保留率和推荐率
*   提升 *流程* (见下文)的概念，让团队能够优先清除浪费
*   平衡创新和改进减少了[技术债务](https://devops.com/?s=technical+debt)，辛劳和计划外工作，因此员工体验也更好
*   在一个连续的循环中，将规划、交付和反馈联系起来，这有助于团队的团结和决策制定

心流是指想法能够以多快的速度在价值流中流动，并被它们的设计对象所利用。它关乎产出和效率。客户体验告诉团队他们的工作成果和效率。这些方法一起使用，确保团队做正确的事情，并朝着正确的方向前进。

在这种模式中，领导者分享他们的权力并授权给他们的团队。他们支持团队自我发现改进，并确保这些本地的知识能够传播到全球。他们帮助他们的团队了解他们花在创新上的时间比花在反应或改进上的时间多。然后，领导者可以帮助他们的团队减少分散注意力和压力的计划外工作，转而投资于提高未来的创新能力。

遵循以下步骤，成为以价值流为中心的组织:

1.  确定你的价值流
2.  确定谁参与了价值流的每一步
3.  通过与您的价值流团队进行可视化协作，让您的价值流可见
4.  使用工具收集价值流数据，并为您的团队提供见解
5.  定期会面，检查并调整你的价值流

所有的组织都是由价值流组成的，不管它们是否可见，大多数大型组织都有复杂的价值流互联网络。这些依赖关系不容易分离。它们也不容易被发现。但是一旦您映射了它们，您就可以开始查看它们在哪里连接和管理依赖项。最终目标是打破这些依赖性，因为它们会导致延迟和风险，但是当它们不可见时，很难做到这一点。

使用这种模式，将组织愿景与价值流联系起来变得更加容易。组织结构扁平化。领导者了解他们对哪些价值流负责，以及他们如何工作。从有一个想法到得到反馈的时间更短，也更长。他们可以看到正在完成的工作，衡量其产生的影响，并对下一步的创新或投资做出更好的决策。
