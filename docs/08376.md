# 利用微服务推动电子商务转型

> 原文：<https://devops.com/driving-e-commerce-transformation-with-microservices/>

毫不奇怪，新冠肺炎加快了数字化转型，但这些变化发生的速度之快令人难以置信。根据最近的研究，公司已经将其数字或数字产品的份额提高了七年。

这些变化正在影响着全球各个地区的各个行业，但在线销售解决方案的公司感受最深，我想也包括贵公司。为了在数字商务的新常态下蓬勃发展，企业需要一个快速、灵活的技术架构来支持，该架构可以适应新产品和上市战略，并且可以快速启动。要做到这一点，最好也是唯一的方法是使用微服务架构。

## 为什么整体架构对商业特别不利

尽管单片架构的开发、测试和部署非常简单，但其局限性很快就会显现出来，尤其是在数字时代。软件变得太大太复杂，无法快速或正确地做出改变。bug 可能是一场灾难，单一架构通常是采用新技术的障碍，因为任何改变都需要大量的时间和金钱投入来更新整个应用程序。

这些限制在订阅时代更加明显。在今天的市场上，如果你在网上销售产品或服务，你可能会提供订阅。事实上， [95%的公司](https://www.appdirect.com/about/press/releases/new-appdirect-study-69-of-companies-have-accelerated-digital-commerce-initiatives-in-response-to-pandemic-era-challenges)至少提供一种基于订阅的产品。对于订阅商务业务，对微服务架构的需求甚至更为关键。经常性收入策略需要频繁调整并与其他系统轻松集成，这是单一架构通常无法满足的要求。

## 规模是关键因素

[微服务架构](https://devops.com/?s=microservices+architecture)的最大优势之一是快速扩展的能力。整体架构更加僵化，因为一切都必须同时扩展。相比之下，微服务架构允许组织扩展应用程序的单个组件，以满足特定的用例。此外，由于微服务让您可以自由更新单个组件，组织可以在不中断更广泛系统的情况下进行更改。换句话说，有了微服务架构，开发者就能在飞机飞行时对其进行升级。

有内部规模，但通过生态系统的外部规模也是一个重要的考虑因素。最近的研究发现，56%的高管认为[参与生态系统](https://www.ibm.com/thought-leadership/institute-business-value/report/ecosystemecon)是进入新市场的最有效方式，而 55%的高管认为通过合作获得的新能力对于建立新能力至关重要。生态系统的参与——无论是作为运营商还是参与者——都需要及时的集成和开发，这是单一架构无法支持的现实。

## 微服务作为招聘工具

创新和数字化转型不仅仅是业务需求。它们也是吸引最优秀人才的最强有力的招聘工具之一。就业市场对未来的员工来说充满了机会——特别是现在，在大辞职时期，大量的员工都在寻求换工作——企业需要抓住一切优势来吸引顶尖人才。

传统技术可以有它的位置，但它也可能成为求职者的危险信号。大多数开发人员希望加入一个重视创新的组织，并给予他们自由来构建独特的产品和解决有意义的业务问题，这正是微服务开发所提供的。微服务架构还可以让新人才的入职变得更容易，因为开发人员没有义务学习架构的每个方面。这对新员工和现有组织都是一个胜利。

## 通过微服务推动转型

任何特定业务的需求总是在变化，但数字和订阅商务业务已被证明是非常动态的。过去的整体架构根本无法为数字化转型提供动力，也无法在电子商务出现时为其提供支持。微服务可能是一个较小的“包”，但从长远来看，它们将为您的业务带来更大的好处。
