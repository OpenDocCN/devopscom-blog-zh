SmartBear 首席执行官 Frank Roe 表示:“我们很高兴将 Pactflow 和开源软件 Pact 添加到 SmartBear API 开发平台，为开发人员提供关键的新兴能力，以加快 API 开发并提供高质量的服务。“随着组织迅速转向数字化转型，SmartBear 加上 Pactflow 将为我们的客户带来更大的价值，确保我们继续满足处于软件开发和 API 旅程中的组织的需求。”

Pactflow 的联合创始人 Matt Fellows 说:“我们对加入 SmartBear 团队感到无比兴奋。“SmartBear 在其 API 工具和 Pactflow 合同测试中采用的多协议方法有很大的协同作用，特别是在客户推动其微服务架构更加完整的情况下。这一点，再加上我们对开源的共同承诺，为 SmartBear 和 Pactflow 客户的未来奠定了良好的基础。”

SmartBear 首席产品官 Dan Faulkner 表示:“组织正在寻求一个平台，在无缝、多协议的开发人员体验中支持完整的 API 生命周期。“Pactflow 合同测试功能的增加代表了我们交付该平台愿景中的一个重要里程碑，所有这些都侧重于可重用性和通过 CI/CD 管道加快上市时间。”

成千上万的团队使用 SmartBear API 生命周期解决方案来管理高质量的应用程序，并将其更快地推向市场。在 SmartBear API 开发平台中添加 Pactflow 是基于对 SwaggerHub、Swagger OSS 和 ReadyAPI 的不断增长的利用，为开发人员提供了一个通用的 UX 来设计或发现现有的 API、标准化、虚拟化和测试，而不管协议如何。

下一代平台将紧密适应开发人员的工作流程，在提高效率的同时允许更好的标准化和治理。该产品最初将集成公司现有的 API 开发工具包，同时快速扩展功能，以全面支持和增强 API 生命周期中的交付和移交。
