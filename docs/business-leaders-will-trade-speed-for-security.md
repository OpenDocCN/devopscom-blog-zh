# 商业领袖会用速度换取安全

> 原文：<https://devops.com/business-leaders-will-trade-speed-for-security/>

CloudBees 对 600 名 C 级高管进行的全球调查发现，当谈到构建软件时，超过四分之三的受访者表示，安全和合规比快速和合规更重要。

因此，超过四分之三(77%)的受访者还指出，他们的组织正在实施左移战略来实施安全性和合规性，尽管 58%的受访者承认这些努力给开发人员带来了额外的负担。

四分之三的受访者还承认，合规性(76%)和安全性(75%)挑战限制了他们公司的创新能力。尽管如此，83%的人表示，尽管 88%的高管表示他们的软件供应链是安全的或非常安全的，但左移对他们的组织来说很重要。然而，只有 33%的受访者表示他们的软件供应链完全合规。

毫不奇怪，与两年前相比，86%的受访者现在更加关注合规性，而且在一系列引人注目的软件供应链违规事件之后，82%的受访者更加担心攻击。

[CloudBees 首席执行官 Anuj Kapur 表示](https://techstrong.tv/videos/interviews/devops-future-anuj-kapur-sacha-labourey-cloudbees)很明显[更需要通过设置护栏来自动化开发人员的流程，从而减轻开发人员的安全和合规负担](https://techstrong.tv/videos/interviews/evolution-of-devsecops-anuj-kapur-and-tim-johnson-cloudbees)。该调查指出，只有 22%的高管认为他们的软件交付供应链是完全自动化的，而另外 37%的人说它接近自动化。只有 22%的受访者表示他们的合规流程完全自动化，35%的受访者认为几乎完全自动化。

五分之三(59%)的高管表示，他们拥有针对安全性和合规性问题的全部或大部分外部工具，29%的高管表示，他们拥有内部和外部工具的组合。只有 11%的人主要使用内部工具。整整 90%的人说他们的风险管理团队拥有构建和/或维护安全的[软件供应链](https://devops.com/?s=software+supply+chain)的工具、知识和专业技能。

当然，真正的挑战是，虽然法规遵从性问题相对稳定，但新的安全问题却一直在出现。在生产环境中部署应用程序后，开发人员会犯错误或发现新的零日漏洞。Kapur 指出，对于组织来说，拥抱自动化以使开发人员能够解决这些问题作为持续自动化更新周期的一部分是至关重要的。

随着更多合规性和安全性责任的转移，开发运维工作流需要调整的程度自然会因组织而异。然而，开发人员在应用程序开发过程中忽略安全问题的日子已经一去不复返了。各种规模的组织的领导层都明确表示，他们希望减少生产环境中出现的安全和法规遵从性问题。

理想情况下，构建更加安全和兼容的应用程序的努力不会降低应用程序构建和部署的速度。然而，CloudBees 调查清楚地表明，更多的高管愿意将安全性和合规性置于应用程序开发速度之上。问题是如何在两个明显相互竞争的优先事项之间取得平衡。
