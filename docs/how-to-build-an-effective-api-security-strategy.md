# 如何构建有效的 API 安全策略

> 原文：<https://devops.com/how-to-build-an-effective-api-security-strategy/>

应用编程接口(API)使组织能够发展业务和推动创新。API 允许应用程序轻松地将数据传输给外部第三方开发人员、业务合作伙伴和内部部门。由于 API 可以访问有价值的敏感数据，它们越来越成为网络攻击的目标。

在 3 月 3 日举行的 [API 安全峰会](https://www.techstrongevents.com/api-security-summit)上，杰出的网络安全领袖将讨论需要注意的顶级 API 安全漏洞以及如何构建有效的 API 安全策略。

[API 安全峰会](https://www.techstrongevents.com/api-security-summit)的议程将探讨以下主题:

*   API 安全性测试的重要性
*   API 在其环境中的角色
*   黑客是怎么进来的？我们能从他们身上学到什么？
*   API 攻击与传统应用程序攻击有何不同
*   当今领先的首席信息安全官如何保护自己
*   API 是如何增加攻击面的
*   为什么 API 管理不足以保证 API 安全
*   “左倾”策略的好处和局限
*   在您的组织中提高 API 安全流畅性的方法
*   如何在开发和安全团队中推动自动化
