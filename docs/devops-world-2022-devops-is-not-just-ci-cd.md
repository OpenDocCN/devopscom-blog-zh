# DevOps 世界 2022: DevOps 不只是 CI/CD

> 原文：<https://devops.com/devops-world-2022-devops-is-not-just-ci-cd/>

曾经有一段时间，权威人士说 DevOps 是症状，CI/CD 是治疗方法。虽然这可能是真的，也可能不是，但不可否认的是，CI/CD 是 DevOps 的核心。虽然这可能仍然是正确的，但是今天的 DevOps 不仅仅是 CI/CD；它已经发展，远远超出了 CI/CD 的核心。

在我为即将于 9 月 27 日至 29 日在佛罗里达州奥兰多举行的 [DevOps World 2022](https://devopsworld.com/) 做准备时，CloudBees 产品团队向我简要介绍了即将发布的产品。我觉得看到 CloudBees 产品线的走向很有意思。在许多方面，CloudBees 产品的发展反映了 DevOps 本身的发展。

如今，DevOps 包括可观察性、安全性和合规性、GitOps、云原生等领域。开发、集成、测试和部署软件只是当今 DevOps 的一个方面。在不透露 CloudBees 将在 DevOps World 上发布的任何具体产品的情况下，我可以告诉你，CloudBees 正在经历一场与 DevOps 本身非常相似的旅程。

在我看来，这是应该的。CloudBees 是 DevOps 运动的先驱之一。通过利用 Jenkins——最初的 DevOps 杀手级应用——cloud bees 帮助 DevOps 进入主流。当然，《詹金斯》的创作者(KK)KOH suke Kawaguchi 已经离开了 CloudBees，詹金斯社区也一直独立于 CloudBees。然而，我们这个领域的许多人认为 CloudBees 和 Jenkins 有着千丝万缕的联系。

虽然詹金斯仍然被广泛部署和使用，但它不再是 DevOps 世界的中心。在过去的几年里，CloudBees 已经不仅仅是 Jenkins 或 Enterprise Jenkins，它仍然是 DevOps 创新的中心。

今年的 DevOps world 将是 CloudBees 展示其在保持卓越的端到端 DevOps 工具提供商地位的同时，从单纯的 enterprise Jenkins 公司取得了多大进步的绝佳机会。如果你想知道到底有多远，请继续关注我们对 DevOps World 的报道，我们将报道所有的产品发布，并与 CloudBees 高管和客户讨论即将发布的新产品、产品线和功能。

我以前写过今年 DevOps World 的主题是 [DevOps Remix](https://devops.com/devops-world-2022-why-devops-needs-a-remix/) 。我们自己的 DevOps 体验虚拟活动将于 11 月举行，主题是 DevOps 无处不在。我认为这两个主题说明了同一个总的想法:DevOps 正在走向它从未去过的地方。它已经超越了 CI/CD，发展成为一种开发、交付和管理软件的新方式，包括保护和测试软件。在接下来的几年里，我想我们会看到我们正在进入一个新的 DevOps 黄金时代。
