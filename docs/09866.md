# 更多的选择，更多的问题:多云的双刃剑

> 原文:[https://devo PS . com/more-choices-more-problems-the-double-edge-sword-of-multi-cloud/](https://devops.com/more-choices-more-problems-the-double-edged-sword-of-multi-cloud/)

多云环境可能是一把双刃剑。一方面，无论供应商是谁，理论上，使用最适合您特定需求的云或工具都有明显的好处。然而，实际上，您的选择越多，您的环境就变得越复杂和昂贵。事实上， [Andreessen Horowitz](https://a16z.com/2021/05/27/cost-of-cloud-paradox-market-cap-cloud-lifecycle-scale-growth-repatriation-optimization/) 认为，与企业使用本地系统相比，多云架构通常会增加企业的成本。

企业需要一个云计算环境，以便在如此快速发展的数字经济中竞争。对于大多数公司来说，完全回到本地并不是一个选择，因为火车已经离开了那个车站。但是，当保持竞争力的成本变得越来越高，讽刺的是让你竞争力下降时，你该怎么办？

### 混合云变成了多重云

当混合云(公共云服务和本地系统的混合)成为企业的筹码时，企业只需要一个公共云服务提供商。大多数都把所有鸡蛋放在 AWS 篮子里，在 AWS 公共云服务和内部系统之间分配工作负载。

这种设置为企业提供了更多的敏捷性和灵活性，并通过减少对昂贵、笨重的本地硬件的依赖降低了成本。运营支出取代了大量资本支出；有了云服务，您只需按使用量付费，无需支付物理基础设施和维护成本。

不可避免的是，云的成本和效率优势促使公司从其内部系统中迁移越来越多的工作负载，随着时间的推移，云成本不断增加。单一供应商的工作负载越多，你就越依赖该供应商。如果 AWS 提高价格，你别无选择，只能支付更多。

很快，混合云发展成为多云，其他公共云提供商如 Azure 和 GPC 也加入其中。有时这是出于选择——当您可以从多家供应商中选择价格更优惠或更适合您需求的产品时，为什么要局限于一家供应商和一套工具呢？

然而，更多情况下，这是不自觉的——开发人员出于自己的目的，在没有咨询 it 部门的情况下，构建了自己的云和资源，从而产生了影子 IT。或者公司收购其他组织，这些组织的云选择各不相同。无论哪种方式，企业都在使用多种云，无论他们是否计划这样做。

不管它是如何发生的，一旦引入更多的云，复杂性就会成倍增加。

### 多云九头蛇

太多的组织低估了每增加一个云所带来的复杂性。每个供应商的操作系统、网络、成本监控和跟踪、工作负载方法和数据汇总都各不相同。你的组织中有多少人知道所有这些差异是什么，以及如何在它们之间导航？当每个公共云的做法不同时，您如何有效地管理、治理和监控使用、成本以及安全性和合规性问题？您必须手动从每个云中收集数据，并将其汇总到电子表格中进行分析。这有多准确，需要多长时间，在人员和时间方面的成本是多少？

此外，您如何能够准确指出浪费和低效？如果您正在追踪工程师以关闭资源，或者必须在一个手动成本效率行动与另一个之间进行选择，浪费将会增加，您的预算将会不准确。

好像这还不够混乱和复杂，分散在企业中的开发团队采用不同的工具，并为他们自己的目的使用它们，没有任何对齐或协调。同一工具被多次购买。团队之间不共享最佳实践。所有的东西都需要定制编码，这就产生了一层层的“意大利面条式代码”,模糊了可见性，容易出现错误配置，从而导致安全漏洞。别忘了影子 IT，IT 部门对此一无所知，更不用说监控了。

您以统一的云战略开始了您的旅程，但现在您正面临着针对不同群体的越来越多的不同云和战略。每个小组都喜欢不同的工具、云、方法和流程，这让您陷入了孤立的团队、无休止的工具蔓延、无限的自定义代码行、不断恶化的安全风险和不清晰的可见性。计价器一直转，让你损失数百万。

那么，你现在做什么？

### 驯服混乱

为了在新的云秩序中生存和发展，企业不能仅仅将爆炸式增长的云成本视为“创新成本”，或者以其他方式证明他们的云混乱是理所当然的。现在有太多的人在这么做，而且不会有好结果，因为我们可能会陷入衰退，市场的不确定性和动荡会越来越大。

首先也是最重要的，驯服云混乱和控制多云成本需要可见性和互操作性。您必须能够从一个位置查看、监控、管理和分析所有云；用电子表格手动收集这些数据已经行不通了。

开发团队不能再彼此孤立。应该有一个单一的存储库，所有的团队都可以访问和利用所有的工具、脚本和先前编码的集成。当所有的东西都可以在一个地方查看和使用时，没有人需要购买同样的工具两次或重新发明轮子。一旦集成了一个工具，任何其他想要使用该工具的团队都可以简单地重用已经创建和审查的集成、配置和策略。如果多个小组在做同一件事，可以创建和共享一个批准的自动化流程。

真正的成本优化还需要组织意识和共同责任。如果人们不知道他们在浪费资源，他们怎么能停止呢？企业需要一个系统来检测浪费和成本超支，并相应地通知利益相关者。

例如，一个开发环境可能已经启动并运行，即使它不再被使用。可以向适当的团队领导发送自动警报，提醒他们关闭它。提供云资源的开发人员可能不会选择最具成本效益的选项，但警报系统会标记他们的活动，并通知他们更便宜的替代方案。夜间或周末未关闭的云资源也会被检测到并关闭。承诺将每天自动进行，不再需要个人记得手动完成。

再也不能回到传统的打地鼠式的云管理和成本控制了。了解这一点并创建基于简单性、可见性、统一性和意识的云管理功能整体框架的企业，将在未来的经济波涛中处于更有利的位置。
