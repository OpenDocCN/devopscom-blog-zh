# 红帽支持 RHEL 8 监控

> 原文：<https://devops.com/red-hat-bolsters-rhel-8-monitoring/>

Red Hat 希望通过扩展 Red Hat Enterprise Linux (RHEL)中包含的监控功能来增加采用最佳 DevOps 实践的组织数量。

本周 RHEL 8.2 的[测试版包括对性能副驾驶(PCP)的更新，这是一个系统性能工具包，现在包括微软 SQL Server 2019 数据库的收集代理。](https://www.redhat.com/en/blog/red-hat-enterprise-linux-82-beta-now-available)

Red Hat 的主要产品经理 Scott McBrien 表示，通过扩展 performance toolkit 的范围，Red Hat 正在寻求使 it 通才和开发人员更容易跨孤岛协作，以更好地保持应用程序的性能水平。对 Microsoft SQL Server 2019 的支持是 2015 年与微软结成的联盟 Red Hat 的产物。然而，McBrien 说，IT 组织应该期待 Red Hat 继续扩大 PCP 的覆盖范围。

RHEL 8.2 测试版还简化了在安装过程中调用 Red Hat Insights analytics 服务的过程，并提供了与扩展的 Berkeley Packet Filter (eBPF)的更紧密集成，以更好地分析网络流量。

最后，最新版本的 RHEL 扩展了应用流功能，将运行时和语言隔离到其他编程环境，并提供了一个工具来测试从 RHEL 6/7 到 RHEL 8 的就地升级。

McBrien 说，Red Hat 正试图保持 Red 个月的升级周期，以便为 IT 组织提供一个更可预测的时间表。当然，大多数 Red Hat 客户仍然在运行 RHEL 6/7 版本。然而，随着开发运维时代应用程序的构建、部署和更新速度不断加快，IT 团队在不断扩展的企业中保持应用程序性能水平的压力也越来越大。不可避免地，IT 通才将需要在堆栈的所有层访问更广泛的可观察性工具，以跟上步伐。

整个企业对 DevOps 实践的采用仍然参差不齐。然而，随着越来越多的 it 通才接触到更丰富的操作系统级监控工具，扩充或取代基于 ITIL 的传统 IT 管理方法就变得越来越容易。事实上，随着供 IT 通才使用的工具变得越来越复杂，对各类 IT 专家的依赖程度也在下降。大多数 IT 组织特别渴望实现这一目标，因为劳动力仍然是最大的 IT 成本。

在未来的几个月和几年中，It 管理的转变程度还有待观察。可以肯定的是，随着基于微服务和无服务器计算框架的新架构的采用，它将变得更加复杂。每个额外的抽象层可能会使开发人员更容易构建应用程序，但是从 it 运营团队的角度来看，它们往往会增加已经相当脆弱的系统之间的依赖性。现在的挑战是在应用程序性能下降成为全面的灾难性事件之前，找出所有这些系统之间存在瓶颈的地方。

— [迈克·维扎德](https://devops.com/author/mike-vizard/)
