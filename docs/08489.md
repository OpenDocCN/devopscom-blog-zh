# Tufin 与 Puppet Labs 合作，为基于主机的防火墙自动化和协调安全策略

## 联合客户现在可以通过 iptables 防火墙的集中自动化管理来提高 IT 灵活性和安全性

By [Tufin Technologies](https://www.sacbee.com/search_results/?sf_pubsys_story_byline=Tufin%20Technologies&link_location=top "Read more articles by Tufin Technologies")Published: Tuesday, Feb. 25, 2014 – 5:08 am

新泽西州莫里斯敦，2014 年 2 月 25 日讯—[安全策略协调](http://www.tufin.com/products-solutions/products/products-securetrack-securechange-overview/)解决方案的市场领先提供商 Tufin Technologies 今天宣布了一项集成，该集成使 Tufin 客户能够利用 [Puppet Enterprise](http://puppetlabs.com/puppet/puppet-enterprise) 、 [Puppet Labs 的](http://puppetlabs.com/)旗舰商业产品的 IT 自动化功能来配置和提供 iptables 的安全策略更改，iptables 是一种常见于物理和虚拟 Linux 服务器上的基于主机的防火墙。自动执行和协调 iptables 策略更改的能力改善了服务器、应用和安全运营团队之间的协作和信息共享，并确保 iptables 保护的服务器的网络连接更改安全、准确地实施，并与业务同步。同时，Tufin 还宣布了[新的安全、自动化和协调功能](http://www.tufin.com/about-us/news-and-media/press-releases/2014/tufin-unveils-unified-security-policy/)以及最近一项调查的[结果](http://www.tufin.com/about-us/news-and-media/press-releases/2014/current-state-of-security-operations-survey/)，该调查强调了在当今高度复杂、动态和分布式的网络环境中，自动化对于有效管理变化的需求。

Tufin 首席技术官 [Reuven Harrison 表示:“这种集成使我们的客户能够利用 Puppet Enterprise 的自动化能力，在内部和以前不受控制的环境(如云部署和虚拟化环境)中实现应用连接自动化。“这对我们的客户非常有用，并确保安全性融入到服务器的连接要求中。随着安全和网络运营的不断融合，该模块使我们的客户能够通过自动化来管理变化，从而带来切实的商业价值。”](http://www.tufin.com/about-us/tufin-management-team/)

[Tufin Orchestration Suite](http://www.tufin.com/products-solutions/solutions/tufin-security-policy-management-solutions-overview/) 使组织能够提高自动化程度，加快网络基础设施变革的步伐，同时保持安全性和合规性。Puppet Enterprise 为系统管理员提供了他们主动管理动态基础架构所需的操作灵活性、效率和洞察力，可以从数十台服务器扩展到数千台内部和云中的服务器。

Puppet Labs 的首席信息官 Nigel Kersten 表示:[这种集成使共同客户能够对 iptables 安全策略进行更改。“我们希望安全策略协调成为我们客户的核心要求，与 Tufin 的集成使我们的客户能够快速有效地满足整个企业的连接要求，而不会影响安全性。”](http://puppetlabs.com/about/management)
