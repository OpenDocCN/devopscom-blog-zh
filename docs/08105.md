# 学习 DevOps 的“第三条道路”——持续改进

> 原文:[https://devo PS . com/learning-the-third-the-way-of-devo PS-continuous-improvement/](https://devops.com/learning-the-third-way-of-devops-continuous-improvement/)

我指的是 DevOps 的第三种方式——持续实验和学习，简单来说就是持续改进。从某种意义上说，没有办法学习 DevOps 的第三种方式，因为根据定义，它是一个移动的目标。讨论为 DevOps 持续改进创建一个持续学习组织需要什么确实是有意义的，它建立在对 DevOps 的第一种方法和第二种方法的掌握之上。

正如我的博客[学习开发运维的“第二种方式”——持续反馈](https://devops.com/learning-the-second-way-of-devops-continuous-feedback/)和我之前的博客[学习开发运维的“第一种方式”——持续流](https://devops.com/learning-the-first-way-of-devops-continuous-flow/)中所指出的，很明显许多组织都在努力实现精心设计的开发运维，甚至是开发运维的第一种方式——持续流。

DevOps 的第一种方式，持续流，是 DevOps 的基础。一旦掌握了，一个组织自然会希望掌握 DevOps 的第二种方法，以实现持续反馈提供的更高水平的好处，如果设计得好的话。一旦实现了开发运维的第二种方式，组织自然会希望掌握开发运维的第三种方式，以维持和发展他们的开发运维能力。

DevOps 是一个不断发展的知识体系。没有涵盖开发运维所需的所有角色和全部技能的一站式商店。具备开发运维能力的员工队伍需要持续致力于开发运维技能的培养和掌握。企业必须战略性地开展 DevOps 培训，以最大限度地利用培训资源。

### 为什么学习持续改进对工程开发很重要？

DevOps 的第三种方式——持续改进——要求组织致力于持续学习和掌握最高水平。所需的典型技能包括微服务设计和运营的高级实践的实施和操作、高级连续测试、多管道、[价值流管理](https://www.vsmconsortium.org/)、SRE、可观察性、DevSecOps 和 AIOps。留住和培养高技能且难以找到的有经验的 DevOps 专业人员和 sre 使得在内部培养技能变得非常重要。

### **学习持续改进是如何完成的？**

以下是学习持续改进的推荐工程实践:

*   持续学习文化鼓励个人不断获取知识，并为 DevOps 和 SRE 交流经验和想法。
*   指导是一个明确的计划。
*   回顾的目的是彻底分析重大的、意外的事件，并确定解决方案，防止将来发生类似事件。
*   维护着一个动态的 DevOps 和 SRE 培训资源的在线图书馆，该图书馆跟踪 DevOps 和 SRE 的最新和最具创新性的方法。
*   DevOps 和 SRE 团队通过与行业思想领袖合作，不断更新培训内容，发展他们的技能，推进他们的职业发展。
*   与值得信赖的教育和培训合作伙伴合作，确保向 DevOps 和 SRE 团队提供最高质量的学习内容和工具。
*   DevOps 和 SRE 团队的一部分时间用于培训和教育。
*   跨团队分享推荐的工程实践是使用横田的概念主动进行的——在团队之间主动分享实践。

### **实施学习持续改进需要什么？**

以下是实施持续改进学习计划的推荐工程实践。

*   为专家分配角色和职责，以定义 DevOps 持续反馈的培训计划(例如，DevOps sensei、培训 PM、传道者)。
*   进行持续反馈 DevOps 技能差距评估。
*   定义一个培训策略，最好地解决差距，并符合组织的需求和优先事项。
*   定义培训预算，包括每个角色和技能的费用和时间分配。
*   确定哪些技能和角色可以被廉价的自学资源所涵盖。
*   确定哪些技能和角色需要更昂贵的商业培训。
*   确定每个角色的培训目标。
*   确保招聘职位符合与组织战略目标最相关的开发运维技能的技能要求。
*   为 DevOps 和 SRE 创造一种持续学习的文化，鼓励个人不断获取知识，交流经验和想法。
*   建立一个动态的 DevOps 和 SRE 培训资源的在线图书馆，跟踪 DevOps 和 SRE 的最新和最具创新性的方法。
*   与行业思想领袖合作，不断更新 DevOps 和 SRE 团队的内容，以发展他们的技能，促进他们的职业发展。
*   与值得信赖的教育和培训合作伙伴合作，确保向开发运维团队和 SRE 团队提供最高质量的学习内容和工具。
*   到位意味着使培训资源可见和可用。
*   为团队和个人设定培训目标。
*   为培训制定激励和认可计划。
*   跟踪培训进度。
