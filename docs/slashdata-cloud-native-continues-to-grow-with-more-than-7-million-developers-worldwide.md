# SlashData: Cloud Native 继续发展，全球有超过 700 万开发人员

> 原文：<https://devops.com/slashdata-cloud-native-continues-to-grow-with-more-than-7-million-developers-worldwide/>

**根据 SlashData 为 CNCF 开发的 [Q3 2021 年云原生发展状况报告](https://www.cncf.io/wp-content/uploads/2022/05/Q3-2021-State-of-Cloud-Native-development_FINAL.pdf) 显示，全球云原生开发者人口在过去 12 个月** 增长了 100 万。公司估计现在全球有**710 万云原生开发者** 。这比我们在 2020 年 5 月使用 SlashData 首次报告 时的 [增长了 51%，当时有 470 万名云原生开发者。](https://www.cncf.io/blog/2020/05/04/the-state-of-cloud-native-development-a-new-survey-report/)

云原生开发人员包括 480 万使用容器编排工具的开发人员和 420 万使用无服务器平台的开发人员。这两个数字分别是从去年年底[](https://www.cncf.io/blog/2021/12/20/new-slashdata-report-5-6-million-developers-use-kubernetes-an-increase-of-67-over-one-year/)以来的 460 万和 400 万。

**Kubernetes 保持稳定，但出现了新趋势**

与此同时，Kubernetes 的使用量保持相对稳定，比去年增加了 2 个百分点。 大约 61%的后端开发人员在过去的 12 个月中使用过容器，其中 30%使用 Kubernetes 来编排它们。据估计，Kubernetes 目前在全球有 510 万用户。

正如我们在 CNCF 的 [2021 云原生调查](https://www.cncf.io/announcements/2022/02/10/cncf-sees-record-kubernetes-and-container-adoption-in-2021-cloud-native-survey/) 中看到的，有几个趋势推动着 Kubernetes 的采用。首先，大型组织更有可能使用 Kubernetes。SlashData 发现，为超大型企业(员工超过 5000 人)工作的近四分之一的开发人员使用 Kubernetes。

我们的云原生调查的另一个趋势是，随着 Kubernetes 走向“幕后”,托管服务正在兴起据 SlashData 称，大多数容器编排工具(包括最常用的六大工具中的五个)都是托管的 Kubernetes 解决方案。此外，选择单一编排工具的开发人员比例有所下降:在 Q1 2021 中，49%使用编排工具的开发人员仅使用一种工具。六个月后，这一比例降至 46%，许多人表示他们将自我管理的解决方案与供应商管理的服务结合使用。

最后， Kubernetes 开发者也更有可能涉足区块链、边缘计算和 5G 等新兴技术。

**无服务器整合即将到来吗？**

在看到上一份报告中的轻微下降后，无服务器工具的使用增加了 5%,从 400 万增加到 420 万。然而，开发人员使用无服务器产品的方式正在发生变化，开发人员同时使用的无服务器工具越来越少。在 Q1 2021 中，23%使用无服务器的开发人员使用了三个或更多的工具。六个月后，这一比例降至 19%。
