# Instana 通过 SmartAlerts 再次改变了性能监控，smart alerts 是第一个包含系统环境的策划警报系统

> 原文：<https://devops.com/instana-changes-performance-monitoring-again-with-smartalerts-the-first-curated-alerting-system-that-includes-systematic-context/>

***公司将自动应用程序性能监控扩展到复杂的警报系统***

**2020 年 2 月 26 日——伊利诺伊州芝加哥和德国索林根—**Instana 正在通过 Instana SmartAlerts 改变开发人员和 IT 运营人员设置、管理和执行警报的方式，Instana smart Alerts 是首个基于环境和情景使用案例自动管理 IT 警报的系统。Instana SmartAlerts 包含在该公司面向微服务应用的自动化应用性能管理解决方案中，使用称为蓝图的精选构建模块来自动配置警报阈值和操作。

Instana 的技术总监兼 APM 策略师 Chris Farrell 表示:“与容器化和微服务应用相关的复杂性不仅会产生可观察性问题，还会影响有效管理警报的能力。“Instana 的 SmartAlerts 提供了预构建的配置模块，称为蓝图，可根据技术、流程和运营情况将上下文注入警报并制定关键的警报规则。”

SmartAlerts 蓝图包含预构建的指标选择和阈值信息，以及对构成可警报情形的因素的精选分析。SmartAlerts 作为 Instana 的 APM 解决方案的一部分提供，它带有现成的蓝图，适用于典型的 IT 和 DevOps 情况，如识别缓慢的服务或网站。用户唯一需要配置的是应用这些检查的范围，比如 URL 或服务名。

Conrad 的 DevOps 工程师 Janek Wonner 表示:“为使用电子商务等关键应用的客户提供最佳体验非常重要。“Instana 的 SmartAlerts 将应用程序监控的自动化和易用性带入了我们的警报系统，帮助我们高效地提供更高的应用程序性能和客户满意度。”

与其他 APM 解决方案不同，Instana 完全自动化应用程序监控的整个生命周期，包括应用程序发现和映射、监控传感器和代理部署以及应用程序基础架构运行状况监控。每当动态应用程序中发生应用程序或基础架构变化时，Instana 都会实时识别变化，即时调整其应用程序服务地图、监控阈值和运行状况控制面板。

SmartAlerts 系统建立在 Instana 的自动化、易用性和最少人工配置的原则之上。开箱即用的蓝图为最理想的警报使用情形(如网站问题)提供完全指定的警报系统，只需最少的配置。用户可以使用无界分析过滤引擎和新的 SmartAlerts 向导(包括任何现有的 SmartAlerts 蓝图)直接从控制面板创建新的警报。

Instana SmartAlerts 附带了该公司的 APM 解决方案，现已可供使用。关键蓝图已可用于即时零接触警报。在 https://instana.com/alerting 了解更多信息。
