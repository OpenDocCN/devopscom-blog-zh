# DevOps 死了吗？我不这么认为！

> 原文:[https://devops.com/is-devops-dead-i-dont-think-so/](https://devops.com/is-devops-dead-i-dont-think-so/)

我在科技界 30 年的经验告诉我，技术和框架永远不会真正消亡——它们会发展壮大。当我读到最近的一篇文章宣称，“DevOps 死了！”而且我们都应该拥抱平台工程，我笑了。

不，DevOps 没有死，关于它灭亡的传言被大大夸大了。事实上，DevOps 比以往任何时候都更有活力，事实是，软件走到哪里，它就跟到哪里。当我回去再次阅读这篇文章时，我意识到它出现在一家风险投资(VC)公司拥有的网站上，并且这篇文章本身是一家推广平台工程的公司的赞助帖子。这是一个很好的提醒，要对那篇文章有所保留。我还意识到，如果不是彻头彻尾的点击诱饵，这个标题也是有争议的。

也就是说，我觉得这篇文章需要反驳。为那篇文章的作者说句公道话，我认为他们试图表达的观点是 DevOps 存在一个主要问题——我们给开发人员带来了太多的负担，再多一根稻草就可能压断骆驼的背。当然，这是有道理的。我们需要平衡我们要求开发人员和团队其他成员之间的关系。我们在 DevSecOps 中看到了这种安全性；我们从测试中看到了这一点，当然也从运营中看到了这一点。事实上，在许多方面，SRE 的崛起更好地定义了老年退休金计划的作用，从而减轻了开发商的负担。

我毫不怀疑平台工程也将减轻开发者的负担，更好地定义平台工程师的角色。然而，SRE 没有杀死 DevOps，平台工程也不会。曾经，谈论一个全栈开发者是非常时髦的。今天，许多人认为所谓的全栈开发人员是一个神话，这个角色并不是大多数组织真正需要的。然而，开发人员的工作从来都不容易，也永远不会容易，即使出现了 AI/ML 生成的代码。在任何给定的时间，开发者总是有许多盘子在空中旋转。就这一点而言，他们获得的高额报酬并非没有根据，也不是不劳而获。

我期待着看到平台工程师角色和平台工程作为一种实践如何发展，以及他们如何与 SREs 和 DevOps 团队以及开发人员、安全测试人员等互动。，但我相当确定，无论角色如何演变，都不会是 DevOps 的丧钟。

正如我们为即将于 2022 年 11 月 16 日举行的 [DevOps 体验](https://www.techstrongevents.com/devopsexperience22)活动所宣布的，它将前往 DevOps 从未去过的地方。我们看到事物的发展；我们看到 DevOps 在边缘。我们也看到分布式开发操作系统。DevOps 已经扎根于我们的软件开发生命周期以及更长的时间。虽然框架可能会改变和发展，但是 DevOps 也会随之改变和发展。

如果你说 DevOps 死了，我不同意。我说德沃普斯万岁！
