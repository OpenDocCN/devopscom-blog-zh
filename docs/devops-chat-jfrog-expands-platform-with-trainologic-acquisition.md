# DevOps Chat: JFrog 通过列车采集扩展平台

> 原文：<https://devops.com/devops-chat-jfrog-expands-platform-with-trainologic-acquisition/>

看到 DevOps 领域的公司成长并扩展其服务范围总是很棒。JFrog 是符合这一描述的最新产品。该公司上周宣布收购开发和 DevOps 技术咨询公司 Trainologic，通过帮助新客户开始他们的 DevOps 之旅并在此过程中支持现有客户来扩大其覆盖范围。

在本次 DevOps 聊天中，我们与 JFrog 首席执行官兼联合创始人什洛米·本哈姆和 Trainologic 创始人兼首席执行官加尔·马尔达就收购以及 Trainologic 如何融入 JFrog 端到端持续更新的愿景进行了交谈。

**Shimel:** 谢谢。加入 Shlomi 的是 JFrog 家族的新成员 Gal Marda，我猜他是——或者说是 Trainologic 的创始人兼首席执行官，对吗？

**Gal Marda:** 正确。很高兴来到这里。谢谢艾伦。

欢迎，欢迎来到 JFrog 家族。

非常感谢。对我来说很刺激。

**Shimel:** 所以，我会让——Gal，我会让你告诉我们的观众今天的重大新闻。这里发生了什么？

Marda: 正如 Alan 所说，接下来发生的事情非常令人兴奋。Shlomi 有一个伟大的想法，JFrog 推出了这个平台，EPlus 平台，在我看来，这是 DevOps 行业的一场彻底的革命，他认为 DevOps 和 liquid software 的愿景，要使它成功，它既是工具也是实践。推出这个平台并确保其成功还需要一些咨询服务，以确保 JFrog 客户在使用企业平台的方式上做出正确的决定。

因此，我们的想法是联合起来，让我们确保客户通过使用 Enterprise Plus 平台获得成功。

**希梅尔:**优秀。那么，Shlomi，让我问你—JFrog 作为一家公司已经取得了巨大的成功，巨大的增长。去年，你们真的转向了市场上的平台型解决方案，对，提供平台。为什么是训练学，为什么是现在？

本哈姆:这是个很好的问题，艾伦。因此，正如您所知，我们过去也谈到过，DevOps 要解决的真正难题不是持续集成、持续部署、持续交付和持续测试的里程碑。这些都是创造和 ________ 革命的非常重要的里程碑。

但是，基本上，如果我们都有一个解决方案来帮助我们实现持续更新，软件快速安全地发布，那么 DevOps 的发展将处于采购级别。我们今年发布的 Enterprise Plus 是一个由七种不同产品组成的捆绑包，它不再是一个可以在服务器端安装和运行，然后为开发人员服务的产品。不仅仅如此。这是一个端到端的二元解决方案。

我们发现，我们需要一些专家来帮助我们在用户中采用和传授知识。现在，我们有 4，500 家企业客户。我们需要 Gal 的团队在现场与我们合作，在客户网站上支持这一目标。

我完全同意，什洛米。你知道，我想起了我在 Interliant 的时候，我们做了早期的云托管，比如 Lotus Notes、PeopleSoft 和 Oracle applications。然后我们发现，你永远不能把这样一个复杂的应用程序从盒子里拿出来，插上电源，然后说它会工作，对吗？总是有，充其量，75%，它的工作。你总是需要 25%的专业服务，来定制它，微调它，让它工作。

实际上，即使在安全领域，许多最成功的安全公司，当你看他们的收入明细时，也是专业服务，因为安全很难。人们就是做不到，所以，对我来说，无论如何，这是一个 DevOps 问题也不足为奇，因为你仍然需要那些专家来做这件事。

不过，我想问你的是，作为你这个位置上的首席执行官，你总是需要平衡我想要多少人，对吗？我们如何衡量这一点？因为这是一个身体密集，这是一个人密集的商业模式。

所以，你知道——而且，你可能有经验，在 Trainologic 领导过。你在那里做什么？你是如何控制它的？

玛尔达:那么，我——什洛米，你拿这个，还是我？

你们中的任何一个。

**本哈姆:**好的，我会谈谈这个比率，而 Gal，你会谈谈从你的经验中得出的最佳实践。我认为，艾伦，你是对的。当你来的时候，不仅仅是技术，不仅仅是市场上正在发生的新事物。DevOps 市场正在蓬勃发展。你可以从任何地方看到它，从筹款到技术采用，新事物，创新，每天早上我们醒来都会听到一些 DevOps 新闻。所以，对于像 JFrog 这样的公司来说，这是非常激动人心的一天。

但是你也看到很多公司开始让大型组织、成千上万的开发人员使用新的方法和新的工具。因此，我们必须在我们向市场发布的内容和它被采用的方式之间保持平衡。因为否则，我们会用没人知道如何使用的技术来淹没我们的用户，他们会——你知道，第一反应将是反射性的反应，只是放弃它，并有这样的咒语，即工作应该是什么，保持原样。如果我们想保持创新，如果我们——抱歉，是推动创新——如果我们想保持市场势头，那么我们也必须通过入职支持市场。这是一件事。

第二件事——JFrog 是一家产品公司。JFrog 不是一家服务公司。但是，如果不帮助我们的客户更好地使用产品，那么开发产品是没有意义的。这就像有一项伟大的技术，但执行不当。因此，我们正在寻找两年前或三年前在我们的领域中做这种事情的公司，以及市场如何采用他们的平台。我们从中学到了很多，我们做了分析。因此，当我们联系培训师 Logic 并邀请 Gal 加入到沼泽团队时，我们首先关注的是，这家公司是否已经安装了 ________ 工厂、x 射线、Bitray 和 JFrog 池？第二，是不是足够大的团队？这是一个 TS 超级指挥单位，所以不太大，如果 Gal 和团队能够建立一个社区，不仅与 JFrog 专业服务，还与我们周围的专业服务公司社区，并在世界各地的这些单位内进行某种业务开发，以帮助我们传播 JFrog 技术。

**希梅尔:**优秀。加尔，你还想补充什么吗？

玛尔达:是的，当然。所以，你问了关于缩放，所以我认为 Shlomi 是正确的。我们将成为一支相对精干和吝啬的指挥力量。我们不会像成千上万的人一样，但是我们可以通过多种方式来扩展。其中之一是，正如 Shlomi 提到的，通过伙伴关系，好吗？我们倾向于创建一个合作伙伴网络和一个有助于我们合作的社区。我们不会独自解决这个问题。

而且，从内部扩大规模、吸纳人才并对他们进行培训来说，这是我们非常习惯于做的事情。我们要确保年轻人是影子，我们要培养他们，我们要教给他们更多我们的实践，我们要确保他们看到许多不同的建筑和许多有不同思维方式的不同客户。通过这种活动，他们在相对较短的时间内变得非常有经验，因为他们看到了很多东西，如果你让有才华的人，你给他们正确的指导，你让他们看到许多不同的解决方案和方法，他们在相对较短的时间内变得非常高效。

**希梅尔:**优秀。那么，这是不是，突击队要进去什么的，Gal，你想象他们吗——那么，是不是每个工作都是一次性的，还是有一种标准化的包装，你会为大多数客户开发？

**Marda:** 嗯，它仍在制作中，但我——我们正计划制作一些产品包，因为在我们去年的会议中，许多不同的公司为他们提供不同的服务，我们意识到有同样的需求一次又一次地回归。

例如，其中之一是某种 DevOps 分析，如对你应该在哪里或你想在哪里的差距分析，以及你当前的 DevOps 状态是什么？因此，肯定会有不同类型的包，在这些包内，当然，许多工作将是量身定制的，因为每个公司都是不同的，正如你所知，DevOps 也是非常文化相关的，并与公司的许多其他事情相关。因此，可以肯定的是，它不可能是完全相同的东西，但软件包中的一般框架将是，不同的软件包将提供现成的。

**希梅尔:**优秀。所以，什洛米，高尔说了一个神奇的词——文化。这是怎么回事——你已经完成了其他收购。你已经收购了 JFrog China，我在中国的一次活动中看到了他们。这些年来你做的其他一些收购。但这些都是与产品相关的公司。这显然是一个服务。我的意思是，你在 JFrog 有很好的文化，对吗？我亲眼目睹了。令人钦佩。你如何制造—

本哈姆:谢谢。

**Shimel:**–_ _ _ _ _ _ _ _ 沼泽中的蛙人？

本哈姆:首先，谢谢你，艾伦。你知道，我们收购了四家不同的公司，从为 DevOps 提供 C++包管理器的柯南，到后来成为 JFrog Insight 产品的 CloudMunch。然后是 DMon，现在是 crenologically，________ 的一个原则是，你应该始终寻求战略和文化之间的结合。

我第一次见到 Gal，是在我们的用户大会上，在收购的想法下。所以，对我来说，没有比这更好的地方了，带一个人给他看 JFrog，不仅仅是从技术的角度和产品的角度，还有社区关系，还有青蛙的氛围和青蛙的 DNA。

然后，当我们开始讨论时，实际上，在这笔交易上努力工作的两个团队是人力资源团队和运营团队。我们与 Gal 分享的第一件事是 JFrog 产品。所以，我认为 Gal 不会加入 JFrog，如果他觉得从文化的角度来看这不是正确的选择，而向前发展的战略可能会改变企业文化的世界。

所以，灵动软件的愿景，加上 JFrog 的文化，我认为已经融入了 Gal 的信念和他的团队在过去 10 年里所做的事情。我们在那里看到的是一张相同的 JFrog 照片。这几乎就像一只手套对一只手，现在，在我们签约并成为 JFrog 的一部分后，我可以说这是一个从我们相遇的第一刻起的爱情故事。

好吧，这是一个快乐的结局。*【笑声】*所以，我不得不问你，什洛米——那么，现在，你已经收购了一家服务公司，突击队，下一步是什么？JFrog 的下一步是什么？

本哈姆:这是个完美的问题。实际上，你现在听起来像我的 ________。*【笑声】*所以，在服务层面上，我认为 Gal 和我们的全球销售副总裁程昕婷现在正在做的是如何将这些功能与企业级，即 JFrog 级融合起来。这是一件事。

JFrog 的第二件事是我们技术的延伸。我们看到 DevOps 市场正在进行整合。有两个方面整合了所有的技术。其中之一是代码库，所以你可以看到 GitHub 和 Bitbucket 等等发生了什么。另一个是藏物库。因此，这两个领域在某种程度上是将所有技术整合到 it 中，我们正在关注 it 和前景，通过我们提供的通用解决方案和混合解决方案，以及作为开发人员优先和开发运维优先公司的自由选择理念，我们将扩展技术。

所以，敬请关注。我向你们保证，在 2019 年初，我们将会有更多关于支持我们目标的技术的消息。

**希梅尔:**妙极了。盖尔，我要给你最后一句话。

马尔达:好的。首先谢谢你。我必须认同什洛米所说的一见钟情和文化。令人惊讶的是。当 Shlomi 打电话给我时，首先，我感到非常荣幸和惊讶。但我很快意识到，我们看到的是同样的事情，但我们试图以不同的方式帮助我们的客户。JFrog 正在创造伟大的技术，而我们正在尽我们所能为他们提供最好的服务。

但我们试图达到完全相同的目标，以完全相同的方式——开发者第一，技术第一，虽然我们欣赏，我们都是人，看到人们，并试图保持一种非常特殊的文化。所以，我认为这是一个非常合适的问题，也是最重要的问题。所以，我非常感谢 Shlomi 给我这个机会成为 JFrog 的一部分，我觉得我们从一开始就非常相似，但加入像 JFrog 这样的公司是我的荣幸。

Shimel: 嗯，恭喜你。
