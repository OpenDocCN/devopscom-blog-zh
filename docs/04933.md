*这是系列文章中的第二篇，重点介绍在 SauceCon 2020**举办的* [*演讲嘉宾。SauceCon 将全球的 Sauce Labs 用户和持续测试专家聚集在一起，互相学习，提升他们的自动化测试和持续交付技能。今年的活动将于 4 月 27 日至 29 日在得克萨斯州奥斯汀举行，DevOps.com 将在现场直播。T13】*](https://saucecon.com/)

SauceCon 2020 演讲人 Hilary Weaver-Robb(在网上被称为 [g33klady](https://twitter.com/g33klady) )希望你知道测试 API 有多重要——单独测试，以及与使用它们的 ui 一起测试。

![](../Images/c1f37c669ae413fd31de445b57116aee.png)T2】

原因如下:如果你只关注 UI 测试，你可能会错过一些重要的东西。毕竟，API 是暴露应用程序所有业务逻辑的地方，它们可以隐藏一些非常讨厌的错误。

Hilary 说:“很长时间以来，我一直倡导人们进行 API 测试，但在组织和举办了一个关于测试 API 的研讨会后，我意识到 UI 和 API 如何相互交流并不经常在讲座或研讨会中涉及到。“这是我很久以前不得不自学的一个领域，我认为这对人们来说是很有价值的信息。”

和很多测试人员一样，Hilary 在一个偶然的机会下碰巧走上了软件测试生涯，并坠入爱河。她目前是 [速贷](https://www.quickenloans.com/) 的软件质量架构师，是一个工程师团队中唯一的测试人员。现在，凭借超过 12 年的软件测试经验，Hilary 专注于 API 测试和 C#自动化，并在她的家乡底特律运营着 [汽车城软件测试人员](https://www.meetup.com/Motor-City-Software-Testers/) meetup 小组。她积极参与 API 测试和测试工具的培训。

任何参加她在 SauceCon 2020 上的演讲的人都会对 UI 和 API 如何协同工作有更好的理解。

她说:“当用户界面出现问题时，这将使人们更容易排除故障。”。"此外，它会给你一个测试下的应用程序的更完整的图片."

当她不为她经常在世界各地举办的课程、讲座和研讨会开发内容时，Hilary 喜欢玩视频游戏，与家人和她的狗共度时光。

在第四届年度酱料实验室用户大会 SauceCon 2020 上，你可以听到来自 希拉里 和其他 30 多位专家的演讲。要查看完整的演示列表或注册 、 [请访问 SauceCon 网站](https://saucecon.com/) 。
