# IT 公司越来越多地向开发运维过渡

> 原文:[https://devo PS . com/transition-to-devo PS-on-demand-跨 it 公司/](https://devops.com/transition-to-devops-on-demand-across-it-firms/)

**DevOps established a new style of IT functioning easing different phases of software process from product development to testing and final market release, all in a rapid and reliable way.**

对更快的产品开发和快速市场发布的需求产生了开发(Dev)和运营(Ops)团队之间增加协作的需要，这最终导致了“DevOps”因此，过渡到 DevOps 已经成为许多 IT 公司讨论的重点之一。本文概述了组织过渡到 DevOps 的需求和趋势。

IT 创新的兴起和蓬勃发展的数字革命导致了对更快开发和更快发布的需求，以便在 IT 竞争场景中脱颖而出。这种转变最终导致了两个关键 IT 团队(开发和运营团队)之间协作的新文化，业界称之为 DevOps。

DevOps 建立了一种新的 IT 运作方式，简化了软件流程的不同阶段，从产品开发到测试和最终市场发布，一切都以快速可靠的方式进行。

## **为什么要进行 DevOps 转型？**

独立软件组件的快速交付及其与生产系统的持续集成构成了 DevOps 功能背后的关键原则**。**

DevOps 经常与敏捷方法混淆。虽然它们在通过快速开发提供改进的 IT 性能方面有相似之处，但是 DevOps 不同于敏捷，因为它在软件部署阶段的效率，确保了两个团队之间的持续沟通。因此，DevOps 通常被称为敏捷的继承者。

协作方法是跨线信息交换的关键，也是推动公司进行 [DevOps 实施](https://www.veritis.com/blog/devops-implementation-strategy-built-philosophy-tools-collaboration/)的主要因素之一。

虽然 DevOps 是众所周知的开发和运营团队的合并，但有效的 DevOps 模型可以扩展到组织的其他关键功能领域。

DevOps 打破了将单个团队限制在各自工作领域的旧趋势，为他们之间的知识共享带来了空间，开发人员可以将自己扩展到测试和部署，运营团队成员可以升级他们的技能集，学习新的语言、架构和工具。

## **对 DevOps 资源的需求增加**

过渡到 DevOps 是快速增长的 IT 市场的一个关键需求，在这个市场中，管理现有的基础架构或资源与开发新的东西几乎同等重要。

撇开绿地项目不谈，DevOps 部署最常见于鼓励在现有资源基础上进行建设的棕色地块项目。

在这种情况下，公司之间对 DevOps 资源也有很多需求。来自 Linux 基金会的“2017 年开源工作报告”显示，在 2000 多名 IT 开源专业人员和 IT 招聘经理中，60%的人需要 DevOps 人力资源。

这清楚地表明了 IT 行业中过渡到 DevOps 的需求水平。

求职网站 Dice 的首席执行官 Mike Durney 说:“如果你的公司中没有 DevOps 这个角色，软件开发人员应该花时间与运营团队坐在一起，或者相反，以了解他们的流程是如何工作的，以及如何通过协作来改进这些流程。”。

**报告发现:**

*   近 50%的招聘经理对那些拥有相关认证等专业证书的 DevOps 资源很感兴趣
*   超过 47%的公司准备资助现有员工的认证项目
*   42%的公司热衷于将 DevOps 技能添加到他们的招聘组合中
*   “DevOps 工程师”简介在 Glassdoor 全美最佳 50 份工作中排名第二

尽管有这些优势，DevOps 仍然有一些与应用程序集成、测试自动化、预算和技能组合等相关的挑战需要克服。
