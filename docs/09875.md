# Sonatype 报告揭示了已知漏洞挑战的范围

> 原文:[https://devo PS . com/sonatype-report-surfaces-scope-of-known-vulnerability-challenge/](https://devops.com/sonatype-report-surfaces-scope-of-known-vulnerability-challenge/)

Sonatype 本周发布了一份软件供应链报告的状态[,发现针对驻留在公共存储库中的开源软件的恶意攻击同比增长了 633%。](https://www.globenewswire.com/news-release/2022/10/18/2536192/0/en/Sonatype-s-8th-Annual-State-of-the-Software-Supply-Chain-Report-Finds-96-of-Known-Vulnerable-Open-Source-Downloads-Are-Avoidable.html)

此外，Sonatype 推出了一个 [Sonatype 安全评级](https://securityboulevard.com/2022/10/how-is-the-sonatype-safety-rating-determined/)系统，该系统采用机器学习算法和其他指标来识别存储在这些存储库中的最安全的开源组件。

该报告发现，96%的开源 Java 下载存在已知漏洞，如果使用更安全的可用版本，这些漏洞是可以避免的。总的来说，该报告估计，如果开发人员只知道软件组件的更高版本可用，就可以避免 12 亿个已知易受攻击的依赖项。

根据该报告，Java 应用程序平均包含 148 个依赖项。Java 应用程序的更新平均每年进行 10 次，因此开发团队每年需要跟踪 1，500 个依赖关系的变化。

Sonatype 首席技术官 Brian Fox 表示，该报告清楚地表明[组织没有为使用具有已知漏洞](https://techstrong.tv/videos/interviews/whats-required-to-make-devsecops-really-work-wayne-jackson-and-brian-fox-sonatype)的开源软件承担足够的责任。他说，实际上，组织盲目地、不计后果地使用软件组件，而没有真正意识到相关的风险。

Fox 说，IT 团队不应该对发现的每个新的零日漏洞做出下意识的反应，而是需要专注于将流程落实到位，以确保使用任何给定软件模块的最新、最安全版本。然而，对报告中包含的 662 名工程专业人员的调查发现，68%的受访者确信他们的应用程序没有使用已知的易受攻击的库。然而，Sonatype 对企业应用程序进行的随机抽样发现，68%的应用程序包含已知的漏洞。

总的来说，该报告推测，开放源码软件的消费将很快超过估计的 3.1 万亿次总请求，每七个漏洞中有六个是由软件组件之间的间接关系造成的传递依赖性造成的。Fox 说，按照目前的消耗速度，很明显，鉴于组织对应用程序的依赖程度，今天用于构建和部署应用程序的流程是疯狂的。

幸运的是，在一系列引人注目的安全漏洞之后，与去年同期相比，如今人们更加关注保护软件供应链。然而，当您考虑已经部署在具有已知漏洞的应用程序环境中的全部应用程序时，挑战的范围是令人难以置信的。组织需要首先确保修复最关键应用程序中的漏洞，同时实施 DevSecOps 最佳实践。这将确保尽可能少的漏洞进入下一代应用程序。

显然，这些应用程序安全问题不会在一夜之间得到解决。DevOps 团队将在接下来的几年里解决技术安全问题，这个问题已经存在了很多年，IT 界的任何人都不愿承认。
