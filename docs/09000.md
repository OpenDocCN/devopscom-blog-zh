# 支持政府采用云

> 原文:[https://devo PS . com/enabling-cloud-adoption-in-the-government/](https://devops.com/enabling-cloud-adoption-in-the-government/)

2022 年， 联邦机构有 [估计](https://www.jdsupra.com/legalnews/us-government-s-cloud-expansion-1512766/) 要在云计算上花费 78 亿美元 。虽然在云计算上的支出是必要的，但政府必须恰当地使用分配给它的资金，以充分利用云计算的潜力 并避免常见的陷阱 。

### 躲避 C 小目 C 高声 收养 P itfalls

我经常目睹政府机构被一家云经销商套牢，这家经销商无法为他们提供由 AWS、谷歌云或微软 Azure 提供的 全部功能。联邦市场之外的大多数人没有意识到，政府并不直接从云提供商那里购买云——除了 一些非常大的合同，如即将推出的 JEDI 替代合同车辆，JWCC。他们通过提供批量折扣的经销商合作伙伴网络购买，并尝试捆绑托管服务来帮助进行云管理。

这种方法赋予经销商而非客户权力，让他们掌握自己王国的钥匙 — ，并使客户难以在表现不佳、缺乏接入或计费透明度的情况下轻松切换云提供商。联邦政府将受益于保持一个小的 、 但是有目的的、、 云项目管理办公室(PMO)内部。从而确保 root 访问对所有帐户、 订阅 和项目可用，以向代理机构提供自由，使其能够随着云成熟度的发展更轻松地更换合作伙伴。

我见过 也见过 的政府客户不注重 的整体终端用户体验，实际上 使用的是 云。结果是不同系统、流程和步骤的分散集合，导致工程师沮丧和低效。客户必须转变端到端云配置流程，以便自助–服务至关重要，并且现有技术投资能够无缝、透明地集成到最终用户。如果做得正确，创新将是副产品，因为可以专注于开发下一代任务系统和工具— 而不是 试图确定如何获得云帐户来开始项目。

### 政府机构可以采取的三个 步骤 从云基础设施中获得价值

挑战是存在的，但政府可以采取一些行之有效的措施来从云基础设施中获得最大价值。

*   定义目的:首先，也是最重要的，理解云基础设施的目的。您是否希望重新设计云架构、变得更具成本效益，或者通过引入新的服务让您走得更远、更快来转变组织内的创新潜力？理解这个目的和你的目标将会对成功大有帮助。
*   量化投资回报率:为了获得最佳投资回报，了解哪些资源已经可用，哪些资源是运营所必需的。为了充分利用云，确保基本成本在控制之下是很重要的，例如确保 AWS EC2 或 Azure VM 实例不会过度配置或利用不足。同样重要的是，要确保利用为可预测的工作负载预留的容量；这些可以为你节省 30%甚至更多。最后但并非最不重要的一点是，专注于充分利用技术解决方案来帮助将所有部分整合到一个跨云提供商的集成式用户友好体验中。
*   考虑外包方法:引入云经销商和承包商可以为您的组织提供指导，并帮助您做出更好的选择。就像建造或翻新房子一样，机构需要总承包商类型的公司来帮助管理和协调迁移到云及其所有移动部分的过程。这些承包商还可以帮助找到在各种特殊情况下完成工作所需的各种专家。从长远来看，拥有一个小型、高效的云 PMO 是每个机构都应该努力实现的目标；然而，考虑到对云工程人才的需求，近期的重点应该是确保与合作伙伴合作的灵活性，以便他们不会阻碍组织的潜力。

随着数字化转型和[云迁移](https://devops.com/5-steps-to-a-seamless-multi-cloud-migration/)的继续，政府跟上它的步伐是很重要的。到目前为止，大多数机构仍然依赖手动步骤来验证控制并获得运营授权(ATOs)批准。简单地转换到自动化程度更高的流程，学习如何充分利用云资源，以及将运营外包给承包商，这些都有助于加快速度。 还有 对云支持的需求从未如此之大。现在是扩大规模的时候了。
