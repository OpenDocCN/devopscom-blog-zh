# 使用 Palo Alto 实现安全性自动化

> 原文:[https://devops.com/automating-security-with-palo-alto/](https://devops.com/automating-security-with-palo-alto/)

今天，自动化比以往任何时候都更加重要。尽早将安全性和自动化集成到 SDLC 中可降低潜在网络攻击的风险，提供早期相关反馈，缩短上市时间并提高产品质量。

在这期 TechStrong 电视节目中，Palo Alto Networks[Prisma Cloud](https://www.paloaltonetworks.com/prisma/cloud)的高级产品营销经理 Taylor Smith 讨论了采用以软件为中心和以云为中心的方法来实现安全性的必要性。他解释了自动化安全以跟上快速变化的应用程序和软件基础设施的重要性。

## 副本

米奇·阿什利:泰勒·史密斯的加入让我倍感荣幸。泰勒是帕洛阿尔托网络公司 Prisma Cloud 的高级产品营销经理。欢迎你泰勒。

泰勒·史密斯:非常感谢。

阿什莉:很高兴今天能和你在一起。给我们介绍一下你自己吧。我知道您最近加入了 Palo Alto Networks，您在该行业有着有趣的背景，当然，请告诉我们一些关于 Prisma Cloud 的信息。

史密斯:对，绝对是。所以，我马上就要满六个月了，我真的很享受。我来自大公司和小公司，所以之前在基础设施公司，然后我在 DevOps 领域做了一点工作，从事一些 DevOps 工具和混沌工程。真的很喜欢，但很高兴回到网络安全，这是我真正的激情，并涵盖了云安全和 Palo Alto Networks 的 shift left 安全。

阿什利:优秀。给我们讲讲 Prisma Cloud 吧。我知道它非常全面，包含了很多东西，但是你如何给出一个简单的概述呢？

史密斯:对，绝对是。因此，Prisma Cloud 的优势在于，我们在云安全的多个不同类别中都是一流的。因此，我们真正覆盖了云安全的整个生命周期，从基础架构一直到代码级别，即运行在基础架构之上的应用程序的代码，我们在云应用程序的整个生命周期中都这样做。因此，我们在构建时和运行时，一直将您的基础架构作为代码保护到您的容器映像。

阿什利:优秀。嗯，作为一个——我自己也用过 Palo Alto Network 的产品。作为一家网络公司，如果你想这样想，也许这是帕洛阿尔托网络公司过时的思维方式，这是我在这里的观点，我们正在谈论应用程序安全和应用程序，实际上，现在一切都在变成软件。所以，网络本身，当然，已经变成了软件，现在在云和本地。但是整个堆栈，整个堆栈，你知道，作为代码的基础设施。

因此，这带来了一些明显的好处，也带来了一些挑战。你对此有什么想法？

史密斯:对，绝对是。因此，现在进入应用领域是一个有趣的时刻。以前，您必须进去物理配置服务器，您必须将线缆连接到路由器和网络交换机，所有这些，然后可能进入 CLI 进行一些更改。

所有这些都被转移到代码中，从基础架构代码一直到您定义它们如何运行的容器，操作系统都包含在代码中。这对开发人员来说很好，他们能够更快地移动，他们可以动态地配置资源，他们可以在需要时使用资源，或者在不需要时缩减资源。它节省了我们的成本，并使事情进展得更快。

这是一把双刃剑，现在它脱离了安全和基础架构，脱离了 it 团队的掌控。开发者只是在部署东西，这有点像狂野西部。因此，为了控制整个世界，确保他们安全并做正确的事情，最好将这种安全性嵌入到他们的流程中。因此，确保您将基础架构作为代码模板进行扫描，在容器映像到达存储库和部署到生产环境之前对其进行扫描。这确实给了我们一个机会，让我们可以更频繁地打补丁，因为现在开发人员得到了直接的反馈，坦率地说，他们比安全团队更有资格进行这些修复。因此，如果他们得到正确的反馈，他们实际上可以进入他们编写的代码并进行修改。

**Ashley:** 是的，作为一名安全工程师，当你想到软件时，我认为这是一件事情，它不是——我现在可以把它想象成一种流体，而不是固体。

史密斯:对。*【笑声】*

阿什莉:一直在变，对吧？这使得它很难保证安全，有时从质量的角度来看也很困难。但是，真的，它几乎是一种物质形式，事物是不断变化的。因此，您不能指望手动流程停止生产线会将安全性带回正轨。你必须配备人员，你必须配备工具链，即创建代码并将其交付给生产的管道，然后通过你想要保护的内容、你想要如何保护它、你要处理的内容以及出现的问题来指导该过程。

史密斯:对，绝对是。微服务确实彻底改变了这一点，我们有几个客户，他们的应用程序在一周开始时看起来一点也不像在一周结束时的样子。他们已经完全重新部署了整个堆栈，这对创新来说非常好，它确实打开了世界，但是它确实打开了世界，作为一名安全工程师，您不希望进入那里并减慢速度。你希望成为开发的一部分，并成为支持者，成为真正实现这一速度的人，但仍要确保它是安全的，并且不会受到应用程序中的错误配置和漏洞的影响。

阿什利:嗯。云又是什么呢——我的意思是，我们谈到了微服务容器，它们不一定要在云上运行，但当然，我们认为它们是云原生架构。是从基础架构管理一直到应用程序本身？当一切都在我们自己的私有数据中心运行时，除了我们所做的以外，我们必须如何重新思考事情？我们可能会尝试以旧的方式在云中实现，但实际上，我们必须从根本上改变你对它的看法，这有什么根本的不同？

史密斯:是的，我认为很大程度上可以归结为一切都是可访问的。我的意思是，有专用链接和 VPN 来确保它被锁定，但最终，您无法实际访问这些数据中心。所以，很大程度上归结于，公众可以以某种方式访问，我们尽可能地锁定它。

但是，如果你在 AWS 中有一个 EC2 实例，或者在 Azure 或 GCP 中有一个 VM 实例，那么它就可以被公开。确保设置了安全组或防火墙规则，这样就不会有问题，我的一位客户向我解释得很清楚。在本地环境中，您会得到这些层层检查，即使有人进入并调配了一个暴露在外且具有所有这些错误配置的虚拟机，它也很可能不会接触到环境。没人能拿到它，所以没什么大不了的。但是云安全，那些错误配置真的会咬你一口，因为它们是暴露的。因此，做好这一点更为关键，确保发现并修复错误配置，这比在本地环境中要重要得多。

阿什莉:真有意思。我想知道你如何接近像 Prisma 这样的平台。显然，已经是帕洛阿尔托客户的人可能会通过他们与你的关系自然而然地找到它。但是，这是怎么回事，是一个网络工程团队说，“我需要找出如何保护云？”是软件开发团队说，“我需要更好的工具来确保我正在配置和编写安全的代码？”通常是谁让你为 Prisma 工作，然后他们如何将这种工作带入组织并传播爱，如果你愿意的话？*【笑声】*

史密斯:对，对。不，这很有趣。与我以前的一些工作相比，我真正喜欢在 Prisma 工作的部分原因是，我不是与一个人或原型一起工作，我是与一些云工程师一起工作，他们是这种新的-坦率地说，对我来说是新的工程类型，他们涵盖了云的所有内容。他们正在为云中的一切资源调配基础架构提供完整的应用程序堆栈。以前，您可能有一名网络专家和一名计算专家，而现在，您只有一名工程师来负责整个云。此外，我们还有开发人员和 DevOps 团队与我们合作，因此不一定是安全团队。此外，我们还与安全团队合作，包括 SOC、安全工程师和云安全专家。

他们所有人都有不同的美国案例，他们带着这些案例进入 Prisma Cloud，但他们最终都试图使用 Prisma Cloud 来简化他们的工作，使安全自动化，并确保检查正常进行。因此，开发人员更担心部署前的时间，他们可以在实际运行容器之前保护他们的应用程序、作为容器映像的代码的基础设施。

然后，安全团队实际上进入我们的系统，查看云中的状况，获得该视图，获得在他们的云环境中运行的所有东西的完整、全面的视图，然后保护他们的容器、他们的 Kubernetes 环境、他们的无服务器环境以及所有实际运行的东西。通过 Prisma Cloud 运行事件管理，运行事件和所有跟踪数据，如果有事件发生，我们会提供给他们。

**Ashley:** 安全工程师/网络工程师的工作发生了怎样的变化，这一角色和所涉及的技能真的很令人着迷。我想知道，我只是好奇作为一名网络工程师/安全工程师，你的经验是什么，处理软件和软件配置，也许自动化，脚本编写，诸如此类的事情。看起来这将使这份工作与软件开发人员的工作更加相关。它们不是一回事，但与我们不久前使用的机架和堆叠、插件以及网络和物理设备有很大不同。

史密斯:对。是的，很有趣，我同意。基础设施工程师的角色越来越像软件开发人员。因此，他们实际上正在学习 Python 或 HCL，这样他们就可以编写 Terraform 脚本，或者编写 Ansible 脚本，并使用代码部署基础架构。现在可以通过管道传输。现在，我听到基础架构人员开始谈论 CI/CD，这在本地环境中是绝对想不到的。他们使用了很多相同的语言，比如对他们的基础设施进行单元测试和静态分析。

因此，他们正在学习成为一名开发人员，但专门针对基础设施。

我记得大概是 11 年前的那一天，我接手了一家 it 公司，一位网络工程师问我:“我应该学些什么？我下一步应该考虑什么？”我说:“学 Python 吧”，他斜眼看着我。

**史密斯:** *【笑声】*

**Ashley:** 最后去参加了一个会议，可能是你的会议，回来后说，“现在我知道你为什么这么说了，现在我们看到了正在发生的事情”——是的，这是在不到十年的时间里发生的巨大转变。想一想，网络堆栈发生了多么大的变化，并作为代码的一部分真正融入了基础架构。

在你的职业生涯中，你的工作发生了怎样的变化？在我们的行业中，这种变化是如何发生的，你有什么进步吗？

**Smith:** 是的，就像我说的，我来自一家大型基础设施公司，当时负责传统网络安全，后来我去了一家 DevOps 公司。所以，我有点看到了这种完全的转变，事情自动化了，人们行动迅速。可靠性是一种——它是云中的一个全新定义。所以，现在，即使是小公司也可以在以前没有的地方裁员。

这对保安人员来说是非常不同的。因此，如果我—现在我再次讨论安全性，但现在从一个更注重开发人员的角度，我会看到自动化比以往任何时候都更加重要，网络边界不再是唯一可以应用安全性的地方。它变得越来越不重要，但它仍然是至关重要的，但所有的事情都发生在里面。因此，在您的基础设施、应用程序内部建立零信任网络，保护您的 Kubernetes 部署，确保您的容器不会被过度授权，或者您不会从易受攻击的开源映像中获取映像。

所有这些现在和我开始职业生涯时所用的传统安全一样重要，甚至更重要。

阿什利:嗯。是啊，这是我们经历的一段迷人的旅程。

史密斯:当然。

阿什莉:嗯，我们的时间不多了。人们在哪里可以找到关于 Prisma Cloud 和 Palo Alto Networks 产品的更多信息？

史密斯:当然，你可以看看我们的网站。所以，你可以去 PaloAltoNetworks.com/Prisma，在那里你可以找到更多的信息，或者我们有很多有用的资源，比如博客或白皮书，所以，如果你想了解更多关于容器安全或云安全的信息，去那里，那里有更多的信息给你。

阿什利:我只是想建议，对，开发者以及安全和网络人员，如果我们想说你的角色或头衔可能是什么的话。泰勒，与你交谈令人着迷，很高兴能与你一起经历我们所用技术的发展和正在发生的创新。

所以，希望你做得很好，你会回来和我们再次交谈。

史密斯:非常感谢你，米奇。这是我的荣幸。
