# Atlassian 调查显示软件开发缺乏协作

> 原文：<https://devops.com/atlassian-survey-shows-lack-of-collaboration-in-software-development/>

一项对 750 名 IT 和工程领导者的调查发现，尽管 83%的受访者认为部门间的沟通对成功的软件开发至关重要，但只有 41%的受访者认为交叉协作是优先考虑的事情。

该调查收集了年收入超过 5000 万美元的组织的反馈，由 Forrester Consulting 代表 Atlassian 进行。

总体而言，超过三分之二的受访者(68%)表示，参与软件开发的部门比以往任何时候都多。总的来说，80%的人说他们需要改善与其他部门的合作，但是 82%的人对他们完成这个目标的能力没有特别的信心。四分之三(75%)的人还指出需要改善他们自己部门内部的协作。

不到一半的受访者(42%)表示，他们的文化、流程和工具支持实时团队合作和自由流动的信息，这有助于领导者做出正确的决策。

然而，在高度合作的受访者中，超过 53%的受访者表示他们看到了更高的客户满意度，59%的受访者表示由于客户满意度的提高，收入也有所增加。

Atlassian 的 DevOps 产品负责人 Suzie Prince 表示，组织面临的挑战是找到一种方法，成功地采用同步和异步通信介质的混合，以更好地实现协作。她补充说，事实上，作为任何 DevOps 文化的一部分，协作需要成为 C 级的讨论，以不断加快软件开发和部署的速度。普林斯说，核心问题是，并不是每个管理团队都同样喜欢组成一个组织的各个部门之间的开放式沟通。

对于高管来说，设定基调并以身作则至关重要；DevOps 最佳实践不能仅仅由开发团队来坚持。整个企业需要采用 DevOps 最佳实践作为业务运营模式，以真正获得最大收益。

不幸的是，开发团队中的 DevOps 实践没有被均衡地采用，所以通常很难让业务的其余部分接受 DevOps 原则。然而，采用 DevOps 的组织往往在成为数字企业的道路上走得更远，并且可以更熟练地利用出现的业务机会，或者根据不断变化的业务条件改变策略。事实上，在经济衰退期间，采用 DevOps 最佳实践的组织可以说表现得更好，经济衰退是全球新冠肺炎疫情的开始，让每个业务和 IT 领导者都感到意外。

从 Atlassian 的调查结果来看，尚不清楚 DevOps 鸿沟是否开始出现，从而使一些组织比竞争对手更具竞争力。至少在理论上，如果每一个组织现在都是一个软件公司，恰好构建一些东西或交付一个服务，那么他们越快可靠地构建和交付应用程序，他们就能获得和保持越大的竞争优势。

当然，决定任何组织竞争力的因素有很多。然而，在数字业务转型的时代，很明显，能够以相当规律的节奏构建和部署软件是简单生存所需的新赌注。
