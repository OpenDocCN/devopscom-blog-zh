# Atlantic.Net 将德沃普斯文化归功于思维敏捷

> 原文：<https://devops.com/atlantic-net-credits-devops-culture-mental-agility/>

Atlantic.Net 创始人&首席执行官马蒂·普拉尼克表示，Atlantic.Net(www.atlantic.net)是一家基于固态硬盘的云托管提供商，在纽约市、旧金山、达拉斯、多伦多、奥兰多，以及(即将推出的)伦敦和新加坡都设有办事处。Atlantic.Net 云系统对于开发人员来说是一个直观的 IaaS 服务器环境。该公司通过在整个组织内进行深度、全栈 DevOps 集成来支持其 24/7/365 运营。

使改变成为可能的 Atlantic.Net 软件栈包括 Linux、Nginx([http://nginx.org/](https://nginx.org/))、PHP([http://php.net/](https://php.net/))/Python([https://www.python.org/](https://www.python.org/))、用于内部应用链接的 RESTful APIs、jQuery([https://jquery.com/](https://jquery.com/))、Ajax(http://www . w3schools . com/Ajax/)、MongoDB(/[www.mongodb.org](https://www.mongodb.org))、node . js([https://nodejs.org](https://nodejs.org))和 KVM 虚拟化。Puranik 解释说:“我们使用串行协议通过 Infiniband(一种快速网络类型)进行通信，因此我还会添加 SRP 或 iSER 串行通信协议。(Infiniband 通常应用于高性能计算应用。)Atlantic.Net 在使用这些工具为其软件、客户和 DevOps 文化服务时，采用了一种特殊的思维模式。

思维敏捷是 DevOps 文化的关键

根据 Puranik 的说法，Atlantic.Net 团队认为，在人力资源中保持思维敏捷，包括所有利益相关者——客户、部门和供应商，他们也经常管理 DevOps 团队——是蓬勃发展的 DevOps 文化的主要元素。“这里的思维敏捷指的是使用新工具或现有工具的升级来重新制定最佳方案的能力。Puranik 解释说:“思维敏捷还意味着不要设计与某项传统技术紧密结合的系统，这样就不可能将它们从该技术中移除。

Atlantic.Net 致力于设计这样的软件，他们可以很容易地从堆栈中添加、更改或删除旧的和新的技术，而无需对代码库进行重大修改。为了做到这一点，Atlantic.Net 团队在实现之前投入了大量时间来仔细考虑设计问题，以确保技术和开发堆栈之间关系的灵活性。团队花了额外的时间来抽象新功能如何工作的不同模型，以便他们可以为每个人做出最佳选择。这是一个比紧密集成新技术更好的选择，因为这对他们来说更容易思考，并使开发进行得更快。“从这个意义上来说，思维敏捷意味着保持选择的开放性，这样你才不会最终陷入绝境，失去最佳选择，”Puranik 说。

这种类型的思维敏捷的好处是双重的:能够持续地进化代码库，而没有从永久的联系到过时的开发活动的任何阻力或摩擦；此外，能够在不破坏代码的情况下，通过最少的代码更改为客户添加功能。

延伸战略以满足利益相关者

Atlantic.Net 敏捷的头脑考虑到了利益相关者，因为对软件的错误更改会对客户、员工和供应商产生影响。“例如，如果我们改变客户访问某个特定功能的方式，这就为客户创造了一个学习过程，”Puranik 说。如果变更没有带来足够的价值，不值得学习，那么开发团队就不应该让客户或他们自己经历整个混乱。

同样，添加来自第三方供应商的软件组件(如 API)的新表达式会使供应商的可靠性受到质疑。“供应商能处理我们将要进行的大量查询吗？否则，对供应商、对我们、最终对我们的客户都是不利的，”Puranik 说。

Atlantic.Net 必须从每个人的角度看待整个发展进程，穿上他们的鞋子，考虑发展选择的结果。Puranik 说:“有时，Atlantic.Net 内部的不同人必须代表内部和外部的不同利益相关者，以使整个团队以不同的方式思考，或者至少思考具体行动的结果可能是什么。这样，连续部署既快速又无害——双赢！
