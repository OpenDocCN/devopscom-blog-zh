# Chef Automate Incident Creation 获得 ServiceNow 的应用程序认证

> 原文：<https://devops.com/chef-automate-incident-creation-receives-application-certification-from-servicenow/>

*新的集成为客户处理事件管理提供了简化的访问权限，大大缩短了解决问题的时间，并提高了最终用户的可见性和工作效率*

**SEATTLE—****2018 年 9 月 11 日—**[持续自动化领域的领导者 Chef](http://www.chef.io/) 今天宣布，其 Chef Automate Incident Creation 应用程序获得 ServiceNow 认证，现已在 [ServiceNow 商店](https://store.servicenow.com/sn_appstore_store.do#!/store/application/02bf0043dba2d30070cd5e25ca9619a3/1.0.10)发售。ServiceNow 的认证仅授予商店中可用的应用程序，表明 Chef Automate Incident Creation 应用程序已成功完成一系列针对 Now Platform 安全性、兼容性、性能和集成互操作性的测试。该认证还可确保在设计和实施 Chef Automate app with ServiceNow 时采用最佳实践。

Chef Automate Incident Creation 应用程序允许用户在其 ServiceNow 事件管理环境中轻松生成事件，无论何时他们的 Chef 托管平台因基础架构自动化更新失败或自动化合规性检查失败而出现故障。这种与 ServiceNow 的集成使 Chef 客户能够捕获其场所中的所有故障，以进行解决和跟踪，从而确保任何基础架构自动化问题得到快速解决，并根据内部最佳实践跟踪和纠正合规性故障。

这种集成的关键组件包括:

*   **针对基础设施和合规自动化的事件管理**–该功能使用户能够创建 ServiceNow 事件单、警报/页面操作或信息安全(分别针对 Chef 故障或 InSpec 故障)，以便尽快处理自动化和合规性故障，并符合客户或监管机构要求的 SLA 或 OLA。
*   **智能数据管理和事件重复数据删除—** 集成智能地使用 Chef Automate 作为中央数据聚合器并删除重复事件，因此重复故障不会导致重复事件单。相反，它们会更新 ServiceNow 中的相关字段，比如上次失败的时间。这有助于降低整体警报噪音，与非 Chef Automate 方法形成对比，后者将数据直接从受管节点发送到 ServiceNow。
*   **与 ServiceNow 平台上其他功能的合规性相关集成机会** —由该集成生成的合规性事件数据流现在可以被 ServiceNow 平台上的其他应用利用。例如，与 ServiceNow 集成的治理和风险合规(GRC)或安全/事件管理(SIEM)系统现在可以访问这些数据。对于那些已经在此类系统上实现标准化的公司来说，通过 Chef Automate 从 InSpec 获得的数据流允许用户使用实时、连续的合规性事件来增强他们的整体风险仪表板，这些事件已经过适当的优先级排序，而不是像以前那样用无意义、与上下文无关的数据来淹没这些系统。

Chef 合作伙伴集成副总裁 James Casey 表示:“这种集成一直是最受欢迎的功能之一，我们的许多客户都要求与 ServiceNow 的首要 IT 服务管理解决方案进行集成。“我们期待与 ServiceNow 合作，为我们的客户提供一个单一位置来跟踪他们自动化基础架构中出现的所有问题，这将提高 ServiceNow 和 Chef Automate 在企业中的整体便利性，并有助于减少任何故障情况的整体影响。”
