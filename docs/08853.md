# 2021 年十大值得感恩的事情

> 原文：<https://devops.com/top-10-devops-things-to-be-thankful-for-in-2021/>

美国从 1861 年开始庆祝感恩节，并于 1863 年由林肯总统正式指定为官方假日。当然，这个传统本身可以追溯到早期的清教徒时代。

当然，过去的一年是从导致全球数百万人死亡的疫情中恢复过来的一年。疫苗拯救了无数更多人的生命，因此尽管我们最近经历了种种考验和磨难，但仍有许多值得感谢的事情。本着这种精神，对于 DevOps 专业人员来说，今年有很多事情需要特别感谢，首先是:

**数字业务转型:**由于购买者行为的重大变化和持续的供应链危机，各种规模的组织都不得不调整其业务模式。DevOps 团队是任何重大数字业务转型计划的核心，因此对 DevOps 专业知识的需求继续超过可用的供应。展望未来，据预测，我们将在未来 10 年看到比过去 40 年加起来还要多的数字化变革。如果没有开发人员和工程师的专业知识，携手以前所未有的速度构建和部署应用程序，这些计划中的绝大多数永远不会问世。

**低代码平台:**随着疫情的到来，推动数字工作流的应用需求激增，专业开发人员更加依赖低代码平台来减少构建应用所需的时间。虽然许多复杂的应用程序仍将使用过程化代码来构建，但很明显，如果不依赖于低代码平台来构建它们，就不可能部署如此多的应用程序。下一代低代码应用程序中有多少将由所谓的公民开发者构建，这仍有待观察，他们可能很快就会用应用程序代码淹没 DevOps 管道。

**微服务:**作为构建应用程序的架构，采用松散耦合的服务来构建应用程序的核心概念可以追溯到 20 世纪 90 年代，当时面向服务的应用程序(SOA)被认为是下一个大事件。当然，微服务本身已经存在好几年了。然而，随着容器作为构建微服务的软件构件的兴起，现在才变得显而易见的是，构建使用微服务构建的应用程序不仅需要更少的时间，而且更易于维护，也更具弹性。

**可观测性:**作为一个概念，可观测性可以追溯到线性动态系统。[可观测性](https://devops.com/?s=observability)以其最基本的形式衡量系统的内部状态可以根据其外部输出的知识推断出的程度。在过去的一年里，许多 IT 厂商推出了各种类型的可观察性平台。这使得 DevOps 团队可以更轻松地查询机器数据，从而在问题造成进一步破坏之前主动发现问题的根本原因。与通常仅提供预定义指标来确定特定平台或应用程序何时在预期范围内运行的监控平台相比，这是一项重大进步。

**拜登的行政命令:**在来自网络安全管理软件产品的针对 IT 服务管理软件的恶意攻击之后，拜登政府发布了一项全面的行政命令，要求联邦机构实施广泛的安全最佳实践。其中最重要的是关于软件供应链安全的指导；这导致许多组织审查他们的整个软件[供应链](https://devops.com/?s=supply+chain)，以确保类似的违规事件不会发生。

作为任何 DevOps 最佳实践的扩展，DevSecOps 通常描述了在使开发人员对应用程序安全性承担更多责任方面的整体转变。虽然 DevSecOps 仍处于相对初级阶段，但随着 DevOps 和网络安全团队寻求弥合与 IT 领域一样广泛的文化鸿沟，它已开始获得关注。最终目标是在不降低构建和部署速度的情况下交付更安全的应用程序。

**GitOps:** 在其核心，GitOps 是一个运营框架，采用了用于自动化应用开发的 DevOps 最佳实践，并将它们扩展到基础设施。基本思想是，为管理基础设施代码而创建的配置软件只是另一个要存储在 Git 存储库中的软件工件。该方法为 IT 团队提供了一种实现持续交付目标的方法，在 DevOps 最佳实践被定义十多年后，这一目标仍然难以实现。就 GitOps 最佳实践而言，现在还为时尚早，但今年 GitOps 已经成为 DevOps 主流话题的一部分。

Kubernetes:Kubernetes 同时也是众所周知的企业 IT 平台中最强大和最复杂的 IT 平台之一，已经成为构建和部署容器应用程序的事实上的标准，这些应用程序可以根据消费 IT 基础设施的需要更有效地扩展和缩减。现在的挑战(和机遇)是让普通的 IT 管理员更容易访问 Kubernetes，这些管理员通常不具备网站可靠性工程师()的编程技能，而且很难找到和留住他们。

**GraphQL:** 作为应用编程接口(API)的查询语言， [GraphQL](https://devops.com/?s=GraphQL) 提供了一种比开发人员过去使用的 REST APIs 更有效的访问数据的方式。GraphQL 最初是由脸书开发的，其采用仍处于起步阶段，但很明显，基于 graph QL 的 API 有一天会像 REST APIs 一样丰富。这对于基于微服务的应用程序尤其重要，这些应用程序依赖 API 在高度分布式的计算环境中访问数据，需要更高效的方法来传输数据。

**风险和华尔街资本投资:**投资公司正在帮助推动 DevOps 的发展，因为他们继续为争夺市场份额的公司提供资本，预计到 2026 年[的复合年增长率将达到 24.7%](https://www.verifiedmarketresearch.com/product/devops-market/#:~:text=the%20DevOps%20Market%3F-,DevOps%20Market%20was%20valued%20at%20USD%203.62%20Billion%20in%202018,24.7%25%20from%202019%20to%202026.)。当然，随着并购活动不可避免地接踵而至，最终将会出现一波整合浪潮。然而，与此同时，如果不是所有的资金都投入到 DevOps 公司，有许多 DevOps 技术可能不会被广泛采用。

当然，最重要的是这么多 DevOps 专业人员仍然在这里构建和部署应用程序。对所有相关人员来说，这是艰难的一年；然而，随着疫情危机继续消退，我们都应该感谢前面的机会。
