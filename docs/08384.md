# 全栈可观测性能帮助减轻 IT 负担吗？

> 原文：<https://devops.com/could-full-stack-observability-help-lessen-its-burden/>

除了显而易见的原因，2020 年是具有里程碑意义的一年。IT 界第一次看到云服务的收入超过了数据中心的支出。总体而言，围绕[云采用](https://devops.com/how-to-respond-to-rising-cloud-costs/)的预算一直在增加。在这种“新常态”下，数字体验将一如既往地保持至关重要的接触点。事实上，新的调查结果显示，对于 85%的用户来说，数字应用程序对于他们的日常生活是绝对必要的。

日益增长的数字需求意味着对云的依赖将会持续。但是，如果组织没有正确调整其云工作负载的规模，费用可能会失控。此外，维持世界一流的用户体验需要对应用、系统和解决方案的整体性能有敏锐的洞察力。如果团队没有持续监控应用程序性能和业务 KPI，底线可能会受到影响。

我最近会见了 AppDynamics 区域 with Joe Byrne，以了解全栈可观测性(FSO)对于当今的云计算应用意味着什么。根据 Byrne 的说法，DevOps 和 [SRE 团队](https://devops.com/how-the-sre-role-is-evolving/)应该维护应用程序整个脉冲的整体视图，包括非功能性指标(NFM)以及传统的安全性和功能性监控。这样做可以减少花在根本原因分析上的时间，并有助于确保代码持续满足业务目标。

## 什么是全栈可观测性(FSO)？

在经典定义中，可观测性侧重于根据外部输出测量系统的内部状态。应用性能监控(APM)通常围绕日志、指标和依赖关系跟踪遥测。例如，IBM 将可观察性趋势称为“APM 数据收集方法的自然进化”

Byrne 说，全栈可观测性(FSO)进一步跟踪业务相关的关键性能指标(KPI)以及其他更传统的软件指标。Byrne 说:“FSO 帮助理解软件正在以最佳状态运行，并且它正在满足业务目标。通过这种方式，FSO 不仅从整体上判断应用程序的功能是否健全，还判断它是否达到了最终目标。

FSO 将考虑关键的性能行动，如用户体验遥测。例如，Byrne 解释说，电子商务网站可以发现跟踪性能水平的价值，如登录时间、结账时间和支付启动。Byrne 说，跟踪这些指标对于查明障碍和创建可操作的步骤来补救用户旅程中的漏洞是必要的。提高可见性还可以帮助工程师快速响应问题，从而更好地优化用户体验。此外，在更高层次上汇总信息(例如，查看每天售出的总产品或每天的总收入)可以了解总体进度。

## FSO 如何减轻 IT 压力

Byrne 说:“我们已经到了这样一个地步，人们已经离开了实体建筑，转而使用网络。这些在线空间比以往任何时候都更无处不在，也更有价值。AppDynamics 2021 应用关注指数发现，普通数字消费者每天使用 42 个应用。此外，85%的人现在表示，数字服务是他们生活中的一个重要组成部分。

日益增长的数字依赖导致了数字复杂性的增加。因此，Byrne 说，公司已经转向云，以获得可扩展性、弹性和云功能的能力。无服务器、Kubernetes、身份和访问管理(IAM)等云原生功能以及其他附加服务有助于避免重新发明轮子。

但是，当然，将您当前的所有架构升级并转移到云是昂贵的；另外，它不一定能消除应用程序的错误。用户之旅越来越漫长，途中接触点的中断可能会影响性能。越来越多的第三方依赖使这变得复杂。然而，Byrne 解释说，通常是应用程序的自定义代码有问题。

也许全栈可观测性的最大好处是减少补救时间。Byrne 说:“如果你没有观点，可能会花很长时间来解决问题。“寻找根本原因的成本可能非常昂贵。”快速了解网络、应用程序和基础架构故障对于快速解决问题和节省潜在的收入损失非常必要。

## 性能监控左移

亚马逊的一项研究发现，每 100 毫秒的延迟会让公司损失 1%的销售额。了解用户高期望的现实后，软件公司必须特别主动地调整规模以满足消费者的需求。由于 72%的用户认为应用程序问题是品牌的责任，公司有责任避免负面的品牌声誉。

为了尽早发现性能障碍，Byrne 鼓励对业务成果的可观察性持左倾态度。迄今为止，左倾口号更多地关注代码验证和安全性——快速部署软件更新的必要元素。现在，添加性能测试可能是下一个迭代，以进一步将每个软件推送与最终业务结果相关联。

Byrne 说:“在生产之前，你需要尽可能多的观察和验证。"性能测试是一个应用程序整体成功的关键部分."

这可能涉及测试特定端点的加载时间——如果一个关键动作没有达到某个阈值，那么它就不会被提升。将性能监控纳入 CI/CD 流程是一种新兴的实践，也称为 [BizDevSecOps](https://devops.com/seeing-the-bigger-picture-with-bizdevsecops/) 。

## 没有回头路了

新常态正在向我们走来，很难想象我们会回到一个对技术依赖减少的世界。从机票预订到银行和杂货订单，软件应用程序现在几乎渗透到了现代生活的方方面面。为了在这样的经济中竞争，不断优化业务成果的微调[模块化客户体验](https://devops.com/digital-customer-experiences-the-future-is-modular/)将真正使公司脱颖而出。

简而言之，为了保持高软件标准，打破操作孤岛并作为一个团队一起工作是至关重要的——devo PS 信条。端到端的生态系统可见性有助于实现这一点，用 Byrne 的话说，就是确保“应用程序以满足业务需求的方式运行。”
