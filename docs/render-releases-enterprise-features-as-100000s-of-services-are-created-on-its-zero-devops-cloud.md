# Render 发布了企业功能，因为在其 Zero DevOps 云上创建了 100，000 个服务

> 原文：<https://devops.com/render-releases-enterprise-features-as-100000s-of-services-are-created-on-its-zero-devops-cloud/>

*自动缩放和读取副本使开发人员能够构建对繁重的工作负载和流量峰值更具弹性和响应能力的应用。*

**加利福尼亚州旧金山，2021 年 6 月 17 日—**Zero devo PS 云 Render 今天宣布全面推出自动缩放功能，这是客户最需要的功能。自动扩展提供了根据不断变化的工作负载级别和流量自动扩展应用程序的能力，因此客户可以提高服务的可靠性和正常运行时间，改善整体终端用户体验，同时控制成本。

今天，Render 还宣布了读取副本的全面上市，这使得数据库或分析使用率较高的客户可以将处理负载转移到主数据库的完全托管的实时副本上，从而使主数据库始终保持响应。

Render 创始人兼首席执行官阿努拉格·戈埃尔(Anurag Goel)表示:“开发者现在已经在 Render 上创建了数十万个服务，在过去的 6 个月里增加了一倍多。我们平台的快速采用不断激励我们为客户带来最先进的功能，而无需他们进行任何额外的工程投资。最终，自动缩放和读取副本等功能可以帮助开发人员提高应用程序的可靠性，控制云成本并提高运输速度，从而为用户提供更好的产品。”

Render 提供了开发人员喜爱的创新公共云替代方案；它为他们提供了两全其美的优势:即时设置和难以置信的易用性，以及以前只有大型云提供商才能提供的功能和灵活性。它通过提供一个直观的托管平台来实现这一点，该平台不需要任何服务器和基础架构知识，并将其与强大的容器化技术相结合，从而实现大规模定制、成本降低和可靠性。

**自动扩展**
自动扩展可用于平台上运行的任何 web 服务、私有服务或任务处理器，并具有内置的便捷仪表板，可配置和查看自动扩展事件和工作负载性能。Render 是第一个提供基于 CPU 和内存使用的本机自动扩展的平台即服务。与将自动缩放限制于昂贵的企业计划的平台不同，自动缩放免费提供给所有渲染客户；仅渲染实际计算使用的费用，按秒按比例计算。

BetaList 和初创公司 Jobs 的创始人马克·科尔布鲁格(Marc klbrugge)说:“Render 上的自动缩放比我以前用过的任何东西都好得多；我们将多个服务从 Heroku 转移到 Render，在 Heroku 中，自动缩放仅在更昂贵的计划中可用，并且依赖于应用程序响应时间。当性能瓶颈在应用程序之外时，例如在数据库中，这通常会使事情变得更糟。Render 查看我的服务的实时 CPU 和内存消耗，因此除了 web 应用程序之外，我还可以将它用于后台工作人员，并在我的所有服务中控制成本。”

渲染时的自动缩放基于精确的实时计算和内存使用，而不是像应用程序响应时间这样的指标，后者通常是实际资源消耗的不准确近似值。这不仅使渲染自动缩放显著地更加灵敏和有效；它还支持其他平台即服务(PaaS)提供商不支持的用例。其中包括响应时间差异很大的应用程序的自动伸缩，或者不支持 HTTP 流量但仍需要基于负载进行伸缩的应用程序，如异步队列处理任务。

**读取副本**
PostgreSQL 读取副本为使用 Render 完全托管的 PostgreSQL 数据库的客户提供增强的性能和恢复能力。使用读取副本，用户只需单击一个按钮，就可以创建生产数据库的只读副本，并且 Render 会自动使该副本始终与主数据库保持同步。客户可以使用读取副本无缝地执行业务关键型功能，如数据分析、报告、迁移和升级，而不会影响其生产数据库的性能或可用性。
