# 释放公民开发者的 7 个步骤

> 原文: [https://devops.com/释放你的公民开发者的 7 个步骤/](https://devops.com/7-steps-to-unleashing-your-citizen-developers/)

随着数字化转型的加速，组织正在寻找各种方法，使其所有员工(不仅仅是 IT、开发或 DevOps 团队)能够根据自己的要求设计、构建、迭代和推出新的流程和应用。这种民主化的方法被称为[公民发展](https://www.gartner.com/en/information-technology/glossary/citizen-developer#:~:text=A%20citizen%20developer%20is%20an,or%20function%20other%20than%20IT.)—业务用户被授权创建自己的流程和应用程序，只需最少的 IT 监督，最终将他们从 IT 积压中解放出来。

## **如何开始公民发展**

运行公民发展项目并不意味着将其边缘化。由于缺乏编程背景，公民开发人员经常容易出错。有限的技能也可能导致创建一个有缺陷的应用程序，或者不知不觉地开发一个类似于现有软件的具有相同用途的应用程序。

因此，公民发展需要一种新的关系方法，这种方法建立在 IT 专家和非 IT 员工之间的透明、信任和对话的基础上。

作为一名商业领袖，你是发起这样一个项目的最佳人选。以下是你可以开始的步骤:

**1。识别公民发展潜力**

您可以跨部门进行调查，或与部门负责人会面，以了解谁在使用各种形式的影子 IT 或已经在使用公民开发工具，他们使用这些工具的目的是什么，以及这些应用程序的用户数量。您的发现可以帮助您评估哪些部门拥有最多的公民开发者候选人。

此外，确定您公司积压的 IT 请求和当前的 IT 难题。了解被压抑的需求会让你知道公民开发者可以寻求解决什么问题。

**2。设定明确的目标**

传达公民发展计划旨在为您的组织实现的目标。确定您打算创建的应用、部署模式、[公民开发者](https://devops.com/?s=citizen+developers)的角色以及哪些部门将从中受益。

**3。获得 IT 和其他业务领导的支持**

在 IT 团队和其他经理中寻找可以帮助您实现公民发展愿景的合作伙伴。

首先，接近将与你合作的潜在经理。尽早获得他们的支持可以帮助你建立一个更全面的愿景，并避免未来的任何冲突。通过帮助他们了解这将如何实际促进他们部门的工作流程，而不是延迟内部任务，你将能够为你的提议赢得信誉。

同时，向您的 IT 团队保证，公民开发人员和 IT 人员之间将有明确的责任划分。在被允许进入发展环境之前，员工将接受自我指导和小组培训。软件开发生命周期也将被记录下来，以使公民开发者了解提交他们想法的过程和必要要求。此外，IT 开发人员仍将负责提供文档，评估想法，并通过定期检查提供技术指导。

**4。根据您的业务需求筛选低代码/无代码平台**

最好是寻找低代码平台提供商，安排演示，并在专门会议期间邀请 It 和业务部门领导。他们可以和你比较特性集，评估学习曲线。

**5。制定治理计划**

治理计划可确保数据安全最佳实践到位，并在您的公民发展计划中明确定义 IT 援助。这一计划应包括:

*   将项目纳入公民开发人员计划或 IT 计划的评估过程
*   为公民发展接受的任何项目的标准符合性和安全性要求
*   标准工作流程协议，包括测试、验证、审计和集成
*   哪些情况需要 IT 部门的参与

**6。获得高级管理层的认同**

您需要与您的高级主管沟通您的意图，并准备正式提交您的公民发展计划。

*   建立计划背后的基本原理。将内部指标与显示当前对公民开发者需求的行业统计数据进行比较。引用成功的故事或案例研究。
*   分享指出当前 IT 积压的内部数字，以及与员工和同事经理的调查和讨论结果。
*   计算并展示贵公司使用低代码/无代码平台进行公民开发的一至五年投资回报率。
*   介绍您提议的治理计划和试点项目。

7 .**。开始训练**

应该给公民开发者时间来学习所选择的低代码/无代码平台。您可以稍后选择一个用例来试验这个工具。确定你的期望和衡量项目成功的标准。
