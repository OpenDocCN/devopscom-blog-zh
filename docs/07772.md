# Catchpoint 宣布对谷歌核心网络要素的本地支持

> 原文：<https://devops.com/catchpoint-announces-native-support-for-googles-core-web-vitals/>

*为数字体验管理提供最准确、可操作和全面的指标；改善用户体验和搜索排名*

**(美国纽约州)**–**2021 年 4 月 27 日—**[数字体验监控(DEM)领域的领导者 Catchpoint](https://www.catchpoint.com/) 今天宣布全面支持谷歌的核心网络重要指标，为公司提供可行的指标，以提供最佳的网络用户体验。结合 Catchpoint 的高级数字体验指标和广泛的监控优势，对核心 web 生命周期的支持为 IT 团队提供了最准确、简单和全面的指标，用于优化 Web 性能以及随后的搜索排名。

有了这一新功能和 WebPageTest 中现有的核心 Web 生命体征支持，Catchpoint 现在提供了最完整的核心 Web 生命体征洞察套件。负责 web 性能的所有利益相关者，从前端开发人员到云架构师、网络工程师、sre 和 IT 运营从业者，都可以利用 Catchpoint 的平台来提供最佳的网站和 web 应用程序性能，取悦用户并推动积极的业务成果。

“对于任何通过网络做生意的人来说，核心网络要素将变得越来越重要。Catchpoint 首席执行官迈赫迪·达乌迪说:“今年夏天，网站的核心网站生命指数得分产生的‘信号’将在谷歌搜索算法中发挥更大的作用。“团队可以利用一组可操作的指标来为用户提供更好的体验，同时提高他们的业务绩效。令人兴奋的是，我们终于有了一套指标，可以让企业将 It 和业务绩效结合起来。”

**博客:**

核心网页活力由三个特定的页面速度和用户交互指标组成:最大内容绘画(LCP)、首次输入延迟(FID)和累积布局偏移(CLS)。这些指标是谷歌“页面体验”得分的一部分，这是谷歌官方搜索引擎排名因素，从今年开始。这对于业务可见性、网络流量以及最终的销售至关重要。

让这些指标变得简单可行的是时间的度量。LCP 是从用户点击链接到他们在屏幕上看到主要内容的时间。许多其他页面速度指标并不代表用户打开网页的感觉。FID 是用户与你的页面进行实际交互所花费的*时间*。这一点非常重要，因为它考虑到了网页开始响应用户的速度。CLS 是页面加载时的稳定性。当页面完全加载时，用户不必重新学习链接的位置。

除了显示测得的 LCP、总阻塞时间(TBT 是 FID 的实验室补充)和 CLS 值，WebPageTest 还提供了“幻灯片视图”，这是一个页面加载的动画预览，可以选择突出显示布局变化。

Blue Nile 的 QA 和发布经理 Robin Schenck 表示:“我们的 web 性能监控计划最重要的方面是我们的网站或应用程序呈现给客户的速度有多快。“Catchpoint 为我们提供了一套最全面的指标来衡量绩效并立即采取行动。”

Catchpoint 的数字体验监控平台中包含了对核心网络生命周期的支持，可立即使用。
