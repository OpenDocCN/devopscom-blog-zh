# 分布式系统:将网络变成你的优势

> 原文:[https://devo PS . com/distributed-systems-turning-networking-to-your-advantage/](https://devops.com/distributed-systems-turning-networking-to-your-advantage/)

无论是办公室、教室、商店还是银行，人类生活的许多领域都已经转移到了网上。每天，我们变得越来越习惯于按需消费应用程序，这些应用程序可以立即将我们与世界其他地方联系起来。我们对互联网日益增长的需求依赖于分布式系统的持续趋势，如今大多数公司都将其信息技术传播到由小型、独立和松散耦合的容器组成的混合和多云架构中。公司通过在不断扩大的表面上部署更多的容器集群来扩展它们的应用。从分布式系统的激增中，企业可以观察到四个关键的教训。

### 不要低估分布式系统的好处

分布式系统的趋势持续存在是有原因的；他们有很多好处。

分布式系统具有难以置信的可扩展性，因为它们允许公司通过在必要时添加、替换或删除节点来扩展应用程序。新的应用程序可以通过多次迭代快速推出，从而加快上市时间。代码库更小，更容易更改。分布式系统优化了开发人员的效率和团队协作，因为许多团队可以同时处理系统的不同部分。

分布式系统具有很高的弹性。当(也只有当)有效构建时，分布式系统将继续工作，即使很大一部分关闭，因为流量可以重新路由到正常运行的服务器——只要看看互联网的弹性就知道了。单点故障越少，稳定性和容错能力就越强。

有了这么多好处，不难看出分布式系统是如何实现我们的互联数字经济的。网飞是一个很好的例子，展示了他们如何支持 SaaS 应用程序的发展。由于一个复杂的分布式系统，地球上的任何人都可以访问一个巨大的电影、电视节目和纪录片库，按需播放他们想要的任何内容，只需支付一个月的订阅费。

虽然网飞被认为是在娱乐行业，但我们认为它首先是一家技术公司，因为其商业模式的成功是基于其分布式系统的功效。网飞的平台是可扩展的、有弹性的、高性能的、高效的，因此是有利可图的，因为它是分布式的。

### 要知道分布式系统是复杂且具有挑战性的

分布式系统天生复杂，因此难以构建和操作。它们由许多必须协同工作的异步组件组成。随着系统的扩展，理解变得更加困难，更不用说操作了。

这些挑战并非网飞独有。任何大规模管理分布式系统的公司都会因为分布式系统错综复杂的本质而遭受损失。Spotify、YouTube 和 Twitter 等其他互联网巨头也在努力应对日益复杂的问题。如果您被您的分布式系统的迷宫所淹没，知道这些挑战(在一定程度上)是不可避免的就可以了。

### 将网络变成你的优势

分布式系统是一组连接到网络的计算机、容器或任何其他节点，通过传递消息与其他实体进行通信。真正允许分布式系统“分布式”和运行的是网络。这使得网络既是一种负担，也是一种机遇。是的，即使是最好的系统也会在不同层面的某一点上失败。在分布式系统中没有完全控制这种东西，但这并不意味着你不能将网络变成你的优势。

开发人员可以通过区分网络的优先级和在整个架构中使用正确的工具来有效地管理他们的分布式系统。网飞已经建立了异常良好的连接系统，服务器部署在全球数千个位置，连接到云中的控制平面。如果一台服务器出现故障，流量可以很容易地被重新路由到其他地方。分布式系统是可扩展的、弹性的、高性能的和高效的。强大的网络解决方案让网飞团队能够了解并控制应用。

联网确实是增强分布式系统的优势和克服其挑战的关键。在你的网络上投资资源，它会照顾你的分布式系统。

### 以简单为目标

在构建网络堆栈时要小心谨慎，将网络转化为您的优势。这里要避免的两个最容易预测的陷阱是，不要选择解决您的独特挑战和竞争优势的堆栈，以及选择对于您的使用情形来说过于复杂的工具。

对于构建和管理分布式系统的公司来说，在没有充分考虑它们的适用性的情况下，采用它们的编排平台附带的流行或默认工具是很常见的。假设对其他人有效的东西对你也有效，复制粘贴其他地方的网络堆栈从来都不是一个好主意。可能会，但不要盲目选择大家都在谈论的堆栈，而不给予他们太多的思考。

许多组织通过采用 Istio 随波逐流，只是后来发现服务网格对于他们的平台来说过于复杂和资源密集型。虽然 Istio 的众多功能对一些人很有帮助，但它们很快就变成了负担。趋势分析工具不一定最适合您的用例。

一些公司走相反的路线，建立自己的定制解决方案，而不是随大流。正如网飞为服务发现构建了 Zookeeper，Spotify 为 API 管理编目构建了 Backstage 一样，拥有庞大分布式系统的全球公司可以从定制工具提供的控制中受益。这种方法既昂贵又耗时，对于非技术型或小型公司来说不可行。

你(可能)不是网飞，所以不要试图成为。你可能认为你有同样的问题，但你没有。是的，分布式系统总是复杂且充满挑战。但是分布式系统的复杂性在规模上复合到了无与伦比的程度。像网飞和 Spotify 这样的互联网巨头需要内部构建的复杂工具或定制解决方案的高级功能(他们有办法投资这些资源)。如果你的平台只是网飞的一小部分，你的分布式系统依赖于一个简单得多的网络。

选择针对您的用例过度设计的解决方案非常容易，但是最好选择更简单的替代方案。从简单的解决方案开始，获得对网络堆栈的控制，然后逐步提高到另一个复杂级别。获得对网络的控制将允许您利用分布式系统的好处，同时避免一些挑战。通过选择简单而非复杂，将网络变成您的优势，并使您的基础架构可扩展。
