# 在 Web 开发项目中使用内部 Wikis

> 原文：<https://devops.com/using-internal-wikis-for-web-development-projects/>

基于团队的 web 开发项目总是一项复杂的工作。它涉及具有不同经验水平、不同专业领域、要完成的里程碑和要实现的时间表的不同开发人员。

为了促进信息共享，仅仅使用像 Asana 或吉拉这样的项目管理工具是不够的。有时，可能需要一个 wiki 来 [帮助将知识](https://tettra.com/article/knowledge-management-process-101/) 和最佳实践分发给项目中的每个人。

在本指南中，我们讨论了使用 wiki 工具在 web 开发人员之间共享信息的实用性。

### 什么是维基？

wiki 是一个程序，它使用户能够协作创建和填充网站内容。这个词来自“维基”，在夏威夷语中是“快”的意思。换句话说，维基是一个允许众包内容创建信息页面的应用程序。

除了 wiki 页面，Web 2.0 工具允许用户通过 RSS、OpenSocial 和 OpenID 等富媒体连接和共享信息。wiki 工具的另一个用途是分享创意内容，如摄影和设计概念。在 wiki 上共享网页的一个非常简单的方法是在页面顶部添加一个共享图标。只需将共享图标添加到维基页面。

### 维基的好处

Wikis 给 [web 开发](https://devops.com/?s=web+development)团队带来了许多好处，例如:

**知识分享**

由于从事该项目的人们具有不同水平的专业知识和经验，wiki 允许主题专家根据他们的专业贡献信息。需要这种信息的其他人可以访问这种有价值的知识，并消除“信息孤岛”的形成，这种信息孤岛通常会阻碍团队彼此孤立地工作。

**可搜索**

存储在 wiki 中的信息按照层次和逻辑结构进行分类和组织。它允许用户根据主题、领域或类别轻松搜索信息，并在瞬间了解他们需要什么。这提高了工作效率，并最大限度地减少了在搜索数据时追赶其他同事的需要。

**真相的单一来源**

作为一个中央信息库，所有数据在进入维基之前都要经过审查。这使得存储的信息得到验证和准确，并且 wiki 充当整个团队的唯一真实来源。

**轻松更新**

wiki 允许贡献者轻松方便地更新信息。因此，所有的参考都是最新的——这是任何 web 开发项目的关键因素。

### 维基和网络开发

Wikis 有可能使与利益相关者共享知识变得更容易，并传播有动力的开发团队的专业知识，从而导致更好的管理和责任。

许多维基以不同的方式得到公共资助。例如，有几个 [开源](https://securityboulevard.com/2022/08/the-4-best-open-source-pki-software-solutions-and-choosing-the-right-one/) 维基，如 BikiWiki、ObjectiveWiki、GrowWiki 和 PDFLabs wiki，但我们也可以在这个领域看到一些基于网络的倡议。其中包括维吉尔技术 wiki 和 KnowledgeVerify。

微软的 SharePoint 平台可以说是网络项目在线协作的最佳解决方案之一。因为 SharePoint 集成了几乎所有的主流应用程序(Windows、Word、Excel、PowerPoint 等。)，用户可以基于 SharePoint 网站模板创建共享网页，并与他们的同事共享这些网页。虽然 SharePoint 确实不能解决与信息共享和保存相关的所有问题，但它为寻找项目开发的共同点提供了一个很好的起点。

### 维基相对于其他协作工具的优势

虽然项目管理工具通常包含协作功能，但在知识共享方面，它们并不像 wiki 那样功能齐全。wiki 因具有其他工具可能缺乏的以下特性而引人注目:

**搜索功能:**维基拥有全功能搜索功能，有助于信息搜索。不需要深入树形结构和分类层次。

**评论:**给页面添加评论是一种高效快捷的贡献和帮助他人的方式。

内容创建:你可以使用任何文本编辑器编写维基文章，也可以使用维基语法编写——这很大程度上取决于平台。wiki 语法与 HTML 格式非常相似。让维基有用的一点是，当你共享一个维基页面时，每个人都能看到所做的修改。因此，共享是通过网络链接进行的，而不是发布到文件中。

wiki 标记是代表页面结构的一系列标签。wikitext 有两种主要类型；维基语法和 HTML 语法。维基语法来自维基百科，这是一个流行的维基。开发人员可以立即使用这两个工具，使内容创建和编辑更加容易。

**访问控制:**由于维基可以根据编辑和添加到页面上的链接数量而改变，所以他们使用访问控制和版本管理系统。

随着软件选项的增加和框架决策的不断变化，web 开发的世界总是需要快速的、有组织的信息补丁。快速的创建和发布选项意味着，对于一项有近 20 年历史的技术来说，维基仍然是当今 web 开发人员的一个有价值的工具。
