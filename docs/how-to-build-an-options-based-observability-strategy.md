# 如何建立基于期权的可观察性策略

> 原文：<https://devops.com/how-to-build-an-options-based-observability-strategy/>

全球化的疫情迫使企业加快数字化转型，在努力适应不断变化的世界的同时，将企业推向新的技术和架构。公司希望快速开发和部署，并乐于牺牲效率、可靠性，有时甚至牺牲安全性来追求目标。如今，在世界许多地方，疫情正在消退，随着企业努力应对在批评声中做出的选择，优先事项正在重新平衡。

可观察性已经成为企业更好地理解不断扩展的基础设施和应用环境的一种方式。[根据 Gartner](https://www.gartner.com/en/documents/3985892/monitoring-and-observability-for-modern-services-and-inf) 的说法，“可观察性使组织能够减少识别影响性能的问题的根本原因所需的时间。特别是，与传统监控相比，操作员可以自由地事后询问数据，而无需对仪表板进行预编程。”

这听起来很有希望，市场迅速做出了反应，但这是可以预见的。大型单片监控平台接受了这个术语，并添加了新的功能，使它们变得更大、更单片。这些单一平台代表了一种失败的基于选择的策略。他们假设所有的可观测性数据都是由一个工具收集、集中、集成和分析的。遵循基于选择的可观察性策略的企业发现自己被锁定在多年投资上，无法灵活应对不断发展的技术。

虽然这对现有监控领域的供应商来说是好事，但最终用户并没有从这些巨大的平台部署中受益，因为基于选择的策略建立在一个错误的假设上，即单一平台服务于从 SREs 到 DevOps 和 SecOps 的所有监控和可观察性受众。

随着技术的不断发展和需求的不断变化，基于选择的策略也迫使客户在未来选择有限。默认基于选择的策略也使得合同谈判、数据集成和数据访问的杠杆作用很小。

基础设施和运营团队需要采取不同的方法来实现他们的[可观察性](https://devops.com/?s=observability)策略。团队需要将他们的思维转移到创造选项上，而不是专注于购买产品和做出选择。你今天的选择会限制你明天的选择。明天是不可预测和不确定的。采取基于选项的方法可以让你在现在和将来都具有灵活性和适应性。

基于选项的可观察性策略建立在三个支柱之上:

*   建立你的可观察性策略原则
*   构建可组合环境
*   履行你的承诺

让我们来看看每一个。

### 建立你的观察策略的原则

当被问及他们的可观察性策略时，许多人通常会跳到涵盖所有可以想象的用例的详细计划。他们谈论其基础架构的当前状态以及三年或五年后的情况，包括他们在此期间需要的技能和功能。如果你已经阅读了以上内容，你会意识到这是一个基于选择的策略:他们在未来几年需要做出的所有选择都已经被安排好了。随着你周围世界的发展，这给适应和选择留下了很少的空间。

相反，把你的可观察性策略想象成一个单一的原则，或者原则的集合，来指导你的可观察性策略。假设您的核心原则是可观察性是跨 sre 和 DevOps 的运营效率的关键促成因素。我们可以在运营效率原则的基础上定义我们的战略必须包括哪些关键要素:

*   数据源具有类似实用程序的可访问性；始终在线并可跨工具使用。
*   指标主要由总拥有成本和优化驱动。
*   治理被集中到向下游系统分发数据的团队。

如果你的核心原则是客户亲密度或风险管理，这些要素就不同了。

有了原则和策略，我们可以专注于下一个阶段:构建可观测性环境。

### 构建你的观察环境

在基于选项的策略中，关键的成功因素是避免限制灵活性的大型生态系统。许多供应商对可观察性采取“给我们一切”的方法。这些方法似乎始于目的平台，也终于目的平台，但它们甚至更早地始于数据源。常见的情况是代理只将数据发送到一个目的地，通过将数据锁定在一个竖井中来限制数据的有用性。

保持模块化意味着选择从目的地提取数据源的工具，并将权力放回到您的手中。通过使用像可观察性管道这样的抽象，您可以将数据从一个源发送到多个目的地，同时以您认为合适的任何方式管理这些数据。例如，这是您的集中治理可以发生的地方。

当引入、升级或废弃新工具时，可观察性管道将您的环境和用户与这些变化隔离开来。模块化方法还可以让您在与其他供应商的合同谈判中拥有更大的权力，因为您不再那么依赖一家供应商来满足您所有的可观察性需求。

### 履行你的承诺

基于选择的可观察性策略是一个巨大的承诺，通常需要数年时间。一旦开始，就不可能停下来。在当今不确定的环境下，这是完全不可接受的。基于选择的策略限制了你测试、实验和学习的能力。与此同时，我们的基础设施至关重要，必须可靠。

保留选择的方法可能包括尝试免费增值产品；只有在产品的价值被证明后才付款。基于云的选项是另一个考虑因素，因为它们通常基于订阅，并提供简单的先试后买的购买风格。更好的是，基于云的选项不需要使用您自己的基础设施来测试新选项。

每个 IT 团队都觉得有必要对业务需求做出反应和响应。这种做出选择、解决眼前问题的紧迫性会对未来的灵活性和敏捷性产生长期影响。通过在保留选项的基础上围绕长期愿景构建您的可观察性策略，您的团队可以随着技术和业务需求的发展而保持适应性。
