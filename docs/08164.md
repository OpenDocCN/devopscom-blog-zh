# PagerDuty 与 JFrog 合作，实现 IT 事件管理的现代化

> 原文:[https://devo PS . com/page duty-alliances-with-jfrog-to-modernize-it-incident-management/](https://devops.com/pagerduty-allies-with-jfrog-to-modernize-it-incident-management/)

PagerDuty】将其 IT 事件管理平台与 JFrog 的持续集成/持续交付(CI/CD)平台相集成，作为简化应用环境故障排除工作的一部分。

PagerDuty 战略业务开发高级总监史蒂夫·格罗斯(Steve Gross)表示，总体目标是让 it 运营团队和开发人员更容易合作发现问题的根本原因，例如，能够识别 JFrog 管道中可能对生产环境产生负面影响的构建。

同时，使用 PagerDuty 的 IT 团队现在还可以接收来自 JFrog Xray 的通知，JFrog Xray 是一种发现开源软件漏洞和许可证合规性问题的工具。

各种规模的组织都试图通过深入了解应用程序的构建方式来最大限度地减少 IT 事故的影响。目标是减少 IT 团队目前花费在寻找问题根源上的时间。

![PagerDuty JFrog](../Images/24a01b1d9e7ad2f37c1e7d1ac388612b.png)

理想情况下，IT 团队应该在这些问题影响最终用户之前发现它们。他们面临的挑战是，发现一个问题的根本原因可能需要几周时间，而开发人员最多只需要几个小时就可以修复。Gross 说，借助 CI/CD 平台的洞察力，it 运营团队可以更容易地主动查明问题的根源，为开发人员提供足够的洞察力来采取行动。

这一点至关重要，因为通常情况下，在生产环境中部署的应用程序的最后一个版本通常是问题的根源，尤其是在该版本升级到生产环境之前，it 环境运行正常的情况下。

page duty 从各种 DevOps 平台收集的数据通过变更事件应用程序编程接口(API)输入到 page duty 工具中。格罗斯指出，随着时间的推移，PagerDuty 还将能够应用机器学习算法和其他形式的人工智能(AI)，在问题达到有意义的破坏水平之前很久就发现问题。

总的来说，有一种趋势是以符合 DevOps 最佳实践的方式实现 IT [事件管理](https://devops.com/?s=incident+management)的现代化，这是在实际部署应用程序之前发现问题的努力的一部分。希望这些努力将减少部署应用程序后为确定性能或安全问题的根本原因而需要召开的“作战室”会议的数量。

It 事故管理流程现代化到这一水平可能还需要一段时间。然而，随着 it 环境变得越来越复杂(部分原因是现在部署了大量基于微服务且具有多种依赖关系的应用程序)，IT 事件管理的现有方法显然无法让 IT 团队对问题做出足够快速的响应。事实上，这比以往任何时候都更具挑战性，因为 IT 团队的许多成员更经常在家工作。

IT 事件管理现代化的压力几乎与 IT 团队的压力水平成正比。现在的挑战是在 IT 团队成员筋疲力尽而无法欣赏这些努力之前实现这些过程的现代化。
