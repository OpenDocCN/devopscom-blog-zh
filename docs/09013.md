# Testlio 将融合测试添加到自动化测试平台

> 原文:[https://devo PS . com/test lio-adds-fused-testing-to-automate-testing-platform/](https://devops.com/testlio-adds-fused-testing-to-automate-testing-platform/)

Testlio 今天宣布，它正在为其应用测试平台增加融合测试功能；IT 组织使用 Testlio 来管理由内部员工和外部承包商组成的应用程序测试团队。

[融合测试](https://devops.com/?s=testing)描述了 IT 组织用来提高整体生产力的自动和手动测试的组合。融合测试在自动化测试和仍然需要人工执行的测试之间取得了平衡。

为了实现这一目标，Testlio 与 Applitools、BrowserStack、Headspin、Mabl 和 Sauce Labs 合作。来自这些工具的失败信号和会话工件可以被推送到 Testlio 平台，以简化测试结果的分类。Testlio 在不增加成本的情况下，每月提供 6 个设备插槽和 100 小时的自动化测试处理时间。

最后，Testlio 建立了一个自动化测试卓越中心(CoE ),作为其专业咨询服务扩展的一部分，以帮助组织实施测试最佳实践。

Testlio 首席执行官 Steve Semelsberger 表示，这些集成将使增加人工测试人员变得更加简单，并提高应用程序的测试速度，因为许多组织正在加快这些应用程序的构建和部署速度。

他补充说，随着组织寻求在 DevOps 工作流的上下文中实施连续测试，显然需要更多地依赖自动化来跟上应用程序开发的步伐。

通过其平台，Testlio 为组织提供了 10，000 多名自由职业应用测试人员。该测试专业人员网络可以使用用 Python、Java 和 Node.js 等编程语言编写的 Appium 和 Selenium 框架来创建、运行、优化和诊断自动化测试。Semelsberger 指出，申请加入 Testlio 网络的自由职业应用测试人员中只有 3%被接受。

组织更喜欢依靠内部员工还是承包商来测试应用程序的程度自然会有所不同。然而，很难预测需要执行多少测试——以及应该何时执行——以保证应用程序开发时间表。Semelsberger 说，承包商为 IT 组织提供了一定程度的测试灵活性，鉴于当前雇用和保留专业应用测试人员的挑战，这种灵活性正成为一种绝对的要求。

当然，不管有多少测试人员，应用程序测试都很难统一应用。除了提供有经验的自由测试人员，Testlio 还提供了一个管理整个测试过程的平台。接下来的挑战是将这些测试任务与组织已经启动的各种应用程序开发项目结合起来。

不管测试的方法是什么，对它的需求从来没有像现在这样重要。随着组织采用广泛的数字业务转型计划，现在有更高比例的应用程序面向外部。客户和合作伙伴对次优应用程序体验的容忍度远低于内部员工。因此，不管喜欢与否，第一印象将决定大多数应用程序计划的成败。
