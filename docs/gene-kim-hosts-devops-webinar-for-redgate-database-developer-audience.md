# Gene Kim 为 Redgate 数据库开发人员受众举办 DevOps 网络研讨会

> 原文：<https://devops.com/gene-kim-hosts-devops-webinar-for-redgate-database-developer-audience/>

*英国剑桥，2018 年 9 月 18 日星期二—*广受赞誉的 DevOps 作者兼研究员 Gene Kim 将于 9 月 25 日为 Redgate 数据库开发工具的用户举办一场网络研讨会。该网络研讨会将讨论 DORA 在 2018 年加速开发运维状态报告中的主要发现，该报告首次将数据库开发作为一项关键技术实践，可以推动开发运维的高性能。

《SQLServerCentral.com》的编辑史蒂夫·琼斯将加入吉恩·金的团队，该社区拥有 190 万数据库管理员、开发人员和 SQL Server 用户。他们将一起研究 DORA 报告中的关键发现，并调查数据库在成功开发运维中日益增长的重要性。

该报告是五年多研究的结果，拥有 30，000 多个数据点，旨在了解软件开发环境中的高性能，并确定可以预测高性能的因素。

今年纳入数据库是因为在执行部署时，对数据库的更改通常是风险和延迟的主要来源。该报告的作者，包括 Jez Humble 和 Nicole Forsgren 以及 Gene Kim，想要调查在实现连续交付时哪些数据库相关的实践有助于提高软件交付性能和可用性。

他们的结论是，将数据库工作集成到软件开发过程中积极地有助于持续交付，这与 Redgate 的一些早期研究产生了共鸣。2018 年 Database DevOps 调查显示，76%的应用程序开发人员现在也负责开发数据库，其中 75%的人构建数据库部署脚本。

使用 DevOps 发布特性的速度更快，这促使了这一变化。这些前端特性通常也需要更新后端的数据库，因此将这两个活动作为同一个流程的一部分是有意义的。

网上技术交流讲座是一个独特的机会，可以看到 DevOps 领域公认的专家和 SQL Server 领域的知名人士聚在一起讨论应用程序和数据库开发之间的共同点，而不是差异。

Gene Kim 是一位获奖的首席技术官、研究员和作家，他是 Tripwire 的创始人和首席技术官长达 13 年，是《凤凰计划:一部关于 IT、DevOps 和帮助您的企业成功的小说》、《DevOps 手册》和《DevOps 加速状态报告》等书的合著者。

史蒂夫·琼斯是微软 SQL Server 的 MVP，从 OS/2 上的 4.2 版本开始，他就是 SQL Server 的拥护者。在多家公司担任数据库管理员和开发人员后，他于 2001 年共同创立了 SQLServerCentral.com，并从 2004 年开始担任全职编辑。

这个一小时的网上研讨会由 Redgate 主办，时间为 9 月 25 日(星期二)英国夏令时 17:00/美国东部时间中午 12:00/太平洋时间上午 9:00，与会者可以在 Redgate 网站或[在线注册](https://register.gotowebinar.com/register/2389527231883004930?source=RG+Website)了解更多信息。
