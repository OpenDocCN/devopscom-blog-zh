# 新思科技同意收购怀特哈特安保公司后开始行动

> 原文：<https://devops.com/synopsys-sets-course-after-agreeing-to-acquire-whitehat-security/>

在[同意以大约 3.3 亿美元的现金从 NTT 安全公司](https://www.prnewswire.com/news-releases/synopsys-to-acquire-whitehat-security-from-ntt-301534684.html)收购 WhiteHat Security 之后，新思系统公司计划在其软件即服务(SaaS)平台中添加动态应用安全测试(DAST)工具。

新思科技软件完整性集团总经理杰森·施密特(Jason Schmitt)表示[怀特哈特开发的 DAST 工具将补充新思科技目前提供的现有静态、交互和软件组成分析工具](https://digitalanarchist.com/videos/devsecops-topic/whitehat-acquisition-jason-schmitt-synopsys)。

他补充说，目标是提供一个端到端的应用安全平台，包括咨询，以帮助组织更好地保护软件供应链。NTT 安保公司此前于 2019 年收购了怀特哈特安保公司，以扩大其提供的服务范围。从那以后，用于测试应用程序安全性的 SaaS 平台变得越来越普及。

此次收购发生在许多组织试图确定将应用程序安全责任转移给开发人员的时候。虽然开发人员有自己的角色，Schmitt 说，将应用程序的责任完全转移到左边是一种谬误。他说，DevOps 团队将需要更多地依赖于提供护栏的平台，这些护栏使开发人员能够以不会产生大量额外摩擦的方式构建安全的应用程序。

不幸的是，针对应用程序的网络攻击在复杂性和数量上都在增加。Schmitt 说，这对应用程序开发人员、开发运维团队和安全专业人员来说都是一个挑战，他们需要跟踪应用程序面临的威胁是如何演变的。随着越来越多的公司依赖软件来推动数字商业计划，Schmitt 说，这些组织将面临的商业风险只会增加。他补充说，在一个开发人员远远超过安全专业人员的时代，应对这一挑战的唯一方法是更多地依赖 SaaS 平台，使组织能够在其现有的应用程序开发工作流程中实施最佳实践。

Schmitt 补充说，今天的挑战是，一个组织的大部分安全预算都分配给了网络和终端，而不是应用程序。目前还不清楚在多大程度上需要将支出转向应用程序安全性，或者是否需要将应用程序开发预算的更大一部分分配给安全性。然而，不管怎样，很明显，为了对抗日益频繁的攻击，在应用程序安全性上的总支出需要增加。

与此同时，随着更多的安全工具对开发人员和 DevOps 团队开放，开发人员和网络安全团队之间的历史鸿沟应该会缩小。否则，安全专家将继续指出可能影响在生产环境中运行的应用程序的安全问题，而开发人员将继续忽略他们认为不相关或他们没有时间修补的问题。不足为奇的是，在生产环境中运行的应用程序中的大多数漏洞仍然没有得到修补；唯一的方法是在部署应用程序之前依靠自动化来消除它们。
