# 日志管理的挑战随着可观察性的采用而增加

> 原文:[https://devo PS . com/log-management-challenges-grow-with-observability-adoption/](https://devops.com/log-management-challenges-grow-with-observability-adoption/)

看起来，在许多 It 环境中，可观察性变得太好了。Dimensional Research 代表日志数据管理平台提供商 Era Software 对超过 315 名 IT 高管、云应用架构师、DevOps 专业人员和站点可靠性工程师(SRE)进行了一项全球[调查](https://www.prnewswire.com/news-releases/era-software-2022-state-of-observability-and-log-management-report-reveals-perfect-storm-of-massive-data-growth-and-need-for-innovation-301504799.html)，发现 96%的人正在见证日志数据的爆炸式增长。

调查还发现，79%的受访者报告说，如果当前的工具不发展，可观察性和日志管理的总成本将在 2022 年飙升。

超过四分之三的受访者(78%)还指出，管理大量日志数据的尝试产生了混合或不想要的结果，例如无法访问数据。近三分之二(65%)的人也在评估他们的可观察性选项，而其他 41%的人正在考虑。

Era Software 营销高级副总裁 Stela Udovicic 表示，该调查清楚地表明，随着组织越来越依赖日志来分析 it 事件，存储所有日志数据的成本正成为一个重大挑战。总体而言，调查发现，随着 IT 团队努力管理日益复杂的 IT 环境，可观察性工具和平台的使用量跃升了 180%。

从好的方面来看，更多的业务用户受益于日志数据。整整 83%的受访者表示，IT 部门以外的业务利益相关方使用日志数据中的见解，96%的受访者表示日志数据正被用于解决业务问题。

[可观察性](https://devops.com/?s=observability)一直是 DevOps 最佳实践的核心原则，但实现它一直是一个挑战。监控工具旨在使用预定义的指标来识别特定平台或应用程序何时在预期范围内运行。例如，被跟踪的度量通常集中在资源利用上。然而，无论何时出现问题，仍然需要几天，有时是几周的时间，通过相当于排除过程的过程来发现问题的根本原因。

相比之下，可观察性将指标、日志和跟踪(一种特殊的日志记录形式)结合起来，以一种更简单的方式对应用程序进行检测，从而解决问题，而不仅仅依赖于预先定义的一组有限的指标来监控特定的流程或功能。DevOps 团队可以使用查询来询问数据，从而更容易发现问题的根本原因。可观察性平台以某种方式关联事件，使得分析工具更容易识别指示 it 问题的异常行为。

出现的问题是，IT 团队需要找到一种方法来存储他们需要分析的日志所生成的所有数据。当然，每个 IT 团队都可以减少他们保留的日志数据量。然而，人们总是担心在发生事故时，关键日志数据将无法使用。

Udovicic 说，大多数 IT 组织在历史上都认为存储日志数据是一种必要的罪恶。她指出，大多数人遇到的主要问题是日志数据不容易处理这一简单事实。

无论组织如何看待日志数据，随着开发运维工作流变得更加成熟，需要管理的日志数据量只会增加。IT 团队现在需要解决的问题是找到一种方法来最大限度地降低存储所有数据的成本。
