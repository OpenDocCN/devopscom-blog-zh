# 是的，你在做 DevOps

> 原文:[https://devops.com/yeah-youre-doing-devops/](https://devops.com/yeah-youre-doing-devops/)

开发人员的 Stack Overflow 年度调查[本月出炉，其中有很多有趣的数据点。我希望他们的问题/方法有很多不同之处，但有一件事引起了我的注意。Docker、Kubernetes 和类似的工具(例如 Ansible 和 Terraform 的比例较低)构成了比标题中带有“DevOps”的人所能解释的更多的回答。](https://insights.stackoverflow.com/survey/2021)

有时差距很大。

这意味着我们有相当数量的开发人员在开发，但是他们不是开发团队的成员。是的，受访者中也有一些系统管理员，但不足以掩盖差异。如果每个处于开发运维和系统管理员角色的受访者都使用 Kubernetes，那么就足以涵盖在过去一年中“广泛”使用 Kubernetes 的受访者人数。不太可能所有人都是 Kubernetes 专家。这并没有触及 Docker，多达 40%的受访者表示他们广泛使用 Docker。任何最近有经验的开发人员都可以解释 Docker——尽管“广泛”对于普通开发人员来说有点过了，但它正在成为开发的一部分，使用容器来托管开发环境、测试甚至构建工具。当然，还有一些部署。但是 40%还是很多。

因此，如果您的头衔是开发人员，并且您经常使用部署工具，那么就称自己为 DevOps 吧。这不是唯一的标准，但是如果您要选择一个，使用部署工具所代表的广度与没有运营人员担心这类事情的含义相结合，使“使用部署工具”成为一个很好的替代测试。

对于使用 Yarn 的近 18% (20%的专业开发人员)，我要脱帽致敬。我在它的早期使用了几年，很高兴地说我不再需要使用它了。我假设您的使用与 Hadoop 无关，并且它在我 2015-2016 年的体验中有了很大的改善。尽管如此，你们都很棒，仅仅是用它来安排资源。这不容易。

总的来说，DevOps/Ops 工具的数量在开发者调查中是惊人的——近 9%的受访者使用 Ansible，近 9%使用 Terraform。毫无疑问有重叠，所以使用一个*或*另一个的实际数量不到 18%,但仍然接近十分之一？那是巨大的。这是 DevOps 或 Ops，而不是纯粹的开发人员。不考虑头衔。

自从这个领域起步以来，这已经成为开发的一个标志——做交付软件所需要做的事情。这只是另一个循环，不管头衔如何，你们都是摇滚明星。你继续提供运行业务的应用程序——让他们爱怎么叫你就怎么叫你。金钱万能；标题几乎毫无意义。不是说你退休后会有头衔，而是奖金或股票奖励？那些你可能会退休。
