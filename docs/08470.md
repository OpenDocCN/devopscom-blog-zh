# Wix 需要可见性，而洞察力是关键

> 原文:[https://devo PS . com/wix-requires-visibility-and-insights-is-the-key/](https://devops.com/wix-requires-visibility-and-insights-are-the-key/)

DevOps 原则的关键要素之一是，您的环境必须为任何事情做好准备。如果您闪亮的现代交付链不支持采用新的 DevOps 工具，或者没有帮助您学习和迭代，那么它将最终落入与瀑布相同的解决方案类别，并且将走上一条清晰的遗留路径。

问题是，大部分的可持续性方法等同于，最后落得个灰头土脸。为了支持现代开发的移动目标，团队需要实现自动化分析，提供可见性并帮助他们响应和学习，而不需要基于手动拉动的学习。远远超出服务器或应用程序日志记录的分析。Wix 在六年前就意识到了这一点，并采取了非常独特的措施来解决这一问题。

我相信你听说过[](https://wix.com)。这是一个受欢迎的工具，让典型的企业主建立一个网络存在。Wix 对于采用现代的 [开发工具](http://stackshare.io/wix/scaling-wix-to-60m-users-from-monolith-to-microservices) 并不陌生。因此，当我有机会与他们谈论他们的 DevOps 环境时，我毫不惊讶地发现他们已经实施了一个全新的概念。Wix 为支持 DevOps 的关键方面做出了不懈的努力——通过建立专门的团队来获得战术洞察力。

大约六年前，Wix 成立了一个小型监控部门，并取得了巨大成功。在一些组织中，NOC 团队似乎与 Wix 有着相同的目标。但是，一个典型的 NOC 的运作方式是，一屋子的 IT 人员只关注生产环境的红色、黄色和绿色指示灯，这是一种非常被动的手动监控方法。Wix 监测小组的独特之处不仅在于他们的存在，还在于他们关注更广泛的渠道，而不是具体的数据。他们打算成为一种服务，而不是一个特定的解决方案。监控团队负责在其 300 多名开发人员中建立和共享一个分析工具库。你可以想象他们的图书馆已经大大增加了。

“我们使用许多监控工具。一些用于服务器日志，一些用于 APM，一些用于我们的应用程序框架，甚至内部工具。但它们是静态的，只能满足特定的需求，”Wix 监控架构师 Mark Sonis 说。所以为了获得全球知名度，团队决定采用一种叫做 [Anodot](http://www.anodot.com/) 的新工具。Anodot 是一款管道分析工具，最近刚刚脱离 隐形 模式。目标:为所有指标和系统提供一个中心位置，并利用 Anodot 拥有的一个关键特性，即能够将数据关联到所有系统的洞察中，以获得全面的可见性。

Anodot 并不是唯一属于这一类的工具。 [Moogsoft](https://www.moogsoft.com/) 和可以说[DataDog](https://www.datadoghq.com/)正在正面解决可见性问题。但是 Anodot 的承诺是远远超越基础设施，进入所有触及交付链的系统。“我们非常依赖商业智能来确定应用程序的功能。数据实时地从各个地方传来。Sonis 补充道:“我们的分析师需要立即审查和回应，无法从每个系统中筛选数据。

分析工具可能会令人困惑。有很多，在许多子类别中。由此产生的一个工件是许多团队采用了适合他们交付链部分的工具。IT Ops 采用服务器监控，开发者采用 APM。但大局往往被忽视，这不仅会危及整个运营的可持续性，还会危及全栈应用和部署的本质。跨指标池关联问题的能力是手动的，如果发生的话。这是 Wix 的一个盲点，这是一个严重的问题，因为他们的产品团队每天要发布 60 多个应用程序数据。

“在所有不同的系统中创建手动警报、阈值和指标是不可扩展的，”Sonis 解释道。他们需要数据之上的机器学习来产生见解。机器学习的承诺也不是新的，但我个人发现它是高度主观的。并且只有在几个月的真实使用案例中，价值才能得到体现。机器学习的性能也很关键。在现代应用程序中学习到的东西保存期限很短，所以它需要近乎实时地发生。

在监测团队部署 Anodot 后，他们知道会有很好的采用，但对其发生的方式感到有点惊讶。“我们只是将其作为服务发布，并提供来自每个应用程序版本、每个服务器等的所有数据。一旦我们这样做了，后端开发人员就可以通过[](https://slack.com/)开始使用它。所有的警报进入一个频道，最好的部分是他们自己设置它。此外，由于通道还集成了 Git，他们可以立即查看新提交是否影响了警报，”Sonis 补充道。

为了跟踪谁得到了什么，Wix 利用了[ZooKeeper](https://zookeeper.apache.org/)，以及他们在其上构建的管理层来收集元数据，并跟踪所有开发人员和团队。管理层知道哪些数据与团队相关，这有助于自动创建松弛通道。因此，团队成员所需要做的就是订阅他们已经所属的小组——这也意味着 Sonis 的团队不必为鼓励和培训新用户做大量的前期工作。

建立专门的监控团队并不常见，大多数情况下也不需要。事实上，有了 Anodot 这样的平台，这可能根本就没有必要。如果一个组织以高级可见性概念为先导，他们将从第一天起就拥有所需的数据，而无需手动汇集所有来源。

Wix 是一家不惧怕新技术的技术公司。但它们也不是全新的。他们对 DevOps 的采用一直令人印象深刻的是，他们跟上了新工具和实践的步伐，而没有分发他们所做的事情(例如，从单一应用程序到由微服务组成的应用程序的巨大转变)。他们通过有意识地监控和学习他们的管道来维持他们的现代运输链。
