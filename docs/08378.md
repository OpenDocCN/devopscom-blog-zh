# 这是怎么回事？

> 原文：<https://devops.com/how-does-that-stack-up/>

有一整类成功的创新是由人们没有意识到他们需要的东西组成的。那些成为障碍的事情，他们找到了解决的方法，直到创新给他们展示了更简单的方法，他们才开始思考。想想类似于[虚拟化](https://devops.com/?s=virtualization)(硬件是瓶颈，但“事情就是这样，”直到它不再是)或网络怪物购物网站(“商店里没有这种东西；我想我没有也能活下去，”直到你可以从易贝和亚马逊这样的网站上订购最奇怪或最小的东西。

我的原则是，只有当我使用产品和服务并从中发现价值时，我才会谈论它们。即使那样，我也尽量保守。但是本着你刚刚读到的创新故事的精神，我将带你去网上最酷的新网站之一。不了解公司；没有和他们做过生意，甚至不能说出负责人的名字。但是我在他们的网站上花了惊人的时间，包括在 *Feed* 部分和*浏览 Stacks* 。

该网站名为 [Stackshare.io](https://stackshare.io/) ，我认为它如此创新的原因是它的目标是什么和谁:列出公司的 DevOps 堆栈，并就 DevOps 堆栈的不同部分提供建议。

*浏览堆栈*部分用于展示其他组织正在做的事情，因此您可以获得自己堆栈的好主意。但该网站的开发者似乎已经认识到这一信息的另一个有用方面:根据技术将组织与未来的员工相匹配。例如，你可以去看看 X 公司正在使用 GitLab，这可能会影响你的求职。

*Feed* 部分只是一个持续的讨论，使用类别来提出问题或提供关于给定组织做出的决策的见解和建议——因此，“选择 X 而不是 Y”实际上是帖子的标准副标题，就像“需要关于 Z 的建议”或“推荐 A”一样。这是很好的阅读材料，毫无疑问，在通读过程中，你会有一些建议提供给“寻求建议”的帖子。

Stackshare.io 参与者的规模似乎从初创到庞大不等，这是我希望他们随着增长而改变的事情之一——查看 DevOps 堆栈的选项是“流行”和“趋势”。我们都希望有更多的过滤器——组织规模、应用数量等——来帮助我们聚焦于与我们相似的环境。我的小公司没有脸书的需求。虽然一些初创公司在他们的工具链中使用的工具确实比脸书多，但你应该明白。

我在 Stackshare.io 网站上遇到的另一个问题(我完全理解，但仍然希望能有所不同)是强制注册/登录以查看一些数据。它甚至不是特别敏感的数据——我知道 ELK 和 Azure 数据服务是什么，所以将定义/描述放在注册页面/登录后面有点傻。但是每个免费网站都需要数据来支付账单，我想我很高兴在 reg 墙后面的其他地方也可以获得这些信息。

该公司提供集成到你的站点/工具和私有公司 StackShare 实例的 API 来帮助你管理我的另一个抱怨——五个(或五十个！)许多组织允许内部开发的不同 DevOps 堆栈。

因此，如果你正在找工作，Stackshare.io 可以帮助你找到做你想做或你擅长做的事情的公司。如果你是一个试图改进或建立一个新工具链的组织(我就是这样找到这个网站的)，它有很多信息要消化，尽管在这一点上并不是所有的信息都是容易访问的。无论哪种方式，继续踢它，并留意做这项工作的工具，无论你在哪里看。
