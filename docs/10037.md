DevSecOps 是一种软件开发方法，它将开发(Dev)、安全(Sec)和操作(Ops)合并成一个团队，在整个软件开发生命周期(SDLC)中集成安全性。目标是快速安全地交付高质量的应用程序。

传统的软件开发生命周期在 SDLC 的后期阶段引入了[安全性。然而，这种方法无法抵御复杂的网络安全攻击。当今的网络安全环境需要更短、更频繁的迭代来快速修补和修复安全问题。DevSecOps 将安全性集成到](https://devops.com/get-comfortable-with-shifting-right-to-improve-resiliency/) [持续集成和持续交付(CI/CD)管道](https://codefresh.io/learn/ci-cd/) 中，以确保团队能够以 DevOps 的速度解决紧迫的安全问题。

## 什么是安全接入服务边缘(SASE)？

[【SASE】](https://www.catonetworks.com/sase/)安全接入服务边缘(Secure access service edge)是一种网络安全模型，基于身份、实时安全上下文和企业安全策略提供安全的网络连接。它符合零信任安全模式，确保默认情况下拒绝连接，除非经过验证的身份有权访问特定的应用程序、服务或边缘位置。

SASE 将网络和安全功能统一到云中提供的一个完全集成的服务中。SASE 包括多项技术:软件定义的广域网(SD-WAN)、安全网络网关(SWG)、云访问安全代理(CASB)、下一代防火墙(NGFW)和零信任网络访问(ZTNA)。

SASE 的一个主要优点是增强了网络安全性。SASE 服务通过精细的网络保护方法来增强网络安全性，通过趋势科技服务器深度安全防护系统扫描来检测和阻止分布式拒绝服务(DDoS)攻击、中间人(MitM)攻击、网络钓鱼、电子邮件欺骗和恶意软件等网络攻击。

SASE 的附加好处包括:

*   与传统网络相比，成本更低，因为 SASE 包括更少的内部硬件组件和以前从多家供应商购买的功能。
*   更易于维护，因为它提供了一个平台，允许 it 团队管理所有应用程序和用户，无论他们身在何处。

## SASE 的组件

### 云接入安全代理(CASB)

CASB 充当用户/设备和云应用之间的桥梁。它允许组织对所有云应用程序应用安全策略、二元身份验证和单点登录，防止未经授权的设备和个人访问敏感资产，并在默认情况下拒绝访问。

### 零信任网络访问(ZTNA)

ZTNA 确保用户或设备具有访问应用程序或资源的明确许可。它是零信任安全架构的一个关键组件，确保内部私有应用对无权访问它们的用户隐藏。它还通过分层身份验证实现了安全的远程访问。ZTNA 提供灵活的安全性来满足现代安全需求。

### 软件定义的广域网(SD-WAN)

SD-WAN 是一种连接架构，将网络硬件与物理控制层分开。它通过增强和简化 WAN 性能和管理来提高网络性能。SD-WAN 支持基于云的现代应用和服务，同时降低连接成本。将 SD-WAN 与高级安全功能相结合，为企业过渡到 SASE 奠定了基础。

### 安全 Web 网关(SWG)

SWG 保护在线设备免受恶意软件感染，执行公司安全政策，并过滤用户发起的互联网流量中不需要的网站。SWG 提供 URL 过滤、应用程序控制、DLP、防病毒、沙箱和 SSL 检查。所有这些功能都可以保护用户免受有害网站的侵害，并执行安全策略，同时确保全面的 web 访问并提高工作效率。

### 防火墙即服务(FWaaS)

FWaaS 是作为基于云的服务交付的防火墙解决方案。它提供超大规模下一代防火墙(NGFW)功能，包括网页过滤、高级威胁防护(ATP)、入侵防御系统(IPS)和域名系统(DNS)安全。通过将 FWaaS 集成到 SASE 平台中，组织可以在边缘提供广泛的网络安全功能。

### 集中统一管理

现代化的 SASE 平台使 IT 管理员能够通过跨网络和环境的集中、统一管理来管理 SD-WAN、SWG、CASB、FWaaS 和 ZTNA。这使 IT 团队成员能够专注于更紧迫的领域，并为组织的混合员工改善用户体验。

## SASE 将如何影响组织中的开发和安全

SASE 可以帮助保护内部交互和应用程序。因此，DevSecOps 团队的负担减轻了。开发运维团队可以使用零信任网络访问(ZTNA)来进一步保护 SD-WAN 上运行的应用。这些技术确保终端和应用程序之间的所有交互安全。

SASE 模型还有助于保护无法在公共互联网上运行的任务关键型应用程序和敏感数据。为了保护这些资产，SASE 对流量进行模糊处理，使用零信任架构限制访问，并使用 NGFW 保护所有网络入口点。它会持续评估应用程序流量中的安全威胁。

### 促成协作

DevOps 团队专注于上市时间，并不断努力加速 CI/CD 渠道。他们需要通过可靠安全的连接进行远程协作。DevOps 团队也在努力减少对网络和安全团队的依赖。SASE 模型有助于解决这些问题。

SASE 通过统一广域网和网络安全来帮助减少协作问题。SASE 解决方案提供了优化的性能和强大的安全性，因此 DevOps 团队无需依赖基础架构员工来调配资源。它让团队能够完全控制运营，而不会影响性能或安全性。

### 提供内置安全性

SASE 提供了保护 DevOps 项目所需的基本网络安全功能。这允许团队将不同的安全机制集成到他们的网络堆栈中，所有的安全服务共享一个集成上下文。这填补了传统安全架构中经常被攻击者利用的漏洞，并保护应用程序免受威胁。

### 改善事件响应

SASE 可以通过嵌入在网络结构中的安全控制立即阻止各种攻击媒介，支持事件响应团队的工作。SASE 提高了混合环境中行为异常和安全事件的可见性。

通过将 SASE 解决方案集成到安全信息和事件管理(SIEM)系统中，事故响应人员可以更好地了解交通异常情况以及 SASE 工具采取的自动化安全措施。像 NGFW、SWG、IPS 和反恶意软件这样的 SASE 功能可能已经包含或消除了许多事件。

### 提供更可靠的基础设施

开发运维团队需要快速、可靠的应用性能。SASE 提供集成的负载平衡和故障转移功能，可以显著提高网络的整体性能。

SASE 架构可以将所有边缘设备、远程用户、数据中心和云资源连接到一个安全、自我修复、完全优化的全球网络。如果电路出现故障或路径变得拥塞，结构可以自动切换到备用路径。

## 结论

在本文中，我介绍了 DevSecOps，并展示了 SASE(一个在边缘提供快速网络连接和安全性的现代平台)如何支持它的采用。DevOps 团队可以使用 SASE 安全组件来保护内部数据中心和云中的应用。SASE 可以加密和检查流量以保护其免受威胁，确保工作负载、用户和开发系统在默认情况下是安全的。

有了 SASE，DevSecOps 团队就有了保护环境所需的基础设施，并可以专注于将安全测试和补救集成到开发生命周期中。
