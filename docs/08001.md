# Atlassian 兑现 Forge 应用开发平台承诺

> 原文:[https://devo PS . com/atlassian-delives-on-forge-app-dev-platform-promise/](https://devops.com/atlassian-delivers-on-forge-app-dev-platform-promise/)

Atlassian 本周发布了一个 [Forge](https://www.atlassian.com/blog/announcements/forge-launch) 平台，用于构建补充其现有产品组合的应用程序。

Atlassian 平台工程负责人 Mike Tria 表示，Forge 旨在为 IT 组织和第三方合作伙伴提供一个应用开发环境。Tria 补充说，它使开发人员能够访问集成的 Atlassian DevOps 工具链，以加速测试、部署、调试和捕获日志数据。Forge 还为开发人员提供了一个命令行界面，通过这个界面，他们可以进入 Atlassian cloud，Forge 在那里运行以访问文档和应用程序示例。它使用 OAuth 2.0 来确保部署在 Atlassian cloud 上的应用程序只能访问他们有权访问的数据。

总体目标是为开发人员提供一个更现代的应用程序开发环境，作为公司以前提供的应用程序开发平台的替代。

![](../Images/567ea11a5003c177c0207410ba493272.png)

一旦应用程序构建完成，开发者就可以在 Atlassian Marketplace 上发布他们的应用程序，作为免费或商业产品。在去年年底开始的测试过程中，已经有超过 3500 个 Forge 应用程序被创建。Tria 指出，Forge 将很快扩展到 Atlassian 投资组合的其余部分。

Forge 旨在补充本月早些时候发起的 [Open DevOps](https://devops.com/atlassian-extends-open-devops-ambitions/) 计划，该计划加强了 Atlassian 提供的吉拉项目跟踪和管理工具与 DevOps 团队常用的其他工具之间的集成。我们的目标不仅是更容易将吉拉列出的问题与各种软件平台中发现的工件关联起来，而且还能从 DevOps 工具中收集数据以自动更新吉拉。

根据 Atlassian 在吉拉提供的图表，这些数据将用于关键绩效指标(KPI ),以跟踪价值流。不需要获得一个单独的[价值流管理](https://www.vsmconsortium.org/) (VSM)平台来跟踪软件开发进度，例如，理解一个项目延迟的影响所需的洞察力成为使用吉拉的红利。

像大多数软件即服务(SaaS)应用程序提供商一样，Atlassian 一直在提供一个平台，用于在构建自己的应用程序的同一平台上构建应用程序。Atlassian 声称，除了内部 IT 团队为其应用程序构建定制扩展之外，还有 1200 多个合作伙伴在构建应用程序。Atlassian 已经表现出收购在其平台上构建应用程序的独立软件供应商(ISV)的倾向。Tria 表示，作为整体 Forge 计划的一部分，Atlassian 将更加紧密地合作，以确定 Atlassian 产品路线图在未来某个日期不会解决的空白区域。

与此同时，独立软件供应商和内部 IT 团队将需要确定在何种程度上他们更愿意建立自己的应用程序开发环境，而不是选择依赖 Atlassian。至少在理论上，基于 Forge 构建的应用程序应该与 Atlassian 产品组合的其他部分更紧密地集成在一起。当然，唯一潜在的缺点是，与同时销售应用软件的应用开发平台提供商之间可能出现的任何潜在冲突。
