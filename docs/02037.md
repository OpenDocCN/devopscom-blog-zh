# Opsview 宣布推出新的 Splunk 结果导出器

> 原文：<https://devops.com/opsview-announces-new-splunk-results-exporter/>

***新导出器实时提取、过滤和重新格式化原始 Opsview 监视器数据，以便与 Splunk Enterprise Security 和其他 SIEM 工具安全集成***

******

 ****BOSTON—2019 年 8 月 28 日** — [统一云和基础设施监控的公司 Opsview](http://icm-tracking.meltwater.com/link.php?DynEngagement=true&H=I%2BqMhXDuIS2RujNbavWSOKShOr7Ezi73JeGvxlkJ09Qb0kDpZZQFeNNF4HHcGlX1zAtuLhv9VWAoJXYL1gIWI1LKJvDVBmkVghFVA72dg6AwJ8dnZwrt6gvEBKf4pbLQraHeUkYDZxI%3D&G=0&R=http%3A%2F%2Fwww.opsview.com%2F&I=20190828172159.00000461152b%40mail6-46-ussnn1&X=MHwxMDQ2NzU4OjVkNjZiODM0YWY4YTQzYWIyZDdhZTIxMTs%3D&S=wSa-PDI_exVG31VeXBWm8OcRsz6v3b-hJPdmiRsCCPc) 今天宣布推出 Opsview Monitor Splunk 结果导出器。新的工具包可以轻松、安全地将大量事件数据实时导出到安全信息和事件管理(SIEM)工具，包括 Splunk 企业安全。通过使用 Opsview 事件数据，Splunk 和其他 SIEMs 能够识别数据泄露检测的异常活动，并揭示低效云资源使用的模式，从而发现节约成本的机会。

**产品特性:**

Opsview 结果导出器提供了与 Splunk 和其他 SIEM 分析解决方案的简单而灵活的集成，仅将相关数据导出并传递到 Splunk 以优化使用成本。通过利用易于使用的工具包，用户可以直接从 Opsview Monitor 的事件总线提取、过滤和重新格式化原始数据，并通过 HTTP 将其转发到目标 SIEM。用户可以在外出前灵活地过滤和重新格式化数据，以优化分析和存储消耗。主要优势包括:

*   并行生成多个输出，每个输出都有自己的自定义字段映射和过滤，因此可以同时支持多个用例。
*   为每种输出方法和要求重新格式化和过滤数据，包括灵活定义特定字段的能力。
*   优雅地支持通用接口，包括 REST 和 Syslog。
*   灵活地支持为需要工具的用例使用定制工具，例如针对 ELK 的 filebeat 或定制脚本。
*   针对真实世界的 Splunk 就绪性进行了测试，包括与 Splunk 云、Splunk 企业和使用 Splunk 转发器的环境一起使用，特别坚持使用 https 以获得最佳安全性。

Opsview Results Exporter 是一个理想的解决方案，可以安全、无中断地添加复杂的安全性、网络流量整形、成本分析和其他操作增强功能。Opsview Monitor 是一款全面的同类最佳监控工具，旨在为内部、云或混合环境中的 IT 运营提供统一的洞察力。当与 Splunk 的 Opsview 结果导出器配合使用时，it 运营团队能够搜索和分析丰富的 IT 事件和指标，使用预先构建的试探法和机器学习来发现安全问题、效率低下、利用率错误和其他优化 IT 支出的机会。
