# Ravello Systems 为 DevOps 提供了在云中克隆生产应用程序的持续集成能力

> 原文：<https://devops.com/ravello-systems-powers-devops-with-extreme-continuous-integration-on-clones-of-production-applications-in-the-cloud/>

Ravello 增加了新的 Apache Maven 插件，并宣布提供 Ruby 和 Python 绑定，以便与构建自动化工具集成。

加利福尼亚州 PALO 阿尔托市——2014 年 2 月 12 日—[业界首家云应用程序管理程序提供商 Ravello Systems](http://www.ravellosystems.com/) 今天宣布，它已添加了新的插件和 RESTful APIs，以在云中提供整个多虚拟机环境以及网络和存储。Ravello 添加了一个新的 Apache Maven 插件，以及 Ruby 和 Python 的编程语言绑定，支持持续集成(CI)系统，包括 Jenkins、Bamboo 和 Teamcity。只需一次 API 调用，开发人员就可以立即配置其现有内部生产应用的完整克隆，并将其部署在公共云中。

**Ravello 在云中的生产副本上启用 CI**

为了实现成功的敏捷开发和测试过程，企业正在迅速采用 DevOps 原则，而使用 CI 是一个关键的实践。CI 的目标是尽可能快地获得可靠的反馈，从而更快地交付更高质量的软件。然而，大多数 DevOps 实践者在实施 CI 时会遇到一些限制，这是由于(1)自动化基础架构方面的困难，这使得提供多层生产应用程序的克隆很成问题，以及(2)数据中心的容量限制，这使得不可能为每个代码提交部署这些克隆。自动化 CI 流程可能非常痛苦、耗时，并且充满了配置噩梦。

借助广受欢迎的构建自动化工具 Apache Maven，开发人员能够从中央存储库管理项目的构建、部署、测试、报告和文档。认识到 Maven 作为构建自动化工具的价值，Ravello Maven 插件旨在帮助基于 Maven 的项目与 Ravello Systems 丰富的 API 集成。这种集成使得自动配置构建、测试和部署过程更加容易，从而使操作更加顺畅。

Ravello 的软件即服务(SaaS)产品允许企业在公共云中构建其现有多层企业应用的克隆，而无需对虚拟机(VM)、网络或存储进行任何更改。随着新 Maven 插件的加入以及 Ruby 和 Python 绑定的可用性，Ravello 现在可以在云中的生产克隆上实现 extreme CI。现在开发人员和测试工程师可以使用 Ravello 的 API 来:

*   调配整个环境；
*   根据需要启动和停止应用程序；
*   自动执行测试，并在测试通过后关闭以释放资源，或者在需要调试问题时保存状态；
*   轻松与诸如 Jenkins、Bamboo 或 Teamcity 等 CI 服务器集成。

所有这些都可以在集成开发环境(IDE)中完成。现在，每个代码提交都可以在云中的生产克隆上进行测试，以获得即时和准确的反馈。

Ravello Systems 的研发副总裁吉尔·霍弗(Gil Hoffer)说:“公共云具有无限的容量，非常有希望实现持续集成，但这是一个与现有环境完全不同的环境。“您的应用程序可能有多个子网，每台服务器有多个网卡，并且有特定的存储要求。借助 Ravello，我们抽象出了私有云和公共云之间的差异，并内置了在云中调配和部署这些多层应用的自动化功能。”
