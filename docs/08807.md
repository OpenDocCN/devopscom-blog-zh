# 微软扩展 Azure DevOps 产品组合

> 原文:[https://devo PS . com/Microsoft-expands-azure-devo PS-portfolio/](https://devops.com/microsoft-expands-azure-devops-portfolio/)

微软已经为其 DevOps 产品组合提供了一系列更新，这些更新旨在增加其在将在扩展企业中推出的应用浪潮中的份额。

在[微软 Ignite 2021](https://myignite.microsoft.com/home) 大会上宣布了微软 DevOps 产品组合的新增内容，其中包括 DevOps 工作流生成器，这是一款旨在帮助 DevOps 团队可视化其工具链的免费工具，以及对 [Azure Automanage](https://cloudblogs.microsoft.com/windowsserver/2021/11/02/5-new-capabilities-to-automate-windows-server-management-with-azure-automanage/) 的更新，这是一个用于在 Windows Server 上以及在 preview 中在 Linux 上自动化虚拟机管理的框架。

与此同时，微软在 Azure Monitor 中添加了对开源 OpenTelemetry 代理软件的支持[，并提供了](https://techcommunity.microsoft.com/t5/azure-monitor/what-s-new-in-azure-monitor-ignite-2021/ba-p/2912049) [Azure Chaos Studio](https://techcommunity.microsoft.com/t5/azure-governance-and-management/announcing-the-public-preview-of-azure-chaos-studio/ba-p/2893050) 的预览版，这是一项测试应用弹性的服务。

微软还提供了 Windows Server 2022 的预览版，为部署在该平台上的工作负载增加了安全代码功能。微软还预览了对后台数据库的更新，如 SQL Server 2022、Cosmos 以及 Redis 实验室创建的 Redis 数据库的实现。

最后，微软还提供了一个用于自动化软件开发任务的 GitHub Actions 和使用 Open ID Connect 的 Azure Active Directory (Azure AD)之间集成的[预览](https://techcommunity.microsoft.com/t5/azure-developer-community-blog/build-secure-apps-on-hardened-dev-environments-with-secure/ba-p/2893917), Open ID Connect 是 OpenID 联盟提出的一种认证协议。

微软首席执行官塞特亚·纳德拉告诉与会者，未来 10 年将见证比过去 40 年更多的数字化，因为它将从一个由移动和云计算定义的时代转向一个由无处不在的计算和环境智能定义的时代。

从程序代码到 Microsoft Power Fx，这是一个低代码工具，Microsoft 使用最初为 Microsoft Excel 电子表格用户构建的编程语言创建。微软设想这些应用程序将主要建立在其云上，并分布在各处。在后端，一系列数据库和人工智能(AI)服务将主要通过应用程序编程接口(API)调用。

当然，每个主要的云服务提供商都有类似的雄心。如今，大多数组织都没有在单一云上实现标准化。相反，他们决定哪类工作负载在哪个云计算平台上运行得最好，而不管该方法是否会增加 IT 的总成本。采用的每一个额外平台通常都需要 IT 团队掌握独立的工具。然而，微软现在也在为[一个可以部署在任何地方来构建、部署和管理应用程序的云原生框架](https://containerjournal.com/features/microsoft-extends-cloud-native-portfolio/)做准备。

与此同时，部署在云服务上的工作负载数量持续增长。市场研究公司科纳仕公司报告称，2021 年第三季度全球云基础设施服务支出达到 494 亿美元，增长 35%。微软目前在整体市场份额上落后于亚马逊网络服务(AWS)，但继续稳步增长。微软和 AWS 合起来约占所有云基础设施服务消费的一半。

DevOps 最佳实践需要不断发展，以应对多云计算的挑战。除了云平台和内部 IT 环境，现在越来越多的应用程序开始部署在边缘。DevOps 团队需要能够在广泛的平台上持续部署应用程序及其相关更新。当然，挑战在于所有这些平台都有独特的属性，这使得真正实现端到端的整个流程自动化成为一个难以实现的目标。
