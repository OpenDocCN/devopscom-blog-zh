# DevOps 如何革新嵌入式开发

> 原文：<https://devops.com/how-devops-is-revolutionizing-embedded-development/>

全球嵌入式开发人员的桌面上正在发生一些奇怪的事情。从家庭办公室到小房间到实验室，在深夜灯光明亮的房间和清晨阳光明媚的房间，开发人员正在做一些不同的事情。他们开始在嵌入式开发中实施 DevOps 实践。

一些相当偶然的事情使这成为可能:云原生计算的平行崛起和物联网(IoT)设备和边缘技术的爆炸。每天每时每刻都有数十亿台嵌入式设备连接到互联网，这就需要一种新的方法来设计、测试、部署和维护这些设备。与此同时，云原生计算引入了[连续交付](https://devops.com/?s=continuous+delivery) (CD)和连续集成(CI)等方法，以实现可重复的自动化软件构建。这正在嵌入式世界掀起一场革命。

直到现在，嵌入式软件被编写和部署，然后永久保持静态，因为它是孤立的。不再是了。

## 长期而言，安全需要新思维

嵌入式软件开发人员传统上面临着构建项目的巨大困难。通常，嵌入式开发人员必须构建一个完整的平台来构建新软件，包括内核、发行版等等。而且所有的软件都需要在每次发生变化时重新构建。所有的东西都必须检查。一切都必须展开。有一个版本控制基础架构团队，一个 CI 团队，一个 QA 团队——他们都必须一起工作，并且保持一致以满足生产进度。

这种开发方法极其缓慢、昂贵和笨拙，而且根本不是为当今的互联设备、物联网和边缘世界设计的。消费品制造商(例如家用电器)根本没有资源来提供频繁的更新。此外，消费者不习惯在非手机或笔记本电脑的消费设备上频繁更新。随着现代应用程序的兴起，构建从不改变或更新的静态软件不再被接受。

随着数十亿台设备连接到一个始终在线的网络，在网络上拥有数百万个节点的公司中，大规模安全漏洞从未成为一个更大的问题或考虑事项。如今，公司需要持续连接到其所有节点和设备，以正确维护和保护它们。

## 进入 DevOps 和嵌入式创新

DevOps 已经改变了云原生应用的开发方式。软件经常被构建，但也经常被部署。这也是 web 应用不断升级的原因。现在，embedded 已经准备好利用这个软件版本控制系统，它可以自动构建软件，并且是可复制的。将 DevOps 引入嵌入式软件生态系统是为了提供一套工具或一个平台，将版本控制、CI 和 CD 全部放在一个包中，这是一个简化公司工作流程的平台，通常需要自己构建所有这些东西，现在可以为巨大的效率和创新提供机会。

想想 60 多年前的航空业。当时航空公司自始至终都是自己造飞机。今天，波音和空客制造所有航空公司的飞机。当然，每家航空公司都可以定制他们的机队，从整体飞行体验到内部品牌，但每个人都在利用相同的核心技术，因为机身真的很难制造，而差异就发生在基础之上。

类似地，嵌入式软件开发人员可以专注于他们产品的定制，而不是耗时的构建，这并不能提供差异化。这是我们在嵌入式软件中看到的文化转变背后的概念:购买机身并定制它，而不是自己构建它。DevOps 模型中固有的还有分布式开发实践，其中不同角色的不同团队成员可以在世界上的任何地方，并且仍然无缝地一起工作。

疫情无疑为这种模式创造了更多的需求，但是分布式团队并没有消失，并且在技术开发中比以往更加普遍。许多嵌入式软件开发人员仍在以他们在 20 世纪 80 年代和 90 年代的相同方式工作。随着 DevOps 实践和平台的发展，嵌入式软件开发人员可以专注于其应用的增值。通过消除自行构建的所有复杂性，如今的公司可以简单地利用易于消费和重用的工具来实现云开发人员多年来一直在做的事情。
