# 英特尔借助 API 和处理器的进步吸引开发团队

> 原文：<https://devops.com/intel-woos-devops-teams-with-api-and-processor-advances/>

在在线[英特尔创新 2021](https://www.intel.com/content/www/us/en/events/on-event-series.html) 活动期间，英特尔今天宣布了一个用于访问应用编程接口(API)的统一开发人员专区，该专区通过 oneAPI 工具包提供给希望更轻松地调用各种处理器类型和类别的 it 团队。

与此同时，该公司还透露，它正在创建一系列 oneAPI 卓越中心，这将使 DevOps 团队更容易使用那些[API](https://devops.com/?s=APIs)。参与该倡议的合作伙伴包括橡树岭国家实验室、加州大学伯克利分校、杜伦大学和田纳西大学。英特尔还将使 Xellence 的英特尔图形可视化研究所成为该计划的一部分。

明年，英特尔还承诺推出 oneAPI 2022 工具包，将增加 900 种额外功能，包括使用英特尔描述为第一个统一 C++/SYCL/Fortran 编译器为 CPU 和图形处理器单元开发应用的能力。

除了扩展其 Advisor accelerator 性能建模工具以支持 VTune Flame Graph 可视化性能热点之外，该公司还计划提供数据并行 Python 工具。

还将有额外的 oneAPI 集成，以添加对 Microsoft Visual Studio 代码和 Microsoft Windows 子系统 for Linux 2 平台的支持。英特尔还创建了英特尔 oneAPI 人工智能分析工具包，以实现开箱即用的人工智能开发，该开发由运行戴尔技术、惠普和联想的 Linux 的工作站支持。此外，微软和英特尔已经同意为 Windows 11 创建一个完整的数据科学工具链，该工具链将成为构建应用程序的 Surface Laptop Studio 框架的一部分。

英特尔首席技术官、高级副总裁兼软件和高级技术事业部总经理 Greg Lavender 表示，确保开发运维团队将英特尔在其处理器中提供的加速引擎和扩展支持添加到他们部署的运行时环境中至关重要。否则，开发人员将永远无法例行调用这些功能来构建下一代应用程序。

该公司还透露，代号为 Sapphire Rapids 的下一代英特尔至强可扩展处理器将拥有额外的加速引擎，以简化大规模部署并优化处理器的核心利用率，从而降低功耗。总体而言，英特尔承诺在这些处理器上运行的人工智能(AI)应用的总体性能将提高 30 倍。

今天宣布的其他硬件进步包括基于 7 纳米处理器的第 12 代英特尔酷睿系列和 Mount Evans 项目，通过该项目，英特尔与谷歌云合作，构建基于专用集成电路(ASIC)芯片的处理器。

该公司还推出了英特尔 ArcTM 炼金术士系列片上图形系统(SOC)，利用机器学习算法来优化性能，并宣传了英特尔 Tofino fabric 处理器(IFP)，该处理器通过支持 P4 编程工具为网络交换增加了智能。

最后，该公司指出，将于 2022 年推出的 Aurora 超级计算机的峰值双精度计算性能将超过 exaflops。

随着英特尔寻求从 NVIDIA、ARM 和 AMD 等竞争对手手中夺回处理器高地，很明显，it 的未来将由各种具有独特功能的处理器类别来定义。现在的挑战是决定那些功能中的哪些真正提供给开发人员，这些开发人员很大程度上依赖于 DevOps 团队决定提供的运行时环境。
