马斯喀斯基:是啊。我是 Platform 9 Systems 的产品副总裁和联合创始人。Platform9 所做的是我们开创了一种独特的模式，一种基于 SaaS 的模式，用于部署复杂的开源云原生框架，如 OpenStack 和 Kubernetes。

艾伦·希梅尔:是的。这是我第一次介绍 Platform9，我被它的商业模式吸引住了。我觉得这很有趣，不像我之前看到的任何东西。正如我提到的，Madhura，除了标语之外，你能给我们的观众一个关于 Platform9 正在做什么的感觉吗？

马斯喀斯基:绝对是。从一开始，我们的根基就来自 VMware。我和我的联合创始人在那里度过了大约七年多的时间。当时，我们在与大中型企业客户的合作中看到的一个共同主题是，他们在尝试实现私有云这一愿景时所经历的大量斗争，当时私有云是一个开始流行了大约一年的概念。

它开始变得流行的原因是因为公共云为消费基础架构的容易程度设置了异常高的标准。但是，企业、VMware 的客户或其他人都没有达到这一高标准，没有创造出一个平等的产品来利用他们的内部资源，这些资源可能基于他们的数据中心或以混合方式使用，但却为他们的内部消费者创造了一个类似的无缝界面来使用这些资源。

因此，Platform9 一开始的愿景是，任何人都可以从任何基础架构端点(无论是私有、公共还是混合)使用资源，这应该很容易，基本上就像使用公共云一样容易。我们希望尽可能使用可用的开源软件来实现这一点，因为我们也强烈认为开源将成为云基础设施消费方式的新规范。

具体到 Platform9，我们的第一个产品是托管 OpenStack，我们的 OpenStack 即服务。该模式意味着我们以 SaaS 托管的方式部署 OpenStack 控制平面。控制平面部署在我们客户的数据中心之外，然后通过代理机制与他们的数据中心或公共云端点集成。

代理，如果您有私有基础架构，代理将成为私有基础架构和我们的云托管控制平面之间的桥梁。如果您有公共云，那么我们有直接与公共云端点交互的驱动程序。这就是该模型的工作方式，它让您在几分钟内就可以启动并运行 OpenStack。

现在，我们用 Kubernetes 重复了相同的模型，这是使用微服务范式构建应用程序的最强大的容器编排框架，其中 Kubernetes 也作为 SaaS 托管的服务部署在设备的基础设施端点上，即私有或公共端点上。

**艾伦·希梅尔:**明白了。当然，当我看到它时，Madhura，我明白了，好吧，所以你有了 OpenStack 产品，OpenStack 即服务，现在你开始挖掘 Kubernetes 和容器。我必须假设——也许我不应该假设。让我问你:OpenStack 市场增长有多快？

**Maskasky:** 我们看到 OpenStack 的客户群逐年大幅增长。2016 年，我们有大约 50 名客户。多项统计数据预测，2017 年 OpenStack 收入的整体增长相当强劲。就在昨天，我读到一篇文章，其中提到整个 OpenStack 市场的收入预计将增长数十亿美元，并且财富 500 强将会越来越多地使用 OpenStack。

因此，即使有了容器，虚拟化基础架构也不会很快消失，我们看到的企业客户面临的巨大挑战是，一方面，他们希望通过将其当前虚拟化基础架构的一部分转换为开源 Linux KVM 并使用 OpenStack 来节省一些资金，而不是完全走 VMware 路线。或者，另一方面，一些客户一开始是 100%的公共云消费者，但后来意识到这是一种非常昂贵的大规模模式，因此希望进行更多的混合部署，其中他们的部分基础架构由私人托管或托管在服务提供商数据中心。

对于这两种情况，Platform9 都非常适合，因为我们同时支持 VMware 和 KVM，还因为我们的“即服务”模式。

**艾伦·希梅尔:**明白了。Kubernetes 怎么样？也许是因为我生活在一个 DevOps 的泡泡里，但那似乎是一种起飞，是吧？

Kubernetes 正在以一种非常重要的方式起飞。我们认为 Kubernetes 将在大约一年前开始建立这种水平的势头，因为我们正在研究不同的容器编排框架，并评估它们的优缺点，另外两个最佳框架是 Mesos 和 Docker Swarm。

Mesos 和 Docker 当时正处于非常早期的阶段，但 Kubernetes 如此受欢迎的主要原因是我认为与它相关的三个重要因素。一个是来自项目成熟度。它们是最成熟的项目——在成为今天这样的开源项目之前，它是在谷歌内部构建和审查的。它有一个非常重要的社区，除了积极为它贡献力量的谷歌开发者。

每三到四个月就会有一个新版本出来，迭代的速度令人震惊。在 3 中，许多企业客户通过展示使用案例和他们如何利用 it 的例子公开为其担保。

**艾伦·希梅尔:**明白了。Madhura，既然我们已经说得很清楚了，那就让我们来谈谈 9 号平台最近的新闻吧。你想和我们的观众分享吗？

马斯喀斯基:绝对是。我们最近分享了两条新闻，实际上就在一周前。第一个是受管理的 Kubernetes 产品，它已经测试了大约六个月，现在已经正式推出。自从我们在 2016 年 6 月左右宣布该产品以来，他们已经有机会与几十个客户一起对其进行测试。因此，现在我们很高兴地与全世界分享，它是普遍可用的。

再说一遍，pure Kubernetes 是作为 SaaS 管理的服务部署的。我们认为这是业内首款与基础设施无关的 Kubernetes 即服务产品，因为它允许您使用私有和公共基础设施，并且内置了许多企业就绪功能，如多租户、单点登录、配额管理，以及能够部署高度可用的 Kubernetes 集群。

艾伦·希梅尔:太好了。当然，再一次，我只是认为有一些东西——我认为市场迫切需要这种产品。我记得 OpenStack 刚出来的时候。实际上，如果你还记得 Cloud.com，我想马丁·尼肯斯参与了那家公司和其他几家公司。这确实是迁移到云的发展过程中的一个转折点；重要的是，人们能够将所有的部分放在一个盒子里，如果你愿意的话——盒子里的云，因为没有更好的词。

但在当今世界，即使是这种模式也行不通。人们需要服务。因此，如果他们能够获得像 Kubernetes 这样的基础设施即服务，并且不可知，那就太棒了。另外，一种叫做裂变的东西也被引入了，对吗？

**马斯喀斯基:**没错。这是我们公告的第二部分，我们宣布了一个新的开源框架，叫做裂变。它是什么，它被设计成 AWS Lambda 的事实上的开源替代品。

艾伦·希梅尔:对不起。我不是故意打断你的。去吧。

别担心，艾伦。所以只是一些背后的背景。谈到无服务器，AWS 确实让 Lambda 变得非常流行，并引入了这种新的功能即服务范式。但我们看到的一个问题是，它将您锁定在 AWS 生态系统中，因为 Lambda 被严格设计为与生态系统中提供的所有其他亚马逊服务集成，但它不一定为您的开发人员可能希望使用的最流行的开源工具和框架提供内置集成。

这可以追溯到我们一直持有的一个重要观点，即开发社区喜欢使用合适的工具，让他们能够很好地完成工作。他们喜欢选择，不喜欢被告知或被要求只能使用一套工具。这就是裂变背后的一些动机。

现在裂变也是建立在 Kubernetes 的基础上，通过专注于 Kubernetes，它还有一个额外的优势，即它也简化了 Kubernetes 的消费。

**艾伦·希梅尔:**明白了。在我看来，该公司已经将这些 Kubernetes 产品定位为真正处于领先地位，因为我们看到 Kubernetes 和容器真的出现了爆炸式增长。Platform9 的下一步是什么？

**Maskasky:** Kubernetes 和集装箱化是我们 2017 年路线图和战略的重要组成部分。托管 Kubernetes 即将全面上市，我们才刚刚起步。作为我们路线图的一部分，我们还有大量工作要做，我们也在寻求客户的反馈，真正实现产品的云无关性，通过内置集成支持客户希望我们提供的尽可能多的当前点，提供跨这些当前点创建联合集群的能力。因此，简而言之，通过托管 Kubernetes 产品真正实现真正的混合容器即服务愿景。

这是一个。第二大部分是裂变。裂变作为一个开源项目刚刚起步。我们希望让它成为 Kubernetes 内部运行功能即服务的实际方式。因此，我们希望与 CNCF，云计算基金会合作，我们希望 Kubernetes 社区能够真正开始投资。

我们看到一些基层早期牵引采用。社区中有人最近为裂变构建了一个. net 客户端，为 Node.js 提供了开箱即用的支持，并且一直在构建对 Go 的支持。我们听到一些人开始在其内部社区部署的基础上使用裂变，他们告诉我们他们的使用案例。这些都是反馈意见，所以我们希望在此基础上发展壮大。
