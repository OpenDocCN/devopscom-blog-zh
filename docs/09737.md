# 解决 DevOps 劳动力不足问题

> 原文:[https://devops.com/solving-the-devops-labor-deficiency/](https://devops.com/solving-the-devops-labor-deficiency/)

我们都知道 DevOps 人才存在长期的不平衡:高需求和有限供应。看看 2021 年 DevOps Institute 的一份报告就知道了，该报告发现，尽管 60%的 [组织正在招聘此类职位](https://insights.devopsinstitute.com/hubfs/Automation%20Downloads/Upskilling%202021-Enterprise%20DevOps%20Skills%20Report.pdf?utm_campaign=Upskilling%202021&utm_medium=email&_hsmi=121425890&_hsenc=p2ANqtz-8Cj74GN1Jbf2p9NPmTGuUya1m5LSvlybLW5ciT-XUhuW4NlesObWAidVOahKkK-9XUBVrcz_GjA22YVPUo3QEEeGabzQ&utm_content=121425890&utm_source=hs_automation)，但 64%的 IT 主管很难找到熟练的 DevOps 专业人员。这不仅仅是你听说的典型的疫情员工短缺。向远程实践的转变加快了各行业的数字化转型，刺激了对开发人员的需求。

那么 DevOps 劳动力市场到底是什么状况呢？这仅仅是对公司而言，还是对求职者而言——自动化能有所帮助吗？为了庆祝劳动节，我们就这个经典的劳工难题询问了一些 IT 领导。

## 双方的劳工壁垒

Fairwinds 的工程运营副总裁 Elisa Hebert 观察到，DevOps 招聘对招聘公司和候选人来说都很难。“由于新冠肺炎疫情，团队在过去几年里一直长期人手不足，这导致公司希望雇佣能够更加自主运营的高级 DevOps 专业人员，”她解释道。“如果你提出专业化的需求——例如，我们只与 Kubernetes 合作——发现领域知识水平与高级经验相结合真的很难。”

另一方面，她表示，行业惯例要求初级职位需要多年的经验，这使得没有经验的人才很难被聘用:“这是一个需要停下来的不可能的车轮。”

## 自动化拯救了…还是没有

你可能会想，“那么，为什么自动化没有帮助解决这些短缺？”DevOps 自动化确实可以减轻开发人员的工作量。开发和运营团队之间的高效协作可以减少摩擦；更快的反馈循环可以加速部署。领导者可以减少繁琐任务所需的员工数量，而他们的工程师可以专注于更有成就感的工作。

所有这些都有助于通过防止[倦怠](https://devops.com/?s=burnout)和促进更健康的工作场所来留住员工。正如 Demandbase 云运营副总裁 Balakrishnan Viswanathan 所说，“DevOps 横跨两个重要支柱:提高开发速度和提高发布质量。自动化是关键，因为否则的话，倦怠会非常高。”

因此，毫不奇怪，DevOps Institute 的【2021 年技能提升:企业 DevOps 技能报告受访者将自动化评为最重要的技能领域。他们将自动化持续集成、持续交付、持续部署、持续运营和支持以及开发人员的熟练程度列为最有价值的知识。

但是自动化解决方案在这里遇到了 DevOps 劳动力问题:对精通自动化的 DevOps 从业者的需求远远超过了可用性。“从 DevOps 的角度来看，整个技术领域很少有人大规模操作生产系统并实现自动化以减少劳动，这使得这些技能在今天的市场上非常有价值，”Viswanathan 说。

## DevOps 招聘、雇用、保留:解决方案是什么？

面对这些挑战，公司如何提高自动化水平，减轻超负荷工作的工程师的压力？赫伯特说，IT 领导者应该“创造脚手架”

“Fairwinds 有一个跨职能(开发、平台、SRE、R&D、服务、UI/UX)的工程领导团队，他们一起规划战略和战术执行，”她说。“我们是伟大的，所以尽管离别是悲伤和快乐的混合，但它不会击垮我们。”她还建议公司修改他们的职位描述，“去掉所有对工作不重要的额外内容”，指出女性通常不会申请工作，除非她们符合列出的所有标准。

Sauce Labs 技术战略副总裁马库斯·梅雷尔(Marcus Merrell)提到历史上的黑人学院和大学(HBCUs)是一个人才来源，他提醒领导者，“HBCUs 和专注于女孩和有色人种 CS 教育的非营利组织是一个优秀的、未被充分利用的招聘资源。这些机构每年都会培养出成千上万的毕业生，但他们都很难找到工作。

Viswanathan 建议组织提供更多的自动化指导。“公司可以立即开始做的一件事是建立思想领导力，努力展示他们的 DevOps 团队通过开发内容(无论是博客、文章还是白皮书)所做的所有自动化工作。”

## 寻找合适的开发机会

对于陷入困境的 DevOps 求职者来说，没有什么灵丹妙药，但 Merrell 鼓励求职者申请职位，即使他们不具备列出的所有资格。“对于那些认为自己是女性和有色人种的人来说，请记住，工作要求通常只要求 60%到 70%的匹配，”他说。"记住，工作描述是有抱负的，没有一家公司真的期望达到 100% . "

赫伯特还建议不要只看薪水。“如果一家公司能为你提供一个机会，让你通过从事有趣的项目或与能指导你的高级工程人才一起提高技能，那就是一笔无价的财富。”

## 以不同的方式定义 DevOps 劳动解决方案

一些组织现在可能在自动化和足够的人员配备两方面都有所欠缺，但是渐进的改进可以帮助行业实现这一目标。IT 领导相信我们会到达那里。“随着云的采用速度加快，所需的技能和劳动力预计也会同步增长，”维斯瓦纳坦预测道。在此之前，DevOps 招聘经理和 DevOps 候选人都可以采取措施在中间相互见面。
