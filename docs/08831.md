# 时代软件新增 SaaS 版日志管理平台

> 原文:[https://devo PS . com/era-software-adds-SaaS-edition-of-log-management-platform/](https://devops.com/era-software-adds-saas-edition-of-log-management-platform/)

年代软件公司推出了其平台的软件即服务(SaaS)版，用于收集和管理基于其建立的时间序列数据库的日志数据。

与此同时，该公司改进了底层电子搜索日志管理平台的用户界面，it 团队可以自行部署该平台，以简化 IT 问题的故障排除。

![](../Images/ef701232a770511f80694b883145de67.png)

Era Software 营销高级副总裁 Stela Udovicic 表示[Era cloud 版本的日志管理平台](https://www.businesswire.com/news/home/20211104005135/en/Era-Software-Announces-EraCloud-for-Managing-Petabytes-of-Log-Data-in-Real-Time-at-Up-to-90-Lower-Cost-than-Alternatives)提供了一种更方便的方式来访问基于对象存储系统的日志管理平台，该系统可以处理任何类型的日志数据。

她指出，还有一个基于机器学习算法的索引引擎，使该平台能够比竞争对手的平台更好地扩展。

EraCloud 可通过亚马逊网络服务(AWS)市场获得，它收集、索引和分析来自云应用程序、Kubernetes 和 IT 基础设施平台的日志数据，以提供一种以较低成本收集数据的集中方式。除了能够持续监控计费成本，电子搜索集群现在可以在几分钟内完成配置。

Era Software，以前称为 EraDB，今年早些时候[额外筹集了 1525 万美元的融资](https://www.businesswire.com/news/home/20210420005403/en/Era-Software-formerly-EraDB-Announces-15.25-Million-in-Series-A-Funding-and-EraSearch-Cloud-to-Modernize-Enterprise-Data-Management)以推动日志管理平台的开发，该平台提供了对 Elasticsearch 等开源平台的替代，这些平台的可扩展性不好。随着通过弹性搜索管理效率较低的日志数据量不断增加，总成本也随之增加。作为这项工作的一部分，该公司还支持 Elasticsearch 应用程序编程接口(API)的子集，以促进从 Elasticsearch 的过渡。

Udovicic 指出，如今许多组织已经有数万亿字节的日志数据需要管理和存储。

目前还不清楚有多少 It 组织正在考虑日志管理平台的替代方案。然而，随着 IT 团队希望能够存储更长时间的日志数据量的增加(部分是为了提高可观察性)，控制保留所有数据的成本的压力也在增加。

[可观察性](https://devops.com/?s=observability)一般来说，需要将日志、指标和跟踪(一种特殊形式的日志记录)结合起来，以便通过查询数据更容易地确定问题的根本原因。它与监控的不同之处在于，可观察性是使用一组预定义的指标来实现的，这些指标跟踪平台或应用程序是否在预期范围内执行。

在大多数 It 团队普遍实现可观察性之前，可能还需要一段时间，但是这个旅程通常是从找到一种更有效的方法来收集专有和开源格式的日志数据开始的。Era Software 没有对所有这些数据进行标准化，而是以任何可能的格式对这些数据进行分析。

无论如何，IT 团队在生产环境中部署的应用程序中有很大一部分必须进行检测，以实现可观察性。现在的问题是确定如何以合理的成本将从这些应用程序收集的所有数据提供给开发运维团队。
