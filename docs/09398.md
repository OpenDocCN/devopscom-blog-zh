# CloudCasa by Catalogic 扩大与微软 Azure Kubernetes 服务的集成；添加基于角色的访问控制和多租户支持

> 原文:[https://devo PS . com/cloud casa-by-catalologic-expands-integration-with-Microsoft-azure-kubernetes-service-adds-role-based-access-control-and-multi-tenancy-support/](https://devops.com/cloudcasa-by-catalogic-expands-integration-with-microsoft-azure-kubernetes-service-adds-role-based-access-control-and-multi-tenancy-support/)

*Catalogic 与 Clastix 合作，为开发人员提供自助式 Kubernetes 备份和恢复服务*

**新泽西州伍德克利夫湖，2022 年 5 月 17 日—**[领先的智能数据保护解决方案提供商 Catalogic Software](https://www.catalogicsoftware.com/) 今天宣布将其云原生备份即服务平台 [CloudCasa](http://www.cloudcasa.io/) 扩展到 Azure Kubernetes 服务，并与帮助用户在 Kubernetes 上构建多租户架构的解决方案 Capsule 的创建者 [Clastix](https://clastix.io/) 建立新的合作关系。

CloudCasa 保护 Kubernetes 环境，为云原生应用提供网络弹性、可扩展的备份和灾难恢复服务。除了亚马逊弹性 Kubernetes 服务(EKS)之外，该服务现在还增加了与 Azure Kubernetes 服务(AKS)的原生集成。CloudCasa 现在允许 AKS 用户注册他们的 Azure 帐户并自动发现他们的 AKS 集群，跨 Azure 区域和 Azure 帐户集中备份策略管理，并大幅减少恢复 AKS 集群及其数据所需的时间。

最新一轮更新还支持面向企业的基于角色的访问控制(RBAC ),面向开发人员的自助备份和恢复，通过本机 Capsule 集成支持多租户集群，以及支持 Longhorn 卷。

“CloudCasa 已经是亚马逊 EKS 最具网络弹性的备份服务。我们很高兴将同样的功能扩展到 Microsoft AKS。我们的 AKS 集成允许简单的统一部署，识别您的集群在 AKS 中的配置方式，并使用该信息在恢复期间启动类似的集群，而无需维护昂贵的备用集群，”Catalogic 的 CloudCasa 总经理 Sathya Sankaran 说。“对于被迫依赖复杂的自主开发脚本、单一集群管理工具和备份产品的 AKS 用户来说，这是一个游戏规则的改变者，而这些工具和产品对底层云平台一无所知。”

借助完全 RBAC，CloudCasa 现在可以对谁可以执行备份和高级恢复以及数据迁移功能进行精细控制。记录集群在云中的配置方式是数据迁移、自动化集群恢复以及智能跨帐户、跨区域和跨云恢复的关键。

由于 Kubernetes 中缺乏本地多租户，许多组织都在努力应对 Kubernetes 集群蔓延。为了帮助解决这个问题，Catalogic 还宣布与 Clastix 合作，增加对 [Capsule](https://capsule.clastix.io/) 的支持，这是一个开源的多租户解决方案，允许多个团队安全地共享 Kubernetes 集群。通过这种集成，可以在 Capsule 租户和 CloudCasa 用户组之间自动创建和维护映射。这允许租户用户仅备份和恢复他们自己的资源，从而为多租户环境中的 Kubernetes 数据保护提供了一个安全的自助式解决方案。

“Clastix Capsule 提供了一个多租户抽象，允许多个团队共享一个 Kubernetes 集群。通过将 Capsule 与 CloudCasa 的新 RBAC 功能相集成，它现在可以提供自助 Kubernetes 备份和恢复的基本和关键功能，”Clastix 的创始人兼首席执行官 Adriano Pezzuto 说。“我们对 CloudCasa 如何在极其复杂的 Kubernetes 数据保护世界中提供简单的用户体验印象深刻。”
