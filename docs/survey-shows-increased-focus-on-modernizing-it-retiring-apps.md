# 调查显示，人们越来越关注 IT 现代化，淘汰应用程序

> 原文：<https://devops.com/survey-shows-increased-focus-on-modernizing-it-retiring-apps/>

今天公布的一项对 140 名企业架构师和 IT 领导者的调查表明，虽然将应用程序迁移到云仍然是一个优先事项，但人们越来越关注通过淘汰应用程序来减少技术债务。

由企业架构管理和价值流管理(VSM)平台提供商 LeanIX 进行的调查发现，40%的受访者将减少遗留应用程序造成的技术债务和实现遗留系统的现代化视为他们的首要 IT 任务。整整 96%的受访者表示，他们的公司今年至少计划了一个旨在减少技术债务的项目，超过一半(52%)的受访者指出，淘汰 IT 应用程序是他们减少技术债务的主要手段。

近 60%的受访者认为将应用程序迁移到云也是一件头等大事。然而，不到四分之一的受访者(24%)认为云计划是 IT 的重中之重。41%的受访者认为第二重要的云计划是增加 SaaS 应用的数量。

无论采用何种云方法，近四分之三(73%)的受访者表示，企业架构师应该与软件开发团队建立更紧密的联系，几乎相同比例的人同意跟踪软件开发的业务影响的重要性。

然而，只有 16%的受访者认为其安全状况的发展和改善是重中之重，尽管超过三分之二的人认为安全计划(如提高风险评估能力和采用更多零信任 IT 产品)是重要的或非常重要的。

LeanIX 产品管理副总裁张秀坤·罗斯表示，随着 IT 环境的不断扩展和发展，企业架构师(或 IT 部门中负责企业架构的人员)正在努力将业务中断降至最低。他们遇到的挑战是软件开发团队和业务的其他部分之间通常存在分歧。这种差距大小因组织而异；不过，Rose 指出，一般来说，数字业务转型计划应该有助于弥合这一差距——假设企业架构师能够与软件开发团队建立并保持密切联系。

LeanIX 正在为企业架构管理(EAM)平台提供案例，该平台提供了对 IT 环境的可见性。该公司还在该平台上提供了一个 VSM 产品，提供对软件交付过程的可见性。采用 VSM 平台的组织数量仍然相对较少，但随着组织越来越依赖软件来驱动数字流程，预计高级业务和 IT 主管对软件交付时间表的需求将会增加。

与此同时，DevOps 团队与业务战略目标的一致性还有待观察。更快地构建和部署软件的能力并不自动等同于更深入地理解业务需求。事实上，也许有一天，软件的构建和部署速度会超过大多数组织的有效吸收能力，除非他们有一个类似于 VSM 的平台。
