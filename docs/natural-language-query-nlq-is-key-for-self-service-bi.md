# 自然语言查询(NLQ)是自助 BI 的关键

> 原文：<https://devops.com/natural-language-query-nlq-is-key-for-self-service-bi/>

现代[商业智能](https://devops.com/?s=business+intelligence) (BI)和分析软件解决方案提供了比以往更多的探索数据和发现见解的方式。然而，确保企业中的每个人都能在需要时独立使用分析仍然是一个挑战。

为了在整个企业中采用分析，您的工具需要为所有用户量身定制。

虽然当今许多 BI 解决方案提供了简化的体验，但要完全理解它们，还需要一些数据素养和知识。例如，仪表板被认为是自助式 BI 工具，但对于普通人来说，它们仍然过于技术性，难以有效使用。

根据 [Ventana Research](https://www.youtube.com/watch?v=8fLPyGTcOmQ) 的数据，如今，只有五分之二(40%)的组织表示他们的员工可以在没有 IT 帮助的情况下分析数据。为了更好地使业务用户能够独立地询问他们的数据，某些 BI 供应商现在提供了一个[自然语言查询(NLQ)功能](https://www.yellowfinbi.com/campaign/natural-language-query-5-key-benefits-of-guided-approach-to-nlq)，它位于他们的仪表板和报告旁边。NLQ 不需要在传统的仪表板或图表中搜索答案，而是允许分析用户直接询问他们的数据问题。

## NLQ 概述

[自然语言查询](https://www.gartner.com/document/3999300) 是一款自助式商业智能工具，能够让分析用户使用非技术语言提出问题，并立即获得自动生成的答案。使用数据建模和机器学习技术来解析关键术语的问题，NLQ 扫描相关的数据库，并生成定制的报告或图表，提供相关的见解。

自然语言查询功能的集成和实现，以及问题的复杂性和支持的数据类型，因供应商而异。最常见和传统的方法是基于搜索的 NLQ，它在用户界面(通常在仪表板上)的某个位置放置一个自由文本搜索栏，允许 BI 用户开始键入问题，并获得作为潜在答案快速生成的报告和图表列表。

少数供应商也在其 NLQ 解决方案中包含对语音命令的支持。

然而，随着 NLQ 日益成为 BI 生态系统的一部分，一些供应商已经发展了他们的自然语言产品，以解决额外的障碍:用户仍然必须*知道*要问什么问题，以及*如何*使用它来避免不准确或不相关的结果。

这种新方法被称为 [引导的自然语言查询，或引导的 NLQ](https://www.yellowfinbi.com/campaign/guided-nlq) ，它通过对解决方案本身进行编程来进一步发展 NLQ 的概念，以指导用户键入问题，帮助他们使用预定义的序列和建议的提示来构建问题。

这消除了在准确解释用户问题的语义(语言)时的知识负担或任何潜在问题，因为系统本身从头到尾都在帮助他们构建它。他们不需要事先掌握技术知识，也不需要求助于 IT 来开始从数据中寻找答案。

当正确使用时，引导 NLQ 可以帮助使分析的使用更加普遍，并为企业中的每种类型的业务人员(不仅仅是专家)开辟分析。

## 适合每个企业用户的自助服务

NLQ 的目标是帮助不习惯传统的结构化 BI 工具的员工找到他们需要的见解和信息，以通知和做出业务决策，同时也是整个企业高级用户的有用工具。

对于非技术分析用户，NLQ 是一种从数据中获得详细答案的简单方法，比浏览仪表板或图表更快，并将其用作即席报告的跳板。

同样，高级用户可以利用 NLQ，无论是引导式还是传统式，快速生成报告或图表，并在此基础上进一步构建。

使用 NLQ 生成的见解可以添加到现有的分析内容中，如仪表板、报告和数据故事，以相对快速、轻松地添加更多详细信息。

最后，NLQ 可以嵌入到用户工作场所应用程序的界面中，使分析在他们的日常工作流程中更加普遍。他们可以问一个问题，得到一个答案，然后开始探索数据，这比仅仅通过仪表板要快得多，也容易得多。

最终，了解用户不同的自助 BI 需求，以及引导式 NLQ 等新工具如何帮助更多人进一步开放分析，将有助于确保您的分析工具被企业中的每个人采用和使用。
