# 你感觉到招聘危机了吗？

> 原文:[https://devops.com/are-you-feeling-the-hiring-crunch/](https://devops.com/are-you-feeling-the-hiring-crunch/)

经过多年的经验，我们现在可以肯定地说一件关于信息技术的事情:总是有招聘危机，而且总是会很快好转。

几年前，我在 InfoSec 上首次注意到了这一趋势。每个人都人手不足，都在寻找合适的候选人。每个人都写下了他们的招聘文本，就像不存在短缺，他们可以要求月亮，每个人都确信这种暂时的人员短缺将很快结束。

它没有。它仍然没有。而且已经扩散了。总的来说，DevOps，特别是 development，这些天的表现完全一样。而且不会变好。

所以，我的建议是？如果你一定要有一个人，写下你的[招聘文案](https://securityboulevard.com/?s=infosec)，以反映你*必须具备的实际技能*。没有花里胡哨，没有“我们总是包括…”没有“但是如果我们…”雇佣你需要的人。或者雇佣着眼于培训你所需要的人。并提供有竞争力的薪酬。这里没有人会说工资太低，但当人手短缺，或者离开的人比进来的人多时，员工的成本就会上升。事情就是这样。如果你只是雇佣那些你*必须*拥有的人，那么你可以多付他们一点。

同时，投资更多的自动化。让我们面对它；同样，我们可以向 InfoSec 学习。他们用更少的员工做更多的事情——即使“更多”并不是“每个人都希望他们能做的一切”。通过增加自动化的使用，他们至少部分地克服了人员短缺的困境。DevSecOps 正在使更多的空间自动化，因此我们最终将克服信息安全人员短缺的问题。

我们也可以这样做，只要跟随他们的脚步。投资自动化，承担风险，启动自动化操作。从小处着手，但是要开始。在可预见的未来，相对较高的人员流动率和较低的合格申请人数可能会成为现状，因此尽可能多地实现流程自动化并不会在这一点上夺走工作岗位，而是改善对可用工作岗位人员短缺的响应。

所以，总结一下:

*   尽可能雇佣员工，但要专注。如果你想找一个 Azure 的人，不要要求在三大云提供商都有经验。不要要求有 15 年的工作经验。现实一点，专注一点，让你需要的人加入进来。
*   招不到人就培训。让我们面对现实吧，有些人非常熟悉应用程序安全性，但他们从未专门做过 DevSecOps，或者从未使用过您的 DevSecOps 解决方案。培训他们——不管他们是内部的还是被雇佣来接受培训的。
*   对于非关键的长期需求，探索自动化选项。这既包括现在手动完成的事情，也包括根本没有完成的事情。在不雇人的情况下想办法搞定。

继续摇摆。你已经完成了工作——我只是推荐一些方法，让你在努力寻找你需要的额外人员的同时，保持领先，完成更多的工作。自动化和招聘之间的平衡会帮助你晚上睡觉，并让组织的其他人确信它是在球上。
