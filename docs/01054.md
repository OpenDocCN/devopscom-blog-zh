# Automic 通过 OpenStack 集成支持云连续交付

> 原文:[https://devo PS . com/automic-empowers-cloud-continuous-delivery-open stack-integration/](https://devops.com/automic-empowers-cloud-continuous-delivery-openstack-integration/)

华盛顿西雅图。&维也纳—(商业资讯)—商业自动化领域的领导者 Automic 今天宣布与领先的云计算开源软件平台之一 [OpenStack](http://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fmarketplace.automic.com%2Fdetails%2Fopenstack-compute-action-pack&esheet=51277101&newsitemid=20160210006087&lan=en-US&anchor=OpenStack&index=1&md5=ccc918abff21bf0545fa309a7662a5d9) 集成。新的 OpenStack 插件与 [Automic Release Automation](http://cts.businesswire.com/ct/CT?id=smartlink&url=http%3A%2F%2Fautomic.com%2Fproducts%2Fautomic-release-automation&esheet=51277101&newsitemid=20160210006087&lan=en-US&anchor=Automic+Release+Automation&index=2&md5=4ed6f29964b9d431018f267c253bae04) 集成，使用户能够简化 [OpenStack](http://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fmarketplace.automic.com%2Fdetails%2Fopenstack-compute-action-pack&esheet=51277101&newsitemid=20160210006087&lan=en-US&anchor=OpenStack&index=3&md5=b63d3f118e66d845fd44a3a93ec3902d) 计算资源以及应用组件和新版本的部署和供应，从而将应用持续交付到云。OpenStack action pack 可立即从[汽车市场](http://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fmarketplace.automic.com&esheet=51277101&newsitemid=20160210006087&lan=en-US&anchor=Automic+Marketplace&index=4&md5=a0603bfbc0fce6e19153c9cde2169fe5)获得。

Automic[OpenStack](http://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fmarketplace.automic.com%2Fdetails%2Fopenstack-compute-action-pack&esheet=51277101&newsitemid=20160210006087&lan=en-US&anchor=OpenStack&index=5&md5=5598e4ca11ad573f2c918d8d2e28bc8e)action pack 通过在云中自动配置或取消配置 open stack 环境，为 DevOps 工程师、开发人员和系统管理员提供独特的支持，帮助他们创建、部署和运行应用。

[Automic OpenStack 操作包](http://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fmarketplace.automic.com%2Fdetails%2Fopenstack-compute-action-pack&esheet=51277101&newsitemid=20160210006087&lan=en-US&anchor=Automic+OpenStack+action+pack&index=6&md5=42aa412e35f09f365117e55c8336f8e9)使用户能够:

*   在 OpenStack 云环境中自动化服务器计算池。活动包括创建、启动、停止、删除和配置计算资源。一些典型的用例包括按需和按计划设置和销毁环境(例如测试环境)。
*   使用自动化动态扩展和缩减现有计算环境。
*   管理现有环境中的配置更改。

“我们很高兴将 OpenStack 添加到我们目前在 Automic Marketplace 上为客户提供的集成列表中，”Automic 首席技术官兼 OpenStack 创始人 John Purrier 表示。“随着组织推动云基础设施和开发运维实践的采用，[应用发布自动化](http://cts.businesswire.com/ct/CT?id=smartlink&url=http%3A%2F%2Fautomic.com%2Fproducts%2Fautomic-release-automation&esheet=51277101&newsitemid=20160210006087&lan=en-US&anchor=Application+Release+Automation&index=7&md5=6709571285d5d9d632a605e519d870de)支持协调和自动化双方的需求，同时为开发运维团队提供完整的[持续交付](http://cts.businesswire.com/ct/CT?id=smartlink&url=http%3A%2F%2Fautomic.com%2Fcontinuous-delivery&esheet=51277101&newsitemid=20160210006087&lan=en-US&anchor=Continuous+Delivery&index=8&md5=f2248e5749e015651d7a63f0ae4906d4)机制。将 OpenStack 添加到我们现有的集成列表中是我们产品的自然发展，是由我们客户的需求驱动的。”

[DevOps](http://cts.businesswire.com/ct/CT?id=smartlink&url=http%3A%2F%2Fautomic.com%2Fdevops-ara&esheet=51277101&newsitemid=20160210006087&lan=en-US&anchor=DevOps&index=9&md5=50c142ed989574d9dbc2ada3afa05439) 运动是企业开源和混合云采用率增长的自然延伸。虽然 DevOps 运动的很大一部分是由开发引领的，但运营通常是企业采用云的驱动因素。DevOps 是持续交付管道的关键推动者，无缝连接开发人员和运营人员。

混合云战略是大多数企业 IT 部门的主要选择，而 [OpenStack](http://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fmarketplace.automic.com%2Fdetails%2Fopenstack-compute-action-pack&esheet=51277101&newsitemid=20160210006087&lan=en-US&anchor=OpenStack&index=10&md5=7bccd7c7b6c3828e83d9e09f95edfc04) 在创建私有云和公共云的开源软件领域处于领先地位。随着[数百家世界领先组织](http://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fopenstack.org%2Fuser-stories%2F&esheet=51277101&newsitemid=20160210006087&lan=en-US&anchor=hundreds+of+the+world%27s+leading+organizations&index=11&md5=f3f8ac8eb6c756957e7a6b952ee60d3a)依赖 OpenStack，以及预计的[CAGR 31%](http://cts.businesswire.com/ct/CT?id=smartlink&url=http%3A%2F%2Fwww.prnewswire.com%2Fnews-releases%2Fglobal-open-stack-market-2015-2019-300117113.html&esheet=51277101&newsitemid=20160210006087&lan=en-US&anchor=CAGR+of+31%25&index=12&md5=dde2d68de77d56daf3be7cb76e4b7c5a)， OpenStack 成为许多组织混合云战略的核心。
