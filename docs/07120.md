# ibm 收购 instant 推进可观察性战略

> 原文：<https://devops.com/ibm-acquires-instana-to-advance-observability-strategy/>

IBM 本周宣布计划[收购 Instana](https://newsroom.ibm.com/2020-11-18-IBM-to-Acquire-Instana-as-Company-Continues-to-Advance-its-Hybrid-Cloud-and-AI-Strategy) ，这是一家为[微服务](https://devops.com/?s=microservices)优化的应用性能管理(APM)平台提供商。

IBM 混合云管理和 AIOps 副总裁 Robin Hernandez 表示，随着组织开始更广泛地构建和部署云原生应用程序，Instana 将填补 IBM 产品组合中出现的可观测性差距。

Hernandez 指出，以前，IBM 通过 Tivoli 工具组合提供 APM 功能，这些工具针对核心静态 Java 应用程序进行了优化。他说，Instana 将添加一个从头设计的 APM 平台，以便在 DevOps 团队部署应用程序时自动检测应用程序。

Hernandez 补充说，Istana 还提供了现有 Tivoli 产品组合所缺乏的分布式跟踪功能。

注入这些微服务的 Instana 代理收集的数据也将在 IBM 将人工智能应用于 IT 运营的整体方法中发挥关键作用，也称为 AIOps。他说，IBM 的 AIOps 方法寻求将来自 Instana 和 Tivoli 等平台的数据与 ServiceNow(T2)等第三方数据源结合起来，以推进 IT 自动化。

Instana 既可作为软件即服务(SaaS)平台使用，也可作为可在内部部署环境中部署的软件使用，随着组织将可观察性作为一套最佳开发运维实践的核心原则，Instana 获得了越来越多的关注。过去，由于 APM 平台的成本和普遍应用代理的内在困难，IT 团队只使用 APM 平台来监控他们最关键的应用程序。

Hernandez 表示，可观察性将监控提升到了一个新的水平，因为 Instana 等平台不是简单地跟踪一组已知的流程，而是使观察交互成为可能，从而发现应用程序和基础架构之间的未知关系和依赖关系。

IBM 计划将其 Instana 工作主要集中在基于微服务的应用程序上。但是，Instana 也可以用于观察现有的遗留应用程序。他指出，作为持续尽职调查的一部分，IBM 仍在研究该用例。

随着组织向微服务过渡，大量资本涌入了试图取代传统 APM 平台的 observability 初创公司。与此同时，传统 APM 平台的提供商正在重新设计他们的平台，以支持基于微服务的应用程序，这些应用程序通常部署在 Kubernetes 集群上。Hernandez 说，IBM 正在通过一项收购来应对这一挑战，这项收购补充了其对混合云计算和人工智能的重新关注。

现在说哪家供应商将在可观察性战争中获胜还为时过早，但有一点是肯定的，那就是不缺乏选择。目前还不清楚向可观察性的转变会在多大程度上导致各种工具的出现来管理它，但是 It 工具简单地监控过程的日子正在开始消逝。
