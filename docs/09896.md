# 构建 IT 系统管理的新价值

> 原文：<https://devops.com/building-new-value-in-it-systems-management/>

几十年来，公司将 IT 系统视为一个成本中心，与[设施维护](https://devops.com/downloads/key-conceptsin-test-maintenance/)没有太大区别。IT 的工作是保持正常运转，确保公司使用的技术正常工作，并确保维持技术正常运行的成本在可控范围内。那些日子早已过去。不仅技术是运营的必需品，而且每个人现在都认识到公司战略性地使用技术的方式——无论是购买、租赁还是制造技术——都对公司业绩有直接影响。这超越了效率。从[客户获取](https://consent.yahoo.com/v2/collectConsent?sessionId=3_cc-session_75f2efc5-4aa7-4b76-a850-5984b07ee0d5)和参与到竞争差异化和优势，技术是一切的关键。

然而，事实证明，并不是每个公司都能够重新思考和重新配置其 it 系统来满足这些新的需求。这并不奇怪。这不仅需要从最高管理层开始转变思维方式，还需要重组 it 职能，以确保 IT 以一致且可衡量的方式支持业务目标。

虽然将 it 从成本中心转变为利润中心不可能一蹴而就，但任何公司都可以做三件事来将 IT 转变为业务推动者和业务驱动力。

### 1.让您的 IT 系统完全透明

任务关键型 IT 系统需要对组织的 IT 环境有一个透彻的了解。花钱购买重复的应用程序和未使用的许可证是一种浪费。未知的 SaaS 应用程序会带来持续的安全风险。优化支出和提高安全性是支持业务的 IT 的重中之重。

事实上，公司里任何持有信用卡的人都可以订阅 SaaS 的服务，这是一个主要的挑战。手动无法有效跟踪所有 SaaS 应用程序和许可证，以及监控 SaaS 的使用情况。与其他单调乏味的任务一样，我们需要的是自动化。例如，通过将 SaaS 管理平台(SMP)与 SSO 和费用管理解决方案相集成，组织可以自动发现 SaaS。当 SMP 还与公司依赖的主要 SaaS 供应商(微软、谷歌、Salesforce 等)集成时。)，它允许组织跟踪使用情况。

这里的目标不是让一个集中的 IT 功能对组织使用的软件进行更多的控制。相反，我们的目标是战略性地利用自动化来产生数据驱动的洞察力，使组织能够就其使用的技术做出更好、更明智的决策。

### 2.提高工程效率

今天所有的公司都是软件公司。这意味着我们需要将工程部门视为整个 IT 产业的一部分。IT 系统不仅需要像支持任何其他业务功能一样支持工程，而且由工程本身创造的技术不可避免地成为由它管理的更广泛的技术领域的一部分。

任务关键型 IT 至少以两种方式支持工程效率。首先，它应该提供提高工程效率所需的工具和能力。在很长一段时间里，衡量工程的效率要么根本不做，要么基于毫无意义的虚荣度量，比如代码行数。最近，DORA metrics 已经成为经过科学验证的方法，被工程团队广泛信任，用于跟踪效率。它需要能够轻松跟踪这些指标。

它还应该便于为工程团队创建和维护服务目录。这个目录的维护不仅促进了服务的重用，这本身就是工程效率的一个标志，而且使得快速缓解与微服务相关的安全问题成为可能( [Log4j](https://securityboulevard.com/?s=Log4j) 只是过去 12 个月中的一个例子)。

然而，除了效率，它还可以通过帮助工程团队进行[价值流管理](https://devops.com/?s=value+stream+management)来支持内部软件创建，以便他们可以跟踪他们创建的软件的业务影响。虽然效率会对业务产生影响(通过节省时间/成本来实现)，但软件在有效满足客户需求时会推动业务向前发展。它可以通过连接客户功能请求、产品变更和(产品和功能的)增加采用之间的点来促进这些见解。

### 3.使企业架构具有战略性

作为组织的任务关键部分的 IT 功能培养了对软件前景的全面概述，并支持工程效率，正如所建议的，通过将开发环境视为更广泛的技术资产的组成部分。然而，IT 系统领导需要做更多的事情来有效地与业务领导合作，并推动业务向前发展。

具体来说，IT 领导需要让企业了解技术影响业务绩效的所有方式。这意味着强调需要解决的风险，发现应该抓住的机会，并为未来的规划提出建议。深思熟虑的战略性企业架构管理是关键。

企业架构管理的战略方法意味着减少对治理(谁使用什么)的关注，更多地关注降低复杂性以促进敏捷性和快速决策。朝着这个方向的第一步是采用一种通用的语言来协调业务和 IT。业务能力和为不同部门创建业务能力图就是为了这个目的。

为了做出有关技术的决策，企业领导人需要知道他们在做什么。可靠的最新数据至关重要。因此，您的企业架构管理需要优先考虑持续的数据收集和数据质量的维护。

最后，为了使目标和选项透明，它需要能够以一种易于理解和操作的方式战略性地显示所有企业架构数据。

### 你从哪里开始？

驯服一个不守规矩的 SaaS 环境，让开发更好的软件和引导技术策略成为可能，说起来容易做起来难。

好消息是，持续的创新创造了更易于使用、有效利用自动化并与所有相关系统集成的解决方案。换句话说，工具就在那里。我们所需要的是致力于 IT 系统的转型，并将其发展成为关键的业务功能。
