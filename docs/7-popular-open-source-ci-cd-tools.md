# 2021 年最佳–7 款流行的开源 CI/CD 工具

> 原文：<https://devops.com/7-popular-open-source-ci-cd-tools/>

随着 2021 年的临近，我们 DevOps.com 想要突出今年最受欢迎的文章。以下是我们 2021 年最佳系列的第四部。

DevOps 是一种软件开发策略，它结合了敏捷实践，用于快速、高效的产品创建和发布。它侧重于开发和运营团队的集成、持续集成/持续交付(CI/CD)以及任务和流程的自动化。

通常，DevOps 团队使用管道来简化和标准化流程。DevOps 管道是工具链，团队可以使用它们来自动化任务，并提供对软件开发生命周期的可见性。在本文中，我们将介绍七种流行的开源 CI/CD 工具。

## DevOps 在不久的将来会有怎样的变化

尽管 DevOps 已经存在了十多年，但它仍在不断发展。[新的开发运维技术和不断变化的基础设施](https://cloud.netapp.com/devops)促使团队不断适应，技术进步将继续推动变革。以下是 DevOps 在 2021 年及以后的几个发展方向。

### 人工智能和数据科学

人工智能(AI)和机器学习具有改善当前 DevOps 流程的巨大潜力。团队可以使用人工智能来识别管道中的低效率，优化测试或评估用户需求。越来越多的关于环境、客户、市场和个人习惯的数据都可以用来提高生产率，并最终增加收入。

### 无服务器架构

无服务器平台，如包括 AWS Lambda 和 Fargate 的 [AWS 无服务器生态系统](https://lumigo.io/aws-serverless-ecosystem/)和包括 Azure 功能和 Azure 应用服务的 [Azure 无服务器平台](https://azure.microsoft.com/en-us/solutions/serverless/#solutions)，正变得越来越受欢迎。这些平台使 DevOps 团队能够专注于他们的代码，并消除许多其他变量，例如对主机操作系统的关注。这些体系结构提供了难以匹敌的可扩展性、部署速度和用户体验。这一价值将在未来几年进一步保障 FaaS 的安全。

### 云原生

云原生应用的增长既推动了云服务的广泛采用，也受其推动。构成许多云应用的容器和微服务在许多管道部署和环境中使用。高效开发云原生应用需要敏捷性和自动化，如果没有 DevOps 战略，这是很难实现的。

基于以上原因，DevOps 几乎已经成为了云原生的代名词。随着云原生技术的改变，工具和流程也会随之改变。此外，由于云增长并没有放缓，您可以期待 DevOps 将继续关注和完善云原生开发。

### DevSecOps

当在自动化 CI/CD 管道中开发应用程序时，安全性不能是事后才想到的。尤其是在复杂的微服务架构中，安全性必须从规划和开发阶段就内置。每个人都必须了解安全要求，包括开发人员、开发运维工程师和基础设施团队。

过渡到 DevSecOps 范式需要文化变革，但也需要新的技术体系。将安全性向左转移需要将[应用安全工具](https://devops.com/what-is-appsec-an-overview/)紧密集成到开发生命周期中，以及[云安全](https://cloud.netapp.com/blog/blg-cloud-security-solutions-compared-6-solutions-to-consider)解决方案的实施，这些解决方案可以帮助获得可见性，并对复杂的云环境应用自动化安全控制。

## 2021 年及以后你会用哪些 CI/CD 工具？

随着技术的变化，工具也在变化。虽然您可以继续使用与开始时相同的工具，但您至少应该考虑其他工具是否可以提高您的效率。下面是一些你可能想要集成的流行的开源工具。

### 阿尔戈光盘

[Argo CD](https://argoproj.github.io/argo-cd/) 是 Kubernetes 开发的 CI/CD 工具。这是一个开源项目，目前正处于云本地计算基金会(CNCF)的孵化阶段。它使用 git 存储库来存储 Kubernetes 应用程序的状态，监控应用程序，并可以将集群重新同步到所需的状态，如 Git 配置中所示。

这种创新的方法还允许您存储 Kubernetes 应用程序的多个期望状态，使用分支、标签，或者使用 Git commit 锁定清单版本。这为在开发过程中管理 Kubernetes 配置提供了一个灵活的环境。

### 绕圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈圈

[CircleCI](https://circleci.com/) 是一款开源的 CI/CD 工具。它包括作业编排、资源配置、缓存、调试、安全性和仪表板报告等功能。CircleCI 集成了多种工具，包括 GitHub、Heroku、Slack 和 Docker。

CircleCI 有三层，其中一层是免费的。您可以在云中使用它，也可以在本地使用 Linux、Mac 或 Windows 机器。

### GitHub 操作

[GitHub Actions](https://github.com/features/actions) 是最近发布的开源工具特性，您可以使用它来自动化工作流。它使您能够直接从 GitHub 构建、测试和部署代码。您可以基于任何 GitHub 事件进行自动化，包括推送或问题创建。

GitHub 操作包括 matrix 工作流功能、所有主要操作系统的托管运行程序、内置秘密存储和实时日志审查。它支持所有常见的编程语言。

GitHub actions 包括托管运行时环境，每月前 2000 分钟免费使用。如果你需要更多的测试时间，你可以按分钟或批量购买时间。您可以通过将 GitHub 动作包含在您的 YAML 文件中来使用它们。

### GoCD

GoCD 是一个开源的构建工具，可以让你构建 CI/CD 开发管道。它的核心是一个持续集成(CI)服务器，允许您处理复杂的发布工作流。它超越了传统的 CI，允许您构建一个完整的连续交付(CD)管道，从自动化和安全的部署一直到生产。

GoCD 使用签入源代码控制的代码定义管道——这种基础设施即代码(IaC)方法允许您跨多个项目测试、管理和重用管道。管道可以用 JSON 或 YAML 模板来表示。

GoCD 的另一个特性是能够顺序和并行运行管道，并具有可配置的依赖关系。使用价值流图，您可以查看从开发到生产的整个工作流程，并跟踪从最初提交到生产部署的功能。

GoCD 非常灵活，提供了一个插件生态系统，该工具还允许您开发和贡献自己的插件。

### Gradle

Gradle 是一个开源的构建自动化工具，已经存在了大约十年。它继续被广泛采用和支持。虽然它不是一个新工具，但是如果您希望取代 Maven 作为您的构建管理解决方案，它可能是一个不错的选择。Gradle 比 Maven 扩展性更强，速度更快，可用于 C/C++应用的原生开发。

Gradle 包括增量构建、构建缓存和并行执行的特性。它支持多种语言和平台，包括 Java、Scala、Android 和 C/C++。您可以将 Gradle 与一系列工具集成，包括 Eclipse、IntelliJ 和 Jenkins。

您可以在本地或云中的虚拟机上使用 Gradle。Gradle 免费使用；但是，如果您想要企业支持或培训，您需要支付额外的费用。

### 舵

[Helm](https://helm.sh/) 是一个开源的 Kubernetes 包管理器。您可以使用它来定义、安装和升级 Kubernetes 编排的应用程序。Helm 包括大量预构建的图表(可分发的 Kubernetes 资源集)，您可以使用这些图表更快、更轻松地部署服务。可以在 Linux、Mac 和 Windows 设备上使用 Helm。

### 詹金斯

[詹金斯 X](https://jenkins-x.io/) 是众所周知的詹金斯的升级版。它建立在 Jenkins 现有功能的基础上，具有针对 Kubernetes 和 Docker 工作流定制的功能。它包括预建管道、集成 GitOps 支持、自动生成预览环境和反馈集成等功能。

詹金斯 X 集成了泰克顿、普罗、赫尔姆、克纳威和斯卡福德。你可以在 Linux，Mac 或 Windows 机器上使用它。

## CI/CD 和自动化齐头并进

敏捷团队需要自动化工具，尤其是在创建和使用 CI/CD 管道时。开发运维环境是动态的，专为流畅的工作流和不断的变化而构建。自动化有助于开发运维团队保持高水平的可见性，并减少在重复性任务上花费的时间。当将人工智能和数据科学添加到组合中时，除了自动警报和响应之外，DevOps 团队还能够做出数据驱动的决策。
