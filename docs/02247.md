# 为开发运维构建安全反馈流程

> 原文:[https://devo PS . com/building-a-security-feedback-process-for-devo PS/](https://devops.com/building-a-security-feedback-process-for-devops/)

过去几年，所有主要云提供商在安全领域都出现了一些重大失误，导致了不确定性和猜测。那是理解；[云安全](https://www.symantec.com/blogs/product-insights/rethinking-security-public-cloud)是一个极其复杂的课题，因为企业构建和部署应用的速度比以往任何时候都要快，以满足业务需求。如今，云中出现的大多数安全问题都与受威胁的帐户、身份盗窃、恶意软件应用程序，当然还有数据泄露有关，在许多情况下，这些都是业务关键型问题。

鉴于 DevOps 的动态环境和云的性质，这是一个需要解决和修复的复杂问题。最有效的方法是在 IT 管道中嵌入安全性，或者通常所说的安全性左移。如果说 DevOps 是一种协作文化，通过改变的大小来降低风险，那么 DevSecOps 就是内置于每个点的自动化安全控制，以建立一种自我防御机制。

DevOps 是[而不是关于工具](https://www.devopsdigest.com/devops-is-not-about-the-tools)；这是关于哲学和过程的。要在 DevOps 取得成功，最重要的流程之一是加强[持续合作](https://www.itbusinessedge.com/articles/why-collaboration-is-crucial-for-devops.html)的反馈循环。

当涉及到与安全相关的信息时，这当然是正确的。在一个健康的 DevOps 组织中，在交付管道中的任何一点检测到的安全问题都会迅速传达给所有的利益相关者，他们可以相应地做出响应。

在本帖中，我们将探讨构建安全流程的步骤，以确保正确的信息快速到达正确的人手中，并确保团队协作，避免重复劳动。

## 自动获得更好的反馈

自动化是第一步，对于 DevOps 来说是显而易见的。如果没有用于代码分析、配置管理、访问控制、漏洞管理和机密管理的自动化安全工具，很难在动态 DevOps 环境中扩展安全性。随着环境的快速变化，云中的手动工作在许多情况下注定会失败。因此，自动化。建议是尽可能自动化每个安全控制和整个反馈过程。

## 管理角色权限并控制其访问

身份和访问管理已经成为云安全的关键。因为很容易给人错误的访问权限，所以弄清楚谁可以访问什么资源就变得很重要。对信息和资源的访问必须是安全的，但又是容易获得的。所有访问权限都应该授予一个组，而不是个人。将安全性访问映射到组织的层次结构是建立通信循环的有效方法。正确的信息到达正确的人群。能够回答以下问题很重要，以确保沟通循环高效:

*   谁可以删除此虚拟机？
*   某个用户可以访问哪些资源？
*   谁拥有服务帐户访问权限？

显示用户、组、角色、权限和资源之间关系的工具有助于回答这些问题。

## 加密秘密，管理它们的密钥

建立有效沟通循环的另一个重要步骤是控制对应用级机密的访问。可以通过加强相关组之间的信息流来提高安全性，这可以通过使用最小特权原则创建和控制对机密的访问来实现。这些秘密的访问密钥可以保存在集中式秘密管理工具或密钥管理工具中。这些键的旋转可以根据需要自动进行。

## 针对故障进行设计并主动管理漏洞

虽然安全人员可能会在提到“失败”这个词时畏缩不前，但为失败而设计意味着提前思考并制定 b 计划。云在设计上是动态的，事情以令人难以置信的速度发生，经常会导致弹性、配置和云能力方面的问题。

上下文成为管理漏洞的重要参数，因为在任何给定时间都有多个漏洞。自动化漏洞管理和配置管理解决方案可降低保护云环境的复杂性。自动化功能和持续监控提供了对云的深入洞察、智能建议和持续保护，从而确保在反馈周期中有效管理信息。

## 记录一切— *一切*

仅仅嵌入安全左移不足以保证安全。当一些代码被部署时，所有相关的协作组知道状态是很重要的。可见性和反馈的主要来源来自日志。有效的日志策略是安全设计的首要属性。应该在任何可能的地方启用日志记录:数据访问日志、身份日志、管理活动日志、云环境中的日志、操作系统、网络平台等。日志虽然笨重，但却是非侵入性的，是一个重要的可见性工具。在决定对事件本身做些什么之前，您需要知道发生了什么，这将我们带到下一步:监控。

## 监控这些事件

测试部门经常被太多的标志弄得不知所措，这阻碍了他们对关键问题进行优先排序。拥有日志和事件数据是重要的第一步，但绝对必要的下一步是将它们传递到自动化策略中，该策略将把不重要的、大量的问题与业务关键的优先级分开。对触发事件的自动监控使反馈循环更加有效。否则，日志会变成噪音——大量的噪音。

## 警报和通知

我们在代码中嵌入了安全性。我们管理访问和角色。我们打开日志并监控它们。一个事件发生了。现在，我们要通知谁？有效的警报通知策略将确保根据通知过滤器(问题类型的重要程度和相关性)将通知发送给正确的个人组。通知策略指定触发将发送给收件人的通知的条件。通知可以通过电子邮件、短信、语音、推送消息等方式发送。然后，收件人可以确认该警报，这向其他用户表明有人正在处理该问题，从而避免重复工作。

## 结论

总之，通过在产品生命周期的早期暴露漏洞，自动化安全控制和反馈周期可以实现高效的 DevOps 生态系统。自动化、简化的通信和对优先级的适当响应确保了可用性、网络连接性和弹性。同时，它们降低了数据泄露和重复工作的可能性。一切的核心是以安全可靠的方式交付代码的能力，以灵活地满足不断发展的业务需求。
