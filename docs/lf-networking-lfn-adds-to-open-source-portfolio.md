# LF Networking (LFN)增加了开源产品组合

> 原文：<https://devops.com/lf-networking-lfn-adds-to-open-source-portfolio/>

在线[开放网络& Edge 峰会+ Kubernetes on Edge Day](https://events.linuxfoundation.org/open-networking-edge-summit-north-america/) 活动 LF Networking (LFN)是 Linux 基金会专注于网络的一个分支，今天增加了几个项目，这些项目有望使基于开源软件的网络服务的编程管理变得更简单。

沃尔玛正在贡献一个 L3AF 项目，用于管理利用扩展的伯克利数据包过滤器(eBPF)功能的网络应用程序，该功能允许应用程序在操作系统的微内核级别的沙箱中运行。与此同时，英特尔和 Aarna Networks 正在捐赠来自 EMCO 项目的种子代码，该项目提供了一个通用控制平面，用于在从边缘到云的分布式计算环境中安全地连接和部署工作负载。

Linux 基金会网络、边缘和物联网(IoT)总经理 Arpit Joshipura 表示，L3AF 项目为 LFN 的产品组合增加了一个管理这些应用的控制平面，解决了负载平衡、速率限制、流量镜像、流导出器、数据包操作、性能调整和其他管理任务。今年早些时候，Linux 基金会成立了一个独立的 eBPF 基金会，以进一步推动 eBPF 在多个操作系统上的开发和采用。

与此同时，LFN 今天还透露，最初由戴尔创建的用于测量数据传输到设备情况的工具 Project Alvarium 和现有开源轻量级物联网数据分析和流平台 eKuiper 现在是 LF EDGE 项目组合的一部分。

最后，最初由华为创建的开源移动边缘计算项目 Edge Gallery 已经捐赠给 Linux 基金会。

Joshipura 表示，随着越来越多的供应商贡献核心技术，最终无法为他们提供足够的差异化价值来继续自行开发，LFN 继续获得动力。今天的 LFN 还透露，VMware 和 F5 Networks 已加入 LF EDGE 联盟成为高级会员，而 mimik 和 Veena，Inc .已成为 LFN 的普通会员，Eclipse Foundation 已成为准会员。

LFN 还提供了一种机制，这种机制能够比任何一家供应商自己所能实现的更广泛地促进这些技术的采用。Joshipura 表示，随着 LFN 项目的不断成熟，交付基于无线 5G 网络的下一代边缘计算平台的总成本也将大幅降低。

Joshipura 说，总的来说，各种 LFN 项目的更新速度继续加快。为边缘计算平台创建蓝图的 Akraino 项目最近发布了第五次更新，以解决涉及智能城市、云原生多租户环境和车载网络的用例。与此同时，一个面向物联网平台的 EdgeX Foundry 项目最近修改了其应用编程接口(API)，使其更加安全，并支持更多基于消息的通信。Home Edge 项目也将很快发布 Drewberries 更新，该更新将解决代码稳定、脚本、API、数据同步和安全问题。

现在，这些 LFN 项目与 DevOps 工作流更加集成只是时间问题，因为 It 团队希望让开发人员对网络服务有更多的编程控制。现在的问题不是决定是否整合网络运营和 DevOps 团队，而是决定这些团队需要在多大程度上相互融合。
