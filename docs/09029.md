# 应对开发运维及安全人员短缺的 4 种方法

> 原文:[https://devo PS . com/4-对抗 devo PS-and-security-劳动力短缺的方法/](https://devops.com/4-ways-to-combat-the-devops-and-security-workforce-shortage/)

大多数人都痛苦地意识到，近年来安全漏洞不断增加，同时他们的方法也变得更加复杂。此外，不断扩展的应用环境和不断演变的工作负载为攻击者创造了前所未有的机会。对于科技泡沫之外的人来说，不那么明显的是:这个世界在应对这些巨大威胁方面装备不良，这很危险。

组织可能希望通过训练有素的技术人员来应对这一挑战，以维护安全运营。但现实是，有能力的 DevOps 专家严重短缺，尤其是那些拥有网络安全技能的专家。这给在现代云和容器环境中运营的所有组织带来了重大问题。

根据网络安全倡导组织 [(ISC)2](https://blog.isc2.org/isc2_blog/2021/10/isc2-cybersecurity-workforce-study-skills-gap-narrows_2021.html#:~:text=The%20global%20cybersecurity%20skills%20gap,the%20field%2C%20the%20study%20found.) 的说法，这种技能差距对开发人员和安全团队来说有着明显而有意义的后果。从世界各地 4500 多名网络安全专业人士那里收集的反馈表明了这个问题的严重性。超过 30%的受访者表示，他们经常遇到配置错误的应用程序，29%的受访者认为他们的团队修补关键系统的速度太慢，27%的受访者无法对针对其 IT 基础架构的所有活动威胁保持必要的意识。

很明显，网络安全团队的人员配备不足以满足当前和预期的未来需求，流程未得到管理，评估未得到解决，监控也不完整。然而，这不仅仅是雇佣更多的人来完成这些任务的问题。保持 DevOps 环境安全所需的必要人员根本不存在，因为角色的变化快于工作人员。

时间是安全的关键因素，不能坐等解决方案的出现。DevOps 领导和 CISOs 必须做点什么，而且要快。幸运的是，他们可以通过采取以下措施来减轻技能短缺的负担，并继续向云、容器和 Kubernetes 过渡:

## 1.采用自动化

安全和 DevOps 团队必须验证安全控制是否真的如预期的那样工作，而且它们没有减缓开发工作。许多企业执行手动检查来确定这一点，但这是不可伸缩的。自动化是有效做到这一点的唯一途径。公司需要无需手动流程即可分析云活动的工具，以了解事情是否按预期运行，即使在最大规模的部署中也是如此。

在自动化中，结合了速度、灵活性和安全性的均衡因素被发现。借助自动化方法，可以分析和解释云活动，并向开发人员和安全团队发出关于其云和容器环境中异常行为的警报。这有助于在漏洞和问题被利用之前解决它们，减少开发过程中的摩擦，并确保安全部署。

首先，开发运维团队应该利用安全工具来自动构建和定制策略。正确的工具，其中一些可以在像 Falco 这样的开源框架中找到，是为适应容器生命周期和集成而构建的。DevOps 团队可以依靠这些工具来执行策略，并对可能指示威胁的异常行为所导致的问题发出警报。

自动化方法中固有的是分担责任的概念；云和容器活动的安全性被直接放到了云客户的领域中。他们可以根据需要自由操作，但没有任何组织可以忽视现代对速度的要求或必要的安全要求。这要求 DevOps 团队认识到并适应安全监督的需求，同时安全团队必须在不减慢开发和交付过程的同时按照自己的职责范围行事。

## 2.为手头的任务培训人员

如果合适的人不存在，就由安全和开发团队来创造他们。组织培训是增加具有合适技能的人员的关键，每个组织都应该将合适的候选人培养成合格的开发人员和安全专家。

这些人中的许多人已经成为应用程序开发人员或在安全部门工作。你可能会发现一些人处于职业生涯的早期，希望提高自己的技能，并渴望参加在职培训。每个组织都应该鼓励由更有经验的员工进行指导，但也应该制定正式的培训计划，让人们了解开发运维安全的独特需求。

一些大学现在提供专门的网络安全学位或[认证](https://devops.com/?s=certifications)。例如，像印第安纳波利斯的 Eleven Fifty Academy 这样的编码学校，就最新的网络安全实践对学生进行培训，可以成为人才的重要来源。美国军方还提供一些最尖端的技术培训，也是招募高技能安全和 DevOps 玩家的沃土。

## 3.放眼全球

没有单一的、规定的方法来攻击或入侵云和容器环境。方法是多样的，这就是为什么负责安全的人才也应该拥护多样性。做到这一点的一个方法是超越你自己的边界寻找人才。在过去两年的新冠肺炎疫情中，全世界都接受了远程工作，比以往任何时候都有更多的组织能够从全球各地吸引员工来帮助他们解决安全和 DevOps 问题。

更广泛地考虑你可以在哪里找到人才，雇佣那些可以通过测试和工作样本证明他们能力的人。这不仅能扩大潜在员工的范围，还能帮助你把有不同观点和方法的新人带进圈子，从而提高你已经在做的工作。

在我工作的公司里，将近一半的员工都在美国之外。这大大增加了我们的人才库。只要努力一点，全球招聘的主要障碍——时差——就不是问题了，你甚至会发现时差会让你更有效率。

## 4.依靠成熟的解决方案

最后，使用商用产品来支持您的开发运维安全需求。自己构建总是一种选择，但是当涉及到安全性时，这种方法可能会比您花费更多的时间；你负担不起的奢侈品。

在这方面，SaaS 产品最为可靠和有效，因为它们会随着团队需求的变化而扩展和调整。这些工具是专为共同责任模式打造的，因此可以减少分心，让您和您的团队专注于必要的安全和速度相关任务，这些任务对帮助您实现目标非常重要。审查网站、试用它们的能力以及开发人员社区的观点也将非常有助于指导您为您的组织做出最佳决策。

管理安全性和开发运维绝非易事。有了激进的时间表和交付期限，很容易让安全所需的纪律溜走。更困难的是，我们无法雇佣足够的人来满足这些需求。然而，通过创新培训、新的招聘视角和技术的有效利用，任何企业都可以克服这些限制。
