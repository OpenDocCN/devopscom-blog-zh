# Snyk 将安全性构建到 AWS 代码管道中，以减轻开发人员和安全团队的开源风险

> 原文：<https://devops.com/snyk-builds-security-into-aws-codepipeline-to-mitigate-open-source-risk-for-developer-and-security-teams/>

*AWS code pipeline 控制台内的最新集成建立在持续协作的基础上，以在 AWS DevOps 部门、AWS Marketplace 和 AWS GovCloud 内交付 Snyk 产品*

**马萨诸塞州波士顿，2021 年 6 月 8 日—** Snyk 今天宣布，它已经在 AWS 代码管道控制台内部建立了一个集成。这种新的集成允许 AWS CodePipeline 用户将自动化安全控制构建到他们的部署管道中，而不必离开 Amazon Web Services (AWS)控制台，将 Snyk 体验直接带给 AWS 用户，并使他们能够在 AWS 上构建云原生应用程序时更有效地找到并修复开源代码中的漏洞。

随着安全性成为开发团队越来越重要的优先事项，Snyk 与 AWS 密切合作，将安全工具自动化到 AWS 的持续交付服务中，在进行应用程序和基础架构更新时，加速分类并增强开发的敏捷性。

Snyk 在 AWS CodePipeline 控制台中的集成是与 AWS 服务的许多关键集成中的最新集成，帮助客户使用直观、开发人员友好的安全平台在技术堆栈的每个级别将安全性构建到应用程序中。Snyk 允许开发人员和工程团队通过首先检测开发人员 IDE 中的漏洞来加速安全测试。使用 Snyk 功能丰富的 CLI，开发人员还可以保护其工作流的各种元素，如 AWS CodeBuild 和 AWS CodeCommit。AWS 用户可以利用 Snyk 与 Amazon Elastic Container Registry(Amazon ECR)和 Amazon Elastic Kubernetes Service(Amazon EKS)的集成来扫描容器映像和 Kubernetes 集群中的漏洞和错误配置，甚至可以将新的漏洞映射到 Kubernetes 集群中已经部署的任何容器映像。对于无服务器工作负载，Snyk 会扫描 AWS Lambda 函数的漏洞，并通过 CI/CD 阶段控制部署，以确保只允许安全工作负载在生产中运行。

AWS CodePipeline 控制台中的这一最新集成扩展了 Snyk 与 AWS 的持续合作，以确保开发和安全团队都能够利用自动化工具来减轻他们在开发过程中的开源风险，从而积极影响他们的工作方式和相互交互。

*   **开发团队**:可以轻松发现、优先处理和修复其开源依赖关系中的漏洞，比竞争对手更快地交付安全代码，而不必采用中断应用部署的门控安全审计。
*   **安全团队**:能够改进开发流程，确保持续扫描所有应用程序的漏洞，而无需安全团队手动满足现代应用程序开发的惊人速度。

Snyk 业务开发副总裁 Carey Stanton 表示:“Snyk 和 AWS 有一个共同的目标:通过支持自动化和开发人员参与的云原生服务来促进我们客户的数字化转型。“部署管道的自动化是在整个应用程序开发生命周期中采用综合安全方法的关键。Snyk 与 AWS CodePipeline 的紧密互操作性现在使开发人员更容易专注于安全地构建应用程序，利用开发人员友好的输出，而不必管理一套新的工具。”

“AWS 希望让我们的客户能够将他们的时间和资源集中在创新业务上，这就是为什么我们与 Snyk 等安全领导者合作，帮助在开发中自动执行手动安全任务，”AWS 开发工具总监 Ken Exner 说。“我们很高兴与 Snyk 合作，让我们的客户在向市场交付安全应用时更加敏捷。”

ActiveCampaign 全球信息安全副总裁 Chaim Mazal 表示:“作为一家高速发展的企业，我们需要一个能够动态适应我们需求的安全合作伙伴，让我们能够快速扩展我们的安全态势。“凭借 Snyk 的开发者至上理念和全面的云原生应用安全平台，我们不断提高可见性、降低风险，同时定位我们的组织，以通过我们的安全软件开发生命周期实现开发人员生产力和创新的宏伟目标。”

作为 AWS 合作伙伴网络(APN)中的 AWS 合作伙伴，Snyk 已经过技术验证，可与多项 AWS 服务无缝协作，并已获得 AWS DevOps 资格认证以及 AWS Lambda Ready 和 Amazon Linux 2 Ready 认证。

Snyk 最近还宣布推出了Snyk Cloud Native Appliance(Snyk CNA)，这是一个自托管的专用 Snyk 实例，可以直接部署到客户的私有 AWS 环境中，甚至可以部署到 AWS GovCloud 环境中，工作负载最高可达影响级别(IL) 5。
