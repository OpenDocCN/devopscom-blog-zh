# Rocket Software 将 DevOps 扩展到了 IBM I 平台

> 原文:[https://devo PS . com/rocket-software-extends-devo PS-reach-to-IBM-I-platforms/](https://devops.com/rocket-software-extends-devops-reach-to-ibm-i-platforms/)

Rocket Software】增加了对 IBM I 平台的支持，作为其重新推出 DevOps 平台的一部分，该平台也可以在 IBM 大型机上运行。

Rocket Software 数据现代化业务部门总裁 Chris Wey 表示，Rocket DevOps 持续集成/持续交付(CI/CD)平台将使采用 IBM I 中端平台和大型机的组织能够使用通用框架实现应用程序开发和部署的现代化。

大型机应用程序开发的现代化已经有一段时间了。Rocket Software 对 500 多名美国 IT 专业人员进行的调查发现，44%的受访者使用多种 DevOps 工具，但略低于四分之一(24%)的人拥有全面的 DevOps 平台。

Wey 说，Rocket Software 正在扩展其 DevOps 范围，以包括 IBM I 平台，因为依赖该平台的组织与其他组织一样迫切需要更快地构建应用程序。

尚不清楚组织在多大程度上仍在为已经成为古老的 IBM 平台构建新的应用程序。Rocket Software [的一项调查](https://www.businesswire.com/news/home/20220329005125/en/New-Rocket-Software-Study-Finds-80-of-IT-Professionals-Say-the-Mainframe-Remains-Critical-to-Business-Operations)发现，虽然 82%的受访者正在将工作负载迁移到云，但只有 4%的人表示他们正在完全迁移到云。在大型机上运行的大多数任务关键型应用程序都是在线事务处理(OLTP)应用程序，这些应用程序需要的处理能力级别是任何其他平台都无法轻易匹敌的。有很多实例表明，某些类别的工作负载已经从大型机中迁移出来，运行在成本较低的平台上。然而，仍然有许多企业应用程序在大型机上运行得比在任何其他平台上都要快。

Wey 指出，在大多数情况下，组织将在未来几年跨混合云计算环境部署和更新应用程序。他补充说，他们面临的直接挑战是管理 IBM I 系统和大型机的 IT 团队缺乏开发运维技能。

Rocket Software 还提供了一组应用程序开发工具，用于在 IBM 平台上构建现代应用程序。最近，该公司[为 Rocket Open app dev for Z framework](https://www.businesswire.com/news/home/20220301005170/en/Rocket-Software-Underscores-Commitment-to-Open-Source-on-the-Mainframe-with-Updated-Rocket%C2%AE-Open-AppDev-for-Z)添加了对 IBM 为 Python 编程语言创建的开源 Python 软件开发(SDK)的支持，用于构建大型机应用程序。

Wey 指出，最重要的目标是让更多的应用程序开发人员和 DevOps 专业人员更容易访问 IBM 平台，他们现在主要在分布式计算平台上工作。

不管怎样，遗留平台和分布式计算系统之间的鸿沟正在不断缩小。许多组织试图通过集中管理这两类系统来降低 IT 的总成本，而不是继续雇佣和保留独立的 IT 团队。

IBM I 平台的历史可以追溯到 AS/400 系统，该系统于 1988 年首次推出，是数字设备公司(DEC)的 VAX/VMS 系统的中型综合小型计算机的竞争对手。30 多年过去了，在 VAX/VMS 系统几乎消失很久之后，仍然有许多 IBM I 平台在运行。
