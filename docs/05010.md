# 2019 年最佳:开发运维工程方法的时代已经到来

> 原文:[https://devo PS . com/the-time-have-for-a-engineering-approach-to-devo PS/](https://devops.com/the-time-has-come-for-an-engineering-approach-to-devops/)

在我们结束 2019 年之际，我们 DevOps.com 想要强调今年最受欢迎的五篇文章。以下是我们为期一周的 2019 年最佳系列的第三部。

你想知道怎么做 DevOps 吗？你在做 DevOps 但是对得到的结果不满意吗？DevOps 很复杂。它不附带处方、安装指南、用户手册或维护手册。它甚至没有一个标准的定义。如果没有清晰明确的指导，怎么能指望 DevOps 会有好的结果呢？有大量的书籍描述了 DevOps 和客户用户故事的不同方面，但直到现在还没有一本书将 DevOps 视为一个工程问题，并提供了一步一步的工程解决方案和一个清晰的推荐工程实践列表来指导实施者。

我的书,《工程开发运维:从混乱到持续改进……及其他》,提供了一个循序渐进的工程处方，领导者和实践者可以遵循这个处方来理解、评估、定义、实施、操作和发展他们组织的开发运维。它为 DevOps 提供了一系列独特的工程实践和解决方案。通过将本书内容的范围限制在工程实践的水平，内容适用于最广泛的实现范围。

这本书的诞生源于我想帮助别人做 DevOps 的愿望，再加上个人强烈的挫败感。沮丧来自于听到领导者和实践者说，“我们认为我们正在做 DevOps，但是我们没有得到我们期望的业务结果。”

有许多文档化的参考资料、课程和其他信息来源解释了 DevOps 的许多不同方面，但没有一个全面的逐步说明性指南详细说明如何为给定起点的任何组织设计 DevOps 解决方案。

工程开发运维采用战略性方法，应用工程实施原则，并专注于运营专业知识，以定义和实现开发运维之旅每一阶段的具体目标。这本书引导读者经历了一个旅程，从定义 DevOps 的工程策略到使用工程实践实现 DevOps 成熟度的三种方式:第一种方式(称为“持续流”)到第二种方式(称为“持续反馈”)，最后是第三种方式(称为“持续改进”)。这本书的目的是作为一个指南，随着时间的推移，随着你的特定 DevOps，以及更一般的 DevOps 的发展，它将继续发挥作用。

## **本书的结构**

本书是一份工程参考指南，分为以下五个部分:

第一部分:什么是工程开发，为什么它很重要？分为三章。第一章，“什么是工程开发？”，解释了本书中使用的工程概念和术语，这些概念和术语已被证明适用于我在咨询经历中遇到的许多客户。我对 DevOps 工程蓝图的描述是作为一个有用的、实用的“大图”参考来讨论 DevOps 的主要工程部分。第二章，“开发运维的九大支柱”，描述了开发运维实践的分类，我发现这是开发运维工程实践的有用特征，在工程开发运维实施时易于应用。第三章，“为什么工程开发很重要？”，解释了采用工程方法进行工程开发的好处。

**第二部分:DevOps 的工程人员、流程和技术**为更高层次的 DevOps 工程蓝图提供了推荐工程实践的全面解释。DevOps 转换通常不使用严格的工程原理来理解或应用。工程实践分为九章。“DevOps 应该如何设计？”包括对三个维度(人员、流程和技术)的 DevOps 工程成熟度级别的讨论，27 个 DevOps 工程关键成功因素和精益 DevOps 价值流管道工程。

在接下来的章节中，我将为 DevOps 工程蓝图的顶层描述推荐的工程实践，包括“价值流管理(VSM)”、“应用发布自动化(ARA)”、“版本管理”、“持续安全(又称 DevSecOps)、“服务目录”、“治理”、“站点可靠性工程(SRE)”、“灾难缓解”和“恢复”

**第三部分:为 DevOps 设计的工程应用、管道和基础设施**全面解释了 DevOps 工程蓝图较低层次的推荐工程实践，分为以下六章:“DevOps 应用工程”、“CI/CD 管道工程”、“弹性基础设施工程”、“持续测试工程”、“持续监控工程”和“持续交付和部署工程”

**第四部分:DevOps 七步改造工程蓝图**为我实现和发展 DevOps 的方法提供了描述和工具。这七个步骤是设想、协调、评估、解决方案工程、实现、操作和扩展。通过使用这些工具，DevOps 领导者和实践者可以在整个组织内创建、实施、操作和扩展他们的 DevOps。本章进一步解释了如何将 DevOps 从成功的第一种方式 DevOps(持续流程)发展到实现更高级的第二种方式(持续反馈)和第三种方式(持续改进)DevOps 实施。这一部分包括对“超越持续改进”的讨论——着眼于塑造未来 DevOps 的新兴技术，以及如何让你的 DevOps 和你自己为未来做好准备。本部分讨论了如何建立有效的开发运维工程培训计划，以支持维护和增强开发运维所需的开发运维工程技能的持续学习。

第五部分:附录、持续学习和参考资料包括我发现对工程开发最有用的材料和资源。
