# Tricentis 收购 SpecFlow 以支持大规模敏捷开发

> 原文:[https://devo PS . com/tricentis-acquisites-spec flow-to-support-agile-development-at-scale/](https://devops.com/tricentis-acquires-specflow-to-support-agile-development-at-scale/)

*Tricentis 通过全球领先的行为驱动开发(BDD)平台扩展了对开源社区的支持。网络*

**加州山景城，2020 年 1 月 15 日**–devo PS 持续测试领域的领导者 Tricentis 宣布收购了 SpecFlow，这是最大、最值得信赖的 BDD 解决方案。全世界的. NET 开发者。SpecFlow 通过实例说明的实用方法已经帮助敏捷开发团队改善了与业务涉众的协作，以构建和交付更高质量的软件。

SpecFlow 将继续为软件开发和测试社区提供免费的开源产品。 [SpecFlow+](https://specflow.org/plus/) 、SpecFlow 的商业产品和[用于用户故事映射的 Azure DevOps 扩展 SpecMap](https://specmap.cc/) 现在将免费提供，以满足在整个企业范围内扩展 BDD 的需求——所有这些都由 Tricentis 在持续测试方面的领先创新提供支持。

通过此次收购，Tricentis 继续扩大其在开源和开发者主导的测试市场的领导地位。SpecFlow 是 Tricentis 在过去一年中在该市场的第二次收购，未来几个月还将进行进一步的收购。2019 年，Tricentis 收购了 [TestProject](https://testproject.io/?utm_source=tricentisnews&utm_medium=PR&utm_campaign=SpecFlow-Tricentis) ，这是世界上第一个利用 Selenium 和 Appium 的免费、基于云的测试自动化平台。收购 SpecFlow 增加了对 BDD 和的一流支持。NET 开发者来完善 Tricentis 的全面开源测试平台。

Tricentis 创始人兼首席战略官 Wolfgang Platz 解释说:“BDD 在企业的数字化转型中正变得越来越重要。“随着业务需求以 DevOps 的速度变化，团队需要一种方法来记录他们的应用程序在高度集成的系统中应该如何运行。如果没有这种共享理解的内置测试自动化，软件交付会导致产品错误和糟糕的用户体验。SpecFlow 创建这种共享业务理解的能力，以及自动化测试，使其成为最值得信赖的 BDD 平台。NET 开发。”

[SpecFlow](https://specflow.org/) 由 TechTalk 开发，作为 Cucumber for 的开源端口。NET 在 2009 年。从那时起，SpecFlow 已经聚集了一个庞大的开源社区，下载量超过 960 万次，用户群继续以每天超过 3000 次的速度增长。

SpecFlow 提供了一种在中定义、管理和自动执行人类可读验收测试的方法。NET 项目。通过使用人类可读的语法，称为 Gherkin(“Given-When-Then”)，SpecFlow 帮助软件交付团队就他们的系统应该如何运行建立一个共同的业务理解。它与 Selenium 和 Microsoft Visual Studio 等流行的开发工具的集成使 SpecFlow 成为 BDD 标准，以弥合开发人员、测试人员和业务用户之间的沟通鸿沟。

TechTalk 的所有者兼管理合伙人 Christian Hassa 表示:“我为 SpecFlow 多年来培育的社区感到非常自豪。“TechTalk 很高兴能在 Tricentis 找到一个致力于 BDD 社区的合作伙伴，并在保持 SpecFlow 独立性的同时显著扩展我们的愿景。”
