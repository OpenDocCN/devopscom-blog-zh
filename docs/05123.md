# 随着产品正式上市，Cloudbees 是 Google Cloud Run 的发布合作伙伴

> 原文:[https://devo PS . com/cloud bees-is-a-launch-partner-for-Google-cloud-run-as-product-goes-ga/](https://devops.com/cloudbees-is-a-launch-partner-for-google-cloud-run-as-product-goes-ga/)

*行业领导者合作提供简化的容器化应用部署，更好地访问 Google 云平台市场上的 CloudBees 解决方案*

**谷歌云接下来，伦敦和圣** **加州何塞。–2019 年 11 月 20 日—**[推动持续经济发展的企业开发运维领导者 CloudBees](https://www.cloudbees.com) ，今天宣布延长与谷歌的合作伙伴关系。作为 Google Cloud Run 的启动合作伙伴，CloudBees 将为开发者提供更多部署容器化应用的灵活性。此外，四种 CloudBees 解决方案——CloudBees Core、CloudBees Jenkins Distribution、CloudBees Jenkins Support 和 Cloud bees Accelerator——在 GCP 市场上的推出，使得获得先进的 Cloud bees 和 Google Cloud DevOps 技术来创建、部署和管理高质量的软件变得前所未有的容易。

[GroupM](https://www.groupm.com/) 已经实施了 CloudBees Core 和 GCP 技术来加速其 DevOps 转型。群邑的工程总监 Moshiur Khondokar 说:“在我们的业务中，我们必须保持敏捷，以确保我们总是为正确的人提供正确的广告。“将 Google Cloud 的无限可扩展性与 CloudBees 的速度、质量和安全性相结合，使我们在不断发展我们的可寻址广告能力以支持全球客户时获得了优势。"

Google Cloud Run 是一个无服务器计算平台，它将基础设施管理抽象化，让开发人员能够专注于构建优秀的应用程序。CloudBees 核心用户第一次可以直接从 Kubernetes 向 Google Cloud Run 部署无状态容器。这简化了开发过程，减少了时间和压力。此外，由于 Google Cloud 仅对使用的资源收费，客户现在可以轻松部署，避免因过度配置资源而产生不必要的成本。

“CloudBees 正在提供关键的 DevOps 功能，使开发人员能够按照自己的方式编写代码，我们很高兴通过谷歌云平台市场提供他们的解决方案，”谷歌云合作伙伴关系总监 Manvinder Singh 说。

GCP 市场提供了 160 多种流行的开发堆栈、解决方案和服务，它们都经过优化，可以在谷歌云平台上运行，并且可以即时访问。CloudBees 还宣布在 GCP 市场推出 [CloudBees Core](https://console.cloud.google.com/marketplace/details/cloudbees/cloudbees-core?q=CloudBees&project=ask-cloudbees-com) 、 [CloudBees Jenkins 发行版](https://console.cloud.google.com/marketplace/details/cloudbees/cloudbees-jenkins-distribution?q=cloudbees&id=eccee7b8-70a5-4ac5-9322-6a6d0b751675)、 [CloudBees Jenkins Support](https://console.cloud.google.com/marketplace/details/cloudbees/cloudbees-jenkins-support?q=CloudBees&id=22f31b54-c5c2-4daf-914d-7299b5585f5e&project=ask-cloudbees-com) 和 [CloudBees Accelerator](https://console.cloud.google.com/marketplace/details/cloudbees/cloudbees-accelerator) 。使这些产品在市场上可用消除了另一层复杂性，使客户能够快速部署应用程序，并将 GCP 服务和 CloudBees 解决方案的支付整合到一张账单上。

“在当今快节奏的商业环境中，企业正在寻找任何可以减少摩擦的方法，”CloudBees 业务发展和战略联盟副总裁 Jason Mero 说。“通过与 Google Cloud 合作，我们已经为组织扫清了道路，让他们能够使用最好的技术，以最佳效率构建和运行应用程序。开发人员可以安心工作，他们可以在其选择的平台上构建和部署容器化的应用程序，同时获得 CloudBees 的统包、弹性、Jenkins 即服务资源的所有优势。"

**关于 CloudBees**

CloudBees 通过提供世界上第一个端到端连续软件交付管理系统(SDM)为连续经济提供动力。对于数百万推动各种规模企业创新的开发人员和产品团队来说，SDM 建立在持续集成(CI)和持续交付(CD)的基础上，使软件交付组织内部和周围的所有功能和团队能够最好地合作，以扩大价值创造。

CloudBees 是 CI、CD 和应用程序发布流程编排(ARO)的核心，建立在其产品的商业成功及其开源领先地位之上。CloudBees 是[詹金斯](https://jenkins.io/)和[詹金斯 X](https://jenkins-x.io/) 的最大贡献者，也是[持续交付基金会](https://cd.foundation/) (CDF)的创始成员。从全栈开发人员实践 NoOps 的创业公司到大型财富 100 强公司，CloudBees 使所有软件驱动的组织能够在正确的时间智能地部署正确的功能。

超过 3，500 个世界知名品牌和超过 50%的财富 500 强企业依赖 CloudBees，因为它能够在任何开发环境中跨任何云工作，并平衡公司治理和控制与开发人员的灵活性和自由度。CloudBees 是世界领先的 DevOps 专家的家园，帮助数千家公司利用“持续一切”的力量，将他们置于从伟大的想法到伟大的软件，再到伟大的商业价值的最快路径上。

CloudBees 由 Matrix Partners、Lightspeed Venture Partners、威瑞森风险投资公司、Delta-v Capital、Golub Capital 和 unregular Ventures 支持，于 2010 年由前 JBoss CTO Sacha Labourey 和一个由持续集成、持续交付和 DevOps 专业人士组成的精英团队创立。在 [Twitter](https://twitter.com/CloudBees) 、[脸书](https://www.facebook.com/CloudBees)和 [LinkedIn](https://www.linkedin.com/company/cloudbees) 上关注 CloudBees。
