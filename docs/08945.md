# 2021 年最佳编程语言:明智的选择？

> 原文:[https://devops.com/programming-languages-choose-wisely/](https://devops.com/programming-languages-choose-wisely/)

随着 2021 年的临近，我们 DevOps.com 想要突出今年最受欢迎的文章。以下是我们 2021 年最佳系列的第一篇。

在这一点上，我们在编程和语言采用方面已经有了几十年的经验，我们可以肯定地说，一般开发人员(特别是 DevOps 工程师)应该知道一些事情。

第一，没你想的那么重要。真的没有。大多数开发人员不会根据优化或一般适用性等重要因素来选择编程语言。他们选择语言是基于易用性、第三方库的可用性以及 UI 等方面的简化。开源版本的可用性有所帮助，但仅限于产生更多的第三方库。所以，使用最适合这个项目的语言，不要太在意它是否是最新的闪亮的语言。

第二，重要的使用和采用的变化——占所有专业编程活动绝大部分的前五到十种语言——不会在一夜之间发生。JavaScript 和 Python 在起飞时都被认为是“快速上升”……但在采用高峰出现之前，两者都存在了*年*。因此，学习任何一种顶尖的语言都比学习最热门的新语言是更好的长期投资。

第三，那些顶级的[语言](https://devops.com/?s=languages)其实变化不大。它们是为了满足需求而编写的，并且不会随着时间的推移而改变。的确，我能想到的唯一一种在其有生之年发生了根本性改变的语言是 C++，它似乎想跟上时代的步伐，而不是继续服务于它最初的小众。Python？Java？仍然和它们流行的时候一样。这是件好事。但这意味着，如果你想尝试一些新的和有吸引力的东西，你需要寻找新兴的语言。在写这篇文章的时候，像 R 和卡夫卡这样的专业语言正在走向成功，这是一件好事。毕竟，我们知道不同的应用程序有不同的需求，不同的平台有不同的需求——并且一直在试图解决第二个问题，目前使用的语言有 Flutter。所有这些将提供新的做事方式，这是很好的曝光。

第四，(虽然我们暂时忽略了这一点)组织确实决定了可用语言的池。坦率地说，从长期维护的角度来看，允许每个团队构建一个单独的架构从来都不是一个好主意…但是相当多的组织都在玩这个主意，并且重新学习了关于技术债务的课程。现在我们回到“我们使用这些语言，选择一种”，这比“我们是 X 商店”更好，并且随着时间的推移提供可维护性，而不消耗大量的工时。

最后，你可以用你的组织提供的语言做任何事情。我见过面向对象的汇编程序，我见过用 C 语言服务的整个网站；这样的例子不胜枚举。您选择的语言使某些事情变得更容易或更难，但是如果您需要完成它，您将会得到语言列表的一个例外，或者您将会找出如何用可用的语言来完成它。但是你*可以*……但是正如我父亲过去常说的，“仅仅因为你*可以*，并不意味着你*应该。他与编程无关，也尽可能少地与计算机有关，但他的逻辑仍然完美地适用。*

所以，找一种被认可的语言，并想出解决方案。就一直把它开回家；你在摇摆。不要停下来，也不要太担心你在使用哪种语言，只要专注于语言，做需要做的事情——就像你一直以来做的那样。让我们开发出更酷的应用。
