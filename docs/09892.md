# JFrog 向 CD 基金会捐赠 Pyrsia 以保护软件供应链

> 原文:[https://devo PS . com/jfrog-gives-pyrsia-to-CD-foundation-to-secure-software-supply-chains/](https://devops.com/jfrog-gives-pyrsia-to-cd-foundation-to-secure-software-supply-chains/)

在本周的 KubeCon + CloudNativeCon 北美会议上，JFrog 宣布[向持续交付(CD)基金会](https://www.businesswire.com/news/home/20221025005480/en/JFrog-Led-Open-Source-%E2%80%9CPyrsia%E2%80%9D-Initiative-to-Secure-the-Software-Supply-Chain-will-be-Contributed-to-the-CD-Foundation)捐赠了 Pyrsia 项目，该项目使用区块链技术保护软件包。

JFrog 开发者关系副总裁兼 CD 基金会董事会成员 Stephen Chin 说，目标是增加项目贡献者的数量。Pyrsia 项目的当前贡献者包括 Docker 公司、DeployHub、Futurewei 和 Oracle。

作为这些努力的一部分，CD 基金会将带来一个集中的治理模型，通过该模型将定义项目的路线图。

Pyrsia 为 IT 团队提供了一个去中心化的[区块链](https://devops.com/?s=blockchain)网络来访问一个不可变的软件包存储库，该存储库被设计为与开发人员已经使用的软件包管理系统无缝集成。它为可以合并到软件材料清单中的代码创建了数字签名的、不可变的证据链。

在去年发现影响 Java 应用程序的零日 [Log4Shell](https://devops.com/?s=Log4Shell) 漏洞后，保护软件供应链成为一个更紧迫的问题。许多开发人员经常重用开源软件，但是这些项目中有许多是由少数程序员维护的，这些程序员自愿贡献他们的时间和精力来构建其他人可以免费使用的组件。像任何其他开发人员一样，这些人拥有的安全专业知识是有限的；确保软件安全的责任落在决定部署软件的组织身上。

问题是，许多依赖开源软件的组织认为它比实际更安全。像 Project Pyrsia 这样的计划是使维护者更容易保护开源软件的更大努力的一部分。

尚不清楚安全问题在多大程度上促使组织审查他们使用的开源软件的数量。大多数组织比他们意识到的更依赖于开源软件，因为大多数打包的应用程序包括大量从任意数量的存储库下载的开源代码。每当发现零日漏洞时，组织可以很容易地花费几个月来寻找可能易受攻击的开源组件的所有实例。

理论上，区块链平台和 DevSecOps 最佳实践应该通过减少生产环境中的漏洞数量来使保护软件包变得更加容易。组织今天面临的挑战是由于遗留应用程序中的漏洞而存在的大量技术安全债务。基于更安全代码的应用程序取代现有解决方案可能需要近十年的时间。

与此同时，保护软件供应链的多种方法正在发展，包括使用某种类型的不可变平台。这些平台被广泛纳入软件供应链可能还需要一段时间，但很明显，现在更多的是需要多长时间的问题，而不是这些技术是否将被应用于保护软件开发环境。
