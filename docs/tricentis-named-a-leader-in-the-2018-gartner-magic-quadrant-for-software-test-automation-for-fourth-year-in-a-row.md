# Tricentis 连续第四年被评为 2018 年 Gartner 软件测试自动化魔力象限的领导者

> 原文：<https://devops.com/tricentis-named-a-leader-in-the-2018-gartner-magic-quadrant-for-software-test-automation-for-fourth-year-in-a-row/>

**Tricentis 位于视轴完整性的最右侧**

**加州山景城，2018 年 11 月 29 日**–持续测试和软件测试自动化领域的领导者 Tricentis 今天宣布，连续第四年被评为 Gartner 2018 年软件测试自动化魔力象限研究报告的领导者。Tricentis 是过去四年中唯一一家出现在领导者象限的公司。

**免费报告访问:** [**Gartner 2018 软件测试自动化魔力象限**](https://www.tricentis.com/resource-assets/gartner-magic-quadrant-software-test-automation/) **。**

根据该报告，“云和数字业务要求向持续质量转变。应用领导者必须支持他们的团队在他们的 DevOps 工具链中集成并有效利用测试自动化工具，以实现持续交付。”

Tricentis 首席执行官 Sandeep Johri 解释说:“数字化转型的目的是确保卓越的客户旅程，如果没有持续的测试，您根本无法实现这一目标。“无论您是通过移动应用开发定制的用户体验，还是将 SAP 工作负载迁移到云(例如，使用 SAP S/4HANA)，这一切都取决于您交付软件的速度。正如研究和实践表明的那样，持续测试对于快速评估每个版本是否满足您的期望或者是否会对您的业务产生负面影响至关重要。”

Tricentis 连续第二年位于视轴完整性的最右侧。在对愿景完整性的解释中，该报告指出，“拥有愿景完整性的领导者在通过令人信服的信息进入市场以及在全球范围内提供可靠的解决方案方面能力很强。随着企业参与到涉及敏捷、开发运维以及双模实践的多年转型中，厂商创建测试自动化愿景的能力将是至关重要的，这种愿景将始终如一地创造价值，并推动积极的变革。现在的关键信息是围绕生产力和帮助技术和流程转型的能力。”

“实现优化的连续测试并不像选择‘最好的’测试自动化工具那么简单；这是一个涉及人员、流程和技术的旅程，”Tricentis 创始人兼首席战略官 Wolfgang Platz 解释道。“在 Tricentis，我们有幸与许多杰出的质量领导者合作，引导数百家顶级企业走上这条道路。我们认为这种承认也属于他们；我们认为，没有他们的持续支持和合作，我们不可能获得这一认可。”

# 最新的补充:新的 Tricentis 敏捷开发测试

Tricentis 正在将 qTest 敏捷开发测试添加到其产品组合中，而 Gartner 正在研究和制作 2018 年软件测试自动化魔力象限。这意味着 Tricentis qTest 不包括在本报告的范围内。

对于敏捷团队来说， [Tricentis qTest](https://www.tricentis.com/qtest-test-management-tool/) 增加了一套软件测试工具，旨在提高整个软件测试生命周期的速度、效率和协作。它提供:

*   **敏捷测试管理**通过手动和自动测试可追溯性，实时跟踪亚特兰大吉拉的问题、缺陷和发布。
*   **利用开源测试自动化管理(使开源测试工具更有生产力)、测试脚本调度和对 BDD 的支持来改变质量策略**。
*   **预测性测试分析**通过管理层仪表板和组合层报告来衡量发布准备情况。

这些新的敏捷开发测试功能补充了该公司的旗舰产品 [Tricentis Tosca](https://www.tricentis.com/software-testing-tools/) 连续测试，该测试提供了跨任何架构或应用堆栈(150 多种技术)的弹性端到端回归测试。此外， [Tricentis Flood](https://flood.io/) 与 Tricentis Tosca 和 Tricentis qTest 集成，使开发人员和测试人员能够即时访问基于云的性能实验室，进行按需负载测试。

完整的平台支持:

*   [**跨所有企业系统(web、SAP、大型机、API、移动…)的持续测试**](https://www.tricentis.com/what-is-continuous-testing/)
*   **跨团队、项目和应用程序的测试协调**
*   **集中** **所有团队的可见性和可追溯性**

关于 Tricentis凭借业界第一的持续测试平台，Tricentis 被公认为是 DevOps 软件测试的革新。通过基于风险的测试、无脚本的端到端测试自动化以及业界最广泛的技术支持，Tricentis 突破了传统软件测试方法所面临的障碍。我们的创新技术简化了即使是最复杂的企业应用程序的测试——将测试从路障转变为创新的催化剂。结果是加快了软件交付速度，提高了成本效率，并降低了业务风险。
