# 提高开发运维工作效率需要可见性和自动化

> 原文:[https://devo PS . com/boosting-devo PS-productivity-requires-visibility-automation/](https://devops.com/boosting-devops-productivity-requires-visibility-automation/)

根据 Software 最近的 [2021 代码时间报告](https://www.software.com/reports/code-time-report) ，工程团队在开发过程中不断与路障和瓶颈作斗争。

这项研究基于超过 25 万名使用[](https://www.software.com/product/code-time)代码时间插件的开发人员的反馈，发现每个开发人员每天的代码时间中位数是 52 分钟，即每周 4 小时 21 分钟。

数据还显示，包括周末在内，只有大约 10%的开发人员每天花费超过两个小时进行编码，而大约 40%的开发人员每天花费超过一个小时进行编码。

该研究还表明，开发人员正在与分心的事情做斗争，这些事情包括太多的会议和太多的中断，以及在流程和系统上太多的等待时间。

## 分心、干扰和会议困扰着 DevOps

结论是，大多数公司没有有效地部署他们的开发和 DevOps 团队，使工程师陷入分心、中断和会议以及系统效率低下的困境，例如缓慢的审查、缓慢的构建和糟糕的工具。

该软件的首席技术官梅森·麦克莱德(Mason McLead)表示，根据该公司的估计，开发人员每天在编辑器中的其他类型的工作上额外花费 41 分钟，如阅读代码、审查[拉请求](https://devops.com/?s=pull+requests)和浏览文档。

“确保 DevOps 团队最高生产力水平的关键是可见性，”他解释道。“我们都倾向于关注显而易见的东西，但最大的瓶颈往往很难发现，也很难衡量。”

McLead 表示，通过以测量系统级指标而不是个人绩效的方式提高整个开发生命周期的可见性，DevOps 团队可以专注于最大的约束，并对他们团队的生产力产生更有意义的影响。

他说，一个隐藏障碍的生产率指标的例子是提前期。

“我与许多经理交谈过，他们过分关注审查拉动式请求需要多长时间，以此作为最大限度缩短交付时间的关键手段，”他说。但是，在发布新功能的过程中有许多阶段可能实际上是关键的限制因素，例如，在转移环境中测试和验证更改需要多长时间，或者将合并的“拉”请求发布到生产环境需要多长时间

虽然这些提前期阶段都是可测量的，但 McLead 解释说，大多数 DevOps 团队都没有设置数据管道或 BI 工具来测量它们。

他补充说，一旦一个组织能够诊断出关键约束，就很容易提出一个有效的解决方案来最小化该约束。

例如:如果创建拉取请求需要很长时间，那么票证的范围是否正确？您能看出拉取请求的大小与批准和合并所需的时间之间的关系吗？

McLean 指出，如果拉动式请求需要很长时间才能得到审核，那么确保设定适当的期望值就非常关键。

他补充说，重要的是要确定是否有一个适当的系统来提醒审查者，并在一天中留出足够的时间来进行审查。

“如果目前将变更部署到生产需要很长时间，采取措施进一步自动化部署过程是另一个潜在的解决方案，”McLead 说。“这实际上可以归结为能够衡量最大的限制，这样你就知道在哪里花时间了。”

## 远程工作和可见性

远程工作也扩大了传统开发运维团队的作用:更加灵活的工作方式的趋势将要求在开发运维方面进行更多投资，如远程基础设施的安全供应、更好的访问管理和改进的开发人员入职，以更好地提高开发团队的工作效率。

“了解您的 DevOps 投资如何产生影响将在决定是留在远程还是回到办公室时发挥重要作用，”McLead 说。

他补充说，吸引和留住高效开发运维团队的一个方法是展示你的开发运维投资正在获得回报。

“我最近和一位工程经理聊过，他的团队推出功能的速度不够快，”他说。“完成一张票据平均需要一个月的时间，这对他们推进主要路线图项目的能力产生了重大影响。”

他解释说，通过实施提高吞吐量的措施(例如投资 CI/CD、减少构建时间和提高测试覆盖率)，他们能够将交付时间从一个月减少到四天。

通过建立数据管道和 BI 工具并提高可见性，工程经理不仅能够满怀信心地向他的管理团队和董事会报告指标，还能提高团队的士气。

从 McLead 的角度来看，留住有才华、训练有素的 DevOps 工程师至关重要，因为许多 DevOps 项目需要更长的时间来实施和衡量他们的成功。

“高效的 DevOps 团队不仅衡量他们的投资如何在短期内提高效率，还衡量吞吐量和生产率的变化在数月和数年内的趋势，”他说。“如果你的团队人员流动率很高，在这么长的时间内持续交付成果就更加困难了。”
