# Fairwinds Insights 最新版本将 DevSecOps 与额外的左移安全增强功能相统一

> 原文：<https://devops.com/fairwinds-insights-latest-release-unifies-devsecops-with-additional-shift-left-security-enhancements/>

*Kubernetes 治理平台增加了自动化的基础设施代码扫描和增强的 GitHub 集成，因此开发团队可以更快地发现并修复错误配置*

**西班牙巴伦西亚(S14 号展位)，****5 月 17 日，20 日** **22 日**–[Kubernetes 治理软件的领先提供商 Fairwinds](http://www.fairwinds.com/) 今天宣布了对团结 DevSecOps 团队的平台[fair winds Insights](https://www.fairwinds.com/insights)的最新增强。最新版本包括增强功能，帮助 DevOps 团队在不断变化的敏捷开发和生产环境中，与安全团队一起管理保持 Kubernetes 安全的复杂性。

Fairwinds 产品副总裁 Joe Pelletier 表示:“Kubernetes 是一个不断变化的环境，它需要不断扫描，以确保开发人员能够更快地交付他们的应用，同时从一开始就确保配置的安全性。“我们今天宣布的安全性增强是我们采取的另一项措施，旨在为 DevSecOps 团队提供一个单一、集中的视图，从拉请求时到运行时，以代码形式查看其容器和基础架构的安全性。通过这样做，我们可以帮助他们节省时间，加快上市速度，并避免部署和管理定制点解决方案的需要。”

pager duty SRE 平台工程经理 Tristan Bates 说:“我们监控的两个重要问题是安全性和合规性标准。我们希望每个团队都符合这些标准，并且需要在不妨碍生产力的情况下易于实施。这就是为什么我们使用 Fairwinds Insights，并将其集成到整个软件开发生命周期中，以便在出现不合规的情况时提醒团队。”

Fairwinds Insights 的新安全增强包括:

*   **自动扫描基础设施即代码以支持 GitOps**–专为需要在多个团队中推广基础设施即代码扫描的平台工程经理而设计，自动扫描可自动发现和扫描 GitHub Repos 中的 YAML 文件和掌舵图，并提供自动扫描，无需单独的 CI 管道集成。借助自动扫描，平台工程团队可以在几分钟内配置基础设施代码扫描，从而创建一个即时反馈循环，以便开发人员可以更快地修复问题。
*   **第三方映像升级建议**–通过推荐漏洞较少的第三方映像的更新版本来加速修复。
*   **Falco 支持—**fair winds Insights 现在可以集成来自开源运行时容器安全项目 Falco 的安全事件。这为 DevSecOps 团队提供了用于监控容器安全性的单一平台，以及用于自动将事件路由到 Slack 和 PagerDuty 等第三方工具的现成集成。
*   **漏洞浏览器–**使用 Fairwinds Insights 识别 Kubernetes 集群中风险最高的容器映像，包括推荐的升级和补救选项。

Fairwinds Insights 还帮助 DevSecOps 团队遵守由美国国家安全局(NSA)和网络安全与基础设施安全局发布的 [Kubernetes 强化指南](https://media.defense.gov/2021/Aug/03/2002820425/-1/-1/0/CTR_Kubernetes_Hardening_Guidance_1.1_20220315.PDF) 技术报告。了解关于 Insights 如何帮助 Kubernetes 强化的更多信息:[https://www.fairwinds.com/kubernetes-nsa-hardening-insights](https://www.fairwinds.com/kubernetes-nsa-hardening-insights)
