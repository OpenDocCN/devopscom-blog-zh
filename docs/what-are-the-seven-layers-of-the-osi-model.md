# OSI 模型的七层是什么？

> 原文：<https://devops.com/what-are-the-seven-layers-of-the-osi-model/>

在软件领域工作，工程师引用不同的“层”并不罕见也许您正在“网络层”处理协议，或者评估位于“第 4 层”或“第 7 层”的解决方案虽然这些概念对一些人来说是显而易见的，但并不是每个人都知道这些层指的是什么。为了让您快速了解，下面，我们将定义 [OSI 模型](https://devops.com/the-7-layers-of-osi/)的七层。

## 什么是 OSI 模型？

开放系统互连模型(OSI 模型)是一个概念模型，有助于可视化一个计算机化的网络。它由七层组成，代表了构成端到端计算系统的不同层次。本着促进开放互操作性的精神，该模型旨在代表一种通用标准，它与特定的技术或供应商无关。

OSI 模型由国际标准化组织(ISO)和互联网工程任务组(IETF)共同构想，并于 1984 年首次发布。此后，它被重新定义为 ISO/IEC 7498-1:1994。尽管 OSI 模型已经存在了几十年，但它仍然经常被引用。

## OSI 模型的七层

OSI 模型分为七个抽象层:物理层、数据链路层、网络层、传输层、会话层、表示层和应用层。您可以将最底层的第 1 层(物理层)视为最接近最基本的电气连接。越往上，就越接近第 7 层(应用层)，这是最面向用户的类别。

### 1.物理层

物理层，即第 1 层，代表数据传输的最低级别。这一类别指的是原始的非结构化*数据位*——以及将它们转换成电信号供设备读取的过程。网络集线器、调制解调器、适配器、电缆和网络控制器等“物理”硬件在这一层工作。蓝牙、以太网和 USB 描述了物理层规范。

### 2.数据链路层

数据链路层处理打包成*帧*的数据。这一层中的技术有助于节点到节点的数据传输，这里的协议描述了如何建立和终止连接以及连接应该如何流动。数据链路层进一步分为两个子层:媒体访问控制(MAC ),负责节点之间的连接；逻辑链路控制(LLC ),负责检查错误并协调帧流和同步。数据链路层协议的一个例子是点对点协议(PPP)。MACsec 也可以在这一层应用加密。

### 3.网路层

网络层负责发送和接收以*数据包*构成的数据帧。网络层的技术使用路由器向不同网络上的节点发送数据包。如果消息超过最大网络数据包大小，网络层可能会将其拆分或分段。如果您查找过您的 IP 地址，您可能对网络层很熟悉—网络层使用 IP 协议(或其他逻辑协议)来查找位置。这种协议指定了消息以及预期接收节点的地址。

### 4.传输层

传输层处理数据*段*的传输，数据段是可变长度的数据序列。传输层的目标是通过执行数据分段和调整数据包大小或传输速率来优化数据传输。传输层协议包括 TCP 和 UDP。隧道协议也在传输层运行。OSI 模型还定义了五类连接模式传输协议。

### 5.会话层

会话层处理两台或多台计算机之间的通信。这里的协议用于创建实体之间的“会话”，这在使用远程过程调用的应用程序中很常见。会话层处理客户端或服务器之间的连接和身份验证，包括登录、查找、注销或会话终止等操作。DNS 和名称解析协议都在会话层运行。

### 6.表示层

表示层是关于数据翻译和格式化的。在这一层，协议处理诸如加密、解密、压缩和解压缩之类的事情。表示层的目标是以这样一种方式转换数据，即数据可以以符合应用层指定的结构的语法通过网络发送。例如，将数据结构序列化为 XML 或 JSON 的技术可以被认为是为表示层服务的。这类数据转化为最终用户的图形显示。

### 7.应用层

应用层是 OSI 模型的顶层，离最终用户应用程序最近。面向用户的软件通过文件共享、消息处理或数据库访问等功能直接与应用层交互。这一层使用 HTTP 和 FTP 等高级协议来共享资源。Web 浏览器和电子邮件客户端是与应用层交互的应用程序的示例。

## 开放系统互连模型:一个有用的分类法

现在你有了它——所有七个 OSI 模型层的入门或复习。值得注意的是，该模型并不打算作为实现规范本身——它纯粹是一个概念框架。但是它确实有助于提供工具在哪里运行以及它们如何与分布式计算系统的元素交互的上下文。

在 DevOps 技术的上下文中，基于特使的[服务网格](https://containerjournal.com/editorial-calendar/kubernetes-in-the-enterprise/when-is-service-mesh-worth-it/)通常被描述为在第 4 层(传输层)和第 7 层(应用层)上工作。或者，由于 [eBPF](https://www.ferrisellis.com/content/ebpf_past_present_future/) 过滤网络帧，可以说它工作在 OSI 模型的第 2 层。
