# 用灵活的劳动力消除开发人员的倦怠

> 原文：<https://devops.com/eradicating-developer-burnout-with-a-flexible-workforce/>

无论人们的工作角色是什么，倦怠都会影响他们，但软件开发人员尤其容易受到影响。很多时候，企业通过长期依赖员工在短时间内交付的繁重工作来追逐利润。开发人员经常承担紧张的工作，并发现他们一次又一次地在巨大的压力下被推到最后期限。但是如果没有适当的照顾，倦怠几乎是不可避免的。毕竟开发者在成为开发者之前首先是人。

编程是一项认知压力很大的工作，需要个人不断解决复杂的问题。对任何人来说，日复一日不间断地连续工作几个小时都是艰难的，这种持续数周的模式对开发人员来说再熟悉不过了。

但是慢慢地，工作场所的面貌正在改变。我们看到后流行病时代对员工福利的关注，当前的就业市场条件意味着专业人士有选择和权利要求更好的照顾，甚至探索自我就业或非传统的工作途径(谢谢，[伟大的辞职](https://www.weforum.org/agenda/2022/06/the-great-resignation-is-not-over/)！).在这种背景下，企业也在以不同的方式思考如何与人才合作，以更有效地使用员工。  为了员工和雇主的利益，这里有机会降低开发人员面临的[倦怠](https://devops.com/?s=burnout)风险。

## 组织承受不起倦怠

开发人员是大多数数字化转型项目的关键，因此组织不能让他们的开发团队精疲力竭。这可能会大大推迟新产品、功能或应用的推出，并减缓开发运维计划，如果没有负责的技术团队持续高效地工作，开发运维计划就无法取得进展。

在疫情之后，开发人员面临的风险尤其高，技术人员面临着额外的压力，要赶上进度，交付可能已经停滞的项目。在此期间，很少考虑员工的福利；一项 [2021 年的研究](https://www.usehaystack.io/blog/83-of-developers-suffer-from-burnout-haystack-analytics-study-finds) 发现，83%的软件开发人员正在经历倦怠。这不仅仅是写代码——技术团队也在与低效的工程流程、积压和技术债务作斗争，并且在试图快速交付工作时更有可能遇到摩擦，导致更大的倦怠风险。

低质量的内部软件也是开发团队的一个挫折点，83%的工程师报告担心他们组织中的软件可靠性 。为这些团队优先选择合适的工具是企业帮助确保项目更顺利运行的一种方式。这是一个简单的解决办法，但却经常被忽视，尽管增强技术团队的能力越来越成为企业的头等大事。

## 开发人员不容易被取代

伟大的开发人员拥有多年的经验，通过解决实际问题来学习他们的技术。他们从来都不容易被取代，但当前全球就业市场的状况意味着这比以往任何时候都更难。这与开发人员的福利有着内在的联系；一些人为了寻求更好的工作条件而换工作，跳槽到其他公司，而另一些人则自己创业。例如，许多人转向自由职业者的角色，以获得独立和控制自己的工作量。

然而，自由职业者可能是孤独的，管理繁重的。尽管灵活性提高了，但缺乏支持也会导致倦怠。人才云通过为自由职业者提供结构化工作和与同行定期互动的封闭社区，为这一问题提供了一个潜在的解决方案。他们的会员人数激增，并推动了更广泛的自主创业。事实上，2021 年，11.2%的开发人员是自由职业者，高于 2020 年的 9.5%，因为他们试图平衡自由职业者的自由和永久工作的一些额外津贴。

这一点至关重要。对于一个被认为“永远在线”的角色，自由职业者决定开发人员何时何地工作的灵活性对于交付高质量、安全的代码至关重要。没有每天浪费在通勤上的时间——否则这些时间可以用来工作，或者，更重要的是，从紧张的工作中减压——它已经成为全世界超负荷工作的开发人员的选择。这是现代企业赖以建立的基础——给这些企业的设计师一些这种灵活性是为了确保他们的福祉而付出的小小代价。因此，产品、项目、功能和应用的及时交付加快了数字化转型计划。

## 这是一条双行道

我们已经看到了自由职业者工作场所灵活性的增加是如何转化为更高的生产力和降低倦怠风险的。同样，[](https://www.usehaystack.io/blog/83-of-developers-suffer-from-burnout-haystack-analytics-study-finds)31%的开发人员担心目标和过程不清晰，因此也有必要改善数字化计划的项目管理，以减轻技术人员的负担。事实上， [58%的开发人员](https://www.revelo.com/blog/software-engineer-remote-burnout) 认为倦怠不是不可避免的，更好的工作实践和管理是预防倦怠的关键工具。

对于希望快速交付项目的企业来说，从不断增加的自由职业者群体中利用按需工人意味着他们可以根据工作类型或使用的编码语言聘请专家，但他们也有能力应对每一个新的挑战——这是经历过倦怠的开发人员的一个主要担忧。通过私人人才云这样做还能确保管理和审查自由职业者的负担从组织中消失，并消除倪星对引进外部人才的潜在担忧。

## 消除倦怠

福祉比以往任何时候都更受公众关注。一方面，开发人员不应该面临高风险的倦怠；另一方面，企业也不能失去推动数字化未来的人才。

双方都可以帮助解决这个问题。开发人员通过探索给他们带来灵活性的自由职业，将职业幸福掌握在自己手中。就他们而言，企业需要考虑创造性的方法来获得人才。在某种程度上，这是为了确保内部技术团队有适当的支持，这样他们就不会过度劳累。在当前的就业市场中，找到这种支持并不容易，但随着开发人员纷纷投身自由职业，在关键时刻，有机会根据需求快速聘用高技能员工。毕竟，管理良好的工作量可以显著降低疲劳的风险。

我们有资源和可用的技能来根除开发人员的倦怠——我们只需要以正确的方式安排他们。
