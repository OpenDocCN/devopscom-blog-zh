# Observe 试图将更多的 IT 指标放入环境中

> 原文:[https://devo PS . com/observe-seeks-to-put-more-it-metrics-in-context/](https://devops.com/observe-seeks-to-put-more-it-metrics-in-context/)

Observe，Inc .今天宣布，它已经更新了它的可观测性平台，增加了对基于时间序列数据的指标的支持。此外，该公司正在改造该平台生成警报的方式，作为展示更多可操作情报的努力的一部分。

Observe 首席执行官杰里米·伯顿(Jeremy Burton)表示，竞争对手的可观察性平台要求 IT 团队筛选数百个指标，这些指标是使用复杂的标记方案生成的，最终会推高存储成本。伯顿指出，观察平台通过将指标与客户等相关联来筛选指标，以提供更相关的背景。

Burton 补充说，这种方法消除了对复杂标记方案的需求，从而降低了存储成本，同时也消除了向 IT 团队提供大量指标列表的需要。

最新的更新利用了这种度量方法，只根据附加到客户、pod、购物车或容器的精选度量生成警报。例如，IT 团队不仅可以收到响应时间下降的警报，还可以轻松看到哪些客户或设备可能会受到影响。这种转变将减少电子邮件收件箱和其他通信平台中已经泛滥的警报数量。

刚刚额外筹集了 700 万美元，今日观察还透露，自从[在亚马逊网络服务(AWS)公共云](https://devops.com/observe-unfurls-observability-platform-built-on-snowflake-dbaas/)上的雪花云服务上推出平台以来，它已经增加了 20 个付费客户。Observe 没有重新发明相同的核心基础设施，而是利用了一个已经在 IT 环境中广泛使用的数据分析引擎。

伯顿说，目前，Observe 将继续主要关注指标，以 IT 运营团队今天准备消费的形式提供见解。然而，随着时间的推移，Observe 将通过 [OpenTelemetry](https://opentelemetry.io/) 和 Jaeger 等工具增加对分布式跟踪的支持，因为它增加了额外的应用性能管理(APM)功能，Burton 补充道。Burton 指出，与其告诉 IT 运营团队他们正在以错误的方式管理 IT，不如让 IT 运营团队了解他们目前的状况。

Burton 指出，这种策略还使 Observe 平台能够管理在 Kubernetes 集群上运行的单片和基于微服务的应用程序。Burton 补充说，当前的目标是为 IT 团队提供传统监控工具的替代方案，通过允许 IT 组织合理化往往不是特别集成的工具，以更低的总成本提供更多的上下文。

从长远来看，Observe 还将采用机器学习算法来更有效地收集数据。然而，伯顿表示，该公司仍然对依赖机器学习算法来不断学习 IT 环境的 IT 未来愿景持怀疑态度，这些环境不仅独特，而且经常发生变化。

可观测性平台提供商之间的竞争已经非常激烈。每个 IT 团队都需要根据成本和他们对某个供应商可能拥有而另一个供应商不提供的特定功能的偏好，来决定哪种类型的平台对他们最有意义。然而，无论如何，IT 环境的管理方式将很快变得更好。
