# Plutora 通过发布见解增强 VSM 平台

> 原文:[https://devo PS . com/plu tora-enhances-VSM-platform-with-release-insights/](https://devops.com/plutora-enhances-vsm-platform-with-release-insights/)

Plutora 增强了其价值流管理(VSM)平台，以提供[仪表盘](https://www.plutora.com/blog/plutora-turning-release-managers-into-the-new-heroes-of-software-development)，使开发运维团队能够更好地优化应用交付。

Plutora 产品营销和战略副总裁杰夫·凯斯(Jeff Keyes)表示，添加到平台的仪表盘将使应用发布经理能够利用预测分析，以更进步的方式管理应用交付，而不是总是必须对意外事件做出反应。

为了实现这种方法，Plutora 改进了它的 release insights 仪表板，除了添加跟踪软件构建和更大项目计划的分析之外，它还暴露了问题领域和热点。

Keyes 说，后一种能力对于目前发现跟踪多个软件开发计划具有挑战性的高管来说尤其重要，特别是那些可能已经落后于计划而高层领导没有意识到其影响的计划。

![](../Images/a6cb5d2be569db542454db25d2034d7f.png)

虽然高级 IT 和业务领导通常使用 VSM 平台来跟踪软件期限如何影响业务的关键绩效指标(KPI ),但 Keyes 表示，应用发布经理需要获得更高级的分析来优化应用部署时间表，这一点变得越来越明显。渐进式发布管理功能通过 Plutora 平台中的仪表板展现洞察力；这使得应用交付经理更容易优化编排资源分配。

当然，发布经理已经可以使用各种平台来自动化应用交付过程。然而，Keyes 说，这些平台没有提供足够的洞察力来优化由不同工具和平台组成的 DevOps 工作流中的应用程序交付。Plutora 使发布经理能够采用价值流流程指标和更传统的 DevOps [指标](https://devops.com/?s=metrics)来减少周期时间并提高整体 DevOps 吞吐量。

Keyes 说，一旦组织获得了这种洞察力，他们也应该能够通过消除这些工作流中的摩擦来降低许多 DevOps 团队目前经历的整体疲劳水平。他指出，事实上，这种疲劳程度是许多 DevOps 团队人员流动频繁的主要原因。

不清楚有多少组织有专门的应用程序发布团队，但是优化软件交付过程对于几乎每个 DevOps 团队来说都是一个重大挑战。虽然许多组织经常使用持续集成(CI)工具来更快地构建应用程序，但应用程序的持续交付已被证明是一个更难以实现的目标。这主要是因为部署应用程序的每个平台都是独一无二的。因此，许多 DevOps 团队仍然依赖电子表格来手动管理应用程序部署。

然而，随着应用程序构建速度的不断加快，应用程序交付显然正在成为一个瓶颈。VSM 平台提供了一种应用分析来优化流程的方法，然后可以使用该方法做出更明智的决策，从而说服更多的组织自动化应用交付。当然，这是假设该组织有先见之明，首先投资于一个能够展示这些见解的 VSM 平台。
