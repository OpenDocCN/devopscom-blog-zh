# 新的 CLOUDBEES 流平台提高了定制发布流程的标准

> 原文：<https://devops.com/new-cloudbees-flow-platform-raises-the-bar-for-custom-built-release-processes/>

*应用资源编排平台旨在消除软件交付链中的焦虑*

**DEVOPS WORLD | JENKINS WORLD，SAN FRANCISCO****–8 月 14 日、20 日** **19 日****–**[cloud bees](https://www.cloudbees.com)^(，推动持续经济发展的企业级 DEVOPS 领导者，今天宣布推出 cloud bees Flow 9.1 版本——业界首个为企业级 DevOps 打造的统一应用发布编排(ARO)平台。新版本建立在最近收购的 Electric Cloud 中收购的前 ElectricFlow 编排平台 CloudBees 的基础上，增加了一系列增强功能，使整个软件交付链比以往任何时候都更容易消除发布焦虑。)

CloudBees Flow 通过自动化和协调软件发布、管道和部署，并利用分析和洞察力来衡量、跟踪和改善结果，从而帮助提高 IT 效率。新功能将进一步简化和定制编排流程，提供更加个性化的仪表盘、新的数据归档以及对版本生产路径的更加实时的可见性。

CloudBees Flow 9.1 包括以下新功能:

*   **发布命令中心定制**–完全可定制的仪表板，允许用户显示他们自己的定制数据，突出显示给定的发布渠道，并更有效地报告他们自己的发布流程。
*   **数据归档**—能够将旧数据归档到他们出于监管和合规目的选择的系统中。
*   **带库存的生产之路**–全面了解发布进度，包括准确查看部署到给定环境的库存的方法。
*   **发布产品组合反馈**–更好地洞察整体发布和所有子管道，能够使用新的直观界面区分各种图例和彩色编码视图。
*   **发布管道执行**–更大的灵活性和对发布管道执行的完全控制，能够:
    *   不按顺序运行任务
    *   在未来阶段预运行任务，以便灵活地无序运行任务，提高效率和控制力
    *   通过将应用程序部署流程标记为要求对环境的独占访问来防止冲突和错误
    *   确保一次运行一个部署

CloudBees Flow 的定位是[*Forrester Wave:持续交付和发布自动化，2018*](https://electric-cloud.com/resources/whitepapers/forrester-wave-continuous-delivery-release-automation/) ¹ 的领导者。该产品还被定位为 Gartner 应用发布流程幻方图 ² 中的领导者。

**关于 CloudBees**

CloudBees 通过提供世界上第一个端到端连续软件交付管理系统(SDM)为连续经济提供动力。对于数百万推动各种规模企业创新的开发人员和产品团队来说，SDM 建立在持续集成(CI)和持续交付(CD)的基础上，使软件交付组织内部和周围的所有功能和团队能够最好地合作，以扩大价值创造。

CloudBees 是 CI、CD 和应用程序发布流程编排(ARO)的核心，建立在其产品的商业成功及其开源领先地位之上。CloudBees 是[詹金斯](https://jenkins.io/)和[詹金斯 X](https://jenkins-x.io/) 的最大贡献者，也是[持续交付基金会](https://cd.foundation/) (CDF)的创始成员。从全栈开发人员实践 NoOps 的创业公司到大型财富 100 强公司，CloudBees 使所有软件驱动的组织能够在正确的时间智能地部署正确的功能。

超过 3，500 个世界知名品牌和超过 50%的财富 500 强企业投资 CloudBees，因为它能够跨任何云、在任何开发环境中工作，并平衡公司治理和控制与开发人员的灵活性和自由度。CloudBees 是世界领先的 DevOps 专家的家园，帮助数千家公司利用“持续一切”的力量，将他们置于从伟大的想法到伟大的软件，再到伟大的商业价值的最快路径上。
