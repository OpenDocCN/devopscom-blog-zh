# DevOps 如何帮助您在 DX 之旅中茁壮成长

> 原文：<https://devops.com/ways-devops-can-help-you-thrive-in-your-dx-journey/>

数字转型(DX)仍然是那些努力寻找新方法来改善业务成果并充分利用现有新技术和技巧的领导者的首要考虑因素。有一个话题继续与这些想法保持一致:DevOps。

DevOps 使企业领导者能够更加自信地看待其市场中的新机遇。这在很大程度上是因为他们公司满足不断增长和不断变化的市场需求的能力有了显著提高。新的软件和功能可以更快地交付，提供令人兴奋的新机会。

DevOps 所需的技能和工具正在被迅速采用。根据 [IDC 和 NetApp](https://urldefense.proofpoint.com/v2/url?u=http-3A__www.netapp.com_us_campaigns_data-2Dvisionary_thrive-2Dwith-2Ddigital-2Dtransformation.aspx&d=DwMFaQ&c=9wxE0DgWbPxd1HCzjwN8Eaww1--ViDajIU4RXCxgSXE&r=YtBJMVdn8YFkZpz9IWJTxRQFpgtqHh3lgEo6UMqh4GU&m=i) 最近的一项调查，38%的组织正在将新的开发运维技能添加到现有组织中，作为其 DX IT 投资和集成战略的一部分。他们正在应用这些技术来实现 DevOps 承诺的一些早期结果，并为持续改进提供基础。

# DevOps 和 DX

虽然并非每个组织都会开发实现真正 DevOps 实施的能力，但以下内容可以普遍应用于支持成功的 DX。

## **不要孤立遗产**

人们自然希望将重点放在新的工作模式上。IT 领导将非常积极地参与新工具的评估和实施、文化的改变、部署和停机率的增加以及所有相关的会议和规划。

成功的组织致力于现代化和发展，以全面改进流程。他们将新的学习和新的模式应用于整个环境，而不仅仅是创新领域。这并不是说，更传统的应用程序将需要代表正在开发的下一代应用程序的价值流。相反，这些应用程序往往不会经常改变。目的应该是将配置管理、云和代码管理等技术应用到环境的这些部分，以提供更稳定、可预测和自助式的环境。这有助于提升整个组织，并提供一个更有效的工作环境。

幸运的是，最新一代的数据中心技术始终如一地提供 API 和 SDK，支持各种新功能。供应商也在提供支持通过通用工具进行控制的本机集成。现在，IT 组织在工具选择和这些工具提供的结果方面都有了更大的灵活性。基础架构投资在支持下一代需求的同时，还能提供对管理更传统的应用程序类型至关重要的弹性和功能。

## **改变你的测量方式**

正如我们在“DevOps 状态”报告中看到的那样，采用 DevOps 的组织通常会经历事件频率和严重性的显著上升。如果继续用平均解决问题时间和平均故障间隔时间来衡量成功，那么它现在已经超过了 MTTR 和 MTBF。

领导者倾向于关注消极的一面，因为它需要被“修复”。哪里出了问题？我们如何防止它再次发生？在实施 DevOps 以帮助促进 DX 时，需要进行转变。

如果这种转变没有发生，组织可能会对正在应用的关键变革产生抵触情绪。团队成员也不太可能探索新的想法或在价值流中“触及脆弱的系统”。

对什么在起作用有更强的理解和关注，这需要成为首要考虑的问题。我们能多快自信地部署一个新特性？我们能够支持多少增长？我们多快就能发现漏洞和潜在问题？通过探索*正在*工作的领域，就有可能将这些收获应用于业务的其他领域。

## **推动整个组织向前发展**

在采用 DevOps 的过程中，工具可能是至关重要的，但是没有什么可以替代良好领导能力的应用。走上 DX 之路的公司将需要重新评估其业务元素，这些元素最初可能被认为与过渡无关。应用于变革创新部分的技能应考虑应用于更稳定的业务元素。这一努力将提高整个组织的效力，同时也使其能够更好地应对下一阶段的转型。
