# DataCore 推出企业级容器原生存储软件，以实现更高的性能和可扩展性

> 原文：<https://devops.com/datacore-debuts-enterprise-grade-container-native-storage-software-for-higher-performance-and-scalability/>

*DataCore Bolt 在云和内部使用 NVMe 为 Kubernetes 环境动态提供快速、持久的存储；将开源 OpenEBS 项目的创新应用于延迟敏感的容器化工作负载*

**佛罗里达州劳德代尔堡和西班牙巴伦西亚，2022 年 5 月 17 日—**[软件定义的存储解决方案的最大独立供应商 DataCore Software](http://www.datacore.com/) 今天宣布推出面向 DevOps 的企业级容器原生存储软件 [DataCore Bolt](https://www.datacore.com/products/bolt-kubernetes-storage/) 。在云中和内部的 Kubernetes 集群上运行的有状态应用程序受益于 Bolt 的高性能持久存储的动态供应。Bolt 的容器化微服务堆栈利用了 NVMe/TCP 的低延迟和高 IOPs 特性，以及 Kubernetes 的内在编排和大规模恢复能力。DataCore 在西班牙巴伦西亚的 KubeCon + CloudNativeCon Europe 上宣布了这一消息，Bolt 将在那里展示到 5 月 20 日。

“我们很早就认识到容器将是一代一次的转变，其中数据存储对基于微服务的应用程序的部署和响应有着独特的影响。DataCore Software 首席执行官 Dave Zabrowski 表示:“这是一个对现代存储技术(如基于结构的 NVMe 和独立于平台的数据服务)有着强烈需求的市场。“有了 DataCore Bolt，我们很自豪能够通过提供高性能、可扩展、灵活、可靠的企业软件来延续 MayaData 开创的创新传统，使组织能够跃进集装箱化世界并快速响应变化。”

Bolt 是该公司第一个获得商业许可的 Kubernetes 存储产品，基于其对 OpenEBS 的原始开发者 MayaData 的收购。传统存储基础架构本身无法满足当今现代有状态工作负载的数据持久性和可移植性需求。在 DevOps 应用中，他们对重复性手动任务的依赖降低了持续集成和持续交付(CI/CD)管道的速度——削弱了集装箱化带来的优势。

Bolt 是一款专为 Kubernetes 打造的存储解决方案，可促进生产就绪型软件的部署和发布，同时实现大规模 DevOps 环境中所需的敏捷性。利用 NVMe 硬件和 NVMe/TCP 协议的优势，Bolt 支持可组合和分解的存储架构，消除了其他存储产品的开销和迟缓。

Evaluator Group 的 HCI、云存储和容器存储高级分析师 Dave Raffo 表示:“Kubernetes 的兴起更加重视为广泛分布的容器化部署而从头设计的存储。“包括 DataCore 在内的容器原生存储厂商正在为开发运维团队和 IT 运营团队带来基本的企业功能，他们希望推出需要更快性能和自动操作的有状态工作负载。”

无论是在云中还是在内部，Bolt 都消除了与存储相关的延迟，并加快了代码进度，以确保通过 DevOps 管道无缝交付应用，从而使团队能够实现其部署目标。Bolt 还允许 DevOps:

*   通过对 NVMe 驱动器的低延迟访问，加快有状态工作负载的响应速度
*   借助直接由 Kubernetes 协调的集装箱存储服务，简化 DevOps CI/CD 渠道
*   用本地 Kubernetes 自动化取代复杂的存储管理
*   获得外部系统无法观察到的存储行为
*   通过消除内核/操作系统依赖性，确保存储服务的可移植性
*   通过跨 K8s 节点的卷复制增强数据恢复能力

Bolt 与用于版本控制、配置管理、CI/CD 自动化代码库和工件管理、测试自动化、监控和部署的其他 Kubernetes 工具一起工作。

Bolt 可用于商业用途，并按 Kubernetes 节点授权。Bolt 典型的 1 年期许可包括年度许可成本和支持费用。欲了解更多信息或与 DataCore 高管会面，请参加 5 月 16 日至 20 日在 Feria Valencia 举行的 KubeCon + CloudNativeCon Europe 展会，展位号 S76，2 号展厅 0 层。
