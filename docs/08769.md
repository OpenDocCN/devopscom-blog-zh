# code 公证人使用不可变的数据库来验证软件工件

> 原文:[https://devo PS . com/code 公证人-使用-不可变-数据库-验证-软件-工件/](https://devops.com/codenotary-uses-immutable-database-to-verify-software-artifacts/)

code 公证人今天为开源工件和容器推出了免费的公证和验证服务，使 IT 组织能够跟踪组成其应用程序的组件的出处。

code 公证人首席技术官丹尼斯·齐默(Dennis Zimmer)表示，[社区证明服务](https://www.globenewswire.com/news-release/2021/10/26/2321016/0/en/Codenotary-Launches-First-Free-Open-Source-Notarization-and-Verification-Service-for-Creating-Software-Bill-of-Materials.html)基于一个不可变的开源 immudb 数据库，该数据库以加密方式为每个软件工件附加一个身份。此外，社区证明服务可以与持续集成/持续交付(CI/CD)平台集成，DevOps 团队使用这些工件来构建和部署应用程序，Zimmer 补充道。

Zimmer 表示，社区证明服务创建的身份可用于验证软件材料清单(SBOM ),这不仅是每个美国联邦机构的要求，也是在一系列高调违规事件后试图更好地保护其软件供应链的组织的要求。Zimmer 指出，社区证明服务还将鼓励开发人员重用他们知道可以在零信任 IT 环境中可靠使用的源代码、构建、存储库和 Docker 容器映像。

![](../Images/9b72a8e81694fd6b92b5a1aefbb3abef.png)

当然，一些开发人员已经签署他们的代码很多年了，而其他人根本看不到这个需要。Zimmer 指出，code 公证人正试图通过其免费服务来扩大开发人员的范围，这些开发人员不仅会签署他们的代码，而且会以一种以后不能被篡改的方式这样做。

与用来提供不可变交易记录的区块链数据库不同，Zimmer 表示，immudb 平台的创建是为了提供一个不可变的数据库，能够每秒处理数百万笔交易。Zimmer 指出，该数据库还用于构建超出社区证明服务范围的广泛应用程序，因此 DevOps 团队可以对底层平台的可行性充满信心。

随着一些令人不安的安全漏洞的出现，安全团队开始询问关于如何构建应用程序的更困难的问题只是时间问题。开发人员可以等待这些网络安全团队最终制定出 DevOps 团队需要执行的策略，或者他们可以更主动地采用一种服务来跟踪代码的来源，希望已经扫描了所有已知的漏洞。事实上，组织决定不使用未经验证的代码可能用不了多久。甚至可能会有人强烈反对使用未签名的代码，这可能会导致许多现有的应用程序被更新或彻底替换。

从积极的一面来看，Zimmer 指出，人们越来越意识到需要签署代码。挑战在于大多数组织没有适当的流程。幸运的是，人们对确保代码完整性的 DevSecOps 最佳实践更感兴趣。然而，意识到需要某些东西并不总是导致采取具体的行动，除非 DevOps 团队能够简单地让开发人员容易地做正确的事情。毕竟，这并不是说开发人员想要使用不安全的代码，只是他们通常太忙了，以至于不能确保所使用的每一个工件都是安全的。
