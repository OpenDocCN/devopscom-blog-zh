# 指数追踪 API 优先的创业公司的显著增长

> 原文:[https://devo PS . com/index-tracks-impressive-growth-of-API-first-startups/](https://devops.com/index-tracks-impressive-growth-of-api-first-startups/)

API 是现代软件的基石，它们的使用在全球范围内持续增长。《邮差》 [2022 年 API 报告](https://www.postman.com/state-of-api/#key-findings)发现，未来 12 个月，对 API 的投资将增加或保持不变。不断发展的 [API 优先方法](https://nordicapis.com/what-it-means-to-go-api-first/)正引领许多创新和创业进入该领域。GGV 资本公司的最新研究集中体现了这一趋势，该公司保持了领先的开发者优先软件公司的 API 优先指数。

API-first 指数追踪的是已经筹集了 5000 万美元或更多资金的 API-First 公司。截至 2022 年第三季度，达到这一门槛的企业数量增加到 60 多家。仅在第三季度，该指数中的公司就筹集了 3.5 亿美元的资金，总的来说，该指数代表了超过 140 亿美元的资金。毫无疑问，对这一领域的兴趣和投资正在扩大，并将在未来一年继续增加。这是一个值得关注的领域，因为对模块化的强调会影响软件的组装。

我最近会见了 GGV 国会的合伙人蒂芙尼·勒克(Tiffany Luck)，她负责 API-First 指数。下面，我们将更深入地探讨这个指数，并了解 Luck 的观点，为什么这些发现代表了 API 优先技术和商业模式作为一个整体的独特时刻。

## API 优先索引

API 优先指数关注通过 API 提供利基功能的公司；也称为 [API-as-a-product](https://nordicapis.com/difference-between-api-first-and-api-as-a-product/) 。该列表包括金融科技、通信、医疗保健、商业和其他领域的各种基于 API 的工具。

例如，Stripe 可能是最著名的 API 即产品，因为它通常被集成为企业接受支付的方式。还有 MessageBird，这是一个通过任何渠道发送通知的 API。其他例子包括 Mapbox，一个用于地理位置数据的 API，Mux，一个用于创建直播和点播视频体验的 API，Nylas，它提供了一个流行的电子邮件 API 和 Alpaca，一个用于股票和加密货币交易的 API 和许多其他 API。

目前，该索引并不包括工具的整个 [API 平台景观，其范围从 API 管理到测试、文档、监控、设计、安全和](https://voyager.postman.com/infographics/2022-postman-api-platform-landscape.png)[可观察性](https://devops.com/?s=observability)等领域。还有中间件集成平台也要考虑。有了这些，资金充足的 API 相关工具集的列表肯定会更长。

## 为什么对 API-First 这么感兴趣？

决定建立 API 优先公司的人数正在迅速增加。这些基于 API 的 web 组件是可互换的，代表了软件开发的未来。“世界正朝着网络和云组件的方向发展，”Luck 说。这一趋势代表了一场全球运动，欧洲、亚洲和美国都做出了很大努力。此外，在持续的经济不确定性和 2022 年不完整的风险投资资金中，API 模型似乎具有弹性。“与更广泛的风险投资环境相比，API 表现得非常好，”Luck 说。

那么，为什么会有这么多关于 API 的兴奋呢？嗯，人们正在寻找同类最佳的结构块，可以用来构建通用功能，如地理定位、人工智能、天气、支付或特定的 CRM 功能。利基 API 可以在没有大型平台的情况下快速提供特定的功能。Luck 说，另一个原因是它们相对具有粘性——基于 API 的交付模式在建立一个随时间扩展的自助服务平台方面非常有效，这是投资者非常热衷的前景。

## Unicorn APIs 解决难题

在评估 API 优先的公司时，Luck 寻找那些为超级困难的共享问题提供 B2B 解决方案的公司。这相当于难以单独构建、维护和扩展的可重用功能。此外，解决方案必须解决跨公司的共同需求，但这不是他们产品的核心。

例如，Luck 关注一些令人印象深刻的 API 优先用例。例如， [Stream](https://getstream.io/) 支持应用内面向用户的功能。 [Knock](https://docs.knock.app/) 提供处理通知的解决方案。但许多最具挑战性的领域包括金融服务，因为从监管和连通性的角度来看，这一功能很难构建。例子包括[风车](https://www.pinwheelapi.com/)，它汇总实时收入和就业数据。方法 API 做了一些类似的事情，但是用于聚集消费者债务。

Luck 将 API 描述为“一种访问技术上无法访问的内容的现代方式。”我们看到这种行为在开放银行中展开，因为全球的新标准现在要求消费者金融数据可以通过编程访问，通常是以 API 连接的形式。

## 开发者体验:新的竞争优势

在一个竞争激烈、面向开发人员的工具层出不穷的市场中，开发人员体验(DX)、[安全性](https://thenewstack.io/its-not-only-banking-apis-that-must-be-secured/)、性能和可靠性等非功能性方面越来越受到关注。根据 Luck 的说法，[开发者体验](https://devops.com/understanding-the-importance-of-developer-experience/)可能位于金字塔的顶端，因为它代表了上述许多特征的总称。具体来说，[具有高质量 DX](https://devops.com/bridging-the-developer-experience-devx-gap/) 的开发者工具拥有优秀的文档，并且必须保持向后兼容性，以避免在消费者端出现破坏性的变化。“最终，您实施的产品会影响该客户的最终客户。客户希望零负面影响和零中断，”她说。

API-First 指数对行业有益，因为它有助于教育公众如何进行私人投资。随着围绕它的讨论越来越多，它带来了对构建 API 优先的[好处的认识和了解。](https://blog.postman.com/why-should-you-be-an-api-first-company/)
