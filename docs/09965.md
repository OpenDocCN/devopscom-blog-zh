# 帕洛阿尔托网络公司收购苹果酒安全公司来封锁管道

> 原文:[https://devo PS . com/Palo-alto-networks-buys-cider-security-to-lock-down-pipelines/](https://devops.com/palo-alto-networks-buys-cider-security-to-lock-down-pipelines/)

Palo Alto Networks 本周同意[以大约 1.95 亿美元的现金收购 Cider Security](https://www.paloaltonetworks.com/company/press/2022/palo-alto-networks-signs-definitive-agreement-to-acquire-cider-security) ，这是一家提供持续集成/持续交付(CI/CD)平台的公司。

定于本季度完成的对 Cider Security 的收购将扩展该公司 Prisma 云平台的覆盖范围，该平台于去年更新，包括一套用于保护 IT 基础设施供应的基础设施代码(IaC)的工具。Palo Alto Networks 之前收购了 Bridgecrew，后者开发了一个开源的 Checkov 策略代码工具。上个月，Palo Alto Networks 也向 Prisma Cloud 添加了软件组合分析工具。

Cider Security 开发了它所称的应用安全操作系统。该解决方案创建了一个图表，使 DevOps 团队能够可视化组成软件开发环境的所有元素之间的关系，包括代码。然后，它可以应用一组控制措施来修复任何漏洞和攻击路径，这些漏洞和攻击路径可能是使用任何数量的第三方扫描工具发现的。

帕洛阿尔托网络公司首席执行官尼科什·阿罗拉告诉行业分析师，此次收购是该公司在现有平台和服务组合的基础上加倍努力保护软件供应链安全的一个例子。

DevOps.com 母公司 Techstrong Research 的总经理迈克·罗斯曼(Mike Rothman)表示，目前还不清楚谁负责 DevOps 平台的安全，但将越来越多地出现某种类型的中央安全功能，与应用开发团队合作，锁定软件供应链。“将会有一个中央安全小组专注于确保管道的安全，”他说。

随着 Palo Alto Networks 继续扩大 Prisma Cloud left 的覆盖范围，它继续通过可通过云集中管理的平台组合来集中管理网络安全。目前还不清楚有多少组织正在集中管理其软件供应链和生产环境的安全，但 Arora 指出，随着组织寻求降低网络安全的总成本，人们对整合安全供应商的兴趣很高。

与此同时，在最近一系列引人注目的违规事件发生后，各种规模的组织都在寻求采用 DevSecOps 最佳实践来更好地保护软件供应链。除了扫描代码寻找漏洞之外，许多组织开始意识到用于构建应用程序的工具和平台也容易受到网络攻击。这些攻击的目标是插入恶意软件，这些恶意软件将在最终部署在生产环境中的任意数量的下游应用程序中表现出来。

目前还不清楚这些工具和平台的危害有多大，但很明显，软件消费者正朝着让开发者对漏洞承担更多责任的方向发展。这些要求中最著名的例子是拜登政府发布的行政命令，该命令要求联邦机构将列出已知漏洞的软件材料清单(SBOMs)包括在内。许多企业 IT 组织预计会采用类似的需求。

然而，无论如何，构成软件供应链的工具和平台，以及它们产生的代码，将受到比以往更严格的审查。
