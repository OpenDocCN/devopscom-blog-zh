# CircleCI 调查显示有必要优先考虑 DevOps

> 原文:[https://devo PS . com/circle ci-survey-shows-the-need-to-prioritize-devo PS/](https://devops.com/circleci-survey-shows-the-need-to-prioritize-devops/)

今天发布的一份 CircleCI [对美国和英国 2013 名商业领袖的调查](https://www.businesswire.com/news/home/20210728005536/en/Businesses-Risk-up-to-126M-Lost-Revenue-with-Ineffective-Software-Delivery)发现，几乎所有的受访者(97%)都认识到，他们未来一年的商业成功取决于他们的软件开发团队。尽管有这些发现，超过三分之一的受访者(39%)承认他们的组织在创新以及获取和保持开发人员技能方面面临挑战。

整整 89%的受访者表示，他们也非常了解如何衡量其工程团队的绩效。然而，40%的公司仍在使用过时的度量标准，如编写的代码行数，而另外 37%的公司衡量故事点。

更令人不安的是，不到三分之一的受访者(30%)表示他们计划优先采用 DevOps 最佳实践。另一方面，只有 15%的受访者表示他们正在首次实施持续集成/持续交付( [CI/CD](https://devops.com/?s=CI%2FCD) )平台。

CircleCI 调查表明，软件交付方式的改进平均每年价值高达 1 . 26 亿美元。近三分之二的受访者(65%)表示，他们的企业参与了这样或那样的数字业务转型计划。另有 20%的人说他们已经完成了一个。

CircleCI 首席执行官 Jim Rose 表示，虽然越来越多的组织现在意识到他们是多么依赖软件，但他们中的许多人还没有解决从根本上影响开发人员生产力的问题。Rose 指出，例如，超过一半(52%)的受访者不允许他们的软件开发人员选择自己的工具。虽然很明显需要一种一致的方法来构建和部署应用程序，但 Rose 指出，DevOps 最佳实践不应该要求开发人员使用一组特定的工具。Rose 说，只有当开发人员能够用不同的工具进行实验时，他们才能真正创新。

总的来说，Rose 说 DevOps 的采用往往会经历不同的周期。如今，许多组织都在寻求减少组成工具链的工具数量。然而，这些努力不应该削弱开发人员的创新能力。Rose 指出，为 DevOps 工作流提供基础的 CI/CD 平台需要易于扩展。

目前还不清楚 DevOps 最佳实践的采用停滞到了什么程度，但在整个企业中实施 DevOps 的组织数量仍然有限。大多数组织都有可能在某种程度上采用 DevOps 的个人团队。在许多情况下，这些团队使用不同的工具和平台。

大多数组织显然希望能够更快地构建和部署应用程序。挑战不在于掌握 DevOps 平台，而在于克服许多限制 DevOps 采用的文化问题。然而，随着时间的推移，更多地依赖数字流程的需求将迫使 DevOps 问题出现。随着一个组织取得进步(部分是由于他们能够有效地采用 DevOps 过程),竞争对手组织将更加热切地确保他们不会失去阵地。企业高管将他们的竞争能力和组织掌握 DevOps 的程度联系起来可能还需要一段时间，但越来越明显的是，这一天不会太远。
