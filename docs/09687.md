# 对比安全为安全平台增加了 API 支持

> 原文:[https://devo PS . com/contrast-security-adds-API-support-to-security-platform/](https://devops.com/contrast-security-adds-api-support-to-security-platform/)

Contrast Security 已经完全扩展了其漏洞扫描和运行时应用程序自我保护(RASP)平台，现在[包括应用程序编程接口](https://www.prnewswire.com/news-releases/contrast-securitys-api-security-solution-further-enhances-protection-against-vulnerabilities-and-zero-day-attacks-301598846.html)(API)。

公司首席技术官 Jeff Williams 表示，Contrast Secure Code Platform [使用集成的静态和动态应用程序安全测试工具](https://techstrong.tv/videos/interviews/series-e-funding-contrast-security)的组合，将代理直接嵌入代码中，以检测漏洞。他指出，这种方法使得在构建和部署应用程序时，在 DevOps 工作流中嵌入漏洞扫描和漏洞防范功能变得简单。

Williams 说，相比之下，传统的应用程序安全方法依赖于网络安全团队部署的 web 应用程序防火墙(WAFs)等遗留平台，以一种不提供 API 级别的粒度控制的方式来保护应用程序。

![](../Images/d473aa87022b8733e262425f4157a80f.png)

对比安全声称，对比安全代码平台将扫描时间提高了 10 倍，同时修复缺陷的速度提高了 45 倍。该公司声称，它还通过提高 200%的准确性来减少误报警报。

随着越来越多的组织开始采用最佳 DevSecOps 实践来更好地保护软件供应链，他们中的许多人正试图在不降低应用程序构建和部署速度的情况下实现这一目标。然而，这种方法需要为开发人员和 DevOps 团队提供构建和部署安全应用所需的工具，而不需要安全运营团队的大量直接干预。

总的来说，随着越来越多的组织构建和部署基于微服务的应用程序，这些应用程序依赖 API 来互连离散组件，现在对 API 安全性的关注也越来越多。Williams 指出，这些 API 应该在部署之前由 DevOps 团队构建、部署和保护，就像任何其他软件组件一样。他补充说，从安全角度来看，当前构建应用程序的过程处于其核心，从根本上被破坏了。

一般来说，应用程序的安全性总是有些问题。网络安全团队倾向于资助他们经常使用的工具和平台。假设应用程序开发团队将投资于构建和部署应用程序所需的工具。唉，网络安全专业知识有限的应用程序开发团队假定网络安全团队正在保护底层基础设施和部署在其上的应用程序。因此，应用程序安全性经常资金不足。

在一系列高调的软件供应链违规事件之后，现在至少有更多的应用程序安全审查正在进行。对比安全是为了保护整个应用程序，包括 API，它使用在构建应用程序时注入应用程序的代理软件来调用。

尚不清楚使用代理软件而不是由安全运营团队管理的传统网络安全平台将在多大程度上解决应用程序安全问题。Williams 说，应用程序安全的责任正在向开发人员和支持他们的 DevOps 团队转移，但是只要人类编写代码，就会出现错误

当然，问题是这些团队的许多成员仍然更多地关注编写代码和部署应用程序，而不是网络安全。
