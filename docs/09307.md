# 数据狗展开应用安全服务

> 原文:[https://devo PS . com/data dog-un furls-application-security-service/](https://devops.com/datadog-unfurls-application-security-service/)

Datadog，Inc .今天发布了一项[应用安全监控(ASM)服务](https://www.datadoghq.com/blog/datadog-application-security-monitoring/)，该服务基于许多开发运维团队已经用于监控应用的相同代理软件。

Datadog 云安全平台产品副总裁 Pierre Betouin 表示，ASM 旨在使 DevOps 团队更容易将其他 DevSecOps 最佳实践添加到他们现有的工作流中。ASM 服务基于运行时应用程序自我保护(RASP)引擎和 web 应用程序防火墙，这是 Datadog 在一年前收购 Sqreen 时获得的。

ASM 然后采用分布式跟踪来发现代码级漏洞，如服务器端请求伪造(SSRFs)、SQL 注入和跨站脚本(XSS)缺陷。它旨在扩展现有的 Datadog 云安全平台，包括云安全状态管理(CSPM)、云工作负载安全(CWS)和云 SIEM 服务。

![](../Images/fa60c5de5ff760b0d609d0615e49aa3b.png)

Betouin 说，总体目标是通过消除在应用程序中部署和维护一组单独代理的需要，使开发运维团队解决应用程序安全问题变得更加简单。他补充说，这种方法减少了 DevOps 团队的阻力，否则他们会担心另一种应用程序安全方法会给应用程序环境带来额外的开销。

在一系列引人注目的违规和零日漏洞披露事件发生后，希望实施 DevSecOps 最佳实践的组织数量预计将在未来几周和几个月内增加。许多组织现在正在对其整个[软件供应链](https://devops.com/?s=software+supply+chain)进行安全审查，作为发现任何可能已被插入应用程序的恶意软件的努力的一部分。

这些组织面临的挑战是，随着应用程序环境变得越来越动态，维护应用程序安全性变得越来越困难，因为代码会不断更新。Betouin 指出，应对这一工程挑战的最佳方式是扩大许多 DevOps 团队已经部署的可观测性平台的范围。最终，我们的目标是尽可能自动化应用安全，但如果 DevOps 团队缺乏所需的环境，这将是无法实现的，他补充道。

尚不清楚组织采用 DevSecOps 最佳实践的比率。然而，显而易见的是，DevOps 团队对应用程序安全性负有更大的责任。大多数开发人员现在都有责任在一夜之间成为应用程序安全专家，因此有效确保应用程序安全的唯一方法是在应用程序部署到生产环境之前发现安全缺陷。

当然，实现这一目标有多种方式。Datadog 认为阻力最小的路径是通过云服务，连接到那些应用程序中插入的代理软件。多年来，DevOps 团队一直在以这样或那样的形式将代理软件插入到这些应用程序中。问题是，每添加一个新代理，就需要部署另一个软件组件并定期更新。如果不仔细跟踪，DevOps 团队可能会发现自己试图为所采用的每个平台管理整个代理软件组合。
