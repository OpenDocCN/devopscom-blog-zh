# 成熟的 DevOps 实践如何增强竞争优势

> 原文:[https://devo PS . com/how-a-mature-devo PS-practice-can-strength-competitive-advantage/](https://devops.com/how-a-mature-devops-practice-can-strengthen-competitive-advantage/)

*Saman Pourkermani，Inspirant Group 运营效率实践主管，本文作者之一。*

软件开发和交付已经成为业务各个方面不可或缺的一部分，不再仅仅是一种支持功能。如今，我们中的大多数人，包括那些在业务运营中的人，都知道不得不等待特定的环境发布日期来部署代码，以便对开发、 [QA](https://devops.com/?s=QA) 、UAT 或生产进行紧急修复或更新，这降低了工作效率。更糟糕的是，一个环境因没有备份环境而崩溃。我们都知道系统必须到位，以确保质量和安全性，但在开发运维方面，是什么让一些公司比其他公司更成功呢？

任何构建和交付软件应用和数字服务的组织都可以从 DevOps 中受益。主要区别在于他们的 [DevOps 文化和实践](https://www.cloudbees.com/blog/dora-devops-metrics-bandwagon)的成熟度。拥有成熟 DevOps 实践和文化理念的公司不仅能够比那些拥有传统 DevOps 实践的公司更快地构建和交付应用程序，还能更快地学习和创新。成熟的 DevOps 功能、工具和文化理念通过自动化和简化开发、测试、部署和基础设施管理流程来实现更快的创新。从交付和学习的角度来看，超越他人的能力是一个巨大的竞争优势。

大多数组织都明白这一点，因此一直在大力投资[完善他们的开发运维能力](https://devops.com/how-dora-metrics-can-measure-and-improve-performance/)和实践。然而，大多数公司都在努力取得有意义的进展，以赶上领先公司。

### 这是为什么呢？

最明显的原因是资源限制——预算、人力和/或系统。许多公司都有固定的预算，必须在建立昂贵的基础设施(配备年度升级)和其他业务需求之间进行权衡，以获得短期/长期利益。

我们认为，一家公司的 DevOps 成熟度与他们在制定业务优先级和权衡决策、选择他们将关注/致力于什么和不关注什么方面的良好和自律程度直接相关，并受到其显著影响——理想情况下是了解他们承诺完成多少工作。

不言而喻，大多数想要但还没有成熟的 DevOps 实践的公司都有很多工作要做，从设置和记录 CI/CD 流程到创建业务连续性和灾难恢复计划，再到确保他们有合适的执行人员。这些关键要素需要时间、精力、专业知识和预算来整合。虽然每个人都希望他们的团队能够及时完成所有工作，但现实是每个人的预算和能力都是有限的。

开发运维能力和自动化的进展经常被搁置，以支持正在进行的客户交付工作。诀窍是建立一个平衡团队能力分配的机制，以便它包括一个健康的工作组合，专注于在交付时变得更加有效和高效(例如，成熟的 DevOps 能力，改进架构和设计，等等)。)并努力满足客户需求。通过这种方式，团队的一部分能力可以始终用于推进 DevOps 交付能力，而其余部分可以用于解决客户请求。

### 如何实现这一点？

**从文化开始。**对于成熟的 DevOps 实践来说，文化是一个关键因素。团队成员必须履行各自的职责，但也要作为一个有凝聚力的单位运作。每个团队成员的心态都必须着眼于整体的更大利益，而不是个人。团队是否已经建立了一种合作的方式，这种方式在促进发展和创新的同时培养了责任感？

**关注过程。没有明确、合理的流程，就无法取得进展。例如，拥有一个良好记录和执行的 CI/CD 过程对于 DevOps 成熟度来说是至关重要的，以解决所有开发人员的痛点并确保平稳的部署过程。关键是要有一个文档化的过程，建立团队的工作方式并解决以下问题:合适的团队成员参与了吗？在开发/配置/部署过程中，是否有需要考虑的特殊情况？每个阶段大家对流程的理解是否一致？**

自动化和工具。您的流程是否实现了自动化，使得开发运维实践每次都以相同的方式进行，而无需人工干预？您是否拥有并知道如何使用所有成功的开发运维实践所需的基础工具 ( )例如，持续构建和测试代码、源代码控制(例如 GitLab)、环境配置管理、性能测量(例如负载平衡)？是否充分识别并解决了部署难点，以防止编码/发布瓶颈？

业务优先级和权衡决策。我们认为这是成为成熟的 DevOps 实践的最重要的方面，因为它允许随着时间的推移而进步。平衡团队的能力，包括提高开发运维交付能力，对于建立和保持竞争优势至关重要。优化您的 DevOps 能力可让您释放和利用团队成员的时间、才能和绩效来进行学习和创新。

正如杰克·韦尔奇曾经说过的，“唯一可持续的竞争优势是比你的竞争对手学得更快，并能够根据你所学的采取行动。”当一个软件组织变得过于僵化而没有一个适应性的 DevOps 边缘时，它将被远远地甩在身后。
