# 利用人在回路中的自动化转变数据运营的 5 种方式

> 原文：<https://devops.com/5-ways-to-transform-dataops-with-human-in-the-loop-automation/>

我们正处于数据复兴的中期。如今，不仅仅是数据工具，还有学习如何让数据运营成为整个组织真正的业务优势。数据已经成为产品不可分割的一部分，用来提升用户体验的质量。可靠的数据访问和数据的完整性是推动创新和业务成功的必要条件。

但是就像任何一个过程一样，会出现问题。将会有管道中断或数据没有被正确检测。问题就变成了，团队如何更快地发现问题并持续改进？数据完整性之旅如何成为迭代和改进之旅？

[DataOps](https://devops.com/?s=DataOps) 团队对自动化绝不陌生——自动化 CI/CD 管道和基础设施即代码(IaC)是他们技能组合的重要组成部分。但是有一些自动化组件在整个数据运营中并不普遍，这些组件可以帮助促进数据完整性并鼓励改进文化。

这一切都始于一种不同的自动化方法——人在回路(HITL)方法。人在回路自动化使部分流程完全自动化，同时使人能够在关键点采取行动、做出决策并决定前进的道路。

以下是使用人在回路中的自动化来推动持续改进和实现最佳数据完整性的五种方法。

### 创造更好、更早的问题中断

就像生活一样，不可避免地会出现问题，团队需要介入进行补救。目标应该是在这些问题发生时能够更好地中断它们——更早地识别问题并更容易和更迅速地将正确的利益相关者聚集在一起。

HITL 自动化不仅可以帮助团队更早地发现问题，还可以启动事件响应，并协调团队和利益相关方。理想情况下，自动化工作流可以由输入信号触发；例如，从像 DataDog 或 BigPanda 这样的可观察性工具。可以自动启动事件响应，从而触发进一步的自动化，例如创建票证、开始缩放会议、创建松弛渠道以及召集合适的团队成员。

通过立即召集合适的人员，团队可以协作调查，并更快地了解问题的状态。然后，HITL 自动化应该使团队能够运行进一步的脚本，以根据需要采取行动和补救措施。

### 通过自我记录的工作流程将改进融入到流程中

如今，许多公司都在问，“我们如何确保我们拥有完整的集成环路，一直到收集点？”团队之间的沟通对这项工作至关重要。但是，即使在团队可以协作改进之前，他们也需要了解数据的当前状态，这样，如果某个团队发现某个报告没有提供所需的内容，就有一种非常干净的方法将该信息反馈给 DataOps 团队，以便首先改进工具。

HITL 自动化通过自动记录整个流程中的每一个人和机器动作，可以改进数据操作工作流程。因为它通过 API 连接到团队的所有工具和服务，一个自我记录的工作流吸收每一个动作并创建一个审计跟踪。这种审计追踪是团队能够更准确地理解数据完整性以及团队如何实现和运行过程的基础。

### 将人类数据带到改进的前沿

数据运营团队在确保收集正确数据方面发挥着重要作用。然而，许多人仍然在使用非常手动的系统。他们可能对自己的日志和监控数据有很好的理解，但对组织周围的人的数据(人们正在运行什么流程和工作流)仍然没有很好的了解。

理解人类如何运行流程与理解技术问题一样重要。上面描述的自记录工作流应该包括人员数据的记录，从手动运行的脚本到 Slack 或团队对话，理想情况下，甚至自动更新票据，这样即使维护记录系统也变得不那么手动。只有全面审视人类和机器数据，团队才能在自动化、沟通和工作流程中找到需要改进的地方。

### 采取渐进的方法

不要让“完美”成为“完成”的敌人。我所见过的每一次伟大的变革都始于一个小团队、一个伟大的使命和来自执行团队的强大支持。增加大量的新技术和雇佣更多的人可能不是团队需要的灵丹妙药。

相反，团队应该首先考虑他们内部拥有的专业知识——人员、技术和流程。然后，开始采取渐进的方法。HITL 通过让团队首先整理机构知识、分析、学习，然后开始小批量地自动化各个部分，来促进渐进的、可实现的自动化。

一头扎进完全自动化的方法会让团队处于比什么都没有更糟糕的状态。自动化是一个旅程——我们通过做来学习，通过记录人和机器的数据，然后以洞察力为指导，开始在少量的自动化中发现真正的价值。这些小步骤可以产生巨大的影响，随着时间的推移，累积起来会产生巨大的商业价值。

### 为行业标准而购买，为差距而构建

正如我上面提到的，团队应该首先评估他们当前的资源。他们已经拥有哪些技能和技术，他们如何使用这些资源来实现他们的数据目标？与此同时，许多开始自动化之旅的团队面临着挑战——缺乏开发资源，脚本存在于一个人的机器上，难以将所有的人和机器数据整合到一个记录的时间表中。

为了帮助数据团队有效、可靠地实施 HITL 自动化，对现成平台的投资有助于弥补团队和组织之间的差距。平台应该通过低代码接口实现自动化，同时提供代码级定制。这种组合使团队能够购买行业标准，同时弥补差距——不管理平台意味着开发资源可以专注于扩展和改进自动化。

平台应该使团队能够容易地连接到 API 并适应定制的 API。DevOps 过程和工具的持续变化与快速变化的 API 共存。因此，最有前途的自动化平台必须管理 API 的复杂性，使用户能够专注于目的，而不是定制 API 的机制。自动化技术应该无缝地处理身份、认证、分页和缓存。

### 寻找 HITL 自动化的优势，推动业务发展

随着数据在产品生命周期和用户体验中越来越根深蒂固，自动化成为加快创新、提高产品质量和可靠性的关键。创造一种持续改进的文化，并实施像 HITL 自动化这样的技术，在整个产品生命周期中促进数据完整性，这是我看到的对组织最大的杠杆作用。
