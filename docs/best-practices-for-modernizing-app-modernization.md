# 应用现代化的最佳实践

> 原文：<https://devops.com/best-practices-for-modernizing-app-modernization/>

根据谷歌云的 [2021 年加速发展状况报告](https://cloud.google.com/blog/products/devops-sre/announcing-dora-2021-accelerate-state-of-devops-report) ，对遗留系统进行现代化改造以采用松散耦合的微服务架构是连续交付成功的最大预测因素。研究继续表明，最优秀的 DevOps 执行者享有更频繁的代码部署、更快的部署准备时间、更低的变更失败率和更快的恢复时间。

网飞、特斯拉和星巴克等领先企业证明，微服务架构最大限度地减少了服务之间的细粒度依赖，然后可以独立测试、部署和扩展。结合 DevOps 最佳实践，这减少了技术债务，并允许基于工厂模型方法的现代化，该方法结合了自动化分析和深度可观察性。

也就是说，到目前为止，应用程序现代化一直是一个手动过程，需要多年的项目，失败率很高。我们如何应用现代工具、已知的 DevOps 最佳实践和期望的业务优势来改善和加速公司的现代化工作？换句话说，我们如何让现代化更现代化？

通过使用自动化、人工智能和数据科学，建立可重复的管道和向左转移架构，建筑师可以在技术债务变得不可收拾之前解决它。在本文中，我们将更详细地探索这些实践。

### 智能地自动化一切可能的事情

DevOps 方法采用手动步骤并应用自动化来创建可扩展的可重复流程。今天的现代化团队迫切需要这些方法，因为他们受困于不能跨整个应用程序资产扩展的手动方法。因此，将智能自动化、人工智能和数据科学应用于现代化——并为重构创建可重复的管道——是过去十年 DevOps 成功的自然延伸。

### 持续集成、持续部署和持续现代化

CI/CD 方法是 DevOps 团队的核心最佳实践，并且已经是开发组织在其应用程序生命周期中使用的熟悉模式。但是，没有持续的现代化就没有持续的交付。精英团队持续监控技术债务积累，并在每个应用程序发布周期解决它，从而在其应用程序生命周期中实现持续的现代化。

### 深度应用可观察性

可观察性定义为监控、记录和跟踪，是云原生架构的基础。对现有的 monolith 进行现代化改造需要超越简单的静态代码分析的深度可观察性，以包括更深层次的领域驱动的行为分析和动态跟踪依赖性交互。这也是发现不再使用的死代码的秘密。

### 确保架构对等

您必须了解您的基础架构，以便为您的应用程序架构和现代化选项提供信息。提升和转移给许多组织带来了头疼的问题，因为整体式应用程序的传统三层架构具有与云原生应用程序完全不同的拓扑结构，云原生应用程序是从头开始设计的，旨在利用云的效率。

无论是混合方法(部分内部部署，部分云)、公共云还是私有云，确保您的现代化战略确保目标应用程序与其在云中的最终目的地之间的架构对等都非常重要。此外，理解您的现代化选项也很重要:重新托管、重新平台、重构、重写等。哪些是你可以举一反三的，哪些是需要更多注意的？

为了找到答案，整合数据科学驱动的分析、自动化服务提取，以及更重要的人，以帮助您了解可能性是至关重要的。

### 为您的现代化项目获得管理层支持

这并不是简单地处理你的积压工作；对于这种规模的项目，您的执行发起人需要从业务角度理解，为什么这对组织至关重要。有大量指标显示了维护老化和有问题的任务关键型应用程序的财务和机会成本。

为了让管理层接受您的现代化计划，您需要制定一个业务案例，传达维护传统应用程序的成本，包括维护的时间和资源，以及对创新、客户获取和保留以及上市速度的负面影响。

DevOps 实践的出现是为了改进 IT 组织如何可靠而高效地将软件从开发交付到运营并交付给客户。将这些自动化、管道和深度可观察性的新兴技术应用于应用程序现代化计划，将使团队能够加速他们交付应用程序转换项目的方式。通过投资在这些方法中培训应用程序团队，并提供现代工具来支持这种培训，组织可以更可靠、更高效地将人员和项目推进到下一代云应用程序中。
