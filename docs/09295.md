# MDR for DevSecOps:托管安全如何帮助您向左转

> 原文：<https://devops.com/mdr-for-devsecops-how-managed-security-can-help-you-shift-left/>

什么是托管检测和响应(MDR)？ 托管检测和响应(MDR)是一项外包服务，可帮助组织检测终端上的威胁，对其做出响应，并执行主动式威胁搜寻。

MDR 服务通常包括部署在公司终端上的终端检测和响应(EDR)平台，以及人类安全专家。MDR 提供商为客户提供安全分析师和工程师，他们可以在端点安全、[](https://www.catonetworks.com/network-security/)、警报分类和响应真实安全事件方面提供帮助(这里有一些进一步的阅读和关于托管检测和响应平台 ) 的 [更多背景信息)。](https://www.cynet.com/mdr/understanding-managed-detection-and-response/)

### MDR 为什么重要？

传统的托管安全服务(MSS)提供基本的监控和警报，无法提供识别、响应和补救真实安全事件所需的背景和指导。它使组织暴露在许多通过这些基本服务的威胁之下。

### MDR 如何帮助保护组织

MDR 提供了一种主动的、结果驱动的方法，超出了 MSS 供应商所能提供的范围。常见的 MDR 服务包括:

*   持续的网络和端点监控。
*   安全编排。
*   威胁狩猎。
*   综合响应行动，如远程威胁遏制。
*   高级 MDR 服务还提供跨云的扩展检测，如 AWS、GCP、Azure 和常见的 SaaS 套件，如 Google Workspace 和 Office 365。其他提供商可以在运营技术(OT)环境中保护事故指挥系统(ICS)和监控与数据采集(SCADA)系统。

### 什么是 DevSecOps？

[DevSecOps 是一种软件开发方法学](https://resources.whitesourcesoftware.com/blog-whitesource/devsecops) ，它统一了以前分散的开发(Dev)、安全(Sec)和运营(Ops)团队。它旨在将安全性集成到所有开发生命周期阶段，包括设计、开发、测试、部署和交付。它涉及使用自动化来集成所有阶段的安全性。

过去，安全任务在开发管道的末端由独立的安全团队执行。接下来，质量保证团队对产品 和 进行了测试。当开发人员一年只发布一次或两次软件更新时，这种方法是有效的。然而，一旦软件开发团队采用敏捷和 DevOps 管道来减少开发时间，安全性就成了瓶颈。

### 【DevSecOps 如何提高安全性

DevSecOps 将安全性集成到敏捷和 DevOps 管道中。它确保团队可以在开发周期的早期阶段解决安全问题。因此，在产品 发布 投入生产之前，团队可以在更容易、更快地修复时应用补丁。

DevSecOps 让安全性成为所有团队的共同责任，包括开发、安全和运营团队。创建安全孤岛不再是一个团队的唯一责任。相反，它使团队能够自动化安全软件交付，在不减慢软件开发周期的情况下优先考虑安全性。

### 受管检测和响应(MDR)解决方案如何让开发人员受益

DevOps 革新了应用程序开发。然而，由于现代开发方法的高速度，它们会引入安全漏洞和缺陷。

除了 DevOps 管道中出现的常规安全漏洞，新的复杂攻击，最重要的是供应链攻击会感染开发系统本身或第三方软件组件。这会导致在被检测到之前就危及数百万设备的大规模攻击。

### MDR 能有什么帮助？

MDR 是一种相对较新的网络安全产品，但开始对寻求更好保护和更低运营开销的公司产生重大影响。MDR 和 DevOps 之间有很多工装裤。为了将网络安全集成到生产和部署管道中，一些组织正在转向 DevSecOps 方法，MDR 可以提供帮助。

DevSecOps 使网络安全成为整个 DevOps 生命周期不可或缺的一部分。随着越来越多的组织实施 DevSecOps，他们可以用 MDR services 提供的安全专业知识来补充内部团队的知识和安全最佳实践。

### MDR 如何让 DevOps 保持领先

MDR 通过让开发人员及时了解新出现的威胁和漏洞，并为开发和部署管道中的持续安全监控和响应提供高级工具，使开发运维团队始终领先于最新的威胁。MDR 的一个主要优点是，它提供了可以采取措施防止攻击的专家，而不仅仅是提供建议。

MDR 成为一个支持元素，允许开发人员编写更安全的代码，并更好地保护他们的 DevOps 环境。在 MDR 技术和专家的帮助下，识别可以绕过安全控制的高级威胁变得更加容易，以免它们影响开发运维环境或最终的软件产品。

### 选择 MDR 解决方案的考虑事项

*   在选择适合 DevOps 环境的 MDR 平台时，有几个重要的考虑事项:
*   了解 DevSecOps 环境中常见的攻击面，以及监控相关环境的能力。
*   采取措施[消除误报](https://devops.com/prevent-false-positives-from-derailing-shift-left/)并对警报进行优先级排序。DevOps 团队高速运转，没有时间费力处理不相关的警报。
*   在开发生命周期的所有阶段实现自动分类和响应。在早期阶段，这应该采取为开发人员提供可行建议的形式，而在后期阶段，则是为生产环境提供完整的事件响应。

当您找到将您的组织转变为真正的 DevSecOps 过程的正确方法时，所有这些都是有用的。
