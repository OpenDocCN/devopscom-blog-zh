# WSO2 将 AI 应用于 API 管理

> 原文：<https://devops.com/wso2-applies-ai-to-api-management/>

WSO2 今天展示了一个 [Choreo](https://www.globenewswire.com/news-release/2021/06/16/2248286/0/en/WSO2-Introduces-Choreo-a-Next-Generation-Integration-Platform-as-a-Service-for-Cloud-Native-Engineering.html) 云平台，通过这个平台，DevOps 团队可以利用机器学习算法和其他形式的人工智能(AI)更有效地创建和管理应用编程接口([API](https://devops.com/?s=APIs))。

WSO2 首席技术官 Eric new merger 表示，集成平台即服务(iPaaS)消除了所有与建立平台和管理基于 API 的服务相关的任务。

Choreo 可用于测试版，它利用人工智能来学习历史活动和表现行为，以便预测开发人员的需求。新人说，随着开发人员构建 API，该平台将提供性能反馈，识别异常，映射数据，甚至完成代码。

Choreo 平台还带有内置的可观察性工具，用于监控 API 使用情况和排除 API 故障，新人补充道。

此外，Choreo 提供了对该公司开发的开源芭蕾舞编程语言的访问，使开发人员能够使用他们认为合适的低代码和程序代码来创建 API。新人说，开发人员创建的低代码图生成源代码，然后开发人员可以在他们想去的任何地方运行。新人补充道，开发者可以关闭 Choreo 提供的整个知识库。

开发人员还可以触发低代码集成逻辑，通过 WSO2 或他们自己创建的预建连接器和模板来控制数百个软件即服务(SaaS)API。

WSO2 还使得在 Kubernetes 环境中通过单击部署 API 成为可能，这消除了管理繁琐的 YAML 文件的需要。这种能力是基于 WSO2 通过[收购 Performer](https://www.globenewswire.com/news-release/2021/06/16/2248292/0/en/WSO2-Acquires-Platformer-to-Extend-Kubernetes-Capabilities-of-Choreo-Next-Generation-Integration-Platform-as-a-Service.html) 获得的技术，该公司今天透露了这一点。

最后，WSO2 提供了一个 API 市场，组织可以通过它共享 API、事件流、数据源和模板。

随着基于微服务的应用的兴起，API 管理变得更具挑战性。每增加一个微服务，就增加一个需要管理和维护的 API。随着更多 API 的使用，DevOps 团队在没有机器学习算法帮助的情况下，跟踪所有这些 API 之间存在的依赖关系变得越来越困难。

机器学习算法不太可能在短期内取代对 DevOps 专业人员的需求。然而，显而易见的是，今天 DevOps 团队例行执行的许多手动任务正在被机器学习算法自动化。在其他情况下。DevOps 团队正在获得新的功能，这些功能要么是他们自己无法完成的，要么是需要太多时间才能完成的。

当然，任何 AI 平台学习 DevOps 环境都是需要时间的。IT 团队需要投入大量时间来训练 AI 模型识别相关问题，就像他们训练 DevOps 团队的任何新成员一样。唯一的区别是，一旦机器学习算法学会了一些东西，它们就永远不会忘记。他们也不会因为在别的地方找到了更好的工作而休假一天或宣布离开公司。

DevOps 团队可能需要一段时间才能在人和机器之间找到正确的平衡，但有一点可以肯定的是，DevOps 环境中的人工智能将会一直存在。
