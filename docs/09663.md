# 分布式云的优势

> 原文:[https://devops.com/the-benefits-of-a-distributed-cloud/](https://devops.com/the-benefits-of-a-distributed-cloud/)

短短几年间，应用程序的创建方式发生了巨大的变化。现在一切都是关于快速组装和集成；使用微服务构建并容器化的应用程序。容器和云无疑是这一进步的催化剂，并将继续成为开发人员创建新的创新应用程序的基础设施。

然而，随着工作负载向云转移，越来越多的企业意识到他们需要一个能够快速适应其特定需求的云。计算环境可能是多种服务器类型的混合体:本地服务器、裸机服务器、共址服务器和跨多个洲分布的服务器。没有一个单一的云架构能够为所有用户提供始终如一的卓越应用性能。

在混合云、内部云、多种云、托管云、集中式云和分布式云之间，很难保持所有术语的一致性。这些云架构有什么区别？为什么使用一个而不是另一个？在本文中，我们将只关注其中之一:[分布式云](https://www.ridge.co/)。

## 什么是分布式云？

与所有计算都在一个中心位置完成的大型公共云不同，分布式云由一个服务器系统组成，这些服务器并不都在同一个位置，尽管它们是通过一个云提供商来管理的。分布式云中的服务器实际上可以在世界上的任何地方。这使得计算可以在离终端用户更近的地方进行，从而从多方面改善用户体验。

## 使用分布式云的优势是什么？

下面是使用分布式而非集中式云的六大优势。

### 一种尺寸不适合所有人

计算环境可能是多种服务器类型的混合体:本地服务器、裸机服务器、共置服务器和跨多个洲分布的服务器。云提供商需要灵活地为其终端用户和业务需求定制云。

### 数据法规合规性

许多国家/地区对其公民数据有特定的合规要求，并且不允许将个人/财务数据托管在其境外。因此，在这些国家拥有分布式服务器可以缓解这个问题。

### 减少延迟

在分布式云模型中，服务器可以放置在各种位置，包括边缘云中。当服务器放置在离最终用户更近的地方，而不是在一个集中的位置时，数据不需要传输很远就能回到服务器。

### 改进的 CDNs

分布式云可用于部署内容交付网络(cdn ),以加速内容和服务交付。大量内容(例如视频)可以由更接近终端用户的分布式云存储和交付，这增强了用户体验。

### 网络弹性

网络连接一直是云提供商的一大担忧。如果他们的系统崩溃，用户的工作负载将无法运行。然而，在分布式云系统中，网络故障的风险降低了，因为如果系统崩溃，云服务可以驻留在本地子网中。

### 位置灵活性

分布式云总是可以变得更加分布式。可以将更多位置添加到网络中，从而拓宽云中可用的全球计算区域。想要联系全球特定位置的用户的公司可以使用分布式云来联系他们。

## 改变人们对云计算的看法

为了满足多功能性的需求，一种新型的云出现了:统一分布式云(UDC ),其中的工作负载可以在地球上任何地方的数千个位置运行。这种云通过一个 API 连接到世界各地的现有基础设施。 企业可以享受在多个地点运行工作负载的好处，而无需维护任何分布式基础设施。从单个用户界面中，用户选择一个现有的数据中心来发送他们的工作负载，并在几分钟内开始部署。

UDC 将异构基础架构转换为同构云计算平台，然后可以利用该平台支持向终端用户交付云服务。它旨在与任何私有、混合云或多云架构进行互操作。因此，企业可以创建一个优化的云计算战略，利用他们选择的位置中现有的可用基础设施。因为它基于现有的基础设施，所以不需要任何额外的资本支出。

与传统的云模式相反，UDC 可以改变人们对公共云的看法。虽然大型公共云提供商受益于规模经济，但分布式云利用了位置和分布经济，因此可以灵活地处理新兴的位置相关应用，如新一轮的物联网应用。
