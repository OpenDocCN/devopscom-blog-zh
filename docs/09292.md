# 利用新的 IT 模式解决多重云的复杂性

> 原文:[https://devo PS . com/用新的 it 模式解决多重云计算的复杂性/](https://devops.com/tackling-the-complexities-of-multi-cloud-with-a-new-it-model/)

疫情给组织带来了难以置信的压力，迫使他们加快数字化转型的步伐，作为回应，许多企业纷纷采用云战略。根据今年 Nutanix 发布的 [企业云指数](https://www.nutanix.com/enterprise-cloud-index) ， 多云—一种结合了多个云(公共云或私有云)的 IT 运营模式—是最常部署的 IT 环境，也是唯一一种处于上升趋势的 IT 模式。事实上，近三分之二( 64%)的组织计划在三年内运营一个 多云 环境。

但是，组织仍在努力应对跨云边界管理的复杂性和挑战，对安全性、数据集成和成本 的担忧依然存在。企业将需要找到利用现有云投资的方法，并重新定义最佳实践以及 it 敏捷性和弹性的含义。

### 当前挑战

传统上，企业是围绕管理成本 、提高效率和简化运营而设计的，但精益 IT 团队发现自己面临新的压力，需要支持我们今天面临的挑战。疫情迫使组织对实施进行优先排序，并将公共云视为快速 解决方案，而没有以优化为代价进行适当的尽职调查。这种方法缺乏对战略业务成果的关注，最终导致复杂性增加，成本 更高 和灵活性更低。根据该研究，87%的回复者同意 多云 成功需要跨混合云基础架构的更简单管理。

企业也继续将提高安全性列为 IT 工作的重中之重，然而管理安全性却是最大的挑战之一。另外， 大多数 受访者(80%)表示，移动应用程序既耗时又昂贵，77%的受访者认为工作负载可移植性是一项重大挑战。这强调了跨不同云环境的统一管理和操作的需要，以避免移动应用程序的高成本和耗时的工作。

### “云智能”IT 方法

当前的云思维认为，每个工作负载或应用都有一个“最佳”的云环境，但是对于 云，没有一种放之四海而皆准的方法。组织将开始摆脱一揽子云优先的 IT 方法，取而代之的是，更加考虑哪些工作负载在哪里运行。将所有应用程序或工作流程永久迁移到云端是最好的选择，这已经不再是一个定局。采用这种“云智能”方法，利用最适合每个用例的云环境，很可能是 多云 激增背后的主要驱动因素。

随着时间的推移，组织对 [IT 基础设施](https://devops.com/the-need-for-agile-adaptable-it-infrastructure/)的使用 变得更具战略性。近四分之三的 IT 专业人员(72%)认为，与一年前相比，现在他们的业务中的 IT 职能更具战略性。虽然企业现在比以前更具战略性地看待和使用 IT，但 多云 的复杂性带来了挑战，阻碍了云的成功。尽管存在挑战，但仍有充分的理由说明为什么 IT 的未来是混合 多云 ，这是一种能够实现跨 c 云的更高运营一致性的模式。事实上，83%的人认为它是理想的。

### 多云 支持新的工作方式

疫情改变了几乎所有组织的运营方式，企业改变其基础设施 和部署模式的首要原因是改善远程工作和协作。虽然组织的远程工作人员可能会有所波动，但在可预见的未来，他们将会一直存在，而且 IT 需要灵活地支持远程、混合的工作人员，并保持人们的工作效率和联系，无论他们在什么位置 。

由于疫情，超过一半(61%)的组织专注于提供更灵活的工作设置，而 云计算 支持这种新的工作方式。它提供了最灵活的 It 环境来支持这种灵活性，方法是分配数据并允许从不同地理位置远程访问业务应用程序，以实现用户接近和业务连续性，并使组织能够实施有助于留住和吸引人才的技术。

多云 将继续存在— 并且是未来几年唯一有望增长的运营模式。2022 年将向混合 多云 部署转变，因为组织希望利用每种云所提供的优势，同时仍然拥有围绕其应用程序和数据的灵活性 和选择自由。这将提供大多数组织在当今数字世界中寻求竞争的灵活性和敏捷性，同时使企业能够显著简化运营。经得起未来考验的 c 响亮的策略是构建 IT 弹性的关键。
