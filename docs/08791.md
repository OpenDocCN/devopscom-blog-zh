# 新独立的 VMware 将专注于多重云

> 原文：<https://devops.com/newly-independent-vmware-to-focus-on-multi-cloud/>

在[正式脱离戴尔技术](https://news.vmware.com/releases/vmware-announces-completion-of-spin-off-from-dell-technologies)之后，作为一家独立的公司，VMware 将再次专注于提供跨多个云计算环境的管理平台。

VMware 负责云原生应用业务部门的执行副总裁 Ray O'Farrell 表示，该公司现在有一个独特的机会与云服务提供商、内部 it 基础架构的构建者和电信服务提供商密切合作。

剥离的条款包括 115 亿美元的特别现金股息，这导致向所有 VMware 股东支付每股 27.40 美元的股息。VMware 声称，该交易带来的简化财务结构将为其提供额外的运营和财务灵活性。

展望未来，大部分开发工作将专注于提供[管理平面](https://devops.com/?s=multi-cloud+management)，这些管理平面不仅跨越多个云，还能解决特定的跨平台需求，如安全性。O'Farrell 指出，不可能有一个单独的超级控制飞机来完成所有任务。相反，IT 团队应该期待 VMware 提供一系列集成良好的控制平面，以打破 IT 孤岛的方式自动化特定任务，他说。

然而，根据其现有联盟的条款，VMware 也将继续与戴尔密切合作，以创建针对 VMware 虚拟机管理程序平台优化的服务器环境，O'Farrell 说。

O'Farrell 指出，该战略的关键是 VMware 计划继续扩展的可观察性工具组合。他补充说，VMware 还将满足在虚拟机上运行的传统单片应用程序以及在 Kubernetes 发行版上运行的云原生应用程序(如 VMware Tanzu)的要求，这些应用程序可能部署在虚拟机管理程序或裸机服务器上。

O'Farrell 说:“鉴于当今 IT 环境的复杂状态，出错几乎是不可避免的。他补充说，VMware 面临的挑战和机遇是为 IT 团队提供必要的观察工具，以便在问题对业务产生负面影响之前发现尽可能多的问题。他指出，一旦发现这些问题，IT 团队应该能够使用 VMware 管理框架产品组合的其余部分来自动化补救过程。

O'Farrell 说，这些管理框架还将为统一传统的 IT 服务管理方法(ITSM)和现在使用更广泛的 DevOps 最佳实践提供基础，特别是在管理云原生应用程序方面。

谈到内部 it 环境，VMware 继续占据主导地位。到目前为止，其虚拟机管理程序是本地数据中心服务器上使用最广泛的。VMware 在过去两年中花了很多时间将其业务扩展到多个云领域，以使 IT 组织能够基于一系列管理框架实现标准化。目标是通过减少对孤立管理框架的依赖来降低 IT 的总成本。VMware 经常遇到的挑战是，管理云计算环境的 IT 团队并不总是管理内部 IT 环境的团队。因此，许多部署了云应用程序的 IT 团队倾向于采用云服务提供商提供的管理框架。

然而，VMware 认为，随着越来越多的组织寻求降低 IT 总成本，使用其框架来整合 IT 管理的经济压力将会非常大，不容忽视。
