# Rust Foundation 与 OpenSSF 和 JFrog 合作保护代码

> 原文：<https://devops.com/rust-foundation-allies-with-openssf-and-jfrog-to-secure-code/>

Rust 基金会今天宣布[将与开源安全基金会(OpenSSF)和 JFrog](https://www.prnewswire.com/news-releases/rust-foundation-establishes-security-team-to-support-and-advance-rust-programming-language-301622579.html) 合作，帮助维护者保护使用 Rust 编程语言创建的开源软件。

Rust 基金会的执行董事丽贝卡·伦布尔(Rebecca rum bul)表示，尽管 Rust 是一种内存安全的语言，但维护者仍需要解决一些安全问题。目标是为那些维护者提供更好地保护他们的软件供应链所需的资源。

作为这项工作的一部分，Rust 基金会正在增加一个专门的安全团队，以帮助使用 Rust 构建的[开源](https://devops.com/?s=open+source)项目的维护者更好地保护他们的软件供应链。该团队由 JFrog 和 Alpha-Omega 计划资助，该计划由 Google 和微软资助，最近由 OpenSSF 发起。JFrog 上周宣布，它将与亚马逊网络服务(AWS)、谷歌、华为、Meta、微软和 Mozilla 一道，以白金级加入 Rust 基金会。

Rumbul 说，安全团队的第一项举措将是进行安全审计和开发威胁建模工具，为维护者提供一套最佳安全实践的基础。该团队还将帮助倡导 Rust 领域的安全实践，包括 Cargo package manager 和 Crates.io 目录项目。

OpenSSF】之前已经呼吁取代像 Java 这样的不安全内存的编程语言,作为保护开源软件供应链的更大计划的一部分。

总的来说，Rust 基金会致力于帮助 Rust 开发者实现他们的规模、安全性和可持续性目标，Rumbul 说。她补充说，鉴于 Rust 编程语言的早期阶段，现在是开始解决应用程序开发人员不能忽视的安全问题的最佳时机。

Rumbul 说，从长远来看，Rust 基金会还致力于为开发人员提供 Rust 安全教育，并致力于提出一种公平的方法来认证哪些项目已经达到了适当的网络安全成熟度水平。

目前还不清楚开发人员是否正在放弃旧的编程语言，转而使用诸如 Rust 这样的内存安全语言。Rust 使用编译器来跟踪可以使用一次的值的所有权，并使用借用检查器来管理数据的使用方式，而不必依赖传统的垃圾收集技术。

当然，替换已经使用各种遗留语言构建的数万亿行代码是一项巨大的任务，可能永远也不会完全完成。但是，随着新的应用程序是使用内存安全的编程语言构建的，组织的整体安全状况将稳步改善。当然，挑战在于知道如何使用这些语言构建应用程序的开发人员数量仍然相对较少。

然而，也许有一天，默认情况下，应用程序的编码方式会比现在安全得多。然而，与此同时，应用程序环境中存在的漏洞数量仍然多得难以计数。
