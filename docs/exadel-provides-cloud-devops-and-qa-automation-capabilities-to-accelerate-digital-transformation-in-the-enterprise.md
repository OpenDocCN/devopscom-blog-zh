# Exadel 提供云、DevOps 和 QA 自动化功能，以加速企业的数字化转型

> 原文：<https://devops.com/exadel-provides-cloud-devops-and-qa-automation-capabilities-to-accelerate-digital-transformation-in-the-enterprise/>

*云、DevOps 和 QA 自动化服务使企业客户能够领先于客户和企业不断增长的需求*

加州核桃溪，2019 年 4 月 16 日——全球领先的数字软件工程解决方案供应商 Exadel (www.exadel.com)今天宣布了其数字转型实践(DTP)的新更新。Exadel 数字化转型实践汇集了现代应用设计和开发(ADD)方面的专业知识，现在包括云优先和云混合设计、微服务架构支持以及 DevOps 和 QA 自动化服务。

凭借数十年与客户在软件开发项目上的合作经验，Exadel 开发了一套打包解决方案，专注于解决与成功的数字化转型计划相关的现代技术。该解决方案包括围绕构建基于 Kubernetes 的应用程序开发模型的最佳实践，使用流行的开源基础设施即代码软件工具 Terraform 完成自动化部署，以及 QA 自动化和微服务开发的内置最佳实践。因为它使用 Terraform，所以该解决方案可以部署到多个云提供商——它目前支持 AWS、Azure 和 Google Cloud。

它是一种通用的解决方案，包括:

1.  防火墙——使用堡垒作为公共互联网和非军事区(DMZ)之间的安全层
2.  DMZ–保护内部资源的公共子网
3.  基于 Kubernetes 的应用基础设施——部署 Kubernetes 和容器基础设施，包括自动扩展
4.  QA 自动化模板——承载 Selenium 和 QA 自动化技术的容器
5.  微服务开发模板–用于全球应用服务和本地应用服务发现的模板
6.  数据服务连接器–为特定云提供商托管数据服务的独立子网

这些组件中的每一个都可以分离出来并独立部署，以加速更传统的 DevOps 项目。

Exadel 工程和数字化转型副总裁 Jonathan Fries 表示:“我们很高兴能够扩展我们的数字化转型能力，以更好地支持我们的企业客户及其数字化转型之旅。“任何正在进行数字化转型的组织都必须考虑云迁移、分离传统架构、DevOps 工具和实践以及 QA 自动化等软件工程趋势。通过 20 多年的经验，Exadel 能够为客户提供一揽子解决方案，旨在解决他们最棘手的软件工程挑战。我们知道每个组织都是独一无二的，可以根据每个客户的目标和需求定制我们的服务。”
