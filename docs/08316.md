# 2021 年网络安全管理软件产品调查报告:数据库的优先权和陷阱

> 原文：<https://devops.com/the-solarwinds-query-report-2021-database-priorities-and-pitfalls/>

混合 IT 的兴起和云原生应用的加速采用，已经改变了传统的数据库设计、管理和监控。工作负载正在快速转移到云，应用程序性能严重依赖于底层数据库的性能。与此同时，全球新冠肺炎疫情迫使企业在去年做出重大改变。尽管这在过去和现在都是必要的，但向远程工作的过渡和许多组织的快速战略转变给负责支持新员工需求和转变业务优先级的技术专业人员带来了巨大的挑战。

因此，组织越来越认识到数据库性能管理和策略的重要性，以确保他们最有价值的资产:数据。网络安全管理软件产品最近公布了[《2021 年网络安全管理软件产品查询报告:数据库优先事项和陷阱](https://orangematter.solarwinds.com/wp-content/uploads/2021/06/SolarWinds-The-Query-Report-FINAL-NA_Stacked-Charts.pdf)的调查结果，该报告调查了[数据库](https://devops.com/?s=database)管理的现状，评估了数据库专业人员今天面临的主要挑战，并概述了数据库专业人员如何才能最好地克服这些挑战。

北美的结果基于对 2021 年 2 月进行的在线调查的回复，其中包括来自美国和加拿大的公共和私营部门中小企业组织的 331 名负责数据库性能管理的技术专业人员的参与。

在许多其他重要发现中，今年的报告揭示了以下内容:

## 日益增加的复杂性

随着复杂性的增加，数据库设计和管理也在不断发展，IT 专业人员即将进行重大战略转变，将云数据库即服务(DBaaS)平台作为关键的数据库性能成功因素。

*   虽然近 80%的受访者表示 Microsoft SQL Server 是目前使用的最重要的平台，但第二、第三和第四大最常用平台之间的比例相对均衡:云 DBaaS (54%)、Oracle (47%)和 MySQL (43%)。
*   随着技术专家与数据库平台的进步保持同步，并表明有意向云 DBaaS 的采用转变(33%的受访者将云 DBaaS 列为未来三年要采用的头号优先数据库平台)，传统/云 DBaaS 平台的新组合也需要新技能。
*   31%的受访者表示，在过去一年中，由于新冠肺炎和向远程工作的过渡，复杂性有所增加，他们将用户数据需求的增加、数据量的增加以及安全性/合规性问题(如 GDPR)列为复杂性增加的三大驱动因素。

业界开始认识到云 DBaaS 平台是数据库性能的关键成功因素。当谈到云迁移时(尤其是在疫情急剧加速云计算和数字化转型的规划和执行的情况下)，数据库专业人员应该从基本的数据和代码清理开始，并确保只迁移真正需要的东西，以降低复杂性和总体成本(我们知道[云计算成本](https://devops.com/?s=cloud+costs)可能会增加)。一个全面、成功的迁移策略还应该包括对数据资产的详细评估——存在哪些依赖性或复杂的编码特性？请记住，与数据库互连的一切都必须与它一起进入云，因此采取这些初步措施将有助于组织避免未来的意外。

## 数据库安全性/合规性影响优先级

云和安全性/合规性是影响 2021 年及以后的优先事项和资源分配的关键因素。

*   近 60%接受调查的技术专家表示，在他们组织内当前管理的数据/系统中，只有不到 25%位于云中。但这种情况有望改变，因为 36%的受访者表示，集成云 DBaaS 是他们组织在 2021 年的首要任务之一。
*   五分之二的受访者预计 2021 年(与 2020 年相比)云服务和安全性/合规性的预算分配会更大(分别为 42%和 39%)。

管理数据库平台是一项艰巨的任务，是主动响应和被动响应之间的平衡。但是技术专家必须确保他们覆盖了所有的基础，特别是当涉及到安全性和合规性(有时，还有成本和许可)时。一次又一次，我们看到这四个大项目被推到优先列表的底部，在其中一个障碍上摔倒会造成很大的伤害。如果你不遵守法律，新一代的守法者就有真正的牙齿。一个简单的成本效益分析可以成为你需要的证据，来强调在其中一个领域不提高技能的潜在影响，并给你一个理由将它移回优先列表的顶部。

## 维护限制

数据库专业人员一天中有超过三分之一的时间用于数据库维护。这限制了创新和提升技能等活动的时间和机会，以应对日益增加的复杂性和数据驱动的计划。

*   与其他任务相比，维护工作占用了数据库专业人员更多的时间。三分之一接受调查的技术专家表示，作为数据库/数据/技术专家，他们花费时间最多的领域之一是维护(33%)和数据库开发(22%)。

日常数据库维护耗费了技术专家本可以用于提升技能或创新工作方式的时间。例如，近一半的受访者表示，性能/查询调优是他们认为可以开发/改进以更好地管理不断增加的数据库复杂性的三大技能/工具之一，其次是安全管理和数据工程(分别为 45%和 41%)。让工作变得更智能而非更困难的最佳方式之一是:减少维护停机时间并提高效率，方法是实施适当的自动化和监控工具。如果实施得当，自动化和监控解决方案可以提高整体性能，并最终降低成本。此外，他们可以腾出技术专家的时间，让他们专注于更主动的数据库性能管理。

该报告揭示了当今组织中数据库体系结构和操作技术专业人员管理的变化有多快，以及伴随这种变化的管理复杂性网络。由于应用程序可用性和性能是关键的业务驱动因素，数据库长期以来一直是当今 IT 环境的关键。然而，面对日益多样化的平台和资源减少的逆风，技术专家必须做好准备，在每天都在增加更多数据库的环境中做更多的被动维护。

技术专家必须抓住眼前的机会，最大限度地发挥云托管数据库的性能潜力，并确保成功优化其组织的数据。
