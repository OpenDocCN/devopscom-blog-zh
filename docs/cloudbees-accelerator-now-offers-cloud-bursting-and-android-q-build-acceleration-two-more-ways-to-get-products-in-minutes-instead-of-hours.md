# CLOUDBEES ACCELERATOR 现在提供云爆发和 ANDROID Q 构建加速:这是在几分钟内而不是几小时内获得产品的两种方法

> 原文：<https://devops.com/cloudbees-accelerator-now-offers-cloud-bursting-and-android-q-build-acceleration-two-more-ways-to-get-products-in-minutes-instead-of-hours/>

*cloud bees Accelerator 版本 11.1 中的新增强功能进一步缩短了构建和测试周期时间*

devo PS WORLD | JENKINS WORLD–加利福尼亚州旧金山。**–2019 年 8 月 14 日—**[cloud bees](https://www.cloudbees.com)^(，推动持续经济发展的企业开发运维领导者，今天推出了其 CloudBees Accelerator 平台的新增强功能，将帮助 IT 团队更快地将产品推向市场。团队第一次可以使用行业领先的加速平台将工作负载分散到所有三大云提供商，并将运行新 Android Q 操作系统的设备的构建和测试时间从几小时缩短到几分钟。)

CloudBees Accelerator 11.1 通过将构建分布在一个大型廉价服务器集群上，大大加快了构建、QA 和测试周期。增加对微软 Azure 和谷歌云平台上的云爆发以及 Android Q 设备开发的支持，将有助于组织更快地进入市场，并为实验和迭代创造更多时间。

试图用专用构建云来加速构建和测试周期是低效的。CloudBees Accelerator 以前允许开发人员将工作负载从数据中心或私有云爆发到 AWS 云平台上，使他们能够缩短这些环境中的构建和测试时间。现在，有了对微软 Azure 和谷歌云平台的支持，团队可以按需将工作负载转移到三大云提供商中的任何一家，从而提高效率，消除供应商锁定。

与此同时，Android 设备制造商面临巨大的时间和成本压力，他们必须与 Android Q 同时发布新设备，Android Q 是新的 Android 操作系统，目前处于测试阶段，计划于 2019 年第三季度发布。使用 CloudBees Accelerator，他们可以将构建和测试时间从数小时缩短到数分钟，从而更快地上市，并可能避免因在新 Android 系统交付日期后发布设备而被罚款。

CloudBees Accelerator 11.1 还包括以下新功能:

*   支持 SSL 和 TLS 加密协议-这确保了 EMake 工具(从程序的源文件控制程序的可执行文件和其他非源文件)和代理组件之间的安全通信，使 cloud bees Accelerator public-cloud 就绪。这使得金融服务和其他垂直行业中对安全性敏感的组织能够确保他们的构建不会被黑客攻击或拦截。
*   对 CloudBees Electrify 的改进——这允许用户加速非 EMake 构建工具，如 MSBuild 和 Scons。
*   Linux 基金会 Yocto 项目的改进——yo CTO 项目是一个开源项目，旨在开发工具和流程，为独立于嵌入式硬件底层架构的嵌入式和物联网软件创建 Linux 发行版。CloudBees Accelerator 使开发人员能够使用 [BitBake](https://en.wikipedia.org/wiki/BitBake) 构建工具或 [Buildroot](https://buildroot.org/) 文件和补丁将构建速度至少提高三倍。
*   改进了对 Windows 的现成 CloudBees 加速器支持，提供了对分发和缓存 Visual Studio 项目的本机支持。
