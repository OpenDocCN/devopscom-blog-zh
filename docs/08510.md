# 人工智能和机器学习驱动的 DevOps 如何实现业务转型

> 原文:[https://devo PS . com/how-devo PS-powered-by-ai-and-machine-learning-is-delivering-business-transformation/](https://devops.com/how-devops-powered-by-ai-and-machine-learning-is-delivering-business-transformation/)

人工智能(AI)和机器学习(ML)的使用正在从根本上改变我们对 DevOps 的看法。最值得注意的是，它提供了一种新形式的 DevOps，这种新形式的 devo PS 认识到需要拥有设计智能并由全面安全性(DevSecOps)支撑的系统。对于许多人来说，如果 DevOps 要缩短所有互联智能系统的软件开发生命周期，确保安全高质量软件的持续交付，这将是至关重要的下一步。

到目前为止，大多数组织都明白开发运维是他们必须采用的一项重要原则——根据 Deloitte 的说法，采用开发运维的组织在上市时间方面看到了 18%-21%的减少[](https://www2.deloitte.com/content/dam/Deloitte/global/Documents/About-Deloitte/gx_about_deloitte-agile-devops-advisory-transformation-delivery_Deloitte-Enterprise-Agility-and-DevOps-Brochure.pdf)。通过打破业务和 IT 运营之间的孤岛，DevOps 可以确保一致的生产力、效率和服务交付水平，所有这些在不确定性增加的时代都举足轻重。

简单地说，DevOps 可以帮助企业在已经拥挤的市场中竞争。通过持续集成(CI)和持续交付(CD)的基础，组织可以确保客户在尽可能短的时间内收到他们需要的产品，同时减轻由于系统工程师和运营团队之间缺乏协调而导致的任何长期挫折。将人工智能和人工智能融入 DevOps 战略将使事情更上一层楼。

## **AI 和 ML 是 DevOps 的下一个进化步骤**

今天的公司是数据驱动的，并被构建为数字平台。然而，只有 [28%](https://www.forbes.com/sites/forbesinsights/2019/09/26/the-secret-to-digital-transformation-in-one-chart/#70674f603555) 的组织目前正在他们的数字化转型之旅中取得成功。越来越多的公司正在考虑人工智能和人工智能可以做些什么来帮助他们实现变革的雄心。

人工智能和人工智能市场预计将在未来三到四年内经历巨大的增长。分析公司 IDC 预测，到 2024 年 ，全球人工智能系统的支出将达到a href = " https://www . marketresearchfuture . com/reports/machine-learning-market-2494">42.8%。

人工智能和人工智能为这些公司揭示了什么，以及人工智能、人工智能和设备可以一起做的事情的价值越来越大，正在改变组织在数字化转型过程中看待世界的方式。将 AI 和 ML 与 DevOps 结合将导致 DevOps 参与的内容发生重大转变。

首先，它将开发运维置于组织更广泛的数字化转型目标的前沿和中心。如果数字公司基于实时数据运行，那么这些智能系统的开发将是一个诱人的环境，让 DevOps 能够以前所未有的方式向组织证明其更广泛的价值。

## **安全成为中心舞台**

根据 TechTarget 对关键 IT 决策者的调查，网络安全和风险管理被认为是今年的头号支出领域，超过 [53%的受访者](https://www.computerweekly.com/news/252482336/IT-Priorities-2020-Compliance-and-risk-are-top-security-concerns) 表示他们看到该领域的预算增加。安全性是任何采用 DevOps 策略的组织都应该考虑的重要因素。

DevSecOps 是一个简单的前提，即参与软件开发周期的每个人都对安全性负责。事实上，“安全是每个人的责任”这句话是一句经常被重复的口头禅，但也经常被忽视。将安全性嵌入开发流程的每个部分，确保安全性和合规性监控工具与 DevOps 提供的速度和敏捷性保持同步。众所周知，安全性是快速无缝开发和部署系统的最大障碍，因为安全解决方案传统上并不是以 DevOps 要求的速度来测试和编码的。

最终，从一开始就实现更多的自动化，减少安全架构师的手动配置需求是 DevSecOps 的亮点。这降低了误管理和发生故障的几率，而误管理和故障会导致停机以及潜在的违规或攻击。通常，保护组织安全的最佳方式是通过简单的卫生因素，如定期修补和软件更新。

现实情况是，每一个使用 DevOps 的组织都应该像 DevSecOps 一样对待它。安全性对每一个组织来说都越来越重要，通过为 it 提供人工智能和人工智能流程，安全性也可以得到增强，从而简化数据处理，轻松识别安全构成中的威胁或潜在漏洞。

## **超越人类能力**

未来将由跨越智能边缘的互联智能系统组成，一直到云。这些系统的期望是基于一个典型的移动应用程序的生命周期和不断的功能增强。但是，您如何知道下一步要交付什么，尤其是当事情进展得如此之快，并且开发人员被大量不同来源的信息和数据淹没时？这种数据洪流可能是强大的，但也可能是压倒性的。

至关重要的是，这就是人工智能和人工智能发挥作用的地方。AI 和 ML 将帮助开发人员理解各种数据仓库中的信息。在 DevSecOps 中，一直有一种自动化一切的方法，AI 和 ML 将有助于自动化数据的分析和处理——这是一项现在远远超出人类能力的任务。

将人工智能和人工智能结合起来，将使开发人员能够更好地理解和使用手头的数据。这将使开发人员不仅了解错误或故障的发生，而且了解故障发生前发生的细节——大大降低再次发生故障的可能性。AI 和 ML 还将负责实现跨任何和所有工具和系统从诊断到预测的转换，使开发人员更容易预测、识别和解决故障或错误。

我们将越来越多地看到组织从实时洞察中获益并创造价值。他们将通过部署在活动系统上的 AI 和 ML 框架来实现这一点，以提供基于实时开发、验证和运营数据的优化。这种级别的实时可操作洞察通过改善和缩短平均问题解决时间、减轻对运营的影响并最终增强最终用户体验，增强了 DevSecOps 实现 CI 和 CD 的能力。将人工智能和人工智能纳入 DevSecOps 周期是否会实现自我修复系统的交付，这种系统可以在不需要人工干预的情况下检测和解决问题，还有待观察。然而，对于真正的人工智能，这很有可能实现。

人工智能和人工智能的优势和潜在用例广受好评，尽管我们距离看到它们的全部潜力还有一段距离，但它们目前拥有的能力正在给许多行业带来革命性的变化。事实是，如果 DevSecOps 要继续与快速增长的数字数据经济保持同步，那么它将变得依赖于 AI 和 ML 等先进技术。如果正确使用，数据是强大的，但是有如此多的数据要处理和分析，这可能是一个挑战。

如果许多组织想要实现其数字化转型的雄心，那么向安全自动化流程的迁移是他们需要考虑的渐进步骤。组织不能继续以传统的方式做事，并期望得到新世界所期待的结果。
