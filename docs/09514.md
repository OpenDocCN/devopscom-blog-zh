# Web 应用程序安全性开发人员指南

> 原文：<https://devops.com/developers-guide-to-web-application-security/>

说到安全性，有许多漏洞会让你的网站或 web 应用程序容易受到攻击。在本文中，我们将讨论 15 个常见的 web 应用程序安全漏洞以及如何预防它们。

### 1.加密不足

[加密技术](https://devops.com/writing-security-acceptance-criteria-devops-story/)是一项重要的安全措施，用于保护传输中的数据和静态数据。然而，许多 web 应用程序没有正确使用加密技术，导致了许多严重的漏洞，包括[潜在的破坏性代码窃取。](https://vpnoverview.com/news/hackers-claim-they-breached-microsoft-leak-37gb-of-source-code/)例如，如果数据没有正确加密，则很容易被截取和读取；如果加密密钥没有得到正确保护，则很容易被猜测或窃取。

为了正确保护数据，使用强加密技术非常重要。这包括使用正确的加密算法、加密传输中的数据和静态数据、正确保护加密密钥等等。让所有软件保持最新也很重要，因为新的加密漏洞会不断被发现。

### 2.中断的访问控制

访问控制是一种安全措施，用于控制谁可以访问系统中的哪些数据和功能。它是任何 web 应用程序的重要组成部分，但是经常不能正确实现。这可能导致严重的漏洞，如敏感数据泄露或攻击者获得管理功能的访问权限。

有许多常见错误会导致访问控制中断，例如未能正确限制对数据和功能的访问，使用不安全的方法存储和传输用户凭据，以及未能正确保护会话令牌。为了防止这类漏洞，在您的 web 应用程序中实现适当的访问控制措施非常重要。

### 3.身份验证和会话管理中断

身份验证和会话管理是任何 web 应用程序中最重要的两项安全措施。然而，它们经常没有被正确地实现，导致许多严重的漏洞。例如，会话 ID 很容易被猜测或窃取，cookies 可能被篡改，密码可能被暴力破解。

为了适当地保护用户数据并防止这种类型的漏洞，实现强认证和会话管理机制是很重要的。这包括使用强密码、双因素身份验证、适当的会话过期和失效等等。这也意味着适当地保护应用程序使用的任何会话 id 和 cookies。

### 4.跨站点脚本

跨站点脚本(XSS)是一种漏洞，使得攻击者能够将恶意代码注入网页。这可用于窃取数据、劫持会话、将用户重定向到恶意站点等。XSS 是最常见的 web 应用程序漏洞之一，尤其是在我们这个远程工作的时代。尽管进行了安全意识培训，但当这些风险没有得到适当解决时，许多员工仍然容易受到社会工程和网络钓鱼策略的攻击。

为了防止 XSS 攻击，净化所有用户输入和输出是很重要的。这包括正确地转义特殊字符，使用允许字符的白名单等等。随着新的 XSS 漏洞不断被发现，保持所有软件最新也很重要。

### 5.不安全的直接对象引用

不安全的直接对象引用(IDOR)是一种漏洞，它允许攻击者直接访问他们不应该访问的数据。例如，攻击者可以猜测或暴力破解敏感文件的 URL，如客户的信用卡信息，然后下载它。IDORs 还可以用来绕过安全措施，比如访问控制检查。

为了防止 IDOR 漏洞，正确验证所有用户输入并将数据和功能的访问权限仅限于那些应该有权访问的人是很重要的。随着新的 IDOR 漏洞不断被发现，保持所有软件最新也很重要。

### 6.授权和认证不足

授权和身份验证不足是一种漏洞，它允许攻击者访问他们不应该访问的数据或功能。这可能是由多种因素造成的，例如弱密码、基于角色的访问控制实施不当、故意越权等等。

为了适当地保护数据并防止这类漏洞，实现强大的身份验证和授权机制非常重要。这包括使用强密码、双因素身份验证(2FA)、适当的基于角色的访问控制等等。由于新的漏洞不断被发现，保持所有软件最新也很重要。

### 7.无法限制 URL 访问

另一个常见的 web 应用程序安全漏洞是未能限制 URL 访问。这使得攻击者能够访问他们不应该拥有的敏感数据或功能。

开发人员面临的最常见的问题之一是忘记正确限制对目录和文件的访问。例如，他们可能忘记将 index.html 文件添加到目录中。这种疏忽会给任何访问该完整目录的人对其中所有文件的读写权限。

另一个常见的问题是，开发人员没有适当地限制对某些 URL 参数的访问。例如，他们可能允许任何人访问“id”参数，该参数可用于查看或修改他们无权访问的数据。

为了防止这种类型的漏洞，确保所有目录和文件都受到适当的限制，并且所有 URL 参数在使用前都经过适当的清理，这一点很重要。

### 8.远程文件包含

远程文件包含(RFI)是一种漏洞，攻击者通常通过脚本或其他类型的应用程序在易受攻击的网页上包含远程文件。这可用于将恶意代码注入页面，然后查看该页面的任何人都可以执行这些代码。

RFI 最常见的问题之一是开发人员没有正确地净化用户输入，这使得攻击者能够将他们自己的文件注入到页面中。另一个问题是，开发人员经常使用静态包含路径，这使得攻击者很容易猜测路径并注入他们自己的文件。

为了限制这种类型的漏洞，确保所有用户输入都经过适当的清理，并使用动态包含路径是很重要的。

### 9.记录和监控不足

日志记录和监控是用于检测和响应安全事件的重要安全措施。尽管有这些至关重要的功能，但许多 web 应用程序并没有正确地记录和监控活动，从而导致许多严重的漏洞。例如，如果没有适当的监控，攻击者可以很容易地掩盖他们的踪迹，或者某个事件可能未被检测到。

为了正确检测和响应安全事件，正确记录和监控活动非常重要。这包括记录所有活动、监控可疑活动等。让所有软件保持最新也很重要，因为新的日志记录和监控漏洞会不断被发现。

### 10.安全错误配置

安全错误配置是当 web 应用程序没有正确配置时出现的一种漏洞。这可能会导致许多严重的安全问题，例如暴露敏感数据，使攻击者更容易获得对系统的访问权限，等等。

为了降低这类漏洞带来的风险，正确配置所有软件和系统非常重要。这包括设置强密码，禁用不必要的帐户和服务，正确配置防火墙等等。由于不断发现新的安全错误配置漏洞，因此保持所有软件最新也很重要。

### 11.篡改数据

当攻击者试图未经许可修改数据时，就会发生数据篡改。这最终会导致许多严重的后果，例如数据损坏、数据完整性丧失等等。

为了防止数据被篡改，通过数据处理和存储最佳实践来正确保护数据非常重要。这包括使用正确的身份验证和授权机制、加密传输中的数据和静态数据、正确保护加密密钥等等。由于新的数据篡改漏洞不断被发现，因此保持所有软件最新也很重要。

### 12.跨站点请求伪造(CSRF)

跨站点请求伪造(CSRF)是一种漏洞，使得攻击者能够诱骗用户提交恶意请求。目标是在用户不知情或未经用户同意的情况下，允许用户请求做一些事情，如更改密码、转移资金等。

为了防止 CSRF 攻击，正确验证所有请求非常重要。这包括使用适当的请求验证机制，比如检查有效的 CSRF 令牌、2FA 等等。

### 注意漏洞

我们在日常和商业生活中使用和依赖大量的网络应用。虽然这些应用程序中的大部分都相对安全可靠，但仍然存在一些常见的安全漏洞，使它们容易受到攻击。

为了确保您的 web 应用程序的安全，了解这些漏洞并知道如何预防它们是非常重要的。这包括使所有软件保持最新，使用适当的身份验证和授权机制，加密传输中的数据和静态数据，以及培训用户识别社会工程和网络钓鱼企图等。通过遵循这些最佳实践，您可以帮助确保您的 web 应用程序尽可能安全可靠。
