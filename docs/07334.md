# 新的 RapidAPI 开发者调查强调加速 API 采用

> 原文:[https://devo PS . com/new-rapidapi-developer-survey-highlights-accelerated-API-adoption/](https://devops.com/new-rapidapi-developer-survey-highlights-accelerated-api-adoption/)

第二份年度 RapidAPI 开发者调查报告提供了关于来年 API 前景和新兴技术趋势的重要见解

**旧金山—2021 年 1 月 27 日— [RapidAPI](https://rapidapi.com/) ，世界上最大的 API 市场，帮助开发者找到并连接成千上万的 API，今天发布了一份衡量开发者趋势和 API 使用情况的新报告。第二份年度 [RapidAPI 开发者调查报告](https://rapidapi.com/developer-survey/)调查了来自 250 个国家、在软件开发中担任不同角色的个人，其中大多数人被认为是专业开发者、管理者或领导者。**

 ****该调查的一个关键观点是，在全球疫情中，开发人员对 API 的依赖在过去一年中有所增加，并将在 2021 年继续增加。来自各行各业的各种规模的组织计划今年加入 API 经济，API 测试和安全性是调查受访者最关心的问题。

**“在疫情期间，许多公司迅速加快了数字化转型之旅。因此，他们在软件开发方面的投资，特别是在 API 经济方面的投资持续增长是有道理的，”RapidAPI 创始人兼首席执行官 Iddo Gino 说道。“RapidAPI 让公司更容易利用 API 经济，将他们的开发人员与数以千计的 API 和微服务联系起来，帮助他们更有效地测试和管理 API。”**

****关键数据洞察****

******除其他见解外，该报告还揭示了:******

******开发者对 API 的依赖持续增加:开发者对 API 的依赖在疫情期间加速，并将在 2021 年继续增加；61%的开发者在 2020 年使用的 API 比 2019 年更多，71%的开发者计划今年使用更多。******

******参与 API 经济是各行业的重中之重:总体而言，58%的高管表示，参与 API 经济是他们组织的重中之重。在某些行业，这一数字甚至更高:89%的电信高管将利用快速医疗互操作性资源(FHIR)等 API 和标准，开放 API 计划正在推动大多数医疗(75%)和金融服务(62%)高管的采用。******

******新 API 技术的采用正在增加:今年的调查还表明，API 技术和使用正变得更加多样化。REST 仍然是被调查者中最受欢迎的标准，他们对新兴技术如无服务器、FaaS、WebSockets 和 gRPC 有着浓厚的兴趣。AsyncAPI 的生产使用量翻了三倍(从 2019 年的 5%增长到 2020 年的 19%)，GraphQL 翻了一倍(从 2019 年的 6%增长到 2020 年的 12%)。******

******API 测试和安全性是重中之重:测试和安全性是几乎所有开发人员的重中之重——95%的人将安全性列为首要考虑因素，92%的人计划在 2021 年测试 API。超过 43%的开发人员将使用工具来编写或管理测试，15%的开发人员计划在未来使用工具。另外 38%的人将在他们的代码中编写测试。******

******超过一半的专业开发人员每周花 10 多个小时在 API 上:组织为 API 开发投入大量资源，包括时间——58%的专业软件开发人员每周至少花 10 个小时在 API 上。近 30%的人说他们每周花 20 多个小时使用 API。******

******JavaScript 继续成为最受欢迎的编程语言:尽管 API 技术发生了变化，但与 2019 年相比，2020 年最受欢迎的五大编程语言保持不变，JavaScript 在最受欢迎的名单中领先。Python、Java、PHP、C#紧随其后。******
