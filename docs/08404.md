# 如何在基础设施上应用 GitOps

> 原文:[https://devops.com/how-to-apply-gitops-on-infrastructure/](https://devops.com/how-to-apply-gitops-on-infrastructure/)

[GitOps](https://www.mediaopsevents.com/gitopsevolution) 已经不仅仅是 DevOps 社区中的一个流行语了。它由一组最佳实践组成，旨在更好地部署、管理和监控 Kubernetes 集群和应用程序。通过采用 GitOps，一切都变成了代码，因此授权 DevOps 团队从头到尾控制开发过程的每个方面。

Upbound 的开发者倡导者 Viktor Farcic 是谷歌开发者专家的成员，也是一名出版作家，他甚至在 2017 年 Alexis Richardson 创造这个术语之前就已经在做 GitOps 了。Viktor 认为，尽管许多开发人员已经开发 GitOps 有一段时间了，但是定义 GitOps 的真正含义并给它起一个名字是帮助它迅速被采用的原因。

Viktor 是出席 [GitOps Evolution 虚拟峰会](https://www.mediaopsevents.com/gitopsevolution)的杰出演讲者之一。在会议期间，Viktor 将讨论 GitOps，并探索与 Flux 相结合的 Crossplane，作为满足所有基础设施需求的潜在解决方案，并看看它是否适合更广泛的 Kubernetes 生态系统。

Crossplane 是一个开源项目，使工程师能够使用 Kubernetes 来提供和管理基础设施、服务和应用。与 Flux 相结合，它允许我们应用 GitOps 原理来持续监控基础设施，并修复当前状态与期望状态之间的任何偏差。
