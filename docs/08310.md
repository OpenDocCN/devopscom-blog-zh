# sophos 收购 refactor 以推进 devsecops

> 原文:[https://devo PS . com/sophos-acquisites-ref actr-to-advance-devsecops/](https://devops.com/sophos-acquires-refactr-to-advance-devsecops/)

Sophos 本周透露，它已经[收购了 Refactr](https://www.sophos.com/en-us/press-office/press-releases/2021/08/sophos-acquires-refactr.aspx) ，这是一个自动化平台的提供商，可以更简单地向 DevOps 管道添加静态和动态安全扫描和应用测试。交易条款没有披露。

Sophos 首席技术官 Joe Levy 表示，Sophos 将扩展 Refactr DevSecOps 自动化平台，以添加安全编排自动化和响应(SOAR)功能。这将为 DevOps 团队提供对其受管威胁响应(MTR)和扩展检测和响应(XDR)平台的访问。Levy 补充说，一旦这些集成在 2022 年初实现，目标是让 DevOps 和网络安全团队的合作变得更简单。Levy 指出，Sophos 还将继续开发 Refactr 平台，作为一个独立的产品，此外还将维护该平台当前的社区版本。

![Refactr Sophos](../Images/df987f284bedeebc06256d46bcb0a8d8.png)

Sophos 已经收购了一系列网络安全公司，因为它重新构建了其核心平台，以创建一个可扩展的安全框架，称为[适应性网络安全生态系统](https://news.sophos.com/en-us/2021/05/05/introducing-the-sophos-adaptive-cybersecurity-ecosystem/)。该平台构建在基于现代微服务的架构之上，简化了收购整合。Levy 指出，总而言之，这些收购使 Sophos 能够收集更广泛的数据，这些数据可用于训练人工智能(AI)模型，并不断更好地自动化 DevSecOps 流程。

Sophos 最近的收购包括 Capsule8 和 BrainTrace，前者是一家提供安全容器平台的公司，后者是一家提供安全管理服务的公司。Sophos 还在 2019 年收购了另一家托管安全服务提供商 Rook Security。

总的来说，Sophos 正在为开发人员和网络安全团队都可以导航的安全平台做准备。除了通过应用编程接口(API)与持续交付和持续部署( [CI/CD](https://devops.com/?s=continuous+integration) )工作流集成之外，还可以通过可视化工具访问 Refactr 平台，这些工具可用于使用拖放工具创建启动测试。

在一系列引人注目的软件供应链违规事件之后，现在人们更加关注实现 DevSecOps 最佳实践。组织面临的挑战是找到一种方法来实现这一目标，而不会过度降低应用程序的构建和部署速度。实现这个目标需要找到更多的自动化方法，例如，在构建过程中扫描代码。

然而，与此同时，最近的一份 Sophos 主动对手剧本 2021 说明了组织如何发现自己被网络犯罪对手超越。该报告详细描述了 37 个不同的攻击组织如何使用 400 多种不同的工具来发动网络攻击。在发现袭击之前，袭击者的平均停留时间为 11 天。

很明显，没有自动化和人工智能的进步，保护软件供应链的可能性微乎其微。组织面临的挑战是找到一种经济实惠的方式来利用人工智能和自动化技术及专业知识。例如，单个组织并不真正有能力构建能够自动化网络安全流程的人工智能模型。大多数组织最终会使用 API 来调用在 DevSecOps 工作流的上下文中通过云交付的高级安全服务。问题是确定哪家服务提供商能以最低的成本提供最完整、最强大的功能。
