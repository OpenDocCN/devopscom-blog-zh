# 开发人员用来减少漏洞的第一招

> 原文：<https://devops.com/the-1-trick-devops-shops-use-to-reduce-vulnerabilities/>

本周发布的一份新报告强调了将漏洞评估信息纳入软件开发生命周期的重要性。由 WhiteHat Security 发布的 2015 年网站统计报告提供了关于 web 应用程序安全的有趣数据。然而，在所有的统计数据中，DevOps 商店也许应该专注于一个主要的收获——统计证据表明，从长远来看，一个单一的实践可以在减少漏洞方面产生巨大的影响。有问题的魔术？将漏洞数据绑定到缺陷跟踪系统中。

“总而言之，应用程序安全性有三个重要指标。WhiteHat 的创始人 Jeremiah Grossman 说:“关键是你有多少个漏洞，你修复它们的速度有多快，以及修复的百分比有多高。“我们发现，当你将漏洞评估数据输入设备时，无论你将这些数据输入到漏洞跟踪系统中，这对 DevOps 都非常重要，这实际上是实现最佳结果的途径。”

正如他所解释的那样，简单地以原始格式提供漏洞评估数据——例如，列出缺陷的 PDF 文档——这对于自动化管道来说毫无用处，而且漏洞仍将得不到修复。在本周发布的报告中，Grossman 报告说，与其他组织相比，将漏洞馈送和开发流程的错误跟踪系统联系起来的组织表现出的漏洞减少了 40%，平均每月修复问题的速度加快了 15%。

虽然 Grossman 承认很难确切知道有多少百分比的客户归因于不同的开发方法，但他粗略的猜测是，大约有一半的客户正在制定某种 DevOps 原则，以便至少每两到三周发布一次。他建议希望更多地参与推动 DevOps 转型的合作精神的安全专业人员应该从指标开始。

“创建一个度量程序，就像 DevOps 的人所做的那样，来测量那个环境的安全性，”他说，并解释说上面提到的三个度量是很好的开始。“然后，您可以跟踪开发运维组在安全性方面的表现，并将数据反馈给该组。然后，当您想要战略性地了解 DevOps 环境中需要工作的流程和改进时，您可以非常专注于激光。App sec 计划应围绕他们希望改进的指标，根据组织的内部流程量身定制。”
