# 当你是一名开发商时，如何应对 CostOps

> 原文:[https://devops.com/tackling-costops-when-youre-a-developer/](https://devops.com/tackling-costops-when-youre-a-developer/)

IT 计划的实施和成功执行通常取决于开发人员。从商业供应商到开源项目、基于云的软件即服务【SaaS】解决方案 、托管服务提供商(MSP)和内部团队，他们都在 IT 和运营团队的成功运营中扮演着重要角色。

开发者和运营团队面临的一个挑战 是 开发和后续运营之间巨大的时间障碍。一个开发人员今天为一个组织构建的东西可能会在几十年后继续使用，并且可能会被其他开发人员维护、修补、更新和修改。虽然这是开发人员的一个骄傲点——他们可能会在职业生涯中继续前进——但对于运营团队来说这是一个挑战，他们不得不年复一年地继续部署它。

随着越来越多的应用出现，开发人员的传统角色正在发生变化。今天，开发人员需要开始将运营成本控制纳入他们的职责。

## 开发商的机会

开发人员很少负责长期的商业决策。一旦代码脱离了他们的控制，他们通常很少或者根本没有对他们的工作进行管理。企业将做出关键的 [IT 成本优化](https://www.solarwinds.com/solutions/it-cost-optimization-solutions) 以及以后对其代码、语言和工具链的遏制决策。开发者关注现在和不远的将来。几年后，花几个小时去设想一万种可能性中的一种获胜的结果，不会得到代码或者实现他们的经理的交付目标。

然而，开发人员有一个经常被忽视的机会，那就是在初始开发阶段加入一些工具来激活成本控制支持。从那里，如果他们的工作是好的，它会持续下去；最终将需要更多的支持。第一步是确保新的现代化应用程序从一开始就包含可靠的云原生监控。这不仅有助于开发运维团队和 SRE 团队，也有利于长期拥有这些应用的整个企业。

集成[以运营为中心的监控](https://devops.com/arctic-wolf-revolutionizes-security-awareness-and-training-with-new-managed-security-awareness-solution/)是开发人员的头等大事。在某些情况下，使用平台管理的无服务器或编排的容器化的应用程序有很多机会为开发人员和操作人员提供详细的性能数据。

开发人员在依赖性分析、代码成熟度和易于集成方面的专业知识是评估工具耐用性的一个很好的测量工具。一个能够轻松将 云原生应用性能 细节与基础设施性能、 [用户体验](https://www.solarwinds.com/solutions/user-experience-monitoring) 和运营费用数据相结合的解决方案，更有可能帮助开发人员和企业管理长期成本。

### 新维度

随着云原生成为主流企业战略，成本控制是一个关键机会。为了迎接 [应用程序管理](https://www.solarwinds.com/solutions/application-performance-management-solutions) 的未来挑战，领导层正在快速扩展他们对应用程序适用性和质量的定义，以纳入应用程序所有权的所有方面。对于一些团队来说，成本控制是高级开发人员成熟度的一个新维度。
