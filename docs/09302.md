# Digital.ai 扩展了 DevOps 平台的范围和影响力

> 原文:[https://devo PS . com/digital-ai-extends-scope-and-reach-of-devo PS-platform/](https://devops.com/digital-ai-extends-scope-and-reach-of-devops-platform/)

Digital.ai 今天发布了与其同名的 DevOps 平台的 Ascension 版本，该平台使用测试透镜功能添加了预构建的指标和最佳实践仪表板，以更轻松地利用分析来识别软件交付瓶颈。

此外，该公司还增加了自动测试工具，可以自动为移动应用程序生成性能和安全性测试。这些工具可以由 DevOps 团队、开发人员或业务用户运行。增强的变更风险预测能力使机器学习算法能够更广泛地利用历史数据。

最后，Digital.ai 还增加了与开源 Argo 持续集成/持续交付(CI/CD)平台的集成，并提供了一个 Agility Sync 工具来构建与传统应用环境的定制集成，并为集成开放了一个在线市场。

Digital.ai 首席营销官 Jeff Moloughney 表示，目标是扩大该公司 DevOps 平台的覆盖范围。该平台的核心是一个价值流管理(VSM)和版本控制软件，最初是由 XebiaLabs 和 CollabNet VersionOne 在两家公司合并创建 Digital.ai 之前开发的。合并后的公司随后收购了 Numerify(一家提供人工智能(ai)分析工具的公司)、Experitest(一套测试工具)和 Arxan(一家提供应用安全平台的公司)。该公司声称其平台被超过 1500 个组织使用。

Moloughney 指出，总的来说，DevOps 正在进入一个新时代，因为组织意识到他们是多么依赖软件来驱动业务流程。从历史上看，在大多数组织中，DevOps 实践的采用是自下而上推动的。他补充说，现在，业务和 IT 领导人也在寻求推动采用一套一致的 DevOps 最佳实践，以获得对软件交付的更多可见性。

然而，这并不意味着组织可以规定单个开发人员可以使用什么工具，所以所采用的 DevOps 平台必须在可用集成方面是可扩展的，Moloughney 说。

与此同时，Molougney 补充说，更多的 DevOps 流程将使用机器学习算法实现自动化。Digital.ai 正在将这些算法嵌入其核心平台，以发现软件交付瓶颈并自动化测试等日常任务。

尚不清楚已经采用 DevOps 的组织可能会在多大程度上考虑替换其现有平台，但无论规模大小，大多数组织都遇到了问题。另一方面，随着更多数字业务转型计划的推出，越来越多的组织正在更广泛地采用 DevOps 实践。随着这些实践的发展，预计许多组织将采用 VSM 形式的分析来更好地优化他们的工作。

与此同时，不同企业采用 DevOps 的比率将继续存在很大差异。组织中的每个开发团队都倾向于基于他们自己的需求定义一组 DevOps 实践。考虑到这些项目之间存在的所有依赖关系，挑战在于确定组织在每个孤立的计划进行过程中的实际进度。
