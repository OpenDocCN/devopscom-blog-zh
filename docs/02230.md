# 解开数据宣布合作伙伴计划和新的管理人员招聘，以满足云迁移和数据运营计划的需求

> 原文：<https://devops.com/unravel-data-announces-partner-program-and-new-executive-hire-to-meet-demand-for-cloud-migration-and-data-operations-initiatives/>

*nravel 合作伙伴生态系统包含一个庞大的创新者网络，帮助企业优化其数据系统和面向未来的云迁移战略*

**加利福尼亚州 PALO 阿尔托市——2019 年 6 月 25 日**—[unallow Data](https://unraveldata.com/)是唯一一个提供全栈可见性和人工智能建议以推动现代数据应用程序更可靠性能的数据运营平台，今天宣布了 unallow 合作伙伴计划，该计划汇集了世界领先的技术创新者，支持 unallow 的使命，即从根本上简化企业优化其现代数据应用程序性能的方式以及为这些应用程序提供动力的复杂管道。新计划首次任命 Mark Wolfram 为业务发展和合作伙伴副总裁，他将指导该计划的战略和扩展。

unallow 的合作伙伴计划生态系统包括一个强大的品牌列表，包括 Informatica、Attunix、戴尔和 RCG 全球服务，以帮助他们的客户解决数据运营挑战并优化他们的云采用计划:

*   **技术和平台合作伙伴:**

unallow 与领先的云平台、数据管理、数据运营和分析合作伙伴的合作关系保证了数据系统的性能、可靠性和成本。客户受益于可操作化的数据应用程序工作负载，以及在生产过程中解决性能和可靠性挑战的能力。找出优化机会，以确保满足服务级别协议并优化资源。随着数据工作负载迁移到云和混合部署模式，unlaunch 可以加快决策过程，同时消除风险。

*   **解决方案合作伙伴:**

unallow 与集成商和解决方案提供商的合作伙伴关系为他们的客户提供经验证的数据运营解决方案，以保证他们的本地或云中数据系统的性能、可靠性和成本。客户受益于获得高度差异化的 unfraud 技术，以解决各种数据操作和云迁移挑战。unallow 的培训和支持加快了实现价值的时间，因为它可以直接和早期接触产品、工程和上市团队，以加速交易和在解决方案设计中提供协作支持。

RCG 全球服务公司的全球管理负责人 Rick Skriletz 表示:“我们与 underlying 的共同目标是建立创新文化，帮助客户从他们的所有数据中受益，而不仅仅是他们有现成解决方案或工具的数据。“虽然这不是普遍真理，但我们通常会发现，可以用来解决问题的数据越多，结果就越好。我们的互补解决方案集以及它们之间的紧密集成旨在为大数据工作流带来自动化，这是我们最常从客户那里听到的需求。

*“作为数字化转型之旅的一部分，将您的数据迁移和优化到云是一项艰巨的任务，但我们看到从能源到金融服务、制造、政府和零售，我们的客户越来越多地要求这样做，”Redapt Attunix 的云服务总裁马特·安东内尔说。“需求和复杂性的推动使我们走上了与 unfralle 合作的道路，为我们的客户提供他们信心十足地迁移所需的复杂工具、支持、愿景和平台专业知识，并从那里监控、管理和改进他们的数据管道，以在推动其业务的应用中实现更可靠的性能。对于寻求这种协同作用来为客户提供价值的客户来说，unlauncher 与领先平台的兼容性和协作是一项重大优势。”*

*unallow 的生态系统合作伙伴受益于支持、上市和售后支持。合作伙伴可以获得独家技术和最佳实践以及销售和产品培训。合作伙伴可以利用各种机会，通过联合营销活动、联合营销宣传资料以及注册合作伙伴销售线索和交易注册来帮助发展他们的业务。通过该计划，合作伙伴可以获得用于客户账户管理、工作流集成、全球专用技术支持的工具和 API，并在 unfralle 社区中获得官方合作伙伴的认可。*

*Mark Wolfram 是一位在销售、业务开发和管理领域拥有 20 多年经验的技术高管，他领导着“解开合作伙伴”计划。Wolfram 在微软和 Azuqua(被 Okta 收购)积累了丰富的云计算和数据分析经验。*

*“unlaunch Data 是一家独特的公司，它在如何解决企业寻求将数据工作负载迁移到云时固有的巨大挑战方面不仅仅是口头说说。它实际上正在采取必要的重要步骤，以便在这个复杂的迁移过程中真正与企业合作，并确保现代数据应用程序的持续可用性，”unlable Data 业务发展和合作伙伴关系副总裁 Mark Wolfram 说。“unallow 开发了业内唯一的人工智能平台，用于在云端规划、迁移和管理现代数据应用。我热情地支持 Unravel 的方法，并期待在我们共同发展 Unravel 合作伙伴计划以服务全球更多企业的过程中分享我的专业知识。”*
