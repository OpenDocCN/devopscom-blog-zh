艾伦·希梅尔:大家好。艾伦·希梅尔，DevOps.com 又来了一次 DevOps 聊天。今天的特邀嘉宾是 IBM 的雷切尔·赖尼茨。瑞秋，欢迎你。

谢谢你，艾伦。和你谈话总是很愉快。

*Alan Shimel:* 是的，实际上我想我们最后一次交谈是在旧金山的 [DevOps 企业峰会](https://www.youtube.com/watch?v=QnwWBTkFj3g)。Rachel，对于那些可能不熟悉或没有机会观看我们在 YouTube 上的 DevOps 企业峰会视频的听众，你为什么不告诉他们一点关于你自己的情况。

Rachel Reinitz: 我是 IBM 的一名杰出工程师，我是 IBM Bluemix Garages 的联合创始人和技术负责人。

*艾伦·希梅尔:*优秀。Rachel，您知道，IBM Bluemix 车库发布了许多公告、新闻和发展，这不是一语双关，我们的一些观众可能不太熟悉。所以给我们一些亮点。首先，IBM Bluemix 车库是什么？它发生了什么？

Rachel Reinitz: 所以我们是 IBM 的顾问。我们与客户合作，让他们成功采用 Bluemix。我们是独一无二的，因为我们住在创业社区。我们与初创客户和企业客户都有合作。实际上，我们所做的是将一个想法、一个问题或一个概念应用到 IBM 的设计思维、精益创业、极限编程、DevOps 中，从这个想法到最小可行产品的定义，然后构建它，并在那里与用户一起测试它，测试业务假设，所有真正好的 DevOps 实践，并利用 Bluemix，因为我们所处的环境中的各种服务，Bluemix 也给了我们很多加速。这就是我们的工作。这是一个令人兴奋的工作场所，我们有——客户可以与设计师、建筑师和开发商接触。我们差不多就是这么做的。

*艾伦·希梅尔:*明白了。Rachel，我有幸参观了 IBM Bluemix 车库。我想大概是在旧金山的 DevOps 企业峰会上。我能解释的最好的方式是它有点像一个实验室，有点像一个活生生的实验室，在那里进行着实验、教育和发展。这似乎是一个非常酷的环境。对于你提到的企业来说，从企业到创业公司都是如此。

雷切尔·赖尼茨:没错。我认为这是一个很好的总结。我们在创业社区，所以我们和那里的创业公司一起工作。这真的有助于企业像创业公司一样创新，从他们的环境进入我们的环境，一个非常合作的空间，正如你所看到的，当你访问最难的核心社区时，我们实际上，在一些环境新闻中，我们已经搬到了旧金山镀锌社区内的一个不同的空间，一个更大的空间，这真的很令人兴奋。我们刚刚在那里开了个小派对。我们还有其他地点。我们在多伦多和伦敦，在尼斯有一个办公地点。每个人都在不同类型的创业社区中。我们正在墨尔本工作。所以你得来调查一下我们的其他地点。

艾伦·希梅尔:当然。我们实际上正在进行一系列路演，其中一场将于 4 月在伦敦举行。我们很乐意邀请一些 IBM Bluemix Garage 的人参加。但我们会离线谈。Rachel，我邀请你参加这个月的 DevOps.com 会议的原因之一是，我们真正关注的是我们所说的开放技术和开放企业。我们指的不仅仅是开源软件，尽管开源软件肯定是其中的一部分。但是除了软件之外，还有更多的东西需要打开，不是吗？

是的，我同意。我认为知识共享是开放的一个重要组成部分，通过不同的媒介分享不同种类的知识。我认为你要去的地方是这样一个事实，即我们现在已经把我们在车库中所做的事情与我们在 IBM 内部的开源和 DevOps 转换中所做的事情合并在一起，我们把它作为 IBM Bluemix Garage 方法公开发布，供人们使用。我们也在寻求捐助。

艾伦·希梅尔:当然。它是开着的，对吗？如果你愿意，整个方法论是开放的。我碰巧知道，因为我以前跟你们说过，你们使用的很多工具都是开源工具，传统的开源软件。但实际上，当我们谈论开放式车库方法时，正如我听到的那样，它是完全开放的，如果你愿意的话，对吗？这确实是一项开放的技术。

雷切尔·赖尼茨:是的。因此，它吸取了我们一直在学习的经验教训，提供了规范的途径和建议，并从我们的软件跨越到我们在开放社区和开源、开源测试工具、第三方工具中正在做的事情。这是我们在正在做的项目中发现的工作，以及围绕特定类型的应用程序(如移动应用程序)的建议。我们推荐什么？我们学到了什么，并且像你公开说的那样分享了这些知识？例如，你如何着手测试它们呢？你有什么不同的选择？诸如此类的事情。

艾伦·希梅尔:太棒了。Rachel，Bluemix 在这个车库方法中有多大作用？

Rachel Reinitz: 就我们认为使用 Bluemix 和其他 IBM 技术是真正加速构建这类应用程序的一种方式而言，这无疑是很重要的一部分。但这并不是唯一的。因此，当我们谈论创新实践，当我们谈论我们的一些工具链时，不，它可以与其他技术一起使用。显然，我认为有很多很好的理由将它与 Bluemix 一起使用。

艾伦·希梅尔:当然，当然。再说一次，你知道吗，瑞秋？我们可能有些人甚至不熟悉 Bluemix，为什么我们不快速给他们一个水平集。当我们说 Bluemix 时，我们在说什么？

Rachel Reinitz: 所以 Bluemix 是 IBM 的平台即服务。它公开存在，然后托管在我们的数据中心，我们的软件公共数据中心。我们还有一个专用版本，在该版本中，我们提供同一功能的一个托管实例。还有我们称之为本地的，它可以部署到防火墙后面的您自己的系统上。而且 Bluemix 有多种模式可以用 Cloud Foundry 容器和 VM 编写应用。它有一整套服务。最近一次统计，我想我们大约有 140 个，这些是 IBM 服务，比如我们的 SQL DB，它是 DB2 的一种形式。但也是像 New Relic 这样的第三方服务。正如我所说，它运行 Cloud Foundry，它运行开源运行时。你可以带你自己的运行时间。所以它有很大的开放性。

*艾伦·希梅尔:*优秀，优秀。Rachel，你提到 IBM 正在寻求与第三方合作伙伴围绕这种开放方法、开源进行合作。什么样的合作伙伴，什么样的贡献是你认为有意义的？

*Rachel Reinitz:* 因此，我们有了一个开放工具链的概念，不同的技术可以集成在一起，再次围绕解决像移动这样的特定类型的问题。我们的工具链团队与不同的合作伙伴合作。因此，我们对合作伙伴非常开放，他们会说，嘿，我们能成为工具链的一部分，并讨论如何集成吗？例如，Slack 是我们在那里讨论的一部分。这是一种整合。我们的另一个集成级别是 Bluemix 本身。因此，我们有第三方服务，我们有一个完整的计划，让提供开发者感兴趣的功能的公司能够集成到 Bluemix 中，成为我们生态系统的一部分。

*艾伦·希梅尔:*明白了，明白了。瑞秋，信不信由你，我们马上就要——我答应过你大概 10 分钟。我们将运行一点，但我想快速旋转，如果你愿意的话。除了本月在 DevOps.com 的开放技术，我们还特别报道了即将在拉斯维加斯举行的 IBM 互联大会。我会想象，对不起，我没有预先检查；但是我想 IBM 的 Bluemix Garage 将会在 Interconnect 上占有突出的位置，对吗？

蕾切尔·赖尼茨:是的，是的。我对互联技术非常感兴趣。据我所知，在世博园区，我们实际上设置了一个迷你车库，甚至还有一辆摩托车。我们主要带您了解我们的创新之路，我们就坐在我们 DevOps 团队的隔壁，他们也支持这种方法，所以您可以来看看我们在为您的创新流程服务方面做了什么，我们当然会展示代码和交付管道。如果您想更深入地了解不同的选择，我们附近就有 DevOps 展位。我们有客户来谈论他们用 Bluemix Garage 方法所做的工作。我们有研讨会，在那里你可以参加圆桌会议并真正投入进去。因此，在 Interconnect 上围绕这个方法有很多事情要做。当然，我也要展示。

*艾伦·希梅尔:*真的吗？你在展示什么，瑞秋？

Rachel Reinitz: 我正在介绍车库和方法。我会发言，我想是周一下午 6 点，然后有各种不同的会议。最令人兴奋的是，我有一些客户和我们一起出席。

*艾伦·希梅尔:*真的吗？如果有人真的参加了 Interconnect，他们很可能会参加，您能告诉我们——我不想让您为难，但您能告诉我们可能的客户以及客户在哪些会议上发言吗？

雷切尔·赖尼茨:我必须查找具体的会议，但我们有 KS2 和 ManualLife，这是我曾经密切合作过的客户。这是几个–哦，我们还有 Spigot，它有一个非常有趣的云采购平台。这些是将要发言的一些客户。可能还有其他几个已经准备好被讨论了。

艾伦·希梅尔:太棒了。当然，在展览大厅，人们可以亲自动手操作车库，看看它是怎么回事。

雷切尔·赖尼茨:当然。

*Alan Shimel:* 再次提醒一下，如果有人在听，我想互连是在 2 月 21 日至 25 日。在拉斯维加斯的曼德勒湾和米高梅大酒店。瑞秋，最后一个问题。下一步是什么？

雷切尔·赖尼茨:更多。更多服务。Bluemix 将提供更多服务。我们刚刚有了一些-有新的分析，有新的沃森服务。未来会有更多的项目，更多的客户项目，很酷的物联网类项目，未来会有更多的车库来建立我们现有的车库。很多事情正在发生。在方法上也是如此。方法中更多的工具，更多的规定路径。
