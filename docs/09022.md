# AWS BugBust 挑战开发者在 re:Invent 2021 上创下 Bug 突破世界纪录

> 原文:[https://devo PS . com/AWS-bugbust-challenges-developers-to-set-world-record-for-bug-busting-at-reinvent-2021/](https://devops.com/aws-bugbust-challenges-developers-to-set-world-record-for-bug-busting-at-reinvent-2021/)

软件 bug 既烦人又耗时，对任何公司来说都是最昂贵的软件开发成本之一。 [*软件测试新闻*](https://www.softwaretestingnews.co.uk/the-real-cost-of-software-bugs/) 引用一项研究发现，仅在美国，软件质量差的成本估计就高达 2.8 万亿美元。让它深入人心。*2.8 万亿美元。*其中 18.22%为技术债务。将这一事实与 Watts Humphrey 在他的书 *【软件工程的一门学科】* 中估计的那样结合起来，软件开发人员平均每千行代码 会犯 [100 到 150 个错误。](https://www.amazon.co.uk/Discipline-Software-Engineering-SEI/dp/0201546108)

可以说软件质量差的代价是巨大的。

### 欢迎来到第一届年度 AWS BugBust re:Invent 挑战赛

AWS 承诺通过他们的全球 [AWS BugBust Challenge](https://devops.com/aws-bugbust/) 来帮助世界各地的软件团队提高软件质量。第一届年度[AWS BugBust re:Invent Challenge](https://bugbust.aws/)即将到来。

挑战赛将于 2021 年 11 月 29 日上午 10 点(太平洋标准时间)至 2021 年 12 月 2 日下午 2 点(太平洋标准时间)举行，招募 Java 和 Python 开发人员使用亚马逊 CodeGuru 集体修复漏洞并消除技术债务。在这段时间内，全球的注册参与者(虚拟参与)和那些在 AWS re:Invent Expo 的 BugBust Hub 现场的人可以竞争修复尽可能多的 bug，以赢得奖品、荣誉并获得最终 AWS BugBuster 的称号。对于您修复的每个 bug，您会根据 bug 的复杂程度获得相应的分数。你修复的 bug 越多，你获得的积分就越多，你在 AWS BugBust 排行榜上的排名就越高。实时排行榜将跟踪每个参与者的进度，并显示修复的漏洞数量和获得的积分。

### 成为创造破虫世界纪录的一份子

在 re:Invent 上，AWS 还希望创造一项最大的代码修复挑战的吉尼斯世界纪录[](https://www.guinnessworldrecords.com/)。迄今为止，还没有这类事件的现有记录。修复至少一个 bug，才有资格成为创造吉尼斯纪录的一员。您可以现场或在线加入。在赢得奖品和获得认可的同时，成为试图创造世界纪录的兴奋的一部分！

### 里程碑奖

虽然开发人员将直接赢得最终 AWS BugBuster 的称号，但他们也将有机会为其里程碑式的成就赢得奖励。 **大奖将是 1500 美元现金。**

开发人员将朝着五个重要的里程碑前进:

*   第一个里程碑—1 分—AWS BugBuster 苍蝇拍
*   第二个里程碑—50 分—AWS BugBust 品牌洗手液(BugSpray)
*   第三个里程碑—150 分—AWS BugBust 夜光错误罐
*   第四个里程碑—300 分—AWS BugBust 连帽衫
*   第五个里程碑—500 分—AWS BugBust 品牌 Echo Dot
*   大奖——顶级 BugBuster 的 1500 美元奖金

除了里程碑奖之外，所有为创造世界上最大的代码修复挑战的记录做出贡献的开发人员都将获得一份独家 AWS 证书。本次现场活动中修复的错误将计入全球总数，AWS BugBusting 团队将在活动中占据重要位置。

### AWS BugBust Hub

BugBust 现场挑战赛将在 BugBust 中心举行。该中心将为开发人员提供一个有趣和轻松的环境，以便他们在 BugBust Live 挑战赛开始时修复 bug 并向 AWS BugBust SWAT 团队提问。不在 AWS 现场 re:Invent？不用担心——开发人员仍然可以参与并赢得虚拟奖品！

欲了解更多信息并预先注册 AWS BugBust re:Invent Challenge，请访问 [AWS BugBust 网站](https://bugbust.aws/) 。

被称为“软件质量之父”的 Watts S. Humphrey 是许多关于软件开发过程和软件过程改进的有影响力的书籍的作者。Humphrey 是卡内基梅隆大学软件工程研究所(SEI)的研究员，他在那里创建了软件过程计划，并为最初的能力成熟度模型(CMM)提供了愿景和早期领导。他也是个人软件过程(PSP)和团队软件过程(TSP)的创造者。他被授予国家技术奖章，这是美国总统授予美国主要创新者的最高荣誉。
