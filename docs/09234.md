# WhiteSource 提供免费的 Spring4Shell 漏洞工具

> 原文:[https://devo PS . com/whitesource-offers-free-spring 4 shell-vulnerability-tool/](https://devops.com/whitesource-offers-free-spring4shell-vulnerability-tool/)

WhiteSource 推出了一个免费的命令行界面(CLI)工具，该工具可以检测易受攻击的开源 Spring4Shell 漏洞(CVE-2022-22965)，这些漏洞会影响使用 Spring 开发框架构建的 Java 应用程序。

WhiteSource 的产品管理总监 Susan St. Clair 说,[white source spring 4 shell Detect](https://www.whitesourcesoftware.com/product-info/news/whitesource-launches-free-developer-tool-to-detect-and-remediate-spring4shell-vulnerability/)工具类似于公司[今年早些时候推出的用于检测 Java 应用程序中 Log4Shell 漏洞](https://securityboulevard.com/2022/01/whitesource-automates-remediation-of-log4j-vulnerabilities/)的工具。它提供了直接和间接依赖的精确路径，以及如何找到 Spring 框架的固定版本来加速补救工作。

Spring4Shell 是另一个零日漏洞，其严重性得分为 9.8。因此，DevOps 和网络安全团队现在正竞相更新受此漏洞影响的应用程序，以防网络犯罪分子利用它。Spring4Shell 漏洞支持远程代码执行(RCE)，受影响的 Java 应用程序使用 Spring framework 的 5.3.17 或 5.2.19 版本以及 Java 开发工具包的 9 或更高版本(JDK)。

该漏洞使得攻击者能够在目标服务器上远程执行任意代码。它允许攻击者滥用 RequestMapping 注释，该注释将传入的 HTTP 请求映射到相应的处理程序函数中。当 RequestMapping 作为处理程序参数与传统的 Java 对象结合使用时，会出现易受攻击的情况。这种攻击可能允许访问所有网站内部数据，包括任何数据库。它还可能允许访问额外的内部资源，以获得更多的权限或转向内部网络的其他部分。

与影响日志工具的 Log4Shell 漏洞相比，尚不清楚 Spring4Shell 对 Java 应用程序的影响有多大，但更多的安全研究人员正在寻找 Java 应用程序环境中的 RCE 漏洞。现在，组织需要决定是维护可能遭受更多零日漏洞发现的遗留应用程序，还是用更安全的编程语言开发的新应用程序来替换它们。当然，不能保证新的应用程序会比旧的应用程序更安全，但是很有可能现代应用程序比旧的应用程序具有更少的漏洞，这仅仅是因为从安全的角度来看，底层的编程工具和平台更先进。

与此同时，DevOps 团队可能会在未来几个月内寻找 Spring4Shell 漏洞的实例。拥有软件材料清单(SBOM)的开发运维团队自然能够比没有的组织更容易地找到那些实例。然而，拥有战略业务管理系统的组织的百分比仍然相对较低。然而，随着零日漏洞成为一个更大的安全问题，随着组织审查其软件供应链，SBOMs 正被更广泛地实施[。](https://devops.com/fixing-spring4shell-starts-with-software-supply-chain-management/)

无论确保应用程序安全性的途径最终是应用于应用程序开发和部署的审查级别，在未来的几个月中，流程将在大大小小的组织中加速最佳 DevSecOps 实践的采用，不管它们喜欢与否。
