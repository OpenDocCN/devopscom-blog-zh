 *## EO:“转变联邦客户体验”

本周首先关注的是:拜登-哈里斯政府发布行政命令，指示其联邦机构做出 36 项具体的客户体验(CX)改善承诺。这很大程度上涉及使用技术来改善 DevOps 团队所称的*UX*——但是政府不喜欢“用户”的**语言**

### 分析:以客户为中心

值得注意的是，许多与政府的互动仍然涉及纸质表格。但是这个 EO 不应该仅仅是把现有的流程放到网上。还需要努力解决潜在的服务复杂性和分散性:关注 CX 是一个有益的框架。

[Brett Samuels](https://thehill.com/homenews/administration/585522-biden-to-sign-order-to-streamline-government-services-to-public "read the full text") : **拜登将签署精简政府服务的命令**

该行政命令旨在减少政府为公众提供服务的官僚作风，如更新护照、申请贷款或改名。[它]影响了 17 个联邦机构的 36 个“客户体验改善承诺”，[针对]旅行、退休、商业、健康和更新个人信息。例如，在线更新他们的护照，而不是处理打印的表格。[它]将为…学生贷款债务创建一个单一的门户，小企业主将有一个更简化的…贷款、赠款和认证流程。
……
自然灾害的幸存者在申请援助时也不再需要填写几个机构的表格……他们将能够提交智能手机照片，并在提交索赔时使用虚拟检查。随着自然灾害变得越来越具有破坏性，这可能是至关重要的，正如周末肆虐肯塔基州的龙卷风所证明的那样，这场龙卷风导致数十人死亡，摧毁了许多社区。

这是技术问题吗？是的，部分原因——但是这也是一个*政府*的问题，詹妮弗·帕尔卡认为:

像《21 世纪创意法案》这样的法律……要求机构“消除或合并重复或重叠的网站或网页……并确保每个网站或数字服务都是围绕用户需求设计的”，这难道不是让这些机构的客户生活更轻松了吗？你说什么？你不确定你注意到了吗？原来你并不孤单。
……
平等机会令是我们政府拥有的众多机制之一，我喜欢称之为“咒语”(但)咒语和它们应该推动的变化之间存在差距。(失败)以如此惊人的规模发生，完全在意料之中。
……
我们倾向于将责任……归咎于政府技术[但是]在某些时候，你只能通过解决[潜在的]复杂性来改善客户体验。……人们陷入的大多数卡夫卡式的噩梦之所以会发生，是因为每项服务只看到了它们强加的负担，而没有看到客户体验到了什么。

如果你像我一样，天生就对政府效率持怀疑态度，你会认为这一努力会彻底失败。但是让我们不要太*愤世嫉俗:警告“选择偏见的危害”，这里是 *[上海比尔](https://news.slashdot.org/comments.pl?sid=20404043&cid=62076791 "read the full text") :**

在过去，我请半天假去车管所更新我的驾照。现在我可以在网上 5 分钟内完成。…我在网上缴纳财产税。我在网上更新我的营业执照。我在网上注册了。

## 每年 5 亿美元——但能持续多久呢？

Mozilla 基金会公布了今年的收入预测。和前几年一样，Mozilla 通过让谷歌成为默认搜索引擎，从谷歌那里获得了最大份额的现金。但是有新的收入来源。

### 分析:一刻也不要太早

浏览器市场的未来应该是 DevOps 团队的战略重点。Mozilla 的谷歌摇钱树可能不会长久。人们通常认为 Chrome 的制造商让 Mozilla 活了下来，因此它可以将 Firefox 视为“竞争对手”。但是随着微软的浏览器逐渐占据使用排行榜，这种情况可能不再是真的了。

Frederic Lardinois:**Mozilla 预计收入将超过 5 亿美元**

对于 2021 年，该组织预测收入超过 5 亿美元。然而，也许最重要的是，Mozilla 的新产品，如其 Mozilla VPN 服务、Firefox Relay Premium、Pocket 和其他商业计划正在缓慢但肯定地开始获得回报。但是从各方面来看，Mozilla 目前仍然完全依赖谷歌。分散收入来源确实是 Mozilla 减少对搜索交易依赖的唯一方法，该公司既是竞争对手，得益于其占主导地位的 Chrome 浏览器，也越来越不符合 Mozilla 的整体理念。
……
这里有一个非 Chrome 浏览器的窗口，用户越来越[怀疑谷歌的动机](https://securityboulevard.com/2021/12/google-nukes-ad-blockers-manifest-v3-is-coming/)和微软的 Edge 团队在最近几个月犯了一些错误。然而与此同时，Mozilla 为 Firefox 带来赞助建议和广告的努力也不一定会让该组织受到自己用户的喜爱。

为什么谷歌要资助它的竞争对手？[u/塞拉龙](https://www.reddit.com/r/technology/comments/rfyamx/comment/hoi2ffu/?utm_source=reddit&utm_medium=web2x&context=3 "read the full text"):

谷歌是 Mozilla 最大的捐赠者。…这基本上是为了避免任何垄断指控，这样他们就可以指着 Firefox 说“看，我们有竞争对手了”。当然，Mozilla 拿走了他们的钱，但是……他们不卖数据，也不欠谷歌的。

但是不要指望这些现金会继续流动，梅尔沃斯说:

随着 Edge 走向市场，这些资金现在可能受到威胁。当你有来自微软的实际竞争时，为什么给 Mozilla 几亿美元来显示竞争？

## Log4j/Log4Shell 每次我们看起来都更糟

易受 Log4Shell 攻击的流行 DevOps 工具的列表在不断增加。仅仅找到你可以利用的服务可能是一项不可能的任务——更不用说修补它们了。

### 分析:纵深防御是关键

搜索并修补易受攻击的代码是不够的。您还需要通过审计互联网可见的服务和收紧您的应用程序防火墙规则来支持您的网络防御。具有讽刺意味的是，密切关注您的日志，寻找**企图利用漏洞和 IOC**的迹象。

史蒂文·j·沃恩-尼克尔斯 : **我们有大麻烦了**

Log4j 用的很多。而且，当我说很多的时候，我的意思是它在数百个应用程序中使用[包括] Druid，Dubbo，Flink，Flume，Hadoop，Kafka，Solr，Spark，Struts，Apple iCloud，… LogStash，GrayLog2，… Steam，Twitter，以及… VMware。
…
Log4Shell 既容易被利用，也可被用来完全控制易受攻击的服务器。…就在你读这篇文章的时候，很有可能你正在被攻击。(但)即使你在系统中安装了补丁，它也很有可能隐藏在其他应用程序中。
……
log 4 shell 可能是一代人以来最糟糕的安全问题。…这将是一段漫长而艰难的旅程。

夸张？即使是 12 月 25 日对政府工作来说也不够近。卡塔林·西姆帕努:

美国网络安全和基础设施安全局已经告诉联邦民用机构修补受平安夜 Log4Shell 漏洞影响的系统。…联邦机构有十天时间…来测试他们的哪些内部应用程序和服务器利用了 Log4j Java 库，检查系统是否容易受到 Log4Shell 漏洞的攻击，并修补受影响的服务器。
…
尽管这是一个最近出现的漏洞，但它已经被认为是迄今为止发现的最严重的安全漏洞之一，主要是因为它在企业软件制造商中几乎无处不在的使用，它简单易用的利用方式，以及远程劫持系统的能力。…在被披露几天后，该漏洞被大量滥用。…第一次袭击发生在 12 月 1 日。

总有另一只鞋。泰瑞·罗宾逊在这里:

也许 Log4j 漏洞就像老鼠一样——每一个可见的漏洞背后都隐藏着许多其他漏洞。… Apache 表示，根据… CVE 2021-45046，解决“Apache Log4j 2.15.0 中的 CVE-2021-44228 在某些非默认配置中不完整”的修复程序。
