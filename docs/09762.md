# 云遣返和 SRE 运营之间的奇妙联系

> 原文：<https://devops.com/the-curious-connection-between-cloud-repatriation-and-sre-ops/>

我喜欢哲学。我还差三个班就拿到学位了，每隔几年我都会告诉自己有一天我会完成它。因此，我非常熟悉统计学和逻辑学中所谓的后验谬误，由此我们得出了“相关性不是因果关系”的说法。这是一个逻辑错误，即假设如果事件 Y 发生在事件 X 之后，那么事件 Y 一定是由事件 X 引起的。这一谬误最著名的注解来自 Bobby Henderson，他用他的图表展示了全球变暖是由世界上海盗数量的下降引起的，从而说明了假设因果关系的荒谬性。是啊，这说不通，但很多人们从中得出因果关系的图表也说不通。仅仅因为两个数据点相互映射并不意味着一个导致了另一个。在许多情况下，将两者联系起来甚至没有逻辑意义。毕竟，海盗和全球变暖？没人真的把那当回事。但是，当我们深入探讨[站点可靠性工程](https://devops.com/emerging-patterns-in-sre-implementation/) (SRE)运营和云遣返之间的关系时，这是很重要的一点。

需要澄清的是，我并不是说采用 SRE 的做法会导致云回归。但我认为这两者之间有着密切而有意义的关系。事实上，云提供商谷歌创造了 SRE 作为一种实践，这并不是一个错误。与 SRE 相关的模式、思维和技能是成功运营云基础设施和服务不可或缺的。

### 云遣返是真的

公有云遣返本身在某些圈子里是有点忌讳的话题。想想安德森·霍洛维茨(Andreessen Horowitz)发表的《云的成本，一万亿美元的悖论》(The Cost of Cloud，a 万亿美元的悖论)所引发的争议吧，该书指出，企业正从云中撤出，并因此实现了显著的成本节约。有些人会让你相信这不会发生，但有足够的数据和轶事证据表明，是的，它会发生。

F5 的《2021 年应用战略状态报告》向市场询问了公共云回归的相关问题。只有 13%的人已经撤回了应用程序，另有 14%的人正计划这么做。一年后，这两个数字分别上升了 40 个百分点，至 37%和 30%。这并不反常，因为有多家可信的分析公司报告了类似的结果。有趣的是，遣返率并不是全球通用的。与 EMEA 和北美相比，APCJ 和拉美遣返的可能性要小得多。

我坚持认为，公司正在从公共云中回收应用，问题不是“是吗？”而是‘他们拉出了多少工作负载，它们将走向何方？’这是一个应用战略 2023 研究状态的问题。

现在，让我们考虑一个可能的遣返推动者——SRE 行动。因为即使云成本的增加推动了回国的愿望，但如果你不具备在其他地方高效运营的技能，从而受益于更低的成本，那么你为什么要回国呢？

我们认为，正是 SRE 的运营实践和技能使公司能够回收并保持证明决策合理性所需的效率和成本节约，无论他们是将这些工作负载迁移到另一个公共云、内部云还是边缘云。

### 挖掘数据

从表面上看，SRE 实践的采用和应用与云遣返之间有很强的相关性，这似乎表明有能力以类似云的方式运营的组织，例如，他们已经采用了 SRE 实践，有效地拿起他们的玩具(应用)并回家(内部或其他地方)，因为他们可以这样做。

换句话说，在没有采用 SRE 实践的组织中，只有 4%的组织从公共云中回收了应用。采用 SRE 做法的人中有高达 73%的人也撤回了应用程序。

当然，采用实践并不一定意味着应用实践。因此，我们研究了组织如何实际操作应用程序、系统和基础架构。具体来说，我们研究了他们使用 SRE 实践的运营百分比。或许不出所料，这产生了类似的结果。

在那些使用 SRE 实践操作 0%应用、系统和基础设施的人中，81%没有回国。相反，在 76%-99%的应用、系统和基础设施运营中使用 SRE 实践的人中，54%已经回国。当组织超过使用 SRE 实践来运行超过四分之一(25%)的应用程序、系统和基础设施时，遣返似乎开始加速。

记得我说过 APCJ 和拉塔姆不太可能回国吗？他们也不太可能利用 SRE 的实践来运营他们的应用、系统和基础设施。事实上，在拉丁美洲和 APCJ，超过四分之一(26%)的受访者(29%)使用 SRE 的实践运营着零百分比的应用、系统和基础设施。在 EMEA？那只是 5%。而在北美，甚至更低，只有 2%。

### 有意义的关系还是好奇的巧合？

将 SRE 作为一种运营实践的组织与公共云返还率之间似乎存在无可争议的关联。但这是一段有意义的关系，还是仅仅是一个奇怪的巧合？

我认为这是一段有意义的关系。

与 SRE 相关的实践和技能完全适合在大规模的多云环境中操作。正如我之前所说，毫无疑问，是谷歌创造了 SRE，并就此写了一本书。云的价值在于其运营模式，这可以大幅降低每笔交易的成本，无论是通过 HTTP 交换还是客户会话来衡量。这实现了应用和数字服务的经济高效的规模。

自动化和实践的使用往往侧重于有意义的事件，而不是无中断的事件，这为负责维护高水平可用性和性能的人员(及其专业知识)提供了经济高效的规模。

SRE 实践的采用和使用使组织能够高效地扩展运营，无论是在公共云中、内部还是在边缘。数据告诉我们，组织似乎正在使用这种能力。
