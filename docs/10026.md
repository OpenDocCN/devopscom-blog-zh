# 从前端到全栈:我如何利用现代开发工具实现飞跃

> 原文:[https://devo PS . com/from-frontend-to-full-stack-how-I-leveraged-modern-dev-tools-to-make-the-leap/](https://devops.com/from-frontend-to-full-stack-how-i-leveraged-modern-dev-tools-to-make-the-leap/)

几年前，我有了一个想法，如何使在互联网上发布食谱更容易获得和有利可图。作为一名前端工程师出身的产品经理，我可以编写代码，只要它涉及 JavaScript。但是我需要一个能处理整个堆栈的人，尤其是后端——你不能只用前端工具来构建一个平台。于是我找了一个技术联合创始人，逛了逛网上论坛，和朋友聊了聊。最终，我找到了一个看起来很合适的候选人(就叫他汤姆吧)。他过去开发过一个食谱应用程序，对前端和后端都很满意。似乎很棒。我们达成了一笔交易，然后就开始了。

汤姆开始构建 API 和数据库，而我用名为[](https://bubble.io/)的无代码构建器制作原型。我喜欢用泡泡。无需编写任何代码，我就可以快速构建东西，并看到我在创建什么。不过，最终，无代码方法有太多的限制，包括缺少用于版本控制和历史的 Git 存储库。所以我们决定改用全代码的 [React 原生](https://reactnative.dev/) 移动应用。T9】

几年前，我改变了职业道路，成为了一名产品经理，但是我已经有一段时间没有深入研究编写代码了。尽管我有 React 的背景，但我对自己构建整个 web 平台的前端感到不安全。但是，唉，当你是一个初创公司的创始人时，你会做各种可怕和不受欢迎的事情！

然而，在投入其中之后，我很快想起我真的很喜欢编码。我很快学会了 React Native，并开始构建屏幕。汤姆负责数据，而我负责用户界面。一切都很好。

好吧，一切都很好，直到它不是。离发布日还有几周，我们就要完成了。我们甚至有几个早期博客排队和我们一起发布。然后，惊喜！汤姆决定在另一家初创公司担任首席技术官。他的临别赠言是，“你够专业；看起来你能处理好。”

我被难倒了。除了前端，我不知道任何东西是如何工作的。我们在[](https://www.pulumi.com/)上运行云功能，在 [谷歌云](https://cloud.google.com/) 上运行，我不知道它们是如何工作的。我不知道如何训练我们用于成分识别的机器学习人工智能模型。关于 SQL，我所知道的最多的就是你用它从数据库中获取数据。更糟糕的是，我们的食谱抓取 API 是用 Python 写的，这是一种我没有任何经验的语言，运行的版本在我的 M1 Mac 上无法运行。我到底应该做什么？

我想我做了任何工程师都会做的事。我一步一步想明白了。当 Postgres 数据库用完了 free [Heroku](https://www.heroku.com/) 计划中的行时，我想出了如何迁移它。当 recipe scraper API 需要更新时，我学会了如何用 Python 编码，并用 [Docker](https://www.docker.com/) 部署。那段时间压力非常大。我面临着迫在眉睫的发布截止日期，代码库随时都可能崩溃——主要是因为我不知道如何坚持下去。

令人欣慰的是 graph QL API 平台,[HasuraT5。这是迄今为止我最容易理解的部分。理解我们的数据是如何布置的非常容易——我可以像浏览电子表格一样简单地浏览它。我不确定如何构建 GraphQL 查询，但 Hasura 网站上的教程帮助我学习了基础知识，控制台中的查询构建器完成了剩下的工作。它没有代码，但是更好——一个没有代码的控制台使添加新表和修改权限变得非常容易，Git 存储库可以检查一切并易于引用。](http://www.hasura.io)

一旦我理解了 Hasura 和堆栈的其余部分是如何工作的，我就能够完成启动所需的屏幕。

如今，该公司已显著增长。我们有了一个新的联合创始人，但她不是技术人员。在弄清楚如何运行整个堆栈后，我认为我有足够的技术来成为首席工程师。我已经学会了如何使用各种各样的 Hasura 功能，这些功能让我们的生活变得更加轻松——监听事件、在我们的 API 中启动操作来运行业务逻辑，等等。通过这一切，我现在认为自己是一个全栈工程师。
