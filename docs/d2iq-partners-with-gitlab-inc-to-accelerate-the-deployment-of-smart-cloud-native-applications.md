# D2iQ 与 GitLab Inc .合作，加速智能云原生应用的部署

> 原文：<https://devops.com/d2iq-partners-with-gitlab-inc-to-accelerate-the-deployment-of-smart-cloud-native-applications/>

*D2iQ 和 GitLab 结合 Kubernetes 和 DevOps 的优势，为新应用提供更高效的运营和更快的上市时间*

**加利福尼亚州旧金山，2022 年 5 月 16 日—**[D2iQ](https://d2iq.com/)，领先的智能云原生应用提供商 Kubernetes 今天宣布与软件创新平台 One devo PS Inc .[git lab](https://gitlab.com/users/sign_in)合作，以自动化和简化容器化工作负载的部署，加快软件交付周期。

通过这种合作关系，用户现在可以在 D2iQ Kubernetes 平台(DKP)上运行 GitLab Kubernetes 运营商，为基础设施和应用提供一致的升级、部署和安全策略。企业组织还可以通过连接到 GitLab 的集群，从持续集成/持续交付(CI/CD)管道安全地将项目部署到 DKP，从而提供增强的灵活性和敏捷性。

“GitLab 和 D2iQ 之间的合作伙伴关系允许联合客户大规模构建、测试、部署和运行现代应用，”git lab 联盟合作伙伴计划和走向市场高级经理 Jaimie Kander 表示。“GitLab 针对 DevOps 生命周期的单一应用程序鼓励持续测试，并允许增加可见性，这使得更容易安全地采用下一代云原生技术。”

根据《Kubernetes 企业年度报告》[](https://d2iq.com/resources/report/kubernetes-in-the-enterprise-annual-report-2021)中的调查结果，“缺乏 IT 资源是整个企业采用 Kubernetes 的最普遍挑战。GitOps 工作流与声明式 API 的完全集成使得在生产环境中管理 Kubernetes 集群变得更加容易。DKP 减轻了开发运维团队的运营负担，使他们能够专注于创新，而不仅仅是支持基础设施。此外，DKP 的用户友好控制平面提供了使用 GitLab 的各种开发方法，包括使用 FluxCD 的持续交付和部署，FluxCD 是为 Kubernetes 设计的，支持多集群和多租户用例。

D2iQ 首席执行官鸢·克纳普说:“各种规模的组织在推动创新、提高性能和速度方面都面临着前所未有的压力。“将领先的 Kubernetes 和 DevOps 平台相结合，使组织能够实现更高水平的业务绩效。D2iQ 和 GitLab 正在提供一流的解决方案和无与伦比的用户体验，帮助我们的客户在第二天的运营中取得成功。”

D2iQ Kubernetes 平台可于 2022 年 5 月 16-20 日在 D2iQ 位于 KubeCon + CloudNativeCon EU 的展台现场体验。更多信息，请访问:[【www.D2IQ.com】](http://www.d2iq.com/)
