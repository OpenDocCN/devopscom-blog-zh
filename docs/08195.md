# 利用云进行大型机访问

> 原文:[https://devo PS . com/levelling-the-cloud-for-mainframe-access/](https://devops.com/leveraging-the-cloud-for-mainframe-access/)

现在，世界各地的组织比以往任何时候都更希望扩展其现有的[大型机](https://devops.com/?s=mainframe)投资，作为更大的数字化转型计划的一部分。作为记录和可靠性的核心系统，大型机继续运行着许多组织最重要的应用程序，替换这些系统的想法带来的[风险大于回报](https://www.microfocus.com/en-us/assets/application-modernization-and-connectivity/continuous-modernization-boundless-business-value)。当组织考虑对其大型机投资进行现代化时，重要的是要记住维护大型机访问并为其用户提供处理数据和系统的新方法是更大项目的重要部分。

## 利用云进行大型机访问

随着最近远程工作的增加，越来越多的公司寻求利用云进行大型机访问也就不足为奇了。这有许多好处，从提高可扩展性、可用性和性能，到降低应用程序成本和提高安全性。将现代大型机访问作为大型云项目的一部分来实施，可以确保组织的现代化之旅不仅是转型之旅，而且是安全之旅。

如果决定利用私有云或公共云进行大型机访问，安全主机访问解决方案的集成可确保团队能够访问完成工作所需的系统和应用，无论他们身在何处。通过将主机访问管理与企业用户目录集成和集中，在云环境中部署这种安全访问变得更加容易。除了简化资源调配和管理之外，最重要的是，集中管理的主机访问比其他形式的访问更加安全。

将企业安全性扩展到大型机有时会在转换中被忽略，但它是加强数据和系统安全性的更大过程中不可或缺的一部分。在理想情况下，这可以通过将主机访问解决方案与现有的身份和访问管理(IAM)系统集成来实现。如果这是可能的，就可以成功利用已经用于授权或拒绝访问大型机的身份验证凭据。

IAM 集成在访问、数据、更强的密码、应用程序安全设置和系统配置方面有许多好处。使用 IAM，最低权限访问的好处意味着用户只能访问完成工作所需的大型机系统和数据。此外，通过使用用户目录，可以从一个中心点更改应用程序安全设置和会话配置。最后，IAM 集成的更强的凭证要求消除了较弱的八字符密码，从整体上增强了安全性。

## 主机访问改善了用户体验

就主机访问而言，拥有一个安全的集中式解决方案不仅可以改善用户的大型机访问体验，还可以改善管理员的体验。借助零占用空间的安全访问，用户可以在需要时转移到更安全、安全标准更高的网站。企业凭证利用大型机会话授权来自动登录，安全性也更强。主机访问解决方案的集成使管理员能够快速部署安全的基于浏览器的会话，并提供无需 Java 的 web 界面，从而消除了对工作站的依赖性。这也与容器的使用相结合，以支持向无数用户的部署和对移动设备的支持。由于这些原因，主机访问解决方案应该是任何云之旅中的常见组件。

大型机及其数据、应用程序和用户访问对组织来说仍然至关重要，因此保持它们的安全和连接非常重要。随着安全性成为一个不断发展的问题，确保这些核心系统的安全，同时确保员工能够访问他们需要的系统和数据应该始终是一个优先事项。总之，希望将公共云或私有云用于其大型机的组织，如果在此过程中也加强其数据和系统安全性，将会取得更大的成功。
