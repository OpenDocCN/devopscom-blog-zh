这本书叫做《数字领导者的发展计划》。正确吗？

Aruna Ravichandran: 是的，没错。

Alan Shimel: 为什么不从一开始就给我们的听众简单介绍一下这本书的内容呢？

Aruna Ravichandran: 听起来不错。所以这本书叫做*面向数字领导者的 DevOps*，它讲述了一个人如何通过一个现代的支持 devo PS 的软件工厂来重新点燃他们的业务。大约一年半前，我和我的两位合著者凯伦·泰勒以及彼得·沃特豪斯开始了写这本书的旅程。

我们查看了市场上所有不同的书籍——当你想到 DevOps 时，市场上并不缺乏书籍——但我们看到的是，尽管市场上有如此多不同的书籍，但没有一本真正谈到我们经常遇到的客户在我们与他们交谈时询问我们的一些痛点。

客户经常问我们的一些棘手问题是，“我希望能够开始这个叫做 DevOps 的旅程。我该如何开始？我的旅程应该是什么？我能从哪里开始？我该如何发展？我的成熟模式应该是怎样的？”这是我们经常遇到的一个问题。

我们从客户那里听到的第二个难题是，“我如何实际衡量开发运维的成功？我可以使用什么样的指标？我可以向我的管理链报告，我与 DevOps 一起开展的任何计划现在都是成功的，这实际上是怎样的成功呢？”

因此，我们与几个客户——超过 100 个——交谈，我们从他们的经验中了解了实际使用的指标以及在行业中实际构建的指标，并在书中以用例以及实际指标的形式分享了这些内容。

因此，这本书主要从旅程的角度关注 DevOps 的采用，以及为了采用 DevOps 可以实际使用的一些关键指标。

艾伦·希梅尔:太好了。阿鲁纳:你发现了一些对我来说很亲近的东西。一个是我已经和很多人谈过的事情，你知道，DevOps 被大肆宣传，很多人将 DevOps 定位为，你知道，这是一个伟大的白色独角兽，为每个人带来彩虹，但它并不总是这样工作，对吗？

有失败，有挑战，有坎，不是每个只想做 DevOps 的人都能挥一挥魔杖，成为 DevOps 工厂或者现代软件工厂。所以我很高兴听到你们在书中揭露了这一点。

另一件事是，你知道，在做《现在 DevOps.com》将近三年的时间里，我发现我们的听众，我们的读者从听他们的同龄人如何处理事情中学到的最多:所以案例研究，案例情景和诸如此类的事情。因此，事实上，你可以接触到与 CA Technologies——CA 合作过的数百位——如果不是数千位的话，也是数百位——客户，这确实给了我们一些深刻的见解。

Aruna Ravichandran: 是的，绝对是。所以如果你看这本书，它是在整个软件工厂中组织的。假设您考虑客户部署，开发运维基本上是从规划到需求，再到开发、测试、发布，然后进入运营。

因此，我们所做的是与一些不同的从业者交谈，他们是现有的 CA 客户，有时我们甚至与非 CA 客户交谈，了解 DevOps 在文化和协作方面所起的作用，以及他们实际利用的工具集，以便在他们的公司内实施 DevOps。

我们研究了——我们 _____ 围绕整个软件交付链的用例。所以如果你看这本书——即使我们与几千名客户交谈过——你知道，我们基本上不能把所有的案例研究都放在这本书里， 因此，我们所做的是，我们将这本书分成了整个软件交付链，其中有一章专门介绍了软件工厂的各个方面，从规划到开发到测试到发布再到运营，我们利用了一个客户示例来说明他们实际上是如何实施 DevOps 最佳实践以及底层工具集的，以及他们在能够展示成功的实际情况方面实际获得了哪些指标或结果。

*艾伦·希梅尔:*明白了。Aruna 我想谈谈你在书中提到的 DevOps 采用的另一个方面，那就是不是每个人都在同一时间开始他们的 DevOps 之旅，或者以相同的速度或速率前进，对吗？一些组织在他们的 DevOps 采用和 DevOps 实践方面相对成熟:其他组织仍然在谈论开始。那么，这本书里是否有适合每个人的东西，或者你需要成为一名开发人员才能真正获得价值？

*Aruna Ravichandran:* 没有。所以，你知道，就像我说的，因为我们知道没有明确的框架，对吧——就像你想一想 I-_____ 就有一个非常规范的框架，规定你实际上可以从哪里开始，以及你实际上可以如何结束。有了 DevOps，人们可以开始许多不同的方式。

因此，我们实际上做的是，我们查看了许多客户实际实施的一些最佳实践，并根据成熟度模型构建了一个框架，我们将其作为最佳实践进行了分享。

因此，如果您想加快我们谈到的成熟度，您知道，客户实际上可以如何开始的三个不同示例。第一种方式是基本建成端到端的发布管理系统；第二个问题是如何操作从开发到运营的反馈循环，反之亦然。

最后一个问题是，如何优化连续性稳定性渠道？

因此，我们分享了一些最佳实践，涉及到人们实际上可以从哪里开始，以及他们如何实际上修改他们的流程，以给他们一个生命周期，而不管他们想从哪里开始。他们可以从开发开始，到运营结束；他们可以从运营部门开始，对开发部门进行调整。但是我们分享了一些客户可以实际实施的成熟度模型方面的最佳实践。
