# 基础设施访问管理日益成为 DevOps 的优先事项

> 原文：<https://devops.com/infrastructure-access-management-a-growing-devops-priority/>

组织计划在未来 12 个月内解决访问管理问题，因为保护和简化基础架构范围内的访问控制是其他计划(如零信任计划)的先决条件。

这是由 Pollfish 发起并由 strongDM 赞助的对 600 名 DevOps 专业人士进行的[调查的关键发现之一。该报告还发现，传统的访问流程导致团队效率严重低下。](https://www.strongdm.com/blog/year-of-access)

这些效率低下的问题需要大量的时间和资源来修复和阻止敏捷开发实践:近 90%的受访组织表示，他们需要两名或更多的员工来审核和批准访问请求，而这些可能需要几天或几周的时间来完成。

调查还表明，组织继续使用不安全的访问管理实践，这使得跟踪和审计关键业务系统的用户和权限变得困难。

strongDM 首席执行官 Tim Prendergast 表示，随着越来越多的工作变得技术化，越来越需要为更多的人提供访问权限，这可能会对公司保持安全的能力产生严重影响。 他解释说，当 65%的组织报告他们的团队使用共享登录，超过 40%的使用共享 SSH 密钥时，你几乎没有办法知道谁在你的基础设施中，或者他们可能造成的破坏。

他说:“这使得确定任何泄漏或丢失变得很困难，因为你有 20 份你家钥匙的副本在到处流动。”。“这是大多数组织在访问速度和便捷性与确保访问安全之间进行权衡的一个例子。”

调查受访者表示，他们最大的挑战是请求和授予访问权限所需的时间(52%)以及分配、轮换和跟踪凭据的任务(51%)。

## 快点等着

普伦德加斯特说:“使用当前的访问方式意味着你在雇佣这些高薪的技术资源，并告诉他们要抓紧时间等待。”

近一半(47%)的受访者表示，他们在入职员工和承包商方面遇到了困难，Prendergast 指出，四分之一的组织表示，仅仅是获得访问许可就需要涉及四个人的流程。

他说:“想想看，在 25%的组织中，你的技术资源基本上在等待访问这个数据库或那个 Kubernetes 集群时无所事事。“现在再乘以您拥有的数据库、服务器、员工和第三方供应商的数量。这还不包括像 Kubernetes 这样的新技术加入到您的基础设施中。最终，即使是您的团队在等待访问时的挫折感也会成为一种负担。”

## 2022:融合的一年

Prendergast 预测，2022 年，开发运维与安全的融合将超越我们在 [DevSecOps](https://www.strongdm.com/blog/year-of-access) 已经看到的情况，在那里，它一直非常专注于左移，并将安全更早地引入开发周期。

他说:“这种融合将以新的工作流程、技术和解决方案为标志，这些新的工作流程、技术和解决方案不仅能提高安全性，还能缩短开发周期。”。“一个很好的例子是优化基础架构访问—如果操作得当，您可以使用零信任方法来改善您的安全状况，同时还可以让开发运维团队更轻松快捷地访问系统。”

他补充说，零信任面临的两个最大的劳动力动态是远程工作和大辞职。

他解释道:“过去，在这种环境下，您必须亲自到场或使用 VPN 才能访问，远程工作打破了这种局面。”。“现在你也有这么多员工离职。你知道他们能进入什么系统吗？你怎么知道所有的访问是否都被关闭了呢？如果他们使用共享凭据会怎么样？”

Prendergast 说，这就是为什么解决访问问题对于应对这一挑战和实现现代安全至关重要——如果您不知道每个系统中谁可以访问什么或他们可以做什么，您就永远无法实现零信任。

“组织需要找到一种方法来理解每个技术专家和每项技术之间的关系，然后能够跟踪和审计这些关系，”他说。“在你做到这一点之前，你将很难达到零信任。这些是现代安全的赌注。”
