# Tricentis 扩展了基于云的测试工具的范围

> 原文:[https://devo PS . com/tricentis-expands-scope-of-cloud-based-testing-tools/](https://devops.com/tricentis-expands-scope-of-cloud-based-testing-tools/)

Tricentis 本周[扩展了一套基于云的测试工具](https://www.businesswire.com/news/home/20220223005518/en/Tricentis-Accelerates-Cloud-Adoption-to-Further-Enable-Customer-Success),承诺简化云原生和移动应用的开发以及现有应用向云的迁移。

产品组合中增加的新功能包括基于云的规划工具的集成和更广泛的移动应用测试功能。例如，DevOps 团队现在可以部署 Tricentis qTest 平台来管理 Kubernetes 集群中的测试流程。Tricentis 还简化了 Tosca 应用测试工具套件的配置和更新。

![](../Images/20a1b1c4864ce519e03a853b18d3773d.png)

此外，Tricentis 还增加了额外的功能，用于测试在云中运行的 SAP 应用程序，这些应用程序之前部署在内部 IT 环境中。

Tricentis 产品营销副总裁 Martin Klaus 表示，目标是让 DevOps 团队能够更简单地将各种类型的应用程序的持续测试集成到他们的 DevOps 工作流中。DevOps 团队可以选择通过软件即服务(SaaS)平台调用 Tricentis 测试工具套件，或者使用 Tricentis 提供的模板将它们部署在云平台上。

虽然开发人员显然越来越多地参与测试，但 Klaus 表示，将测试功能集成到 DevOps 管道中至关重要。每个开发人员通常只处理大型应用程序的一个组件。他指出，除非在端到端的基础上进行测试，否则一个组织将无法在生产环境中部署软件之前确定应用程序体验的真实质量。

Klaus 补充说，因此，关于使用什么工具作为持续测试过程的一部分的决定现在是由 DevOps 团队领导而不是单个开发人员做出的。

最终，目标是尽可能多地自动化[连续测试](https://devops.com/?s=continuous+testing)过程。克劳斯说，实现这一目标将需要增加对人工智能(AI)技术的依赖，尽管程度不同。然而，这些能力不可能消除对专业测试人员的需求；相反，人工智能将使这些团队能够更有效地测试日益复杂的应用环境，他补充说。

Klaus 指出，事实上，如果没有例如应用计算机视觉来分析用户界面的算法的帮助，测试人员手动检查应用程序的每个元素几乎是不可能的。他补充说，总会有需要人类的更高级别的任务，但随着组织开始构建更多的应用程序，人工智能技术将在实现大规模测试方面发挥重要作用。

与此同时，很明显，随着组织继续推出数字化业务转型计划，测试的难度会更大。在许多情况下，组织只有一次机会说服最终用户，即将推出的新应用程序比他们以前采用的流程更好。因此，在 DevOps 工作流环境中简化测试的需求变得至关重要——对于降低应用程序构建和部署速度的瓶颈，几乎没有容忍度。
