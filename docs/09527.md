# 从芯片到云的物联网:迈向 Web3 的一步

> 原文:[https://devops.com/chip-to-cloud-iot-a-step-toward-web-3-0/](https://devops.com/chip-to-cloud-iot-a-step-toward-web-3-0/)

在 2021 年的前六个月中，物联网设备被攻破[15.1 亿次](https://www.iotworldtoday.com/2021/09/17/iot-cyberattacks-escalate-in-2021-according-to-kaspersky/) ，与 2020 年全年仅观察到的 6.39 亿次攻破相比大幅增加。这个问题可以归因于物联网(IoT)和 Windows Server 消息块(SMB)的广泛采用，在[远程工作](https://devops.com/?s=remote+work)的时代，这两者都无法避免。

虽然这些技术为工业环境中许多互联的数字和物理系统提供了前所未有的数据移动和监督水平，但如果不加保护，它们也使恶意行为者更容易渗透到我们的生活中。

这使得 DevOps 团队和开发人员在创建物联网系统时优先考虑网络安全变得更加重要。解决固有的 [安全缺陷](https://devops.com/iot-security-concerns/) 很重要，比如容易被破解的用户名和密码，或者在远程访问服务器上无保护地保存数据。

开发人员和 DevOps 专业人员有一种方法可以帮助遏制日益增长的网络犯罪，这就是所谓的芯片到云物联网。让我们探索它是如何成为一种可行的方式，为每个人构建一个更安全、更有价值、更分散的系统。

## 物联网:了解固有的安全挑战

最好先调查您的组织可能面临的潜在网络安全问题，然后再想办法使用解决方案来保护您的物联网平台，如 [左移安全](https://www.crowdstrike.com/cybersecurity-101/shift-left-security/) 方法(意味着安全性嵌入在应用程序开发流程的最早阶段)或物联网分散方法，我们将在下面展开讨论。

一些常见的安全问题包括:

### 软件和固件中的漏洞

由于以下安全缺陷，保证物联网系统的安全具有挑战性:

*   缺乏计算能力来提供足够的内置安全性
*   物联网系统中低效的访问控制
*   用于充分测试和进一步开发固件安全性的资金不足
*   由于物联网设备的技术和预算限制，缺乏定期修补和升级
*   无法修补的漏洞，因为用户没有定期更新他们的小工具
*   旧设备可能与新软件更新不兼容
*   易访问性意味着攻击者可以插入损坏的芯片或通过无线电波进行黑客攻击

### 云漏洞的可能性

黑客可以通过拦截未加密的消息来获取你的物联网系统处理的数据。

您的设备可能包含敏感信息，如您的位置、银行账户信息和医疗记录。攻击者可以通过许多其他方式访问关键信息，包括利用保护薄弱的通信。

即使每一份数据都被传输到 [云存储](https://devops.com/tiering-cold-data-to-the-cloud-without-tears/) ，入侵者也有可能将目标锁定在云托管的应用上。因此，数据泄露的风险不仅来自设备本身，还来自设备所连接的云环境。这就是为什么 DevOps 专业人员、开发人员和软件工程师必须从一开始就找到保护技术的方法，并防止黑客侵入云。

### 恶意软件的风险

Zscaler 最近的一项 [调查显示，智能电视、机顶盒和智能手表最容易受到恶意软件的攻击。](https://ir.zscaler.com/news-releases/news-release-details/zscaler-study-confirms-iot-devices-are-major-source-security)

如果攻击者成功渗透物联网系统或插入恶意软件，他们可能会改变其功能，获取个人数据并实施其他可能危及设备所有者和用户的恶意活动。

麻烦在于，制造商努力维护适当的软件安全，以保证他们设备的安全。当这些小工具被制造出来时，安全性往往被放在次要位置，让购买者在感染恶意软件后处理善后事宜。

## 芯片到云物联网如何最大限度降低安全风险

组织有两种方法来增强其物联网安全性。

第一步是确保设备制造商和软件开发商都遵守可靠的安全原则。如果[价值链](https://devops.com/?s=value+stream+management)暴露，解决方案必须是芯片到云。

其次，他们必须采取深入的方法，通过分层安全控制来保护关键技术，以创建一个可靠的物联网基础设施。

芯片到云架构支持创建安全、低能耗的物联网设备网络，这些设备可以与 [云托管](https://www.atlantic.net/vps-hosting/what-is-cloud-hosting/) 平台直接通信。构建云优先技术堆栈的集体努力成本高昂、效率低下且不可靠。

传统上，IT 专业人员使用防火墙等非机器托管的安全工具来保护物联网设备。芯片到云架构旨在修复这一潜在缺陷。

## 芯片到云物联网的特点

板载安全性和计算能力是物联网设备的两个最大缺点。除了比以前的型号更强大、更安全之外，采用芯片到云安全性构建的物联网设备也更节能。

以下是使芯片到云技术如此有效的特性:

*   机载加密引擎
*   随机数发生器
*   足够的随机存取存储器(RAM)

### 它是如何工作的

可靠的软件设计是 [物联网设备](https://devops.com/why-trust-and-integrity-are-critical-to-iot/) 等物联网设备必不可少的。它可以防止黑客出于不可告人的目的窃取您的身份或复制您的设备。芯片到云在各个方面都有所贡献。

这些芯片组特性带来了额外的安全优势。每个物联网节点在密码上都是唯一的，这使得黑客几乎不可能冒充它并访问它所连接的更广泛的公司网络。芯片到云通过消除逻辑程序和准备好对信息采取行动的边缘节点之间的流量延迟来加快速度。

物联网的芯片到云架构在设计上是安全的。正在开发新的工具来为定制和旧设备提供数据移动能力，就像当前的物联网一样。

尽管如此，芯片到云芯片组总是连接到云。因此，资产的可用性和跨节点、部门和设施的数字通信速度将得到显著提高。芯片到云物联网是物联网向 Web3 演进的[](https://zephyrnet.com/chip-to-cloud-iot-a-decisive-step-toward-web-3-0/)的重大一步。专家预测，与集中式提供商相比，物联网设备客户和价值生产者将从这些设计原则中受益更多。

## 芯片到云实现物联网去中心化

物联网的传统方法是用第三方防火墙和其他安全措施覆盖现有的联网设备云。它的等效图像将是一把雨伞覆盖一个 12 口之家——或者至少试图这样做。管理安全的协议必须是权力下放的核心。

芯片到云物联网是一种功能强大、独特安全的芯片组，可以安装在每个设备中，使其在物联网链中的连接远远超过普通的物联网设备。

云管理器或分析应用使用芯片到云技术实时接收来自每个节点的数据。该过程不会增加另一层保护；相反，它减少了数据包从一个目的地传输到另一个目的地时的延迟和丢失。

## 物联网的未来

在最近的全球事件之后，通用云技术和芯片到云架构正在获得巨额投资。物联网可以提供公司和组织所需的数据，以推动其客户参与门户、机器维护平台和企业规划工具。

也就是说，即使有了芯片到云技术，物联网也不会完全安全。企业必须拥有 [以安全为中心的文化](https://devops.com/the-6-pillars-of-devsecops-pillar-one-collective-responsibility/) 、管理新 IT 投资的标准以及挑选最佳技术合作伙伴的专业技能。

Web3 过渡可能会与未来的 Web3 创新(如区块链)相结合，以提高其实用性和安全性。看到世界上真正的价值创造者如何使用这些新的坚实的工具将是有趣的。

## 结论

在任何物联网项目的一开始就考虑安全性是至关重要的。由于网络攻击频率上升和系统漏洞日益复杂，在物联网环境中维护全面的网络安全极具挑战性。

物联网设备的安全对于许多企业乃至整个经济的运营至关重要。工程师们面临着越来越大的压力，要确保快速增长的互联设备的安全。这就是为什么 [芯片到云技术](https://intersog.com/blog/chip-to-cloud-the-next-billion-dollar-technology/) 有望成为该领域的下一个大事件。你准备好改变了吗？
