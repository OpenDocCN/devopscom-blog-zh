# Verica 推出共享 IT 事故报告的数据库

> 原文:[https://devo PS . com/verica-launchs-database-for-sharing-it-incident-reports/](https://devops.com/verica-launches-database-for-sharing-it-incident-reports/)

Verica 是一家事故管理平台的供应商，本周宣布将提供一个数据库，通过一个单一的存储库向公众提供事故报告。

Verica 的高级研究分析师 Courtney Nash 表示， [Verica 开放事件数据库](https://www.businesswire.com/news/home/20211005005193/en/Verica-Announces-The-Verica-Open-Incident-Database-The-VOID-to-Bring-Transparency-to-Software-Based-Failures-and-Outages) (VOID)计划是帮助组织从软件导致的中断和故障中学习的努力的一部分。她补充说，如今，关于此类事件的报道往往很难找到，因为它们分散在互联网上。

Nash 指出，缺乏透明度使得人们很难通过比较和对比事件来获得有意义的见解。

VOID 的发布合作伙伴包括 Indeed、NS1、Adaptive Capacity Labs、Security Scorecard 和 Auxon。每个人都将提供支持，帮助资助该项目产生的知识的开发、研究和传播。

Nash 补充说，该网站不仅仅包含标准的公司事后分析或状态更新。Verica 致力于应用元数据来更好地了解个人、公司、媒体和其他人在事件中的反应。

从该计划中得出的一个早期发现是，由于事件数据的不同分布，平均解决时间(MTTR)被证明是一个不准确的指标。其他发现包括，只有四分之一的事件报告花时间确定问题的根本原因，而不到 1%的人花时间研究未遂事件报告。

总的来说，现在比以往任何时候都更加依赖 IT 环境的性能和可用性。随着 COVD-19 疫情带来的经济衰退，数字业务转型变得风靡一时。随着经济持续复苏，企业现在希望它能对不断变化的环境做出反应和调整，无论这种变化有多突然。

当然，DevOps 团队已经学会了通过采用基于 DevOps 最佳实践的现代事件管理来应对意外情况，使 IT 团队能够灵活地应对任何停机或应用性能的突然下降。关于事件管理，要记住的最重要的事情是熟能生巧。虽然实践运行可能永远不会准确反映实际的危机，但拥有 IT 事故团队熟知的流程和实践“消防演习”的组织往往会做出更快、更好的响应。毕竟，需要调用的肌肉记忆越多，组织的弹性就越大。

尚不清楚 It 组织愿意在多大程度上共享事故报告。纳什说，然而，IT 专业人员有机会走到一起，这类似于不同航空公司的工程师如何合作，为公众提供更安全的飞行。事实上，现在有如此多的过程依赖于软件，令人惊讶的是没有出现更多由于软件问题导致的灾难性事件。甚至有争议的是，这种级别的合作得到授权只是时间问题。

与此同时，实现这一目标的第一步可能只是与同事分享一份报告，这样他们就可以从以前的错误中吸取教训。
