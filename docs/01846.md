# DevOps 学院宣布新的现场可靠性工程(SRE)基础课程

> 原文：<https://devops.com/devops-institute-announces-new-site-reliability-engineering-sre-foundation-course/>

*SRE 基金会是 DevOps Institute 认证矩阵的最新成员，是 SKIL(技能、知识、理念和学习)框架的一部分*

**拉斯维加斯，2019 年 10 月 29 日–DevOps 企业峰会****–**devo PS institute([https://devopsinstitute.com/](https://hubs.ly/H0lwqnK0))，一个推进 devo PS 人力要素的全球会员制协会，今天宣布将 Foundation℠站点可靠性工程(SRE)加入其认证矩阵。新认证引入了 SRE 的原则和实践，使组织能够可靠、经济地扩展关键服务。

从 2020 年 1 月开始，SRE 基金会将通过 DevOps Institute 的全球注册教育合作伙伴渠道提供服务。虽然参加考试需要通过认证合作伙伴的培训课程，但参加培训没有任何先决条件。

DevOps Institute 的课程基于行业研究和最新资源，以实现基于角色的开发方法，该方法侧重于采用 DevOps 和 SRE 模式和实践的组织所需的现代能力。该公司提供八种基于能力的 DevOps 认证和三种非认证，包括超过 160 小时的学习材料，是当今市场上最多的。

**欲了解更多关于 DevOps 学院认证的信息，请访问:(**[](https://hubs.ly/H0lwq7X0)****)。****

**DevOps Institute 首席执行官 Jayne Groll 表示:“我们选择从基础级 SRE 认证开始，以便让 IT 社区对 SRE 的核心原则、实践和词汇有一个共同的理解。“站点可靠性工程可能是自 ITIL 早期以来在 IT 运营和 IT 服务管理(ITSM)指导方面最重要的进步。SRE 现在不仅是一个实际的工作角色，而且作为一个服务管理框架，它与 DevOps、持续交付和敏捷最为一致。”**

**在很大程度上基于谷歌网站可靠性工程书中描述的指导，认证内容还包括实用建议、相关实践和自动化需求，以减少被称为 toil 的手动工作。它是为专注于大规模服务可用性的从业者和领导者量身定制的。**

**SRE 基金会认证的引入响应了企业兴趣的快速增长和 SRE 作为服务管理替代方案的采用。站点可靠性工程师负责可用性、延迟、性能、效率、变更管理、监控、应急响应和容量规划。**
