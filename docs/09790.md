# 利用 AIOps 扩展预测分析，推动下一代 SRE

> 原文：<https://devops.com/scaling-predictive-analytics-with-aiops-to-drive-next-gen-sre/>

企业系统的价值取决于它们的可靠性，也就是说，它们不会遭受过多的故障。否则，由于管理问题的额外负担，公司会经历代价高昂的停机时间，并增加工程师的压力。这一确保系统在生产规模下以最少的人工干预可靠、优化运行的关键功能，是工厂可靠性工程( [SRE](https://devops.com/?s=SRE) )团队的职责。

SRE 专业人员的工作是通过开发和实现优化软件和应用程序的可靠和高度可扩展的系统来减少人类的劳动。他们的目标是主动识别并在可能的情况下预测潜在的中断，以最大限度地降低风险并确保最佳的系统性能。然而，要取得成功，SRE 团队需要一定程度的企业范围的可见性，如果没有合适的工具来捕获和分析适当详细程度的数据，这是很难实现的。

让我们来看看 IT 运营人工智能(AIOps)如何通过连接企业范围内的数据来支持 SRE 任务，以实现系统和流程的前所未有的可见性和控制，实现大多数问题的自动解决，并更有效地对 SRE 团队必须围绕修复进行合作的少数剩余案例进行分类。

### 追求 SRE 卓越的复杂性和规模挑战

随着越来越多的企业将其运营数字化并转向更高程度的自动化，他们的 IT 运营必须巧妙地利用所有数据资产，以提高可靠性并减少人工劳动。SRE 专业应运而生，专注于监控系统、辅助自动化发布、了解变更影响以及自动化一些最常见的系统流程。

为了有效地完成工作，SRE 团队需要各种各样的数据和分析能力。这包括深入描述性和诊断性分析的能力，以回顾过去和当前状况，发现过去发生了什么及其原因，并确定当前运营的基准。但这仅仅是开始。SRE 的真正价值  来自预测性和规范性分析的扩展，以利用历史数据并应用详细的分析来生成对未来最有可能发生的事情的预测性见解；这些见解是确定主动措施的基础，这些措施可用于解决任何潜在问题并优化这些未来结果，最大限度地减少对运营的不利影响

这些分析能力必须由来自整个 IT 产业的强大数据提供支持。被分配到 SRE 角色的开发人员在计划、构建、测试、发布、监控和保护系统时，面临着一个特别强的要求，即拥有这种全面的可见性；他们的工作受到一定程度的阻碍，组织孤岛或规模问题阻碍了这种可见性的实现。正是在这一点上，AIOps 可以通过扩展预测性和规范性分析的能力和范围，帮助 It 运营变得更加主动和自主。

### AIOps 在企业范围内支持 SRE 使命

AIOps 是 SRE 社区在减少操作员压力、将 IT 系统配置为更加稳定和高效运行以及减少人工干预的战斗中的一个重要工具。AIOps 采用人工智能(AI)和机器学习(ML)进行可观察性、上下文、正常行为分析和自动化健康诊断。这反过来又可以实时检测异常，并以闭环方式自动解决大多数问题。

例如，AIOps 可以发现这样的模式:90%的情况下，组织支付系统中的某个特定警报会在 15 分钟内触发一个看似无关的警报。向前看，这给了 SRE 团队 15 分钟的时间来处理二级警报；这一发现构成了自动解决或分类方案的基础，该方案揭示了两种警报的潜在原因，并提供了永久的修复方法。

这只是说明 AIOps 如何使用高级分析来识别数据中的微妙模式，以预测问题可能在何时何地发生，从而可以自动制定主动修复措施来避免这些问题。通过这种方式，AIOps 成为技术支柱，扩展并自动化了 SRE 团队对所有系统资产和依赖关系的洞察和控制，是在最少人工干预的情况下持续优化性能的关键工具。

### 结论:SRE AIOps 的改变游戏规则的角色

AIOps 是现场可靠性工程关键功能的游戏规则改变者。AIOps 通过对来自整个企业系统的强大数据进行高级分析，为 SRE 团队提供了前所未有的可见性和控制力，以减少工作量并确保企业系统的可靠性和弹性。其结果是减少了人工干预，提高了 IT 系统的价值，使其更强大、更稳定、更高效。
