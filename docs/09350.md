# 进步扩大了法规遵从性能力的范围

> 原文:[https://devo PS . com/progress-expands-scope-of-compliance-as-code-capabilities/](https://devops.com/progress-expands-scope-of-compliance-as-code-capabilities/)

本周的进展[扩展了其 DevSecOps 产品组合——构建在 2020 年收购的 Chef automation 框架](https://investors.progress.com/news-releases/news-release-details/progress-extends-power-devsecops-cloud-native-assets-first)之上——现在包括了以编程方式解决合规性要求的能力。

与此同时，Progress 更新了 Progress Chef InSpec 框架，用于自动发现合规性问题，增加了对 SAP ASE、IBM DB2、Mongo、Cassandra、Oracle、MS SQL 平台以及 RHEL、CentOS、Ubuntu 和 macOS Monterey 的支持。

Progress 还更新了 Progress Chef Enterprise Automation Stack(EAS ),增加了将基础架构配置处理与合规性审核结合在一个整合的策略定义中的功能，以及确保平台始终可访问的高可用性功能。

Progress 产品管理副总裁 Prashanth Nanjundappa 表示，Progress Chef 云安全使 DevOps 团队能够利用单一策略即代码平台，在多个公共云和内部 IT 环境中一致地实施合规性策略和安全控制。

Progress Chef 云安全利用经认证的行业基准，如互联网安全中心(CIS)基准，在 DevOps 管道内的多个阶段持续扫描和自动检测安全配置问题。Progress 现在已经扩展了针对亚马逊网络服务(AWS)、微软 Azure 和谷歌云平台(GCP)的 CIS 基准配置文件覆盖范围，以包括 100 多个可定制的预建服务和资源模板。DevOps 团队可以利用这些资源的代码、测试和文档工件的自动化创建。最后，添加了对合规性扫描的状态和预期完成时间的可见性，以及对控制和测试的高级 Shell 命令支持，而无需文件输出。

一般来说，解决 DevSecOps 工作流中的法规遵从性需求的能力变得至关重要，因为云中发生的[错误配置](https://devops.com/?s=misconfigurations)的数量已经激增。通常很少或没有安全或法规遵从性专业知识的开发人员正在以编程方式配置云基础架构，例如，让端口完全开放。Nanjundappa 指出，随着 IT 组织接受更多的云计算平台，安全问题只会变得更加复杂。

更具挑战性的是，随着基于 Kubernetes 等平台的云原生应用的兴起，云计算环境正变得越来越复杂。随着每个微服务的部署，开发人员出错的几率只会增加。

Progress 认为，随着云计算的发展，安全性和合规性的责任将越来越多地转移到 DevOps 团队；他们将负责实施由网络安全和合规专家定义的政策。Nanjundappa 说，目标是提供一套减少配置错误可能性的护栏，因为策略是在 DevSecOps 工作流的上下文中测试的。

在一系列引人注目的违规事件之后，软件供应链安全审查将迫使更多组织内部出现 DevSecOps 问题，这只是时间问题。当然，目标应该是消除将 DevSecOps 从 DevOps 工作流中分离出来的需要。然而，不管采用哪种方法，所有这些关注的一个好处是应用程序环境将不可避免地变得更加安全。
