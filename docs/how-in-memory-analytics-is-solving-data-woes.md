# 内存分析如何解决数据难题

> 原文：<https://devops.com/how-in-memory-analytics-is-solving-data-woes/>

数据对业务决策和运营的重要性不可低估。每天都会产生 [2.5 万亿字节](https://www.forbes.com/sites/bernardmarr/2018/05/21/how-much-data-do-we-create-every-day-the-mind-blowing-stats-everyone-should-read/#6285d91d60ba)的数据。管理如此多的数据，以便轻松获取必要的情报，或者确保不遗漏任何见解，是一项极具挑战性的工作。幸运的是，确实存在可以帮助显著缓解这种数据洪流压力的技术:内存分析。

最初由金融行业采用——在该行业需要复杂、即时的决策——内存分析非常擅长快速处理大量数据的分析。随着数据变得更加普遍，公共云服务的发展使得各种规模的公司都可以负担得起专用硬件，内存分析技术正在应用于从零售到供应链到医疗保健的各个垂直领域。此外，鉴于内存计算实现了一系列对各种业务都非常有用的分析功能，而这些功能在基于磁盘的平台上很难实现，该技术确实有望快速扩展。

## **以任何方式切片和切块**

如今，作为一名数据专业人员，最困难的部分之一是利用您可以支配的所有数据。不幸的是，商业智能(BI)专业人员通常受限于他们可以分析和不可以分析的公司数据部分，这取决于预定义的数据查询和他们的 BI 工具的限制。

然而，内存分析使分析师能够不受任何限制地探索数据。这创造了一种思路分析的体验，帮助企业进行更深入的分析，并发现对其市场和自身表现有价值的新见解。这意味着，例如，零售商可以对市场有一个全球性的、精确的看法，并交叉引用来自销售数字、客户收据和忠诚度计划的数据，以更好地了解客户对他们的期望。

## 弄清楚“如果……会怎样？”

由于涉及到大量的数据以及物理存储磁盘的限制，计算任何业务决策对复杂 KPI 的影响可能需要很长时间。

通过利用内存分析，BI pros 可以不断比较和对比“假设”情景(如对金融机构进行压力测试或为供应链业务建立中断测试案例),以清晰、即时地了解潜在结果，从而尽可能为未来做好准备。这意味着企业能够限制他们措手不及的次数，或者减轻任何意外变化的影响。

## **最新数据**

当今世界发展如此迅速，拥有尽可能最新的数据至关重要。在某些情况下，每天只更新一次或两次的静态数据集可能会导致重大的、代价高昂的误判。另一方面，内存分析可以支持来自多个来源的连续数据流，因此分析师总是使用最相关的数据集。例如，这意味着供应链企业可以对运营进行实时监督，以更好地应对任何发生的中断，并最终节省数百万美元的潜在损失。

数据科学的繁荣为内存分析开辟了一个新的用例领域，从验证从 ML/AI 派生的新算法到工业化构建新的分析视图和仪表板。如果您在数据科学方面进行了大量投资，内存分析可能是从您组织的每个数据基础架构中获得最大产出的关键因素。
