# 不再烫手山芋:协作如何成为应用安全的关键

> 原文:[https://devo PS . com/no-more-hot-potato-how-collaboration-is-key-in-application-security/](https://devops.com/no-more-hot-potato-how-collaboration-is-key-in-application-security/)

技术正以前所未有的速度加速发展，在应用安全方面几乎没有数字足迹或经验的公司正在努力跟上。随着在应用程序中检测到越来越多的违规和漏洞，人们开始推卸责任，就像这是一场烫手山芋的游戏。但不一定非要这样。

随着虚拟会议的召开，Rebecca Auguste 与 ShiftLeft 的首席执行官 Manish Gupta 坐在一起，讨论开发人员和应用程序安全团队如何更好地合作以提高应用程序的安全效率。两人谈论了 ShiftLeft 的战略，创新和安全如何相互矛盾，采用现代解决方案以及介于两者之间的一切。

**什么是 ShiftLeft？**

ShiftLeft 是一家下一代应用安全公司。随着各公司迅速采用云技术，开发软件的速度越来越快，因此他们可以向客户提供功能并推动收入不断增长，在这场竞争中，我们开始将安全性抛在身后，因为我们仍在使用传统工具。这个想法是，我们需要向左转，我们必须在这个过程中尽早开始修复漏洞，从而降低我们的网络风险。

创新和安全现在有冲突吗？

从历史上看，他们是。在现代管道中使用遗留技术迫使客户在敏捷、更快地交付特性功能和提高安全性之间做出选择。在 ShiftLeft，我们开始通过客户的成功向他们展示，这种选择并不一定是那样的。我们可以变得更加敏捷，也可以变得更加安全。

以下是 ShiftLeft 客户的一些重要统计数据:

1.  如果你在 ShiftLeft 中自动执行，你就修复了 91.8%的漏洞
2.  如果你没有进行自动化执行，而是使用了像 ShiftLeft 这样的现代解决方案，那么你在第一次冲刺中就修复了 58%的漏洞。
3.  如果你仍然停留在传统解决方案上，你的团队将永远无法修复 60%的漏洞——忘记这是第一次、第二次还是第三次冲刺。就变成了技术债。

那么，为什么人们不采用像 ShiftLeft 这样的现代解决方案，抛弃传统解决方案呢？

传统解决方案要求开发人员放慢速度，因为他们必须花几个小时去扫描和转储一长串漏洞。作为开发者，看到成百上千的漏洞就像车灯前的小鹿。因为这是一项堆积如山的工作，所以开发人员很容易想忽略这些问题。问题不应该停留在“我们的应用程序由什么组成？”当我们开始问我们的应用程序包含什么，它们有哪些可以被利用的漏洞，因此我们能多快修复它们时，真正的问题就解决了。一切都是为了变得主动，而不是被动。

**开发和安全团队如何更好地合作，以提高应用程序和软件的安全性** ？

对我们来说，认识到开发者不对安全性负责是非常重要的。开发人员是被衡量的，他们的报酬是按时交付特性功能，并以最少的错误提供特性功能。但是 bug 不是漏洞。业界有很多关于开发人员应该对应用程序安全性负责的讨论。我们今天不在那里，但我们最终会在那里。

有两个重要的应用程序角色:

*   应用程序安全性:应用程序安全性对应用程序以及应用程序安全性改进的补偿对象有深入的了解。
*   开发人员:开发人员对软件有深入的了解，因为他们太专注于自己的代码。

这两种角色都是必需的，因为应用程序安全性无法修复应用程序中的漏洞，开发人员必须这样做。但是开发人员不会想这么做，除非应用程序安全优先考虑这项工作。

**关键外卖？协作对于应用程序安全性至关重要。**

我们需要创建一个工作流，让应用程序安全满足他们的最终目标，例如:应用程序有哪些漏洞？有哪些关键漏洞？我如何在开发人员工作流中向开发人员提供这些漏洞，而不要求他们转到不同的控制台，以便他们可以尽快修复漏洞？我们需要开始看到一个允许应用程序安全性和开发人员角色协同工作来提高应用程序安全性的工作流。
