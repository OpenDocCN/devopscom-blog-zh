# DevOps Institute 的“提升技能 2020:企业 DevOps 技能报告”强调了人类转型的需求

DevOps Institute 首席执行官 Jayne Groll 表示:“人员转型是为企业 IT 组织提供 DevOps 实践和模式的最重要的成功因素。“传统的技能提升和人才培养方法不足以让企业保持竞争力，因为对拥有核心人力技能的 IT 专业人员的需求越来越大，这是企业领导者一生中从未见过的。我们必须像我们的技术一样，经常通过新的技能来更新我们的人类。”

**来自《2020 年提升技能:企业开发运维技能报告》的主要发现包括:**

*   ***对于超过 50%的受访者来说，DevOps 转型之旅仍然非常艰难:*** 管理与 DevOps 转型相关且必要的人员、流程和技术都非常困难。每个类别都收到了相同数量的挑战回复。
*   ***在 2020 年，寻找和吸引熟练的开发人员仍然是一个挑战:*** 报告中 58%的受访者[PA(2 )]表示寻找熟练的开发人员是一个挑战，48%的人表示很难留住熟练的开发人员。根据 Robert Half Technology 2020 薪资指南(【https://www.roberthalf.com/salary-guide】T4)，有经验的 DevOps 工程师的薪资可以超过 179250 美元。结合目前的技能差距，2020 年对于招聘经理和人力资源主管来说将是艰难的一年。正如人力资源公司 Robert Half Technology 的执行董事杰夫·韦伯(Jeff Weber)所说，“各种类型和规模的组织都在寻找开发人才。一家公司的 DevOps 专业人员在另一家公司可能需要不同的技能和经验，这使得招聘和雇用非常困难。”
*   ***devo PS 人力必须是混合型的，并在三个关键技能类别内配备:*** 流程技能和知识(69%的受访者)、自动化技能(67%的受访者)和人力技能(61%的受访者)是 2020 年前三大必备技能类别。这与 2019 年的研究有所不同，在 2019 年的研究中，自动化技能是最重要的技能类别，流程技能和知识被列为第二重要的必备技能类别。
*   *如今，提升技能需要商界领袖的关注:提升技能的挑战并不新鲜，根据世界经济论坛的说法，这是全世界科技工作面临的最大挑战。不幸的是，超过 38%的受访者的组织没有技能提升计划，21%的受访者目前正在制定，7%的受访者甚至不知道他们的组织是否有技能提升计划。然而，31%的受访者表示，他们的公司已经在其组织内实施了正式的技能提升计划。*
*   ****敏捷、DevOps 和 ITIL 正面临来自现场可靠性工程(SRE)的强劲竞争:*** 自 2019 年基准报告以来，敏捷采用率(81%)、DevOps 采用率(75%)和 ITIL 采用率(25%)均有所增长，而 SRE 的采用率已从 2019 年的 10%上升至 2020 年的 15%。价值流管理(19%)和系统思维(13%)等其他理念也在被利用。许多这些规程和/或框架在组织内共存。*

***介绍 DevOps 的“E 形”人类***

*2019 年[报告](https://u7061146.ct.sendgrid.net/ls/click?upn=4tNED-2FM8iDZJQyQ53jATUWOCZJAfw6mgVqizRpj2tzemlIU-2FZ-2FefHlsj7e77Nv4k5lXIdWZLW5-2B65qsO2uy-2FFw-3D-3DjlJ__4wMuA-2F3DC3uSAmQL6V-2BkQu0afFz1GruHJkIzmAji2oFWOkivY6H-2Fp9T3bCOJ3ooKRz9vTLvGBrxu4NSl8wWP04mJ78cyl5HVOh9GNSav5uASOyF69uYH90gzZulBSPOtIqff85Xo3N2vnElvejANezh7qgSp3tajOjuAPVquxLBnusAeIcL7WCPW7dDsr0rzmx0fUt0fXNtX9hSEnmrKNGD3mTiBcZogXyNI5bSeuB6J4VjlgLmciSfuFFecWQWdb2EaVmz-2BCogkfOF1ZASw2Z-2BurG1rjOP-2F1dpB-2F7jHa36AQN0WOyfz0lhOsW9jsLvEAHWrbbd3ugFDpoAmk12SIw-3D-3D)中强调的 DevOps 专业人士的“T 型”人类必须进化为“E 型”专业人士——根据 Sarah DaVanzo 的说法，这一想法“青睐具有‘广度’和‘深度’知识以及有形和无形技能的个人，这些技能意味着作为从业者的大视野和对细节的关注。“E 型人”拥有“四个 E”的组合:经验和专业知识，探索和执行。”*

*   *根据 2020 年技能提升调查的最新数据分析，“E 形”DevOps 人力由以下技能类别组成:三个横向技能类别，包括自动化技能、功能知识技能和技术技能。*
*   *第二组技能——过程和框架技能——这是一组垂直技能，侧重于流程和对不同实践和方法的理解，如 Scrum、敏捷开发和价值流映射。*
*   *另一个垂直技能类别侧重于人类技能，如协作和人际交往技能。*

*Eveline Oehrlich 说:“现在是提升您的 DevOps 团队和个人技能的时候了，但是，这必须超越技术和职能技能。“我们在 2019 年的研究中已经看到了对各种人类必备技能的巨大需求，今年我们看到了所有人类技能的巨大增长，例如协作、人际交往技能、同理心和创造力等。不过，最重要的是，在我们的调查中，来自管理层和商业领袖的人的技能的价值增加了。我们的研究表明，随着从“软”技能向“人”技能的转变，思维方式正在发生转变，但更重要的是，今天的领导者必须改变思维方式，认识到人的技能将给团队和组织带来的价值。”*
