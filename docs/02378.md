# Instana 为 Red Hat OpenShift 发布自动应用程序监控代理操作员

> 原文:[https://devo PS . com/instana-releases-automatic-application-monitoring-agent-operator-for-red-hat-open shift/](https://devops.com/instana-releases-automatic-application-monitoring-agent-operator-for-red-hat-openshift/)

*新运营商为 Instana 的全栈 Kubernetes 应用性能监控解决方案提供部署灵活性*

波士顿红帽峰会——2019 年 5 月 8 日——针对容器化应用的自动[应用监控](https://instana.com/application-monitoring)解决方案的领导者 Instana 正在通过其作为 Instana 代理操作员的第一个代理技术版本，使 DevOps 团队更容易管理编排的应用，并已获得 Red Hat OpenShift 操作员认证。这种新的部署选项为用户提供了部署灵活性，用户也可以将代理作为自己的容器、主机进程或微服务来运行。

Instana 联合创始人兼首席运营官 Pete Abrams 表示:“Instana 运营商让公司部署和运营他们的微服务应用变得更加容易和高效。“Instana 将监控代理作为操作员部署在 Red Hat OpenShift 上的能力允许 DevOps 团队利用 Instana 的能力，在行业最全面的企业 Kubernetes 平台 OpenShift 上自动监控配置和应用根本原因分析。”

Instana 面向微服务的自动 APM 解决方案是唯一一款自动化应用监控解决方案，可自动提供全栈可见性，并收集每个用户请求的端到端跟踪。该解决方案不仅能识别容器和编排的存在，还能自动为这些基础设施组件部署监控传感器。

与传统的监控解决方案不同，Instana 的自动 APM 解决方案可发现应用基础设施和服务组件，为应用的技术堆栈部署监控传感器，并跟踪所有应用请求，无需任何人工配置，甚至无需重启应用。该解决方案实时检测应用环境中的变化，调整自己的模型，并在几秒钟内可视化变化和对性能的影响。

Red Hat 云平台合作伙伴生态系统总监 Julio Tapia 表示:“Kubernetes 运营商之所以有吸引力，是因为它们有助于对管理运行在 Kubernetes 原生应用上的服务所需的人工操作逻辑进行编码，并旨在简化日常操作。“我们很高兴 Instana 代理操作员获得了 Red Hat OpenShift 操作员认证。通过在 Red Hat OpenShift 上提供操作员，用户可以开始体验 Kubernetes-native 基础设施的下一级优势，其服务旨在跨 Kubernetes 运行的任何云“工作”

Instana 的[微服务监控](https://instana.com/apm-for-microservices)功能包括深入了解几乎十几种不同的语言和 100 多种不同的微服务应用基础设施技术，包括 Docker、Kubernetes、Spring Boot、NGINX、AWS 等等。对每个技术组件的监控包括相关的应用程序级可见性，以及与跟踪中的每个请求相关的信息。

对应用和基础设施的关联监控扩展到 OpenShift Kubernetes 发行版，提供了对关键任务应用、其依赖关系及其性能的自动化可见性。

Instana 将在波士顿红帽峰会(310 号展位)上展示其针对 OpenShift 应用的 [APM 解决方案，这是 Instana 年度 Kubernetes 路演的第二站。请访问 www.instana.com](https://www.instana.com/supported-integrations/openshift-monitoring/)，了解有关该公司自动 APM 和应用基础设施监控解决方案的更多信息。

Red Hat 和 OpenShift 是 Red Hat，Inc .或其子公司在美国和其他国家的商标或注册商标。
