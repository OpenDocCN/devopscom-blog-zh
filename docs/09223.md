# 远程开发运维团队的基本要素

> 原文:[https://devo PS . com/essential-ingredients-for-a-remote-devo PS-team/](https://devops.com/essential-ingredients-for-a-remote-devops-team/)

企业业务流程运行在不断发展的应用程序上，DevOps 团队需要和他们创建的应用程序一样高效。不幸的是，新冠肺炎疫情对 DevOps 的生产力产生了深远的影响——这种影响并不总是积极的。

在一个遥远和混合的世界里，跟踪 DevOps 团队的表现变得更具挑战性。像其他人一样，开发人员已经成为遍布疫情的远程工作者，这意味着要适应新的工作方式。关于在家工作对生产力的影响，已经有很多研究了。当然，衡量任何开发人员的生产力已经是一项复杂的任务。好像你不能使用一个简单的度量标准，比如每天产生的代码行数；比这复杂多了。

在家工作使得开发人员保持高效工作变得更加困难。当你在家工作时，会有很多干扰，研究表明，一旦干扰被打断，即使是最熟练的开发人员也需要 15 到 20 分钟才能恢复工作。远程工作时，平衡工作和家庭生活的压力也更大。据 [催化剂](https://www.catalyst.org/) 称，92%的员工表示他们因远程工作的压力而感到精疲力竭。

### 软件开发在远程工作中蓬勃发展

在许多方面，远程工作的趋势对开发人员来说是一笔意外之财。编码的本质总是有助于混合或远程工作环境，但对保护知识产权毫无根据的担忧总是一个障碍。

最近的一项 Gartner 调查 显示，如果员工在工作地点、方式和时间上有灵活性，他们成为高绩效员工的可能性会增加三倍。远程工作还提高了灵活性，消除了通勤时间的浪费。在家工作减少了照顾孩子的责任。Catalyst 发现，32%有育儿问题的女性如果可以选择在家工作，就不太可能辞职。

远程工作还消除了地理障碍，这是软件行业众所周知的优势。能够在世界任何地方雇佣在家工作的程序员增加了人才库的规模，也更容易找到拥有合适技能的开发人员。

当然，采用远程工作模式的开发运维团队也面临着挑战。沟通可能是最大的挑战。缩放会议可能不如面对面会议有效，因为它更难读懂面部表情和捕捉非语言线索。由团队领导或高级经理来促进有效的沟通。拥有合适的工具、合适的沟通渠道以及保持相互尊重也很重要。即使在远程工作环境中，团队动力也会受到职责滥用、行为问题和不尊重行为的不利影响。

### **建立远程开发团队**

支持混合工作环境中的协作需要新流程、新指南和新工具。最好的起点是选择正确的协作平台和流程。例如，电子邮件不是即时请求的好渠道，但是你不想回答连续的电话。建立使用电子邮件、聊天、电话会议和会议的协议。采用云通信工具也给了你更多的灵活性和可伸缩性。

选择合适的协作工具时，请考虑安全性。异地工作会降低您对企业访问的控制力。远程员工使用不同的设备以及公共和私有网络访问企业系统。为了减少易受网络攻击的脆弱性，提供具有多个安全层的安全可靠的网络访问。采用虚拟专用网络(VPN)、远程桌面协议(RDP)客户端和恶意软件防护。员工在远程工作时往往缺乏安全意识，所以尽可能采取措施降低风险。

除了拥有合适的工具，您还必须拥有合适的方法。为了不断提高效率，你需要透明度、检查和适应，就像你使用 DevOps 一样。首先为远程团队建立并记录一套清晰的期望。此外，建立一个领导阶层。即使他们在家工作，每个人都需要一个工作结构。创造一个稳定的工作环境取决于团队领导。缺乏明确性会消除结构感和安全感，并会破坏团队。

保持透明是任何远程工作文化的最佳方式。建立一个以诚实和开放为动力的工作场所，在这里每个声音都能被听到，确保每个人都在努力实现一个共同的目标，并保持高昂的士气。

维持一个成功的远程工作环境取决于尊重四个 c:沟通、协作、连接和文化。这四个 c 相互依存，共同创建一个有效的数字化工作环境，提供跨地点、设备、渠道和情况的员工体验。

### **创造高效的环境**

一旦为远程协作奠定了基础，下一步就是监控工作效率。有几种方法可以考察 DevOps 团队生产力。您可以查看整体工作的最终输出，评估作为最终产品的应用程序。您还可以衡量整个团队的生产力。

为了监控评估团队生产力的关键指标，项目经理必须准备好深入评估热点:

*   度量特定开发人员的产出，并查看它与其他团队成员的生产率相比如何。
*   维护排行榜，突出排名靠前的贡献者。使用游戏化可以创造良性竞争，当顶级表现得到认可时，可以激发更高的生产力。
*   通过提供一套合适的工具并激励工程师发挥最佳水平，创造一个有利的环境。
*   监控分心的事物。编码是一项密集的活动，中断会损害生产力。

不要根据编码来衡量生产力。开发人员的贡献还包括设计新功能、功能讨论、代码审查、文档、测试和错误修复、与客户会面以及其他任务。

提高 DevOps 生产力也是自动化的产物。除了简化与即时消息和微软团队等工具的实时协作之外，通过自动化测试工具和人工智能测试框架缩短开发时间，还可以消除手动流程，腾出时间来完成更高效的任务。

如果你在疫情寻找一线希望，考虑一下远程工作环境的发展。支持远程 DevOps 开发人员并不是什么新鲜事，但是强迫员工在家工作已经使得混合工作环境更加高效。DevOps 团队继续发明更好的工作方式，促进灵活的工作时间、更高的生产率和更好的工作生活平衡。
