# SmartBear 用 LoadNinja 重新发明了 Web 应用程序的负载测试

> 原文:[https://devo PS . com/smart bear-reinventies-load-testing-for-web-applications-with-load ninja/](https://devops.com/smartbear-reinvents-load-testing-for-web-applications-with-loadninja/)

*SmartBear 发布了开创性的技术，将脚本创建时间减少了 50%，用真正的浏览器取代了模拟器，并为敏捷团队配备了重要的指标，以忍者的速度诊断问题*

马萨诸塞州萨默维尔。–2018 年 10 月 29 日–团队软件质量工具的领导者 SmartBear 发布了 LoadNinja，这是一个基于云的平台，面向加载测试 web 应用程序的工程师和性能专业人员。LoadNinja 提供了效率、准确性和可操作的性能数据的罕见组合，允许团队在敏捷和 devops 环境中轻松合并负载测试。与传统的基于协议的负载测试平台不同，LoadNinja 允许工程师将更多的精力放在构建可扩展的应用程序上，而不是构建负载测试脚本。

SmartBear 性能测试和监控副总裁 Anand Sundaram 表示:“应用性能对于确保数字业务的持续成功至关重要。“由于使用传统工具获得正确的负载测试脚本需要不合理的时间，因此团队很难对他们的应用程序进行性能测试。LoadNinja 消除了这种手动工作，帮助您使用真实的浏览器生成最真实的负载，并提供关于压力条件下 web 应用程序行为的可操作性能数据。”

传统的基于协议的工具具有陡峭的学习曲线，并且需要对记录的脚本进行冗长的编程，然后才能回放。LoadNinja 的 InstaPlay 记录器消除了这一步骤，因此录制的脚本可以立即播放。这种方法带来的效率提升，尤其是当负载测试在敏捷和 devops 环境中频繁运行时，使团队能够将精力集中在负载测试上，并节省了之前在负载测试之前创建和修改脚本所花费的时间。

与使用供应商开发的专有浏览器模拟器的传统工具所产生的负载不同，LoadNinja 中的 TrueLoad 技术使用真实的浏览器进行大规模负载测试，在支持测试中的 web 应用程序的基础设施上创建最真实、最准确的负载表示。

LoadNinja 返回的性能数据也显示了使用真正的浏览器生成所有负载的强大功能。传统上，负载测试工具在测试结果中返回请求-响应时间，需要进一步内省和解释，以便开发人员和性能测试人员可以使用。LoadNinja 省去了这个额外的步骤，直接以基于浏览器的导航计时的形式提供可操作的数据，这是开发人员和性能测试人员每天都习惯使用的。

LoadNinja 还以 VU 检查器和 VU 调试器的形式提供了另外两项创新。VU 检查器允许您在虚拟用户对应用程序产生负载时可视化降级。当虚拟用户遇到问题时，可以使用 VU 调试器来连接出错的虚拟用户的浏览器并与之交互，以收集关于该问题的附加诊断信息。
