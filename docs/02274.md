# Humio 宣布增强企业级控制和可见性

> 原文:[https://devo PS . com/humio-announces-enhanced-enterprise-grade-controls-and-visibility/](https://devops.com/humio-announces-enhanced-enterprise-grade-controls-and-visibility/)

*更新的平台在实时和无约束测井中提供了新的特性和功能*

圣何塞，2019 年 6 月 11 日 Humio 是一款专门构建的实时可观测性解决方案，它不受平台限制，提供快速、高效和易于扩展的日志管理，今天宣布了一系列旨在帮助企业用户扩展系统数据的交互性和可见性的新功能。Humio 通过丰富的功能增强了其强大的日志管理平台，帮助数据驱动的开发人员、运营人员和安全专业人员在不增加复杂性的情况下无限制地记录日志，并提供无限制的接收价格。

“企业正在各种类型的环境中运营，整合传统的整体应用，过渡到新的微服务堆栈，并通过 Kubernetes 实现云原生。Humio 首席执行官 Geeta Schmidt 表示:“这种复杂程度要求我们能够利用单一解决方案实时监控整个环境，从而为 DevOps 团队提供所需的观察能力，以发现和解决问题，无论他们身在何处。“更新后的平台在高可用性和集群管理 UI、基于角色的访问控制、更快的正则表达式引擎以及支持领先日志记录实践的其他技术等领域实现了创新。凭借这些新的企业级功能，我们很高兴能够为客户提供改进的功能，轻松管理我们快速、可扩展的平台，从而有效地记录一切，不遗漏任何内容。”

新功能包括:

*   高可用性——Humio 提供了整个数据路径的高可用性，从解析和消化到存储数据。高可用性使管理员能够通过群集节点管理用户界面轻松配置冗余。为了补充存储规则，该公司引入了摘要规则，允许管理员轻松指定哪些摘要节点彼此冗余。
*   主/辅助存储–可以将 Humio 配置为具有主存储和辅助存储，也称为热/冷存储。这种本地功能允许 Humio 管理员为旧数据配置和利用低成本存储技术，并仅将主存储用于最新数据。这样做是为了从主存储中获得最佳的查询性能。
*   基于角色的身份验证和访问控制—基于组成员资格，一个组可以拥有许多权限以及对特定存储库和视图的访问权。这允许管理员为 Humio 的用户分配各种权限和访问权，这可以通过权限文件或管理 UI 来完成。它还允许从 SAML、LDAP 或 Auth0 读取组成员，简化了大量用户的管理。
*   删除事件 API–可以通过本地和云客户均可使用的[删除事件 API](https://www.humio.com/blog/2019-10-4-delete-events/) 从 Humio 存储中删除单个事件，从而使用户能够删除无论出于何种原因都不应再出现在日志中的数据。访问此功能需要管理员分配的特殊权限。
*   事件背景–Humio 改进了用户处理事件的方式。该平台支持显示给定事件的事件上下文的能力，支持在原始上下文中查看导致该事件的日志条目。通过简单地右键单击给定的事件，用户现在可以在其原始上下文中看到它。
*   无限滚动——通过用户界面，可以滚动到所需的最远位置，查询结果不再局限于前 200 个事件。这与新的事件上下文功能相结合，为用户检查日志文件提供了一种强大的方法，现在所有客户都可以使用这种方法。
