# 掌握现代云原生企业的 API 网关

> 原文:[https://devo PS . com/mastering-API-gateways-for-the-modern-cloud-native-enterprise/](https://devops.com/mastering-api-gateways-for-the-modern-cloud-native-enterprise/)

随着现代企业不断向云迁移，以及对互联应用和设备的需求急剧增加，API 驱动的架构正被广泛采用。除了能够高效连接和共享资源的便利之外，组织还受益于 API 在多云环境中带来的创新和加速开发速度。API 并不新鲜，但是随着越来越多的企业采用现代应用架构，它们越来越多地占据了软件开发团队的大量交付工作。

当企业实现现代化并迁移到云时，挑战自然会出现，需要在它们成为棘手问题之前加以解决，尤其是当它涉及复杂的微服务环境时。API 网关是帮助企业克服许多与 API 相关的挑战的基本要素。API 网关部署在基础设施的边缘，负责将客户端 API 请求路由到后端[微服务](https://containerjournal.com/?s=microservices)。随着团队和基础设施的同步发展，现代 API 网关还带来了额外的好处，组织可以从中受益。

让我们深入了解一下组织在 API 方面面临的一些主要挑战，以及如何最大化它们的优势。

## 快速采用遇到成长的烦恼

面向 API 的架构大大提高了 IT 部门的灵活性，因为它们已经成为现代应用架构的一种众所周知的模式。然而，这种方法可能是欺骗性的，因为许多关于 API 的关键问题并不总是从一开始就显而易见。如果没有正确地识别它们，它们就会成为团队每天都要面对的成长中的烦恼。

第一个挑战是当用户发出请求时，需要对他们进行身份验证。API 网关允许组织使用他们选择的协议来集成和整合身份验证提供者。借助计费和分析，组织需要了解处理交易时消耗的资源数量，以及他们的客户如何使用他们的 API。API 网关成为收集这些数据的自然点。

在 DevOps 计划方面，一些 API 网关使用服务发现来动态配置自身，这大大加快了开发和运营团队在发布新功能或修复时的速度。为了帮助团队交付高可用性和弹性平台，API 网关是使能器，为他们提供高级负载平衡、速率限制、断路器、镜像等拼图。这些因素使 DevOps 思维成为现实。团队可以根据需要对其进行扩展，以实现生产级环境，不断根据软件变化更新部署，并避免任何无意的单点故障(SPOFs)。

近年来，API 网关的发展解决了云中的成长难题，帮助组织利用云原生技术和工作流。将云原生技术与 API 主导的集成平台相结合，为数字企业创建了一个有效的架构来提高生产力。自动化生产、运营和服务不再是一个目标，而是必须具备的。

这个环境中的现代 API 网关特性远远超出了人们的预期。API 网关可以成为部署策略转换的基石。随着数百个(如果不是数千个的话)服务的持续部署，引入平台回归的风险会激增，并且能够立即解决这个问题是不够的。API 网关可以帮助您过渡到现代部署策略，蓝/绿或淡黄色部署变得司空见惯—帮助部署成为一项易于管理的日常任务。

## API 网关拯救云原生架构

作为向现代应用架构转变的一部分，企业应该投入资源来实施 API 策略，作为其软件开发计划的一部分。反过来，他们应该利用 API 网关来减轻成长的烦恼并简化交付工作。API 应该得到保护、管理、遵守，甚至货币化。

API 网关还可以帮助解决与身份验证、安全性和 SSL、计费和分析、生命周期管理、高可用性和弹性相关的关键问题，它们甚至可以帮助您启动向开发运维心态的文化转变。通过将 API 驱动的架构和云原生技术相结合，组织可以为数字时代提供有效的架构。API 网关有助于克服挑战，展现当今以云为中心的世界中错综复杂的网络。
