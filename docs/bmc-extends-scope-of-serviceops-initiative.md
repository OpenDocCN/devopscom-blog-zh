# bmc 扩展服务范围计划

> 原文：<https://devops.com/bmc-extends-scope-of-serviceops-initiative/>

在本周举行的 [BMC Exchange](https://exchange.campus.bmc.com/#lct_last=lobby&lct=entrance) 会议上，BMC 透露[已经更新了其 Helix IT 服务管理(ITSM)平台](https://www.bmc.com/newsroom/releases/bmc-helix-saas-delivers-serviceops-and-enterprise-orchestration-to-accelerate-innovation.html)，以包括地图绘制和发现功能。这些更新使跨职能团队能够更轻松地关联事件、警报和数据，并更快地查明根本原因。

此外，随着指标、事件和警报数量的不断增加，BMC 现在正在将预训练的人工智能(AI)模型应用于[可观察性](https://devops.com/?s=observability)数据及其跟踪的服务模型，以评估服务运行状况、预测服务影响并主动管理 it 环境。

最后，BMC 还宣布，BMC Helix Control-M 编排引擎现在支持开源的 Apache Airflow，这是一个使用 Python 编程工具定义数据管道的工作流管理平台。

BMC 副总裁兼数字服务和运营管理总经理玛格丽特·李表示，这些更新扩展了该公司去年推出的统一 ServiceOps 计划，以统一 ITSM、DevOps 和客户服务台工作流。虽然这些领域中的每一个都可能有不同的目标，例如速度和可靠性，但这些目标往往相互冲突，Lee 说，很明显，it 组织需要减少当前在管理 IT 时经常遇到的摩擦。

作为一个软件即服务(SaaS)平台，Lee 表示，BMC Helix 平台通过一个类似消费者的用户界面来实现这一目标，该用户界面可以轻松实现个性化。她补充说，我们的目标是简化虚拟代理、知识库、实时聊天和票证的创建，使客户和员工能够轻松请求 it 帮助。

然而，实现这一目标将需要 IT 组织更加依赖人工智能模型来自动化日常任务，因为 IT 环境变得更加复杂。她指出，根本没有足够的 IT 专业人员来管理使用传统流程的 IT 环境。

此外，最终用户和客户希望服务请求能够自动化，从而允许他们通过移动设备解决问题，Lee 补充道。在许多情况下，现在 IT 流程的繁琐程度会直接影响客户和员工的忠诚度。主要目标是减少完成某项任务所需的交互次数，例如在需要新员工入职时更改密码或配置笔记本电脑。

虽然 IT 管理的发展速度会因组织的不同而有很大差异，但很明显，许多 DevOps 原则正在被吸收到传统的 ITSM 平台中。挑战在于开发运维团队和集中式 IT 团队之间的文化差异仍然很大。大多数组织仍在试图确定他们希望将责任转移给应用程序开发团队，还是继续依赖 IT 管理员使用图形工具集中管理 IT 工作流。这很少会是一个要么全有要么全无的命题。相反，问题是找到一种平衡的方法，使组织尽可能地敏捷，而不会给弹性带来不可接受的风险。
