[![Continuous-Delivery-Infographic-Image](../Images/2983c6d6acf2128fe1b0321bf3eea693.png)](http://blog.zend.com/2014/03/11/companies-investing-in-devops-continuous-delivery/#.UydID_ldXjg) 本周，DevOps 和持续交付领域出现了一些好东西，专家称这将是 2014 年的一个主要支出领域。

首先是[devopsgoys](http://blog.devopsguys.com/2014/03/17/why-companies-are-investing-in-devops-and-continuous-delivery/)带着一点评论转发了这张来自 [Zend Technologies](http://blog.zend.com/2014/03/11/companies-investing-in-devops-continuous-delivery/#.UychRsJOWbw) 的信息图。它展示了开发运维及持续交付的真正价值。我注意到的一件事是，根据数据，99%接受调查的公司现在认识到比以前更需要 DevOps 战略。Zend 利用来自各种可靠行业来源的数据制作了这张图片。

当然，你不能在不提及 Jez Humble 的情况下谈论[连续交付](https://continuousdelivery.com/)，毕竟他确实就此写了一本书。他最近在他的博客上发布了 Nhan Ngo 的四个可视化效果，Nhan Ngo 是 T2 Spotify T3 的质量保证工程师。这四个可视化是在阅读 Jez 的书《持续交付》时创建的。一些相当激烈的东西。Nhan Ngo 让它看起来几乎很简单，就像一个连环漫画，但没有什么好笑的。

如果你真的想进入战壕，这里有一个有趣的视频，我发现所谓的连续交付和网站夜间黑客零停机时间。场景是你的“持续集成系统运转良好，并在每次提交后开始运作。您的代码已编译。你的测试是绿色的。您觉得已经准备好将您的项目带到下一个级别，并转移到持续交付。”

我们对 DevOps 和持续交付的研究还没有结束， [DevOps 反应](http://devopsreactions.tumblr.com/post/78637209483/spreading-additional-load-on-already-busy-servers)可能最能说明持续交付是如何工作的。*这在模型中行得通，但在现实世界中需要真正的技巧。*
