# Exadel 推出 Adobe Experience Manager 创作工具包，作为面向数字营销社区的开源工具

> 原文：<https://devops.com/exadel-launches-adobe-experience-manager-authoring-toolkit-as-open-source-tool-for-digital-marketing-community/>

*AEM 创作工具包让任何人都可以利用新的 Adobe 创作功能*

加利福尼亚州核桃溪，2019 年 9 月 10 日——数字软件工程解决方案的全球领导者 Exadel([www.exadel.com](http://www.exadel.com))今天宣布推出其 Adobe Experience Manager (AEM)创作工具包。作为一个由 Exadel 数字营销技术团队开发和维护的开源项目，该工具包作为一个打包的、完全受支持的解决方案提供给 Exadel 的企业客户，并作为一个开源工具提供给社区。该工具包支持为 AEM 作者自动生成通用且可扩展的 UI 元素，并在 Adobe 的 Coral UI 驱动的环境中提供下一代创作体验。

Exadel Solutions 总裁 Lev Shur 表示:“基于开源的产品开发从第一天起就是 Exadel 的核心工作，我们很高兴能够将这种奉献精神扩展到我们的 AEM 创作工具包。“Adobe 显然是数字营销解决方案的领导者，但跟上 AEM 生态系统的步伐可能具有挑战性。该工具包为 AEM 创作体验带来了速度、可靠性和自动化，帮助客户和用户最终创建更强大、更健壮和更漂亮的网站。”

AEM 创作工具包:

*   基于添加到后端 Java 类和/或其字段的注释，生成丰富多样的 AEM 组件对话框和就地编辑工具。
*   将 AEM 开发人员从为创作界面编写复杂的 XML 标记中解放出来，防止难以跟踪的错误。
*   允许跨组件“继承”某些特性和标记特性，减少了要存储和维护的 Java 代码量。
*   支持重用和微调可用的创作小部件，并重新定义它们的属性。
*   几乎不需要 Java 代码重构。它还减少了开发 AEM 安装的“创作”方面的 UI 团队工作。
*   通过注释值和 Maven 配置高度可配置。
*   可在任何操作系统中与 AEM 6.4 及更高版本的 JDK 1.8+兼容。
