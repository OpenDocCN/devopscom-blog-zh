# Sonatype 报告显示供应链攻击激增

> 原文:[https://devo PS . com/sonatype-report-shows-spike-in-supply-chain-attacks/](https://devops.com/sonatype-report-shows-spike-in-supply-chain-attacks/)

Sonatype 今天发布了一份报告,发现针对上游公共存储库的供应链攻击同比增长了 650%。根据该报告，网络罪犯希望通过将恶意软件注入许多组织可能正在使用的软件组件来破坏这些存储库。

第七份年度软件供应链状况报告基于对上个月完成的 100，000 个应用程序和 400 万个组件迁移的分析，以及对 702 名软件工程专业人员的调查。具体来说，该报告重点关注与 Java (Maven Central)、JavaScript (npmjs)、Python (PyPI)和。Net (nuget)生态系统。前四大开源生态系统现在总共包含 37，451，682 个不同版本的组件，同比增长近 20%。

在 2021 年期间，开发者还从四大生态系统下载了超过 2.2 万亿个开源包，比上一年增长了 73%。

然而，尽管有大量可用的[开源](https://devops.com/?s=open+source)软件，这种软件的使用主要集中在少数流行的项目中。生产应用程序只使用了 6%的可用开源项目。

毫不奇怪，受欢迎的项目更容易受到损害。共有 29%的流行项目版本包含至少一个已知的安全漏洞。相反，在没有广泛使用的项目中，只有 6.5%的项目存在漏洞。

Sonatype 产品创新副总裁 Stephen Magill 表示，流行的开源项目往往有更好的安全实践，因为有大量的维护者和贡献者在项目上工作。Magill 说，矛盾的是，不太受欢迎的开源软件不太可能成为网络犯罪分子的目标，他们往往专注于最有可能感染下游应用程序的目标。报告发现，流行的开源项目包含漏洞的可能性是其他项目的 2.8 倍。

平均更新时间(MTTU)更快的项目通常也更安全。调查发现，他们有漏洞的可能性低 1.8 倍。

该报告还发现，开发人员在更新第三方依赖关系时，有 69%的时候会做出次优选择，即使项目的新版本已经普遍可用。开发人员可能会避免这些更新，因为他们可能认为这些更新不像他们希望的那样稳定。但是，这些更新通常包含解决最近发现的漏洞所需的最新安全补丁。该报告发现，商业工程团队只管理他们使用的 25%的组件，使他们的大多数开源依赖变得陈旧，容易受到安全风险增加的影响。

最后，该报告提出，对自动化有更大的需求。配备了智能自动化，一个拥有 20 个应用程序开发团队的中型企业每年将节省总共 160 个开发人员工作日。Sonatype 估计这可以为组织每年节省 192，000 美元。

总的来说，Magill 指出，在拜登总统发布行政命令之后，软件供应链的安全性应该会继续改善。然而，与此同时，大多数组织最好尽早开始审查组成其软件供应链的组件和依赖关系。毕竟，网络罪犯很有可能已经在做同样的事情了。
