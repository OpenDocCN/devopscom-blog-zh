# Puppet 宣布推出 Puppet Enterprise 2017.2，提供库存发现和可视化流程编排工作流

> 原文：<https://devops.com/puppet-announces-availability-puppet-enterprise-2017-2-delivering-inventory-discovery-visual-orchestration-workflows/>

**Continued Innovation Provides Enhanced Situational Awareness to Deploy Software Faster**11 May 2017, Portland, OR

[Puppet](https://puppet.com/) ，为我们周围的一切提供动力的软件自动化交付和操作的标准，今天宣布 Puppet Enterprise 2017.2 的正式上市，以及 Puppet 5 平台的即将上市，这是开源 Puppet 的最新版本。Puppet Enterprise 2017.2 的重大更新现已推出日文版，使客户能够通过可视化工作流在其软件中编排更改和分阶段部署，并更好地了解正在运行的软件包，无论它们是如何管理的。此外，Puppet 的核心开源平台 Puppet 5 平台包括升级功能，为开源客户提供一种更强大的通用语言来交付和操作他们的所有软件。

“今天的组织面临着越来越大的压力，要更快地交付更好的软件。对于企业 IT 而言，这意味着从每隔几个季度发布一次更新，转变为按需交付优秀的软件，这是一个重大转变，”Puppet 首席产品官 Omri Gazitt 说道。“木偶有助于这种转变。Puppet 为企业 IT 提供了开发灵活性、可预测性和安全性所需的现代自动化。这一版本使 it 部门比以往任何时候都更容易开始自动化，从了解不受管理的基础架构，到更好地控制软件变更的交付。”

Puppet Enterprise 2017.2 的重要新功能包括:

*   **包检查:**用户通过快速识别没有被自动管理但应该被自动管理的包来开始自动化。借助 package inspection，Puppet Enterprise 用户可以报告未受管理的基础架构，从而轻松查看最能从自动化中获益的系统，以确保其最关键的软件拥有正确的维护更新、安全补丁和许可证。
*   **可视化编排工作流程:**用户完全在 Puppet Enterprise web UI 中编排基础设施和应用程序部署，无需编写任何 Puppet 代码。企业可以使用下拉菜单和过滤器来定位特定的基础架构环境，定义工作流，实时监控部署状态，并根据需要启动、停止或重新启动部署，所有这些都只需单击一个按钮即可完成。

今天还宣布了 Puppet 最新的开源创新，包括[LUMO gon](https://lumogon.com/)和 Puppet 5 平台。该公司的核心开源产品 Puppet 5 平台的更新，包括 Puppet Agent、Puppet Server 和 PuppetDB，使用户更容易下载、实施和升级他们的 Puppet 部署。Puppet 5 平台的组件套件保证能够协同工作，而不需要其他开源工具通常需要的额外测试和故障排除。

Puppet Enterprise 2017.2 现已面向新老客户发售，Puppet 5 平台将于 2017 年 5 月上市。
