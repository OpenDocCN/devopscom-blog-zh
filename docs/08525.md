# IBN 的成功源于制定正确的测试验证策略

> 原文:[https://devo PS . com/ibns-success-is-driven-by-formulating-the-right-test-validation-strategy/](https://devops.com/ibns-success-is-driven-by-formulating-the-right-test-validation-strategy/)

基于意图的网络(IBN)被广泛认为是网络行业的下一个重大创新。IBN 旨在简化在任何网络环境中实施新升级和变更的过程，以确保结果符合最初的意图。IBN 可以被视为软件定义网络(SDN)的继任者，因为它寻求自动化网络运营，同时更好地将网络与业务意图相结合。

IBN 有助于通过基于意图的模型自动执行传统上由网络分析师或网络支持工程师执行的操作。这是通过创建一个抽象层来实现的，该抽象层通过特定的顺序指出要添加的服务或应用程序，从而自动化所有网络配置和管理活动。

## 基于 SDN 和 NFV 的网络带来的挑战

许多组织正在对云进行长期投资，并且刚刚开始认识到 NFV 和 SDN 的优势。因此，在转型完成之前，我们仍处于过渡阶段。随着企业继续进行这种转变，从 INB 架构的角度来看，在开始这一旅程之前，需要考虑几个关键因素:

*   在将 IBN 整合到网络架构中之前，分析您路线图中的每个联网项目，并确定是否可以将基于意向的联网添加到这些[项目](https://www.sdxcentral.com/directory/projects/)中。
*   确保建立健全和全面的监控和保证系统。
*   确定并检查网络架构中的应用编程接口(API ),确定它们是否适用于 IBN。例如，ODL/ONOS 控制器中的 Nemo API。
*   IT 网络团队应该拥有跨架构、数据中心、园区、分支机构和云平台的单一视图。
*   确保网络架构考虑了所需的必要[安全](https://www.sdxcentral.com/security/)元素。

## IBN 如何确保网络智能验证的整体方法

与传统的网络验证不同，我所指的智能网络验证能够考虑网络设计、配置和主要或现有的网络状态，以准确评估网络活动。它不考虑实时流量和预定义的测试场景，因此有助于验证负面场景，这是传统测试很少能完成的。

由于 IBN 有能力将整个网络视为一个单一的实体，这有助于全面的网络核查方法。因此，组织能够更有效地验证网络的行为，基于其配置和状态，具有更高层次的意图。

## IBN 要求网络设计和测试发生根本性转变

IBN 正在将网络 IT 模式从被动的方法转变为更加主动的软件驱动的方法，在这种方法中，对当前网络设计的自动分析几乎可以消除人为错误和错误配置。自动化验证有助于复制和增强 IT 人员在停机期间故障排除和问题分类、验证供应商提供的修复以及记录程序和工作流程方面的网络专业知识。

由于多厂商软件平台、协议和 API 在这种环境下协同工作，出现了新的测试挑战。

在设计新的测试方法之前，要采用的一个好的实践是深入研究设计 IBN 解决方案中的关键依赖项。但是，从现有网络迁移到虚拟网络时，两个网络都必须彻底重新验证。

## 设计交付价值的测试验证策略

要充分利用这些优势，很明显，SDN 和 IBN 必须整合在一起，充分发挥各自的潜力。因此，将测试方法集中在五个重要领域是至关重要的。

这些领域以及需要考虑的相关测试如下所示:

1.  **业务策略**:策略转换为技术和映射测试、服务设计和逻辑拓扑测试、用户名到主机映射测试和 REST API 功能测试。
2.  **自动化**:服务放置测试、服务功能测试、服务性能测试、策略验证测试(用户意图和策略变更验证)。
3.  **保证**:端到端监控测试、用户意图和资源使用性能测试、SLA 性能和服务配置文件测试(流量吞吐量、延迟、抖动)。
4.  **可编程网络基础设施**:南向 API 测试(NEMO API)、OpenFlow 测试、Ovsdb 测试、网管测试。
5.  **安全**:服务融合测试，一致性测试(IBN 客户规范一致性，意图漏洞报告)。

## 测试对于确保 IBN 自动化的成功至关重要

随着[网络](https://devops.com/accelerate-network-processes-for-devops-by-focusing-on-high-level-policy-requirements/)的不断转型以及网络工程师面临的相关技术复杂性，验证成为 IBN 在基于 SDN 的 IBN 网络架构中实现其目标的重要一步。虽然公司仍然允许软件完全驱动其 IT 升级，但专注于定义明确的测试方法对于提供成功实施 IBN 所需的可预测性和保证至关重要。

采用分层和集成的方法提供了与早期检测缺陷、通过自动化改进测试周期、结果的可预测性、信心水平的增加、增强的可扩展性、改进的安全指南和确保顺利交付的充分测试相关的显著优势。

毕竟，当意图是正确的，并且更重要的是得到验证时，所有的部分都聚集在一起来驱动业务敏捷性。
