# 降低云基础架构的复杂性

> 原文：<https://devops.com/reducing-cloud-infrastructure-complexity/>

云的影响及其在主流 IT 中的作用是不可否认的。根据 IDC 的数据，2023 年，全球[云 IT 基础设施支出预计将达到 920 亿美元(1260 亿新加坡元)。值得注意的是，云支出正在超过非云 IT 基础设施支出，凸显了云在现代 IT 部署中的影响力。](https://devops.com/?s=cloud%20infrastructure)

## 向云快速推进

为什么企业和创业公司都转向云？最重要的是它的可扩展性，以及它如何实现快速上市。云基础设施可以在几分钟内启动并投入使用，而不是等待几周或几个月的硬件系统。此外，托管云数据库等基于云的资源降低了部署新服务的复杂性，从而使较小的开发团队能够完成更多工作。

随着云的成熟，新的以云为中心的方法也应运而生。例如，基础设施即代码(IaC)使得通过配置自动管理和配置整个技术体系成为可能，取代了容易出错的手动部署。除此之外，IaC 还支持 CI/CD 部署和灾难恢复计划，能够轻松部署测试环境甚至整个基础架构堆栈。

然而，不同组织对云的熟练程度各不相同。对于大多数人来说，云计算之旅始于简单易行的迁移或相对简单的云实现，如网站或在线商店。其他企业可能已经在其组织内开始使用云原生应用程序，作为快速试验的一部分。

像大多数 IT 项目一样，云部署可能是由当今的技术实现的，并且是由更喜欢快速结果而不是效率的开发者和架构师实现的。尽管这种自由放任的方法产生了快速的结果，但它可能是低效的，并最终导致系统难以维护。随着时间的推移，这种情况会变得更糟，因为附加服务或新功能是在初始设计的基础上构建的。

## 云放大的复杂性

毫不奇怪，部署设计良好的云系统变得越来越困难。随着组织寻求以更高级的方式利用云，一些组织被迫重新设计现有系统，这使他们无法实现缩短上市时间的好处。

增加云复杂性的其他因素包括:

*   **云的快速发展:**公共云的飞速发展催生了前所未有的新云技术和战略。例如，无服务器架构和容器只是在最近几年才成为主流。
*   深厚的专业知识比以往任何时候都重要:尽管如今有更多合格的云工程师，但鉴于对他们的需求，专家仍然短缺。一个由“去过那里，做过那里”的专家组成的团队可以让一个组织更快地前进。
*   **更大的攻击面:**随着越来越多的系统和工作负载转移到云，IT 团队面临着越来越大的压力，要跟上急剧增加的攻击面。从发现配置偏差到修补技术堆栈中使用的软件库，拥有经验丰富的团队成员是非常宝贵的。

现代公共云平台提供了细微而广泛的功能，这些功能继续快速发展。利用正确的特性不仅可以在成本方面带来巨大的回报，还可以提供更好的可靠性和可维护性。相反，忽视良好的设计会导致云蔓延和云成本膨胀，使组织陷入阻碍新云实施的螺旋式复杂性网络中。

## 更好的云基础设施设计

最终，云之旅不会随着单个系统成功实施或迁移到云而结束。云计算的飞速发展意味着最佳实践和最佳策略在不断发展，这使得内部 it 部门很难确定。此外，随着新的云功能变得可用，企业有责任利用正确的功能来实现最佳的云解决方案，这需要不断优化。

要做到这一点，需要精通最新云实践和自动化策略的专家的参与，这些知识只能从经验、对最新云方法的深刻理解以及每个公共云平台不断发展的功能中获得。随着这些云专家专注于核心设计，内部 IT 可以继续做他们最擅长的事情:让用户满意，提高工作效率和用户体验。

例如，印度尼西亚的流媒体服务提供商 Genflix 希望将其视频点播(VOD)工作流迁移到亚马逊网络服务(AWS)。对于拥有 10，000 小时视频点播内容的库，效率和无缝处理至关重要。CloudCover 帮助 Genflix 在短短三个月内利用无服务器计算设计构建了一个端到端的媒体工作流。

无论您是从头开始部署新的云原生系统，还是希望改进现有的云部署，对您的云部署进行审查可能是合适的。这并不像看起来那么令人畏惧，可以通过简单的发现和分析阶段开始。
