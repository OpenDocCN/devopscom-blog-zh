# 英特尔扩大 OneAPI 工具包计划的范围

> 原文:[https://devo PS . com/Intel-expands-scope-of-one API-toolkit-initiative/](https://devops.com/intel-expands-scope-of-oneapi-toolkit-initiative/)

英特尔已经发布了 oneAPI 工具包的 2022 版[，通过该工具包，开发运维团队可以访问处理器扩展。](https://www.intel.com/content/www/us/en/newsroom/news/intel-empowers-developers-oneapi-2022-toolkits.html#gs.lm9gbk)

英特尔软件产品的首席宣传员詹姆斯·莱因德斯(James Reinders)表示，目标是让 DevOps 团队更容易在开发人员使用的运行时中暴露任何公司处理器的扩展。

在其核心， [oneAPI](https://devops.com/?s=OpenAPI) 工具包是英特尔为 x86、图形处理单元(GPU)、专用集成芯片(ASICs)、现场可编程门阵列(FPGAs)和其他类别的处理器创建的 API 集合。

Reinders 指出，开发人员以前从未像现在这样容易接触到这么多不同的处理器类别。然而，组织面临的挑战是让开发人员更简单地利用这些功能，而不需要他们编写低级代码，他补充说。

oneAPI 2022 工具包增加了额外的功能，包括世界上第一个实现 C++、SYCL 和 Fortran 的统一编译器，面向 CPU 和 GPU 的数据并行 Python，面向人工智能(AI)和光线跟踪可视化工作负载的高级加速器性能建模和调整以及性能加速。总的来说，英特尔表示，去年它已经通过 oneAPI 工具包提供了 900 项新的和增强的功能，包括更深入的微软 Visual Studio 代码集成，对微软 Visual Studio 2022 的支持以及微软 WSL2 在 Windows 上的 Linux 开发。

Reinders 说，随着它的不断发展，开发人员在同一应用程序中调用不同种类的处理器将变得越来越常见。他补充说，这些能力将推动软件开发创新在这个十年的剩余时间和未来爆发。

Reinders 说，英特尔特别关注让开发人员更容易调用推理引擎，这些引擎利用处理器扩展来运行人工智能工作负载。展望未来，大多数应用将利用机器学习算法，这些算法运行由英特尔通过扩展其处理器优化的推理引擎。Reinders 指出，随着这些工作负载的构建，在 DevOps 工作流中集成人工智能模型将成为下一个主要挑战。

现在说对异构处理器的简化访问(也称为 [XPUs](https://digitalanarchist.com/videos/featured-guests/the-rise-of-xpus-intel?) )可能会改变应用程序开发前景还为时过早。有一点很清楚，IT 组织现在比以往有更多的选择。现在的问题是找到一种方法来调用所有这些功能，而不会让开发人员不知所措。DevOps 团队将在决定向其开发团队提供哪些处理器和相关扩展方面发挥关键作用。

与此同时，对处理器主导权的争夺在未来几个月只会更加激烈。开发运维团队需要决定他们使用英特尔等单一公司生产的处理器的舒适程度，或者他们是否希望混合使用多家厂商提供的处理器。不管他们做出什么决定，看来 API 将在使处理器技术的进步更容易受益方面发挥关键作用。
