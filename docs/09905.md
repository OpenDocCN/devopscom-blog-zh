# 那只狗吃了我的一打 DevOps

> 原文:[https://devops.com/the-dog-ate-my-devops-dozen/](https://devops.com/the-dog-ate-my-devops-dozen/)

作为 Techstrong Group 团队的新成员，我对 DevOps 十几个奖项并不熟悉。基础知识:这是一个很酷的计划，它认可 DevOps 世界中许多类别的优秀。获胜者将在 1 月 12 日我们的[预测 23 会议](https://www.techstrongevents.com/predict2023)上宣布。我不得不告诉你，提交过程很有趣。T3】

许多提交的材料都是在最后一天提交的，这并不奇怪。有些是在最后一天的最后一小时。但令人惊讶的是，当公司不可避免地要求延期时，我们得到的一些借口。

没有人真的使用“狗吃了我们提交的东西”这个借口——主要是因为，嗯，这是一个在线提交过程，所以，除非狗吞下了一台电脑，否则这真的不再是一件事了。今年似乎诚实占了上风。我们有一群人说他们正在进行一个大型发布会；其他人说他们一直在为 KubeCon 做准备，并要求多一点时间。

我们有点同情，因为我们也在为 KubeCon 做准备，完全理解为一个重大事件做准备所需的所有人都在甲板上的推动。我们的支付处理器也遇到了一些问题，导致一些提交无法完成。我们已经解决了这个问题，并从经验中吸取了教训，所以明年不会有问题。

在类别方面，不出所料，吸引最多提交的前三名是“最佳 DevSecOps 解决方案”、“最佳端到端 DevOps 工具和服务”和“最佳新 DevOps 工具/服务提供商”。考虑到将安全性集成到 DevOps 环境中的趋势以及将各种 DevOps 工具集成到一个平台上的趋势，这两个类别的流行是意料之中的。

下一波类别包括“最佳可观察性解决方案”、“最佳云原生安全解决方案/服务”和“最佳 CI/CD 工具”，从而完成了前六名。同样，在第二组中，少一些战略，多一些战术。

如果你想去人不多的地方，那么最不拥挤的三个类别是“最佳开发运维相关视频系列(视频)”、“最佳开发运维转型(非供应商)”和“年度最佳开发运维演示”也许这些不是最性感、最值得关注的类别，但竞争是最小的。

那么，我们从今年的提交过程中学到了什么？尽量不要等到最后一分钟，最有竞争力的类别是那些处理热点问题的类别。请关注公众投票的开始，我们将在 Predict 23 期间的颁奖典礼上与您见面！
