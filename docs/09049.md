# 异构硬件需要通用软件

> 原文：<https://devops.com/heterogeneous-hardware-needs-universal-software/>

随着硬件的进步和多样化，我们正在进入许多人眼中的计算机架构的新黄金时代。然而，对于软件开发人员来说，不断扩张的硬件环境的想法可能会令人望而生畏，因为尽管硬件多样性对于价格和创新来说是好事，但它可能会导致后端软件复杂性的螺旋上升。如此多的软件工具专用于它们运行的硬件，以至于一个 CPU、GPU、FPGAs、ASICs、DSP 等自由混合的世界让人感觉势不可挡；互操作性问题和螺旋上升的技术开销的邀请。开发人员心中的问题是，如何在这个不断变化的生态系统中导航，以利用硬件加速的新世界，而不会在这个过程中让自己发疯。

开发人员面临的挑战是，如果我们回顾历史，每次一款改变世界的芯片出现，它通常都会带来一套新的软件工具。随着异构硬件创新以几乎不可追踪的速度发展，跟上软件端的步伐开始变得难以为继。作为回应，一种新的理念正在出现:我们应该能够拥有一套通用的工具、库、框架和编译器，能够以“硅不可知”的方式处理各种各样的加速器。

今天从事开发运维工作的任何人都应该问:“我们正在采用的工具为异构硬件的未来做好准备了吗？”我们应该考虑抽象层，它允许我们迁移到跨多个供应商和架构工作的开放软件堆栈，而不是使用单个供应商专有的加速概念。

这一理念是许多运动的核心，例如 [oneAPI](https://devops.com/intel-expands-scope-of-oneapi-toolkit-initiative/) 要求更多用途的库、编译器和分析工具。任何有未来意识的开发人员都有责任分析他们正在使用的工具，并询问他们是否在为未来做准备，或者将他们锁定在一个将你排除在其他创新之外的解决方案中。

在应对不断变化的开发环境时，关注多供应商尤为重要。计算和电子学的一个显著优势是它的互操作能力。如果平台之间不兼容，我们最终会被单一解决方案所束缚，无法利用我们可用的多样化创新前景。如果系统是从零开始构建的，并考虑到混合和匹配，我们不必改变我们的软件工具来适应新技术，我们可以使用我们习惯的相同的语言、库和分析工具。

我们现在着手这项工作非常重要，因为硬件多样性不仅达到了历史最高水平，而且其势头还在不断增长。在过去，当硬件环境更加有限时，为不同的 CPU 切换到不同的工具集是一种可行的解决方案，尽管这种解决方案不够优雅。然而，今天的计算机拥有来自多家厂商的计算能力。如果我的工具不能延伸到所有的生态系统，我很快就会陷入复杂的混乱。

超级计算是一个特别相关的例子，它是开发人员社区中最激动人心、发展最快的创新领域之一。超级计算最让你兴奋的是什么？它是异构的，允许多种架构为它们最适合的不同任务承担专门的硬件加速。例如， [Aurora 超级计算机](https://www.alcf.anl.gov/aurora)计划实现高达 2 exaflops 的性能——这是一个惊人的数字(我帮助设计的第一台超级计算机具有 1 teraflop——或 Aurora 的百万分之二的性能)。这些机器的惊人能力之所以成为可能，是因为异构内存技术、存储能力、复杂的 CPU 和 GPU 的进步，它们能够无缝地协同工作。

我们看到的超级计算机的进步，一如既往，最终将渗透到低级系统中。在不久的将来，不管你做什么类型的工作，巨大的计算能力即将上线。我们需要确保我们的库和工具能够在它到来时充分利用它。
