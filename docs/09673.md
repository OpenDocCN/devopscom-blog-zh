# 打破 5 个常见的数据库现代化神话

> 原文:[https://devo PS . com/busting-5-common-database-modernization-myths/](https://devops.com/busting-5-common-database-modernization-myths/)

现代化数据库是提高数据库和应用程序性能的最佳方式之一。然而，许多公司对做出重大改变犹豫不决，即使这些改变会带来更好的结果。为什么？因为有很多关于数据库 e 现代化项目未能达到预期的神话。 尤其是五大神话，似乎阻碍了领导人追求现代化的机遇。我们将在下面详细介绍这些神话:

*   误区一:现代 在云中存储数据库几乎没有性能优势
*   误解二:云数据库不会降低我的 IT 成本
*   误解三:出于安全性和可用性原因，我最好将数据留在本地
*   误解四:在云数据库中，我能做的事情没有多少是我不能在内部完成的
*   谣言五:商业 商业数据库与云数据库不兼容

幸运的是，我们已经揭穿了这些数据库现代化的神话，并且可以谈论在云上实现数据库现代化的难以置信的好处。我们将使用[亚马逊网络服务](https://aws.amazon.com/what-is-aws/) (AWS)云，具体来说，作为这里的一个例子，因为它是 提升数据库性能和创新的常用选项。

### 谬论:云中的数据库 的现代化几乎没有性能优势

现代云计算技术在最近的历史中取得了巨大的进步。随着这一进步，对数据库性能的期望也越来越高。事实是商业数据库正在落后。

商业数据库总体上是可扩展的。他们可能会遇到延迟问题，尤其是在支持现代应用程序时。此外，找到足够的处理能力也是一个挑战。公司仍在处理与持续供应链中断相关的芯片和内存短缺问题。

另一方面，云提供商并没有遭受太多损失。他们批量购买组件，并拥有大量设备、基础设施和内存来运行云数据库。谁从这种可访问性中受益？Compa 已经将他们的内部基础设施和商业数据库换成了云。

云数据库的最佳特性之一是它们的按需可扩展性。组织可以根据客户需求轻松扩展。相比之下，内部或私有 ate 云解决方案将工程团队与他们的硬件绑定在一起。这些限制对于云用户来说是不存在的。

云上的公司可以从各种即插即用的解决方案中进行选择，以优化解耦服务的性能(也称为 microservi ces)。这些微服务可以各自拥有自己的数据库引擎，可以根据需要进行扩展。

### 谬论:云数据库不会降低我的 IT 成本

阻碍公司发展的第二个神话是认为 IT 成本不会从云端降下来。从理论上讲，内部和云数据库的成本可能一样高，但当您考虑所有因素时，这在现实世界中是不太可能的。

有了 on -premises 数据库，无论是直接购买还是通过私有云购买，公司都要全额付款。他们通常还必须购买固定价格的许可证。即使组织没有最大限度地利用他们可用的资源，他们仍然需要支付全部费用。AWS 不是这样操作的。

在 AWS 上，如果组织选择按需使用资源，他们只需为消耗的资源付费。如果 IT 团队愿意做出最小量的 c 承诺，他们可以从不同的节约计划中进行选择。与固定价格合同相比，这些方法确保成本与处理能力更紧密地成比例。对于想要部署现代应用程序的现代公司来说，这种级别的弹性是必不可少的。

### 谬论: 为了安全和可用性，我最好把数据留在本地

除了性能和价格问题，一些人仍然认为从安全性和可用性的角度来看，商业数据库更好。然而，云胜这里出了一个 的井。云数据库最大限度地减少与相关的手动工作

*   加密静态数据
*   加密传输中的数据
*   定义港口津贴
*   准备合规审计

这一点至关重要，因为手工操作为人为错误敞开了大门。内部数据库 s 将这些任务放在 IT 部门的肩上，让管理员负责确保数据的完整性。没有犯错的余地，这在管理复杂的应用程序时是很可怕的。好消息是云数据库缓解了这一压力。

例如，亚马逊 Aurora 跨三个可用性区域维护六份数据副本。该解决方案拥有 99.99%以上的耐用性和< 30 秒的实例故障转移。同样，亚马逊 DynamoDB 保证了高达 99.999%的可靠性。AWS 的托管数据库还提供了 256 位加密，带有 SSL 认证和旋转访问密钥。

Moreover, the cloud provider offers an identity and access management (IAM) solution that simplifies authentication and role assignments for users and resources. AWS also lets users set up security groups, which take away much of the complexity associated with protecting cloud-based infrastructure.

### 谬论:在云数据库中，我能做的事情没有多少是我不能在内部完成的

k eeps 领导现代化数据库的第四个神话是，他们不会因为留在本地而错过太多。但事实是，当试图提升遗留应用程序时，商业数据库在很多方面都存在不足。大多数都是由 des ign 开发的，这与云的敏捷本质是对立的。

想要交付现代化应用程序的组织必须拥有现代化的后端基础设施。如今，这意味着利用微服务，并为不同的应用 功能提供不同的数据库。采用这种方法，组织可以根据需求分别扩展应用程序的不同部分。

### 谣言:商业数据库与云数据库不兼容

减缓数据库现代化进程的最后一个神话是流行的商业数据库，如 Oracle 和 SQL server，与云数据库 不兼容。这不是真的。例如，Amazon RDS 允许用户在完全托管的数据库服务中运行 Oracle 和 Microsoft SQL server 工作负载。亚马逊 Aurora 还完全兼容 MySQL 和 PostgreSQL 数据库。

在很大程度上，云数据库允许公司继续使用熟悉的代码、驱动程序、应用程序和工具，而不必进行太多的重构。如果领导者决定完全放弃商业解决方案，有 导入和导出工具可以帮助加速这一过程。

### 结论

总而言之，许多关于保存商业数据库的传统智慧已经过时了。云提供了一条更加引人注目和激动人心的道路。C loud-native 解决方案使公司能够在不增加任何复杂性的情况下将数据库性能提升到新的水平。
