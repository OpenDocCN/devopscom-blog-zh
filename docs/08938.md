# Graph 捐赠了 4800 万美元用于推进 GraphQL 平台

> 原文：<https://devops.com/the-graph-donates-48m-to-advance-graphql-platforms/>

Graph 是一家为区块链平台设计的基于 GraphQL 语言的索引和查询工具提供商，它今天宣布向 Guild 提供 4800 万美元，Guild 是一个开发人员团体，旨在推进开源应用程序编程接口(API)平台，以进一步开发基于 GraphQL 的技术。

该协议由 Graph Foundation 管理，这是一个监督开源图形数据库 ONgDB 开发的非营利联盟，该协议还呼吁行业协会成员在未来四年内为 Graph 推进的索引和查询工具的开发做出贡献。

Graph Foundation 主任伊娃·贝林(Eva Beylin)表示，目标是资助利用 GraphQL 的项目的开发，Graph QL 是一种开源数据查询和操作语言，用于应用程序编程接口(API)，最初由脸书开发，用于创建在以太坊等区块链平台上使用的子图，以一种更易于开发人员访问的方式索引数据。

Beylin 指出，开源社区正开始围绕区块链平台发展，也称为 Web3，现在是一系列 GraphQL 项目的主要贡献者。这些区块链平台现在正被用于驱动不依赖于任何单一平台来处理交易的去中心化金融(DeFi)应用。这些平台正被全球金融服务的初创和现有提供商所采用。

然而， [GraphQL](https://devops.com/?s=GraphQL) 社区将如何解决安全问题还有待观察。[graph QL API 就像任何其他 API 一样，很容易受到对象级认证问题的困扰](https://devops.com/graphqls-greatest-strength-is-also-its-greatest-weakness/)。问题是单个 GraphQL 查询可能比典型的 API 请求泄露更多的信息。如果 GraphQL 提供程序没有对每个特定的方法和资源进行粒度授权检查，GraphQL 端点可能会成为一个主要的漏洞。

网络罪犯也倾向于对 API 发起暴力攻击。使用 REST APIs，这种恶意行为更容易被标准 web 应用程序防火墙(WAFs)捕获，因为它更容易注意到数百个特殊的畸形请求。然而，使用 GraphQL，暴力攻击更难检测。通过为每个资源分配访问参数并一次处理一个授权检查，REST APIs 可以更容易地限制这种恶意流量。

GraphQL APIs 不可能在一夜之间取代 REST APIs。然而，随着管理 GraphQL APIs 的后端平台变得更加强大，DevOps 团队在生产环境中使用的这些 API 的数量将会急剧增加——假设网络安全团队没有异议。

与此同时，开源社区显然开始团结在 GraphQL 周围，这应该会带来适用于广泛用例的创新。现在的挑战不仅是决定使用哪种类型的 API 以及何时使用，还要决定是否需要替换任何现有的遗留 API。然而，不管怎样，API 的管理和保护——不管喜欢与否——在 2022 年只会变得更具挑战性。
