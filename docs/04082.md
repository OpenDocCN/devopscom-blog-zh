# 开发和安全就像熏肉

> 原文:[https://devops.com/devops-security-like-smoking-meat/](https://devops.com/devops-security-like-smoking-meat/)

并不是所有人都觉得“Ubuntu 没让你想起野猪吗？”或者一边盯着容器，一边在他的慢烤猪肩肉坑里劳作。或者当牧豆树的烟雾弥漫在排骨上时，还在 e2e 试验。

但是，[阿波罗·克拉克](http://www.apolloclark.com/)([@阿波罗·克拉克](https://twitter.com/apolloclark?lang=en))知道。阿波罗是熏肉的美食家(那是肉食还是烟？)，努力掌握这门手艺:了解不同的切工，每种木材添加的东西，以及酱汁的微妙之处。但是，唉，这是一种爱好；安全和 DevOps 是他的职业。

他对两者都充满热情，自然看到了两者之间的相似之处，因此阿波罗在 2016 年[全天 DevOps](https://www.alldaydevops.com/) 会议上发表了题为“熏肉教会我关于 DevOps 和安全的事情”的演讲

可以理解的是，你在问自己，“熏肉跟 DevOps 和安全有什么关系？”阿波罗笔记都有巨大的复杂性和细微差别，有多种方法来完成工作，有许多方法来搞砸它，有几种方法来做好它，你总是在学习。

![apollo 1.png](../Images/90e143f3bad29b4d23d490b6b603968e.png)

对于这两者，有许多工具和过程来完成工作。对于吸烟来说，你的木材是一个重要的组成部分，还有时间。让我们来看一些相似之处:

*   Oak =单元测试
*   枫叶=覆盖率
*   苹果=动态分析
*   桃子=静态分析
*   梅斯基特= e2e 测试。人们喜欢它，但它很难处理
*   酒桶=浏览器支持。这是你做得非常好的时候
*   波旁桶=设备支持。相当复杂
*   吸烟时间=测试。你可以测试或抽五分钟或五小时，但有一个甜蜜点。你可以过度吸烟，过度测试。

饿了吗？

![apollo2.png](../Images/fe0344db24d0b7b80d86c40fad19628d.png)

肉是一餐熏肉的关键，但许多其他成分也是如此。同样，软件应用程序需要一套其他工具和组件来交付完整的包。准备好你的味蕾，因为阿波罗将熏肉带来的所有好处比作你用来保持应用程序运行的工具:

*   酱汁=自动缩放。它们都会变得相当复杂。总是有点不同，但你必须让它为你工作。
*   面包=监控，这是基础设施的面包和黄油。确保你的筹码上下都有。
*   沙拉=系统日志。不是最性感的东西，但你可以依靠它们。
*   Fruit =自定义应用程序日志。把它们配对需要很多时间，但是要用一个真正好的东西，让它们变得更好。
*   啤酒=防火墙。你应该两者兼得。
*   入侵检测系统。这些是你的好酒。
*   威士忌= IR 训练。这需要时间，而且有很多方法可以做到。当东西坏了，我们有处理它们的程序。

在熏肉和 DevOps 和 security 中，Apollo 会问和回答:“我们必须做所有的事情吗？不，但是我们做得越多，味道就越好。

“要花钱和花时间吗？是的，但是即使没有钱和时间，你也可以做大事。你可以使用便宜的肉，也可以使用开源软件。从简单开始，逐步增加复杂性，并不断学习。”不断地问自己，“我今天比昨天更好吗？”如果没有，那就更好。
