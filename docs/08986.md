# 通过自动化实现多云网络

> 原文：<https://devops.com/enable-multi-cloud-networking-with-automation/>

随着我们进入一个新的技术时代，我们正在经历云创新的“[第三波](https://www.forbes.com/sites/forbestechcouncil/2021/05/18/the-intersection-of-cloud-30-and-the-clouds-third-wave/)”，这给 IT 领域带来了新的复杂性，主要集中在[多云](https://devops.com/?s=multi-cloud)网络。随着云技术的每一次进步，网络组件变得越来越复杂，并分布在多个群体中。此外，多云网络带来了新的挑战，即如何连接和管理由私有云和公共云以及物理数据中心组成的分散环境。组织必须统一他们的网络运营，将他们的能力提升到新的技能和效率水平。

### 增强策略

期待自动化工具提供的新功能，技术领导者可以增强他们的 2022 IT 战略。怎么会？

*   支持在混合 IT 基础架构中更加无缝地过渡到云。
*   培养网络工程师给组织带来的新价值。
*   使执行领导能够轻松协调多个团队的运营，从而在整个组织中传播创新。

### 连接混合 IT 基础架构

随着越来越多的 IT 领导者投资各种数字工具来简化网络运营，一些工程团队可能会更不愿意放弃旧流程，或者对唾手可得的众多新功能感到恐惧。随着现代工具和流程逐渐融入 IT 部门的工作流，在这种“混合”环境中找到适当的平衡至关重要，这种环境将包括传统的内部系统和新的云应用程序的组合。自动化可以通过简化以前依赖的手动流程来帮助平稳过渡到新技术，同时还可以将过多的系统连接在一起以简化管理。

### 利用云原生自动化

当将不同的系统连接在一起时，利用提供 API 功能的云原生自动化解决方案可以带来很大的不同。这些解决方案不仅在不同应用程序之间建立了所需的适当连接，还提供了以云友好方式管理网络的灵活性。此外，可以部署自动化来确保跨内部网络设备以及连接到基于云的基础架构的设备配置的合规性。例如，通过自动化，可以快速执行配置备份、检查合规性并执行“智能补救”。这意味着可以备份一个网络设备或一组设备，并将当前配置与现有的黄金配置标准进行比较。可以自动进行更改，以确保网络符合当前的合规性标准。此外，自动化正在改变 IT 团队应对日常运营挑战的方式。

### 网络团队的价值演变

网络工程师在组织中扮演着独特的角色，他们执行的任务需要多年的培训，并且需要了解内部系统的复杂细节。尽管这些手动任务的性能可能被认为是工程师的价值驱动因素，但新技术的采用已经突破了网络专业人员所熟悉的界限。反过来，这也让人们对什么是真正的成功和影响产生了期望。组织正在寻找工程师来利用这些新技术的力量，并执行现代战略——利用自动化来支持这种新的思维模式。

### 简化集成式多云环境的采用

[打破技术和团队之间的孤岛](https://digitalcxo.com)对于了解如何从投资中获得最大收益至关重要。对于网络团队来说，要完全接受多云网络，他们需要对云网络进行更深入的管理、细分、安全和控制。与细分一样，自动化可以帮助网络团队摆脱使用覆盖，转而使用 AWS、Azure 等原生软件工具。此外，许多网络团队可能会错误地认为自动化需要他们学习编程和编写代码，而在许多新工具中，网络元素可以直接从应用程序的代码中管理和编排。这将使网络可编程性成为云软件结构的一部分，并简化集成多云环境的采用。网络团队可以实施合规性和自动化来标准化内部或新的云程序，同时培养他们的自动化技能，而无需编写任何代码。

### **调整运营效率**

当跨组织投资新的 IT 基础设施时，IT 领导通常在每个部门内“垂直地”建立他们的计划。然而，这造成了团队之间的差距，因为每个团队都在独立的模型和功能中运营，从而增加了不必要的成本和低效的业务实践。为了在 2022 年成功调整多云计划以提高运营效率，IT 领导者需要在整个组织中进行更多的“横向”投资，并在此过程中整合自动化工具以减少时间和压力。

### 实施 NaaS

实现运营效率可以减少 IT 团队经历的时间和压力，这就是采用网络即服务(NaaS)工具可以发挥有益作用的地方。IT 领导者可以直接从云中投资网络服务，而不是构建新的网络。在接下来的一年里，随着云基础设施的扩展，NaaS 实施的重要性也将随之增长——企业和其他组织(包括服务提供商)将有机会利用云基础设施，而无需自己构建新的网络。自动化工具可以帮助加快完成时间，并自动化活动中最大比例的任务，尽可能消除空闲时间。

### 移动模型并进行改进

当考虑投资这些类型的工具时，重要的是要记住它们是如何影响整个过程而不是单个任务的。通过自动化整个流程，而不仅仅是单个任务，操作员可以灵活地用更少的资源在更短的时间内完成更多的工作，从而使创新成为可能。随着灵活性和可用时间的增加，工程和运营团队可以将注意力集中在更有价值的战略活动上，而不是执行常规的时间密集型活动。当 IT 领导者能够从“以人为中心”的范式转向“以自动化为中心”的模式时，运营就会得到改善。例如，对于维护检查，该团队可以让系统在几秒钟内运行越来越多的预检查，从而使流程更加高效和安全地执行。关键是采用自动化整个流程而不是单个任务的模型。

### 走向

明年，随着越来越多的工作场所以混合模式运营，IT 领导者将需要改进他们的流程，不仅要统一远程和现场员工的运营/方法，还要推进这一新的云现代化浪潮。通过这样做，自动化可以帮助减轻和管理多云网络带来的挑战，提高技能和效率以实现长期业务影响。
