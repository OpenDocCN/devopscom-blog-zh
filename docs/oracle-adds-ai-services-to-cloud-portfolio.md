# 甲骨文将人工智能服务加入云产品组合

> 原文：<https://devops.com/oracle-adds-ai-services-to-cloud-portfolio/>

甲骨文今天推出了甲骨文云基础设施(OCI)人工智能服务，这是一个服务集合，使开发人员更容易使用应用编程接口(API)来调用基于甲骨文代表他们训练的算法的各种服务。

甲骨文负责人工智能服务和数据科学的产品管理副总裁 Elad Ziklik 表示，主要目标是将与机器学习操作( [MLOps](https://devops.com/?s=MLOps) )相关的数据工程任务从那些只想让开发人员在其应用程序中包含人工智能功能而不必雇佣和保留专门的数据科学团队的组织中卸载。

然而，他指出，组织也可以选择将自己的数据上传到 OCI 人工智能服务中，以重新训练甲骨文提供的模型。

最终，目标是通过简化人工智能模型的使用，使 OCI 人工智能服务成为 DevOps 工作流的简单扩展；Ziklik 说，这些模型成为另一个软件工件，可以与 DevOps 团队管理的所有其他工件一起在 Git 存储库中存储和更新。他补充说，实际上，甲骨文希望让人工智能变得“无聊”，因为这些功能现在可以被任何开发团队随时调用。

OCA 人工智能服务包括 OCI 语言和 OCI 语音，前者是一种文本分析功能，允许开发人员将情感分析、关键短语提取、文本分类、命名实体识别和其他类似功能添加到他们的应用程序中，后者将包含人类语音的基于文件的音频数据转换为文本转录。

甲骨文还包括用于图像识别和文档分析任务的 OCI 视觉，用于创建模型以标记流程中的违规行为的 OCI 异常检测，用于跟踪长期业务指标的 OCI 预测，以及用于创建数据集以训练人工智能模型的 OCI 数据标签。

Ziklik 说，这些服务旨在解决一般企业最关键的人工智能要求，而不是试图在一系列不能反映业务开发团队要求的人工智能基准上超越对手。

此外，Ziklik 指出，Oracle 还使解决人工智能合规性要求变得更加容易，最终将要求组织准确记录他们的人工智能模型如何工作。

不久的将来，几乎每个应用程序都会在某种程度上调用人工智能功能。这种转变给开发运维团队带来了巨大的工作流挑战。人工智能模型通常是由数据科学团队创建的，这些团队通常一年不会在生产环境中成功部署人工智能模型超过几次。然而，这些人工智能模型需要与应用程序集成，在许多情况下，这些应用程序通过 DevOps 工作流不断更新。

Oracle 表示，与允许数据科学团队在需要由 it 专业人员维护的平台上构建一套定制的 MLOps 工作流相比，使用它创建的模型可以更容易地实现这一目标。例如，使用 OCI 人工智能创建的模型可以存储在现有的 Git 存储库中，而不是必须建立一个功能库来跟踪和管理人工智能模型。

当然，每个组织都需要确定他们融合 MLOps 和 DevOps 最佳实践的程度。然而，甲骨文认为实现这一目标的最简单方式是通过云服务提供人工智能模型，开发者可以随时调用这些模型。
