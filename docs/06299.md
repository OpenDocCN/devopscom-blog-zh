认识到开发运维确实是一次旅程,《开发运维之旅行动手册》的章节支持“从您所在的地方开始”的方法，这种方法符合不同的开发运维成熟度水平和不同的兴趣领域，例如:

1.  探索和发现:这部分包括围绕 DevOps 人类之旅的愿景、使命和目标的关键主题。它通过不同的案例研究和侧重于挑战、解决方案和结果的垂直市场来考察成功是什么样子的。它还提供了当今“开发运维状况”的概况，以及对市场和开发运维专业人员角色的其他重要理解。
2.  理解和学习:本节提供了理解 DevOps 和其他相关行业实践的框架。它强调了成功的 DevOps 转型之旅所必需的不同原则和概念，还提供了与 DevOps 人员相关的功能和技术内容，以及术语表和认证信息。
3.  计划、实践和行动:本节概述了与核心技能、工作描述/角色相关的 DevOps 之旅的文化方面，并将文化发展与评估和发展工作场所结构联系起来。它提供了跨关键功能和面向流程的任务的内容，并侧重于自动化，提供了来自行业研究、协会和实践的资源链接。
4.  优化和改进:本节重点关注绩效管理，特别是围绕关键绩效指标、目标和关键结果的测量和度量。这包括一个由社区提供的资源库，用于支持与提高日常工作绩效相关的讨论和活动。

在 DevOps 研究所研究主任 Eveline Oehrlich 的领导下，其他 DevOps 研究所行动手册正处于规划阶段，包括 DevOps 工程师行动手册、现场可靠性工程行动手册和 DevOps 人员行动手册。

DevOps Institute 首席执行官 Jayne Groll 表示:“现在，DevOps 社区比以往任何时候都更需要一个动态的知识体系，以捕捉最新的思维，编排来自其他来源的工件和资产，并提供可操作的指导和资源。“DevOps Institute 一直倡导集体知识体系(cBok ),我特别自豪的是，DevOps Institute SKILup 行动手册库可以利用领先的行业分析师、从业者、供应商、思想领袖和其他具有广泛实践经验的人的贡献，创建一个完整的行动手册系列。”

DevOps Institute 首席研究主任 Eveline Oehrlich 表示:“剧本对于提高团队的效率和许多个人的生活至关重要，因为通过与行业知名模式和实践相关的数据和信息的集合，人们可以采用行动计划来改善他们如何开始新的旅程以及如何成功取得进展。“DevOps 之旅行动手册是第一本为个人和组织如何启动或导航其 DevOps 转型之旅奠定基础的出版物，旨在为其业务团队并最终为其客户或客户取得成功。”

**关于 DevOps 学院**

DevOps 学院致力于推动 DevOps 成功的人文因素。作为一个全球性的会员制协会，DevOps Institute 是连接 IT 从业者、教育合作伙伴、顾问、人才获取和业务高管的首选学习中心，旨在为支持数字化转型和新 IT 铺平道路。

我们基于以人为本的 SKIL 框架，包括技能、知识、想法和学习，在 DevOps 社区内帮助促进职业发展并支持新兴实践。我们的所有工作，包括认证、研究、活动和持续学习计划，都专注于提供“人类知识”,以实现 IT 现代化，并使 DevOps 取得成功。
