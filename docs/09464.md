 *## 1.IPv6 被又一次踢皮球运动扼杀了

本周首先关注的是:释放大量“保留”IPv4 空间的努力正在向前推进。愿景是释放额外的 4 . 19 亿个 IP 地址。

### 分析:在 IPv6 上插个叉——搞定了

这个善意的项目只是 IPv6 僵尸中最新的一把匕首。虽然它也解决了其他问题，但 IPv6 的主要目的是给我们更多的地址空间。但每次我们像这样延长 v4 的寿命，我们就会把 v6 越踢越远。最终，我们会到达**没人再关心**的地步。

你好，[朱哈·萨里南](https://www.itnews.com.au/news/freeing-up-of-hundreds-of-millions-of-ipv4-addresses-mooted-580689 "read the full text") : **释放数亿个正在讨论的 IPv4 地址**

Seth Schoen 是……Let ' s Encrypt 的联合创始人，他正在进行一个 IPv4 清理项目，该项目将采用目前未在公共互联网上路由的地址，并使它们普遍可用。[他]表示，上世纪 80 年代做出的将多个 IPv4 地址范围保持为“特殊”地址范围的决定，导致了大量编号资源的浪费……“尽管这些决定背后的原因尚未得到证实。”
…
采用 240/4、0/8、127/8、225/8-232/8 范围，并使它们可用…将增加一些*4.19 亿个* IPv4 地址。[但是]让当前保留的和不可路由的范围可用带来了挑战。[例如，]许多组织使用当前受限的范围作为其内部网络的私有地址空间，这带来了需要解决的政策问题。

答案当然是推动 IPv6 的采用，而不是让 IPv4 维持生命？[塞思·舍恩发言](https://blog.apnic.net/2022/05/31/cutting-down-on-ip-address-waste/ "read the full text"):

有些人…担心我们的提议会与 IPv6 竞争或威胁到 IPv6 的采用。…这是一个错误的困境:任何改善 IPv4 地址空间供应的努力都不需要被解释为对 IPv6 的攻击，也不需要被解释为任何组织不实施 IPv6 的理由。
…
即使是已经广泛采用 IPv6 的组织，出于多种原因，通常也会继续对 IPv4 空间有需求。我们认为在很长一段时间内都是如此。

这有点儿严肃的一厢情愿的味道。这里是[佐林尼克斯的惨淡总结](https://tech.slashdot.org/comments.pl?sid=21436256&cid=62582223 "read the full text"):

这是一个双向的先有鸡还是先有蛋的问题:……在*客户端*没有使用 IPv6 的动机，因为如果你有 IP v4……你可以访问整个互联网，[所以]你不会“错过”任何东西。……在*服务器*端没有使用 IPv6 的动机，因为如果你有 IPv4，每个人都能联系到你。
……
在*不*使用 IPv6 变得更加痛苦之前，全面采用 IPv6 不会发生，遗憾的是，这在短期内不会发生。…我们一年比一年接近，但每年我们都要走大约一半的距离，所以我们不可能很快到达那里。 🙁
