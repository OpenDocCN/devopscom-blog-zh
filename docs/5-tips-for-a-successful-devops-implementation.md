# 成功实施 DevOps 的 5 个技巧

> 原文：<https://devops.com/5-tips-for-a-successful-devops-implementation/>

越来越多的企业转向 DevOps 实施，以更高效地交付软件和更新。尽管业务领导知道 DevOps 可以帮助他们变得更快、更敏捷，但他们中的许多人不确定如何开始。许多企业甚至开始实施 DevOps，结果却遇到了不可预见的问题，阻碍了这一过程。

虽然开发运维是有益的，但你不希望轻易做出改变。你需要知道你将进入什么样的环境，并准备好应对变化对公司的影响。拥有一个优秀的团队，并为他们提供合适的 [基础设施即代码](https://bridgecrew.io/infrastructure-as-code-security/) (IaC)工具会有所帮助，但在开始之前，你需要考虑的还有很多。

以下是成功实现 DevOps 的五个重要提示。

## 评估您的流程

你需要从学习 [开始，什么是 devo PS](https://devops.com)以及它对你的组织意味着什么。团队中的每个人都需要了解 DevOps 可以做什么，以及它可以为您的公司解决什么类型的问题。

从头到尾评估你的整个开发周期。查看进入开发的不同过程，找出不同的瓶颈和低效之处。通过找到您当前流程的不足之处，您将能够确定在实施 DevOps 时要关注的领域。

## 协作和交流

协作和沟通是 DevOps 成功实施的核心。您可以获得所有可用的最佳 DevOps 工具，但是如果您没有准备好一起工作和交流的团队，您将无法获得 DevOps 实施的全部好处。

如果你想要成功，你需要在团队中灌输强烈的协作和合作意识。除此之外，您还需要为您的团队成员提供工具，帮助他们交流和协作。通信平台和协作软件与直接促进开发的工具一样重要。

## 设定目标

DevOps 实施不仅仅是获得正确的工具和改变流程。你需要评估公司的运营情况，并为实施设定目标。如果您只是购买新工具而没有设定新的方向，那么实现不会带来太大的变化。

另一个很好的建议是从一个更小的、更容易实现的目标开始，然后逐步扩大。一次改变一切可能会造成太大的干扰。相反，选择一个较小的目标，一旦实现了，就向更大更复杂的目标前进。

## 自动化

自动化是使 DevOps 对现代企业有益的重要因素。如果你要实现变化的全部好处，你应该尽可能的自动化。无论是扫描错误配置的代码，还是自动化测试[](https://www.softwaretestinghelp.com/devops-and-software-testing/)，通过使用各种不同的自动化工具，您可以做很多事情来使您的操作更加高效。

您可能还想一步一步地实现自动化。如果您试图一头扎进自动化，您的团队可能有点难以跟上。取而代之的是，从你的团队中选择一个耗费大量时间和精力的手工过程，并实现一个自动化工具。让你的团队监控几个星期，看看进展如何，然后利用你学到的东西，将反馈应用到自动化更多的过程中。

## 了解您的关键指标

从实施流程的一开始就设定关键指标。没有[指标](https://devops.com/?s=metrics)，你将无法跟踪你取得的进步。缺乏跟踪也会使你很难发现可能阻碍你进步的问题。

大多数组织需要关注三点:交付时间、部署时间和平均恢复时间。除此之外，在定义关键指标时，试着想想你的用户。他们喜欢什么？他们关心什么？什么样的经历更令人愉快？

DevOps 实现并不简单。通过采取正确的准备步骤，您可以避免最糟糕的问题，并使自己能够很好地应对实施过程中出现的问题。
