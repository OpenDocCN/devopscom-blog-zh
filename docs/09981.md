# Zilliz 展开托管矢量数据库服务

> 原文:[https://devo PS . com/zilliz-un furls-managed-vector-database-service/](https://devops.com/zilliz-unfurls-managed-vector-database-service/)

Zilliz 本周推出了一种可管理的云服务，现在不到五分钟就可以启动。这项服务基于该公司的开源 Milvus 矢量数据库。

Zilliz 的运营总监刘中基表示，运行在亚马逊网络服务(AWS)云上的托管服务 Zilliz Cloud 将使更广泛的应用程序开发人员更容易访问矢量数据库。

向量数据库正在获得牵引力，因为它们为人工智能(AI)应用程序开发人员提供了一种更有效的方法来管理训练 AI 模型所需的非结构化数据。目前，许多这样的团队都依赖于标签和标记来分类和管理数据。刘说，Milvus 矢量数据库采用深度学习算法和自己的人工智能模型来创建和存储、索引和管理矢量，以更有效地对数据进行分类。

他补充说，基于云原生架构，例如，利用 Kubernetes 更有效地扩展云资源的消费，Zilliz Cloud 还可以扩展为数据密集型应用程序处理数百亿高维向量数据。该公司声称，Zilliz Cloud 为数据密集型工作负载的读写操作提供了高达 10 倍的吞吐量和 5 倍的延迟。

![](../Images/edaad97ed9ca62e4646786d8aaaf3e86.png)

受管 Zilliz 云的其他属性包括支持多语言软件开发工具包(SDK)、基于 web 的图形用户界面以搜索和浏览数据、SOC 2 合规性、使用 IP 白名单的网络隔离和传输中的完全数据加密(TLS)。

未来，Zilliz 还计划在图形处理器单元(GPU)之外添加对微软 Azure Cloud 和谷歌云平台的支持，以优化其深度学习算法的性能。

向量数据库的潜在使用案例涵盖了从图像检索和视频分析到欺诈检测到药物发现的方方面面。已经使用 Milvus 的组织包括易贝、Shopee 和 SmartNews，Milvus 是在 LF AI & Data 基金会的支持下推进的。

总体而言，数据密集型工作负载的增加正推动更多组织采用[数据运营](https://devops.com/?s=DataOps)最佳实践来优化海量数据的管理。鉴于训练人工智能模型通常需要大量数据，建立人工智能模型的组织通常处于这些努力的最前沿。组织历来面临的挑战是，没有多少组织拥有管理矢量数据库所需的内部专业知识。Zilliz Cloud 依靠 Zilliz 培训的数据库管理员团队的专业知识来运营托管服务，从而消除了这种需求。

目前尚不清楚各组织以何种速度采用矢量数据库来构建人工智能模型。随着下一个应用时代的构建，用于构建人工智能模型的机器学习操作(MLOps)实践与用于构建嵌入这些模型的应用的 DevOps 实践融合只是时间问题。挑战在于，构建人工智能模型的数据科学团队往往也有自己独特的数据运营文化；DevOps 团队需要找到一种方法来接受和扩展这一点。

与此同时，用于构建应用程序的数据库类型的数量只会继续增加，因此每个开发运维团队都需要知道何时自己管理每种数据库，而不是依赖托管服务。
