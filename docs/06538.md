# Barracuda Networks 与微软合作开发 SD-WAN 服务

> 原文:[https://devo PS . com/barracuda-networks-partners-with-Microsoft-on-SD-wan-service/](https://devops.com/barracuda-networks-partners-with-microsoft-on-sd-wan-service/)

Barracuda Networks 今天推出了一项[软件定义的广域网(SD-WAN)服务，该服务原生运行在微软 Azure 云](https://www.prnewswire.com/news-releases/barracuda-introduces-first-global-secure-sd-wan-service-built-natively-on-microsoft-azure-301094835.html)上。

Barracuda Networks 网络安全副总裁 Klaus Gheri 表示，微软已经使使用 Azure Virtual WAN Hubs 推出 SD-WAN 服务成为可能，Azure Virtual WAN Hubs 是一种基于轴辐式架构的网络服务，它在一个公共接口下统一了安全和路由等网络服务。这些服务涵盖分支机构连接、多种类型的虚拟专用网络、云互联、防火墙和加密服务。

Gheri 说，Barracuda Networks 不是必须复制所有这些网络服务，而是在 Azure 虚拟 WAN Hubs 之上分层 Barracuda CloudGen WAN 服务，以提供像任何其他云服务一样可以按计费小时消费的服务。

他说，Barracuda CloudGen WAN 服务是与微软合作开发的，提供了将 Azure 虚拟 WAN 中心转换为 SD-WAN 服务所需的管理框架，IT 团队可以使用该服务安全地将远程办公室连接到云应用程序或内部 IT 环境。

如今，SD-WAN 服务有四种基本形式。IT 团队可以部署和管理他们自己的 SD-WAN 设备。或者，电信运营商在其网络上提供 SD-WAN 服务，或者托管服务提供商(MSP)可以通过在其管理的全球网络的接入点(PoP)上部署设备来创建自己的 SD-WAN 服务。最近，云服务提供商已经通过他们建立的连接所有数据中心的网络提供 SD-WAN 服务。

Gheri 说，微软现在通过增加由 Barracuda Networks 创建的管理门户，将 SD-WAN 功能向前推进了一步。

Vanson Bourne 最近代表 Barracuda Networks 对 750 名业务和 IT 专业人员进行的调查发现，超过一半(52%)的人希望采用通过云交付的 SD-WAN 服务。相比之下，调查发现 16%的人更愿意从独立供应商处获得 SD-WAN，15%的人会选择电信合作伙伴，12%的人会依赖增值经销商。

随着越来越多的应用程序工作负载迁移到云中，网络的部署和管理方式也在快速发展。越来越多的组织不再将分支机构连接到本地数据中心来访问云应用，而是采用 SD-wan 来提供对云应用的直接访问，而不必通过本地数据中心回程网络流量。这种方法显著改善了整体应用延迟，并且在许多情况下，减少了对电信运营商租用 MPLS 线路的依赖。

许多 IT 组织希望能够通过门户访问和管理 SD-wan，而不是自己构建 SD-wan，通过门户，他们可以按照自己认为最合适的方式为应用程序分配带宽。尚不清楚 It 组织将在多大程度上继续倾向于部署设备，而不是使用 SD-WAN 即服务。然而，鉴于管理 SD-WAN 的复杂性，通过现有云网络基础设施交付的 SD-WAN 服务很有可能很快成为广泛采用的替代方案。
