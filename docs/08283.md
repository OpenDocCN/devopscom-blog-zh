# Logz.io 在开源可观察性方面与微软结盟

> 原文：<https://devops.com/logz-io-allies-with-microsoft-on-open-source-observability/>

Logz.io 本周宣布[与微软](https://www.businesswire.com/news/home/20210728005515/en/Logz.io-Announces-Strategic-Partnership-with-Microsoft-to-Rapidly-Accelerate-Cloud-Native-Application-Monitoring)结盟，使 DevOps 团队能够在 Azure 公共云上部署其可观测性平台——基于 ElasticSearch、Logstash、Kibana (ELK)、Prometheus 和 Jaeger 等开源平台。

Logz.io 的 Jonah Kowall 表示，多年的努力使这些团队能够通过 Azure 控制台点击一个按钮来部署 Logz.io 平台。他补充说，Azure 平台具有数据收集功能，可以聚合操作系统事件，从而简化 Prometheus 监控解决方案的使用。

然而，DevOps 团队仍然需要从他们在 Azure 平台上开发的应用程序中收集数据，他补充道。

Logz.io 在单个可观察性平台内集成了基于指标、日志和跟踪的多个开源监控工具，否则需要内部 IT 团队自己投入大量时间和精力。

随着组织在解决问题时试图为内部 IT 团队提供更多的上下文，对可观察性的兴趣正在上升。每当出现 IT 问题时，大多数团队都会召集一个“作战室”，在那里他们试图通过比较各种监控工具生成的分析来发现问题的根本原因。可观察性平台承诺使 DevOps 团队能够通过在单个平台内集成日志数据、指标和分布式跟踪，以提供更多上下文的方式来替换其中的许多工具。

在许多情况下，现有的[应用性能管理](https://devops.com/?s=application+performance+management) (APM)平台正在通过添加工具来监控 IT 基础设施，从而转变为可观察性平台。相反，管理 IT 基础设施的平台正在增加对监控应用程序的工具的支持。在其他地方，一系列初创公司已经出现，提供他们从头开始开发的可观测性平台。

无论采用何种平台，许多组织内基于微服务的应用程序的兴起都在推动可观测性问题。基于微服务的应用程序中存在的依赖性太复杂，如果没有一个提供某种程度的可观察性的平台，就无法管理。当然，大多数现有的单片应用程序不会很快退役，因此对传统和现代云原生应用程序的可观测性框架的需求只会变得更加迫切。

当然，Logz.io 平台可以部署在其他云平台以及内部 IT 环境中。随着混合云计算的不断发展，很明显，可观测性平台将需要将其覆盖范围从云扩展到网络边缘。随着时间的推移，人工智能(AI)功能将被添加，以便更容易地(例如)简化复杂 it 环境中生成的警报数量。

与此同时，关于可观察性最重要的事情可能就是简单地开始。大多数 IT 团队开始分析日志，然后使用开源的 Jaeger agent 软件进行跟踪。毕竟，唯一比应用程序在部署前更难检测的东西，可以说是已经在生产环境中运行的应用程序。
