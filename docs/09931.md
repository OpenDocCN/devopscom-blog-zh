# 开发人员仍然更喜欢远程工作

> 原文：<https://devops.com/developers-continue-to-prefer-remote-work/>

一项[调查](https://www.netlify.com/press/developers-drive-remote-work-future-jamstack-survey-shows/)发现 83%的开发人员一半以上的时间都在远程工作，超过四分之三(76%)的人报告说他们在去年保持或增加了远程工作的频率。事实上，五分之三的受访者(62%)至少 90%的时间都在远程工作。

这项调查由 Netlify 发布，Netlify 是一个使用 Jamstack 框架构建 Web 应用程序前端的平台提供商，调查了 Jamstack 社区的 6544 名开发人员，发现超过一半(55%)的人如果被迫回到办公室，就会辞职。

目前还不清楚有多少远程工作已经成为组织吸引和留住开发人员的事实要求，但很明显，在新冠肺炎疫情之后，开发人员和雇用他们的组织之间的动态已经完全改变了。

Netlify 首席执行官 Mathias Biilmann 表示，不管行业如何，显而易见的是，DevOps 团队将继续需要做出调整，以促进分布式团队之间的合作。他指出，高级管理人员会发现，管理员工的工作流程越来越具有挑战性，员工流动率也很高。三分之一的受访者(33%)称去年换过工作。目前还不清楚办公室缺乏互动会在多大程度上影响创新，但在对开发人员的需求远远超过供应的时候，组织需要适应似乎已经成为永久新现实的情况。

与此同时，Netlify 调查指出，新兴技术被接受的速度继续快速发展——这可能是因为而不是尽管有远程工作。例如，调查发现 70%的开发人员报告去年使用了无服务器计算框架。调查发现，近一半(47%)的人也在尝试边缘动态网站。很明显，公司文化，至少在技术采用方面，在吸引开发人员方面起着很大的作用。

总体而言，96%的受访者指出 JavaScript 是他们的首选编程语言，21%的开发人员现在专门使用 TypeScript 来构建应用程序。与此同时，71%的开发人员正在使用 React 框架来构建应用程序，47%的开发人员表示他们使用了该框架的 Next.js 版本。其他获得关注的 React 框架包括 Vite (32%)、Sveltekit (15%)、Astro (11%)、Remix (10%)和 SolidJS (6%)。

然而，新兴的 Web3 技术还没有获得牵引力。虽然 7%到 10%的开发人员已经尝试了加密货币和 NFT 等技术，但只有 3%的人报告定期使用这些技术。调查发现，总共有 42%的开发者要么不知道 Web3 是什么，要么不关心它，而 31%的人对它持否定态度。

最终，组织投资开发运维的主要原因是为了提高开发人员的工作效率。挑战在于，随着开发人员采用更多的工具和平台来构建应用程序，部署这些应用程序的过程变得更加难以有效管理。实现这一目标的一种方法是将前端和后端软件开发过程分开，使开发人员能够将更多的精力放在编写代码上，而不是管理部署代码所需的晦涩难懂的过程。
