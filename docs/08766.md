# Eclipse 基金会发起开源边缘计算倡议

> 原文:[https://devo PS . com/eclipse-foundation-launchs-open-source-edge-computing-initiative/](https://devops.com/eclipse-foundation-launches-open-source-edge-computing-initiative/)

今天，Eclipse 基金会与 OpenAtom 基金会合作[启动了 Oniro 项目](https://www.globenewswire.com/news-release/2021/10/26/2320349/0/en/Open-Source-Leader-the-Eclipse-Foundation-Launches-Vendor-Neutral-Operating-System-for-Next-Generation-Device-Interoperability.html)和工作组，以创建 OpenHarmony 的独立实现，open harmony 是一个[开源](https://devops.com/?s=open+source)操作系统。

OpenHarmony 操作系统由 OpenAtom 基金会创建，基于华为最初创建的鸿蒙系统。它被设计成支持多个内核，但是如果一个设备有大量的内存，它就使用 Linux 内核。

它还包括 DSoftBus 功能，用于将物理上分离的设备集成到单个逻辑超级设备中，除了促进分布式设备之间的数据共享之外，还使一个设备能够控制其他设备。

华为还与 Linaro、Seco、Array、NOITechPark 和 Synesthesia 合作，为更大的 Oniro 项目中的持续集成/持续交付(CI/CD)平台做出了贡献。

Eclipse Foundation 的执行董事 Mike Milinkovich 表示，目标是提供一个通用的抽象平台层，可以在广泛的嵌入式系统中使用，这些系统通常部署在网络边缘。Milinkovich 指出，围绕平台的这种碎片化水平不能提供很多显著的差异化价值，只会导致应用程序的部署速度变慢。

作为这一努力的一部分，成立了一个专门的工作组，以提供一个供应商中立的结构，通过这一结构，可以以更大的透明度为项目做出贡献。OpenAtom 基金会于去年在北京成立，得到了阿里巴巴、百度、华为、浪潮、奇虎、腾讯和中国招商银行的支持。与 Eclipse Foundation 的联盟为 OpenHarmony 项目带来了更多构建开源软件的传统治理规则。

一般来说，DevOps 团队应该密切跟踪边缘计算平台事实上的标准提案的开发。现在，越来越多的应用程序被部署在边缘计算平台上，这使得在数据被创建和消费时处理和分析数据成为可能。与在高度分布式计算环境中构建和部署应用程序相关的 DevOps 挑战是相当大的。

在许多情况下，当需要支持运行在基于各种处理器的嵌入式系统上的数百甚至数千个应用程序时，从未真正尝试或掌握应用程序及其相关更新的持续交付的组织将面临严峻挑战。

此外，许多相同的 DevOps 团队将开始看到传统的面向批处理的应用程序被基于事件驱动架构的应用程序所取代，这些应用程序可以近乎实时地处理数据。在大多数情况下，这些应用程序将构建在基于微服务的应用程序上，这些应用程序的构建和部署通常比传统的整体应用程序更具挑战性。

可以说，IT 组织正处于一个新的 IT 时代的尖端，假设大多数组织不仅掌握了 DevOps 最佳实践，而且现在也在采用 DevSecOps 最佳实践来保护这些应用程序环境。

Orino 项目开发的开源软件进入生产环境可能还需要一段时间，但很明显，降低网络边缘的复杂性应该是所有 DevOps 团队在 2022 年的首要任务。
