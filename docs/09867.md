# JFrog 添加模块以更好地保护软件供应链

> 原文:[https://devo PS . com/jfrog-adds-module-to-better-secure-software-supply-chains/](https://devops.com/jfrog-adds-module-to-better-secure-software-supply-chains/)

JFrog 今天在其 Artifactory 存储库中添加了一个 JFrog 高级安全模块，使 DevOps 团队能够扫描二进制文件和源代码中的漏洞和错误配置。

JFrog 开发人员关系副总裁 Stephen Chin 表示，这种方法将使 DevOps 团队能够确保应用程序在部署到生产环境之前是安全的。

Chin 说，除了发现漏洞，JFrog 高级安全模块还可以检测暴露的秘密，如密码、访问令牌和私钥。他补充说，利用 JFrog Xray 软件组合分析工具提供的现有扫描功能，它还可以用于检查用于提供基础设施即代码的文件。

![](../Images/f76d5a5dac50927bc2774d15fa7cd38c.png)

Chin 指出，最重要的是，它为开发团队提供了确定应用程序的哪些组件正在使用封装了实际可利用代码的容器所需的上下文。

DevOps 团队目前面临的一个主要问题是，网络安全团队经常向他们提供一长串漏洞，其中许多不会对未暴露在互联网中的应用程序产生有意义的影响。然后，开发运维团队需要手动报告为什么任何给定的漏洞都不需要修复。Chin 说，JFrog 高级安全模块有助于自动化这一过程。

Chin 说，总的来说，JFrog 继续追求二进制优先的方法来确保软件供应链的安全。他指出，尽管尽可能解决源代码中的漏洞很重要，但网络罪犯要利用的是生产环境中运行的应用程序二进制文件。

实际上，网络犯罪分子通过在软件组件部署到生产环境之前将恶意软件注入软件组件，开辟了一条危害 IT 环境的新战线。希望是，一旦在这些环境中部署，恶意软件可以在以后的某个日期被激活。因此，DevOps 团队现在被要求在他们的软件开发过程中改善网络安全卫生，Chin 说。

目前还不清楚各组织采用 DevSecOps 最佳实践来实现这一目标的速度有多快，但是在拜登政府发布了要求联邦机构更好地保护其软件供应链的[行政命令](https://securityboulevard.com/2022/09/white-house-releases-software-supply-chain-security-guidance/)之后，预计更多的企业 It 组织将会效仿。面临的挑战是找到一种方法来构建和部署更安全的应用程序，而不会对它们的构建和部署速度产生负面影响。今年早些时候，JFrog 启动了开源 [Pyrsia 项目](https://jfrog.com/blog/pyrsia-open-source-software-that-helps-protect-the-open-source-supply-chain/)，该项目采用区块链技术来确保开源软件包不会受到漏洞和恶意代码的危害。

从长远来看，随着应用程序变得越来越安全，DevOps 团队和网络安全专业人员之间存在的历史性分歧开始缩小只是时间问题。当前最大的挑战是，许多部署在生产环境中的应用程序已经存在已知的漏洞。在接下来的几个月中，许多 DevOps 团队将被要求评估这些漏洞的严重性，作为持续改善应用程序安全性的工作的一部分。这些修补工作无疑会对编写构建新应用程序或用新功能更新现有应用程序所需的代码的可用时间产生重大影响。

然而，无论如何，过去几十年来允许积累的技术安全债务的数量需要得到解决。
