# Civo 报告提出了越来越多的云锁定问题

> 原文：<https://devops.com/civo-report-surfaces-growing-cloud-lock-in-concerns/>

基于 Kubernetes 的云服务提供商 Civo 对 100 名 IT 专业人士进行的调查发现，目前共有 82%的人依赖亚马逊网络服务(AWS)或微软的公共云基础设施。然而，超过三分之一(34%)的受访者也报告说，他们现在感到被公共云服务提供商束缚住了。

调查受访者认为这种感觉的主要原因是数据传输成本过于昂贵，无法从当前的云环境中迁移出来(65%)。

简单地说，惯性似乎也是一个因素。三分之一的受访者(33%)表示他们一直使用同一家云服务提供商。近三分之二(62%)的受访者还表示，他们担心替代云提供商将遭受更多的停机。然而，与此同时，45%的受访者承认他们面临着越来越大的降低云成本的压力。

Civo 首席执行官 Mark Boost 表示，很明显，云中断不仅仅局限于替代云提供商。他指出，事实上，大多数应用程序工作负载相对简单，因为它们可以在任何数量的云服务上以相同的弹性水平运行，成本低于主要超大规模服务器目前的收费。

事实上，除了一些更复杂的工作负载，云计算应该在很大程度上被视为一种商品，增加了推动力。

当然，云服务提供商不鼓励使用过高的数据出口费用进行迁移，这通常会使从他们的平台上移动数据的成本过高。事实上，大多数应用程序部署在单一云上，而不是真正的混合云模型上，因为数据管理可能会变得非常复杂。理想情况下，IT 组织希望能够让一个[云服务](https://devops.com/cloud-misconfigurations-threaten-cloud-migration/)提供商与另一个提供商合作来降低成本。然而，在实践中，一旦应用程序被部署，它很少移动到另一个平台。

然而，从好的方面来看，部署在 Kubernetes 集群上的基于微服务的应用程序更容易移动，因为 Kubernetes 集群的每个实例都公开了一组一致的 API。然而，部署在 Kubernetes 环境中的有状态应用程序可以像任何其他应用程序一样依赖于专有数据库。

当然，代价是企业 IT 环境中添加的每个云平台都会增加总成本。IT 人员通常需要雇用额外的专家，这些专家需要工具来管理每个云。当然，存在跨多个云的平台，但是购买、供应和维护这些平台的成本可能相当高。与此同时，越来越多的组织发现他们在分别管理和尝试保护多个云，仅仅是因为各个部门更喜欢一个[云服务](https://digitalcxo.com/article/selecting-the-right-cloud-service-provider-for-your-digital-transformation-initiatives/)而不是另一个。

尚不清楚组织可能会在多大程度上很快决定整合云工作负载，作为降低成本的一部分。然而，在新冠肺炎疫情之后，许多工作负载被转移到了云上，而几乎不考虑成本。现在，企业正在适应所谓的新常态，至少，这些决策中的许多正在被重新评估，看看什么可能是最有财务意义的，因为整体经济似乎越来越不可预测。
