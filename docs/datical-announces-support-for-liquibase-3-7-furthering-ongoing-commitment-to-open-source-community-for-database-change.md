# Datical 宣布支持 liqui base 3.7，进一步致力于开源社区的数据库变革

> 原文：<https://devops.com/datical-announces-support-for-liquibase-3-7-furthering-ongoing-commitment-to-open-source-community-for-database-change/>

***其他计划包括与数据库供应商的战略合作伙伴关系，专门为 Liquibase*** 招聘人员

**得克萨斯州奥斯汀——2019 年 8 月 8 日——**[领先的数据库发布自动化解决方案提供商 Datical](http://www.datical.com/) 今天宣布支持 [Liquibase](https://www.liquibase.org/index.html) 3.7，继续努力为数百万使用开源项目管理数据库模式变更的软件专业人员提供支持。此外，Datical 正在投资与数据库供应商建立战略合作伙伴关系，包括 Cassandra、蟑螂实验室、Couchbase 和 SAP HANA 等，以增加社区贡献。Datical 目前还雇佣了多名专门负责 Liquibase 的工程师，并计划在今年雇佣更多的工程师和一名社区经理。

Liquibase 是领先的开源解决方案，使应用程序团队能够版本化和跟踪数据库的变化。版本 3.7 包括 DB2 for z/OS 的错误修复和改进，以及基于状态的管理和变更集生成的开发。它还支持 Oracle 约束上的 NOVALIDATE 关键字，因此用户能够创建或重新创建约束，而不会产生验证现有数据的成本。

“Liquibase 是开源数据库变革的无可争议的领导者，Datical 将 100%致力于继续努力支持 Liquibase 社区，”Liquibase 创始人兼 Datical 架构师 Nathan Voxland 表示。“Datical 有多名工程师专注于 Liquibase，无论是关于更新还是 bug 修复，未来几个月将有更多的社区更新。随着新版本的发布，我们很高兴能与数据库市场的领导者合作，并雇佣一名社区经理来更好地支持这些工作。我们致力于通过增加功能、新闻、文档和社区贡献来增加对 Liquibase 的投资。”

依靠 Liquibase 的功能来协调环境或为数据库迁移生成更改集的用户将受益于 Liquibase 3.7 中的关键问题修复和改进，包括:

*   **生成的变更日志中变更的原子性增加。**diffChangeLog 操作现在在每个生成的变更集中返回一个——且只有一个——变更。这种增加的原子性使用户更容易从部署错误中恢复，因为他们不必担心手动更正他们的更改日志和跟踪他们已部署到的数据库中的更改的额外开销。
*   **改进了多个目录/数据库项目的快照。**管理单个项目或应用程序的多个数据库(或目录)变更的用户将受益于跨目录功能的改进。通过比较两个环境(在线或离线)产生的依赖排序和变更集的准确性的改进，使得理解和协调应用程序管道中的差异变得容易。
*   **支持 Oracle 约束上的 NOVALIDATE 关键字。**此更新允许用户创建或重新创建约束，而不会产生现有数据的验证成本。
