# 现在是自动化测试的时候了

> 原文：<https://devops.com/the-time-for-automated-testing-is-now/>

熟练的软件测试人员仍然非常吃香，而且这种趋势似乎不会很快改变。当前的劳动力短缺和迫在眉睫的经济衰退给了软件开发领导者一个很好的理由来考虑如何吸引和留住合格的人才，更重要的是，让他们尽可能高效地工作。除此之外，软件开发周期中的趋势是[左移](https://katalon.com/resources-center/blog/shift-left-testing-approach)测试，这增加了开发人员承担测试责任的压力，这通常需要采用新的视角和使用新的工具。自动化测试解决了这些问题，它使得学习如何在更短的时间内建立一个一致的、可靠的测试程序变得更容易，需要更少的维护。

再加上[人工智能或人工智能增强的](https://devops.com/accenture-aims-to-infuse-ai-into-app-testing/)测试平台的未来前景和当前的机器学习(ML)能力，如测试脚本的自我修复，你就有了一个既能拉平学习曲线又能加快效率的平台。组织越早从手工测试过渡到自动化测试，就越能更好地抵御内部和外部市场的波动。

### 一个开发人员和一个测试人员走进一个酒吧…

传统的软件测试是一个手工的过程，在这个过程中，测试人员被雇佣来复制用户，并被期望将这种观点带到他们的工作中。因此，软件测试人员与软件开发人员有着非常不同的思维方式。当开发人员保护他们的创造性产出时，软件测试人员被好奇心驱使去探究系统的局限性。显而易见，使一个人在这些角色中脱颖而出的品质是不一样的。然而，随着测试和持续集成/持续开发(CI/CD)趋势的左移，业界已经模糊了编码人员和测试人员之间的区别，这让双方都感到沮丧。

自动化测试消除了许多与重复测试计划相关的单调工作。它允许测试人员专注于编写更好的脚本，扩展他们测试计划的范围和安排更强大的负载测试。从组织的角度来看，一旦你建立了一个可重用的测试脚本库，你的测试程序将会免受未来就业市场动荡的影响，包括衰退和减员。

### 低代码/无代码

无论您使用什么工具，如果您正在创建软件，代码都会被创建——甚至是测试脚本。然而，理想情况下，低代码/无代码工具可以隐藏一些复杂性并促进最佳实践。使用低代码/无代码工具可以通过记录手动交互和提供建议，将测试人员从从头开始创建测试脚本中解放出来。在左移测试的情况下，低代码工具可以消除学习整个测试框架的需要，同时仍然使开发人员能够在需要时访问代码。自动化测试还消除了重复性手工测试中固有的人为错误。简而言之，测试自动化不会降低熟练的软件测试人员和开发人员的价值；它增加了它。

在熟练的测试人员手中，自动化测试只会变得更强大，更有效，对你的开发周期更关键。它提升了测试人员的技能，使他们更加高效、准确和彻底。测试自动化也有助于更平稳地过渡到左移测试，允许开发人员花更少的时间测试他们的软件，花更多的时间做创造性的工作，使他们茁壮成长。在一个吸引和留住顶尖人才只会变得越来越困难的市场中，你能做的任何减轻员工负担、提高他们工作效率的事情都将是你企业的一大胜利。因此，问题不是你是否应该从手工测试过渡到自动化测试，而是什么时候。
