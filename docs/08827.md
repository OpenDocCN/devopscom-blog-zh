# 谷歌推出模糊测试集成工具

> 原文:[https://devo PS . com/Google-makes-fuzz-testing-integration-tool-available/](https://devops.com/google-makes-fuzz-testing-integration-tool-available/)

Google 今天发布了一个工具，使得将模糊测试整合到 DevOps 工作流程中变得更加简单。

模糊测试是指一种自动化的软件测试技术，涉及提供无效的、意外的或随机的数据作为应用程序的输入，以发现诸如潜在的内存泄漏等问题。

谷歌高级软件工程师乔纳森·梅兹曼(Jonathan Metzman)表示，ClusterFuzzLite 是谷歌向其内部开发团队提供的一个工具的轻量级版本。它本身不是一个模糊测试工具，而是一个工具，它使得在 DevOps 工作流中显示模糊测试的结果更加容易。一旦这些结果浮出水面，开发人员在将应用程序部署到生产环境之前解决问题就变得更加容易了。

ClusterFuzzLite 支持的核心功能包括持续模糊化、杀毒支持、语料库管理和报告生成工具。Metzman 指出，我们的目标是让更多的开发人员能够更容易地进行模糊测试，这也是采用 DevSecOps 最佳实践的整体努力的一部分。

ClusterFuzzLite 目前支持 GitHub Actions 和 Google Cloud Build，但 Metzman 表示它很容易扩展到其他 DevOps 平台。通过向持续集成/持续交付(CI/CD)工作流添加几行代码，它被设计为作为 DevOps 工作流的一部分运行。

Google 已经为开源项目提供了 ClusterFuzzLite，但是现在为商业开发者提供了额外的访问。自 2016 年发布以来，已有超过 500 个关键开源项目集成到谷歌开源软件模糊测试计划中，导致超过 6500 个漏洞和 21000 个功能错误被修复。

在美国国家标准和技术研究所(NIST)最近发布的指南(这些指南本身是对拜登总统发布的行政命令的回应)之后，使用模糊测试的组织数量预计将会增加，因为 NIST 已经确定模糊测试是代码验证的最低标准要求。

几年来，谷歌一直在努力提高开源软件的安全性。所有开发人员面临的挑战是找到一种方法，在不降低部署速度的情况下构建更安全的应用程序。模糊测试使得在应用程序开发生命周期的早期发现问题成为可能，而不是在应用程序部署之前。

当然，不太清楚的是，开发人员是否开始自己使用这些工具，或者越来越多的采用是因为他们的组织一致努力采用 DevSecOps 最佳实践。然而，不管动机如何，随着开发人员调用更高级的测试工具变得越来越简单，应用程序安全性的整体状态应该会继续改善。

与此同时，在一系列引人注目的安全违规事件发生后，各种规模的组织都在审视自己如何构建软件。至少在理论上，对软件供应链越来越多的关注应该可以防止问题在部署后出现，因为修复这些问题的成本会高得多。
