# Instana 成为第一个自动收集预先存在的 W3C 用户计时事件的 APM 解决方案

> 原文：<https://devops.com/instana-becomes-first-apm-solution-to-automatically-collect-pre-existing-w3c-user-timing-events/>

***公司发布自动支持 W3C 用户计时和跟踪上下文 API***

**伊利诺伊州芝加哥****——2020 年 5 月 28 日——**面向云原生微服务应用的实时[应用性能管理](https://instana.com/application-management) (APM)解决方案的领先提供商 Instana 今天宣布推出首款 APM 解决方案，集成来自 W3C 跟踪上下文和用户计时 API 的数据。在这个版本中，Instana 自动识别定制的 W3C 事件并使用它们，不需要修改代码。

Instana 技术总监兼 APM 策略师 Chris Farrell 表示:“开发人员和运营人员需要最大的可观察性，以便尽可能有效地工作。通过将来自专有来源的数据与 W3C 用户计时和跟踪上下文相结合，Instana 提供了两种可见性世界的最佳组合，这是一种以最小的努力提供最大可见性的强大方法

Instana 对 W3C 用户计时 API 的支持实现自动收集所有预先存在的 W3C 用户计时事件，而无需开发人员添加自己的 JavaScript API 调用来将事件放入他们的分析平台。与 Instana 解决方案中的其他性能实体类似，W3C 用户计时 API 可用于 Instana 的 Unbounded Analytics 引擎中的分析，允许跨多个配置文件的模式被发现和协作分析。

“W3C 分布式跟踪工作组通过[跟踪上下文规范](https://www.w3.org/TR/trace-context/)定义了分布式跟踪上下文传播的格式和语义，为各种跟踪工具、库和平台的用户提供了集成体验，同时为创新和实验留出了空间，”W3C 分布式跟踪工作组编辑兼联合主席 Sergey Kanzhelev 说道。“随着云计算和现代平台的兴起，Trace Context 等标准变得越来越重要，尤其是随着应用程序开发进入新的抽象级别，标准提供了实现不同应用程序组件的兼容性和互操作性的构建块，从而简化了开发流程并加快了上市时间。”

Instana 对 W3C 跟踪上下文的支持包括将跟踪与 AutoTrace 相关联，AutoTrace 是 Instana 的自动化分布式跟踪技术。然后，Instana Analytics 引擎(称为 Unbounded Analytics)中的每个跟踪(无论来源如何)都是可用的，允许开发人员和运营人员找到任何问题的根源，并跨不同的跟踪源轻松地将上下文信息从一个事务传递到下一个事务。

与传统的 APM 工具不同，Instana 的自动化[应用性能监控](https://instana.com/application-monitoring) (APM)解决方案可发现所有应用服务组件和应用基础设施，包括 AWS Lambda、Kubernetes 和 Docker 等基础设施。Instana 自动为应用技术堆栈的每个部分部署监控传感器，跟踪所有应用请求并分析每个流程，无需任何人工配置，甚至无需重启应用。该解决方案实时检测应用程序和基础架构的变化，调整自己的模型，并在几秒钟内可视化这些变化和任何性能影响。

在[https://instana.com/w3c](https://instana.com/w3c)了解更多关于 Instana 及其 W3C 跟踪上下文和用户计时 API 的信息。
