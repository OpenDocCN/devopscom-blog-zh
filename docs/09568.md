# 宗教推出脆弱性优先化平台

> 原文:[https://devo PS . com/rezi lion-launchs-漏洞-优先级-平台/](https://devops.com/rezilion-launches-vulnerability-prioritization-platform/)

Rezilion 今天宣布[一个平台的正式上市，该平台通过识别哪些漏洞在内存中运行并实际影响可执行的类或函数，使开发运维团队能够更好地确定补救工作的优先级](https://www.prnewswire.com/news-releases/rezilion-unveils-broad-lineup-platform-enhancements-providing-organizations-with-holistic-and-automated-toolset-to-accelerate-software-security-301583841.html)。

Rezilion 首席执行官 Liran Tancman 表示，DevSecOps 组织目前面临的最大挑战是，开发人员负责修复的大部分漏洞并不构成重大的网络安全威胁。

Rezilion 创建了一个可搜索的专有下一代漏洞数据库(NGVDB ),可以识别哪些问题会影响特定的类和功能。塔克曼说，这种能力使 DevOps 团队能够降低近 95%的软件漏洞的优先级。

他说，此外，Rezilion 平台将提出补救建议，降低中断量和/或成本。例如，全自动修复功能将[自动升级和测试持续集成(CI)管道中被发现可利用的易受攻击的软件组件和软件包](https://digitalanarchist.com/videos/interviews/automating-devsecops-rezilion)。塔克曼说，从长远来看，Rezilion 还致力于提出补救建议，这些建议可能会使用修补的替代方法来解决。

他补充说，Rezlion 平台还与 ServiceNow 等 IT 服务管理平台和吉拉等项目管理工具相集成，以便更容易地解决现有工作流程中的漏洞。

![](../Images/92fb818e62fcf47309e9872f57823ba0.png)

塔克曼指出，如今大多数开发人员只是被网络安全团队例行要求他们修补的漏洞数量所淹没。他说，目标应该是使开发者能够将他们有限的时间和精力集中在修复对确保应用程序安全真正重要的漏洞上。

此外，开发人员需要更有信心，他们被要求做的补丁不会破坏他们现有的应用程序，塔克曼补充说。

当然，漏洞优先级是 DevOps 和网络安全专业人员之间长期分歧的核心。经常会创建充满漏洞长列表的电子表格，但是通常没有提供任何给定漏洞造成的威胁有多严重的上下文。因此，开发人员会分配一小部分时间来应用补丁，这些补丁是基于直觉和复杂程度而不是严重性来划分优先级的。

很自然，开发人员会将大部分时间分配给创建新的应用程序和功能。任何 DevSecOps 计划要取得成功，关键是要确保最佳利用分配给补救的时间。网络安全团队将继续编制漏洞列表，但该列表现在可以以引起最小摩擦的方式进行分类。

在一系列引人注目的安全漏洞之后，现在比以往任何时候都更加关注保护软件供应链。挑战在于找到一种方法，在应用程序部署到生产环境之前和之后更好地保护它们。部署应用程序后，总会发现一些漏洞。然而，仅仅因为漏洞存在，并不意味着它是可利用的。如今应用程序安全性的问题是，补救工作往往是浪费时间。
