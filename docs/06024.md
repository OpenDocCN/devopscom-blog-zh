# 通过人工智能的可观察性，在 Kubernetes 的复杂性上穿针引线

> 原文:[https://devo PS . com/threading-the-needle-on-kubernetes-complexity-with-ai-powered-observability/](https://devops.com/threading-the-needle-on-kubernetes-complexity-with-ai-powered-observability/)

如果你仔细查看主要行业的清单——运输、零售、医疗保健、汽车、金融服务——你会发现每个行业的主要参与者基本上都变成了软件公司。他们做什么，如何服务他们的客户，完全取决于他们的软件工作是否完美。支撑这一点的是一场大规模的数字化转型，这场转型让组织摆脱了企业 IT 铁板一块的陈旧、静态方式。相反，今天的企业正在更加动态的 IT 环境中工作——敏捷、多云，尤其是基于容器和微服务的新数字服务。所有这些都意味着比以往任何时候都更加复杂的 IT 环境。

## **Kubernetes 带来了更多的数据，但这影响了可观察性**

这种更复杂的环境的一个关键部分是过去几年 Kubernetes 的采用率飙升。在 [对 5，000 家企业进行的一项调查](https://www.cncf.io/blog/2018/08/29/cncf-survey-use-of-cloud-native-technologies-in-production-has-grown-over-200-percent/) 中，40%的企业表示他们目前在生产中使用 Kubernetes，超过 80%的企业表示他们采用了 Kubernetes，而不是其他任何集装箱管理系统。 [另一项最近的调查](https://www.dynatrace.com/global-cio-report/) 显示，68%的首席信息官已经在使用容器，总共有 86%的首席信息官预计将在明年部署容器。企业 IT 越来越倾向于使用容器，尤其是 Kubernetes 来编排他们的容器化应用程序。

推动组织转向容器和 Kubernetes 的是给予开发团队更快发布服务、功能或应用的灵活性。但是，更快并不一定意味着更好或更高的质量，除非您也应用云原生架构模式，将质量关纳入您的交付渠道，采用新的部署模型(如 canary 或功能切换),并提供从生产到工程的全面可见性和短反馈循环。

Kubernetes 本身提供了对节点或 pod 健康状况的深入了解。Kubernetes 甚至通过其内置的健康检查机制来确保您的 pod 的健康。多亏了服务网格等组件，您还可以获得关于 pods 中运行的服务之间的依赖关系的实时信息。理论上，这些数据可以为组织提供对其环境的必要洞察，以了解他们的组织是否正在更快地向最终用户释放更好的价值，并释放新的业务价值。

我之所以说这些是理论上的好处，是因为在实践中，所有这些额外的数据并不一定意味着更多的可见性。这通常意味着更少的可见性，以及对环境中不同元素如何影响生产部署速度、整体系统性能和最终用户体验的更少了解。我见过 Kubernetes 环境，其中充斥着应用程序、容器、微服务和云之间数十亿的相互依赖关系。要求人类手动跟踪这种级别的数据现在是一项不可能的任务。依靠传统的手动方法来管理和监控 IT 环境会降低组织应该从 Kubernetes 中获得的价值。

## **Kubernetes 环境需要全栈可观测性****——****这意味着需要确定性 AI**

充分发挥 Kubernetes 的潜力意味着能够观察到整个技术堆栈和数十亿个相互依赖项。但正如越来越多的组织发现的那样，没有人工智能，这是不可能发生的。具体来说，一个确定性的人工智能引擎。事实上，在[2019 年的一项调查](https://www.dynatrace.com/global-cio-report/) 中，88%的首席信息官表示，他们将人工智能视为克服 IT 复杂性的关键解决方案。只要 Kubernetes 在助长复杂性，克服这种复杂性就意味着让确定性人工智能成为任何 Kubernetes 环境不可或缺的一部分。

确定性人工智能应用特定算法来检测系统异常，并将其收集的依赖性信息用于检测这些异常背后的根本原因。当确定性人工智能引擎吸收关键的 Kubernetes 事件时，如状态变化和工作负载变化，它提供了对 Kubernetes 堆栈的相互依赖性的可见性。这种人工智能驱动的全栈可观察性非常重要，因为它为 it 环境提供了新的功能，包括检测和映射容器，管理和分配容器资源，以及深入研究工作中的容器运行时。所有这一切都是可能的，因为你只能通过确定性人工智能获得可观察性水平；传统的指标和仪表板根本无法与之竞争。

通过在 Kubernetes 环境中提供完整的堆栈可见性和相互依赖性映射，确定性人工智能使 IT 团队能够实时识别业务服务可用性下降的根本原因——从可靠性和资源使用到系统性能和用户体验——比手动识别速度更快。这种自动化的速度和可见性也使得在这些问题影响到最终用户之前，可以在源头上立即解决这些问题。确定性人工智能提供的自动化也融入了 DevOps 的生产管道，加快了新部署的发布和反馈周期，并允许团队更快地将用户反馈纳入下一次迭代，以获得更好的体验。

简而言之，确定性的人工智能引擎让 IT 和他们要服务的客户的生活变得更美好。确定性人工智能承担了过去人工完成时耗时、乏味且违反直觉的工作，并快速自动地执行这些工作。这就解放了过去从事这项工作的团队，让他们专注于更令人满意的工作，并推动新的商业价值。

## **人工智能驱动的可观察性依赖于质量数据和质量监控**

尽管如此，确定性人工智能在 Kubernetes 环境中提供可靠、精确的全栈可观测性的能力依赖于为其提供可处理的一致质量的数据。算法依赖于人工智能引擎摄取的数据。如果没有全面使用的标准监控功能，那么您将会遇到不同团队使用他们自己的首选监控工具的情况。Kubernetes 集群越大，监控工具就越多样化。监控工具的多样性越大，输入确定性人工智能引擎的数据就越不一致。

摆脱这种困境的方法是利用单一监控工具作为 Kubernetes 集群的自助服务功能，该工具可以支持引入堆栈的各种技术，并提供开发人员、IT 管理员和确定性人工智能都依赖的系统和事件数据的基准标准。换句话说，当你向人工智能输入一致的、高质量的数据时，你就让它变得更聪明了。一个更智能的确定性人工智能引擎提供了组织所需的全栈 Kubernetes 可观察性。

## **在 Kubernetes、确定性人工智能和可观测性之间建立正反馈回路**

Kubernetes 和确定性 AI 是天生的一对。Kubernetes 为管理和检测容器化应用程序提供了一个平台。确定性人工智能提供了绘制 Kubernetes 堆栈的工具，提供了对相互依赖和退化的洞察，并在它们影响最终用户之前解决它们。来自 Kubernetes 的数据让人工智能变得更好，更智能的确定性人工智能引擎意味着在你的技术栈中更精确的可观察性。

这是一个自我强化的反馈循环，消除了容器导致的 It 复杂性的常见缺陷。相反，它创建了可见、一致和高质量数据的新标准，组织可以利用这些数据来实现更快、更好的部署，进而带来更好的用户体验和更大的业务价值。
