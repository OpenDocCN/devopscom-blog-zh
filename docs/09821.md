# ServiceNow 收购 Era Software 以统一可观测性

> 原文:[https://devo PS . com/service now-acquisites-era-software-to-unify-observability/](https://devops.com/servicenow-acquires-era-software-to-unify-observability/)

ServiceNow 今天宣布，它已经同意[收购 Era Software](https://www.businesswire.com/news/home/20221005005278/en/ServiceNow-to-Acquire-Observability-Innovator-Era-Software-Helping-Businesses-Turn-Data-Driven-Insights-into-Action) ，作为该公司去年收购的 Lightstep [observability 平台](https://devops.com/servicenow-acquires-lightstep-to-gain-observability-platform/)增加日志数据支持的努力的一部分。

ServiceNow 的 Lightstep 业务部门总经理本·西格曼(Ben Sigelman)表示，与其重新发明 Era Software 已经拥有的相同日志管理功能，不如将该功能与收集指标和跟踪的 Lightstep 平台集成起来更有意义。

他补充说，这两个平台的集成将使统一可观测性成为可能，而不是要求 DevOps 团队导航两个独立的筒仓。

Sigelman 说，随着可观测性的不断发展和成熟，最直接的目标是使导航警报风暴变得更简单，警报风暴在任何时候出现中断都不可避免地随之而来。他指出，警报风暴通常发生在发生变化之后，但揭示这种变化究竟如何影响 IT 环境是一项挑战。

Sigelman 补充道，可观测性平台通过呈现一组指导开发团队完成该过程的统计数据，减少了实现该目标所需的时间和精力。当然，挑战总是在于找到一种有效的方法来存储产生的所有遥测数据。事实上，随着组织开始部署基于微服务的云原生应用，许多组织开始看到遥测数据量呈指数级增长。

Era Software 使用一个对象存储系统来索引和分析从云应用程序、Kubernetes 和 IT 基础架构平台收集的日志数据。该公司还推出了测试版 Era Streams 工具，该工具通过实时动态重新配置数据管道，在不牺牲保真度的情况下减少了收集的可观测性数据量。

一旦收购完成，日志数据分析将与 IT 服务管理(ITSM)工具共享，这些工具是 ServiceNow 与 Lightstep observability 平台一起提供的。Sigelman 说，我们的目标是更容易地将这些 it 事件与其对业务的潜在影响联系起来。

Era Software 首席执行官托德·珀森(Todd Persen)表示，10 年的长期目标是达到自动检测和修复问题的地步。与此同时，IT 团队遇到的上下文切换的数量可以通过统一的可观察性平台大幅减少。

当然，目前不缺乏统一的可观测性平台。DevOps 团队多年来一直依赖跟踪特定指标的遗留监控工具。问题是这些传统的监控工具只能跟踪范围很窄的预定指标。随着 IT 环境变得越来越复杂，启动更多开放式查询来确定 IT 中断的根本原因的需求变得越来越明显。

在可观察性平台被广泛采用之前可能还需要一段时间，但是对于任何管理现代 It 环境的人来说，对它们的需求是显而易见的。不太清楚的是，可观测性平台将在多大程度上消除对遗留监控工具的需求；然而，现在的重点应该是获得尽可能多的可见性和透明度，以了解仍然不透明的 IT 环境。
