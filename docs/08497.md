# SaaS:为有开发意识的人打造工具

> 原文:[https://devo PS . com/SaaS-building-tools-for-the-devo PS-minded/](https://devops.com/saas-building-tools-for-the-devops-minded/)

开发运维、持续交付、持续集成和持续测试不再仅仅是方法和开发策略，它们都是科技行业中不断增长的经济的一部分。

伟大的软件即服务(SaaS)公司，如 [Chef](https://www.chef.io/chef/) 、 [Docker](https://www.docker.com/) 和 [New Relic](https://newrelic.com/) 在早期发现趋势并围绕新一波开发流程构建他们的工具和服务方面做得非常出色。据报道，仅厨师和码头工人就筹集了近 2.5 亿美元，New Relic 于 2014 年上市。

事实上，这个网站是这些新的发展实践如何产生影响的另一个例子——因为几个专门的新闻媒体已经从这些革命性的发展战略中诞生了。因此，DevOps 方法不仅帮助开发人员更快地创建更好的产品；但这一过程为技术专家提供了机会，让他们可以在像这样的网站、电子书、网络研讨会、会议以及任何其他人们交流的方式上分享他们的经验。

在 DevOps.com，你会看到网上讨论不同组织如何定义 DevOps 的最佳文章，为什么财富 500 强公司正在将旧的实践转向 CD，以及来自行业专家的 CI 最佳实践。

在成为对 DevOps 最友好的解决方案的竞赛中——无论是哪个领域——已经有数百家初创公司成立并获得了资助，这只是增强了数千名开发人员的工具箱。你很难找到一家好的 SaaS 公司不提供与 Jenkins CI、TeamCity 或 Chef 的集成，这是因为 SaaS 的产品不仅应该解决一个重要的问题，而且应该对你当前的开发过程产生最小的干扰。

综上所述，[持续测试](https://devops.com/2015/05/27/continuous-testing-what-exactly-is-it/)是当今技术中最令人兴奋的领域之一。持续测试是一个旨在使测试和 QA 与开发和运营同步的过程，最终更快地产生更好的产品。

在一个团队的软件开发生命周期中，有几个因素使得持续测试成功——例如开源协作解决方案、测试自动化 API 和跟踪应用程序历史性能的工具。

随着[Load Impact 3.0 版本](https://loadimpact.com/)的发布，全球性能和负载测试 SaaS 公司宣布了它的最新功能，正好符合 DevOps 的心态:[性能趋势](http://www.virtual-strategy.com/2015/06/15/load-impact-releases-version-30-performance-testing-platform#axzz3d8QAlG2Q)。

性能趋势分析允许用户绘制一段时间内相同测试的运行情况。这为公司提供了一个清晰的数据视图，可以定位性能下降或改善的模式，从而轻松验证代码和基础设施的变化。

本着持续测试的精神，DevOps、CD 和 CI —性能趋势分析提高了用户获得其系统性能的低级和高级视图的能力。随着 DevOps 专业人员运行的测试数量的增加，性能趋势为关注其应用程序性能的公司提供了有价值的信息——应该是每个人。

在 DevOps.com 和世界各地的许多大型会议之间，可以肯定地说，这种开发工具的经济在未来几年只会继续增长。
