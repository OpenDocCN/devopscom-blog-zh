# Rocky Linux 成为 CentOS 的替代品

> 原文：<https://devops.com/rocky-linux-emerges-as-a-centos-replacement/>

在 Red Hat 最近决定将其重点从[CentOS](https://www.centos.org/)—[Red Hat Enterprise Linux(RHEL)](https://www.redhat.com/fr/enterprise-linux-8)的开源版本转移之后，Rocky Linux 被创建来为社区提供一种替代方案。然而，Rocky Linux 的创造者 Gregory Kurtzer 也是 Ctrl IQ 的创始人兼首席执行官，他说，Linux 发行版的流行也可以归因于它的创造者如何基于 CentOS 开发过程中获得的经验教训寻求提供一个更好的替代方案，而不是对红帽有争议的举动的强烈反对。

“正如人们可能想象的那样，从 CentOS 的成立中吸取了很多教训——我们做了很多正确的事情，但我们也犯了一些错误，这导致了项目经历的一些困难，”Kurtzer 说。“我过去没有——现在也没有——对红帽或其他任何人不满。重要的是要明白公司有法律责任去做他们需要做的事情。但我要说，这就是为什么开源社区不应该由一家公司控制，特别是如果这家公司有开源项目的竞争产品。”

除了应用从 [CentOS](https://devops.com/?s=CentOS) 项目中获得的经验来帮助“在新项目中重建 CentOS 的初始目标”，Kurtzer 说，在 Red Hat 决定终止(EOL) CentOS 之后，他被鼓励创建 Rocky Linux。此举“影响了我的公司，我们的客户，以及整个行业，”库尔茨说。

“坦率地说，我的知识——在自己做这件事方面——已经严重过时了。幸运的是，在我宣布我正在考虑重建 CentOS 后，成千上万的人找到我，希望成为这个项目的一部分并做出贡献。"由此，我们的开发团队和开源社区诞生了."

此外，CentOS Stream 通过允许“我们和其他人”参与开发上游企业 Linux 标准，为社区提供了价值，Kurtzer 说，他补充说，他的团队将继续调试 Rocky Linux，以便解决方案能够正确集成到[企业 Linux 标准中。](https://www.redhat.com/fr/resources/red-hat-enterprise-linux-server)

“必要性是绝对清楚的，所以我创立了洛基 Linux 和[洛基企业软件基金会(RESF)](https://linux.developpez.com/actu/315628/Rocky-Enterprise-Software-Foundation-annonce-la-sortie-de-la-premiere-release-candidate-de-Rocky-Linux-8-3-le-projet-Linux-choisi-pour-remplacer-CentOS-fait-un-pas-vers-sa-premiere-version-stable/) ，”库尔策解释道。“我们与我们的贡献者、赞助商和合作伙伴一起，为了一个共同的目标而团结起来:创建一个开源社区，为我们所有人提供一个经得起时间、损耗和商业议程考验的企业操作系统。”

当 Kurtzer 第一次向社区提出这个项目的想法时，人们对 CentOS 替代品的开发产生了浓厚的兴趣。根据 Rocky Linux Tier0 mirror 的数据，自 6 月发布以来，ISO 下载总数为 56，000 次。

“我们对来自其他镜像的 ISO 下载一无所知，但如果让我猜的话，我会说总共会多两到三倍，”Kurtzer 说。“当我们都有时间休息时，看看有多少人在使用 Rocky Linux，并接受社区提供的所有反馈，这将是一件有趣的事情，”他说。

Kurtzer 说，这个开发团队花了大约 7 个月的时间来构建通用(GA)版本。“我们并不是直接投入到构建操作系统中，因为我们的目标是创建一个基础设施，让许多贡献者和开发者都可以参与其中，而不会将产品的安全性置于风险之中。构建基础设施花了大约四个月的时间。“一旦基础设施完成，我们花了大约两个月的时间来构建第一个发布的测试候选，然后花了一个月的时间来确保我们做的一切都是正确的。”
