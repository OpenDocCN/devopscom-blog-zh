# Sentry 收购 Specto，为可观察性增加分析工具

> 原文：<https://devops.com/sentry-acquires-specto-to-add-analytics-tool-for-observability/>

Sentry 本周宣布已经收购了 Specto T1，这是一家分析移动应用性能的工具提供商。

随着部署的移动应用数量不断增加，开发人员被要求管理这些应用，这是整体左移的一部分，要求他们对其应用在生产环境中的运行情况承担更多责任。

Sentry 首席执行官米林·德赛(Milin Desai)今天表示，Specto 工具的重点是让开发者更容易确定移动应用程序如何运行，但计划是将这种分析能力扩展到更广泛的应用程序。

随着应用程序的责任继续向开发人员转移，开发人员对易于使用的可观察性和监控工具的需求变得更加迫切。Sentry 已经提供了一个 JavaScript 软件开发工具包，使开发人员能够在他们的应用程序中插入少量代码来跟踪，例如，应用程序失败了多少次。Desai 表示，Specto 工具将使开发人员能够分析更广泛的收集指标，从而将这些可观察性能力推向更深层次。

当然，DevOps 团队也已经在收集指标了。Sentry 与 GitLab 和 GitHub 等合作伙伴合作，提供了一个插件，使 DevOps 团队可以看到开发人员现在用来管理应用程序的相同指标。德赛说，目标是让开发者和 it 运营团队更容易围绕一套通用指标进行合作。

当然，管理应用程序的责任向左转移的速度差别很大。一些组织继续倾向于让开发人员将主要精力放在编写代码上。然而，Desai 表示，Sentry 正在遇到更多的情况，即站点可靠性工程师(SREs)向开发人员提供 Sentry 工具，作为向他们提供更多关于他们的应用程序在生产环境中如何运行的见解的努力的一部分。

过去，IT 运营团队依赖应用性能管理(APM)平台来管理 IT 运营。这些平台现在正在演变成[可观察性](https://devops.com/?s=observability)平台，这些平台承诺通过在应用程序和 IT 基础架构平台上关联事件来提供更多上下文。然而，这种方法通常需要开发人员将代理软件插入到他们的应用程序中。一旦插入，该代理软件需要在更大的 DevOps 工作流的上下文中进行维护和更新。

Sentry 方法为开发人员提供了一种从应用程序中收集数据的工具，在嵌入到应用程序中后，只需消耗几十千字节的内存。这种方法消除了依赖大得多的代理软件来实现可观测性的需要。

不管开发人员如何检测他们的应用程序，随着应用程序环境变得越来越复杂，可观察性的整体状态显然需要改进。现在的问题是确定如何以对开发人员和 DevOps 团队都有利的方式最好地实现这个目标。
