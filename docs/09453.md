# LogicMonitor 扩展了可观测性平台的范围

> 原文：<https://devops.com/logicmonitor-expands-reach-and-scope-of-observability-platform/>

在 LogicMonitor 的 [LM Elevate](https://elevate.logicmonitor.com/?) 用户大会上，logic monitor 今天正式推出了 [LM Envision](https://www.businesswire.com/news/home/20220602005392/en/LogicMonitor-Delivers-Unified-Observability-to-Address-Today%E2%80%99s-IT-Data-Supply-Chain-and-Digital-Business-Unveils-LM-Envision-Platform) ，这是一个包含人工智能(AI)的可观察性平台，可以发现问题，然后进行补救。

LogicMonitor 首席执行官 Christina Kosmowski 表示，LM Envision 是唯一通过软件即服务(SaaS)模式提供的可观测性平台，可应用于内部和云应用程序。她指出，作为开放式 LM Envision 生态系统的一部分，该平台总共已有 2000 多个集成。

在过去的一年中，已经开发了额外的功能并添加到可观测性平台中。其中包括用于识别瓶颈和优化性能的应用性能监控(APM)工具、对公共云和 Kubernetes 环境的扩展支持、与 StackStorm automation 平台的集成、与 Terraform infra structure-as-code 工具的集成以及用于搜索日志的更高级工具。

该公司还扩展了其人工智能操作工具，通过聚合日志和包括拓扑工具和图表，在解决问题时提供更多上下文。

最后，LogicMonitor 添加了推送指标应用程序编程接口(API)以简化数据接收、预集成仪表板和对开源 OpenTelemetry 代理软件的支持。

Kosmowski 说，可观察性和 AIOps 最终能够花更少的时间管理平台，这只是时间问题。IT 团队面临的主要挑战之一是，他们使用的每个平台都需要自己的框架来监控和管理它。她指出，随着可观察性和管理框架的融合，与管理现代 IT 环境相关的总体开销应该会稳步下降，因为机器学习算法使问题在造成重大破坏之前更容易浮出水面。

Kosmowski 补充说，总体目标是关闭发现 IT 问题和补救之间的环路。

当然，可观察性是任何 DevOps 最佳实践的核心原则之一。然而，实现它一直是一个挑战。现在的目标是最终用一个可观察性平台取代跟踪有限的预定义指标集的监控工具的大杂烩，该平台不仅聚合指标，而且使开发运维团队能够发起查询和解决 IT 问题。在某些情况下，监测工具的合理化将有助于证明购买可观察性平台的合理性。矛盾的是，随着 it 环境变得越来越复杂，管理这些环境实际上可能变得越来越容易，因为更多的应用程序被工具化，而将 IT 团队相互隔离的各种孤岛开始崩溃。

与此同时，不缺乏可观测性平台选项。Kosmowski 说，它们之间的主要区别是它们提供的集成程度和它们采用 AIOps 自动解决问题的能力。她补充说，DevOps 团队没有花在管理 IT 平台上的每一分钟都可以用来刺激创新。

无论如何，DevOps 团队将很快获得比以往更多的关于其 IT 环境的信息。不太清楚的是，可观察性平台收集的数据中有多少将真正构成可操作的情报。但是随着可观察性的不断发展，现在更多的是目标实现的程度，而不是是否或何时实现。
