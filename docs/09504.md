# TechStrong 缺点:经济低迷带来了更多的开发紧迫感

> 原文：<https://devops.com/techstrong-con-downturn-brings-additional-sense-of-devops-urgency/>

不管整体经济是否正在经历调整或处于衰退的边缘，随着资源变得更加有限，组织都将优先考虑一些项目。

在“DevOps:泡沫破裂了吗？”由 DevOps 高管和风险投资公司投资者组成的小组今天启动了虚拟的 tech strong Con 活动，小组成员表示，组织需要继续投资 DevOps 自动化和网络安全，以确保他们的长期生存。

然而，Rain Capital 的普通合伙人王晨曦表示，企业能够同时投资的 IT 项目数量可能会下降。“我们开始看到项目放缓，这是肯定的，”她说。

从根本上说，组织需要决定的不是他们是否要现代化遗留的单一应用程序，而是以什么样的速度。王指出，超过 70%的组织代码库仍然基于遗留的整体架构。

JFrog 首席执行官什洛米·本哈姆说，没有人确切知道当前的经济紧缩会持续多久，但是，正如在任何经济低迷时期一样，各组织都将寻求节省现金。DevOps 计划将继续得到优先考虑，因为否则，组织可能会远远落后，可能永远无法恢复，他补充道。“DevOps 是一个好地方，”本哈姆说。

Insight Partners 董事总经理 Thomas Krane 指出，事实上，到目前为止，许多 DevOps 公司尚未看到需求下降，因为组织正在投资自动化以降低其技术债务成本。“这是生死攸关的问题，”他说。

与此同时，Krane 表示，组织可能希望认真审视他们所依赖的推动创新的供应商的盈利能力。他指出，那些一直依赖风险资本资金维持运营的供应商可能很快就会发现，随着资本变得稀缺，资金变得更加难以获得。

Incredibuild 首席执行官塔米·马泽尔·沙查尔补充说，核心问题将是如何快速产生投资回报。“将会有更多的关注投资回报率，”她说。

与此同时，关于什么是 DevOps 的争论仍在继续。一些人将 DevOps 狭义地定义为一组围绕持续集成/持续交付(CI/CD)平台的开发活动。DevOps 的其他定义包括从应用程序测试到可观察性和应用程序安全性的所有内容。然而，无论众所周知的 DevOps 帐篷有多大，越来越多的组织开始采用 DevOps(现在是 DevSecOps)工作流。

面临的挑战是，在管理层对未来充满不确定性的情况下，将这些努力集中在为企业带来最大价值的项目上。当然，没有什么比迫在眉睫的危机更能让人集中注意力了；不管是好是坏，这将要求更多的组织诚实地评估他们当前的能力。
