# CloudBolt 向混合管理平台添加云成本工具

> 原文:[https://devo PS . com/cloud bolt-adds-cloud-cost-tool-to-hybrid-management-platform/](https://devops.com/cloudbolt-adds-cloud-cost-tool-to-hybrid-management-platform/)

CloudBolt Software 本周为亚马逊网络服务(AWS)和微软 Azure 公共云增加了成本管理功能，以及与其他平台的无代码集成到其混合云管理平台。

CloudBolt 首席营销官 Grant Ho 表示，该公司平台的 [Spring Release](https://www.globenewswire.com/news-release/2021/06/08/2243688/0/en/CloudBolt-Enhances-Cost-Management-Compliance-and-Governance-and-Codeless-IT-Integrations-with-New-Technology-Release.html) 现已全面上市，使 IT 运营团队能够更少地依赖定制代码来集成混合云计算环境的各种元素。Spring 版增加了对 VMware vRealize Automation 8 自动化平台、由 ServiceNow 作为云服务提供的配置管理数据库以及来自网络安全管理软件产品的 IP 地址管理平台的支持。

![CloudBolt](../Images/74467ded4a9e2521d7ba6c06a6364dc7.png)

在新冠肺炎疫情之后，CloudBolt 打赌更多的组织将寻求集中化 IT 管理，作为降低 IT 总成本的一部分。Ho 指出，添加到平台中的成本管理功能使 IT 团队能够根据成本将自己定位为某类工作负载应该在哪里运行的独立仲裁者，而此时组织从来没有像今天这样有这么多的选择。

CloudBolt 最近发布的[调查](https://devops.com/survey-illuminates-long-dark-road-to-hybrid-cloud/)发现，80%的受访者优先考虑实时洞察成本优化和安全补救。总共有 71%的人希望自助服务对最终用户来说更容易，而 62%的人希望一种不需要深厚的领域知识或特殊专业知识的集成方法。

整整 94%的受访者还指出，混合云计算对于数字化转型至关重要。共有 92%的受访者还指出，通过自助服务为终端用户提供支持是数字化转型的加速器，而 99%的受访者表示，成功的自动化依赖于简化集成。

超过四分之三(78%)的受访者表示，他们缺乏优化云部署所需的可见性。超过一半(56%)的人也承认他们的自助服务产品无法满足用户需求，超过四分之三(76%)的人在集成时仍然依赖某种形式的定制编码。

随着 IT 环境变得更加分散，大多数企业 IT 组织都试图在成本和灵活性之间取得平衡。每次向 IT 环境中添加新平台时，IT 的总成本都会上升，因为需要为管理这些平台配备额外的控制台。面临的挑战是找到一种方式来集中管理 IT，以减少必须掌握的控制台数量，而不要求每个业务部门在一个平台上实现标准化，从而将一个组织局限于一个提供商。

目前还不清楚 It 组织将在多大程度上依赖由管理员操作的传统 IT 服务管理(ITSM)平台和由站点可靠性工程师(SREs)部署的 DevOps 平台来实现这一目标。然而，无论如何，分布式计算环境的成本需要降低，即使它们变得越来越复杂。现在的主要问题是，在每个应用程序所有者都倾向于将自己视为自己 IT 领域的主人的 IT 环境中，如何最好地实现这一目标。
