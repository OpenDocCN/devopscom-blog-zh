肯·切尼:首先，AWS re:Invent，这是一场令人印象深刻的表演。所以如果人们还没有去，我建议他们明年把它写进去。但在 re:Invent 上，我们宣布——实际上是 AWS 宣布，这是他们的产品，一个名为 AWS OpsWorks for Chef Automate 的新产品。它首次将 Chef 商业平台的全部功能作为托管服务提供给 AWS。因此，他们提供支持，并百分之百地支持该产品。

因此，这是一种新的产品，它使云以及内部基础架构中的配置、部署和扩展工作变得简单而安全。我们通过真正做 Chef 多年来最擅长的事情来实现这一目标，那就是将基础设施自动化为代码。

我认为为了突出我所说的，re:Invent 的 Jassy 特意强调了混合动力是他们眼中的新常态。因此，我们使用 AWS 所做的有趣的事情是，您可以使用 AWS 在云中管理您的本地工作负载，并且您可以在如何管理两者之间的资源方面保持一致。

肯·切尼:对。如果是这样的话，那么可管理性和可管理性中的这种一致性就变得非常重要，这就是 Chef 发挥重要作用的地方，它是真正允许您拥有一个平台的桥梁，您可以在一个地方存储这些配置，在这个地方您的资源每次都可以完美地工作，您可以在这个地方进行更改，并将这些更改传播到这些环境中。因此，无论您是在大型机环境下运行，还是在云中进行开发、测试和生产，您都会获得可重复性。
