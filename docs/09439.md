# Broadcom 收购 VMware 以扩大软件产品组合

> 原文:[https://devo PS . com/Broadcom-acquisites-VMware-to-expand-software-portfolio/](https://devops.com/broadcom-acquires-vmware-to-expand-software-portfolio/)

Broadcom 今天扣动扳机，以 610 亿美元的现金和股票收购 VMware，这将极大地扩展其软件产品组合。

Broadcom 软件集团总裁 Tom Krause 告诉投资者，一旦交易完成，Broadcom 基础设施软件集团将成为 VMware 的一部分。然而，该交易包括一个为期 40 天的“去商店”选项，VMware 可以向另一个可能愿意以更高价格收购 VMware 的追求者寻求帮助。

Krause 表示，如果交易完成，Broadcom 将致力于将其 DevOps、AIOps 和安全产品组合与 VMware 产品组合相融合。博通在 2018 年收购了 CA Technologies，在 2019 年收购了赛门铁克的企业部门。

Krause 表示，Broadcom 计划投资于核心的 VMware vSphere、vSAN 和 vRealize 管理软件平台，这些平台为 VMware 带来了最大的收入。Broadcom 还承诺在未来三年内将 VMware 收入从 47 亿美元增加到 85 亿美元。Krause 表示，这一增长的主要驱动力将是通过消除合并后公司的重复职能以及 VMware 平台上运行的工作负载量的预计增长而实现的节约。他指出，目前有 300，000 个 vSphere 客户正在增加他们在云和内部 IT 环境中部署的工作负载数量。

Krause 补充说，Broadcom 还致力于将尽可能多的客户从永久软件许可证转向订阅，从而产生更稳定的经常性收入。VMware 今天公布的 2023 财年第一季度营收为 30.9 亿美元，比 2022 财年第一季度增长 3%。订阅和软件即服务应用占该季度总收入的 29%。第一季度订阅和 SaaS 收入为 8.99 亿美元，同比增长 21%。

DevOps.com 母公司 Techstrong Research 的首席分析师兼董事总经理丹·基尔希(Dan Kirsch)表示，鉴于交易的规模，许多现有的 VMware 客户自然会感到紧张。他指出，主要关注的是 Broadcom 可能会在多大程度上试图提高许可费来实现其财务目标。

收购 VMware 正值云环境中部署的工作负载数量远远超过内部 IT 环境中部署工作负载的速度。这些工作负载中的大部分运行在非基于 VMware 的虚拟机管理程序上。然而，如今在数据中心运行在 VMware 上的许多工作负载正在转移到云中。VMware 还投资构建了一个 Tanzu 平台，用于在 Kubernetes 集群上构建和部署云原生应用程序。

Krause 说，Broadcom 致力于对合并后的软件产品组合的研发进行投资。

合并后的公司将保留多少 VMware 员工仍有待观察。基尔希指出，在一次重大合并之后，出现“人才流失”的情况并不少见。

无论交易是否完成，随着新的 Broadcom-VMware 实体的发展，许多 IT 团队至少将在未来几周内重新审视他们的 VMware 软件产品组合。事实上，收购 VMware 很可能会在经济低迷时期掀起一波并购浪潮。这意味着定期审查供应商关系将很快成为众多 IT 组织的议事日程。
