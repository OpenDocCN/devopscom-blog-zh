## 1.“恐龙宝宝”和“高龄产妇”的劳动力困境

本周第一条新闻:IBM 在一场就业年龄歧视诉讼中的辩护看起来被粉碎了。内部邮件“似乎显示了高层关于削减老员工级别的讨论。”

### 分析:不好看

据原告的一名律师称，一位未透露姓名的“IBM 高层管理人员明确地相互密谋驱逐年长的员工……为千禧一代员工腾出空间。”从表面上看，他们似乎在做与《就业年龄歧视法》截然相反的事情。

诺姆·斯凯伯 : **让“恐龙宝宝”灭绝:IBM 推动年轻劳动力**

IBM 的高层管理人员直接参与了关于需要减少老年员工比例的讨论…有时用“恐龙宝宝”这样的艺术术语来贬低他们。…联邦地区法院公布的文件…显示高管们正在讨论逐步淘汰老员工的计划。引用公司内部电子邮件的新公开文件显示，IBM 高管特别强调需要精简老员工队伍，雇佣更多年轻员工。一封电子邮件提到了 IBM 的“过时的产妇劳动力”，这明显是在暗示老年妇女，邮件中写道:“这是必须改变的。他们真的不懂社交或参与。”
……
IBM 发言人亚当·普拉特(Adam Pratt)[声称]“IBM 从未参与系统性的年龄歧视。”[他]说，电子邮件中的一些语言“不符合 IBM 对其员工的尊重”，“也不反映公司的做法或政策。”

咦，年龄歧视*和*性别歧视？经验丰富的员工很有价值，MrBanana 说:

我在 IBM 工作过一段时间，记得我被邀请参加一个新团队的会议，该团队由一位新的“思想家”领导，他有一些东西要向我自己和一位同事展示。我们走过桌上足球桌，来到会议室。自然，房间里没有椅子，贴满了彩色的便利贴。一旦他们完成了情绪板和这个新想法的非线性展示，我就转向房间里的另一个老头。“8 年前我们不也是这么看的吗？”

“是的，我们发现仅仅运行一个会话的 UI 的计算能力在几十年内都是不可用的。”孩子们有些失望，(幸运的是)我们再也没有被要求回到他们的游戏室。

但是经验可能是一件好事也可能是一件坏事，这是 T2 学到的:

我在一家高速发展的公司工作了五年，经历了这两个方面。…我在那份工作中学到了很多:…在第一年，我下定决心要完成 X，并忽略了一些比我在公司时间长的同事的质疑。我失败了，浪费了一些时间。在第 4 年和第 5 年，新同事决心完成 X，无视我怀疑的评论。

## 2.IBM Wazi 即服务的困惑

“大型机”不是一个肮脏的词。它安全、可垂直扩展，并将*虚拟机*放在 VMware 中。现在，正如 IBM Wazi 为本地 LPARs 所做的那样，Wazi aaS 承诺将扩展到托管 z/OS 实例的**。**

### **分析:重塑大型机即服务**

**大型机和云原生 DevOps 不必是不兼容的。使用混合云实例进行开发/测试(甚至生产)以实现传统应用的现代化，可以极大地提高工作效率。但是，就在几个月前，IBM [推出了](https://devops.com/devops-deeper-dive-ibm-to-manically-focus-on-hybrid-cloud-ai/)其大型机即服务云产品**，这难道不奇怪吗？****

**[迈克·维扎德](https://devops.com/ibm-shifts-mainframe-app-testing-to-the-cloud/ "read the full text") : **IBM 将大型机应用测试转移到云端****

**虽然多年来许多工作负载已经从大型机中迁移出来或者干脆退役了，但是今天在 z/OS 上运行的应用程序仍然是组织运行的最关键的任务。… IBM 本周透露，它已经增加了在 IBM Cloud 上为 z/OS 大型机应用程序设置测试环境的能力。
…
IBM 计划在今年下半年推出面向 z/OS 的 IBM Wazi as-a-Service (Wazi aaS)，【这】将使 DevOps 团队能够在运行于 IBM Cloud 上的安全私有空间上自行配置 z/OS 虚拟服务器的实例。…与此同时，通过一组安全的 API 使大型机应用程序更容易被其他应用程序访问，从而使大型机应用程序的现代化变得更加容易…使用一组在 Kubernetes 环境中运行的云原生 DevOps 工具。
……
与迁移大型机应用程序相比，现代化大型机应用程序并不复杂。挑战在于，构建、部署、维护和保护这样做所需的所有 API 意味着大量的工作。**

**好了，布玛儿，*为什么我们要谈论 DevOps.com 上世纪 60 年代的大型机呢*？这里是阿奇邦克:**

**大型机完全是不同的野兽，当你把它们与 PC 硬件相比时，它们几乎是异类。你有 20 世纪 60 年代为 S/360 系统编写的二进制文件吗？好消息是，您仍然可以在本机运行它们，无需重新编译。

你认为虚拟化和容器是热门话题？20 世纪 70 年代，IBM 在一个虚拟机中运行多个操作系统实例。…工程令人难以置信。**

 **但是 z/OSaaS？这是*似曾相识*的时刻[被围攻的灰胡子](https://forums.theregister.com/forum/all/2022/02/15/ibm_cloud_to_offer_cloudy/#c_4415902 "read the full text"):**

**“z 的云托管服务”又名“zCloud ”,已经存在了 10 多年。… IBM 用 Kyndryl spin 摆脱了它，所以很奇怪他们现在想重新发明它。如果这是一个战略平台，他们为什么不抓住它，而把它剥离给 Kyndryl？听起来像是更愚蠢的领导决策。**

## **3.IBM 的重返工作计划有疫苗授权**

**IBM 希望在员工完全远程办公后回到办公室工作时“保证他们的安全”。实际上，这意味着**疫苗强制**。**

### **分析:IBM 数字健康通行证是分裂的声音少数**

**尽管有基于医疗或宗教原因的豁免，IBM 已经承诺让异议者无薪休假。正如你所料，这项政策引起了一些员工的强烈抗议。**

**[Cassie Werber](https://qz.com/work/2115173/return-to-work-how-salesforce-ibm-and-hsbc-are-thinking-about-it-now/#:~:text=more%2C%20he%20said.-,IBM,-When%20the%20pandemic "read the full text"):**sales force、IBM 和汇丰银行如何重返工作岗位****

**当疫情来袭，IBM 将其 95%的员工转移到完全远程的地方，它惊喜地发现生产力没有下降。…尽管如此，他们还是计划回到办公室。公司认为这是保护员工安全的一种方式。…一个例子…是它开发了 IBM 数字健康通行证，作为疫苗卡或纸质疫苗证书的替代品，他们用它来确保所有进入办公室的员工都接种了疫苗。
……
与此同时，IBM 首席执行官 Arvind Krishna 提出了他所谓的 IBM 在家工作承诺。[它]包括承诺灵活性，善良，对家庭的敏感性，设定界限，并承认视频疲劳。它以“保持联系”的承诺结束。**

**如果不允许你回到大楼里，就很难“保持联系”。[安妮·德伊诺森齐奥、希瑟·霍林斯沃思、程昕婷·阿贝尔、梅·安德森、肯·斯威特、马特·奥布莱恩和迪-安·德宾](https://apnews.com/article/coronavirus-pandemic-health-pandemics-f78e81ca7e93e03141250e78a7d70ebb "read the full text"):**

**IBM 软件工程师贾斯汀·阿尔巴诺(Justin Albano)说，他的公司最近通知员工，将取消没有接种疫苗或没有证明自己接种疫苗的员工的徽章。“我们现在不能进入 IBM 的办公室，”Albano 说。他说一些工人已经成功地寻求宗教豁免，但是“我的想法是我不应该向我的公司解释我的信仰”来逃避一项“从根本上来说是不道德的”政策**

**有什么选择？从某种角度来看，这是克里斯·哈尼:**

**或者，打疫苗。保护好自己和身边的人。你们这些人真是脱离现实。**
