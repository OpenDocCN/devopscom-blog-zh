# 新遗迹调查发现可观测性挑战

> 原文:[https://devo PS . com/new-relic-survey-unouths-observability-challenges/](https://devops.com/new-relic-survey-unearths-observability-challenges/)

一项新的遗迹调查发现，只有 27%的受访者实现了全栈可观测性，只有 5%的人声称他们有成熟的可观测性实践。调查发现，三分之一(33%)的受访者还表示，他们仍然主要是手动或根据投诉来检测停机。

这项全球调查调查了 1，614 名受访者，包括 1，044 名从业者和 570 名 IT 决策者，并于今天由 New Relic 发布。

从好的方面来看，调查还发现，近四分之三的受访者表示，他们组织中的首席执行官是可观察性的倡导者，超过四分之三的受访者(78%)认为可观察性是实现核心业务目标的关键因素。

然而，超过一半(52%)的受访者表示，他们每周经历一次或更多的高业务影响停机，29%的受访者表示，他们需要一个多小时来解决这些停机问题。

New Relic 负责战略和用户体验的高级副总裁彼得·佩扎里斯(Peter Pezaris)表示，解决问题需要这么长时间的一个重要原因是，使用多种工具确定问题的根本原因需要时间。整整 82%的受访者表示，他们使用四种或更多工具来监控系统的运行状况。只有 7%的人说他们的遥测数据是完全统一的，只有 13%的人说数据的可视化或仪表板是完全统一的。近一半的受访者(47%)表示，他们更希望有一个单一的、整合的可观察性平台，但只有 2%的人表示，他们目前使用一种可观察性工具。

当然，New Relic 正在为跨越应用程序环境和应用程序所依赖的底层基础设施的单一可观测性平台做准备。如今，大多数 IT 团队都在使用各种工具来监控基础架构和应用程序。

几乎所有受访者都表示，到 2025 年，他们预计将部署网络监控、安全监控和日志管理等功能，以及 Kubernetes 监控。超过三分之一(36%)的人认为可观察性提高了他们的生产力，使他们能够更快地发现和解决问题。不到三分之一(32%)的人认为可观察性能够促进跨团队协作，而超过四分之一的人认为可观察性提高了他们的创新能力。

近四分之三(72%)的受访者预计明年将维持或增加他们的可观察性预算。超过一半(52%)的受访者，包括 57%的首席执行官，预计可观察性预算将在下一年增加。总共有 14%的受访者和 16%的 IT 决策者预计将大幅增加可观察性预算。IT 决策者预计，未来三年，人工智能(51%)、物联网(48%)、边缘计算(38%)和区块链(36%)最需要可观察性。

就可观测性平台的采用而言，现在还为时尚早，但是很明显，随着应用程序环境变得更加复杂，DevOps 团队所依赖的过多监控工具将需要简化。可观察性平台承诺以某种方式统一日志、指标和跟踪，使得发起查询以识别问题的根本原因变得更简单。

DevOps 团队接受可观测性的速度自然会有所不同，但最大的障碍可能不是平台本身。相反，问题是了解哪些查询可以帮助开发团队在出现重大中断之前更好地了解 IT 问题的根本原因。从长远来看，预计机器学习算法将利用可观测性平台收集的数据，在实际发生之前很久就自动识别可能导致中断的问题。

与此同时，人们希望监控工具将让位于可观察性平台，这将使管理复杂的 IT 环境对所有相关人员来说都更加简单。
