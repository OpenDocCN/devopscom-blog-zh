# SauceCon 2020 演讲嘉宾简介:迪伦·莱西

> 原文：<https://devops.com/saucecon-2020-speaker-profile-dylan-lacey/>

*这是系列文章中的第三篇，重点介绍在 SauceCon 2020**上演讲的人。SauceCon 将全球的 Sauce Labs 用户和持续测试专家聚集在一起，互相学习，提升他们的自动化测试和持续交付技能。今年的活动将于 4 月 27 日至 29 日在得克萨斯州奥斯汀举行，DevOps.com 将在现场直播。*

软件工程师 Dylan Lacey 在 Sauce Labs 支持团队工作，帮助客户解决问题，排除集成和测试故障。此外，作为一名狂热的家庭厨师，他喜欢创造和改造食材。![](img/aeca972a7accf9de7c3ec1e64b7d72c6.png)

他认为需要转换的一个测试领域与测试命名有关。

迪伦说:“在支持部门工作意味着我不仅会看到大量的测试，还会看到大量的测试出现故障。“许多客户惊讶地发现，通过更好地识别和管理测试，他们会获得多少好处。”

迪伦解释了这一点的重要性。

迪伦解释道:“有太多名不副实的测试影响了太多的客户。“用大量冗余数据测试名称。没有识别信息的测试名称。没有名字的测试。具有超长名称的测试。我在各种规模的企业和各种级别的测试复杂性中都遇到过这种情况，每次都会让客户感到沮丧。”

在 2020 年的 SauceCon 大会上，迪伦将在他的题为“名字有什么意义？为了乐趣和利润的元数据。”与会者将带着实用的信息离开，这些信息包括什么是好的测试名称，更批判性地思考测试失败的技能，以及使用标签、构建 id 和自定义数据的建议，以帮助他们组织和共享测试。

迪伦说:“即使与会者离开时对他们的元数据感到内疚，我也希望他们能准备好并兴奋地去修正它。”。

不工作时，迪伦有一系列令人印象深刻的爱好和休闲活动。他热爱烹饪，有大量的烹饪书籍，甚至每周两次靠[](https://twitch.tv/DylanTonic)做菜为生。他还养蜂，自制蜂蜜酒和奶酪，写短篇小说，是一名业余皮革工人。

在第四届年度酱料实验室用户大会 SauceCon 2020 上，你可以听到来自 迪伦 和其他 30 多位专家的演讲。要查看完整的演示列表或注册 、 [请访问 SauceCon 网站](https://saucecon.com/) 。
