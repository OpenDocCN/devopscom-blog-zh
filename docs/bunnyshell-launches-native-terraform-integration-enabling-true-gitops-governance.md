# Bunnyshell 启动本地平台集成，实现真正的 GitOps 治理

> 原文：<https://devops.com/bunnyshell-launches-native-terraform-integration-enabling-true-gitops-governance/>

*集成消除了环境漂移，可跨基础设施、微服务和数据库自动更新*

**加利福尼亚州三藩市，2022 年 5 月 16 日—** 领先的环境即服务(EaaS)平台 Bunnyshell 今天宣布与基础设施即代码工具 **Terraform** 进行本机集成。新的集成为开发团队提供了完整的端到端解决方案，以在软件开发生命周期的所有阶段复制他们的生产环境。

现代云原生分布式企业软件的发布管理可能会带来挑战，因为本地开发环境与代码最终部署的复杂生产环境截然不同。这些差异会影响代码的行为方式，导致生产中的错误、回滚和长测试周期。即使一个公司能够构建类似生产的开发和测试环境，对真实系统的更改也会导致复制品和真实事物之间的不断漂移。

Bunnyshell 的 EaaS 平台为开发和测试创建了不断更新的短暂环境。通过 Terraform 集成，Bunnyshell 的自动更新和 GitOps 治理能力从微服务扩展到了基础设施。提交给代码或基础设施的变更将触发所有环境的更新，确保整个组织总是在最新版本上工作。该平台可以从 Github、GitLab、Bitbucket 的存储库中提取基础设施配置和代码，并将其部署在亚马逊 AWS、谷歌云平台(GCP)、微软 Azure 和 DigitalOcean 上。

Bunnyshell 的联合创始人兼首席技术官 Roxana Ciobanu 表示:“我们的使命是让开发人员的生活更轻松，Terraform 是这一旅程中的又一个里程碑。在 GitOps 时代，基础设施和代码之间的区别正在消失，两者都在以前所未有的速度变化。我们很自豪能够帮助开发人员跟上他们不断变化的生产环境，以便他们能够快速而自信地创建和部署优秀的代码。"

**关于邦尼谢尔**

Bunnyshell 是一个多云环境即服务平台，可以创建、管理和维护生产副本，无论架构有多复杂，使用什么服务。它会随着每个拉取请求或作为软件发布周期的一部分，自动启动新环境，用于开发、测试、演示或部署目的。Bunnyshell 为 sprints 带来了可预测性，支持频繁发布，并帮助开发人员发布更多代码。[https://www.bunnyshell.com](https://www.bunnyshell.com)
