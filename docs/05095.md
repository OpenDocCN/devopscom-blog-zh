# Instana 通过零代码修改为 AWS Lambda 函数提供了自动化 Node.js 跟踪

> 原文:[https://devo PS . com/instana-delives-automated-node-js-tracing-for-AWS-lambda-functions-with-zero-code-modification/](https://devops.com/instana-delivers-automated-node-js-tracing-for-aws-lambda-functions-with-zero-code-modification/)

*公司提供跨多个无服务器功能和基于服务器的微服务的端到端跟踪*

**拉斯维加斯 AWS re:Invent–2019 年 12 月 3 日**–针对微服务和云原生应用的实时应用性能管理解决方案的领先提供商 Instana 今天宣布能够跟踪运行在 AWS Lambda 上的无服务器 Node.js 函数。这些新功能无需修改代码，可自动提供卓越的本机无服务器跟踪，提供超越 X 射线且独立于 X 射线的性能指标。

Instana 的联合创始人兼首席执行官 Mirko Novakovic 表示:“随着公司继续推动创建更小、更动态的服务，开发人员将更频繁地在更关键的应用中使用无服务器功能。“通过在 AWS Lambda 上提供本机 Node.js 跟踪，Instana 可以帮助 DevOps 等应用程序利益相关方了解各个函数是如何逐个调用执行的，同时还可以提供开发人员所需的必要详细信息，以使他们的代码按预期执行。”

新功能是 Instana 独特的 AutoTrace 特性的扩展，该特性无需采样就能捕获所有请求的端到端分布式跟踪。新的无服务器监控和跟踪功能包含在 Instana 针对微服务和云原生应用的应用性能管理解决方案中，无需额外成本。

Instana 首席技术官兼联合创始人 Pavlo Baron 表示:“应用环境和这些环境中使用的技术总是会随着平台的进步而改进。“Instana 致力于为应用和应用平台提供最全面的可见性，并提供我们的用户所期望的同类最佳自动化。”

对于负责开发、部署和维护包含使用 Node.js 构建的无服务器功能的应用程序的开发人员和 DevOps 团队来说，Instana AutoTrace for AWS Lambda 是一大进步。

*   为所有受支持的 Node.js 库和框架自动跟踪用 Node.js 编写的 AWS Lambda 函数
*   零代码修改，代表着开发、开发运维团队的巨大生产力提升
*   通过其他编程语言、平台和部署进行跟踪

Lambda function traces 的所有数据都无缝集成到所有 Instana 分析功能中，包括最终用户监控、Unbounded Analytics 和 Application Perspectives。新功能还包括现成的上下文 Lambda 函数概述仪表板，其中包括函数名称、函数版本、AWS 区域、与其他函数和服务的相互依赖性以及所有其他关键的 Lambda 运行时指标。来自 Instana 的 Lambda Node.js 跟踪的信息与其他性能信息相结合，包括 CloudWatch，以提供最全面的服务健康和性能视图。

与其他 APM 解决方案不同，Instana 完全自动化应用程序监控的整个生命周期，包括应用程序发现和映射、监控传感器和代理部署以及应用程序基础架构的运行状况监控。当动态应用程序发生任何变化时，Instana 会识别出发生了变化，并自动对地图、监控阈值和健康仪表板进行适当的调整。

Instana 在拉斯维加斯的 Amazon re:Invent(511 号展位)上强调了新的 Lambda 无服务器追踪功能。更多信息，请访问 https://instana.com/serverless。
