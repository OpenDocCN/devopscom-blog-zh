# VMware 扩展 Equinix 联盟，推动托管服务

> 原文：<https://devops.com/vmware-extends-equinix-alliance-to-drive-managed-hosting-service/>

VMware 今天宣布扩大与 Equinix 的联盟，推出托管的[分布式云](https://devops.com/?s=distributed+cloud)服务，在托管的 it 环境中运行其云计算平台的实例。

在 [VMware Explore 2022 Europe](https://www.vmware.com/explore/eu.html) 大会上宣布，[VMware Cloud on Equinix Metal](https://news.vmware.com/releases/equinix-and-vmware-expand-global-cloud-services-partnership)服务利用全球 71 个主要大都市地区的 240 多个 Equinix 数据中心来部署运行在该公司虚拟机软件之上的 VMware 云平台实例。

VMware 云解决方案副总裁 Narayan Bharadwaj 表示，这项 VMware 服务将公司的多云计算战略的范围和影响力扩展到了大型超大规模和传统内部 IT 环境之外。

他指出，目标是提供一个受管的 IT 环境，为部署延迟敏感型应用程序提供一个更接近网络边缘的替代方案，在网络边缘可以近乎实时地处理和分析数据。此外，Equinix 的托管 it 环境也使 IT 团队能够更轻松地满足高度监管行业的要求。

目前还不清楚相对于公共云或本地数据中心，在托管环境中运行的应用程序占多大比例。但是，随着越来越多的组织决定不自己运营数据中心，他们中的许多人发现托管服务比云更受欢迎，原因涵盖从应用程序性能、成本和安全性到合规性的方方面面。

Bharadwaj 指出，在某些情况下，组织确定最初部署在云中的某些类长期运行的应用程序在内部 IT 环境中运行实际上更便宜。他补充说，托管选项通常以较低的成本为这些应用程序提供云的灵活性优势。

随着云计算的发展，它变得比以往任何时候都更加分散。许多组织发现，他们现在拥有在公共云、托管设施和传统内部 IT 环境中运行的混合应用程序。面临的挑战是，随着每个新平台用于运行应用程序，管理它的总成本也在增加。不管任何潜在的基础设施成本节约，每个平台通常都需要一组专门的专家来掌握管理它所需的框架。事实上，最大的 IT 支出仍然是劳动力成本。VMware 正在为一个熟悉的 IT 框架寻找理由，该框架可以部署在任何地方以控制这些成本。

当然，DevOps 的创建是为了提供一种大规模自动化 IT 管理的新方法。然而，DevOps 的使用仍然不均衡，很大程度上是因为没有足够的 IT 专业人员具备成功管理代码化 IT 所需的编程技能。托管服务越来越受欢迎，因为在大多数情况下，它们将管理开发运维工作流的责任转移给了 VMware 等供应商。

在未来的几个月和几年中，It 管理的自动化程度如何，还有待观察。然而，很明显，在云时代，越来越多的组织选择依靠第三方来管理基础架构，这样他们就可以投入更多的资源来构建和部署应用程序。
