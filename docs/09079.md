# 分析 SRE 招聘信息

> 原文：<https://devops.com/analyzing-sre-job-postings/>

关于什么是[现场可靠性工程](https://devops.com/site-reliability-engineering-sre-comes-of-age-in-2022/)以及什么是[现场可靠性工程师](https://rootly.com/blog/what-is-an-sre) 的工作，你可以在网上找到很多高层次的定义。

但是如果你想了解[作为一名现场可靠性工程师](https://devops.com/day-in-the-life-of-a-site-reliability-engineer-sre/)实际上是什么样的，也许没有比工作描述更好的来源了。SRE 的招聘广告解释了真正的公司在寻找什么，以及他们在日常工作中的期望。

为了深入了解 SRE 的实际情况，我们来看看几家领先公司的招聘广告。正如我们将看到的，每个公司对站点可靠性工程师角色的要求略有不同，但是可以确定一些强调企业期望的核心主题。

为了理解 SRE 职位描述的内容，我们来看看四家领先公司的例子。

### GitLab 的招聘广告

GitLab 的[【SRE】职位描述](https://about.gitlab.com/job-families/engineering/infrastructure/site-reliability-engineer/) 包含了你对典型网站可靠性工程师招聘广告的大部分期望。它强调了混合技术技能和协作技能的重要性。它还强调在 Unix/Linux、基础设施即代码和 Kubernetes 等领域的专业知识。

然而，可能有点令人惊讶的是 GitLab 的工作描述对编程技能的强调。GitLab 希望 SREs 知道如何用特定的语言编码，比如 Ruby 和 Go。要点是 sre 需要知道的不仅仅是如何使用代码来解决 IT 运营问题。他们还必须能够开发应用程序或为应用平台贡献代码(职位描述提到“在 GitLab 中贡献代码”是 SRE 的一项必备技能)。

同样有趣的是，GitLab 希望网站可靠性工程师能够写博客。当你想到网站可靠性工程师的职责时，写博客可能不是你首先想到的技能之一，但至少在 GitLab，它是工作的一部分。

### LinkedIn 的招聘广告模板

LinkedIn 提供了一个 [SRE 招聘广告模板](https://business.linkedin.com/talent-solutions/resources/talent-engagement/job-descriptions/site-reliability-engineer) ，企业可以在网站上发布自己的 SRE 招聘信息时使用。

LinkedIn 的广告水平相当高(这并不奇怪，因为它是一个模板，而不是针对特定工作的广告)。在很大程度上，它没有提到具体的工具——尽管主要的例外是编程语言，它说 SREs 应该知道如何用 C、Python、JavaScript 和 Ruby 等语言编码。和 GitLab 一样，LinkedIn 显然将发展视为 SRE 的核心工作职责。

LinkedIn 的职位描述也非常强调“收集和分析指标”的能力它没有使用“可观察性”这个词，但这似乎是 LinkedIn 想要表达的意思。

### 在微软的 SRE 工作

继续以发展为中心的 SRE 工作的主题，最近的一则招聘广告专门招聘[【SRE】在微软](https://careers.microsoft.com/us/en/job/1032757/Site-Reliability-Engineer) 工作的人，读起来很像 DevOps 工程师的职位描述。“设计、编写和交付软件”的能力是广告中列出的首要职责，其中也提到了对 CI/CD 工具的熟悉程度。

微软的职位描述也提到了书面和口头沟通技巧的重要性。与 GitLab 的广告不同，这个没有说 SREs 将需要写博客文章；不过，很明显，公司希望这个职位的人除了是技术专家之外，还是有创意的沟通者。

同样值得注意的是，微软想要既懂 Windows 又懂 Unix 的站点可靠性工程师。如果你需要进一步的证据来证明 Unix 技能对于 SRE 的工作是绝对重要的，看看这个来自创造 Windows 的公司的招聘广告就知道了。

### 亚马逊网站可靠性工程师广告

即使是亚马逊——一家更像是平台提供商而不是软件供应商的公司——也在其最近的 SRE 招聘广告[](https://www.amazon.jobs/en/jobs/1665128/site-reliability-engineer)中强调 SREs 的编码能力。

有趣的是，亚马逊的这份工作也强调社交技能是一项主要资格。鉴于 AWS 云中网络的复杂性，这是可以理解的。但这有点令人惊讶，因为大多数其他网站的可靠性工程师招聘广告并没有将网络专业知识列为关键技能——尽管网络工程对 SRE 的工作 至关重要 。

亚马逊的职位描述也提到了应聘者愿意随叫随到的重要性。我们怀疑这在几乎所有 SRE 的工作中都至关重要，但至少亚马逊坦率地承认，有时你会在凌晨 4 点被叫醒去处理事故。

### 结论:外卖

从这些不同的 SRE 职位描述中，有一些关键的要点。

*   要成为一名 SRE，开发技能绝对至关重要。为管理或 IT 运营目的编写脚本本身是不够的。
*   清晰沟通的能力——包括口头和书面——对 SRE 的工作来说比你可能意识到的更重要。
*   网络工程技能也是 SRE 工作的重要组成部分，至少在一些公司是如此。
*   实际上，SRE 的职位描述听起来比你想象的更像 DevOps 的招聘广告。

当然，这并不是说所有的 SRE 职位都是一样的，也不是说所有的公司对他们的 sre 都有相同的期望。但我们调查的招聘广告或模板确实突出了对 SRE 的高层理解如何转化为实际公司的实际情况。
