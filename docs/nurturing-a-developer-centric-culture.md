# 培养以开发人员为中心的文化

> 原文：<https://devops.com/nurturing-a-developer-centric-culture/>

科技世界一直在快速变化，许多科技公司正在成长。他们增加了产品组合，增加了员工数量，服务了更多的用户，扩大了全球范围等等。通常，一个组织的成长不会一蹴而就。要让商业理念开花结果，需要付出大量的努力、吸纳新的人才和合作文化。这种增长通常涉及整合新的角色、方法和文化，以帮助企业取得成功。DevOps 就是这些新出现的文化之一。

DevOps 的发展源于对开发人员工作方式采取新方法的需求。随着技术的进步，开发人员寻求新的方法来实现他们的策略，DevOps 作为一种解释这一过程的方法出现了。它源于缩短系统开发生命周期和关注高质量软件的持续交付的概念。这种方法使开发人员能够执行端到端的任务，从开发到测试，再到生产中的部署和操作。许多人现在认识到，DevOps 是一套结合了软件开发(Dev)和 IT 运营(Ops)的实践，最终演变成一种支持开发人员完成手头工作的文化。

DevOps 文化建立在工程[文化](https://devops.com/?s=culture)的基础上，致力于增强工程师的能力，与持续交付的概念交织在一起，赋予开发人员端到端部署其产品的责任和能力。这意味着开发人员可以快速行动并独立工作。虽然 DevOps 通常指的是软件开发生命周期的操作方面，但更进一步可能是有益的。在一个真正以开发人员为中心的文化中，工程师参与到软件的整个生命周期中，从开始、产品、QA 一直到在生产中操作它。例如，从事产品开发的工程师可以进步很快，因为他们可以像在自己的创业公司一样工作。工程师一旦开发出一个特性，就可以在几分钟内发布。然而，要发展壮大，这种文化需要整个组织的支持。

参与并不一定意味着完全负责。例如，产品经理负责产品，但是工程师参与产品定义是非常重要的。这适用于产品开发生命周期的所有方面，工程师需要参与所有的事情。

领导层有一些有效的方法来支持以开发人员为中心的文化。越是接受这种文化，公司和行业的工程师就会越成功。组织可以通过多种方式有效地支持 DevOps，并建立和维持以开发人员为中心的文化。

1.  **理解以开发者为中心。**不了解 DevOps 是什么，就很难培育它的成长。重要的是要记住，这是一个让开发人员能够通过 SDLC 无缝工作的概念。许多人认为 DevOps 是一个角色或一个具有特定职责的工作描述，但它远不止于此。这是一种文化。
2.  设定指导方针，而不是规则。以开发者为中心不能也不会装进一个盒子里。大多数公司为流程创建了待办事项清单，但这些清单最终都失败了。相反，公司应该专注于创建鼓励创造力和批判性思维的指导方针，以便工程师能够有效地计划、评估和反思他们对最终产品和过程所做贡献的每一部分。关键是让工程师对他们的工作和他们努力的目标感到自豪。
3.  **培养工程师承担所有权和责任**。让工程师参与产品生命周期的每一步，从产品开始。 [C 级领导](https://digitalcxo.com)和其他经理需要相信团队能够主动思考并有效执行。雇佣这个团队是有目的的；不要微观管理并接受一些开发人员可能会以不同于其他人的方式做事——不一定只有一种“正确的方式”
4.  营造一个协作的环境。工程师之间以及工程师和其他部门之间的合作能够激发创造力，找到解决问题的方法，或者创造出全新的产品。此外，倾听和尊重不同的观点可以激发整个组织的积极变化。
5.  拥抱错误，鼓励持续改进。做事没有对错之分，因此让工程师有创造力和犯错的灵活性非常重要。他们的错误可以带来洞察力和知识，否则可能不会被发现。痛苦等于成长。犯错和冒险让个人、部门和组织变得更好。
6.  **授权给组织内外的工程师。**随着公司的成长，他们对技术社区的责任也在增加。术语“授权工程师”应该被视为组织内外的一种责任。工程师越强大，越有能力，整个行业就越强大，越成功。

总之，随着公司的成长，重要的是要适应为实现目标而变化的角色和文化。以开发人员为中心是文化进化以适应和解决业务中新需求的一个例子。如果你想到一个在装配线上的软件开发过程，我们把装配线做成一个圆圈，这样编写软件的开发人员负责从编写软件到操作软件的整个生命周期。这是一种以发展为中心的文化。既然以开发为中心已经存在，它需要得到公司内外领导的培养和支持才能茁壮成长。
