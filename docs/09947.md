# HPE 将 VMware 云平台添加到托管绿湖服务中

> 原文:[https://devo PS . com/hpe-adds-VMware-cloud-platform-to-managed-green lake-service/](https://devops.com/hpe-adds-vmware-cloud-platform-to-managed-greenlake-service/)

在 [VMware Explore 2022 Europe](https://www.vmware.com/explore/eu.html) 大会上，惠普企业和 VMware，Inc .本周扩展了联盟，将 VMware 云平台的[实例集成到托管的 HPE 绿湖服务](https://www.hpe.com/us/en/newsroom/press-release/2022/11/hpe-and-vmware-advance-partnership-to-drive-digital-transformation-with-integrated-hybrid-cloud-experience.html)。

HPE 首席战略官 Hang Tan 表示，除了服务器和存储平台之外，HPE 首次通过基于消费定价模型的服务来管理整个 VMware 云体系。他说，我们的目标是通过不要求 IT 团队过度调配 IT 基础架构资源来运行 VMware 云，从而降低 IT 总成本。

Tan 补充说，以前，IT 团队可以以更分散的方式在 HPE 绿湖服务上部署 VMware 云，但这需要单独的计费流程。

Tan 指出，在日益严峻的经济形势下，许多组织目前都在重新审视其 IT 战略。他补充说，托管的 HPE 绿湖服务使 IT 团队能够将更多的内部 IT 资源用于构建和部署应用程序，而不是管理 IT 基础设施。

目前还不清楚有多少组织正在向使用内部 [It 基础架构](https://devops.com/?s=IT+infrastructure)的订阅模式过渡，但在云时代，很明显更多的组织乐于使用 IT 即服务。HPE 报告称，HPE 绿湖服务支持全球超过 120，000 名用户访问超过 1 EB 的数据。该公司声称，在 HPE 的前 100 名客户中，近 80%已经采用了 HPE 绿湖平台。

许多组织面临的挑战是，出于性能原因和满足各种法规遵从性要求，他们仍然需要在内部 IT 环境中部署和更新应用程序工作负载。HPE 格林莱克服务旨在将云运营模式引入内部 IT 环境，不仅运行 VMware 云，还运行 HPE 已经支持的各种其他竞争软件堆栈。

自然，采用 DevOps 工作流的组织需要将构建和部署应用程序的方式与托管服务保持一致。Tan 指出，如果 IT 团队觉得这很有挑战性，他们可以选择 HPE 绿湖服务的自助服务版本，这样 IT 团队可以更好地控制作为服务使用的底层基础架构。

不管采用哪种方法，很明显，在高度分布式的计算环境中，某种类型的消费 it 基础架构的“即服务”方法正在得到更广泛的应用。每个组织都需要自己决定在多大程度上将其视为运营支出而非资本支出最有经济意义。

然而，毫无疑问，当基础设施周期作为多年服务协议的一部分时，其破坏性通常会更小。it 团队也可能会发现，当 IT 环境由供应商代表他们管理时，他们更容易利用正在进行的人工智能(AI)进步。

无论如何，IT 基础架构的管理方式都在不断发展，而此时企业开始将更多精力放在越来越难以构建、部署、保护和维护的应用程序上。
