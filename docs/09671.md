# 克服多重云管理挑战

> 原文:[https://devo PS . com/overcoming-multi-cloud-management-challenges/](https://devops.com/overcoming-multi-cloud-management-challenges/)

我们一直在谈论云是怎样的未来。但是随着 [绝大多数企业数据现在存储在云](https://www.statista.com/statistics/1062879/worldwide-cloud-storage-of-corporate-data/) 中，很明显，大多数企业现在意识到这是当今日益苛刻和敏捷的商业环境中的必备条件。

在过去几年中，云已经成为工作场所远程重组的支柱，使组织能够访问数据流和关键工作负载，无论其员工身在何处。以至于部署不同服务提供商作为[多云](https://devops.com/?s=multi-cloud)战略一部分的企业数量急剧增加。这使他们能够在混合环境中追求更高的敏捷性和灵活性。T3】

最近来自 Wanclouds 的[](https://wanclouds.net/assets/pdf/Wanclouds_2022_CloudNative_Trends_Outlook.pdf)研究发现，如今大多数 IT 专业人员使用两个或更多的云平台，不到一半(48%)的人表示他们混合使用公共云和私有云。然而，现实情况是，您的组织使用的云环境越多，管理和保护它们就变得越复杂。以下是当今影响 IT 团队的三个多云管理挑战，以及他们如何克服这些挑战。

### **面对日益严峻的安全挑战保持正常运行**

**避免停机从未如此重要，也从未如此难以实现。极端天气事件变得越来越频繁，增加了自然灾害或停电的风险，导致数据丢失。事实上，65%的美国和英国 IT 决策者表示，他们的组织去年至少经历过一次数据丢失事件，而 21%的人承认他们经历过两次数据丢失事件。**

**但是，对 it 部门和他们支持的开发团队来说，最大的威胁可能是安全挑战。自新冠肺炎病毒爆发以来，组织成为网络攻击目标的频率要高得多，被网络犯罪分子渗透的速度也快得多。据 [迈克菲](https://www.mcafee.com/enterprise/en-us/assets/reports/rp-quarterly-threats-apr-2021.pdf) 统计，仅 2020 年就有 310 万次基于云的网络攻击。毫不奇怪，这些攻击加剧了 IT 界的担忧。88%的美国和英国决策者表示，他们对 2022 年的云工作负载安全性至少有点担忧，15%的人对此感到忧虑。**

**虽然多云环境为开发人员带来了灵活性，但它们也导致了固有的配置和可见性问题，而这正是网络攻击者可以利用的弱点。跨不同云平台和操作系统迁移数据的复杂性也可能导致修补延迟。关键通常是确保跨这些平台的可见性并维护互操作性。必须建立适用于多云环境的安全平台，以便在提供商和平台之间共享威胁信息。否则，威胁很容易被忽略，或者即使检测到了，整个基础架构的响应也太慢，无法最大限度地减少损失。**

**随着企业加速实现在未来几年内实现 100%云原生的目标，基于云的灾难恢复策略也必须嵌入其管理计划中。如果没有，每个企业都有丢失重要数据的风险，并且他们的系统、运营或服务会因自然和人为灾难、安全风险、硬件故障和停电而关闭。**

### ****顶着隐藏的罪名****

****在过去的两年中，云支出已经飙升，所有迹象表明，在未来几年中，组织将继续 [将更多的 IT 预算分配给云](https://www.infoworld.com/article/3650768/it-spending-will-soon-shift-to-mostly-cloud.html%23:~:text=Cloud%2520computing%2520budgets%2520will%2520hit,development%2520and%2520operations%2520by%25202025.) 。但并不是每个企业都拥有像苹果一样的资源，据报道，根据 2019 年 [美国消费者新闻与商业频道](https://www.cnbc.com/2019/04/22/apple-spends-more-than-30-million-on-amazon-web-services-a-month.html) 的报道，苹果每月在亚马逊的云上花费超过 3000 万美元。对于大多数 IT 团队来说，现实情况是他们一直在与隐藏费用和意外的云成本作斗争，这些费用有可能会超出他们的整个 IT 预算。T9】****

****事实上，在 Flexera 的 [2022 年云状态](https://info.flexera.com/CM-REPORT-State-of-the-Cloud?lead_source=Website%2520Visitor&id=Blog) 报告中，受访者自我估计他们的组织浪费了不到三分之一(32%)的云支出，高于去年的 30%。与此同时，Wanclouds research 发现，54%的英国和美国 IT 专业人士表示，他们在过去一年中遭遇了意想不到的云成本。****

****从多云计划的迁移阶段到管理和维护，这是一个让 It 团队陷入循环的严重问题。当然，像亚马逊这样的云提供商提供了资源跟踪和支出可见性的基本工具。但事实上，这些功能通常无法提供关于如何优化成本的实时和企业范围的见解。****

****为了消除意外云成本带来的巨大冲击，组织需要制定明确的云成本管理计划。同样，公司范围内对其云平台计费系统的可见性是必不可少的，也是跟踪其多个账户支出的一种方式。对于公司来说，正确标记基础架构以了解成本并维护所有资源的最新清单至关重要。****

### ******满足多云合规性要求******

******每个云用户都必须遵守特定的监管标准，遵循行业指南以及当地、国家和国际法律。例如，医疗保健领域的组织必须遵守 HIPAA 规则，HIPAA 规则实质上是一套国家标准，用于保护敏感的患者健康信息在未经患者同意或患者不知情的情况下不被泄露。同时，制造 IT 团队必须遵守严格的知识产权安全协议以及针对 PCI 和 PII 等其他敏感数据的协议。******

******确保跨多个云环境的合规性是必要的，但无疑是一项挑战。在当今充满攻击的安全环境中尤其如此，内部威胁对数据安全造成的风险不亚于外部攻击者。(瓦罗尼斯 调查的组织中有 32%发现分配给用户的访问授权和权限不正确)。更不用说法规遵从性要求在不断变化，这使得组织更难满足需求。******

******这就是为什么每个投资于云的 IT 和安全专业人员都需要一个解决方案，让他们能够针对其云基础架构设置合规性策略，从而在发生任何不可预见的自然灾害或安全事件时轻松确保合规性要求。我们已经看到了一次数据泄露对企业造成的损害——花费数百万美元进行清理，并给他们的声誉带来相当大的损害。组织的整个云计算战略必须优先考虑敏感数据的合规性和安全性。******
