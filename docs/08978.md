# LogDNA 调查表面可观察性 ROI 问题

> 原文：<https://devops.com/logdna-survey-surfaces-observability-roi-concerns/>

Harris 对 200 名高级工程专业人员的调查表明，大多数开发团队在他们的 it 环境中实现真正的可观察性可能还需要一段时间。

这项调查是代表 LogDNA 进行的，LogDNA 是一个可观察性平台的提供商，调查发现近四分之三的受访者(74%)尽管投资了数十万美元，但仍难以实现他们的可观察性目标。事实上，超过三分之一(38%)的人表示，他们已经每年在可观察性工具上投资 30 万美元。

尽管有这些投资，不到一半的受访者对他们使用日志数据的能力非常满意。令人沮丧的关键领域包括与跨多个团队的同事协作的挑战(67%)、工具不易于使用的事实(66%)以及安全事件的路由(58%)。

尽管如此，大部分受访者表示，他们对可观测性持乐观态度。整整 85%的参与者说他们相信真正的可观测性是可能的。

LogDNA 首席执行官 Tucker Callaway 表示，许多组织未能认识到的核心问题是，可观测性实际上是一种数据素养和管理挑战。组织正在从各种工具和平台收集日志、指标和跟踪，而没有考虑最佳的数据管道构建。他说，结果是一个没有被标准化以简化表面可操作情报的数据集合。Callaway 说，事实上，许多组织很快发现，他们真正做到的只是增加了数据存储成本。

Callaway 指出，在缺乏数据标准化的情况下，跨职能团队查询存储库中收集的所有数据变得极其困难。

其核心是，可观察性承诺在 it 问题造成破坏之前更容易识别它们。这种或那种形式的可观察性一直是任何 DevOps 最佳实践的核心原则。最初，DevOps 团队将持续监控作为主动管理应用环境的最有效方式。然而，发现问题的根本原因仍然需要几天甚至几周的时间。可观察性平台使关联事件成为可能，从而使分析工具更容易识别可能表明 it 问题根本原因的异常行为。

相比之下，传统的 IT 监控侧重于预定义的指标，以确定特定平台或应用程序何时在预期范围内运行。可观察性平台将指标、日志和跟踪(一种专门的日志记录形式)结合起来，使 it 专业人员能够更轻松地查询由各种 DevOps 工具和平台生成的数据。

当然，假设 DevOps 团队实际上知道启动什么查询来确定 IT 问题的根本原因可能是什么。从长远来看，机器学习算法应该更容易识别 it 环境中的异常。然而，与此同时，实现可观测性仍然需要相当数量的 DevOps 专业知识，而这些知识并不总是容易获得的。

最终，可观测性将变得比现在更加自动化。随着这种转变的发生，IT 团队应该会发现可观察性的 ROI 正在稳步提高。
