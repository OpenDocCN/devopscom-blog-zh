# CloudBees 进一步扩大了 Jenkins CI/CD 的覆盖范围

> 原文:[https://devo PS . com/cloud bees-further-extends-Jenkins-ci-CD-reach/](https://devops.com/cloudbees-further-extends-jenkins-ci-cd-reach/)

CloudBees 宣布，它正在将其 Jenkins 开源持续集成/持续开发(CI/CD)平台的范围和影响力扩展到合规性和无服务器计算框架领域。

在法国尼斯举行的 DevOps World | Jenkins World 大会上宣布，Jenkins 的一个新插件[增加了对亚马逊网络服务(AWS)定义的 AWS 无服务器应用模型](https://www.businesswire.com/news/home/20181024005511/en/Jenkins-Community-Establishes-Native-Support-AWS-Serverless) (SAM)的支持。该模型涵盖了 AWS Lambda 函数、应用程序编程接口和 Amazon DynamoDB 表，并由 AWS CloudFormation 提供本地支持。AWS SAM 插件支持经典的自由式作业和 Jenkins 管道，使读取和部署无服务器应用程序成为可能。

实际上，CloudBees 使得将 AWS Lambda 视为任何 CI/CD 管道的自然延伸成为可能。从长远来看，无服务器计算框架也将在 CloudBees 的云原生实例中扮演更重要的角色。

与此同时，CloudBees 还扩展了其 CI/CD 平台，以解决身份管理带来的合规性问题。CloudBees Core 现在与 Google 在其云身份服务中提供的基于 LDAP 的目录集成在一起。这种能力使得通过使用相同的 Google Cloud Identity 或 G Suite 凭证连接到 CloudBees Core 来集中用户管理策略成为可能。

CloudBees 的首席产品官 Christina Noren 指出，随着更多的安全责任继续向开发者转移，治理和合规性正在成为最佳 DevOps 实践中的更大问题。

CloudBees 社区继续增长，尽管 CI/CD 竞争对手的数量继续增加。在大会上，CloudBees 高管[透露，在 2017 年 9 月 30 日至 2018 年 9 月 30 日期间，Jenkins 社区增长了 24%](https://www.businesswire.com/news/home/20181024005509/en/Jenkins-Community-Achieves-Record-Growth-Driven-Major),达到 192，623 个活跃安装。该数字仅适用于允许将使用信息传输到 Jenkins 项目的安装，因此该数字不代表 Jenkins 用户总数。总的来说，据估计全球可能有 1500 万到 2300 万开发者依赖于 Jenkins。CloudBees 还指出，在同一年度期间，詹金斯管道和其他工作的总数量从 10509706 万增加到 20664054 万，詹金斯管道工作比去年增长了 309%。

Noren 说，这种持续成功的主要原因之一是，与其他 CI/CD 平台不同，这些平台提供了只能作为云框架访问的高度固执己见的框架，Jenkins 环境使得在内部或公共云中部署更灵活的框架成为可能。

当然，仍有数百万开发人员尚未接触到任何类型的 CI/CD 框架。但是随着 DevOps 实践继续成为应用程序开发和部署增长的主要方法，现在大多数开发人员在 CI/CD 平台的上下文中工作只是时间问题。

事实上，有争议的是，那些没有很快采用 CI/CD 平台的组织将被竞争对手淹没，这些竞争对手能够以越来越多的天、小时而不是周、月来衡量的速度更新应用程序。
