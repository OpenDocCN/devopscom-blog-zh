# Quali 宣布增强对 Helm 的支持，以改善容器管理，同时加快部署

> 原文:[https://devo PS . com/quali-announcements-enhanced-support-for-helm-to-improve container-governance-while-accelerating-deployment/](https://devops.com/quali-announces-enhanced-support-for-helm-to-improvecontainer-governance-while-accelerating-deployment/)

*Quali 的基础设施自动化平台通过简化掌舵图的部署和管理，增加价值并帮助企业实现开发运维规模*

**、德克萨斯奥斯丁、2022 年 3 月 15 日 —** [Quali](https://c212.net/c/link/?t=0&l=en&o=3471820-1&h=89548733&u=https%3A%2F%2Fwww.quali.com%2F&a=Quali) ，环境即服务基础设施自动化解决方案的领先提供商，今天宣布其 [Torque](https://c212.net/c/link/?t=0&l=en&o=3471820-1&h=3158979496&u=https%3A%2F%2Fwww.quali.com%2Ftorque%2F&a=Torque) 平台现在支持与 Helm 的无缝集成。Helm Charts 允许工程师使用所有必需的微服务和网络组件为 Kubernetes 定义他们的应用程序——本质上，充当 Kubernetes 的包管理器。

Quali 首席技术官 Edan Evantal 表示:“许多 IT 组织都在努力应对集装箱基础设施的复杂性、成本和责任管理。“我们的目标是帮助组织利用 Helm 提高开发和测试过程的速度和可靠性，从而获得更大的价值和控制力。同时，我们允许他们通过重用经过验证的现有掌舵图来维护治理并减少摩擦，以便他们可以更快地创新，同时降低与规模相关的业务风险。”

借助 Torque 对舵图的本地支持，客户现在可以:

*   利用完整的应用程序环境保护和扩展他们现有的 Helm 投资
*   在一个或多个环境蓝图中获得对新的和现有掌舵图的更大控制和责任
*   通过使舵图易于重用并将容器外的基础设施组件组合到可管理的环境中，在扩展容器采用和使用方面提供价值
*   通过将头盔的使用可视化和报告为蓝图的一部分，并将容器的使用与结果联系起来，提供对头盔使用的更大控制，而不抑制头盔的功能
*   通过给予人们使用容器的能力来更好地优化资源，不管他们的技能水平如何。这使得更熟练的开发人员能够创建舵图并扩展环境，以包括容器之外的元素，同时受益于 Torque 固有的治理和控制
*   使用自助服务功能按需创建容器，无需创建和管理图表

领先企业已经采用 Torque 来加速软件开发和部署管道，同时提供无摩擦的管理和治理。借助 Torque，组织可以将完整的应用环境(包括基础设施、数据、服务和应用工具)定义到统一的可重用蓝图中，以支持软件交付管道和 QA/测试环境中所需的基础设施的自动化部署。

Torque 还通过自助式访问控制支持安全要求，允许集中式开发运维团队定义最终用户可以自由配置的环境元素以及不能更改的元素，从而提供灵活性和管理。Torque 通过包括自动标记、自定义标记和基于角色的访问控制(RBAC)在内的内置功能来实现这一点。
