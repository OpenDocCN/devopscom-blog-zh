# 对云提供商的信任是开发者的问题

> 原文：<https://devops.com/trust-in-cloud-providers-is-a-developer-issue/>

今年夏天早些时候，我们与 ClearPath Strategies 合作，询问开发人员对不同云服务提供商的信任程度。我们了解到的情况很有说服力:虽然开发者对亚马逊、微软和谷歌的熟悉和熟悉使这些知名的云提供商成为市场的领跑者，但开发者往往不认同这些公司的价值观。

该研究项目收集了 800 多名中小型企业开发人员的见解，这些企业的全职员工不到 500 人，遍布全球各行各业和各个地区。您可以在[下载总结报告(无需注册)。](https://www.linode.com/content/cloud-provider-trust-clearpath-strategies-report/)

在 ClearPath Strategies 的报告中，高级分析师斯蒂芬妮·费尔柴尔德(Stephanie Fairchild)表示，“大约 20%的开发人员认为，三大超大规模公司将从事以下行为，包括利用其品牌扼杀竞争，关注股东多于客户需求，托管或与开发人员认为道德上令人反感的客户合作，将用户数据货币化供自己使用，并用专有工具锁定他们。对于开发人员来说，安全性和性能等标准显然比这些[信任](https://devops.com/?s=trust)问题更重要，但有趣的是，有相当一部分开发人员正在考虑他们的价值观如何与其服务提供商的价值观保持一致。”

*   信任在开发者选择云托管提供商的过程中扮演着重要的角色，但需求优先；当优先考虑需求或价值时，开发人员报告说他们 73%的时候优先考虑需求而不是价值。
*   开发者选择第三方云提供商的三大原因是:安全、信任和服务质量。
*   调查受访者表示，超大规模和替代云提供商都能提供同样有效的性能和安全性。

该报告列举了选择第三方云提供商的三大标准:安全性、信任度和服务质量。调查受访者认为，超大规模和替代云提供商提供了同样有效的性能和安全性。有趣的是，尽管 73%的受访者表示在做出购买决定时，他们优先考虑需求而不是价值，但大约 20%的人表示，他们不信任三大提供商(AWS、GCP 和 Azure)在可被视为关键道德领域的服务。

众所周知，超大规模云的广泛功能和品牌知名度使其成为开发人员的热门选择，但 Linode、DigitalOcean、OVHcloud 或 Hetzner 等替代云提供商可能是创新型新贵更值得信赖的合作伙伴，它们可能提供更好的价格，不会威胁或竞争他们的业务。当替代云提供商满足开发人员的功能需求时(几乎总是如此)，开发人员可以两全其美:一个他们可以信任的提供商可以满足他们的需求，另一个提供商可以分享他们的价值观。
