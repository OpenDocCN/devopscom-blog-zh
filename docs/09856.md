# 随着攻击面的增长，保持 DevOps 管道的畅通

> 原文:[https://devo PS . com/keeping-the-devo PS-pipeline-flow-as-attack-surfaces-grow/](https://devops.com/keeping-the-devops-pipeline-flowing-as-attack-surfaces-grow/)

当今企业和公共实体必须管理的攻击面从未像现在这样复杂和难以保护。在过去十年中，云和 SaaS 产品的引入导致了看似不可逾越的 IT 蔓延，现在有三分之一的成功攻击是通过不受管理或几乎未知的资产发生的。虽然影子 IT 是许多组织面临的紧迫威胁，但软件开发生命周期向云的迁移以及机器身份的兴起正在以深刻的方式加剧攻击面。DevOps 产生于 2010 年代的敏捷和精益软件开发策略，早于云计算的主流化。

但是，一旦基于云的 IT 资源的价值得到广泛认可，开发团队就开始大规模地将他们的开发人员环境迁移到云上，以至于今天许多人认为开发运维与云密不可分。Gartner 最近的研究表明，企业 IT 支出的很大一部分现在都投向了云项目。事实上，对于应用软件、基础设施软件、业务流程服务和系统基础设施等 IT 类别，到 2025 年，超过 51%的 IT 支出将从传统解决方案转移到公共云。到 2025 年，仅在应用软件上的支出就将有近三分之二投向云。

云开发运维的爆炸式增长带来了受管理的特权、凭据、身份和访问权限数量的大幅增长。我们都很熟悉[数字身份和访问管理](https://devops.com/akamai-to-add-customer-identity-access-management-capability-to-enhance-digital-trust-by-acquiring-janrain-inc/) (IAM)如何通过用户名和密码的组合在传统设置中工作。在 DevOps 世界中，云容器可以作为与其他进程交互(机器对机器通信)的独特微服务进程运行，身份和权限通常通过 SSL/TLS 证书进行管理。这些机器身份已经成为多个行业的关键安全优先事项，不仅因为它们呈指数级增长，还因为 IT 安全团队缺乏有效的工具和策略。

### 机器身份—问题的范围

最近一项针对 1000 名首席信息官的全球调查发现，到 2022 年，平均每个组织管理着 250，000 个机器身份，这一数字预计到 2024 年将翻一番。该研究还发现，83%的组织在过去一年中遭遇了与机器身份相关的停机，57%的组织在同一时期至少经历了一次与受损机器身份相关的数据泄露或安全事件。

由于多年来构成 IT 安全最佳实践基础的边界防御解决方案和策略在云环境中用处不大，安全管理员不得不依赖现有的 IAM 和特权访问管理(PAM)工具来保护数据、帐户和开发管道。毫不奇怪，身份和访问权限已经成为云中的主要攻击媒介，为攻击者提供了比几年前更多的企业漏洞。我们已经看到这些弱点在最近的高调入侵和攻击中发挥作用，如网络安全管理软件产品、Codecov 和 PHP。

### 拥抱零信任原则，确保 DevOps 管道的安全

幸运的是，对于今天的 DevOps 团队来说，面向云的权限管理技术近年来发展迅速，特别是在云基础设施权限管理(CIEM)和基于时间的角色访问管理或即时权限管理类别中。CIEM 解决方案针对云环境特有的安全弱点和流程漏洞。这些问题包括缺乏对用户、组和角色权限的深入了解，这使得难以对云基础架构和应用内的用户活动进行监督和控制。IAM 和 PAM 等传统方法主要用于保护组织内的数据、应用和用户，而 CIEM 旨在管理权限风险，以防止云漏洞和数据窃取。

新兴的 CIEM 解决方案带来了发现、监控和审计方面的高级功能，大大拓宽了 IT 安全专业人员对开发运维流程的了解。借助 CIEM 解决方案，安全团队可以扫描云应用程序中的现有用户、组、角色和授予的权限，直至资源级别。可以对每个用户、组、角色和特权进行分类，以识别和验证特权用户是否具有适当的访问级别。可以生成访问报告，简明扼要地列出具有最高权限和最高风险的用户和组。

在审计方面，CIEM 解决方案提供了跟踪和报告权限级别以及与这些权限相关的所有活动的能力。通过这种方式，可以生成特权访问报告供内部和外部审计人员使用，以确保符合监管和其他要求，如 SOX、GDPR、HIPAA、GLBA、PCI-DSS、ISO 27001、SOC 2、FedRAMP 等。同样，由于能够监控所有云应用程序的特权访问，安全团队可以应用行为分析来识别用户的特权滥用和异常活动。与传统 SIEM 工具一样，CIEM 解决方案将向管理员发出可疑活动或行为警报，例如，如果用户试图规避阻止对云服务进行直接特权访问的政策。

### 即时特权和零特权

重要的是，CIEM 解决方案还基于最低权限访问和零信任原则，有效地消除了对云资源的长期特权。权限的自动授予和过期—即时(JIT)权限授予—在最大限度地减少攻击面方面非常有效。因为这些 JIT/零长期特权(ZSP)解决方案工作在零信任模型上，所以没有人和任何东西被信任来长期访问云帐户和数据。

有了 JIT 许可，提升的特权可以在会话或任务的持续时间内扩展，也可以在一定的时间内扩展。任务完成后，这些提升的权限会自动撤销，完全不需要系统管理员的参与。用户以前拥有的长期访问权限可能会持续几个月，转换为 JIT 授权会将攻击面压缩到每月几个小时。此外，JIT 许可在很大程度上使组织不必维护和支付特权和非特权帐户。动态密码生成——动态密码是根据需要生成的，对于客户端来说是唯一的，而不是预先定义和共享的静态密码——也为保护临时部署的服务和功能提供了更好的模型。

在更广泛的计算机科学和网络安全领域中，DevOps 和 DevSecOps 仍然是新的、快速发展的概念。毫无疑问，DevOps 在加速自动化和加快创新应用和业务服务的上市时间方面取得了巨大成功。然而，迄今为止，安全解决方案提供商一直在努力解决云访问管理问题。换句话说，他们一直在努力加速特权访问解决方案，以保护开发运维团队使用的设备、数据和资源，尤其是在跨云环境中。使用 JIT 特权授予和采用 ZSP 原则的动态许可平台在解决这些问题方面显示出巨大的希望。
