# Toptal 推出新的按需开发运维专业化计划，为企业向云的转变提供服务

> 原文：<https://devops.com/toptal-launches-new-demand-devops-specialization-serve-enterprise-shift-cloud/>

旧金山—2017 年 11 月 16 日—Toptal 是一个由顶级工程师、设计师和金融专家组成的全球网络，旨在帮助公司加速、适应和扩展，今天宣布推出一项新的按需人才专业化计划，专注于满足对熟练 DevOps 工程师不断增长的需求。利用 Toptal 的高技能 IT 专业人员私人网络，新的专业化服务将企业和组织与虚拟化、容器化、自动化、持续集成和交付以及云应用程序分发领域的自由 DevOps 专业人员联系起来。

自 2016 年以来，Toptal 的客户对 DevOps 职位的需求年增长率为 109%，在过去四个季度中，每个季度的增长率都超过 25%。来自 Linux 基金会 2017 年开源工作报告的类似数据发现，60%的开源专业人士和招聘经理目前正在寻找空缺的 DevOps 工程职位。Andreessen Horowitz 的 Scott Weiss 称 DevOps“不仅仅是一种方法论。这是现代程序员必备的技能，并且越来越成为一个独立的部门。”他同意“超大规模云数据中心的兴起让这项工作变得更加困难。” [2017 年开发运营状况报告](https://puppet.com/resources/whitepaper/state-of-devops-report)发现，高绩效组织在计划外工作和返工上花费的时间减少了 21%，在新工作上花费的时间增加了 44%。

Toptal 人才运营副总裁 Alvaro Oliveira 表示:“随着组织进行 DevOps 转型，以更快地交付应用和服务，并在市场上更有效地竞争，大多数组织仍然无法获得合适的人才来实现这一目标。“我们新的 DevOps 专业化为合格的 IT 专业人员提供了可靠的渠道，他们可以实现更短的开发周期、更高的部署频率和更可靠的版本，以及其他敏捷软件开发目标。”

Toptal 的新 DevOps 专业化是基于最近与摩托罗拉解决方案和 Arduino 等客户的成功合作。Toptal 的 DevOps 专业化为组织提供了成功将业务应用迁移到云并更有效地竞争所需的人才。

Toptal 网络中的一位 DevOps 专家是 Andrea Villa，他是 Atlassian 的前 DevOps 工程师和 CloudReach 的系统架构师，在系统管理、信息安全和网络方面拥有 15 年的经验，是云迁移、部署和自动化方面的专家。

“为了提高效率，今天的 DevOps 工程师需要有开发背景，并拥有广泛的系统管理知识和网络知识，”Villa 说。“重要的是，他们能够弥合原始代码和企业每天必须面对的挑战之间的差距，例如减少想法、交付和消费者之间的摩擦。”

另一位在 DevOps 领域拥有 12 年以上经验的顶级专家是 Fredrik H rd，他擅长后端产品开发和生命周期维护。赫德是麦肯锡的高级云架构师，在 Wondersign，他确保了他们最终向云的成功迁移。

赫德说:“优秀的 DevOps 工程师必须是组织可能不会优先考虑的所有领域(如数据安全、灾难恢复、缓解和审计)的倡导者，并对这些领域负责。”。“你在 DevOps 中做出的选择会对一家公司产生长期影响。”

Toptal 吸引了软件开发、设计和商业领域的最优秀人才。申请者经过严格的筛选过程，只有不到百分之三的人被录取。然后，Toptal 寻找人才专长和客户需求之间的完美契合。

通过利用专家驱动的精选匹配流程，确保无需耗时的简历审查和面试即可找到合适的个人或团队，Toptal 的专有技术平台实现了招聘流程的自动化，提高了速度和效率。结果是拥有专业技能的高素质员工队伍能够满足公司的需求，从而节省时间和金钱。这种模式使 tal 成为世界上最大的分布式劳动力，年收入超过 1 亿美元，并在过去两年中以大约 300%的速度增长。
