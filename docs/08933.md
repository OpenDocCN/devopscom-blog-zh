# NetApp 调查显示混合云正在走向成熟

> 原文：<https://devops.com/netapp-survey-shows-hybrid-cloud-is-maturing/>

NetApp 对 79 家中大型企业进行的一项小型[调查](https://investors.netapp.com/news-releases/news-release-details/netapp-customers-say-hybrid-cloud-future-enterprise-it-new)表明，企业 IT 组织可能最终会在涉及备份和恢复的用例之外采用混合云计算。

调查发现，20%的受访者目前同时使用内部资源和云资源来支持生产环境中的相同工作负载，另有 17%的受访者计划在 24 个月内这样做。

NetApp 混合云高级产品营销经理 Darnell Fatigati 指出，云计算的混合方法将占据主导地位，超过四分之三的受访者表示他们的组织计划在可预见的未来运营混合云环境。只有 17%的受访者预计他们的组织最终会将所有 IT 环境迁移到云中。

Fatigati 指出，决定不将所有工作负载迁移到云的一个主要因素是[在内部 IT 环境中运行的传统应用程序所产生的数据量](https://digitalanarchist.com/videos/featured-guests/storage-is-the-key-netapp)。他补充说，成功地将万亿字节甚至千兆字节的数据迁移到云中所需的工作量[简直太大了](https://containerjournal.com/features/defying-data-gravity-with-container-native-storage/)。

调查受访者提到的混合云计算的主要优势是创新速度更快(26%)、响应速度更快(25%)和协作能力更强(22%)。受访者还指出，他们希望提高基础设施的灵活性和规模(27%)，优化成本(21%)和提高数据可访问性(13%)。涉及灾难恢复、备份和归档的数据保护仍然是混合云最广泛使用的使用案例，29%的人目前正在使用它，或者计划在未来 24 个月内使用它。

如今，大多数混合云计算环境由多个云和内部 IT 环境组成，这些环境相互独立管理。事实上，越来越多的组织开始在云服务和内部 IT 环境中部署相同的工作负载，这对于需要管理这些工作负载的开发运维团队来说意义重大。如今，大多数开发运维工作流无法跨越多个平台。然而，随着越来越多的组织受到偶然发生的[云中断](https://devops.com/aws-outage-exposes-weaknesses-of-devops-resilience/)的影响，混合云计算实现的弹性级别变得更加重要。

在许多情况下，新冠肺炎疫情加速了向混合云计算的转变，促使许多组织增加在云中部署的工作负载数量。Fatigati 指出，现在，进入 2022 年，这些组织正在寻求通过尽可能集中 IT 管理来降低 IT 成本。每当 IT 组织添加一个新平台时，管理该平台所需的工具和人员都会增加任何组织的 IT 总成本。向混合云的转变在很大程度上是由希望降低这些成本的高管推动的。

大多数企业 It 组织可能还需要一段时间才能跨多个平台部署工作负载。然而，随着云和内部 IT 环境的不断发展，以及部署在本地数据中心的更多平台越来越多地使用云运营模式进行管理，这些平台的按需访问方式将继续融合。事实上，可能有一天云计算这个术语会从 IT 词典中消失，因为默认情况下，每个 IT 平台都会利用一种或另一种类型的云。
