# 低代码/无代码选项是让 Web3 成为主流的关键

> 原文:[https://devo PS . com/low-code-no-code-options-key-to-bring-web 3-mainstream/](https://devops.com/low-code-no-code-options-key-to-bringing-web3-mainstream/)

在加密货币和 NFT 的推动下，Web3 是过去几年中最受关注的技术产业浪潮之一。但是即使有了这么多的收入和媒体的报道，Web3 仍然没有像 Web2 那样得到主流的认可。这种差距的主要原因之一是基于区块链的技术仍然很复杂，需要开发人员学习新的语言和开发环境，以及获得加密、哈希、oracles、零知识证明、工作证明、利益证明等概念的专业知识。即使用户理解了这一点，他们也需要弄清楚这些部分是如何组合在一起的。所需的时间和金钱投资减缓了开发人员的采用速度——直到现在。

Web3 最终获得了自己的地位，这在一定程度上要归功于 SDK 和 API 提供的抽象。这项技术使得所有开发者，而不仅仅是 Web3 开发者，越来越有可能参与到这一新的技术颠覆浪潮中并进行创新。

如果你看看 Web3 中的许多提供商，例如 Infura ( [ConsenSys](https://consensys.net/) )、NFT 港和 OpenSea，以及它们的增长，在过去几年中，这些提供商发生了重大变化，从简单地为底层区块链提供节点到现在提供更高级别的 API。这意味着开发人员和公司不再需要设置自己的节点来提供 SDK 和更高级别的 API，它们依赖于数据索引器和预先审计的智能合同来简化令牌制造和从区块链读取。T3】

随着 API 和 SDK 的发展，使用这些程序的开发人员不再需要学习在不同的区块链、多种开发人员工具、语法和语言上创建合同的复杂性，也不必对这些不同的区块链有深刻的理解。相反，开发人员可以依赖抽象底层复杂性的 API，减轻开发人员的负担，他们只需要传递参数，如索引链、令牌或智能契约 id 以及其他通用信息。这些 API 将演变为提供更多的[低代码/无代码](https://devops.com/?s=low-code/no-code)产品，这些产品具有更大的图形 UI 组件，开发人员可以简单地拖放这些组件来提取价值，并创建 dapp，这些 dapp 也可以改造到他们现有的应用程序或应用程序基础架构中。T3】

这种进化将类似于云的进化方式；以 AWS 和 Azure 的基础设施即服务(IaaS)为例。现有的应用程序可以升级并转移到云上。这导致了平台即服务(PaaS)产品，其 API 是云就绪的；低代码或无代码产品增加了开发人员的延迟、安全性和可伸缩性需求。这也类似于微软的 Power 平台，它在许多云组件上使用拖放操作，允许在短时间内集成和扩展微软平台。这催生了新一代的生态系统参与者，因为参与和创新云计算的门槛降低了；现在，每种类型的开发人员，无论他们的专业领域是什么，都可以利用这一点。这也意味着所有开发人员都可以使用新技术来更快地解决业务挑战，同时显著缩短上市时间。

你已经可以在 Web3 世界中看到这种相似之处。许多开发开发工具的主要参与者开始瞄准业务和技术决策者。他们在谈论 Web2 开发人员如何轻松地利用他们的高级 API 和 SDK(web 2 开发人员非常了解的东西)来使用 Web3 技术解决新的业务挑战。他们可以做到这一点，而不需要建立昂贵的创新组织，也不需要寻找稀缺(昂贵)的 Web3 开发者。这一演变的下一步将是扩展这些用于 NFTs、分析、流、mev、估计气体等的 API，以拥有无代码、自动生成的组件，最终允许更多的 Web2 开发者过渡到 Web3，从而将 Web3 带入主流。
