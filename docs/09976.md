# VMware 为 Spring Boot 框架带来 Java 17 支持

> 原文：<https://devops.com/vmware-brings-java-17-support-to-spring-boot-framework/>

VMware 本周对 Spring Boot 3.0 进行了更新，用于构建基于微服务的 [Java](https://devops.com/?s=Java) 应用，该应用基于 Java 标准版(se)平台的最新长期支持版本。

VMware 软件工程高级主管迈克尔·米内拉(Michael Minella)表示，该框架的最新版本支持 Java 17。希望增加这种支持将鼓励组织从 Java 8 迁移到该平台的最新长期版本，而不需要多次升级到其他以前发布的 Java SE 版本。

此外，Spring Boot 3.0 现在还本机支持称为 GraalVM 的高性能 Java 开发工具包(JDK)以及使用 OpenJDK 构建 Java 应用程序的 Spring framework 6.0 版，open JDK 是 VMware 本月早些时候推出的 Java SE 的开源版本。

VMware 增加了对微米和微米跟踪的支持，以提高 Spring 应用程序的可观察性，这些应用程序将越来越多地包括多个微服务依赖项。

最后，VMware 还增加了对 Jakarta Enterprise Edition (EE) 10 的支持，Jakarta Enterprise Edition(EE)10 是 Java SE 的一个开源扩展，它是在 Eclipse Foundation 的支持下发展起来的。

Minella 表示，对 Java 17 和 GraalVM 的支持为构建 Java 应用程序提供了稳定的基础，这些应用程序比依赖 Java 8 的应用程序要快得多，Java 8 仍然是该平台使用最广泛的版本。Minella 说，DevOps 团队应该开始采用一种新的标准来构建和部署仍然主导企业 IT 环境的 Java 应用程序。

当然，用于构建企业应用程序的编程语言并不缺乏。然而，三十年后，Java 继续占据主导地位，这在很大程度上是因为拥有 Java 编程技能的开发人员数量庞大。

目前还不清楚传统 Java 应用程序更新到 Java 17 和 GraalVM 上运行的速度有多快。现在 Java 17 在 Spring Boot 得到了支持，这是构建 Java 应用程序最广泛使用的框架，Java SE 平台最新长期版本的采用速度应该会在 2023 年加快。

此外，Minella 指出，Spring 框架的最新版本将使组织能够更好地利用即将到来的进步，如 [Project Loom](https://openjdk.org/projects/loom/) ，这是 OpenJDK 的一项努力，旨在用基于纤程的轻量级并发模型来取代线程，纤程通过 Java 运行时而不是底层操作系统来管理。

总的来说，这些技术和其他技术将降低运行 Java 应用程序的成本，这些应用程序比前几代应用程序要快得多，他补充道。

大多数 DevOps 团队会发现，在未来的许多年里，他们管理着一个混合的 Java 应用程序——使用核心 Java SE 平台的多个变体构建。然而，很可能大多数新的 Java 应用程序将基于 Java 17。此外，还将努力在 Java SE 以前版本的基础上实现遗留应用程序的现代化。因此，大多数在企业 IT 环境中工作的 DevOps 团队应该会看到，在未来的几个月中，使用微服务的基于 Java 的项目总数会有所增加。
