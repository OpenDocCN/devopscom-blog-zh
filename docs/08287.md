# 调查显示 SRE 劳动水平略有下降

> 原文:[https://devo PS . com/survey-reveals-light-decline-in-level of-sre-toil/](https://devops.com/survey-reveals-slight-decline-in-level-of-sre-toil/)

尽管 IT 环境总体上变得越来越难以管理，但站点可靠性工程师(sre)的日常工作量在去年略有下降。

IT 监控平台提供商 Catchpoint 与 VMware 和 [DevOps Institute](https://devopsinstitute.com/) 合作对 300 份 sre 进行的年度[调查](https://www.catchpoint.com/press-releases/catchpoint-sre-study-reveals-a-global-drop-in-toil-warns-of-looming-scalability-ceiling-and-highlights-the-need-for-new-operational-capabilities)显示，过去一年中，被定义为低级手动任务的工作量下降了 15%。

只有 22%的受访者声称在科学地衡量辛劳，因此，在某种程度上，调查结果是伪造的。总体而言，本周发布的调查发现，毫不奇怪，sre 将大部分时间花在应对事故和停机上，其次是参与事后分析和随叫随到。调查中提到的主要问题是技术债务过多、优先事项和目标不一致、缺乏商业价值评估、缺乏培训和支持以及缺乏协作。

就自动化采用而言，最常见的用例是发布管理、基础设施管理和应用程序管理。sre 最常用的工具用于监控基础设施、网络和应用性能。采用人工智能(AI)工具来管理 IT 运营( [AIOps](https://devops.com/?s=AIOps) )的比例仍然很低。调查还发现，受访者在管理 it 的集中式方法(42%)和分散式方法(38%)之间大致平分秋色。另有 20%的人表示他们采用了混合方法。缺乏对整个体系的可见性(53%)是最常被提及的云应用监控挑战。

调查还发现，只有一半的受访者不断完善他们的服务水平目标(SLO)。

Catchpoint 的产品营销总监利奥·瓦西利乌表示，很明显，在预防和遏制 it 事故的影响方面，sre 正变得更加积极主动。随着这些 IT 环境变得越来越复杂，他们面临的挑战是缺乏可见性和上下文。复杂性正在增加，部分原因是运行基于微服务的应用程序的云原生计算平台的兴起，这些应用程序具有许多依赖性。

目前不太清楚的是目前由 SREs 管理的 IT 环境的百分比。理论上，通过使用各种 DevOps 工具而不是依赖传统的图形工具来自动化流程，单个 SRE 可以完成与多个 IT 管理员相同的工作量。然而，sre 很难找到和留住。要成为一名 SRE，需要达到什么样的自动化水平，或者就此而言，可能需要什么样的认证，也并不总是显而易见的。成为 SRE 的兴趣正在上升，部分原因是他们通常要求更高的薪水。

与此同时，很明显，组织需要找到更大程度地自动化 it 管理的方法。大多数组织无力雇佣一小群 IT 专业人员来管理日益复杂的 IT 环境。从 DevOps 最佳实践到承诺为人工智能平台提供更多上下文的下一代可观察性工具，一切都将成为组合的一部分。面临的挑战是在 IT 管理进一步失控之前找到实施这些工具的方法。
