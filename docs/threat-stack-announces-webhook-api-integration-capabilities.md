# 威胁堆栈宣布 Webhook API 集成功能

> 原文：<https://devops.com/threat-stack-announces-webhook-api-integration-capabilities/>

*定制的工作流程支持协作以简化安全性*

**BOSTON–2016 年 4 月 14 日**–[云安全管理和合规领域的领导者威胁堆栈](http://www.threatstack.com/)今天宣布推出其 webhook API 集成功能，以减少安全和运营工作流挑战。该功能允许威胁堆栈客户启用 webhook API 来基于安全警报构建自定义工作流。这些自定义工作流扩展了威胁堆栈[云安全平台](https://www.threatstack.com/platform)[^(TM)](https://www.threatstack.com/platform)[【CSP】](https://www.threatstack.com/platform)，并帮助客户将安全性进一步集成到他们现有的监控操作中。

对于每天面临越来越多威胁的 IT 领导和 CISOs 来说，安全问题仍然是一个挑战。组织内部各部门协作以简化和自动化流程比以往任何时候都更加重要。威胁堆栈的 webhook API 集成使安全和运营团队能够更轻松地自动触发警报和补救，从而加快问题的解决。

“在云环境中，自动化是监控和解决事件的关键。Threat Stack 产品和客户宣传副总裁 Venkat Pothamsetty 表示:“Threat Stack 的 webhook API 集成使自动化超越了简单的警报，实际上隔离了实例，直到问题得到解决。“这些类型的功能对内部团队效率有着显著的影响，也是威胁堆栈开发自动化以缩短解决时间的众多方式之一。”

使用 webhooks 的威胁堆栈客户现在可以更加轻松高效地:

*   将威胁堆栈云安全平台与定制的内部应用程序相集成
*   与其他监控平台集成，以便更轻松地发现问题
*   基于威胁堆栈警报构建新的上报和/或补救工作流

威胁堆栈用户可以自定义配置自己的 webhook 集成，以订阅特定的优先事件。当一个事件被触发时，webhooks integration 以 JavaScript Object Notation (JSON)格式发送一个警报信息摘要，每个警报都会导致一个对 webhooks API 的单独调用。然后，用户能够解释他们的警报，并根据他们的独特需求采取行动。

云安全团队的自定义工作流示例包括:

*   基于严重性 1 威胁堆栈警报生成自定义 AWS 简单通知服务(SNS)通知
*   根据严重性为 2 的威胁堆栈警报创建吉拉票证
*   将严重性为 3 的威胁堆栈警报归档到 S3 存储桶中，以便长期存储

完全集成的云原生威胁堆栈 webhook API 集成支持协作，从而加快关键安全事件的平均解决时间。医疗保健、金融科技、SaaS 领域和其他行业的威胁堆栈客户受益于更高的效率、简化的服务以及来自整个基础设施的安全事件的自动化集成。
