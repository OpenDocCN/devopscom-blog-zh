因此，不仅是 IT 和产品团队，高管和高级领导也可以发现到底发生了什么，以及我们开发软件的能力和管理能力有多好。

阿什利:事实上，任何研究都必须从一些假设开始；我们对市场有什么看法，或者你认为我们对市场有什么看法？这里的假设是，我们管理整个软件交付过程的能力的成熟程度如何？将 it 连接到业务，而不仅仅是 IT 内部。因此，我们从这一点开始，我们有一个目标受众，我们试图与他们交谈，既有个人贡献者，软件开发部分的领导者，也有高管，产品人员，首席信息官。所以我们真的试着撒下一张足够大的网，但不要太宽，以至于全部都被冲淡了，你无法说出谁说了什么。

我们真的花了一些时间关注我们想和谁交谈。这不是一个很短的调查，但也不是很长。确实有一些很有见地的问题围绕着你，你知道你的积压看起来像什么吗？你知道延迟一个发布或者用一个特性换取另一个特性的代价是什么吗？你真的知道你什么时候会得到你花钱买的版本或功能，这样你就可以根据这些计划推动业务向前发展，或者你只是在瞎猜，你只是希望它在下一季度的某个时候出来，我们很好，然后我们就可以从那里开始运行一切？所以我们做了一个调查，但是我们做了很多调查，在我们做问卷之前，和不同角色的人交谈。

所以不仅仅是一个研究公司说，“好吧，你想让我们问什么？让我们为您建立一个调查。”你知道，Sanjeev 和加速策略小组的其他分析师都是从业者。他们做过这个，他们在这个领域工作过，他们领导过软件团队、项目、公司，他们为大型企业、小型创业公司、中型公司提供咨询。所以，所有这些都是我们最好的尝试，当我们走出去，询问人们他们认为他们的成熟度曲线在哪里，他们真正看到了挑战，让我们获得我们可以获得的最好信息，我们得到了一些很棒的结果。看到我们认为是真实的事情和一些可能是真实的事情，这是非常非常有洞察力和令人兴奋的，嗯，这是一个小小的惊喜。

**Ashley:** 你知道，每个人——我不想说每个人——这项研究的最大成果之一是每个人了解他们将从投资中获得的价值的能力，对吗？在这个软件 IT 组织、产品团队上花了多少钱，我最终会得到什么？所以很明显，对资助这项工作的人来说，他们根本不清楚他们会得到什么。你只是不能得到那些信息或者得到一个直接的答案。

我们还没有很好地了解它，这是第二个关键发现，即围绕整个软件创建过程的沟通已经中断。它可能在一个真正紧密合作的开发团队中起作用，或者在一个敏捷团队中，他们有一个每天都在一起工作的产品负责人，但是除此之外，在组织中，无论是横向还是纵向，高级领导级别的认知与构建软件的个人和团队级别的认知可能会有很大的不同。所以你能想象我们做得对吗，艾伦？–根据你认为正在发生的事情做出数百万，也许更多，更多，更大的决定，但事实并非如此。

所以这是一个大问题。如果我们真的相信软件正在引领世界，相信所有的企业都是软件公司，那么你必须——你必须改变这一点。那件事必须处理好。第三个领域基本上是通过交付渠道和理解价值流。你在哪里创造了价值，你在哪里花费了精力却没有获得价值？也许从软件过程中得到错误的输出。因为我喜欢用的一个例子是软件团队喜欢测量我们每天发布多少版本，有时是每小时。我的意思是这是一件好事。这听起来是一件非常了不起的事情。但是如果它是错误的东西，错误的软件，即使我们每小时发布一百万次也没关系。如果是正确的事情，我会一个月发布一个版本，而不是每天发布一百万个版本，而且所有版本都没有达到目标。

是的，我同意。

**Ashley:** 这是一个极端，但这才是真正的极端，是对整体的理解——同样，我们谈论的不是一个工作流程，而是一个工作价值流；它跨越多个领域，因为几乎每个应用程序都涉及各种系统、数据，你知道，特别是在大型企业中——大型机、云中的东西、可能是较老的客户端服务器但仍然存在的东西，无论它可能是什么。因此，这是我们确定的三个关键领域，如果你能评估你在哪里，你在软件开发的这一部分做得如何，这将在提升你的游戏和交付软件的能力方面产生最大的差异。
