# JFrog 为应用程序二进制文件添加了私有分发网络

> 原文:[https://devo PS . com/jfrog-adds-private-distribution-network-for-app-binaries/](https://devops.com/jfrog-adds-private-distribution-network-for-app-binaries/)

在今天的在线会议上，JFrog 发布了一个私有分发网络(PDN ),使得在一个扩展的企业中分发应用程序二进制文件变得更加简单和快速。

JFrog 首席技术官 Yoav Landman 表示，PDN 提供了相当于[内容交付网络](https://devops.com/?s=CDN) (CDN)的功能，专为使用 JFrog 管理的对等网络分发应用程序二进制文件而设计。兰德曼补充说，每个分发的二进制文件都是一个不可变的签名版本，提供了可见性和可追溯性，以验证源目的地以及二进制文件的完整性。兰德曼指出，这种方法本质上是将区块链网络的概念应用于应用程序二进制文件的分发。

兰德曼说，PDN 基于正在申请专利的网络优化技术，该技术采用缓存、分发节点和轻量级加速器，使 IT 团队能够通过仅入站防火墙或有限连接的环境分发二进制文件。

最后，除了基于角色的访问控制(RBAC)之外，PDN 还提供了工具来分析二进制文件的开源安全漏洞和许可证政策的合规性。

Landman 表示，目标是提供一种通过单击将二进制文件分发到边缘计算平台的方法，这种方法与持续集成/持续交付(CI/CD)管道和软件开发生命周期(SDLC)流程紧密集成，DevOps 团队已经采用了这种方法。

目前通过私人测试版提供，PDN 计划在 2021 年第三季度上市，定价基于不需要前期成本的每次使用模式。

多年来，JFrog 一直致力于将 DevOps 最佳实践应用于二进制文件的部署，而不仅仅是管理源代码。随着企业 IT 环境变得更加分散，大规模管理二进制文件部署的需求变得更加迫切。IT 团队将无法派遣软件工程师在所采用的每个边缘计算平台上手动安装二进制文件。Landman 说，我们的目标是为 IT 团队提供软件分发的完全透明性，使满足合规性要求变得简单。

大多数组织已经发现部署软件是一个重大挑战。CI 流程通常被理解为任何应用程序开发流程的一部分。CD 已经被证明是一个更加难以捉摸的目标，因为软件注定要在其上运行的每个目标平台都是独一无二的，这使得自动化交付过程具有挑战性。随着组织试图在数百甚至数千个可能基于多种平台类型的边缘计算平台上安装软件，这个问题只会变得更加难以解决。兰德曼指出，PDN 通过为 IT 团队提供集中管理流程的网络，缓解了许多此类问题。

此时，大多数组织面临下一个重大 DevOps 挑战只是时间问题。随着组织寻求在边缘处理和分析数据，以近乎实时地推动任意数量的数字流程，部署在云或传统数据中心之外的软件数量将会激增。当然，现在的挑战是找到最有效的方法来保护所有代码的交付。
