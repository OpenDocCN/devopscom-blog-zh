# 云时代调查表面可观测性的挑战

> 原文：<https://devops.com/survey-surfaces-observability-challenges-in-the-cloud-era/>

最近的云可观测性调查发现，近三分之二(64%)的受访者表示，采用公共云使得实现可观测性变得更加困难。近一半的受访者(48%)表示，他们现在认为提供对应用程序/基础架构环境的实时洞察是重中之重。

这项调查调查了 357 名 IT 专业人员，由企业战略集团(ESG)代表 Yotascale 进行，yota scale 是云计算环境成本优化工具的提供商。

Yotascale 首席执行官 Asim Razzaq 表示，随着每一个新的云平台的采用，实现和保持可观测性变得更加具有挑战性。调查发现，71%的组织现在依赖三家以上的云基础设施提供商。

整整 84%的调查受访者还指出，跟踪成本和云成本分配既麻烦又耗时，61%的受访者承认他们对哪些团队拥有哪些云资源缺乏足够的了解。随着基础设施成本的上升，近四分之一(24%)的企业正在考虑改变其监控和可观察性实践。

[可观察性](https://devops.com/?s=observability)一直是 DevOps 的核心宗旨，但实现和保持这一目标具有挑战性。如今，大多数 DevOps 团队都渴望能够保持某种程度的持续监控。然而，随着检测应用变得越来越容易且成本越来越低，人们对可观测性平台的兴趣越来越大，这种平台可以简化对异常情况的调查，这些异常情况表示可能会破坏应用环境的问题。Razzaq 说，随着这种转变的发生，更多的组织将寻求将事件与成本相关联，作为控制 IT 成本的努力的一部分。

新冠肺炎疫情之后，部署在公共云上的工作负载数量急剧增加。理论上，许多组织签署了企业许可证，如果他们持续消耗一定水平的基础架构资源，就可以降低云成本。然而，在实践中，单个开发团队根据其工作负载的属性，表现出对一个云平台的偏好。然而，这样一来，不仅要持续观察不同的平台，还要控制成本，就变得更具挑战性。

就可观测性平台的采用而言，现在还为时尚早，但是很明显，随着应用程序环境变得更加复杂，DevOps 团队所依赖的过多监控工具将需要简化。对于 DevOps 团队来说，拥有多个监控工具来呈现一组预定义的、但经常相互冲突的度量标准并不罕见。可观察性平台统一了日志、指标和跟踪，使得发起查询以识别问题的根本原因变得更加简单。

DevOps 团队接受可观测性的比率自然会有所不同。在 2022 年，最大的障碍可能不是技术，而是简单地了解哪些查询可以帮助开发运维团队在出现重大中断之前更好地了解 it 问题的根本原因。从长远来看，预计机器学习算法将利用可观测性平台收集的数据，在实际发生之前很久就自动识别可能导致中断的问题。然而，无论采用哪种方法，在未来的几个月和几年中，管理复杂 IT 环境的能力都将继续稳步提高。
