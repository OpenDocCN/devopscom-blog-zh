# AI 是否正在接管 DevOps 功能？

> 原文:[https://devops.com/is-ai-taking-over-devops-functions/](https://devops.com/is-ai-taking-over-devops-functions/)

人工智能将技术世界带到了自动化的新水平。今天，几乎每个专业化都需要机器学习的干预，以开发人工智能技术，帮助企业用更少的时间和资源做更多的事情。尽管如此，一些组织质疑以人工智能 为首的 [是否是一个好的投资。](https://devops.com/why-you-must-lead-with-ai/)

对 DevOps 来说，答案是响亮的是。AI 可以增强 DevOps 实践，加快软件发布的步伐，帮助企业 [实现持续交付](https://devops.com/what-is-continuous-delivery/) 。这使得程序员发布软件的速度提高了约 10 倍，并允许在发布程序之前对其进行审查。

AI 还改善了 DevOps 文化，从实现更有效的决策过程到代码质量增强和自动化操作。所以，让我们更深入地探讨一下 AI 是如何影响 DevOps 的。

## DevOps 的最新变化

监控和管理 DevOps 环境中生成的所有数据涉及高度的复杂性，使得团队很难有效地收集和使用数据。此外，任何给定团队可能收到的数据量都可以达到数十亿字节，因此人工智能工具提供了急需的帮助。毕竟，手动分析海量数据集会耗费太多的时间，而且也不能满足现代企业的需求。

更有甚者，调查显示，美国 57%的[](https://brainstation.io/coding-bootcamps)开发者的经验不足五年。出于这个原因，软件测试必须是一个持续的、深入的过程，以检查和反复检查没有漏洞影响代码的安全性。人工智能可以补充人工审查，以提高查找 bug 的速度和准确性。

也就是说，将 AI 集成到 DevOps 功能中并不意味着开发人员没有必要，即使他们的角色已经发生了很大的变化。后端开发人员参与开发和运营方面，因为他们承担运营任务和测试，特别是围绕 [基于云的基础设施](https://devops.com/how-to-improve-cloud-management/) 和网络安全。创意设计的技术支持将永远需要前端开发人员——没有人能比人类做得更好。

在当今动态和快节奏的世界中，开发人员需要比以前更快地发布代码，同时运营团队要确保现有系统的运行中断最小。当集成人工智能时，这种合作关系更容易，因为它使开发和运营团队之间的协作更有效。人工智能驱动的系统为复杂的开发链中的系统和故障提供了统一的视图。

## AI 要接管 DevOps 吗？

简而言之:不会。AI 正在 DevOps 中补充人类的活动，以使事情更加高效和安全。

由于 DevOps 需求的复杂性，人们一直在寻找一种自动化程度更高的解决方案，可以帮助人们提高性能并加快流程。此外，你可以期待在美国为一名经验丰富的后端开发人员支付大约每小时 60 美元的[](https://www.waveapps.com/freelancing/web-development/back-end-web-developer-salary)，这意味着使用人工智能来自动化繁重的工作也可以节省大量成本。

以下是 AI 转换 DevOps 函数的几种方式:

### 增强的安全性

近来，分布式拒绝服务(DDoS)和相关攻击已经成为对网站和在线服务安全的普遍威胁。人工智能工具有能力识别和缓解这些威胁。此外，AI 还可以在仍然使用传统 DevOps 的团队中帮助 pivot to[DevSecOps](https://www.whitesourcesoftware.com/resources/blog/devsecops/)。

人工智能安全工具根据实时数据并通过分析过去的行为来检测异常和威胁。这样，AI 通过补充人类安全专家，在保护许多组织、 [包括学校和学院](https://www.gettingsmart.com/2020/07/artificial-intelligence-is-solving-cybersecurity-staffing-shortages-in-higher-education/) 免受网络攻击方面发挥了至关重要的作用。

### 改进的数据访问

DevOps 中需要收集、组织和分析的大量数据超出了人类的能力。然而，人工智能通过汇编来自多个来源的数据并将其组织起来进行分析，解决了这个问题。这极大地改善和加速了开发运维的过程。

### 快速软件测试

软件开发和软件测试 [随着 AI](https://www.forbes.com/sites/forbestechcouncil/2021/07/12/autonomous-software-testing-the-next-peak-to-conquer/?sh=4a15e0a33a3c) 的集成而加速。因此，DevOps 使用的各种 [测试类型](https://www.neliti.com/publications/236745/software-release-management-evolution-comparative-analysis-across-agile-and-devo) ，比如用户验收测试、回归测试、功能测试，比以往任何时候都更加高效和准确。

这些测试产生了大量的数据，这些数据需要人类花费大量的时间来手工识别和收集。在人工智能的帮助下，识别导致错误和漏洞的模式和编码实践变得方便快捷。DevOps 团队随后可以使用这些数据来提高他们的效率和改进开发实践。

### 及时提醒

在事件响应期间，开发运维团队有时很难对警报做出反应和响应。由于没有附加优先级标签，团队很难有效地对[](https://devops.com/sres-guide-to-pragmatic-incident-response/)事件进行优先级排序。一个更有效、更全面的警报系统应该能够立即发现缺陷，并将更严重的缺陷标记为更高的优先级。这样，团队就可以更系统地处理问题，并且成功地解决问题。

AI 可以根据某些因素，包括警报的强度、警报的来源和过去的行为，来帮助确定对警报的响应的优先级。这导致在系统被大量数据淹没并且必须立即响应的情况下的高效管理。人工智能驱动的警报 [在 DevOps 安全中特别重要](https://www.darkreading.com/cloud/embracing-devsecops-5-processes-to-improve-devops-security/a/d-id/1333947) ，因此这是人工智能在该领域的一个关键优势。

### 卓越的实施效率

DevOps 团队经常被基于规则的环境的管理任务所累，这减少了他们在更具创新性和创造性的领域花费的时间。当人工智能接管时，这些任务变得自治，提高了效率，减少了人为干预。人工智能系统可以独立工作，让人类从这些任务中解脱出来，专注于创新和创造力。

通过这种方式，人工智能将 DevOps 变成了自治系统，不再需要基于规则的人类分析管理。这将解决 DevOps 团队努力实现的研究的复杂性，并实现更快的适应。

### 反馈回路

DevOps 的主要功能是使用监控工具收集每个运营阶段的反馈，然后使用这些反馈来改进软件开发和交付过程。性能监控工具使用机器学习来收集诸如性能矩阵、日志文件、数据表等信息。然后，收集的反馈用于提前跟踪潜在问题，并相应地提出解决问题的建议。然后通过应用这些建议来改变应用程序，使其更有效。

人工智能正在提高 DevOps 功能的效率。然而，说它正在接管是言过其实的，因为仍有许多领域需要人类干预。AI 整合 背后的 [理念不是改变玩家而是改变游戏。](https://devops.com/ai-automation-wont-replace-jobs-it-will-replace-processes/)

尽管人工智能正在帮助人类加速软件开发，但开发人员仍然在许多方面处于领先地位。但如果没有人工智能的集成，似乎也不可能以所需的速度执行 DevOps 功能。因此，如果你希望以跟上现代技术趋势的速度发布软件，你需要在你的 DevOps 功能中加入人工智能。
