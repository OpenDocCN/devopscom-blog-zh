# 结合 SecOps 和 DevOps

> 原文:[https://devops.com/blending-secops-devops/](https://devops.com/blending-secops-devops/)

对于大多数正在或已经迁移到云的公司来说，安全性必须是首要考虑的问题。此外，企业知道他们需要迅速采取行动，以确保他们构想的任何新产品或服务不会给他们带来风险。

当云的快速移动特性将适当的安全实践抛在脑后时，他们如何做到这一点呢？今天的问题是，今天的安全解决方案是专有的，太慢，占用太多资源。

总而言之，为大多数企业提供良好服务的安全模式从来不是为了速度而构建的，在当今以云为中心的世界中，这种模式根本无法抓住机遇。今天，安全解决方案必须是敏捷的、轻量级的、松散耦合的和可扩展的。

在各种公司的前线，你会看到一种新型的合作——在这种合作中，最直接参与潜在解决方案、开发和安全的两个团队正在打破传统的(往往是对立的)孤岛，更加开放地合作，从一开始就推动安全创新。

我们看到了 SecOps 和 DevOps 的新“联姻”,这为推动组织内外的创新创造了一种全新的思维方式。这种新的心态使安全和开发人员能够找到共同点，使组织能够更容易地将其安全目标与当今快速发展的业务中的产品交付保持一致。

虽然企业正在改变内部结构以解决云安全问题，但也有许多新的解决方案可以针对公司的云进行部署，并将安全情报反馈给开发运维团队。

这类工具对于面临独特挑战的初创企业和中小型企业尤为重要，这些企业面临着保护其业务免受网络攻击和防止数据泄露的挑战。

通过鼓励安全部门和开发运维部门加强协作，并在开发新产品和服务时依靠外部解决方案来评估和识别潜在威胁，各种规模的企业都可以更加自信地认为，他们正在通过实施行业领先的安全实践来解决安全问题。
