 Rafay Kubernetes Operations Platform(KOP)现在具有完全集成的 GPU 资源仪表板，可以可视化关键的 GPU 指标，因此开发人员和运营团队可以从一个统一的平台无缝监控、操作和提高基于 GPU 的容器工作负载的性能。

Kubernetes 已经迅速成为需要在云和边缘/远程位置提供和操作支持 GPU 的人工智能和机器学习应用程序的企业的首选编排层。

根据《2022 年 Gartner 新兴技术:边缘技术提供强大的机会——采纳者调查结果》,受访组织投资和采纳边缘技术的主要目标是提高员工生产力(41%)和自动化业务流程(39%)。这与 Gartner 的现有研究(参见新兴技术:边缘人工智能中的用例模式)相一致，即边缘人工智能正被用于改善业务流程，提供自动化和生产力收益，这些收益转化为可衡量的投资回报，如成本节约。”*

然而，随着企业快速增加人工智能和机器学习工作负载的数量，解决一些挑战，如可见性和监控，有助于防止应用程序部署的重大延迟以及与集群中闲置或性能不佳的 GPU 相关的浪费成本。

例如，一个越来越依赖由人工智能支持的实时视频检测应用的工厂需要一种标准化的方法，以便跨职能团队管理 IT 基础设施和应用。以下挑战通常会导致操作脆弱性和缺乏可重复性，从而影响工作效率:

*   需要 GPU 指标按需调整和优化 GPU 工作负载的开发人员和运营人员的访问和可见性存在缺陷或限制过多。
*   雇佣或培训专家团队，花费数月时间开发、运营和维护定制的监控基础设施，以收集和集中汇总 GPU 指标。
*   开发和维护与企业单点登录(SSO)系统的集成以提供对指标和仪表板的基于角色的访问的复杂性。
*   说明由外部实体(例如，合作伙伴和 ISV)开发和维护的组织的支持 GPU 的工作负载。这些实体还需要了解 GPU 指标，以确保工作负载以最佳方式运行。

Rafay KOP 通过为企业和受信任的外部实体提供零接触体验来解决这些挑战，为整个 Kubernetes 集群的 GPU 提供自动化和集中化的关键运营指标聚合。Rafay 的零信任访问服务与 SSO 集成，实现了基于角色的无缝访问，确保只有授权的开发人员、外部合作伙伴和运营人员才能从控制台获得对 GPU 指标的安全访问和可见性。
