# 英特尔扩大 oneAPI 计划的范围

> 原文:[https://devo PS . com/Intel-expands-of-one API-initiative/](https://devops.com/intel-expands-scope-of-oneapi-initiative/)

英特尔已加大努力，推动采用一套面向人工智能(AI)应用的开放编译器、库、分析和移植工具及框架，这些工具和框架通过 oneAPI 计划共同提供，以包括对其最新一代处理器的支持。

oneAPI 计划在三年多前启动，旨在创建一套多架构工具来构建可部署在任何 CPU、图形处理器单元(GPU)和现场可编程门阵列(FPGAs)上的应用。[现在支持 oneAPI 的处理器包括即将推出的第四代英特尔至强可扩展处理器、英特尔至强 CPU Max 系列和英特尔数据中心 GPU](https://www.intel.com/content/www/us/en/newsroom/news/oneapi-2023-tools-maximize-value-intel-hardware.html)。

英特尔软件产品的首席宣传员 James Reinders 表示，现在显然需要开放的工具和框架，以提供对多种处理器的平等访问。例如，作为这一努力的一部分，oneAPI 的最新 2023 版本增加了对英特尔(Intel)旗下 Codeplay 开发的 Nvidia 和 AMD GPUs 插件的支持，以简化 SYCL 代码的编写，使代码可以跨其他处理器架构移植。

还有一个 CUDA 到 SYCL 的代码迁移功能，使用基于开源 SYCLomatic 项目的英特尔 DPC++兼容性工具，支持 100 多个 CUDA 应用编程接口(API)。英特尔还提供工具来使用英特尔 VTune 分析器识别大规模失衡，英特尔顾问工具现在为英特尔数据中心 GPU Max 系列提供自动化的屋顶线分析，以识别内存、高速缓存或计算瓶颈的根本原因，并洞察优化涉及 CPU 到 GPU 卸载的数据传输成本。

Reinders 指出，随着它的继续发展，开发人员在同一应用程序中调用异构类的[处理器](https://devops.com/?s=processors)将变得越来越常见，但说服其他类型处理器的提供商支持 oneAPI 计划是一项挑战。历史上，处理器提供商依靠编译器和库将开发人员锁定在特定的处理器类别中。

然而，开发人员构建调用多种类型处理器的应用程序变得越来越普遍。英特尔 oneAPI 计划旨在简化对异构处理器(也称为 xpu)的访问。这一点至关重要，因为处理器创新的步伐已经加快，开发人员需要能够调用额外的功能，而不必等待多个提供商更新特定的编译器和库。Reinders 指出，一旦 oneAPI 被更广泛地采用，就有可能显著降低应用程序开发和部署的成本。

尚不清楚 oneAPI 计划是否会被更广泛地采用，是在英特尔的支持下，还是由某种类型的行业联盟提供一个中立的场所来构建一套通用的编译器和库。有一点是肯定的，DevOps 团队致力于在尽可能多的平台上实现编译器和库的标准化。当然，挑战在于确保开发人员只调用可以跨越多个处理器类别的编译器和库，而不是针对特定类型的处理器架构进行了优化的编译器和库。
