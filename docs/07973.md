# Kentik 为公共云环境带来了网络可观察性

> 原文：<https://devops.com/kentik-brings-network-observability-to-public-cloud-environments/>

*公司解决企业的三大云问题:可见性、性能和成本控制*

**三藩市，加州，2021 年 5 月 18 日 —** 网络观察公司 Kentik 今天宣布推出适用于公共云环境的新产品套件 Kentik Cloud。作为 Kentik 平台的一部分，Kentik Cloud 将流量和遥测数据与业务、安全、互联网和云元数据融合在一起，为企业客户提供更高的可见性、更好的性能以及对连接整个云环境的网络的更好的成本控制。

像 Box、易贝、Dropbox、IBM、Kayak 和 Zoom 这样的云计算公司使用 Kentik 来规划、运营和修复他们的网络。Kentik Cloud 的推出将可观察性扩展到了公共云，提出了业界最全面的方法来可视化云网络性能、解决连接问题、控制和降低流量成本、优化服务交付、将应用迁移到云以及保护云环境免受网络攻击。

在最近的一篇文章中，美国职业棒球大联盟的首席网络自动化工程师杰里米·舒尔曼讨论了 Kentik 如何帮助体育联盟了解“有多少流量在云中进出各种项目”。[Kentik]实际上是向您展示您的物理设备如何通过云中的虚拟专用连接进行连接的可视化表示。它还为我们提供了本地、云端和本地的延迟、吞吐量和抖动等信息。”

Kentik Cloud 通过四个核心新功能集扩展了公司现有的云监控功能:

*   **Observation Deck:**一种集成的用户体验，带有可配置的小部件，允许用户将其最重要的数据和见解放在前面和中心，并使用所有类型的网络遥测和上下文数据快速回答任何有关网络的问题。
*   **Kentik Map:** 对虚拟专用云(VPC)、内部网络以及它们之间的连接性的端到端可视化描述，以查看进出云网络元素的流量，从而更快地进行故障排除。该功能还汇集了 AWS CloudWatch 和基础设施健康指标。
*   **云性能监视器:**自动检测云网络内部使用的最重要网络路径以及与本地网络的互连的功能。主动测试测量延迟、抖动和丢包数据，并与 Kentik 的大型公共云代理网络接口，以便轻松监控互联网和多云服务的性能。
*   **云流量成本浏览器:**一种工具，可自动计算云网络流量随时间的实际成本，以及按组和帐户进行的成本预测，通过快速发现流量低效问题来帮助降低数据传输成本。

“我们建立了 Kentik 云产品套件，因为我们看到我们的客户正在努力以他们习惯的精确度和控制力运行云网络，”Kentik 首席产品官 Christoph Pfister 说。“大多数云监控工具并不关注基础架构，或者无法超越单一云环境，而旧的网络工具根本跟不上云操作的步伐，即使它们了解云网络结构或 VPC 流日志。借助 Kentik Cloud，我们的客户可以在一个解决方案内大规模管理云网络的性能、可见性和成本。”
