# Instana 发布了上下文指南，这是应用系统模型的第一个图形用户界面表示

***公司的自动化 APM 解决方案现在向用户显示上下文，无需任何培训或管理应用知识***

禁运截止时间:美国东部时间 2020 年 4 月 7 日上午 8:10

**伊利诺伊州芝加哥——2020 年 4 月 7 日**—面向微服务的自动化应用性能管理(APM)解决方案的领先提供商 Instana 今天宣布推出 Instana 上下文指南，提供对该公司底层系统模型动态图的 GUI 访问。随着 Context Guide 的发布，Instana 成为第一个通过托管应用服务架构提供图形导航的 APM 解决方案。

Instana 技术总监兼 APM 策略师 Chris Farrell 表示:“现代开发和运营团队面临的一个主要挑战是能够看到并理解复杂应用生态系统中正在发生的事情。Instana 的新上下文指南允许任何用户即时了解应用程序中任何对象或服务的上游和下游依赖关系，而无需任何培训或技术知识

与传统的 APM 工具不同，Instana 的自动化应用程序性能监控(APM)解决方案可以发现所有应用程序服务组件和应用程序基础架构，包括 Azure 等云基础架构、Kubernetes 和 Docker 等编排基础架构、应用程序服务和 DevOps 流程。Instana 自动为应用技术堆栈的每个部分部署监控传感器，并跟踪所有应用请求，无需任何人工配置，甚至无需重启应用。该解决方案实时检测应用程序和基础架构的变化，调整自己的模型，并在几秒钟内可视化这些变化及其对性能的影响。

这种自动化的一个关键部分是通过该公司的动态图实现的，这是一种高级的 uber 模型，包括依赖关系、单个基础设施健康状况、配置信息和性能信息，所有这些都随着系统的变化而实时更新。Context Guide 的发布标志着用户第一次可以真正使用动态图来浏览应用和基础架构可视化，这是 Instana 应用性能监控解决方案的核心。

“容器编排和面向服务的架构帮助我们为客户提高了应用程序的可伸缩性和可靠性。但是它们也带来了更多的复杂性，这使得我们很难获得有效监控性能所需的可见性。“虽然 Instana 一直在帮助我们处理这种复杂性，但新的上下文指南允许我们以简单直观的方式在其中导航。新的上下文指南不需要任何介绍；人们刚刚开始使用它，这使得查看单个有问题的应用程序组件的整体性质变得更加容易，从下游依赖关系到应用程序基础架构，只需点击几下鼠标。”

动态图是 Instana 自动化和人工智能功能的关键组成部分。动态图不仅仅是一个模型，它还是所有单个实体、配置数据、性能指标、依赖关系和更改日志的清单，所有这些都包含在一个清单中，并且包含系统中的所有其他实体。

作为旨在帮助开发和运营团队优化其应用性能的一系列产品发布中的最新产品，今天的发布提供了更深入的信息，特别是关于每项服务、服务和整体应用之间的关系的情景环境。有了 Context Guide，现在用户只需点击几下鼠标，就可以从用户请求到缓慢的事务处理，再到有问题的软件或基础架构。

你可以在[https://instana.com](https://instana.com)了解更多关于 Instana 和上下文指南的信息。
