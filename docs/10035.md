# 企业软件架构师如何设计可持续性

> 原文：<https://devops.com/how-enterprise-software-architects-can-design-for-sustainability/>

各种规模的组织都感受到了引入更积极的可持续发展措施来应对气候变化的压力。例如，大多数云服务提供商已经宣布[重大气候承诺](https://www.wired.com/story/amazon-google-microsoft-green-clouds-and-hyperscale-data-centers/)减少或抵消他们的碳排放。但是这些举措只能起到推动作用——这取决于各个公司和部门参与到[遏制其气候影响的行动中来](https://devops.com/10-actions-it-can-take-to-curb-climate-impact/), devo PS 无疑在制定积极的变革中发挥着作用。

设计可持续性的一个方法是优化信息技术系统以减少浪费。企业 IT 通常会支持大量[不必要的技术债务](https://devops.com/enterprises-face-growing-technical-debt/)，并延续[导致能源效率低下的繁重任务](https://devops.com/shedding-light-on-toil-ways-engineers-can-reduce-toil/)。删除未使用的实例和功能或没有贡献价值的空闲工作负载可能是减少浪费的一种简单方法。然而，其他优化将需要更多的努力。

我最近与 LeanIX 的联合创始人兼首席执行官 André Christ 聊天，探讨企业级软件架构师可以使用哪些技术来充分利用他们的技术现代化工作，同时留下最少的碳足迹。根据 Christ 的说法，优化现有流程、使用云原生技术和利用价值流管理是架构师可以采取的“事半功倍”的一些策略

## 1.将技术与可持续发展目标相匹配

首先，技术领导者必须了解他们的业务有助于实现可持续发展目标的能力。“这需要理解和系统的绘图，”基督说。“这实际上是了解作为一家公司，你可以做些什么来以更全面的方式为可持续发展做出贡献。”

例如，路由和物流提供商可能会通过重新设计其平面图来优化运营，从而最大限度地提高效率并降低能耗。类似地，Christ 建议软件专业人员评估他们现有的能力，以确定应该改变什么。这相当于发现帮助一个部门的内部微服务，并将它们扩展到整个组织。

优化数据传输是创建更精简 IT 系统的另一种方式。Christ 说，例如，团队可以通过消除[过度抽取和不足抽取](https://nordicapis.com/how-to-avoid-overfetching-and-underfetching/)或者采用新的 API 标准[，比如 GraphQL](https://devops.com/graphql-as-a-meta-layer/) ，来减少持续轮询。删除这些多余的呼叫可以减少不必要的能量消耗。

## 2.确保云架构，而不是搬来搬去

云迁移可以在可伸缩性和灵活性方面获得显著优势。但并不是每个云迁移都能带来切实的可持续发展优势。AWS 确定了将应用程序迁移到云中的六种独特策略:重新托管(称为提升和转移)、重新搭建平台、重新购买、重构、淘汰和保留。虽然早期的收益可能只是通过重新托管应用程序来满足，但用云原生组件完全重新平台化或重构它们可能会提供最佳的优化。

“如果你只是将应用程序提升并转移到云中，它们将像虚拟机一样低效地运行，”Christ 说。因此，架构师应该尽可能为云原生设计，而不是简单地将遗留应用移植到云。另一方面，谨慎的做法可能是用 SaaS 替换特定的遗留组件，或者让它们完全退役。因此，在进行云迁移之前*分析您现有的环境以确保变化既[经济高效](https://devops.com/how-to-respond-to-rising-cloud-costs/)又能推进可持续发展目标是至关重要的。*

## 3.实践价值流管理

实践价值流管理是通过确保产品团队总是为最终客户创造更多价值来削减膨胀的另一个潜在策略。 [Planview](https://www.planview.com/resources/articles/what-is-value-stream-management/) 这样定义价值流管理:

> “价值流管理是一种管理技术或实践，其重点是增加从客户请求到客户交付的商业价值流。”

这种心态可以帮助工程师对他们正在创造的东西做出更合理的决定。但是，为了实现价值流管理，组织将需要[增强可见性](https://devops.com/increasing-use-of-slos-to-enable-observability/)以了解各个服务如何执行、它们在幕后的依赖性以及它们的大致排放量。

Christ 说，你应该问产品团队的核心问题是*相对于你所服务的用户数量，你的运营碳效率如何？*答案将强调业务的哪些方面最需要优化。

## 4.将可持续发展与业务底线联系起来

许多可持续发展承诺建立在碳抵消的基础上，即公司购买其排放的“抵消”，通常是通过植树或保护森林。但是，碳抵消的做法受到了批评，被认为对阻止全球气温上升无效。那么，一个比绿色清洗更诚实的气候承诺是什么样的呢？嗯，也许还有一个与底线联系更紧密的[。](https://devops.com/the-rise-of-the-cloud-economist-the-other-cfo/)

Christ 将价值流管理的理念向前推进了一步，他认为真正的气候承诺将涉及到每产生一美元，就减少一定比例的碳足迹。随着企业规模的扩大和收入的增加，它应该在增长的同时不断减少排放。克里斯说，这相当于为每吨二氧化碳分配一个美元价值，并将其计入资产负债表。

## 少花钱多办事

各国都在努力通过全面立法来减少碳排放。然而，联合国报告称，国家目标不足以实现到 2030 年减少 43%和到 2050 年净减为零的目标。因此，这也需要私营部门的努力。

我们已经确定了上面的一些步骤，以达到事半功倍的效果，但是许多当前的不可持续的做法在文化上是根深蒂固的，这使得它们很难改变。这将是企业软件架构师的工作，以打破它，并将更智能的设计应用到云原生应用中。这可能需要集中内部服务的视图来增强可见性和报告。“你需要很好地了解你在建造什么，它由什么组成，以减少足迹，”基督说。
