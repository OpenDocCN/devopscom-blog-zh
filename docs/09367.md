# EDB 加速创新，为 Kubernetes 提供全球首个开放管理的 Postgres 运营商

> 原文：<https://devops.com/edb-accelerates-innovation-with-worlds-first-openly-governed-postgres-operator-for-kubernetes/>

*发布 CloudNativePG，为现代云应用提供前所未有的灵活性*

**马萨诸塞州贝德福德。2022 年 5 月 16 日—**[EDB](https://www.enterprisedb.com/)，全球企业级 Postgres 加速领导者，宣布发布首款针对 Kubernetes 优化的开源运营商 CloudNativePG，并向云原生计算基金会(CNCF)提交沙盒应用。CloudNativePG 基于云原生原则构建，支持高级 IT 决策者为基于微服务的云原生架构奠定基础。 新的开源 Kubernetes 运营商简化了 Postgres 数据库平台在企业 Kubernetes 部署中的集成，加快了应用和服务交付，并加速了企业创新。

Kubernetes 已经改变了全球企业通过容器创建、部署和管理大规模应用程序的方式。随着容器化应用的增长，在容器化应用中管理和利用数据的挑战已经成为在一些应用场景中使用 Kubernetes 的障碍。CloudNativePG 为 Kubernetes 上以数据为中心的应用程序带来了一个社区管理的开源解决方案。

EDB 首席技术官 Marc Linster 表示:“这是 Kubernetes 和 Postgres 社区之间的首次合作，结合这两种基础技术的可能性令人兴奋。“作为 CloudNativePG 社区的创始成员，我们致力于支持开源，并通过利用我们对这两个社区的深入了解，提升现代应用程序的云能力。凭借专业知识、增强的企业级功能、全球支持和 Kubernetes 认证，我们在帮助最终用户加快基础架构标准化、现代化和云转型方面拥有独一无二的优势。”

开源的 Kubernetes operator 在设计时就考虑到了 DevOps 和敏捷开发，旨在帮助最终用户为他们的项目增加技术多样性，并实现更快的上市时间。EDB 是 CNCF Kubernetes 认证服务提供商，CNCF & Linux 基金会的白银会员，Kubernetes 社区数据的创始赞助商，并拥有 Red Hat OpenShift 的认证运营商。利用社区版本的最终用户将能够通过 EDB 的 [社区 360 订阅](https://www.enterprisedb.com/plans-postgres-pricing-subscriptions-for-services-and-support) 产品从 EDB 获得对 CloudNativePG 的支持。

“90%的人认为 Kubernetes 已经为有状态工作负载做好了准备，但许多人认为运营商面临的挑战是采用的主要障碍，”Kubernetes 社区数据总监 Melissa Logan 说。“我赞赏 EDB 努力让 Kubernetes 系统上的数据更加开放，并让 Postgres 用户轻松管理 Kubernetes 上的数据密集型工作负载。"

全球的组织都认识到了在 Kubernetes 上运行 Postgres 的力量。 “在勃林格，我们做出了对开源技术进行标准化的战略决策，我们选择了 Postgres 运营商 CloudNativePG 进行快速开发项目，”勃林格殷格翰 容器化和 Kubernetes 全球主管 Simon Adamczyk 说道。“虽然我们对 Kubernetes 的采用还处于早期阶段，但我们计划在不久的将来投入生产。I 对于我们来说，拥有简单的解决方案以快速部署和弃用，而无需考虑许可和锁定问题至关重要。”

为了向企业最终用户提供工具、增强功能和支持，EDB 还为 Kubernetes 提供了 EDB Postgres，它利用 CloudNativePG 使最终用户能够利用企业级功能和支持。EDB Postgres 为 Kubernetes 提供了以下功能和好处:

*   **轻松访问**:Kubernetes 运营商使用本地云流复制，覆盖了具有主/备用架构的 Postgres 数据库集群的整个生命周期。
*   **对部署的控制** :无论是在本地还是在云中，EDB Postgres for Kubernetes 在任何地方都可以运行，为最终用户提供一致可靠的体验。这也包括红帽 OpenShift。
*   **自动修复和管理** :如果 Postgres 中断，EDB Postgres for Kubernetes 会恢复功能，保护故障转移、切换、备份、恢复和滚动更新

“EDB PostgreSQL 运营商帮助我们的 DevOps 工程师在我们的 Kubernetes 云原生平台上构建基于开源技术的应用，”梅赛德斯-奔驰公司混合云和数据中心架构 IT 产品负责人 Eduard Diner 说道。“Kubernetes 帮助我们加速新应用的构建和现有应用的现代化。”
