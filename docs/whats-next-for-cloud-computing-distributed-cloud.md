# 云计算的下一步是什么？分布式云

> 原文：<https://devops.com/whats-next-for-cloud-computing-distributed-cloud/>

公共云通过允许团队与他们的数据交互并以云原生的方式编写新的应用程序，极大地推进了 IT。云简化了应用程序开发过程，按需提供服务，并允许您按需付费。公共云的使用出现了惊人的增长，所有的估计都表明，在不久的将来，这一数字可能会继续增长。

然而，大多数 IT 计算仍然发生在公共云之外，原因很简单，因为在超大规模云提供商运营的数据中心(或“区域”)之外，没有有效的方法来消费云原生服务。这种位置限制在三个主要方面对云计算产生负面影响:性能(延迟、吞吐量)、数据主权(驻留、法规)和商业考虑(传统架构、定价)。

因此，为了实现云的最大潜力，许多企业正在转向云计算的最新创新:不受特定位置或区域限制的分布式云。

这是如何工作的？通过 API，[分布式云](https://www.ridge.co/)连接到任何位置的任何 IT 堆栈，并提供云功能，就像在超大规模云上运行一样。但是，它不是在专用位置运行，而是在任何 IaaS、虚拟化环境或裸机之上的现有基础架构上运行。它与堆栈的底层物理无关，不需要任何安装或资本支出。它可以在任何环境中运行工作负载:商业数据中心、内部设施、边缘微中心，甚至是多设施混合环境。

### 云如何改变企业和中小型企业

现代应用程序开发离不开云计算。它一直是现代应用程序(例如专用于物联网、智能城市和个人健康的应用程序)最近取得许多进展的推动因素，并将继续成为企业和小企业开发新创新的基础设施。但是，随着越来越多的企业采用云优先的方法，使用[云原生](https://www.gartner.com/en/information-technology/insights/top-technology-trends?utm_source=press-release&utm_medium=promotion&utm_campaign=RM_GB_2022_ITTRND_NPP_PR1_TTT22&utm_term=hubpage)架构和技术，“IT 预算”已经成为“云预算”的同义词这通常是一个非常昂贵的提议。

因此，许多企业的下一步是管理[云计算成本](https://devops.com/?s=cloud+cost+management)，同时支持传统和云原生应用以及云迁移。他们的计算策略可能是跨多个大洲的本地、裸机和混合。因此，他们需要一个具有架构灵活性的云来满足他们的需求和预算。

这种灵活性由可定制的分布式云提供。通过促进与现有基础设施的互操作性，它使企业能够在任何需要的地方部署和无限扩展应用程序。他们在运行应用程序方面越开放，他们在优化预算方面就越自由。

### 谁在推动云计算创新？

亚马逊、微软和谷歌已经从云计算的创新者变成了现任者。云计算领域的创新来自初创公司和传统 IT 公司。这里有几个例子:

*   当财富 500 强企业寻求创新和现代化其 IT 和应用基础设施时，Kyndryl 已成为首选的系统集成商之一。Kyndryl 已被大公司聘用来解决云计算带来的许多挑战。他们支持的创新之一是分布式云。

*   为了构建云产品组合，NetApp 已经花费了 20 多亿美元进行了十几次收购。他们的重点是创建一个产品组合，让企业能够将其各种计算和存储基础架构连接到云。与假设他们的客户将在云基础架构上进行全新实施相反，NetApp 构建了一个产品组合，使他们能够满足客户当前的架构需求，并从那里转移到更多云原生功能。

*   Zenlayer 已经在不同的地理位置建立了云基础设施，使他们的全球影响力甚至是超大规模公司都无法匹敌的。Zenlayer 声称提供的云基础设施分布在全球各地，世界上 85%的互联网用户的延迟在 25 毫秒以内。对于许多现代应用程序来说，这种对支持全球性能的物理基础设施位置的关注是一项重要的创新。

### 新兴的云计算创新

分布式云以编程方式集成到任何现有的本地基础设施中，是一项创新，通过为企业提供可针对特定位置需求定制的云，推动企业向前发展。无需任何安装，其云原生构造块利用现有服务器在任何环境(私有、公共、内部和混合)中运行应用程序工作负载。

因此，应用程序所有者能够构建优化的云计算战略，利用他们选择的一个或多个位置的现有可用基础架构。
