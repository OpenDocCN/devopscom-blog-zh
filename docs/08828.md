# 关于边缘计算，开发人员需要了解什么

> 原文:[https://devo PS . com/what-developers-need-to-know-about-edge-computing/](https://devops.com/what-developers-need-to-know-about-edge-computing/)

越来越多的公司生活在“边缘”，因为他们从“我会考虑边缘计算”转向“我不认为我们的业务能够在没有边缘计算的情况下以最佳方式发展和竞争*”开发人员将在帮助组织使用 edge 方面发挥关键作用，但他们的有效性将取决于是否有与 edge 一致的流程、产品和人员。*

边缘计算发生在用户或数据源的物理位置附近。这是组织跨多个位置使用和分发公共资源池的一种方式。边缘模式为用户提供了更快、更可靠的服务，同时为组织扩展了混合云计算的潜力。客户和企业都将受益于边缘计算通过在更接近数据源的地方运行数据密集型应用程序来优化人工智能和机器学习的使用和应用的方式。

虽然在很多方面这是一个架构问题，但应用程序是有效利用这一优势的关键。以下是开发人员及其组织需要知道的一些事情。

**边缘计算和云计算并不相互排斥。**最佳解决方案将利用[两个平台](https://www.gartner.com/smarterwithgartner/gartner-predicts-the-future-of-cloud-and-edge-infrastructure)。例如，集中式计算可能用于存储和计算密集型工作负载，而边缘计算将用于在源位置近乎实时地利用数据。基础设施和公司文化需要到位来支持这一点，但开发人员必须考虑构建可以跨越和桥接计算平台的应用程序。

一致的工具是必不可少的。开发传统物联网非常特殊，需要专业的工具和技能。相比之下，现代边缘是技术连续体的一部分，开发人员需要可以在该连续体中使用的工具，包括:

*   **提供商/企业核心:**“非边缘”层，由公共云提供商、电信服务提供商或大型企业拥有和运营。
*   **服务提供商边缘:**层位于核心或区域数据中心和最后一英里接入之间。
*   **最终用户驻地边缘:**最后一英里接入的最终用户侧的层可以包括企业或消费者边缘。
*   **设备边缘:**通过非互联网协议直接连接传感器/执行器的独立(非集群)系统。这代表网络的远端。

**容器化支持灵活性和规模** : Linux 容器技术不是为 edge 而构建的，但该模型完全符合 edge 对模块化、隔离和不变性的要求。T3】

保持“开放”的心态。开放 API 开启新机遇。使用开放 API 的开发人员可以以编程方式访问实时数据，这使得构建能够在数据“诞生”时利用数据的应用程序成为可能同样，关注开源开发工具和平台将有助于确保组织能够随着模型的发展继续利用优势。T3】

不同的是设备。为 edge 开发就像为任何其他平台开发一样——除了开发人员必须编写可以无缝复制到所有设备类型的一致且安全的代码。T3】

边缘计算可能不会从开发人员开始，但开发人员将在组织利用边缘的能力中发挥巨大作用。对灵活性、开放性和未来的关注将有助于确保开发者构建出充分利用和扩展优势的应用。
