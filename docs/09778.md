# Linus Torvalds 表示支持内存安全的 Rust 语言

> 原文:[https://devo PS . com/Linus-Torvalds-signals-support-for-memory-safe-rust-language/](https://devops.com/linus-torvalds-signals-support-for-memory-safe-rust-language/)

Linus Torvalds 今天告诉欧洲开源峰会的与会者，他希望为 Rust 编程语言添加支持所需的基础工作将在即将到来的 Linux 操作系统 6.1 版本中奠定基础。

要增加对 Rust 的完全支持，还需要几个新的 Linux 版本，但是更容易地构建和部署用 Rust 编写的应用程序的能力将大大有助于提高 Linux 平台上应用程序的整体安全性。

开源安全基金会(OpenSSF)是 Linux 基金会的一个分支，作为保护开源软件供应链的更大计划的一部分，它呼吁取代像 Java 这样不安全的编程语言。Rust 是一种内存安全语言，它使用编译器来跟踪可以使用一次的值的所有权，并使用借用检查器来管理数据的使用方式，而不依赖于传统的垃圾收集技术。

本周早些时候， [Rust 基金会联合 OpenSSF 和 JFrog](https://devops.com/rust-foundation-allies-with-openssf-and-jfrog-to-secure-code/) 来解决在 Rust 中开发应用程序时可能出现的其他安全问题。

目前还不清楚有多少开发人员正在采用 Rust，但托瓦尔兹说，如果 Linux 的维护者同意，他个人很期待学习 Rust 语法——使用最近获得的基于 Arm 处理器的笔记本电脑。然而，Torvalds 也留下了一种可能性，即增加对 Rust 的支持可能不会像最初希望的那样成功。

托瓦尔兹还哀叹操作系统层面网络安全透明度的整体状况。在硬件级别遇到的许多错误不能与整个开源社区共享，因此对这些工作的任何修复都不会像操作系统的其余部分一样受到相同级别的同行审查。他表示:总体形势正在改善，但从发展的角度来看，仍然非常痛苦。他说，保密与开源精神背道而驰。

随着使用内存安全编程语言构建新的应用程序，组织的整体安全状况将稳步改善。当然，挑战在于知道如何使用这些语言构建应用程序的开发人员数量仍然相对较少。当然，替换已经使用各种遗留的非内存安全语言构建的数万亿行代码是一项巨大的任务，可能永远也不会完全完成——但是在应用程序安全性方面，转向现代编程语言是朝着正确方向迈出的重要一步。

甚至有一天，默认情况下，应用程序的编码方式会比现在安全得多。这可能不一定能解决所有的应用程序安全问题，但随着编程语言变得越来越安全，开发人员成为网络安全专家的压力应该会减轻——当然，前提是他们愿意学习一种新的编写代码的方法。
