如今，寻求数据驱动的企业面临着机器生成和业务数据泛滥的挑战。安全性、可用性、日志分析和其他机器生成的数据的庞大数量压倒了当前的解决方案。对于业务分析，每个新用例都需要几周甚至几个月的时间来建立；数据保存在筒仓中，限制了其对数据消费者的有用性；数据以有限的保留期存储；迁移和操作数据的需求会带来巨大的成本和法规遵从性风险。

ChaosSearch 使客户能够更好地了解 ^(TM) ，提供大规模的数据洞察，同时实现数据湖经济学的真正承诺。ChaosSearch 数据湖平台连接到客户自己的云存储环境中的数据并对其进行索引，使其完全可搜索，并利用开放 API，利用目前使用的现有数据工具进行数据分析。独特的多模型方法使数据驱动的团队能够从同一个平台访问日志和 BI 数据，使组织能够通过治理来民主化数据洞察。

虽然其他解决方案需要由解析或模式更改组成的复杂数据管道，但 ChaosSearch 会按原样索引所有数据，无需转换，同时自动检测本机模式。企业可以使用现有的分析工具在几分钟内分析他们的数据，而其他解决方案则需要几周或几个月。凭借其革命性的方法，ChaosSearch 克服了竞争解决方案的成本和复杂性，并提供了无限的可扩展性、业界领先的弹性以及大量的时间和成本节约。ChaosSearch 提供:

*   无与伦比的洞察时间(几分钟或几小时，而不是几天或几周)，无需数据转换或移动
*   通过 AWS 或 GCP 上的一个多模型数据湖实现经济高效的大规模性能，无需基础架构管理
*   通过自动发现和模式检测实现业务敏捷性和全面的 IT 治理；没有数据移动或搭建；存储一次，随处访问
*   通过用户选择的工具轻松访问数据，如 Tableau、Looker、Kibana、Grafana 或 Elastic API

Blackboard 的 DevOps 工程总监 Joel Snook 表示:“在我们于 2020 年底与 ChaosSearch 合作之前，我们的 SRE 团队一直在努力管理大量日志，以便在我们的所有产品线中以一致的方式实时支持数百万用户。“借助 ChaosSearch，我们能够为各种日志使用单一的解决方案，而无需管理日志工具。”

“当今的企业正在应对激增的数据量，如果他们想保持竞争力，就必须实时了解这些数据并采取措施。如果没有 ChaosSearch，他们需要针对 BI 和日志分析的多种解决方案，所有这些都会带来巨大的成本、复杂性和时间消耗，”ChaosSearch 首席执行官 Ed Walsh 说道。“ChaosSearch 帮助组织实现数据湖的真正承诺。只需将您的数据放入云中，ChaosSearch 就会激活您的数据湖。最后，您的所有数据都可用于分析，无需移动数据或更改数据分析师、安全团队和 IT 运营人员日常使用的工具。IT 领导者被我们的多模式平台的经济高效的性能和立竿见影的效果所折服。

基于这些功能，ChaosSearch 是当今普遍部署的弹性搜索解决方案和商业智能用例的理想替代品。通过 ChaosSearch，企业可以在 AWS S3 和 GCP 上执行可扩展的日志分析，使用熟悉的 Elasticsearch API 进行查询，使用 Kibana 进行日志分析和可视化，同时降低成本，提供无限保留并降低复杂性。借助 ChaosSearch SQL API，客户可以对 S3 和 GCP 的数据执行即席查询，并使用 Looker、Tableau 和 Metabase 等工具部署可扩展的商业智能。
