# 网络研讨会:为企业扩展 DevOps 和持续交付的经验教训

> 原文：<https://devops.com/12542/>

企业组织比以往任何时候都更需要平衡风险和快速响应不断变化的市场条件的能力。然而，许多因素似乎为采用 DevOps 制造了障碍，包括遗留系统、官僚机构、命令和控制文化等等。

在本次网络研讨会中，我们听到了企业 DevOps 运动中三位德高望重的声音——著名作家、研究人员、演讲者和 DevOps 传播者、Capital One 产品经理 Gene Kim、Tapabrata (Topo) Pal 和 XebiaLabs 的 DevOps 战略副总裁 Andrew Phillips，他们聚在一起讨论通过持续集成、持续测试和持续交付实施 DevOps 所需的流程、人员和技术。

该会议还将包括来自大型复杂组织的近 100 份经验报告的关键发现，这些组织在最近的 DevOps 企业峰会上展示了他们的 DevOps 转型，包括 Macy's、Capital One、Disney、Target、GE Capital、Western Union、Nordstrom 等。

与会者还了解到:
—成功转型的最佳成功模式
—2015 年 DevOps 企业峰会的主要经验
—大型复杂组织中 DevOps 转型的主要障碍
