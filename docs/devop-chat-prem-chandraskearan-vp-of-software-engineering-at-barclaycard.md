# DevOps 聊天:Prem Chandraskearan，巴克莱银行软件工程副总裁

> 原文：<https://devops.com/devop-chat-prem-chandraskearan-vp-of-software-engineering-at-barclaycard/>

在这次 DevOps 聊天中，我们采访了 Barclaycard 软件工程副总裁 Prem Chandraskearan。Prem 领导了那里的 DevOps 转型。他这样总结他的学习:

> 组织处理 DevOps 的最佳方式是从协作的角度。当开发和运营专家积极地团结协作时，从开发和维护的角度来看，解决方案都是设计良好、高效和智能的。当运营被视为一个软件问题时，开发人员和运营人员可以协同工作来实现一个工具，该工具从开始到结束都是整个流程的一部分。
> 
> DevOps 始终是一项进行中的工作，但该计划只能通过高质量的资源来改进，这些资源实现了开发人员和运营人员在一起工作时的各种背景和观点。由于身处信用卡/银行业，Barclaycard 在 DevOps 方面遇到了一些有趣的挑战，例如，“您构建它，您运行它”的口号对我们来说并不像对实施 DevOps 的其他企业那样管用。这肯定是一个学习曲线，我们的目标和最佳实践仍在不断取得成果，但我们仍在继续寻找自动化的方法。
> 
> 让团队一起工作的最佳方式是简单地确保他们不在孤岛中，这意味着如果有不同的开发和运营团队，他们必须在彼此的领域中花费时间来了解困难、功能如何工作、是什么使其运转等等。当开发和维护视角被一前一后地制定策略时，它会自动地产生一个更好的、更精简的工作流。
> 
> 确保 DevOps 计划长期成功的最佳方式是确保计划的基础是协作和对每个功能如何工作的理解。因此，自动化故障测试对于发现通常只能在生产中发现的问题是必不可少的。如果所有的基础都包括在内，并且流程是精简的和协作的，那么你的 DevOps 功能的整体质量将会更高。
