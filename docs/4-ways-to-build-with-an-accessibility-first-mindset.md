# 以易访问性第一的心态构建的 4 种方法

> 原文：<https://devops.com/4-ways-to-build-with-an-accessibility-first-mindset/>

超过 13 亿人患有某种形式的残疾 。这几乎占美国今天劳动力的 20%[虽然在缩小劳动力中的残疾包容差距方面取得了长足进步，但仍有更多工作要做。](https://www.bls.gov/news.release/pdf/disabl.pdf)

随着混合和[远程工作](https://devops.com/?s=remote+work)的不断普及，残障人士和在家工作的员工越来越需要为他们提供可访问的协作工具。

为了满足这种需求，开发人员需要在构建时考虑可访问性。通过这样做，他们可以创建更易于访问的软件，帮助残疾员工克服他们在与技术交互时可能遇到的任何独特挑战，同时为每个人创造一个愉快的工作环境。

有很多方法可以做到这一点，但是，这里有一些最佳实践和重要的学习，可以帮助开发人员以可访问性优先的心态进行构建。

### 挑战现状

创新总是挑战常规，尤其是在开发更公平的产品时。有许多方法可以做到这一点，这取决于现有的流程。尽管如此，我认为有一个领域是可以调整的，那就是内部对可访问性的思考和管理方式，尤其是在产品设计方面。

虽然今天绝大多数公司都以某种方式优先考虑可访问性，但通常是通过一个团队来完成的，或者在较小的部门中，一个大使来负责管理这项工作。这种方法经常导致易访问性被看作是设计的附加物，发生在过程的最后。实际上，对于开发人员和 DevOps 团队来说，采用中心辐射式方法来实现可访问性更好。在这个模型中，有一个集中的专家资源，人们可以去那里寻求建议，但是单个的团队同时也要致力于并注意可访问性。作为一个集体，其结果将更具协作性和影响力。

同样，挑战现状可以简单到鼓励与团队讨论可访问性。如果可访问性是你想和你的团队讨论的一个话题，但不知道如何提出，记住大多数可访问性问题源于缺乏意识。这通常是一个疏忽，一旦对话开始，想法就开始流动。那么，为什么不开始对话呢？准备好用户故事和潜在的解决方案，你的团队将立即看到所有需要做出的适当改变。

### 直接与残疾人一起工作

当构建任何软件时，从各种各样的人那里收集反馈是很重要的，这样你就可以确定你正在构建人们需要的功能和体验。

当我们开始一个项目或更新时，我们努力理解当前的功能或其他类似的产品是如何可访问的(或不可访问的),以便我们能够更好地理解我们客户的期望或挫折。有很多方法可以做到这一点，但也许最有影响力的是雇佣残疾人，并与他们就产品进行公开讨论。除了招聘，与雇佣残疾人的特定公司签订合同来测试产品并给出他们的反馈也是很有价值的。

收到这些反馈是开发成功产品的关键，通常会带来更加灵活和全面的用户体验。例如，为盲人或弱视人士设计的基于文本的应用程序具有广泛的文本可扩展性，对于喜欢小文本或在需要不同字符串长度的语言中使用该产品的用户同样有用。类似地，为失聪或听力困难的人设计的字幕对任何可能处于嘈杂环境中或喜欢阅读书面内容而不是音频内容的人都很有用。当你让一个产品为一个特定的群体很好地工作时，你自然地为每个人创造了一个更好的、适应性更强的体验；你在用户所在的地方与他们见面。

### 不要做假设

当你与残疾人交谈或设计新产品时，不做假设是非常重要的。例如，在我职业生涯的早期，我会假设一个基于图像的程序对于一个盲人来说是完全没有意思的；我现在明白这绝对不是真的。通过将有趣和生动的图像描述整合到产品中，一个基于图像的程序可以变得容易被所有人接受和参与。

通过避免假设某人能做什么或不能做什么是基于他们的能力，我们将自己开放给一个创造性的解决方案和可能性的世界，否则我们可能不会考虑。

虽然残疾人士最终会提供最好的反馈，但不要忘记花时间独自或作为一个团队积极挑战您的设计，并寻找提高其可访问性的方法。作为一个团队花一两个小时使用你的产品和屏幕阅读器，比如 VoiceOver 或 TalkBack，或者只用键盘而不用鼠标来使用你的应用程序。通过不断参与和测试您自己的设计，您将更好地理解用户体验，并可能自己发现错误或可访问性问题。

### 为试错做好准备，分享你所学到的东西

12 年前，当我第一次开始关注优先开发可访问性时，这个领域几乎没有关于可访问性的文档——尤其是在为移动技术设计时。这导致了很大的挫败感，并且当我们试图(并且失败了)把它做好时，经常会减慢我们的工作。有时这涉及到一个微妙的平衡，既要坚持你产品的既定标准(例如，东西应该如何贴标签或看起来应该如何)，又要同时解决可访问性和效率。

从那以后，开发人员领域发生了很大的变化。今天，有更多的关于这个主题的材料以及需要遵守的行业标准。大学现在在计算机科学课程中提供更多的可访问性培训，残疾内容创作者分享更多的经验，公司在共享可访问性资源方面更加开放。

然而，试错法对于以易访问性为先的思想来构建仍然是至关重要的，特别是当我们在这个领域继续创新的时候。所以要有耐心，相信这个过程。

包容性设计为每个人带来更好的产品，特别是为世界上所有有某种残疾的人。请确保您是解决方案的一部分，并与您的开发人员社区分享您的经验，以继续这项运动。

作为一名开发人员，你花越多的时间学习和倾听你的社区，让他们成为你的头号资源，你就会越了解他们的需求，以及你如何才能创造最终的用户体验。反过来，这使用户无论其能力或环境如何，都能做到最好。
