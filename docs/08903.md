# 测试不是铁板一块

> 原文：<https://devops.com/devops-unbound-ep-8-testing-is-not-a-monolith/>

DevOps 带来了 CI/CD、自动化和测试的实施。这足以实现“左移测试”吗？或者测试需要进一步左移？我们需要新的或额外的测试来接受(ATDD)或行为测试(BDD)吗？

在这一集的[DevOps unbounded](https://digitalanarchist.com/videos/devops-unbound/devops-unbound-ep-8)中，我们将与 NAIC 的 Christine Fisher、Qualitest 的 Leandro 梅伦德斯(又名 seor Performo)、Ken Pugh Inc .的 Ken Pugh 和 Accelerated Strategies Group 的 Mitch Ashley 一起探索 devo PS 和 CI/CD 管道中的测试状况，以及测试可以在哪些方面发展以跟上软件开发和交付的步伐。
