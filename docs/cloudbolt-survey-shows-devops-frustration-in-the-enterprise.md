# CloudBolt 调查显示 DevOps 在企业中的挫败感

> 原文：<https://devops.com/cloudbolt-survey-shows-devops-frustration-in-the-enterprise/>

今天发布的一项针对拥有 1000 多名员工的组织的 200 多名高级 IT 领导的调查发现，只有 4%的人认为他们的组织在持续集成/持续交付( [CI/CD](https://devops.com/?s=CI/CD) )方面是专家。

由为许多 IT 环境提供平台的 CloudBolt Software 进行的调查还发现，只有 11%的人认为他们的 CI/CD 基础架构是可靠的。另外 69%的人说他们的 CI/CD 基础架构有些可靠，而 21%的人说他们的平台有些不可靠或根本不可靠。整整 85%的受访者还表示，他们会定期测试他们赖以驱动开发运维流程的基础架构。

总体而言，CloudBolt 调查发现，只有 5%的受访者报告他们的组织每天部署多次代码。大多数受访者(85%)表示，他们的组织仅每周或每月部署一次代码。

整整 70%的人说，平均来说，建立一个管道也需要几天甚至几周的时间。近三分之二(63%)的受访者表示，在设置管道基础设施时，涉及太多的手动流程。超过一半(56%)的受访者还指出，他们没有针对部署后任务的自动化流程。

超过一半(53%)的人说主动检测基础设施问题也是一个挑战，而 55%的人说很难在多个开发团队之间创建一个一致的 CI/CD 管道环境。改善这些管道环境需要通过自动化流程加快基础设施的供应过程(70%)；调查受访者表示，持续检测基础架构问题以减少测试挑战和失败(62%)并主动简化基础架构问题的补救(56%)。

CloudBolt 软件调查特别确定了组织在使用 Terraform 管理基础架构即代码(IaC)时遇到的问题。只有 5%的受访者表示他们对使用 Terraform“非常满意”,另有 75%的受访者表示他们有些满意。主要挑战包括难以在内部基础架构部署中使用 it(62%)，缺乏对已部署资源的第二天管理(53%)和治理(42%)。只有 2%的受访者表示他们正在使用 Terraform 部署 75%或更多的基础架构。整整 90%的人指出，Terraform 需要更多的定制代码与其他关键工具和系统集成才能正常工作。

最后，29%的人说他们的组织缺乏特定的 Terraform 专业知识。最受欢迎的 Terraform 功能使非 Terraform 专家能够使用 Terraform 轻松部署基础设施；确保只执行带有 IT“护栏”的 Terraform 计划，以及可视化和成本优化 Terraform 部署的基础设施的能力。超过三分之一(35%)的人希望在使用 Terraform 时简化内部基础设施的部署。

CloudBolt 首席执行官 Jeff Kulkowski 表示，调查清楚地表明，没有足够的专业知识来大规模实施 DevOps 最佳实践，而不支持通常缺乏编程专业知识的 it 管理员来管理基础架构和 DevOps 平台。他补充说，这一要求产生了对基于一套可视化工具的 IT 管理平台的需求，使 DevOps 专业人员和传统 IT 经理能够更好地协作。开发运维团队可以随时管理 IaC，但让 it 运营人员与这些团队一起管理 IT 基础架构也很重要。

显然，DevOps 作为一门 it 学科，在被企业 IT 组织广泛采用之前，还有很长的路要走。现在的挑战是消除许多阻碍 IT 现代化计划的低级障碍。
