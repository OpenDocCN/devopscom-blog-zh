# 新遗迹:可观测性是 2022 DevOps 议程的重中之重

> 原文：<https://devops.com/new-relic-observability-tops-2022-devops-agenda/>

如果 2021 年是 IT 组织意识到可观测性需求的一年，那么在接下来的一年里，可观测性的愿望应该会实现。

一份由可观察性平台提供商 New Relic 发布的[报告](https://newrelic.com/resources/report/2021-observability-forecast)认为，至少有 11 个因素汇聚在一起，将使可观察性成为组织的绝对要求。

最重要的是一个简单的事实:现在需要可观察性来推动通常跨越多个流程的数字业务转型计划。事实上，New Relic 对全球 1，300 名 IT 领导者、软件工程师和开发人员进行的调查发现，90%的受访者认为可观察性对业务至关重要。

其次，报告指出，结果是，随着组织采取更全面的方法来管理软件开发生命周期，可观察性现在开始在生产前和生产后环境中应用。

第三，该报告表明，IT 团队希望做出更明智的决策，更多地基于事实而不是直觉。为此，团队使用从整个企业应用程序环境中收集的数据。事实上，提高企业可观察性的第四个因素是检测应用环境的成本正在下降，报告指出这使得收集相关数据变得更加容易。

第五和第六个因素反映了对 IT 团队当前使用的监控工具数量进行合理化的愿望，以及基于使用的定价同时降低了根据需要利用可观察性平台的成本。

第七个因素是向开源软件的整体转变，这使得可观测性平台提供商能够更快地共同利用创新。与开源趋势密切相关的第八个因素是 Kubernetes 等更复杂的开源平台的兴起，这些平台需要更高的可见性才能成功部署和管理。

报告中确定的第九个趋势是越来越多地采用 DevOps 最佳实践，这鼓励 IT 团队将可观察性视为任何 IT 管理策略的核心原则。报告引用的第十个因素是需要推动 IT 专业人员之间的合作。

最后，推动进一步采用可观察性的第十一个因素是对提高应用程序可靠性的需求，因为越来越多的组织意识到他们依赖软件来交付基本服务。

New Relic 的集团副总裁兼总经理 Buddy Brewer 表示，现在可观测性平台变得越来越容易访问，下一个重大挑战将是教 IT 专业人员在对传统单片应用和各种新兴的基于微服务的应用进行故障排除时应该问什么问题。他补充说，这些查询的结果需要与开发人员和内部 IT 运营团队轻松共享，他们必须共同努力，最大限度地提高应用程序的性能和可用性。布鲁尔指出，后一个要求就是为什么 New Relic 决定[收购 CodeStream](https://devops.com/new-relic-acquires-codestream-to-close-observability-gap/?) 以使遥测数据可用于集成开发环境(ide)。

当然，IT 团队并不缺少可观察性平台。事实上，现在最大的挑战是让每个应用程序利益相关者同意在一个可观察性平台上标准化——这不是一个小的壮举，因为许多 DevOps 团队可能已经偏好一种方法而不是另一种。
