 *## 1.不可阻挡的铁锈

本周的第一件事是:突然之间，似乎每个人都在谈论 Rust 语言。窃窃私语很快变成了喋喋不休的咆哮。从 12 年前的卑微开始，现在听起来像是**下一个大事件**。

### 分析:更安全、更快、*更好*

你得努力在 Rust 里写内存相关的漏洞。但是不像其他“更安全”的语言，比如 C#，你不需要因为过多的检查或者**垃圾收集**而导致运行缓慢。

莉莉·海·纽曼就是这样一个话匣子:**接管科技的语言**

**`Solution to memory-safety issues`**
人都会犯错，所以代码不可避免地会包含错误。…但是，用一种叫做 Rust 的语言编写软件的运动正在兴起，因为代码在一个重要方面是防错的:通过设计，开发人员不会意外地创建最常见类型的可利用的安全漏洞。通过用 Rust(而不是 C 或 C++)编写新软件，程序员可以确信他们没有引入任何内存安全缺陷，而这些缺陷构成了所有软件漏洞的主要部分。…因为 Rust 产生了更安全的代码，而且最重要的是，它不会降低性能，所以这种语言一直在稳步赢得追随者，现在正处于一个转折点。…支持者强调，Rust …可以与用其他语言编写的软件很好地互操作，它之所以至关重要，只是因为它满足了一个迫切的需求。然而，过渡到 Rust 的最大挑战之一恰恰是开发人员已经花了几十年时间用不安全的内存语言编写重要代码。[但是]似乎每天都有针对内存安全问题的某种解决方案出现。就在上周，如果用一种内存安全的语言编写该机制，那么无处不在的安全通信库 OpenSSL 中的一个高危险度漏洞本来是可以避免的。

转行是什么感觉？[马丁·法森](https://blog.startifact.com/posts/random-rust-impressions/ "read the full text")几十年来一直使用 C、C++、TypeScript 和 Python:

Rust 工具很棒:Cargo 做了很多，Rust analyzer 在复杂的设置中工作，autoformat 工作，Clippy 给出了很好的建议，有时甚至可以自动重构。默认值很好！事实上，我认为 Rust 工具是我迄今为止使用过的最好的编程语言。
…
有很多高质量的锈库可用。图书馆生态系统提供的内容也相当广泛。… Rust 包(称为板条箱)很容易在网上找到[并且]通常很容易找到文档[但是]文档质量参差不齐。
……
测试驱动开发帮助你用清晰和最小的契约设计代码。…如果你想测试一个依赖于其他东西的子系统，你可以把它和一个伪实现组合在一起，这个伪实现提供足够的行为让你测试你自己的代码。…这真的很强大。

和[纳比勒·埃尔卡提卜](https://n-eq.github.io/blog/2022/11/01/rust-fiddling-2-years "read the full text")分享了他的印象:

大约两年前，Rust……被作为一种强类型语言“卖给”我，它承诺提供很好的工具来防止内存安全错误……通过在编译期间跟踪对象生存期和所有引用的可变范围。…当时，我很难接受这个新概念。
…
我参与的一个项目包括将 90 万多行 C 代码从 Rust 中桥接出来，并将其桥接到 Rust 中。没有太大的困难…而且这个用例似乎已经建立得相当好了。…你仍然需要一些“管道”代码…但这是你要付出的代价，而且…相当低。
……
严格的所有权和借用有助于确保安全高效地访问数据。它的现代语法和设计允许更容易理解和使用…模式和最近的范例。此外……Rust 在确保线程并发运行而没有竞争条件和类似问题方面做得非常好。

等等。*暂停。*不久前，我们不也是这么说戈朗的吗？[乔纳森](https://developers.slashdot.org/comments.pl?sid=22338023&cid=63029281 "read the full text")比较和对比:

Rust 比 Go 更容易引入现有的 C 代码库。因此，如果您正在进行一个已经有几百万行代码的项目，明智的决定是将所有处理外部输入和数据解析的接口重写到 Rust 中，以帮助将程序与常见的与内存相关的攻击媒介隔离开来。您不必重写几百万行代码来大幅提高应用程序的安全性。

但是 Rust 的治理需要发展——而且要快。[奥利维耶·福雷](https://poignardazur.github.io//2022/11/05/political-messages-in-the-rust-blog/ "read the full text")在客厅里摆出一副政治厚皮动物的样子:*【你被解雇了。]*

让我们提一下房间里的大象。…宣布 Rust 编程语言 1.65 版本的文章包括:……“我们声援伊朗人民争取人权的斗争。”…在 1.59 版本公告中:……“我们与乌克兰人民站在一起。”[以及]对于 1.44 版本:……“表明立场，反对目前在美国和全世界发生的警察暴行。”许多人反对利用科技论坛传播政治信息，对一些人来说，这反映了他们不一定认同的世界观。…很多人…在 Rust 博客发表这些政治言论时感到被欺负了。…如果你真的想推动人权，而不仅仅是美国进步议程的淡化版本，你需要有一些选择有价值的事业的方式，而不仅仅是主流渠道上的任何东西。你需要一套标准，并记录在案。

离开[虚拟世界](https://developers.slashdot.org/comments.pl?sid=22338023&cid=63029413 "read the full text")的草坪:

我并不是在为 COBOL 辩护，但是它有很多，你会对它的数量感到惊讶。它也是最早的语言之一，所以尊重你的长辈。我还听说如果你说三遍 COBOL，格蕾丝·赫柏的鬼魂就会出现在你浴室的镜子里。

## 2.埃隆·马斯克说:你必须回到办公室

Twitter 员工将不再能够在家工作。新任首席执行官表示，他们甚至不能实行混合工作制——要么工作 40 多小时，要么辞职。

### 分析:那是不行的

人们很容易认为埃隆的“非传统”想法不可行。但是真的很难看出这怎么可能是一个好主意。

[Kurt Wagner](https://www.bloomberg.com/news/articles/2022-11-10/musk-s-first-email-to-twitter-staff-ends-remote-work "read the full text") : **马斯克给 Twitter 员工的第一封邮件**

Twitter 公司的新老板埃隆马斯克(Elon Musk)周三晚些时候第一次给员工发电子邮件，让他们为“未来的困难时期”做好准备，并禁止远程工作，除非他亲自批准。…即将生效的新规定要求员工每周至少在办公室工作 40 小时。在马斯克到来之前，Twitter 已经建立了一个永久的随处工作的安排。…他还取消了 Twitter 员工日历上的“休息日”……这是在疫情期间推出的每月一次的全公司休息日，表明马斯克对 Twitter 现有的工作文化不耐烦。马斯克领导 Twitter 已经将近两周了，在此期间，他已经解雇了大约一半的员工和大部分高管。

没人能活捉 T2 的黑夜:

LOL，埃隆。你不明白 Twitter 不是 SpaceX 类型的东西，它有一个雄心勃勃、非常雄心勃勃的目标，尽管工作繁重，但它可能会让人们留下来。现在在 Twitter 工作的人都不在那里，因为他们都在奉承你。通过灌输一种不信任的气氛，让他们一天比一天更怨恨你……让你更接近完全失去你的 440 亿美元的错误。…每个人都会辞职。

和 [@MartinSFP](https://twitter.com/MartinSFP/status/1590645199868678144 "read the full text") 之前都看过了:

典型的“我的房子我做主”来自新老板。但是与这些天其他地方工作的数量以及想要 T1 工作的工人数量非常不一致。
