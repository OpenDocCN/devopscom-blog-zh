# Zebrium 推出“根本原因即服务”,使流行的可观察性工具能够自动找到软件问题和停机的根本原因

> 原文：<https://devops.com/zebrium-launches-root-cause-as-a-service-enabling-popular-observability-tools-to-automatically-find-the-root-cause-of-software-problems-and-outages/>

*RCaaS 将平均解决时间减少了 90%，并具有超过 95%的验证准确率*

**加利福尼亚州圣克拉拉–2022 年 6 月 15 日**–Zebrium，使用机器学习自动查找软件问题根本原因的领导者，今天宣布推出 Zebrium 根本原因即服务(RCaaS)，这是一种新的解决方案，增加了监控和可观察性工具的功能，如 Datadog、New Relic、Elastic、Dynatrace、Grafana、AppDynamics、ScienceLogic 和其他工具，以自动查找软件和基础设施事件的根本原因。

当生产软件发生事故时，Zebrium RCaaS 会自动找到根本原因，并直接在现有的监控仪表板上显示问题摘要，以及显示指标、跟踪和 APM 数据的其他图表。这使得站点可靠性工程师(SREs)、DevOps 人员和开发人员能够将解决(MTTR)软件或基础设施问题的平均时间减少 90%。

今天，当技术团队遇到新的服务中断或问题时，他们通常依靠可观察性工具来促进故障排除过程。如果没有 Zebrium，这需要查看指标以确定问题“何时”开始，深入跟踪或 APM 数据以缩小问题来源(“何处”)，最后梳理来自应用程序和基础架构堆栈的大量日志以确定根本原因(“为什么”)。此过程可能需要数小时，需要大量团队资源，同时关键服务仍会受到影响。现在有了 Zebrium RCaaS，挖掘日志的艰苦过程实现了自动化。最终结果是，RCaaS 很快发现了技术团队通过手动梳理日志最终会发现的根本原因指标。

RCaaS 在 95%以上的事故中找到正确根本原因的准确率得到了验证。“思科技术援助中心(TAC)每月花费数千小时分析软件日志，以找到客户事件的根本原因，”思科系统全球 TAC 创新、自动化和中断总监 Koree Mires 表示。“多年来，我们一直在研究帮助自动化这一过程的方法。当我们看到 Zebrium 时，我们立刻被深深打动了。为了验证其有效性，我们用四个产品线和 192 个实际客户事件测试了 RCaaS。我们惊讶地发现，RCaaS 在 95%以上的时间里自动正确地找到了根本原因。我们现在正在利用这项技术来加快客户事件的解决，并将在今年继续将其推广到更多的产品线。”

Zebrium RCaaS 旨在使 sre、Devops 工程师和开发人员已经在使用的相同工具和工作流程中提供根本原因的详细信息。RCaaS 已经完成了与流行的可观察性和监控工具的“开箱即用”集成，包括 Datadog、New Relic、Elastic、Dynatrace、AppDynamics、Grafana、ScienceLogic 等。它还本机集成了事件管理和响应平台，包括 PagerDuty、Opsgenie、Victorops、Slack、Teams 和电子邮件系统。另外 3 个 rd party 工具也可以通过一组开放 API 轻松集成。

Zebrium 首席执行官 Ajay Singh 表示:“停机成本不断上升，让工程师来解决这个问题并不是一个可扩展的解决方案。“当软件团队需要解决应用程序事故时，速度和准确性是至关重要的，因此唯一的解决方法是自动化的根本原因分析(RCA)。Zebrium RCaaS 是一种行之有效的方法。由于我们的平台不需要任何手动培训或规则，客户可以在几分钟内开始，并利用 RCA 作为几乎任何类型的可观察性工具。”
