# 随着开发人员角色的演变留住人才

> 原文：<https://devops.com/retaining-talent-as-the-developer-role-evolves/>

由于远程工作的增加和大量的辞职，传统的开发角色正在迅速演变。公司领导层也必须适应，以留住人才，并确保他们的开发团队的成功。

美国劳工统计局报告称，2021 年 11 月有创纪录的[450 万美国人自愿离职](https://www.nytimes.com/2022/01/04/business/economy/job-openings-coronavirus.html)——2021 年 12 月， [CompTIA 的科技就业报告](https://www.comptia.org/newsroom/2021/12/03/tech-hiring-momentum-and-job-gains-continue-in-face-of-headwinds-comptia-analysis-finds) 显示，科技和 IT 职位的公开招聘职位同比增长 53%。随着 2022 年对技术和 it 人才的需求持续存在，普遍的倦怠和技术技能差距使得寻找和留住顶级 IT 人才变得更加困难。

但这不是唯一的挑战。远程工作的影响引发了开发人员角色的根本变化，科技公司越来越多地向云基础设施迁移进一步加剧了这种变化。在组织内部，缺乏基础知识和正式的公司培训正在造成开发方法应用上的差异。

随着开发人员期望和技能的发展，[敏捷开发](https://devops.com/?s=Agile+development)方法比以往任何时候都更加相关，这要归功于它的适应性和灵活的迭代过程。公司领导层必须将这一事实内在化，并将正式的实践落实到位，以在他们的组织内标准化方法。通过这样做，领导者可以用他们需要的工具武装开发人员，以适应快速变化的开发标准和方法——并在高流动的市场中留住顶尖的 IT 人才。

## 开发人员角色的变化带来了新的可能性和挑战

疫情极大地促进了对基于远程的工作结构的需求，随着员工离开办公室，许多组织开始将迁移到基于云的基础架构(包括物理基础架构和组织基础架构)置于更高的优先级。这种推动给开发团队带来了巨大的负担，他们需要以高于正常的速度将组织和项目数据转移到云基础设施。对于 IT 专业人员来说，疫情引发的灾难响应加速了向云的永久迁移，以此来保护对数字基础设施和流程的访问。特别是对于开发人员来说，这种迁移促使许多团队接受基础设施即代码(IaC)。

IaC 是 DevOps 的核心实践，允许通过代码而不是手动过程来构建和管理基础设施(如网络、云服务和虚拟机)。这意味着开发人员越来越多地编写他们的软件运行的实际基础设施，这对于自动化、可追溯性和可靠性具有改变游戏规则的优势，并影响几乎所有的开发运维实践。由于办公室里没有人执行管理和操作信息技术工作，IaC 特别有用。但是由于开发人员除了他们的传统代码之外，还必须具备必要的技术技能来编写这种基础设施，IaC 创造了一个灰色区域，在这个区域中，开发人员也成为 IT 操作员和系统设计师。

向 IaC 的转变已经为开发人员发展了标准，并且它要求开发团队和公司作为一个整体具有更高的灵活性。采用 IaC 与加强和细化敏捷过程是并行不悖的，因为创建标准化的敏捷实践不仅能够实现更大的敏捷性，，还能实现更准确的可见性——随着人员配备模型的发展以及开发人员和 IT 操作员的角色变得更加交织，这两种能力都是必要的。但是，公司领导可以定位他们的开发团队，以便在这个不断变化的技术环境中取得成功，并获得 IaC 和基于云的技术提供的增强协作和效率的好处。

## 领导支持开发团队的 3 种主要方式

开发团队的角色正在快速变化，领导者必须跟上步伐以留住 IT 人才并保护其组织的健康发展。记住这一点，当您的开发团队在不断变化的开发标准中导航时，有三种方法可以支持他们。

1.  **规范最佳实践。**即使在一个单一的组织或单一的开发方法中，IT 和开发专业人员之间的关键实践也可能不同。为了成功地执行敏捷，团队必须有标准化的最佳实践来确保基础知识的一致性。敏捷的迭代过程可以带来有用的速记和即兴创作，但前提是开发人员使用相同的方法标准和坚实的基础知识。
2.  增加对正式培训的承诺——尤其是对新员工。为了充分受益于敏捷的能力，所有员工都必须具备敏捷流程的基础知识——尤其是新员工。提供关于行业认可的敏捷标准的正规教育和培训，以消除技能差距，并在您的组织内培养机构知识。机构知识确保质量标准是明确定义的和可实现的，最终加强您的开发团队的协作能力。
3.  **优先考虑减少倦怠的措施。**疫情揭示了各行各业的倦怠状况，IT/技术界也不例外。实施旨在减少员工倦怠的切实措施取决于领导层，比如管理现实的工作量，允许灵活的工作时间表，以及实施鼓励工作与生活平衡的政策。一致的标准导致有意义的报告，这确保了团队绩效和工作量的准确测量。

## 为未来的发展做准备

远程工作无处可去，未能采用远程技术的公司面临员工流失和过时的风险。随着开发方法的发展，以适应疫情引发的挑战和 IT 招聘缺口，开发人员的角色将继续变化。在 5 到 10 年内，它将与今天的样子大不相同，公司必须愿意接受这些变化，以确保其 it 和开发团队的成功。

敏捷开发方法旨在适应快速的技术创新和组织变革。向远程工作的转变和 IaC 革命是未来的趋势 — ，它就在眼前。这些力量正在推动混合开发方法，为它们强加的新标准服务。接受这些变化是组织的当务之急，通过在您的开发团队中致力于标准化知识，您可以留住您的顶级 IT 人才，并确保您的团队做好准备迎接这些新挑战。
