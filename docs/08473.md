# 总的来说，OCP 是 CoreOS 和集装箱的胜利

> 原文:[https://devo PS . com/OCP-is-a-victory-for-coreos-and-containers-in-general/](https://devops.com/ocp-is-a-victory-for-coreos-and-containers-in-general/)

一个由主要科技公司组成的广泛联盟已经联合起来创建了开放集装箱项目——或称 OCP。围绕软件容器开发和维护通用标准的举动证明了 CoreOS 从 Docker 的分离是正确的，并将两者重新结合到一个共同的事业中，这将使所有的容器平台和技术更加强大。

一份新闻稿宣布开放集装箱项目解释了 OCP 的整体愿景。“位于 Linux 基金会下，OCP 的使命是使用户和公司能够继续创新和开发基于容器的解决方案，并相信他们现有的开发成果将得到保护，不会出现行业分裂。”

Docker 和 CoreOS 位于 OCP 的中心，但它们绝不是孤独的。DevOps 和 container technologies 的一个名副其实的“名人录”已经招募来支持 OCP，包括 Amazon Web Services、Cisco、EMC、Google、HP、IBM、Linux Foundation、Microsoft、Red Hat 和 VMware。

Docker 曾经有过一个标准的容器宣言，阐明了它认为所有容器都应该具备的一些基本属性。标准容器宣言呼吁容器以标准的方式运行，不依赖于内容，不依赖于基础设施，并且为自动化而设计。当 Docker 删除宣言并似乎转向更专有的策略时，CoreOS 分离出来并推出了自己的容器技术，以继续容器标准的愿景。

“容器的全面成功和采用需要一个开放的、定义良好的规范。在 [CoreOS](https://devops.com/2014/12/03/shooting-rocket-dockers-bow/) 时，当我们开始应用容器规范(appc)时，我们的目标是拥有一个设计良好的软件容器规范。通过 OCP，我们现在已经与 Docker 和许多行业巨头走到了一起，努力统一这一目标。”

OCP 的推出和支持它的公司名单表明，CoreOS 并不是唯一一个追求集装箱标准愿景的公司。微软 Azure CVP Jason Zander 表示:“我们很高兴能够支持这一努力，让整个行业就容器的核心[标准达成一致，我们认为这将使所有客户受益，无论他们使用哪家云提供商。“通过过去一年与 Docker 的密切合作，我们展示了我们对开放的承诺，并欢迎有机会帮助为客户创建一个开放的平台。”](https://devops.com/2015/04/13/hyper-v-containers-combine-virtualization-with-containerization/)

VMware 云原生应用副总裁兼首席技术官 Kit Colbert 解释说:“客户需要一个开放、可互操作的软件容器未来，我们很高兴看到业界携手建立一个通用标准。“我们对开放式容器项目的参与强化了我们的目标，即帮助客户更快地创新，并在他们的环境中采用该技术时从容器中受益。我们期待着与社区在容器可移植性和安全性方面进行合作，这是企业 IT 的重点领域。”

根据 OCP 的新闻稿，Docker 将向该项目捐赠其软件容器格式和运行时的代码，以及相关的规范。应用容器规范(appc)的领导团队也将为 OCP 提供技术领导和支持。

Docker 的首席执行官 Ben Golub 总结道，感谢大家的努力和对 OCP 的支持。“我们要感谢业内许多人共同支持这一重要举措。我们要感谢 Linux 基金会，他们做了令人难以置信的工作，帮助把这个广泛的群体团结在一起。考虑到容器和操作系统交叉的重要性，我们也要感谢 Red Hat 和微软在这项计划中的早期领导作用。我们真诚地感谢 IBM 和 Intel 对开放治理的不断鼓励。最后，我们要感谢 appc 社区的成员，包括 Google 和 CoreOS，感谢他们愿意与我们一起建设性地解决这些关键问题。”

既然所有的主要参与者都团结在一个共同的事业中来定义和维护容器的行业标准，那么开发者和组织将更容易毫无保留地接受容器技术。相信你投入的时间、金钱和努力不会被浪费，也不会让你陷入困境，这是一种解放。
