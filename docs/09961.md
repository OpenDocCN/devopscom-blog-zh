# 可视化地为您的开发团队配备人员

> 原文:[https://devops.com/visually-staffing-your-developer-teams/](https://devops.com/visually-staffing-your-developer-teams/)

在[开发团队](https://devops.com/?s=developer+teams)中的有效合作是创造和改进任何产品的关键，所以你选择谁加入这个团队——以及它的结构如何——会有很大的不同。做出这些决定需要考虑很多因素，由于混合工作环境的广泛流行，挑战甚至更大。为了使流程变得更简单、更透明，并从每个项目的一开始就为团队的成功做好准备，使用可视化的人员配置方法会很有帮助。

## 组建团队时的考虑事项

当一个总经理在组建职业体育团队时，他们必须考虑到个性、能力和领导风格，以确保团队顺利运作并赢得比赛。组建开发团队时，也需要考虑类似的问题。

例如，一个团队必须拥有正确的专业知识平衡。拥有一个有前端经验但很少或没有后端经验的团队可能会有问题。此外，一个全是实习生或初级工程师、很少或根本没有高级工程师的团队，将缺乏成功所需的领导力和专业知识。

在组建团队时，编码风格也是一个主要的考虑因素。一些工程师可能是“牛仔编码员”——在开发产品时不怕冒险的编码员。然而，其他工程师可能患有“分析麻痹”，花费太多时间来确保代码尽可能完美。对于团队来说，混合使用这些风格是很重要的，这样开发过程就不会向任何一个方向倾斜太多，这样整个团队就能在手头的项目中找到合适的平衡点。

## 使用可视化协作来集合团队

作为一个开发团队的领导者，我发现使用可视化的方法来为我们的团队配备人员——以及在这些团队中进行可视化的协作——是非常宝贵的。它使我们更容易确保在正确的团队中拥有正确的专业知识和编码风格，同时管理当今商业环境中的快速变化。随着每个项目人员的可视化，领导者和团队成员可以很容易地看到谁领导这些项目，以及每个团队中有多少人。

经理或项目领导应该能够看到人员配备的所有相关因素，然后才能做出明智的决定，决定谁应该在团队中，或者他们是否需要额外的支持。要开始可视化地为您的团队配备人员，首先要有一个图表，除了显示每个人所在的团队之外，还要清楚地显示每个人的职责和标志(如果他们是团队的新成员)。当决策者能够一眼看到这些因素时，这有助于他们快速确定团队是否有差距，是否需要额外的支持。

为了使可视化人员配备更加有效，您可能希望在查看团队结构时获得更多细节。这可能包括给团队成员贴上相关属性的标签，如头衔、资历、位置等。因此，如果一个团队需要一名高级工程师或从事过类似项目的人，可以快速将他们的名字拖到你的图表上，然后将他们放到另一个团队中。使用这种方法，团队可以看到一个新的人才组合如何一起工作，以及这种调整如何影响其他团队。T3】

我们还能够绘制报告结构图。了解每个人向谁报告以及每个领导有多少个直接下属是很重要的。这有助于防止精疲力竭，并确保每个经理都有足够的时间领导他们的直接下属，并了解团队的需求。

一旦创建了团队，工作会议，如项目增量(PI)计划会议，也将受益于可视化协作解决方案，如虚拟白板。在混合和远程工作最近占据主导地位之前，许多规划会议都包括在物理白板前的面对面讨论。此外，对决策的投票包括物理的、彩色涂层的便利贴，每个人将把它放在白板上以表明他们的投票。

![](../Images/2d9ada6b7d9919c5207ce443c082b76a.png)T2】

*描述:开发团队可以参加虚拟 PI 规划会议，实现更无缝的协作。*

使用可视化协作不仅有助于在混合环境中重现这种体验，还会使其更加有效和高效。例如，使用虚拟白板解决方案可以改善团队使用通用项目管理工具(如吉拉)的体验。合适的可视化协作解决方案可以允许团队提取故事，并为每个人显示最新的项目信息，以便在协作空间内直接提供输入。结果是，团队领导能够通过实时协作和来自世界各地团队的异步输入来创建高水平的计划，同时大大减少了在物理白板前进行协作时常见的缓慢的手动步骤。这些缓慢的手动步骤可能包括将故事从项目管理软件转移到物理卡片上，随着讨论的继续移动这些卡片，然后——一旦决定是最终的——手动将新信息上传回吉拉或另一个项目管理工具。由于今天对开发团队的依赖，为了更有效的过程而去除这些步骤是非常宝贵的。

## 直观地为您的团队配备人员

即使在混合环境中，为团队配备人员和支持协作也可能是一种顺畅的体验。我发现可视化团队如何合作提供了做出最佳决策所需的清晰性。如果有必要的话，它还允许容易地改变团队的组成。可视化的团队人员配置允许团队领导让开发人员有一个良好的开端，并且从开始到结束始终朝着项目目标努力。
