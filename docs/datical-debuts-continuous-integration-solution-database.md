# DATICAL 首次推出数据库持续集成解决方案

> 原文：<https://devops.com/datical-debuts-continuous-integration-solution-database/>

*Datical DB 4.0 使数据库管理员不必审查绝大多数数据库变更*

**得克萨斯州奥斯汀市—2016 年 7 月 7 日—[领先的敏捷数据库自动化解决方案提供商 Datical](http://www.datical.com/)** 今天宣布推出 Datical DB 4.0。这一最新版本将开发驱动的数据库变更管理无缝映射到成熟的持续集成(CI)工作流。这些现代化的数据库更改工作流与在高度自动化的环境中管理应用程序代码的方式非常相似。

Datical DB 4.0 包括一个新的自动化部署打包程序，它允许通过更普遍的、无需干预的自动化来不断验证、打包和集成数据库更改。Datical DB 的自动化功能大大减少了数据库管理员(DBA)、开发人员和 QA 团队管理这些从构思到生产部署的变化所花费的时间。

Automated Deployment Packager 通过在开发人员每次请求更改数据库时卸载大部分手动审查、批准和部署活动，使数据库管理员能够跟上应用程序发布的步伐。通过使用 Datical 的自动部署打包程序，数据库开发人员现在能够将 Oracle、Microsoft SQL Server 或 IBM DB2 等流行数据库平台的更改脚本检查到 Git、Apache Subversion 或 Team Foundation Server 等源代码控制解决方案中。这触发了一个过程，该过程根据经验丰富的 DBA 定义和定制的验证例程，自动接受或拒绝提议的更改。

验证例程确保变更是精心设计的，并且符合组织标准和最佳实践。除非有特别重要的原因，否则这些例程会阻止 DBA 参与进来。

“90%以上的开发人员脚本和数据库更改不需要数据库管理员参与，而有了 Datical 4.0，数据库管理员只需参与需要他们关注和专业知识的一小部分更改，”Datical 的联合创始人兼产品副总裁 Pete Pickerill 说。“这一版本让数据库管理员重新获得了时间、精力和理智，帮助他们回到了他们应该扮演的角色。”

随着时间的推移，自动化验证和详细的报告帮助开发人员理解可接受的变更和会破坏构建的变更之间的区别。

一旦一组更改获得批准，它将被打包和标记，以便通过与 Jenkins、IBM UrbanCode、XebiaLabs XL Deploy 或 Electric Cloud ElectricFlow 等流行的构建和发布自动化工具紧密集成，或者通过使用 Datical 的图形或命令行界面，轻松部署到其他环境中。数据库变更管理成为开发、测试和运营团队的自助服务体验。因此，数据库管理员可以腾出时间从事更具战略性的工作，如性能调整、容量规划以及新技术和数据平台的评估。

最后，Datical 的 Automated Deployment Packager 还消除了协调部分更新的数据库环境的痛苦任务，确保环境始终反映最新批准的版本，或者恢复到最近完全批准、备份和存储的版本，所有这一切都只需最少的停机时间。

“当我们寻求将数据库纳入持续集成时，我们发现 Datical 代表了理想的途径，”Radial(前易贝企业)的数据库架构师 Devon Siegfried 说。“老实说，Datical 允许我们以如此有效、密切跟踪和受控的方式部署数据库更改，这是无可替代的。最重要的是，Datical 的实施和产品团队无与伦比。”
