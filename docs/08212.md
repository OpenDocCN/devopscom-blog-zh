# 现代应用的新标准:安全可观察性

> 原文:[https://devo PS . com/the-new-norm-for-modern-apps-security-observability/](https://devops.com/the-new-norm-for-modern-apps-security-observability/)

可观察性已经出现在所有类型的运营和安全活动中。对企业的需求不断增加，要求企业在处理潜在问题时更加积极主动地应对变化。特别是，安全可观察性有望帮助减少检测网络攻击的时间。更进一步，它可以帮助在攻击发生之前检测安全漏洞。

**什么是安全可观察性？**

过去，大多数运营依赖于对其系统和应用程序的可见性来管理安全性。可见性将通过监控应用程序、系统及其日志来实现。监控解决方案将收集和分析信息，以检测可疑行为或未经授权的系统更改。定义了哪些类型的行为应该触发警报后，解决方案将根据需要采取措施。

也许更确切地说，监视让您可以检测一组已知的问题或情况。因此，它对于预警和分析长期趋势至关重要。简而言之，监控让用户知道他们的系统和应用程序是如何运行和使用的。

然而，对复杂的分布式应用程序进行安全监控的问题在于，这是一种针对高度动态操作的被动方法。并且它通常不提供任何对根本原因问题或异常的洞察，而这些问题或异常可能是正在形成的问题的前兆。

安全可观察性通过支持数据的关联和检查来提供更深入的见解，从而扩展了监控。可观察性通常需要日志、指标和深度跟踪。所有数据都用于建模和分析。公司可以挖掘数据，寻找模式，使用人工智能和[机器学习](https://devops.com/traceai-machine-learning-driven-app-and-api-security/)来修复问题，并主动防御问题。

**为什么现在需要安全可观察性**

由于应用程序的开发方式，对安全性可观察性的需求正在增长。

总体行业趋势是转向基于微服务的云原生架构。这种方法有很多好处。应用程序可以快速扩展。公司可以轻松地更改或更新大型应用程序的一个小方面，而不会影响应用程序的其余部分。企业可以利用新技术。例如，企业可以添加一个新的前端，或者实施更复杂的人工智能或机器学习建模。

这种开发方法可能会引入潜在的盲点和复杂性。特别是，使用云原生架构和微服务构建的现代应用带来了新的挑战。这种应用程序的松散耦合、它们的分布式本质以及增加的复杂性使得理解漏洞变得更加困难。结果，传统的网络安全方法崩溃了。他们忽略了构成现代应用程序的许多元素之间的相互作用和相互依赖。

此外，传统的安全监控解决方案可能会受到限制。例如，可见性通常必须跨工具进行划分，导致不安全的平台或者相关数据甚至不用于监控的情况。

除了[安全挑战](https://devops.com/8-security-considerations-for-ci-cd/)之外，还有低代码/无代码开发方法的广泛采用。这种技术向业务单位和公民开发者开放了开发。这些团队可以构建复杂的应用程序，通常很少受到监督或指导。因此，应用程序可能是通过组装过时的、未打补丁的或只是有漏洞的构建块来创建的。

**DevSecOps 需要安全可观察性**

鉴于所有这些趋势，企业探索新的安全方法也就不足为奇了。正如使用自动化测试将质量保证融合到开发过程中一样，安全操作正在与开发融合在一起。因此，DevSecOps 方法在当今的组织中得以快速实现。

虽然在使用云原生开发方法时，安全问题可能会增加，但还有一个鲜为人知的安全优势。使用 API 和微服务构建的应用和服务比其他系统更有优势，因为开发人员和安全人员可以更容易地观察它们内部发生的事情。此外，企业可以添加 API 和微服务的链接，并使用工具来收集此类操作的详细信息。

考虑到如此丰富的数据，安全可观察性扩展到了监控方面。DevSecOps 不仅仅是检测在开发过程中引起法规遵从性问题的漏洞和事件，而是使用安全可观察性来寻求识别问题并自动采取纠正措施。

展望未来，企业应该会更多地采用 DevSecOps 和基于安全可观察性的更多自动化。这只会有助于加强跨系统和应用程序的安全性和合规性。
