# DevOps 是竞争优势吗？一场辩论

> 原文：<https://devops.com/devops-competitive-advantage-debate/>

DevOps 成为竞争优势的想法并不新鲜。服务提供商、软件销售商和内部 IT 经常使用这一论点来证明各种投资的合理性。但是这个论点有什么真正的价值吗？DevOps 真的是可持续竞争优势的来源吗？或者仅仅是炒作和销售产品/服务的新方式？

以下是来自两党的一些更受欢迎的观点。

| **“绝对是”** | **“不，去远足吧”** |
| **速度是性感的**在移动、社交和网络世界中，客户期望速度和响应能力。比竞争对手更快地将想法推向市场绝对是一种优势，DevOps 让我们能够做到这一点。 | **DevOps 不是海里唯一的鱼**同意速度是性感的。但是在这个特定的时刻，有多种技术/理念声称做同样的事情，即云、大数据、分析等。是什么让 DevOps 如此与众不同？ |
| **DevOps 很难复制**虽然购买工具很容易，但真正的好处在于获得正确的文化和流程。任何能够获得这种权利的组织都将能够形成可持续的竞争优势。 | 其他战略也是如此还记得外包开始的时候吗？这也被吹捧为一种可持续的竞争优势。现在，它实际上是一种消除早期采用者所拥有的任何优势的策略。DevOps，如果它真的起飞了，也会沿着同样的路径。 |
| **新的 IT 运营模式** DevOps 代表了企业 IT 运营的新方式。它打破了竖井结构，释放了现有组织中的价值。 | **企业不是初创企业**它在初创企业环境中工作过，并不代表它能在大企业中工作。 |
| **倾听客户的意见**速度只是 DevOps 优势的一个方面，另一个方面是更快的反馈循环，以便您可以确定自己是否走在正确的方向上。将两者结合起来，你就拥有了强大的竞争优势。 | **不是 IT 职责/能力**这不正是大多数组织已经在做的吗？我有完整的业务/产品/营销组织。专注于此。我现在最不需要的就是它跳进来把一切都搞砸。 |
| **新的软件交付生命周期(SDLC)** DevOps 代表了企业 IT 运营的新方式。它打破了竖井结构，释放了现有组织中的价值。 | **良好的营销**这只是把一瓶旧酒重新包装成一瓶新酒……devo PS 只是一个新事物。像所有其他时尚一样，这也将会过去。 |

**吾取**

请允许我作为一个典型的顾问，从我的两个方面发言。DevOps 是也不是竞争优势。现在，在你关闭浏览器继续前进之前，让我解释一下。

DevOps 在短期内绝对是一个优势。将组织从传统方法转移到开发运维方法并不是一件容易的事情。任何能够成功实现这一转变的公司都将最先受益——从更快的上市速度到更快地响应客户需求。而且好处是实实在在的！问问网飞和 Etsy，他们已经能够在创新上超越竞争对手，并在市场上获得相当大的领先优势。

然而，从长远来看，随着越来越多的公司投资 DevOps，优势将继续缩小。DevOps，如果你剥去所有的市场行话，回到最基本的定义，是关于重组 IT 的内部管道(例如，它如何执行软件交付生命周期内的各种任务)。虽然在短期内管道可以让你做一些整洁的东西，但最终它只是内部管道，很少是持久竞争优势的来源。

要看到这是行动，看看汽车，金融，然后我们自己的信息技术产业。在所有这三个例子中，能够转型的公司能够在早期确立相当大的领先地位，获得回报，行业的其他公司最终被迫迎头赶上。

在汽车行业，对精益和 TQM 的早期投资为丰田赢得了客户忠诚度，并显著领先于其传统竞争对手。虽然领先优势持续了十年(在产品生命周期为 4-5 年的行业中，这是很短的时间)，但竞争对手很快开始投资并迎头赶上。你现在可以说，美国制造商在质量和速度上都与丰田不相上下。竞争对手还没有赶上的地方是感知。

在 20 世纪 70 年代，根据当时的“行业标准”，购房者等待 90 天才能获得信贷决定和贷款并不罕见。花旗银行重新安排了内部流程，让买家在 30 天内做出决定。它席卷了整个行业。竞争对手最终赶上来了，现在 30 天是常态。

至于我们自己的信息技术产业，外包被认为是许多先行者的竞争优势。像其他战略一样，它也需要结构性的改变才能实现。许多最初的先行者能够削减超过 30%的 IT 成本，并确立了相对于竞争对手的成本优势。但在十年内，竞争对手复制、储存了储蓄，并侵蚀了成本优势。

在我看来，DevOps 也将遵循类似的道路。一些领导者将会出现(已经发生了)，他们将会为这个行业设定变革的步伐，其余的将会紧随其后。不管他们喜不喜欢。
