# GitHub 的 2FA 指令对世界各地的开发者意味着什么

> 原文：<https://devops.com/what-githubs-2fa-mandate-means-for-devs-everywhere/>

多因素认证(MFA)在软件开发组织中变得越来越标准，GitHub 最近宣布 到 2023 年底，双因素认证(2FA)将对所有代码贡献者强制执行。

这是明智之举。近年来，不良行为者频繁发起攻击，通过使用窃取的开发人员凭据， [访问源代码](https://www.helpnetsecurity.com/2021/09/17/open-source-cyberattacks/) 。一旦进入，它们会迅速渗透到组织的整个代码库，并提升对公司范围系统的访问。2021 年 [网络安全管理软件产品](https://www.cnbc.com/2021/01/01/solarwinds-hackers-accessed-microsoft-source-code-the-company-says.html) 妥协案就是如此，最终危及联邦机构、企业和政府机构。

但是对于一些有崇高目标和集中时间表的开发人员来说，MFA 是一个额外的步骤，有可能会减慢他们的速度。随着组织继续实施 MFA 和其他基础安全控制，他们可能会面临阻力，但他们不应该被吓倒。MFA 是一个关键的攻击屏障，可以提高个人保护，并有助于加强源代码的安全性。

如果组织深思熟虑地处理实现，他们可以为后续的安全控制奠定基础——并在他们的用户和更大的开发社区中建立信誉。

### 个人开发者安全松懈的风险

组织的源代码通常支持大量的上游程序和产品，因此对于不良行为者来说，通过渗透他们的库来发起攻击是很有价值的。攻击者还可以在源代码中隐藏恶意软件，这些恶意软件可以升级到破坏整个系统和第三方组织。许多公司仅仅依靠反病毒扫描和其他表面级监控工具来标记源代码漏洞，而不是将它们与[动态测试](https://devops.com/?s=DAST)和渗透测试结合起来，这一事实加剧了这种风险。

攻击者通常通过窃取的开发人员凭据来获得对源代码的访问权限。虽然单一身份认证系统(即用户名和密码)是最容易被攻破的，但 MFA 为攻击者制造了更难逾越的额外障碍。虽然这个工具确实在开发过程中制造了额外的摩擦，但是这一额外的步骤是值得的。即使对于业余爱好者或开源开发人员来说，受损的源代码也会引起更大的反响，使其他开发人员处于危险之中。说到网络攻击，最好假设任何实体都是潜在的目标。

除了个人安全之外，MFA 和其他基础控制对于维护组织声誉和遵守法律隐私标准也至关重要。到 2023 年， [全球 65%的人口](https://www.gartner.com/en/newsroom/press-releases/2020-09-14-gartner-says-by-2023--65--of-the-world-s-population-w) 将拥有受隐私法规保护的个人数据。在未来几年中，保护身份验证凭证的安全措施将变得更加重要，尤其是对于生产支持面向消费者的产品和程序的源代码的组织而言。通过有意的实施方法，MFA 可以帮助减轻潜在的源代码攻击，并形成全面安全计划的基础。

### 成功实施 MFA 的四个步骤

MFA 的好处远远超过用户摩擦和开发者抵制的成本。遵循以下四个实施步骤，充分利用这种身份认证技术。

1.  **从小处着手，建立信誉。**逐步实现，帮助开发者理解 MFA 的价值。成功的实现将为您的领导和安全团队提供未来安全计划的可信度，并且它将改进将安全集成到您更大的软件开发生命周期中的过程。
2.  **检查你的安全程序的价值链。**实施基础安全控制(如 MFA)提供了一个检查安全计划整个价值链的机会。花点时间寻找源代码库之外的潜在漏洞，并标记出开发生命周期中可能产生风险的区域。
3.  **层后续安全控制。**借助 MFA，使用其他工具和控制来构建您的安全计划，如动态和渗透测试、源代码扫描、软件成分分析(SCA)和漏洞管理。你使用的安全工具越多，攻击者就越难得逞。
4.  **计划持续优化。**随着越来越多的组织要求 MFA，攻击者将会想出更多的方法来阻挠它。例如，当攻击者开发出控制手机的方法时，使用短信作为第二种身份验证形式的流行程度就下降了。为了跟上不断发展的攻击者行为，不断优化您的 MFA 技术和后续安全控制。

### **MFA 将成为行业标准**

随着不良行为者发展攻击方法和隐私法规的加强，MFA 无疑将成为保护源代码库的行业标准。

为了保持领先，通过确保 MFA 保护每个系统入口点，在整个网络中关注 MFA 的概念。这一额外的步骤是改进源代码保护的一个简单代价——并最终保护您的代码所能保护的每一项资产、产品和程序。精心实施 MFA 可以为全面的源代码保护奠定基础。
