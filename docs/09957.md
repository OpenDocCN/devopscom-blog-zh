# SAP 添加低代码平台以更快构建定制应用

> 原文:[https://devo PS . com/sap-adds-low-code-platform-to-build-custom-apps-faster/](https://devops.com/sap-adds-low-code-platform-to-build-custom-apps-faster/)

在 [SAP TechEd 2022](https://www.sap.com/about/events/teched.html) 大会上，SAP SE 今天宣布在其产品组合中添加了一个名为 [SAP Build](https://news.sap.com/2022/11/sap-teched-sap-build-business-expertise-coursera-partnership/) 的低代码平台，以简化组织构建定制应用的过程。

SAP 副总裁兼低代码/无代码解决方案负责人 Sebastian Schrö tel 表示，SAP Build 旨在使公司的低代码工具更容易被更广泛的开发人员使用，包括通常没有太多编程专业知识的所谓的[公民开发人员](https://devops.com/?s=citizen+developers)。

目标是通过利用 SAP Build 中 500 多个可用模板来监控、分析和自动化流程。Schrö tel 说，我们的目标是不仅可以更快地构建应用程序，还可以通过一次单击来部署它们。

![](../Images/cd33d65c15422ade1951814737f182f0.png)

该公司构建定制应用程序的方法是通过在一个独立的云中运行 SAP 企业资源规划(ERP)应用程序的原始实例，将其核心应用程序与他人开发的代码分开，该云称为 SAP digital core。然后，在单独的云中部署一个自定义应用程序，并通过应用程序编程接口(API)访问 SAP 数据。这种方法使 SAP 能够更新其 ERP 环境，而不必担心破坏定制扩展。

当然，并不是每个组织都已经过渡到这种模式，因此仍有大量 SAP 软件运行在使用 Java 或专有 ABAP 编程语言编写的代码扩展的内部 IT 环境中。随着时间的推移，SAP 希望定制代码将被淘汰，以支持其已添加到 ERP 平台的流程，或者由该流程的更现代的实例与部署在单独的 SAP 云平台上的定制应用程序一起运行。

SAP 还与 Mendix 和 OutSystems 等低代码平台提供商建立了合作伙伴关系，组织可以利用这些平台来构建应用程序，但随着 SAP Build 的推出，该公司正在做出更协调一致的努力，以鼓励组织采用它开发的低代码平台。

虽然还不清楚普通开发人员和专业开发人员对低代码工具的热情有多高，但可以肯定的是应用程序开发的步伐正在加快。在许多情况下，专业开发人员正在采用低代码平台作为过程代码的替代方案，以减少由[数字业务转型](https://digitalcxo.com)计划带来的不断扩大的应用开发积压。在其他情况下，专业开发人员在构建和部署驱动这些过程的应用程序时，使用低代码工具来更轻松地与最终用户协作。

不管动机如何，应用程序开发的加速将对 DevOps 团队产生深远的影响。这些应用程序不仅必须部署在生产环境中，而且开发运维团队还必须保护和维护更大的应用程序组合。因此，自动化开发运维工作流的需求将变得更加突出。
