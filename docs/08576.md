# BMC 收购 StreamWeaver 推进 AIOps 战略

> 原文:[https://devo PS . com/BMC-acquisites-streamweaver-to-advance-ai ops-strategy/](https://devops.com/bmc-acquires-streamweaver-to-advance-aiops-strategy/)

BMC 已经[收购了 StreamWeaver](https://www.prnewswire.com/news-releases/bmc-acquires-streamweaver-301389603.html) ，这是一个平台提供商，可以更简单地实时捕获流数据，作为扩展 BMC Helix 人工智能 it 运营平台(AIOps)的努力的一部分。交易的财务条款没有披露。

BMC 数字服务和运营管理总经理兼高级副总裁玛格丽特·李表示，StreamWeaver 与多个数据源进行了广泛的集成，BMC 将使用这些数据源来训练人工智能模型并提高整体可观测性。Lee 指出，虽然连接数据源相对简单，但现在的挑战是以一种 IT 运营团队可以轻松重复的方式实时收集数据。

总的来说，随着 IT 环境变得越来越复杂，AIOps 将继续受到关注。Lee 说，如果不给 IT 人员配备一些人工智能，大多数 IT 运营团队将无法管理高度分布式的计算环境。

自然，许多曾经由 IT 运营专业人员手动执行的机械任务将变得越来越自动化。然而，对 IT 人员的需求总是存在的，Lee 指出。

可以说，有一天，IT 专业人员可能不愿意为没有 AIOps 平台的组织工作，因为分配给他们的任务会被认为是过时的，特别是如果这些任务涉及许多 IT 专业人员不愿意执行的一系列乏味的任务。

与此同时，AIOps 应降低整体风险，让 it 运营团队在中断应用服务之前更容易主动发现问题。这一点尤其重要，因为 IT 环境(越来越多地使用容器构建基于微服务的应用程序)和无服务器计算框架正越来越频繁地与传统整体应用程序一起部署。

早在 2017 年，BMC 就一直在为 AIOps 做准备。然而，从那以后，一波创业公司和 IT 运营现任者要么从头开始建立 AIOps 平台，要么用 AI 功能更新现有平台。很难评估哪些厂商在 AIOps 竞赛中领先，但有一点是肯定的，It 组织现在有很多选择。

然而，从文化的角度来看，并不是每个组织都完全适应 AIOps。尽管一些 IT 专业人士仍持怀疑态度，但其他人将人工智能视为对他们生存的威胁。大多数 IT 团队也担心任何有缺陷的人工智能模型可能对 IT 运营产生的潜在影响——人类犯错是一回事，但人工智能模型犯大规模错误可能是灾难性的。

很少有 IT 组织全面采用 AIOps，但是大多数 IT 组织都认识到 AIOps 平台以某种形式存在是不可避免的。毕竟，机器学习算法永远不会忘记它们学习的任何东西。当得到更高的薪水时，他们也不会休息一天或决定离开。挑战在于弄清楚 IT 人员和算法之间的接口将会在哪里——不仅是今天，而且是在不久的将来。
