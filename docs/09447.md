# Eclipse Foundation 为 OpenJDK 二进制文件打开了市场

> 原文：<https://devops.com/eclipse-foundation-opens-marketplace-for-openjdk-binaries/>

Eclipse Foundation 本周开放了一个 [Adoptium Marketplace](https://www.globenewswire.com/news-release/2022/05/26/2451097/0/en/OSS-Leader-the-Eclipse-Foundation-and-the-Adoptium-Working-Group-Launch-the-Adoptium-Marketplace.html) ，DevOps 团队可以通过它访问基于 OpenJDK 规范的 Java 二进制文件。OpenJDK 是基于编程语言和虚拟机的 Java 标准版(SE)创建的。

Eclipse 基金会的执行董事 Mike Milinkovich 表示，目标是让 it 团队更容易通过供应商中立的市场访问开源二进制文件，该市场符合 Java SE 技术兼容性工具包(TCK)，创建该工具包是为了确保基于 OpenJDK 的运行时符合原始 Java 规范。

这个市场是与 Adoptium 工作组合作建立的，adopt ium 工作组是由 Eclipse Foundation 建立的，用于管理 OpenJDK，open JDK 是 Java SE 的一个开源实现，最初由 Sun Microsystems 创建，后来被 Oracle 收购。

Adoptium 工作组成员包括阿里云、Azul、华为、IBM、iJUG、Karakun AG、微软、New Relic 和 Red Hat。该小组创建了一个 Eclipse AQAvit 项目来测试二进制文件是否符合 Java SE TCK 规范。

OpenJDK 是多个厂商用来创建自己的二进制文件的参考实现，只保证在下一个更新到来之前与标准兼容；大约每六个月发布一次更新。Eclipse Adoptium 项目将使开发团队更容易识别哪些二进制文件与 OpenJDK 的特定实现兼容。最终，我们的目标是鼓励开发人员使用框架来构建应用程序，当新的 OpenJDK 特性被添加到平台中时，这些框架会利用这些新特性，而不是继续依赖旧的二进制文件。

对于在企业 IT 组织中工作的开发人员来说，Java 仍然是主导语言。然而，在过去的几年中，OpenJDK 获得了发展，因为组织采用了该规范的开源实现，这使得应用程序开发框架的提供商能够降低许可成本。

尚不清楚企业 It 组织用 OpenJDK 替换 Java 专有版本的速度。然而，在经济低迷时期，更多的组织倾向于依赖开源软件。许多组织现在也希望使用基于 OpenJDK 运行时的 Java 框架来构建基于微服务的应用程序。

OpenJDK 社区显然花了相当多的时间来构建一个厂商中立的市场。与此同时，各种各样的其他编程语言在企业中获得了发展。在构建后端应用程序和服务时，仍然有很多对 Java 的依赖，但现在比以往任何时候都更常见的是，使用基于 JavaScript 的框架来访问这些后端应用程序和服务的前端。

当然，Java 不会很快消失。依赖它来构建应用程序的企业开发人员的绝对数量将确保它的持续寿命。然而，需要注意的重要一点是，随着越来越多的开源平台被采用，整个社区的创新步伐应该会继续加快。
