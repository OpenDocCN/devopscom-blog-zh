# SREs 可观测性的主要优点

> 原文:[https://devo PS . com/the-key-benefits-of-observability-for-sres/](https://devops.com/the-key-benefits-of-observability-for-sres/)

在当今的技术环境中，组织努力倡导创新理念、技巧和技术，以取得成功并超越竞争对手。由于这个原因，站点可靠性工程(SRE)已经成为发展最快的企业角色之一，以及一套快速可靠的软件交付的组织实践。

SREs 使用各种工具和实践来大规模管理服务，例如可观察性。[可观察性](https://devops.com/?s=observability)是推断系统内部状态的能力。它提供了关于系统中何时出现错误以及更重要的是为什么会出现错误的可操作的见解。对于 SREs 来说，这些可操作的数据对于提供安全可靠的应用非常重要。

为了更深入地了解可观测性对 SREs 的好处，我邀请了[](https://www.devopsinstitute.com/all-events/?utm_campaign=SKILup%2520Days%25202021%2520%255BGeneral%255D&utm_source=devopsdotcom-Observability-SKILup%2520Day%2520Article)SKILup Day 的参与者和 [DevOps Institute 的大使](https://www.devopsinstitute.com/about-us/ambassadors/?utm_campaign=Brand%2520Awareness%25202021&utm_source=devopsdotcome-Observability-SKILup%2520Day%2520Article) 来参与讨论。以下是他们分享的:

**赞助商，** [**普瑞亚·萨蒂什**](https://www.linkedin.com/in/priyacsatheesh/) **，CEO，Instana**

“顾名思义，站点可靠性工程师的任务是保持应用程序、架构和网站的正常运行和可用。可观察性解决方案在问题出现的第一时间向他们发出警报，并提供适当的背景和信息，以便他们能够在问题变成事故并开始影响其客户之前采取行动。此外，AIOps 和机器学习可以帮助他们预测事件，并对常见事件的响应进行标准化，以减少响应和修复时间。”

[**海伦·比尔**](https://www.linkedin.com/in/helenjbeal/?originalSubdomain=uk) **，DevOps Institute 首席大使**

“接受可观察性通过以下方式支持 sre 的目标:

*   减少与事件管理相关的工作——尤其是与原因分析相关的工作——提高正常运行时间和 MTTR。
*   提供一个根据 SLO 进行检查和调整的平台，并最终提高团队满足 SLO 的能力。
*   当 SLO 未得到满足且错误预算超支时，提供潜在的改进解决方案。
*   减轻团队在处理大量数据时的认知负荷——减少疲劳。
*   将人类和团队从繁重的工作中解放出来，同时提高生产力、创新和价值的流动和传递。
*   支持多功能、自主团队和“我们建造它，我们拥有它”的 DevOps 口号。
*   通过提供可反馈到创新阶段的价值结果洞察，完成价值流循环。"

[**马克·彼得斯**](https://www.linkedin.com/in/tinycyber/) **，技术负责人，诺维塔**

“真正的系统可观察性意味着任何操作或开发阶段的任何过程或事件都可以通过 SREs 进行监控。具有内置可观察性的系统为作为主题专家的 sre 提供了从不同角度检查事件的机会。系统的可观察性越强，可以检查的不同地方就越多，从而有助于改善整体流程。SREs 然后可以创建反馈并设计实验，从不同的角度进行观察。如果事件表现出功能上的困难，并且唯一的监控点是功能的输入/输出，并且没有过程可观察性，则 sre 必须回到开始来寻找事件。如果他们可以观察从提交到部署的整个流程链，就可以发现不同的区域。想想 A/B 周期:计划有限的部署，并逐渐转移负载，以确保新系统能够处理增加的功能。如果 SRE 不能观察功能，那么他们不能帮助提高可靠性。”

[**瑞安**](https://www.linkedin.com/in/ryansheldrake/) **，现场 CTO，花边**

“如果您能够‘看到’或观察到 sre 负责管理、创建甚至破坏的复杂且不断变化的环境，您就可以管理与这些环境和相应工作负载相关的风险状况。

[**Vishnu Vasudevan**](https://www.linkedin.com/in/vishnuvasudevan811/)**产品工程与开发负责人 Opsera**

“现场可靠性工程(SRE)团队的最大好处是有机会与业务利益相关方和技术利益相关方合作，以设定目标并持续改进。基于这些目标，SRE 可以创建一个审批流程，这将有助于他们了解自己的规划、安全策略、质量标准和运营实践。一旦实现了健壮的可观察性实践，度量每个发布和验证项目的能力有助于提高上市速度和其他因素。借助跨越所有 DevOps 工具、发布的应用和相关不同团队的 DevOps 流程编排，SRE 团队可以将可观察性实践转化为竞争优势。”

*   获得有关服务状态的实时健康更新和透明信息。
