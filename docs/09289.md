# Checkmarx 发现恶意开源 PyPi 库

> 原文：<https://devops.com/checkmarx-finds-malicious-open-source-pypi-repository/>

Checkmarx 是一家测试应用程序安全性的平台提供商，该公司本周披露，它发现了一个恶意的 Python 代码 PyPi 库实例，该代码已经被下载了 70，000 多次。

供应链安全负责人 Tzachi Zorenshtain 表示，这一发现代表着网络犯罪分子提供了一个流行的开源软件包的恶意副本，该软件包包含注定会进入下游应用程序的恶意软件。他指出，网络犯罪分子通过一种称为“劫星”的策略，创建一个包含虚假统计数据的网页，如 GitHub stars，使其看起来像是开发人员可能下载的来自合法开源项目的软件包。

Zorenshtain 指出，网络罪犯将劫持和域名抢注结合起来，建立网页来模仿合法的开源项目。

“劫星”是又一次破坏软件供应链完整性的努力，这些供应链依赖于大量开源软件组件来构建应用程序。Zorenshtain 指出，阻止劫持的唯一方法是开发人员或 DevOps 团队在将组件集成到应用程序之前对其进行验证。否则，可能只有在恶意软件已经进入生产环境中部署的下游应用程序之后，才会被发现。

当然，挑战是记住花时间验证任何软件组件的来源。通常试图尽快构建应用程序的开发人员不会总是停下来确保他们使用的所有组件都来自合法来源。

尚不清楚鼓励开发者下载恶意软件的虚假网站有多普遍。然而，随着网络犯罪分子扩大其危害软件供应链的努力，很明显，他们的策略正在演变为利用开发者对开源软件的信任。这些攻击不太可能会破坏开源生态系统，但它们确实增加了使开源软件更加安全的紧迫性。

开源安全基金会(OpenSSF)是 Linux 基金会的一个分支，已经筹集了超过 1000 万美元来开发工具和定义保护开源软件项目的最佳实践。谷歌承诺最终将花费 100 亿美元来提高开源软件的安全性。拜登政府还通过扩大合规性要求，将改善政府机构内外广泛使用的开源软件的安全性作为优先事项。白宫显然也在试图向 IT 厂商和大型企业施压，要求他们为保护开源软件做出更多贡献。

问题是许多开源项目是由少数程序员维护的，他们贡献时间和精力来构建其他人可以免费使用的组件。他们中的许多人认为，确保软件安全的责任在于决定部署该软件的组织。他们也没有责任追踪那些利用域名仿冒技术传播恶意软件版本的网络罪犯。

不幸的是，许多依赖该代码的 IT 供应商和大型企业 IT 组织并没有为项目做出任何有意义的贡献，无论是在融资方面还是仅仅帮助开源维护者发现和修复漏洞方面。然而，许多同样的组织现在也在评估他们所使用的开源软件，从安全的角度来看，在没有这些贡献的情况下是否是可持续的。因此，一个酝酿已久的开源软件安全问题爆发成一场重大的信任危机可能只是时间问题。
