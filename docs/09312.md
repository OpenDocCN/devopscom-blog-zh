# CloudBolt 添加模块化自动化框架来驯服混合云

> 原文:[https://devo PS . com/cloud bolt-adds-modular-automation-framework-to-tame-hybrid-clouds/](https://devops.com/cloudbolt-adds-modular-automation-framework-to-tame-hybrid-clouds/)

CloudBolt Software 本周[增加了一个由八个模块](https://www.globenewswire.com/news-release/2022/04/28/2431525/0/en/CloudBolt-Brings-Order-to-Multi-Cloud-Multi-Tool-Chaos.html)组成的框架，这些模块共同简化了混合云计算环境的自动化、优化和管理。

CloudBolt 首席执行官 Jeff Kukowski 表示，目标是为 IT 团队提供一种更模块化的方法来管理多种云，他们可以在获得更多专业知识时使用这种方法。

CloudBolt 提供的模块解决了从提供称为自动化交换的存储库(通过它可以共享可重用的自动化内容)到实施治理策略和跟踪云支出的所有问题。

其他模块处理测试和保证、集成、自动发现和可用于自动化流程的机器学习算法。

虽然大多数组织在内部 IT 环境中采用了多种云，但整个扩展企业的 IT 管理远未实现统一。CloudBolt 最近发布的一项调查发现，整整 88%的受访者需要一个“经理中的经理”,使他们能够统一在其 IT 环境中成倍增长的各种孤岛。CloudBolt 软件正在为一个 IT 框架提供一个抽象层，通过这个抽象层可以统一多个云和内部 IT 环境的管理。

尚不清楚组织是否有意采用多种云，或者是否由于简单的惯性，他们发现自己由于各种应用程序开发团队做出的决策而管理多个云平台。一些 IT 组织采用多种云，因为他们担心被某个特定的云计算服务所束缚；其他人担心一个平台的中断可能会导致他们的所有应用程序离线。其他人只是利用一个云服务提供商提供的服务比另一个更好。一个组织收购另一个依赖完全不同的平台来部署应用程序的组织也并不罕见。

无论 IT 组织如何发现自己管理着多个云，随着每个新平台添加到 IT 环境中，IT 的总成本都会上升。同样具有挑战性的是，每个新平台都公开了某种类型的自动化框架。用不了多久，IT 团队就会发现他们不得不在高度分布式计算环境中实现的多个自动化孤岛中导航。在许多情况下，这些自动化实例正在使用完全不同的工具集重新发明已经在另一个平台上自动化的相同过程。

这更多的是一个时间和程度的问题，It 组织最终将在某种类型的统一控制平面下整合不同 IT 环境的管理，从而实现真正的混合云计算。一旦实现了这一功能，IT 团队就会发现将工作负载从一个平台迁移到另一个平台变得更加简单。

与此同时，在可预见的未来，大多数组织都将管理多个 IT 环境。唯一不清楚的是，在任何给定时间，这些平台上可能运行的工作负载量。挑战和机遇是重新获得对日益分散的计算环境的控制。
