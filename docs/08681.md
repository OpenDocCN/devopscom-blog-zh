# Instana 发布首个 EUM 解决方案，提供与微服务性能自动关联的浏览器响应时间无界分析

> 原文:[https://devo PS . com/instana-releases-first-eum-solution-to-provide-unbounded-analytics-of-browser-response-time-automatically-correlated-to-microservice-performance-2/](https://devops.com/instana-releases-first-eum-solution-to-provide-unbounded-analytics-of-browser-response-time-automatically-correlated-to-microservice-performance-2/)

#### 首个包含免费 EUM 的 APM 解决方案使开发人员能够精确确定响应时间，并立即优化最终用户体验

**德国索林根和加利福尼亚州圣马特奥**–2019 年 1 月 16 日–APM 提供商 Instana 消除了性能分析的界限，提供了一种无限灵活的方法来分析和了解终端用户体验。Instana 的无界分析现在可用于该公司最新发布的最终用户监控(Insana EUM)，包括在 Instana APM 服务中。最新的 Instana EUM 版本可以捕获 100%的用户请求，提供简单的过滤功能来查找特定用户 id 的所有页面负载，并通过服务器端后端对浏览器前端的每个请求进行详细的性能分析。

Instana 联合创始人兼首席执行官 Mirko Novakovic 表示:“随着当今应用的复杂性不断增加，从浏览器到所有微服务，分析每个请求的能力(对如何分割数据集没有限制)对于 DevOps 和 CI/CD 至关重要。“事实上，由于最终用户监控是整体性能管理的关键部分，Instana 在我们的 APM 解决方案中免费提供了这一功能！新的 EUM 实例包含在我们的 APM 产品中。”

Instana 的无界分析允许用户以有效的无限组合对跟踪数据集应用过滤器，使开发人员能够找到确切的请求或需要关注的跟踪。由于现在包括 web 性能数据在内的所有跟踪都是自动可用的，所以对于整个 DevOps 组织来说，发现具有挑战性的性能问题更加容易。

Instana EUM 包括大量浏览器端数据，如页面负载、资源、HTTP 请求和 JavaScript 错误，允许网站可靠性工程师和开发人员识别和分析各种情况，包括:

*   确定特定类型和版本的 web 浏览器的浏览器端性能退化
*   分析来自特定用户会话的每个请求，以了解导致错误或性能问题的确切情况
*   立即了解问题是出在浏览器还是服务器端，并找出用户请求执行不佳的根本原因

Edmunds.com 云服务技术负责人 Stephanie Byrd 表示:“借助新的 EUM 分析功能，结合 Instana 捕获每一个最终用户请求的惊人事实，我们的 devops 团队能够准确了解我们的客户如何体验 Edmunds.com 服务。“例如，我们可以分析西雅图在过去两个小时内的性能，并将其与我们的微服务性能相关联，从而帮助我们隔离问题并优化我们的客户体验和保留。让 Instana 数据集中的每个请求都在我们的指尖，这是非常强大的。”

对于前端开发人员，Instana EUM 收集浏览器端开发人员工具的网络选项卡上可用的相同数据，但记录每个用户请求的历史信息。有了 Instana，开发人员再也不需要通过猜测用户如何发出 web 请求来解决重现问题。相反，他们将在 Instana UI 中随时获得实际的诊断数据，以快速了解任何缓慢或错误的用户请求的根本原因。

Instana 的自动应用程序监控解决方案可发现应用程序基础架构和服务组件，为应用程序的技术堆栈部署监控传感器，跟踪所有应用程序请求，无需任何人工配置，甚至无需重启应用程序。该解决方案实时检测应用环境中的变化，调整自己的模型，并在几秒钟内将变化和对用户的影响可视化。

该公司的 APM 解决方案免费包含了新的 Instana EUM 功能。详情请访问 [https:instana.com/eum](https://www.instana.com/press-releases/instana-releases-first-eum-solution-to-provide-unbounded-analytics-of-browser-response-time-automatically-correlated-to-microservice-performance/instana.com/eum) 。
