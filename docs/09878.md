 *## 1.哎哟:停电愤怒

本周的第一件事是:Kakao 使用的一个数据中心发生火灾，导致 32，000 台服务器损失，停电持续数天，甚至首席执行官辞职。事实证明，把所有鸡蛋放在一个篮子里不是一个好主意。谁知道呢？

### 分析:*你的* DR 计划如何？

2022 年，停机是*畏缩*。你的应用程序对国家的重要性可能不如 Kakao 对南韩的重要，但是如果你要满足用户的**正常运行时间** **期望**的话，区域故障转移和灾难恢复规划的其他元素同样重要。

[常澈和于今杨](https://www.nytimes.com/2022/10/19/world/asia/korea-kakao-ceo.html "read the full text") : **韩国超级 App 下线**

<big>**`The result of ‘negligence’ and inadequate preparations`**</big>
在刚刚过去的这个周末，韩国的生活变得一团糟:Kakao 应用套件因为一个数据中心的火灾而宕机，并导致一名联席首席执行官辞职。十多年前，KakaoTalk 以一款消息应用起家，如今它已经成为一个自成一体的世界，服务涵盖银行和支付、打车、地图和游戏。在韩国，超过 90%的手机都有这个功能。…关于为什么该公司似乎缺乏更快恢复服务的应急计划的问题仍然存在。
…
多个客户团体准备对 Kakao 进行集体诉讼。…律师声称事故是“疏忽”和准备不足的结果。

有人说*辞职*吗？ [Manish Singh 和 Kate Park](https://techcrunch.com/2022/10/18/kakao-co-ceo-resigns-after-fire-incident-that-caused-mass-outage/ "read the full text") 补充:**公司投资超过 3 亿美元建造数据中心**

Kakao 的联席首席执行官 Whon Namkoong 已经辞职，这是对企业责任感的一次显著展示。此前，位于首尔以南盘京市的 SK C&C 数据中心发生火灾，导致大规模停电。[他]为“如此长时间”的大规模停电道歉，并表示他对该事件感到“责任重大”。
…
KakaoTalk 是韩国最受欢迎的消息应用程序，每月访问韩国 5170 万人口中的 4700 多万人。这款应用还被政府官员和企业使用，包括银行、打车服务和支付公司。……韩国总统尹硕益说[它]实际上是一个国家通信基础设施。

闻所未闻。邪恶的无神论者(玛丽莲·梦露)说，除了在朝鲜(不是朝鲜— *韩国*):

韩国和日本文化受儒家思想的影响由来已久。因为失败而辞职，不管你有多应该立即受到谴责，都被认为是一件值得尊敬的事情。荣誉不再是西方人理解的东西。

尚未完全确定，但目前的情况表明，这很可能是一个电火花…点燃了附近发电机房的应急燃料。

故障转移呢？[格威希尔](https://slashdot.org/comments.pl?sid=22244519&cid=62980445 "read the full text")一直在燃烧，因为这个世界一直在转动: *[ [你被解雇了](https://www.youtube.com/watch?v=H5SADK50TEM "WDStF")-Ed。]*

## 2.IaaS 通货膨胀

百分之二十:这是对一年后您将为公共云支付的额外费用的估计。基础设施即服务的成本正受到能源通胀和利率飙升的双重打击。

### 分析:DevOps 必须节省现金

所以找储蓄吧。但是你不能管理你不能测量的东西。你知道你在云上花了多少钱吗— **为什么？**

[Paul Kunert](https://www.theregister.com/2022/10/20/public_cloud_price_rise/ "read the full text") : **明年美国和欧洲的公共云价格将飙升**

渠道分析公司科纳仕公司(channel analyst)首席执行官史蒂夫·布拉泽(Steve Brazier)表示，挑战正在逼近，他指出，不断扩展的公共云基础设施的成本“极其昂贵”…对于美国，预计公共云价格将上涨…五分之一。借款成本正在上升。…看似无情的通货膨胀也是一个问题:…这包括建筑、网络设备和其他 IT 设备。

面对通货膨胀，DevOps 是否束手无策？不，[布罗 9](https://news.ycombinator.com/item?id=33259065 "read the full text") 说:

工程师应该像了解 CPU 负载、文件系统 I/O 或内存使用一样了解它们的成本。工程师应该知道他们扩展到了多少个实例，以及何时将会缩减。
……
不了解成本是不负责任的表现:……现在我们正处于一个应用程序的规模可以超出公司财力合理范围的时代，对应用程序成本的深入了解是一项基本的核心技能。…即使在高速增长期间，你也应该意识到你的成本。

迁移回本地的时间？ [UK_Bedders](https://forums.theregister.com/forum/all/2022/10/20/public_cloud_price_rise/#c_4551850 "read the full text") 有:

我们可以将虚拟机/容器整合到更少的物理机上，将空调温度调高一定程度，淘汰最后一批能效低于新设备的旧设备。如果所有东西都由 Azure 或 AWS 托管，我们会感激他们，他们会把任何价格上涨都推给我们，没有快速的方法来节省成本。

有什么减少 IaaS 账单的建议吗？ [pid-1](https://news.ycombinator.com/item?id=33258645 "read the full text") 有一个或两个:

特别是 AWS 的设计非常漂亮，所以你无法回答…钱去了哪里…以及让人们在没有必要的情况下花更多的钱。
–默认情况下，每资源成本关闭。打开它需要钱。多少钱？**`¯\_(ツ)_/¯`**
——像 S3 这样的服务会按 API 调用向你收费。你怎么知道谁在调用你的 API？使用默认情况下关闭的云轨迹。对于一个合理使用的桶来说，打开它进行这种分析可能要花费很多$$。
–cloud watch 将根据自定义指标向您收费，并快速累计。为什么我有这么多自定义指标？祝你好运。
–许多成本与非常难以控制的因素相关，例如出口流量。迄今为止，使用多个账户是分离不相关项目的最佳方式。但是，这样一来，你将最终为许多未被充分利用的 NAT 网关、VPN 连接、负载平衡器等买单……
……
这样的例子不胜枚举。我知道相当多的咨询公司靠减少 AWS 账单赚钱。
