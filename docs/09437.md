# 调查显示多云计算和成本挑战

> 原文：<https://devops.com/survey-surfaces-multi-cloud-computing-and-cost-challenges/>

一项针对美国和英国 360 名首席信息官和 IT 主管的调查发现，随着组织采用的云平台数量的增加，他们需要部署和掌握的工具数量也在增加。

管理云迁移的平台提供商 Virtana 本周发布的调查发现，近三分之二(63%)的受访者报告称，他们的组织依赖至少五种独立的工具来进行迁移、云成本优化、集成性能监控、应用性能管理和云基础架构监控。整整 83%的人花费一定程度的手动工作来整合来自所有这些工具的数据，或者只是孤立地使用它们。只有 17%的人说数据整合是完全自动化的。

近四分之三(73%)的受访者还指出，孤立的工作限制了他们实现云全部潜力的能力。

似乎多云计算只会让管理 It 变得更具挑战性。更不用说多种云方法给开发人员和开发人员带来的更大压力了，他们已经掌握了大量的开发工具、开发工具等等。当然，[云错误配置](https://devops.com/cloud-misconfigurations-threaten-cloud-migration/)的风险随着组织使用的云平台数量的增加而增加。

根据 Virtana 的调查，共有 82%的受访者拥有多云战略，超过四分之三(78%)的受访者在三个以上的公共云中部署工作负载。整整 59%的公司现在在公共云上运行超过一半的工作负载。超过一半(51%)的受访者还计划今年增加他们支持的公共云实例的数量，超过三分之一(34%)的受访者计划使用五个或更多云平台。

更具挑战性的是，超过四分之三(76%)的企业正处于实施无服务器计算框架的早期阶段。总体而言，超过一半(52%)的企业将继续维护混合环境，作为其核心 IT 战略的一部分。整整 96%的受访者看到了工作负载可移植性的价值，但只有 71%的人处于实现这一目标的早期阶段。预期的主要优势包括最大限度地节约成本(58%)、降低云服务提供商风险(46%)和提高业务灵活性(43%)。然而，近四分之三(75%)的企业没有或只有很少的云治理能力(34%)，或者将不同的工具拼凑在一起以实现这一目标(41%)。

Virtana 产品管理副总裁 Jonathan Cyr 表示，随着全球经济变得越来越具有挑战性，对云成本的敏感性开始上升只是时间问题。每次整体 IT 环境扩展时，复杂性的增加都会推高 IT 的总成本。

尚不清楚对云成本的敏感性会在多大程度上推动更多 It 组织集中管理 IT 环境。理论上，单个控制平面可以用来管理多个云计算环境，但今天大多数云仍然是彼此隔离管理的。

IT 团队也发现自己无意中被锁定在了云平台上。开发人员调用专有应用编程接口(API)越多，将工作负载从一个云平台迁移到另一个云平台就越困难。

几乎可以肯定的是，未来几个月将会有一场云计算管理清算。出于多种原因，开发团队可能更喜欢一个云平台，但随着 IT 成本的不断增加，财务办公室很快会提出更棘手的云成本问题。
