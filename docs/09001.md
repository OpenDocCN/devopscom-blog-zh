# 基础设施即代码(IaC):为什么协作是关键

> 原文:[https://devo PS . com/infra structure-as-code-IAC-why-collaboration-is-key/](https://devops.com/infrastructure-as-code-iac-why-collaboration-is-key/)

基础设施即代码(IaC)是一项重要的 DevOps 实践，允许开发人员和运营团队通过代码自动管理和配置基础设施，而不是通过手动流程进行配置更改。IaC 提供转型灵活性，可以显著加快交付速度、降低成本并帮助降低风险。

然而，当人们谈论 IaC 时，有时他们会忘记提到它的主要成分:协作。IaC 的全部目的是使基础设施和软件开发团队能够协作。

软件自动化公司 RackN， 的联合创始人兼首席执行官罗布·希施菲尔德说:“它(IaC)是为了让运营更像开发者。

Rob 将加盟 [预测 2022](https://www.techstrongevents.com/predict2022) 演讲嘉宾阵容畅谈基础设施 as Code 和 DevOps。在“添加 IaC 缺失的成分:协作”会议期间，Rob 将讨论如何帮助运营团队变得更加协作和成为业务的一部分，以及 IaC 如何帮助在系统内创建和重用自动化。

Rob 将展望 IaC 的未来，2022 年基础设施和运营团队将面临什么，并分享他自己的预测。

在对我们的杰基·惠特泽尔的一次简短采访中，罗伯让我们先睹为快，并分享了他对 2022 年的一个预测。

“人们对复杂性越来越紧张，但他们无法定义它，这就是为什么它如此模糊，”罗布说。“我们真正需要开始做的第一件事是弄清楚如何谈论复杂性，以及如何管理和驯服它。我先给你看一眼。来听听我的演讲，我们将讨论如何利用协作、团队合作和管道来驯服这种复杂性。”
