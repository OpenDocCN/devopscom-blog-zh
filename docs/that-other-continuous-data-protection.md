# 另一个持续:数据保护

> 原文：<https://devops.com/that-other-continuous-data-protection/>

四处寻找可以添加到我们当前项目工具链中的东西(这是我们做这些兼职项目的原因之一；为了检验各种各样的新想法/工具等)，并研究一位前同事是 CMO 的公司，我被吸引到了持续数据保护(CDP)的复兴世界中。有趣的是，我们最终可能会将 CDP 添加到“持续”活动的列表中，如 CI/CD/… CDP，尽管 CDP 的命名远在我们的 DevOps 或敏捷术语诞生之前。

CDP 的整体理念是它总是有备份的，因此您会受到持续的保护。最初推出时，它基本上失败了，因为它大大增加了您的存储环境的成本和复杂性。本质上，它需要您的所有[存储](https://devops.com/?s=data+storage)的热拷贝。这不是大多数企业愿意投资的。我是说，那太贵了。而且它使用了虚拟化存储，没人想乱用它。长话短说，当时，虚拟化存储带来了没人想处理的问题。

但是这些年来一些事情已经发生了变化，看到 CDP 概念在哪里出现是很有趣的。你看，今天它正被用来保护公司免受勒索软件。这仍然有点复杂，但我们已经成为隐藏复杂性的专家。它仍然需要大量的存储空间(我认为我看到的当前版本比最初的 CDP 解决方案需要更多的存储空间)，但当您的环境被勒索软件接管时，可以使用干净的机器将数据恢复到勒索软件接管之前的某个点。从“持续保护”的角度来看——按照“持续测试”的思路赋予它现代的含义——这是一个惊人的壮举。

它的工作方式——至少是我所看到的当前实现的工作方式——本质上非常简单，尽管正如我的前同事喜欢说的那样，“概念简单，实现困难。”它将所有写入存储的数据保存下来，并带有时间和货币标记。因此，它知道当前的磁盘*是什么样子，但它也可以知道昨天上午 12:33 磁盘是什么样子。或者一月一日，或者一年前…你明白了。*

它占用了大量的空间，因为这里隐含着一个真理——没有东西会被破坏。在传统存储中，如果您更改文档并点击保存，文档将被覆盖。在 CDP 的新版本中，如果您更改文档并点击 save，更改会被记录下来，但原始文档不会被更改。如果你愿意，可以称之为“非破坏性写入”。

值得一试。这些概念是巨大的，并提供了保护，否则在供应短缺。如果你有一个专门的 DevSecOps 团队，我会把他们作为重点，因为使用它的主要原因是本质上的安全原因(尽管还有其他用途；我目前正在滔滔不绝地谈论安全性，因为这是一件大事)。

下周，我将回到正轨，直接在我们最新的 DevOps 项目的工具链中汇报事情。我认为这是值得写的，因为你们大多数人都可以使用这个概念，即使只是为了内心的平静。你太棒了！保护您的作品免受最新范围的攻击者的攻击只是常识。CDP 的现代迭代是否适合您，值得一看。
