# Protegrity 宣布继续平台创新，推出新的动态数据屏蔽功能

> 原文：<https://devops.com/protegrity-announces-continued-platform-innovation-with-new-dynamic-data-masking-capabilities/>

*客户现在可以在单个数据存储中监控、屏蔽或标记数据，从而可以更灵活地根据业务发展速度保护数据*

**美国犹他州盐湖城，2021 年 6 月 9 日—**[全球数据安全领域的领导者 Protegrity](https://www.protegrity.com/) 今天宣布发布最新版本的 [Protegrity 数据保护平台](https://www.protegrity.com/platform)。Protegrity 行业领先平台的 8.1 版引入了动态数据屏蔽和监控功能，为客户提供了多种数据保护方法，所有这些都在单个数据存储中。该公司还推出了:增强了对保留语言的 Unicode 标记化的支持；Protegrity Cloud API，允许企业轻松地将数据保护功能嵌入到云 ETL 工作流中；以及重新设计的日志记录体系结构，为数据隐私计划提供更精细的可审计性。

Protegrity 总裁兼首席执行官 Rick Farnell 表示:“公司需要使用他们的数据来推动创新，但保持数据安全往往会延迟访问，这可能会对任务关键型人工智能、人工智能和分析计划产生重大影响。“我们最新的平台创新为数据保护方法提供了更多的选择和控制，有效地扩展了可用于保护数据的工具，从而加速了创新。”

监控和动态数据屏蔽为敏感数据访问提供了可见性和控制

凭借监控和动态数据屏蔽功能，Protegrity Data Protection Platform 8.1 版为客户提供了更多的选择，并让他们能够更好地控制为满足业务需求而实施的数据保护级别。监控功能为客户提供了在整个组织中如何访问和使用数据的基本可见性。为了更好地控制敏感数据，企业可以利用 Protegrity 的动态数据屏蔽，在用户访问时强制执行数据可见性规则，为未经授权的用户隐藏或“屏蔽”敏感数据，而不会改变静态数据。

“对于不太敏感的数据和轻量级用例，监控和动态数据屏蔽可以提供强大的安全基线水平，以立即降低风险，”Protegrity 产品和战略执行副总裁杰弗里·布林说。“随着组织的安全和隐私状况日趋成熟，我们将继续提供更强大的保护方法，如令牌化，这可以通过同一策略轻松实施。这确保了 Protegrity 可以帮助任何规模、任何行业的组织保护他们的敏感数据，无论他们需要处于数据隐私连续体的哪个位置。”

Unicode 标记化为全球企业提供了保留语言的数据安全性

Unicode 是一种国际标准，用于对世界书写系统中表达的文本进行一致的编码、表示和处理。在 8.1 版本中，Protegrity 现在将其行业领先的标记化的所有优势引入到以单字节和双字节 Unicode 标准编码的所有文本中。改进的功能提供了字符编码和保持长度的标记化，而没有性能损失，允许组织轻松标记所有西方字母的文本。

布林继续说道，“现有的记号化方法可以被征税以支持更大的字母，所以我们很自豪能够为我们的客户提供一种创新的 Unicode 记号化方法。这一新功能是 Protegrity 持续使命中的一个重要里程碑，它为全球组织提供不折不扣的选项来保护其最敏感的数据。”

新的云 API 和日志架构增强了平台功能

除了监控、数据屏蔽和 Unicode 标记化等新的数据保护方法之外，Protegrity 还推出了以下关键的平台增强功能:

*   Protegrity Cloud API:面向 AWS 和 Azure 等流行云服务的“功能即服务”令牌化产品，Protegrity Cloud API 允许组织轻松地将 Protegrity 的数据保护功能嵌入到 ETL(提取、转换、加载)工作流、原生云服务和自定义应用程序中。Protegrity Cloud API 构建于无服务器架构之上，无论数据量要求如何，都能提供弹性扩展和成本效益。

*   重新设计的日志记录架构:Protegrity 重新设计的日志记录架构是多版本功能的延续，该功能最初是在 8.0 版本中引入的，使用 Elasticsearch 为审计日志建立了一个新的存储数据存储。
