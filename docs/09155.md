# Styra 为现代应用程序授权开创了上下文丰富的权利

> 原文：<https://devops.com/styra-pioneers-context-rich-entitlements-for-modern-application-authorization/>

*开放策略代理的创建者和维护者通过策略即代码解决 IAM 和应用团队之间的授权鸿沟*

**加利福尼亚州雷德伍德城，2022 年 2 月 22 日—** [Styra，Inc .](https://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fwww.styra.com%3Futm_medium%3Dpr%26utm_source%3Dbusiness_wire%26utm_campaign%3Dawn%26utm_content%3Dwebsite&esheet=52582199&newsitemid=20220222005134&lan=en-US&anchor=Styra%2C+Inc.&index=1&md5=0fb22fa38883d1cb58d66d1044e6965d),[开放策略代理](https://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fwww.openpolicyagent.org%2F&esheet=52582199&newsitemid=20220222005134&lan=en-US&anchor=Open+Policy+Agent&index=2&md5=9dcdace1d283b2d62e62b1d89bf9a0ae) (OPA)的创始人和维护者，云原生授权的领导者，今天宣布 [Styra 声明式授权服务(DAS)面向云原生授权](https://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fwww.styra.com%2Fcloud-native-entitlements-with-styra-das-and-open-policy-agent%3FhsLang%3Den%26utm_medium%3Dpr%26utm_source%3Dbusiness_wire%26utm_campaign%3Dawn%26utm_content%3Dwebsite&esheet=52582199&newsitemid=20220222005134&lan=en-US&anchor=Styra+Declarative+Authorization+Service+%28DAS%29+for+Cloud-Native+Entitlements&index=3&md5=e0091f84066283b2817e3b3c31744db8)。这一新功能解决了传统和现代应用程序授权之间的差距，使身份和访问管理(IAM)团队能够充分利用现有的记录系统，同时为开发人员团队提供他们需要的东西——云原生、上下文丰富的授权，跨云、区域、可用性区域和集群复制。借助 Styra DAS，组织可以使用其现有的集中式 IAM 系统将应用无缝迁移到云，并通过统一授权加快云迁移。

随着组织继续其数字化转型，他们发现自己开发的授权解决方案阻碍了发展，因为他们的功能也必须迁移到云。这些服务建立在 LDAP、AD、SCIM 等之上。，如果留在内部，会给现代应用程序带来高延迟、单点故障，这显然是云应用程序的反模式。

Styra 的联合创始人兼首席技术官 Tim Hinrichs 表示:“对于以前不得不自己开发授权创可贴的 IAM 团队来说，我们的技术改变了游戏规则。“现在，借助 Styra DAS for Cloud-Native Entitlements，IAM 团队终于可以迁移到云原生技术，并扩展到其组织所需的水平。我们是第一家也是唯一一家为企业环境提供支持的组织。”

**面向云原生授权的 Styra DAS**

[Styra DAS for Cloud-Native Entitlements](https://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fblog.styra.com%2Fblog%2Fstyra-accelerates-cloud-migration-with-cloud-native-entitlements%3Futm_medium%3Dpr%26utm_source%3Dbusiness_wire%26utm_campaign%3Dawn%26utm_content%3Dblog&esheet=52582199&newsitemid=20220222005134&lan=en-US&anchor=Styra+DAS+for+Cloud-Native+Entitlements&index=4&md5=c7454176444af8d3eb21f9fe0f696a71)允许组织将规则、法规和政策从应用中分离出来，并使用现有的数据源，这些数据源:

*   缩短上市时间:无缝地为应用团队提供无状态、易于运行的云原生授权服务，该服务可与其应用一起部署。
*   避免成本:使用现有的集中式 IAM 系统，避免昂贵的大修，并了解政策对应用程序的影响。
*   提供审计就绪授权:访问与决策日志一致的预建策略包—授权实施的证据清晰且可导出。

KuppingerCole 创始人兼首席分析师 Martin Kuppinger 表示:“Styra DAS for Cloud-Native Entitlements 解决了许多企业 IAM 团队在迁移到云原生环境时面临的难题。“非常需要对这些权利进行基于政策的管理。Styra 正在推出一种创新的解决方案来应对这些挑战。”

Styra 经历了大幅增长，最近在 2021 年 5 月的 B 轮融资中获得了 4000 万美元。2022 年，Styra 将为 OPA 添加其交钥匙控制面板，以提供跨云原生堆栈的统一授权。随着开源项目同比增长 300%,迄今为止下载量超过 1 . 25 亿次，OPA 已经成为云原生环境中事实上的授权标准。 [Styra DAS](https://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fwww.styra.com%2Fpricing%3Futm_medium%3Dpr%26utm_source%3Dbusiness_wire%26utm_campaign%3Dawn%26utm_content%3Dwebsite&esheet=52582199&newsitemid=20220222005134&lan=en-US&anchor=Styra+DAS&index=5&md5=74b4733254e8531a7c23e7c0a3e9a13f) 提供了基于 OPA 构建的统一政策即代码平台，以确保云基础设施、Kubernetes、应用和服务网格部署的安全性和合规性。OPA 和 Styra DAS 共同为整个云原生堆栈提供安全性、运营和合规性防护。
