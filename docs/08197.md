# 容器如何简化 DevOps 工作流和 CI/CD 管道

> 原文:[https://devo PS . com/how-containers-simplify-devo PS-workflows-and-ci-CD-pipelines/](https://devops.com/how-containers-simplify-devops-workflows-and-ci-cd-pipelines/)

DevOps 使组织能够实时利用云资源的灵活性，这是一个巨大的游戏改变者。正如采用[敏捷方法](https://agilemanifesto.org/)可以使开发团队反应更快一样，DevOps 彻底改变了 IT 目标的实现，从瀑布式的流程控制公司生产力到使高质量软件快速简便的动态流程。

组织一直在利用越来越灵活的资源，并通过硬件和软件资源规划、订购和集成来摆脱传送带式的项目交付。DevOps 提供了一种方法来解决这个问题，“如果我们可以快速、轻松地提供资源，我们如何以类似的响应能力完成整个项目？”

由于 DevOps 降低了开发和运营之间的壁垒，保持理智和负责并不一定会使流程陷入困境。结构化的交流仍然会发生，但是是以一种迭代的、渐进的方式，就像打磨宝石一样。

我们可以创造、部署和调整切实可行的解决方案，而不是在遥远的未来设定崇高的目标。该过程更快地将应用程序送到最终用户手中，使用实际的用户反馈来平滑任何粗糙的边缘，并帮助组织不仅对不断变化的需求做出更快的响应，而且更有效地利用宝贵的软件开发和操作资源。

## 为什么开发和运营团队需要协作

一个组织的文化很难适应这种程度的变化。但是开发和运营团队都有一些棘手的问题，这些问题有助于激励他们更加动态地合作。

公司的开发(Dev)方希望交付更及时的解决方案，确保对新应用程序的更大接受度，并更有效地使用其资源。运营部(Ops)在硬件资源规划方面面临挑战；他们的维护和降低成本项目被推迟了。Ops 也对新功能、效率和集成有深刻的见解，但没有办法实施它们。

当传统的瀑布式流程限制了沟通和实现时，就会出现这些和其他棘手的问题。在两个团队之间创建新的流程有助于减少这些棘手问题。

DevOps 并不局限于特定的技术，而是一种旨在提高软件开发效率的方法。集装箱化将开发和 IT 运营之间的差距缩小到最小。[容器](https://devops.com/?s=containers)是支持 DevOps 工作流并简化开发和交付管道的伟大工具。容器保存运行应用程序所需的代码和依赖关系，并且独立存在。这使得团队能够在这些封闭的环境中开发、测试和部署应用。这不会影响交付的不同部分，使得测试人员和开发人员的生活变得更加容易。

## 启用 CI/CD

DevOps 创造了一种自动化流程的方法，可以更快、更可靠地构建、测试和编码。持续集成/持续交付(CI/CD)并不是一个新概念，但是像 Jenkins 这样的工具已经做了很多来定义 CI/CD 管道应该是什么样子。虽然 DevOps 代表了组织中的文化变革，但 CI/CD 是推动 DevOps 成功的核心引擎。

使用 CI，团队必须更频繁地实现较小的变更，但是他们用版本控制库检查代码。因此，在应用程序的构建、打包和测试中有了更多的一致性，带来了更好的协作和软件质量。CD 从 CI 结束的地方开始。因为团队在几个环境中工作(生产、开发、测试等)。)，CD 的作用是将代码自动部署到这些环境中，并执行对数据库和服务器的服务调用。

CI/CD 概念并不是全新的，但直到现在，我们才拥有正确的工具来充分获得 CI/CD 的好处。容器使得实现 CI/CD 管道变得非常容易，并且支持更加协作的文化。容器非常轻量级，可以无限扩展，可以在任何环境下运行，并且非常灵活。没有比这更容易的了。

现在，您可以跨容器(或容器集群，如 Kubernetes)移动代码，而不是在不同环境中的各种虚拟机(VM)之间移动代码。虚拟机是静态的，利用整体应用架构，而容器使用分布式微服务模型。这为弹性、高可用性和资源使用带来了新的优势。

作为 DevOps 面向团队的方法的一部分，团队成员可以接触他们的同事并分享创新，将他们的知识和经验融入到演进过程中。开发团队可以使用新的资源，利用容器和 Kubernetes 等技术来构建解决方案。通过共同努力，这种合作让 Ops 能够运行该解决方案，对其进行扩展、分发和升级，而不会出现延迟。

## 把所有的放在一起

DevOps 已经帮助组织重新构想了他们的整个开发生命周期。虽然 DevOps 带来了文化变革，但 CI/CD 是 DevOps 成功的核心引擎。像容器这样的新工具现在使组织更容易充分利用 CI/CD。但是，必须考虑数据存储，因为它是动态 DevOps 环境的基础。它还提供故障转移、灾难恢复和其他基本的可用性和可靠性功能。软件定义的存储提供了动态开发运维环境所需的灵活性。这是 DevOps 项目成功的关键因素。
