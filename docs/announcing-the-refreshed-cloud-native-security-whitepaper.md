# 发布更新的云原生安全性白皮书

> 原文：<https://devops.com/announcing-the-refreshed-cloud-native-security-whitepaper/>

CNCF[安全技术顾问小组(TAG)](https://github.com/cncf/tag-security) 刚刚发布了一份更新的 [云原生安全白皮书 v2](https://github.com/cncf/tag-security/tree/main/security-whitepaper) 来帮助教育社区关于保护云原生部署的最佳实践。该白皮书旨在为组织及其技术领导层提供对云原生安全性、其在生命周期流程中的整合以及确定 NIST SSDF 等标准适用性的注意事项的清晰理解。白皮书的 [第一版](https://www.cncf.io/blog/2020/11/18/announcing-the-cloud-native-security-white-paper/) 于 2020 年 11 月 18 日发布。这篇论文已经被翻译成不同的语言，第一版的音频可以在 [这里](https://soundcloud.com/user-769472014/sets/cncf-tag-security-cloud-native-security-whitepaper-version-v1) 找到。

“在过去的 18 个月里，安全行业和我们的集团随着对供应链和勒索软件的攻击而发展。作为长达数月的[](https://www.cncf.io/blog/2021/10/12/cloud-native-security-microsurvey-more-than-80-of-organizations-want-to-build-modern-security-systems-with-open-source-software/)回顾过程的一部分，我们收到了对论文第一版改进的极好反馈。自本文第一版发布以来，我们的团队还通过多种补充材料不断壮大，如 [安全供应链安全白皮书](https://www.cncf.io/announcements/2021/05/14/cncf-paper-defines-best-practices-for-supply-chain-security/)[云原生安全地图](https://cnsmap.github.io/)[词典](https://github.com/cncf/tag-security/tree/main/security-lexicon) 。作为充满活力的社区的标志，就像第一个版本一样，整个社区的人们带来了他们的热情、技能和专业知识，以解决最终用户的反馈和云原生安全领域的发展难题。

虽然原始论文的大部分内容经受住了时间的考验，但这一更新版本通过浏览勒索软件事件处理的具体使用案例以及如何根据欧盟法规保护金融机构的安全，揭开了安全保证和合规性的神秘面纱。来自原始论文读者的反馈也通过包含这些新的部分来处理:

*   **安全默认–云原生 8:** 关于实施默认安全的云原生应用的高级 [指南](https://github.com/cncf/tag-security/blob/main/security-whitepaper/secure-defaults-cloud-native-8.md) 。
*   **SSDF v1.1 映射:** 将 [NIST SSDF](https://csrc.nist.gov/publications/detail/sp/800-218/final) 实践和任务映射到云原生安全应用生命周期
*   **ATT & CK 集装箱威胁矩阵:** 概述 [威胁矩阵](https://attack.mitre.org/matrices/enterprise/containers/) 如何提供一种结构来应用本文所述的指南
*   关于如何分享反馈的指导: 关于如何分享论文反馈的指导现已成为论文的一部分，并简要总结了第一版发表后如何收集和处理反馈。

CNCF 安全标签旨在促进厂商中立的协作，以发现和生产资源，为云原生生态系统中的运营商、管理员、开发人员和最终用户实现安全访问、策略控制和安全性。如果您有兴趣参与安全标签，请查看 [章程](https://github.com/cncf/tag-security/blob/master/governance/charter.md) 了解更多信息，并欢迎前来打招呼在[# tag-security](https://cloud-native.slack.com/archives/CDJ7MLT8S)频道上 CNCF 懈怠。

白皮书可在 [GitHub](https://github.com/cncf/tag-security/tree/main/security-whitepaper) 上获得。
