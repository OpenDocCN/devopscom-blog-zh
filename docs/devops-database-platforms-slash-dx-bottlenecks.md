# DevOps 数据库平台大幅降低了 DX 瓶颈

> 原文：<https://devops.com/devops-database-platforms-slash-dx-bottlenecks/>

对于拥有许多数据库和各种数据需求的企业来说，DevOps 持续交付带来了一个问题。例如，测试数据供应是当今的一个挑战，尤其是在数据库更新的预发布测试中。标准实践涉及到共享的数据库环境，需要从一个测试到下一个测试进行维护，这减慢了测试周期，并且通常不适合自动化的 CI 管道。正如我在[中解释的，需要一个新的 DevOps 数据库平台](https://devops.com/a-new-devops-database-platform-is-needed/)，充其量，它使事情变得混乱；最坏的情况是，数据库与 DevOps 集成的问题严重阻碍了组织的整体灵活性，这破坏了 DevOps 的初衷。

最近的白皮书[一种新的 DevOps 数据库平台](https://windocks.com/database-devops-whitepaper)指出，需要一种新的 DevOps 数据库平台有多种原因，它们是:

*   兼容性要求
*   与当前 CI/CD 管道集成
*   与价值流管理组合的集成
*   基础设施成本效率
*   开发人员、测试人员和操作人员的生产力
*   安全和治理
*   DevOps 性能

该白皮书解释了 DevOps 数据库平台需要以下功能来满足这些要求。

*   **自动化数据库供应:**使用自动化过程部署数据库的能力。
*   **数据库克隆:**可写数据库环境基于构建的版本化映像按需交付。
*   **分布式存储库:**数据库映像可以驻留在 Linux 和 Windows 服务器以及存储系统上。
*   **简单的声明性构建**:一种简单的预定义方法，使版本化数据映像的创建变得简单，同时支持用户/组访问控制、数据屏蔽和其他数据工程功能。
*   **可扩展的数据克隆:**数据库克隆按需提供，从单个映像同时提供许多克隆。
*   **全面的数据库支持**:repo 支持所有常见的关系、NoSQL、机器学习和基于文件的数据存储。
*   **版本化数据库:**每个映像都成为版本化的、不可变的 DevOps 工件。
*   **同等部署到所有云环境:** DevOps 数据库可以在任何内部基础架构或云上部署和运行。
*   **跨平台:** DevOps 数据库可以在 Windows、Linux 等常见操作系统环境下部署和运行。
*   **安全特性包括访问管理、数据屏蔽和加密:**数据库映像通过 SQL 脚本、批处理文件或命令行工具轻松整合访问控制、数据屏蔽和合成测试数据。
*   **兼容 CI/CD 平台:** DevOps 数据库可以在 Azure DevOps、Jenkins、CloudBees 等多个 CI/CD 平台工具中同样良好地部署和运行。
*   **兼容容器和容器编排器:** DevOps 数据库可以在 Docker、Kubernetes、Red Hat OpenShift 等多个容器环境中同样良好地部署和运行。
*   **以 DevOps 的速度交付可写数据:**数秒交付可写数据库。
*   **易于与 DevOps 工具集成的 API:**有一个 API 可用于支持 DevOps 数据库控制平面与其他工具的集成。
*   **审计日志记录:**devo PS 数据库包括一个完整的操作审计日志。

## 向新数据库 DevOps 平台转型

企业中的 DevOps 数据库平台需要的不仅仅是简单地购买和安装软件并告诉每个人使用它。随着多个根深蒂固的数据库平台已经就位，必须克服一些实际和政治上的考虑。向企业中的一大组应用程序引入任何更改都可能会很混乱，如果更改不引人注目，通常会花费更多的成本和太多的时间。

为了具有足够的吸引力，数据库 DevOps 平台需要经济实惠、易于理解、易于设置并与各种 CI/CD 管道一起使用。以下是足以推动采用的诱人优势:

*   作为 DevOps 工件管理的版本化、安全的数据库环境或映像。
*   DevOps 团队可以构建和管理映像，无需新的基础架构。
*   几秒钟内即可建立多数据库和多 TB 数据环境。
*   对于 CI/CD 管道，可以按需为数据库环境提供服务。
*   简化了较低级别的数据库环境基础架构，单个虚拟机能够支持多个并发的数据库环境，这推动了直接的成本节约和积极的投资回报。
*   版本化数据库环境的数据安全性和治理优势，以及内置和可审计的安全策略。

### **这意味着什么**

数字化转型是现代企业和组织保持竞争力的必要条件。然而，许多企业和组织的转型受到将数据库集成到其 DevOps 应用程序管道和价值流中的挑战的严重阻碍。一个新的 DevOps 数据库平台可以允许像对待 DevOps 工具链中编排和自动化的任何其他 DevOps 工件一样对待数据库。

用于自动化数据库环境交付的建议资源包括 [Windocks](https://windocks.com/) 、Cohesity 和其他存储设备，这些设备提供可适用于 DevOps 目的的虚拟化(克隆)数据库环境。我要特别感谢来自 Windocks 的意见，他推出了一个 DevOps 数据报告，该报告的特点是在标准 Linux 和 Windows 服务器上对版本化数据环境和 CI 管道进行数据库虚拟化，并为容器、Kubernetes 和传统实例的数据交付提供云原生支持。
