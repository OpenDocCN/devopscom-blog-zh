# DevOps 必须在 2023 年实现自动化

> 原文:[https://devops.com/devops-must-embrace-automation-in-2023/](https://devops.com/devops-must-embrace-automation-in-2023/)

对美国 500 名工程和软件开发专业人员的调查表明，在新的一年里，比以往任何时候都需要更多的开发团队决心在更深的层次上拥抱 T2 自动化。

这项由可观察性平台提供商 Chronosphere 赞助的调查发现，工程师平均每周花费超过 10 个小时来尝试分类和了解事件。这相当于一周工作 40 小时的四分之一。

总体而言，96%的受访者表示，他们将大部分时间花在解决低级问题上，88%的受访者表示，这些时间对他们和他们的职业生涯产生了负面影响，因为他们花了太多时间来解决 IT 问题。

超过三分之一(33%)的人说这些问题扰乱了他们的个人生活，39%的受访者承认他们经常感到压力很大。不到四分之一(22%)的人说他们想戒烟。

Chronosphere 首席执行官 Martin Mao 表示，DevOps 团队遇到的大部分压力都是由于依赖组织自己构建的 DevOps 平台造成的。他补充说，今天由供应商构建和支持的平台通常提供更高水平的自动化，从而降低整体压力水平。

调查发现，40%的受访者经常从他们的可观察性解决方案收到警报，但没有足够的上下文来对事件进行分类。共有 59%的人表示，他们从当前的可观察性解决方案中收到的一半事件警报实际上没有帮助或不可用，49%的人报告说，他们在使用当前的可观察性方法时遇到了不一致的性能。近一半(45%)的受访者表示，他们当前的可观察性解决方案需要大量的手动时间和人力。

总体而言，使用供应商解决方案的受访者中有 42%表示他们每个季度都会遇到严重程度较高的事件，相比之下，使用他们构建的平台的受访者中有 61%表示会遇到严重程度较高的事件。在没有使用供应商解决方案的组织中，大多数表示他们会考虑这样做来提高团队生产力(61%)或提高可靠性(54%)。

随着组织部署新兴的云原生应用程序，压力水平可能会增加。整整 87%的受访者表示，云原生架构增加了事件发现和故障排除的复杂性。

毛说，在充满挑战的经济时期，对于 DevOps 的领导者来说，了解构建和维护平台的总成本更为重要。在没有其他可行选择的时候，许多组织决定构建自己的平台。他指出，支持这些平台所需的手动流程导致劳动力成本上升，从而阻碍了经济高效地扩展工作流的能力。

目前还不清楚 2023 年，任何已经建立自己的 DevOps 平台的组织对变革的胃口会有多大。面临的挑战是，考虑到已经部署了传统平台，更换商业平台的初始成本可能会很高。但是，购买新平台的成本需要与持续运营的总成本进行权衡。许多软成本表现在，例如，DevOps 专业人员的离职率更高，他们可能会在其他地方找到一份更有回报、不那么单调乏味的工作。无论如何，在 2023 年初，很明显有比以往更多的困难问题需要被问和回答。
