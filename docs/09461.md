# 告别传统系统

> 原文:[https://devops.com/saying-goodbye-to-legacy-systems/](https://devops.com/saying-goodbye-to-legacy-systems/)

你还在使用旋转式电话、显像管电视或名片夹吗？大概不会。但是，如果您是，您仍然会使用比许多企业仍然每天依赖的技术更新的技术来驱动我们每天在购物、银行或其他数百项正常任务中使用的任务关键型应用程序。

至少有三分之二的大型企业[](https://www.businesswire.com/news/home/20210624005072/en/78-of-Enterprises-Have-Started-at-Least-One-Modernization-Program-as-a-Direct-Result-of-Covid-19)运行着 10 到 20 年前的基于大型机的应用程序，超过 25%的企业运行着 20 到 30 年前的应用程序。如今，企业通常运行和维护基于传统数据库的传统应用，同时构建基于微服务的应用来加速新服务和创新。“如果它没坏，就不要修理它”这句口头禅当然适用于这里；虽然有许多原因导致组织继续依赖大型机和传统系统，但也有许多组织希望开始现代化计划，但不知道从哪里开始。

然而，过去几年，各行各业涌现出一波数字优先的新公司，给传统的基于系统的业务带来了压力——从零售到金融，从制造到技术。新冠肺炎疫情加快了许多组织的现代化计划，并明确表示，对于那些想要或需要现代化以保持竞争力的人来说，避免这个问题不是一个选项。公司发现，如果他们不主动进行现代化，当他们需要更新和发展时，被动反应的成本会非常高。

加快 [IT 现代化](https://devops.com/?s=IT+modernization)计划可能是一项重大挑战:安全地现代化这些遗留应用程序和大型机系统以及其中的关键数据看似不可能，但却是可以做到的。例如， 像 [克罗格](https://blog.yugabyte.com/kroger-distributed-sql-architecture-at-scale/)这样的公司已经实现了这一飞跃，有很多东西可以向他们学习。这里有三件事可以帮助任何企业告别大型机和遗留系统:

### 您需要一条无摩擦的途径来运行传统应用程序

许多现代解决方案只关注绿地应用，并帮助新生的云公司和项目；然而，现实情况是，大多数财富 1000 强公司都有许多现有的应用程序，他们不能忽视或一次性完全重建它们。使用 Postgres 等标准接口支持现有应用程序，同时提供可伸缩性和弹性来支持现代应用程序，这对于提供一座桥梁至关重要，从公司已经走过的地方到他们正在尝试的地方。分布式 SQL 数据库为现有应用的现代化提供了一条无缝的前进道路，同时支持新的和当前的计划。

### 拥抱自动化

是时候停止依赖复杂的代码或带有附加第三方工具的脆弱系统了。能够依靠数据层的智能和自动化来实现分发、弹性和可用性，对于最大限度地减少对当前应用程序的更改和影响至关重要。这样做将降低风险，卸载传统应用程序任务，并确保系统具有正常工作的智能。借助分布式 SQL 数据库，您可以交付单个逻辑数据库，该数据库在应用程序看来就像一个整体式的单服务器数据库，但其底层智能允许它跨混合云、私有云和公共云延伸。

### 寻找灵活性，面向未来

为了避免被特定的系统或架构所束缚，是时候采用一种面向未来的思维模式了。企业可以通过实现这一点

*   **多云/混合云** :借助分布式 SQL 数据库，您可以交付单个逻辑数据库，该数据库看起来像一个整体式的单服务器数据库，但具有跨混合云、私有云和公共云延伸的智能，可灵活地部署到任何需要或最适合应用的地方。
*   **任何硬件** :没有两个数据中心看起来是一样的，并且有各种各样的硬件需求，从裸机到虚拟机和容器。合适的现代分布式数据库在数据层的安装位置上提供了极大的灵活性，以匹配任何已在使用的硬件和配置。
*   **任何部署** :现有组织有各种各样的环境和部署类型，从单个数据中心到两个或更多私有数据中心，再到越来越多的跨内部和公共云的混合环境。现代分布式数据库提供了支持各种部署选项的自由，包括同步复制、地理分区和跨集群异步复制。
*   **开源自由** :虽然许多数据库提供了一些开源组件，但仍有少数 100%开源的解决方案致力于 Apache 2.0 许可。用户可以从这些开源解决方案中的全套企业特性和功能中受益，也可以从 100%开源解决方案带来的安全性和透明性中受益。

关于传统系统的现代化，是时候将古老的格言“如果它没有坏，就不要修复它”更新为“如果它没有修复，它可能会坏”。如果您正在考虑开始现代化之旅，现在就节省时间、资源和资金来进行升级，这对于您的内部团队来说可能是昂贵、耗时和难以承受的。
