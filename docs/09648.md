# 组织努力实现应用程序现代化

> 原文:[https://devo PS . com/orgs-奋斗获得应用程序-现代化-权利/](https://devops.com/orgs-struggle-to-get-app-modernization-right/)

今天发布的一项调查发现，93%的 IT 领导者受访者承认应用程序现代化工作极具挑战性。

这项调查由 IT 服务提供商 abase as 进行，调查了 150 名至少拥有 1000 名员工的组织的 IT 领导。调查发现，受访者确定的主要问题是确定正确的工具和技术(30%)，其次是找到具有正确经验的员工(20%)，选择正确的外部合作伙伴(18%)，让用户接受新系统培训(17%)以及获得管理层的支持(11%)。

超过四分之三的受访者(78%)表示，组织在进行应用程序现代化时的首要任务应该是“正确使用”

像许多事情一样，应用程序现代化的定义是在旁观者的眼中。三分之一的受访者将现代化定义为使用基于云的基础设施，而 32%的受访者表示正在将传统系统迁移到云中。另有 22%的人将其定义为使用基于云的工具。

超过一半的受访者(54%)表示，使用微服务意味着他们正在现代化，而 59%的人表示，使用任何云原生服务都表明了这一点。

satellite as 的实践主管 Scott Wheeler 说，虽然有很多关于应用程序现代化的定义，但很明显，将应用程序迁移到云的 T2 时代即将结束。相反，组织更喜欢使用各种各样的云原生平台和平台即服务(PaaS)环境来构建和部署应用程序，而无需管理这些环境。

面临的挑战是，这些应用程序中有许多是在多个云上构建和部署的。超过四分之一(27%)的受访者表示，最大的多云挑战是保护应用程序，其次是与内部系统和其他云平台的集成(26%)。另有 20%的人认为确保平台间的可移植性是最大的挑战。

总共 80%的人还指出，提高内部员工的技能和改进过时的流程对于应用程序现代化极其重要。Wheeler 说，事实上，组织仍然需要解决的主要问题之一是“糟糕的开发操作”实践在现代应用环境中的应用程度。他补充说，这个问题很大程度上源于一个简单的事实，即许多 IT 团队仍然不知道如何有效地管理云计算平台。

超过三分之一(36%)的人表示，关键的应用程序现代化最佳实践是给自己充足的时间来完成项目，同时有同等比例的人认为要避免走捷径、分心和/或合作伙伴做出过多承诺。另有 34%的人表示，最重要的教训是为工作选择最佳工具，而近四分之一(24%)的人认为，对一个显然需要现代化的系统进行现代化的时间太长了。

整整 100%的受访者表示应用程序现代化对他们的业务非常重要或有点重要，45%的受访者表示他们选择进行现代化的应用程序非常重要。略多于四分之一(28%)的人还指出，现代化使他们能够更快地创新和扩展，而 25%的人说这可以提高最终用户的工作效率，11%的人说这可能会使他们的公司对员工和客户更具吸引力。

不管应用程序现代化的动机是什么，几乎每个组织都在不同程度上从事这些工作。问题似乎不是开始这些项目的意愿，而是确保它们足够快地完成以与业务相关。
