# 简化云工具、平台管理

> 原文:[https://devops.com/streamlining-cloud-management/](https://devops.com/streamlining-cloud-management/)

开发运维及微服务改变了组织开发、部署和管理应用的方式，以及支持应用所需的底层基础设施，无论是内部部署的云。一方面，开发运维服务和微服务能够实现更高的速度和敏捷性。另一方面，这种速度和灵活性也伴随着复杂性的增加。

当 DevOps 开发的所有元素和谐地工作时，这是一件美好的事情。然而，所使用的各种工具和平台往往缺少 IT 团队所需的关键安全特性和功能。

[Apcera](https://www.apcera.com/) 拥有全面的开源项目组合，为客户提供跨各种平台和云基础设施有效管理和保护应用的能力。Apcera 的产品组合包括 [NATS](http://www.nats.io/) ，一个云原生消息系统； [Kurma](http://www.kurma.io/) ，一个灵活的容器运行时；以及 [Libretto](https://www.apcera.com/blog/using-golang-virtual-machine-management) ，一个虚拟机配置库。该公司最近宣布，它正在扩展其平台，以支持 Kubernetes。

“Apcera …允许客户安全地迁移到任何云，并在任何基础架构上部署、协调和管理最广泛的工作负载，包括内部部署，”Apcera 创始人兼首席执行官德里克·科利森在一份新闻稿中说。“在部署基于容器的工作负载时，客户和开发人员迅速支持 Kubernetes 作为他们的 API 选择。凭借其快速增长的生态系统和强劲的市场势头，这显然是我们的发展方向。”

Kubernetes 是一个开源的容器编排平台，最初由 Google 创建，现在由云本地计算基金会(CNCF)管理。Kubernetes 使组织能够将大量的容器集合作为一个系统来管理。拥抱 Kubernetes 扩大了 Apcera 的影响范围，将混合云管理纳入其中。Apcera 还将积极参与 Kubernetes 开放源码项目。

“我们认为向云提供信任至关重要，而 Apcera 在这一领域拥有深厚和成熟的专业知识，”谷歌云平台开发人员 Kelsey Hightower 表示。"我们欢迎他们对 Kubernetes 的支持以及未来对该项目的参与."

集装箱尤其是一个主要的安全问题。来自 Gartner 的一篇[博客文章指出，“容器的安全属性在很大程度上是一个未开发的领域，关于容器是否包含有很多有争议的讨论。有时，讨论似乎是由(隐藏的)业务议程、合作伙伴关系和财务依赖驱动的，而不是由简单的技术驱动的。所以，抛开所有这些不谈，你能让你的容器包含或不包含吗？”](http://blogs.gartner.com/joerg-fritsch/can-you-make-your-containers-contain/)

在过去的一年中，在更安全的容器技术方面取得了很大的进展。微软的 Hyper-V Containers、英特尔的 Clear Containers 和 T2 的 CoreOS 最近的改进都说明了容器世界已经意识到了安全问题，并且正在努力解决这个问题。

组织需要了解其基础架构的安全性，并能够回答诸如“我如何快速有效地投入生产并具有可扩展性？”简化各种云工具和平台的管理使 IT 组织能够在创新和速度与安全性和稳定性之间取得平衡。将 Kubernetes 支持融入 Apcera 产品组合可能是在混合云环境中有效管理和保护应用和基础设施的关键。
