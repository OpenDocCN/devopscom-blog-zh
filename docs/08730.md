# New Relic 推出业界最大的快速入门和合作伙伴集成开源生态系统，帮助每个工程师在几分钟内实现可观察性

> 原文：<https://devops.com/new-relic-launches-industrys-largest-open-source-ecosystem-of-quickstarts-and-partner-integrations-to-help-every-engineer-embrace-observability-in-minutes/>

**Cribl、Fastly、Gigamon、Kentik、Lacework 和趋势科技等领先的企业技术有助于加快所有工程师对可观测性的采用**

 ***美国加州三藩市**–**2021 年 10 月 13 日**–[新遗迹](https://newrelic.com/) (纽约证券交易所代码:NEWR)，可观测性公司，今天推出了 [新遗迹即时可观测性](https://newrelic.com/platform/about-instant-observability) (I/O)，这是一个快速启动的开源生态系统，让所有软件工程师能够在几分钟内对他们的整个技术堆栈进行仪表化、仪表盘和警报。作为 New Relic 承诺使可观测性成为每个工程师的开放、数据驱动和日常实践的一部分，New Relic I/O 提供了与 400 多个云服务、开源工具和企业技术的端到端集成，由社区贡献和维护。此次发布会展示了 Cribl、Fastly、Gigamon、Kentik、Lacework 和 Trend Micro 等合作伙伴的贡献，旨在将世界领先技术的力量带给世界各地的工程师，无论是技术还是使用案例。New Relic I/O 是 New Relic 慷慨的免费定价层的一部分，因此每个工程师都可以在不与销售人员交谈或提供信用卡的情况下开始工作。

根据 New Relic 的 [2021 年可观测性预测](https://newrelic.com/reports/2021-observability-forecast) ，虽然大多数工程师和开发人员熟悉可观测性，但在采用实践方面存在巨大差距:只有 26%的受访者拥有成熟的可观测性实践，最常引用的可观测性成功障碍是缺乏资源(38%)和技能差距(29%)。New Relic I/O 通过引入一个以知识为中心的不断扩展的开放资源生态系统来解决这一差距，该生态系统将世界上可观测性专家和从业者的集体经验整理成文，以帮助世界各地的工程师更快地释放他们数据的力量。

**延续新遗迹对“开放”可观测性的承诺**

今天的新闻强化了 New Relic 的承诺，即通过继续投资全球开源社区，使可观测性成为每个工程师的日常最佳实践。在过去的 12 个月里，New Relic 开源了 10 多年的代理 R & D，标准化了 OpenTelemetry，并向云本地计算基金会(CNCF)贡献了 Pixie。New Relic I/O 是这一战略的延续，旨在大幅降低工程师在规划、构建、部署和运行出色软件时的可观察性障碍，这些软件为客户、员工、合作伙伴和供应商提供完美的数字体验。

**New Relic 首席执行官 Bill Staples** 表示:“作为我们开发人员第一的战略和对开源和开放社区承诺的一部分，我们很自豪地推出一个新的开放生态系统，通过一个充满活力的遥测数据源、仪表盘和警报配置快速入门社区，帮助所有软件工程师将可观测性作为一种数据驱动的方法和日常实践。“我们很荣幸与一个不断扩大的行业领先合作伙伴团队一起启动，他们将帮助我们兑现我们的承诺，让所有人都可以观察，这样每个工程师都可以在几秒钟内立即观察任何硬件或软件系统、云服务、SaaS、物联网设备或应用。”

“在未来的几年里，我们预计将会有成千上万的遥测源、预建仪表板和快速入门由工程社区管理，并被数百万需要遥测数据的工程师使用，以便在软件生命周期中做出更好的决策。只有 New Relic 致力于这种开放生态系统的可观测性，”Staples 补充道。

与其他只关注仪器仪表的交易所不同，New Relic I/O 引入了一个包含 400 多个 quickstarts 的开放式中心，它不仅仅是仪器仪表，还提供了预建的仪表盘、警报配置、用户指南和文档，以帮助软件工程师通过引导式安装流程开始观察。此外，New Relic I/O 是由世界领先的可观测性专家建立的社区，每个贡献都由 New Relic 审查。用户和合作伙伴可以轻松地添加新内容，改进现有的快速入门，并随时向快速入门库提供建议。

**即时观察领先合作伙伴的技术**

作为 New Relic 提供开放可观测性承诺的一部分，此次发布采用了来自 Cribl、Fastly、Gigamon、Kentik、Lacework 和趋势科技的 quickstart 贡献，以增强对每个工程师日常工作流程中应用性能的理解。此外，New Relic 正在推出云服务、开源工具和企业技术的快速入门，包括 Kubernetes、Istio、Apache 和 Cassandra，因此每个工程师都可以根据自己的首选工具和特定用例获得定制的可观察性视图。

*   Cribl 是 observability pipeline 公司，让客户解析和发送任何类型的数据。[Cribl quick start](https://developer.newrelic.com/instant-observability/cribl-logstream/e67f2859-80c1-4234-bbcf-bcbeeb31d70d/)可让您直接从 New Relic One 获得整个环境的可见性，而无需创建自己的仪表盘和警报，从而简化您的工作流程并缩短价值实现时间。
*   **Fastly** 是一个边缘云平台，使其客户能够快速、安全、可靠地创建出色的数字体验。借助[Fastly CDN quick start](https://developer.newrelic.com/instant-observability/fastly-cdn/c5c5dd30-dcdf-46b6-9412-f9a1bba5a600/)，您可以监控 fast ly 内容交付网络的关键指标，从而帮助您提高服务可靠性，确保最终用户获得出色的在线体验。
*   **Gigamon** 是云可视性公司。Gigamon Hawk 混合云可见性和分析平台提供对所有网络流量的访问并从中提取情报。[giga mon quick start](https://developer.newrelic.com/instant-observability/gigamon-newrelic/d6b36ecf-bf99-475c-938f-370153db8e70)提供高级安全功能，可对高级威胁进行网络检测和响应，包括影子 IT 活动、加密挖掘和 torrent 活动、SSL 密码版本以及托管和非托管主机的到期日期，如 IoT/OT 和 containers。
*   **Kentik** 是网络观察公司。 [Kentik 快速入门](https://developer.newrelic.com/instant-observability/?search=kentik) 帮助网络和开发团队快速识别和排除与网络流量性能和健康数据相关的应用性能问题。
*   **Lacework** 是一个数据驱动的云安全平台，可以收集、分析和准确关联组织的 AWS、Azure、GCP 和 Kubernetes 环境中的数据，并将其缩小到少数重要的安全事件。 [花边快速入门](https://developer.newrelic.com/instant-observability/lacework/8a7a7220-e8ec-4959-b35d-0fe082be8039) 在可观察性和安全团队之间架起了一座桥梁，并与 New Relic 的数据库集成，直接在 New Relic One 中显示安全事件和警报。
*   **趋势科技** 是全球网络安全的领导者。 [趋势科技 Cloud One quickstart](https://developer.newrelic.com/instant-observability/trendmicro-cloudone-conformity/c3ae3b79-260a-436f-ba25-1d97c61da3ad) 将云安全态势管理(CSPM)数据从 compliance 吸收到 New Relic One 中，将其与工作负载遥测数据关联起来，提供人工智能支持的可视化和快速洞察。这使得安全和云团队能够立即采取行动改善其安全性和合规性状况。

“云安全与可观察性有着相同的核心需求——组织需要速度、自动化和环境。Lacework 总裁 **和 CRO 安迪·拜伦** 说:“无论你是主动识别漏洞，还是试图找到漏洞的根本原因和影响，你都需要尽可能地简化。”。“我们的云安全解决方案与 New Relic 的全面可观察性功能相结合，为我们的共同客户提供了开发运维与安全用例之间无缝一致的体验。我们一起弥合软件交付和安全性之间的差距，提供更深入的可见性、共享的上下文和效率，同时不抑制您的开发速度。”

作为 New Relic One 平台的一部分，New Relic Instant Observability 现已正式推出。用户今天可以浏览 [新遗迹 I/O](https://newrelic.com/instant-observability) 中的所有快速入门。更多信息，请访问[【newrelic.com】](https://newrelic.com/)或 [阅读我们的博客](https://newrelic.com/blog/nerdlog/instant-observability-quickstarts) 。
