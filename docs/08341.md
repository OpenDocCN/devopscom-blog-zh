# 如何成功扩展敏捷

> 原文：<https://devops.com/how-to-successfully-scale-agile/>

敏捷已经证明了它对不同行业和地域的企业的价值。事实上，疫情将其提升为组织成功和生存的绝对必要条件。世界各地的商业领袖积极投资于[敏捷](https://devops.com/?s=Agile)转型，他们争先恐后地通过新的混合和远程劳动力来满足不断变化的客户需求。毫不奇怪，他们越快大规模采用敏捷方法，它为他们带来的价值就越高。[研究表明](https://www.infosys.com/services/agile-devops/insights/agile-radar-2021.html)敏捷交付的价值有可能超过同行 63%。

美国企业现在需要的是移动正确的敏捷杠杆来最大化业务成果。同一项研究[提出了实现这一目标的议程](https://www.infosys.com/services/agile-devops/insights/agile-radar-2021.html):

### 绘制客户旅程

这包括评估公司的产品服务于什么目的，以及价值如何通过围绕它的系统流向客户。通过识别价值流的关键路径和在价值节点区分敏捷投资的优先级，客户价值的数量和速度都会受到影响。例如，在高盛的 Marcus by Goldman Sachs，每个团队，无论他们属于哪个部门，在回答关于他们做什么的问题时，都有一个明确的客户焦点:支持客户入职、改善客户服务体验或帮助客户密码之旅。

### 利用客户洞察

这可以通过 A-B 测试技术和分析平台来实现，以挖掘导致客户流失的态度和动机，建立忠诚度，并将情绪转化为行动。这包括通过调查、讨论收集客户反馈，对于数字渠道，还包括从渠道本身获得的洞察力。此外，随着客户的加入，从设计到运营，这一杠杆支持不同的观点。例如，卡夫亨氏利用雪花的平台利用数据为消费者带来新的数字体验。雪花为他们提供了实时感知、响应和发展的能力，其洞察力可以让他们与客户进行更有意义的互动。

### 围绕产品而不是功能组织团队

这包括汇集技能和能力，在市场上推出颠覆性产品，赢回市场份额。以产品为中心的敏捷价值交付在涉及内部团队之间多次交接的场景中尤其有用，这可能导致延迟和孤立的开发。例如，戴姆勒借鉴 swarm 的原则，从不同部门引入专业知识，并创建跨职能团队，在开发新产品时进行合作。

### 提高劳动力技能

对于组织来说，在自主的、跨职能的团队中提升员工的技能以实现持续交付、战略愿景和技能一致性是非常必要的。满足不断变化的客户需求所需的快速开发通常会造成需要弥补的技能差距。西门子就是一个很好的例子。他们加快了 2020 年的数字化转型之旅，并意识到轻松获得技能提升和再技能提升机会对每个员工来说是多么重要。为了保持领先，他们为员工部署了一个灵活的数字学习平台，同时利用并简化了他们现有的学习生态系统。

### 采用最小可行产品(MVP)方法

该杠杆评估实际价值交付，验证假设，并根据需要实现快速过程修正。预算总是有限的，而敏捷 MVP 方法还可以通过更快的交付、优化的流程和增加的灵活性来最大化投资价值。最近的一个例子是英国石油公司在开始集成能源即服务(EaaS)产品时采用的方法，该产品将为客户的能源资产和服务提供端到端的管理。他们的第一步是在创新中心/工作园区试点该计划，然后探索在工业和商业园区更广泛地管理能源的机会。

### 扩展敏捷工作场所

公司需要通过为会议、销售执行和 IT 支持以及高级协作工具创建远程模拟来扩展和适应混合现实的敏捷工作场所概念。一个很好的例子是通用电器；在疫情的最初阶段，他们通过遍布全球指挥中心和服务台的自动化驱动的敏捷服务支持，加速了工作场所的发展。

### 利用云增强敏捷性

在没有可靠的云基础设施的情况下扩展敏捷是成功的一半，尤其是当公司想要用业务弹性来支撑速度和效用的时候。事实上，最优秀的公司会从云每年为各行业和地区创造的 4140 亿美元利润中分得一杯羹。
