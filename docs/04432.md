# DevOps 标准化面临的挑战

> 原文:[https://devops.com/challenges-devops-standardization/](https://devops.com/challenges-devops-standardization/)

许多组织在企业范围内采用 Devops 的过程中仍然面临着挑战，特别是在工具、不可重复的流程和即席实践以及文化障碍方面没有标准化的时候。作为 DevOps 转型之旅的一部分，以下是标准化方面的一些挑战以及在 DevOps 中实现标准化的可能解决方案。

## 商业案例

在多个客户团队中进行的一项调查显示，由于缺乏端到端协调、手动干预、传统做法、不一致的环境、更长的配置时间以及其他一些因素，他们在工具标准化和简化流程差距方面面临挑战。综上所述，它们使得整个企业的开发运维标准化成为必要，以使其可重复、可预测且一致。

## 开发运维标准化面临的挑战

### 工具挑战

1.  使用多个工具进行评估，工具中有一个或多个重叠的功能，评估/验证概念需要学习曲线，选择工具时要有短期目标，而不是长期目标。
2.  对工具的了解有限，从特定领域自动化的角度来看，导致自动化孤岛化，并使用手动的临时流程。
3.  缺少与 CD 管道中使用的大多数工具的现成集成，团队投入时间构建定制集成。

### **流程挑战**

1.  没有中央 DevOps 团队可以评估和推荐要使用的正确工具集，各个项目团队/小组试图设置他们想要的工具并为他们的团队服务(开发/QA/运营)，不清楚可跨小组共享的基础架构的可用性。
2.  没有定义的过程框架来详细说明 DevOps 中的不同原则，包括需要什么活动和方法来移动到期望的状态，使得团队在没有结构化方法的情况下实现他们期望的自动化状态。
3.  缺乏关于为每个开发运维功能定义正确 KPI 的知识，以及如何衡量指标以确定开发运维自动化流程的效率和确定优化范围的知识。
4.  SME 缺乏在开发和运营团队之间流动的应用程序、应用程序堆栈和环境管理方面的知识，无法实现标准化流程。
5.  多个供应商正在管理一个应用程序的不同阶段，每个供应商都遵循他们自己定义的开发生命周期过程。

### 文化挑战

1.  如果没有一个中央机构(一个中央 DevOps 团队)在整个企业中推动这些实践，那么对 DevOps 实施的度量、流程、实践、治理和组织级控制的标准化将是一项挑战。
2.  团队在孤岛中运作，遵循他们自己的 DevOps 实践，在没有整体视图的情况下在孤岛中思考，导致不一致的破碎流程。
3.  团队仍然缺乏 DevOps 中关于 CI、持续测试、持续部署和操作中要遵循的工具、实践的培训。缺乏 DevOps 熟练资源/专业知识给实施端到端自动化带来了挑战。
4.  开发运维被认为是一个概念，每个组织的每个开发运维实施都是独一无二的，并且没有采用开发运维的标准方法，这导致了从哪里开始以及如何开始的困惑。此外，在对工具进行标准化的同时，还存在利益冲突和来自多个利益相关者的影响力。

## 建议的解决方案

### DevOps 采用框架(DAF)

DevOps 采用框架(DAF)是一个全面的框架，包括尽职调查、工具支持和解决方案加速器，旨在创建系统且规划完善的过渡路线图，实施连续交付工具链，并克服 DevOps 实施中的常见挑战。

### 选择正确的工具和堆栈

工具的选择应该基于需求和最适合需求的东西，根据所附白皮书中讨论的各种因素来决定，并使用一种方法，例如为开箱即用功能和不同因素/约束给出的分数。

### 具有基于模板的基础设施和工具链编排的 DevOps 自动化平台

DAAS (DevOps 即服务)是一个自动化平台，它作为一个中央平台来推动工具使用的标准化、调配基础架构、工具链、管理虚拟机的生命周期、引入中央权威和治理、定义 KPI、监控和衡量指标、监控端到端管道以提供实时分析、跨多个团队提供自助服务功能、引入审计和合规性、持续改进以及推动整合和标准化。

## 未来方向

*   更加注重开发运维指标的标准化，通过分析和监控尽早发现瓶颈，并通过交付渠道将大数据用于预测性分析。
*   整个企业快速采用 DevOps，利用支持端到端管道的自助式 DevOps 自动化平台，实现规模、合规性和利用率的标准化和整合关键因素。

## 结论

跨企业需求实现 DevOps 采用的关键成功因素侧重于中央 DevOps 团队推动 DevOps 采用，通过 DevOps 采用框架实现治理，通过 DevOps 自动化平台实现流程整合、指标和工具标准化。
