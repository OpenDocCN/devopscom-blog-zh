# Alert Logic Cloud Insight 为您的云基础架构带来基于云的安全性

> 原文:[https://devo PS . com/alert-logic-cloud-insight-将基于云的安全性引入您的云基础设施/](https://devops.com/alert-logic-cloud-insight-brings-cloud-based-security-to-your-cloud-infrastructure/)

DevOps 背后的主要驱动力之一是其流动性。从创建应用程序的开发人员到部署和管理应用程序的 IT 管理员，再到下载和使用应用程序的个人，这是一种多米诺骨牌效应。传统安全无法跟上步伐，因此 [Alert Logic 创建了 Cloud Insight](https://www.alertlogic.com/blog/vulnerability-management-in-the-cloud-a-fresh-approach/)—一种针对运行在亚马逊 Web Services (AWS)上的云基础设施的云原生漏洞和配置管理解决方案。

我以前写过关于[持续不断的 DevOps](https://devops.com/2015/04/22/the-continuous-continuousness-of-devops/) 的文章。在静态基础设施上，任何曾经是手动任务的事情，现在都实现了第 n 次自动化。“持续的开发和持续的测试导致持续的部署和持续的交付，这需要持续的支持。持续的监控产生持续的集成和持续的变化。”

所有这些也意味着，在不断变化的环境中，要跟上当前的配置或准确评估漏洞几乎是不可能的。如果您的整个云基础架构只需按一下按钮就可以改变，那么您还需要部署安全控制措施，这些措施同样能够自动进行，并且能够随着变化而变化，并提供持续、实时的安全性。传统的安全解决方案根本行不通。

Alert Logic 同意这一点，这也是它开发 Cloud Insight 的原因。Alert Logic 首席执行官 Gray Hall 在发布云洞察的[新闻稿中解释道:“在保持云敏捷性的同时管理安全性，这与其他任何环境都有着根本的不同。借助 Alert Logic Cloud Insight，我们提供了一种创新的解决方案，为企业提供了对运行在 AWS 上的工作负载的额外可见性，这是一种可操作的智能，可跨实例和应用程序识别和消除风险，同时不会降低业务速度。”](https://www.alertlogic.com/press-releases/alert-logic-launches-cloud-native-vulnerability-and-configuration-management-solution-for-amazon-web-services/)

Cloud Insight 会自动跟踪 AWS 平台内的更改，包括添加、移动和删除。它持续识别动态环境中的威胁、漏洞和潜在风险。警报逻辑使用 AWS APIs 与云基础架构集成，以检测任何更改，并检查新的、更改的或修改的资源是否存在漏洞或配置问题。

Cloud Insight 生成一个按优先顺序排列的补救措施列表，以便组织能够实时了解发生了什么故障以及面临的风险是什么，并且 it 可以专注于解决问题并保持强大的安全态势，而不是只是努力跟上不稳定环境中的漏洞评估。

AWS 全球合作伙伴生态系统副总裁 Terry Wise 表示:“安全即服务解决方案构建于 AWS 之上，旨在补充 AWS 的本地安全功能，使客户能够保持最高级别的保护，同时实现 AWS 云的敏捷性、灵活性和成本节约。”

Alert Logic Cloud Insight 已经过预先批准，可在 AWS 上运行。您可以使用 Cloud Insight 在 AWS 内进行漏洞扫描和渗透测试，而无需任何额外的许可。Cloud Insight 还包括面向客户的 API，可用于与其他 DevOps 工具集成，并且可以集成到您现有的 DevOps 管道中。

Alert Logic Cloud Insight 的定价类似于 AWS 定价模式。它基于给定一天的主机或受保护实例的数量，以按需购买的方式运行。每天每个实例向客户收取 1 美元的费用。您可以申请 Alert Logic Cloud Insight 的[免费试用，以确定它对您的组织来说是否是一个好的解决方案。](https://www.alertlogic.com/products-services/cloudinsight/)
