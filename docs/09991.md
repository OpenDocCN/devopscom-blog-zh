# 为什么 SaaS 厂商需要回馈开源

> 原文：<https://devops.com/why-saas-vendors-need-to-give-back-to-open-source/>

没有多少科技组织不利用开源技术。拒绝进口软件的公司将依靠免费的库工具或[代码库](https://devops.com/new-survey-highlights-codebase-size-problems/)。开源是软件传播的最大和最强大的机制之一。然而，开源正处于危险之中。

### 开源开发的两个方面

近年来，开源工具的普遍存在和主导地位已经导致公司集体采用它们。像 Elasticsearch、Spring 和 Akka 这样的产品作为开源、免费的工具非常受欢迎。但不幸的是，通常只有一半的开源哲学得到了尊重。

### 开源哲学

那是什么意思？开源取决于两件事。客户使用软件，这些客户为代码库做出贡献。这是一个由志趣相投的人组成的反馈循环，他们对构建和使用优秀的产品有着共同的兴趣，并乐于贡献他们的时间来实现他们的集体目标。该协议推动了过去二十年中一些最激动人心的软件成就。

唉，虽然使用开源软件变得很普遍，但贡献的受欢迎程度并不一样。突然间，专注的开发人员组成的小团队被大企业的需求淹没，这些需求将“开源”等同于“免费”。这导致开源项目的所有者，如 Elasticsearch、MongoDB 和 Akka，重新评估他们的地位，并转向越来越流行的 [source available](https://en.wikipedia.org/wiki/Source-available_software%5C) license。

### 开源社区面临越来越大的风险

如果用户为他们所利用的代码库做出贡献，那么这些代码库就可以繁荣发展，没有人需要担心订阅费或罚款。此外，当用户依赖一个软件时，他们可以保持确定性，他们没有做任何非法的事情。如果没有工程界的贡献，开源项目的商业和实践基础很快就会瓦解。

### 那么，SaaS 供应商为什么要回馈呢？

SaaS 解决方案占据市场的特定角落，帮助公司解决交叉问题。比如可观察性。这是一个重要的细节，因为许多开源工具也在这个领域运行。举例来说，SaaS 的供应商经常与开源的本土供应商竞争。许多 SaaS 解决方案实际上将其核心基础设施建立在相同的开源软件上。

这意味着 SaaS 提供商经常与他们用来构建产品的开源解决方案直接竞争。

### SaaS 供应商依赖开源软件

支持开源软件重要部分的社区，如 Apache Kafka 或 OpenDistro，正在帮助世界各地的企业进行创新。这种创新的持续步伐与它所依赖的开源工具的持续增长和发展密不可分。

对 SaaS 的供应商来说，唯一明智的做法是尽一切努力为开源空间做出贡献。他们不需要放弃自己的独特功能，也不需要发布自己的专有软件，只需要简单地维护补丁、提高性能、增加兼容性，甚至纠正糟糕的文档，所有这些都是对整个 SaaS 生态系统的贡献。

### 这些贡献将孕育专业化

使用开源软件的一大挑战是找到理解它如何工作的工程师。许多工具的即插即用特性意味着用户永远不需要钻研库或应用程序的细节。然而，这是有代价的。用户通常不知道为什么软件会崩溃或者行为不可预测。

如果一个 SaaS 供应商开始对他们所依赖的开源工具的核心做出贡献，他们不仅会使他们的产品和开源社区受益，而且他们还会在他们的团队中播种专业知识。这种专业知识意味着他们能够以新的、令人兴奋的方式利用开源特性，从而获得超越竞争对手的洞察力和优势。

### 开源贡献具有商业意义

当开源工具顺利工作时，SaaS 社区就会繁荣起来。相反，如果像 Kafka 或 OpenDistro 这样的基本开源工具开始动摇，那么 SaaS 社区的成功也将面临风险。它们高度相互依赖。这意味着对核心开源工具的贡献就是对 SaaS 产品稳定性的贡献。此外，工程团队中不断增长的专业技能意味着，当需要利用新的开源特性时，您的开发人员可能拥有其他公司所没有的技能和洞察力。这是一个明显的商业优势，所有这些都是为了做正确的事情，保持自由、开源软件的梦想。
