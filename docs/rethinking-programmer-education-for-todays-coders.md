# 反思当今程序员的程序员教育

> 原文：<https://devops.com/rethinking-programmer-education-for-todays-coders/>

低代码和无代码应用程序的激增正在改变软件的前景。在过去十年中，这些领域的发展使员工能够以数字世界的速度制定解决方案，而不管他们的技术背景如何。

这就引出了几个问题:企业真正需要什么样的程序员？他们还应该依靠在四年制大学接受过正规培训的程序员吗——你可能会称之为“白领”程序员？或者，相反，是拥有自学技能的蓝领程序员还是那些从训练营学到的程序员更有价值？什么能让组织获得正确的编程技能组合？

简而言之，所有的程序员都有巨大的价值。总的来说，蓝领程序员毕竟在计算环境中连接事物，而白领程序员创造被连接的事物。低代码的本质是它允许更多的人将事物联系起来，并利用他们的创造力变得更有生产力。但是它并没有消除对专业编程和工具的需求。

在他们的核心，低代码和无代码是关于民主化。它们允许没有技术技能的人创造性地工作，本质上使每个人都成为程序员。与早期计算一代类似的是 Excel 电子表格，它让没有编程背景的人执行计算任务，并最终从简单的格式化发展到成熟的编程。但是在某种程度上，完美的编程技能仍然是必要的，无论无代码或低代码可以多么容易地创建这些元素。

在基于云的计算环境的飞速发展中，新服务不断被开发和部署到边缘，值得考虑白领和蓝领程序员之间的区别以及哪种教育适合他们。

## 四年制大学对新兵训练营

对于旁观者来说，程序员的职业生涯似乎令人生畏。当这些工作涉及 C++和 Java 等复杂语言时尤其如此，这些语言通常用于大型项目，这些项目可能需要数年的开发时间，并涉及数百万行代码。其中一些仍然是正确的，但是多年来编程向需要更少代码的语言(如 Python)和特定领域语言(DSL)的发展也开始降低编程的门槛。

的确，你从来不需要四年的计算机科学学位就能成为一名程序员。事实上，许多从大学辍学或者根本没有上过大学的人已经为这个行业做出了巨大贡献。然而，大学确实提供了理论和算法的基础，这些基础一直为程序员提供良好的服务，并使他们能够扩展到新的领域，如人工智能或其他学科，如生物信息学。对他们来说，编码训练营可以提供 DSL 或 Rails 或 React.js 等框架的强化培训，这些培训可以使公司的计划受益，给蓝领程序员提供白领程序员可能不具备的实用技能。它们还吸引了其他学科(包括人文科学和科学)的四年制学位的转行参与者，这为该行业带来了急需的新思路。

虽然每种方法都是有效的，但四年制院校和训练营也有其缺点。大学深入研究软件开发理论，但通常不强调工作的关键方面，如团队合作、测试和敏捷过程。他们也不太关注像云计算这样的核心业务领域。通过瞄准特定领域和新兴技术和语言，新兵训练营可以帮助人们获得实习或入门级工作，但不提供更广泛的编程理论知识。他们可能会有失误，其中一些人在支付和工作安置方面使用粗略的做法。

白领和蓝领培训中的固有弱点可以通过学徒和实习来解决，但只能在一定程度上解决。问题仍然是:谁将在现代开发和编码实践中培训低代码程序员？许多公司抛出诸如敏捷和 CI/CD 之类的术语，但它们通常只是旧的、低效的实践的新标签。公司将需要白领程序员来提高蓝领程序员的工作效率。

## 面向大众的编程

无代码软件是一个很好的使能器，允许没有正规教育或经验的人精通编程环境。另一方面，对于接受过正规培训的人来说——无论是正规教育还是新兵训练营经历——低代码简化了他们的工作，让他们有时间专注于更复杂的项目。但是不管是不是程序员，如果他们希望他们的软件可靠和有用，无代码和低代码解决方案的用户需要理解的不仅仅是部署和测试。

专业开发人员可以通过创建和维护用于构建、测试、存档和部署低代码软件的管道来发挥作用；他们将需要构建新的工具来适应低代码框架。在熟悉当前开发技术的同时，他们可以成为不涉及编码的基本计算实践的老师。

白领和蓝领程序员之间合作的、富有成效的关系对于推动软件开发至关重要，因为这将允许双方继续获得低代码之外的新技能和经验，低代码只是编程的产物，无论它可能多么普遍。

## 建立联系

今天，我们正在进入一个几乎每个人都需要编码的商业世界，这使得低代码和无代码框架成为必要，特别是对于没有经过正式培训的用户。然而，职业程序员的重要性仍然存在。

我们很可能会看到为解决特定问题而创建的 DSL 激增，最终将向通用编程语言发展。程序员将需要构建 web 框架、云能力等等，包括从 web 小部件到让用户工作的高级工具的一切。随着越来越多的设备上线，越来越多的连接产生，世界变得越来越依赖自动化，这可能是我们开始满足更多人编程需求的唯一方式。
