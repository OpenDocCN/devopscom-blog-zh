# 用 AI/ML 驱动软件交付自动化

> 原文:[https://devo PS . com/driving-software-delivery-automation-with-ai-ml/](https://devops.com/driving-software-delivery-automation-with-ai-ml/)

如果我们可以使用人工智能(AI)引擎来预测下一行要构建的软件代码，会怎么样？如果机器学习(ML)可以触发自动化测试用例的创建来验证我们的解决方案，会怎么样？

多年来，自动化一直是信息和通信技术(ICT)行业的焦点，有些经验也适用于软件开发人员社区和其他垂直行业。如今，运营和软件交付流程的执行更加简化，如应用开发中的持续集成、持续交付和持续部署(CI/CD ),继续推动成本降低到新的水平，使产品能够更快地上市和更新。

然而，随着数据量的增加，会出现更复杂的生态系统，并需要更高级别的自动化。超自动化，或使用 AI/ML 进一步简化流程，可以建立在这些 CI/CD 流程的基础上，彻底改变多个行业交付软件和优化运营流程的方式。

## 当今网络的复杂性和挑战

在电信行业，来自超大规模和其他创新参与者的干扰，以及 5G 技术投资水平的不断提高，正在迫使通信服务提供商(CSP)采取行动。这导致他们转向自动化来提高运营效率和创新水平，以满足频繁更新的需求。

然而，挑战仍然以下列形式存在:

*   **总拥有成本和运营成本:**随着升级现有网络的需求越来越频繁，需要包括 AI/ML 和超自动化在内的额外自动化水平来管理和维护当今复杂的网络。T3】
*   **上市时间**:在竞争日益激烈的市场中，通信服务提供商能够对市场需求和/或新的营销理念做出反应，并迅速向最终用户和企业客户推出这些需求和理念，这是一个关键的竞争优势。T3】
*   **客户体验:**数字化业务流程、提供更高水平的自助服务以及寻求优化客户整体旅程的方法仍然是一项重要需求。
*   **员工工作效率:**为了增加收入、更有效地发展网络并改善最终用户的客户体验，电信运营商还需要提高员工的工作效率。他们可以通过让员工参与解决更大的问题来做到这一点。因此，最简单和最重复的低级任务可以自动化。

## 人工智能和人工智能的力量

自动化之旅始于将复杂的流程分解为较小的部分，并使用脚本、操作手册自动化(RBA)和/或[机器人流程自动化](https://devops.com/?s=robotic+process+automation) (RPA)来自动化离散的部分。T3】

随着人工智能通过机械和计算过程模拟人类智能以及人工智能使用从数据中学习的算法进行预测和决策的引入，CSP 现在可以将自动化和超自动化带到下一个水平。

电信软件解决方案的交付和维护的潜在用例是无限的，其中许多经验教训可以应用到其他领域。一些例子包括:

### 针对云原生解决方案的自主故障和异常检测

今天的网络解决方案部署在容器和微服务上。随着成千上万的工作负载并行运行，识别问题和保证服务质量变得越来越具有挑战性。容器故障会导致严重的服务中断。事实上，单个服务失败或异常行为可能会对整个系统产生蝴蝶效应，并对客户满意度产生负面影响。

基于人工智能的主动监控解决方案可以持续搜索环境中的异常行为。不同的容器指标和日志可以主动自动分析和检测问题，并帮助为应用程序工作流找到正确的纠正措施。这是基于识别与 pod、工作负载和命名空间相关的容器编排指标中的异常。这些技术支持更可靠、更可预测的应用运行时环境，提供更高的服务质量，更快地识别和解决系统故障，并最终提供更好的整体客户体验

### 机器学习触发的闭环自动化

在整个数据中心内管理大型计算、存储和网络资源集群，以支持定制工作负载的部署。由于在虚拟化基础架构和云基础架构上运行的应用程序的固有特性，使用情况会有高有低，需要不断部署基础架构组件或重新配置群集，以确保经济高效的服务可用性。

基于机器学习的解决方案可以检测基础设施中的异常。具体来说，通过闭环自动化，企业可以在当前设置的基础上引入一个智能层，以监控应用性能和基础设施指标，并识别需要系统立即关注的异常情况。如果发现这些问题，它们可以触发向基础架构自动添加容量。当异常平息下来，并且利用率能够由原始容量处理时，该解决方案可以持续监控指标，以回滚虚拟或云原生功能。这允许高效和全自动地使用网络容量，最大限度地减少人为疏忽。

### 智能代码建议

对于程序员来说，工具可以减少软件开发工作量的最明确的方式之一是通过代码补全。人工智能驱动的代码完成是一种访问函数描述及其参数列表的便捷方式。这些解决方案还通过减少键盘输入和名称记忆的必要性来加速软件开发。它们还允许用户不那么频繁地参考外部文档，因为交互式文档以工具提示的形式动态出现。人工智能驱动的代码完成也可以使用自动生成的内存数据库，其中包含基于机器学习的类、变量名和其他构造或参考，以开放代码库。

### 人工智能驱动的测试推荐

大量时间花费在理解解决方案需求和确定测试的关键场景上。这之后通常是手工测试计划的创建和执行。为了解决这个问题，可以应用 AI/ML 技术来识别在软件开发阶段的每个周期中需要做什么测试，包括自动化所需文档的分析。这可以用来增加总的测试周期时间和提高生产率。

### 人工智能支持的测试自动化

API 测试对于验证应用程序逻辑至关重要，并且需要超越 GUI 层来验证应用程序的核心。这意味着复杂性，因为协议通常难以理解，API 清单通常很大，API 调用的顺序对于成功的测试规范和执行至关重要。利用 AI/ML 和自然语言处理技术，可以自动分析解决方案测试计划，以获得关于使用什么测试框架的建议，识别和建议每个测试用例的参数，识别 API 调用序列，并基于可重用功能自动开发测试用例。这可以提高服务的整体质量，简化变更管理，提高测试团队的生产力，并提供更可靠的测试用例。

### 测试失败分析

在自动化测试用例的执行过程中，可以在各种日志中观察到多个失败。手动分析日志并将它们与应用程序问题关联起来可能非常耗时。此外，识别影响系统稳定性的不可靠测试是复杂的。

AI/ML 技术，如矢量化、K-means 聚类或主题建模，可用于自动收集跨应用程序的测试日志，并标记故障，包括集群失败测试的类型。测试主题专家(SME)被提供了一个失败分析报告，该报告从应用程序的许多日志中自动总结了问题，简化了测试自动化区域或脚本中问题的识别。这些技术可以显著提高测试生产率和总的故障排除时间。

### 加速根本原因识别和分析(RCI&A)

当电信网络中出现问题时，往往会引起连锁反应，影响其他集成服务并引发警报风暴。在这种情况下，[根本原因](https://en.wikipedia.org/wiki/Root_cause_analysis)识别会变得复杂而繁琐。需要事件聚合和同步来降低环境的复杂性和变化。T3】

机器学习加权分析技术可用于确定单个 IT 指标在影响托管应用和服务的整体健康和性能方面的影响，以便运营网络的团队可以更容易地实时查明任何服务影响的根本原因。加速 RCI 和分析的有效人工智能/人工智能技术可以显著缩短时间。

### 人工智能/人工智能推动运营创新和市场灵活性

在电信行业，AI/ML 驱动的用例有很大的潜力来进一步改善云原生之旅，并为未来的解决方案实现类似 IT 的软件生命周期管理流程。

更广泛地说，高度自动化和自我修复的网络将推动更高水平的效率和客户满意度。这包括用更少的资源进行更快的测试；更高效地利用网络资源和云基础设施；更快的上市时间，每年发布更多具有类似容量的软件，以及更高的质量，以更快、更灵活地实现 CSP 功能的货币化。
