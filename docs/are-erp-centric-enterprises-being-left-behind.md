# 以 ERP 为中心的企业被甩在后面了吗？

> 原文：<https://devops.com/are-erp-centric-enterprises-being-left-behind/>

许多大型企业公司都有依赖定制 ERP 软件应用的部门。这些系统适应从库存管理到订购功能、会计程序到 CRM 平台的重要业务任务，所有这些都是为满足企业的特定需求和满足每个独特组织的 KPI 而定制的。这些应用程序倾向于维持等级、补丁和返工，有时导致几十年中的多次重新设计，并且这些通常没有被适当地记录。更糟糕的是，这些 ERP 软件系统往往是用过时的编程语言编写的，如 SAP 的 ABAP 和 Oracle 的 PL/SQL，这使得它们很难更新。

例如，在最近 ASUG 对 110 家 SAP 公司的调查中，91%的受访者表示他们使用定制代码，而 [这些应用程序中有 41%是至少六年前创建的](https://www.pillir.io/asug-market-research-custom-code-and-digital-transformations) 。这些公司中的大多数表示，这些定制代码环境的存在已经成为数字化转型的障碍，并阻碍了创新。

这些情况造成了企业市场中常见的臃肿且过于复杂的遗留环境。广泛分布的定制核心业务系统让企业的数字化转型变得尤为艰难。他们需要大量的开发人员资源、过多的时间投入和大量的费用来为云环境拆除和重新设计这些定制系统。

## 缺乏开发人员——以及开发人员的兴趣

美国劳工部 2020 年 12 月的统计数据表明，全球将短缺 4000 万技术工人，进一步放大了这些问题。这个数字预计到 2030 年将增长到 8520 万。此外，进入市场的新开发人员倾向于更现代的编程语言，如 JavaScript 和 Python，而不是想修改大多数 ERP 解决方案中使用的古老代码，如 ABAP。

因此，在将任务关键型软件应用升级到云或更现代的架构时，一系列垂直行业的大型企业都落后了。这些组织清楚地意识到，他们可以从数字化改造其系统中获得显著的生产力优势，例如，可以通过移动设备访问库存数据或在基于云的平台上从任何地方共享库存数据。然而，这些大型组织的现代化努力停滞不前。一些公司甚至放弃了他们目前的系统，开始从头开始重新编程他们的软件应用程序——这是一项非常昂贵、漫长和费力的努力，也充满了风险。

## **利用低代码简化企业 IT 环境**

在解开这些过度紧张的环境方面已经显示出进展的一种方法是通过低代码集成来应用自动化。理想情况下，有效的低代码开发解决方案将自动将现有业务流程转换为现代的云原生应用程序，使组织能够维护多年来构建到这些应用程序中的业务逻辑，同时大幅减少时间投入、成本和与传统的长形式集成相关的潜在中断。例如，在 SAP 环境中，使用低代码快速应用程序开发解决方案可以将应用层从核心 SAP 系统中分离出来，在保留“干净的”SAP 核心的同时，自动完成大部分迁移。

在持续不确定的经济形势下，许多组织都面临着控制预算的压力，这些大规模公司正在寻找在更可控的时间表内实现运营现代化的方法。当企业使用低代码解决方案时——此外，不必寻找这些更古老语言的开发人员——他们可以自由地以更具增量和成本效益的方式进行数字化转型。

例如，SAP 解决方案的 1200 万用户被迫升级到其产品的现代化版本 S/4HANA。然而， [美国 SAP 用户组(ASUG)研究](https://s3.us-east-1.amazonaws.com/asugv5-assets/archive/uploads/images/2021-POSAPC-Infographic-All-Member.pdf) 显示，尽管当前流行版本即将寿终正寝，但迄今为止只有 33%的组织接受了这一挑战。这是因为缺乏低代码开发解决方案，公司被迫一次性转换和重建所有业务流程。很少有组织有资源来承担这样一项激烈而又有风险的任务。

快速应用程序开发计划为那些希望实现业务现代化的公司搭建了一座桥梁，允许他们根据自己的计划逐步升级应用程序。低代码可以成为技术债台高筑的企业的催化剂，允许他们简化复杂的遗留环境，并着手创新他们的系统。
