# 持续测试实践差距评估

> 原文:[https://devo PS . com/continuous-testing-practices-gap-assessment/](https://devops.com/continuous-testing-practices-gap-assessment/)

持续测试是 DevOps 的重要组成部分。为了从持续测试中获得最大价值，使用推荐的最佳实践是很重要的，包括人员、过程和技术。差距评估是一种有效评估组织的开发运维实践并确定改进策略的好方法。差距分析为制定改进持续测试的策略和路线图提供了有价值的输入。

我的幻灯片“持续测试的差距调查、评估和分析”解释了我的持续测试差距评估工具如何大大加快持续测试的差距调查、差距评估和差距分析过程。

澄清这个博客的一些定义是很重要的，因为我发现不同的人和组织使用这些术语是不同的。在这个博客和我的咨询工作中，我使用了以下定义。

**差距调查**是一个发现工具，用于收集关于持续测试等主题的实践的当前状态的信息。
**差距评估**是确定某个主题的当前实践和推荐实践之间的差异(差距)的过程。差距评估也用于指本清单中的所有活动。
**差距评估** **研讨会**是在差距分析之前验证差距评估的会议。
**差距分析**是确定减少差距评估中发现的差距的优先级的过程。
**差距评估工具**是帮助执行差距调查、差距评估和差距分析的软件。

我定义并使用差距评估的七步流程。

步骤如下:

1.**挑一个题目来分析。相同的过程和工具可以同样适用于几乎任何主题，比如持续测试。
2。**确定该主题的推荐做法。**一些值得进行差距评估的事情，比如持续测试，通常会相当复杂，如果没有上百个实践，也有几十个。将实践分类成组(我称组为“支柱”)，使实践列表更容易理解和有意义。要做到这一点，需要一些研究和发现工作。例如，我有一个 DevOps、持续测试、DevSecOps 和 SRE 以及其他一些主题的实践和支柱的数据库。
3。**将第二步的实践输入差距评估工具**。该工具可以是您创建的简单调查工具、商业调查工具或定制工具。
4。**确定谁需要参与差距调查**。
5。**使用差距调查**收集数据。
6。**执行差距评估**。
7。**执行差距分析**。**

### 第四步:确定差距调查的对象

为了使差距分析变得全面，需要对受持续测试实践影响的角色中的人进行调查，或者至少代表他们，以确保他们的观点被包括在内。以下是通常包含的角色示例。

商业领袖——因为他们影响持续测试的文化和预算。开发人员——因为他们需要根据持续的测试实践进行设计。
·项目负责人——因为他们影响产品工作的优先级。
QA 测试人员——因为他们需要根据持续测试实践进行测试。
·运营——因为持续的测试实践会影响运营。
安全性——因为持续的测试会影响安全性实践。

调查可以针对企业中的单个应用程序、一组应用程序或所有应用程序进行。但是，对作为改进目标的组织部分进行差距评估和差距分析是很重要的。

### 第五步:使用差距调查收集数据

差距调查应允许每个被调查人输入重要性水平分数、实践水平分数和对每个实践的评论。所有这些信息对于差距评估和差距分析都是必不可少的。在我的差距评估工具中包含的差距调查中，调查受访者被要求将每个实践的重要性分为 0 =不相关、1 =不重要、2 =很好、3 =重要、4 =非常重要或 5 =关键。实践水平选项为 0 =不确定，1 =很少，如果有的话，2=有时，3 =大部分时间，4 =总是或 5 =我们真的很擅长这个。还应输入与限定或解释为每项练习输入的分数相关的注释，尤其是如果有任何疑问或模糊之处。

### 第六步:进行差距评估

差距评估流程要求收集从调查中收集的所有实践分数，并进行汇总，以计算一组总分数。差距分数是使用一个公式计算的，该公式用相应的重要性等级分数对每个实践等级分数进行加权。可视化的表示有助于识别实践领域和具有最高、最重要差距的单个实践。

无论练习和分数定义写得多么专业，一些人误解并输入他们本来不会得到的分数是很正常的。因此，在进行差距分析之前，确保对收集的数据进行验证非常重要。首选方法是与参与调查的每个角色的主要代表一起召开差距评估研讨会。在研讨会期间，将对调查结果之间偏差较大的每项练习的分数进行讨论，并在必要时进行调整。

### 第七步:进行差距分析

差距分析过程包括提取差距较大的实践，并根据顾问或主题专家确定的解决方案类别对每个实践进行标记和排名。差距分析的结果指出了解决方案策略和实施路线图需要关注的地方，以缩小最重要的差距。
