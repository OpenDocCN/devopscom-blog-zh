# 语言和 DevOps:建议

> 原文:[https://devops.com/languages-and-devops-recommendations/](https://devops.com/languages-and-devops-recommendations/)

根据这些文章中提出的观点，对于在给定的项目中使用哪种语言，我的建议是什么？我总是给开发团队同样的建议:为工作使用正确的工具。我的父亲是一名木工，他把这个概念灌输给了我。我把它带到发展，这是真的。不同之处在于，在现有的 IT 环境中，“合适的工具”是一个比木工复杂得多的问题。你需要凿子还是螺丝刀是显而易见的。Python 或 Go 是否是合适的工具……不太合适。需要考虑的事项:

### 该工具有广泛的开发者基础吗？

这是这个列表中最简单的问题。你能让人们用这种语言工作吗？我在一个仍有 RPG III 代码在大型机上运行的地方工作。我们曾经开玩笑说，澳大利亚的那两个退休开发人员通过维护它变得富有了——而我们仍然用这种语言做了一些新的开发。我们离开是一个错误的选择。在开发新平台的同时维护旧平台是有意义的。将任何新的功能打包到一个很少有人知道其来龙去脉，甚至更少人接受过培训的工具中？毫无意义。

### 该工具是否专门针对您的目标进行定制，或者拥有针对您的目标进行定制的模块/附件？

虽然在 50，000 英尺的高度，理论上你可以用任何语言做任何计算工作，但这并不意味着你的首选语言是明智的选择。确保工具支持您需要做的事情是最重要的。这是最接近“正确工具”的思维——它能完成工作吗？

### *开发者群体是在增长还是在萎缩？*

虽然这对于生命周期较长的项目更重要，但是您确实希望上面第一点中提到的开发人员基础仍然存在。这也延伸到附加组件。如果你今天选择[ASP.NET](https://dotnet.microsoft.com/apps/aspnet)，明天会是一个可靠的选择吗？(我不是说它不会，在这里，我只是选择了一个框架来说明这一点)。

### 该工具是否如您所愿，或者您是否会绕开故障以达到目的？

老实说，你可以——人们已经这样做了——用 c 写网站。这并不意味着它是正确的选择。与开发实际的网站相比，你最终会花费更多的时间来回避 C 不适合这个任务的事实。评估这种语言，看它实际上能多好地完成你需要做的工作。这与上面的第二点不同——这是“仅仅因为你能，并不意味着你应该”的部分。

### 该工具适合您的环境吗？

这是一个太多人苦苦挣扎的问题。新的开发团队希望带来他们最熟悉的工具，因为它在短期内缩短了开发周期。这里的问题很简单。管理一个有意识地努力尽可能控制我们不断增长的工具、语言和库的环境要容易得多。如果整个环境都是服务器端的节点，仅仅因为有人对它感兴趣而引入 Go 来开发服务器是没有意义的。利用你现有的环境，这样你可以更容易地维护事物。一切都经过更彻底的审查——从包括哪些附加组件到安全性——使用组织最常用的语言；利用这一点。从“我们在 70%的应用程序中使用 Java”开始，然后只有当你朝着放弃 Java 的目标前进时才偏离这个目标(同样，不是说你应该，只是一个例子)。

## **作为一个新 DevOps 的人，该学哪个？**

不久前，对新开发人员的普遍建议是，“学习 Java”它无处不在，无所不能，即使它不是工作的最佳工具。

同样，现在答案是“学习 Python！”在所有的语言中，它是使用最多的语言。我没有数据来支持下面的说法，但我敢打赌大多数开发人员已经在一个或多个项目中使用过 Python，相当多的运营人员也使用过。如果 Java 是过去，Python 就是未来。

也就是说，去那些你真正喜欢工作的公司看看他们在做什么。一般来说，如果他们 70%的代码都是用 Go 编写的，他们就不会雇佣 Python 开发人员。我不建议根据一两个组织的环境来定制您的学习(除非您已经在其中一个组织中工作过)，但是请考虑本系列中列出的流行语言，并首先学习您的目标组织使用的语言。

所有这些语言都有免费/便宜的学习选择。没有人阻止你学习围棋；的确，有些网站的目的是帮助你边做边学。大多数语言都有相同类型的学习，只是实现细节稍有不同。如果这是你学习一门新语言的首选方法，那么有大量的好书可供选择。

最后，学习一门新语言永远不会是一件坏事。它将帮助您了解上面的列表，并在不可避免的“这是正确的工具吗？”中提供有意义的输入对话。

所以，先从 Python 开始，然后再通过其他。附带说明:如果你倾向于 Ops 方面，了解不同的 Ops 工具需要什么，并考虑学习其中的一些。例如，有一个著名的 Ops 平台仍在使用 TCL，我们都知道 CICS。所以要意识到可能需要什么，至少要熟悉这一点。

## **继续摇摆**

现代 IT 环境异常复杂。从权限管理到存储，再到数据重力，再到各种语言和设备，您和您的团队成员都在努力工作。坚持做下去。在新语言对你的工作变得必要之前学习它们，但是永远不要认为你落后了。如果公司还在发展，你就是这份工作的合适人选。
