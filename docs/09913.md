# 开源 ZAP 工具的更新提高了 DAST 的性能

> 原文:[https://devo PS . com/update-to-open-source-zap-tool-improves-dast-performance/](https://devops.com/update-to-open-source-zap-tool-improves-dast-performance/)

OWASP Zed 攻击代理(ZAP)开源动态应用程序安全测试(DAST)工具的更新今天发布，通过采用多线程被动扫描引擎提高了性能。

ZAP 的 2.12.0 版本也增加了对 HTTP/2 的支持，并通过使发现新资源的蜘蛛成为一个现在能够发现更多 URL 的插件，使更新漏洞扫描工具变得更简单。

ZAP 是由开放 web 应用程序安全项目(OWASP)维护的开源渗透测试工具，专门用于测试 Web 应用程序。

![](../Images/12aa372870032980236ec7ab33b3af04.png)

ZAP 的创始人兼首席维护者西蒙·贝内特(Simon Bennetts)表示，最新的更新将使开发者更容易更快地发现他们代码中的漏洞。他说，开发者经常不频繁扫描代码漏洞的原因之一是因为扫描被认为很慢。当然，开发人员在扫描代码之前等待的时间越长，完成扫描的时间就越长。

DAST 工具是任何 DevSecOps 工作流的关键元素，因为它们使开发人员能够像攻击者一样通过前端分析 web 应用程序来查看应用程序安全性中的弱点。这种方法并没有消除对[静态应用安全测试](https://devops.com/the-scariest-things-about-sca/) (SAST)工具的需求，以发现源代码中的漏洞；Bennetts 指出，DAST 工具有助于开发人员更好地了解网络攻击如何针对他们应用程序中的特定类型的漏洞。他补充说，作为一个开源工具，ZAP 旨在通过让开发者更容易下载和安装来消除采用 DAST 工具的障碍。

当然，在一系列针对软件供应链的高调攻击之后，人们对 DAST 工具的兴趣正在上升。随之而来的争论是在多大程度上将应用程序安全责任转移给应用程序开发人员。越来越多的组织开始要求开发人员在其应用程序开发工具中嵌入安全工具。其他人也在他们的 DevOps 工作流中嵌入了各种类型的安全工具，以确保在开发人员未能执行扫描时也能执行扫描。

当然，没有开发人员会着手构建一个不安全的应用程序。Bennetts 说，主要的问题是，许多提供的 DAST 工具不是为开发者设计的。他补充说，相反，它们是为网络安全团队设计的，在应用程序构建完成后，他们可以与开发人员分享结果。Bennetts 说，这种方法在应用程序开发和网络安全团队之间建立了一种功能失调的关系，因为开发人员在最初编写代码后很久才被要求修复漏洞。他指出，在大多数情况下，这些开发人员已经转移到其他项目，因此代码是如何构建的任何背景往往都会丢失。

自然，实现 DevSecOps 最佳实践与文化和所提供的工具一样重要。然而，有一件事是清楚的，开发人员自愿采用的工具将比其他人代表他们选择的工具得到更广泛的使用。在许多情况下，改善应用程序安全性整体状态的最短路径将是开发人员自己可以随时访问的开源工具。
