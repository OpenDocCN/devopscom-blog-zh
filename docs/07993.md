# IT 运营占据中心位置

> 原文:[https://devops.com/it-ops-takes-center-stage/](https://devops.com/it-ops-takes-center-stage/)

当人们谈论过去的 18 个月时，大多数人会记得这是一段混乱的时间，没有什么是确定的。然而，对于许多 IT 运营和网络运营中心(NOC)的领导者来说，这是他们走出黑暗走向聚光灯下的一年。突然之间，谁在维护我们购物、工作、吃饭、生活所依赖的服务变得非常重要——而且很明显，it 运营团队——他们以前只被认为是需要密码重置的号码——一直在维持我们的业务运营。

虽然在疫情之前可能已经有了一些认识，但这种认识肯定在 2020 年期间获得了动力，而且现在没有停止的迹象。

正如索尼 Playstation Network 的高级运营经理 Ben Narramore 在最近的一次客户会议上所说，“今天，我能够与全公司的领导者交谈，而一年前我绝不会与他们交谈。他们不关心运营，不关心 NOC，不关心事件管理，现在他们就像，'哇，这真是太棒了，'”他说。“我现在很兴奋。我们的团队在运营方面取得了进步，不仅仅是解决问题。”

听到这样的观点我很兴奋，因为在过去的 10 年左右的时间里，应用程序开发一直是 IT 的黄金孩子。你编写代码的速度越快，业务发展的速度就越快。多年来，每一美元和每一项资源都被用于构建令人惊叹的应用程序和服务，而让 IT 运营人员来保持它们的可用性和性能。当然，像 Hulu 和 Expedia 这样的一些公司已经研究出了动态，并建立了一个现代化的优化组织，可以像它建立时一样快地运行，但大多数公司在时间、重点和资源上付出了高昂的代价，以保持他们的服务正常运行。

对大多数公司来说，这就像是一场速度与可用性的战争。当工程能够快速交付业务需求时，他们就赢了。他们不需要在群体之外寻找太多东西，而且——如果它如预期的那样起作用——他们很容易做到。另一方面，IT 运营需要对整个技术环境有一个宽广的视野。他们需要考虑所有服务的可用性，这要求他们能够了解整个企业体系，并经常处理跨网络、硬件、软件和云的停机和事故的下游影响。

现在是 2021 年，战争结束了。事实证明，运营的兴起并不是一场革命。这是关于认可。IT 运营团队需要一个席位。他们终于有机会解释说，他们也希望快速行动，以便与工程合作伙伴一起简化运营、巩固意识、尽可能实现自动化，并帮助支持和实现业务所需的速度。

Jeremy Bragg 是 Abbott Labs 的基础设施自动化架构师。我认为他的角色描述非常引人注目；他是这样解释的:“自动化最有趣的一面可能是我们打破了孤岛。我们不能只是一个服务器团队、网络团队或移动团队。我们与所有团队全面合作，努力推进他们的事业。我们问自己:我们今天在哪里做我们可以自动化的手工工作？”

在未来的五年里，我相信更多的团队将开始利用人工智能和机器学习来为现代 IT 组织设定步伐；更多的日常工作将被自动化，因此他们可以自由地领导创新，而不仅仅是对创新做出反应。

本·纳拉摩尔也对这个话题充满热情。“如果我看看过去十年我的团队，我们是消防员；出于需要，我们擅长于此。现在，我们正处于云计算和人工智能的转折点，这让我们有机会重新评估我们的团队。我对我们的团队如何从解决日常问题发展到着眼未来看待我们的业务感到兴奋。能够自动化手动操作和工作流使我们能够做新的、更令人兴奋的工作，并帮助进行事件管理和问题管理。”

我自己认为这种转变正在发生，当 IT 运营团队走出阴影走向光明，作为战略合作伙伴在谈判桌上占据应有的位置，并帮助推动业务向前发展时，我很高兴能够坐在前排。
