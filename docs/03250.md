# CollabNet VersionOne 宣布其 VS 解决方案的新版本，这是一种创新产品，可帮助组织实现企业价值流管理

> 原文：<https://devops.com/collabnet-versionone-announces-new-release-of-its-vs-solution-an-innovative-offering-that-enables-organizations-to-achieve-enterprise-value-stream-management/>

**圣地亚哥–敏捷 2018，2018 年 8 月 6 日**–企业价值流管理、ALM、敏捷管理、Git 版本控制和 DevOps 的全球领导者 collab net version one([www.collabnet.com](http://icm-tracking.meltwater.com/link.php?DynEngagement=true&H=oGJ1pkZyysOST8btnWeAdWfazdP60O3PLkuOZioW2RiI0IVhceklygOqF8GRG%2FIL4EvS1rfH3txDVd1esR2Y89Hrf%2BZ%2BBJpZKs%2BblySLcq8tkiGs1lzWTQ%3D%3D&G=0&R=http%3A%2F%2Fwww.collabnet.com&I=20180806130225.0000000ddff1%40mail6-94-usnbn1&X=MHwxMDQ2NzU4OjViNjYyZGFlZjE0ZGQzNTJhYWRhNTMyNTs%3D&S=HCblRLAVsRgnVBvK8kBXZyYXG6MWoOLpoC9rZT4qRvw))宣布其最新发布的 [VS 解决方案](/cdn-cgi/l/email-protection#2c4458585c5f1603035b5b5b024f4340404d4e02424958035c5e4348594f585f035a5f)。这个版本使组织能够在软件开发和交付的企业级建立和实践价值流管理(VSM)。

VS 的这个新版本通过克服建立价值流模型的行业障碍，使企业 VSM 变得可行和可实现。现有工具的障碍可以总结为在整个企业中扩展敏捷和扩展开发运维的困难和无能。这些障碍和限制是由规划、版本控制之间的脱节以及开发运维对快速创新的利用不足造成的。这些严重的脱节增加了风险和成本。因此，客户希望消除它们并进入下一个级别，这在企业价值流管理中是可能的。

VS 消除了那些障碍和挣扎。客户可以扩展敏捷，应用 DevOps 编排和商业智能，并在整个企业中利用 Git 来使软件开发和交付流程与业务目标保持一致。

CollabNet VersionOne 首席执行官 Flint Brenton 表示:“这一发布意义重大，因为组织希望建立 VSM，使其在下一波数字化转型中具有竞争力，这将要求更高的速度、质量、效率和市场一致性，但他们缺乏企业级有效 VSM 的解决方案。我们解决了这个问题。”

“除了充分利用扩展敏捷和 DevOps 流程编排的潜力，没有其他方法可以从企业 VSM 中获益。Brenton 说:“VS 之所以能够做到这一点，是因为它以独特的方式将规划、Git 版本控制和 DevOps 结合在一起，以缩小差距，消除障碍，使整个企业的价值流更加顺畅。

知名主题专家和技术专家 Eric Robertson 领导该公司的产品营销管理，他进一步解释说:“我们在 VS 方面的创新领先于同行。它让组织建立企业 VSM，这是由与更广泛的业务操作相一致的透明软件开发和交付所锚定的。我们的 VS 解决方案让企业可以放心创新，这对于数字化转型至关重要。”

**VS 可用性和解决方案**

VS 一般都有。深度创新在于 VS 特性，这些特性允许用户扩展敏捷并将规划、Git 版本控制和 DevOps 连接成变革性的价值流，允许企业测量软件价值对企业的吞吐量。

VS 解决方案的这一版本由 VersionOne 及其用于扩展敏捷的新特性、用于 Git 企业版本控制的 TeamForge SCM 以及用于 DevOps 编排和快速创新的 Continuum 组成。与工具链集成、分析和治理等其他功能一起，开发团队获得了在整个生命周期中规划和管理其软件开发和交付过程的灵活性和可见性。该公司在培训和咨询方面的专业知识与技术相结合，使 VS 解决方案能够提供企业 VSM。

“我为这个解决方案感到自豪，并荣幸地在 CollabNet 和 VersionOne 成为一个组织并推动我们行业发展的周年纪念日宣布它，”Brenton 补充道。“我邀请所有希望了解更多信息的领导者与我们会面，分享你们在敏捷、DevOps 和企业 VSM 方面的挑战和目标。”
