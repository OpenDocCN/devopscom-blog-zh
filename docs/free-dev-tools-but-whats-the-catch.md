# 免费开发工具！但是有什么问题呢？

> 原文：<https://devops.com/free-dev-tools-but-whats-the-catch/>

软件公司，尤其是年轻的初创公司，总是处于开发竞赛中——顾名思义，他们试图最大化成就，最小化烧钱率。这意味着开发团队自然会面临一场内部的拉锯战。一方面，他们寻求使用市场上可以提供的最新开发工具，目标是让他们的应用程序变得更好(更安全、更高效并符合最新标准)。另一方面，他们不能花钱买这样的工具，因为他们需要节省每一分钱。在当前的经济环境下，许多新的投资都被推迟了，或者至少被复核了。

为了吸引开发者，许多供应商和软件开发工具公司提供免费增值项目。有些是由开源和社区的精神和善意驱动的；另一些是企业对开发商(B2D)公司，依靠产品导向的增长(PLG)模式。如果开发者看到了免费工具的价值，他们更有可能成为付费用户，以获得更多的功能。

在这里，我们将回顾流行的免费开发者工具的公共参数，并区分它们在免费层(也称为社区层)、入门级付费层和其他付费程序产品之间的服务。我们将看看用于可观察性、日志记录、漏洞扫描、合规性、认证和 VPN 的工具。

### 1.音量阈值

日志被认为是可观察性的三大支柱之一，是故障排除和调试的基础。产品中的代码行越多，收集、存储和聚合的日志行就越多。日志积累可能会很快失去控制，并成为非常沉重的预算。一家日志聚合供应商 [Sumologic](https://www.sumologic.com/pricing/) 提供免费增值服务，允许每天收集高达 1gb 的日志。如果您需要更大的交易量，您可以转到允许更大交易量的付费层级，或者支付更多费用而没有任何限制。

### 2.交易阈值

交易和测试可能是免费增值服务最重要的特征。它包括有数量限制的核心产品，让你有机会通过实际使用来尝试产品，但你可能必须快速进入第一付费层。例如， [Snyk.io](https://snyk.io/plans/) 将帮助您在代码创建之初就扫描开源软件的缺陷和容器漏洞，每月多达 200 次开源测试和 100 次容器测试。

### 3.认证:安全性和合规性

比如说，玩一个不涉及敏感数据的免费 AutoCAD seat 是一回事，而起草不能泄露给竞争对手的下一代设计则完全是另一回事。安全和合规认证通常包含在免费增值层级中；更详细的安全性和合规性报告和分析仅适用于付费层级。

### 4.可用性和 SLA 支持

拥有一个免费的工具是很好的——但是如果这个工具被绑定到一个专有的服务，那么你可能应该要求这些工具满足你提供给你的客户的相同的标准和 SLA。observability platform[Rookout](https://www.rookout.com/pricing/)为所有层级的企业客户提供 24/7 支持和专门的客户经理和工具。

### 5.法律方面

自由层的一个关键方面是使用该工具的法律方面。开发工具可能会与敏感数据交互，不仅在您自己的开发环境中，而且在您客户的环境中。您的客户可能要求您的开发操作满足特定的标准，这使得免费层成为一个潜在的法律问题。例如，如果你使用 [JFrog](https://jfrog.com/pricing/) 的自由层来管理你的供应链，你将无法使用 Artifactory 向第三方发布数据；如果你想这样做，你将不得不转向一个付费的商业节目。

### 6.使用阈值

一些免费工具是基于使用模式运行的——在达到一定数量的用户或动作之前，你有权使用免费层。 [Auth0](https://auth0.com/pricing) 例如，一个用户认证解决方案(现在 Okta 的一部分)，免费为多达 7000 个用户提供他们的服务(慷慨！)但如果你的增长超过这个水平，你就需要升级到付费级别，根据你的月使用量收费。

### 7.速度档

一些免费和免费增值服务基于连接速度、带宽甚至虚拟专用网络进行调整。今天，你甚至可以免费获得一个安全的 VPN，但仅限于特定的阈值。例如， [ProtonVPN](https://protonvpn.com/) 将为您提供一个免费的中速 VPN，如果您转到最高速度的 VPN，您将需要转到付费层。

在大多数情况下，免费增值层将根据一个或几个主要特征限制开发工具的使用，但允许您执行基本操作并享受工具的本质。一些参数可能是无关紧要的，比如测试量，而其他参数需要开发人员和 DevOps 团队仔细监控使用情况。免费和免费增值开发工具有很多价值——但是一定要阅读细则。
