# GitLab 更新 CI/CD 平台以提高开发效率

> 原文:[https://devo PS . com/git lab-updates-ci-CD-platform-to-improve-dev-productivity/](https://devops.com/gitlab-updates-ci-cd-platform-to-improve-dev-productivity/)

GitLab 为其同名的持续集成/持续交付(CI/CD)平台提供了[14.9 更新](https://about.gitlab.com/releases/2022/03/22/gitlab-14-9-released/)，增加了 epic 链接、集成安全培训、扫描结果策略的规则模式和改进的设计等功能。GitLab 说，更新的环境使发现应用程序开发项目的状态变得更加容易。

GitLab 产品营销总监 Cindy Blake 表示，这些更新是将技术嵌入 CI/CD 平台的持续努力的一部分，这将使应用程序在部署到生产环境之前更加安全，并符合各种要求。

例如，DevOps 团队现在可以利用用户界面，通过该界面他们可以使用规则模式来设计和编辑策略，而无需直接访问 YAML 文件。

![](../Images/b0855836b340da673a9ac028ffdfcd92.png)

GitLab 还与 Kontra 和 Secure Code Warrior 合作，提供将推动安全培训功能的内容。只要发现漏洞，就可以通过 GitLab 平台访问这些内容。

Blake 指出，总体目标是使组织更容易向左转，因为期望每个开发人员都成为安全专家是不太现实的。将开发人员面临的许多安全任务自动化的能力是对他们面临的巨大挑战表示同情的一种方式。

总的来说，随着应用程序开发变得越来越复杂，GitLab 正在努力使其 CI/CD 平台更容易被更广泛的组织访问，包括那些可能刚刚开始采用 DevOps 最佳实践的组织。例如，epic 链接功能可以更容易地识别“相关”、“阻塞”或“阻塞”关系，以便更好地跟踪和管理 GitLab 组之间的 epic 依赖关系。

当今组织面临的两个最大的挑战是保护他们的[软件供应链](https://devops.com/?s=supply+chain)和提高开发人员的整体生产力。在大多数情况下，这两个目标紧密相连；开发人员在安全性上多花一分钟，就可以少花一分钟编写代码。事实上，许多组织发现，除了工资和福利之外，许多开发人员还将他们在构建和部署应用程序时会遇到的摩擦程度作为决定他们希望为哪些组织工作的指南。因此，许多组织现在都希望实现应用程序开发和部署环境的现代化，以期吸引最优秀的开发人员。

当然，与此同时，组织也在寻求提高现有开发人员的生产力。在数字业务转型的时代，有更多的应用程序开发计划正在启动。组织面临的挑战是雇佣和留住开发人员仍然是一个重大挑战。

目前还不清楚组织有多渴望更换他们现有的 CI/CD 平台。然而，在组织越来越依赖软件的时候，改善整体开发人员体验的压力也在增加。可以说，展望未来，任何组织的竞争能力都将取决于应用程序开发团队的生产力。阻碍组织更快地构建和部署更安全的软件的平台和工具将被丢弃。
