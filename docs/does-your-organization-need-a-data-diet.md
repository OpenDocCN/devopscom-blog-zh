# 您的组织需要数据饮食吗？

> 原文：<https://devops.com/does-your-organization-need-a-data-diet/>

这个场景太熟悉了:出现了一个安全漏洞，之后，受影响的组织询问它必须做什么来更好地保护它的数据。

但是，如果该组织从一开始就没有收集和存储过这些敏感信息呢？通常，防止令人尴尬且代价高昂的违规行为的最佳方法是只收集对一个实体的使命至关重要的数据。

一些在计算机隐私领域工作的人将此称为“[进行数据节食](https://p97.com/marketing-meets-gdpr-part-1-maintaining-a-healthy-data-diet/)”他们知道，对于组织来说，积累各种数据的诱惑是巨大的。毕竟仓储成本低。随着向云的迁移，组织甚至不需要投资硬件和维护来存储其收集的信息。那么，为什么不获取顾客或客户愿意提供的任何数据呢？

因为我们生活在这样一个时代，对计算机安全来说，唯一明智的方法就是想知道*你的实体什么时候*可能被攻破——而不是*如果*。这就是为什么不仅要坚持数据饮食，而且要采取一种方法来保持组织的数据库精简，从而保持健康的客户关系。

很多时候，公司收集数据只是因为它可能有用或者他们认为它是无害的。我亲眼目睹了这出戏。销售部门寻找他们能找到的任何小优势；谁知道在寻找潜在客户的过程中，什么样的数据碎片会被证明是有用的呢？对于一个组织的营销部门来说也是如此，他们的员工正在努力为他们的产品或信息赢得更广泛的受众。如果信息自由地从潜在的和当前的顾客那里流出，他们收集信息是为了在吸引人们注意力的竞争中取得优势。

但是，除了数据存储的价格标签之外，组织还有其他成本。这包括声誉风险、诉讼风险以及这些和其他问题带来的干扰。越多的信息看起来越好——直到黑客侵入系统，组织的领导被迫解释为什么这些数据首先被收集。

### 将看似无害的数据武器化

就在不久前，似乎还存在两种截然不同的数据类型:一种是个人身份信息(PII)，如姓名、地址、电话号码和电子邮件地址，另一种是重要的财务信息，如银行账户和信用卡号。加入个人健康信息(PHI)和其他明显敏感的信息；每个人都知道这类数据必须得到严格保护。

第二桶是其他一切:我们生活在数字世界中留下的看似无关的东西，比如你什么时候访问了一个网站，你为网站选择的语言，你浏览了什么产品。把它想象成一个拖把桶——如果洒了会很脏，但对公司安全来说并不重要。

但近年来，这些区别变得模糊了。一个单独的数据片段可能看起来无伤大雅，但当与其他不属于法律规定的 PII 定义的数据集结合时，就不是这样了。让大数据对一个组织的销售和营销团队如此有前途的原因，也是让收集大数据变得潜在危险的原因:一个聪明的算法和当今令人敬畏的计算能力可以提供在庞大的数据存储中爬行的联系。如果你链接的信息是个人敏感的(一个人的品味或习惯或活动)，但不是技术上的 PII，它可以被武器化，以伤害个人。

举个例子，想想人们的手机提供的追踪数据。如果一个人每天早上 8:30 左右离开同一个街道地址，来回走同样的路，很快就能知道某人住在哪里，在哪里工作。鉴于其他公开记录，一个有着黑暗动机的黑客可以找到我们通勤者的名字和其他信息。将这些信息与盗窃甚至跟踪勒索等犯罪联系起来并不是什么大的飞跃。

### 采用新的数据规程

考虑到这些风险，有远见的组织现在正在采用隐私保护计划，迫使他们的团队在收集信息之前问一些棘手的问题。对于这些隐私从业者来说，任何时候有人在内部提议以新的方式获取或使用新的数据，随之而来的就是关于其有用性和收集数据的合法性的对话。

也许这项政策是通过内部隐私委员会来执行的，该委员会开会讨论其适当性。也许这是一个严格的过程，意味着整个组织的制衡。但是不管用什么方法，只收集对一个组织的运作至关重要的数据。

另一个想法在关注隐私的人中间越来越受欢迎:抛弃旧数据的过程。(举个例子:一个组织是需要知道一个人最后的几个地址，还是只需要知道可以联系到他们的当前地址？)在这样的数据饮食下，一个组织继续存储超出其直接使用的所有信息都需要一个具体的理由。

毕竟你没有的信息是最容易保护的。
