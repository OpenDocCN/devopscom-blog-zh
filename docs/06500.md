# Instana 推出企业增强功能，帮助组织更有效地管理任务关键型应用程序

> 原文:[https://devo PS . com/instana-launchs-enterprise-enhancements-to-help-organizations-manage-mission-critical-applications-more-effectively/](https://devops.com/instana-launches-enterprise-enhancements-to-help-organizations-manage-mission-critical-applications-more-effectively/)

*公司为 Redis Enterprise 和 IBM MQ 提供定制仪表板、基于角色的访问控制、NGINX 跟踪和监控传感器*

芝加哥，2020 年 7 月 9 日——面向微服务应用的自动化[应用性能管理](https://www.instana.com/application-management)解决方案的领先提供商 Instana 今天宣布了其云原生应用监控解决方案的多项新特性和功能，以帮助企业更有效地管理其任务关键型应用。新功能包括 UX 改进、操作增强和新的企业技术监控传感器。

Instana 的技术总监兼 APM 策略师 Chris Farrell 表示:“随着企业和云原生微服务应用之间的界限变得模糊，开发和运营团队需要更广泛的技术支持和企业级功能。“Instana 今天宣布的功能集将为组织提供所需的可见性和易用性，以跟上持续的数字化转型和创新。”

Instana 推出的新功能包括更强大的企业用户体验、针对复杂企业平台的监控和跟踪，以及针对大型组织的运营增强功能:

*   自定义仪表板
*   NGINX 跟踪
*   Redis 企业监控传感器
*   IBM MQ 监控传感器
*   基于角色的访问控制

运行企业规模环境的 IT 组织对数量更多、范围更广的用户群有操作需求。Instana 的新仪表板和基于角色的访问都旨在为所有用户提供他们需要的准确细节，而不必接受有关 Instana 数据和用户界面细微差别的培训。

随着组织通过更大的团队和更多的利益相关方来监控云原生应用程序，自定义仪表盘使用户能够基于各种 UI 小部件和设置(如图表、降价和时区)来构建个性化的仪表盘，使他们能够在单个屏幕上可视化任何数据。

随着基础架构和 IT 环境日益复杂，用户越来越难以从大量数据和噪音中抽身出来，专注于他们需要的准确信息。借助基于角色的访问控制，组织能够创建和管理具有相应权限和访问区域的组。这提供了减少可见应用视角(按应用、单个服务所有者和团队的独特动态分组)、按团队的服务和事件的数量的控制，以便每个用户都有他们需要看到的确切内容的定制视图。

Instana 的 APM 解决方案还增加了 NGINX 跟踪功能，该功能可自动与 Instana 终端用户监控(EUM)集成，以原生方式提取通过 NGINX 的跟踪请求，消除 NGINX 代理的盲点，并收集 NGINX 内外运行的服务的性能和延迟指标。

新的 Instana enterprise 监控传感器包括 IBM MQ 和 Redis Enterprise。对于遗留消息传递系统中的主要平台 IBM MQ，Instana 的解决方案现在直接从 IBM MQ 收集指标，以增强全面理解消息、应用程序和 IBM MQ 系统本身的健康和性能所需的上下文细节。

此外，Instana 的 APM 解决方案现在可以在单个组件和应用程序级别监控 Redis 企业关键绩效指标(KPI)。这些 KPI 用于实时监控每个实例和集群的运行状况，并用于了解 Redis 企业在整个应用程序中的性能。最终结果是能够在用户对业务关键型应用程序产生负面影响时发出警报。

与传统的 APM 工具不同，Instana 的自动化应用性能监控(APM)解决方案可以发现所有应用服务组件和应用基础设施，包括 AWS Lambda、Kubernetes 和 Docker 等基础设施。Instana 自动为应用技术堆栈的每个部分部署监控传感器，跟踪所有应用请求并分析每个流程，无需任何人工配置，甚至无需重启应用。该解决方案实时检测应用程序和基础架构的变化，调整自己的模型，并在几秒钟内可视化这些变化和任何性能影响。

Instana 应用程序性能管理解决方案的 SaaS 版和本地版中都提供了新功能和传感器。
