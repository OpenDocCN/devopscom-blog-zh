# 利用数据库开发运维实现业务成熟度

> 原文：<https://devops.com/achieving-business-maturity-with-database-devops/>

在过去的一年半时间里，公司经历了前所未有的变化，IT 团队试图通过满足客户对以新方式交付服务和产品的需求，同时适应远程工作，来帮助企业度过难关。随着越来越多的公司将混合方法引入他们的工作场所，一个特殊的趋势开始出现:DevOps 的采用。

许多企业采用 DevOps 来适应新的环境，通过先进的应用和数据库开发实践来提高软件交付和运营性能。事实上， [来自 Redgate Software](https://www.red-gate.com/solutions/database-devops/report-2021) 的研究表明，DevOps 的采用在今年早些时候达到了顶峰，26%的企业在所有项目中利用 DevOps，这比以前的采用率有了显著提高。

尽管并非所有公司都因为担心中断现有工作流、技术技能差距或缺乏管理层的支持而迅速采用，但有充分的理由说明为什么[采用 DevOps 实践](https://devops.com/?s=devops+adoption)，包括数据库，可以帮助企业更好地管理不断发展的技术环境。

## **采用自动化部署**

在过去的一年里，世界各地的公司都将数据库迁移到了云中，而且这一趋势没有放缓的迹象。IDC 的分析师 [预测](https://www.idc.com/getdoc.jsp?containerId=prMETA46165020) 到 2022 年，超过 90%的企业将依靠内部/专用私有云、多种公共云和遗留平台的混合来满足其需求。帮助维护这些数据的最佳方式之一是为这些数据库实施开发运维流程，并实现部署自动化。这提供了很多好处，包括:

*   **防止软件错误:**手动数据库迁移经常会出现大量错误，导致停机、回滚和返工。更好的方法是依靠一个自动化的过程，这个过程是健壮的、可重复的，并且可以在所有数据库中实现，以确保它们以相同的、可靠的方式更新。
*   **留在圈子里:**关于如何发布更新和变更的知识通常仅限于少数人，甚至只有一个人。相反，如果它是系统和开发过程的一部分，它确保每个人都与更新和开发同步，并允许您的 it 团队中的任何人处理部署。
*   **保证质量和一致性:**对于存在安全性和合规性问题的公司，自动化数据库部署引入的可预测且可重复的流程可确保万无一失，同时提供可审计的记录，准确记录更改的内容、时间和更改者。

尽管研究显示 20%的公司将成本列为自动化的障碍，但避免自动化将使企业在为来年做准备时处于不利地位。

## **创建实践社区**

当被问及如何决定 DevOps 的战略方向和规划时，受访者最普遍的回答是[实践社区](https://hbr.org/2000/01/communities-of-practice-the-organizational-frontier)。这一概念在 20 世纪 90 年代初首次提出，最近这一实践在 IT 团队中越来越受欢迎。它被描述为一群人，他们对自己所做的事情有着相同的热情或关注，并经常根据他们定期、频繁的互动来学习如何更好地完成任务。

鉴于当前的环境，这些不同的社区(无论是面对面的还是虚拟的)有助于培养员工的内在动机，改善整体决策并鼓励创新。根据 Redgate 的研究，超过三分之一的受访者指出，在开发运维战略和规划实施之前，实践社区会做出决策、提供建议或被征询意见。这项研究的另一个有趣之处是实践社区的采用与公司规模相关。大型企业和中型企业的采用率分别为 38%和 36%，相比之下，小型企业的采用率仅为 28%。

随着公司的成长和团队的成熟，这些社区不仅通过在决策时将更多不同的想法带到桌面上来使组织受益，而且通过支持发展、实现网络和提高整个组织的认可来使个人受益。

## **管理跨数据库资产**

Redgate 的研究发现，70%的组织现在使用不止一个数据库管理系统，48%的组织有团队使用三个或更多不同的数据库。一个可能的原因是越来越多的人采用云数据库。随着大量云数据库和平台的使用，每一个都提供不同的优势，团队现在根据特定的用例做出选择。例如，与去年相比，使用云原生数据库(如 Azure Cosmos DB、MongoDB 和 Snowflake)的受访者比例有所增加。

由于疫情和对可远程管理、操作和访问的技术的需求，云的采用正在加速，引入 DevOps 流程以维护数据库变更并确保版本控制的优势从未如此明显。拥有自动化和高级开发实践的团队可以继续像疫情之前一样顺利地构建和部署软件，即使是在远程工作的时候。

成熟的 DevOps 团队可以将精力集中在解决问题和确保更高的代码质量上，而不是努力构建流程和过程，不需要费力的手动任务或代码审查，自动化系统执行起来更可靠。

## **为数据库开发做准备**

当团队分散在不同的地方，按照不同的时间表工作，并在家里处理各自的挑战时，公司就可以做更多的事情来减少工作量，帮助他们专注于增值工作，从而创造更快乐的工作场所，提高企业成功的机会。面向应用程序和数据库开发的 DevOps 的兴起有助于缓解组织在数字化转型过程中面临的诸多挑战。它将帮助 It 团队在远程工作时保持高效，并鼓励他们探索新的数据库平台，包括内部和云中的平台。

随着远程和混合工作成为未来的新常态，着眼于在整个公司范围内采用 DevOps 的公司只会看到这些好处的增加。
