# bmc 希望采用先进的服务方法来管理 it

> 原文:[https://devo PS . com/BMC-looks-to-advance-service ops-approach-to-managing-it/](https://devops.com/bmc-looks-to-advance-serviceops-approach-to-managing-it/)

在一次在线[BMC 交流](https://exchange.bmc.com/)活动中，BMC 宣布扩展其 it 管理产品组合，作为在集成的 ServiceOps 框架下统一传统 IT 服务管理和开发运维流程的持续努力的一部分。

BMC 产品组合的最新成员[包括对其 Control-M 工作负载自动化平台的更新，该平台增加了用于监控应用程序和数据工作流的可观察性工具，以及对云服务的额外集成，如 AWS Glue、Google Cloud Dataflow 和 Function 以及 Microsoft Azure 数据工厂和平台，如 Databricks 的 data lake。](https://www.prnewswire.com/news-releases/bmc-unveils-new-innovations-to-unleash-the-power-of-enterprise-data-301409768.html)

BMC 还提供了 Control M Python 客户端的预览版，这将把工作负载自动化平台的范围扩展到代码数据科学团队用来构建人工智能(AI)模型的领域。

此外，BMC 还增加了一个 BMC Helix Digital Workplace studio，使 it 团队能够更轻松地定制其为 [IT 服务管理](https://devops.com/?s=ITSM) (ITSM)提供的软件即服务(SaaS)平台。最后，BMC 还提供了一个为通信服务提供商优化的 BMC Helix 版本。

BMC 首席执行官 Ayman Sayed 表示，总体而言，当前对数字业务转型的重视推动了对多种 IT 平台管理的融合需求，因为各组织正在重塑核心流程在整个组织中的工作方式。他指出，如果不使用统一 DevOps 和现有 ItSM 平台的 ServiceOps 框架来集中管理这些平台，就不可能推动跨多个 IT 平台的重新设计。

与此同时，Sayed 表示，组织需要在人工智能方面进行更多投资，以实现 IT 运营的自动化，也称为 AIOps，以增加因管理新兴和传统 IT 平台而受到压力的 IT 员工。BMC 对 AIOps 的最新投资是本月早些时候对 StreamWeaver 的收购，StreamWeaver 是一家提供一套工具的公司，可以更容易地收集大量数据，这些数据可以用来更容易地训练人工智能模型。

现在说 It 管理流程会在多大程度上实现统一还为时过早。除了集中的 IT 团队之外，大多数组织都有业务部门，这些部门经常推动他们自己的 IT 计划。然而，随着组织意识到一个数字流程需要访问多少个平台，集中管理 IT 的需求变得更加明显。每个组织需要确定的问题是集中化需要达到的程度。

很明显，在新冠肺炎疫情之后，许多 It 组织从未像现在这样敏捷。在某些情况下，这些 IT 团队现在能够推动变革的速度之快，许多企业甚至无法吸收。事实上，整个企业需要采用 DevOps 原则来管理这种变化率是有争议的。

与此同时，IT 团队将需要更仔细地了解在手动任务上花费了多少时间，这些手动任务可以通过自动化的方式来提高工作效率。
