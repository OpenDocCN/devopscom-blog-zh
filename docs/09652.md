# 技术人员努力应对混合 IT 复杂性

> 原文：<https://devops.com/tech-workers-struggle-with-hybrid-it-complexity/>

随着技术团队的工作变得更加复杂，IT 员工的信心水平直线下降。IT 工作者焦虑背后的原因是前所未有的，标志着一种类似于其他大流行后行业的转变。跨云和内部基础架构运行应用程序、具有[分布式工作人员的混合办公环境](https://devops.com/zscaler-2021-exposed-report-reveals-corporate-cloud-infrastructures-more-at-risk-than-ever-from-expanded-attack-surfaces/)以及新技术和自动化技术的加速采用都导致了 IT 复杂性的增加。

在我们最近发布的【2022 年网络安全管理软件产品 IT 趋势报告——正确应对:管理混合 IT 复杂性中，我们研究了 IT 的当前状态以及技术专业人员的机会领域，他们必须管理混合 IT 加速增长带来的复杂性。“继续向混合 it 转变推动了 IT 管理复杂性的增加，但技术专家对如何最好地管理 IT 缺乏信心，”该报告称。

### 管理混合 IT 复杂性

解决方案是必要的，因为研究显示，在接受 IT 趋势报告调查的技术专家中，只有 8%的人对自己管理日益复杂的网络的能力“非常有信心”。超过一半的受访技术专家表示，他们的组织通过培训员工和采用 IT 监控/管理工具来管理混合 IT 复杂性。然而，预算、时间限制和混合工作环境加剧了这一挑战。这些反应表明需要进行干预和评估，因为 IT 专业人员的工作比以往任何时候都更具业务关键型。强大的 IT 基础设施对于所有行业和业务环境的基本流程都是必要的。

### 一半缺乏能见度

根据市场情报公司 IDC 的一份报告，随着越来越多的任务关键型工作负载迁移到跨越公共云、私有云、混合云和多云环境的互联云架构，企业认识到需要投资工具来帮助他们确保一致的策略和性能，尽管存在预算和可观察性实施问题。《网络安全管理软件产品 IT 趋势报告》显示，近一半的受访者对其组织的大多数应用程序和基础架构缺乏了解。

事实上，对于负责在这些多样化的分布式环境中管理更高级别复杂性的技术专家来说，可见性正日益成为明确的答案。为了发展他们的团队和业务以取得长期成功，组织必须留出时间和资源来提升和培训技术专业人员，以帮助他们正确实施可观察性战略并更有效地管理混合 IT 现实。

### 场外效力丧失

但是，远程革命的某些方面，包括培训 IT 员工进行日益复杂的网络管理、头脑风暴会议、提供敏感反馈和新员工入职，在非现场工作时会失去效力。皮尤研究中心(Pew Research Center)的一项研究显示，对于那些转向远程办公的人来说，大多数人发现工作与生活更加平衡，但与同事的联系或来自领导的持续支持更少。

软件公司 Culdesac 宣布将员工转移到网上，并关闭其旧金山办公室。在疫情期间，全球超过 14，000 名谷歌员工搬到了新办公室或完全远程办公。微软等其他公司表示，希望 50%以上时间远程工作的员工必须让他们的经理批准他们的时间表。

随着我们进入大流行后 IT 基础设施时代，情况正在迅速变化。

### 自动化网络管理标准

解决方案的可用性越来越强。互联网工程任务组(IETF)是一个由网络设计者、运营商、供应商和研究人员组成的国际团体，它正在制定自动化网络管理的标准，旨在随着网络规模和复杂性的不断增加，改进网络管理并使其更加高效。

虽然由于混合 IT 复杂性的增加，组织也缺乏对其网络对投资回报的影响的洞察力，但展望未来，他们必须重新审视过去几年的投资。其中一部分要求公司了解其 IT 环境，以了解哪些可行，哪些不可行。

### 愿意适应的人的经济增长

技术变革可以带来经济增长和机遇，但只针对那些愿意适应的人。IT 基础设施比大多数企业所理解的要复杂得多，正如现有数据所证实的(该领域的大多数专家也同意)，投资 IT 基础设施是必要的。企业的网络——及其速度和安全性——可能是满意或失望的员工和客户之间的差异。

由于混合 IT 加速增加了网络复杂性，降低了技术专家的网络管理信心，这些大流行后的问题为组织提供了发展的机会。“健康的做法是向前看，接受现实，并开始为转变做准备，”华盛顿大学信息学院的副教授 Chirag Shah 说。“因此，与其感到焦虑和戒备，不如把这当成一个进步的机会。”
