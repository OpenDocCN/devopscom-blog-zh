# 无代码将如何改变 DevOps

> 原文:[https://devops.com/how-no-code-will-transform-devops/](https://devops.com/how-no-code-will-transform-devops/)

在低代码和无代码开发工具出现的这些年里，这种运动的步伐一直很慢。直到最近，绝大多数应用程序和工作流开发仍然牢牢掌握在专业程序员手中。

这种情况即将改变，而且会很快改变。Gartner 预测，全球[低代码开发技术市场](https://www.gartner.com/en/newsroom/press-releases/2021-02-15-gartner-forecasts-worldwide-low-code-development-technologies-market-to-grow-23-percent-in-2021)有望在 2021 年达到 138 亿美元，比 2020 年增长近 23%。[其他研究表明](https://resources.formstack.com/reports/rise-of-the-no-code-economy/tech-revolution)今天 20%的员工使用无代码工具，包括 56%的程序员。在 IT 部门使用无代码工具的人中，有 71%是在去年采用的。

无代码开发正在达到一个临界点，这种采用的涓涓细流已经变成了一股洪流，每天驱动着越来越多的业务需求。

## 无代码运动的驱动力

这个加速的原因很简单:[数字转换](https://devops.com/?s=digital+transformation)需要速度。如果企业要将跨所有部门的全部流程数字化，那么他们需要能够快速采用的工具，并且几乎不需要加速时间。

无代码工具是这个问题的理想解决方案，而不仅仅是短期的修补。如果投资仅被视为解决眼前问题的短期措施，那么公司的数字化转型努力很可能会继续胎死腹中。数字化转型不是“一蹴而就”的过程；这只是基于持续创新的新业务战略的开始，新的应用和工具一次又一次地被构建和重建，以跟上不断变化的市场动态和客户/员工需求。

两个短期趋势加速了数字化转型的长期趋势。首先，显然是新冠肺炎·疫情。用于控制病毒的社交距离措施意味着，如果企业想要生存，就必须创建新的应用程序和工作流(在几小时或几天内)。无代码工具在帮助企业快速适应方面发挥了重要作用，并让他们体验了直观应用和工作流开发的可能性。

第二，缺乏熟练的开发人员。全球 2400 万开发人员仅占总劳动力的 1%,这使得企业人手不足，并在寻找方法让非编码人员控制自己的应用开发。除了填补技能缺口，no-code 还满足了员工对更多自主权以及对其运营、项目和工作流程更多控制的需求。

## 开启创新

这些驱动因素的最终结果是，无代码现在是一场真正的运动，大大小小的技术公司都在利用市场创新来改变网站、软件应用程序、工作流程和移动应用程序的构建方式。

对企业的影响将是深远的。正如埃森哲在其 [2021 年技术愿景](https://www.accenture.com/gb-en/insights/technology/technology-trends-2021)中所强调的，无代码是更广泛的 IT 民主化的一部分，其中创建技术解决方案来解决业务挑战的能力扩展到了组织中的每个人。这一趋势将引发一波“草根创新”浪潮，帮助企业在未来几年重塑自我。那些现在没有抓住这个机会的人，以后可能会努力跟上更有创造力和更敏捷的竞争对手。

## 无代码对开发者意味着什么

对于开发者来说，这种趋势无疑会引发很多疑问。在一个无代码的世界里，受过专业训练的程序员扮演什么角色？DevOps 会随着“公民开发者”的接管而灭绝吗？无代码和低代码不仅不会造成有害的影响，反而可能会被证明是非常有益的。以下是四个主要原因:

**工作量/压力减少。**有了基于非代码的角色，组织能够扩展他们的技术能力，而不需要雇佣更多的开发人员或编码人员。此外，通过将基本的应用和工作负载开发任务分散到整个组织，开发人员的时间得到了释放。对于已经人手不足并面临压力的团队来说，这可以在工作场所体验中产生巨大的差异，并使开发人员能够花更多的时间专注于交付高质量的软件。

企业中更具战略性的角色。随着战术软件开发在业务部门内进行，开发人员可以将更多时间花在“下一件大事”上，无论是混合现实、量子计算还是人工智能。开发人员和更广泛的 IT 团队可以充当“未来学家”，评估最适合业务的新工具，并指导购买决策，以确保尽可能好的结果。

**加速创新。**对于需要编码专业知识的更复杂的软件开发，开发人员可以利用无代码的速度，在投入时间和资源进行全面开发之前，进行概念验证和测试。事实上，[的研究表明](https://resources.formstack.com/reports/rise-of-the-no-code-economy/tech-revolution)许多开发人员已经在采用这种方法，38%的人声称他们使用无代码来开发新产品原型。

新的角色和机会。无代码将对企业产生深远而广泛的影响，改变工作的完成方式，并在流程中创造新的角色。其中许多角色将为开发人员提供新的职业道路和机会，例如无代码咨询(帮助企业找到满足其需求的最佳无代码框架和工具)，以及围绕优化流程、构建解决方案和发现轻松自动化和简化其组织内日常工作的方法的角色。

长期以来推动数字化转型的 DevOps 团队现在终于感受到了这些努力的效果。一个新的工作世界正在出现，每个人都是技术专家；使用直观的工具在业务需求出现时快速构建解决方案。这种方法将把开发人员从传统的创新把关人角色中解放出来，使他们能够专注于战略性技术决策，并将他们的专业技能引导到他们将产生最大影响的领域。

无代码的兴起对每个人来说都是好消息。它加速数字化转型，释放所有员工的创新潜力。这对开发者来说也很棒；让他们的工作更有趣，压力更小，并有新的职业道路和发展机会。无代码可以为所有企业提供增长动力，并为早期采用者提供显著的竞争优势。
