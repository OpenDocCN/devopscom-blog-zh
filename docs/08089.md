# PagerDuty 和 JFrog 宣布集成 DevOps 和安全生命周期

> 原文:[https://devo PS . com/page duty-and-jfrog-announce-integrations-for-devo PS-and-security-life cycle/](https://devops.com/pagerduty-and-jfrog-announce-integrations-for-devops-and-security-lifecycle/)

*新的集成为客户提供了更快的事件解决方案，提高了正常运行时间，改善了客户体验并推动了收入增长。*

**加利福尼亚州旧金山，2021 年 6 月 8 日—** 全球领先的数字运营管理公司 PagerDuty，Inc .(纽约证券交易所代码:PD)今天宣布与灵动软件公司 JFrog(纳斯达克代码:Frog)进行新的整合。与 JFrog Xray 和 JFrog Pipelines 的集成——两者都是 [JFrog DevOps 平台](https://jfrog.com/platform/)的一部分——允许客户观察他们的整个软件交付生命周期，了解任何变化，并在整个过程中识别安全问题。这款端到端 DevOps 解决方案包括 DevSecOps、DevOps 可观察性和软件开发生命周期(SDLC)监控，适用于本地混合和多云部署。

“传统上，信息安全是软件交付的瓶颈，”PagerDuty 的合作伙伴和联盟全球副总裁 Timm Hoyt 说。“这种新的集成不仅有助于消除这一瓶颈，还能帮助组织在解决事故时茁壮成长。我们很荣幸能够与 JFrog 合作，并提供一种解决方案来帮助团队管理紧急的关键任务工作，这对于保持数字服务始终在线至关重要。”

借助 [JFrog Xray-PagerDuty 集成](https://jfrog.com/blog/stay-alert-to-security-with-xray-and-pagerduty/)，DevSecOps 团队可以收到针对开源安全漏洞和许可证合规违规的 PagerDuty 通知，这些漏洞和违规是由 [Xray 对包和容器图像的持续扫描](https://jfrog.com/xray/)检测到的，以便快速修复问题。然后，PagerDuty 可以将任何安全或许可证策略警报转换为事件报告。这对于跨软件开发和发布生命周期主动管理安全性和合规性、定制通知以及接收受影响组件及其相关依赖项的持续更新列表非常有用。

使用 [JFrog 管道和 PagerDuty，](https://jfrog.com/blog/jfrog-and-pagerduty-extend-ecosystem-integration/)用户可以简化和精简如何识别影响生产环境的错误构建。IT 操作员可以快速检测事件、确定根本原因、回滚构建并缩短解决时间。这有助于 IT 操作员和开发人员提高工作效率，确定其环境中发生了哪些变化，并使他们能够更好地防止停机，增加关键业务应用程序和数字服务的正常运行时间、稳定性和可靠性。

“我们很高兴与 PagerDuty 合作。JFrog 开发人员关系副总裁 Steve Chin 表示:“我们领先解决方案的集成——来自 JFrog 的端到端通用 DevOps 平台和 PagerDuty 的数字运营管理功能——使 DevOps 团队能够在整个 DevOps 生命周期中实现强大的可观察性和事件管理。
