## **花时间发展你的技能**

会议的头两天将会有很多研讨会和培训课程。不管技能水平如何，每个人都有培训。那些刚刚熟悉 Jenkins 的人可以参加免费的“[Jenkins 简介](https://jenkinsworld20162017.sched.com/event/AMM6/introduction-to-jenkins-free?iframe=no&w=&sidebar=yes&bg=no)”培训，学习如何轻松创建管道，同时实施持续集成(CI)和持续交付(CD)的最佳实践。

那些在 Jenkins 之旅中走得更远的人可以参加研讨会，如[“最后一英里:Jenkins 管道和运营模式](https://jenkinsworld20162017.sched.com/event/AMMF/the-final-mile-jenkins-pipeline-and-patterns-for-operations-99?iframe=no&w=&sidebar=yes&bg=no)”，与会者将回顾 CD 环境中运营团队的常见问题和解决方案的模式，以及如何创建满足运营需求的 Jenkins 管道。

当然，所有与会者都应该确保参加每个主题演讲！主旨发言人包括 CloudBees 首席执行官 Sacha Labourey 詹金斯项目创始人川口康介；以及 DevOps Research and Assessment LLC 的首席技术官 Jez Humble。

除了顶尖的主题演讲人之外，还有许多 DevOps 专业人士，您在展会期间也想听听他们的意见。尝试至少参加其中一个会议:

*   [DevOps 领导小组](https://jenkinsworld20162017.sched.com/event/BcIk/devops-leadership-panel)——将邀请 WatchGuard Technologies 的 Kolby Allen 出席；Andrey Falko，Salesforce 乔治·斯旺，欧特克；斯塔斯·兹维尼亚茨科夫斯基，埃森哲；以及 Forrester 的 Rob Stroud(主持人)。
*   [与我谈论开发人员:Jenkins 和 Alexa 负责语音激活部署](https://jenkinsworld20162017.sched.com/event/ALN2/talk-devops-to-me-jenkins-and-alexa-for-voice-activated-deployments?iframe=yes&w=100%25&sidebar=yes&bg=no)—由埃森哲的 Patrick Hampton 领导，他是亚马逊的高级 ISV 宣传员 Alexa。
