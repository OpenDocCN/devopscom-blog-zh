 **除了测试流程的改进之外，这些集成使开发人员能够使用任何测试框架和编程语言，轻松快速地将代码更改与 UI/UX 更新关联起来，以实现跨 web 和移动应用的可视化 UI 版本控制。

创建特征分支后，Applitools 立即镜像特征分支，以允许一组新的基线图像。这个过程简化了所有特性分支的自动化测试，而不会影响主干及其测试。当 PRs 被合并回时，集成自动地用源分支的基线图像更新目标分支，同时确保分支之间不存在冲突，消除了测试维护开销。

![Image](img/00c170805d6e835139e5f9d91cdb3522.png)
(上图:Applitools / GitHub 集成)

使用 GitHub Actions，开发人员可以直接在工作流程中实现自动化。现在，开发人员可以通过 GitHub Actions 工作流 集成 Applitools 来自动对他们的代码进行可视化测试。在那里，他们可以通过使用 Applitools 根本原因分析功能快速处理和解决回归问题。这有助于团队定制他们的工作流，以满足开发人员的需求，同时采用可视化人工智能来帮助自动化和加速构建、测试和发布管道。

GoodRx 质量工程高级经理 Priyanka Halder 表示:“Applitools 与 GitHub 的集成合并了两个基本产品，使我们的团队能够持续交付。这种协作在不改变我们工作流程的情况下实现了新的效率。使用了一段时间后，我无法想象我们的团队没有 UI 版本控制和合并时的自动基线更新。”

借助 Visual Studio 应用中心，开发人员可以简化移动应用的工作流程，包括自动化 UI 测试。通过将这一功能与 Applitools 相结合，开发人员现在可以无缝地自动化功能和视觉测试，并在发布前找到任何设备类型上的错误，同时确保应用程序正常运行，并在所有设备和屏幕尺寸上保持视觉合规。开发人员会收到测试结果、每个步骤的全分辨率截图以及性能指标。

Applitools 的联合创始人兼首席运营官 Moshe Milman 表示:“扩大我们与 GitHub 和微软的集成有助于开发人员更快地提供创新解决方案。“这些集成让开发人员可以毫不费力地将视觉人工智能和跨浏览器测试功能添加到他们的管道中，使他们能够更快地将产品推向市场，并提高产品质量。”
