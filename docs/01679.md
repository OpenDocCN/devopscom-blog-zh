# 网络研讨会:引领变革

> 原文：<https://devops.com/webinar-leading-the-transformation/>

与《引领变革》和《大规模敏捷开发的实用方法》的合著者 Gary Gruver 一起深入探讨大型组织中的软件开发过程变革。

这本书和演示的重点是向高管们展示如何领导企业规模的软件开发过程的转变。这本书没有采用传统的敏捷和 DevOps 方法来提高团队的效率，而是针对大多数大型组织中最大的效率低下问题，高管们有独特的定位来解决这些问题，即协调团队之间的工作。

## 关于小组成员

实用大规模敏捷公司总裁加里·格鲁弗

Gary Gruver 是一位经验丰富的高管，拥有在大型组织中改造软件开发和交付过程的良好记录。他目前正在与整个行业的高管合作，帮助他们转变开发和交付流程。作为《领导转型》的合著者，他向大型传统组织的高管展示了如何通过大规模应用敏捷和 DevOps 原则来领导他们的转型。作为《大规模敏捷开发的实用方法》一书的合著者，他记录了在他担任 LaserJet 固件开发实验室主任期间，他们如何革新了惠普的软件开发流程。作为 Macy's.com 的 QE、发布和运营副总裁，他领导了他们向持续交付的过渡。
