# GitHub 预览将于 2023 年推出的免费 CI/CD 迁移工具

> 原文：<https://devops.com/github-previews-free-ci-cd-migration-tool-due-in-2023/>

在其 GitHub Universe 活动上，GitHub 今天公布了其云服务的一系列更新，包括即将推出的一套迁移工具。这些工具将自动从遗留的持续集成/持续交付(CI/CD)平台迁移到 GitHub 操作。

一小部分企业 IT 组织已经在使用 GitHub Actions Importer 来迁移工作流和模板化 GitHub CI/CD 平台。

GitHub 开发者权益高级经理 Damian Brady 告诉与会者，这些免费工具将在 2023 年更广泛地提供。GitHub Actions Importer 将作为 Docker 容器分发，该容器显示为 GitHub 命令行界面(CLI)的扩展。

他指出，该公司在推动其平台采用方面面临的最大挑战之一是，许多 DevOps 团队不想重建他们已经在现有 CI/CD 平台上使用的模板和工作流。

其他 GitHub Actions 扩展包括与软件开发工具的本机集成，用于构建物联网(IoT)应用程序，并支持个人访问令牌(pat ),实现对权限和存储库访问的更细粒度控制。

其他安全功能包括 GitHub 企业版中对审计日志事件的扩展访问、开源库维护者私下共享漏洞报告的能力以及对 Ruby 的支持，Ruby 是一种现在由 CodeQL 工具支持的用于发现漏洞的编程语言。

还预览了对开发人员现有服务的其他扩展，包括与 GitHub Copilot 工具交互的能力，以使用语音命令自动编写代码。配音为“嘿，GitHub！”该工具是 GitHub Next 研究团队正在推进的一项实验。

最后，该公司预览了 Codespaces 工具中代码搜索和查看功能的增强，现在将向开发人员免费提供每月 60 小时的服务。

GitHub 声称其 CI/CD 平台现在每天被用来处理 1000 万个构建，其中大部分都使用了通过其存储库访问的代码。目前还不清楚它与其他 [CI/CD 平台](https://devops.com/?s=CI/CD+platforms)相比如何，但很明显 GitHub Actions CI/CD 平台已经获得了很大的吸引力，因为 GitHub 继续为在其存储库和 CI/CD 平台之间创建更紧密的集成提供理由。GitHub 也明确表示，它将花更多的时间来争取已经安装了传统 CI/CD 平台的组织。

与此同时，随着工作负载在高度分布式计算环境中构建和部署变得更加复杂，下一个 DevOps 时代已经到来。组织部署的许多 CI/CD 平台现在都不堪多年来进行的多次定制和扩展的重负。每个 DevOps 团队都需要决定维护这些平台是否最具技术和经济意义，或者他们是否应该迁移到更现代的平台。

不管他们做出什么决定，有一件事是肯定的，那就是 DevOps 团队不会变得更大，因此投资更高水平自动化的需求变得迫切。
