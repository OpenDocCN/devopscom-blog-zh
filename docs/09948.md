# IBM 向云产品组合添加完全托管的 VMware 服务

> 原文:[https://devo PS . com/IBM-adds-fully-managed-VMware-service-to-cloud-portfolio/](https://devops.com/ibm-adds-fully-managed-vmware-service-to-cloud-portfolio/)

在 [VMware Explore 2022 Europe](https://www.vmware.com/explore/eu.html) 大会上，IBM 本周宣布了一项[IBM Cloud for VMware-as-a-Service](https://newsroom.ibm.com/IBM-is-Helping-Clients-Modernize-VMware-Workloads-with-Speed-Launches-New-Managed-Service)产品，该产品将 VMware 云实例与 IBM 在云中管理的基础架构-as-a-service ( [IaaS](https://devops.com/?s=IaaS) )平台集成在一起。

IBM Cloud 产品管理和战略副总裁 Rohit Badlaney 表示，现在 VMware Cloud 可以作为订阅服务使用，IBM 现在可以将其与云平台集成，以创建一个完全托管的服务。他指出，以前，IT 组织必须单独许可 VMware 云，并自己在 IBM 云上部署它。

总的来说，Badlaney 表示，IBM 专注于与高度监管行业中需要在云中运行复杂工作负载的组织合作。他补充说，IBM 没有兴趣与其他云服务提供商竞争在商用基础设施上运行工作负载。

如今，运行在云中的大多数应用程序消耗的内存和存储资源范围很窄。IBM cloud 显然更专注于提供利润更高的服务，以补充其通过数据中心提供的云基础设施。不太清楚的是，在内部 IT 环境中运行的单一应用程序在多大程度上被提升并转移到云中。虽然很明显有一部分组织不想再运营自己的数据中心，但大部分应用程序仍在内部 IT 环境中运行。其原因包括从性能要求和安全考虑到法规遵从性问题的方方面面。

然而，由于大多数内部部署的应用程序都在 VMware 虚拟机上运行，因此对云迁移的关注主要集中在不需要 IT 团队重构应用程序以在不同的虚拟机平台上运行的平台上。IBM 还认为，鉴于当前的全球经济状况，更多的组织将很快选择不管理自己的数据中心。由第三方管理的云平台将使这些组织能够将更多的精力放在应用程序开发和部署上。

每个组织都需要根据哪种方法最有经济意义来决定是否将其视为运营支出或资本支出。已采用开发运维工作流的组织需要将构建和部署应用的方式与托管服务保持一致。

当然，理论上，托管云计算服务使 IT 团队更加敏捷和高效。较少的 IT 基础设施最终被开发人员过度配置，这些开发人员可以通过编程按需增加资源。当然，除了 VMware Cloud 之外，还有很多 IT 堆栈可以部署在云中。例如，IBM 提供 Red Hat OpenShift 平台，在 Kubernetes 集群上运行云原生应用。IBM 还没有承诺为运行这些类型的应用程序提供 VMware Tanzu 平台。

并不是每个应用程序最终都会在云中运行，但可以肯定的是，在下一个云计算时代，应用程序环境变得越来越复杂，难以管理。
