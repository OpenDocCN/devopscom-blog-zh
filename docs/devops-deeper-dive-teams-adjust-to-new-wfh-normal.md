# DevOps 深潜:团队适应新 WFH 常态

> 原文：<https://devops.com/devops-deeper-dive-teams-adjust-to-new-wfh-normal/>

在新冠肺炎·疫情事件之后，软件开发活动的水平保持相对稳定，但是开发人员——和其他人一样——现在大部分都在家工作，投入了更多的时间。组织现在正在努力解决在软件开发项目[上远程工作是否代表了“新常态”](https://devops.com/the-way-forward-digital-resiliency-wins/)

GitHub 本周发布的一份[报告显示，虽然随着疫情病毒的蔓延，世界各地的活动略有下降，但公共 GitHub 知识库的活动迅速恢复。考虑到家里所有潜在的干扰，很难断定开发人员是否在项目上花了更多时间，但根据 GitHub 的报告，每个开发人员花在平台上的时间增加了。这可能是由各种因素造成的，包括开发人员没有通勤上班或在延长的工作日内管理多项任务(其中一些与工作直接相关)。其他人可能只是在活动间隙多休息一下。](https://github.blog/2020-05-06-octoverse-spotlight-an-analysis-of-developer-productivity-work-cadence-and-collaboration-in-the-early-days-of-covid-19/)

GitHub 的产品战略和营销副总裁 Kelly Stirman 说，尽管独立工作，但开发人员似乎仍然很有效率。"更多的编码时间可以提高生产率."

其他 DevOps 平台的提供商在疫情期间观察到类似的活动增加。CloudBees 高级副总裁兼软件交付管理和软件交付自动化云总经理 Moritz Plassnig 表示，许多组织一直在推出定制应用程序开发项目，以推动数字化业务转型计划。现在，这些项目正成为该组织业务连续性战略的核心。

“目前的形势只是加速了这一趋势，”普拉斯尼格说。

JFrog 首席数据科学家弗雷德·西蒙(Fred Simon)表示，管理高度分散的劳动力的最大挑战是确保在正确的时间编写正确的代码。“每个人都需要反馈。”

Simon 补充说，为了提供这种反馈，经理们采用了 Zoom 等协作工具，并在必要时将工作转移到作为云服务提供的 DevOps 平台上。

与此同时，DevOps 平台中的通信工具水平比以往任何时候都重要，桑德拉产品副总裁艾姆拉·萨姆丹(Emrah Samdan)说，他是一家提供保护和观察云原生微服务平台的公司。开发人员可以使用异步通信工具更有效地相互协作，例如共享代码。

“设置 Zoom calls 会损失很多生产力，”Samdan 说。

现在的问题是远程从事软件开发项目是否代表了新的常态。普华永道最近发布的一项针对首席财务官的调查发现，近一半的受访者(49%)计划将远程工作作为其组织中允许远程工作的职位的永久选择。

员工对这种变化的反应会有所不同。根据 GitLab 最近对员工进行的[调查结果，52%的员工表示他们远程工作的效率更高，而 48%的员工表示效率更高。当然，这些结果也表明，一半的员工对在家工作没有同样的欣赏程度。](https://www.globenewswire.com/news-release/2020/03/10/1998243/0/en/The-Remote-Work-Report-by-GitLab-The-Future-of-Work-is-Remote.html)

GitLab 的高级开发人员布道者 Brendan O'Leary 说，值得注意的是，在疫情之前，许多软件开发团队已经开始更多地远程工作。事实上，他指出，像 GitLab Remote Playbook 这样的资源在过去的几个月里被下载得更频繁了。

“我认为这是一个老板和领导者真正分离的时代，”O'Leary 说，并指出领导者将超越简单的跟踪指标，使组织能够继续推出创新的软件开发项目。

显然，在疫情之前以分布式方式工作的开发运维团队比那些主要在同一个中心位置工作的团队有明显的优势。然而，考虑到组织现在面临的所有不确定性，很可能 DevOps 团队的每个成员都将更加频繁地在家工作。
