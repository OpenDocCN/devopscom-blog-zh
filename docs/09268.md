# 对开发部门人才不断增长的需求

> 原文:[https://devops.com/the-rising-demand-for-devsecops-talent/](https://devops.com/the-rising-demand-for-devsecops-talent/)

对 DevSecOps 人才和技能的需求增长速度甚至超过了对 DevOps 的需求。

[全球 DevSecOps 市场](https://www.digitaljournal.com/pr/development-security-and-operations-market-size-to-hit-usd-23-42-billion-by-2028-exhibit-a-cagr-of-32-2)预计未来几年的 CAGR 将达到 32.2%，收入预计将从 2020 年的 25.5 亿美元增长到 2028 年的 234.2 亿美元。难怪 DevSecOps 的招聘数量在上升。

许多组织正在他们的工作流程中实现 DevSecOps。一些著名的名字包括网飞，Etsy，推特，脸书和谷歌。那些已经精通 [DevOps](https://wesecureapp.com/blog/what-is-devsecops-and-its-role-in-devops-architecture/) 并有安全背景的人在寻找工作时会比其他人更有优势。

然而，这不仅仅是工具和技术的问题——还有更多的东西需要开发。这也是关于文化和新的工作方式；关于自动化安全流程并使安全成为持续交付管道的一部分。

什么是 DevSecOps？是什么激励顶级商业领袖投资这一领域？获得这些技能对你的职业生涯有什么帮助？

## 什么是 DevSecOps？

DevSecOps 是一组实践，它将 DevOps 的速度和敏捷性与安全控制结合起来。它旨在帮助组织更快地交付软件，同时确保应用程序的安全性。“发展警察”一词是由约翰·威利斯和达蒙·爱德华兹在 2009 年首次提出的。他们将其定义为:

DevSecOps 是运营和开发工程师共同参与整个服务生命周期的实践，从设计到开发流程再到生产支持

后来，Gartner 给出了 DevSecOps 的如下定义:

“DevSecOps 是一种动态软件开发方法，它将安全性作为从设计到生产的应用程序交付过程中的一个关键要素。”

简而言之，DevSecOps 可以被定义为一种文化和一组过程，帮助组织更快地交付安全的软件。随着对 DevSecOps 专业人员的需求上升，这是软件工程师、开发人员和 DevOps 专业人员考虑在该领域发展职业的绝佳时机。以下是您应该考虑在 DevSecOps 工作的一些主要原因:

### 1.这是一个迅速发展的领域

DevSecOps 市场预计将在 2020 年至 2028 年间以 32.2%的 CAGR 增长。这种快速的增长为 DevSecOps 专业人员创造了大量的就业机会。

### 2.这是一个跨学科的领域

DevSecOps 不仅仅是工具和技术。这是一个跨学科领域，需要开发、运营和安全方面的技能。

### 3.高薪角色

根据最近的一项调查，一名开发人员的平均年薪为 123，000 美元或更高；远高于其他领域的平均工资。

### 4.使用最新技术

DevSecOps 完全是关于软件交付过程的自动化。它需要对最新的 DevOps 工具和技术以及自动化、人工智能和机器学习有深刻的理解。

### 5.引领最新潮流

作为 DevSecOps 专业人员，您需要站在行业最新趋势的最前沿。像这样的角色可以帮助你保持最新的技能，并保持领先地位。

## 为什么是主要投资？

以下因素激励着顶级商业领袖投资 DevSecOps:

### 1.对速度的需求

在当今快节奏的世界中，企业必须更快地交付软件。DevSecOps 帮助组织加速软件交付过程，同时确保应用程序的安全性。

### **2。对敏捷性的需求**

DevSecOps 使组织更加敏捷，能够适应不断变化的市场趋势。快速交付软件的能力有助于组织在竞争中保持领先，同时降低数据泄露、网络攻击和恶意软件的风险。

### 3.对安全的需求

随着网络攻击的不断增加，企业正在投资 DevSecOps 以确保其应用程序的安全性。DevSecOps 帮助组织在软件开发过程的早期检测并修复 [安全漏洞](https://wesecureapp.com/blog/red-team-assessment-versus-penetration-testing/) ，因为修复这些漏洞更容易且成本更低。

2021 年，全球约有 414，02809 起网络安全事件，这一数字可能会增加。在 DevSecOps 的帮助下，企业可以保护他们的应用程序免受攻击。

### 4.合规的必要性

在 DevSecOps 的帮助下，企业可以实现合规流程的自动化。它有助于节省花费在手动合规流程上的时间和资源。

### 5.节约成本的需要

DevSecOps 通过自动化软件交付过程来帮助组织节省资金。自动化有助于减少对人工的需求，从而有助于降低成本。

## 专业技能和要求

像任何其他领域一样，DevSecOps 也有自己的一套成功所需的要求和技能。虽然许多是特定于技术和工具的，但其他的是更一般的软技能。

### 1.对跨职能技能的需求

由于 DevSecOps 是一个跨学科的领域，担任该角色的专业人员需要既了解开发又了解运营。他们还必须熟悉软件开发、IT 运营、安全、CI/CD、容器等领域的最新工具和技术。

### 2.协作的需要

DevSecOps 是关于不同业务领域和不同团队之间的协作。专业人员需要能够与其他团队成员密切合作。

### 3.持续学习的需要

随着 DevOps 和安全领域的不断变化，专业人员需要不断更新技能，熟悉行业的最新趋势。

### 4.耐心的必要性

软件交付过程通常缓慢而复杂。对于习惯在快节奏环境中工作的专业人士来说，这可能会令人沮丧。

### 5.对适应性的需求

由于 DevOps 和安全领域在不断变化，专业人员需要具有适应性。他们应该能够快速学习新的工具和技术，并知道如何应用这些知识来解决技术问题。

## 入门指南

如果你想成为一名 DevSecOps 专业人员，你需要既懂软件开发又懂运营。您还应该熟悉最新的工具和技术。你可以阅读其他专业人士的博客，参加正式课程或参加编码训练营，在社区学院或大学参加课程或自学。你应该参加会议，与其他技术专家交流，分享知识。此外，你需要提高你的软技能，包括沟通、协作、团队合作和领导力。
