# 对比安全发布新功能以保护企业软件供应链

> 原文:[https://devo PS . com/contrast-security-releases-new-capabilities-to-secure-enterprise-software-supply-chains/](https://devops.com/contrast-security-releases-new-capabilities-to-secure-enterprise-software-supply-chains/)

*领先的 AppSec 提供商结合了针对构建、测试、生产和云原生环境中的定制和第三方代码的漏洞测试*

**加利福尼亚州洛斯阿尔托斯，2022 年 3 月 15 日—** [对比安全](https://www.contrastsecurity.com/)(对比)，代码安全领域的领导者，使开发人员能够保护他们的代码，今天宣布其安全代码平台现在集成了从开发人员桌面到生产系统的整个开发生命周期的软件供应链安全。新的集成使 Contrast 成为市场上第一个允许企业识别其最大供应链风险并防范这些风险的平台。

作为对 2021 年关闭殖民地管道的勒索软件攻击的直接回应，乔·拜登总统的[网络安全行政命令](https://www.contrastsecurity.com/security-influencers/president-bidens-executive-order-secure-the-software-supply-chain)对出售给联邦机构的任何软件实施了严格的标准。最近的零日事件，如流行的 Log4j Java 库中嵌入的日志注入漏洞，也迫使私营部门的企业重新评估开发人员进口、构建和使用的软件的安全地位。

Contrast Security 的联合创始人兼首席技术官杰夫·威廉姆斯(Jeff Williams)表示:“开源和定制代码共同构成了企业构建、购买和发布应用的要素。“单独测试这些软件成分缺乏上下文，会导致误报和漏报。为了准确识别漏洞，组织必须对整个集成应用程序或 API 进行安全测试，以揭示定制代码和开源代码是如何交互的。”

Contrast 将软件组合分析(SCA)与它的每个安全测试和保护解决方案集成在一起，包括业界领先的交互式应用安全测试(IAST)、运行时应用自我保护(RASP)和无服务器应用安全解决方案。与 Contrast 的静态应用安全测试(SAST)解决方案的集成即将推出。对比安全代码平台通过以下方式帮助企业缩小其软件供应链中的安全差距:

*   在原生 CI/CD 管道和云原生环境中同时测试自定义和第三方代码漏洞。
*   制定全面的软件物料清单(SBOM ),以帮助衡量软件供应链风险并满足监管和合规要求。
*   通过标记运行时实际调用的库，消除了对从代码库中提取的非活动库进行修复的需要。
*   在无服务器功能(如 AWS Lambda)等云原生工作负载中发现第三方安全问题
*   保护生产应用程序和 API 免受有针对性的攻击，无需打补丁或更改代码。

Contrast 的联合创始人兼首席技术官杰夫·威廉姆斯(Jeff Williams)计划于美国东部时间 2022 年 4 月 6 日星期二下午 2 点与 ESG Research 的高级分析师梅林达·马克斯(Melinda Marks)进行虚拟炉边聊天，讨论 Log4j 等最近的事件如何优先考虑软件供应链安全、SBOMs 的作用以及企业需要考虑的其他技术，以嵌入第三方组件的安全编码实践。要注册参加即将举行的 ESG 联合研究和对比网络研讨会，请访问[https://www.contrastsecurity.com/webinar-esg](https://www.contrastsecurity.com/webinar-esg)。

**关于对比安全性**

对比安全保护全球商业所依赖的代码。它是业内最现代、最全面的代码安全平台，消除了低效的安全路障，使企业开发人员能够更快地编写和发布安全的应用程序代码。Contrast platform 将代码分析和攻击预防功能直接嵌入到带有工具的软件中，在开发人员编写代码时自动检测漏洞，消除误报，并提供特定于上下文的操作指南，从而轻松快速地修复漏洞。这样做可以让应用和开发团队更有效地协作，更快地创新，同时加快数字化转型计划。这就是为什么越来越多的世界上最大的私营和公共部门组织依靠 Contrast 来保护其开发中的应用程序，并将保护扩展到生产中的云和本地应用程序。
