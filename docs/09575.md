# Styra 推出云服务以实现法规遵从性

> 原文:[https://devo PS . com/styra-un furls-cloud-service-for-implementing-compliance-as-code/](https://devops.com/styra-unfurls-cloud-service-for-implementing-compliance-as-code/)

Styra，Inc .今天推出了基于开放策略代理(OPA)软件的授权服务，可以通过应用程序编程接口(API)调用该软件。

Styra 开源副总裁 Torin Sandall 表示， [Styra Run](https://www.businesswire.com/news/home/20220713005153/en/Styra-Announces-Styra-Run-a-New-Application-Authorization-Service-for-Developers-Built-on-Open-Policy-Agent) 云服务将使在应用程序中嵌入企业级授权功能变得更加简单。如今，开发人员花费太多时间来构建、维护和操作他们自己的授权平台，以至于无法管理多个应用程序之间的权限。

除了使开发人员能够访问预建的 OPA 逻辑，Sandall 指出，该服务还将最终用户权限数据的存储和全局复制卸载到云服务。

他补充说，这项服务目前在亚马逊网络服务(AWS)上运行，但 Styra 计划未来在其他云平台上提供这项服务。

OPA 利用通用策略执行引擎，开发人员可以使用减压阀语言将该引擎嵌入到应用程序中。OPA 现在正在云原生计算基金会(CNCF)的支持下向前推进，并在云原生应用程序开发人员中获得支持。OPA 提供了一种方法，可以在各种微服务中强制执行代码合规性，这些微服务现在定期访问各种分布式数据。

然而，Sandall 指出，Styra Run 应该使 OPA 更容易被更广泛的组织使用，特别是那些希望将安全性和合规性的责任进一步留给开发人员的组织。他补充说，没有很多合规专业知识的开发人员现在可以依赖云服务，使他们能够将更多的时间和精力放在编写代码上。

Styra Run 的到来也正值在一系列高调的安全漏洞之后，更多的组织正在审查他们的[软件供应链](https://devops.com/?s=software+supply+chain)流程。尚不清楚合规性和网络安全团队对 OPA 的重视程度，但随着越来越多的组织接受合规性即代码，这些团队的压力应该会减少，因为这些应用程序本质上比传统应用程序更安全。

桑德尔说，当然，会有一些开发人员喜欢将 OPA 嵌入到应用程序中，但通过 API 访问云服务将是大多数开发人员构建软件即服务(SaaS)应用程序阻力最小的途径。问题可能只是让开发人员意识到存在另一个低延迟选项；他补充说，这不需要他们为了达到合规的唯一目的而深入掌握减压阀编程语言。

无论采用何种方法，未来的普遍预期是，应用程序将能够更轻松地满足广泛的法规遵从性要求，而无需 IT 团队部署和维护专用的法规遵从性基础架构。因此，可能很快就会有一天，任何不使用某种代码遵从性框架的应用程序都将被直接拒绝。
