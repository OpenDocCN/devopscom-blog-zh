# 尽管民主化，IT 部门比以往任何时候都更加重要

> 原文:[https://devo PS . com/尽管-民主化-it 部门比以往任何时候都更加集中/](https://devops.com/despite-democratization-it-department-more-central-than-ever/)

多年来，人们一直在谈论 IT 在组织内部的分散化，近年来的事件无疑加速了这一趋势。然而，它对本组织的成功从未如此重要。

根据 ManageEngine 最新发布的[IT in Work:2022 and Beyond](https://www.manageengine.com/survey/it-at-work-2022/us-canada.html)研究，越来越多的 IT 相关决策是在正式 IT 部门之外做出的。然而，调查还发现，正如非 IT 员工和业务部门在选择他们的技术组合时发挥更大的作用一样，IT 和其他业务领域之间的合作实际上有所增加，而不是像许多人担心的那样有所减少。

调查发现，76%的北美决策者报告说，他们的组织鼓励非 IT 员工使用低代码/无代码平台开发自己的应用程序，大多数部门，最值得注意的是质量控制(24%)和财务(21%)，正在独立地转向 AI/ML 解决方案来完成他们的工作。

这些发现呼应了早先由 IDG 为斯诺软件公司进行的调查。这项调查发现，67%的受访者认为，他们的 IT 预算至少有一半是由各个业务部门而不是 IT 部门控制的。有趣的是，78%的人认为这对于他们的组织来说是一个积极的发展，这将最终使他们的业务变得更加创新和敏捷。

也许是这样，但许多参与 ManageEngine 调查的北美 IT 决策者(48%)也认为，缺乏培训和基本技术知识(47%)阻碍了他们的员工充分利用现有技术。尽管如此，76%的受访者表示，他们的员工如今比疫情之前更懂技术。

除了这些优势之外，整整 99%的受访者表示，他们的组织还面临着 IT 分散化带来的挑战，最突出的是维护 IT 安全性(56%)、整体质量维护(41%)和持续支持的可靠性(37%)。

矛盾的是，尽管组织内部的技术自主性增加了，但 ManageEngine 调查发现，82%的北美业务和技术领导者认为 IT 和其他业务部门之间的协作在过去两年中有所增加。此外，89%的人认为 IT 部门的成功与其组织的总体成功直接相关。

S&P 全球市场情报公司 451 Research 的分析师 Scott Crawford 说:“在这方面取得成功的企业不仅要有鼓励 IT 和业务线协作的正确政策，还要有能够识别业务线经理独立部署的应用程序和云服务的技术。

很明显，[低代码/无代码平台](https://devops.com/?s=low-code/no-code+platforms)、云服务和其他云平台使非 It 决策者可以轻松获得业务技术，这使得 IT 决策分散化和民主化。有趣的是，IT 部门发现他们在组织中的相关性和重要性在增加，而不是像许多人担心的那样减少。那些已经成功并将继续成功的 IT 部门是那些发现自己有能力支持其各个部门已经做出的技术决策，而不是试图集中控制这些决策的部门。

ManageEngine 调查基于 3，300 名全球 IT 和业务决策者的反馈，其中 500 名受访者居住在北美。
