# Catchpoint 通过自动化 Web 应用程序测试实现实验

> 原文:[https://devo PS . com/catch point-enables-experiments-by-automating-we B- app-testing/](https://devops.com/catchpoint-enables-experimentation-by-automating-web-app-testing/)

Catchpoint 本周[扩展了其开源 WebPageTest 性能测试工具套件](https://www.globenewswire.com/news-release/2022/06/01/2454515/0/en/Catchpoint-Accelerates-Web-Optimization-Prototyping-with-No-Code-Experiments-on-WebPageTest.html),使 IT 团队能够运行即时测试，以提出改进整体 web 应用程序和网站性能的建议。

此外，Catchpoint 现在在 WebPageTest 工具套件中提供对私有测试、批量测试和优先级测试功能的支持。

WebPageTest 的工程总监 Tim Kadlec 表示，目标是让开发人员和 DevOps 团队更容易试验应用程序和网站的变化如何使用现已添加到核心套件的功能来提高性能。

![](../Images/d3316e36c30d7a5dc6fcc4cf35efd495.png)

Kadlec 指出，确定提高性能的机会通常需要花费数周时间进行繁琐的手动测试。大部分努力都是实验性的，因为没有人确切知道正在构建的测试是否会对性能产生实质性的影响。

机会和实验能力为团队提供了一个基于 Catchpoint 定义的一组最佳实践构建测试的低代码工具。这种方法使得自动生成和测试调整变得更加简单，例如，消除阻塞脚本、优化图像渲染和最小化布局偏移。Kadlec 说，结果是优化测试表明，他们的应用程序和网站如何从具体的、可操作的改进中受益，而不必改变任何已经部署的代码。

这是一个关键的能力，因为它将鼓励以前 it 团队回避的实验水平，仅仅因为实现不确定的结果所需的工作量太大。

目前还不清楚谁在进行这些类型的测试，因为测试应用程序的更多责任继续转移到开发人员身上。Kadlec 指出，WebPageTest 的最新添加使得开发人员更容易进行这些测试，而不一定需要 DevOps 团队的帮助。

虽然 web 应用程序和网站性能一直备受关注，但在数字业务转型时代，各组织的关注度都有所提高。许多通过 web 应用程序实现的流程对延迟极其敏感。随着越来越多的应用程序成为例行测试，应用程序性能的整体水平应该会稳步提高。

与此同时，大多数组织支持的 web 应用程序和网站的数量也在继续增加。这些组织中的大多数都在寻找测试应用程序的方法，而不必扩大测试团队的规模或显著增加花费在测试上的时间。

不管采用什么方法，很明显测试正在变得更加自动化。我们的目标并不是消除对专门测试团队的需求，而是为了节省时间，减少导致测试回避的机械任务的数量。理论上，测试运行得越多——越早——devo PS 团队稍后需要处理的问题就越少。
