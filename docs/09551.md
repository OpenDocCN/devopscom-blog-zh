# 超过一半的 DevOps 专业人员可以通过后门访问 IT 基础架构

> 原文:[https://devo PS . com/超过一半的 devo PS-pros-have-back door-access-to-it-infra structure/](https://devops.com/more-than-half-of-devops-pros-have-backdoor-access-to-it-infrastructure/)

strongDM 是一个管理[访问 IT 基础设施](https://devops.com/?s=infrastructure+access)的平台，该平台对 600 名 DevOps 专业人员进行的[调查](https://www.prnewswire.com/news-releases/new-research-57-of-organizations-take-days-or-weeks-for-infrastructure-access-to-be-approved-301573622.html)发现，近三分之二(64%)的人因为访问问题每天或每周的工作效率都受到影响。

调查显示，毫不奇怪，DevOps 团队因此创建了许多变通方法来获取访问权限，尽管这些方法不符合网络安全最佳实践。调查发现，超过一半的受访者(55%)保持对系统的后门访问，42%的受访者采用影子 IT 来访问其角色所需的工具和系统。

更糟糕的是，调查发现，53%的受访者承认他们在团队之间共享凭证。

超过一半(57%)的受访者表示，访问请求可能需要数天或数周才能获得批准，40%的受访者表示，向现有访问解决方案添加工具需要数天时间。近四分之三(73%)的受访者还指出，访问基础架构需要 15 分钟或更长时间。

超过一半的受访者(52%)表示，由于访问基础架构的问题，他们错过了最后期限，其中 53%的受访者对错过最后期限负有责任，即使这是因为他们无法访问基础架构。

strongDM 的 CTO 贾斯汀·麦卡锡(Justin McCarthy)表示，尽管围绕身份管理和零信任 IT 进行了大肆宣传，但很明显许多组织仍在努力解决基本的访问控制问题。他补充说，自从两年多前新冠肺炎疫情成立以来，由于需要让更多的 DevOps 专业人员在家工作，这个问题变得更加严重。

McCarthy 补充说，许多这些组织仍然依赖虚拟专用网络(VPN ),尽管事实证明窃取访问这些 VPN 的凭据相对来说微不足道。

代替这些访问 IT 基础设施的方法，strongDM 正在为管理和审计对数据库、服务器、集群和 web 应用程序的访问的专用基础设施访问平台进行论证。它基于本地客户端软件、网关中介和跟踪角色和权限的配置工具。

McCarthy 指出，这种方法的好处涵盖了从降低 IT 支持成本到加快新 IT 员工入职的方方面面。

当然，管理 DevOps 团队的基础架构访问始终是一项挑战。开发运维团队由工程师组成，是最难支持的终端用户类型之一。他们中的许多人会利用自己的专业知识来规避任何他们认为效率低下的流程。问题是，这些终端运行中的许多都不符合负责访问管理的网络安全团队的要求。每个网络安全团队需要应对的挑战是，他们希望花多少时间来寻找这些网络安全政策变通办法，还是让安全访问 it 基础架构变得更容易。

可悲的事实是，DevOps 和网络安全专业人士都在浪费大量的时间和精力，试图以智取胜，而这些时间本可以更好地利用。
