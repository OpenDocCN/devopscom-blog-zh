# 多速 IT 与双模 IT

> 原文：<https://devops.com/multi-speed-versus-bi-modal/>

我想继续回顾上个月在拉斯维加斯举行的 IBM 互联会议。周一，我们发布了我们的第一批视频,以此拉开序幕。从那时起，我们刚刚更新了新一批可用视频的 [YouTube 播放列表](https://www.youtube.com/playlist?list=PLotLY1RC8HovnuFHxrmFzTG24oe5YW4Dq)。我们将在接下来的几天里添加这些内容。在这篇文章中，我想专注于一个我感兴趣并且以前探索过的主题，即多速 IT 与双模 IT。

我知道 Gartner 引领了术语“双峰 IT”的普及。许多人已经接受了这个术语，意思是 IT 组织的一些遗留部分将不能以组织的一些更新的 DevOps 授权部分的速度运行。可以在工作。所以不同的部分以不同的速度运行是可以的。你实质上已经不止双峰，而是几乎分叉了。

我对这个场景的问题始于双峰 Bi。Bi 表示两个，就好像这个等式中只有两个部分。一个以一种速度运动，另一个以另一种速度运动。在现实生活中，组织有两个以上的部分以不同的速度运行。所以多速比双峰合适多了。

不过，我的问题不仅仅是比较两个和多个。我不同意认为遗留系统不能从开发运维、敏捷、精益等中获益。正如我们所看到的，即使是大型铁主机也可以用 DevOps 制造得更快。因此，虽然 IT 组织的不同部分可以以不同的速度移动，并且自然会这样做，但这并不意味着遗留记录系统不能使用 DevOps 来加速。

在我看来，多速 IT 的整个组织更像是瑞士表中的齿轮。不同大小的不同齿轮，以不同的速度运转，但所有这些齿轮都像一个精致的计时器一样相互配合。
