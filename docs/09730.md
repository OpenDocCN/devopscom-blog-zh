# VMware 采用图形技术来管理多个云

> 原文:[https://devo PS . com/VMware-enchances-graph-technology-to-manage-multi-clouds/](https://devops.com/vmware-embraces-graph-technology-to-manage-multiple-clouds/)

在今天举行的 [VMware Explore 2022](https://www.vmware.com/explore/us.html) 大会上，VMware 推出了一个被称为 [VMware Aria](https://news.vmware.com/releases/vmware-explore-2022-multi-cloud-management) 的多云计算管理工具产品组合，该产品基于一个基于图形的数据存储，可捕获跨多个云计算的资源及其相互关系。

VMware Aria Graph 通过 VMware Aria Hub 平台(以前称为 Project Ensemble)进行访问，可在由数百万个节点组成的云计算环境中近乎实时地提供应用程序、资源、角色和帐户的通用定义。

VMware 首席执行官 Raghu Raghuram 告诉与会者，Aria Graph 是一项努力的核心，旨在使 IT 组织能够超越“云混乱”,采用下一代“云智能”方法来管理跨多个平台的分布式计算环境。

基于事件驱动的平台，VMware Aria Graph 将从公共应用程序编程接口(API)提取的数据聚合到驻留在云上的数据存储中。数据存储基于联邦模型，因此不需要将数据移动到中央存储库中。相反，VMware 正在利用 [GraphQL](https://devops.com/?s=GraphQL) 来启动跨联合数据模型的查询。此外，IT 团队可以发现依赖关系并分析历史配置数据，以发现任何问题的根本原因。现在，所有运营数据都以一种创建单一真实来源的方式进行聚合。

VMware 还提供了一组 API 服务，可用于将 Aria Graph 数据存储与第三方平台集成。Raghuram 表示，总的来说，该公司致力于加速由[数字业务转型](https://digitalcxo.com)计划推动的 IT 环境的“大平台化”。他指出，IT 组织面临的挑战是各种障碍，包括缺乏技能、现有企业应用程序的重量以及分散的运营和网络安全模式。

Raghuram 说，VMware Aria graph 数据存储将使该公司能够自动化广泛的管理任务。Raghuram 补充说，这包括为正确的应用程序使用正确的云，并以尽可能低的成本管理和访问它们。他说，最终目标是实现统一的开发者体验，在一套一致的企业基础设施上提供无摩擦的多云体验。

利用 VMware Aria 图表的 VMware 产品包括用于实施 IT 策略的 VMware Aria Guardrails、使用 VMware HCX 平台自动执行评估、规划和执行的 VMware Aria 迁移工具，以及使用机器学习算法展示业务见解的 VMware Aria 业务见解工具。VMware Aria 产品组合的其余部分围绕现有平台的扩展，以由 CloudHealth、VMware Aria Operations 和 VMware Aria Automation 支持的 VMware Aria Cost 的形式存在。

VMware 今天还宣布，其现有的用于跨多个云部署软件堆栈的 VMware 跨云服务产品已经扩展到添加对 Microsoft Azure 云的支持。

总体而言，VMware 声称其生态系统中现在有 500 万开发人员，他们每周共同启动 350，000 个项目。根据 Raghuram 的说法，这相当于大约每分钟 35 个新应用程序。总而言之，VMware 声称全球有 8500 万个工作负载在其平台上运行。

如今，这些工作负载中的大部分仍在内部 IT 环境中运行。事实上，Raghuram 指出，云平台仍然只占每年 4130 亿美元的 IT 支出，这只是每年达到 4.4 万亿美元的总 IT 支出的 10%。

VMware 预计将在未来几个月内完成 Broadcom 对其[的收购，并押注随着越来越多的工作负载迁移到云，集中管理异构 IT 环境的需求将发挥其优势。](https://devops.com/broadcom-acquires-vmware-to-expand-software-portfolio/)
