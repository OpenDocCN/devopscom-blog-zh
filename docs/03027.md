# DevOps 聊天:中断:运营使安全性更接近 DevOps

> 原文:[https://devo PS . com/devo PS-chat-disrupt ops-bring-security-closer-to-devo PS/](https://devops.com/devops-chat-disruptops-brings-security-closer-to-devops/)

对于公司来说，转移软件开发过程中留下的安全性可能很困难，因为在开发运维领域，实现安全性必然会伴随着大量的操作性工作。如果没有某种护栏的帮助，很容易失去对安全流程的控制。

一家全新的公司正在帮助做到这一点。我很高兴向大家介绍 disrupt:Ops，它自称是一个 guardrails 平台，旨在帮助公司在安全和运营方面纳入和实施最佳实践，从一开始就将安全性引入开发运维流程。我的朋友 Mike Rothman 是 disrupt:Ops 的联合创始人兼总裁，他和我一起参加了这个 DevOps 聊天，讨论他的公司的愿景，以及它希望如何帮助安全真正成为 DevOps 的一部分。

你可以从公司的另一位联合创始人，我在 https://devops.com/building-great-cloud-security-guardrails 的朋友 Rich Mogull 那里了解更多关于 disrupt:Ops 和他们对云安全和 devsecops 的护栏方法
