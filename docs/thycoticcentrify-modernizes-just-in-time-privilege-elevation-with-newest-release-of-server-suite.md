# ThycoticCentrify 通过最新发布的服务器套件实现了即时特权提升的现代化

> 原文：<https://devops.com/thycoticcentrify-modernizes-just-in-time-privilege-elevation-with-newest-release-of-server-suite/>

*业内领先的增强型特权提升平台利用 Centrify 客户端的强大功能，确保对时间关键型活动的及时访问*

**加利福尼亚州圣克拉拉市和华盛顿特区**，**2021 年 7 月 14 日** ― [由特权访问管理(PAM)领导者 Thycotic 和 Centrify 合并而成的云身份安全解决方案领先提供商 thycoticscentrify](https://www.centrify.com/)，今天宣布了其对服务器套件的最新更新，包括通过 Centrify 平台动态更新 Centrify 客户端来优化[即时(JIT)](https://www.centrify.com/solutions/capability/just-in-time-jit-pam/) 特权提升工作流的新功能。该公司的旗舰产品特权提升和授权管理(PEDM)解决方案的最新版本现在包括用于 UNIX 的会话审计数据屏蔽功能，以降低暴露潜在敏感或高度受限数据的风险。

最小特权原则被认为是支持零信任和零特权的基本 PAM 最佳实践。当系统停机或违规调查等管理任务需要额外的访问权限时，时间是至关重要的。然而，尽管十多年来 Active Directory (AD)已经证明了其作为中央角色管理平台的价值，但将更新的角色传播到端点可能需要几个小时，这可能会带来灾难性的后果。

在 2021 版中， [Server Suite](https://www.centrify.com/resources/data-sheets/centrify-server-suite/) 通过来自平台的相互认证的通信通道同时更新 AD 和 Centrify 客户端权限策略，从而克服了这一问题。一旦管理员的访问权限获得批准，本地客户端就可以实施更新的策略，允许用户立即登录并根据需要提升权限以进行调查和修复。因此，访问被及时授予和可用，而不损害最少的特权。这一功能之所以成为可能，是因为 Server Suite 基于客户端的体系结构，该体系结构还可以实施更高级的 PAM 功能，如实时密码协调、委派的机器凭证和代理身份验证。

“网络攻击的速度正在加快，这意味着管理员需要更快地更新和保护资源，同时仍然保持实施最低权限的控制，”ThycoticCentrify 首席技术官大卫·麦克尼利(David McNeely)说。“在最新版本的 Server Suite 中，我们通过删除额外步骤来简化即时特权访问，使组织能够通过消除基于角色的特权访问权限分配来采用“零特权”安全模式。我们的轻量级客户端和 PAM 平台在所有特权身份(无论是人还是机器)之间建立了信任的基础，以更好地区分朋友和敌人，并降低风险。”

服务器套件的[审计&监控服务](https://www.centrify.com/pam/audit-monitoring/)还包括新功能，旨在限制暴露审计日志中捕获的密码或其他敏感事件。UNIX 数据屏蔽解决了高度管控行业面临的一个关键挑战，在这些行业中，静态数据通常是可见的，例如，当审计数据被转发到 Splunk 等第三方事件管理工具时。现在，日志文件中的敏感数据在服务器上被屏蔽，因此原始数据永远不会暴露。Server Suite 还增加了审计功能，如定制提示(包括语言)，审计向 AD 报告状态，以及提高 Windows 10 上的 CPU 利用率。

多重身份验证(MFA)和芯片组支持的其他增强功能包括:

*   MFA 后对重复 Radius 密码提示的静默身份验证
*   控制台和远程桌面协议(RDP)会话的宽限期控制
*   支持 MacOS 的 M1 芯片
*   AMD ARM 处理器架构的 DirectControl 支持(aarch64)
*   支持 Ubuntu 工作站的 AD 用户证书智能卡认证
