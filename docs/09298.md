 *## 1.“言论自由不是一个口号，”EFF 告诉马斯克

本周第一件事:埃隆·马斯克正在收购 Twitter(以防你一直生活在岩石下)。他说他想让平台上的言论更加自由。但是怎么做呢？什么是优先事项？。

### 分析:关注意图，而不是内容

忽略那些关于左派对右派或者觉醒对红皮的政治废话。马斯克先生应该把目标对准机器人、天文学家和足球运动员——特别是那些由民族国家控制的旨在散播社会不和谐的机器人。

EFF 的 Jillian C. York，Gennie Gebhart，Jason Kelley 和戴维·格林 : **Twitter 有了新主人；下面是他应该做的**

埃隆·马斯克对 Twitter 的收购凸显了当任何一个人完全控制影响近 4 亿用户的政策时，人权和人身安全面临的风险。[它]在社会和政治话语中扮演着越来越重要的角色，而且[因此]有责任确保……决策既透明又负责。言论自由不是一句口号。…它长期以来一直在努力应对机器人和主要人物的令人不安的推文…让错误或虚假信息迅速传播。与此同时……Twitter 的社区标准限制受法律保护的言论,[它]定期删除不违反*标准的内容。
……
更好的内容审核是迫切需要的:更少的自动化，更多的专家对政策的投入，以及更多的整体透明度和问责制。不幸的是，[关于]内容适度的讨论是令人沮丧的二元对立，评论者要么呼吁更多……要么就像马斯克的情况一样，呼吁少得多。*

埃里克·比尔认为这是垂直整合的终极:

[Twitter]被证明是最有影响力的“注意力导演”，迫使所有新闻媒体重复少数超级用户希望他们重复的内容，而且成本非常低。他一直在练习如何通过他的粉丝放大他的推文，这是一个他非常擅长的方法。如果他不能被平台取消，谁知道一年后我们会被什么疯狂的推文所左右。

给你一个激进的建议，这是[罗伯特](https://news.ycombinator.com/item?id=31164634 "read the full text"):

雇佣私人雇佣兵去燃烧弹机器人农场。他拿到钱了。雇佣那些 YouTube 黑客，他们用 pwn 欺骗呼叫中心来找到坏人，赶走所有人，烧毁服务器、电话和大楼。现代社会太讲礼貌太讲文明了。

## 2.我们会看到一个没有密码的未来吗？

这是一个诱人的愿景——用户不再需要记住唯一的凭据。不再有不一致的密码构造规则。不再有单点攻击密码管理器。也没有更多善意的建议如 **[正确的马电池钉书钉](https://xkcd.com/936/)** 。

### 分析:细节中的魔鬼—稍后再问

我的*魔法 8 号球*让我失望了(问问你的父母)，但它看起来并不好。智能手机是证明你身份的一种不完美的方式，像 FIDO 这样的标准似乎已经被 megacorps 劫持了**。**

小汤姆·哈德斯顿 : **一些科技巨头想要一个没有密码的未来——这就是未来的样子**

梦想一个没有人必须不断更新和更改在线密码以领先于黑客并保持数据安全的未来是如此诱人。好消息是:一些科技界的大腕已经表示，无密码互联网的梦想即将成为现实。
…
超过 80%的数据泄露是由弱密码或密码泄露造成的。(但是)完全取消密码并非没有风险。首先，通过电子邮件或短信发送的验证码可以被拦截。…其次，当今的一些无密码选项仍然要求您创建 PIN 或安全问题来备份您的帐户。这和有密码没多大区别。…第三，还有一个广泛采用的问题…虽然绝大多数美国人都拥有智能手机，但这些设备在年龄和内部硬件方面存在巨大差异。
……
换句话说，密码完全消失可能还需要一段时间。尽情地在登录框中输入你的长而复杂的字符串吧。

人们担心的是智能手机的角度。齐特奇斯:

我们手机丢了怎么办？…提高数字安全性是好事，但仅仅将这一切建立在拥有一部特定手机的基础上似乎是危险的。对我认识的每个人来说，他们与智能手机的关系一直是，“这不是‘如果’它会坏的问题，而是‘它多久会坏？’“
…
手机发生了很多事情，我不想因为它发生了什么事情而冒着失去我整个数字世界的风险。我听到不止一个安全专家说，他们完全认为他们的手机在任何时候都会受到威胁，不管它有多新或最近重置过……并相应地对待它。

但是菲多呢？上个月，当我为我们的姐妹刊物编辑[这篇文章](https://securityboulevard.com/2022/03/bad-dog-everyone-hates-this-fido-passwordless-idea/ "Everyone HATES This FIDO Passwordless Idea")时，我有点沮丧:

揭开表面，FIDO2-WebAuthn 似乎让“侵犯隐私的大公司”从你的私人数据中获利。通常的嫌疑人…已经接管了。他们试图将你锁定在他们的生态系统中，用他们惯用的“热门企业垃圾”品牌通常在 SB Blogwatch 中，我会尝试讲述故事的双方。但是今天，我找不到任何人认为 FIDO 是个好东西。

## 3.欧盟将迫使苹果允许在 iOS 上使用其他浏览器

如今，苹果不允许在 iPhones 和 iPads 上使用其他浏览器。当然，你可以安装一些叫做 Chrome 或 Firefox 的东西，但在引擎盖下，它们都是 wrappers 的圆 WebKit。如果您的应用程序是 PWA、渲染网页、运行 JavaScript 或出于任何目的调用渲染器 API，也是如此。它甚至不是 Safari 使用的最新版本的 WebKit 批评者说**苹果是反竞争的**。

### 分析:欧盟出手相救？

数字市场法案(DMA)来了——一项旨在解决许多高科技不公正问题的欧盟法规草案。最新的草案包括一项条款，该条款将阻止苹果公司实施这一噱头。

[Thomas Claburn](https://www.theregister.com/2022/04/26/apple_ios_browser/ "read the full text") : **根据最新的欧盟规则草案，苹果对 iOS 浏览器引擎的控制是不允许的**

欧洲的[DMA]包含旨在终止苹果 iOS 浏览器限制的语言。…苹果要求竞争对手的移动浏览器…使用自己的 WebKit 渲染引擎，这是其 Safari 浏览器的基础。结果是 iOS 上的 Chrome、Edge 和 Firefox 或多或少都是 Safari。苹果从现状中获利的程度已经引起了欧洲、英国、美国和其他地方的监管审查。…现在这些努力已经转化为 DMA 的文本，它与数字服务法案一起…定义了欧洲将如何管理大型技术门户。
……
【我们】向苹果公司寻求评论。我们没有期待得到答复，也没有收到答复。然而，苹果清楚地认识到，它不能一边指出网络应用是原生 iOS 应用的竞争对手，一边又要求使用外人认为阻碍网络发展的浏览器引擎，从而逃避反垄断监管。

值得庆祝的原因，认为[Gary market 27](https://news.ycombinator.com/item?id=31165867 "read the full text"):

如果真的发生，这是个好消息。苹果公司显然有一点点动机来提高移动网络应用的能力。从长远来看，这将带来巨大的变化，使真正的跨平台开发变得更加容易。

开发人员应该关心吗？是的，特兰希尔认为:

这不是(仅仅)你必须使用 WebKit，而是可以在应用程序(或竞争浏览器)中使用的 WebKit 版本…不是最新的。……Safari(最近)支持我们需要的 API ,(但是)Safari 以外的任何设备使用的旧 WebKit 引擎都不支持这些 API。
