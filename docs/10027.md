# CloudBees 调查发现 VSM 正在推动数字化转型

> 原文:[https://devo PS . com/cloud bees-survey-sees-VSM-driving-digital-transformation/](https://devops.com/cloudbees-survey-sees-vsm-driving-digital-transformation/)

CloudBees 对 600 名高级管理人员进行的一项全球调查(T1)表明，在价值流管理(VSM)和数字业务转型(T3)的成功(T2)之间有很高的相关性。

调查发现，90%的受访者认为，致力于 VSM 的组织可能会在实现数字业务转型目标方面走在前面，92%的受访者指出，VSM 还创造了更多时间来专注于创新。

CloudBees 平台高级总监霍普·林奇(Hope Lynch)表示，挑战在于太多高管将 VSM 等同于传统的六适马流程，而不是推进数字业务转型计划的应用开发和部署流程。调查发现，虽然 90%的高管声称他们熟悉 VSM 概念，但只有一半的人能在被问及时选择正确答案。事实上，五分之二的受访者混淆了价值流管理和价值流图的概念。近三分之二(72%)的受访者承认，对于 VSM 是什么仍然存在困惑。

VSM 将其血统追溯到精益制造方法，这要求对制造过程的每一步进行连续测量。随着软件开发的发展，组织意识到他们对软件的依赖程度，越来越多的人开始意识到监控事情的价值，比如错过软件开发期限对业务的影响。VSM 平台的核心是聚合 DevOps 指标，然后将它们与业务领导者定义的一组关键绩效指标(KPI)相匹配。一旦收集并关联了这些指标，组织就可以应用高级分析。

Lynch 指出，最终目标是应用算法和其他形式的预测分析，使组织能够更容易地确定软件开发过程对业务的精确影响。

与此同时，CloudBees 的调查将经理承诺视为采用 VSM 的主要障碍，35%的受访者表示让中层员工承诺，31%的受访者表示让高层领导承诺是最大的障碍。这是一个很重要的问题，因为超过一半的受访者(56%)表示，VSM 的投资决策是由首席执行官做出的。

调查受访者还表示，在整个组织中推广 VSM 工作是三分之一(33%)受访者的一个重要问题。

林奇指出，总的来说，VSM 的许多倡议都被新冠肺炎·疫情搁置了。然而，她补充说，各组织现在将着眼于评估这些努力的结果。96%的受访者表示，数字化转型计划已经到位并且进展顺利，只有 18%的受访者表示正在实施，但速度不够快。

然而，Lynch 补充说，所有软件开发过程都可以通过呈现实时洞察来改进。她补充说，如今大多数高级管理人员对推动多项数字化转型计划的软件开发流程缺乏足够的了解。

理论上，VSM 平台有一天应该有助于缩小 IT 和业务其他部分之间长期存在的鸿沟，因为仪表板使每个相关人员都更容易理解软件开发决策对整个业务的级联影响。
