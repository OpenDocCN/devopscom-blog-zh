# 哈希公司增加了 Terraform 的企业吸引力

> 原文:[https://devo PS . com/hashi corp-increases-terra forms-enterprise-appeal/](https://devops.com/hashicorp-increases-terraforms-enterprise-appeal/)

在今天的在线[hashi conf Europe](https://hashiconf.com/europe/)会议上，HashiCorp 首次亮相[HashiCorp Terraform 1.0 版本的正式发布以及 hashi corp terra form 云服务的更新](https://www.globenewswire.com/news-release/2021/06/08/2243463/0/en/HashiCorp-Delivers-Terraform-1-0-Milestone-Release-for-Multi-Cloud-Provisioning-Product-with-100-Million-Downloads-and-Over-5-000-Ecosystem-Add-Ons.html)。

HashiCorp 的产品营销高级总监 Meghan Liese 表示，在这两种情况下，HashiCorp 都在努力使组织能够更好地保护和管理使用开源 Terraform 工具供应的 IT [基础设施。](https://devops.com/?s=infrastructure+as+code)

HashiCorp Terraform 版提供了稳定的应用程序编程接口(API)，可促进互操作性并简化未来升级。

Terraform 云服务消除了 IT 团队设置和管理 Terraform 平台的需要，为 Terraform 云添加了工作流增强功能，包括将管理的模块发布到私有注册表的能力，使工作区元素可视化更容易的增强功能，以及使用与 Terraform 运行相关的第三方工具进行检查的预览功能。

Liese 说，总的来说，目标是使用模块和公共及私有注册选项，使组织更容易组合、协作和重用基础设施即代码。公共注册处已经有 5000 多个模块。Terraform Cloud 现在提供了一个本机工作流，可以将模块从公共注册中心直接发布到组织的私有注册中心。

当然，自 2014 年首次亮相以来，Terraform 已被广泛采用。在 Terraform 1.0 发布之前，已有 15 个主要更新，旨在提供一个基础，以确保未来版本之间的向后兼容性。

在某些情况下，Terraform 可能是太多的好东西。它大大加快了在开发运维工作流环境中调配 It 基础架构的速度。然而，面临的挑战是，在网络犯罪分子越来越擅长扫描这些漏洞的时候，开发人员往往缺乏防止基础设施被错误配置所需的安全专业知识。为了使 IT 团队能够防止这些错误配置，HashiCorp 提供了 Terraform 企业版及其 Terraform 云服务。超过 1，200 家企业使用 Terraform Enterprise，而 Terraform Cloud 拥有超过 120，000 名用户，每天执行超过 30，000 次运行。

尚不清楚安全问题是否有一天会导致对将更多责任转移给开发人员以提高生产率这一概念的强烈反对。在某些情况下，开发人员只是更快地部署更不安全的应用程序。HashiCorp 正在为 Terraform 平台提供一个案例，该平台允许集中式 IT 团队对如何以代码形式供应基础架构进行更多控制。对这种方法的需求正变得越来越明显，因为组织不仅采用 Terraform 来调配云基础架构，还采用内部 IT 环境。

无论开发人员如何调配基础架构，随着部署的工作负载比以往任何时候都多，云内外基础架构的安全性仍然是重中之重。实际上，需要防御的攻击面已经急剧扩大。当然，最好的安全问题是从未发生过的问题，因为基础架构在第一次和以后每次都是正确配置的。
