# 白宫会议聚焦 OSS 可持续发展

> 原文：<https://devops.com/white-house-meeting-puts-spotlight-on-oss-sustainability/>

IT 行业领袖和白宫官员之间最近的一次会议强调了开源软件可持续发展的问题，因为高调的违规和零日攻击已经使许多组织审查他们的软件供应链。

白宫发表了一份声明,其中描述了参与者如何就如何在开源软件的安全性方面有所作为，同时继续有效地参与和支持开源社区进行了“实质性和建设性”的讨论。

与会者包括负责网络和新兴技术的副国家安全顾问 Anne Neuberger、国家网络主任 Chris Inglis，以及来自国家网络主任办公室、科学和技术政策办公室、国防部、商务部、能源部、国土安全部、网络安全和基础设施安全局(CISA)、国家标准和技术研究所以及国家科学基金会的官员。Akamai、亚马逊、阿帕奇软件基金会、苹果、Cloudflare、脸书/Meta、GitHub、谷歌、IBM、Linux 基金会、开源安全基金会、微软、甲骨文、红帽和 VMware 都派出了代表。

白宫报告称，讨论的重点是防止代码和开源包中的安全缺陷和漏洞，改进发现缺陷和修复缺陷的流程，并缩短分发和实施修复的响应时间。

几乎没有具体的建议，但在 Java 应用程序中的零日 [Log4j 漏洞](https://devops.com/?s=Log4j)在企业 IT 环境和政府机构中造成严重破坏后，白宫显然施加了更多压力。该漏洞清楚地表明了组织对开源软件项目的依赖程度，并使人们注意到这样一个事实，即许多开源软件是由少数志愿者维护者和贡献者创建和维护的。创建这些项目的个人并不总是有很多网络安全专业知识。事实上，他们中的许多人会争辩说保护开源软件的责任在于那些使用相当于自由软件的组织。开源软件的贡献者和维护者没有责任放弃一切，按需创建补丁来解决零日漏洞。

然而，联邦政府已经通过[行政命令](https://securityboulevard.com/2021/05/how-bidens-eo-impacts-cybersecurity-players/)明确表示，它希望依赖开源软件的 it 供应商和大型企业采取更多措施来保护它。同时，IT 团队将需要评估他们对开源软件的依赖；尤其是从安全的角度来看，仅仅因为没有足够的具有必要专业知识的贡献者，软件就不可持续。

自然，这是一个复杂的问题。在许多情况下，组织依赖开源组件，甚至没有意识到这一点。独立软件供应商(ISV)将这些组件嵌入到应用程序中，通常不会透露该应用程序是如何构建的。当零日漏洞被披露时，网络安全团队可能会花费数周时间来寻找他们自己构建或许可的应用程序实例受到影响的所有方式。

出版《DevOps.com》的母公司 Techstrong 集团的一个分支机构 Techstrong Research 的负责人米奇·阿什利(Mitch Ashley)说，“没有办法真正知道。“漏洞可能在任何地方。”

开源软件安全的可持续性到底有多重要还不太清楚。Ashley 指出，相对于目前使用的开源软件的数量，[遇到的安全问题相对较少](https://devops.com/log4j-is-there-such-a-thing-as-too-much-open-source/)。

“开源软件带来的创新收益远远大于风险，”他说。

开源社区可能还需要一段时间来适应日益增长的软件供应链问题。然而，很明显，更多的 it 社区成员将为此承担责任。
