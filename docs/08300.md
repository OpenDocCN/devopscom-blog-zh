# 开发运维团队利用更新的云成本控制工具

> 原文:[https://devo PS . com/harness-updates-cloud-cost-control-tool-for-devo PS-teams/](https://devops.com/harness-updates-cloud-cost-control-tool-for-devops-teams/)

Harness 今天宣布，它已经为其同名的持续集成/持续交付(CI/CD)平台更新了云成本管理( [CCM](https://harness.io/platform/cloud-cost-management/) )模块，除了提供商业智能工具来分析整体云成本之外，还在节点级别添加了对 Kubernetes 的支持。

Harish Doddala 是 Harness 的产品管理高级主管，他说，这次更新对 DevOps 团队因云资源被浪费而产生的云成本有了更深入的了解。CCM 的最新版本使 DevOps 团队能够创建他们自己的那些资源的定制视图。

随着整体经济继续从新冠肺炎疫情中复苏，越来越多的组织希望更好地控制 IT 成本。在疫情之后，许多 DevOps 团队在不考虑成本的情况下消耗云资源，以使组织能够推进各种数字化业务转型计划。现在，财务团队要求开发运维团队考虑这些云成本。

我们的目标不仅是尽可能降低成本(例如，通过快速识别云资源消耗的突然峰值)，还包括每月准确预测预算成本，以便 IT 和财务团队能够协作跟踪支出。典型的财务团队对准确预测成本的关注程度不亚于对总支出的关注程度。

![Harness Cloud Cost](../Images/6548eb5ac1d3aedbe90f6112a0abf071.png)

最后，Doddala 说 Harness 的 CCM 使组织能够更好地评估基于分配的云资源优先考虑哪些软件开发项目。随着业务条件的变化和发展，组织不得不突然改变软件开发的优先级，这种情况并不少见。一个今天可能是最高优先级的项目很容易被降级，因为，例如，一个竞争对手已经能够推出一些组织现在需要快速匹配的新功能。

不幸的是，DevOps 团队并不总是关注基础设施成本。多达拉说，Harness 的 CCM 通过自动停止亚马逊网络服务(AWS)、微软 Azure、谷歌云平台(GCP)或任何 Kubernetes 集群上运行的闲置资源来帮助降低这些成本。

当然，云服务提供商可以提供任意数量的现有工具来帮助 IT 团队降低成本。然而，Doddala 指出，CCM 在 DevOps 平台的上下文中提供了这种能力，可以跨越多个云计算环境。例如，开发运维团队可以在 CI/CD 渠道中创建和实施云基础架构的成本策略，并防止组织在月底收到云服务提供商的账单时感到惊讶。

尚不清楚成本控制模块将在多大程度上影响一个 CI/CD 平台的选择。然而，很明显，组织消耗云资源而不考虑成本的日子即将结束。事实上，可能很快就会有比疫情之前更多的对软件开发项目的成本控制。当然，挑战在于找到一种方法来自动化这些控制的实现，并最小化对软件开发速度的影响。
