# OpsRamp 将人工智能扩展到事件管理

> 原文:[https://devo PS . com/ops ramp-extends-ai-reach-to-incident-management/](https://devops.com/opsramp-extends-ai-reach-to-incident-management/)

OpsRamp 今天将其注入了机器学习算法的 IT 运营平台，也称为 [AIOps](https://devops.com/?s=AIOps) ，扩展到事故管理领域。

该公司同名平台的【2021 年夏季版本增加了用于防止中断和事件的警报预测、用于更快事件故障排除的警报丰富策略，以及用于自动获取从阿里云环境和开源 Prometheus 监控软件以及日立、保利、VMware 和戴尔 EMC 平台收集的指标数据的增强功能。

IT 运营团队现在还可以使用仪表图直观显示在固定范围内增加或减少的指标值。IT 团队现在还可以添加警报上下文细节，以提供更多相关信息。现在，苹果 iOS 和谷歌 Android 设备都有一个移动应用程序，IT 团队可以通过它来响应警报和事件。

最后，OpsRamp 还提供了动态工作流的测试版，使 IT 团队能够维护一个决策表来处理特定的操作场景。

![](../Images/11daca2a45f9f87a63b9cbecee5f7df5.png)

OpsRamp 首席收入官 Sheen Khoury 表示，目标是利用人工智能(AI)来提高应用程序的可用性，使 IT 团队能够更好地预测何时会出现由季节性流量增加等因素导致的事件迹象，从而导致停机。

与此同时，IT 运营团队可以通过利用嵌入平台的自动监控工具来提高整体价值实现时间，他补充道。

OpsRamp 认为，许多 IT 团队将选择部署一个新的 IT 管理框架，而不是等待现有的 IT 运营平台提供商添加人工智能功能。Khoury 指出，如今 IT 运营团队经常执行的许多手动任务可以更快地实现自动化。

自然，机器学习算法的训练需要时间，所以 AIOps 的好处不是瞬间的。然而，这些算法将不断学习 it 环境的独特属性，从而使 IT 团队更容易更主动地管理 IT 环境。这些算法提供了额外的好处，使新成员更容易加入 it 运营团队，而不必花几个月的时间学习该环境特有的所有细微差别。

每个 IT 团队都需要自己决定在多大程度上依赖 AIOps 平台来管理他们的 IT 环境。最初，大多数 IT 团队需要评估人工智能平台提出的建议。但是，随着他们对这些建议越来越有信心，他们会相信该平台能够自动执行更多流程，以实时响应 IT 环境中发生的事件。

不管 it 团队对人工智能有多少信心，很明显，企业 IT 环境只会变得更加复杂。如果没有人工智能平台的帮助，许多 IT 团队将无法管理某些环境。许多 IT 专业人员可能不想为那些不提供一些使用人工智能增强技能的方法的组织工作——压力水平可能会太高。

无论如何，高度致力于无情地应用自动化的 DevOps 团队将处于企业采用人工智能的最前沿。
