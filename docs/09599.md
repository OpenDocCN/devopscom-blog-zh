# 甲骨文的免费托管服务与微软 Azure 集成

> 原文:[https://devo PS . com/Oracle-free-managed-service-integrated-with-Microsoft-azure/](https://devops.com/oracles-free-managed-service-integrates-with-microsoft-azure/)

甲骨文今天扩大了与微软的联盟，纳入了一项托管服务，使 it 组织能够更轻松地采用混合云计算。[面向 Microsoft Azure 的 Oracle 数据库服务](https://www.oracle.com/news/announcement/oracle-database-service-for-microsoft-azure-2022-07-20/)将 Windows 应用程序与运行在 Oracle 云基础设施上的 Oracle 数据库相集成。

甲骨文产品和战略副总裁 Leo Leung 表示，托管服务是 2019 年推出的现有功能的扩展。然而，许多 IT 组织发现自己调配混合云计算环境太难了。Leung 说，托管服务使通过跨越 11 个全球区域的高速网络访问 Oracle 数据库变得更加容易。

Leung 说，在 OCI 和 Azure 之间移动数据时，Oracle Database Service for Microsoft Azure、Oracle Interconnect for Microsoft Azure、data egress 或 ingress 是免费的。组织将只为他们消费的其他 Azure 或 Oracle 服务付费。

DevOps 团队经常花费大量时间[试图集成](https://devops.com/devsecops-in-azure/)不同的云计算环境。现在，只需点击几下鼠标，甲骨文就可以将 Azure 订阅连接到 OCI。该服务自动配置 OCI 和 Azure 环境，联合 Azure 活动目录身份，并支持使用 Azure 应用洞察进行监控。

Oracle 和微软当然陷入了激烈的数据库竞争，因为微软希望推动采用 SQL Server 作为 Oracle 关系数据库的替代方案。但是，目前有许多组织在内部 IT 环境中的 Windows 服务器上运行 Oracle 数据库。Leung 指出，Oracle 希望以最小的中断更轻松地将这些环境迁移到云中。

从理论上讲，组织正在采用多云计算，以确保他们不会发现自己被单一提供商所束缚。挑战在于每个云计算环境都有自己的一套需要掌握的管理工具。随着组织添加额外的云，IT 的实际总成本会随着更多专家参与管理这些环境而上升。Leung 说，Oracle 的方法通过使 IT 团队能够通过驻留在 Azure 云上的单一控制台管理 Oracle 数据库来解决这个问题。

甲骨文一直试图通过降低云服务的总成本来获得云市场份额。将 Azure cloud 与 OCI 集成的免费托管服务也降低了任何现有 Oracle 客户在迁移到云时更换 Oracle 数据库的动机。

尚不清楚 Oracle 服务将在多大程度上推动混合和多云计算，但它确实为 It 组织应该期待的东西设立了一个新的基准。云服务提供商在很大程度上相互忽视的日子即将结束。在当今时代，IT 团队不应该仅仅因为云服务提供商不想承认应用程序工作负载通常部署在多个云上这一事实，就不得不投入资源来集成低级 API。
