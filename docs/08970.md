# 2021 年最佳表现——6 个迹象表明你正在正确地进行开发工作

> 原文：<https://devops.com/6-signs-youre-doing-devops-correctly/>

随着 2021 年的临近，我们 DevOps.com 想要突出今年最受欢迎的文章。以下是我们 2021 年最佳系列的第 21 期。

许多组织声称专注于开发运维，因为他们使用特定的工具，并且松散地遵循某些实践。但是，要真正成为一个以开发运维为中心的组织，有具体的开发运维原则和[流程](https://devops.com/?s=processes)来确保您的组织尽可能优化和自动化。为了确保你不只是使用 DevOps 头衔，而没有遵循作为一个组织的基本原则，这里有六个迹象表明你实际上做得对。

**部署完全自动化**

从头到尾完全自动化部署是您正确遵循 DevOps 原则的最明显的标志之一。自动化部署可以节省大量的时间，尤其是当事情出错并且您需要快速恢复到先前的构建版本时。手动部署可能会导致通常由人为错误引起的问题。高度关注[自动化](https://devops.com/?s=automating)流程是以 DevOps 为中心的组织应该持有的核心信念。自动化提高了组织的效率，减少了错误。

**你有频繁而快速的发布周期**

一旦软件准备就绪，就应该部署并交付给客户。我们生活在一个竞争对手每天都在开发更好的软件的世界里。不断部署新功能和缺陷修复程序以保持在竞争中的领先地位对组织是有益的。DevOps 是关于持续交付的，所以让开发人员产生新代码并快速发布到生产中是 DevOps 的核心特性之一。

**您使用了正确的工具和平台**

您用来部署特性和修复的工具应该使构建应用程序变得容易，测试变得简单。一般来说，您应该能够构建、测试并部署到生产环境中，一路上不会出现任何问题。如果您花费大量时间来修复出现的任何问题，那么您可能没有合适的工具或流程。幸运的是，今天有广泛的[持续集成](https://www.cloudbees.com/continuous-delivery/continuous-integration)软件选项，可以适应任何现有的基础设施。虽然工具是 DevOps 的重要组成部分，但重要的是要记住，虽然工具使 DevOps 成为可能，但真正使变化发生的是实践。

**你有一个连续的反馈回路**

在第一次迭代中，没有什么是完美的，在这个过程中必然会出现错误。你必须有一个连续的反馈循环系统来检测何时何地出错。拥有监控流程并在出现错误时快速通知您的软件工具，将使解决突然出现的问题变得更加容易和快速。

如果你建立了一个系统来识别问题，解决问题并改进它们，你就正确地完成了开发工作。如果你遇到问题，只是简单地解决它们，而没有花时间去分析它们为什么会发生，或者如何防止它们再次出现，你可能要考虑做一些改变。

**开发和运营团队合作**

开发、测试和 IT 运营团队之间的沟通对于 DevOps 的成功至关重要。这些不同团体之间的合作确保了管道的顺利运行。当开发人员和 IT 运营人员能够轻松地相互协作时，任何确实发生的问题都会更快地得到解决。

**明确制定目标和目的**

虽然不同团队之间的沟通是必不可少的，但每个人都努力实现的明确目标也同样重要。最终，所有的团队都有相同的目标:让组织更有效率；然而，他们为实现这一点而遵循的个人实践可能会有所不同。团队成员应该清楚要达到目标需要满足的精确需求，而不是做出假设或听天由命。
