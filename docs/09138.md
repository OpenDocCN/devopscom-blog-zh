*Opsera 和 Octopus Deploy 将无代码 DevOps 编排与持续部署自动化相结合，使软件交付团队能够简单轻松地进入现代 DevOps 实践*

旧金山，2022 年 3 月 8 日——devo PS 的持续编排平台 Opsera 今天宣布与 Octopus Deploy 合作。通过这种集成，软件交付团队可以快速、安全、无缝地为端到端软件供应链管理创建无代码编排层。

如今，组织面临着在多个云和环境中部署应用程序以跟上创新和客户需求的复杂性。为了满足对部署效率的需求，Octopus 提供了一种工具来加速跨云和内部的可靠、可重复和可跟踪的部署。但是，向部署流程添加编排层对于确保实现连续部署至关重要。通常需要多种专有脚本语言、特定领域语言和配置文件方面的专业知识来协调工具之间的交付任务。使用 Opsera 的无代码编排解决方案降低了这种复杂性，并允许组织通过自由选择工具和完全自动化的部署管道和工作流来快速扩展开发运维流程。Opsera 还消除了编写手动粘合代码和定制脚本的需要。

**通过 Octopus 和 Opsera 的集成，用户可以享受:**

**速度提高，进入壁垒降低**

作为代码的配置可能很复杂，但是 Octopus 提供了一个简单的作为代码的配置解决方案，它平衡了 Git 的全部功能和 Octopus 的可用性。然后，Opsera 通过让开发人员能够并行管理跨多个云和环境的 Octopus 部署(即，虚拟机、Kubernetes、EC2、AKS、EKS、函数、无服务器、数据库部署等)提供了进一步的优势。)通过无代码编排平台。

**提高安全性和合规性**

Octopus 提供了内置的手动干预步骤，Opsera 对其进行了扩展，因此软件交付团队可以通过单一的事实来源获得报告和跟踪，以确保符合严格的审计和合规性措施。Opsera 提供了无代码脚本，允许开发人员在管道流程(安全、质量、变更管理等)中插入审批关口。)进行可重复的、可审计的、符合规范的软件交付。

**安全和机密管理**

Octopus 为管理配置变量和秘密提供了内置支持，这些变量和秘密随着软件发布在环境中的推广而变化。Opsera 通过与 HashiCorp Vault 等第三方机密管理解决方案的集成扩展了这一功能，只需几次点击即可支持任何部署类型。Hashicorp Vault 保护、存储并严格控制对令牌、密码、证书、加密密钥、自定义部署脚本的访问，并保护机密和其他敏感数据。Opsera 提供了与 Hashicorp Vault 的内置集成，使客户能够快速、安全地将 Vault 变量集成到 Octopus 部署中。

**可定制的报告和指标**

至关重要的是，从 CXO 到工程和产品负责人，所有的利益相关者都要理解使用哪个 KPI 来衡量部署的成功，并充分了解这些指标。Octopus 提供了指示板，向您显示您的应用程序的哪个版本被部署到了哪个环境中。用户可以看到谁在什么环境下部署了哪个版本，何时部署的，以及部署是否成功。Opsera 用 100 多个 KPI 补充了这一点，这些 KPI 符合来自 DORA、DoD 和 NIST DevSecOps metrics 的设定标准。

Opsera 产品负责人 Vishnu Vasudevan 表示:“Opsera 和 Octopus 的集成提供了速度和控制软件交付团队与现代 DevOps 保持同步所需的速度和控制软件。“多云和多环境部署极具挑战性，但在企业中正在激增。将工具选择的灵活性和 Opsera 的无代码自动化与 Octopus Deploy 卓越的部署自动化功能相结合，意味着发布工程师和开发人员可以更快、更好和更安全地跨环境进行部署，并采取严格的安全和合规措施，从规划到运营(包括价值流管理)具有 360 度的可见性。”

“我们很高兴看到 Opsera 的 Octopus 集成部署。这将允许客户利用 Octopus Deploy 在持续部署自动化方面的优势。通过这种新的集成，软件团队将能够利用 Octopus 的部署能力协调其 CI/CD 管道，”Octopus 产品副总裁 Colin Bowern 说。

**关于 Opsera**

Opsera 是第一个面向下一代 DevOps 的连续编排平台，可在整个软件生命周期中实现选择、自动化和智能化。它提供简单的自助式工具链集成、拖放管道和统一的见解。通过持续的流程编排，开发团队可以使用他们想要的工具，运营团队获得了更高的效率，业务领导拥有了无与伦比的可见性。Opsera 认为，DevOps 已经从一种愿望转变为一门实用科学，持续的协调是未来帮助组织加快 DevOps 的采用并达到峰值创新速度的方向。
