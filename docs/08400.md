# 换位调查显示开发运维对首席信息官和首席技术官关系的影响

> 原文:[https://devo PS . com/transposit-survey-shows-devo PS-impact-on-CIO-CTO-relationship/](https://devops.com/transposit-survey-shows-devops-impact-on-cio-cto-relationship/)

DevOps 流程编排平台提供商 [Transposit](https://www.transposit.com/blog/) 对 415 名首席信息官和首席技术官进行的调查发现，随着组织继续更广泛地采用 DevOps 流程，许多组织内这些类型的高级 it 领导之间的摩擦正在增加。

作为一个高管级别的角色，“首席技术官”最初是在互联网热潮之后受到关注的。近年来，许多传统企业在首席信息官的基础上增加了首席技术官，因为他们意识到组织已经变得多么依赖软件。在新冠肺炎疫情之后，现在比以往任何时候都更加关注依赖于软件及时交付的数字业务转型计划。

Transposit 首席执行官 Divanny Lamas 表示，随着越来越多的组织结构围绕着产品团队，首席技术官也越来越多。在某些情况下，首席信息官向首席技术官报告。

接受调查的大多数组织都有这两种角色。整整 83%的首席信息官表示他们的组织中也有首席技术官，而 86%的首席技术官表示也有首席信息官。不到四分之一的受访者(24%)表示他们同时扮演两种角色。

与此同时，98%的调查受访者表示，在过去三年中，他们的组织更加关注[数字化转型](https://devops.com/?s=digital+transformation)，四分之三(75%)的受访者表示，自疫情开始以来，他们的关注度有所提高。整整 85%的受访者表示他们的预算在过去三年中有所增加。

毫不奇怪，92%的受访者表示他们的职责在过去三年中发生了变化，超过一半(56%)的人表示他们的角色发生了重大变化。两者现在都根据与管理收入相关的关键绩效指标(KPI)进行评级，66.3%的首席信息官和 78%的首席技术官表示，他们的职位现在要么直接参与，要么需要领导业务战略。

但是，只有 34%(略多于三分之一)的人表示，对数字化转型计划的大多数方面的所有权是 CIO 和 CTO 的共同责任。具体来说，不到三分之一(32%)的人指出他们组织中软件开发的所有权是一项共同的责任。整整 88%的受访者应用 DevOps 实践，83.4%的受访者认为 DevOps 对其组织的数字化转型非常重要。几乎所有人(98%)都计划扩展他们的 DevOps 业务，83%的人已经有了这样做的计划。

Lamas 说，软件工程第一的原则不仅改变了应用程序的构建方式，也改变了 IT 团队的构建方式，从而推动了创新。

超过三分之一(35%)的受访者表示，拥有内部开发的软件运营是他们组织的共同责任，而 28%的受访者表示，管理 SaaS 或运营商业现货(COTS)软件是他们组织的共同责任。三分之一(33%)的受访者还指出，在他们的组织中拥有外部客户体验所需的技术是一项共同的责任。

整整 95%的调查受访者还确认，他们团队的绩效是通过与收入相关的关键绩效指标(KPI)来衡量的，特别是收入增长率(75%)、创新率(60%)和正常运行时间(54%)。

现在谈论首席信息官和首席技术官之间的关系将如何发展还为时过早，但有一点是明确的:这些角色更有效地合作的压力正在上升。
