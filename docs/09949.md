# Aqua Security 声称遵守拜登的行政命令

> 原文:[https://devo PS . com/aqua-security-claims-compliance-with-Bidens-executive-order/](https://devops.com/aqua-security-claims-compliance-with-bidens-executive-order/)

Aqua Security 本周声称，它是第一家满足 T2 认证要求的软件供应链安全平台提供商，认证要求由拜登政府去年向联邦机构发布的行政命令定义。

拜登政府发布的补充备忘录要求联邦机构在 2023 年 9 月 14 日之前收集他们使用的所有软件的合规证明函。这些证明信需要确认他们采用的软件开发环境是安全的，代码来源是可信的，代码漏洞已经得到补救，数据和代码的来源正在使用软件材料清单(SBOM)进行跟踪，并且这些流程正在得到一致维护。

Aqua Security 的供应链安全高级主管 Eilon Elhadad 表示，Aqua Security 能够在备忘录发布后的 100 天内满足这些要求。用于保护软件供应链的 Aqua Security 平台基于它在去年年底收购 Argon 时获得的平台，Argon 是一套工具的提供商，提供了一种无代理方法来集成映射开发环境的持续集成/持续交付(CI/CD)平台。Elhadad 指出，有了这些见解，就有可能根据安全最佳实践和合规性规则来优先考虑和自动修复漏洞。

Argon 平台还包括代码篡改检测技术，该技术执行有效性检查，以确保构建过程的完整性没有受到损害。

自然，其他安全平台提供商有足够的时间来满足拜登行政命令中提出的要求。然而，软件开发团队也在与时间赛跑。在一系列引人注目的违规事件之后，软件供应链的脆弱性变得越来越明显。埃尔哈达德指出，网络犯罪分子已经加大了危害开发者证书的力度，这样他们就可以在软件组件中嵌入恶意软件，如果未来的应用程序包含该组件，他们就可以在下游激活它们。

目前还不清楚企业 It 组织对美国政府保护其软件供应链的努力有多密切，但任何为这些机构开发软件的实体都将受到影响。企业 IT 组织中 DevSecOps 最佳实践的倡导者无疑将使用拜登政府的要求来定义保护软件供应链的最低基线。

与此同时，网络安全战显然开辟了一条新战线。目标应用程序越多，将责任进一步转移给应用程序开发团队的动力就越大。Elhadad 指出，挑战不仅仅是扫描开发人员编写的代码，还要确保开发团队创建的软件版本不会被那些将自己插入到过程中的不良行为者破坏。

考虑到需要付出的巨大努力，不可能很快保证每个软件供应链的安全。然而，保护这些软件供应链的责任更多地落在了 DevOps 团队的肩上。
