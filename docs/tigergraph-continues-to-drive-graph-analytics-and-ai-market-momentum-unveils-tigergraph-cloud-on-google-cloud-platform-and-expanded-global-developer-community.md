# TigerGraph 继续推动图形分析和人工智能市场的发展，在谷歌云平台上推出 TigerGraph 云，并扩大全球开发者社区

**加利福尼亚州雷德伍德城——2021 年 4 月 21 日——**[领先的图形分析平台提供商 TigerGraph](https://www.tigergraph.com/) 今天宣布，该公司将继续加快图形分析在云上的采用、应用和使用，并扩大对所有云提供商的支持。TigerGraph Cloud 可以通过亚马逊 AWS、微软 Azure 以及现在的 GCP 访问。该公司还宣布了雪花和 Tableau 的连接器，这意味着用户可以直接从他们的雪花和 Tableau 仪表板访问关系分析；现在，只需点击几下鼠标，就能获得有价值的数据见解。TigerGraph 在 [Graph + AI Summit 2021](https://tigergraph.com/graphaisummit) 上发布了这些公告，并分享了有关其不断增长的全球开发者社区的详细信息，这是业内唯一一个致力于通过图形算法加速分析、人工智能和机器学习的公开会议。

TigerGraph 创始人兼首席执行官徐雨博士表示:“人工智能增强将在 2021 年创造近[3 万亿美元的商业价值，【graph 被视为当今组织的一项基础能力。“企业一直在努力利用人工智能，因为它需要上下文感知和对多个实体数据的理解。Graph technologies 可以在几分钟内回答这些关键的业务问题，而其他数据管理系统需要几个星期。我们举办 Graph + AI 的愿景是团结生态系统，今年我们将看到更多的参与者。Graph + AI 峰会注册人数增长了 200%，这一事实表明，在第一届 Graph + AI World 活动举行仅 6 个月后，当今有远见的公司对 Graph 的兴趣越来越大。”](https://www.gartner.com/en/newsroom/press-releases/2019-08-05-gartner-says-ai-augmentation-will-create-2point9-trillion-of-business-value-in-2021)

***扩展产品功能***

TigerGraph 扩展了其产品功能，新增了对 Google 云平台工作负载的 TigerGraph 云支持。在宣布 1.05 亿美元的 C 轮融资(图形数据库和分析市场迄今为止最大的一轮融资)后仅两个月，TigerGraph Cloud 现在是唯一一个在所有三大云平台(AWS、Azure 和 GCP)上可用的分布式图形数据库即服务。

TigerGraph 的分布式原生图架构有助于组织快速扩展并分析数据的许多不同方面，以形成新的模型并产生新的见解。TigerGraph 云用户可以:

*   使用免费层实例免费使用 TigerGraph 云
*   保护他们在图形平台上的投资，该平台旨在随着业务增长而扩展
*   选择他们首选的图形工作负载云平台(无供应商限制)
*   购买 TigerGraph 云点数以获得额外折扣，并使用他们在 GCP 上的预付或承诺消费进行支付
*   将他们的 TigerGraph 平台与其他数据服务和应用程序连接起来，包括 Kafka、Spark、Snowflake、Tableau、SageMaker 和包括 Python (PyTigerGraph)在内的开发人员服务
*   利用包含 20 多个初学者工具包的行业用例库构建分析或机器学习应用程序，这些工具包包含预构建的模式、样本数据集和针对所有主要用例的查询，包括欺诈、反洗钱、实体解析、客户 360 度、推荐、知识图、网络安全、网络分析、影响者检测等

随着交易和分析工作负载转移到云，TigerGraph 已成为连接、分析和从数据中学习新见解的首选图形数据库。该公司为雪花和 Tableau (Salesforce)发布的新连接器使公司更容易将 TigerGraph 与上游和下游数据服务集成。连接器功能包括:

*   TigerGraph 的雪花连接器，它简化并加速了从雪花数据仓库到 TigerGraph 的数据传输
*   Tableau 和 Microsoft Power BI 的专用驱动程序为用户提供了丰富的无代码分析和可视化仪表板功能
*   标准的 JDBC 和 ODBC 数据库连接器为大量的集成提供了基础

***扩展的全球开发者社区***

除了扩展其产品功能，TigerGraph 还通过推出新的支持和服务来倾听其开发人员社区的意见。 [TigerGraph 的开发者社区](https://community.tigergraph.com/)是一个面向开发者的全球网络，旨在营造一个协作、支持和以开发者为中心的环境。开发人员被邀请加入位于 TigerGraph 网站(【community.tigergraph.com】T2)上的虚拟生态系统，在这里他们可以分享信息，提出问题，并与所有技能水平的开发人员合作。当其他技术公司依赖他们的用户在他们的社区中共享故障诊断信息时，TigerGraph 积极地与其产品团队互动回答问题。这项正在进行的工作由 TigerGraph 的开发人员关系和产品团队联络经理 Jon Herke 领导。

TigerGraph 使开发人员能够在类似团队的环境中工作，而不是在没有资源的情况下独自解决问题。开发者可以分享他们创建的课程、黑客马拉松参赛作品、他们正在写的博客，并询问关于使用 TigerGraph 的一般性问题。Herke 在数字分销平台 [Discord](https://discord.gg/F2c9b9v) 上主持了一个每日会议，开发者们可以在会上提出即兴问题，讨论巧妙的解决方案，或者只是简单地聊天。事实上，其他公司和产品已经复制了 [TigerGraph 的 Discord 社区](https://discord.gg/F2c9b9v)，因为它成功地吸引并维持了一个活跃的开发者社区。

如果您想沉浸在最新的 TigerGraph 生态系统活动中，请在每个工作日美国中部时间上午 8:15 加入 [TigerGraph Discord(开发者)社区](https://discord.gg/F2c9b9v)的开发者关系团队，收听或积极参与当前的开源工作，包括:

*   [TG cloud-jupyter](https://pypi.org/project/tgcloud-jupyter/)–JupyterLab 扩展，允许数据科学家在数据科学环境中与 TigerGraph 服务进行交互
*   [TG-RASA-Pipeline](https://github.com/TigerGraph-OSS/RASA)–一个连接 TigerGraph 和 RASA(聊天机器人平台)的数据管道工具，允许 RASA 用户拥有知识图机器学习管道
*   [pyTigerGraph](https://pytigergraph.github.io/pyTigerGraph/Functions/)–TigerGraph 的 Python 连接器，允许用户将 tiger graph 集成到他们的数据科学工作流或运营服务中
*   tiger flow——一个基于流(低代码或无代码)的数据管道编排工具，使用户能够将节点拖放到画布上，从而快速设置流入 TigerGraph 的数据流

***扩展开发者社区活动参与度***

TigerGraph 社区的力量在 Graph + AI Summit 2021 上得到充分展示，该峰会包括 40+场会议、现场研讨会以及来自摩根大通、NewDay、Pinterest、捷豹路虎等公司的演讲人。4 月 21 日至 23 日的虚拟会议包括主题演讲、演讲人、真实世界的客户案例研究以及面向数据、分析和人工智能专业人士的实践研讨会。

TigerGraph 的 Graph + AI Summit Web App Hackathon 吸引了来自 30 多个国家的 200 名开发者。参与者使用 TigerGraph 的 20 多个云初学者工具包之一，在 TigerGraph 云之上设计创新的 web 应用程序解决方案。TigerGraph 与行业领先的合作伙伴 Streamlit(面向数据科学家的 UI 仪表板解决方案)、Graphistry(图形可视化产品)和 Galvanize(教育培训公司)合作。两位女性获奖者将在 Graph + AI 峰会上展示他们的项目:江和基于“机场数据集:探索全球机场的图形和网络分析”创建了图形最短路径和中心性即使在比赛结果公布后，TigerGraph 的网络应用黑客马拉松开发者参与者仍然积极参与 TigerGraph 开发者社区。
