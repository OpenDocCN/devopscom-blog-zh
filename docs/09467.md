# CDF 报告展示 DevOps 工作流收益

> 原文:[https://devo PS . com/CDF-report-surfaces-devo PS-workflow-gains/](https://devops.com/cdf-report-surfaces-devops-workflow-gains/)

在今天的 [cdCon 2022](https://events.linuxfoundation.org/cdcon/) 会议上，持续交付基金会分享了一份[2022 年持续交付状态](https://www.prnewswire.com/news-releases/cd-foundation-announces-state-of-cd-in-2022-report-opens-third-annual-cdcon-with-new-project-cdevents-new-members-301563033.html)报告，该报告发现近一半的开发人员(47%)使用持续集成或持续部署平台，但只有约 20%的人报告同时使用两者来自动化所有应用构建、测试和部署。

总体而言，在 SlashData 代表 CDF 调查的 18，585 名开发人员中，只有不到四分之一(23%)的人表示他们没有参与任何与 DevOps 相关的活动。

CDF 总经理法提赫·德吉蒙西表示，相对较低的百分比表明 [DevOps 工作流](https://devops.com/?s=DevOps+workflows)现在比以往任何时候都得到更广泛的应用。监控软件和基础设施性能以及使用持续集成来自动构建和测试代码更改仍然是调查中引用的最受欢迎的 DevOps 相关活动，各占 36%。

调查还发现，在大型企业中，在代码更改的交付时间方面，表现不佳的比例在过去六个月中从 34%下降到 24%，而表现优秀的代码更改所需时间不到一天的比例也从 13%上升到 21%。

根据该报告，大型企业也看到越来越多的开发人员报告说，他们可以在不到一小时内恢复服务(22%)，而那些仍然需要一周时间的开发人员(18%)。

该调查还表明，采用微服务、[容器和 Kubernetes](https://containerjournal.com) 等现代后端技术与改进代码更改的交付时间和恢复时间之间有很强的相关性。使用集装箱的受访者交付周期不到一天的可能性是使用集装箱的受访者的两倍，而采用 Kubernetes 的受访者成为最佳表现者的可能性是使用集装箱的受访者的三分之一。

然而，调查发现这对部署频率没有影响。该报告发现，能够每天多次部署代码的优秀员工比例为 11%。事实上，不使用这些技术的开发人员每天进行多次部署的可能性是普通人的两倍。总体而言，只有 53%从事后端开发的开发人员在使用微服务。

德吉蒙西表示，很明显，在简化 DevOps 工作流程方面，仍有许多工作要做。到 2022 年第一季度，DevOps 从业者使用的平均技术数量为 4.6。CDF 上个月启动了一个 [CDEvents](https://devops.com/continuous-delivery-foundation-adds-interoperability-project/) 项目，以创建一个供应商中立的规范，用于定义跨多个服务、平台和系统共享事件数据的格式。

与此同时，CDF 调查还指出，超过四分之一的受访者(26%)现在依赖托管持续集成/持续交付(CI/CD)服务，而不是自己构建和维护 DevOps 平台。

现在说组织依赖托管服务的复杂程度还为时过早，但随着应用程序开发速度的不断加快，越来越多的组织开始关注开发人员的工作效率。因此，许多组织正在审查他们现有的开发运维方法，以及这是否实际上是一个瓶颈，而不是更快地构建和部署应用的催化剂。
