# 开发者的熟悉度是数字化成功的关键

> 原文：<https://devops.com/developer-familiarity-is-key-to-digital-success/>

有一段时间，数字化转型的目标是取代传统系统。这变成了一场新的和闪亮的与旧的、过时的和“必须淘汰的”之间的战斗。然而，随着企业意识到两个关键事实，这一理念已经发生了转变。

首先，遗留系统对于传统组织来说太重要了，不能完全淘汰——例如，考虑到 SQL 的灵活性和开发人员对它的熟悉程度。因此，许多这些遗留的应用程序和系统正被转换成它们以前的虚拟版本。

此外，尽管一些组织完全依靠新的尖端技术将自己重塑为绿地企业并非不可能，但对许多公司来说，至少在可预见的未来，这种方式通常是资源密集型和耗时的。

第二，抛弃遗留系统会破坏现有的技能——而且不是一种好的方式——特别是考虑到 64%的数字架构师 [报告说](https://www.couchbase.com/press-releases/pressure-on-digital-architects-more-than-doubles-during-covid-19-couchbase-research-finds) 被锁定使用它们，因为他们在相关技能上投入了大量资金。此外，公开关注新人才排除了许多拥有良好技能和大量业务流程和系统关系知识的 IT 专业人员。与此同时，数字技能是昂贵的——不仅是获取，而且是提升技能。即使是初创公司——通常有一张字面上的“系统空白支票”——也面临着技能限制，许多人正在考虑将更多成熟的技能引入新兴的技术环境。

## 开发人员技能问题

由于新冠肺炎疫情，数字化转型现在成为关注的焦点，项目经理因此积累了关于转型项目成功的宝贵经验。451 Research 最近进行的[](https://webinars.dev/couchbase/451-research-why-your-modern-applications-wont-survive-without-new-database-foundation.md/)调查显示，44%的公司已经进入转型战略的执行阶段，其中 66%的公司开始看到成功。另有 10%的人宣称他们的转型项目取得了圆满成功。

阿基米德说他可以用足够大的杠杆移动地球。对于数字化转型目标而言，技能是构造杠杆。根据英国 WorldSkills 的一份[](https://www.worldskillsuk.org/news/new-research-highlights-uk-digital-skills-gap/)报告，60%的企业认为他们对高级数字技能的依赖将会增加。与技能要求相关的压力不会消失，而新旧技术之间的竞争只会进一步加剧这种压力。

正如您所料，开发人员被夹在中间。随着世界的数字化——见证了过去两年数字项目的加速——开发者已经成为英雄。也许不出所料，一项 couch base[](https://resources.couchbase.com/c/cio-survey-developers-2020?x=p2kCzD#zoom=100)调查的 92%受访者同意这一说法。大约 63%的人还表示，适当的发展[文化](https://devops.com/?s=culture)允许他们灵活地改变与数字化转型成果相关的目标。

然而，每当数字项目陷入困境时，开发商往往会被指责为延误的原因。根据同一项研究，至少五分之一的开发人员不具备所需的技能或技术。许多人也没有得到支持或空间来交付，令人震惊的是 49%的人说他们被要求在太少的时间里做太多的事情。

数字化转型项目越来越成功，我们应该感谢开发人员的成功。但是这些成功并没有减轻围绕技能的压力或问题，因此许多项目仍然停滞不前。此外，Gartner 最近的一份 [报告](https://www.gartner.com/en/newsroom/press-releases/2021-09-13-gartner-survey-reveals-talent-shortages-as-biggest-barrier-to-emerging-technologies-adoption) 发现，IT 高管正在努力寻找支持云和边缘采用的技能。

## 遗产不是敌人

如果我们将这些要点结合起来(对遗留系统的持续支持的需求、新兴的现代化需求、日益增长的开发人员技能短缺以及成熟技术专家的命运)，它们似乎是相互矛盾的。有些东西必须付出，对吗？

不一定。越来越多的人认识到抛弃传统技能是一个错误。这样做会削弱应该保留的技能和商业知识。我们还意识到，我们不需要将旧技术和新技术并置。现代系统可以合并甚至吸收遗留环境、编码语言和过程。

现代多模型数据库提供了一个很好的例子。在高层次上，我们可以将数据库分为关系型(SQL)和非关系型(NoSQL)系统。SQL 数据库无法提供众所周知的 NoSQL 系统的灵活性，但这种技术方法仍能带来许多好处。在多模型数据库系统中，SQL 开发人员可以在 NoSQL 系统上使用 SQL 编码，从而实现这两个目标。

## 成熟开发人员技能的新去处

创造这种熟悉感对于成功的数字项目至关重要。仅仅关注广泛的再培训或不断寻找新的人才并不是满足现代企业需求的唯一途径。厂商不要问开发商能不能上山，应该问:我们能不能把山带给开发商？

部署能够利用现有开发人员技能的新技术正变得越来越流行。除了某些数据库，我们还看到通过 API 连接到 web 服务的[大型机，以及使用微服务来强调遗留环境的特定部分。](https://devops.com/the-role-of-apis-in-mainframe-modernization/)

考虑一下雷曼著名的软件进化法则，其中包括熟悉度和组织稳定性。大多数现代化项目都忽略了这两个定律，而倾向于强力推进。但是数字现代化是一次旅程，每一次成功的旅程都需要所有人的参与——而不是沉重的负担和偷渡者。为开发者提供他们在新环境中最熟悉的东西是数字化成功的关键。
