## 2.自动辅助功能覆盖:盲人用户看不到重点

你担心你的内容对依赖屏幕阅读软件的用户的可访问性吗？**你应该是**。

### 分析:当心神奇的快速修复

你可以购买“覆盖图”,承诺让盲人和弱视用户可以访问你的网站。但是越来越多的视力障碍者愤怒地齐声说，这些事情实际上使生活变得更糟。

[Amanda Morris](https://www.nytimes.com/2022/07/13/technology/ai-web-accessibility.html "read the full text") : **对于失明的互联网用户来说，修复可能比缺陷更糟糕**

超过 12 家公司提供…自动化无障碍网络服务。……这些公司的客户包括 Hulu、易贝和优衣库等大公司，以及医院和地方政府。在他们的推销中，通常会保证他们的服务会……让公司免于……诉讼。但是事情并不是这样的。用户……说该软件提供的帮助很少，一些使用 AudioEye、accessiBe 和 UserWay 的客户正面临法律诉讼。…除了标记不良的图像、按钮和表单之外，盲人用户还记录了覆盖图的问题，包括无法使用键盘浏览网页，原因可能是页面上的标题没有正确标记，或者是页面的某些部分不可搜索或不可选择。

这些抱怨肯定会引起[u/mael trom](https://www.reddit.com/r/ketochow/comments/tefng8/comment/i0qpn2j/?utm_source=reddit&utm_medium=web2x&context=3 "read the full text")的共鸣:

作为一个建立网站并尽力让网站易于访问的人……它们不仅不利于网站的可访问性，公司还因使用[覆盖]代替实际解决可访问性问题而被起诉。…因此，如果不使用这种快速解决方案对自身非常有利。问题是，对于那些不太了解可访问性的人来说，这些产品中的许多演示得非常好(这似乎是他们关注的地方)。虽然我不是专家，但幸运的是(在我看来),我从前端开发那里学到的东西总是强调可访问性，我一直试图继续关注并尽可能放大宣传它的声音。我肯定我犯了很多错误，但我很高兴人们……会大声说出并分享他们的经验，这样那些愿意的人就可以从中学习。

对开发者有什么建议吗？这里是[罗斯兹](https://tech.slashdot.org/comments.pl?sid=21710630&cid=62701342 "read the full text"):

要使网站具有可访问性，首先查看没有样式表的网站。…如果你的网站对视力不好的人来说是不可用的，那对盲人来说就更糟了。
…
重写你的网站，使其不需要样式表也能使用。对长菜单之类的东西重新排序，让它们出现在末尾，而不是开头。你这样做是因为每次加载新页面时，如果它在顶部，文本到语音会首先读取所有该死的菜单元素，这真他妈的烦人。如果你有能力编写样式表，你可以为视力正常的用户把菜单放在页面上任何你想放的地方。是的，没有样式表，网站看起来就像****。那不重要。*怎么读*？
…
在每一张图片中包含【可选文字】。大多数情况下，你不需要说得那么详细，比如“孩子抱着鱼”就足够了。会有例外的情况，你可能需要更多的描述，只是使用一些常识。例如，如果鱼的种类很重要，就把它包括进去。如果孩子的种族很重要，那就包括在内。仅仅遵循这些简单的步骤就能让你达到 90%的成功率，让你比世界上任何一个网站都更容易访问。

## 3.终于！谷歌云做 ARM

AWS 有。Azure 拿到了。见鬼，就连甲骨文云也有。ARM 实例现在是 IaaS 提供商的必备条件。**欢迎来到派对，谷歌**。

### 分析:更快、更环保、更酷

正如我在之前[说过的，ARM 芯片在云和本地数据中心越来越常见，尤其是那些重视“性能功耗比”的数据中心而且，好像是为了强调我的观点，谷歌正在**为这些虚拟机积极定价**。](https://devops.com/facebook-bans-innocent-users-azure-and-arm-ampere-great-resignation-no-end-in-sight/#tlv2 "read the full text")

[Frederic Lardinois](https://techcrunch.com/2022/07/13/google-cloud-launches-its-first-arm-based-vms/ "read the full text") : **谷歌云推出首款基于 Arm 的虚拟机**

这是一个漫长的过程，但谷歌云今天宣布了其第一个基于 Arm 的虚拟机，继 AWS 及其 Graviton 实例和 Azure 之后，Azure 最近也推出了 Arm 虚拟机。…谷歌云使用 Ampere 的芯片，跟随 Azure 的脚步。这些新虚拟机目前处于预览阶段，将加入谷歌云的 Tau 虚拟机系列，以“Tau T2A”的名字提供更好的性价比。
…
除了将这些虚拟机作为谷歌云计算引擎的一部分，谷歌现在还将它们作为其 Kubernetes 引擎、数据流和批处理服务的一部分来支持。…新虚拟机现已在少数地区推出，包括美国中部(爱荷华州-A、B、F 区)、欧洲西部 4(荷兰-A、B、C 区)和亚洲东南部 1(新加坡-B、C 区)，但随着时间的推移，这些虚拟机还会出现在其他数据中心。

迪伦·马丁认为，一点也不早:

对于 Arm 来说，这是艰难的一年。首先，这家英国芯片设计商失去了一笔财务增长，因为它出售给英伟达(Nvidia)的交易被监管机构审查否决了。然后，Arm 在计划首次公开募股(IPO)时裁员，现在市场条件看起来对 IPO 不利。对 Arm 来说，好消息是云世界对替代指令集架构越来越感兴趣。
…
虽然谷歌云没有提供任何性能对比… Ampere 跳出来说，一个拥有 32 个虚拟 CPU 的 T2A 实例比谷歌使用相同数量虚拟 CPU 的英特尔 Ice Lake silicon 的 N2 实例快 31%。这是基于… SPEC CPU 2017 整数比率的估计分数。
……
可以肯定地说，云提供商采用 Arm 肯定是现在的趋势。[它]现在由六家全球最大的云服务提供商提供支持:亚马逊网络服务、微软 Azure、谷歌云、阿里云、腾讯云和甲骨文云。其他云提供商也在支持 Arm，如 JD Cloud、UCloud 和 Equinix Metal。

谁能想到会发生这种事？ [Hannu Krosing](https://twitter.com/PostSQL/status/1547472782694518784 "read the full text") 举个例子:

五年前，我惊讶地发现我的 Android 手机…运行 PostgreSQL 比我的英特尔笔记本电脑还快。这是对即将发生的事情的强烈暗示。
