# DevOps 如何帮助银行优化服务

> 原文:[https://devo PS . com/how-devo PS-helps-banks-optimize-their-services/](https://devops.com/how-devops-helps-banks-optimize-their-services/)

一家世界知名银行的首席执行官最近表示，虽然现在有成千上万家人工银行，但未来只会有少数几家数字银行。与世界上其他重要的商业部门一样，绝大多数银行也在向虚拟平台转移。这次迁移由全球金融科技领域最优秀的 DevOps 资源精心策划。

始终按需提供的个性化服务的便利性刺激了银行业对开发运维工作流的依赖。当新冠肺炎疫情袭击并限制实体银行服务时，企业银行系统向云的迁移已经全面展开。大多数银行员工不得不转向在家工作，这加速了虚拟服务交付的使用。

根据 IBM 商业价值研究所的估计，大约 59%的受访公司正处于数字迁移的不同阶段，银行业自然也在跟进。DevOps 团队的任务是编写软件扩展，以敏捷的方式将数字银行服务连接到最终客户。

本文探讨了为什么 DevOps 正慢慢成为任何成功的银行机构的重要基础。我们还将讨论传统机构如何应对这些变化，以及一旦它们实现了向 DevOps 的飞跃，它们可以期待哪些好处。

### 敏捷流程促进数字化转型

如今，大多数银行都在采用敏捷流程来保持竞争优势。这背后的主要动机是为了跟上频繁的业务需求，这些需求伴随着改善客户银行体验的努力而来。微调驱动这种改进的项目规划和执行过程是 DevOps 团队主要要做的事情。

在敏捷方法下，银行应用程序将通过称为 sprints 的小而可控的步骤来创建。每一步都要经过细致的测试，才能被认为是可执行的或完整的。在继续构建已交付的组件之前，遵循适当的渠道进行实时反馈和过程修正。最终目标是客户必须在比其他开发过程更短的时间内获得有价值的产品。

甚至比管理它的过程和规则更重要的是，所涉及的 DevOps 团队和他们的协作是重中之重，也是最终的结果。最佳银行服务和产品背后的敏捷项目必须包括持续的计划、测试和成品组件的集成。

### 银行业开发运维团队更快取得成功

为了成功推动新的数字银行服务，大型银行需要数十亿美元的资产，并需要在此过程中获得数百万新客户。这一切都归结为将他们的流程转变为更细粒度的流程，并在此基础上不断改进。

在需要主动采用 DevOps 实践的情况下，客户的业务线(LOB)团队会很快转向敏捷规划策略。他们使用各种工具来实施敏捷和 DevOps 实践，以满足客户对其改进的数字银行服务不断增长的需求。

但是为了无缝地工作，LOB 和 IT 团队的开发人员体验都需要简化。如果 DevOps 平台通过内置的敏捷功能和双向集成实现自动化，则可以整合业务线和 IT 团队的工作。这确保了无缝协作、更快的开发运维实施以及银行服务和产品的快速交付。

### 专注的洞察力和更好的可预测性

大型银行可以在工程生命周期管理平台上标准化内部 IT 开发团队，同时通过敏捷问题跟踪仪表板为 LOB 团队提供支持。敏捷生命周期管理工具扩展了银行的集成解决方案。使用这些工具，DevOps 团队可以获得开发人员数据的完全透明性和可追溯性。

特定于开发人员的功能，例如源代码管理、构建自动化、持续集成和交付以及敏捷规划，都可以通过这种方式更容易地获得。IT 开发人员社区将敏捷工作流管理平台视为高效和持续业务流程的基准。工作流管理平台和问题跟踪门户是大多数敏捷计划发生的地方。

然后，敏捷生命周期集成适配器将 LOB 定义的敏捷故事与特定的开发人员任务链接到易于管理的包中。在劳动力管理工具中跟踪开发人员对应用程序代码的更改，以便从数据中得出有用的、可量化的见解。由于在项目、团队和工具之间建立了联系，应用程序交付的速度进一步加快。

### 最大限度地减少服务交付事故

使用工作流管理工具，团队可以提供持续的构建和集成功能。他们可以在专用云或内部环境上部署自动化集成构建，从而最大限度地减少服务交付中的错误。在最终的审计跟踪中，为每个构建生成一个软件材料清单(SBOM ),这也让开发人员可以追溯性地解决错误。

创建了一个广泛的图片，其中包含了构建中的新任务和代码片段，以及创建了什么软件资产。DevOps 实现确保了银行应用程序在测试和生产环境中的持续交付。在每项银行服务或产品的生产生命周期中，错误都会减少。这减少了从开发到生产的时间。创建了一个完全自动化且可审计的交付渠道，以加快未来的部署。

### 缩短银行产品发布周期

DevOps 为银行的业务发展提供了一个非常健壮和可重改的基础环境，同时减少了每次迭代的发布周期时间。这意味着每次交付新的银行服务时，开发生命周期都会一点一点地改进。在某些情况下，发布周期可以比原来缩短一半以上。

在服务交付得到改进的同时，这也将发布工程师从对部署窗口至关重要的紧急任务中解放出来。发生更多的部署，同时仍然保持银行或金融机构符合监管准则。
