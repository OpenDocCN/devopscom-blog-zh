# Instana 发布了对 NGINX 和 Envoy 的性能监控和跟踪支持

> 原文:[https://devo PS . com/instana-releases-performance-monitoring-and-tracing-support-for-nginx-and-envoy/](https://devops.com/instana-releases-performance-monitoring-and-tracing-support-for-nginx-and-envoy/)

*Instana 是首个 APM 解决方案，可在代理层将每个应用请求的跟踪自动绑定到端到端跟踪*

2019 年 3 月 21 日——加利福尼亚州旧金山和德国索林根——针对容器原生应用的自动 APM 解决方案的领先提供商 Instana 今天宣布，为 NGINX 和 Envoy 应用代理提供监控和跟踪支持。通过 Instana APM 服务的当前版本，开发人员和开发人员可以从浏览器、每个服务到代理层获得性能可见性和每个事务的单独跟踪。所有这些新数据都是自动捕获的，并与 Instana 的其余传感器和跟踪技术集成在一起。

Instana 联合创始人兼首席执行官 Mirko Novakovic 表示:“虽然应用程序代理是为微服务应用程序创建按需可扩展性的重要组成部分，但它们会产生性能可见性差距，因为它们是作为黑盒运行的。“Instana 通过这些基础系统跟踪应用程序请求的能力将提高整体应用程序质量，并提供业内最全面的应用程序性能可见性。”

Instana 的 NGINX 和 Envoy 监控和跟踪功能包括通过代理服务器将性能指标和跟踪与 Instana 的 AutoTrace 技术捕获的端到端应用程序跟踪联系起来的能力。通过跨代理进行跟踪，前端用户跟踪会一直跟踪到后端微服务请求，而无需添加任何自定义跟踪代码。Instana 成为唯一可以自动做到这一点的 APM 解决方案。

首席运营官联合创始人 Peter Abrams 表示:“随着支撑应用的技术和架构不断发展，监控解决方案也必须跟上这些变化。“Instana 通过 NGINX 和 Envoy 监控性能和跟踪请求的能力表明，该公司致力于提供完整的微服务应用。”

Instana 的自动应用程序监控解决方案可发现应用程序基础架构和服务组件，为应用程序的技术堆栈部署监控传感器，跟踪所有应用程序请求，无需任何人工配置，甚至无需重启应用程序。该解决方案实时检测应用环境中的变化，调整自己的模型，并在几秒钟内可视化变化和对性能的影响。
