# Trilio 宣布推出“连续恢复”技术预览版，可在几秒钟内跨不同的基础架构提供云原生应用程序的可移植性和可恢复性

> 原文：<https://devops.com/trilio-announces-technical-preview-of-continuous-restore-delivering-cloud-native-application-portability-and-recoverability-in-seconds-across-disparate-infrastructure/>

*该功能释放了数据引力，让数据驱动型组织能够在任何地方快速复制生产级云原生应用*

**西班牙瓦伦西亚，KubeCon + CloudNativeCon 欧盟，2022 年 5 月 17 日—** [领先的云原生数据保护提供商 Trilio](https://trilio.io/) ，今天宣布了一项新的“连续恢复”功能的技术预览，该功能将在 KubeCon + CloudNativeCon 欧洲大会上向云原生社区展示。连续恢复提供了更快的数据和元数据复制、恢复和迁移级别，可从任何存储或云平台迁移到另一个存储或云平台，大大缩短了云原生应用程序的恢复时间。

tri lio 的“连续恢复”功能完全不受存储、云和分发的限制，使用户能够在多个异构云中连续存放数据。这意味着无论应用程序位于何处，它们都将能够利用这些数据并在几秒钟内上线，从而实现出色的恢复时间目标(RTO)。该功能将作为 TrilioVault for Kubernetes 云原生数据保护和管理平台的一部分引入。

Trilio 创始人兼首席技术官 Murali Balcha 表示:“由核心、边缘和云资源组成的分布式环境正成为当今企业的主导架构。“公司经常需要在这些不同的环境中移动有状态的应用程序及其数据量，以实现成本效益、性能、安全性和灾难恢复要求。 遗憾的是，目前还没有一种易于使用、价格合理的解决方案能够跨异构基础架构环境持续复制数据量。”

Balcha 继续说道:“Trilio 的连续恢复功能可在数秒或数分钟内完成有状态应用程序的迁移和复制，因此所有公司都可以在任何地方保护和使用其数据，无论应用程序运行在什么平台上或数据存储在什么位置。这使组织能够以合理的成本满足领先水平的应用程序正常运行时间，实现生产级应用程序的服务级别协议(SLA)。 连续还原是一项改变游戏规则的技术，它将使当今的现代企业更具竞争力和弹性。”

**连续恢复功能的使用案例包括:**

1.  ****灾难恢复:** 用户将能够实现可用性目标，并在几秒或几分钟内从中断或故障中恢复，而不是几天或几周。使用连续恢复， 与传统方法相比，恢复时间目标(RTO)将提高 80%以上。**
2.  ****应用程序迁移** :持续恢复允许 IT 团队 通过选择最适合当前需求的基础架构来优化性能并降低总拥有成本(TCO)。 连续恢复功能还将使组织能够统一其基础架构，特别是那些快速发展并采用各种计算平台和存储解决方案来满足独特需求的组织。连续恢复将使应用程序跨基础架构孤岛快速移动成为可能，使它们不再是孤岛。**
3.  ****测试/开发:** 开发人员可以 通过为多个测试/开发环境暂存数据来提高 CI/CD 管道的速度。这些测试/开发环境可以在几秒钟内启动，持续复制生产数据，并加速将经过验证的更改推向生产。 事实上，DevOps 团队可以使用这一功能来测试他们的“恢复”协议，以确保恢复在需要时能够工作。**
4.  ****来自边缘云的数据监管:** 如上所述，随着分布式环境在未来的激增，大量的数据将被收集在“边缘”，来自不同架构的这些信息将需要在分布式系统中快速复制和移动，以便被任意数量的不同应用程序吸收和集中分析。**
