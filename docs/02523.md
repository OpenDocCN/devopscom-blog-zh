# 新的 XebiaLabs DevOps 预测引擎使用机器学习来为软件交付提供“天气预报”

波士顿—([商业资讯](https://www.businesswire.com/))—[XebiaLabs](https://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fxebialabs.com%2F&esheet=51960195&newsitemid=20190327005061&lan=en-US&anchor=XebiaLabs&index=1&md5=6f3ec73b61c8642800577fb09e75b8de)，企业级 DevOps 和持续交付软件工具的公认领导者，今天宣布了第一个针对 DevOps 的预测风险解决方案，该解决方案警告团队软件交付过程中的延迟和风险。XebiaLabs DevOps 平台的风险预测模块使用新的 DevOps 预测引擎，将机器学习与专有算法相结合，在发布开始之前为团队提供发布的“天气预报”。

**软件发布经常受到不可预见的发布延迟和失败的困扰**

DevOps 团队很难预测会延迟发布或导致发布失败的因素——并且团队经常在发布过程已经开始后对失败感到惊讶，使他们不得不匆忙回滚更改并解决错误。

**DevOps 预测引擎可在发布问题发生之前发现它们**

XebiaLabs DevOps 预测引擎可以在发布流程开始之前突出潜在的瓶颈，因此发布经理、开发人员和 DevOps 工程师可以采取预防措施，调整时间表，并让业务保持在循环中。强大的预测风险分析使开发运维团队能够回答以下问题:

*   这个版本失败的可能性有多大，我们能做些什么？
*   这个发布会被推迟吗？会推迟多久？
*   团队成员跳过特定发布任务的可能性有多大，为什么？
*   在过去，是什么导致了像这样的发布失败或延迟？

**预测性风险分析防止发布灾难**

XebiaLabs 已经提供了可定制的风险评分，可以在发布过程中识别警告信号，以表明何时不在正轨上。现在，该公司在 XebiaLabs DevOps 平台上增加了新的风险预测功能。它由基于机器学习的 DevOps 预测引擎提供支持，提供以下功能:

*   **风险警报**在发布管道开始运行之前，当发布可能被延迟或失败时，向团队发出警告，以便他们能够采取预防措施
*   全面的**风险预测**视图，总结了发布过程中每个任务的预测延迟和失败，因此团队可以深入到过程中可能导致问题的特定部分

*   类似发布的**统计数据的概述，这样团队可以一目了然地看到历史分析，并识别发布和任务失败的趋势，跨越他们发布管道中的所有工具**
*   **发布取证**,发现不可靠的自动化测试、有问题的构建设置、长时间运行的部署和耗时的手动任务，因此团队可以确定他们最大的痛点，并做出对他们快速可靠地交付软件的能力有最大影响的改变

XebiaLabs 产品战略副总裁 Andreas Prins 表示:“XebiaLabs DevOps 平台长期以来一直包含高级功能，可帮助组织降低风险故障的可能性。“XebiaLabs 预测性 DevOps 引擎是提高 IT 团队能力的又一重大进步，使他们能够从被动管理发布转变为主动预防失败并优化成功渠道的主动方法。"
