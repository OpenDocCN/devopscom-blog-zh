# 低代码操作如何改变开发运维

> 原文：<https://devops.com/how-low-codeops-can-transform-devops/>

在过去的一年里，随着新冠肺炎的经济后果推动了前所未有数量的数字化计划，企业采用[低代码](https://devops.com/category/blogs/low-code-no-code/)软件开发平台[在全球经济中激增](https://www.gartner.com/en/newsroom/press-releases/2021-02-15-gartner-forecasts-worldwide-low-code-development-technologies-market-to-grow-23-percent-in-2021)。企业正在[转向](https://www.gartner.com/en/newsroom/press-releases/2021-02-15-gartner-forecasts-worldwide-low-code-development-technologies-market-to-grow-23-percent-in-2021)低代码平台，以快速构建新的应用程序，让客户、远程员工和外部终端用户能够开展业务运营或转向新的机会。

然而，加速软件开发只解决了数字优先方法的一半难题。将解决方案快速送到用户手中需要仔细规划，以确定这些新应用程序将如何在组织的整体软件运营战略中发挥作用。这通常在多个云部署中进行。

但是当 low-code 的自动化和抽象的基本原则被扩展到软件维护和管理的 DevOps 领域时，真正的奖励就来了。只需点击一下鼠标，就能推出连续的软件更新流程，并提供简化的测试和质量保证，这需要一个平台来自动执行与在企业级生产环境中构建、测试、发布和维护应用程序相关的大量耗时任务。这不可避免地导致了“低代码操作”的概念或者，套用一句话，“低成本运营”

事实上，除非抽象和自动化的好处——在这种情况下，主要是速度——扩展到操作和部署，否则低代码开发的真正好处是难以捉摸的。一个让你在六天内构建一个应用程序，但需要六个月来部署的平台根本不是解决方案。

## 利用云实现改变游戏规则的结果

要见证无缝云运营在业务创新中发挥的重要作用，只需看看生物技术巨头 Moderna 及其在“曲速运营”中的成功。Moderna 使用公共云将地理上分散的合作者聚集在一起，整合科学见解并优化其生产周期，使该公司在新冠肺炎病毒基因组测序后仅 42 天就交付了第一批 mRNA 疫苗样本。该解决方案的价值实现时间无疑是惊人的。

而且，可以肯定，价格不菲。云工程师和 DevOps 系统管理员是高技能的专家，他们在凌晨 2 点卷起袖子，修复深埋在软件操作的互联生态系统中的意外故障。他们想出如何让软件实际运行。毫无疑问，企业希望能雇佣更多这样的人。

但是，同样的熟练 IT 专业人员的短缺也影响了 DevOps，这种短缺推动了当今对低代码软件开发的需求。在这种情况下，相同的低代码平台是解决方案，授权一个更广泛的跨职能，协作团队，以减少运营负担-一句话，低代码操作。

## 低成本运营的基本特征

为了在将应用程序和数据迁移到云中时实现无摩擦移交，利用低代码原则的生产环境将提供以下低代码操作的关键原则:

**性能监控:**平台基础设施的日常监控和日志检查是现代 DevOps 运营的支柱，确保复杂的相互依赖关系平稳运行。自动化仪表板监控可以将花费在这些重复性任务上的时间转化为工程团队专注于创新的机会，同时仍然确保企业软件组合的健康和性能。该工具需要支持对上游和下游服务的应用程序工作负载和性能特征的自动化端到端可见性。

**合规:**每个经济部门都受特定政府法规的约束。软件[平台](https://www.mendix.com/evaluation-guide/enterprise-capabilities/security/)和开发团队在将新功能集成到软件升级中时，必须跟踪每一个需求。银行客户必须在获得 ISO/IEC 27001 认证的基础设施上部署数字解决方案。医疗保健解决方案必须符合联邦强制要求的 HIPAA 要求。国防承包商必须通过 FedRAMP 认证。数据是任何数字解决方案的关键，它受制于不同国家和边界的隐私法。想象一下，您必须手动重新检查应用程序部署的每个迭代，以确保合规性。幸运的是，低代码操作意味着有一个自动化的、系统范围的合规框架。这缩短了服务组织(SOC)审计和报告的工作。

安全:在我们数字化互联的世界里，导致系统中断和破坏的网络攻击是[日益增长的](https://www.forbes.com/sites/chuckbrooks/2021/03/02/alarming-cybersecurity-stats-------what-you-need-to-know-for-2021/)担忧。资源丰富的云计算巨头也不能幸免于破坏性的网络攻击，比如去年的[网络安全管理软件产品](https://securityboulevard.com/?s=SolarWinds)黑客攻击。弹性是关键。持续的安全扫描可以在检测到可疑变化时停止自动软件部署。预防总是最好的。监控工具识别漏洞，使系统能够快速恢复在线，并帮助防止可能持续数小时或数天的停机。

**基础架构独立性:**广泛的企业[采用](https://www.forrester.com/report/A+Clear+Multicloud+Strategy+Delivers+Business+Value/-/E-RES128781)多云和混合云来提高运营效率，这使得灵活的基础架构成为新的标准。组织正在绕过单一供应商锁定，将工作负载分散到多个云合作伙伴。这意味着 DevOps 需要简化的配置和管理工具，这些工具可以跟踪资源之间的关系，并跟踪合规性问题。当云提供商与反对其他全球贸易伙伴的政府关系密切时，地缘政治考虑可能会进入等式。为使用企业数据和应用程序的众多供应商提供可观察性的监控工具是至关重要的。

恢复能力:混乱发生了。平台可以自动执行连续的域级指标，以关联不同来源的性能(包括日志、指标、应用程序跟踪和外部渗透测试)，从而加快检测和响应时间。快速审计和调试工具，结合实时仪表板监控和系统范围的警报，加上一盎司的检测，等于一磅的治疗。对于敏捷开发团队来说，被动消防被转化为弹性火力。

## 采用一体化低成本方法

低代码支持软件构思、创建和部署的频繁迭代的能力似乎与 DevOps 最小化风险的使命背道而驰。然而，同样激进的抽象和自动化使得软件开发速度更快，质量更高，更容易被大批开发人员使用，也可以减少景观管理的操作负担。每个低代码供应商都吹嘘其平台中的治理特性。但是治理的具体细节需要一些装配，包括系统范围内可操作的监控、测试警报和合规性保障。低代码操作结合了最高效、敏捷、灵活的开发平台和强大、简化的操作工具，为这两个领域注入了坚实的护栏，以实现操作控制和可预测性。采用这种方法将大大提高企业的价值实现时间，只需点击一下鼠标，就能将创新的数字解决方案直接送到最终用户手中。
