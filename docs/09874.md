# Datadog 扩展了集成 DevOps 平台的范围

> 原文:[https://devo PS . com/data dog-extends-reach-of-integrated-devo PS-platform/](https://devops.com/datadog-extends-reach-of-integrated-devops-platform/)

在其 [Dash 2022](https://www.dashcon.io/) 会议上，Datadog 今天宣布，它正在扩展其同名云交付监控和可观测性平台的覆盖范围，以解决持续测试、应用安全和成本管理问题。

此外，Datadog 还推出了测试版的数据流监控工具，可以更轻松地识别可能影响开发运维工作流的上游问题。测试版中还有一个动态检测工具，它可以利用 Datadog 代理软件从特定源代码中动态收集数据，以调查性能问题。Datadog 还在测试版中提供了一个资源目录，以便更容易地导航 it 团队正在使用的云资源，并添加了一个 Datadog 工作流工具来创建蓝图，以实现可重复流程的自动化。

Datadog 还在扩大其注入机器学习算法的看门狗遥测分析服务的范围，以增加对容器的测试支持。现在还有一个事件管理工具的测试版，DevOps 团队可以使用它来更容易地调查使用 Watchdog 服务的相关警报流。

Datadog 持续测试服务现已正式推出，它为 DevOps 团队提供了一个生成和维护测试的工作台，这些测试可以集成到[持续集成](https://devops.com/?s=continuous+integration) (CI)工作流中。Datadog 还在测试版中添加了一个智能测试运行器工具，通过分析代码执行路径来提高开发人员的工作效率，使 DevOps 团队只运行需要在发生变化时运行的测试。

Datadog 产品高级副总裁 Renaud Boutet 表示，目标是使 DevOps 团队能够使用一组无代码工具，在 Datadog 用户界面内并行创建和运行测试，而不必使用脚本。他补充说，该功能将使使用 Datadog 平台收集的数据自动生成测试成为可能，作为无限 DevOps 工作流循环的一部分。

与此同时，Datadog 正在全面提供云安全管理服务，该服务结合了其现有的云安全状态管理(CSPM)服务和云工作负载安全(CWS)服务，以在不增加成本的情况下提供集成的云原生应用平台(CNAP)。此外，Datadog 还可以根据使用目前处于测试阶段的工具检测到的已知漏洞，实时阻止对受到攻击或可能受到攻击的应用程序组件的访问。

这种方法通过为 IT 团队提供单一服务来降低网络安全的总成本，通过该服务，他们可以使用去年收购 Sqreen 获得的平台 Datadog 来管理应用程序安全。

最后，Datadog 正在全面提供云成本管理工具，该工具利用其监控功能，使组织能够自动将支出归因于应用程序、服务和单个团队。影响成本的支出模式变化与 IT 团队已经在使用的监控云计算环境的工具一起浮出水面。这种统一的可见性提高了成本意识，鼓励开发运维团队优化多类云资源的利用。

正在添加的服务是 Datadog 平台的扩展，该平台使用它提供的一组代理。Datadog 平台还支持开源的 OpenTelemetry 代理来提供广泛的服务。这种方法消除了 IT 团队每次添加额外的监控、安全、测试或成本管理工具时部署多个代理的需要。

Datadog 一直在为一个集成平台提供理由，该平台可以减少开发运维团队在部署单独的单点产品以解决监控、可观察性、安全性和成本管理问题时需要花费的集成工作量。这种方法使开发运维团队更容易协作，并降低了开发运维的总成本。相比之下，每次 DevOps 团队添加单点产品来解决特定问题时，他们都需要部署和维护一组单独的代理。

从长远来看，尚不清楚一旦开源的 OpenTelemetry 代理软件得到更广泛的应用，代理问题会有多严重。然而，开源软件要提供今天使用专有代理软件所能提供的相同功能，可能还需要几年的时间。

可能需要一段时间才能让每个应用程序都得到应有的充分利用，但随着监控和可观察性平台的不断发展，更主动地管理应用程序环境是可能的。
