# 数字颠覆者——建立 DevOps 领导形象

> 原文：<https://devops.com/digital-disrupters-building-devops-leadership-profile/>

虽然 DevOps 方法的进步对于许多组织来说仍然是相当初级的，但是一个不断扩大的领先实践者阶层正在发展一系列相关的最佳实践。

除了“独角兽”这一老生常谈的概念——为数不多的几个作为 DevOps 涅槃神话范例的组织——这些更广泛的早期采用者已经开始拉开他们努力的帷幕，为孕育这一当前成功模式的战略提供了进一步的细节。

上周，行业分析师 Freeform Dynamics 发布了[有价值的研究](http://rewrite.ca.com/us/articles/digital-transformation/exploiting-the-software-advantage-lessons-from-digital-disrupter.html)，强调了这类 DevOps 创新者正在进行的工作，这些创新者因其在追求更大规模数字化转型方面的成就而被称为“数字颠覆者”。由 CA Technologies 赞助的报告**“利用软件优势”**概述了一系列对开发运维前景至关重要的计划。

带头冲锋的那些组织的具体特征是什么？别人在哪里苦苦追赶？基于 1442 名受访者的许多 Freeform Dynamics 调查结果重申了广泛的 DevOps 观察结果，而其他发现则有助于阐明新观点，包括:

*   **每个人都在开发软件:**感知实际上就是现实，为了让数字化转型深入人心，数字颠覆者越来越认同这句话。虽然 95%的领导者承诺[变得更加以应用为中心](http://www.channelworld.in/interviews/app-economy-changing-cio-role%3A-stephen-miles-)，但只有 51%的主流组织采用了这种角色。类似地，84%的颠覆者直接认为自己是“软件公司”，而不考虑行业，相比之下，大众中只有 40%。

*   这一点看似显而易见，但确实值得重复。高达 92%的领导者积极行动起来，打破[开发和运营之间的传统孤岛](http://www.infoq.com/articles/patterns-devops-culture)，其中 51%的组织已经完成了这项工作。相比之下，只有 18%的主流组织声称已经做了同样的事情，至少有 59%的人赞同这种更具协作性的 DevOps 愿景。
*   承诺至关重要:这似乎也是显而易见的，但数字证明了这一要求。大约 77%的颠覆者拥有敏捷开发方法，70%的颠覆者“广泛”实施了开发运维活动。对于主流组织，这些指标的比率分别只有 31%和 21%；领导者正在全力以赴，享受其中的好处，那些跟随的人[也需要做同样的事情](https://devops.com/2015/10/12/building-winning-software-team-devops/)。
*   **实验无处不在:**为了获得持续交付的好处，[接受试错](https://enterprisersproject.com/article/2015/9/learn-how-leading-edge-enterprises-are-experimenting-their-way-success-devops)是必须的。在颠覆者中，93%的人认识到“在通往成功的道路上快速试验和失败”的必要性，而主流组织中只有 58%的人认识到这一点。大约 95%的领导者已经从固定的产品周期转向持续创新，而只有 56%的主流领导者实现了这一飞跃。
*   **API 无处不在**:无处不在的集成，无论是内部集成还是与第三方的连接，都增加了[对 API 的重视](http://www.infoworld.com/article/2911639/apis/6-ways-to-get-a-grip-on-the-api-economy.html)。颠覆者非常重视 API 管理，68%的人使用 API 将第三方服务集成到他们的应用程序中，66%的人提供对系统和数据的外部访问。主流同行的比例分别只有 26%和 24%，这表明 API 管理差距相当大。

这些仅仅是少数几个最引人注目和最明显的领域，在这些领域中，DevOps 倡导者已经明确地重新构建了他们的方法，将他们自己与那些尚未全力以赴的组织区分开来；它不止于此。

正如研究中所强调的，差异化和二分法的其他领域包括跨职能协调、对[数字渠道](https://www.internetretailer.com/2015/09/29/digital-channels-will-influence-64-holiday-purchases)的重视、使用数字技术提高核心业务效率，甚至是数字投资的融资。

除了这些遍布 DevOps 成功之路的关键哲学里程碑，调查还强调了这些组织寻求实现的大量战略目标。这些结果凸显了从业者认为 DevOps 可以带来的大量重大业务优势。

主要使用案例包括:

*   *提高客户满意度* (49%)
*   *更高的工作效率* (44%)
*   *改善客户体验* (41%)
*   *提高竞争优势* (38%)
*   *更好的敏捷性/上市速度* (37%)
*   *开拓新市场* (35%)
*   *创造新的商业模式* (30%)
*   *开辟新的市场路线* (26%)

如果要断章取义，与 DevOps 的影响无关，公平地说，这些结果读起来像是领先商业倡议的通用列表，进一步强调了该运动的整体相关性。

这并不是说这一切都很容易实现。报告受访者详细描述了来之不易的胜利，这些胜利需要大量的时间和投资来实现所有这些 DevOps 驱动的进步。毕竟,“颠覆”的概念并不能准确地唤起轻松或舒适的形象。
