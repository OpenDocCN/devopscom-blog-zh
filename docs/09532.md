# 调查显示应用程序现代化的高成本

> 原文:[https://devo PS . com/survey-reveals-high-cost-of-application-moderation/](https://devops.com/survey-reveals-high-cost-of-application-modernization/)

一项针对美国 250 名软件开发人员和架构师的调查发现，近四分之三(74%)的受访者报告称，一个[应用程序现代化](https://devops.com/?s=application+modernization)项目的平均成本接近 150 万美元，其中 79%的人指出这些项目中至少有一个失败了。

这项调查是由 Wakefield Research 代表 vFunction 进行的，v function 是一家自动将整体应用程序转换为微服务的平台提供商，调查对象是拥有 5，000 名或更多员工的组织中的开发人员和架构师，这些员工从事业务至少 15 年，并且正在维护 Java 应用程序。调查发现，应用程序现代化项目失败的主要原因是预期设置不正确(43%)以及需要进行组织结构变革(37%)。

尽管存在成本和固有风险，但调查还发现，整整 92%的受访者计划或正在对应用程序进行现代化。近三分之二(62%)的受访者表示，拥有合适的资源和工具是应用程序现代化成功的关键。

超过一半的受访者(58%)还指出，完成这些项目平均需要 16 个月，超过四分之一的受访者(27%)指出需要两年或更长时间。不足为奇的是，整整 97%的人说他们预计他们组织中的某个人会推迟一个提议的项目。

当然，应用程序现代化是一个包罗万象的短语，可以涉及从为应用程序提供 web 前端到将它们提升并转移到云中或者完全重构它们以在云平台上本地运行的一切。

vFunction 首席执行官默蒂·拉法林表示，无论该计划的范围如何，大多数组织都应该首先根据对业务最有意义的内容来选择这些项目。他补充说，一旦确定，鉴于失败的高成本，it 团队就必须制定周密的战略来实现这一目标。

Rafalin 指出，组织还应该明智地确定工作角色将如何演变，因为应用程序现代化的最大障碍之一是担心任何工作可能会受到影响。

目前尚不清楚组织选择更新现有应用程序还是直接替换它们的速度有多快。然而，随着越来越多的组织选择在云中运行应用程序，使用微服务来构建应用程序的兴趣水平急剧上升。目标是利用微服务使应用程序更具弹性和安全性。代价是基于微服务的应用程序往往有许多依赖性，这使得它们更难以管理和保护。

无论采用哪种方法，这更多的是一个应用程序何时现代化或退役的问题，而不是是否。随着时间的推移，每个应用程序的技术债务水平最终会变得不可持续。当然，问题是大多数组织没有一次性实现每个应用程序现代化所需的资源，因此可能需要十年或更长时间来完成整个应用程序组合，即使该组合在继续增长。事实上，随着技术的不断发展，应用程序现代化可能不再是一项需要完成的任务，而是 it 事务的一个永久状态。
