# 在开发现代 z/OS 应用程序时利用开源生态系统

> 原文:[https://devo PS . com/leveling-the-open-source-ecosystem-when-developing-modern-z-OS-applications/](https://devops.com/leveraging-the-open-source-ecosystem-when-developing-modern-z-os-applications/)

今天的开发人员面临着比以往任何时候都更快地创建应用程序的巨大压力，他们必须不断更新应用程序，添加新功能并融入新技术。许多公司采用 DevOps、云原生架构和低代码开发解决方案来加快开发和更新周期。但是现代应用程序开发的另一个重要方面是利用开源软件的众多优势。

开源软件的优势包括新版本的快速周期(与许多商业应用程序相比)、对添加用户请求的功能的更快响应、适应新技术的频繁更新以及带来各种不同方法来解决问题的庞大开发人员基础。然后是商业利益。开源软件通常比商业产品成本更低，更灵活(更少被厂商锁定)，也更透明。此外，开源软件完美地补充了今天的云原生应用程序，这些应用程序将松散耦合的组件结合在一起，作为旧的整体应用程序的替代方案。

这些好处推动了企业对开源软件的使用，尤其是在过去几年里。过去的一些研究强调了开源软件的广泛使用和日益增长的使用。[一项研究](https://www.securitymagazine.com/articles/92368-synopsys-study-shows-91-of-commercial-applications-contain-outdated-or-abandoned-open-source-components)发现，基本上所有被检查的代码库(99 %)都包含至少一个开源组件，其中 70%的代码是开源的。在去年发布的另一项[调查中，75%的企业表示开源软件非常或极其重要。在同一项调查中，77%的公司表示，他们计划在未来 12 个月内增加开放软件的使用。](https://enterprisersproject.com/article/2020/2/10-stats-enterprise-open-source)

**开源和 z/OS**

对开源软件的热情延伸到了 z/OS 应用程序开发社区，原因和普通应用程序一样。也就是说，在开发现代 z/OS 应用程序时，公司可以依赖一个庞大的解决方案和开发人员生态系统。

特别是，有许多 z/OS 开源软件小组、项目和成果可供借鉴。

首先，有一些开源编程语言和工具，如 Python、Git 和 cURL，让开发人员无需学习新技能就能使用 z/OS 系统的强大功能。

有像[开放大型机项目](https://www.openmainframeproject.org/)这样的组织，其目标是成为在大型机计算环境中部署和使用 Linux 和开源的焦点。

这个小组的一个项目是 Zowe，它是一个现代 z/OS 系统的开源框架。Zowe 弥合了现代应用程序和大型机之间的鸿沟，在来自多个供应商的产品和解决方案之间提供了更容易的互操作性和可伸缩性。

还有像 Ansible 这样的开源项目，这是一个用于应用部署、配置管理和编排的开源社区项目。它适用于 z/OS。Ansible 允许开发人员自动化 z/OS 应用程序。借助 Ansible，开发人员可以使用现有的 JCL、REXX 和 z/OSMF 来自动化基于 z/OS 的软件。这使得开发人员可以对混合应用程序采用通用的方法，并以一致的方式将 z/OS 集成到企业自动化策略中。

使用这些开源 z/OS 产品的一个巨大优势是，拥有广泛可用技能的开发人员可以创建新的 z/OS 应用程序。随着拥有原生 z/OS 技能的人才库逐渐减少，这是一个重要的优势。许多从事大型机和 z/OS 编码的人正在退休。许多年轻的开发人员在工作中学习为 z/OS 编程。

**企业能力仍然是必须的**

正如从 Linux 到 Apache webserver 到 Kubernetes 的许多开源解决方案一样，使用它们的公司必须用额外的功能来补充解决方案，以确保可用性、安全性和性能。

自己动手(DIY)的方法当然是可能的。但是这需要大量的时间和资源。它还需要一家公司可能不具备的技能。雇佣具备这些技能的员工或让内部员工接受这些领域的培训也是既费钱又费时的。

另一种选择是与一家技术提供商合作，该提供商将带来 z/OS 专业知识、开源知识以及使解决方案为企业所用的特性和人员。
