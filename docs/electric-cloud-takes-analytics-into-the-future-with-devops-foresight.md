# 电子云以“DevOps 远见”引领分析走向未来

> 原文：<https://devops.com/electric-cloud-takes-analytics-into-the-future-with-devops-foresight/>

*“electric flow devo PS Foresight”使用深度学习来识别发布渠道中的模式，衡量软件发布成功的可能性，并将基于改进渠道提供建议*

加利福尼亚州圣何塞，2018 年 6 月 25 日——自适应发布流程编排和持续交付的领导者 Electric Cloud([http://electric-cloud.com/](http://electric-cloud.com/))宣布推出新的预测分析解决方案 Electric flow devo PS Foresight([http://electric-cloud.com/solutions/devops-foresight/](http://electric-cloud.com/solutions/devops-foresight/))。DevOps Foresight 将机器学习算法应用于工具链生成的大量数据，以开发一种风险评分指标，在产品投入生产之前预测发布的结果。预测分析更进一步，它还将说明基于开发人员影响、代码复杂性影响等在哪里改进管道。

ElectricFlow DevOps Foresight 通过更清晰地洞察现有的开发实践和发布模式，为 DevOps 团队提供了消除“发布焦虑”的进一步来源的必要步骤。现在，软件交付过程中的瓶颈和低效更容易被发现和纠正。了解新的和复杂的应用程序和环境需求的资源分配的能力也有助于确保获得成功软件发布的最佳途径。

**要安排 ElectricFlow DevOps Foresight 的演示，请访问:([http://electric-cloud.com/solutions/devops-foresight/](http://electric-cloud.com/solutions/devops-foresight/))**

基于对过去成功和失败模式的深刻理解，DevOps Foresight 预测一个版本成功的可能性。与信用评分非常相似，版本风险评分数值的创建基于开发人员、代码和环境概况，并为利益相关者提供了一种简单、直观的方式来解释特定构建或管道的成功可能性。如果得分很高，DevOps 团队可以查看这些配置文件，以确定是什么(特别是在这些配置文件中)增加了风险。

为了说明改进管道的领域，DevOps Foresight 着眼于促成因素以及过去有助于改进它们的因素，并将建议在团队、代码或环境中进行适当的更改。经理将能够主动回答以下问题:

*   我们会按时完成发布吗？
*   我们能不能走得更快或者做得更多？
*   这个版本会引起或多或少的质量问题吗？
*   生产部署失败的可能性有多大？

Electric Cloud 首席执行官 Carmine Napolitano 表示:“Electric Cloud 长期以来为其客户提供了深入了解正在发布的版本的好方法，但对于组织来说，在版本最终确定甚至开始之前了解版本的风险一直是一个挑战。“改进渠道通常基于试错或最佳猜测。我们使用 ElectricFlow DevOps Foresight 的目的是通过查看过去的成功案例、构建复杂性、作者简介等，在流程的更早阶段提供数据驱动的洞察，然后根据事实显示可以在哪些地方改进管道。我很自豪地说，Electric Cloud 的团队已经开发了一个工具，可以为我们的客户节省数百个小时的工作，并缓解他们交付的每一个产品不必要的发布焦虑。”
