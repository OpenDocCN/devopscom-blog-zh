# CircleCI 将使用人工智能来增加集体开发人员的智能

> 原文:[https://devo PS . com/circle ci-will-use-ai-to-increase-collective-devo PS-intelligence/](https://devops.com/circleci-will-use-ai-to-increase-collective-devops-intelligence/)

CircleCI 承诺为其持续集成/持续交付(CI/CD)平台添加[额外的集体智能功能，该平台将利用机器学习和其他形式的人工智能(AI)来优化应用程序开发和交付。](https://www.businesswire.com/news/home/20221116005775/en/CircleCI-Achieves-Significant-Growth-Puts-Big-Bet-on-Collective-Intelligence)

CircleCI 首席执行官吉姆·罗斯(Jim Rose)表示，随着该公司的同名 CI/CD 平台继续从多个客户收集更多遥测数据，创建将优化从应用测试到可靠性和性能的一切的人工智能模型变得可行。

作为该战略的一部分，CircleCI 还增加了一些功能，如测试分割建议，这些建议采用机器学习算法来估计在测试作业上启用并行性的预期时间节省。

此外，CircleCI 正在创建一个分析师和数据科学家团队，以基于通过该公司平台的云版本收集的数据开发额外的人工智能能力。

今年早些时候,[收购了单元测试平台提供商 Ponicode](https://devops.com/circleci-acquires-ponicode-to-gain-ai-testing-tool/) ,随后决定加大对开发人工智能能力的投资。该平台通过提供语法验证、悬停提示和自动完成功能等功能，使用人工智能来增强应用程序测试人员，从而提高使用 Visual Studio 代码开发的软件的准确性。目标是将人工智能测试技术集成到 CircleCI 平台中，以便在 DevOps 工作流中不断更新时，可以将算法应用到单个开发人员的代码以及从一个构建中的多个开发人员聚合的代码。

Rose 指出，工作流构成了一个软件供应链，在许多方面，它的功能类似于任何其他类型的供应链。他补充说，因此，有很多机会应用人工智能模型来优化这些工作流程，就像人工智能被应用于优化其他类型的供应链一样。

就将 AI 应用于 DevOps 流程而言，现在仍处于早期阶段，但很明显，有必要进一步自动化广泛的手动流程。人工智能不太可能很快取代对 DevOps 工程师的需求，但它将减少目前阻碍生产力的大量辛劳。当然，挑战在于训练一个人工智能模型需要大量的遥测数据，而这些数据通常只能通过云平台有效地收集和存储。不太清楚的是，人工智能功能是否会推动更多组织从部署在内部 IT 环境中的传统 CI/CD 平台转移，在内部 IT 环境中可能没有足够的遥测数据来有效地训练人工智能模型。

有一点可以肯定的是，现在有一场将人工智能应用于 DevOps 工作流的竞赛。采用 DevOps 的组织通常高度致力于自动化，但迄今为止应用于这些工作流的人工智能数量有限。然而，在未来一年，应用于 DevOps 工作流的人工智能数量将会稳步增长，尤其是在许多 DevOps 团队努力跟上复杂应用环境的步伐时。
