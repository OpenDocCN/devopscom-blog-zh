# Devs 会回办公室吗？

> 原文:[https://devops.com/will-devs-return-to-the-office/](https://devops.com/will-devs-return-to-the-office/)

沙琳·奥汉伦和 CRO 的迈克尔·戴维斯在 [Rollbar](https://rollbar.com/) 讨论最近[开发者](https://devops.com/?s=developers)调查的结果，调查探索开发者对重返工作场所的感受。下面是视频，后面是对话的文字记录。

戴维斯:是的。这是一个很好的介绍，虽然我可能会把我们推回到 COVID 之前。我认为这与我们合作的许多软件开发团队尤其相关。如果你追溯到 COVID 之前，我们有一些客户，他们的整个开发团队甚至没有笔记本电脑。他们几乎每天都在办公室。因此，现在不仅公司在 COVID 后挣扎于这种转变，而且他们在 COVID 前经历了巨大的转变，对吗？这无疑给公司带来了很大的压力，但更重要的是给实际的开发者带来了压力。

正如你所描述的，我们现在即将结束，我们正在经历另一个转变。我同意，我认为调查开始表明，这不是非黑即白的事情。由于你已经提到的许多原因，它不会是一个或另一个的全部。该调查非常清楚，在第一次报价中，它说 77%的开发人员肯定受到了 COVID 的显著影响，以及这对他们意味着什么，现在当您开始采访人们并进行调查时，您会发现这是一个真正的大杂烩。有些人已经享受到了自由，享受到了灵活性，并开始在某种程度上适应这种情况。

但我认为最有趣的发现之一是他们缺少工作中面对面的部分。如果你想想开发人员和他们的工作，这是一项团队运动，对吗？

奥汉隆:对，对。

戴维斯:他们真的很怀念那种日常的、面对面的互动，尤其是考虑到赌注的增加，这也是这次调查的另一个结果。我们发现这种发展的外部压力甚至更大了。为什么？因为客户需要通过软件与公司互动。因此，更快地构建更多软件的压力更大了。这是一些有趣的事情，我们看到了它们之间的对立。

奥汉隆:那么你认为这里的答案是什么？我理解开发者渴望面对面的交流。我也看到有些人现在正面临某种程度的社交焦虑，这种焦虑可能是他们一生中从未面对过的。如果你离开某人或一大群人一年，让自己重新融入那个环境会很难。我们都说我等不及让事情回到过去的样子，我们可以去听音乐会，我们可以去购物，不用担心戴口罩或其他什么，你把人们放在这些情况下，这对他们来说很难。有些比其他人多。

有些就像，是啊，无论如何，你知道，让我们回到过去，一切都会好的。但我不确定当人们说我想和所有这些人一起工作，我想回到 scrum 中，我想做这个做那个，我只是不确定当他们真的这样做时，那真的是他们想要的。如果你明白我的意思，你认为会有买家后悔吗？

戴维斯:是的，有可能。我的猜测是——同样，这是基于我们看到的一些调查数据——你确实对你的生活平衡有这种复杂的感觉，并不一定是你一直在家所认为的那样。但我想你会看到同样的事情。如果我们全职回到办公室，人们会在通勤和随之而来的事情上感受到同样的压力。所以我的直觉告诉我，既然公司已经调整，人们也更加适应离开，我们将会看到这种混合环境。

你将会看到这样一种环境，在这种环境中，人们会在特定时间呆在办公室里，但他们也会在特定时间在家工作，这又会带来一系列问题。我们已经看到在工具方面的大量投资，Zoom 是我们目前正在进行的投资之一，但还有很长的路要走。因为调查显示的另一件事是，有了这些距离，就更难得到那些答案，比方说，作为一个开发人员，你可能在那一刻需要这些答案。当有人离你不在两个隔间之外时，你只能希望他们在偷懒。如果他们不是，那么你有一个延迟和问题。

因此，我认为我们看到的下一件事是，我们看到在更多工具、更多类型的流程方面的更多投资，从而为这种混合环境提供更多帮助。

欧汉伦:你认为我们会看到全职的分布式劳动力增加吗？这可能是疫情大规模人口外流的直接结果，也可能是因为他们认识到我们不必一直呆在办公室里。或者乔走了，他可以全职做，所以我要做同样的事情，他们会怎么做？

戴维斯:对。[笑]

奥汉隆:剥夺我的生日？这将是其中之一——我认为将会有这种推拉效应。所以我想知道你同意我吗？这是很有可能发生的事情吗？你看到了什么？组织如何——你期望他们如何对此做出反应？

戴维斯:是的。除了你提出的观点，我还想补充一点，我认为公司已经认识到，一旦你不再需要每个人都在办公室，这也开始释放你更广泛地招聘的能力，这给你带来了灵活性，无论他们坐在哪里，你都需要这种类型的人，对吗？

奥汉隆:对。

戴维斯:所以我认为这也将会发生，这就是为什么我认为它真的会是这种混合环境。那么他们能做些什么呢？嗯，他们必须非常关注人们是如何工作的，他们必须进行投资。公司必须投资于正确的工具。我会在这里吹响我们的号角。Rollbar 就是这样一个平台，它可以让你在整个团队中看到你代码中的特定错误。

这是一个例子。如果公司有这样的工具，现在你不必坐在两个隔间之外。你需要的数据就在你面前，实时的，这样你才能做出这样的决定。这是一个例子。还有很多很多，但我们需要以不同的方式思考这些工作流，并确保您拥有正确的流程和数据，以便人们能够更高效地运营。

奥汉隆:是的。我认为这将是有趣的一年，因为我们在乔维德之后甚至是乔维德最后的阵痛中回归生活，因为在过去的 12 年里发生了太多的事情，对我们所有人来说都有很多事情要处理。现在，对于我们来说，回到过去，我认为这是不可能的。我不认为我们会回到 COVID 之前的样子，因此这意味着组织内部和员工之间以及他们对工作的理解和日常工作方式都将发生巨大变化。我认为，如果你愿意的话，公司将不得不做出很多让步，在雇佣和留住员工方面，他们将不得不采取全新的策略。

但员工也需要理解，在一天结束时，有一项工作需要完成，有时完成这项工作的最佳方式可能是在办公室或面对面。因此，这里没有放之四海而皆准的解决方案。迈克尔，很好的结果。关于开发人员，以及他们如何看待工作或工作/生活平衡或工资或你们在这次调查中涉及的任何类似的事情，调查中还出现了其他事情吗？

戴维斯:我们实际上并没有太多地谈到薪水。我认为最重要的一点是这对开发者的影响有多广泛，以及他们想做什么。这是第一件事。第二件事是，与此相关的许多压力和紧张是一种正常的过渡，但也是对软件的更高需求，我不认为我们中的任何人真的预测到了这一点，这是对世界上已经存在的软件的补充。每个公司都是软件公司，对吗？这就是挑战，也是 Rollbar 和其他公司正在努力帮助这个社区解决的挑战，因为这很困难。这是一项艰难的工作。

奥汉隆:是的。至于你的观点，我认为没有人会预料到，但是你认为这将是可持续的吗？我的意思是，如果你愿意的话，这是否会成为一种新的常态？至少从产品的角度来看不一定——即使从生产率的角度来看也是如此，但你认为需求会像过去 18 个月左右那样高吗？

戴维斯:我对此毫不怀疑。我是说，这个系统已经承受了巨大的压力。这些团队已经感受到了巨大的外部压力，要求他们能够更快地交付特性和功能，对吗？我们都听说过迭代速度。因此，我现在没有任何疑问了——再说一遍，我认为公司已经认识到，也许他们不需要同样的零售业务来服务他们的客户群。这又给软件需求带来了更大的压力。

所以，是的，这一切都有助于我认为更具挑战性的环境，但肯定是一个有益的环境，因为这些人现在正处于一切的中心。

奥汉隆:是的。但我确实担心精力耗尽。我确实担心这样一个事实——我认为很多人在疫情的开始和整个过程中都挺身而出，因为他们确实认识到了它的价值。他们实际上可以看到他们的工作如何影响他们的公司。我不确定一旦我们回到那种，不管我们叫它什么，那会持续下去，它真的会给开发者和其他员工他们需要的那种推动。我确实担心，我们很可能会看到开发人员的集体倦怠，这对每个人都不好。
