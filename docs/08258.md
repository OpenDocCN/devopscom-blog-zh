# 可编程性是敏捷转换的关键

> 原文:[https://devo PS . com/programm ability-is-key-to-agile-transformation/](https://devops.com/programmability-is-key-to-agile-transformation/)

企业敏捷性已迅速成为企业长期弹性的最关键变量之一。随着新冠肺炎疫情、全球供应链永无止境的中断以及几乎每个行业的典型流程颠倒过来，从来没有比现在更重要的时间来优先考虑敏捷性。

虽然敏捷性的回报很高，但组织正在努力实现关键技术，并坚持能够实现它们的方法。事实上，[第 14 次年度敏捷状态报告](https://stateofagile.com/)指出，59%的组织遵循敏捷原则，但是只有 4%的组织获得了全部好处。但快速适应、抓住新机遇和降低成本的能力对于在不断发展和高度竞争的数字时代生存至关重要。

敏捷性是指快速适应变化和提高生产率。重要的是，敏捷方法本身并不敏捷，因此不会神奇地将企业转变为敏捷组织。相反，重点需要放在实践正确的原则上，以成为真正的敏捷。考虑到这一点，让我们来看看这两个关键原则——可编程性和可观察性——如何提高生产率，并允许整个企业的创新。

## 可编程序性

可编程性描述了硬件和软件改变和接受一组改变其行为的新指令的能力。让我们来看看这两种形式。

使用可编程性的一种方法是将模拟器件与软件相连，例如软件支持的网络路由器。最近，可编程性已经扩展到使用网络来控制所涉及的硬件并与软件系统桥接。基础设施即服务(IaaS)的日益流行增加了对可编程性的需求，并将基础设施即代码(IaC)等概念带入现实世界。

可编程性的另一个用途是扩展现有的软件功能。这方面最广为人知的两个例子是[应用编程接口](https://devops.com/?s=APIs)(API)和软件开发工具包(SDK)。使用这两种方法，开发人员可以扩展现有程序的默认功能。API 作为业务 API 的重生极大地扩展了应用软件系统中可编程性的引入。该方法特别受应用服务提供商的欢迎，因为它使最终用户能够扩展和定制由产品或服务提供的核心能力。

## 连接敏捷性和可编程性

那么这两个概念是怎么联系起来的呢？敏捷性需要基础和合适的环境来提供最佳结果。可编程性是为敏捷性创建适当路径的使能器。具有高度可编程组件的组织增加了自动化可重复任务的能力。让我们看两个场景:一个业务用例，一个技术用例。

## 商业中的可编程性

当采购订单(PO)到达时，客户服务代表必须验证组织满足请求的能力，然后在后端客户关系管理(CRM)系统中创建订单。传统上，这个过程需要大量的人工工作:首先，提取 PO 数据；然后使用其他系统或电子表格验证细节，最后将数据输入 CRM 系统。

相比之下，考虑一个人在一个具有高度可编程性的现代环境中工作。PO 可以以电子格式接收，例如 JavaScript 对象符号(JSON)、可扩展标记语言(XML)或电子数据交换(EDI)。然后，可以通过连接到其他系统和参考数据存储库来验证它，提示用户验证它并将数据输入 CRM 系统。

## 技术环境中的可编程性

在一个典型的技术用例中，软件开发人员将完成编写代码；[质量保证](https://devops.com/?s=quality+assurance) (QA)工程师必须测试发布；发布工程师必须验证它，DevSecOps 团队必须部署发布的工件。与业务用例类似，这种技术工作流传统上需要大量繁琐的手动工作。

然而，当开发组织使环境可编程时，该过程变得明显更加简化。在这种情况下，一旦开发人员提交了代码，自动化流程就会启动，应用提交的代码，运行整个生命周期阶段，并将其部署到生产中。

通过可编程性支持和扩展的自动化，组织可以简化其可重复的任务，并为用户提供更多的时间和空间来处理基本的或创新的任务。

现在，让我们看看两个用例的实现细节。

## 数字供应链中的可编程性

在业务用例中，工作流的自动化，例如 PO 处理，被视为现代数字供应链计划的一部分。这种流程中涉及的系统、消息传递和数据使用集成技术连接在一起。

集成方法首先涉及点对点连接，然后发展到使用集中式企业服务总线(ESB)。最近，现代方法围绕着以 API 为中心的集成微服务和集成平台即服务(iPaaS)解决方案，这些解决方案使用分散模型和云原生技术。

现在，组织通过结合 API、集成和安全技术来创建数字供应链流，从而构建或收购数字平台。任何数字供应链成功的关键是能够选择具有内置 API 功能的系统，并使现有系统能够将应用程序和服务作为 API 公开，并使整个企业可编程。

## 安全软件供应链中的可编程性

在前面讨论的技术用例中，DevSecOps 自动化为软件生命周期的各个方面的自动化提供了基础。在这里，开发工具、测试工具和治理相关的工具都使用管道连接。

IaC 和自动化构建服务器是自动化这种流程中使用的两种流行技术，我们称之为安全软件供应链。Puppet、Chef 和 Ansible 是广泛采用的 IaC 脚本语言，而 Bamboo 和 Jenkins 是流行的构建服务器。然而，随着 DevOps 空间随着云原生和去中心化技术的发展，GitHub 和 GitOps 也在构建有效管道方面发挥着至关重要的作用。

与业务用例类似，这里所涉及的组件需要连接到构建管道，因此所涉及的每个元素的可编程性在自动化工作流中起着重要的作用。此外，为了构建全面的管道，所涉及的代码质量、单元测试、自动化测试、基础设施供应、工件构建、依赖性管理和部署工具必须使用 API 进行连接，并使用 IaC 扩展所需的功能。

## 可观察性

通过分析业务和技术用例，我们对自动化的影响以及如何通过可编程性实现自动化有了清晰的认识。然而，我们错过了一个重要的元素:[可观测性](https://devops.com/?s=observability)。让我们在这里检查一下。

自动化和可编程性创造的环境为团队提供了一个很好的操场，让他们练习变得更加敏捷，尝试新的想法，并迅速将它们投入生产。虽然敏捷性提高了，但运行健康的系统以提供持续的数字体验的责任是至关重要的。

可观察性有助于使用分析、机器学习(ML)和人工智能(AI)来检测和预测问题。现代云基础设施在运营层面提供了一定程度的可观察性。与此同时，一些现代编程语言，如 [BallerinaLang](https://ballerina.io) ，内置了对与特定语言相关的应用程序级可观察性的支持。当使用缺乏此功能的语言时，开发人员将不得不使用与编程模型兼容的第三方跟踪库，或者直接插入可观察性指令以及跟踪和调试指令。

当今的企业正在迈向敏捷性，因为他们寻求优化他们的数字战略。然而，组织的敏捷性需要赶上项目的敏捷性——然后，赶上每个人的敏捷性——才能更有效。这就是“结构化敏捷性”的概念，它是由“双披萨团队”或自组织团队推动的。

API 是可编程性的基础，这使得连接模拟和数字系统以及扩展软件系统成为可能。可编程性提高了交付速度，使企业变得“可组合”,具有高度的可重用性，并通过开放创新来实现灵活的环境——这是敏捷性的核心原则。
