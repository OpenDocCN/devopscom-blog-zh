# Catchpoint 发布 WebPageTest API 的增强版

> 原文：<https://devops.com/catchpoint-releases-enhanced-version-of-the-webpagetest-api/>

*(美国纽约)-2021 年 4 月 6 日-[数字体验监控(DEM)领域的领导者 catch point](https://www.catchpoint.com/)宣布全面推出 WebPageTest API，该 API 之前仅限于少量用户使用，同时还进行了多项升级，包括更深入的性能指标、即时测试结果、专门支持、开发人员资源和 CI/CD 集成。*

 *增强的 WPT API 提供对网页测试数据和测试基础设施的即时编程访问，包括深入的前端性能指标和全球最新浏览器用户体验的并排视频比较。

“我们很高兴整个开发社区都可以使用 WebPageTest API，”WebPageTest 的创始人兼 Catchpoint 的工程研究员 Patrick Meenan 说。“看到开发人员利用 API 的灵活性来优化网站性能并将页面速度指标集成到他们的开发工作流程中，这将是一件令人兴奋的事情。”

**持续交付更快的网页**

API 的可移植性使开发人员能够在模拟的用户环境中运行性能测试，而不用离开他们的 CI/CD 工具来持续提供快速的 web 页面。

Catchpoint 为 WebPageTest 推出了官方的 GitHub Action，使开发人员能够根据性能预算测试他们的代码更改，并在他们现有的工作流中获得详细的性能信息。

Catchpoint 首席执行官 Mehdi Daoudi 表示:“将 WebPageTest 数据与您的 CI/CD 流程相集成，可以将性能带入开发对话中，确保网站在发布并由用户在生产中体验之前达到速度和可用性标准。“在开发的所有阶段，开发人员将从缩短的反馈循环、加速的交付和更深入的性能可见性中受益。”

**更深入的性能调查，自动化**

除了开发集成之外，WebPageTest API 还为性能工程师和技术 SEO 顾问提供了巨大的价值，他们现在可以自动进行更深入的性能调查和审计，以发现提高网站性能的新机会。

“WebPageTest 是页面加载性能分析的行业标准。谷歌 Chrome 软件工程师亚历克斯·罗素表示:“能够以高度的信心和可重复性隔离出各个因素，就能很容易地查明和沟通渐进式变化如何能让事情变得更好。”。“我所知道的最好的专业 web 性能调查人员将 WPT 作为他们工作流程的重要组成部分，这也是我的工作中心。”

**支持长期核心网络生命战略**

谷歌计划从 2021 年 5 月开始将性能指标(核心网络生命指标)纳入其搜索排名算法，这突显了进一步加快页面加载速度的需求。随着利润丰厚的搜索流量岌岌可危，核心网络生命指标已经成为一个通用的性能和自上而下的组织必要性的晴雨表。

Catchpoint 的性能工程研究员 Tim Kadlec 说:“WebPageTest 突出了影响核心 Web 生命的因素，因此开发人员可以优化其性能，并随后优化其首页的搜索排名。“在发布过程中强制执行核心 Web Vitals 性能预算，可确保所有未来页面的长期高性能。”

**SaaS 平台整合**

此外，Cloudinary、Cloudflare 和 Snyk 等世界一流的 SaaS 提供商已经利用 WebPageTest API 来构建产品集成，以执行图像和漏洞分析，从而提高客户的页面加载速度和安全性。

Snyk 开发人员关系部的 Liran Tal 表示:“如今，漏洞管理不再是一个不错的选择，而是任何运行 web 应用程序或交互式静态网站的人的一项关键要求。“这些面向公众的资产是恶意行为者寻求未经授权访问系统和数据的常见攻击媒介。Snyk 使用 WebPageTest 来执行第三方库检测和漏洞扫描，以揭示您网站页面中的潜在安全漏洞。”

WebPageTest API 服务现已推出，包括开箱即用的集成和对 Catchpoint 屡获殊荣的支持组织的访问，以及更新的文档中心和论坛。这个新的 API 服务不会影响免费版的 WebPageTest。在这里了解更多:[https://www.product.webpagetest.org/api](https://www.product.webpagetest.org/api)
