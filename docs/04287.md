# DevOps 工具中 UI 和 UX 的出现

> 原文：<https://devops.com/emergence-ui-ux-devops-tools/>

市场上大多数流行的 DevOps 工具，如 Puppet、Chef 和 Jenkins，都是开源的，需要很高的熟练程度。随着复杂性而来的是学习新的框架和范例的挑战，这对那些想要挑战自我和学习新技能的人来说是一种好处。

但是，虽然这些工具已经被广泛采用，但 DevOps 的广泛采用正在创造一种对界面的需求，以满足所有用户的需求，从大多数技术工程师到那些不想学习、看不到学习价值或不需要复杂脚本界面提供的一些顶级功能的人。

正是因为这个原因，CloudBees 为其持续交付工具开发了新的界面 Blue Ocean，Red Hat 为 Ansible 发布了用户界面(UI)。

## UI 的合适时机？

为什么之前没有 ui？

我会说两个原因:

1.  到目前为止，用户并不需要也不关心它。在许多圈子里，UI 被认为对可以实现的东西有更多的规定/限制，只有脚本/编程接口才能向工程师展示真正的力量。此外，许多最进步的工程师喜欢脚本；他们从学习新技能中获得满足感，尤其是那些最需要的技能。并且，我认为很多工程师喜欢的东西是因为它们很难做；这是一个挑战，他们有吹牛的权利，这意味着他们可能会得到更多的报酬/很受欢迎。
2.  开源社区不擅长或不适合构建 ui。

为什么 UI 现在变得重要了？

在 MidVision，像许多其他商业自动化工具供应商一样，我们一直认为用户体验和可访问性至关重要，因此我们为我们的工具提供了命令行、脚本、API 和 GUI 界面。但这并不是常见的方法。

让这个时刻变得不同的是 DevOps 自动化工具被那些至少想要图形用户界面(GUI)和用户体验改进的群体更广泛地采用，这样你就不需要非常精通技术就能使用它们。

我们看到的主要群体是:

*   **组织内的业务用户。**持续交付/发布自动化工具极大地促进了协作、互动和自助服务，允许发布经理查看管道状态、批准发布或启动部署。经理可能希望看到仪表板和指标，生成报告或使用数据。其他非技术人员、离岸团队、二级支持等。可能希望能够调配、克隆环境或部署更改。
    在地下室说一种语言的书呆子和说另一种语言的商业用户之间有一条鸿沟，易访问性、用户界面和 UX 希望能够弥合这条鸿沟。
*   **操作。**IT 运营的某些部分已经张开双臂欢迎基于脚本的技术工具，这是 UNIX 系统管理员成为开发运维/自动化工程师的自然演变。他们仍然可以在 shell 中编写脚本(或者他们打算全部用 Python 来做吗？？).同样，一些安全工具、存储等。一般来说，在草丛里会更舒服。然而，随着您在堆栈中向上移动，朝向业务应用程序部署，用户对 ui 更加熟悉和舒适。此外，在这个级别的仪表板上，指标、访问控制和报告要重要得多——如果没有合适的 UI，所有这些都很难实现。
*   **更广泛的行业采用。**devo PS 世界中经常提倡的工具和技术的采用始于相同的海报子公司——网飞、Twitter、脸书、谷歌——以及云原生的、具有开源优先政策且不受传统技术栈阻碍的其他新公司。

我们已经看到部分企业采用了 DevOps 实践和工具。显然，从小处着手，建立价值，然后扩大规模是很重要的——像在企业内部创业一样运作一项计划效果很好。

既然有令人信服的证据表明 DevOps 实践提供了竞争优势、成本节约、改进的上市时间等。由于各种原因，其他严重依赖技术的行业还没有采用这些实践。金融和医疗保健行业的某些部分就是两个这样的领域。这些行业对安全性和稳定性有所顾虑，这可能减缓了它们的采用速度。

然而，总的来说，这种更广泛的采用意味着我们不仅需要迎合新一代的 DevOps 工程师，他们既是系统管理员又是程序员，可以用 Shell、Rudy 和 Python 编写脚本。我们还需要迎合更广泛的群体，他们要么没有使用这些工具的技能，要么不关心它们。

那么，我们可能会看到什么变化呢？

我相信 DevOps 工具将会不断发展，提供新的接口来迎合业务用户和自助服务，并使之更容易实现同样的事情，而不需要在被认为是合格用户之前有多年的使用工具的经验。

我会把今天的许多工具比作地图和指南针。定向运动或登山运动的爱好者可能会认为在手机上使用 GPS 或地图是作弊，不可靠，或者不会提供与 1:10，000 地图上的等高线相同的信息。或者比起 Spotify，你更喜欢黑胶唱片。

这两件事都没有错；他们并不是唯一的权利。如果你想带着你的地图和指南针、黑胶唱片和 Python 脚本呆在外面，我建议你穿上靴子。所有这些，顺便说一句，我有一些热情！然而，如果你也想进入你的汽车，输入一个邮政编码并知道它会带你去那里(如果你做了更彻底的研究，接受那里可能是一条稍微快一点的路线)，这对我来说非常有意义。

用户体验是我们选择消费的大多数其他产品的核心，我认为许多新用户和现有用户都会对 UX 变得越来越重要感到高兴，因为像我们和其他人这样的工具将这一点放在了他们设计的最前沿。
