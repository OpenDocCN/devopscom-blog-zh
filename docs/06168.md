# Deque 引入自动化源代码可访问性检查器

> 原文:[https://devo PS . com/deque-introduces-automated-source-code-accessibility-checker/](https://devops.com/deque-introduces-automated-source-code-accessibility-checker/)

Axe Linter 只需最少的努力就能检测到并建议可访问性修复； *新的 GitHub 应用对个人存储库免费*

弗吉尼亚州赫恩登——2020 年 5 月 5 日——专注于数字辅助功能的领先软件公司 Deque Systems 今天推出了 axe Linter，这是一款基于 GitHub 的自动化应用程序，可以检查源代码中常见的辅助功能问题，自动找到并建议修复方法。

开发人员可以通过使用 axe Linter 在开发过程的早期捕捉可访问性问题来补充现有的可访问性测试工作，从而大大减少未来的测试和补救工作。一旦为 GitHub 组织启用了 axe Linter，它就可以在每个存储库上安装或强制执行，在每次更改或请求时自动运行，而无需开发团队的进一步输入或处理。

Axe Linter 现在可以在位于 https://github.com/apps/axe-linter 的 GitHub 网站或者位于 axe-linter.deque.com T2 的 Deque 网站上获得，并且对所有个人仓库都是免费的。Axe Linter 目前支持 React (JSX / TSX)、Vue、HTML 和 Markdown 格式。

Deque Systems 首席技术官 Dylan Barrell 表示:“Axe Linter 通过将检查转移到开发的最早阶段来节省时间和资金，因此可访问性问题甚至可以在代码合并之前就被发现和修复。“一旦安装到 GitHub 存储库中，它就不需要开发人员做更多的工作来获得代码更改的反馈。Axe Linter 只分析新的或更改的代码，并就如何解决可访问性问题提出建议。我们预计它将成为我们 axe DevTools 解决方案的必备元素。”

Axe Linter 建立在基于 WCAG 2.1 和大量最佳实践的不断增长的规则列表之上。该应用程序已经过优化，返回零误报。

Axe Linter 可以对开发工作的可访问性产生立竿见影的影响，有助于使网站和应用程序对包括残疾人在内的所有人更加无障碍。

“Axe Linter 为源代码提供了比按钮更简单的可访问性检查，确保可访问性检查适用于每个更改或拉取请求。Deque Systems 首席执行官 Preety Kumar 表示:“当与广受欢迎的免费 axe 浏览器扩展(用于在浏览器中进行测试)或任何 [Deque 的企业产品(用于测试)相结合时，开发团队就拥有了他们所需要的一切，可以对他们的应用程序产生巨大的影响，使其更易于残疾人使用。”。](https://www.deque.com/tools/)
