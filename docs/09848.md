# DataOps Vs. DevOps:有什么区别？

> 原文:[https://devops.com/dataops-vs-devops-whats-the-difference/](https://devops.com/dataops-vs-devops-whats-the-difference/)

今天有令人难以置信的大量数据；为了测量它，需要使用一个称为 zettabyte 的字节度量，它是一个六次方字节(即 21 个零)。目前，由于存在如此大量的数据，结束浪费的数据处理变得越来越紧迫。在这种环境下，DataOps 诞生了。与过去企业采用 DevOps 来形式化和简化浪费的开发实践的方式类似，今天，许多大型组织转向 DataOps 来形式化现代数据管理实践。

## 数据行动到底是什么？

首先，许多公司采用这些原则是因为他们试图避免或纠正数据债务，这是由于数据流程管理不善而修复数据问题所需的资金。数据债务是修改过时流程和政策的强大动力，尤其是当决策者和利益相关者在实施变更之前需要指标时。未支付的数据债务可能对企业不利；不付费的时间越长，维护数据环境的成本就越高。

通过实施 DataOps 原则和数据治理，组织可以有效地减少其数据债务，并防止其进一步增长。此外，DataOps 实践和软件工程可用于检测低效率，最大限度地减少知识损失，并利用与数据使用相关的错失机会。

## DataOps 和 DevOps 的相似之处

许多支持数据操作的流程最初都借鉴了构建开发运维的基础。同样，正如公司需要 DevOps 为软件和功能开发提供高质量、一致的框架一样，数据企业也依赖这些相同的功能来实现快速数据工程和分析开发。对于已经部署了 DevOps 框架的组织来说，利用 DataOps 非常简单。DataOps 采用的几个重要 DevOps 概念包括:

*   敏捷开发
*   专注于交付商业价值
*   持续集成和持续交付(CI/CD)
*   自动化测试和代码升级
*   重用和自动化

## 差异

尽管 DevOps 和 DataOps 的基础有相似之处，但仍有几个主要差异。

**人的因素:**使用 DataOps 和 DevOps 的人有不同的个性和技能。DataOps 参与者可能精通技术，但他们的知识通常是理论性的。DataOps 专业人员包括专注于创建模型和可视化辅助工具的数据工程师、数据科学家和分析师。然而，DevOps 是为软件开发人员和工程师设计的——编码是他们的基因。

**流程:**data ops 和 DevOps 的生命周期确实共享相似的交互属性。但是，前者的不同之处在于，它由数据管道和分析开发过程组成，两者都是活动的和交叉的。虽然从概念上讲，DataOps 的管道类似于 DevOps 的开发流程，但专家指出，DataOps 流程通常更具挑战性。

**编排:**在 DevOps 流程中，应用代码不需要复杂的编排。然而，对于 DataOps，数据管道和分析开发流程编排都是必不可少的组成部分。尽管 DataOps 管道中的编排经常发生并驱动数据流，但在应用程序开发和 DevOps 流程中通常没有这种管道协调。

**测试:**同样，DataOps 的两条管道与 DevOps 产生了显著的差异；DataOps 中的测试发生在数据管道和分析开发过程中。这些测试试图捕捉异常，标记异常数据值，并与开发运维不同，在部署前验证新的分析。同样，这些测试被嵌入到数据质量框架中，用于持续监控。

**测试数据管理:**在大多数 DevOps 环境中，测试数据管理几乎不占优先地位；借助 DataOps，加速分析开发至关重要，这样创新才能跟上敏捷迭代的步伐。

**工具:** DevOps 是 DataOps 的“父亲”，因此，支持后者所需的工具仍处于起步阶段。虽然 DevOps 中的测试主要是自动化的，但 DataOps 却没有这种奢侈——大多数用户修改测试自动化工具或从头开始构建自己的工具。

**探索性环境管理:**一般来说，数据团队比软件开发团队使用更多的工具。此外，从工具和数据的角度来看，数据分析中的探索性环境更具挑战性；数据团队也很自然地脱离了企业中的数据孤岛。

虽然从根本上来说，DevOps 的概念是 DataOps 的起点，但后者涉及到在操作数据和分析产品时最大限度提高效率的额外考虑。然而，两者都服务于他们的目标受众，减少数据债务和发展数据产品或缩短系统开发生命周期或提供连续交付。对于希望提高内部数据相关流程效率的企业，他们应该从检查与开发运维相关的最佳实践开始。
