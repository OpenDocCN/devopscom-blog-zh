# 2022 年最佳:伟大的 DevOps 倦怠

> 原文：<https://devops.com/the-great-devops-burnout/>

随着 2022 年的临近，我们 DevOps.com 想要突出今年最受欢迎的文章。以下是我们 2022 年最佳系列的最新内容。

DevOps 需要一个心态转变来拯救过度工作的工程集体。现在有很多关于大辞职的讨论。但是如果你从事软件工程，你可能会更加熟悉巨大的倦怠。

在过去的几年里，软件开发领域的一切都在加速发展。工程师们每天都以极快的速度工作，并且非常精确。对新的、更新的或更好的应用程序的需求从未减缓。但是你的 DevOps 团队只能执行这么快。

## DevOps 工作过度，心力交瘁。不，真的。

整整 83%的软件开发人员对他们的工作感到倦怠，而 81%的人报告说由于疫情，倦怠有所增加。导致这种疲惫感的最重要的因素包括高工作量、低效的过程和定义不明确的项目目标。

过度工作、超负荷工作和耗尽开发人员的精力是导致灾难的原因；这导致了跳槽和低生产率。组织需要改变他们关于我们如何开发应用程序和培养开发者的思维模式，以使软件开发的未来可持续发展。

## 不均衡的工程需求对您的底线造成的负担

软件交付的可持续性取决于员工的快乐程度。今天，工程师的供求完全不平衡，两者之间的鸿沟继续扩大。

全球数量最多的 SaaS 公司位于美国。绝大多数(73%)的美国企业正计划在不久的将来转向 vast 雇佣优秀工程师并找到平衡的需求将在未来几年持续增长。

此外，到 2028 年，软件开发人员的就业预计将增长 21%，高于所有职业 5%的平均预计增长率。

对于工程师来说，这种增长是个好消息。但对于领导层来说，对工程师的不均衡需求意味着，考虑开发团队的痛点，并为过度工作等常见问题提供解决方案，比以往任何时候都更加重要。

## 基础设施需要改变以减轻开发人员的疲劳

在已经超负荷的工程文化上投入更多的钱和更多的人不会使 SDLC 更可持续，也不会帮助您实现业务目标。

如果我们从“五个为什么”中抽出一页，深入到通用 DevOps 耗尽的根本原因，在所有软件工程方法中有一个共同的线索。DevOps、敏捷、六适马、精益、Scrum，你能想到的——这些都是久经考验的方法，如果只解决了基础设施问题的症状，这些方法就会失败。如果我们要适当地利用这些可持续发展的框架，我们需要重新考虑我们当前的工具选择。

组织可以通过专注于节省开发人员的时间和减少辛劳来转移进一步的工程倦怠和反弹:劳动工程师致力于重复、枯燥或手工任务。

有必要通过自动化来推进基础设施，并采用注重开发人员体验的下一代工具来阻止进一步的开发过度。

## 让我们尝试一些不同的东西:自动化 API

要拯救 DevOps 免于灭亡，需要转变观念。您可以通过采用自底向上的方法并将更多智能自动化引入到您的软件开发基础设施中来实现这一点。

自动化将开发人员从平凡或低价值的任务中解救出来，将他们的时间和才能集中在规划、创建和交付高质量的应用上。通过提高 DevOps 功能的质量，工程师们对自己在公司中的角色感到更有价值，更有自豪感。

自动化任务不只是帮助当前的 DevOps 团队。拥抱自动化解放了您现有的劳动力，降低了招聘需求，同时保持了他们的 DevOps 团队以最高效率运行的信心。这种方法解决了问题的根源——太多的工作，没有足够的开发人员——并允许组织通过使他们的工作生活更好、更适合居住来长期留住开发人员！

让技术去做平凡的事情，这样开发人员就可以在重要的地方应用他们的专业知识。少雇人，多做事。自动化工作流，以便开发人员可以做他们真正的工作:编写代码。回避就业需求问题，保护开发商。

当然，这种方法并非没有缺点。如果需要高度专业化的开发人员进行过多的定制，即使是近乎完美的产品也可能导致失败。然后，由于为定制和数据库管理开发 API 所需的人工劳动，成本开始飙升。

当有效地完成时，持续开发是改进 DevOps 的基础设施和用自动化影响倦怠的一个光辉的例子。持续集成和持续部署(CI/CD)可以更快地从您的客户群获得反馈，让您能够对功能进行调整，添加或删除实际适合的内容，并更快地为您的客户解决问题。

## 3 个下一代开发工具，让您的团队从持续的辛劳中解脱出来

通过整合旨在改善开发人员体验的下一代开发工具，解决开发人员超载问题。利用三种新兴的开发工具来修复 DevOps 倦怠，让工作日更易于管理，而不是更复杂。

1.自动化代码建议
上下文感知代码完成工具在编码过程中向开发人员建议代码。自动化的代码建议节省了开发人员宝贵的时间(和人力),因为它减少了击键次数，并且旨在提高整个编码过程的速度。
2。 **Instant API** 无论您需要支持 web 应用程序还是移动应用程序，将您的 API 和数据库托管在一个平台上的云中，都允许团队配置、测试和管理 API，而不会给开发人员带来太熟悉的手动负担。
3。**神经网络**
人工神经网络是机器学习的一个分支，是从复杂或杂乱的数据中提取意义以检测趋势和模式的计算模型。神经网络虽然不完美，但可以在程序员写作时向他们提出改进建议。

我们对工程人员造成了严重的伤害，因为我们忽视了巨大的倦怠，也没有解决从根本上纠正开发过程所需的转变。如果没有必要的基础设施自动化进步和以开发人员为中心的工具的采用，团队将继续与速度需求和流失作斗争。是时候接受一种新的思维模式，开创一个 DevOps 生产力和开发人员满意度的新时代了。
