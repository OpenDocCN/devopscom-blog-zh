# 运用测试者培养质量文化

> 原文:[https://devo PS . com/applying-testops-to-culture-a-quality-culture/](https://devops.com/applying-testops-to-cultivate-a-quality-culture/)

质量保证是一个普遍关注的问题，尤其是在应用程序开发中。企业应用程序比以往任何时候都更快、更可靠。然而，除了性能之外，应用程序开发人员还必须考虑数据安全性、法规遵从性和成本效益等因素。组织正在接受[质量工程](https://devops.com/survey-sees-application-testing-shifting-further-left/) (QE)作为创建软件质量文化的基础，以应对这些挑战。创建一个高质量的工程或测试环境并不简单，尤其是当 QE 部门必须在多个企业环境中测试数百个应用程序时。无论是解决方案提供商还是跨国公司，应用程序经常需要跨平台、跨行业和跨地域进行测试。

TestOps 包含了比 QE 更广泛的概念。有了 [TestOps](https://devops.com/?s=testops) ，测试人员和工程师使用自动和手动测试来验证软件的完美应用性能。在 enterprise DevOps 中，TestOps 团队充当开发和部署之间的桥梁，因为他们对应用程序有很好的理解，并且对可能影响性能的上下文和变量有更好的看法。

持续测试是任何软件开发周期不可或缺的一部分。质量工程师必须作为敏捷开发的一部分尽早加入进来，以保持开发在正轨上，并阻止任何可能升级或在以后变得更难解决的问题。在敏捷中，测试变得更加重要和频繁，以确保完美的性能。

在设置 TestOps 方法时，横向考虑是有好处的。例如，测试应该在不同的垂直环境中进行，如媒体、制造和金融。目标是创建一个可以适应各种行业的测试环境。

### 跨环境的测试者

在 SaaS 和通用数据访问的时代，跨多个平台进行测试是必不可少的。除了确保企业性能，还必须测试移动 web 和远程数据访问。对于流媒体服务，跨移动平台、Roku、Apple TV、Android TV 和 Fire TV 等 OTT 平台，甚至 PlayStation 和 Xbox 等游戏机进行测试至关重要。

当发送软件更新时，地理位置很重要(尤其是在跨时区发送时)，因为人们可能永远不知道软件何时会被使用，或者无法控制他们计划使用它的环境。

例如，考虑一家领先的抵押贷款机构在移动应用程序测试方面面临的挑战，以及地理位置是一个怎样的因素。首先， [Google Analytics](https://analytics.google.com/analytics/web/) 用于评估特定设备和用户需求，并确定美国不同地区的性能，因为不同地区使用不同的运营商。由于大多数银行和抵押贷款应用程序需要一次性密码(OTP)和短信相关功能，因此测试在 iOS、Android 及其组合等移动平台和运营商之间进行。对生物识别和面部识别等新的身份认证策略进行了进一步的测试。最后，按地理位置进行性能测试，以确定新特性或版本是否会产生问题。

### 针对地理位置和区域合规性的测试

本地化测试非常重要，因为每个地区和每个国家都有独特的合规问题和法律。为了确保正确的本地化，必须执行完整的测试，包括较低的环境和生产系统。

在任何 TestOps 环境中，持续测试的理念应该已经是设计环境的一部分，包括理解产品的细微差别。发布就绪并不意味着在某些地区只有几个已知的 bug。这意味着该应用程序已经在每个可能使用它的地区进行了合规性测试。

### 跨行业测试

在特定行业环境中测试应用性能会带来一系列独特的挑战。例如，在制造业中，物联网设备中的每个传感器都必须根据一致性规范进行建模，以确定是否可以通过模拟实现自动化。在媒体领域，人们需要测试不同的平台、内容管理系统和业务工作流程。

为了加快上市时间，人们需要支持双班开发生命周期来衡量消费者体验并获得持续的反馈，还需要一个具有多种自动化工具的测试环境，如 Selenium、Appium、Cypress、Tricentis、UiPath、Test Complete 和 Winium。

人工智能(AI)在测试、缩短上市时间和确保优质软件方面发挥着更大的作用。人工智能驱动的测试工具针对特定领域的应用进行了优化，例如媒体、制造和金融科技等行业的测试。人工智能可以填补空白以实现优化，并在有限的人工干预下提供更快的周转。

任何培养 TestOps 文化的组织都需要有特定的技术和工具。当评估您的测试技术栈时，创建一个端到端的测试生态系统是很重要的。这意味着环境必须具有包容性，并且能够针对独特的应用程序、环境和场景进行测试。为了支持当今敏捷应用程序开发环境的需求，我们必须规划一个敏捷测试基础设施来保持开发的正常进行。
