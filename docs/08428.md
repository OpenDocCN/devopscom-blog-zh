# Sumo 逻辑扩展了可观察性的范围和深度

> 原文:[https://devo PS . com/sumo-logic-extends-observability-scope-and-reach/](https://devops.com/sumo-logic-extends-observability-scope-and-reach/)

在在线 [Illuminate](https://www.sumologic.com/illuminate/) 用户大会期间，Sumo Logic 透露了它如何将 2021 年 6 月收购开源监控工具提供商 Sensu 获得的能力与其持续智能平台相集成，此外还增加了对通过开源 OpenTelemetry 软件收集的数据的支持。

Sensu Go 是一套实现代码监控的工具，现在是持续智能平台的核心部分。该公司现在还提供 beta Sumo Logic OpenTelemetry 发行版和生态系统支持，这是一个单一代理，通过它可以从仪表化应用程序中收集所有日志、[指标](https://devops.com/?s=metrics)和跟踪。Sumo Logic 还支持 AWS OpenTelemetry 发行版，该发行版由 AWS 提供，提供类似的功能。Sumo Logic 还免费增加了 Sumo Logic 的功能，包括 Sensu 创建的代码监控工具。

产品和战略副总裁 Bruno Kurtic 表示，随着管理应用程序的责任转向开发人员，提供可通过编程方式注入应用程序的可观察性功能至关重要。作为这一努力的一部分，Sumo Logic 现在还提供 Sensu Plus，它为平台收集的数据提供了一个分析引擎。

Sumo Logic 还添加了机器学习算法，以通过作为核心平台一部分的警报响应功能，为平台生成的警报提供更多上下文。

该公司还增加了对 Red Hat 创建的操作员软件的支持，以简化部署 Sumo Logic 连续智能平台以及 Red Hat OpenShift 平台，从而在 Kubernetes 集群上构建和部署应用程序。此外，Sumo Logic 还提供与 38 个新的或更新的可观察性应用程序的集成，这些应用程序可与各种第三方平台集成。

最后，Sumo Logic 正在扩展其基于其平台提供的安全可观察性平台，以添加对容器和无服务器计算功能以及虚拟机的支持，并集成第三方端点检测和响应(EDR)、网络检测和响应(NDR)、威胁情报和托管检测和响应(MDR)平台。该公司还为特定的第三方平台增加或更新了 26 个应用程序。

Kurtic 说，Sumo Logic 使开发人员和 it 运营团队能够通过单一窗口端到端地观察 IT 环境。在某些情况下，IT 运营团队的成员需要一个用户界面来管理该环境。同时，开发人员可能更喜欢应用程序编程接口(API ),通过它可以从持续集成/持续交付(CI/CD)平台中访问可观察性数据。

然而，不管用户的类型如何，IT 团队都不应该雇用他们自己的数据科学团队来分析可观测性数据。他说，相反，可观测性平台应该嵌入机器学习算法，这些算法可以展现数据科学的所有好处，而不需要雇用额外的员工。

无论如何，DevOps 团队将很快从可观察性平台提供的额外的上下文级别中受益。接下来的挑战将是确定如何最好地解决许多 DevOps 团队根本不知道存在的问题。
