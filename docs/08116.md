# Grafana Labs 收购 k6 添加开源负载测试工具

> 原文:[https://devo PS . com/grafana-labs-acquisites-K6-to-add-open-source-load-testing-tool/](https://devops.com/grafana-labs-acquires-k6-to-add-open-source-load-testing-tool/)

在今天的 GrafanaCONline 2021 会议上，Grafana Labs 透露它已经被 T2 收购了 K6 T3，一个开源负载测试工具的提供商。

Grafana Labs 首席执行官 Raj Dutt 表示，负载测试将补充 Grafana Labs 目前基于开源 Prometheus 软件提供的[可观测性](https://devops.com/?s=observability)平台。他指出，任何负载测试过程的大多数输出通常都是使用分析指标的平台来评估的。

Dutt 说，k6 平台在需要使用一种工具的开发人员中受到了欢迎，这种工具可以更简单地根据需要动态扩展负载测试，而不是猜测测试应用工作负载可能需要多少基础设施。Dutt 补充说，这种能力不仅使在 DevOps 工作流程中集成测试变得更加简单，而且有助于降低测试成本。

总的来说，Dutt 指出，随着开发人员对质量保证承担更多的责任，测试正在进一步左移。Dutt 补充说，面临的挑战是找到一种方法，使开发人员能够自动将测试纳入持续集成(CI)过程，使他们能够轻松地观察合成测试是如何执行的。

毫无疑问，在将工作负载提升到生产环境之前，开发人员正在进行更多的测试，以减少在生产环境中部署应用程序时可能遇到的问题。在许多情况下，现在出现的支持电话直接打给开发人员，他们现在对减少这些电话的数量有既得利益。

开发人员面临的问题是，随着他们采用微服务，跨多个应用程序存在的依赖性使得成功部署这些应用程序变得更加困难。Grafana Labs 一直在为一个云平台提供案例，该平台可以更简单地集中查看从分布在多个应用程序开发环境中的 Prometheus 实例收集的数据。Prometheus 现在在负责监督 Kubernetes 开发的云本地计算基金会( [CNCF](https://cncf.org) )的支持下得到发展，现在正被 IT 团队更广泛地部署，以观察传统的单片应用程序以及新兴的基于微服务的应用程序。

还不清楚组织在多大程度上也雇佣了专门的应用程序测试人员。在许多情况下，那些测试人员会暴露出开发人员在紧张的应用程序交付期限面前忽略(或者干脆选择忽略)的问题。

然而，从长远来看，人工智能(AI)显然将在应用测试中发挥更大的作用。仍然需要有人审查这些测试，但是更多的机械任务将逐渐自动化。

不管怎样，测试的总成本正在下降，部分原因是由于对开源测试工具的依赖增加，在许多情况下，开发人员只是自己下载。这些工具减少了开发人员可能遇到的大量许可摩擦和成本限制。

事实上，测试应用程序越容易，在未来的几个月甚至几年中，应用程序的整体质量就会变得越好。在组织比以往任何时候都更加依赖软件时候，应用程序质量整体状态的潜在改进不会来得太快。
