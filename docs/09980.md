# 在云中实现左移安全性

> 原文:[https://devo PS . com/implementing-shift-left-security-in-the-cloud/](https://devops.com/implementing-shift-left-security-in-the-cloud/)

虽然勒索软件在过去几年中一直是企业安全团队的主要担忧，但软件漏洞也紧随其后。基于云的应用和服务的蓬勃发展以及工作的日益数字化对黑客来说是一个福音，他们正在利用开发人员和 DevOps 团队更快更智能地工作以满足需求的尝试。一项估计称，在过去十年中，10 起零日攻击中有 4 起发生在 2021 年。

许多因素导致了这一增长。开发人员捉襟见肘，并且正在[重用代码](https://devops.com/software-quality-is-the-heartbeat-of-the-best-organizations/)，这使得错误配置和漏洞在不同的程序中意外地重新出现，并且多个云服务的使用破坏了安全措施，降低了运行许多企业功能的代码的可见性。这就是为什么开发人员和安全专业人员在整个软件开发生命周期(SDLC)中，尤其是在早期阶段，更加关注安全性。

### 左移安全原则和挑战

零日激增导致对左移实践的兴趣增加，作为一种使安全性成为开发过程中的优先事项的方法。左移文化在软件生命周期的更早阶段，在软件部署之前，就将安全性带入了等式中，而不是在用户报告错误之后修补它们。这种先发制人的方法有助于防止可能影响应用程序安全状态的漏洞。

当开发人员为云平台(如亚马逊网络服务、微软的 Azure 或谷歌云)构建应用程序时，左移原则也可以增强安全性，因为平台的专有代码和安全工具的可见性可能是有限的。在左移文化中，DevOps 将最低权限策略嵌入到云工作负载的日常工作中，以保护网络基础设施并避免对这些工作流授予过多权限。

例如，在 Kubernetes 容器上设置基于角色的访问控制(RBAC)会在这些集群上强制实施最低权限模型，并避免可能导致违规的过度权限，同时从持续集成/持续交付(CI/CD)工作站中删除管理员凭据会阻止黑客将这些管道用作攻击媒介。

左移安全性是一个很好的概念，但在执行中遇到了障碍，主要是开发团队之间的内部冲突，他们希望与业务保持同步，而安全人员则采取更谨慎的方法。这导致了摩擦，开发人员警告拒绝访问会影响他们的工作并导致部署延迟，而安全分析师则担心太多用户拥有未经检查的管理权限会导致数据泄露。

解决这种摩擦需要思维方式的转变，DevOps 明白安全性是开发过程的一部分，管理层可以相信开发人员已经掌握了安全性。这还需要工具和培训来集成左移实践，例如可以桥接 CISO 的安全需求(例如，通过授予对网络基础设施代码的更大可见性)和开发运维团队的敏捷性需求(例如，通过自动化安全护栏和授权)的平台。最终目标是创建一个 DevSecOps 模型，将开发、安全和运营整合到一个敏捷、安全和高效的工作流程中。

### 如何在云上采用左移安全性

四个最佳实践会有所帮助:

●获得深入的可见性:由于企业经常在多个云环境中工作，并且将公共云和私有云混合到混合基础架构中，对资产的可见性通常会受到影响，这使得获得对风险的清晰视图变得困难。能够发现环境中的所有身份、权限、配置和资源以及特定资源的所有访问路径，有助于构建云资产的上下文清单，从而更好地管理它们并执行策略分析。

●优先考虑风险:风险管理是任何安全计划的关键实践，但来自多个云提供商和系统的大量警报可能会淹没安全运营中心，这些中心通常缺乏人才和时间。自动化工具现在可以在从开发到生产的整个云环境中搜索风险并确定其优先级，发现问题组合，并减轻在多个孤岛中手动整理警报的工作量，使员工能够专注于优先级更高的缓解和补救任务。

●即时访问:零信任架构已经成为大多数安全防御的标准，但它经常受到管理访问权限的困难的挑战。授予有限时间可撤销权限的实时(JIT)访问是构建零信任安全架构和实施最低特权策略来维护该架构的有用工具。主动管理和监控开发人员对云环境的访问的技术工具，包括对特权活动的强大审计跟踪，为安全性提供了一种实施 JIT 访问和零信任的方法。

●发现并防止漏洞和错误配置:基础设施即代码(IaC)使得云中的软件部署(通过使用虚拟机、容器、微服务等)更加敏捷和高效。然而，在建立云基础架构时，安全性通常是事后才想到的，无论是生产前还是生产后。组织应该扫描代码以发现违反安全策略的情况和其他风险，以便在缺陷进入生产环境之前将其消除，并检测和修复生产环境中的问题。通过开发工作流中内置的反馈和自动化护栏，开发人员和安全团队可以降低代码中的安全风险。

像零信任一样，左移正在成为网络安全保证的标准做法。但是构建一个 DevSecOps 模型需要的不仅仅是良好的意愿。它需要实践、工具和培训来执行该模型，以及跨企业和与外部云和安全合作伙伴的合作心态。只有团队的努力才能提供敏捷企业所需的云安全保证。
