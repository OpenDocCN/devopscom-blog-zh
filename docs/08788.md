# 超越 IaaS:使用 EaaS 的好处

> 原文：<https://devops.com/beyond-iaas-the-benefits-of-using-eaas/>

**超越 IaaS:使用 EaaS 的好处**

云基础设施市场正在快速增长。驱动因素包括 IT 需要削减成本、提供更多服务、更快地响应业务需求，以及实现创新和数字化转型。多年来，基础设施即服务(IaaS)产品一直是企业消费的第一个也是最常用的云服务。但是越来越需要更多。这为环境即服务(EaaS)打开了大门，有些人称之为 IaaS Plus。

要正确看待对这类服务的需求，请考虑云支出。虽然大多数组织在 COVID 之前就已经在执行他们的云迁移战略，但疫情为那些落后者敲响了警钟。Gartner 预测今年全球公共云服务支出将从 2020 年的 2700 亿美元增长到 3323 亿美元。Gartner 还[预计](https://www.gartner.com/smarterwithgartner/cloud-shift-impacts-all-it-markets/)到 2024 年，几乎一半的软件、系统基础设施和流程外包方面的 IT 支出将从传统解决方案转向云。

在这些数字中，IaaS 细分市场预计将在未来几年取得最大的增长(百分比)。IaaS 市场将从 2020 年的约 60，000 美元增长近一倍，至 2022 年的约 107，000 美元。展望明年，ResearchAndMarkets.com[预测](https://www.businesswire.com/news/home/20210813005334/en/Infrastructure-as-a-service-IaaS-Global-Market-Report-2021---ResearchAndMarkets.com)IaaS 市场将在 2025 年达到 746.3 亿美元，复合年增长率(CAGR)为 13.8%。

为什么要大跳？基于云的服务模式可以为企业提供显著的财务优势，允许他们快速访问新功能，而不需要大量的前期投资。云服务将一些资本支出成本转化为运营成本，基础架构即服务(IaaS)使组织能够将基本的存储和网络功能从内部硬件转移到云中。

**输入 EaaS**

公司在过去几年中遇到的一个最大的根本性变化是，对新应用程序和服务的业务需求迅速增长，因此需要加快应用程序开发。

最重要的是，开发不再是一次性的工作，应用程序必须不断地修补、更新和增强。每次更改或添加都需要一个或多个开发环境，以及用于性能测试、质量保证和部署的基础设施和应用程序。

这就是 EaaS 发挥作用的地方。EaaS 将 IaaS 的功能和优势扩展到了开发环境中。它提供了开发运维团队开发、部署和维护应用所需的基础设施、应用和数据。

理想的 EaaS 解决方案将提供某些功能来提高 DevOps 人员的效率，降低成本，并帮助企业保持对快节奏开发环境的控制。

例如，一个非常适合的 EaaS 将使用自动化来为特定的应用程序配置服务器。这将是一个自助服务产品，集成了正常的开发运维工作流、工具和方法。例如，一个企业可能会寻找一个 EaaS 产品，让用户可以选择如何访问服务。这可能包括通过 web 门户、命令行界面或直接通过开发人员的 CI/CD 工具进行访问。

鉴于应用程序的开发和维护方式，最后一项功能在今天尤为重要。EaaS 必须集成到现有的 DevOps 生态系统和 CI/CD 管道中，以支持当今对按需交付的完整、随时可用的环境的需求。确保这一点的方法是寻找一个 EaaS 提供者，它支持开箱即用的插件和一个用于通用 CI/CD 和 DevOps 工具的扩展 REST API。

企业需要解决的另一个问题是成本控制。随着云服务的使用变得如此简单，以及一些提供商提供自助服务选项，成本可能会迅速增加。在许多情况下，云服务用户直到每月账单到达后才能了解他们的云支出。即便如此，对于成本发生在哪里，通常也没有详细的了解。

我们所需要的是一个整合了云成本管理功能的 EaaS 产品，其实现方式不会降低应用程序开发和交付的速度。特别是，成本可见性和控制是必需的，但不能以增加流程摩擦的方式实施。许多成本管理方法在这方面都失败了。好的 EaaS 解决方案将提供所需的可见性和控制，同时不会阻碍开发和部署。

**最后一个字**

现代应用程序开发必须基于 CI/CD，以便及时交付新的应用程序、服务和更新，从而满足快速变化的业务机会和需求。

虽然 IaaS 平台满足了许多核心基础架构要求，但 EaaS 通过提供开发运维所需的按需交付的完整环境来补充这些功能。EaaS 还提供了与 DevOps 工作流和 CI/CD 管道的紧密集成。
