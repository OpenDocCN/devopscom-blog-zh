# DevOps 十二大奖 2022 提名现已开始

> 原文：<https://devops.com/devops-dozen%c2%b2-awards-2022-nominations-now-open/>

DevOps 十几个奖项第八年回归，提名已经开始。这些奖项旨在突出、认可和表彰在过去一年中为 DevOps 社区做出重大贡献的杰出和鼓舞人心的领导者。我们为我们社区的爆炸性增长和持续成功感到非常自豪，我们很高兴庆祝它的辛勤工作和成就。

连续第三年，我们扩展了 DevOps 十几项大奖计划，纳入了两个不同的组别:DevOps 十几项社区大奖和 DevOps 十几项工具和服务大奖。

每组有 12 个奖项:

*   **DevOps 十几个社区奖**—表彰对 DevOps 社区产生积极影响的领导者。
*   **DevOps 十几种工具和服务奖**—表彰那些开发和交付卓越解决方案以增强开发人员、DevOps 和 IT 运营团队能力的公司。

提名是由观众做出的。任何人都可以提名一个工具或服务，尽管工具和服务奖提名需要 250 美元的报名费。我们收取这项费用是为了减少垃圾邮件和多余的提名。但是，社区奖提名不收费。请注意，商业工具和服务是 *而不是* 有资格获得社区奖项，并且不会进入决赛。

将选出决赛选手，并从决赛选手中选出获胜者。最终入围者将由公众投票(占最终总分的 40%)和我们的评委评选(占最终总分的 60%)决定。

提交您提名的最后一天是**2022 年 10 月 22 日** 。要投票，请按照 [DevOps 打网站](https://devopsdozen.com/) 上概述的步骤进行。

## **2022 年 DevOps 十几个类别**

### **DevOps 十几个工具和服务奖项**

*   **最佳端到端开发运维工具/服务** 开发运维领域的整合为提供端到端解决方案的平台带来了更多选择。该奖项旨在表彰为开发运维团队提供最多样化、最强大解决方案的平台。

*   **最佳 DevOps Repo/GitOps 工具/服务** GitOps 已经成为 DevOps 工具箱不可或缺的一部分。该奖项颁发给最杰出的代码库/Git/GitOps 工具。

*   **最佳 CI/CD 工具** 这个奖项表彰帮助团队更快交付高质量代码的最佳 CI/CD 工具。

*   **最佳价值流管理工具** 价值流管理已经成为衡量开发运维及应用开发业务价值的重要工具。该奖项旨在表彰最有效地帮助组织可视化价值流、跟踪指标和衡量改进的价值流管理工具。

*   **最佳可观察性解决方案** 该奖项旨在表彰了解单个组件、系统和整个应用性能的最佳可观察性解决方案。

*   **最佳 DevSecOps 解决方案** 该奖项旨在表彰最佳 DevSecOps 解决方案，该解决方案使安全性能够更早地包含在 SDLC 中并持续贯穿整个 SDLC。

*   最佳测试工具/服务
    测试是 DevOps 工具链中的支柱。这个奖项是对确保代码质量的最佳测试工具的认可。

*   **最佳 Kubernetes 平台/服务** 容器的采用为整个解决方案生态系统铺平了道路。Kubernetes 可能是最广泛认可和使用的容器编排工具。有许多平台和服务提供基于 Kubernetes 的解决方案。该奖项旨在表彰最佳 Kubernetes 平台/服务。

*   **最佳云原生安全解决方案/服务** 由于可用解决方案的深度和广度以及专为云原生环境构建应用和服务的优势，云原生运动席卷了 IT 界。该奖项旨在表彰最佳的云原生工具、解决方案和服务。

*   **大型机最佳开发运维解决方案**尽管年代久远，大型机仍是 IT 界的中流砥柱，有多种解决方案可用于在大型机和混合环境中实现开发运维实践。这个奖项是为了表彰他们中的佼佼者。

*   **数据运营/数据库解决方案的最佳开发运维**数据库经常被排除在开发运维对话之外，但随着数据运营概念越来越受欢迎，这种情况正在发生变化。该奖项旨在表彰数据库和数据运营领域的最佳解决方案。

*   **最佳新 DevOps 工具/服务提供商**该奖项旨在表彰过去一年中为 DevOps 领域带来创新的最佳新服务提供商或工具(必须成立不到 24 个月)。

### **DevOps 十几个社区奖项**

*   **最佳 DevOps 行业实施**该奖项旨在表彰成功采纳并实施 DevOps 原则和实践的组织。

*   **最佳 DevOps 调查/分析/研究**该奖项旨在表彰对 DevOps 社区产生重大积极影响的研究。

*   **最佳 DevOps 相关视频系列(视频)**此类别突出了最具娱乐性、教育性和励志性的 DevOps 视频系列——那些最具吸引力和最高质量的内容。

*   **最佳 DevOps 相关音频播客系列**今年你最喜欢的 DevOps 播客是哪个？这个类别庆祝了许多伟大的 DevOps 播客，这些播客推动了关于从 CI/CD 和测试到 DevSecOps、Kubernetes、容器等等的讨论。

*   **年度最佳 DevOps 书籍/电子书**每年都会有大量制作精良的 DevOps 相关书籍和电子书问世。这个类别突出了今年最受欢迎和最受欢迎的作品。

*   **顶级 DevOps 布道者**这个类别是我们的最爱之一！有这么多伟大的福音传播者传播德沃普斯福音，但我们想知道谁做得最好。被提名者可以在 DevOps 内部或外部工作。

*   **最佳 DevOps 转型(非供应商)**在持续数字化转型的一年里，我们想知道哪些组织在通过 DevOps 实现业务转型的努力中脱颖而出。

*   **年度最佳 DevOps 赛事**虚拟赛事成为整个疫情的常态。但随着越来越多的人渴望面对面的互动，面对面的活动慢慢卷土重来。哪个 DevOps 活动脱颖而出？

*   **最具创新性的 DevOps 开源项目**开源项目旨在促进创新，提高效率、速度和质量。虽然有许多伟大的开源项目，但最好的是那些创建一个社区的人，他们使用该技术并相互支持。

*   **DevOps 年度高管(非供应商)** 该奖项旨在表彰高管(或同等职位)级别的 DevOps 领导者的卓越表现。

*   **年度最佳 DevOps 演示**该类别从今年举行的众多(虚拟)会议中评选出最佳 DevOps 主题演示。

*   **年度最佳 DevOps.com 文章** 该类别旨在表彰首席执行官(或同等职位)级别的 DevOps 领导者的卓越表现。

有这么多才华横溢、鼓舞人心的领导者在为我们的社区赋权，选出一位获胜者绝非易事。虽然不是每个人都可以拿回家一个奖项，但被提名仍然是一个了不起的成就，并证明你走在正确的道路上。我们非常感谢您对 DevOps 社区的奉献和服务。
