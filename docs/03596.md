# SmartBear 为大众带来跨浏览器测试自动化

> 原文：<https://devops.com/smartbear-brings-cross-browser-test-automation-to-the-masses/>

SmartBear 宣布推出新的 CrossBrowserTesting Record & Replay 功能，通过利用基于云的设备实验室实现测试自动化，使所有技能水平的团队都能节省时间和资金

马萨诸塞州萨默维尔。–2018 年 4 月 11 日–团队软件质量工具的领导者 SmartBear 今天宣布了 CrossBrowserTesting 的新记录和重放功能，这是唯一一个一体化测试平台，允许团队在云中的数千种真实桌面和移动浏览器上运行自动化、可视化和手动测试。记录和重放功能使所有技能水平的团队能够快速轻松地从手动测试过渡到自动测试，通过利用基于云的设备实验室在更短的时间内测试更多的浏览器来节省时间和金钱。

鉴于目前使用的终端用户设备的巨大碎片化，软件团队需要确保他们的 web 应用程序能够在任何浏览器、操作系统或设备分辨率下完美运行，但是测试所有这些场景会给软件交付流程带来巨大的成本和时间延迟。此外，作为左移趋势的结果，今天每个人都被要求参与 QA 过程——但与大多数熟悉脚本和使用 Selenium 等框架的开发人员和 QA 工程师不同，一些测试人员、设计人员和产品经理缺乏创建自动化测试的技术技能。借助新的 CrossBrowserTesting Record & Replay 功能，所有团队成员都可以像专家一样通过记录和自动化他们的手动测试来自动化他们的 UI 测试，而不管他们的技术水平如何，并立即进入 CrossBrowserTesting 基于云的设备实验室，以确保他们的软件在任何最终用户设备上都能完美运行。

许多参与这项新功能测试的组织已经认识到了它的价值。“CrossBrowserTesting 中的记录和回放功能非常直观，我的团队可以在几秒钟内开始自动化，”MacMillan Cancer Support 的测试经理 Tom Newman 说。但是这一新功能的价值并不止于此。Newman 继续说道，“除了它的易用性之外，我特别欣赏它创建的测试视频，你可以在许多浏览器上重新运行测试，而不会中断测试。记录和重放特性对于从手工测试过渡到自动化测试非常有帮助。”Solidbrain 的 QA 主管 Mateusz Piasta 补充说，新功能“非常容易使用，令人印象深刻的是，它成功完成了 90%的脚本。”

CrossBrowserTesting 中的 Record & Replay 特性是对未来功能进行持续创新的第一步，对于那些不习惯复杂脚本的用户来说，这是向测试自动化过渡的第一步。正如 SmartBear 产品、测试和开发副总裁瑞恩·洛伊德所说，“在 CrossBrowserTesting 中实现记录和回放功能后，我们收到了客户的大量反馈，帮助许多人开始向自动化过渡。展望未来，我们将继续投资于能够帮助我们的客户有效扩展其自动化工作的功能。”

作为团队软件质量工具的领导者，SmartBear 为任何技术水平的用户提供了最多的选择来利用测试自动化的好处。SmartBear 提供了 CrossBrowserTesting 来针对基于云的设备实验室运行 web 测试，使用直观的 Record & Replay 或通过上传 Selenium 脚本，TestComplete 来测试桌面、web 和移动应用程序，SoapUI Pro 来测试 REST、SOAP 和其他流行的 API 和物联网协议。SmartBear 还提供 TestLeft 和 TestServer，使所有开发测试人员能够在他们的 IDE 中创建自动化测试。

所有交叉浏览器测试自动化计划中都包含新的记录和重放功能。您可以通过参加他们在 4 月 12 日举办的网络研讨会，了解更多关于 CrossBrowserTesting 和新的记录和重放功能的好处:[https://smartbear.com/<wbr>资源/网络研讨会/记录和- <wbr>重放/](https://smartbear.com/resources/webinars/record-and-replay/)

关于 CrossBrowserTesting
CrossBrowserTesting 是一个基于云的 web 测试平台，可以加速数千种不同桌面和移动浏览器的手动和自动测试。CrossBrowserTesting 专注于测试基础设施，通过让测试人员或开发人员直接在云中轻松访问，减少了与拥有和更新移动设备和具有最新浏览器配置的虚拟机相关的维护和成本。更多信息请访问:[https://crossbrowsertest。<wbr> com/](https://crossbrowsertesting.com/) 并在 Twitter 上关注@ xbrowsertesting。
