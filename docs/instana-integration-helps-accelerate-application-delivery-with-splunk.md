# Instana 集成有助于通过 Splunk 加速应用交付

> 原文：<https://devops.com/instana-integration-helps-accelerate-application-delivery-with-splunk/>

***Instana App 将问题解决信息直接传送到 Splunk IT 服务智能***

旧金山–2018 年 10 月 30 日–Instana 宣布与 Splunk IT 服务智能(ITSI)进行新的集成，以解决组织采用持续集成和交付(CI/CD)应用交付策略时可能出现的挑战。新的 Instana 应用程序为 Splunk 用户提供了对 Instana 全栈基础架构和应用程序监控信息的集成访问。Splunk 的 Instana 应用程序直接连接到 Splunk ITSI，提供 Instana 的性能和配置数据，以及服务问题的事件和自动分析。

Instana 联合创始人兼首席运营官 Pete Abrams 表示:“随着组织实施 CI/CD 战略，代码和基础设施的不断变化会导致其动态微服务应用中的性能可见性差距。“Instana 与 Splunk 的集成不仅限于性能数据，还包括应用交付事件、服务问题和问题分析，所有这些都成为 Splunk 集成分析的一部分。”

Instana 的自动应用程序监控解决方案可发现应用程序基础架构和服务组件，为应用程序的技术堆栈部署监控传感器，跟踪所有应用程序请求，无需任何人工配置，甚至无需重启应用程序。该解决方案实时检测应用环境中的变化，调整自己的模型，并在几秒钟内将变化和对用户的影响可视化。

新的 Instana 集成今天在 Splunkbase 上提供，为 Splunk 用户提供最完整的数据和分析，以实现动态应用程序的可观察性，允许 Instana 的数据与 Splunk 中的其他数据相关联。

当服务问题发生时，自动分析会接管，利用人工智能和机器学习来隔离任何问题，确定问题的可能原因，并向用户显示从哪里开始调查。服务事件、相关事件和可能的触发事件都是从 Instana 发送到 Splunk ITSI 的信息的一部分。

451 Research 的应用和基础设施性能高级分析师 Nancy Gohring 表示:“在当今的 IT 运营环境中，有两个相关的常数——新技术不断涌现，随着新技术的采用，应用的复杂性也不断增加。“典型的结果是 IT 运营陷入混乱，尤其是在绩效管理领域。然而，新的工具和流程可以最大化可见性，同时向整个应用交付组织提供即时反馈。结果是，即使在这些动态复杂的环境中，团队也能快速识别和解决问题。”

Instana 应用程序通过开箱即用的 APM 和终端用户监控 KPI 为 Splunk ITSI 客户提供了独特的价值，可帮助开发运维团队实现更快的 MTTR、减少停机时间并提高工作效率。独特的功能包括:

*   应用服务指标，包括 KPI
*   包括基础架构在内的整个应用体系的性能数据
*   灵活地将任何实例指标导入 Splunk，包括丰富的事件数据和问题解决建议

有了这些广泛的综合性能数据、事件数据、事故信息以及在哪里寻找和如何解决问题的建议，AI Ops 正在成为现实。

面向 Splunk 的 Instana 应用程序现已推出，请访问 www.instana.com 和 Splunkbase 了解更多信息。
