# Instana 宣布 APM 与 LogDNA 日志分析解决方案集成

> 原文:[https://devo PS . com/instana-announcements-APM-integration-with-log DNA-log-analysis-solution/](https://devops.com/instana-announces-apm-integration-with-logdna-log-analysis-solution/)

*从自动化 APM 解决方案到 LogDNA 分析工具的一键式导航*

伊利诺伊州芝加哥——2020 年 3 月 17 日——面向云原生微服务应用的实时应用性能管理(APM)解决方案的领先提供商 Instana 今天宣布扩展该公司与日志分析工具的一键式集成，现在支持 LogDNA。

在对微服务和云原生应用进行故障排除时，Instana / LogDNA 集成为开发和运营团队提供了无缝体验。当应用程序出现问题时，Instana 会自动隔离有问题的服务，以及引发问题的触发事件和有问题的应用程序代码、微服务或基础架构平台。

Instana 的技术总监兼 APM
策略师 Chris Farrell 表示:“虽然应用的复杂性不断增加，但单个基础设施平台和微服务也是它们自己的复杂生态系统。Instana 和 LogDNA 的集成解决方案提供了全方位的监控、跟踪和日志分析，使任何应用程序利益相关方能够在服务级别受到影响之前识别、隔离和修复应用程序问题

Instana 与 LogDNA 的集成是从自动化 APM 解决方案到日志分析工具的最新一键式导航，通过关键组件仪表板(如服务器、容器和 Kubernetes)上 Instana GUI 中的按钮进行处理。单击该按钮时，Instana 会启动日志分析工具的一个实例，以及一个预执行的分析过滤器，过滤需要检查的应用程序组件和相关基础架构的准确日志。时间是同步的，这意味着用户不必搜索解决问题所需的特定日志或时间范围。

与传统的 APM 工具不同，Instana 的自动化应用程序性能监控(APM)解决方案可以发现所有应用程序服务组件和应用程序基础架构，包括 Azure 等云基础架构、Kubernetes 和 Docker 等编排基础架构、应用程序服务和 DevOps 流程。Instana 自动为应用技术堆栈的每个部分部署监控传感器，并跟踪所有应用请求，无需任何人工配置，甚至无需重启应用。该解决方案实时检测应用程序和基础架构的变化，调整自己的模型，并在几秒钟内可视化这些变化及其对性能的影响。

与 LogDNA 和其他日志分析工具的集成现已在所有 Instana APM 解决方案中提供。更多信息可在[https://www.instana.com](https://www.instana.com/)找到。
