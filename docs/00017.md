科罗拉多州博尔德—(美国商业资讯)—面向 DevOps 和 IT 专业人士的领先自动化服务器管理工具 JumpCloud 今天公布了与 SoftLayer 联合举办的 DevOps 大会的调查结果。调查证实，对自动化程度提高的需求和完成任务的持续压力推动了 DevOps 的快速采用。当被要求描述他们最大的挑战和最耗时的任务时，主要的回答提到了部署和修补活动。用户管理、日志文件分析和取证也是 DevOps 会议与会者回答最多的问题。

“我们决定与 Foundry Group、TechStars 和 Bullet Time Ventures 公司一起举办这次 DevOps 会议，以便对 DevOps 运动的状态有一些基本的了解和认识”

不管具体的挑战或任务是什么，与会者都一致认为时间越来越少，责任越来越大，生产周期越来越快。调查结果特别指出手动任务是时间管理的障碍。为了解决这一问题，专注于开发运维的组织正在寻找自动化任务和更有效地管理在线基础架构爆炸式增长的方法。

JumpCloud 首席执行官 Rajat Bhargava 表示:“我们决定与 Foundry Group、TechStars 和 Bullet Time Ventures 公司举办这次 DevOps 会议，以便对 DevOps 运动的状况有一些基本的了解和认识。“会议旨在通过分享最佳实践和汇编来自 DevOps 运动中主要声音的高度有针对性的受访群体的调查数据来推进 DevOps 运动。来自 100 多家公司的 200 多位业界领袖能够出席并帮助我们获得并分享对这一日益重要的运动的一些见解，这让我们感到无比高兴。”

超过四分之三的受访者将自己归类为 DevOps 组织。另外 15%正在成为开发运维组织。

当被问及受访者使用了哪些云提供商时:

AWS–55%
Rackspace–14%
SoftLayer–6%
Digital Ocean–6%
Heroku–6%
misc .–13%
当被问及如何提高组织的自动化水平时:

超过 75%的人报告使用某种形式的配置自动化工具，如 Chef 或 Puppet。
大约三分之一的受访者表示使用了某种形式的补丁解决方案。
超过一半的人在使用预构建的映像。
当被问及哪些任务最需要手动管理流程时

1.部署和补丁并列第
2。用户管理
3。日志文件分析和取证
