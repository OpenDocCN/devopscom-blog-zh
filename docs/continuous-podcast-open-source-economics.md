# 持续播客:开源经济学

> 原文：<https://devops.com/continuous-podcast-open-source-economics/>

欢迎来到 To Be Continuous，这是一个关于持续交付和软件开发的节目，由 [Heavybit](https://twitter.com/heavybit) 成员、 [CircleCI](https://circleci.com/) 的创始人保罗·比格和[launch blackly](http://blog.heavybit.com/blog/to-be-continuous-16#)的首席执行官伊迪丝·哈博主持。

在这一集的《未完待续》中，伊迪丝和保罗将与肖恩·伯恩斯、 [Outlier.ai](http://outlier.ai/) 的首席执行官和[纳迪亚·艾格巴尔](https://twitter.com/nayafia)一同出席。该小组讨论了开源经济学，并研究了为开源等式增加资金的几种潜在演变。

Edith:Nadia，你最喜欢开源的哪一点？

Nadia: 我喜欢开源，在那里我感觉我和很多有创造力的人在一起。这有点像是创造性人才的最佳状态。它直接反馈到其他的创业公司和技术，以及我喜欢的一切。

但是似乎一切都是从开源开始的。这让它变得非常神奇。

Edith:Sean，你最喜欢开源的哪一点？

肖恩:是社区。我喜欢向别人学习。我喜欢能够找到我永远不会联系到的人，我有共同的兴趣。一个有帮助的社区，因为我认为开源不仅仅是写代码，而是分享想法，分享概念和分享问题的解决方案，虽然我可能不会总是使用那些相同的解决方案，但我总是会受到启发，找到解决其他问题的新方法。

保罗:这非常有趣，因为没有人说他们喜欢通过分成制来支持他们在硅谷的创业。或者…没有人说，我喜欢我的工作方式。

伊迪丝:嗯，保罗，那会在这一集的后面，但这将是我们的客人自我介绍的好时机。

Nadia: 我是 Nadia Eghbal。我最近一直在写作，思考了很多关于如何支持开源基础设施的问题。

最初，我有非营利组织和影响力投资以及与基金会合作的背景，但最近我是一家初创公司的创始人，然后我进入了风险投资领域。从各个不同的角度来看这个系统，并且喜欢将所有不同的兴趣和主题结合在一起，这在开源中似乎会发生。

肖恩:嗨，我是肖恩·伯恩斯。我大半辈子都在写软件。我也是公司的创始人和投资者。我不认为自己是开源方面的专家，我也不想自吹自擂，但我是一些开源库的维护者，这些库看起来很受欢迎，我也为其他人做了贡献，我认为我很幸运在我的生活中认识了很多开源开发者，虽然不是大多数，但至少足以让我感觉自己是社区的一部分。

保罗:我们在节目开始前讨论的话题之一是开源开发者如何获得报酬。或者，开源开发者是否应该获得报酬。

Nadia，听起来你已经谈论这件事有一段时间了，或者考虑这件事有一段时间了。你能把我们踢出去吗？

Nadia: 是的，实际上，Node.js 基金会的负责人[麦克尔·罗杰斯](https://twitter.com/mickrogers)对这个话题有了新的想法。他最近说，我想，在我写的最后一篇文章中，每个人都可以从开源中获得报酬。我认为这是一个很好的开始思考这个问题的方式，而不是说没有人会得到报酬。因为最终我们会为开源贡献时间，最终，有人会付钱给你让你有时间去做开源。

无论是你的雇主，然后你可以在你的工作岗位上全职工作。或者你是全职员工，这给了你足够的带宽，让你可以在周末或其他时间从事开源工作。

所以当我想到为开源工作付钱时，我想人们会对这个想法感到愤怒，因为他们认为开源根本不涉及金钱。我真正喜欢迈克尔观点的是，最终，金钱实际上与开源紧密相连，而我们只是不谈论它。

肖恩:我明白了。这很有趣，因为我的观点是它实际上更微妙，我 100%同意你，但我认为这是非常微妙的，许多人确实为开源而获得报酬，我们甚至没有渴望让他们获得报酬，对吧。因为有很多种开源软件。很有趣，因为在灵媒上，娜蒂娅和我有过交流。

我把它量化为我认为的开源的四个类别。这绝不是普遍标准，但至少在我看来，在有些地方，开源是你公司产品的一部分。它是整体的一部分。

就像，黑暗发射。伊迪丝的公司有开源 SDK。这并不是因为他们在做什么伟大的事情，如果他们的客户能够在他们将代码集成到他们的应用程序之前看到代码，这将减少他们的销售摩擦。所以这是一种开源。

第二个是我所说的企业开源，有一个开源项目，但有一个公司围绕它销售服务和产品。所以《合流》为卡夫卡做了这些。Databricks 为 Spark 做了这些。我认为，MySQL 是第一批真正用它做好模型的公司之一。你可能会说，Red Hat 的成功有好有坏，但这是另一种情况，公司的目标和销售都是这个项目。

我称之为第三种，有点像企业营销，就是你从一家公司拿出一个项目，然后像 LinkedIn 和 Kafka 一样推出，他们实际上是公司在维护它，他们只是让它可用。于是谷歌、Angular 和脸书的工作做出了反应。我认为这也是。你可以争辩为什么公司要这么做，有些是为了招聘，有些是为了营销。有些人这么做是因为他们是非常好的人。

我认为第四类是有点奇怪的地方，我称之为有人开始一个项目的社区。也许是个人，也许是几个人，去解决一个问题。今天许多最重要的开源产品都是这样开始的。甚至 Linux 也是这样开始的，对吧。它成长了，变得更大了，最终，变得不清楚了，因为其他人在用这个项目建立其他公司和其他产品，赚更多的钱。

我认为第四类是我认为比较少的地方，因为前三类通常很清楚谁拿工资做什么，钱从哪里来，但在第四类中它变得非常模糊，非常快。因为如果我开始做某件事，我喜欢用的比喻是，我写的开源库叫做 [Likely.js](https://www.npmjs.com/package/likely) 这是一个没有 js 的推荐系统，就像亚马逊产品推荐。我知道世界上有些人把它作为公司的一部分。很明显，这些公司赚钱是因为他们推荐产品，对吧。所以，问题是，你知道，在那个世界里，他们是否欠我什么，因为我创造了他们产品背后的东西？然后，反过来，我应该对我可能在其上构建的项目做些什么？那么世界上的经济流向哪里呢？

保罗:我对此有个有趣的看法。几年前，我参与了一个名为 PHC 的项目，这是一个 PHP 编译器。有几个人开始了这个项目，一年后我也加入了。当他们开始的时候，他们把它变成了 GPL。当我加入的时候，我认为，我们应该从 GPL 转变为 BSD，因为如果有人来了，并使用该技术来构建一些很酷的东西，我宁愿这种情况发生，而不是现在不发生，如果是 GPL，它可能不会发生。所以，本质上，在某种程度上与 Likely.js 有关，你由许可证决定。

无论你是否打算公开它，或者如果你不希望人们在它上面建立业务，你可以让它成为 GPL 或 AGPL 或类似的东西，你可以以任何你想要的方式限制它，但你选择不这样做。这其中隐含着一些东西，可以传达给那些在此基础上建立公司的人，等等。你的意图是什么。

肖恩:不，那是真的。虽然这里有一个细微的差别，那就是，即使 GPL 也遭受了这一点，Richard Solomon 今天仍然抱怨的是，你仍然可以在服务器上使用 Likely.js。

保罗:对，对，对。所以 AGPL 会解决这个问题。

**Sean:** 当然，我还可以选择其他许可证，但是现在大多数许可证都是这样，但是即使这样，我还是要提出反对意见，不是因为我同意它，只是因为，嘿，让我们提出反对意见，为什么不呢！你知道，作为一个个人开发者，我没有时间资源去追求人。我不会去起诉任何人。真的吗？不，拜托，就像不会发生一样。所以，我认为，你只有在 GitHub 上开源发布它才是正确的。我做了一个决定，人们可以用这种方式来使用它，或者，我不打算，他们可以在丝绸之路上用来推荐药物，据我所知，说真的，对。

或者，用我不会从中受益的方式赚钱。我做这个决定的时候非常清楚这件事的影响。实际上，我认为大多数开发人员都是这样做的。问题是当后来发现，等等，我真希望我没有做那个决定，我可以回到过去的时候，是否会有悔恨。

保罗:嗯，我认为这是很常见的事情，人们创造了一个东西，然后他们说，嗯，如果这很受欢迎，我可以保留它，或者我可以，这里有价值。我可以把它留给自己，或者把它的价值留给自己。或者试图以某种方式从中获利。但他们做出的本质上是营销决策。我要把它放出来，看看会发生什么。我要看看它会不会扩散，如果扩散了，我就可以决定，我怎么可能从中赚钱。或者我可以从中获得价值。但是在那一点上，很明显，就人们实际使用软件而言，秘密已经泄露了。

纳迪亚:是的，我的意思是，我认为这实际上应该从原则出发，就像你说的，当人们把一些东西送出去，然后人们可以用它们做他们想做的事情，这真的很棒。不管是为了一个爱好项目。不管是为了非法毒品还是什么，你知道的。商业，无论什么，就像，开源最酷的事情之一是你可以用它做任何你想做的事情，这就是你注册的原因。

如果你不想这么做，那就不要以这种方式授权。我不知道，我不会框定，至少我看到的问题和我感兴趣的问题，并不真的是关于卖家的悔恨，我猜测，“哦，我做了这个东西，现在它真的很受欢迎，我怎么能从中赚钱。”

但我认为更重要的是对我们都在使用的基础设施进行投资，这需要一些维护和保养。这就是为什么我喜欢“基础设施”这个术语，并且我试图在没有商业模式或业务与之相关的事物的意义上使用它。就像你一样，不能对 Python 收费。这是不可能的。或者与之相关的包装等等，但是仍然有数百万人在使用它。它感动了数十亿人。实际上，你有一个人在一个图书馆或项目上工作，这看起来不像是确保它正常工作符合每个人的最佳利益。

伊迪丝:我认为娜迪亚提出了一个很好的观点，那就是创造的乐趣在于创造。艰苦的工作是所有的维护、支持和文档。肖恩你基本上得到了乐趣。

肖恩:哦，完全正确。我的意思是，如果你回想一下几年前最初的“大教堂和集市”的文章，那是埃里克继承的基本主题之一，我想它叫做来自别人的流行邮件。

他已经厌倦了维护它，所以他希望有人接管它，他找到了埃里克，埃里克接管了它。我的意思是，你知道，我认为，是的，作为工程师，我们喜欢创造，我们喜欢在空闲时间创造东西，我们不喜欢维护它们，维护感觉更像工作。

因此，我同意 Nadia 的观点，如果每个人都从它身上获取经济价值，就必须有人愿意站出来支持它。我认为问题在于，我也很有趣，你有非营利组织的背景，因为我也曾为非营利组织工作过，我在非营利组织中看到类似的哀叹。

那就是——非盈利公司，非盈利组织，他们的创始人通常真的很想解决这个问题，他们通常很擅长这个。他们不擅长筹款

在这种情况下，非营利组织，就像筹集资金或获得捐赠。但是他们有一个好的使命，他们有一个好的组织去完成它。他们有这种哀叹，这就是为什么他们不能得到更多的支持，以他们想解决的方式来解决他们看到的问题。

我在开源项目的维护者身上也看到了同样的情况，我认为挑战将会是，我想知道普通经济是否有可能支持他们，做他们想做的事情，用他们想做的方式。或者他们是否要开始像将军那样做—

保罗:对，因为他们有其他选择。他们可以开一家咨询公司。被二类公司或类似的公司雇佣。以这种方式建立并支持他们自己，然后他们的工作没有，然后他们的工作不是他们想要的工作，那就是制作酷软件。他们的工作是，你知道，为一个没有灵魂的大企业做维修合同。

伊迪丝:你说就像是，我是说，没有灵魂的企业，我是说…

保罗:我非常喜欢没有灵魂的企业。但是我只是想把这个—

肖恩:他们有很多钱。我们必须卖掉它们。

如果任何没有灵魂的企业想要购买 CircleCI，他们应该联系我以获得企业定价。

肖恩:再多提 10%的代码 OSS。

顺便说一句，谢谢你今天穿上你的黑色 t 恤，保罗。

保罗:一直都是。Nadia 和 Sean 一周前在 Twitter 上有过这样的对话，这让他们上了这个节目。其中一条推文来自雅各宾派，他是 Python 的资深成员之一。他说的是，开源软件开发者应该得到报酬。SV，硅谷，通过剥削免费劳动力获得了巨额利润。这需要结束。

我认为在开源中有这种感觉，至少有很多人最终站在维护的一边，这里有一种分成制。这是需要发生的事情。

伊迪丝:我认为娜迪亚在文章中把这一点写得非常好；喜欢酷，我希望我写了那篇文章，它是辉煌的。

娜蒂娅: Aww。

Edith: 但是，就像 GitHub 的爆炸一样，现在开始一个开源项目真的很容易。

Nadia: 是的，我觉得现在只是一个我们习惯于以某种方式思考开源的时代，事情已经发展了很长时间，以至于我们没有注意到或者后退一步，过去的五年看起来与过去的 30 年完全不同。一切都变了。在某种程度上，我认为这应该被视为一件真正的好事，因为这意味着开源在某种程度上赢了。

公司希望东西是开源的。他们想学习如何开源他们自己的项目。他们想了解整件事。每个人都默认使用开源软件，这是一件非常积极的事情，而问题更多的是，哦，我们在剥削人，这真的很糟糕，但我们如何才能帮助支持底层，让一切变得如此伟大。它显然需要某种形式的再投资。

伊迪丝:是的，我是说，写于 30 年前的《大教堂和集市》最吸引人的一点是，他们把莱纳斯·托沃兹称为这个迷人的家伙。是的，我知道，你会说哇，这是一个迷人的，风度翩翩的家伙，他喜欢通过纯粹的意志力说服人们参与他的项目。

肖恩:是的。

伊迪丝:我的意思是，回到肖恩关于非营利组织的例子，现在工作室里的每个人都像是，哇，那不是…

娜迪亚:一点也不。

伊迪丝:现在任何人都这么认为，我不知道这是因为不得不继续维持一个项目还是…

**Sean:** 首先，我要感谢，我不知道他是否真的叫 Jacobian，不管他叫什么，感谢他对 Python 的服务和工作，因为我认为，不管我们在这里达成了什么共识，我们都同意这样做的人是在为整个社区服务。

娜迪亚:肯定的。
