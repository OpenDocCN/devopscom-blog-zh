# 掌声报告暴露了功能测试问题

> 原文:[https://devo PS . com/plateau-report-surfaces-functional-testing-issues/](https://devops.com/applause-report-surfaces-functional-testing-issues/)

一项由 plateau 收集的超过 340，000 个 bug 的分析发现功能性 bug 占问题的 68 %, plateau 是一个旨在与一系列 DevOps 平台集成的测试平台的提供商。该研究收集了 13，000 台移动设备和 1，000 台运行 500 个版本操作系统的独特台式机的数据，发现大多数问题都可以追溯到功能缺陷，而视觉(17%)、内容(9%)、崩溃(4%)以及滞后和延迟(2%)问题加起来仅占 32%，报告发现。

该报告还发现，屏幕阅读器占所有可访问性错误的 66%，相比之下，键盘导航问题和色彩对比度不足仅占 12%。报告发现，在本地化方面，质量差和缺失的翻译占错误的三分之二以上(67%)。根据 plateau 不断从客户那里收集的反馈数据，将近一半的组织(47%)认为货币和数字格式是本地化过程中最有价值的错误。

总体而言，组织将崩溃(75%)、功能性错误(61%)以及滞后和延迟问题(53%)的发现列为非常有价值的发现。

掌声的首席增长官卢克·达米恩(Luke Damien)表示，就测试而言，总的来说，对用户和客户旅程的关注仍然不够。他指出，随着越来越多的组织投资于主要由移动应用程序驱动的数字业务转型计划，缺乏重点正成为一个更大的问题。他补充说，支付尤其成问题，因为它们通常依赖于第三方提供的应用编程接口(API)，导致次优的应用体验，对收入产生直接影响。

挑战在于，还不完全清楚[应用测试](https://devops.com/?s=application+testing)的责任转移了多远。在某些情况下，开发人员正在承担责任。在其他情况下，专门的测试团队仍然负责。当然，开发人员会在构建应用程序时对其进行测试，但是本地机器上的应用程序体验可能无法在生产环境中复制。问题是，当移动应用程序未能达到预期时，今天的大多数最终用户并不特别宽容。仅仅因为一个功能没有在生产环境中测试，几个月的开发工作就可能被浪费掉。

不太清楚的是未来测试自动化的程度。机器学习算法使得功能和用户界面的自动化测试成为可能。不太可能很快消除人类参与测试的需求，但在未来的几个月或几年中，许多限制应用程序测试速度的例行繁琐任务应该会大幅减少。

与此同时，一个组织的整个品牌声誉现在与它在移动设备上实现的应用程序体验的质量联系在一起。如果移动设备的用户仅仅因为某个功能被破坏而选择了一个应用程序而不是另一个，那么收入目标现在很容易被错过——即使是很短的时间。因此，鉴于组织对软件的依赖，测试从未如此重要。
