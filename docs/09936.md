# SRE 调查揭示了主要的技术和文化挑战

> 原文:[https://devo PS . com/sre-survey-reveals-major-technology-and-cultural-challenges/](https://devops.com/sre-survey-reveals-major-technical-and-cultural-challenges/)

Catchpoint 与 Blameless 合作，今天公布了一项对 559 名现场可靠性工程师(SREs)的年度调查[发现 59%的受访者不认为工具蔓延是一个主要问题。另有 40%的人认为工具泛滥是一个中度(33%)到严重(8%)的问题。](https://resources.catchpoint.com/hubfs/eBooks/SRE%20Report%202023%20Catchpoint.pdf)

超过一半的受访者表示，他们自己开发了多达 30%的工具，超过一半(54%)的人指出，他们的组织跟踪三种或更多种不同类型的遥测数据，以推动[可观测性](https://devops.com/?s=observability)。收集的前三种遥测数据是基础设施(62%)、应用(58%)和网络监控(55%)，平均而言，调查显示 SREs 的中值 [toil](https://devops.com/?s=toil) 水平同比下降了 5%。

不到一半的受访者(46%)还指出，他们认为人工智能对 IT 运营没有价值或价值很低，而 30%的受访者认为价值中等或较高。另有 25%的人仍不确定。

最后，调查得出结论，根据谷歌定义的 DevOps 研究和评估(DORA)指标，采用“公正文化”的组织(在这种文化中，组织确保事件报告不会遭到惩罚措施)成为精英执行者的可能性增加了 500%。

Catchpoint 产品营销总监 Leo Vasilou 表示，调查结果为 SREs 面临的挑战提供了一些见解，因为他们在努力实现 DevOps 目标和经济衰退期间招聘变得更加受限的事实之间取得了平衡。

然而，他指出，这种压力与其说与管理工具链有关，不如说与确保驱动关键业务流程的应用程序始终可用有关。

近年来，SREs 已经成为一种独特的 DevOps 功能，几乎完全专注于以编程方式管理各种应用程序及其所依赖的底层基础架构之间的依赖关系。在云原生时代优化应用变得更具挑战性，因为构成现代分布式计算环境的微服务之间存在的依赖关系数量继续呈指数级增长。在许多情况下，应用程序不断添加到 IT 环境中，而 SRE 团队的整体规模却没有相应增加。

不管经济环境如何，考虑到对具备这些技能的 IT 专业人员的需求，招聘和留住高级研究人员仍然是一个挑战。当然，没有被广泛认可的正式的 SRE 证书，所以谁有资格成为 SRE 往往因组织而异。人们普遍认同的一点是，SRE 可以自动执行大量原本需要多名 IT 管理员才能完成的任务。Glassdoor 还报告称，SRE 的平均收入超过 125，000 美元，这是所有 IT 组织中最高的薪资水平之一。

根据所在的地理区域，每个组织都需要确定如果雇佣 SRE，他们可以替换多少 IT 管理员。这并不意味着依赖各种图形工具来管理 IT 任务的 IT 管理员会很快离开，但随着 DevOps 的不断发展，SRE 和 IT 管理员之间的互动也将继续发展。
