# 解读可观察性市场

> 原文：<https://devops.com/deciphering-the-observability-market/>

“可观察性”已经成为 IT 和网络安全领域使用最频繁的术语之一。如今，该术语被供应商用来指代从应用程序性能到网络监控、网络安全以及数据和分析的一切。

虽然这个词的普遍存在给从最终用户到记者的每个人都带来了困惑，但在过去两年里，该领域的初创公司也吸引了超过 20 亿美元的风险投资。这种势头促使 TechCrunch 询问这一领域，特别是数据可观察性，是否有效地[](https://techcrunch.com/2022/05/31/is-data-observability-recession-proof/)。

这一观察引发了两个问题。第一，什么是可观性？第二，有哪些不同的种类或变体？

## 什么是可观测性？

[可观察性](https://devops.com/?s=observability)作为一种广泛的实践或能力，起源于 20 世纪 60 年代，是工业控制理论的一部分。这个想法是，通过观察一个系统的输出，你可以弄清楚黑匣子里面发生了什么。这听起来很像监控，这是 IT 和安全团队的一种流行做法。但是，可观察性采用的方法不同于基于警报的监控方法；它允许您询问有关系统的问题，这些问题比您的监控系统的预定义阈值更深入。

可观测性要求从系统、网络和应用程序中收集大量数据，以支持其发现过程。随着系统和应用程序变得越来越复杂，找出哪里出错了以及为什么出错变得越来越困难。从高层次来看，这是该领域许多初创公司正在应对的挑战。每一种都采用不同的方法，解决可观测性问题的不同部分。

## 可观察性的类型

最常提到的类型是数据可观察性，它与通过数据管道传输的数据的健康和质量有关，用于分析用例，如馈送数据仓库。Acceldata、Bigeye 或 Monte Carlo Data 等供应商通常将他们的产品面向负责构建和运营分析数据管道的数据工程师。

另一组公司处理应用程序。这些公司，如 Honeycomb 和 Observe.ai，从应用程序中收集数据，以帮助站点可靠性工程师(SREs)了解性能问题，并帮助他们进行调试和故障排除。

也有专门研究机器学习可观测性的公司，这些公司关注的是生产中模型的性能和漂移。这些公司，如 Arize 和 WhyLabs，目标是数据科学家。

最后，在由日志、事件、度量和跟踪组成的可观察性数据空间中有公司，这对于所有其他形式的可观察性和监控工作都是必不可少的。这一领域的公司，如 Cribl，使用专门设计的管道连接可观测性和安全性数据的来源和目的地。这些管道允许公司将数据路由到多个目的地，在传输过程中丰富数据，并在摄取之前减少数据量。

如您所见，该领域有数十家公司。在当今纷繁复杂的 IT 和安全环境中，每一类供应商都面临着独特的挑战。

## 当心可观察性——清洗

随着围绕这个话题的炒作越来越多，希望从竞争对手中脱颖而出的公司可能会突然更名为可观察公司。这已经发生了。传统监控和警报公司称自己为可观察性公司，希望吸引与该领域中实际上拥有差异化产品和定位的其他公司相同的关注。
