# Instana 宣布为 Aws Lambda 提供首个人工智能监控解决方案；可在 Aws 市场上获得

> 原文:[https://devo PS . com/instana-announces-first-ai-powered-monitoring-solution-for-AWS-lambda-available-on-AWS-market place/](https://devops.com/instana-announces-first-ai-powered-monitoring-solution-for-aws-lambda-available-on-aws-marketplace/)

*公司的自动化监控扩展到 S3、SQS、RDS、迪纳摩和 Kinesis*

2018 年 6 月 6 日–柏林在 AWS 峰会上–监控动态容器化微服务应用的 APM 解决方案领导者 Instana 今天宣布将性能监控支持扩展到广泛的亚马逊网络服务(AWS)产品，包括亚马逊首屈一指的无服务器计算平台 AWS Lambda。除了发布用于 AWS 原生平台的新监控传感器，Instana 还宣布，他们的人工智能驱动的 APM 解决方案现在可以通过 AWS 市场获得。

“越来越多的业务应用在混合环境中运行，整合了传统的整体应用，包装了新的微服务堆栈，并使用原生 AWS 服务。Instana 首席执行官 Mirko Novakovic 表示:“现在，客户开始将 AWS Lambda 函数用于特定的微服务也很常见。这种复杂程度要求能够使用单个解决方案来监控整个环境，从而为开发人员提供所需的观察能力，以便发现和解决问题，无论问题位于何处

Instana 的新 Amazon 功能包括连续自动发现特定的 AWS 组件，包括 Lambda，收集每个平台的配置、性能和健康信息，以及整个环境中的应用程序性能，包括 AWS Lambda。目前正在监控的其他 Instana AWS 监控传感器包括:

*   S3——亚马逊的简单云存储服务，一个面向网络应用的网络级存储解决方案
*   SQS——AWS 消息中间件平台
*   RDS——AWS 本机关系数据库
*   迪纳摩-自动气象站 NoSQL 数据库
*   kine sis——基于 AWS 的应用程序的流优化平台。

Instana 的人工智能驱动的 APM 解决方案自动化了关键业务应用的管理，即使是在云、容器、微服务或无服务器环境中运行的应用。这种自动化通过提高服务和应用交付的速度和效率，优化了 IT 组织内的应用和服务交付。该公司的解决方案利用自动化和人工智能来自动监控、可视化和了解动态容器化微服务应用的应用性能。持续发现和映射提供了实时基础架构和应用服务可视化，即使在不断变化的情况下也是如此。

通过添加 AWS Lambda 支持，Instana 扩展了其微服务和基于容器的应用程序监控，以包括在亚马逊功能即服务(FaaS)平台中运行的代码。作为 Instana 下一代监控解决方案的一部分，支持的其他平台扩展了多云应用监控的功能，既包括传统组件，如 SQS 消息和 RDS 数据库实例，也包括非传统组件，如 Kinesis(视频流)和 AWS Lambda (FaaS)。这种灵活性使应用程序开发人员能够自主选择所需的确切平台，因为他们知道 DevOps 能够有效地监控任何选定的平台。
