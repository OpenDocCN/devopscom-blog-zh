# 持续交付:5G 成功的秘密

> 原文：<https://devops.com/continuous-delivery-the-secret-to-5gs-success/>

新技术正在改变社会运作的方式，从而改变商业运作的方式。在网络行业，这些新兴技术加上过去一年的长期影响——如[远程工作](https://devops.com/?s=remote+work)——正在改变客户和通信服务提供商(CSP)的互动方式。为此，这些行为上的转变带来了对如何交付通信服务的新需求。

近年来，网络技术取得了一些显著的进步。例如，[片上系统](https://en.wikipedia.org/wiki/System_on_a_chip)的进步将物联网设备提升到了一个新的水平。此外，边缘计算、高性能计算以及软件、分析和人工智能领域的革命极大地提升了实时本地数据处理的价值主张。但也许是 5G 带来了一些最令人震惊的增强。5G 连接实现了安全可靠的商业和工业运营，所有这些都可以零延迟、实时地发生，从而推动了 OT/IT 融合等的加速。

随着每一代新的无线网络的出现，通信服务提供商必然会提出一个迫切的问题:有助于带来更多客户和收入的“杀手级应用”是什么？过去，每一代人都在移动设备使用的演变中找到了答案。然而，正如我们所知，5G 改变了游戏。通过 5G，通信服务提供商现在可以利用以前无法进入的企业市场中的各种新机会。正因为如此，没有一款杀手级应用适合所有垂直行业。例如，交通运输行业的 5G 杀手级应用程序将与医疗保健行业的应用程序截然不同。

那么，5G 的杀手级应用是什么？首先，我们应该后退一步，认识到对于 5G 来说，杀手级应用将是网络本身。这是因为该网络能够快速创建和推出定制的 5G 服务，以满足不同垂直市场的独特客户需求和使用案例。在 5G 时代，在优化成本的同时以更快的上市时间带来新服务的能力是产生真正商业价值的关键。

实现 5G 服务所需的灵活性和敏捷性首先需要一个既定的基础。构建这一基础的关键步骤是建立云原生网络核心，并采用自动化驱动的持续集成和持续部署(CI/CD)功能。这样，通信服务提供商就可以开始为他们的网络增加真正的价值。

## 为什么 5G 云原生核心是关键任务

云自动化不仅仅是一个时髦词，而且变得越来越重要。对于 4G，云原生内核可能是首选，但对于 5G 服务——支持一系列应用，并将不断需要更新的功能——云原生设计是至关重要的。通过 5G 云原生核心，可能性是无限的，因为它支持基于服务的架构、改进的会话管理、端到端网络切片和接入无关连接，以实现无缝移动性等。

具有云原生核心的 5G 网络也将继续提供新的通信功能和服务，其中自动化将是一个关键的推动因素。这就是 DevOps 的用武之地。DevOps 可以更大程度地提高自动化程度，帮助 CSP 实现交付、安装和测试的自动化。因此，随着开发运维自动化变得更加以云为中心，电信运营商应该将持续交付作为开发运维的基础。如今，许多公共和私有云计算提供商在其平台上支持云 DevOps 系统，如持续集成和持续开发工具。当您考虑到连接服务的高度复杂性时——多供应商和多领域软件供应链以及多种生产环境——增加一个侧重于交付操作的流程级别至关重要。

## 拥抱持续交付，驾驭 5G 的力量

对于云世界来说，连续交付可能是一个熟悉的概念，但对于电信行业来说，这是一个相当新的概念。对于电信运营商而言，在电信网络中实现持续交付需要具备满足独特环境需求及其相关挑战的能力。CSP 可以做到这一点的一个方法是实施交付运营(DelOps)解决方案:一个重新构想的 DevOps。DelOps 非常适合电信环境，因为它能够利用多对多的交付和运营流程，包括设计、部署、测试和运营阶段。

在电信行业，每个网络和运营环境都是独特的；因此，解决方案不是“一刀切”对于电信运营商来说，实施最佳的 DelOps CI/CD 解决方案需要他们与运营商建立正确的合作关系。这样，他们可以建立一个灵活的 CI/CD 体系结构，补充他们的运营环境，并可以与任何云体系结构集成。

借助 DelOps CI/CD 解决方案，交付和运营的无缝自动化将成为现实。它是 5G 成功的“秘方”，因为它:

*   利用敏捷流程缩短推广周期和产品/服务上市时间
*   确保通信服务提供商的网络是最新的，并高效快速地推出新服务
*   进行验收测试、跟踪和管理发布特性所需的工作更少
*   快速解决问题，确保无缝的客户体验和满意度

随着 5G 时代的不断发展和演进，电信运营商应该尽早采用 CI/CD 服务，以实现平稳的运营之旅。通过这样做，通信服务提供商可以将其网络发展到最佳性能水平，最终提高其竞争力，创造新的收入并取得长期成功。
