# Harness 开放了持续交付服务，提供免费的源代码许可，以无缝采用世界级 CI/CD 平台

> 原文:[https://devo PS . com/harness-opens-continuous-delivery-service-with-free-source-available-license-enable-seamless-adoption-of-world-class-ci-CD-platform/](https://devops.com/harness-opens-continuous-delivery-service-with-free-source-available-license-to-enable-seamless-adoption-of-world-class-ci-cd-platform/)

*DevOps 团队现在可以将 Harness CD 与免费的开源社区版一起使用，而企业用户则可以从开放的公共资源库中获得 100%的 Harness CI/CD 代码*

加利福尼亚州旧金山，2021 年 1 月 19 日—寻求世界级持续集成/持续交付(CI/CD)平台的 DevOps 团队现在在 [Harness](https://harness.io/) CI/CD 平台中有一个开源和源代码可用的选项。该公司今天宣布，它将开放其 DevOps 平台的 CD 组件，该组件现在是免费的，可以在源代码可用的许可证下访问，以补充其 CI 平台，后者已经在开源许可证下可用。

100%的 Harness CI/CD 源代码现在可以在[公共存储库](https://github.com/harness)中免费获得。用户现在可以快速、轻松地开始利用 CI/CD 来构建敏捷软件交付解决方案，而无需支付许可或使用费。扩展到额外的企业级功能和 SaaS 计划是一条简单的升级途径。

Harness 的创始人兼首席执行官 Jyoti Bansal 表示:“我们向社区开放了我们的源代码和产品路线图，以简化团队的采用，他们需要一个全球采用的类别领先的 CD 平台，一个具有人工智能的智能平台，可以自动进行部署和回滚，以便他们可以找回他们的夜晚和周末。

在今天的公告中，Harness 在免费和可获得源代码的 Polyform Shield 许可下，以社区版的形式开放其 CD 服务。这种以用户为中心的许可选择使 Harness 用户能够自由和开放地访问代码，包括使用、修改和分发代码的权利，但不能使用它来推出竞争产品。

Harness CD Community Edition 加入了 Harness CI Community Edition(以前称为 Drone Open Source)，这是一个非常受欢迎的开源、云原生 CI 解决方案，在 Apache 2.0 许可下提供。

此举正值 Harness 的快速增长期。该公司现在拥有 500 多名员工，并继续增加其 CI/CD 平台的企业用户，以及其产品组合中的个人服务。

Redmonk 的首席分析师 Steve O'Grady 说:“目前，市场倾向于以二元方式看待软件许可，要么是开源的，要么是专有的，这给源代码可用的许可带来了一些耻辱。“这是不幸的，也是短视的，因为源代码许可是两者的合法替代品。这些许可证的问题不在于它们不是开源的，而在于它们被不太关注行业标准定义的各方周期性地描述为开源的。虽然在本质上不同于开放源码，并且没有授予相同的许可范围，但是源代码可用的许可证仍然可以为用户和企业提供切实的好处。因此，Harness 明确应用于其 CD 和 CI 组件的不同许可的决定是受欢迎的。"

**关于线束**

Harness 是完整软件交付的唯一端到端平台。它为工程和开发团队提供了一种简单、安全和可靠的方法来将应用程序发布到生产环境中。Harness 使用机器学习来检测部署的质量，并自动回滚失败的部署，从而节省时间并减少对定制脚本和手动监督的需求，让工程师们重新享受周末时光。Harness 的总部设在旧金山。
