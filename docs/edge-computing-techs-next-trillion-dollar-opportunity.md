# 边缘计算:技术的下一个万亿美元机会

> 原文：<https://devops.com/edge-computing-techs-next-trillion-dollar-opportunity/>

即使技术继续向云转移，随着计算越来越多地向边缘转移，同时也在向相反的方向转移。一个关键原因是:根据 IDC 的数据，到 2025 年，使用中的联网设备数量预计将超过 560 亿台。边缘计算不是云的继任者，而是技术范围的另一种扩展，代表着投资和增长的绝佳机会。

随着边缘设备和计算的采用持续飙升，组织需要在本地处理来自这些设备的激增数据，以便能够实时处理这些数据。这将需要对适应边缘的基础设施以及相关工具和平台进行投资。企业技术即将再次发生转变，在数据被创建和消费的同时处理和分析一些数据的需求正成为一项强制性的业务需求，就像将技术堆栈的其他部分移动到云中的中心位置一样。

组织已经必须使用基于必须持续刷新的数据的边缘设备提供令人信服的客户体验。另一种方法——向云服务或本地数据中心上传和下载数据——通常需要很长时间。无论数据是在消费者的智能手机上还是在生产车间，组织都需要处理和分析数据，尽可能接近需要实时数据驱动响应的程度。

提供这些功能的竞争已经非常激烈。从云服务提供商到电信运营商，再到服务器和存储基础设施的长期提供商，许多供应商都将边缘计算视为他们的下一个万亿美元机会。

## 为什么我们在边缘计算

边缘计算已经以各种形式存在了几十年。有无数的独立软件被部署在例如工厂车间使用的设备上的例子，这些设备使制造过程的各个方面自动化。近年来，这些系统中有许多已经连接到互联网，使各组织能够更容易地收集和分享数据。现在面临的挑战是，在边缘生成的数据量通常会超过通过广域网(WAN)传输大量原始数据的实际可行范围。需要能够在网络边缘处理和分析大量数据的平台。然后，在边缘平台上处理的数据的汇总结果可以更有效地与云中、内部 IT 环境中甚至网络本身上运行的其他应用程序共享。

低延迟要求是当今最大的挑战。在许多情况下，在电信网络边缘的网关上运行的应用程序需要能够在几分之一秒内响应移动计算应用程序的数据请求。在其他情况下，边缘应用程序正在推动自动化工厂运营，需要动态调整以适应在扩展企业边缘处理的分析。

还有更高级的用例，下面将介绍其中一些。其中包括为无人驾驶汽车实时处理计算机视觉数据的车载计算机，以及有望改变制造业的机器人应用。

## 硬件进步支持边缘计算

低延迟网络连接、高性能计算和大容量存储需求的竞争需求正在推动边缘硬件的创新。例如，电动汽车制造商特斯拉为其处理器添加了定制设备，能够预处理大量数据，执行机器学习推理，以做出快速驾驶决策和预测，甚至在车辆上进行一些深度学习模型训练——然后将数据子集发送到中央系统进行更多训练。

今年早些时候，特斯拉推出了一款拥有 60 亿个晶体管的处理器。据该公司称，与其在特斯拉 Model S、Model 3 和 Model X 模型中取代的 Nvidia GPU 相比，其设备的性能提高了 21 倍。除了 AMD 和英特尔等传统提供商的 CPU 之外，这些功能强大的设备还用于信息娱乐等繁重应用。利用这种边缘计算能力，特斯拉汽车可以处理传感器数据，使其能够识别和自动驾驶行人、路上的其他汽车、紧急路边标志和潜在的危险移动物体。该传感器数据也通过预加载的地图数据和 GPS 连接进行处理。此外，特斯拉的车载计算机通过特斯拉的网络将信息传递到大规模存储设施，在那里对数据进行分析，以改善特斯拉汽车的自动驾驶和其他功能。车辆软件栈的改进然后通过网络下载到它。

医疗保健是一个对先进的机器学习设备和软件有类似需求的行业。部署在许多地方的具有推理计算能力的 MRI 和其他扫描设备的图像也需要类似于特斯拉汽车所需的计算能力和网络基础设施。然而，大多数医院没有兴趣或必要的资金来构建自己强大的边缘计算硬件，需要从技术供应商那里获得这种能力。

随着半导体电路的发展接近 7 纳米和更小，以高能效方式在边缘运行应用的能力对于更广泛的组织来说变得更加现实。随着处理器的每一次更新换代，edge 平台的总成本将继续下降，而运行应用程序的能力将稳步提升。

## 边缘网络

无论有多少数据在边缘被处理和分析，庞大的数据量都会增加对网络带宽的需求，从而导致潜在的瓶颈。一个主要瓶颈是回程，即边缘塔与中央服务器和网络之间的距离。未来的边缘网络基础设施是否将由 5G、更强大的 WiFi 连接和设备、低延迟菊花链系统、谷歌正在开发的光纤系统、卫星连接、尚待开发的技术或这些技术的组合组成仍不清楚——但有一点是肯定的:数据量的增长速度快于网络带宽。

如今，组织可以在多种可用的有线和无线网络方案中选择连接边缘平台。涉及边缘计算的用例数量正在扩大，部分原因是无线 5G 网络使跨多个互联边缘平台共享数据成为可能。例如，佩戴增强现实耳机的制造业员工不仅可以与他们在现场的同事，还可以与数百英里外的同事近乎实时地共享数据和分析。

随着电信运营商继续虚拟化他们用来提供 5G 服务的网络基础设施，在越来越多的地理区域提供 5G 服务的成本应该会稳步下降。今天，4G 网络每平方英里可以支持大约 4000 台设备。一个 5G 网络可以在同一区域打包多达 300 万台设备。

就 5G 而言，现在仍处于早期阶段。凯捷对 1000 家计划将 5G 纳入其运营的工业组织进行了调查，发现不到三分之一(30%)的组织已经进入试验和实际实施阶段。然而，运营商已经在定义下一代 6G 无线标准，这些标准有望在下一个十年为无线网络带来又一次巨大的飞跃。

无论采用何种网络，组织仍需要在需要处理、存储并最终在边缘处置的数据量与需要通过 WAN 传输并整合到集中处理数据或用作记录系统的各种其他平台中的聚合数据量之间取得平衡。

## 考虑到安全因素

可以说，边缘计算的最大障碍是安全性。每当一个组织部署一个边缘平台，他们需要防御的攻击面就会增加。许多边缘平台也由运营技术(OT)团队管理，这些团队不像谷歌的数据中心安全团队那样拥有丰富的网络安全专业知识。大多数组织都试图将他们的 OT 和传统的信息技术(IT)团队融合在一起，努力部署、管理和保护边缘平台。然而，这两个团队的文化有很大的不同。组织需要时间来协调这两个群体的工作。

更具挑战性的是，网络安全专家很难找到和留住。网络犯罪分子已经发现了 OT 环境的脆弱性。隶属于不同民族国家的攻击者已经证明，如果战争爆发，控制电网和其他关键基础设施系统的工业控制系统是主要目标。从他们的角度来看，每一个连接到网络的边缘计算平台都是另一个将恶意软件引入 IT 环境的工具。恶意软件在整个平台上横向传播只需要几分钟。

最终，组织将很快花费数十亿美元来保护边缘平台。一旦边缘系统被锁定和安全，边缘处理可以改善整体网络防御-在边缘处理更多数据而不移动数据，可以大大减少攻击面。

## 边缘计算生态系统如何发展

一旦适当级别的基础设施到位，下一个挑战就是构建应用程序。容器技术的广泛采用允许开发人员使用需要更少内存和存储的工件来构建应用程序。今天的大多数容器环境，如基于 Docker 和 Kubernetes 的环境，在某种意义上是去中心化的，但在数据中心这样的中心化环境中运行。但是，更小的容器格式的工作正在取得进展，这使得在远离任何服务器或网关数百英里的石油钻塔上部署应用软件变得更加容易。

开源 Kubernetes 容器编排引擎的轻量级实例开始获得关注。 [Kubernetes](https://containerjournal.com/?s=Kubernetes) 通过呈现一组一致的应用程序编程接口(API)而不管它部署在什么平台上，提供了在高度[分布式计算](https://devops.com/?s=distributed+computing)环境中集中部署和管理基于现代微服务的应用程序的机会。

机器学习、数据科学和其他支持边缘计算的技术的行业 API 规范和框架也开始出现。一个例子是 oneAPI，由英特尔和其他公司支持，它为开发边缘应用程序提供了一个框架。

开发平台的提供商也在竞相创建框架，以简化在边缘架构内构建应用的过程，无缝调用运行在云中的后端应用服务。像 Apache Airflow 和 Kafka 这样的分布式事件流平台可以在允许边缘计算平台与那些后端平台大规模共享数据方面发挥作用。一些组织还将调用内容交付网络(cdn)的服务来提高部署在已经构建的全球存在点(pop)网络上的应用程序的性能。

最后，部署在边缘的有状态应用程序可能需要本地数据库来提供对持久数据形式的访问。今天部署的大部分容器应用程序都是无状态的——它们将数据存储在外部存储系统上。在本地 Kubernetes 集群上构建访问容器中数据的有状态应用程序更具挑战性。

已经有数百万熟悉容器平台的开发人员在做准备，直到改进的硬件和软件尽可能地简化和保护高度可移植的应用程序的部署。

## 即将到来的人工智能和数据管理挑战

未来最令人兴奋的边缘创新可能将由机器学习和人工智能的兴起推动。今天，人工智能模型通常在位于中央的云计算平台中训练。然后可以在生产环境中创建和部署人工智能和机器学习推理引擎，在生产环境中，一个经过训练的人工智能引擎可以在发现新数据时推断出要采取的行动。

随着时间的推移，这些推理引擎会随着收集的新数据量超过原始人工智能模型所基于的参数而发生漂移。数据科学团队与 IT 和 OT 团队合作，然后需要训练一个新的人工智能模型来取代边缘的推理引擎。新人工智能和机器学习能力的进步，加上更低功耗和更高性能的处理器和其他组件来帮助降低成本，可以提高预测系统的准确性和性能，但通常不会降低这种漂移发生的程度。

如果人工智能模型可以至少部分地在边缘重新训练，这可以帮助减少预测精度漂移。然而，在 edge 平台中有效地重新训练人工智能模型所需的软件和硬件基础设施仍然处于非常早期的阶段。也就是说，随着越来越高性能的硬件成本下降，以及边缘机器学习软件系统的改进，在边缘进行训练的能力将变得更加可行。
