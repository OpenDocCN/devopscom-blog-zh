# 云计算:WASM 将引领下一个 IT 时代

> 原文：<https://devops.com/cloudnativeday-wasm-to-drive-next-it-epoch/>

WebAssembly (WASM)将引领下一个真正可以在任何地方运行的云原生应用开发时代。

Cosmonic 的首席执行官 Liam Randall 是一家构建和部署 WASM 应用程序的平台提供商，他在一次虚拟的 [CloudNativeDay](https://www.techstrongevents.com/cloud-native-days-2022#agenda) 活动中告诉与会者，WASM 将成为一个新 IT 时代的核心，这将使组织能够更快地构建高度可移植的应用程序，这些应用程序不依赖于任何 IT 基础架构平台，并且更加安全。

WASM 的核心是一种可移植的二进制指令格式，用于构建描述内存安全的沙盒执行环境的软件。万维网联盟(W3C)推动了 WASM 的开发，这是为执行 JavaScript 代码的浏览器创建通用格式的努力的一部分。Wasm 现在正被扩展到浏览器和 JavaScript 之外，以使开发人员能够创建一组通用的二进制文件，无需修改就可以在任何平台上工作。

这种方法取代了当前主流的构建软件的方法，这种方法依赖于软件组件的集合，而这些组件之间往往缺乏明确的界限。这种方法的一个问题是，恶意软件感染应用程序的所有组件变得相对简单。

云计算原生计算基金会(CNCF)现在正试图推动 Cosmonic 的 WASM 运行时的采用，这使得在任何平台上运行 WASM 应用程序成为可能，而不是局限于构建只能在 Windows 或 Linux 上运行的应用程序。

Randall 指出，随着越来越多的组织寻求在广泛的边缘计算平台上推动应用部署，这种可移植性水平将被证明是至关重要的。

WASM 成为构建应用程序的首选工件可能还需要一段时间，但很明显，将一个 It 平台与另一个平台分隔开来的墙终于开始倒塌了。一旦应用程序变得更加可移植，编写一次应用程序，然后将其部署到任何地方的长期目标可能最终会实现。这不仅对应用开发人员的整体工作效率至关重要，而且对简化开发运维工作流的管理也有重要意义，因为初始部署应用和持续更新应用变得更加容易。

与此同时，随着组织继续审查他们的应用程序开发实践以更好地保护[软件供应链](https://devops.com/?s=software+supply+chain)，越来越清楚的是，构建应用程序的传统方法存在根本缺陷。与其试图重新设计流程来鼓励开发人员更好地保护应用程序，不如改变构建应用程序的方式可能更有效。毕竟，考虑到保护应用程序开发的遗留方法所需的所有过程，采用新的应用程序开发方法可能更便宜、更容易。

当然，挑战总是在于，从遗留应用程序开发方法的惯性中迈出最初的第一步。
