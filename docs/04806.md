# Red Hat 通过最新版本的 Red Hat OpenShift 容器平台推动了 DevOps 的采用和应用程序的可伸缩性

> 原文:[https://devo PS . com/red-hat-drives-devo PS-采用-应用-可扩展性-最新版本-red-hat-open shift-container-platform/](https://devops.com/red-hat-drives-devops-adoption-application-scalability-latest-version-red-hat-openshift-container-platform/)

北卡罗来纳州罗利([美国商业资讯](http://www.businesswire.com/))–全球领先的开源解决方案提供商 Red Hat，Inc. (NYSE: RHT)今天宣布推出 Red Hat open shift Container Platform 3.3，在一个安全的企业容器平台上带来了新的开发人员特性和部署可扩展性，以改善混合云的应用交付。red Hat open shift Container Platform 支持组织构建和维护云原生和传统应用程序，提供一个可以在整个应用程序生命周期(从开发到生产)中使用的解决方案。

Red Hat open shift Container Platform 3.3 由开放式行业标准提供支持，构建在 Red Hat Enterprise Linux 的可信基础上，提供了一个单一平台，在一个强化的企业就绪型产品中提供 Linux 容器和容器编排创新，以及应用程序开发和部署功能。Red Hat 是 Docker 和 Kubernetes 项目的主要贡献者，最新版本的 Red Hat OpenShift 容器平台提供了 Kubernetes 1.3 的企业版和 Docker 容器运行时，构建在 Red Hat Enterprise Linux 7 的基础上，帮助客户享受快速创新的好处，同时保持企业平台的稳定性、可靠性和安全性。

Red Hat OpenShift 容器平台 3.3 的新特性包括:

**增强开发人员的生产力和灵活性**

在现代 IT 组织中，开发人员需要更快地交付更好的应用程序和应用程序更新。red Hat open shift Container Platform 通过基于 Jenkins Pipeline 项目的新连续交付管道和显示应用程序管道统一视图的聚合开发人员仪表板，为应用程序开发生命周期带来新的自动化功能，从而帮助满足这一需求。额外的新功能通过 OpenShift 的集成路由和软件定义的网络功能对应用路由配置进行精细控制，从而提高 A/B 测试的自动化程度。

**云级部署**

由于向云的迁移要求计算平台具有更高的可扩展性和灵活性，Red Hat open shift Container Platform 3.3 有助于提供大幅改善的平台可扩展性，这是大规模运行任务关键型应用程序的组织的实质性需求，每个计算集群支持多达 1，000 个节点。这使得更大规模的部署不仅可以支持新应用，还可以支持混合云环境中现有的传统应用，从而帮助企业在不牺牲创新的情况下保留现有的应用投资。此外，Red Hat open shift Container Platform 固有的 Kubernetes 功能现在可以主动监控并防止资源匮乏和计划外停机。

**企业级安全性**

安全性是采用 Linux 容器技术的组织的主要关注点之一。OpenShift 提供了一个安全的、企业就绪的容器平台，它包括从 Linux 操作系统扩展到容器平台和集成应用服务的支持。OpenShift 包括 Red Hat Enterprise Linux，这是最受信任和安全的企业 Linux 平台，并继承了其所有的安全功能和持续更新。安全特性扩展到 Docker 容器运行时，该运行时与安全性增强的 Linux 相集成，以实现额外的多租户安全性，并扩展到 OpenShift 的集成容器注册表，该注册表允许管理员管理其 Docker 格式的容器映像，并允许他们控制访问和映像更新。OpenShift Container Platform 3.3 增加了额外的注册表增强功能，涵盖用户、团队及其项目，包括查看图像细节和管理图像访问的能力。OpenShift 还为与企业 LDAP 系统集成的 Kubernetes 提供集成的用户授权和基于角色的访问控制，以及用于容器执行的集成安全策略，因此管理员可以在一个地方控制用户访问、权限、配额和访问，以拉取和运行容器映像。

Red Hat 扩展的容器产品组合涵盖私有和完全托管的公共云产品，在一个解决方案中支持应用程序开发流程的不同方面，该解决方案可以跨多个基础架构并提供容器即服务。这些基础设施环境包括 Red Hat OpenShift Container Local，包含在[Red Hat Container Developer Kit](http://cts.businesswire.com/ct/CT?id=smartlink&url=http%3A%2F%2Fdevelopers.redhat.com%2Fproducts%2Fcdk%2Foverview%2F&esheet=51425407&newsitemid=20160922005195&lan=en-US&anchor=Red+Hat+Container+Developer+Kit&index=1&md5=5126f45a319e9f04e5b4488c551a5aa5)中，Red Hat OpenShift Container Lab，OpenShift Online 和 OpenShift Dedicated。Red Hat 的容器优化解决方案组合包括存储、应用服务和管理产品，以及免费开发工具。

**可用性**

Red Hat OpenShift 容器平台 3.3 将于 2016 年 9 月 27 日提供给客户下载。OpenShift Online 和 OpenShift 专用平台计划在 Red Hat open shift Container Platform 3.3 发布后不久进行更新。

**红帽开放创新实验室:通过容器和 DevOps 启动创新**

2016 年 4 月，红帽推出了[红帽开放创新实验室](http://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fwww.redhat.com%2Fen%2Fabout%2Fpress-releases%2Fred-hat-launches-red-hat-open-innovation-labs-introducing-collaborative-open-source-cloud-and-devops-residency-program&esheet=51425407&newsitemid=20160922005195&lan=en-US&anchor=Red+Hat+Open+Innovation+Labs&index=2&md5=3e51d6199244ae8c5691efbbdf0569d8)作为一项咨询服务，帮助公司整合人员、方法和技术，以加速的方式解决业务挑战。在开放创新实验室项目中，客户在面向派驻的环境中协同工作，使用 OpenShift Container 平台作为应用精益和敏捷原则的基础，以连接传统和当代应用程序开发。客户可以在自己的站点或 Red Hat 站点参加开放式创新实验室。
