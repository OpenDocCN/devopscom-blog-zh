# 推动更多 API 采用的数字业务转型

> 原文：<https://devops.com/digital-business-transformation-driving-more-api-adoption/>

今天，[应用程序编程接口](https://devops.com/?s=API) (API)中心提供商 RapidAPI 公布了一项针对 2200 名开发人员的全球年度[调查](https://stateofapis.com/)，调查发现[数字业务转型计划](https://digitalcxo.com/digital-business-transformation-requires-forethought/)已经导致对面向外部的 API 的依赖增加。

根据调查结果，近四分之三的开发人员(74%)正在为内部应用程序使用 API。然而，将近一半(49%)的人说他们正在开发第三方 API，44%的人在开发面向合作伙伴的 API。这比一年前的 35%有所上升。

RapidAPI 首席执行官 Iddo Gino 表示，很明显，API 是推动数字业务转型计划的关键使能技术。

根据调查结果，超过四分之三(75%)的开发人员表示，参与 API 经济是他们的组织现在或在不久的将来的首要任务。公司正在意识到新收入流的机会，API 货币化战略的增长将从 2020 年的 35%增加到 41%。

此外，超过三分之二的开发者(68%)表示，他们预计 2022 年将增加 API 的使用，其中 61%的人在 2021 年比 2020 年更依赖 API。

然而，超过一半的人(56%)指出，开发人员短缺是最令人担忧的问题，其次是缺乏围绕数字化转型工作的内部优先级(34%)、缺乏 API 开发资源和工具(32%)以及缺乏与其他部门的协调(29%)。整整 90%的人还指出，安全性和数据隐私是使用 API 时要考虑的一个关键问题。

结果还显示，2021 年，REST 和 SOAP APIs 在生产环境中的使用分别下降了 3.7%和 1.4%。Gino 说，这种下降反映了 GraphQL 和异步 API 等替代 API 技术的兴起，然而，他指出，任何新的 API 类别都不太可能在短期内取代传统 API；相反，在未来的许多年里，大多数组织将会使用各种各样的 API。

调查结果显示，JavaScript 是构建 API 最常用的(71.6%)和首选的(51.9%)编程语言。TypeScript 的使用率(20%)和受欢迎程度(12%)也越来越高。

然而，调查中最令人不安的一面可能是，超过一半的受访者(56%)表示他们对跳槽持开放态度。开发人员表示他们可能跳槽的五大原因是更高的薪酬或改善的福利(60%)，对不同公司的特定产品或技术的兴趣(54%)，对更灵活的工作环境的渴望(38%)，对特定公司或市场的兴趣(25%)以及与前队友或朋友合作的机会(17%)。

目前还不清楚所谓的大辞职现象会在多大程度上影响开发团队。然而，很明显，随着全球经济继续从新冠肺炎疫情复苏，没有任何一个职业能免于普遍的不安感。DevOps 领导者面临的挑战和机遇是鼓励最优秀的人才加入并留在他们的组织，而不是去其他地方寻找工作，至少从远处看，那里似乎有更好的工作。
