# 在 Gartner 的 2019 年应用发布流程幻方图(ARO)中，CloudBees(电子云)因其愿景的完整性而获得认可

> 原文：<https://devops.com/cloudbees-electric-cloud-recognized-for-completeness-of-vision-in-gartners-2019-magic-quadrant-for-application-release-orchestration-aro/>

CloudBees(电子云)连续第四年被评为 ARO 魔力象限的领导者。CloudBees 相信 CloudBees Flow 提供了一种安全可靠的方式，以前所未有的洞察力和控制力来扩展软件发布

October 29, 2019 11:30 AM Eastern Daylight Time

加利福尼亚州圣何塞([美国商业资讯](https://www.businesswire.com/))—[推动持续经济发展的企业开发运维领导者 CloudBees](https://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fwww.cloudbees.com&esheet=52118758&newsitemid=20191029005629&lan=en-US&anchor=CloudBees&index=1&md5=b11804d5aa3bc1566be8428908653671) 今天宣布，它已连续第四年被任命为 Gartner 2019 年应用发布流程幻方图的领导者。Gartner 分析师根据公司的执行能力和愿景的完整性来评估公司。CloudBees(电云)因其愿景的完整性在七家厂商中被定位在最右边。

CloudBees 首席执行官 Sacha Labourey 表示:“我们相信，CloudBees 连续第四年成为 Gartner 魔力象限的领导者，今年在愿景完整性方面排名最靠右，这证明了我们在软件交付领域实现市场领先卓越的承诺。“CloudBees Flow 提供了对复杂企业管道的高级控制和可见性，帮助我们的客户消除整个软件交付链中的发布焦虑。结合我们在基于 Jenkins 的 CI/CD 方面无与伦比的深度，CloudBees 提供了从代码到发布的强大软件交付解决方案。随着客户继续投资开发运维，我们比以往更加专注于帮助他们将软件交付变成真正的竞争优势。”

**缩放发布活动**

Gartner 将应用程序发布协调(ARO)类别定义为拥有使组织能够跨多个不同团队(例如开发运维)、技术、开发方法(例如敏捷)、交付模式(例如持续)、管道、流程和工具链扩展发布活动的解决方案。

Gartner 根据以下市场理解类别评估了供应商的愿景完整性:部署自动化、管道和环境管理、版本编排、可扩展性和性能，以及可用性和访问。根据这些标准，CloudBees 被定位在轴的最右侧，以保证视觉的完整性。

**ARO 市场被打乱**

在 2019 年的报告中，Gartner 表示，“这项研究将成为 ARO 供应商的最终魔力象限。”

“随着 ARO 市场被严重扰乱，Gartner 认为应用程序发布不再是理解和评估厂商前景的正确角度。各种工具(ARO、持续集成/持续交付(CI/CD)、无服务器、平台即服务(PaaS)等)之间的功能划分。——随着这些市场的融合，变得越来越模糊。因此，我们决定在 2020 年及以后，根据具体情况，为客户提供清晰、可行的 ARO 解决方案指导，”报告称。 ¹

“我们同意，ARO 市场正在迅速发展，这是客户连接其软件交付管理实践所有方面的整体行动的一部分，”Labourey 说。“组织在多个不同的团队、技术、流程、堆栈和工具链之间扩展发布活动的需求并没有消失，只是变得更大了。我们相信这种融合必须包括过程、工具和数据，这就是为什么我们坚信软件交付管理。”

**软件交付管理**

当前的软件交付实践限制了组织更快、更频繁地交付软件以满足业务需求的能力。组织依赖不同团队使用的孤立工具来执行大部分手动任务，这阻碍了他们在交付范围内进行协作和共享可见性的努力。CloudBees 正在通过构建客户想要的内聚系统来应对这一挑战，而不会强迫他们放弃现有的工具或流程。它将连接产品利益相关者和开发团队与业务的其余部分，实现从想法到交付的完全可见性和自动化。

CloudBees Flow 是该公司软件交付管理战略的关键组成部分，使团队能够快速实施并轻松适应，对任何规模的所有类型的发布都有前所未有的洞察力和控制力。
