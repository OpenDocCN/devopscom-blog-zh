# 谷歌寻求保护软件供应链

> 原文:[https://devo PS . com/Google-looks-to-secure-software-supply-chains/](https://devops.com/google-looks-to-secure-software-supply-chains/)

在[Google Cloud Next’22](https://cloud.withgoogle.com/next)大会上，谷歌今天推出了托管[软件交付屏蔽](https://cloud.google.com/blog/products/devops-sre/introducing-software-delivery-shield-from-google-cloud) (SDS)服务，使 DevOps 团队能够在工件注册表中存储、管理和保护构建工件。SDS 的核心是基于信任的策略引擎，在整个软件供应链中建立、维护和验证信任链。

此外，Google 已经更新了其容器分析工具，以包括针对 Maven 和 Go 容器的漏洞扫描，以及对非容器化 Maven 包的预览支持。

托管软件交付盾服务涵盖一系列 Google 运行时环境，包括 Google Kubernetes 引擎(GKE)、云代码、云构建、云部署、工件注册和二进制授权。除了为容器化的应用和非容器化的 Maven 和 Python 包生成认证的和不可伪造的构建来源之外，Cloud Build 现在还默认地正式支持软件工件的供应链级别(SLSA)3 级最佳实践。

在 preview now 中，GKE 提供了详细的评估，分配了严重性等级，并展示了集群和工作负载的安全状况，包括对操作系统漏洞、工作负载配置和其他可行见解的洞察。现在还可以捕获日志数据，同时将安全事件信息发送到安全信息和事件管理(SIEM)平台等。

Google Cloud Run 无服务器平台还可以在预览中显示软件供应链安全洞察，如 SLSA 构建级合规性、构建来源和运行服务中发现的漏洞。

与此同时，谷歌还提供预览云工作站，这些工作站在谷歌云上提供完全托管的安全开发环境，可以通过浏览器访问。内置的安全功能包括虚拟专用云(VPC)控制、无源代码本地存储、私有入口/出口、强制映像更新和身份访问管理(IAM)策略。

最后，面向集成开发环境(IDEs)的云代码插件现在包括一个 Source Protect 预览，以便在开发人员编写代码时，以漏洞依赖性和许可证报告的形式向他们提供实时安全反馈。

总的来说，谷歌声称它现在提供了超过 250 个跨 Java 和 Python 的精选包，这些包都经过了出处验证。它还会自动生成一个[软件材料清单](https://devops.com/?s=SBOM) (SBOM)来创建所有组件和依赖项的清单。

从历史上看，云服务提供商一直倡导云安全的共同责任方法，这要求组织不仅要保护其应用程序，还要保护云基础架构的供应方式。面临的挑战是，开发人员通常自己提供云基础设施，即使他们通常具有有限的网络安全专业知识。太多开发人员认为底层云基础设施比实际更安全。

现在，云服务提供商正在扩展他们提供的安全服务，而此时组织正在将安全性作为决定在何处构建和部署应用程序的更大因素。

谷歌云平台副总裁兼总经理 Sunil Putti 表示，谷歌正试图让网络安全对开发者不可见，因为护栏将自动嵌入平台。

问题是，随着新的云原生应用程序类别与传统的单一应用程序一起部署，云计算环境正变得越来越复杂。一如既往，复杂性是网络安全的敌人。
