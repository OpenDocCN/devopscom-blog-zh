# Instana 成为首个支持 GitOps 代理管理的 APM 解决方案

> 原文:[https://devo PS . com/instana-becoms-first-APM-solution-with-gitops-enabled-agent-management/](https://devops.com/instana-becomes-first-apm-solution-with-gitops-enabled-agent-management/)

***Instana Agent 自动支持基于 Git 的配置管理，以简化跨多台主机的配置部署***

**伊利诺伊州芝加哥** **，2020 年 7 月 22 日—**面向云原生微服务应用的实时应用性能管理(APM)解决方案的领先提供商 Instana 今天宣布推出首款采用 GitOps 支持代理的 APM 解决方案。随着该版本的发布，Instana 的代理现在可以跨任意数量的主机自动支持基于 Git 的配置管理。

“对于当今的组织而言，在生产环境中更新和配置代理是一项繁琐而耗时的工作。当然，运行的代理越多，任务就越繁琐，”Instana 技术总监兼 APM 策略师 Chris Farrell 说。通过这一最新的独特功能，Instana 为用户提供了更多的控制，同时减少了手动工作，使他们能够在代理启动和重新启动期间自动管理代理配置，并通过他们首选的 Git 存储库将 Instana 代理版本锁定到

传统的监控解决方案让组织选择使用静态代理或放弃版本控制。Instana 允许这两种选择，但是现在让使用 Git 存储库的客户端更加容易。Instana 代理支持基于 Git 的配置管理，为特定主机提供版本锁定控制，在代理启动和重新启动时自动获取配置更新，灵活地随意打开和关闭此功能，并能够根据所监视的环境将不同的主机代理组映射到 Git 分支。

除了手动代理配置是一个复杂且耗时的过程之外，它还引入了很高的出错风险，这可能会中断对关键系统的监控。Instana 的 GitOps enabled host agent 降低了这种风险，减少了配置错误，同时提供了在更新导致问题时轻松回滚的选项。通过这一新颖的功能，Instana 的代理为用户提供了更好的客户体验和更多的控制，允许他们在最方便的时候保留最新版本的代理。

与传统的 APM 工具不同，Instana 的自动化应用性能监控(APM)解决方案可发现所有应用服务组件和应用基础设施，包括 AWS 和 Lambda 等云基础设施、Kubernetes 和 Docker 等流程编排基础设施、应用服务和 DevOps 流程。Instana 自动为应用技术堆栈的每个部分部署监控传感器，跟踪所有应用请求并分析每个流程，无需任何人工配置，甚至无需重启应用。该解决方案实时检测应用程序和基础架构的变化，调整自己的模型，并在几秒钟内可视化这些变化和任何性能影响。

通过访问 https://instana.com 的，了解更多关于 Instana 的 [GitOps enabled 代理](https://instana.com/gitops-agent)和基于 Git 的[配置管理](https://www.instana.com/docs/setup_and_manage/host_agent/configuration/git_ops)的信息。
