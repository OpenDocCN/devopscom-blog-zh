# Atlassian 通过收购 AgileCraft 增加业务规划应用

> 原文：<https://devops.com/atlassian-adds-business-planning-app-by-acquiring-agilecraft/>

Atlassian 宣布将收购 agile craft,以使业务领导能够以一种与开发运维流程驱动的 it 项目紧密关联的方式了解战略项目。

这笔交易预计将于下月完成，价值估计为 1.66 亿美元。

许多 DevOps 团队已经广泛使用了 Atlassian 开发的吉拉项目管理软件。AgileCraft 于 2013 年出现，是一家协作应用程序提供商，旨在帮助业务用户跟踪各种主要依赖于 DevOps 流程的战略性 IT 计划。

Atlassian 和 AgileCraft 之前建立了一个技术联盟来促进这一目标的实现。Atlassian 总裁 Jay Simons 表示，此次收购为 Atlassian 提供了一个机会，可以将其触角延伸到商业用户领域，这些用户越来越多地试图跟踪依赖于多个 IT 项目的复杂数字业务转型计划。他说，许多数字业务计划往往会很快落后于计划，因为各种 IT 计划之间的依赖关系对于所有关键的业务利益相关者来说都不清楚。

此外，Simons 说，许多企业现在正试图在 IT 组织中更广泛地应用 DevOps 团队开创的敏捷方法。他指出，为了实现这一目标，业务用户需要一个经过优化的项目管理应用程序，使他们能够跟踪多个相互关联的 DevOps 项目。

大多数组织还没有达到 IT 组织完成 IT 项目的速度超过业务吸收能力的程度。但是，随着敏捷方法和最佳开发运维实践越来越广泛地被采用，有一天，IT 组织完成 IT 计划的速度可能会超过大多数企业的适应能力。事实上，对于采用 DevOps 的组织来说，一周向应用程序交付多个更新并不罕见。AgileCraft 规划软件将使商业用户更容易理解与这些更新相关的所有商业含义。具有讽刺意味的是，关于无法使 IT 与业务保持一致的持续争论通常归结为 IT 组织无法跟上业务需求的步伐。很快，事实可能会证明，该业务不再能够吸收新应用程序推出的速度。

与此同时，AgileCraft 等应用程序应能让 it 领导者更容易与业务领导者就有限资源的应用领域和应用时间进行更明智的对话。现在，企业领导人将能够亲眼看到任何一组错过的最后期限都可能对任何战略计划产生什么影响。事实上，可能有一天，业务领导会主动提供更多可用资源来实现特定目标，而不需要 IT 领导参加会议来详细解释每次延迟如何对业务的大计划产生负面影响。取而代之的是，企业领导人将能够自己准确地看到在哪里需要额外的资源来实现特定的业务目标。
