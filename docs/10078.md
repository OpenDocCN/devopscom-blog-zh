# 产品评估讨论

> 原文:[https://devops.com/product-evaluation-discussions/](https://devops.com/product-evaluation-discussions/)

与其像 99%的权威人士一样写预测博客，我想我会写一个认知博客来总结 2022 年。每年，您都会遇到各种各样的人，他们试图告诉您对您的 IT 工作可能有用(也可能没用)的产品。有些产品甚至进入了评估阶段，虽然有一些产品已经安装，但只有一小部分被使用。

在谈论令人眼花缭乱的新 DevOps 技术和产品时，最大限度地减少时间投入的方法之一是了解您所消费的信息来自哪里。下面是我的一个信息来源列表，它们所传递的信息来自哪里，以及如何最大限度地利用这些来源。

任何组织的营销团队都是为了宣传产品的优点而存在的。大多数营销专业人士只有在受到压力时才会承认潜在产品的弱点。这是意料之中的——如果你工作的唯一目的是告诉世界为什么需要这种产品，那么喋喋不休地谈论它的弱点是不会奏效的。但是我们都知道没有完美的产品，甚至在你考虑到我们正在安装的所有环境差异之前。所以听着——他们会尽可能好地展示产品，并拥有有用的知识，尤其是在评估过程的早期。

**销售:**销售团队想要销售产品，但他们也投资于快乐的客户。一个好的高科技销售代表正在建立关系，因为如果产品的价值在他们的网站上实现，组织可以创造一个收入流。这使得它们很实用。他们的目标肯定是积极地展示产品和公司，但是要承认更多的弱点，并为交易破坏者提供变通办法。

**销售工程师:**销售团队的很多实用性来自团队的工程师。他们必须帮助您安装和运行东西，所以他们在组织中有既得利益，提前知道什么需要工作和什么是风险。如果可以选择，在签订合同之前单独与销售工程师交谈——不是因为销售谎言，而是因为销售工程师专门负责记录可能出现的问题以及如何解决这些问题。

**人群评估:**关于人群评级网站，要记住两件事——参与者在最好的网站上是志愿者，在其他网站上是付费贡献者。对于付费的贡献者来说，动机很明显。对于许多自愿参与者来说，动机是他们要么喜欢要么讨厌这个产品。这些网站中的大多数允许供应商对评论做出回应，有些甚至允许供应商删除评估。没有人提供一种服务，可以纠正热情洋溢的供应商评论…记住这一点。这些网站是有用的，其中一些主要是想提供帮助的人。所以找到好的(我没有在这里做任何广告，所以没有链接——抱歉)并使用它们。

分析师:我认识的几乎所有分析师——无论是 GigaOm 的分析师还是竞争对手——都有这个领域的经验。他们已经这样做了，而且他们中的大多数人以某种方式继续参与其中。虽然我已经几个月没有写(或维护)一行代码了，但我会尝试定期这样做。在单个分析师层面，要关注的是它已经存在了多久，以及它们现在有多流行。有些人做纯分析，十年或更长时间没有接触过玩具，对于 C 级分析来说，这很好…但对于实施建议，最好找到不同的来源。

分析公司:这是分析中真正的问题出现的地方。作为一名企业架构师，我和一家分析公司打过交道——至少在当时——他们会说我们想让他们说的任何话。在评估中完全没有用，但在说服我们的技术顾问委员会批准正确的方法时，它工作得很好。每个公司都有自己的视角来看待问题，意识到一个特定公司的倾向是理解如何使用它们的一个很好的步骤——或者你是否应该依赖它们。

纯粹出版的时代基本上已经过去，留下了许多“付费出版”。虽然仍有非赞助内容在开发中(Techstrong Group 为我写这篇博客制定的少数规则之一是，我应该避免提及产品；例如，提供赞助时，他们可能会更多地关注他们的供应商关系，以及你们中有多少人在阅读我的胡言乱语)，以“文章”形式发布的大部分内容都是“营销”，也应该如此对待。阅读这些内容可能对早期教育有用，但对评估没什么用。

**内部支持者:**不是大多数项目要求的强制性支持者，而是推动组织寻找解决方案的人。这些人通常是关键。他们了解环境，他们了解产品，他们对产品如何帮助环境有想法，他们在组织中有历史。例如，他们关于各种产品如何帮助公司的想法是否在其他工具上得到证实。当然，通常的警告仍然适用…安全人员仍然认为您应该做更多的工作来阻止不良行为者——这是他们得到报酬的原因，但这会影响他们对新安全产品的评估以及该产品在组织中的位置。对这种既得利益的认识有助于过滤内部拥护者的意见。

当然，还有其他的，但我将保持这篇博客的篇幅，大多数——如 MSP 或主机服务提供商——可以被视为其中一种的延伸(例如，MSP 和主机服务提供商通常就像他们选择的产品的销售和营销的延伸)。

继续摇摆。所有这些人都可以帮助你理解如何让环境变得更好，或者让你的角色变得更轻松。他们不能替你做；那是你的技能。听听他们都说了些什么，然后按照组织的最大利益行事。当我不是专家时，我倾向于使用营销/人群评级来了解这个空间，然后分析师和系统工程师分别告诉我这个空间和其中的具体产品。但是要弄清楚什么对你的组织有用，并意识到不同的参与者给你的学习带来了什么观点。
