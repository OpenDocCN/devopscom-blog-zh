spherex 推出了企业级可插拔数据服务增强平台:spherex d b-Plus-Suite v 1.0 Business Suite，包括 spherex-Boot、spherex-Console、SphereEx DB Plus 引擎。该商业套件旨在为企业提供一站式解决方案，包括 spherex d B- Plus-Engine 集群的安装、部署、O&M 监控和插件管理，并为企业提供高效、灵活和全面的管理能力。

**安装和部署:SphereEx-Boot**

SphereEx-Boot 是安装和管理各种集群管理工具的理想解决方案。

SphereEx-Boot V1.0 是 Python 开发的命令行工具，用于轻松管理 SphereEx DB Plus 引擎代理集群，提供了显著的可用性和兼容性改进。

*   在可用性方面，spherex-Boot 提供了离线安装模式，允许用户在离线环境中通过一个命令安装和部署产品。
*   在兼容性方面，spherex-Boot 支持 Prometheus 的安装，允许轻松监控 spherex 的产品组件。

在不断增长的 Apache ShardingSphere 生态系统的背景下，SphereEx-Boot 管理 SphereEx DB Plus 引擎生态系统中的许多组件，并在企业生产环境中充当集群管理工具。

**数据管理和控制:SphereEx-Console**
SphereEx-Console 是一个数据可视化平台，用于管理和控制 SphereEx 企业数据服务平台，提供了用户友好的体验。它集成到一个解决方案中，包括资源、实例、插件等，以 SphereEx DB Plus Engine 为核心提供一站式用户体验。

新发布的 SphereEx-Console V1.0 进一步扩展了集群适配、DistSQL(分布式 SQL)和可视化方面的产品功能。

*   集群管理:spherex-Console 可以一键管理 spherex d B- Plus-Engine 集群，无缝访问用户现有平台，显著提高产品兼容性。
*   DistSQL: SphereEx-Console 完全支持 DistSQL 的操作，使用户能够创建和绑定各种类型的表——极大地提高了 DBA 的可用性。
*   可视化:spherex-Console v 1.0 支持用户定制监控指示器和界面管理可视化。

SphereEx-Console 是一个以管理 Apache ShardingSphere 为中心的 SphereEx 业务数据服务集群平台，它简化了用户配置、操作和部署过程。降低新手入门门槛在解决企业管理和控制痛点的同时，为企业提供增强的运营和管理。

企业级数据服务增强引擎:SphereEx DB Plus Engine
SphereEx DB Plus Engine 是一个企业数据服务平台，以 Apache ShardingSphere 为核心构建。除了 ShardingSphere 的功能之外，DB Plus Engine 还提供了诸如企业安全性、高性能集群、高可用性集群等特性。它由两个产品组成，SphereEx DB Plus 引擎驱动程序和 SphereEx DB Plus 引擎代理，可以组合部署也可以独立部署。

新发布的 SphereEx DB Plus Engine-Proxy V1.0 主要关注加密和解密、双路由和用户权限方面的更新能力。

数据特性:
数据加密解密:支持用户自定义数据类型。
·双路由:基于双路由功能的 JDBC 和代理集成。

用户权限管理:
增加了新的登录模式:LDAP 认证。
·增加角色管理。
增加了表级权限控制。
增加列级权限控制。
用户权限可以在线更改。

SphereEx DB Plus Engine-Driver 和 SphereEx DB Plus Engine 都可以提供标准化的数据横向扩展、分布式事务和分布式治理功能，适用于 Java 同构和异构语言场景以及云原生环境。

——

SphereEx Business Suite 将继续遵循分布式数据库系统的 Database Plus 设计概念，在分散的异构数据库之上构建一个生态系统，以扩展和增强原始数据库计算能力。应用程序和数据库之间的交互将面向 Database Plus 的标准，最大限度地减少上层服务上的数据库碎片，实现 Database Plus 的三个核心目标:连接、增强和可插拔。

“云”是 SphereEx 的一个关键技术发展趋势。今天，云原生已经成为托管大规模应用程序和掌握未来技术的关键。SphereEx 将主动适应云原生场景的数据需求，使云原生数据库更贴近用户，并为企业提供灵活高效的云数据治理，以加速其从内部部署向云的过渡。
