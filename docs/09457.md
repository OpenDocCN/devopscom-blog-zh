# Orca Security 添加 CLI 以提高云安全性

> 原文:[https://devo PS . com/orca-security-adds-CLI-to-improve-cloud-security/](https://devops.com/orca-security-adds-cli-to-improve-cloud-security/)

Orca Security 通过命令行界面 (CLI)扩展了其云安全平台[，使其更容易与各种 DevOps 工具集成。](https://www.businesswire.com/news/home/20220511005350/en/Orca-Security-Unveils-Industry%E2%80%99s-First-Context-Aware-Shift-Left-Security-to-Identify-and-Prevent-Cloud-Application-Security-Issues-Earlier-in-the-Development-Cycle)

Orca 安全平台不依赖代理，而是使用只读访问来创建风险概况，以阻止通过亚马逊网络服务(AWS)、微软 Azure 或谷歌云平台上托管的运行时访问存储。这种方法被称为 SideScanning，它消除了 DevOps 团队部署和维护代理软件以确保云安全的需要。

然后，该平台扫描工作负载和云配置元数据，以构建风险图，从而更好地帮助开发运维团队确定云安全工作的优先级。

Orca Security 首席执行官 Avi Shua 表示，CLI 现在可以更容易地将云安全的责任进一步转移给开发人员和支持他们的 DevOps 团队，通过更大范围的 DevSecOps 最佳实践来更容易地扫描漏洞。

云安全仍然是一个重大挑战，因为基础架构通常是由很少或没有安全专业知识的开发人员提供的。犯错误几乎是不可避免的。Orca Security 正在开发一种工具，使组织能够识别这些安全问题，而无需在广域网上部署额外的代理软件。

当然，挑战在于确定将应用程序安全性的责任转移到什么程度。Shua 指出，即使对安全问题有所警觉，许多开发人员可能也没有完全意识到这个问题的严重性。许多开发人员还认为他们的云服务提供商正在提供他们实际上没有的安全级别。部署应用程序的实体有责任保护它和用于部署它的相关配置。他补充说，将云安全工具集成到 DevOps 工作流中变得至关重要，因为它使 DevOps 团队的更多成员能够在应用程序构建时而不是在生产环境中运行后评估业务的潜在风险。

许多现有的 DevSecOps 工具没有提供足够的上下文；Shua 指出，他们所做的只是提供已经部署的代码的静态分析。他说，开发人员需要更深入地了解哪些问题代表了需要他们立即关注的风险水平。

当然，随着越来越多的组织在一系列备受瞩目的网络安全违规事件后审视其[软件供应链](https://devops.com/?s=software+supply+chain)的构建方式，云安全受到了前所未有的关注。许多人发现，他们对云计算环境的安全可见性水平充其量也是有限的。

安全问题不太可能会降低应用程序在云中部署的速度。相反，挑战在于更好地理解如何使用与在内部 IT 环境中保护应用程序完全不同的工具和流程来保护这些应用程序。
