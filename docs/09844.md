# 数据操作的分类

> 原文：<https://devops.com/the-taxonomy-of-dataops/>

数据操作，或简称为 DataOps，是许多人使用的 IT 术语之一，但很少有人能准确定义。像云或 DevOps 一样，数据操作没有意义，除非你坐下来真正思考它需要什么。

为此，让我们在本文中了解一下数据运营的分类，或者换句话说，是哪些 IT 学科、专业领域和工作类型促成了数据运营。我不会解释关于 DataOps 如何工作的一切(这需要不止一篇博文)，但我会提供一个关于 DataOps 工作流如何崩溃的初级读本。

## 什么是数据操作？

在深入研究数据操作的分类之前，让我首先在较高的层次上定义我使用术语“数据操作”时所指的内容

我使用 DataOps 作为所有类型的数据操作的简写，从数据收集和存储到数据管理、监控、分析和性能优化。

我提到这一点是因为当一些人听到术语 DataOps 时，他们只会想到" [DataOps Manifesto](https://www.dataopsmanifesto.org/) "和类似文章中描述的实践和想法，这些文章侧重于将 DevOps 软件交付理念的核心原则扩展到数据操作。我是“数据操作宣言”的忠实信徒，但因为它是针对数据科学家而不是整个 it 社区的，我认为它呈现了数据操作所需要的狭隘形象。

对我来说，数据操作包括任何以任何方式使用数据的人所执行的工作。虽然数据科学家和数据工程师可能会执行与数据转换和数据分析相关的大部分繁重工作，但开发人员和 IT 工程师在数据操作中也扮演着重要角色。当我使用数据操作这个术语时，我也想到了它们。

## 数据操作的分类

澄清之后，让我们来探索一下数据运营的内容。

**数据收集**

没有数据就没有数据操作，因此，数据收集是数据操作的核心部分。

当然，数据收集指的是积累数据并将数据存储在某个地方一段时间的所有过程。

值得注意的是，数据收集可以是被动的，也可以是主动的。如今，组织存储和分析的许多数据都不是人们出去手工收集的数据。它是通过这些组织的日常运作产生的信息——可以说，它们的[数据耗尽了](https://whatis.techtarget.com/definition/data-exhaust)。

**数据存储**

数据存储看起来简单明了。毕竟，你真的需要考虑多少数据的存储？你不就是挑一个存储位置，把数据放在那里吗？

不，数据存储要比这复杂得多，这就是为什么数据存储是 DataOps 中一个独特的学科。

有效的数据存储需要找到正确类型的存储基础架构，即能够在数据性能、可用性和成本之间实现最佳平衡的基础架构。它还涉及定义数据保留策略，以确定数据将存储多长时间。它还包括数据备份，这对于确保您打算保存在存储中的数据在意外发生时确实可用非常重要。

**数据管理**

数据管理是一个广泛的学科，涉及到在收集和存储数据后保持数据可用和可访问所需的所有任务。它需要将数据从一种格式转换为另一种格式，以使其与给定的工具集兼容。它还涉及在存储位置之间移动数据，以及识别和解决整个数据操作管道中降低性能的瓶颈等任务。

**数据监控**

在 IT 界,“监控”一词通常是指对应用程序或基础设施的问题监控。但是，如果您想有效地进行数据操作，您还必须监控您的数据。

数据监控包括诸如识别数据质量问题(如丢失数据或数据格式问题)之类的任务，这些问题会降低数据的可靠性或增加数据处理的难度。这还意味着检查故障迹象，如数据库故障或基础设施崩溃，这可能会导致整个数据操作管道破裂。

**数据分析**

正如 Gartner [所言](https://www.gartner.com/smarterwithgartner/turn-data-into-dollars/)，每个人(至少是每个人的老板)都想“将数据转化为金钱”。数据分析，或在数据集内寻求洞察力的过程，是实现这一目标的关键。

复杂的数据分析是这个列表中唯一一个通常由数据科学家独自完成的学科，而不是 IT 团队的其他成员——尽管随着越来越多的简化数据分析和可视化工具进入市场，这种情况正在发生一些变化，这使得在没有统计学博士学位或不知道如何用 r 编程的情况下理解大型数据集变得很容易。

**数据安全**

数据安全是一个大话题，尤其是在当今备受瞩目的数据安全漏洞时代。保护数据安全需要一系列流程和工具，从为数据设置正确的访问控制策略，到监控未经授权访问的迹象，以及扫描数据以根除数据集中可能存在的恶意软件或其他威胁。

## 数据行动是每个人的行动

您可能已经注意到，上面的 DataOps 分类中描述的许多任务通常是由 IT 部门的员工(或者在某些情况下是程序员)执行的，而不是专门从事数据操作的数据科学家或工程师。

这是非常重要的一点，因为当许多人谈论数据操作时，他们假设每个组织都有一个数据专家团队，他们处理与数据相关的所有工作，从收集和存储数据到分析和保护数据。

实际上，许多公司没有专门的数据运营团队。即使他们这样做了，一般的 IT 部门仍然在一些任务中发挥着巨大的作用，例如为数据收集配置工具和平台，提供数据基础设施以及保持数据库健康运行。安全工程师也是实施数据访问控制和监控数据安全问题迹象的关键。

因此，无论您的公司是否有专门的数据运营团队，数据运营都是每个从事相关工作的人的责任(至少是部分责任)。这是每个人的行动。
