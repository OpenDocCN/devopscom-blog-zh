# Splunk 调查显示可观察性有所提高

> 原文:[https://devo PS . com/splunk-survey-surface-gains-in-observability/](https://devops.com/splunk-survey-surfaces-gains-in-observability/)

Splunk 今天发布的一项针对 1，250 名可观察性从业者、经理和其他专家的全球[调查](https://www.splunk.com/en_us/newsroom/press-releases/2022/state-of-observability-2022-report-reveals-organizations-with-mature-observability-practices-significantly-reduce-costs-while-increasing-innovation.html)发现，经验丰富的可观察性从业者能够削减 90%的停机成本。这个数字是根据新来者每年 2380 万美元到 250 万美元的估计费用计算的。然而，与 59%的初学者相比，只有 9%的受访者足够先进，可以被认为是可观察性领导者。

这些领导者表示，由于在可观察性方面的投资，他们看到计划外停机或性能下降的平均解决时间(MTTR)缩短了 69%。调查还指出，去年，这些组织从应用程序开发计划中推出的产品或收入流增加了 60%。

可观察性一直是 DevOps 的核心原则，但是实现和维护是一个挑战。如今，大多数 DevOps 团队都渴望能够保持某种程度的持续监控。然而，随着对应用程序进行检测变得越来越容易且成本越来越低，人们对可观测性平台的兴趣也越来越大，这使得启动查询和调查异常变得更加简单。

Splunk 高级副总裁兼可观察性总经理 Spiros Xanthos 表示[可观察性平台明显降低了停机成本](https://digitalanarchist.com/videos/devops-topic/observability-pace-quickens-spiros-xanthos-splunk)。他指出，在接受可观察性的组织和已经部署的基于微服务的云原生应用程序的数量之间也有明显的相关性。Xanthos 说，事实上，复杂性的增加正在推动组织转向可观察性平台来管理云时代的 IT 环境。

调查发现，75%的受访者拥有在多个环境中运行的多个云原生应用。略超过三分之一(34%)的内部开发应用基于使用容器构建的微服务，28%的受访者报告称他们在公共云基础设施上专门运行云原生应用。

总体而言，调查还发现，三分之二的领导者表示他们对应用程序性能的可见性非常好，相比之下，只有 44%的初学者表示非常好，而 64%的领导者表示他们对安全状况的可见性非常好，相比之下，只有 42%的初学者表示非常好。

调查还发现，对于大多数内部开发的应用程序，59%的领导者可以按需将代码投入生产，相比之下，新手只有 28%。共有 41%的领导者表示，他们可以在几分钟内检测到与内部开发的应用程序相关的问题，相比之下，初学者的这一比例为 20%。

就可观测性平台的采用而言，现在仍处于早期阶段，但从理论上讲，即使使用的平台数量继续增加，管理起来也会变得更加容易。Xanthos 说，分析和人工智能(AI)的进步正在提高信噪比，因此不仅更多的问题会更快地浮出水面，而且会提出更可信的建议。

当然，可观察性是一个旅程。大多数组织甚至不太确定[他们应该制定什么类型的问题](https://containerjournal.com/features/the-power-of-asking-questions-container-scaling-and-observability/)来从可观察性平台的投资中获得最大价值。Xanthos 说，组织应该从小处着手，通过一个有限的项目来了解如何在更成熟的时候更广泛地利用可观测性。

无论采用何种方法来实现可观察性，有一点是肯定的，那就是在 IT 环境日益复杂的时代，传统的 IT 监控已经不够用了。
