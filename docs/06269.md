# 戴尔技术推进混合云战略

> 原文:[https://devo PS . com/Dell-technologies-advances-hybrid-cloud-strategy/](https://devops.com/dell-technologies-advances-hybrid-cloud-strategy/)

戴尔技术公司与 VMware 和 Google 合作，今天[通过在 Google 云平台(GCP)上提供其 CloudFS 存储系统，并在其用于内部 IT 环境的 VxRail 超融合基础架构(HCI)平台中添加对 VMware Tanzu Kubernetes Grid 和 VMware Cloud Foundation 4 的支持，推进了其混合云战略](https://www.prnewswire.com/news-releases/dell-technologies-cloud-and-google-cloud-launch-hybrid-storage-solution-301062384.html)。

戴尔技术产品营销副总裁 Varun Chhabra 表示，这两项举措都是戴尔持续努力的一部分，旨在为 IT 组织提供跨内部 IT 环境和公共云的一致云运营模式。

在新冠肺炎疫情带来的经济衰退之后，戴尔更加积极地通过整合多个云的管理来降低 IT 成本。

VMware 云平台业务部门的副总裁兼首席技术官 Kit Colbert 表示，目前在多个云平台上的支出水平是不可持续的。然而，这并不意味着 IT 组织应该整合云。相反，他说，组织需要简化多种云的管理以降低成本。

VMware 一直致力于在内部和来自 Google、Amazon Web Services (AWS)和其他公司的公共云上部署其 VMware vSphere 平台，以便更轻松地构建和部署组织认为最合适的应用程序工作负载。例如，仅仅因为应用程序工作负载是在特定的公共云上创建的，并不意味着工作负载需要部署在该平台上，Colbert 说。

与此同时，VMware 一直在为在 VMware vSphere 之上部署基于 Kubernetes 的云原生工作负载寻找理由。 [VMware Cloud Foundation 4 提供对一整套基础架构软件的访问，其中包括嵌入到 VMware vSphere 最新更新中的 Kubernetes](https://containerjournal.com/topics/container-ecosystems/vmware-delivers-on-kubernetes-promise-for-vmware-vsphere/) 分发版。VMware 还提供了 VMware Tanzu 云服务，可用于跨多个 Kubernetes 发行版构建、管理和部署容器化的应用程序。

事实上，VMware 和戴尔正在推动扩展已经广泛部署在内部 IT 环境中的管理平面，以涵盖在云中运行的整体式和基于微服务的应用程序。可能会有比以往更多的工作负载转移到云中，但 Colbert 指出，大部分应用程序工作负载和数据仍然驻留在内部 IT 环境中。

虽然戴尔和 VMware 可能有优势，因为许多 IT 组织已经依赖其平台来管理 IT，但竞争平台的每个提供商都在制造类似的混合云计算案例。例如，AWS 正在支持 VMware vSphere 在其云上运行，同时[推动在内部 IT 环境中采用其基础架构](https://devops.com/aws-adds-gui-to-snow-edge-computing-family/)，作为戴尔和惠普企业(HPE)平台的替代方案。

It 组织定义他们的混合云计算战略可能还需要一段时间。如今，大多数 IT 组织继续孤立地管理 IT 环境。然而，减少管理每个 IT 平台所需的专家数量的压力越来越大。无论 IT 环境变得多么复杂，大多数 IT 组织都被要求做到事半功倍。现在的挑战是找出如何最好地自动化管理，无论从哪方面来看，现在都是令人困惑的平台和工作负载组合。
