# 如何在不破坏安全性的情况下快速移动

> 原文:[https://devo PS . com/how-to-move-fast-without-breaking-security/](https://devops.com/how-to-move-fast-without-breaking-security/)

“快速行动，打破常规”是一句耳熟能详的格言。这句话出自脸书首席执行官马克·扎克伯格之口，有助于解释该公司在过去十年中因产品创新而取得的惊人增长。然而，尽管对于软件开发来说这是一个有用的哲学，但是在其他领域，比你计划的更快是一个危险的方法，正如全球的组织在新冠肺炎疫情期间所认识到的。虽然 2020 年数字化转型计划的时间表[提前了七年](https://www.mckinsey.com/business-functions/strategy-and-corporate-finance/our-insights/how-covid-19-has-pushed-companies-over-the-technology-tipping-point-and-transformed-business-forever)，但企业快速向云迁移也意味着一些事情在此过程中受到了损害，包括安全性。

我们最近与云安全联盟进行的调查显示，超过一半的组织现在在公共云中运行超过 41%的工作负载，而 2019 年只有四分之一，到 2021 年底这一比例将进一步增加。企业正在快速向云迁移，但他们也发现在此过程中会出现问题。

该研究显示，去年有 11%的组织报告了云安全事件，其中三个最常见的原因是云提供商问题(26%)、安全错误配置(22%)和拒绝服务利用等攻击(20%)。就这些中断性云中断的业务影响而言，24%的受访者表示恢复运营需要长达三个小时，26%的受访者表示需要半天以上。

因此，组织非常关注在云中实施和管理安全性也就不足为奇了。他们最关心的问题是维护整体网络安全、缺乏云专业知识、将工作负载迁移到云时出现问题，以及没有足够的员工来管理扩展的云环境。那么，这些云安全问题和挑战的根本原因是什么，企业应该如何解决它们呢？

## 对云控制的困惑

当被问及在公共云中运行应用程序时，他们最担心安全的哪些方面时，绝大多数受访者表示要清楚地了解整个混合网络资产的拓扑和策略，其次是检测风险和错误配置的能力。

这些担忧的一个关键原因是，组织正在使用一系列不同的控制措施来管理[云安全](https://devops.com/?s=cloud+security)，作为其应用程序编排的一部分；52%使用云原生工具，50%报告使用编排和配置管理工具，如 Ansible、Chef 和 Puppet。然而，近三分之一(29%)的受访者表示他们使用手动流程来管理云安全。

此外，对云安全的整体控制也存在竞争:35%的受访者表示他们的安全运营团队管理云安全，其次是云团队(18%)和 IT 运营(16%)。其他团队，如网络运营、开发人员和应用程序所有者，也都参与其中。让不同的团队使用多种不同的安全控制会限制整个混合云环境的整体可见性，还会显著增加安全流程的复杂性和管理开销。任何时候你需要做出改变，你都需要在这些不同的控制和团队中重复工作。这会导致安全漏洞以及我们前面提到的基于错误配置的事件和中断。

## 如何快速移动*而不是*打破东西

那么，组织如何解决这些安全和管理问题，并获得对其云和内部环境的一致控制，从而充分利用云的灵活性和可扩展性，而又不影响安全性呢？以下是四个关键步骤:

**获得整个网络的整体可见性:**缺乏云资产的可见性是调查受访者面临的主要安全挑战。组织需要获得所有不同云帐户的整体可见性，包括在云中使用的安全控制以及管理进出云环境的流量。这使 IT 和安全团队能够快速识别任何可能导致停机的潜在安全问题或风险。

**用一个管理控制台来管理所有策略:**当在云和内部混合使用云提供商的安全控制和其他控制时，对组织来说，一致地管理策略是一个巨大的挑战。如果使用一组命令和语法从一个控制台管理所有这些不同的安全控制，就可以一致地应用安全策略。这避免了重复工作和容易出错的手动流程，从而避免了配置错误和停机。它还提供了对每项变更的完整审计跟踪。

**自动化安全和合规性流程以提高速度:**自动化对于高效管理云环境，同时协调复杂的安全控制变化至关重要。它为管理云环境中的安全变化带来了速度和准确性。它还加快了审计准备，并有助于确保持续的合规性。自动化还可以帮助组织克服技能差距和人员限制，超过四分之三的调查受访者认为这是一项挑战。

**随时随地加快事件响应:**我们的调查还显示，组织担心云中的整体网络安全。与 SIEM 或 SOAR 工具集成的安全管理解决方案可以通过加快事件响应来帮助解决这些问题。一旦 SIEM 或 SOAR 产品检测到威胁，管理解决方案就可以识别威胁可能影响的所有应用程序和服务器。然后，该解决方案可以通过自动将任何受影响的服务器或设备与网络隔离(无论是在云中还是在内部)来防止横向移动，从而降低威胁的风险。

借助处理这些步骤的网络安全自动化解决方案，组织可以获得跨其所有公共云帐户以及私有云和内部部署的整体单一控制台安全管理。这有助于他们解决云复杂性挑战，并确保更快、更安全、更合规的云管理，使组织能够快速响应不断变化的业务需求，而不会造成任何破坏。
