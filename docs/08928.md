# 红帽为微软 Azure 云带来了可实现的自动化

> 原文:[https://devo PS . com/red-hat-bries-ansi ble-automation-to-Microsoft-azure-cloud/](https://devops.com/red-hat-brings-ansible-automation-to-microsoft-azure-cloud/)

Red Hat 本周宣布，Red Hat Ansible Automation 平台的私人预览版现已在微软 Azure Cloud 上发布。

Red Hat Ansible automation 副总裁汤姆·安德森表示，让开源自动化框架的 Red Hat 实例在 Azure 上可用是让 it 团队能够跨多个云使用 IT 自动化框架的第一步。

我们的目标是让 it 团队能够更简单地使用通用框架连接多个 IT 环境，以便在混合云计算环境中自动配置从操作系统到安全平台的一切。

Ansible 正在成为自动化 it 环境的事实上的标准，这主要是因为它为 IT 团队提供了一种自动化 IT 流程的声明性方法。IT 团队可以编写剧本，描述他们对平台的期望状态，而不必了解平台的当前状态。

随着部署在云和内部 IT 环境中的工作负载数量稳步增加，人们对 IT 自动化的兴趣也随之高涨。一个简单的事实是，许多组织无法跟上应用程序构建和部署的速度，因为他们无法雇佣和留住足够的 IT 员工。挑战在于编写剧本来自动化任何 IT 流程需要大量的专业知识，而许多组织都缺乏这些专业知识。然而，现在有大量的现有模板可供组织重用。

![](../Images/c5a8079aae1c6c5944acb4173d051a60.png)

安德森还指出，安全问题也将迫使 IT 自动化的问题。云计算环境中运行的许多应用程序工作负载配置错误，因为它们是由几乎没有安全专业知识的开发人员提供的。安德森解释说，剧本大大降低了应用程序工作负载在云中被错误配置的可能性，而不必将每个开发人员都变成安全专家。

Anderson 补充说，最终，IT 自动化框架减少了开发人员和 IT 运营团队其他成员之间经常存在的摩擦，这实际上有助于提高开发人员的生产率。

不管 IT 组织如何自动化 IT 流程，现在的问题是目标实现的速度，而不是是否会实现。当然，DevOps 团队一直处于自动化的最前沿，但现在很明显，随着 it 环境变得更加复杂，IT 组织正在更加积极地拥抱自动化。这些最初的努力大多集中在重复性、[乏味的](https://devops.com/?s=toil)和手动的 IT 任务上，但是以前所未有的规模实现 IT 流程自动化的潜力从未如此之大。

事实上，许多组织可能很快就会发现，如果不投资自动化，很难留住 it 专业人员。毕竟，最有才华的 IT 专业人员也不会愿意为一个要求他们总是手动配置 IT 资源的组织工作。许多 IT 团队可能还会发现，像 Ansible 这样的自动化框架是拥抱其他 DevOps 最佳实践的门户。当然，挑战在于当有如此多的 IT 团队不能忽视的紧迫问题时，找到时间开始自动化过程。
