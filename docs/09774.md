# 在人工智能增强的 DevOps 中寻找价值

> 原文：<https://devops.com/finding-value-in-ai-augmented-devops/>

虽然人工智能(AI)和[机器学习](https://devops.com/five-challenges-of-machine-learning-devops/) (ML)是新兴技术，但我们知道它们可以帮助组织解析大型数据集并收集可操作的见解。但是，人工智能注入的流程真的对采用 DevOps 的组织有影响吗？Tricentis 最近的一项调查揭示了这个问题的答案。Tricentis 与 [Techstrong Group](https://techstronggroup.com/) 合作，调查了全球 2600 多名 DevOps 从业者关于其人工智能增强的 DevOps 流程的情况。结果表明 DevOps 有一个光明的未来。

在这项调查中，大多数受访者表示他们已经从人工智能驱动的开发运维流程中获得了价值。三分之一的调查受访者认为人工智能增强的 DevOps 今天非常有用，近一半的人认为非常有用。尽管该技术仍然相对年轻，但 IT 基础设施自动化公司 Puppet Labs 的 2021 年报告发现，83%的组织正在实施 DevOps 流程。这种快速的采用证明了这项技术的前景。

有趣的是，近三分之二的受访者表示，测试是 DevOps 中人工智能将产生最大影响的领域。测试是 DevOps 组织的一个主要痛点，因为扩展的 DevOps 环境需要复杂的测试场景和大量数据，许多组织努力将其自动化扩展到 DevOps 要求的水平。如果没有自动化，手动测试会大大延迟发布。在注入 AI 的 DevOps 流程中，AI 可以帮助加速测试的创作，帮助测试人员了解风险在哪里，并修复损坏的测试。

### 人工智能最大的潜力在于功能测试

我们通常认为软件测试要么是功能性的，即程序使用户能够正确执行任务，要么是非功能性的，即应用程序是安全、快速和稳定的。回答者被问及哪些测试学科会从人工智能中获益最多，其中 65%的人表示是功能测试。单元测试和 UI 测试是最常被选择的子类别。

功能测试过程的复杂性是人工智能的亮点。功能测试是开放式的，需要许多排列来确保适当的测试覆盖率。人工智能介入并管理这些排列，解析大量数据并提供见解。此外，人工智能可以帮助争论测试环境中没有发现的生产信号。

在整个调查中，受访者特别提到了 AI 在 UI 测试方面的承诺。在大多数组织中，UI 测试可能是最耗时和最手工的测试。由于存在许多不同的用户体验路径，人工智能可以帮助模仿真实终端用户的行为，根据应用程序的细微差别或体验的变化进行调整。AI 和 ML 也可以加速你的构建过程，提高重复任务的效率。快速识别和解决问题的能力可以提高自动化的效率。

虽然调查受访者认为测试潜力最大，但他们承认人工智能驱动的 DevOps 可以解决其他业务挑战。超过 40%的人声称，这些流程可以减少技能差距，使初级员工能够执行复杂的任务，改善客户体验，降低成本，促进创新，并提高开发人员的效率。近 40%的人还认为，人工智能驱动的 DevOps 为领导层提供了持续改进所需的洞察力。

### 自动化是扩展开发运维的关键

值得注意的是，在从 AI 和 ML 中看到好处的 DevOps 组织中，大多数是成熟的组织；他们采用 DevOps 工作流管道、工具链、自动化和云技术，并且他们已经自动化了超过一半的测试。只有 21%刚刚接触开发运维的从业者达到了这种自动化水平。即使在成熟的组织中，测试仍然是主要的瓶颈；只有 40%的公司自动化了一半以上的测试。

扩展 DevOps 实践的关键是开发测试自动化，以跟上不断增长的发布量和发布速度。但是通往 DevOps 的道路并不总是一帆风顺。从业者列举的障碍包括缺乏技术技能(44%)、预算不足(25%)和工具选择(19%)。想要实施 DevOps 的组织需要依靠外部供应商和内部资源来应对这些挑战并有效地进行扩展。

### 希望采用更多 DevOps 实践？

那么，一个想要实现 DevOps 的组织从哪里开始呢？带着大问题。确定您想要解决的具体业务问题至关重要。你想要的收益是多少？你希望在软件生命周期的哪个阶段发现好处？AI 和 ML 可以增加商业价值，但它们不能解决整个生命周期的所有问题。

在一个组织确定了它的 DevOps 目标之后，它的焦点应该转移到自动化上。为了获得人工智能测试的好处，一个组织应该自动化重复的功能测试，尤其是单元和回归测试。大多数组织从实现持续集成和持续交付开始。一旦一个组织采取了这些步骤，它将开始积累大量运行测试用例产生的结果和工件的缓存，人工智能可以挖掘这些结果和工件，以提高测试稳定性并识别重复出现的问题。自动化程度越高，收获就越多，交付速度就越快。

刚刚开始使用 DevOps 的组织也应该了解人工智能和 ML 之间的区别。大多数人把 ML 称为 AI，但是它们是不可互换的，而且大多数组织还没有发展到使用 AI。一个真正的人工智能是广泛强大的，而 ML 技术是狭隘的，解决一个模型被训练的特定领域的问题。

当您评估潜在供应商时，有许多关键问题要问，包括:

供应商使用的是什么特定的人工智能技术？是现成的图书馆还是他们自己建的？

他们如何使用人工智能？哪些算法在起作用，它们采用什么方法？

他们如何训练他们的模型？不同类型的模式出现在数据集中，这取决于行业、数据更新的频率和应用。供应商正在培训的数据群体必须看起来像您的生产数据，否则模型可能会将您引向错误的方向。

他们如何确保不存在偏见？例如，如果他们在不同的领域或垂直领域训练模型，你可能是那个供应商的局外人。

他们如何处理失败？他们如何以一种让你信任数据的方式监控人工智能的错误分类和误用？

你的数据会被用来训练和改进你的学习模式吗？如果是这样，这是否会引发隐私问题？对数据隐私的期望应该明确，不留任何疑问。

在您的垂直行业中，是否有其他具有类似业务特征的客户正在获得您所寻求的好处？供应商能提供例子吗？

### AI 注入的 DevOps 在实践中是什么样子的？

假设一家金融服务公司有 20 多个应用程序需要管理。面对如此多的应用程序，有必要进行大量的 UI 测试，而且测试必须稳定。UI 测试的最大挑战之一在于定位各种 web 元素；大多数人从基于脚本的手动自动化开始，但是创建和维护它会很有挑战性并且很耗时。使用人工智能和智能定位器来识别这些元素可以节省时间，并大幅减少 UI 测试维护。

调查受访者被问及他们目前如何使用人工智能来增强他们的测试流程。37%的人说他们正在加速创建他们的自动化测试用例，而 44%的人将他们的测试集中在风险最高的领域。几乎一半的人能够通过自我修复来减少测试用例维护，43%的人使用人工智能来确定失败测试的根本原因。31%的人声称人工智能提供了对测试过程改进的见解，34%的人使用人工智能来帮助识别基于应用程序内的变化而运行的测试。

### 人工智能驱动的 DevOps 下一步将走向何方？

展望未来，人工智能将越来越多地嵌入测试过程。现成的供应商正在取得巨大的进步，他们的产品比过去更容易获得。随着工具的不断成熟，这些产品将减少对专业技能的需求。像数据湖或数据仓库这样的内部计划将会持续，但建立大型数据科学团队以获得人工智能优势的需求将会减少。
