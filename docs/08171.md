# SREs 说 AIOps 没有达到宣传的效果

> 原文:[https://devo PS . com/sres-say-ai ops-donts-live-up-to-the-hype/](https://devops.com/sres-say-aiops-doesnt-live-up-to-the-hype/)

投资人工智能做 IT 运营( [AIOps](https://devops.com/?s=AIOps) )可以期待什么？海量信息的实时可见性。闪电事件相关和异常检测。自动化补救和自我修复，运营人员无需动手。

这听起来很神奇——至少，如果你相信市场宣传的话。但是这些 AIOps 的好处真的能转化到现实世界中吗？根据最近对近 300 名现场可靠性工程师的调查，情况并非如此。

在接受调查的 sre(包括一些世界上最大、最具创新性的数字公司的工程师)中，只有 12%的人将 AIOps 作为他们监控工具包的日常组成部分。近 40%的人根本不用。

为什么当前的 AIOps 解决方案没有达到宣传的效果？对于 IT 监控的总体状况，这一差距能告诉我们什么？让我们仔细看看。

## 庞大的复杂性

理论上，AIOps 应该非常适合 SREs。毕竟，这些人一只脚在操作领域，一只脚在开发领域，他们的工作是使分布式软件系统高度可伸缩和可靠。能够将关于这些系统的海量数据转化为可操作的见解的解决方案应该是 SREs 所需要的。事实上，53%的 sre 认为缺乏整个体系的统一可见性是他们最大的云应用程序监控挑战。

然而，如果我们用利用率来衡量，今天的 AIOps 工具似乎并没有兑现这一承诺。当被问及 AIOps 的价值时，平均只有 7.5%的 sre 认为 AIOps“价值高”，甚至低于那些认为其价值低的 sre(9%)。

为什么 AIOps 这么慢才流行起来？最终，这些工具面临的障碍与人类工程师面临的障碍是一样的:IT 环境中巨大且不断增长的复杂性。随着数字产品越来越依赖于第三方云服务，随着企业想要跟踪的事物数量的增长(从基础架构到应用程序到体验)，监控数据的数量、速度和种类都呈爆炸式增长。

## 多样化的数据来源

使问题更加复杂的是，企业越来越依赖多个“相同服务”提供商提供 IT 服务。也就是说，他们使用多个云提供商、多个 DNS 提供商、多个 API 提供商等。这样做有很好的商业理由，比如增加弹性和利用不同供应商在不同领域的优势。但是，即使两个提供者在做基本相同的事情，他们也使用不同的接口和工具，并且他们的数据源通常使用不同的度量、数据结构和分类法。

无论你是要求人类还是人工智能驱动的工具来解决这个问题，这种异构性都使得可视化整个基础设施的完整画面变得极其困难。这也造成了如何最好地利用每个供应商不同的规则和工具集的灰色区域。所有这些都使得自动化除了最表面的操作任务之外的任何事情都非常具有挑战性，如果有可能的话。

## 优化 AIOps

AIOps 确实是这个问题的解决方案——或者至少是解决方案的重要部分。毕竟，机器学习是专门用来分析大量观察和参与数据并实时做出反应，以及不断识别改进机会的。但是在我们到达那里之前，我们需要改变我们对 AIOps 的想法。

不要试图同时做所有的事情。

如果我们能够部署 AIOps，让它了解我们的环境，并看着它神奇地开始自行修复问题，那就太好了。但我们还没到那一步。(如果供应商试图让你相信我们是，你一定要持怀疑态度。)然而，如果我们把期望分解成更小、更易管理的部分，AIOps 可以完成很多事情。

例如，事件关联、拓扑分析和智能警报都是 AIOps 目前可以提供重要价值的领域。对于这些更有限的目标，AIOps 绝对可以减少人类操作员繁琐的手动工作，减少平均维修时间，提高可见性等，数字企业*应该*依靠 AIOps 来做到这一点。

**投资 AI 和 ML 培训。**

AIOps 工具的采用仍然缓慢的部分原因源于这样一个事实，即许多团队仍然没有完全理解如何使用它们。sre 接受 AI/ML 培训的次数越多，他们就越有机会发现具体的、有意义的 AIOps 应用。例如，只有 27%的 sre 说，“试验或接受培训以扩展知识或技能”是他们工作的一部分。

**拥抱“平台运营”**

我们可以采取的最大步骤是重新思考我们如何在组织内使用 AIOps。今天，AIOps 工具——就像 SRE 资源本身一样——通常专用于特定的数字产品或服务。因此，企业最终会尝试应用他们的 AIOps 工具以及使用这些工具的 sre 来支持与该产品相关的一切(DNS、API、CDN、云监控等)。然而，随着 IT 环境变得越来越复杂，企业可能会发现转移到共享平台工具和团队会产生更好的结果。

不要试图将 AIOps 应用于数字服务的每一部分，想象一下将它应用于该服务的一部分(例如，只关注 DNS，或者只关注 API)。通过这种方式，AIOps 工具以及使用它们的团队获得了跨所有不同接口和用于该功能的工具的深入专业知识，并在跨这些接口和工具标准化多提供商数据源方面不断取得进步。在这一点上，这种特定于平台的 AIOps 功能可以在整个公司范围内扩展和重用，从而提供更高级的 DNS AIOps(或 API，或 CDN 等)。)针对多个数码产品和团队。

现在，您已经有了一个高级的、高度专业化的工具集来优化一个重要的平台功能，无论哪个数字产品碰巧在使用它。您缩小了使用给定 AIOps 实现的范围，同时比更通用、更广泛的实现更深入地研究了该功能。突然间，更先进的自动化和自我修复成为可能。AIOps 最崇高的营销主张开始看起来不像是炒作，而更像是您业务的可行现实结果。
