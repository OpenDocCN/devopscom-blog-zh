# Datadog 宣布开放遥测协议支持

> 原文：<https://devops.com/datadog-announces-opentelemetry-protocol-support/>

*Datadog 通过在 Datadog 代理中启用本机 OpenTelemetry 协议摄取来扩展其对 OpenTelemetry 的现有支持*

**纽约州纽约市，2022 年 5 月 16 日—**[云应用监控和安全平台 Datadog](https://www.datadoghq.com) ，Inc. (NASDAQ: DDOG)今天宣布在 Datadog 代理中全面提供 OpenTelemetry Protocol (OTLP)支持。这一新功能为 OpenTelemetry 仪表化应用程序带来了 Datadog 平台的全部监控功能，无需安装单独的 OpenTelemetry 收集器。

OpenTelemetry 是云本地计算基金会(CNCF)的一项倡议，它为检测服务和应用程序提供开放的、厂商中立的标准和工具。许多组织使用 OpenTelemetry 的 API、SDK 和工具集合来从他们的环境中收集可观测性数据并将其导出到他们首选的后端系统。

Datadog 代理现在提供了对 OTLP 的本机支持，使得从使用 OpenTelemetry 库的应用程序接收指标和跟踪变得容易。此外，由于 Datadog 代理还可以收集其他遥测数据，如来自 500 多个集成的应用配置文件、网络数据和基础设施指标，Datadog 客户现在可以访问其 OTLP 跟踪的更丰富的上下文，并对其系统和应用有更深入的了解。Datadog 客户还可以将 OpenTelemetry 跟踪与日志连接起来，以获得其堆栈的更完整的图片。

Datadog 产品和社区高级副总裁 Ilan Rabinovitch 表示:“OpenTelemetry 为应用程序开发人员、系统可靠性工程师和中间件提供商提供了一致的 API 和有线协议，以独立于供应商的方式检测应用程序并收集指标和跟踪信息。“在这个版本中，我们增加了另一个选项，使客户能够将 OpenTelemetry 与 Datadog 一起使用，而不必经历改变其应用的仪器的耗时练习。”

有关 Datadog 对 OpenTelemetry 支持的更多信息，请访问:[https://www . data dog HQ . com/blog/ingest-open telemetry-traces-metrics-with-data dog-agent](https://www.datadoghq.com/blog/ingest-opentelemetry-traces-metrics-with-datadog-agent)。
