# 可靠的云平台需要自动化

> 原文：<https://devops.com/automation-is-needed-for-reliable-cloud-platforms/>

在不到五年的时间里从六个微服务增加到 1400 个？乍一看，这听起来非常复杂。您真的能看到跨所有微服务的整个系统吗？这是高度可用和可靠的 SaaS 云平台的关键。但是这意味着什么，SaaS 提供商如何实现它？

在过去十年中，云已经成为大多数公司业务流程和通信发生的虚拟空间。仅考虑去年加速的数字化进程，[云计算](https://devops.com/?s=cloud+computing)在提高分布式团队的工作效率方面发挥着越来越重要的作用。由于这些发展，应用程序、微服务和接口的数量不断增加，形成了更加复杂的 IT 基础设施。

因此，云服务的可靠性和可用性必须是提供商的重中之重。薄弱的服务和停机时间不仅会让客户感到沮丧，还会给客户带来巨大的经济损失。因此，SaaS 提供商必须建立流程和方法来帮助他们提供安全、高度可靠的云基础设施，否则将面临失去客户的风险。

## 微服务是可靠性的基础

每一个 SaaS 云平台都隐含着复杂性。这可以通过两种不同的方式来管理。一方面，管理可以通过一个集中的整体来完成。这样一来，只有一个外部系统接口的单一系统就变得复杂了。另一方面，复杂性可以通过微服务形式的较小系统单元来分布。这意味着应用程序或服务的各种功能被分解并分配给几个相互通信的独立实例。这样做的好处是，如果一个微服务出现故障，整个应用程序不一定会受到影响。

一段时间以来，云环境中的微服务越来越受欢迎。事实上，作为 2019 年 Forrester 研究的一部分，76%的受访公司表示，微服务将在未来成为许多公司的重要业务议程。当时只有 4%的人不同意这种说法。2020 年， [TechRepublic 的一项调查](https://www.zdnet.com/article/survey-microservices-worth-the-hype/)发现，大多数受访专家(73%)已经将微服务集成到他们的应用流程中，66%的受访者以云服务的形式使用微服务。微服务无处不在！

## 服务级别目标和自动化

由于微服务会产生大量接口，并且需要进行监控，这种架构乍一看似乎比传统的单片系统更容易出错。一些公司已经依靠服务水平目标(SLO)——即确保服务质量的目标和可衡量的 KPIs 来实现更高级别的安全性。然而，这些通常仅限于整个系统的外层。

为每个微服务制定适当的 SLO 并添加额外的警报可以抵消这一点。结合起来，这些为系统提供了针对爬行误差的深度保护。

为了保持关于安全和安全相关事件的准确概述，从而确保系统的可靠性，必须密切监控微服务及其各自的 SLO 并持续测试。当我们在 2019 年调查软件和 IT 专业人员时，我们发现 71%的受访者报告说基于[微服务的架构比单片测试](https://www.atlassian.com/blog/software-teams/modern-software-development-trends)容易得多。在自动化的帮助下，运营团队可以确保系统的每一个实例的整体可靠性水平接近 100%，无论多么复杂。一个独立的集中式自动化工具监控所有正在使用的微服务，并能够向适当的团队发出有关违反单个 SLO 的任何异常活动的警报。此外，该工具应链接到事件管理平台，以便可以直接补救任何违规行为。通过遵循这一框架，软件开发人员将享受到更易于管理和扩展的微服务系统的好处。

我们生活在一个消费者期望技术定期更新和可靠运行的时代。如果不是这样，他们就会选择更好的替代方案。为了防止这种情况，公司需要投资于更有效的服务结构。微服务与自动化测试和管理的中央平台相结合，是实现可靠性和可扩展性的基础。
