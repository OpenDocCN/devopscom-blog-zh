# Catchpoint SRE 公司的研究揭示了全球工作量的下降，警告了即将到来的可扩展性上限，并强调了对新运营能力的需求

**纽约州纽约市，2021 年 6 月 15 日—** [数字化体验管理的领导者 Catchpoint](https://www.globenewswire.com/Tracker?data=KuVSFHgl1H_P_B7wTWApQPiawBfvGjau34s-21Cv1_VPAvhP0i2mfIZmof7s94OUBGk7iKM5Ql_iseMY4Qn5sg== "Catchpoint") 与 VMware Tanzu 和 DevOps Institute 对近 300 名现场可靠性工程师(sre)进行了一项研究。SRE 报告是同类研究中最有数据支持的研究之一，自四年前发布以来，在定义 SRE 的本质方面发挥了关键作用。今年的报告强调了多重云的挑战，指出了 AIOps 的利用不足，并显示了核心基准数据的系统性转变。该报告最后为 sre 提供了一条可操作的路径，以始终如一地为客户提供价值。

Catchpoint 首席执行官 Mehdi Daoudi 表示:“SREs 应对跨越转型和运营活动的广泛挑战。“这份报告为他们提供了帮助应对这些挑战所需的洞察力——在构建和运行大型、分布式和可靠的系统时，平衡敏捷性需求和稳定性需求。”

**世界各地的辛苦程度有所下降**

辛劳是与生产服务相关的工作，往往是手工的、重复的、自动化的，缺乏持久的价值。Google 建议软件工程师不要做超过 50%的运营工作(包括辛劳)和 50%的开发工作。今年，SRE 的报告指出，劳动的年平均降幅为 15%。

Catchpoint 技术运营副总裁托尼·费瑞里(Tony Ferrelli)表示:“这是一个如此有影响力的见解，因为所有地区的工作量都在下降。”。“如果这种辛苦程度的下降是因为自新冠肺炎导致在家工作以来，工作变得更有意义，那么随着人们回到办公室或混合工作环境，明年报告的辛苦程度会上升吗？”

多个提供商的加速使用警告了即将到来的可扩展性上限

如果云是您的新数据中心，那么 DNS 和 CDN 等第三方服务就是您的新机架和机柜。当将多个相同服务平台(例如，多云)的使用增加与监控数据的数量、速度和种类的增加相结合时，难怪缺乏跨堆栈的可见性(53%)是最常提到的云应用监控挑战，或者为什么 sre 不断完善服务级别目标(50%)。调查反馈提出了一个关键问题，即公司如何最有效地扩展 SRE 实施？

“跨越每个提供商提供的接口和数据之间的鸿沟增加了 SRE 团队在多个提供商之间实现自动化的难度。除了最表面的方面，这些集成很少是简单的。SRE 建筑师 Kurt Andersen 说:“在多供应商环境中，有效地将不同的数据模型映射在一起可能是 SRE 的下一个前沿。”。

**向 AIOps 的转变缓慢**

AIOps 被广泛吹捧为可以减少繁重的运营工作，并智能地筛选组织不断面临的不断增长的数据量。然而，该报告显示，许多 sre 从未使用过 AIOps，他们对其接收价值的评级均匀地跨越了 1-9 的值范围。

根据 Twitter SRE 员工 J. Bobby Dorlus 的说法，“大多数这种规模的 SRE 已经在利用机器学习，特别是在数据中心(位置、冷却和 it 内部发生的所有事情)的网络和基础设施建设效率方面……下一个合乎逻辑的步骤是将其发展为 AIOps。”

**可观察性应包括数字体验指标和业务 KPI**

未能交付客户价值的 sre 有陷入运营困境的风险。相反，没有认识到 SRE 活动重要性的企业面临着失去优秀员工和竞争优势的风险。

成功实施 SRE 的最大驱动因素是事件解决(60%)，而扩展业务排在倒数第五位(33%)。这些发现表明，sre 仍然向内关注 IT 运营，而向外关注交付客户价值的业务成果。为了缩小这种 IT 到业务的差距，SRE 团队必须扩大可观察性的边界，以包括数字体验指标和业务 KPI。

DevOps Institute 首席研究和内容官 Eveline Oehrlich 表示:“创新与提供卓越运营之间的平衡工作迫使许多 IT 团队非常重视提高服务和应用的可靠性和稳定性。“SREs 现在需要做的是确保客户理解这些可靠服务和应用的价值。”

**建议**

*   业务和 SRE 需要围绕核心 SRE 原则和业务水平指标建立基线计划，以了解情况是变好了还是变坏了。
*   应组建平台运营团队，以实现更高水平的规模和效率。平台运营部门应开发标准化的功能，供整个组织的 sre 利用(即使底层平台具有不同的接口)，并将这些功能视为向企业内其他团队销售和营销的产品。
*   为了实现 AIOps 的承诺，SRE 和经理必须将 AIOps 分解成更小的组件，并在此基础上进行增量开发，此外还要为 SRE 团队投资 AI 和 ML 培训。
*   找到弥合 SRE 和业务目标之间差距的方法至关重要。例如，围绕能力展开对话，而不是关注低层次的监控指标和高层次的业务成果。
