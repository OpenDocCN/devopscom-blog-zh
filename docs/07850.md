# Applitools 与 Rally 集成，用于快速和自动化的 Bug 管理

> 原文:[https://devo PS . com/appli tools-integrated-with-rally-for-fast-and-automated-bug-management/](https://devops.com/applitools-integrates-with-rally-for-fast-and-automated-bug-management/)

*开发人员和 QA 工程师通过本机 Rally 集成直接在 Applitools 中高效记录 bug 细节，从而全面了解 appdev 生命周期流程*

加利福尼亚州圣马特奥，2021 年 5 月 11 日——由视觉人工智能和超快测试云提供支持的下一代测试自动化平台提供商 Applitools 今天宣布了一项新的 Rally 集成，作为 [Eyes 10.11 版本](https://info.applitools.com/udlWA)的一部分。Rally 是一个流行的应用程序生命周期管理平台，用于跟踪敏捷软件开发的每个阶段，从计划到缺陷跟踪。这种集成允许开发人员和 QA 工程师在单个工作流中轻松记录和管理收集的所有关于 bug 的数据，以及如何修复它们。

**在这篇博文中了解更多拉力赛积分:(**[**【https://info.applitools.com/udlWg】**](https://info.applitools.com/udlWg)**)。**

Applitools Eyes 视觉 AI 技术允许客户自动测试他们的应用程序，并识别视觉和功能错误。一旦发现问题，客户通常会将其记录在一个 bug 跟踪工具中，如 Rally。以前，客户必须在 Eyes 和 Rally 之间切换，以手动记录、管理和修复问题。随着 Rally 的集成，Eyes 收集的所有关于 bug 的数据都与 Rally 的 bug 跟踪系统无缝链接。与开发人员和 QA 工程师分别管理 Applitools 和 Rally 中的 bug 不同，他们只需点击一下鼠标，就可以在一个地方记录所有信息，包括截图、重现步骤和环境细节。

Applitools 的首席运营官 Moshe Milman 表示:“确保我们与客户现有的工具和流程无缝集成一直是 Applitools 的首要任务与 Rally 整合是很自然的下一步。我们知道，对于我们的客户来说，能够在一个地方记录和跟踪有关其应用程序生命周期的所有信息是多么重要，这样他们就可以分配适当的移交，并有效地结束反馈循环。"

通过这种集成，使用 Rally 和 Applitools 的客户现在可以全方位了解整个应用程序开发生命周期，从 UX 设计和开发到执行自动化测试、分析测试结果、报告和修复缺陷。appdev 流程中的任何利益相关者，从产品经理到 UX 设计师、开发人员和测试人员，都可以使用 Applitools [根本原因分析](https://info.applitools.com/udlWB)功能自动查明问题的确切原因，并能够当场修复缺陷。

IHG 全球质量工程总监 Tammie Davis 表示:“我们依赖 Applitools 等工具来实现快速高效的测试自动化和视觉质量，它们是我们自动化工具箱中不可或缺的一部分。“随着新发布的 Rally 集成，团队将拥有一个无缝的缺陷分类工作流，并优化整体用户体验。”

Rally integration 是 Applitools 承诺扩展其集成生态系统的延续，它提供了 70 多个 SDK 和集成，可以无缝融入客户现有的工作流程和工具。
