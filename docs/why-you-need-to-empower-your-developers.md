# 为什么你需要授权给你的开发者

> 原文：<https://devops.com/why-you-need-to-empower-your-developers/>

近年来，全球对软件开发人员的需求不断增加。然而，2020 年，当疫情来袭时，许多首席信息官大幅削减了他们的 it 支出。这一趋势已经迅速逆转。2022 年，全球 IT 支出预计将达到 4.4 万亿美元。此外，根据美国劳工统计局的数据，软件开发行业的就业率预计将上升 22%。

尽管多个行业的组织都在加大对 IT 的投资，但技术人才的供应却难以跟上。随着对创造力的需求，开发人员明白了创造力的重要性，并能更好地规定他们的条款——包括更高的薪酬和更好的福利。不满足这些期望的组织有失去梦寐以求的人才的风险。

对于许多开发人员来说，远程工作仍然是重中之重，还有[伟大的公司文化](https://devops.com/management-and-culture/)。为了给开发人员创造一个积极的工作环境，企业领导人必须提供一种文化，让技术员工在远程工作时有归属感和使命感。许多开发人员还希望有发展的机会，并且能够提供敏捷和创新的工具集来帮助提高生产力和协作。

让我们讨论一些现代组织仍然可以授权给开发人员的方法，以及企业领导人如何建立一种工作场所文化来吸引一些企业中最好的开发人员。

### 提供支持性的数字工作场所

开放源码的使用在工作场所已经显著增加，并继续成为开发人员的一种形成性资源。开源社区提供了许多新技术，开发者寻求甚至需要这些新技术来发挥他们的最大能力。例如，开源提供了一些最流行的编程语言，包括 Javascript、C++和 Python。多种语言的出现使得开发人员更容易高效地编写代码并产生高质量的工作。

开源社区帮助开发人员建立他们的技能，并允许他们定制他们的技术栈，从而允许他们优化他们的生产力，并从其他开源项目中学习不同的技术。毫无疑问，开源对于科技行业和开发者来说是至关重要的。拥抱开源可以提高满意度，并创建一个更好的整体劳动力。

### 激发软件开发人员之间的协作

过去，免费食物和协作社交区等办公室福利被用来吸引新员工和留住现有员工。然而，在远程工作环境中，这些不再是优先事项。相反，企业领导人应该优先考虑组织为员工提供的整体职业体验。特别是对于开发人员来说，建立这种体验的责任落在了首席信息官和 IT 领导者身上，以确保他们提供工作场所满意度、现代技术和灵活的流程。要做到这一点，他们需要专注于促进生产力和沟通的工具。

大多数员工都熟悉许多工作场所提供的工作流软件和协作工具。如果没有这些服务，远程工作人员最糟糕的情况是对自己的工作量感到困惑，最乐观的情况是与同事疏远。然而，对于开发人员来说，这些通用的协作工具不能胜任，因为它们不太适合工程工作流。一般的协作软件不提供深度定制的能力，也不适合在本地或私有云中部署。因此，开发人员很少或根本没有使用这些工具。

这种缺乏实用意识造成了一个重要的不同。开发人员的倦怠达到了前所未有的高度，低效的工作流会让团队无法发挥出最佳水平。或者，更糟糕的是，它会让员工个人感到沮丧，导致更低的保留率和对潜在员工更没有吸引力的产品。希望为开发人员提供改进的工作环境的企业领导应该提供一个更加集成的、面向开发人员的技术栈。

### 授权的软件开发人员是高效的开发人员

虽然市场已经发生了变化，但对于许多开发人员来说，分散的工具仍然是最大的生产力挑战。投资现代工具和编程语言，同时消除技术复杂性，是提高生产力并同时创造更好的开发人员体验的关键。这就是为什么组织必须实现开源应用程序，以便开发人员可以完全了解和控制整个堆栈。

由于工具碎片会导致许多干扰，开发人员可能会变得沮丧，并花费时间来应对每一个单独的事件。这给开发人员带来了失去对工作的热爱的威胁。通过从旧技术中去除碎片并选择相关的解决方案，商业领袖允许开发人员做他们最擅长的事情:开发软件。

当组织领导授权给他们的开发人员时，他们反过来会发现他们的产品团队更有生产力，并且感觉与组织、客户以及彼此之间的联系更加紧密。因此，对开发人员最好的就是对整个组织最好的。
