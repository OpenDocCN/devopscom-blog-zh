# Snapt 宣布推出在公共云中运行 Kubernetes 的单一安全包

> 原文：<https://devops.com/snapt-announces-the-one-security-package-to-run-kubernetes-in-public-cloud/>

*Snapt 赞助了 KubeCon Europe，并为公共云中的 Kubernetes ingress 带来了基于人工智能的威胁情报、WAAP 和 bot 保护——只需一次点击。*

**西班牙巴伦西亚，2022 年 5 月 16 日—** 领先的面向未来、云中立、平台无关的负载平衡和安全解决方案提供商 Snapt 在巴伦西亚的 KubeCon Europe 展示了一个强大的人工智能平台，用于确保 Kubernetes 进入公共云环境。Snapt Nova 是一个集中管理的负载平衡和应用安全平台，专为云本地和混合环境而设计，包括实时威胁情报、web 应用和 API 保护以及 bot 保护。Snapt 将于 5 月 16 日至 20 日在 KubeCon Europe 展会上展示 Snapt Nova，展位为展厅中的 S32 号展位或我们的虚拟展位:https://bit.ly/38VKVwo(链接仅供注册与会者使用)。

KubeCon 的与会者还可以与 Snapt 交谈，请求免费的健康检查，以测试其基础设施的安全性、规模和速度。此优惠仅适用于 KubeCon Europe 2022 的与会者。

Snapt Nova 是应用交付控制即服务，通过集中式控制器和 UI 向每个节点按需提供负载平衡、全局流量路由以及 web 应用和 API 保护。Snapt Nova 的集中式控制平面通过与数百万个数据平面节点的实时低延迟连接，支持软件定义的负载平衡。Snapt Nova 可满足 DevOps、NetOps 和 SecOps 团队的需求，确保 Kubernetes 在公共云环境(包括多种云)中的大规模进入。

“Snapt Nova 使公共云 Kubernetes 投入生产变得轻而易举，”Snapt 联合首席执行官 Dave Blakey 说。“Snapt Nova 确保您的应用程序高度可用，免受拒绝服务等威胁，并针对性能进行了优化。它提供了对 Kubernetes 和其他平台的可见性，甚至在多云环境中也是如此。我们为现代应用团队设计了 Snapt Nova，因此我们使用完整的 REST API、集中式控制面板、单一控制台监控以及与现代工具堆栈的集成来构建它。”

Snapt Nova 简化了云中 Kubernetes 应用程序的安全保护和扩展。它可以与所有公共云和 Kubernetes 轻松集成，包括用于快速配置和部署应用服务的原生上游服务发现。Snapt Nova 的集中式控制器提供了弹性可扩展性，响应需求的延迟极低，因此应用服务始终与应用流量保持同步。Snapt Nova 的 worker 节点是轻量级的、容器化的、快速的，甚至支持在 ARM 架构上的部署。

Snapt Nova 还降低了 Kubernetes 与其他平台和架构一起部署的复杂性，使 NetOps 和 SecOps 团队能够确保其网络每个部分的可靠性和安全性。集中式策略、安全性和全局流量路由有助于组织保持全球网络和混合架构的一致性、简单性和合规性。具有一键式部署、向导和数据可视化的现代 web GUI 使运营商能够快速启动并高效解决问题，而无需以代码形式部署基础设施。

Snapt Nova 为 DevOps 从业者提供了完整的 REST API，用于完全自动化和与 CI/CD 工作流的集成。

基于角色的访问控制允许 NetOps 或 PlatformOps 团队使用 Snapt Nova 的 GUI 设置全局策略，并使用 API 将每个应用程序的控制权委托给多个开发运维团队。这种能力使应用程序团队能够快速、自动地开发、测试和启动应用程序，同时使网络运营团队能够设置护栏并监控整体网络健康、安全和性能。

Snapt Nova 包括实时威胁情报，因此 Kubernetes 部署受益于应用程序安全性，该安全性始终与最新出现的威胁和趋势保持同步，允许自动先发制人地阻止当前和潜在的威胁。Snapt Nova 的威胁情报由高级机器学习和来自 Snapt 全球威胁传感器网络的独特实时真实威胁数据提供支持，可识别、跟踪和阻止僵尸网络、恶意软件、网络犯罪和平台滥用。

Snapt Nova 包括强大的第 7 层 Web 应用程序和 API 保护，可阻止黑客、拒绝服务(DoS)、僵尸程序、数据泄漏、SQL 注入、XSS、CSRF 等，同时确保您的 Kubernetes 入口合规、安全。Mutual TLS 支持保护敏感数据的端到端加密。API 网关实施 API 访问控制和 SLA，并防范 OWASP 十大 API 漏洞。Snapt Nova 的应用安全性通过机器学习和人工智能支持的自主响应得到增强，实现了复杂的流量分析、异常检测、持续的自我学习以及在检测到威胁时的闪电般反应。

Snapt Nova 实时自动检测和阻止恶意僵尸程序。它结合了节点上的活动识别，集中的人工智能驱动的行为分析，以及已知机器人的全球数据库。Snapt Nova 的分布式应用交付控制器(ADC)使用强大的活动识别引擎来识别可疑的浏览模式、非人类输入和来自可疑来源的流量。

分布式 ADC 还可以提交一个可疑用户，由机器人 AI 进行集中评估。Snapt Nova 使用机器学习来开发正常和自动化浏览行为的深度配置文件，可以一致地识别其他系统失败的机器人。这种自我学习系统不断进化，提高其速度和智能。

对于准备了解更多有关保护和简化 Kubernetes 入口的用户，Snapt 将于 5 月 16 日至 20 日在 KubeCon Europe 的 S32 展位上展示 Snapt Nova。欲了解更多信息，请下载 [Snapt Nova 和 Kubernetes 数据手册](https://www.snapt.net/content-hub/snapt-nova-kubernetes-datasheet)。
