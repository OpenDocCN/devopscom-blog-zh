# 避免云迁移期间的预算超支

> 原文：<https://devops.com/avoiding-budget-overruns-during-cloud-migration/>

许多组织已经开始向云迁移，或者计划很快迁移。这是有道理的:由于多种原因，云是一个有吸引力的选择，包括其成本节约的潜力、可扩展性和对持续应用程序开发的支持。

然而，除了好处之外，企业很容易失去对云迁移的财务控制。在迁移到云时，潜在的代价高昂的错误是巨大的，组织经常最终受到令人惊讶的账单的打击，最终停止他们的脚步。事实上，[在 2020 年的一项调查](https://cdn2.hubspot.net/hubfs/1624046/2020%20Cloud%20Computing%20executive%20summary_v2.pdf?__hstc=164992608.c9073e6fef8e5767d0c95f899e530d29.1628011616454.1628011616454.1628011616454.1&__hssc=164992608.2.1628011616454&__hsfp=3641277094&hsCtaTracking=ab53c411-75c2-445f-87fc-c00c684da70c%7C91996eca-14bf-4e5d-923d-6bbfeccf1994)中，94%的参与者声称充分利用公共云资源对他们来说是一项挑战，40%的人认为需要[控制云成本](https://devops.com/?s=cloud+costs)是他们最大的障碍。

这些财务错误通常是由于云服务的按使用付费模式和不断增长的云技能差距造成的。IT 团队成员经常被迫在他们不熟悉的云环境中工作，而且随着每周都有新功能发布，这些人几乎不可能跟上。最近的一项研究表明，90%的企业发现他们的云技能差距在三年内翻了一番。

## 两种不同类型的云迁移

通常会发生两种类型的云迁移，即提升和转移，或者平台/重新架构迁移。提升和转移方法快速且相对简单，但它无法让组织真正优化其云使用。重新架构方法是最受欢迎的，因为它让组织有机会通过利用云原生技术和服务的优势来解决任何突出的问题。

但是，一旦重新架构迁移开始，组织通常会发现完成迁移所需的时间、精力和资源比他们最初想象的要多得多。这可能会成为一个真正的挑战，特别是当面临关键的最后期限或经历重大事件，如新冠肺炎疫情或全球各地经历的恶劣天气事件。

## 了解和评估绩效

对于这两种类型的云迁移，理解和评估性能和底层架构是一项持续的任务，因为云提供商总是会发布新的或改进的服务或降低价格。为了确保实现最佳性能并构建弹性架构，应用程序开发人员和 IT 团队必须不断寻找改进应用程序的方法。在云中，这通常集中在四个关键领域:网络、计算、存储和应用程序架构。

从网络的角度来看，了解数据在哪里被使用以及数据在哪里被访问至关重要，可以帮助组织大幅削减成本。例如，云提供商通常有昂贵的数据进出费用，因此在必要时评估如何最好地访问和使用数据可以帮助公司应对这些费用。

关于计算，有多种资源可以帮助管理系统负载，对计算基础架构进行调整，并找到性能和成本优势。例如，利用来自原生云报告工具的数据和指标可以帮助组织了解和预测需求，利用自动扩展等服务使他们能够根据应用程序使用和需求的变化适当地扩大或缩小规模。

在存储方面，云提供商提供了许多节省资金和提高性能的方法。从发布 [GP3 卷](https://aws.amazon.com/blogs/aws/new-amazon-ebs-gp3-volume-lets-you-provision-performance-separate-from-capacity-and-offers-20-lower-price/)到基于访问频率的分层长期存储选项，存储是发现云节约机会的简单方法。此外，云提供商通常会提出建议并设置分层策略，以帮助将数据自动移动到更冷、更深的存储中。

随着云提供商不断增加新的服务，审视您的应用架构对于创新和成本节约至关重要，尤其是在包括劳动力和精力的情况下，并真正实现云的承诺。

## 解决常见的代价高昂的错误

虽然有许多[公司在管理云实例时犯的常见财务错误](https://cloudacademy.com/blog/10-common-aws-mistakes-how-to-avoid-them/)的例子，但缺乏可见性和理解是站出来维护其组织架构的开发人员的主要痛点。由于对云的需求如此之高，而且他们的注意力被转移到了别处，开发人员和工程师通常会抱着“如果它没坏，就不要去修复它”的心态。这阻碍了公司评估和优化云运营的能力。他们没有时间持续审查他们的申请，以寻找省钱的方法，甚至不知道他们的决定要花多少钱。

这一普遍存在的问题可以通过提高云迁移和管理流程的可见性以及将实时财务直接与既定的企业预算保持一致来解决。此外，在云服务的按使用付费模式中苦苦挣扎的组织应该采取必要的措施(无论是内部还是第三方供应商)来确保云资源和服务的使用得到有效监控。这使得开发人员在他们的老板来敲门之前，更容易理解他们的行为在哪里以及如何影响组织的财务状况。

如果企业为 IT 团队提供必要的资源，让他们更加了解如何在云中使用他们的应用程序，并解决员工的云技能差距，他们将能够防止成本超支，节省成本，并重新控制预算。这可以采取多种形式，但企业需要建立持续的培训计划，以确保团队成员了解最新的云功能和服务，并利用工具和自动化来减少手动干预。在整个云迁移过程中提供这种关键可见性通常意味着成功的预算内迁移与停滞不前且成本高昂的迁移之间的差异。
