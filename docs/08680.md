# DevOps 职业培训和发展的可扩展途径

> 原文:[https://devo PS . com/scalable-path-devo PS-career-training-development/](https://devops.com/scalable-path-devops-career-training-development/)

“我们公司是一个高端人才市场，拥有按需领导能力，客户主要在湾区、硅谷、纽约和其他大都市地区，”由一千多名软件架构师和开发人员组成的地理分布网络, [Scalable Path](https://www.scalablepath.com/) 的合伙人托马斯·古铁雷斯(Tomas Gutierrez)说。Scalable Path 为科技初创公司和产品公司等客户提供服务，帮助他们建立软件团队并填补技术空白。鉴于 Scalable Path 有助于软件设计，其客户肯定需要 DevOps 方法。Scalable Path 的 Gutierrez 向 DevOps.com 介绍了 DevOps 的职业培训和发展重点以及构建模块。

基础知识

在 Scalable Path，对于希望使用 DevOps 方法的软件工程师来说，对[开放系统互连模型](http://www.webopedia.com/quick_ref/OSI_Layers.asp)的[七层](https://support.microsoft.com/en-us/kb/103884)的良好工作知识非常重要。Gutierrez 说:“为了对在哪里发生的事情有一个完整的心理图像，软件工程师必须具有跨越所有这些层的对话知识。”。

对于任何软件应用程序，你将把它放在硬件上，你将有一个操作系统，主机将通过网络相互通信。Gutierrez 解释说:“DevOps 工程师必须了解所涉及的七个层次，以便将基础设施的目标视为代码，并帮助发布软件、部署基础设施、联网和配置。

适用特性

Gutierrez 说，在工作中，一个好的 DevOps 工程师必须是一个好的问题解决者，能够跳出框框思考，对软件产品和组织的需求有同理心。对细节的关注和对安全的关注也是值得考虑的。

古铁雷斯说:“尽管接受正规培训、获得大学学位和证书是件好事，但良好的人格，即自学成才的人格，是优秀开发人才的关键。”。这些工程师将在全球范围内检查技术挑战及其组成部分，并寻找和组装工具和流程，以提供完整的解决方案。

此外，对软件和部署软件的企业的需求感同身受的人会意识到他们何时需要使用一个简单的、成本有效的软件部署，而不需要所有额外的东西。

对 DevOps 工程师来说，将注意力集中在软件工程细节上的能力是另一个重要的特质。否则，会出现编程错误，可能会造成问题。Gutierrez 说:“如果你没有正确地构建一些东西，你可能会因为在编程中犯了一些错误而导致你的部分架构失败。随着体系结构的增长，这些事情会传播并扩大成更大的问题。

对于 DevOps 工程师来说，增强安全性也很重要，这样他们的编程就不会暴露关键数据。Gutierrez 说，这是关于程序员如何构造数据层，他们如何与传输层一起工作，以及数据的端到端路由是什么。

证书

尽管强调的是工作中的特点和应对挑战的方法，但对 DevOps 工程师来说，还是有一些认证是有价值的。“例如，思科认证，尤其是网络认证将会很有帮助，因为你在 DevOps 中做的很多事情都是关于传输中的数据的，”Gutierrez 说。Gutierrez 解释说，DevOps 工程师需要注意流量是如何在互联网上移动以及如何进出数据中心的。Gutierrez 补充说，网络配置和网络安全专业知识，尤其是通过高级思科认证的专业知识，对于 DevOps 工程师来说非常重要。

应用安全认证也很有用，如厨师、木偶和码头工人的供应商认证和课程，以及 AWS、[数字海洋](https://www.digitalocean.com/)等的平台供应商培训。

个性化学习

不同工具(如容器和配置管理、自动化和部署工具)的用户文档和“试用”机会对自发者和遇到问题或有疑问的人很有帮助。Gutierrez 说:“从事这类培训的人可以简单地在本地机器上为这些工具创建本地实验室，以模拟他们试图学习的更大的应用程序或架构。

为了学习如何在系统和应用程序架构之间的深度领域导航，DevOps 学生应该考虑这些主题的书籍。仅举几个例子，包括，“DevOps:一个软件架构师的视角(软件工程中的 SEI 系列)”，“有效的 DevOps:大规模构建协作、亲和和工具化的文化”，以及“Salt Essentials”。
