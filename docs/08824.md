# Akamai 将 Web 应用程序和 API 安全性结合在一起

> 原文:[https://devo PS . com/akamai-brings-web-application-and-API-security-together/](https://devops.com/akamai-brings-web-application-and-api-security-together/)

Akamai Technologies，Inc .本周推出了一项服务，该服务整合了保护 web 应用程序和应用程序编程接口(API)的过程。

Akamai 产品管理和战略副总裁 Amol Mathur 表示，随着 [App & API Protector](https://www.akamai.com/newsroom/press-release/akamai-redefines-waap-simplicity-and-automation-with-new-app-and-api-protector) 的推出，托管安全服务提供商正在让 it 团队更容易通过融合的 web 应用和 API (WAAP)服务来采用 DevSecOps 最佳实践，而不是使用单独的工具和服务来单独保护它们。

该服务还被设计为持续发现 [API](https://devops.com/?s=API) 请求，然后自动检查恶意代码。可以根据内部 IT 团队定义的策略实施可选的 API 安全控制。

此外，该公司还提供了一个名为 Adaptive 的多维威胁评分模型，该模型将 Akamai 收集的数据与来自每个 web 和 API 请求的数据和元数据相结合，以帮助组织更好地了解哪些威胁需要立即关注。我们还使用机器学习算法持续分析安全事件，以提供高度准确的逐个策略调整建议，只需点击一下鼠标即可实施。

Akamai 安全研究人员还利用机器学习和数据挖掘技术来持续分析超过 303TB 的日常攻击数据，这些数据用于自动更新防护。或者，组织可以选择自己手动评估这些数据，以最大限度地减少这些更新可能对其应用程序产生的任何意外影响。

其他功能包括内置的 bot 缓解工具，这些工具由超过 1，500 个已知 bot 的目录提供信息，与 Akamai 命令行界面(CLI)、Terraform 或用于驱动跨持续集成/持续交付(CI/CD)平台的自动化管道的脚本集成。

Akamai 最近发布的一份报告发现，从 2020 年 1 月到 2021 年 6 月，在 18 个月的时间里，共有超过 110 亿次攻击企图。最常见的攻击媒介是 SQL 注入(SQLi)，攻击次数为 60 亿次，其次是本地文件包含(LFI)，攻击次数为 33 亿次，跨站点脚本(XSS)，攻击次数为 10.19 亿次。

该报告还发现，凭据填充攻击达到了 10 亿次以上，并在 2021 年 1 月至 2021 年 5 月期间达到峰值。分布式拒绝服务(DDoS)攻击在 2021 年 1 月达到了 90 次的峰值。

Akamai 一直在为通过其创建的内容交付网络(CDN)外包 web 应用程序和 API 安全管理提供理由。它的 CDN 在面向 web 的应用程序和企业的其他部分之间提供了一个隔离层。组织现在经常面临的挑战是，随着他们开始部署基于微服务的云原生应用程序，安全责任变得更加分散。Mathur 说，Akamai 提供的安全服务创造了一个机会，可以通过单一平台更好地管理应用程序安全，此时人们比以往任何时候都更加关注保护软件供应链。

目前还不清楚组织将在多大程度上更多地依赖托管安全服务，作为采用 DevSecOps 最佳实践的更大努力的一部分。然而，在应用程序安全专业知识难以获得的时代，越来越明显的是，在许多情况下，依靠外部服务提供商来保护应用程序环境是快速改善任何组织的整体安全状况的最简单的方法。
