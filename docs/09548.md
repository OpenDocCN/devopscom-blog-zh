# 将自动化应用于开发运维

> 原文：<https://devops.com/applying-automation-to-devops/>

在《[生活与工程艺术，](https://twitter.com/advancedenguk/status/953219532046438401)》一书中，作者 Haresh Sippy 说，“自动化是通过收紧角落来削减成本，而不是削减成本。”如今，企业和组织一直在寻找提高生产率的方法，同时降低运营中的低效率。自动化已经成为自然的答案，因为它解决了这些问题，同时使组织能够扩展其运营。

这与 DevOps 非常契合，devo PS 旨在自动化、简化并加速从 [代码生成](https://securiti.ai/developers/) 到应用性能监控的开发流程。自动化有望带来更好的结果、更高的效率以及减少或消除错误，但许多组织仍然不愿意在关键运营领域部署它。

但是开发运维自动化有许多组织不应忽视的优势:

*   一致性:devo PS 自动化最重要的好处是一致性。由于自动化工具以同样的方式重复过程，不需要人工就可以消除错误，从而确保开发环境始终保持高质量。

*   **可伸缩性:** DevOps 自动化不仅允许快速发布新代码，还允许大量发布新代码。自动化还允许团队在多个环境中同时交付多个项目，减少了手动过程。

*   **速度:**devo PS 自动化带来的第一个明显优势是产品交付流程的绝对速度。DevOps automation 简化了配置流程、开发环境和部署新软件的其他方面，大大加快了流程。有了 DevOps 自动化， [软件交付](https://devops.com/the-benefits-of-software-delivery-management/) 变得不那么依赖人了。

*   灵活性:灵活性常常是自动化中被忽视的一个方面。然而，借助 DevOps automation，组织可以根据新的需求更改和修改关键代码，并实时部署这些更改，而不必培训团队根据新的更改来管理运营。

## 关键开发运维自动化流程

一旦一个组织意识到了这些好处，很明显下一步就是找出哪些流程要自动化。

虽然 DevOps 的核心原则是将逻辑上可能的一切自动化，但这是一个长期目标。在更直接的基础上，组织最好将注意力集中在一些关键的标准流程的自动化上。

这将允许更平稳的过渡以及更有效的结果。但是，自动化哪些确切的流程是一个主观决定，取决于每个组织的独特需求和技术要求。然而，任何组织都可能从 DevOps 自动化中受益的一些基本领域包括:

*   **软件测试:**任何提供数字服务或产品的组织都会在推出新产品或更新之前花费大量的时间和人力资源来测试它们。自然地，自动化测试过程会给测试结果带来更高的效率，同时显著减少任何错误，尤其是人为错误。可以自动化的附加测试包括单元测试、冒烟测试和 UI 测试。

*   **监控:**大多数组织都明白采取积极主动的监控方法有多重要。DevOps 的一个关键方面是始终如一地读取、记录和维护来自定期监控的关键数据。通过自动化这一流程，组织可以进一步提高宝贵的 [B2B 数据](https://coresignal.com/blog/b2b-data/) 和洞察力的提取，同时确保更高的安全性，防止可能的违规和数据窃取。

*   **基础设施:**基础设施管理是一个关键领域，在确保一切顺利运行方面发挥着至关重要的作用。这包括持续需要维护、配置修改和初始设置的网络或服务器。通过部署自动化基础架构，组织可以提高效率，同时释放宝贵的人力资源以部署到其他地方。

*   **持续集成/持续交付(CI/CD):** CI/CD 的全部目的是快速应用程序开发和交付。就其本质而言，这项任务自动化的时机已经成熟，因为它需要 [测试和监控](https://webinars.devops.com/testing-and-monitoring-and-broken-things) 每一个微小的变化或修改。  同样，自动化 CI/CD 的部署阶段将确保成功更新的分发在准备就绪的第一秒就主动执行。因此，组织将看到更新更有效地推出，最终更新中的错误或缺陷最少。
