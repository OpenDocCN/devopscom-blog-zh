跟上信息是一项挑战。面对所有这些变化，您可能会疑惑，“谁能指导我了解所有这些信息？”幸运的是，[Kubernetes](https://www.mediaopsevents.com/cloudnative2021?utm_campaign=Cloud%20Native%20Con&utm_source=email&utm_medium=invite&utm_content=invite)的云原生日致力于为与会者提供精通云原生技术所需的工具。在虚拟峰会上，与会者可以 收听我们由专家主导的专题会议，并 了解如何利用云原生技术实现更快、更安全的发展。

随着企业不断加快开发步伐以满足业务需求，本地技术正迅速成为 IT 环境不可或缺的一部分。正因为如此，向改变我们所知的 it 世界的行业专家学习是非常有价值的。除了云原生领域的行业思想领袖和实干家，还有谁更值得我们学习呢？通过涉及安全性和可观察性等主题的会议，我们的特色技术巨头将让您更好地了解云原生及其对 IT 领域的影响。

如今，CI/CD 管道主要基于开源工具——Kubernetes、Jenkins、GitLab 等。安全传统上是封闭的，但现在很清楚，创新的步伐不能受到专有开发模式的限制。主题演讲人 Sysdig 的首席技术官和创始人洛里斯·德吉奥安尼将讨论安全云和容器环境所需的关键实践，包括:提高云中的安全性、可见性和简单性需要什么，从卡塞亚和网络安全管理软件产品等重大攻击中吸取的经验教训，利用安全 DevOps 不仅“左移”，而且“右滚”进行检测和响应的方法，以及开源如何成为在不断变化的环境中查看所有信号的关键因素。

在他的文章中， [【开源接管安全】](https://thenewstack.io/open-source-is-taking-over-security/) 发表在去年的 New Stack 上，洛里斯·德吉奥安尼详细阐述了开源对于维护高安全性至关重要的观点，他说:“我认为，尽管开源的渗透率较低，但安全是从开源方法中受益最多的行业。我还认为，开源将逐渐占据主导地位，成为做安全工作的标准方式。”为了进一步理解这一点，洛里斯将其分解为几个关键问题:为什么开源是实现安全的最佳方式？而且，开源会成为做安全的标准方式吗？

让我们从最重要的问题开始:为什么开源是最好的安全方式？洛里斯坚信开源是安全的关键因素，因为一件事——创新的速度。洛里斯指出，像 Kubernetes 这样被广泛采用的开放式计算平台的兴起，对分布式、协调的生态系统如何在 100%的时间里击败“闭门造车”的单一供应商产生了巨大影响。洛里斯写道，“一方面，Kubernetes 和 cloud native 的创新步伐如此之快，供应商很难独自跟上。另一方面，由于 Kubernetes，我们第一次有了一个完全开放的平台来运行我们的应用程序，安全性可以在本地集成，而不是固定在上面。”

开源也加快了创新的速度，因为它让最终用户能够与供应商合作。洛里斯说，任何用户，无论大小和复杂程度，都可以在一个紧密的循环中提供反馈，他们也可以做出贡献。那么，为什么这与安全性相关呢？正如洛里斯所解释的，“脆弱性和威胁的内在分散性使得这种以集市为导向的贡献方式非常有效。”

这让我们想到了洛里斯的第二个问题:开源会成为实现安全的标准方式吗？洛里斯不仅说是的，开源将成为做安全的标准方式，但事实上这是无法逃避的。洛里斯用一个坏人 vs .好人的比喻来描述网络安全之战，他说，“坏人的军队很大，很聪明，很有侵略性。而且形势对他们有利。”

那么，在这种情况下，好人是如何获胜的呢？洛里斯说，最好的机会是，“作为一个团队一起工作，利用彼此的力量，确保没有能量被浪费。”洛里斯坚持认为，“更好的方法是利用一套通用的标准，这些标准深度集成在平台中，由社区拥有，由共识驱动，所有参与者都可以在此基础上自信地创新”，而不是让各种供应商坚持竞争标准
