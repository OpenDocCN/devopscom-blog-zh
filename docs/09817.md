# API 是什么和如何:新的开发者指南

> 原文:[https://devo PS . com/the-whats-and-hows-of-APIs-a-new-developers-guide/](https://devops.com/the-whats-and-hows-of-apis-a-new-developers-guide/)

对于新开发人员来说，你可能不太了解应用编程接口([API](https://devops.com/?s=APIs))。然而，很可能你以前作为消费者或最终用户使用过它们，例如，无论是在网上购物，使用商务通信软件联系同事，还是在社交媒体平台上分享来自 [monogram 徽标制作商](https://logo.com/logos/monogram) 的图像。不管你是打算自己制作和使用一个，还是想了解你使用的工具是如何工作的，了解它们都是有帮助的。

虽然编码和开发工具可能会让外界望而生畏，但我们将把 API 变成一种新开发人员(甚至非开发人员)可以理解的语言。在本指南中，我们将介绍什么是 API 以及存在的不同类型。您还将了解它们的工作原理以及如何使用它们。希望到最后，您会对 API 更加熟悉，并且知道它们如何影响您和您的业务。

## 什么是 API？

应用程序编程接口是开发者用来在不同设备、系统或应用程序之间进行通信的一段代码。它允许信息传递，允许跨多个平台遵循程序。根据 API 的不同，这可能是传输数据、图像、视频、文本、文件和其他内容，或者允许开发人员通过使用 [【后端即服务】](https://blog.back4app.com/backend-as-a-service-baas/)【BaaS】进行连接，以将其应用程序链接到云。API 通常用于商业环境和个人项目或其他目的。

一般来说，API 关心的是原始信息并将其传递给另一个程序，而不是这些应用程序的外观或视觉方面。另一组编码被用来建立界面的美感和可用性，而 API 对用户来说不太可见。尽管如此，它们仍然是我们今天技术堆栈的重要组成部分，允许系统集成并变得更加敏捷，使用户更容易浏览应用程序。

例如，当您刷新您的全渠道通信应用程序时，会向 API 发送一个请求，以检查您社交平台上的其他链接程序的新消息，或拨打您的 [800 商务号码](https://www.dialpad.com/blog/how-to-get-800-number-for-business/) 。这些程序通知 API 新的通信，并发送适当的数据。API 将这些数据返回到您正在使用的应用程序，使您能够从这里读取消息或通知您没有新的调用。

## API 的类型

### 1.私人的

私有 API 在团队内部使用，由为公司或组织工作的开发人员创建。它有助于根据您的业务需求制作高度个性化的 API，并在需要时调整 API。外部开发人员不能使用或处理私有 API，因为它纯粹是为您的内部团队服务的。这些 API 倾向于帮助您公司的生产力和跨多个应用程序的项目管理。

### 2.伙伴

或者，合作伙伴 API 在外部用于与您的业务合作的公司。通常，这在企业之间传递信息，例如在联盟计划内或在线订单的外部邮资。但是，API 也可以共享和更新文件，如您的 [责任合同的发布](https://www.pandadoc.com/release-of-liability-template/) 或紧急程序模板。两家公司的开发人员可以改进 API 系统，以便在合作伙伴之间有效地共享数据。

### 3.开放/公开

由于开放 API(也称为公共 API)不受特定企业的限制，任何人都可以使用。根据您使用 API 的频率，您可能需要订阅才能访问该 API，尽管也有其他完全免费使用的 API。一旦你理清了你的访问，任何软件开发人员都可以使用 API 来进行修改，并在不同的软件之间更容易地传输信息。

![](../Images/25e676173a4f5ce499201819f305b290.png)

[图片来源于 mailchimp.com](https://mailchimp.com/developer/marketing/api/root/list-api-root-resources/)

### 4.复合材料

通过获取多个程序的响应并将它们整合到一个平台上，复合 API 对于商业和个人使用来说是必不可少的。反之亦然，复合 API 允许您同时向多个平台发出请求。有助于全渠道管理，跟踪客户互动， [在线传真](https://www.dialpad.com/features/virtual-fax/) 以及跨各平台购买。这些往往只需要开发人员很少的输入，并且是为任何人设置的。

## 如何使用 API

*   ### Select your API 【T1]

正如你可能已经发现的，API 可以有多种用途。在选择 API 时，寻找一些与业务相关的东西，帮助您完成已经完成的任务。你用的软件和工具可能有推荐的 API 可以试试，也可能有类似商家用的 API。做你的研究，以确保你的 API 执行正确的功能，并与你的 [重组工具](https://writer.com/guides/rephrasing-tools/) 、日历软件或你使用的其他附加软件一起工作。

如果你已经习惯了 API，你可能想从一个免费的选项开始。这意味着如果它不像你希望的那样工作，你没有因为尝试而失去任何东西。许多 API 都有一个免费的试用期或一个包含基本功能的免费订阅层，因此您可以感受一下这个 API，并决定它是否适合您的业务。当您对自己的选择有信心时，您可以投资于其他订阅层，这些订阅层往往允许更多用户和更多功能。

*   ### Find your API key

API 密钥将您标识为用户，使用 API 验证您的操作并跟踪您的使用情况。拥有它是至关重要的，尤其是对于基于订阅的 API，因为没有它，您可能无法执行某些操作，或者您的 API 用户可能会受到限制。即使您选择的 API 可以免费使用并公开访问，仍然可能有一个 API 密钥来保存您的信息和偏好，并在您的平台之间进行链接。

![](../Images/a8886f1c4962d8f9300f5faa94e265b9.png)

[图片来源于 developers.google.com](https://developers.google.com/maps/documentation/javascript/get-api-key)

您的 API 密钥可能会在您注册该服务时发布，因此请记住这一点，并确保您企业中需要它的人能够访问它。否则，您可能需要请求或支付一个 API 密钥，尽管这在您注册您选择的 API 时应该是清楚的。为了提高您的 [API 安全性](https://webinars.containerjournal.com/the-2022-guide-to-api-security) 并避免您的偏好被更改，请将您的 API 密钥保持为私有，并限制谁可以访问它。如果你的账户被黑了，你可能需要申请一个新的 API 密匙。

*   ### Read the instructions

假设你通过实践和实际使用掌握了要点，很容易忽略 API 附带的文档和说明。尽管在某种程度上这可能是真的，但是您可能会错过有用的功能或者没有尽可能有效地使用 API。同样，您可能会让自己陷入混乱，没有正确地 [设置资源请求和限制](https://webinars.containerjournal.com/building-stability-pitfalls-of-not-setting-kubernetes-resource-requests-and-limits) ，导致您的业务在未来使用 API 时出现更多问题。

留出时间阅读 API 的说明和文档，或者在管理团队之间分配工作量。您可能会发现它附带了安装说明或有用的教程，您可以与团队的其他成员一起使用。当你向更多的人介绍这个 API 时，这些说明会确保每个人都以同样的方式使用它。它可以防止你的团队产生困惑，并使他们更快地建立对 API 的信心。

*   ### Set your request

根据 API 的不同，可能会有一些建议的请求需要您来设置，或者您可能需要创建并编写自己的代码。遵循你的 API 的指令可以帮助你做到这一点，允许你的内部 [开发人员在 API 中发挥他们的全部潜力](https://webinars.containerjournal.com/unleashing-developers-full-potential-in-the-cloud-native-era-2) ，并告知你要使用的编码结构。尽管如此，网上有很多教程介绍了如何在不依赖编码专业知识的情况下设置 API 请求。

![](../Images/44a34646aeaedebc413b802405741175.png)

[图片来源于 developer.paypal.com](https://developer.paypal.com/api/rest/requests/)

仅仅因为一个请求是可能的，并不意味着它总是与你的业务相关。考虑你为什么使用 API，一旦你设置了你的请求，你想让它做什么。如果你是使用 API 的新手，从一些基本的请求开始，随着你获得信心和经验，在这些基础上，使用 [CI/CD 最佳实践](https://www.computer.org/publications/tech-news/trends/the-best-ci-cd-practices) 来确保它运行良好。它允许您熟悉每个请求的使用，并确保它以您想要的方式工作。

## 你准备好使用 API 了吗？

随着技术的进步和软件的开发，应用编程接口变得越来越有用。能够在应用程序之间进行通信可以简化您的 [虚拟接待员服务](https://www.dialpad.com/features/virtual-receptionist/) ，例如，传入视频电话和传出消息，以及加速云原生应用程序的开发。在一个地方提出请求并显示来自多个平台的信息，可以让您更好地管理任务。

如果你还不熟悉 API，你不必马上就开始使用它们。花时间学习在特定的功能中使用它们，熟悉它们，并使您的团队能够适应使用它们。在此基础上，根据需要添加新的 API 或请求，根据您的业务和您要完成的任务进行个性化设置。随着时间的推移，您将对使用 API 充满信心，并建立一个强大的请求网络来有效地运行您的业务。
