# Trifacta 将数据准备扩展到数据操作，为数据工程师提供了新功能

> 原文：<https://devops.com/trifacta-extends-data-preparation-to-dataops-with-new-functionality-for-data-engineers/>

*包括 RapidTarget 和 Automator 在内的新功能使数据工程师能够在生产中更高效地操作数据准备工作流程*

旧金山，2018 年 11 月 14 日—**[tri facta](https://www.trifacta.com/)****[数据准备](https://www.trifacta.com/data-preparation/)** 的全球领导者，今天宣布了一组新的功能，旨在使数据操作(DataOps)实践中的数据工程师能够更高效地开发、测试、调度和监控生产中的数据准备管道。通过 **RapidTarget，** Trifacta 客户可以利用现有的数据模型智能地指导转换，并加快生成与预定义模式相匹配的新输出数据集的过程。 **Automator** 提供调度、监控和细化准备工作流程的端到端管理。这些新功能，以及新的**部署管理器**框架，有助于数据工程师管理数据管道，为分析、机器学习和数据科学计划提供信息。

 Trifacta 产品副总裁郑伟表示:“当今的组织越来越多地采用数据运营来更有效地扩展他们的分析和数据管理实践。“数据工程师处于这一转变的中心，在数据准备实践中发挥着关键作用，这些实践构成了他们数据工作的支柱。我们的团队广泛研究了数据工程师在客户组织中的新兴角色，并将这些最新的增强功能集中于提高他们将数据准备工作流转变为可扩展、可重复的数据管道的能力。我们对这些新功能的引入证明了 Trifacta 在我们客户的数据管理实践中发挥着越来越大的作用。"

数据运算明显呈上升趋势，73%的组织表示他们计划在 2018 年投资该方法。鉴于 DataOps 实践的日益普及，数据工程师在配置、管理和扩展数据管道中的作用已经上升为数据管理中的一个关键职位。Trifacta 通过与包括 **[、纽约人寿](https://www.trifacta.com/customers/new-york-life/)** 、 **[、葛兰素史克](https://www.trifacta.com/customers/glaxosmithkline/)** 和 **[德意志交易所](https://www.trifacta.com/customers/deutsche-borse/)** 在内的大型企业合作，亲眼见证了数据工程师如何推动数据准备从临时活动走向标准企业数据管理流程。Trifacta 平台中的这些新功能进一步支持了数据工程师在促进由分析师最终用户开发的数据准备流程进入可扩展的数据管道，从而为更广泛的组织提供价值方面所发挥的作用。

“数据工程是 Malwarebytes 的一项关键职能，最终，我们负责在整个组织内安全采用新的分析用例，”数据科学与工程总监 Manjunath Vasishta 说道。“Trifacta 的数据准备平台有助于实现这一目标，使我们能够为业务用户提供更强的数据可访问性，同时保持治理。随着数据准备使用的持续增长，该平台的功能为数据工程师提供了更进一步的保证，以扩展和运营数据准备工作流，从而跟上业务需求的步伐。”

tri facta 的新特性进一步支持数据工程师和以数据操作为中心的组织包括:

*   **RapidTarget:** 允许数据工程师设置一个预定义的模式目标，该目标为如何准备不同的数据源并将其连接在一起以映射到该目标提供自动指导。将不熟悉的或外部的数据集成到分析中时，将转换过程与现有的数据模型保持一致是至关重要的。RapidTarget 提供了智能建议，以加快用户准备和混合数据源的速度，从而使其输出符合所需的数据模型。*T3*
*   **Automator:***tri facta 的系统，智能管理生产中数据准备工作流程的缩放、调度和监控。用户可以设置流的时间表，以便在数据更新时或使用 API 以编程方式在生产中自动运行。在调度时，用户可以定义参数或变量，以自定义将什么数据输入到工作流以及如何发布输出。最后，他们可以监控作业的状态和性能，并在输入数据发生变化或异常时发出警报。*
*   ***部署管理器**:在数据准备工作流程过渡到企业范围的数据管道时，提供测试、版本控制和管理数据准备工作流程的框架。在一个新的工作流程能够为更广泛的组织带来价值之前，该流程必须进行大规模的测试。借助 Trifacta 的部署管理器，数据工程师可以在过渡到生产流水线之前，在开发、测试和试运行环境中无缝测试新工作流。该框架还允许数据工程师管理流的版本，甚至在需要时回滚到流的先前版本。*T3**

*根据 Gartner 最近的一份报告，“在正确的时间向正确的人提供正确的数据是一项持续的挑战。数据工程师正在成为应对这一挑战的关键角色。因此，数据和分析领导者必须发展数据工程学科，作为其数据管理战略的一部分。”**
