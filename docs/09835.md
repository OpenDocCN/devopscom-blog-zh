# GitLab 与谷歌结盟，提高开发人员的工作效率

> 原文:[https://devo PS . com/git lab-alliances-with-Google-to-improve-developer-productivity/](https://devops.com/gitlab-allies-with-google-to-improve-developer-productivity/)

GitLab Inc .与谷歌合作，今天推出了一款开源云种子工具，让 DevOps 团队更简单地消费谷歌云平台(GCP)资源。

在虚拟[Google Cloud Next’22](https://cloud.withgoogle.com/next)会议上发布的云种子工具集成在 GitLab web UI 中，使开发人员能够更简单地使用 Google Cloud 资源，例如用于构建和部署容器化应用程序的 Cloud Run 和托管云 SQL 数据库服务。

GitLab 联盟全球副总裁尼玛·巴迪(Nima Badiey)表示，目标是减少摩擦，否则会通过要求开发人员编写代码来使用一组低级应用程序编程接口(API)来消费这些服务。

他补充说，这种自助服务方法不仅提高了生产率，还提供了额外的好处，即减少了这些服务可能被错误配置从而产生网络安全漏洞的可能性。

随着谷歌寻求在亚马逊网络服务(AWS)和微软上取得进展，谷歌在与 DevOps 平台提供商合作以使开发者更容易使用云资源方面有着既得利益。挑战在于开发人员在以编程方式调用云服务时经常出错。这些由此产生的错误配置往往直到发生某种类型的网络安全事件后才被发现和补救。

在一系列高调的软件供应链漏洞事件之后，人们更加关注安全性。当然，挑战在于找到一种方法来确保应用程序开发平台尽可能安全，同时又不降低软件开发和部署的速度。Badiey 说，实现这一目标需要政策、培训和系统的正确组合。

大多数组织都试图在对更高安全性的需求和吸引并留住开发人员的需求之间取得平衡。理论上，DevSecOps 最佳实践的兴起将更多的安全责任推给了开发人员。然而，大多数开发人员几乎没有网络安全专业知识，因此很明显需要在 DevOps 工具和平台中内置更多的防护栏，以防止犯网络安全错误。

尚不清楚开发人员工作效率的提高和更高的安全性将在多大程度上推动组织转向一个云平台，但在这个时代，越来越多的组织现在经常采用两个或更多云平台，这可能会产生重大影响。开发人员并不特别喜欢在将应用程序部署到生产环境中之后对其进行补救，因为他们通常已经转移到了其他项目中。IT 运营和网络安全专业人士也可能会将开发人员推向运营开销最小的平台。

无论如何，有一点是肯定的，随着未来几个月对软件供应链的审查加强，对所有相关人员的合规要求将变得更加严格。现在的问题是如何在这些需求变成更大的问题之前主动解决它们。
