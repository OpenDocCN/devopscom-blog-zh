# TechStrongCon:是时候组建一支公民开发者大军了

> 原文：<https://devops.com/techstrongcon-time-to-build-an-army-of-citizen-developers/>

DevOps 团队需要开始起草一个策略来支持即将成为公民开发者大军的队伍。

DXC Technology 现代应用开发全球总监 JP Morgenthal 在虚拟的 techstrong con T1 活动上发言时告诉与会者，现在是建立一支由 T2 公民开发者组成的队伍的时候了。

公民开发人员通常是最终用户，他们已经开发了足够的专业知识来使用低代码或无代码工具构建应用程序。Morgenthal 指出，大多数组织没有足够的专业开发人员来处理他们的应用程序开发积压，因此他们必须让最终用户能够构建应用程序。

然而，这些公民开发人员需要 DevOps 团队的支持，以确保为构建可扩展的安全应用程序提供保护。此外，DevOps 的参与还减少了公民开发者在构建应用程序时肯定会遇到的摩擦，他补充道。“DevOps 是基础，”摩根索说。“我们需要在这方面大力投资。”

否则，公民开发者——他们通常有其他需要他们关注的任务——会简单地放弃他们的应用程序开发工作，他指出。“这不是他们的主要任务，”摩根索说。“他们需要被培养。”

大多数专业开发人员专注于记录系统的应用程序。Morgenthal 指出，问题是根本没有足够多的人来构建必要的应用程序来推动互动系统。

他补充说，事实上，对公民开发者越来越多的依赖是更大的 IT 民主化浪潮的一部分，在这一浪潮中，没有先进技术学位的个人可以利用自动化来构建和部署应用程序。

然而，Morgenthal 说，当组织未能测试这些应用程序，确保配置没有被手动更改或没有解决随着时间的推移不可避免地积累的技术债务时，他们就会遇到问题。

倦怠也可能成为一个问题，所以 Morgenthal 说，组织需要确保敏捷开发实践用于首先创建最低可行产品(MVP ),然后计划开发冲刺来更新和维护它。

最后，还需要有一个适当的计划，当创建应用程序的个人最终离开组织时，将应用程序的责任转移给其他人，他补充说。

当然，公民开发者并不是唯一使用低代码和无代码工具的人。许多专业开发人员正在使用同样的工具来更快地构建应用程序。事实上，不清楚有多少最终用户拥有成功构建应用程序所需的技能，尽管低代码和无代码工具自动化了大部分过程。无论如何，许多专业开发人员现在也更频繁地与最终用户合作，使用低代码/无代码工具来构建应用程序，这使得最终用户更容易(至少)描述他们想要构建和部署的应用程序的工作流。

不管是谁构建了这个应用程序，在未来的几个月甚至几年中，都会有更多的代码通过 DevOps 管道传输。因此，DevOps 团队将不可避免地需要重新审视工作流和平台，以持续更新和大规模维护这些应用程序。
