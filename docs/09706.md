夏琳·奥汉隆:大家好。欢迎回到 TechStrong 电视台。我是 Charlene O'Hanlon，现在和我在一起的是 Jason Beres，他是 Infragistics 开发工具的高级副总裁。Jason，非常感谢你花几分钟时间和我一起讨论这个问题。我很感激。T3】

杰森·贝雷斯:谢谢你邀请我们，夏琳。谢谢你。

*O'Hanlon:* 所以我想和你们谈谈你们在过去一年半的时间里所看到的，与开发人员有关的，大辞职，以及缺乏合格的开发人员来填补这些职位，以及这对开发人员组织的影响，特别是与工具和工具有关的影响。所以我想知道我们是否可以稍微布置一下舞台。那么，让我们来谈谈你们在过去一年左右的时间里在市场上看到了什么，因为它与开发人员队伍有关。我们看到很多人在四处走动吗？

贝雷斯:是啊。这很有趣，这次采访的动力是我们刚刚得到了一份调查结果。它持续了六周的时间，我们问了很多关于过去 18 个月里开发人员发生了什么的问题。这是一个大问题。这对每个人来说都是一个巨大的问题，缺乏推动数字化转型所需的技能。因此，开发人员可能正在寻找其他职业机会，可能在开发空间中有他们想做的新事情，而不是他们过去做的事情。也许他们不想再做全职 web 开发人员了，他们想进入 AI，机器学习。因此，我们看到有些地方确实很难招到人。你会看到 50%到 60%的公司很难找到有技能的开发人员。基础统计学也不例外。我们的一些职位已经空缺了将近一年，我们在全球都有办事处。我们会在任何地方招人。这是一个巨大的挑战，我认为它正在改变组织对应用程序开发和工具的思考方式，以及他们前进真正需要的东西。

我一直听说大辞职，事实上每个人……不是每个人……但是很多人都在重新思考他们的人生目标。所以他们开始做不同的事情。但我几乎觉得这些人会去某个地方。那么，这真的是伟大的辞职，还是你认为这只是伟大的重新洗牌？T3】

贝雷斯:哦，哇。这是一个很好的说法。我确实认为这是一次伟大的改组，而不是辞职。我认为，在经济的某些领域，这无疑是一种巨大的无奈。我哥哥是一家货运公司的首席财务官。他们看到许多在疫情期间不能运输的卡车司机已经退休了。或者他们会说，“伙计。卡车运输真的很难。我要去做一些不同的事情，一周 60 个小时不在路上。”所以我认为某些经济领域比软件开发者受到的影响更大，但是我认为软件开发者是需要挑战的那一类人。所以他们总是在寻找下一件大事，也许不一定是恢复填充。更重要的是我需要更多的挑战，我正在寻找其他的选择。顺便说一下，这样做我可以加薪 25%。这是…我认为驱动它的是你看到的大技术…有趣的是，现在你看到的是，就 Meta 而言，例如，元宇宙，大技术只是…他们现在用荒谬的薪水从彼此那里抢人。所以，如果一个人赚了 X 美元，他们会提供三倍于 X 美元的价格在脸书、谷歌和微软之间交易。

所以，在小企业中，我不认为有那么糟糕，但这真的取决于你在哪里。Infragistics 的总部恰好位于新泽西州，距离纽约市只有 45 分钟的火车车程。纽约有很多大型科技公司，所以人们真的会说，“嘿。我应该冒这个险吗？我还能在家工作吗？”现在在家工作是一个很大的好处，我认为大型技术公司看到的风险之一是，“嗯，这是…嘿，我们要让人们开始走进办公室。”很多人会说，“不，请不要让我们这么做，否则我会去别处看看。”所以我认为大洗牌可能是一种更好的方式，正如你所描述的，特别是在软件开发中。

*奥汉隆:*我也看到许多开发人员，在疫情的过程中，他们使用了大量的…我不会说是空闲时间，因为我不认为任何人真的有空闲时间，但他们利用一些休息时间来提升他们的技能。所以他们现在正在转移到更新的发展领域，在那里，就像你之前说的，也许他们对人工智能更感兴趣。所以他们现在有能力做到这一点。那么，你们是否看到了某些级别的开发商，或者某些领域比其他人有更多的空缺职位？这些天，我们是否会看到更多的初级职位无人问津？

*Beres:* 我想我们看到的是，我们在世界各地都有办事处，在美国、南美、东欧和西欧、日本和印度。每个办公室都有独特的问题。例如，在南美，我认为我们看到的只是发展技能水平在持续提高。所以，也许五年前你可以雇佣很多真正初级的人，而且你可以花更少的钱得到他们。现在你雇佣了更多的高级员工，他们希望得到更多的钱，但是他们希望得到更多的挑战。我认为可能在东欧有更多初级员工，这是我们以前没有看到的，但风险在于你雇佣了初级员工，然后他们会相对快速地跳槽，可能在一年后，也可能在 18 个月后，一旦他们觉得，“嘿，等一下。我可以去赚钱改变生活，当你 20 出头刚从学校毕业时，这可能是 20%的加薪，但这就像是你前进的一块巨大的垫脚石。

所以我确实看到在每个地方都很有挑战性。但同时，我认为每个人都提高了水平。如果你认真对待你的事业，去年和前年你确实有更多的时间。你已经一个半小时没上路了。如你所知，过去在纽约工作的人可能会花两到三个小时在火车上，和其他人挤在一起，他们甚至不能看书。

奥汉隆:是的。去过那里，做过那个。

贝雷斯:所以现在他们在家里，上免费课程。所以你看看这些培训公司，甚至是 LinkedIn Learning，它们在过去几年里爆炸式增长。还有十年前什么都不是的培训公司。现在，由于在线培训，他们成了价值数十亿美元的公司。所以我认为，如果你对你的职业是认真的，你已经做到了，你正在寻找其他的机会。还有一个我之前没提到的是数据科学。当然，数据科学碰巧刚刚爆发，可能三到五年前还没有数据科学家。他们很难找到。他们都是职业转换者。也许我是一个非常善于分析的人，所以我要尝试一下这种数据科学。现在，他们学会了。数据科学家在那里，但他们赚的钱更多。因此，随着时间的推移，工资也随着市场的变化而变化，这很有意思。

奥汉隆:我们到处都能看到这样的剧变。我们考虑供应链。我们想到那些正在换工作的人，这些天有太多的变化。很难确定事情。看看接下来一年左右事情如何解决会很有趣。我认为我们会在接下来的 12 个月里看到一些稳定，因为，引用/不引用，事情会恢复正常，或者某种意义上的正常。

我们希望。

奥汉隆:是的。当我们看到所有这些事情发生时，所有这些不同的力量都在发挥作用，这必然会对整个组织产生影响，超越人的因素。那么，从组织的角度来看，你们对工具蔓延和 DevOps 工具链受到影响的其他方式有什么看法，或者只是整个开发人员体验正在发生变化。

贝雷斯:我认为我们确实看到，无论如何，业务都在向前发展。因此，企业，甚至在我们的调查中，他们期待 10%的增长，20%的增长。没有人报告说他们预计会出现负增长。他们期待正增长。我认为很多企业…不仅仅是小企业。这是中型企业，甚至像大型科技公司的筒仓意识到他们落后了。所以疫情说，或者它真的告诉他们，他们需要赶上来。因此，企业想要发展，他们想要前进。因此，当我们开始认识到技能组合和培训中的这些差距，以及我们真的想做但我们真的不能做的事情，因为我们没有人来做，或者我们没有合适的技能组合来做，我们等了太久。我们没有迁移到云。那你为什么不搬到云上呢？Gartner 十年来一直在说迁移到云，但你没有迁移到云。

因此，开发团队面临的压力越来越大，积压的工作越来越多。因此，影响工具链的方式，也就是我们在典型企业中看到的影响工具链的方式，甚至在 SMB 中，是像低代码、无代码工具这样的工具变得越来越被采用，可能更受信任。对于组织来说，这是一种开放，他们可能会允许一些 API，营销或销售团队可以使用这些 API 来实现自助式 BI，或者创建不是业务关键型的应用程序，但会帮助他们提高工作效率。因此，我们已经看到了一些变化，我认为我们也看到了市场的增长，在过去的几年里，嵌入式分析和低代码(无代码)的业务仍然是数十亿美元，但在更高的规模上，大型技术也有数十亿美元。现在，它被更广泛地全面采用，并在中小型企业中推广。

就像我之前说的，我认为随着事情的稳定，我们将会在这一年中看到很多变化。你认为我们会看到组织如何选择他们的工具，并做出重大决策的变化吗？你认为会有什么变化？

疫情再次教导所有人，你不能像今天这样受制于它的资源。否则你会被落下。数字化转型需要发生，团队需要更有生产力。你看看微软和微软团队的爆炸式增长，以及这种工作空间协作。所以，我们看到了这类工具的增长。我们看到越来越多的人将留在远程工作，然后软件公司的管理层、IT 管理层和执行管理层意识到我们需要生产力。因此，我们如何注入工具，使我们在安全性、功能、用户体验方面仍能得到我们需要的东西，但又能推动我们如何在内部和外部交付应用程序。所以我们看到很多…在疫情之前，有很多关于 UX 的谈论。因此，UX 是每次高管会议的首要议题。但是，他们真的关心 UX 吗？现在每个人都是，消费化已经发生，您的内部应用程序体验与基于客户的体验一样重要。您需要工具来帮助您做更多的工作。因此，引入更好的用户体验，能够帮助交付看起来更好、感觉更好、体验更好的应用程序的工具，将推动内部增长，显然，外部客户增长，真的变得至关重要。在接下来的 12 个月，在接下来的 18 个月，当事情开始具体化的时候，这些工具将会继续存在。

同样，也没有足够的熟练工人。例如，如果我们可以引入低代码、无代码工具来自动化流程，并且市场营销或销售团队中的超级用户，甚至初级开发人员都可以做到这一点，而不是必须雇佣一个团队来从头开始构建，这就是我们看到的节约之处，也是我们看到的企业数字化转型加速之处。

*奥汉隆:*我真的真的很喜欢这样一个事实，在过去的几年里，我们已经看到了整个技术的民主化。事实上，就你的观点而言，财务或营销部门或其他地方的人实际上可以进去进行这些更改，并且十有八九实际上比他们将更改发送给 it 团队更快地完成。我认为确实发生了翻天覆地的变化，而且这种变化还会继续下去。它最终会如何影响 it 组织，现在几乎人人都在猜测。但是，随着大辞职的发生，以及仍在发生，或大改组…我只是更喜欢这个词…大改组发生，我认为，我们将看到许多变化，组织如何看待他们…不仅仅是他们如何利用技术，还有他们如何利用他们的员工。那可能是另一个时间的另一次谈话。

但是 Jason，非常感谢你今天和我谈话。
