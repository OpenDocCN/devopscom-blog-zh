# Linux 基金会列出了顶级开源库

> 原文：<https://devops.com/linux-foundation-lists-top-open-source-libraries/>

Linux 基金会今天发布了一份[报告](https://linuxfoundation.org/wp-content/uploads/LFResearch_Harvard_Census_II.pdf)，提供了组织正在使用的前 500 个开源库的八个列表，作为帮助更好地保护软件供应链的持续努力的一部分。

自由和开源软件的普查 II——应用程序库报告基于软件组合分析(SCA)工具提供商的使用数据，如 Snyk、Synopsys 网络安全研究中心(CyRC)和 FOSSA。这份报告是由哈佛创新科学实验室编辑的。

这八个列表由四个包含版本号的列表和四个版本不可知的列表组成。列表被分为 npm 和非 npm 包，因为 npm 包将支配任何创建的排名。正在使用的顶级非 npm 包包括 maven、掘金、Go 和 cargo。

哈佛商学院助理教授 Frank Nagle 表示，该报告旨在为组织提供一些指导，帮助他们了解在广泛使用的用于管理 Java 应用程序日志的软件中发现漏洞后，使用了哪些开源包。

受该漏洞影响的许多组织并不知道 Log4j 在其企业 IT 组织中的部署有多广泛。IT 组织应该根据致力于确保这些库的安全性的贡献者和维护者的数量来评估这些项目的可持续性。

Nagle 指出，最大的挑战之一是找到一种方法来标准化软件组件的命名模式和不同库版本的整体管理。

Nagle 指出，最终目标是说服更多的企业 IT 组织和支持它们的供应商提供更多的资源来保护这些库。大多数最广泛使用的开源软件都是由少数贡献者开发和维护的。许多贡献者认为，虽然他们免费提供了该软件，但使用该软件的组织有责任保护它。

Linux 基金会正在成为一个渠道，通过这个渠道，它将提供额外的资源来帮助更好地保护开源软件。开源安全基金会(OpenSSF)是 Linux 基金会主办的一个联盟，本周早些时候宣布 [19 个其他组织已经加入 OpenSSF](https://www.prnewswire.com/news-releases/open-source-security-foundation-attracts-new-commitments-advances-key-initiatives-in-weeks-since-white-house-security-summit-301492720.html) 来帮助识别和修复开源软件中的安全漏洞，并改进工具、培训、研究、最佳实践和漏洞披露实践。

OpenSSF 的执行董事布莱恩·贝伦多夫表示，除了提供资源，很明显，随着各组织在其软件供应链中更广泛地使用开源软件，有必要对开源软件进行某种形式的第三方审计。

OpenSSF 的新卓越会员承诺现在来自 1Password、花旗、比特币基地、华为技术、JFrog 和 Wipro。Accuknox、阿里云、Block，Inc .、区块链技术合作伙伴、Catena Cyber、Chainguard、DeployHub、Gravitational .、MongoDB、NCC Group、ReversingLabs、Spotify 和 Wingtecher Technology 正在做出新的一般会员承诺。新的准成员包括中国科学院软件研究所、MITRE 和 OpenUK。

OpenSSF 领导的其他工作包括更好地保护开源安全态势的 Alpha-Omega 项目、识别超过 100 万个项目中的依赖关系风险的记分卡、多因素身份验证令牌以及签名、验证和保护开源代码的 Sigstore 项目。

开源软件达到 It 行业所追求的安全水平可能还需要一段时间。然而，从好的方面来看，用于实现这一目标的资源从未如此之多。
