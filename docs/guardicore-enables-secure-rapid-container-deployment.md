# GuardiCore 支持安全快速的容器部署

> 原文：<https://devops.com/guardicore-enables-secure-rapid-container-deployment/>

GuardiCore Centra 安全平台可降低合规风险，在容器化应用和工作负载中实施安全策略

以色列旧金山和特拉维夫 2018 年 4 月 17 日电/美通社/ —内部数据中心和云安全领域的领导者 GuardiCore 今天宣布了 GuardiCore Centra 安全平台中的几项新功能，旨在帮助安全架构师可视化、细分、检测和修复容器化应用或工作负载中的威胁，同时保持开发敏捷性、应用性能和可扩展性。

GuardiCore 利用其在数据中心和云安全创新方面的血统，保护容器化的应用程序，为 DevSecOps 团队提供各种关键能力，而不会影响速度或创造力。GuardiCore Centra 安全平台通过使 IT 安全团队能够查看每个容器、pod 和服务，可视化其通信流并使用微分段策略保护它们，同时检测攻击并展示在任何基础设施中的大规模合规性，而不会对性能产生任何影响，从而保护容器的生产和运营元素。

“随着企业竞相将其数据和应用转移到低成本、高度灵活和可扩展的云平台，他们越来越依赖容器来帮助提高灵活性和可扩展性。然而，缺乏对容器的可见性削弱了他们在运行时/生产环境中保护应用程序和工作负载的能力。“正如虚拟机和虚拟服务器改变了安全检查和保护网络和服务器的方式一样，容器给那些负责保护它们的人带来了更大的挑战。GuardiCore 屡获殊荣的云工作负载保护平台增加了特定于容器的功能，使现代企业能够根据需要尽可能快地部署容器，而不会牺牲安全性。GuardiCore 应用我们平台的创新功能来解决容器带来的特定安全挑战，为企业和服务提供商提供实施安全控制、解决合规性挑战、检测、调查和遏制威胁所需的可见性和可扩展性。”

**支持复杂云工作负载中的容器安全**

“云工作负载保护平台供应商需要能够提供对容器的可见性，并基于单个容器区分和应用策略，包括网络分段。对于使用容器来支持微服务风格的架构和快速开发工作流的组织来说，这是一个新兴的关键要求，”Gartner 副总裁杰出分析师 Neil MacDonald 在报告[云工作负载保护平台市场指南](https://www.gartner.com/doc/reprints?id=1-4U3BRXM&ct=180327&st=sb)中写道。

GuardiCore 将 Centra 的实时可见性功能扩展到包括容器，包括完全整合容器编排、元数据的能力和利用本机 pod 标签的能力。安全和应用程序开发团队可以在 pods 中查看流程级别的通信流，并部署精细的微分段策略来保护和控制通信流免受攻击和误配置。此外，该平台提供了检测单个集装箱内的威胁的能力，并且在集装箱受损的情况下，隔离它并防止攻击的传播。凭借这些新增功能，GuardiCore 扩展了其已经广泛的平台支持，包括 Docker、OpenShift 和 Kubernetes 容器，为所有数据中心和云环境提供了一个集成的解决方案。

**实施云工作负载保护和微分段**

除了保护容器化应用的功能之外，GuardiCore 还在 Centra Security Platform 中提供了新功能，使企业和服务提供商能够实施云工作负载保护和微分段。这些功能包括基于角色的访问控制以及保存、共享和调用可视化地图的能力。基于角色的访问控制允许管理员根据特定标签(如业务部门、业务应用程序或工作负载)定制访问范围。保存、共享和调用可视化地图允许安全运营人员保存常用的通信流地图，从而节省时间和精力，实现协作，并加快事件调查。
