# DevOps 世界 2022:为什么 DevOps 需要混音

> 原文：<https://devops.com/devops-world-2022-why-devops-needs-a-remix/>

我今天在查看 DevOps World 网站收集信息，名为 [Techstrong。电视](https://techstrong.tv/) 将于 2022 年 9 月 28 日和 29 日在佛罗里达州奥兰多进行直播。今年 DevOps World 的主题是“DevOps Remix”我想得越多，就越清楚 DevOps Remix 是 DevOps 世界的完美主题——也是 DevOps 实践的完美主题。

DevOps 混音是什么意思？好吧，我不会假装为该活动的主办方 CloudBees 说话，但下面是他们在 DevOps World 网站上的内容:

“随着 DevOps 转型步伐的加快，节奏将会更快、更有活力、更和谐。 ***渐进式交付、价值流管理以及从 AIOps 到 XOP 的一切现在都是组合的一部分*** *创造新路径、新节拍和更快的代码发布。”*

这就是 DevOps 混音对他们的意义；以下是 DevOps remix 对我的意义。

我相信一个 DevOps remix 表明了 DevOps 作为一种方法和实践的不断成熟，也表明了 DevOps 市场和社区的不断成熟。DevOps 不再模糊不清；不再有“是真是假”的问题了？DevOps 到底是什么？”类似的事情。不，DevOps 是软件开发生命周期(SDLC)中的一员。如此之多，以至于随着数字化转型继续在以前没有的地方涉及软件，DevOps 正在帮助它实现这一目标。DevOps 正在走向它从未去过的地方。

Techstrong Group 自己的 DevOps 体验活动(现已进入第五年，将于今年 11 月举行)也有类似的主题:DevOps 无处不在。作为 CI/CD 管道的一部分，DevOps 仍然存在，但今天的 DevOps 已经远远不止这些了。为了在今天取得成功，各种规模的组织(尤其是企业)需要掌握 DevOps 基础知识，并在构建、部署和管理软件的任何时间和任何地点实施它们。

重新组合的想法如此合适的另一个原因是，随着世界从新冠肺炎封锁中走出来，重温现场活动、会议和培训——疫情之前发生的“正常”行业活动和事件——会令人感到安慰。对于我们这些 DevOps 领域的人来说，DevOps 世界就是其中之一。我一直去 DevOps 世界，甚至在它被称为 DevOps 世界之前——那时它还是 Jenkins 世界。这是给你的混音！

DevOps 确实需要一次重组，而且现在就在进行。虽然我确实认为能够恢复许多相同的活动、参加会议和接受培训非常好，但我不希望今年的 DevOps World 像旧金山或在线的前几届一样。这是使它成为如此了不起的事件的一部分——旧的、新的、真实的、前沿的和熟悉的混合。所以，如果你认为自己是 DevOps 社区的一员，并能在九月底到达奥兰多，你应该考虑成为 DevOps Remix 的一员。

今年， [DevOps World](https://devopsworld.com/) 将在佛罗里达州奥兰多的世界万豪中心举行。CloudBees 的人发布了一个折扣代码， DW22，在注册时使用，应该可以为您节省一些钱来购买一张完整的演出门票，包括一些培训，一些精彩的会议和互动活动。我们迫不及待地想在那里看到你的 DevOps 混音！
