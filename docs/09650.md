## 2.“是家庭，不是数据中心，”市民们大声疾呼

人们指责人口中心附近的云数据中心把人们赶出了住房市场。随着地区电网运营商威胁在 2035 年前禁止新的连接，人口密集的地方如英国西伦敦也感受到了影响。

### 分析:ARM 和 RISC-V 可能会有所帮助

问题不在于太多的发电，而在于电力分配:现有的电网布线能力即将耗尽，升级绝非易事。部分解决方案是[让数据中心**更高效**](https://devops.com/red-hat-ceo-out-blind-users-revolt-arm-google-joins-party/#tlv3 "Finally! Google Cloud Does ARM") 。

[劳伦·莱弗](https://gizmodo.com/data-centers-london-tech-electric-grid-1849342150 "read the full text") : **太多的服务器可能意味着没有新家**

<big>**`Proliferation of data centers`**</big>
数据中心导致伦敦部分地区电力需求暴涨。现在，新的住房建设可能会被禁止…在一些社区。…太多的数据中心消耗了太多的电力。
…
近年来，伦敦西部及其周边地区建造了多个数据中心，尤其是 M4 走廊沿线，这是微软、甲骨文、亚马逊、惠普、索尼、戴尔、华为等公司的主要技术中心。(但)伦敦正面临一场严重的房地产危机。政府已经承诺解决这个问题，部分是通过建造更多的房屋——但是这些电力基础设施的限制可能会使这个承诺无法实现。
……
其他地方也面临着类似的电网和供电问题:……近年来，数据中心的激增给爱尔兰的电网带来了压力[。[和]加密货币开采一直是德州电网的负担。](https://devops.com/twitter-bluesky-adx-algorithm-cloud-energy-use-tripled-apple-staff-are-revolting/#tlv2 "Irish Data Center Energy Use is Way Up")

这是 ARM 及其电源效率的工作吗？[上海比尔](https://news.slashdot.org/comments.pl?sid=21819002&cid=62754730 "read the full text"):

如果我们要在数据中心从 x86 大规模转向更高效的处理器，我们应该转向 RISC-V，而不是 ARM。RISC-V 更高效，也是更现代的架构。
…
RISC-V 为非嵌入式版本内置了代码压缩。代码比 ARM 或 x86 更紧凑。所以缓存可以多保存大约 30%的代码。…代码大小是 CISC 优于 RISC 的少数几个优势之一，有了 RISC-V，这一点也没有了。

“天塌下来了。11 点拍。”这是利物浦橡子:

这个故事应该在一周内刊登在每一个新闻简报上。我们正前往瀑布。没有桨。

## 3.英特尔是最新的 Wi-Fi 7 芯片制造商

早在四月份，我就告诉过你博通、联发科和高通推出 Wi-Fi 7 芯片。现在英特尔的**加入了这个派对**。

### 分析:在人口密集的地区很棒

802.11be 自然应该更快。但它也将更具确定性，受干扰更少，并改善延迟:**这不仅仅是速度的问题**。

[金智雄](https://english.etnews.com/20220801200002 "read the full text")迷失在翻译中:**英特尔将下一代 Wi-Fi 7 商业化**

<big>**`Advanced technologies`**</big>
英特尔将于 2024 年推出 802.11be。数据处理速度比现有的 Wi-Fi 6E … 802.11ax 快两倍以上
…
英特尔预测，Wi-Fi 7 …的应用将利用高端游戏、增强现实(AR)、虚拟现实(VR)、机器人等先进技术扩展其产品应用。

这么远，这么蓬松。为什么我们需要这么快的速度？ [drinkypoo](https://mobile.slashdot.org/comments.pl?sid=21818968&cid=62754666 "read the full text") 有一个围棋:

记住，带宽是在同一个 AP 上的每个人之间平分的，大多数人只有一个。…如今，除了 AP 之外，家里几乎没有什么东西是有线连接的。这样的速度支持多个用户共享文件、流式传输等。
…
对于一两个客户端来说，【802.11n】就可以了。对于一堆用户来说，就没那么多了。
