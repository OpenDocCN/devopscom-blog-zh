# 连接您的云和开发运维工具以支持 CI/CD

> 原文：<https://devops.com/connecting-your-cloud-devops-tools-to-enable-ci-cd/>

一个有效的持续集成/持续交付(CI/CD)管道必须无缝集成云和 DevOps 工具，以确保应用程序的构建、编译、测试和发布高效且无摩擦地完成。

应用程序架构、内部过程和个人偏好意味着组织和团队以不同的方式建立 CI/CD 管道，导致整个软件交付管道中基础设施和工具的多样性。虽然创新要求使用交付应用所需的任何东西的自由，但由此产生的多样性可能是一个重大的集成挑战。我们需要的是一个为 DevOps 构建的平台，无缝连接 DevOps 生态系统和 CI/CD 管道工具。

有两种方法可以做到。

一种是使用 REST APIs。REST 是一种流行的选择，因为它很简单，而且它建立在现有的 HTTP 系统和特性之上。它避免了创建新标准、框架和技术的需要。使用基于 REST 的集成方法的另一个好处是，许多开发人员熟悉 REST 架构的关键元素，例如安全套接字层(SSL)加密和传输层安全性(TLS)。此外，REST 是独立于语言的。因此，开发人员在创建 REST APIs 时可以使用任何使用 HTTP 的语言，这样开发人员就可以选择他们熟悉的、最适合他们需求的技术。

另一种方法是寻找一个支持与常用云和 DevOps 工具现成集成的平台。自然，每个公司和 DevOps 团队都使用一套特定的工具。但是可以肯定的是，有一些广泛使用的工具应该被包括在任何列表中。一些有用的插件集成包括亚马逊 S3、Azure DevOps、Bamboo、CircleCI、Github、Jenkins、TeamCity、Terraform 等等。

**整合的好处以及如何扩大这些好处**

将云和 DevOps 工具集成到 CI/CD 管道中有几个好处。

内置集成将帮助开发者从管道中获得最大价值。例如，可以利用开箱即用的集成来帮助提供更多的自动化测试和更快的发布时间。为什么？如果工具没有集成，开发人员需要经历一个缓慢的手动过程来为他们的开发环境部署、配置和优化工具。这是一个耗时的过程。那些参与这个过程的人在他们真正的工作上浪费了时间。如果缺乏集成，整个组织中的多个团队可能会浪费时间去做相同的过程来让工具工作。也许最麻烦的是，手动集成工具会引入错误和潜在的安全漏洞。

高度集成的工具有助于鼓励协作。在许多公司中，不同的项目和不同的开发团队很容易陷入孤立的工作模式。不同的小组将使用不同的工具。将云和 DevOps 工具集成到 CI/CD 管道中可以实现事实上的流程标准化。由此，最佳实践通常来自于对使用相同工具的熟悉程度。聪明的公司会利用学到的经验，并确保在团队中分享机构知识。一旦达到了基本的共享水平，不同的团队可能会直接工作，互相询问，并一起工作以充分利用这些工具。

需要什么？

要实现在 CI/CD 管道中集成云和 DevOps 工具的优势，需要一个合适的平台。正确的平台应该毫不费力地将基础设施和应用程序资源交付给开发团队。

提供自助式基础架构访问的解决方案需要的不仅仅是简单的配置[自动化](https://devops.com/enabling-developers-with-infrastructure-automation/)。解决方案必须帮助公司监控资源利用和成本。并特别提供了一种将成本与业务单元、项目和个人开发人员联系起来的方法。

有了这样的产品，公司就能获得将 [cloud](https://www.quali.com/blog/what-is-cloud-governance/) 和 DevOps 工具集成到 CI/CD 管道中所带来的全部好处。
