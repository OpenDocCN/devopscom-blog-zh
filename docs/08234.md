# 集成 z/OS 系统以获得跨渠道体验

> 原文:[https://devo PS . com/integrating-z-OS-systems-for-cross-channel-experiences/](https://devops.com/integrating-z-os-systems-for-cross-channel-experiences/)

**集成 z/OS 系统，实现无缝的跨渠道客户体验**

在许多行业中，客户体验是 z/OS 系统的关键优势。这包括零售、金融服务、运输、酒店、政府等等。因此，将 z/OS 系统集成到任何面向客户的应用程序或前端应用程序中是必不可少的。

这个问题到底有多重要？超过三分之二的公司现在主要基于客户体验进行竞争。这比十年前的 36%有所上升。像捷蓝航空公司和丽思卡尔顿酒店公司这样的公司在历史上因其卓越的客户服务而闻名于世。眼镜供应商 Warby Parker 因其客户体验而广受好评，并因一个传奇的超范围服务而闻名于世，当时[一名高管将一副眼镜](https://www.forbes.com/sites/stanphelps/2014/08/01/heroic-customer-service-by-a-senior-executive-at-warby-parker/?sh=2d30391f4c9f)归还给一名将眼镜遗忘在 Acela 火车上的客户。

**从多渠道到全渠道**

许多企业为客户提供了多种与公司互动的方式。对于一个公司来说，通过网络、面对面、手机、语音、文本和聊天来支持交互并不罕见。这样，每个客户都可以使用他们喜欢的方法来提问、查看帐户信息、检查订单状态、获取信息、搜索、下订单等等。

一家公司与另一家公司的区别在于，一家公司何时能提供无缝的跨渠道客户体验。无缝的跨渠道体验通常被称为全渠道，是一种服务客户的多渠道方法，无论客户以何种方式或在何处接触，都可以提供集成的、有凝聚力的客户体验。

我们都经历过单独处理每个渠道的孤立方法。您正在进行在线操作，例如下订单、检查服务请求的状态或查询您的帐户。一个问题发生了，比如电话线掉线，你的手机屏幕太小，无法执行任务，或者你只是想与某人面对面交谈。你从手机切换到网络，或者从手机切换到店内，你已经做的和现在做的没有任何联系或遗留。您需要重新输入信息，回答相同的问题，并完成相同的步骤。

相比之下，与提供无缝跨渠道体验的企业的互动就大不相同了。电话中的代理知道您在网上下订单时遇到了问题，店内代表希望您留下一件商品进行维修，当您在线登录您的帐户时，您会被带到您在移动设备上所处的流程部分。

**整合是关键**

全渠道客户体验无缝集成了企业用来与客户沟通的不同沟通渠道。客户对无缝跨渠道体验的需求正在推动企业进行变革。投资全渠道客户体验的公司数量从 2010 年的 20%跃升至 2020 年的 80%以上。

因此， [z/OS](https://devops.com/bringing-z-os-systems-into-todays-modern-application-development/) 用户竞相适应全渠道客户体验的新现实。用户期望数据和应用程序在移动设备、网络或电话上像在桌面客户端一样容易获得。

开发无缝的跨渠道用户体验意味着大量的新应用程序编码。我们需要的是一种使用现代 API 轻松转换经验证的应用程序和记录系统数据的方法，以加快新服务的交付并满足不断变化的客户期望。

为了紧密集成 z/OS 系统，需要采用现代应用程序开发方法。这包括为开发人员提供解决方案，让他们能够保留几十年来在 z/OS 系统上运行的核心应用程序中所做的努力，同时构建能够利用这些系统的新应用程序。

为了实现这一点，企业必须向他们的开发人员提供 z/OS 环境中常用的开源开发、工作流自动化和 CI/CD 工具。这种策略将帮助企业加快新应用程序的开发，以满足客户不断变化的需求。它允许企业利用其 z/OS 系统上的数据和应用程序，并创建创新的新应用程序。

开发新的客户应用程序，同时保留 z/OS 系统上的核心业务功能、流程和数据，确保所有前端访问都利用关于客户的单一真实来源。每个客户只有一个关联记录。因此，z/OS 系统的这种集成使所有渠道保持一致变得更加容易，并使企业能够提供宝贵且高度期望的无缝跨渠道客户体验。
