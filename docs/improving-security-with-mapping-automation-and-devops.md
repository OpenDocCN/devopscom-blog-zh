# 通过映射、自动化和开发运维提高安全性

> 原文：<https://devops.com/improving-security-with-mapping-automation-and-devops/>

强大的网络安全对每个组织都至关重要。但是很多时候，安全团队与正在进行的开发过程脱节，或者被视为将产品推向市场的障碍，或者被视为事后的想法。这导致新平台的成本增加，攻击面可能出现缺口。

虽然许多组织转向安全自动化来帮助他们管理和缓解不断扩大的威胁，但快速修复很少奏效。需要计划来帮助组织确定他们的痛点，在软件开发过程的早期增加安全性，并确定自动化在哪里可以提供最好的 ROI。

## 什么应该自动化？

安全自动化可以通过执行作为持续改进管道一部分的任务，以编程方式测试、设置护栏、提供反馈和补救网络威胁。反过来，这可以通过简化复杂而耗时的测试过程来帮助人手不足的安全团队，使团队能够最大限度地利用时间，提高工作效率，并专注于风险优先级。一旦取消了手动流程，自动化工作流就可以使用智能和逻辑决策来执行各种更细微的操作。

可能受益于自动化技术的以安全为中心的领域包括但不限于:

*   用于发现安全漏洞的静态和动态代码分析
*   软件测试工具，用于在各种条件下运行程序单元，以发现并监控其行为和输出
*   上游代码分析，检查代码，如完整的代码块或补丁
*   漏洞测试、管理和补救，以发现和修复易受攻击的弱点
*   渗透测试用于模拟对组织计算机系统的网络攻击，并评估安全状况
*   基于数据分类和标记策略的自动化执行
*   基础架构组件的部署和管理

## 关注投资回报率

尽管有这些优势，但许多组织在开发、实施、测试和优化使用安全自动化来应对不断变化的威胁形势所需的全面、综合、长期的方法时，仍然面临着巨大的障碍。对于组织来说，以自动化作为结果开始项目，而不是严格地映射过程并确定从哪里可以获得自动化的价值，这种情况太普遍了。

假设一个组织决定将防火墙部署到 AWS 云的过程自动化。一个团队花费数周时间开发和测试一个自动化剧本，该剧本可以启动虚拟防火墙、许可它们、将它们连接到中央管理器、部署配置并应用它们，以及将日志发送到适当的位置。测试证明自动化是可行的。

但当被问及他们预计今年将安装多少防火墙时，答案是两个——也许。明年有多少？没有。因此，团队花费了比仅仅部署系统更多的时间来自动化这一功能，并且组织至少在两年内不可能实现这一工作的任何直接回报。

## 价值流图

实现真正价值的第一步是理解自动化是一种二阶操作，它依赖于对实现真正的技术和业务成果所需的所有步骤的全面评估和计划。这意味着执行[价值流图](https://www.vsmconsortium.org/)来分析、设计和管理最适合您环境的工作流。

映射将帮助您分析和定义安全流程。如果不分析您的流程，您将不知道瓶颈在哪里或者如何消除它们。更好地理解这些过程也将允许您根据需要对它们进行改进。如果不这样做，您将无法证明您正在实施的自动化的价值，也没有令人信服的商业理由。

## DevOps 的角色

很少有组织能够从一开始就有效地实现自动化的好处。价值流图使组织能够确定他们的痛点，准确定位他们希望在哪里使用自动化，并确定不能或不应该自动化的流程。清晰的计划和评估将帮助团队定义一个过程，并确定自动化某些过程的回报价值。

此外，为了使安全性有效，我们必须将安全性转移到软件开发生命周期中。安全从业者应该参与到开发管道中，包括在自动化计划上的合作，以及随着工作的进展构建利用和测试用例。这将节省时间、提高质量并最终增加价值，确保安全自动化项目的成功。
