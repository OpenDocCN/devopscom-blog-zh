# Puppet 宣布 DevOps 市场细分报告的状态，按行业、地区、公司规模等更深入地探讨趋势和模式

> 原文:[https://devo PS . com/puppet-announces-state-of-devo PS-market-segmentation-report-dives-deep-into-trends-and-patterns-by-industry-region-company-size-and-more/](https://devops.com/puppet-announces-state-of-devops-market-segmentation-report-dives-deeper-into-trends-and-patterns-by-industry-region-company-size-and-more/)

报告揭示了 IT 性能方面令人惊讶的行业差距，以及开发运维起点等方面的差异

俄勒冈州波特兰市。，2018 年 5 月 31 日(环球新闻网)——[Puppet](https://www.globenewswire.com/Tracker?data=5sEU5WnQT2Nf4CvSlWL9I0BTgluSSKUYnav47gdHA0hG0YB0TmaCyWaR6L2NWYah)，为我们周围的一切提供动力的软件的交付和运行自动化的标准，今天公布了 DevOps 状态:市场细分报告的调查结果。这份新报告是同类报告中的第一份，根据 2017 年 DevOps 调查数据，按行业、地区、部门和公司规模揭示了更多见解[1]。在过去的六年中，Puppet 调查了 27，000 多名技术专业人员，使《DevOps 状况报告》成为关于 DevOps 主题的最全面、持续时间最长的研究。

这份新的市场细分报告揭示了不同行业和地区的公司在发展之旅中的进展情况。调查结果包括:

*   大多数组织从解决他们最棘手的问题开始他们的开发运维之旅。
    部署自动化、版本控制、持续集成和基础设施自动化是 DevOps 之旅最常见的起点。最不常见的起点是精益实践，例如小批量工作以允许单件流，关键质量和生产率指标的可视化，以及对正在进行的工作设置限制。
*   公司规模越大，低 IT 绩效者的比例越高。我们发现，在不同规模的公司中，高水平 IT 人员的比例几乎没有差异。然而，随着公司规模的增长，低 IT 绩效者的比例也在增加。
*   自动化并不像你想象的那样普遍。
    我们的大多数受访者表示，在配置管理、部署、测试和变更批准流程中需要大量的手动工作。当他们采用 DevOps 实践时，自动化是整个过程中的一个关键促成因素，尽管它在他们的组织中仍然是不一致和不稳定的。
*   **行业问题。** 虽然所有行业中高水平 IT 人员的比例大致相同，但不同行业中低水平 IT 人员的比例却大相径庭。媒体、娱乐和零售行业的低绩效员工比例最低，而金融服务、保险和制造公司的低绩效员工比例最高。

“今天，世界各地的每家公司都有同样的优先事项——大规模自动化——他们正在通过 DevOps 实现这一点，”DevOps 报告作者兼 Puppet 首席技术策略师 Nigel Kersten 说。“虽然数据显示各种类型和规模的公司都在取得进步，但要消除阻止公司扩大自动化成功的手动工作，我们还有很长的路要走。更重要的是，DevOps 的大量研究表明，每个企业都有潜力成为顶级企业，我们很高兴能在实现普遍自动化的过程中提供帮助。”
