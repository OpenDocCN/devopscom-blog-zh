# GitOps 演进虚拟峰会亮点

> 原文：<https://devops.com/highlights-from-gitops-evolution-virtual-summit/>

GitOps 在 DevOps 领域越来越受欢迎。为了更好地部署、管理和监控 Kubernetes 集群和应用程序，DevOps 团队一直在采用 GitOps。9 月 23 日，DevOps 社区聚集在 [GitOps Evolution 虚拟峰会](https://www.mediaopsevents.com/gitopsevolution)上，探讨组织如何使用 GitOps 解决实际问题。

由 MediaOps 主办的虚拟活动提供了由专家主导的会议和小组讨论，重点讨论各种主题，包括:

*   GitOps 和 DevOps 的发展
*   采用 GitOps 的开发人员指南
*   在 GitOps 环境中要测量的指标

GitOps Evolution 虚拟峰会[现已推出，可在现场活动结束后的 30 天内点播](https://www.mediaopsevents.com/gitopsevolution)。[立即注册](https://www.mediaopsevents.com/gitopsevolution)即可独家访问按需虚拟环境。

## GitOps 演进会议

**GitOps——介绍，作者艾伦·希梅尔和保罗·柯蒂斯**

在本次会议中，Paul 和 Alan 讨论了 GitOps 的四个原则，增强的开发人员体验如何提高工作效率，如何提高 Kubernetes 环境的稳定性和可靠性，以及如何从一开始就确保安全性。

特雷西·拉冈著

GitOps 需要许多手动步骤，从更新。yaml 文件，并管理拉取请求、合并和提交。在本专题讲座中，Tracy 探讨了如何将 GitOps 迁移到完全自动化的 DevOps 流程，并回顾了 GitOps 的基本架构、如何手动管理以及 GitOps 如何发展成为持续交付渠道的一部分。

**如何在基础设施上应用 GitOps，作者 Viktor Farcic**

在这次演讲中，Viktor Farcic 探讨了 Crossplane 与 Flux 的结合，作为满足我们所有基础设施需求的潜在解决方案，以及它是否适合更广泛的 Kubernetes 生态系统。

**GitOps 盲点——实用 GitOps 简介，作者 Ravi lach man**

查看 Ravi 的会议，了解 GitOps 如何成为旅程中的连续体，并介绍实用的 GitOps 实践和方法，无论您处于软件开发方法的哪个阶段。Ravi 谈到了采用纯 GitOps 方法可能会留下的常见盲点。

**保罗·柯蒂斯《商业成功的 GitOps 成熟度演变》**

在本专题讲座中，Paul 进行了四步 GitOps 之旅，从操作单个集群和应用开始，到管理大型部署和设备群。他讨论了每个步骤的需求和业务优势，并将它们与提高软件交付和运营(SDO)性能联系起来，包括部署频率、更短的交付周期、平均恢复时间和变更失败率。

杰恩·格罗尔、特雷西·拉冈和维克多·法西克撰写的《炉边谈话:GitOps 的人员、流程和技术》

加入杰恩·格罗尔、特雷西·拉冈和维克多·法西奇的小组讨论，深入探讨 GitOps 的人员、流程和技术——包括下一步，文化和理论与实际实施的挑战。
