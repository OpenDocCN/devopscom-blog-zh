# CloudTruth 收购 Tuono 以推进配置管理

> 原文:[https://devo PS . com/cloud truth-acquisites-tuono-to-advance-configuration-management/](https://devops.com/cloudtruth-acquires-tuono-to-advance-configuration-management/)

统一配置管理平台提供商 CloudTruth 今天透露，它已经[收购了云秘密管理平台提供商 Tuono](https://cloudtruth.com/blog/cloudtruth-raises-5-25-million-to-solve-cloud-configuration/) ，作为简化使用代码提供的基础设施安全的努力的一部分。交易条款没有披露。

CloudTruth 首席执行官格雷格·阿内特(Greg Arnette)表示，云计算环境中普遍存在错误配置。随着组织可以采用的云服务数量不断增加，IT 组织现在需要一种自我记录的方法来管理配置。Arnette 说，CloudTruth 平台将自动跟踪对 IT 环境进行的所有配置更改。

Kubernetes 和 T2 微服务的兴起让事情变得更具挑战性，带来了一系列错误配置云平台和应用的机会。

刚筹集了 525 万美元的种子资金，Arnette 说，配置挑战中最不受重视的一个方面是组织对部落知识的依赖程度，这种知识在开发运维团队中的共享不均衡。他补充说，由于缺乏用于管理配置的集中存储库，让新成员加入这些 DevOps 团队变得非常困难，从而导致出现更多错误的机会。

Tuono 增加了一个云原生秘密管理平台，可以跨多个云服务部署。目标是提供一个配置管理平台，作为元数据包装器，与配置工具兼容，比如来自 Amazon Web Services (AWS)的 Terraform 或 CloudFormation。Arnette 说，这种方法减少了 DevOps 团队的认知负荷，否则他们必须记住每个服务是如何以及为什么以这种方式配置的。

Arnette 补充说，另一个目标是使 DevOps 团队能够将护栏放置到位，以减少出现错误配置错误的机会，同时继续使开发人员能够快速构建和部署应用程序。Arnette 指出，每个组织都需要决定如何在这些相互竞争的议程之间取得平衡。

如今，组织面临的挑战是软件供应链有多脆弱，一系列网络安全攻击已经非常明显。组织正在采用 DevSecOps 最佳实践来更好地保护这些供应链。然而，DevOps 团队在安全性方面的发展速度不够快，不足以保护即将部署的应用程序。同时，在云平台中可能部署了数百个应用程序，它们都有不同程度的错误配置。网络安全团队现在被要求审查这些应用程序的配置情况，作为解决整个软件供应链安全的努力的一部分。

目前还不清楚，随着网络安全团队审查应用程序开发和部署实践，对 DevOps 的反弹是否正在形成。毫无疑问，必须对这些过程进行调整，尽管大多数组织已经意识到他们是多么依赖软件。问题是，更快地部署更不安全的应用程序最终会适得其反。
