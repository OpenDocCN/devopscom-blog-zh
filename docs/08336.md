# IT 的未来:智能工具、智能技术

> 原文：<https://devops.com/the-future-of-it-smart-tools-intelligent-technology/>

在过去几年中，信息技术和发展之间的界限已经模糊。开发人员需要了解基础设施，管理员需要知道如何编码。容器化、流程编排、[持续集成](https://devops.com/?s=continuous+integration)和基础设施即代码(IaC)等进步与 DevOps 的崛起同步发生。DevOps 的优点是可以快速构建和部署产品。然而，这是有代价的，这种代价就是基础设施的复杂性。管理这种复杂性的答案在于人工智能(AI)和机器学习(ML)等技术。

## AI 又能如何呢？

近年来，智能技术取得了长足的进步。机器学习和人工智能正在越来越多的技术中找到出路。可以相对容易地监控和管理复杂系统的新工具开始出现。在 IT 中使用这些系统有几个明显的好处:

*   出现问题时的即时通知。
*   繁琐的 IT 任务的自动化。
*   动态基础设施调整有助于应对流量高峰和其他变化。

今天，训练一个人工智能来保持系统最佳运行，应用更新并实时提醒 IT 员工注意问题是完全可能的。通过自动化大量繁琐的手动工作(如挖掘日志), IT 可以专注于关键任务目标，而不是照看服务器和监控仪表板。

## 这对 IT 工作意味着什么？

一些人担心人工智能的进步将导致机器人取代人类工作的世界。虽然毫无疑问，在某些领域，人工智能和人工智能比人更有意义，但这些技术也将创造新的就业机会。随着 IT 团队从繁琐的任务中解脱出来，他们可以扩展到使用 ML 和 AI 技术。IT 团队非常适合指导业务决策，决定哪些任务从智能技术中获益最大。因此，尽管人工智能可能会改变 IT 工作，但没有理由相信它们会消失。相反，将人工智能视为 It 的新工具会更有帮助。

## 它能在哪里实现 AI 工具？

人工智能最适合用于有大量数据的任务。人工梳理日志文件、数据库和仪表板是一个缓慢的过程。人工智能可以随时监控一切，让 IT 人员专注于解决方案，而不是跟踪基础设施的哪一部分出现了问题。

可以从 AI/ML 中受益的一个特别的领域是欺诈检测。机器学习可以分析数据，以比人类更快的速度发现欺诈交易。随着时间的推移，它会越来越快，越来越准确地检测。

人工智能的另一个潜在领域是面向客户的服务台。能够访问票据解决历史和设备数据的智能系统可能能够处理大多数请求，而不需要占用实际技术人员的时间。如果人工智能无法解决问题，它可以将问题升级到技术人员。随着这些工具的成熟，它们也在影响客户服务流程；让人工智能尽可能地发挥作用，然后将不太常见和/或更复杂的问题上报给技术人员。这是一个很好的例子，说明了人工智能和信息技术如何合作来提供最好的结果。

AI 和 ML 工具是 IT 团队的未来。这些工具将能够处理大量目前由人完成的繁琐工作，如解析日志、监控流量和提供某种程度的技术支持。这使得它可以专注于任务关键型问题和人工智能和人工智能无法轻松解决的问题。
