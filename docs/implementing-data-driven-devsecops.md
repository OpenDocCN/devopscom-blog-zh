# 实现数据驱动的 DevSecOps

> 原文：<https://devops.com/implementing-data-driven-devsecops/>

现在，DevSecOps 通常实现的方式根本不适合快速敏捷的 DevOps CI/CD 管道。这就像将 19 世纪的灭火方法应用于现代森林火灾。

那时候，消防队员雇佣了一支“水桶队”，他们会排成一队，把水桶从一只手传到另一只手去灭火。毫无疑问，这需要大量的工作，但是大部分的努力都被浪费了。当水桶从一个人手中传递到另一个人手中时，水不可避免地会溢出，当水桶被倒入火中时，一半的水已经用完了。不仅浪费了如此多的努力，而且对于抗击我们今天面临的这种野火来说，速度太慢，效果太差。

同样，当前 DevSecOps 计划的大部分手动方法在对抗现代移动应用程序面临的数字威胁和网络攻击方面是无效的。就像现代的特大火灾一样，这些威胁和攻击每秒钟都在增长和变化，寻找新的载体在其他地方引发新的攻击。传统的 DevSecOps 工具，如代码扫描和渗透测试，可以识别漏洞，然后安全团队启动手动“水桶大队”，在应用发布前尽可能地增加保护。但是威胁和 CI/CD 进程都没有暂停。添加的新功能带来了新的漏洞，威胁格局也发生了变化。它的效率很低，因为 pentesting 无法提供关于攻击和威胁的实时数据，而开发人员需要这些数据来防范当前的威胁。结果是易受攻击的移动应用的发布。

## 数据驱动的流程

在商业领域，[首席执行官](https://digitalcxo.com)正在努力将他们的公司转变为数据驱动的组织，在这种组织中，决策不是基于直觉或专家意见，而是基于对硬数据的分析。移动应用的安全实现也需要采用同样的方法。

此外，在移动应用中实现安全性需要自动化。随着如此多的 CI/CD 流程已经实现自动化，安全性也不能落后。

这两个元素结合起来创建了数据驱动的 DevSecOps。在这种方法中，开发和安全团队拥有一个记录系统，可提供有关该领域应用程序的实时网络攻击和网络威胁信息，从而推动团队就最紧急且必须包含在下一版本中的保护措施做出决策。

现在，移动应用程序开发人员完全有可能收集关于他们的移动应用程序在现场面临的确切威胁和漏洞类型的近实时信息。当结合位置、网络和其他类型的数据时，开发人员不仅可以详细了解他们的应用程序遇到的最常见的威胁，还可以了解哪些威胁在特定的地理区域最普遍。他们还可以主动识别快速增长的威胁，这些威胁将在不久的将来成为一个巨大的问题。

有了这些数据，开发团队就可以做出明智的决定，在下一个版本中应该优先考虑哪些保护，以充分利用他们的时间和资源，为最终用户提供最大限度的保护。

## 对自动化的需求

但是有数据还不够。开发团队需要能够根据所提供的见解快速采取行动，手动实施方法太慢、太麻烦，无法跟上移动应用快速变化的威胁形势。一旦做出决定，还必须有一个系统来自动整合 CI/CD 管道内的移动应用安全、反欺诈和反欺骗保护，以便安全实施与功能创建一样顺利。

数据驱动 [DevSecOps](https://devops.com/?s=DevSecOps) 的优势有很多。首先，发行商、开发者和安全团队可以准确地了解网络罪犯对他们自己的真实用户应用程序做了什么。有了这些数据以及组织可以从其应用程序中收集的大量其他数据，他们可以确定哪些威胁是最紧迫的，哪些威胁正在快速出现，甚至它们在不同地理区域的分布情况，所有这些都是在决定在下一版本中构建哪些保护时非常有价值的信息。

最后，一旦应用程序发布，团队可以使用他们收到的关于威胁和攻击的数据来证明所包含的每种保护的有效性和价值。这不仅是持续改进的重要信息，也是证明数据驱动的开发团队对管理层和高管层的价值的重要信息。

借助数据驱动的 DevSecOps，组织可以向 CI/CD 流程中的所有利益相关方提供有关安全性的数据、透明度和可见性，同时使安全性的整合更加高效和有效，并提供实时数据来证明这一点。
