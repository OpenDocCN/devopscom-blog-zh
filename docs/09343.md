# Red Hat 在 Azure 上添加托管的 Ansible 服务

> 原文:[https://devo PS . com/red-hat-adds-managed-ansi ble-service-on-azure/](https://devops.com/red-hat-adds-managed-ansible-service-on-azure/)

Red Hat 今天宣布在微软 Azure cloud 上提供 Red Hat Ansible Automation 平台的托管实例[。](https://www.businesswire.com/news/home/20220510005642/en/Red-Hat-Delivers-Managed-Ansible-Automation-for-Microsoft-Azure)

在 2022 年红帽峰会[活动上宣布，该产品使在](https://www.redhat.com/en/summit)[微软 Azure](https://devops.com/?s=Microsoft+Azure) 云上部署 Ansible 的控制平面成为可能，作为对亚马逊网络服务(AWS)或内部 it 环境的现有支持的替代方案。

此外，在自动化平台的 2.2 版本中，Red Hat 正在预览对内容签名技术的支持，该技术可以自动验证内容以提高整体安全性。

红帽 Ansible 副总裁托马斯·安德森(Thomas Anderson)表示，这项托管服务使 IT 团队能够利用集成的微软计费，以及为该平台提供的最新功能和内容以及为优化微软 Azure 环境而添加的其他功能。

开源 Ansible 平台作为一个自动化框架已经获得了越来越多的关注，它使 IT 团队能够以声明的方式自动化任务。它比竞争对手的自动化框架需要更少的编程专业知识，Anderson 说这是这么多第三方供应商采用 Ansible 来自动化一系列 It 任务的主要原因。

如今，许多组织面临的挑战是管理跨多个云和内部 IT 环境的多个自动化孤岛。Anderson 说，Ansible 为 IT 团队提供了在单一框架下统一所有这些孤岛的能力。

目前还不清楚采用竞争对手自动化框架的 DevOps 团队会在多大程度上愿意用 Ansible 取代它们。然而，随着越来越多的 it 团队采用自动化来弥补 IT 人员的短缺，Ansible 显然正被广泛用于从开发运维到网络管理的各种 IT 领域。

无论采用哪种方法，随着 IT 环境变得越来越复杂，自动化尽可能多的任务的需求变得至关重要。IT 团队自己创建的许多脚本根本无法扩展。更具挑战性的是，它们通常不被记录，所以每次创建脚本的 IT 专业人员离开组织时，维护脚本代码就变得很困难。

当然，从长远来看，自动化框架将越来越多地注入能够学习个体 IT 环境独特细微差别的机器学习算法。这些算法不太可能很快取代对 It 专业人员的需求。然而，越来越多的 IT 专业人员并没有抵制自动化，而是得出结论，认为所需的任务对于手动执行来说太具挑战性了。许多人会得出这样的结论:他们只是更愿意为拥有一致的 it 自动化方法的组织工作，这种方法使他们能够管理广泛的 IT 任务。

无论如何，IT 自动化正在成为一项标准功能，现在已经远远超出了一些孤立的开发运维工作流的范围。问题是要找到一种方法，让具有不同技能水平的 IT 专业人员能够在一个既容易访问又高度可扩展的通用框架上进行协作。
