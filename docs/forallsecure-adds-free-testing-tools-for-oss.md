# ForAllSecure 为 OSS 添加免费测试工具

> 原文：<https://devops.com/forallsecure-adds-free-testing-tools-for-oss/>

ForAllSecure 正在投资使[开源软件](https://devops.com/?s=open%20source%20software) (OSS)更加安全，并向任何使用 GitHub 库的人提供[免费的个人使用版本](https://www.prnewswire.com/news-releases/forallsecure-launches-2-million-mayhem-heroes-program-to-make-open-source-more-secure-301518096.html)的 Mayhem 应用程序安全测试工具，其中注入了人工智能(AI)功能。

公司首席执行官大卫·布鲁姆利(David Brumley)表示，作为公司 Mayhem 英雄计划的一部分，任何将 Mayhem 集成到用于构建开源软件的合格 GitHub 存储库中的人都将获得 1000 美元。

Mayhem 现在有两个免费的个人使用版本，可用于分析代码或应用程序编程接口(API)。这些工具基于模糊测试技术，该技术将数据随机注入到应用程序中以检测漏洞[，作为自动化 DevSecOps 流程](https://digitalanarchist.com/videos/featured-guests/autonomous-security-tools-forallsecure)的一部分。

随着影响企业 IT 组织的开源软件中零日漏洞的发现，人们越来越关注如何创建、维护和保护开源软件。许多构建和维护开源软件的开发人员通常没有很多网络安全专业知识，也没有必要的时间和资源来持续修复发现的漏洞。Mayhem 应用安全测试工具旨在使用人工智能技术更快地发现漏洞，以减少误报警报的数量。

现在说开源软件多快会变得更安全还为时过早，但是大量的资源正在变得可用。开源安全基金会(OpenSSF)是 Linux 基金会的一个分支，已经筹集了超过 1000 万美元来开发工具和定义保护开源软件项目的最佳实践。谷歌承诺投入 100 亿美元来提高开源软件的安全性。拜登政府还通过扩大合规性要求，提高了政府机构内部(和外部)使用的开源软件的安全性。白宫还试图鼓励 IT 供应商和大型企业为保护开源软件做出更多贡献。

开源软件的开发者和消费者都面临着确保开源软件安全的压力。然而，许多开源项目是由少数程序员维护的，他们贡献自己的时间和精力来构建其他人可以免费使用的组件。他们中的许多人认为，部署该软件的组织有责任确保该软件是安全的。许多依赖该代码的 it 供应商和大型企业 IT 组织没有为该项目做出任何有意义的贡献，无论是在融资方面还是仅仅帮助开源维护者发现和修复漏洞方面。具有讽刺意味的是，许多同样的组织现在正在评估他们使用的开源软件，从安全的角度来看，在没有这些贡献的情况下，实际上是否是可持续的。
