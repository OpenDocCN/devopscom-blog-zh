# 多云计算是一个主要的 IT 难题

> 原文:[https://devo PS . com/multi-cloud-computing-is-a-major-it-health/](https://devops.com/multi-cloud-computing-is-a-major-it-headache/)

CloudBolt Software [对 500 名高级 IT、DevOps、安全和财务主管的调查](https://www.globenewswire.com/news-release/2022/02/22/2389450/0/en/The-Data-Is-In-Cloud-Has-Hit-a-Wall-and-There-s-Only-One-Way-to-Vault-Over-It.html)发现，79%的人在管理多个云计算环境的重压下苦苦挣扎。

云管理平台提供商 CloudBolt Software 的调查特别发现，超过四分之三的受访者(78%)表示，他们在整个企业中有太多的云工具，80%的人难以全面了解云的使用和支出。80%的受访者还指出，他们当前的云计算方法产生了新的安全漏洞。

整整 88%的受访者表示，他们需要一个“经理中的经理”，使他们能够统一在其 IT 环境中成倍增长的各种孤岛。

然而，78%的受访者表示，与在内部 IT 环境中运行应用程序相比，迁移到云使他们能够实现成本节约。

CloudBolt Software 首席执行官杰夫·库考斯基(Jeff Kukowski)表示，问题不在于 IT 组织后悔将工作负载转移到云上；相反，问题是找到一种方法来管理多个云，每个云都有自己需要掌握的一套工具和框架。

尚不清楚组织是否有意采用多种云，或者是否由于简单的惯性，他们发现自己由于各种开发团队做出的决定而管理多个云平台。例如，一些 IT 组织采用多个云，因为他们担心要么被锁定在某个特定的云计算服务上，要么担心停电会使他们的所有应用程序同时离线。其他人只是试图利用某个供应商比另一个供应商提供的更好的服务。

无论 IT 组织如何发现自己管理着多个云，随着每个新平台添加到 IT 环境中，IT 的总成本都会上升。CloudBolt Software 正在为提供抽象层的 IT 框架[提供案例，通过该抽象层可以统一多个云和内部 IT 环境的管理](https://devops.com/cloudbolt-software-aims-to-bridge-devops-and-it-divide/)。Kukowski 说，这更多地是一个时间和程度的问题，It 组织最终将在单一控制平面下整合不同 IT 环境的管理，这为真正的混合云计算提供了基础。

当然，有一些组织已经在单一云计算平台上实现了标准化。例如，随着针对特定云环境进行优化的新工作负载的构建，或者随着公司收购在其他云平台上运行工作负载的其他组织，这些组织维持该标准的程度自然会有所不同。在许多情况下，组织发现调用的云服务暴露了一系列专有的应用程序编程接口(API)和不同的虚拟机，这使得迁移工作负载成为一大挑战。

不管喜欢与否，在可预见的未来，大多数组织都将管理多个 it 环境。唯一不确定的是，在任何给定时间，这些平台中的任何一个上可能运行的工作负载量。
