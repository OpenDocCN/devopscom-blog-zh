# 可观测性和安全性的融合对开发人员意味着什么

> 原文:[https://devo PS . com/what-the-convergence-of-observability-and-security-means-for-devs/](https://devops.com/what-the-convergence-of-observability-and-security-means-for-devs/)

电影 [*中有一个场景，阿波罗 13 号*](https://www.imdb.com/title/tt0112384/) 任务控制飞行主任问为什么指令舱中的二氧化碳洗涤器与登月舱中使用的形状不同(因此不兼容)。工程师简单地回答道，“这根本不是我们曾经考虑过的意外。”

当他们检查安全和开发实践时，类似的场景在许多组织中上演。当代码进入生产阶段时，每个开发人员都希望他们的代码能按计划运行——顺利无误。不幸的是，在将应用程序部署到生产环境后，会遗漏一些东西，性能可能会降低，或者会出现新的、无法预料的问题。

开发人员害怕“未知的未知”这让他们夜不能寐。

当一个“未知的未知”问题发生时，团队需要时间来进行取证以识别问题。团队可能需要几天甚至几周的时间来完成最初的调查。如果一个任务关键型应用程序由于取证调查而长时间离线，那么它可能会对您客户的体验产生负面影响，并最终影响您组织的利润。

“未知的未知”问题是开发人员在当今不可预测的宏观环境中面临的另一个挑战。这包括海量数据、日益增加的网络威胁、日益分散的基础设施、远程和混合劳动力，以及现代云原生和单一应用的混合。

为了帮助开发人员构建未来并解决安全问题，他们必须重新思考从各种来源收集、操作和存储不同类型数据的方式。

### 安全性和可观察性:一起更好

如今，安全性和[可观察性](https://devops.com/?s=observability)最近开始重叠，这是由于组织越来越需要更好地了解其环境内部的活动。许多具有前瞻性的组织正在运营当前生成的大量日志和事件数据，以了解和评估其基础架构和应用中的问题，并使用这些可操作的见解来优化工作负载、应用资源可用性、安全性和正常运行时间。

那么，开发人员如何利用安全性和可观察性的融合趋势呢？

目前，开发者拥有两个代理作为他们技术堆栈的一部分是很常见的。一个可观察性代理，它收集日志和事件等数据，以实时了解应用程序的运行状况和性能。另一个代理是安全代理，在他们的开发测试、集成和生产环境中收集数据。

由于这两个代理经常收集相同的数据，因此有机会将其整合到单个位置，例如具有单个代理的统一安全平台，其中这些数据可以覆盖时间序列分析，以向用户提供相关的见解。使用这种将安全和非安全数据放在一个地方的方法，将大大加快将这些有价值的见解传递到开发人员手中的过程。

除了更快地获得洞察力之外，将这些数据放在一个位置的另一个好处是能够访问历史数据。让开发人员能够长期存储和管理数据，使他们能够从这些历史数据中获得洞察力，从而更好地了解警报、事件和对手。这可以帮助他们了解应用程序面临的风险，例如潜在的“未知的未知”威胁，他们可以在造成任何损害之前主动解决这些威胁。

采用融合的安全性和可观察性方法是组织可以采取的最成熟的步骤，以帮助开发人员为其客户和用户提供最佳结果和最佳体验。随着越来越多的组织依赖云以及业务速度的快速增长，开发人员必须转变他们对应用程序和基础架构的运行状况、性能和安全性的看法。给予开发人员这种视角将极大地帮助他们在不利事件影响到他们公司的底线之前采取快速行动。
