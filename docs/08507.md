# 2020 年预测:2020 年值得关注的基础设施和运营趋势

> 原文：<https://devops.com/predictions-2020-infrastructure-and-ops-trends-to-watch-in-2020/>

今天，系统现代化不仅仅是简单地将技术转移到一个新的地方。它需要新一代技术和工具的 It 堆栈才能工作。

随着新的一年的到来，让我们来看看将决定我们如何开发和交付软件的八大趋势。

## **Kubernetes**

是的，很大，而且越来越大。到 2020 年，企业采用这种领先的容器编排解决方案的速度只会越来越快。从 KubeCon 2015 年(500 人)到 KubeCon 2019 年(12，000 人)，观众人数增加了 2，300%。随着越来越多的公司开始看到 Kubernetes 的灵活性、可扩展性、自动化、高可用性和可移植性带来的直接财务和运营好处，对这一复杂软件的开发人员和工程专家的需求也将相应增长。一个典型的例子是:在 LinkedIn 上进行的 2019 年全国求职搜索中，出现了 16744 个与 Kubernetes 相关的职位[](https://enterprisersproject.com/article/2019/1/kubernetes-jobs-9-facts-and-figures-0)。鉴于 Kubernetes 社区已经很大并且仍在发展中，我们还应该看到功能发布的速度和数量的增加。

## **微服务**

据预测 到 2022 年，90%的应用将采用微服务架构。软件工程团队在越来越短的周期内生产和发布软件的日益增长的业务需求——连续交付——已经为微服务的巨大增长铺平了道路，微服务是 CD 的理想架构。随着企业在 2020 年及以后继续加速向云迁移，他们越来越重视尝试微服务提供的新技术堆栈的灵活性。所有企业都希望优化资源，最大限度地减少停机时间，降低基础架构成本，并避免技术或供应商锁定。到 2020 年，随着从 monoliths 迁移的速度加快，微服务架构将继续提供更多更好的优势。

## **服务网格**

微服务的兴起带来了服务网格的平行繁荣。这只会加速，因为当前的服务网格类别(Istio、Linkerd、Consul、Envoy)对于微服务的成功运行是多么不可或缺。作为一款旨在使组成应用程序的数千个微服务能够相互交流的软件，它很快成为厂商生态系统中一个必不可少的组成部分，为大幅增长做好了准备。由 IBM、Google 和 Lyft 支持的功能丰富的 Istio 是领先的 Kubernetes 服务网格，并且在 2020 年没有改变的迹象，特别是考虑到设置和运行它相对容易。如果您的企业已经踏上了从整体服务到微服务的旅程[](https://learning.oreilly.com/library/view/monolith-to-microservices/9781492047834/)，它将很快与服务网格交叉。

## **可观察性**

在最近的过去，现代分布式系统的复杂性已经超过了允许团队观察它们的工具。输入可观察性——这不是监视。就像好的直升机父母一样，软件工程师、应用程序开发人员、SREs 和 DevOps 团队希望随时了解和理解他们的系统在做什么。仅举几个例子，LightStep、Datadog 和 Honeycomb 是准备为快速增长的构建云原生系统的企业团队提供日志记录、跟踪和度量解决方案的供应商。密切关注 OpenTelemetry，这是一个开源的可观察性框架，目前是 CNCF 沙盒的成员。

## **SRE**

这个[](https://cdn2.hubspot.net/hubfs/5595333/Research%20Papers/2019%20SRE%20Report.pdf)新兴领域面临的巨大挑战之一是，不同规模和工作文化的企业要找到一种有效的 SRE 形式。在谷歌取得的成功可能并不是在小型科技企业或初创公司实施 SRE 的秘诀。SRE 的角色以及随之而来的期望仍然不确定。SRE 对自动化的持续需求只会增加诸如 Terraform、Ansible 和 Jenkins 等关键 SRE 和 DevOps 工具的知名度。深思 [亚马逊 Prime Day 2018 年亏损高达 9900 万美元](https://www.businessinsider.com/amazon-prime-day-website-issues-cost-it-millions-in-lost-sales-2018-7) 对于未来几年 SREs 的业务关键重要性有一个清晰的认识。就业市场反映了这一新的现实。

## **安全**

软件开发生命周期中的左移运动开始了。附加的后期制作安全的日子正在迅速消逝。尽管 DevSecOps 仍然是一个时髦的术语，但随着更多的实践和用例加入到理论中，它将继续存在。对于支持软件开发工作流的自动化应用程序安全工具来说，这里有巨大的机会。

另外还有隐私问题。《GDPR》于去年 5 月生效，《加州消费者隐私法》于 2020 年 1 月 1 日生效。除此之外，还有 HIPAA 和企业在安全领域面临的法规遵从性压力，这些压力已经非常大，再加上云安全的巨大挑战。2019 年赛门铁克关于 [云安全威胁](https://www.symantec.com/security-center/cloud-security-threat-report) 的调查报告称，云中 65%的企业仍未使用多因素身份认证。同一调查还指出，令人震惊的是，85%的赛门铁克云客户没有采用安全最佳实践。期待企业在流程的早期关注安全性。

## **无服务器**

随着 2014 年 AWS Lambda 的推出，谷歌的 Functions、微软的 Azure Functions 和 IBM 的 OpenWhisk 都提供了他们的无服务器解决方案，这是无服务器的繁荣时期。期待更多企业在升级传统系统时采用无服务器优先的方法，并看到大量的成本节约和应用程序开发优势。初创公司也将看到无服务器带来的成本削减优势和灵活性。作为无服务器的结果，应用程序开发和云中架构的变化是很难夸大的。有了现在提供的供应商解决方案，开发人员可以自由地做他们最喜欢做的事情:更快地运行代码并向他们的应用程序添加功能，而不用担心可扩展性和可用性。像微服务一样，随着整体服务器时代的消逝，无服务器服务的使用将会大幅增加。

## **DevOps**

随着云计算的需求以及企业对专门团队的投资大幅增加，2020 年的 DevOps 将从政策和原则过渡到实际实践。将内置安全性添加到 DevOps 领域(DevSecOps)增加了构建、供应和部署安全软件自动化的紧迫性。ML 和 AI 驱动的应用已经到来，DevOps 实践将在开发工作流中发挥重要作用。零接触自动化的目标迫在眉睫，这增加了对干净数据、组织买入和更多集成系统的需求的紧迫性。AIOps 虽然得到了大量的关注，但仍然很年轻，其对 IT 运营的好处仍然是理论多于实际，但请保持关注。

随着 2020 年的到来，看到这些趋势的发展将会令人兴奋。开发运维团队可以期待我们开发和部署应用的方式——在任何架构中——都将继续发展。通过掌握 Kubernetes、微服务和其他新技术的新发展，他们将为将其业务带入新的十年做好充分准备。
