# 梭鱼网络调查:开发警察很难

> 原文：<https://devops.com/barracuda-networks-survey-devsecops-is-hard/>

采用 DevSecOps 流程的组织在提高网络安全方面可能正在取得进展，但这绝非易事。Dimensional Research 代表 IT 安全软件和设备提供商 Barracuda Networks 对 618 名 IT 决策者进行了一项调查，发现一半的人采用了 DevOps、DevSecOps 或持续集成/持续开发(CI/CD)平台。整整 93%的 IT 决策者表示，他们面临着将安全性集成到这些实践中的挑战。

列举的主要挑战包括:现有的安全技术没有与应用程序开发环境集成(63%)，应用程序开发人员对安全性不放心(48%)，以及安全流程没有发展(48%)。

Barracuda Networks 负责公共云的副总裁蒂姆·杰斐逊(Tim Jefferson)表示，这种挫败感很大程度上可以归咎于对防火墙的依赖，这些防火墙没有应用编程接口(API)或与公共云天然集成。每个云服务提供商都有一个独特的环境，要求 IT 安全软件提供商编写与该平台本地集成的代码。IT 组织不能简单地剪切和粘贴为内部 IT 环境创建的安全控制，这主要是因为云提供商已经在基础架构级别实施了自己的控制。IT 组织需要为应用层开发他们自己的控制措施，每个控制措施都必须位于云服务提供商实施的控制措施之上。

在缺乏这些功能的情况下，开发人员会感到沮丧，因为在公共云上的虚拟机上运行的防火墙向他们提供的是图形用户界面(GUI ),而不是 API。Jefferson 说，云时代的开发人员不会使用任何他们不能通过 API 以编程方式调用的技术。

毫不奇怪，调查发现，要求最多的防火墙功能是云原生集成(74%)、云开发人员易于部署和配置(59%)、管理内部 IT 环境和公共云之间的流量(56%)以及分布式策略执行(53%)。

因为许多 IT 专业人员不具备这些能力，所以他们中的许多人认为内部 IT 环境比公共云更安全。超过一半的受访者(56%)表示，内部网络安全仍然优于公共云。很明显，这些评估中有很多技术和情感因素，但是不管真相如何，感知仍然是真实的。

然而，无论采用哪种方法，在云中部署防火墙的组织都希望按消费为这些服务付费，这并不总是与在虚拟机上运行防火墙，然后将其部署在公共云上相一致。

Jefferson 表示，随着云时代的网络安全不断成熟，人们对防火墙应该能够提供的功能的期望越来越高，特别是在控制进出公共云的流量时。

显然，现在任何与 DevSecOps 有关的东西都还在发展。但是，无论网络安全的未来如何，内部部署和云安全之间的关系远比许多人意识到的更加微妙。
