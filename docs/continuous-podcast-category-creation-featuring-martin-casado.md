# 成为连续的播客:马丁·卡萨多特色的类别创建

> 原文：<https://devops.com/continuous-podcast-category-creation-featuring-martin-casado/>

在这一集的《未完待续》中，保罗和伊迪丝将与[安德森·霍洛维茨](https://twitter.com/a16z)的普通合伙人[马丁·卡萨多](https://twitter.com/martin_casado)一起。他之前是 Nicira 的联合创始人兼首席技术官，该公司于 2012 年被 VMware 收购。在 VMware 期间，Martin 曾担任网络和安全业务部的高级副总裁兼总经理。

该小组讨论了非常复杂和困难的类别创建过程，特别关注技术基础设施产品。

这个播客是由 heavy bit(T1)带给你的，这是一个帮助以开发者为中心的公司将他们的产品推向市场的项目。

* * *

伊迪丝·哈博:你喜欢连续交货的什么？

**Martin Casado:** 我认为围绕持续交付有一种更广泛的趋势非常有趣，即传统上，基础架构和运营来自一个独立的团队。我本科毕业后的第一份工作是在一个国家实验室从事模拟代码工作。

我无法控制任何基础设施。我真的没有预算去买一支铅笔。你得到了基础设施，你有一些应用程序要编码，你要在那个应用程序上工作。我们会摆弄我们的米格文件什么的，差不多就是这样。

现在关于广泛运动的酷的事情，我认为 CI/CD(持续改进/持续交付)只是其中的一部分，是越来越多的应用程序，应用程序开发人员，实际上正在承担运营的责任。

这是供应，这是交付，这是部署。这是一个巨大的变化，不仅是创造了什么样的技术，而且是谁在使用它以及他们为什么使用它。我认为这实际上是目前更有趣的趋势之一。

伊迪丝:这是你自我介绍的好时机。

马丁:我叫马丁·卡萨多，是安德森·霍洛维茨律师事务所的普通合伙人。

伊迪丝:但是你在安德森之前就做了一些事情。

马丁:没错。正如我提到的，我曾经是劳伦斯利弗莫尔国家实验室的研究科学家。我离开了那里，在斯坦福大学攻读博士学位。我的研究方向是网络，所以我是启动软件定义的网络的团队的一员。

我在那里工作了四年，从那里脱离出来，创办了一家名为 Nicira 的公司，我们在那里做软件定义的网络。我这样做了五年。被 VMware 收购。我在 VMware 负责网络安全业务部门将近四年，四个月前，我决定做出改变，成为一名风投。

伊迪丝:所以你基本上创建了一个类别。是什么驱使你做那件事？是什么让你决定开始一件事，尽管知道它很难？

Paul Biggar: 你在谈论软件定义的网络？对于在家的观众，马丁粗略地发明了软件定义的网络。

马丁:有一大群人都做出了贡献，但我认为很多最初的动力来自我的论文。你知道，我会完全诚实。如果我知道这有多难，我绝不会这么做。

伊迪丝:真的吗？

马丁:我想是的。

伊迪丝:我想你会的。

马丁:嗯，也许吧。但那会更令人生畏。我从斯坦福毕业时相当天真，就像，“这很容易！”现在，九年过去了，头发都白了，我才意识到，伙计，这是一场战斗。我不认为我有四分之一是容易的。一切都是一场战斗。

我是这样想的。如果你是一名企业家，你打算做品类创造，除了产品开发，你基本上只有两份工作。第一项工作是，会有一些你要销售的客户群，而这些客户群，他们每天早上醒来都会想到各种各样的事情，但他们不会想到你的东西，因为它还没有被创造出来。

思维空间中的那个对象，那个对象甚至还不存在于他们的大脑中。您必须做的第一件事是创建该对象。就像《盗梦空间》里的莱昂纳多·迪卡普里奥你进去了，你就在别人脑子里植入了一个想法。它必须成为一种东西，一种人们思考的东西，一种人们了解并整合的东西。

这是从无到有，这很难。对我们来说，这花了五年多的时间，它触及了思想领导力的各个方面，如分析师关系、技术会议和广泛营销。

**Paul:** 所以你在这个类别创造中所做的活动很大程度上是营销，让人们听到并了解这个活动？

马丁:我想是的。我想也许技术专家会认为营销是一件非常肤浅的事情，但我认为你是完全正确的。是营销。但往往是非常技术性的营销，比如打着开源的幌子。

在发布开源项目的过程中，你会让人们使用它，你会去找标准机构，你会去参加会议。这是一个令人难以置信的技术问题，但最终，从某种意义上说，这是营销，你实际上是在创造意识和市场。

作为一名企业家，你要做的第一件事是，如果你在创造品类，没有人会考虑你的东西。你必须让他们考虑你的事情，这是一个很大的工作量。以我的经验，五到七年。

然后你要做的第二件事，也是同样多的工作，就是你要给这个东西赋予一个价值。我认为我们作为技术人员，有这样一种谬误，“我要创造这个新的部件。”假设我创造了 iPhone。

保罗:每个人都会知道这将会有多棒。

马丁:完全正确。

保罗:只要建起来，他们就会来。

马丁:是的，我的产品有内在价值。我把我的小工具放在人行道上，然后有人把它捡起来。他们说，“这太棒了。值几百万呢！”事实完全不是这样。

事实是，只有当人们兑换了钱并且使用了它，他们才会看到价值。

如果你在做品类创建，人们怎么知道那个价值是什么？这是一件非常困难的事情。

保罗:你说的附加价值是指货币化吗？或者还有其他原因？

马丁:我指的是更多的想法。最终，这实现了货币化。假设您是我的支持者，我创建了这个新的小部件。首先我要让你认为这是真的。我打算把它叫做新手机。所以，你需要这部新手机。首先我需要说服你你需要新手机。

然后，我需要说服你的第二件事是，它对你来说价值数万美元，因为你需要在精神上真正赋予它这种价值。事实证明，引入新概念不一定有价值，如果它们是新概念的话。

通常，该值来自于实际的交易。围绕这个有一个心理学。或者至少有一些使用它的经验，知道它将会，我不认为你可以只是坐在沙发上实验，就像，“哦，我认为这将会节省我的钱，因此它是值得的。”真的需要付出很多努力才能产生价值。

**Paul:** 因此，我看到正在创建的类别，或者至少是最接近我的类别，是容器和 Docker 的想法，更具体地说，是 Kubernetes，那里有云基础设施的想法。我意识到它的方式正是你谈论它的方式。你可以使用开源软件，你也可以自己动手。那里有让人大吃一惊的谈话。

马丁:我认为实际上这一点，当然对所有听众来说，我认为这是这十年来最突出的一点，实际上，这将是一件很奇怪的事情。至少基础设施是这样的。

在过去，我们如何创造思想份额和客户群？嗯，有这整个 pmm 功能和这整个销售功能。如果你看看大型企业公司，如果你看看思科、IBM 和甲骨文，他们真正的力量就是这个功能。他们拥有销售渠道。

他们与分析师有很好的关系，他们与客户有很好的关系，他们有认证和课程。所以每当他们有一个新的概念，他们就把这个概念塞进机器。

作为一家创业公司，你的技术有多酷并不重要，最难的是真正渗透到那个渠道。

但当谈到这个开源的新时代，现在你实际上有了这种以前从未存在过的黑客，买家正在从运营商转向开发商。开发者喜欢开源，实际上你可以通过开放源代码并让他们使用它来创造这些概念。

保罗:我们一直在讨论这类事情，只是认为现在是自下而上还是自上而下。

马丁:不仅如此，这是一个完全不同的买家，这是一种完全不同的审美。我不认为开发人员会如此关心分析师。我不认为开发人员会关心那个 42 岁、长着白牙和船的销售人员。我不认为开发人员会关心这些。

保罗:他们甚至不接他的电话。

马丁:完全正确。关于渠道，我觉得他们不在乎复杂的采购流程。

伊迪丝:我自己也卖过一点软件，如果你进入一家更大的公司，你仍然不会开始碰到这种情况。

保罗:你会到达那里，因为开发者首先要求它存在。

伊迪丝:在我们的销售周期中，最长的部分就是你被采购和法律部门踢来踢去。

马丁:绝对是。我认为这是这个论点最重要的细微差别，正是你所说的。我一直认为开源和开发者正在取代市场营销和售前，而不是销售。

你获得了客户的信任，你可以从技术上完成交易和技术验证，你可以获得客户拉动。所以我仍然相信，这就像是给管道加油，完成交易。在很长一段时间里，你需要一个专业的、直接的销售团队，在那里推动采购。

**伊迪丝:**这只是很多工作，因为你通过了技术评估。他们会说，“好吧，去找采购部吧。”对采购部来说，你只是个小工具。

马丁:没错，但好消息是，现在有可能了。在过去，获得这么多的合法性，让事情在技术上接近，是非常困难的。你没有认证，你没有接触买家的渠道，你没有客户代表来做介绍。你什么都没有。

伊迪丝:的确如此。

**马丁:**现在你做到了，现在只是实际通过物流。你完全正确，这很难，这是巴洛克风格。我称之为部落流血。所有这些都是，但你可以作为一家初创公司来做。

保罗:我认为真正有趣的是事情的反过来。现在，在开源领域工作的人正在成为不可思议的营销人员。他们并不这样称呼自己，但他们在社交媒体上有着惊人的影响力。他们建立了大量的渠道，无论是邮件列表还是推特粉丝。

伊迪丝:好像每个人都在重现营销。每个人都喜欢，“哦，我们不需要营销。”然后你会想，“好吧，你有一个电子邮件列表的这个东西是什么？”哦，那不是营销。

保罗:这不是营销，因为我是一名开发人员。

马丁:我真的认为那些知道如何做的人是整个行业中最有价值的人，因为这是产品进入客户群的新入口。我的意思是，这里有另一种潜在的陷阱，也就是说，你使用开源软件进行营销。

现在，大概有 70 亿美元被投资到开源公司，而只有大约 10 亿美元的回报。我不确定这些数字是不是真的，但我总是听到有人引用它们。但事实是，大量的资金投入其中，却没有多少钱回来。

原因是因为现在，如果你使用开源来做营销，你几乎是在蚕食你的销售活动。所以在很长一段时间里，我们知道如何工作的唯一开源模式是红帽模式，这是一个服务公司。只有红帽成功过。这非常困难。

但是在过去的几年里，另一个转变发生了，人们已经认识到，实际上你可以使用开源来进行营销，但是你不必把它和你实际提供的产品混淆起来。

随着服务的兴起，你实际上可以使用开源来吸引客户。你可以将服务货币化，当涉及到服务时，不要讨论开源或闭源。所以我认为，我们实际上已经开始开启可行的商业模式，在这种模式下，你可以使用开源进行营销。

伊迪丝:你认为谁在这方面做得好？

马丁:我认为 GitHub 是一个很好的例子，对吗？Git 甚至不是由 GitHub 构建的。这是 Linux 的标准。但是如果你看看 GitHub，它是一家发展非常迅速的基于开源的公司，因为 Git，每个人都知道它。所以 Git 在做营销。而这只是一个例子，我觉得还有很多。

保罗:有趣的是，你说这是基于开源的，因为他们的堆栈几乎没有一个是开源的。

马丁:没错，这是一项服务。我就是这个意思。

保罗:当你说服务时，你指的是软件即服务，而不是咨询中的服务？

马丁:绝对是。我应该说得更清楚些。对我们来说，一个长期存在的问题，当然是从 1995 年开始，就是你如何建立一个开源公司，拥有一个闭源软件公司的利润？因为你是开源的，基本上你变成了一个服务和支持公司。

过去十年发生的事情正是你所说的:人们会意识到，你可以通过开源获得客户可信度和客户吸引力，以及客户知识。但是，您可以提供后端服务产品作为服务产品，并将其货币化，这通常是您正在做的后端的封闭源代码堆栈。

但是对于服务产品，你很少谈论开源/闭源。通常情况下，如果你在本地吃东西，你会想到这些东西。

保罗:你对此有何看法？建造的人，让我们说 CircleCI 是一个很好的模型，但是他们有很多。构建一个 SaaS 产品，一个托管的 SaaS 产品，然后销售给在他们自己的 VPC 和伪本地使用它的人。

伊迪丝:天啊。

马丁:我来自这个有点传统的销售企业软件的世界。在那里，有一个完整的讨论，我也是其中的一部分，讨论它是在本地还是一般在异地？我认为你完美地提出了这个问题。我意识到这是一个错误的二分法。这实际上不是一个合适的问题。我认为作为一种服务，它实际上是一种运营模式和交付模式吗？无论是在本地还是在异地，都没有关系。

我认为向公司出售闭源软件的日子已经结束了。

但是你可以向他们出售服务，实际上你可以向他们出售设备。但是如果你看看电器，电器看起来几乎像服务。它们是提供服务的机器，升级通常是远程进行的，管理也是远程进行的。

我确实认为这是这些公司的正确模式，我认为 CircleCI 是一个很好的例子，你有客户使用的开源软件，你有 SaaS 产品，但你也有本地交付产品，只要你不改变运营模式，只要它看起来仍然像 SaaS 模式，无论是托管服务还是类似的东西。

我认为，一旦它成为可发布的软件，在这种情况下，您必须管理一组异构的环境，您必须管理所有的第二天操作。它变得笨拙。

前几天，我和 CircleCI 的销售副总裁以及微软的一些人进行了一次非常有趣的讨论。CircleCI 基本上是这样的，“我们喜欢它在内部销售。它在帮助你。”微软就像是在说，“我们正拼命地离开本地。”所以这不是你认为会发生的争论。

保罗:我想这可能是因为这个 IP 的敏感性。如果你说的是像这样的事情，那么，文档是相当敏感的，但是源代码是非常敏感的。因此，人们并不像我们曾经认为的那样乐于将产品运送到异地。

Edith: 微软的理由是，这可能是微软的做法，他们有 10 年的维护协议。

马丁:我们有，“如果你有问题，升级到最新版本，然后我们会处理你。”我认为这是每个人都有的。过去有这样一种想法，你会创建一个分支，然后你会将东西反向移植到那个分支或旧版本，或者类似的东西。

我不知道有谁在做这种服务，SaaS 有一个托管的本地版本，除了“最新版本就是版本”之外，它什么也不做
