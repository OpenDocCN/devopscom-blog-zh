中国所有的抖音表示，将阻止在俄罗斯直播和上传新内容，此前克里姆林宫将传播其认为的有关入侵乌克兰的假新闻定为刑事犯罪。……英国广播公司、美国有线电视新闻网和其他全球媒体机构已经暂时停止在俄罗斯的报道，以保护他们的记者。抖音发言人 Hilary McQuaide 表示，这款应用现在出现在“仅观看”模式下，不允许人们发布或观看新视频或直播。他们仍然可以看到旧的视频，但如果它们来自国外就不行了。

俄罗斯总统弗拉基米尔·普京周五签署了一项新法律，禁止该国所谓的关于军方的“假”新闻。该法律将针对任何将入侵乌克兰称为“入侵”的言论，任何试图诋毁武装力量的行为，或呼吁制裁俄罗斯的行为。那些被发现传播所谓假新闻的人可能面临长达 15 年的监禁。

## 2.专利巨魔仍在攻击

“专利主张实体”，也称为“非实践实体”，是不生产任何东西，只是简单地收购和囤积专利的组织——后来对不知情的组织主张专利。目的是向他们声称侵权的人勒索金钱。

问题？许多专利申请就像一只生病的小猫一样虚弱无力。

### 分析:PAE 又名 npe 是 NBG-SNAFU

你可能认为你是安全的，但是巨魔也会追踪有争议技术的用户。例子包括播客、JPEG 图像、订单确认电子邮件、汽车地理定位，甚至**缓存数据**。

史蒂文·沃恩-尼科尔斯 : **开源区在专利巨魔面前苦苦挣扎**

专利流氓，又名专利断言实体(PAEs)，已经困扰了…软件很长时间了。…攻击不断升级。原因很简单:有利可图。由 200 多家企业组成的国际组织“统一专利”向巨魔发起了挑战。它通过使专利巨魔的获胜成本过高来阻止专利巨魔攻击其成员。…随着联合专利公司在淘汰劣质专利方面的高成功率，专利流氓正被缓慢但肯定地击退。
……
例如，一项加速内存技术专利 6，513，062 被 troll IP Investments Group 用来声称……管理云上缓存资源的 Redis 侵犯了该专利。[巨魔]追踪 Hulu、Citrix Systems、Barracuda Networks、Kemp Technologies 和 F5 Networks 使用 Redis 软件的情况。IP 投资集团放弃了，而不是战斗到底。每个使用 Redis 的人都是赢家。

禁止软件专利？T2 叔叔认为，小心不要把婴儿和洗澡水一起倒掉:

专利本身是一个好主意:……有创造力的人应该因为他们对经济的贡献而受到奖励。

【但是】专利巨魔是寄生虫，对公民的繁荣毫无贡献。…不幸的是,“专利巨魔”商业计划相当可行。你不发明任何东西，也不资助新的发展。有了更好的律师，你就有足够的资金击败任何挑战你的人。

## 3.Chrome Mac Perf 上的谷歌 Gloats

macOS 上的谷歌 Chrome 比 M1 MacBook Pro 上的 Safari 拥有更好的基准分数。搞笑的是，这是在用苹果自己的 ***车速表*基准工具**。

### 分析:基准并不是一切

伟大的公关，和值得爆米花库比蒂诺 trolling。但是谷歌难道没有更好的优化方法吗？

弗雷德里克·拉迪诺瓦:**MAC OS 上的 Chrome 现在比 Safari 还快**

macOS 上的 Chrome 99 版本在速度表基准测试中获得了 300 分，该测试最初由苹果的 WebKit 团队开发。谷歌指出，这是所有浏览器中性能最快的。浏览器市场的竞争聚焦于速度已经有一段时间了。…但是，当新的芯片出现时，总会有优化的空间。最初，苹果 M1 芯片上的 Chrome 性能尚可，但是……Chrome 现在运行速度快了 43%。
……
速度计 2.0 测试响应性，这使其成为用户体验的良好代理。…让软件感觉更灵敏并不需要太多，所以 M1 MAC 电脑上的 43%肯定会让你感觉更有效率。

如何？尽量不要幸灾乐祸，[麦克斯·克里斯托夫](https://blog.chromium.org/2022/03/a-new-speed-milestone-for-chrome.html "read the full text"):

基于去年的许多性能变化，我们在 M99 中启用了 ThinLTO，这是一种构建优化技术，可以内联代码库的速度关键部分，即使它们跨多个文件或库。…最近对 Chrome 速度做出贡献的另外两个主要因素是 V8 斯派克普拉编译器和短内置调用。斯派克普拉是一个新的中间层 JavaScript 编译器。V8 JavaScript 引擎使用简短的内置调用来优化生成的代码在设备内存中的位置，通过避免调用函数时的间接跳转来提高性能。我们知道，性能指标评测只是衡量浏览器速度的众多方法之一。[但是]结合最近的图形优化(即直通解码器和进程外光栅化)… Chrome 现在比… 17 个月前快了 43%。

基准/标杆，认为[代码库](https://news.ycombinator.com/item?id=30598115 "read the full text"):

浏览器开发者在优化错误的东西。…优化速度通常会导致内存消耗增加，这增加了调用交换的机会。如果系统开始交换，速度会慢几个数量级。如果浏览器可以在使用两倍 CPU 时间的情况下使用[一半]的内存，那将是完美的。试想一下，有多少拥有 2gb 或 4 GB 内存的笔记本电脑会再次变得可用。
