# 德勤在可观察性方面与 Dynatrace 保持一致

> 原文:[https://devo PS . com/Deloitte-aligns-with-dyna trace-for-observability/](https://devops.com/deloitte-aligns-with-dynatrace-for-observability/)

德勤[扩展了其云可观察性实践，将 Dynatrace 软件智能平台](https://www.prnewswire.com/news-releases/deloitte-expands-its-cloud-observability-practice-with-targeted-solutions-for-clients-301549658.html)纳入其中。

德勤董事总经理兼现代交付联席主席杰伊·麦克唐纳(Jay McDonald)表示，这家 IT 服务提供商现在将提供传统的 DevOps 和可观察性咨询专业知识，以及一组将代表组织管理的 Dynatrace 软件智能平台实例。

麦克唐纳说，德勤选择 Dynatrace 平台是因为它采用单一代理来收集指标、日志和痕迹，然后由戴维斯人工智能(AI)引擎进行分析，以实现[可观察性](https://devops.com/s?=observability)。

虽然人们越来越重视可观察性，以使 DevOps 团队能够在 IT 问题成为主要问题之前识别它们的根本原因，但是部署可观察性平台需要大量的专业知识。McDonald 说，许多组织选择依赖第三方组织，如德勤，在更大的 DevOps 工作流程中管理这项任务。

他补充说，随着许多组织转向构建和部署基于微服务的应用，管理这些工作流对于他们的部署和维护来说变得尤其具有挑战性。

Dynatrace 全球合作伙伴副总裁 Michael Allen 表示，现在已经有超过 4000 家组织在使用 Dynatrace 软件智能平台，随着可观测性平台作为一种服务被消费，这一数字还将继续增加。

目前还不清楚 DevOps 团队接受可观测性的速度。这个概念一直是 DevOps 的核心原则，但是，在大多数情况下，更成熟的 DevOps 团队只能实现对预定义指标的持续监控。可观察性平台承诺在 IT 问题升级之前发现异常情况；然后，开发运维团队可以启动查询，以更好地确定其根本原因和潜在的严重性。

艾伦指出，戴维斯人工智能引擎的好处之一是，机器学习算法可以发现许多问题的根本原因，而不需要 DevOps 团队确切知道如何构建正确的查询。

然而，许多 DevOps 团队遇到的最大挑战是他们缺乏部署和管理可观测性平台所需的技能。事实上，他们中的许多人现在依靠第三方来管理各种各样的 DevOps 平台，以便他们可以将精力集中在实际的工作流上。雇佣和留住 DevOps 专业人员也变得越来越具有挑战性，因为正在构建和部署的应用程序总数已经超出了 DevOps 团队的有效管理能力。

无论如何，IT 团队通过排除过程来确定 IT 问题根本原因的日子终于要结束了。无论谁设置和管理可观察性平台，DevOps 团队都可以获得对其应用环境的更多可见性，从而获得更有意义的可操作情报。DevOps 团队有机会利用这种智能来提高整体应用弹性，并推动广泛的数字化业务转型计划。
