# 调查揭示了大量应用程序漏洞

> 原文：<https://devops.com/survey-surfaces-massive-number-of-application-vulnerabilities/>

软件漏洞自动修复平台提供商 Rezilion 今天公布了一项针对 16，510 名 IT 和 IT 安全从业者的调查,发现近一半的受访者(47%)在拥有大量易受攻击的应用程序的组织中工作。超过一半(66%)的人表示，他们的积压包括超过 100，000 个漏洞，平均需要补救的漏洞数量为 110 万个。

这项由 Ponemon Institute 进行的调查还发现，超过一半的受访者(54%)表示，他们只能修补不到 50%的漏洞。

阻止团队进行补救的因素包括无法确定需要修复的内容的优先级(47%)、缺乏有效的工具(43%)、缺乏资源(38%)以及没有足够的关于利用漏洞的风险的信息(45%)。

超过四分之一(28%)的人还认为补救措施太耗时。调查发现，77%的受访者表示，仅检测、优先处理和补救生产中的一个漏洞就需要 21 分钟以上的时间。

此外，85%的人说他们花费超过 16 分钟来检测开发中的一个漏洞。整整 82%的受访者还指出，修复开发中的一个漏洞需要 21 分钟以上的时间。大多数受访者表示，修复应用程序中的漏洞非常困难(36%)或困难(25%)。

另一方面，超过一半(56%)的受访者表示他们使用自动化进行漏洞补救，43%的受访者表示响应时间因此显著缩短。

Rezilion 首席执行官 Liran Tancman 表示，该调查清楚地表明仍有很大的改进空间。他说，需要解决的漏洞数量惊人。他指出，IT 团队需要将当前的工作重点放在识别最关键的漏洞上。

随着更多专注于保护软件供应链的法规生效，补救这些漏洞将很快成为更高的优先级。塔克曼说，事实上，这些法规要求的[软件材料清单](https://devops.com/?s=SBOM)(sbom)将暴露当前存在于应用环境中的许多脏衣服。

最终，目标需要是使组织能够解决漏洞问题，而不降低应用程序的构建和部署速度，塔克曼补充说。随着更多的安全责任转移到应用程序开发人员身上，实现这一目标的程度会有很大的不同。然而，Tancman 指出，向开发人员提供工具，使他们能够解决漏洞问题的需求从未如此明显。

很明显，组织需要解决的应用程序安全漏洞债务的水平不能再被忽视了。现在的问题是确定如何最好地解决这些问题，要么修复现有的应用程序，要么用新的应用程序替换它们。不管怎样，很明显需要采用 DevSecOps 最佳实践来更好地保护软件。

与此同时，随着应用程序安全性成为董事会层面的问题，整个软件供应链将面临更多的审查。
