随着 2021 年的临近，我们 DevOps.com 想要突出今年最受欢迎的文章。以下是我们 2021 年最佳系列的第三部。

Linux 确实偶尔会引发安全问题。虽然许多用户认为它是最安全、最健壮和最通用的操作系统——这也是作者的观点——但安全预防措施仍然必须采取。

最近一个广为人知的案例说明了这一点；Linux 创建者 Linus Torvalds 警告不要使用 Linux 5.12 版本。他描述了一个“令人讨厌的错误”，并写道，由于在添加 Linux 更新时使用了交换文件，情况变得“一团糟”。事实上，这个讨厌的 bug 有可能破坏整个根目录。

这场“混乱”之后的一些主要收获包括:安装早期 Linux 版本时要非常小心，尤其是那些涉及交换文件而不是分区的版本，尤其是，尽管 [Linux](https://devops.com/?s=Linux) 众所周知的安全优势，但不要自满，因为 Linux 安全并不总是万无一失的。

因此，尽管“今天的 Linux 安全状况相当好，并且已经以积极的方式发展，构建了更多的可见性和安全特性，但像许多操作系统一样，您必须在考虑安全的情况下安装、配置和管理它；这就是网络罪犯利用人际关系的方式，”特许访问管理(PAM)解决方案提供商 thy cotic 的首席安全科学家兼 CISO 顾问约瑟夫·卡森(Joseph Carson)说。

## Linux 的优点

事实上，Torvalds 对这个 bug 如此坦率，以及 Linux 内核提供的透明度，也证明了 Linux 仍然受欢迎的众多原因之一。提供网络安全和合规软件的新网络技术公司(NNT)的安全研究全球副总裁德克·施拉德说，鉴于 Linux 内核以这样或那样的形式“不仅用于世界上约 50%的互联网服务器，而且用于我们所有智能手机的很大一部分，很高兴看到这种‘根级别’的透明度。

“Linux 的安全性基于它的透明性；“审查发行版代码的能力，”施拉德说。“经常被遗忘的是，透明度还包括谈论错误、失误和那些讨厌的缺陷。”

引用[国家标准与技术研究所(NIST)](https://www.nist.gov/) [漏洞数据库统计](https://www.cvedetails.com/)，施拉德描述了与 Windows 系列桌面和服务器操作系统相比，例如，Linux 内核在整体漏洞方面显示出更好的结果。根据 NIST 国家漏洞数据库，漏洞的数量在过去四年也有所下降，而微软的操作系统没有表现出同样的趋势。

由于 Linux 著名的内核是开放源代码和透明的，因此可以推断，与那些监控封闭系统中的漏洞的监管者相比，潜在的漏洞监管者数量更多。有些人可能会说，微软有时在检测漏洞和发布急需的补丁方面不太成功。

然而，Linux 用户仍然必须保持警惕。

“尽管如此，对于任何 Linux 发行版，任何使用早期版本候选的人——特别是 RC1——都应该确保他们自己的开发或构建过程正在接受变更控制，这样就不会有任何事故将令人讨厌的错误转移到生产环境中，”施拉德说。
