# 不完全买 Web3？你并不孤单

> 原文:[https://devops.com/not-totally-buying-web3-youre-not-alone/](https://devops.com/not-totally-buying-web3-youre-not-alone/)

对 Web3 的兴趣正在升温。我们已经看到围绕[区块链](https://devops.com/some-trends-to-anticipate-for-blockchain-in-2022/)和新的 [Web3 协议](https://devops.com/how-do-we-bring-apis-into-decentralized-web3/)的重大兴奋，它们将重塑我们所知的网络。但很难隐藏怀疑和怀疑大型加密货币硬币筹款者背后有不可告人的动机。毫无疑问，Web3 运动是有好处的——web 3 的去中心化性质带来了共享所有权的创新形式。然而，营销炒作和投资热情都可能阻碍该行业的实际进展。

我最近会见了 Netlify 首席执行官马特·比尔曼，讨论了 Web3 缺失的部分。对于那些不知道的人来说，Netlify 一直是 JAMstack 运动的大力支持者，这是一种融合了 JavaScript、API 和 Markdown 的设计风格(因此有了 JAM)。无头方法将前端和后端解耦，实现了更简单的多平台支持和其他好处。

根据比尔曼的说法，Web3 背后的理论是令人兴奋和合理的，它非常适合无头运动。然而，Biilmann 说，在实践中，Web3 将需要更加强调互操作性和开放标准。如果没有链间的可移植性和更好的数据控制，这种趋势很可能会成为支撑当前网络的围墙花园和集中化的牺牲品。关于 Web3 和区块链，还有一些其他需要注意的问题，我们将在下面探讨。

## 回到开放网络

首先，Web3 到底是什么？嗯，Web3 是一个宽泛的术语，包含松散相关的现象，但它可以总结为一个新的网络，它是用去中心化的技术设计的，建立在拥有分布式所有权的区块链之上。“Web3 运动背后有一种普遍的感觉，即作为一个非集中化的实体，它可以追溯到 Web 的根源，”Biilmann 解释道。

在 Web2 中，用户是在参与别人的围墙花园，而 Web3 背后的概念是它从根本上是开放的。但是，我们如何在没有单一所有者的情况下，使更多的堆栈部分成为协作标准呢？根据比尔曼的说法，这是该行业应该扩展的方面。

## Web3 的问题

有很多猜测。比尔曼预见到的 Web3 的一个大问题是猖獗的投机。建立在区块链基础上的新 Web3 项目的长期成功被这样一个事实所迷惑，即创造者希望人们购买代币，这样代币就会升值。“Web3 的基础有多少是真正解决实际问题的，有多少是纯粹的投机性投资？”他问道。在浮华、别致的外表背后，区块链的解决方案是否能解决真正的问题，这一点并不总是显而易见的。

缺乏链间的互操作性。许多新兴的区块链风格之间缺乏互操作性也可能给这个新兴的空间带来风险。“在多大程度上，他们实际上是在围绕标准构建东西，有可能从一个玩家转移到另一个玩家？”比尔曼问道。"我们需要开放的标准，你可以从一个移植到另一个."

**集权依然存在**。无论围绕分权的文案怎么说，在资本主义经济中，仍会有玩家拥有更多控制权和权力。根据 Biilmann 的说法，重要的是使平台从根本上开放和可扩展，以避免创建更多的围墙花园。他承认，只要这些[开放平台](https://nordicapis.com/why-api-platforms-should-be-open/)提供互操作性，拥有强大的中央提供商并不是一个大问题。

**缺乏身份可移植性**。当一个人拥有一个 web 域时，他可以在服务器之间自由地移动内容。同样，Biilmam 认为，在 Web3 中，类似的自由对于身份认同是必要的。理想是增加个人数字身份的所有权和可移植性。然而，比尔曼质疑区块链是否是实现这一目标的最佳工具，尤其是在区块链协议可能会干扰便携性的情况下。

**区块链仍将依赖于外部来源的权威**。另一个问题是区块链将如何面对现实。通常，一个智能契约会指向一个外链上下文的 URL，而不严格保证外链发生了什么。如果区块链必须依赖一个外部来源的权威，这在某种程度上违背了一个相互依赖的网络的理想。“大多数时候，在区块链发生的事情和外部世界发生的事情之间，你必须相信一个提供者，”比尔曼解释说。在这种情况下，区块链的完整性最终可能会围绕一个单一的实体。

将区块链与外部世界联系起来很困难。关于这一点，与外链结构接口的确切过程仍在开发中。这通常依赖于第三方 oracle 模型，尽管[已经取得了一些进展](https://devops.com/how-do-we-bring-apis-into-decentralized-web3/)来消除 oracle 概念，以支持集成链外计算的更现代的标准。

**每一次操作都会产生很高的处理成本**。区块链需要过多的处理能力来完成简单的任务，这在货币和资源方面都是大规模的浪费。

**NFT 的好处被大大夸大了**。NFTs 不太可能帮助大多数创作者。正如[金泊利·帕克](https://thatkimparker.medium.com/most-artists-are-not-making-money-off-nfts-and-here-are-some-graphs-to-prove-it-c65718d4a1b8)解释的那样，NFT 平台的销售数据显示“极少数艺术家通过少量销售赚取了巨额财富，而大多数艺术家被卖给了一个被可怕地夸大了的巨大利润的梦想。”

[NFT 骗局很常见](https://futurism.com/nft-developer-absconds-millions-dollars)，由于铸造一个 NFT 需要高昂的平台费，艺术家收回前期投资的可能性微乎其微。“我认为受益最大的是像文克莱沃斯双胞胎这样的人，他们已经是百万富翁，拥有一个 NFT 平台，”猫·格拉夫姆在 Time.com[节目中说。“最大的输家是绝大多数试图打入 NFT 销售市场的未成名艺术家。”](https://time.com/6093982/nft-art-teens-money/)

## 保留可供选择的方案

“我们不要妄下结论，认为加密货币和区块链是解决方案，”比尔曼说。正如他指出的，几十年来，人们一直在修补开放身份和数据的方法。这些思考者可能也有相关的见解和不应该被忽视的替代解决方案。

例如，万维网和 HTML 编程语言的发明者蒂姆·伯纳斯·李一直在倡导一项名为 [Solid](https://solid.mit.edu/) 的新计划。Solid 是一套“基于关联数据构建分散式社交应用的约定和工具”该项目的目标是实现真正的数据所有权和更好的隐私。“如果我们追求开放网络的目标，我们应该同样好奇，想知道他们多年来一直在研究什么，”比尔曼说。

Biilmann 说，我们可能会看到 Web3 项目的出色只是因为对加密货币的潜在投机兴趣，而不是因为它们在技术上是最好的解决方案或拥有最好的开放所有权模型。其他解决方案包括用于大容量分布式存储的 [IPFS](https://ipfs.io/) 。或者，以 DNS 为例，它是网络核心的分散式命名系统——“现有网络的这一部分已经非常开放，属于用户所有，”比尔曼说。

## 目标:种植没有围墙的花园

“我对此感到兴奋——就我个人而言，我对这项技术没有任何怨恨，”比尔曼澄清道。“让身份和底层数据从根本上变得更加开放是非常棒的。”

Biilmann 说，但是，与此同时，这个领域的投机性太强，离为用户创造有用的东西太远了。区块链行业也迫切需要互操作性和可移植性。由于对代币的兴趣引发了如此多的噪音，社会信任一个更中立的团体来构建我们的数字生活结构可能会更有益。

“如果人们忘记了我们已经可以建立的东西，那将是一种耻辱，”比尔曼说，他指着像 IPFS 和 DNS 这样已经存在的技术。尽管这些开放标准不会让早期的利益相关者在短期内变得富有，但是它们可以帮助我们更快地达到开放状态。

Web3 仍处于早期阶段。如果我们将货币与这种雄心分开，它可能会带来巨大的成果。“让我们弄清楚如何让身份和数据公开，而不是局限于特定的围墙花园，”比尔曼说。“作为一个行业，这是我们应该努力的方向。”
