# 云迁移的现实:在变好之前会变得更糟

> 原文：<https://devops.com/the-reality-of-cloud-migration-it-gets-worse-before-it-gets-better/>

国税局上个月占据了头条新闻，当时在纳税日出现了计算机故障，导致该机构无法收集数百万份纳税申报表，并迫使最后期限延长。但是在这些可预见的“尴尬的政府混乱”的故事之下，隐藏着一个更大、更系统的问题:IT 现代化。具体来说，*不*现代化的潜在后果。

现代化的优势众所周知:敏捷性、灵活性、创新性和可伸缩性。但是也可以理解为什么组织——特别是像国税局这样的大型组织——对现代化犹豫不决。该组织几十年来一直依赖相同的计算机和系统。在很大程度上，这些计算机和系统已经工作了几十年。随着变化而来的是风险和不确定性，更不用说大量的时间和精力了。虽然技术供应商说现代化是“无缝的”和“容易实现的”，但我们知道这不是全部。事实是，情况在好转之前会变得更糟。

## 残酷的现实

云迁移就是一个典型的例子。我曾经共事过的一位首席信息官将遗留应用程序比作埃及金字塔:庞大、复杂，而每个知道它们是如何建造的人都早已不在了。除了基本技能挑战之外，您和您的团队还必须加倍努力，在迁移到云的同时保持现有运营的正常运行。残酷的现实是，当您将传统应用程序迁移到云时，您的工作负载的总规模和复杂性会先增加后减少。

成功的云迁移始于直面以下事实，以便为未来做好准备。

### **这份工作将会比它初看起来更大**

就像金字塔一样，你从挖掘和清点开始。您需要深入您的应用程序，了解它们的依赖性以及它们在您的业务中的当前角色。这是一项耗时的工作——您拉起一个线程，它通常不会结束，直到您清点了大部分应用程序。线很快变长，绕成圈，缠绕在你身上。你有的只是绳结。

把结一个一个解开。随着时间的推移，这条线会变直，你会对你的系统有新的认识。无论有没有云，这都是有价值的。在清点了您的产品组合之后，通过找到对业务来说独立但重要并且明显需要改进的系统来寻找快速的胜利。在您对重要但独立的应用程序进行了一些实践并取得成功之后，处理具有复杂依赖关系的系统。

### **日复一日颠倒过来**

团队通常最初都很渴望，但是当变革真正开始时，他们会退缩。人们不喜欢改变他们的责任，最初有许多未知，当然，双重责任意味着比以往更多的工作。不要隔离从事云迁移的团队，因为他们会发现很难获得他们需要的广泛支持。让移民成为每个人的责任。在这种情况下，领导力至关重要。透明而明确地传达目标和潜在障碍。获得反馈并进行调整，但不要忽视最终目标。以身作则，无论是参加更多直接下属的会议还是占用办公时间。保持信息流稳定，确保每个人都知道“为什么”和“什么”

### 你会在一段时间内跨越两个世界

成功的“大爆炸”转变是一个神话。在现实世界中，你不能一下子把你的传统技术推到一边，去寻找更新更好的版本。你也不能让你的传统应用程序像云原生应用程序一样运行。在向新方法现代化的过程中，您需要保持现状。明白你不会一夜之间变成云原生的连续发货店。接受时间和成本来维持你所拥有的，因为你正在向你想要的现代化迈进。

如果做得好，云中的迁移和现代化将提高效率，清理垃圾，加速创新。然而，要实现这些好处，您必须首先了解您今天拥有什么，然后领导您的组织进行变革。由于没有企业可以简单地从头开始，因此您必须计划在云迁移和现代化的同时维护现有系统。假设您会发现意外的依赖性、不情愿的参与者和您自己系统的不一致的安全状态，开始您的工作。预料到这些问题会让你和你的团队做好处理这些问题的准备。不要抓着云锤，像钉子一样对待一切——为每个应用程序使用正确的解决方案。最终目标不是让*进入*某事，而是超越业务需求。成功的云迁移没有单一的途径，因此请保持选择的开放性，让您的目标决定您的技术决策。在生活变得更好之前，为生活变得更困难做好准备，回报将会是值得投资的。
