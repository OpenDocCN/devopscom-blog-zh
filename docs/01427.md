# Jelastic PaaS 以同样的价格将 CPU 能力提高了一倍

> 原文：<https://devops.com/jelastic-paas-doubles-cpu-power-price/>

*je elastic 云托管在计算性能方面变得更加经济高效和优化*

加州帕洛阿尔托(PRWEB)2016 年 7 月 12 日

Jelastic 云平台 for DevOps 和 hosting business 激动地宣布，所有计算环境的 CPU 能力都增加了一倍，资源单位的价格保持不变。

每个 [Jelastic cloudlet](https://docs.jelastic.com/cloudlet) (动态和保留)现在拥有比以前多两倍的 CPU 能力。一个新的 Jelastic cloudlet 大小等于 128 MiB RAM 和 400 MHz 的 CPU。

“在对开发人员的应用需求进行更深入的研究后，我们决定为他们提供这样的奖励。增强的 cloudlet 能力使 Jelastic 的性能更高，价格更实惠。从现在起，最终用户将获得两倍多的 CPU 资源，并且他们的发票保持不变。”Jelastic 的首席执行官鲁斯兰·辛尼茨基说。

那么，这对 Jelastic PaaS 用户来说意味着什么呢？

Jelastic 云环境中每个容器的 CPU 性能提高了 100%。新的 cloudlet 规模调整变得更适合更广泛的应用程序和工作负载，包括 web 和游戏项目、分析服务器和数据库、构建和批处理等。

这些变化将显著加快大量使用 CPU 的云应用程序的速度。因此，Jelastic 云托管在计算性能方面变得更加经济高效和优化。

“许多 web 应用程序的 CPU 负载并不明显，因此我们发现客户会以非常小的 cloudlet 限制来部署它们。”提供英美服务的 Jelastic 白金合作伙伴 [Layershift](http://www.layershift.com/jelastic) 的服务总监达米安·兰塞姆(Damien Ransome)说。“我们的测试表明，提高峰值 CPU 功率通常会带来显著的性能优势。由于我们只对超过一小时的平均 CPU 消耗收费，这种节流效应对我们的客户来说不一定是显而易见的——但是仅仅因为您可以在一个缓慢的 CPU 上运行某些东西并不意味着您应该这样做！将每个 cloudlet 的 CPU 能力提高一倍，可以确保我们所有的客户都能从这种额外的性能提升中获益，而且不会增加额外的成本。实话实说吧；谁不想免费获得更好的性能？”

所有更改都会自动应用到现有的和新创建的环境中，因此不需要任何手动步骤来利用这些新的超高性能 cloudlets。

关于 Jelastic 资源消耗和定价的完整信息可以在[文档](https://docs.jelastic.com/pricing-model)中找到。Jelastic 中容器的当前性能可以通过[在一个云托管服务提供商](https://jelastic.cloud/)注册免费试用期来检查。

关于 Jelastic

Jelastic 是一款面向企业的 DevOps PaaS 和 CaaS，可以部署在裸机硬件或任何 IaaS 上。该平台为 Java、PHP、Ruby、Node.js、Python 和。NET 和使用定制 Docker 容器的能力。Jelastic 提供灵活的部署模式，无需对专有 API 进行编码，为无状态和有状态应用程序、协作、访问控制、监控、备份和灾难恢复、内置计费和业务分析工具提供灵活的自动扩展，同时通过高密度和硬件利用率降低总拥有成本。欲了解更多信息，请访问我们的网站[http://www.jelastic.com](http://www.prweb.net/Redirect.aspx?id=aHR0cDovL3d3dy5qZWxhc3RpYy5jb20=)
