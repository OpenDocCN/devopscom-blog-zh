# 选择事件管理平台

> 原文:[https://devo PS . com/choosing-an-incident-management-platform/](https://devops.com/choosing-an-incident-management-platform/)

当您感受到手动管理事件和事件响应的压力和痛苦时，决定寻找事件管理工具是显而易见的。但是，如何选择最适合您、您的团队和您的企业的方案呢？

你可能会问自己，“我从哪里开始？我需要知道什么？我问什么问题？有哪些选择？我如何确定我们选择了正确的工具？”

为了帮助确定最适合您的组织的解决方案，以下是需要考虑的四大事项。

**1。您是否拥有支持事件管理解决方案所需的文化和流程？**

不同的业务模式需要不同的[事件管理](https://devops.com/?s=incident+management)方法。您应该寻找一个平台来满足您组织的独特需求。更重要的是，即使你找到了一个看起来合适的解决方案，你有必要的人员、文化和流程来支持一个新工具吗？如果没有，您是否有计划将它们落实到位？

成功实施解决方案需要计划，因此您需要确保您和您的团队能够处理工作量。

**2。你的痛点和目标是什么？**

您寻找事件管理解决方案是有原因的，但是您确切知道您希望解决哪些棘手问题吗？你的最高目标可能是“让事情变得更简单”，但是每一个解决方案都可能吹嘘它们让事情变得更简单。确定你希望完成什么。您希望流程自动化吗？让你的团队更快地团结起来？跟踪从申报到追溯的每一个动作？以上全部？如果你把你想解决的问题按优先顺序排列，当你调查你的首选时，就更容易勾选这些选项。

**3。谁应该参与这一搜索？**

您可能是寻找更好的事故管理方法的领导者，但是知道谁应该参与决策过程会使寻找更有效。你需要知道你是否得到了团队和领导的支持；谁将批准新工具的预算，谁必须签署您的决定。

不要忘记团队之外的其他利益相关者！事故会影响组织的所有部分，因此考虑谁需要成为选择过程的一部分—从您的客户成功团队到您的销售团队。

**4。你将如何做出决定？**

寻找和实施事件管理解决方案需要时间和资源。尤其重要的是，要考虑到组建团队和配置解决方案所需的时间，包括分配个人角色、设置与现有工具的集成以及试驾您的[事件管理](https://securityboulevard.com/?s=incident+management)流程。

最糟糕的情况是你购买了一个平台，但是没有时间去实现和使用它。你不会开心，你的团队或高管利益相关者也不会开心。

因此，要清楚地知道做出最终决定需要什么，无论是考虑谁参与其中，你需要保证多少预算，你的时间表是什么，或者你将如何解决任何法律或安全问题。
