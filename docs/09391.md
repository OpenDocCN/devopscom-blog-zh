# Red Hat 向社区发布开源 StackRox

> 原文：<https://devops.com/red-hat-releases-open-source-stackrox-to-the-community/>

Red Hat 很兴奋地宣布,[Red Hat Advanced Cluster Security for Kubernetes](https://www.redhat.com/en/technologies/cloud-computing/openshift/advanced-cluster-security-kubernetes)现已作为开源项目推出: [StackRox](https://cloud.redhat.com/blog/red-hat-releases-open-source-stackrox-to-the-community?hs_preview=GJzYpWQz-69825238357) 。Kubernetes 的用户和开发者可以加入社区，在 [GitHub](https://github.com/stackrox/stackrox) 上测试和贡献项目的代码库。

Kubernetes 是云原生开发的基础，也是发展最快的开源项目之一。根据[云本地计算基金会(CNCF) 2021](https://www.cncf.io/wp-content/uploads/2022/02/CNCF-AR_FINAL-edits-15.2.21.pdf) 调查，96%的受访者表示他们正在使用或评估 Kubernetes。Red Hat Advanced Cluster Security 的开源为组织提供了一种帮助塑造未来的方式，即如何更好地管理和保护他们的 Kubernetes 环境。

**增强构建、部署和运行时的应用安全性**

过去一年的数据泄露和供应链漏洞再次证明了安全性对于构建和运行 Kubernetes 应用程序的组织是多么重要。StackRox 社区旨在提供一个开源项目，帮助用户尝试新的方法来保护他们的 Kubernetes 环境。

StackRox 项目旨在通过在开发和部署生命周期中集成安全功能，有效地将应用程序安全性“向左”转移到软件创建中，从而帮助简化 DevSecOps。这有助于在开发周期的早期更有效地识别和解决问题，同时在整个 IT 堆栈和应用程序生命周期中提供更具凝聚力的云原生安全性方法。该项目旨在鼓励采用 DevSecOps 原则，有助于解决常见的云原生安全挑战，包括可见性、漏洞管理、配置管理、网络分段、合规性、威胁检测和事件响应，以及风险预测。

Red Hat 计划成为该社区的活跃成员，StackRox 项目将成为用于 Kubernetes 的[Red Hat Advanced Cluster Security 背后的代码，该项目有助于推动未来的产品路线图。Red Hat Advanced Cluster Security for Kubernetes 为使用 Red Hat OpenShift 的客户提供企业级的 Kubernetes 原生安全性，以在整个应用程序生命周期中增强基础架构和工作负载的安全性。](https://cloud.redhat.com/products/kubernetes-security)

**开放式创新**

在 Red Hat，我们相信使用开放的开发模式有助于创造安全、稳定和创新的技术。这种对开源原则的承诺是我们今天商业模式的核心，StackRox 的开源是我们如何坚持开放代码和开发价值的另一个例子。

有关 StackRox 社区和未来更新的更多信息，请访问 [StackRox.io](http://StackRox.io) 。
