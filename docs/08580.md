# Snyk 扩展工具组合以推动 DevSecOps 的采用

> 原文：<https://devops.com/snyk-extends-tools-portfolio-to-drive-devsecops-adoption/>

在本周的 SnykCon 2021 在线会议上，Snyk 扩展了 Snyk Code，这是一个静态应用安全测试(SAST)工具，已经支持 Java、JavaScript 和 Python 编程语言，包括对 C#、Ruby、PHP 和 Go 的[支持。](https://snyk.io/news/snykcon-2021-gathers-thousands-of-developers-to-advance-the-global-devsecops-movement/)

与此同时，Snyk 开源(一个用于修复开源漏洞的平台)已得到扩展，以提供与 Atlassian BitBucket 和 AWS CodePipeline 平台的原生集成，用于驱动 DevOps 工作流。Snyk 还加强了与 DigitalOcean 和 HashiCorp 平台的整合。

Snyk 还增加了对 Elixir 编程语言和包管理器 Yarn 2 和 poem 的支持，以及与 foster id 的 C++扫描工具的集成，foster id 是 Snyk 今年早些时候收购的开源代码软件组合分析(SCA)工具的提供商。

Snyk 容器平台现在集成了开源的 Trivy 容器扫描工具和 Snyk 的漏洞数据库，此外还增加了对 Quay、GitHub Container Registry、GitLab、Google Artifact Registry 和 Harbor 等容器注册表的支持。

Snyk 基础设施即代码平台现在还可以通过与亚马逊网络服务(AWS)、微软 Azure 和谷歌云平台(GCP)的云平台兼容的方式，检测 Terraform 代码中 Kubernetes 清单中的配置问题。

Snyk 还增加了对 OpenAPI 第 3 版的支持，该公司称其速度是前一版本的三倍。该 API 也是该公司将继续推出的 Snyk 应用程序的基础。

最后，Snyk 推出了一个新的免费开发人员安全教育计划，称为 Snyk Learn，开发人员可以通过它获得并衡量他们的 DevSecOps 专业知识水平和 Snyk 影响，这是一项促进开发人员之间合作的努力，涉及广泛的社会经济问题。

公司总裁 Guy Podjamy 表示，随着 DevsecOps 的不断发展，主要焦点仍然是开发人员和 DevOps 团队。然而，作为一个问题，DevSecOps 在组织内的 C 级以及各个安全运营团队中也变得越来越重要。因此，在最近备受瞩目的安全违规事件之后，围绕 DevSecOps 的紧迫感大大增强了。Podjamy 指出，组织正在努力解决的根本挑战是找到一种方法来提高整体应用程序的安全性，而不减慢应用程序的构建速度。

Snyk 自己透露，它已经获得了额外的 5 . 3 亿美元融资来解决这些问题。迄今为止，该公司已经筹集了 7.75 亿美元，估值为 85 亿美元。Snyk 还透露，其年度经常性收入(ARR)同比增长了 154%，客户群现已覆盖 1200 多家组织，包括 Asurion、Google、Intuit、MongoDB、New Relic、Revolut 和 Salesforce。该公司声称，Snyk 的客户和用户在过去 12 个月中总共进行了超过 3 亿次测试，在 90 天的时间内修复了超过 3000 万个漏洞。

大多数组织在每个开发运维工作流中嵌入安全工具可能还需要一段时间。然而，考虑到投入 DevSecOps 的资源，最终实现 DevSecOps 可能更成问题，而不是如果实现。
