# 调查警告迫近的软件测试危机

> 原文：<https://devops.com/survey-warns-of-looming-software-testing-crisis/>

一项针对参与应用程序测试的首席执行官和 IT 专业人员的调查发现，在发布未经适当测试的软件的可接受程度方面，存在巨大差距。

这项调查由市场研究公司 Censuswide 代表测试自动化平台提供商 Leapwork 进行，调查了美国和英国 480 名首席执行官，他们来自员工人数超过 500 人的组织，以及 480 名质量保证、测试人员、发布和交付经理和站点可靠性工程师。

美国 85%的首席执行官和英国 69%的首席执行官表示，发布未经适当测试的软件是可以接受的，只要补丁经过测试并在以后应用。不到一半的受访 IT 专业人士(49%)同意这一观点。整整 94%的首席执行官还表示，他们相信发布的软件会定期测试。

然而，79%的 IT 专业人员承认，多达 40%的软件未经充分测试就进入市场。因此，68%的英国和 52%的美国 IT 专业人员表示，他们的团队每年花 5 到 10 天时间修补软件。

调查还清楚地表明，每个人都很清楚缺乏测试所带来的风险。几乎每个首席执行官(95%)都担心在软件故障后丢掉工作。略多于四分之三的受访 IT 专业人士(76%)有同样的恐惧。

超过四分之三的首席执行官(77%)还指出，在过去五年中，软件故障损害了他们公司的声誉。

如今，许多组织认为敏捷开发和 DevOps 最佳实践将使他们能够在出现问题时快速应用补丁。问题是，在缺乏足够的测试的情况下，许多组织实际上是在终端用户身上做实验。

Leapwork 首席执行官 Christian Brink Frederiksen 表示，如此多的软件没有得到正确测试的主要原因是大多数组织继续依赖手动测试过程，无法跟上应用程序开发的速度；一个主要因素是使用低代码和无代码工具。

在发布未经充分测试的软件的组织中，整整 40%的 IT 专业人员将测试自动化投资不足列为一个因素，只有 43%的受访者使用某种类型的自动化。缺乏适当测试的其他原因包括缺乏时间(34%)和部署率增加(29%)。

超过三分之一的首席执行官(34%)和大约十分之四的测试人员(42%)的公司使用或开发未经测试就上市的内部软件，他们也提到缺乏熟练的开发人员。超过三分之一的首席执行官(34%)还提到对测试人员的投资不足。

如今，越来越多的组织依赖软件来推动数字业务转型计划，很明显，应用测试危机迫在眉睫。唯一有待确定的是，由于现有流程无法应对挑战，其后果会有多严重。
