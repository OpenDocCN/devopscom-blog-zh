# 企业面临越来越多的技术债务

> 原文：<https://devops.com/enterprises-face-growing-technical-debt/>

技术专家总是支持新的、流行的开发解决方案。更健壮的框架，增加的交付，更好的集成。但是这些支持者很少考虑引入新技术的长期副作用。因此，技术债务在大多数公司都是司空见惯的事情。

将近三分之一的 IT 预算专门用于解决技术债务。将如此多的精力投入到维护遗留代码上，留给支持现状的资源就更少了，更不用说开发新的解决方案了。事实上，最近的一项研究发现，69%的 IT 领导者将技术债务视为对其公司创新能力的重大威胁。

“技术债务的威胁越来越大”，来自 OutSystems 的一份报告调查了 500 名 IT 领导对这个难题的看法。下面，我们将探索技术债务的主要原因，并考虑防止它失控的方法。

## 什么是技术债？

技术债务似乎违反直觉。技术不是应该帮助组织变得更有效率吗？为什么会成为负担？技术债务通常是由短期内做出的草率决定引起的，这些决定会产生长期的后果。这通常也是公司玩弄许多竞争技术的结果，其中一些已经过时。

技术债务可能会带来沉重的代价。但这种债务并不是以托管遗留技术或运行破损代码的成本来衡量的，而是以维护和修补这些技术所损失的时间来衡量的。该报告发现，平均而言，企业将 28%的 IT 预算用于解决技术债务。在大型企业中，这个数字增加到 40%。

## 常见原因

造成技术债务的因素不止一个。组织中的许多问题都会导致这个问题，包括对陈旧技术的依赖、短期交付的压力、不断的变化、开发人员流失和不正确的架构决策。

技术债务的主要原因是太多的开发语言和框架，52%的受访者认为这是一个大的或关键的问题。传统技术会拖累 IT 部门。但是，不一定只有旧技术在碍事，它可能支持太多的竞争议程。

第二大原因是开发团队内部的高离职率。在今天的竞争环境中，高质量的工程师供不应求，招聘也很有挑战性。因此很难吸引和培养稳定的工程人才。如果开发人员经常离开，特别是在记录他们的过程之前，最佳实践很容易丢失，并且技术的有效使用受到阻碍。

该研究发现，其他常见原因包括接受已知缺陷以满足最后期限，使用过时的编程语言和框架，以及应对服务新市场或细分市场的挑战。有时，团队发布带有已知缺陷的软件只是为了满足最后期限。大型组织的庞大规模会导致技术债务的增加，而不解决这个问题会使问题进一步复杂化。

就行业而言，报告发现医疗保健和社会援助类别受到的冲击最大。在这一领域，平均 35%的 IT 预算用于解决技术债务。第二个负担最重的群体是金融类，根据该报告，这一比例为 29%。

## 技术债务的成本

技术债务的成本是什么？维护成本很高，创新时间减少，对软件整体性能有不利影响。团队动态也可能招致持续技术债务的负面反应。

技术债务需要不成比例的大量时间来维持。平均来说，工程师花 33%的时间处理技术债务，Stripe 的"[开发者系数](https://stripe.com/files/reports/the-developer-coefficient.pdf)"研究发现。这相当于平均每周工作 13.5 小时。

负债越多，用于维持现状运营和建设新能力的预算就越少。支持技术债务不仅会产生财务影响，还会扼杀未来的前景；事实上，69%的受访者表示，技术债务限制了他们的创新能力。在今天的加速经济中，不为 T2 的数字创新留出资源可能会毁掉一个企业。

OutSystems 首席执行官兼创始人保罗·罗萨多表示:“旧代码与新一代移动应用、堆栈应用和 SaaS 蔓延的结合正在掠夺组织的资源、时间和创新能力。

技术债务也可能阻碍业绩。调查显示，61%的受访者表示，这严重拖累了他们公司的业绩。陈旧的项目可能会干扰精简[客户体验](https://devops.com/digital-customer-experiences-the-future-is-modular/)和[数字化转型](https://devops.com/the-role-of-apis-in-digital-transformation/)目标。

最后，技术债务也会对士气产生负面影响。很少有工程师以重复低效的流程或使用过时的技术为荣。

## 减轻技术债务:有意为之

与金融债务类似，技术债务也有复利。如果不加以控制，这种兴趣可能会成倍增长。那么，组织可以采取什么措施来减少他们的技术债务呢？

为了帮助缓解这个问题，该报告建议通过阻止开发团队的人员流动和限制你采用的新编程语言和框架的数量来扭转技术债务的两大原因。

其他方法包括评估您的资产，以了解遗留技术的存在和影响。接下来，重新思考并重新定位你的策略，从简单地维护遗留的依赖关系到解决它们并用现代的风格取代它们。

乔恩·桑顿在 Squarespace 博客上说:“很多不良技术债务来自于建设过多，并且在维护和漏洞修复上花费了比预期更多的时间。“关键是要有意识地投入时间，并意识到自己承担的成本。”

当然，企业 IT 必须承担某种形式的技术债务才能发挥作用——完全摆脱它是徒劳的。因此，正如桑顿建议的那样，要有意识地对待它，并意识到你所承担的成本。当采用新技术时，最好考虑维护问题，并在计划时考虑寿命。
