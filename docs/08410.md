# Google DORA 报告显示适度的 DevOps 收益

> 原文：<https://devops.com/google-dora-report-shows-modest-devops-gains/>

谷歌今天发布了由其开发运维研究和评估(DORA)团队创建的年度[加速开发运维状态报告](https://cloud.google.com/blog/products/devops-sre/announcing-dora-2021-accelerate-state-of-devops-report)，该报告发现，26%接受调查的开发运维团队现在可以被归类为精英执行者。相比之下，2019 年上次进行这项研究时，这一比例为 20%。

DevOps 团队由来自全球的 32，000 名 IT 专业人员组成，我们根据他们频繁部署、编码、变更准备时间、恢复服务时间、变更失败率和运营绩效的能力对其进行了评估。精英团队有望在停电不到一小时的情况下恢复服务。

谷歌用户体验研究员 Dustin Smith 博士表示，增长率的提高表明，尽管新冠肺炎疫情提出了各种挑战，DevOps 的成熟度仍在稳步提高。整整 96%的受访者表示，在疫情之前，他们没有定期在家工作。

《多拉报告》发现，高水平的精英演员占受访者的三分之二。与之前的研究相比，精英们能够集体缩短变革的准备时间。同样值得注意的是，只有优秀的员工将他们的变更失败率降到了最低。前几年，中高绩效者也能做到。

总体而言，超过一半的受访者(52%)报告他们在某种程度上采用了[现场可靠性工程](https://devops.com/?s=site+reliability+engineering) (SRE)最佳实践。

精英人员部署代码的频率是低水平人员的 973 倍，低水平人员每六个月部署不到一次。标准化的年度部署数量范围从表现最佳的每年 1，460 次部署到表现不佳的每年 1.5 次部署。不足为奇的是，对于优秀员工来说，从提交到部署的交付时间比表现不佳的员工快 6，570 倍。

安全实践也有所改善，60%的企业进行了安全审查，58%的企业报告审查包括测试。近三分之二(63%)的人说他们与[信息安全](https://securityboulevard.com/2021/09/state-of-devops-2021-benefits-of-a-secure-software-supply-chain/)团队密切合作。该报告发现，拥有高质量文档的团队在整个开发过程中集成安全性的可能性是其他团队的 3.8 倍。

对云平台的依赖也有所增加，56%的受访者表示他们使用公共云，比 2019 年增加了 5%。只有 21%的受访者报告部署到多个公共云。超过四分之一(26%)的受访者这样做是为了利用每个云提供商的独特优势，紧随其后的是 22%的受访者提到可用性。

略超过三分之一(34%)的受访者表示他们正在使用混合云，而 29%的受访者拥有私有云。总共有 21%的人仍然不使用公共云。

该报告还发现，精英员工满足美国国家标准与技术研究所(NIST)定义的所有基本云特征的可能性是普通员工的 3.5 倍。这些特征被定义为按需自助服务、广泛的网络接入、资源共享、快速弹性或扩展以及可衡量的服务。只有 32%表示正在使用云基础设施的受访者同意或非常同意他们符合 NIST 定义的云计算的所有五个基本特征，比 2019 年增加了 3%。总体而言，云计算的这些特征的使用增加了 14%到 19%，其中快速弹性增长幅度最大。

DORA 报告清楚地表明，在更广泛地实现 DevOps 涅槃之前，还有很长的路要走。然而，从积极的一面来看，在某种程度上采用 DevOps 的组织数量持续稳步增长。
