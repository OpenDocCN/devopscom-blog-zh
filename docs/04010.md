# Puppet 与 Google Cloud 合作，支持云中的迁移和管理

> 原文：<https://devops.com/puppet-partners-google-cloud-support-migration-management-cloud/>

**新的 GCP 木偶模块帮助新老木偶用户利用谷歌云平台**

俄勒冈州波特兰–( market wired–2017 年 10 月 10 日)–[Puppet](http://ctt.marketwire.com/?release=1322394&id=12288190&type=1&url=https%3a%2f%2fpuppet.com%2f)，为我们周围的一切提供动力的软件交付和操作自动化的标准，今天宣布与谷歌云建立新的合作伙伴关系，包括 Puppet 批准的模块，以加快云的采用和专门的工程团队。这种合作有助于 Puppet 客户快速采用谷歌云平台(GCP)服务，包括其先进的机器学习和数据分析功能，同时降低 IT 成本，获得云的敏捷性和灵活性。

随着越来越多的企业希望利用云的优势，在应用程序迁移到云的过程中对其进行迁移和管理仍然是一个挑战。迁移通常意味着在云中重新构建您的环境，这非常耗时，并且手动完成时容易出错。使用 Puppet，用户只需将基础设施定义为代码，就可以在内部和云环境中应用这些配置。Puppet 加速了向云的迁移，并提供了第 2 天管理，从而提高了动态混合环境中的运营效率。作为共同努力的一部分，客户现在能够利用一组谷歌开发的模块，自动提供、配置和持续管理 GCP 服务，以帮助傀儡客户快速有效地利用 GCP。

Google Cloud 最近在 Puppet Forge 上发布了一组 Puppet 批准的模块，用于管理基础设施即服务组件和 Kubernetes，包括；谷歌计算引擎(GCE)、云 DNS、云 SQL、谷歌容器引擎(GKE)、云存储和谷歌认证凭证。Puppet Approved 是对通过特定质量和可用性要求的模块的指定。在 PuppetConf 2017 上，Google Cloud 将发布额外的模块来增加 Google Cloud 产品的覆盖范围，下一组模块专注于数据处理，包括 Google Cloud Pub/Sub、Cloud Spanner 和 Stackdriver。在接下来的几个月里，他们将继续发布新的模块。

Google Cloud 还将发布他们用来生成模块的技术，以便木偶模块生态系统可以更快地移动，并跟上云中快速变化的 API。Google 的 Puppet 代码生成工具对模块开发人员来说是一个福音，因为它支持更快的开发和更广泛的覆盖。

谷歌工程经理埃里克·约翰逊表示:“随着企业转向云计算，他们正在寻找利用现有基础设施投资的方法。”。“借助我们针对 Google Cloud 的新 Puppet 模块，我们可以帮助客户更快速地迁移到云，并使用 Puppet 管理他们的内部和云环境。”

Puppet 首席技术策略师 Nigel Kersten 表示:“我们的客户需要选择、灵活性和能力来管理他们拥有的一切，从物理基础架构到云资源，以实现最大的运营效率和规模。“凭借 Google Cloud 在提供世界一流基础设施方面的专业知识和 Puppet 广泛采用的企业管理平台，我们正在帮助客户加速向云迁移。”
