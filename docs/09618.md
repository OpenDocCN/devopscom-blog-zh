# 开源技术的好处

> 原文:[https://devo PS . com/the-benefits-of-open-source-technologies/](https://devops.com/the-benefits-of-open-source-technologies/)

在过去的 20 年里，我们已经看到了[开源软件](https://opensource.org/)和技术的激增，它们成为商业产品的一部分，或者是为产品提供动力的技术堆栈的一部分，或者是产品本身的一部分。尽管这两个用例有很大的不同，但它们都展示了开源是多么有用，不仅对开发人员社区如此，对企业也是如此。

使用开源技术或项目的好处之一是自由分享想法。开源将人们聚集在一起，集思广益，开发一项共同的技术。对他们所做事情的共同热爱跨越了文化和地理的界限。最著名的——也可以说是最重要的——开源项目之一是 Linux 内核和操作系统。Linus Torvalds 通过邮件列表向开源社区发布了第一个版本的内核，引发了世界范围的协作开发和广泛采用。

对开源的热情源于一个根深蒂固的信念，即贡献者正在解决一个重要的问题，他们正在为改善整个世界而努力。在许多情况下，这是显而易见的。Linux 内核现在支持全球大多数数据中心，为数十亿用户提供服务。但是，在建立开源技术的过程中，不仅仅是主观热情发挥了关键作用。开源项目的同行评审过程意味着每个代码提交都要通过一个严格的、公开的审计和验证过程。事实证明，这是提高代码质量、减少错误和退化以及防范漏洞的极好方法。[研究表明](https://www.zdnet.com/article/coverity-finds-open-source-software-quality-better-than-proprietary-code/#:~:text=Open%20source%20code%20quality%20surpasses,standard%20for%20good%20quality%20software)，客观地说，这一过程比标准的、专有的质量保证和测试程序产生更高质量的软件。

当选择在开发中使用开源技术时，为了保证产品的质量，需要考虑几个问题。一个关键的考虑是不要追逐最新的技术潮流。相反，要为工作选择合适的技术。开发人员的 DNA 中有对令人兴奋的新技术的痴迷，但这并不总是意味着它们是合适的。虽然如果你需要在管理维护和成本的同时，以网飞那样的规模为数百万用户提供服务，Kubernetes 可能是正确的解决方案，但它可能不是发布其产品第一次迭代的新创公司的最佳选择。在这种情况下，开发速度和进行更改的灵活性比可伸缩性更重要(因此，初创公司使用 Ruby on Rails 或 Django 可能会更好)。这三个项目都有一个重要的共同特征:它们有活跃的社区，不仅保护项目特性，还维护它们并提供对等支持。

当用特定的技术实现一个新特性时，这个支持社区非常重要，因为他们可以极大地加速你的产品的开发。这些人热爱他们所做的事情，并希望看到其他社区成员在这项技术上取得成功。在这方面，考虑回报你受益的项目和社区也很重要。您可以参与社区并自己一对一地帮助他人，或者通过分享您对开源技术的改进来做出贡献。例如，如果您已经扩展了 API，或者发现并修复了 bug，那么没有理由不反馈您的更改并进一步丰富生态系统。你以这种方式建立的信誉也是一种好处，因为人们会把你、你的团队和你的产品视为开源项目的支持者。这是一种良性的合作循环，带来的好处不可低估。

随着您和您的产品成为开源生态系统的重要组成部分，或者因为您使用开源组件，或者因为您产品的核心是开源的，请记住，信任是出了名的难以建立，并且非常容易很快失去。

不管人们对这件事的感觉如何，开源社区对他们信任的人非常小心，很容易被一家似乎没有考虑到他们最大利益的公司疏远。为了建立信任，团队建立可信度是很重要的。开源社区很大程度上是基于价值的，所以除非社区看到专业知识和知识的证明，否则他们不会自动信任自称的专家。这种可信度甚至可以延伸到个人，比如许多开源开发者，他们在社交媒体上拥有大量追随者，并将他们的可信度借给他们工作的项目和公司。

最后，在处理开源技术、项目和社区时，关键是要记住开发人员对明目张胆的推销非常敏感。真实性关系到一个项目、公司或品牌展现自身的方式。开源是关于协作和社区的；这是一种建立在信任和互利基础上的关系，而不是利益。
