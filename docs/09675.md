# 非结构化数据管理:避免内部知识缺口

> 原文：<https://devops.com/unstructured-data-management-avoiding-insider-knowledge-gaps/>

大多数 IT 领导者都知道，管理非结构化数据环境是一项艰巨的任务。在开始非结构化数据管理项目之前，有许多因素需要考虑，包括存储设备的数量、特定数据集的大小、数据集中的文件数量，以及可能最令人生畏的非结构化数据所处的独特环境。

一刀切的解决方案往往会无意中掩盖非结构化数据环境的重要特征。诚然，这种短期的简单解决方案比投入工作为每个组织的数据集定制最适合的解决方案更容易。然而，任何承担非结构化数据管理项目的人都应该预先得到警告——这种方法可能会产生长期的有害影响，称为“内部知识”

当一群人花时间在一个组织内建立[数据管理](https://devops.com/?s=data+management)系统，然后离开他们的角色，而没有将关于数据管理系统的关键信息传递给接任者时，就剩下了内部知识。这对那些继承了聚集系统的人来说是难以置信的困难，并且有可能，目前有这个问题的组织还没有意识到他们有这个问题。

### 结束:创造内幕知识的开始

今天的存储管理员有很多工作要做，而且总是很分散。他们的日常职责通常包括:

*   创造份额和出口
*   设计文件系统布局
*   为每个应用程序或数据集分配设备
*   搜寻未使用的数据
*   组织工作流并将其与数据集配对

通常，在他们繁忙的日常环境中，IT 团队的决策和行动都是通过孤立的方法进行的。随着项目的进展，一种只有监督项目的 IT 专业人员才知道的独特语言被创造出来。他们可以创建自己的标签，并按照一定的组织系统放置文件。然而，这些大部分都没有被写下来，这意味着它就像一个没有答案的密码一样被破译。这种语言就是“内部知识”

### 内幕消息和大辞职

为什么这是一个问题？只要有人知道数据的整体情况就可以了，对吗？嗯…

仅仅依靠创建工作流的 IT 专业人员团队意味着他们必须永远和公司在一起，他们的系统才有意义。毕竟，一旦拥有这种知识的人离开，公司就永远失去了这种知识。

一家公司的员工仅仅会留下来看数据管理项目完成，这是不值得的。内部知识给组织带来的威胁是，只有知道系统如何工作的人才能正确地参与下一次招聘。这对于了解系统所有信息的 IT 专业人员来说是一个很好的职位，因为他们有一种工作安全感，但对组织来说不是很好。

对于组织来说， [劳动力中 44%的员工](https://www.cnbc.com/2022/03/22/great-resignation-continues-as-44percent-of-workers-seek-a-new-job.html%23:~:text=Investing%2520Club-,The%2520Great%2520Resignation%2520continues,%2520as%252044%2525%2520of%2520workers,look%2520for%2520a%2520new%2520job&text=Forty-four%2520percent%2520of%2520employees,the%2520Great%2520Reshuffle,%2520is%2520continuing.) 都是主动求职者。[大辞职](https://fortune.com/2022/07/21/great-resignation-40-percent-want-to-quit-where-are-they-going/)显示了求职者在今天的市场上占了上风，优秀的人才可以在不经意间被公司收买和挖走。当(而不是如果)组织的 IT 团队成员离开时，必须为非结构化数据管理系统建立一个记录完善、组织有序的基础，以便下一个团队能够从他们离开的地方继续工作。

否则，当团队成员离开时，他们将不可避免地留下知识空白，并会出现以下问题:

*   您的 NAS 上是否托管着 20 TB 过时的应用程序数据？
*   这 20 TB 的数据是存储管理员计划删除的吗？或者它很重要吗？
*   您是否会花钱将这些数据保存在第 1 层磁盘上？
*   您是否有归档到其他存储设备但从未删除的旧员工文件？

在开始处理数据管理项目之前，记住内部知识的含义是很重要的。这实际上就像预测未来和决定不让少数人控制整个企业的非结构化数据存储一样简单。

### 留给技术:找到解决方案时要问什么

为了管理非结构化数据，现在市场上有许多工具都是基于内部知识(以及组织内部预先存在的代码)构建的。通过尽可能地自动化流程，组织可以在 IT 团队继续前进时避免头疼的问题。一些有助于找到合适工具的问题包括:

*   **数据位置:**有多少个存储设备？有多少个数据中心？数据中心分布在多少个国家？有生活在云中的吗？
*   **扫描能力:**数据的上述位置会影响工具扫描数据的能力吗？可以执行哪些功能，从仅仅扫描到利用大数据能力进行分析？
*   **演示:**你如何从工具中接收信息？可消化可利用吗？会不会很容易突出被遗忘的数据集？

*   **时间:**需要多长时间？引擎可以扫描多少数据？这会影响一切照旧吗？

这些都是 IT 领导者在为其组织的长期成功决定最佳解决方案之前必须找到答案的问题。无论 IT 团队人手不足、工作过度，在当今的现代数据中心中，最不应该做的事情就是让一些 IT 专业人员完全控制企业的非结构化数据管理系统。

员工流动从来都不好玩。在 it 中，如果 IT 团队被迫从零开始项目，这可能会严重影响整个组织，因为没有人能够翻译基于内部知识构建的系统语言。

鉴于每天创建的数据量，现在是尽可能清理和自动化该流程的时候了，因为昨天是决定适用于今天的解决方案的时候了。花时间评估新技术和全面数据管理解决方案的 IT 领导者将准备好明智地使用预算，以降低挫折、延迟、成本超支甚至 IT 团队更替的风险。
