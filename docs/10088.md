# 调查揭示了云、API 专业技能差距

> 原文:[https://devo PS . com/survey-sheds-light-on-cloud-API-expertise-skills-gap/](https://devops.com/survey-sheds-light-on-cloud-api-expertise-skills-gap/)

企业 IT 面临三重冲击:数字化转型、开放 API 和安全威胁。首先，大多数组织仍在进行中的[数字化转型](https://devops.com/the-role-of-apis-in-digital-transformation/)和[云迁移](https://devops.com/90-of-orgs-say-multi-cloud-helps-achieve-business-goals/)。第二，开发团队继续增加对[开放 API](https://www.forbes.com/sites/googlecloud/2021/01/14/your-api-strategy-is-a-business-requirement-not-just-a-technological-solution/)的使用，导致[蔓延问题](https://devops.com/api-sprawl-a-looming-threat-to-digital-economy/)。第三，我们已经目睹了一系列[网络安全威胁](https://devops.com/the-age-of-software-supply-chain-disruption/)，这些威胁似乎越来越复杂。技术领导者正试图控制这种新工具和风险的涌入，但在确保敏感数据始终受到保护的同时快速创新可能是一项挑战。

Axway 2022 Open Everything 调查揭示了投资 API 开发和支持 API 的托管文件传输(MFT)的人的上述一些挑战和态度。最值得注意的是，调查发现，86%的受访者担心未来几年将出现专业知识缺口，他们的组织将没有创新所需的熟练软件和 IT 基础设施人才。下面，我们将回顾调查中的其他发现，重点关注那些表示他们投资于 API 开发的人的回答。

## 云迁移仍在进行中

大多数组织仍在进行数字化转型。只有 12%的受访者表示完全数字化并引领创新。其余的要么刚刚开始转型，要么正处于中期，要么仍主要是实体企业。超过三分之一(38%)的人还表示，他们公司的数字化妆是新旧技术的混合体。

采用最多的计算环境是公共 AWS 云，占 47%。然而，其他选项紧随其后，包括混合(37%)、托管云服务(32%)、Azure (28%)、私有(25%)和谷歌云平台(18%)。其他报告证实了 IT 变得越来越现代化[混合云与多云](https://accelerationeconomy.com/cybersecurity/hybrid-multi-cloud-is-the-new-normal/)。随着数字化之旅的不断转变和发展，一些组织正在采用[多用途云](https://accelerationeconomy.com/cybersecurity/how-to-know-if-multi-cloud-is-right-for-your-organization/)。

虽然多重云提供了无与伦比的灵活性，但主要缺点是安全性。超过一半(55%)的受访者表示，跟踪和控制个人数据流入和流出云是多云战略的一个关键障碍。53%的受访者表示，处理云基础架构的区域实例的不同安全要求是一项最大的挑战。这种担忧是有根据的，因为在国际上开展业务的公司必须小心翼翼地遵循许多特定的地区数据法规。

## 开放数据策略比比皆是

正如*开放一切*的名字所表明的，许多不同的部门正变得更加[开放](https://nordicapis.com/open-everything-the-role-of-open-apis-across-6-sectors/)该报告强调了供应链中[API 的采用习惯，在供应链](https://nordicapis.com/api-security-is-paramount-to-protect-the-software-supply-chain/)中，它们最常用于配置应用程序、操作和监控应用程序以及集成内部数据源。这里流行的用例包括支付和跟踪系统。API 在金融领域也扮演着重要的角色，41%的金融从业者认为开放银行是发现新商业模式的机会。但是，最需要开放数据的地方莫过于医疗保健领域，该领域被认为是最需要数字化转型的领域。

我们认为 API 革命是由 [API 第一的初创公司](https://devops.com/index-tracks-impressive-growth-of-api-first-startups/)和 SaaS 服务推动的。虽然这些趋势得到了关注，但它并不是 API 最常见的用例。事实上，44%的时间 API 用于支持 IT 咨询和系统集成需求。其次是为客户提供云应用(35%)、发展合作伙伴网络(35%)和为客户或合作伙伴提供云服务(34%)。尽管[通过 API](https://nordicapis.com/5-ways-to-generate-direct-revenue-with-apis/)产生收入肯定有商业用例，但它们最常被用来补充现有平台，或者将内部服务和数据粘合在一起。

## 大多数技术领导者担心 API 蔓延

安全性是所有 IT 部门都关心的问题，但在开放数据时尤其重要。受访者表示，安全性是创建 API 时最重要的考虑因素(30%)。这种担心是有根据的，因为近年来对 API 的攻击呈指数级增长。其他最受关注的是 API 采用或消费(15%)、客户体验(14%)、API 管理的便利性(14%)以及设计和开发(11%)。

随着 API 和微服务在组织中变得越来越丰富，蔓延变得越来越容易。不受约束的技术蔓延会对设计标准产生恶化的影响，并导致[未知的影子 API](https://nordicapis.com/the-risks-of-zombie-apis-and-what-to-do-about-them/)，这可能会造成安全漏洞。因此，超过三分之二的受访者表示，API 蔓延的前景令他们担忧。

为了缓解蔓延并解决其他问题，大多数公司都在投资工具来帮助管理不断增长的 API。大多数人(83%)表示，在过去两年中，他们用于 API 管理的预算要么增加了，要么保持不变。对于 18%，这个预算上涨了 25%以上。操作化 [API 生命周期管理功能](https://blog.postman.com/api-lifecycle-blueprint/)可以带来一些好处，比如标准化设计风格和增加内部服务的可发现性。

## API 安全性必须仍然成熟

在一个完美的世界里，我们可以共享数据而不用担心滥用。但实际上，[API 是黑客](https://securityboulevard.com/2021/07/for-hackers-apis-are-low-hanging-fruit/)的常见目标。访问控制被破坏或缺少对象级授权的公开终端很容易被滥用于权限提升和数据泄露等行为。尽管这些风险持续存在，许多 API 并没有采用关于 API 安全性的[行业最佳实践](https://curity.io/resources/learn/api-security-best-practices/)。

尽管大多数 API 使用认证和令牌方案，但不到一半(43%)实现了 OAuth。此外，只有 30%的企业实施了带有 API 网关的策略。OAuth 2.0 和网关已经成为服务 API 的最佳实践，因为有太多的漏洞不能仅仅依靠 [HTTP 基本认证和 API 密钥](https://nordicapis.com/the-difference-between-http-auth-api-keys-and-oauth/)。因此，随着 API 安全性的成熟，我们可能会看到这些采用率的增加。

与其他网络安全努力一样，这实际上是关于采取整体观点和应用深度防御。幸运的是，大多数技术专业人员都明白这一点——开发更全面的安全方法被列为保护个人数据的首要策略。其他策略，如数据加密、高级多因素身份认证、数据丢失预防工具和高度安全的数据交易，也是保护云原生资产所必需的。
