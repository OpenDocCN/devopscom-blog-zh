# 打造敏捷企业架构

> 原文：<https://devops.com/crafting-an-agile-enterprise-architecture/>

在这个数字技术和快速变化的时代，数字化转型、业务敏捷性和企业架构是经常听到的术语。数字化应用程序、解决方案和流程的使用使组织能够提高效率。反过来，这使他们能够以更灵活敏捷的方式响应不断变化的客户需求和趋势。

数字化转型、移动性的增加和远程工作人员的出现，导致了商业模式范式的转变。采用更敏捷的方法已经越来越流行，正在将敏捷性融入流程的企业正在看到回报。结果是更高的生产率、更快的上市时间(TTM)和更高的消费者满意度。

### **敏捷的文化**

数字化转型在一个更具协作性和灵活性的商业环境中取得成功。从简单地执行敏捷业务流程转变为全面的敏捷需要企业思维模式的转变。当企业开发一个真正敏捷的架构时，企业中的每个人都参与交付解决方案；从业务运营和技术到营销、财务、合规性和安全性。敏捷文化从面向公司的文化转变为面向人或客户的文化，包含协作、共同创造、所有权、灵活性和沟通的原则。

### 短期周期和流体规划

敏捷性正在从传统的集中式治理、冗长的开发周期、预定的季度评审和依赖瀑布交付方法的模式中重新定义企业架构。敏捷企业架构涉及以客户和市场为中心的短期周期和流动计划，而不是长期运营和战略计划。语言从自上而下的项目命令和管理转变为协作产品团队中的持续产品开发、实验、测试和监控。

### **打造敏捷企业架构的要点**

真正的[敏捷架构](https://devops.com/?s=agile+architecture)的蓝图需要商业动态的根本转变。对于在整个企业中实现敏捷性，有三个突出的基本要求。

*   **协作****–**成功的企业级敏捷性的基石是协作。当创新被分散化时，它将蓬勃发展，孤立的筒仓让位于跨职能、敏捷和自组织的团队。孤立的 IT 团队会导致项目延期、预算超支、生产力难以衡量以及业务和运营脱节。每个部门都必须参与支持和实现关键业务目标。

由业务线和 IT 专业人员组成的团队加速了开发和交付，大大缩短了上市时间。基于共同的以客户为中心的目标和愿景，在整个企业中存在成果的共享所有权和更深层次的参与。日常交流和协作反馈培养创造力、解决问题的能力，并推动持续集成和持续发展。来自组织内部和外部同事的协作输入对于企业、技术和解决方案架构师来说是必不可少的。

*   **当前状态映射****–**在整个企业中采用敏捷实践和原则意味着潜在的巨大变化。如果没有一个清晰的流程路线图，企业可能会举步维艰。没有实现更精简的简单性，反而增加了复杂性和模糊的愿景。在一个企业能够成功地采用敏捷架构之前，必须完全理解当前的运营状态。本质上，必须精心设计功能、能力、应用程序和技术堆栈的蓝图。从数据目录到业务流程工作流的一切都必须映射。企业领导可以提出以下问题来分析当前状态并评估影响点:

*   我们应该优先考虑哪些项目，它们需要什么能力？
*   哪些流程依赖于哪些软件应用？
*   如果我们删除一两个应用程序，会对流程产生什么影响？

一旦绘制出当前状态的蓝图，企业就可以以结构化的方式轻松地按照自己的节奏创建其架构的未来状态。

*   **平衡敏捷转型与日常运营****–**继续使用当前的一些架构一段时间，可以让组织的高层轻松地接受敏捷思维和文化。在开发敏捷架构的迭代过程中，架构师、高管和技术领导者需要在转换计划和有效维护日常运营之间取得平衡。他们将不得不辨别需要多少方向和支持，以及他们将在多大程度上设定速度。

如果首先关注的是交付给客户的价值，那么将根据最具战略意义的项目、功能和应用来确定架构的优先级。一个敏捷的架构将采用一种“刚好够用”的方法来进行项目设计和交付。

### **敏捷架构优化协作**

企业敏捷性促进了跨职能团队中企业架构师更直接的沟通和参与。通过日常讨论和各种接触点，架构师可以提供关于 IT 决策的指导、建议和反馈，并帮助团队共同拥有解决方案。

通过合作，企业和解决方案架构师可以开发一个完整的 IT 环境视图，以及如何修改它以适应未来的需求；最大限度地减少浪费和优化资源。解决方案架构师可以查看所有应用、解决方案和系统的架构图，并协作进行改进。它们还可以帮助企业架构师在整个业务中扩展标准化。敏捷转型核心的协作基石使架构师和非技术领导者能够获得更多的清晰度和一致性。

### **敏捷的价值**

当企业架构结合敏捷实践时，商业价值会大大提高。根据麦肯锡&公司的一份报告，过渡到更灵活模式的企业可以将财务绩效提高 20-30%，其中 30-50%可以提高运营绩效、TTM 和客户满意度得分。在流程、人员和技术中嵌入敏捷模型将培养对 CI/CD、突破性创新、协同合作的持续支持，并最终最大化客户成果。
