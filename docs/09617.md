# DevSecOps 实践差距评估

> 原文：<https://devops.com/devsecops-practices-gap-assessment/>

DevSecOps 实践对于在 DevOps 价值流中安全可靠地交付软件至关重要。

为了从 DevSecOps 获得最大的安全保护，使用推荐的最佳实践是很重要的，包括人员、流程和技术。差距评估是有效地评估一个组织的 DevSecOps 实践并确定改进策略的好方法。差距分析为制定策略和路线图提供了有价值的输入，以改进包括 DevSecOps 在内的主题。

我的幻灯片分享[DevSecOps 的差距调查、评估和分析](https://www.slideshare.net/MarcHornbeek/gap-survey-assessment-and-analysis-for-devsecops)解释了我的 9 大支柱差距评估工具如何大大加快了 DevSecOps 的差距调查、差距评估和差距分析流程。

为了这些目的，澄清一些定义是很重要的，因为我发现不同的人和组织使用这些术语是不同的。在这篇博客和我的咨询工作中，我使用了以下定义:

**差距调查**是一种发现工具，用于收集关于某个主题实践的当前状态的信息。
**差距评估**是确定某个主题的当前实践和推荐实践之间的差异(差距)的过程。差距评估也用于指本清单中的所有活动。
**差距评估**研讨会是在差距分析之前验证差距评估的会议。
**差距分析**是确定减少差距评估中发现的差距的优先顺序的过程。
**差距评估工具**是帮助执行差距调查、差距评估和差距分析的软件。

对于那些想更深入了解这个主题的人来说，我的书 [Engineering DevOps](http://mybook.to/engineeringdevops) 中阐明了与我进行差距评估的方式相关的其他定义和上下文信息。

我为差距评估定义了一个七步流程。步骤如下:

**1。挑一个题目来分析。**同样的流程和工具可以同样适用于几乎任何主题。
2**。确定该主题的推荐实践。**一些值得做差距评估的事情，比如 DevSecOps，通常会相当复杂，有几十个甚至几百个实践。将实践分类成组(我称组为“支柱”)，使实践列表更容易理解和有意义。这样做将需要一些研究和工作。例如，我有一个 DevOps、持续测试、DevSecOps 和 SRE 以及容器等其他主题的实践和支柱的数据库。
**3。将选择的实践输入差距评估工具。该工具可以是您创建的简单调查工具、商业调查工具或定制工具。
**4。确定差距调查需要包括哪些人。**
**5。使用差距调查收集数据。**
**6。执行差距评估。**
**7。执行差距分析。****

**第 4 步:确定差距调查的对象**

为了使差距分析更加全面，需要对受容器实践影响的角色中的人进行调查，或者至少代表他们，以确保他们的观点被包括在内。通常包括的一些角色示例:

商业领袖——因为领袖影响 DevSecOps 的文化和预算。开发人员——因为开发人员需要根据 DevSecOps 实践进行设计。
·项目负责人——因为他们影响产品工作的优先级。
QA 测试人员——因为他们需要根据 DevSecOps 实践进行测试。
·运营——因为开发运营实践会影响运营。
安全——因为 DevSecOps 实践会影响安全实践。

调查可以针对企业中的单个应用程序、一组应用程序或所有应用程序进行。但是，对作为改进目标的组织部分进行差距评估和差距分析是很重要的。

**步骤 5:使用差距调查收集数据**

差距调查应允许每个被调查人输入一个“重要性水平”分数，一个“实践水平”分数和对每个实践的评论。所有这些信息对于差距评估和差距分析都是必不可少的。在我的差距评估工具中包含的差距调查中，调查受访者被要求对每个实践的重要性进行评分，0 =不相关，1 =不重要，2 =很好，3 =重要，4 =非常重要或 5 =关键。实践水平的选择是 0 =不确定，1 =很少，如果有的话，2=有时，3 =大部分时间，4 =总是或 5 =我们擅长这个。还应输入与限定或解释为每项练习输入的分数相关的注释，尤其是如果有任何疑问或模糊之处。

**步骤 6:执行差距评估**

差距评估流程要求收集从调查中收集的所有实践分数，并进行汇总，以计算一组总分数。差距分数是使用一个公式计算的，该公式用相应的重要性等级分数对每个实践等级分数进行加权。可视化的表示有助于识别实践领域和具有最高、最重要差距的单个实践。

无论练习和分数定义写得多么专业，一些人误解并输入他们本来不会得到的分数是很正常的。因此，在进行差距分析之前，确保对收集的数据进行验证非常重要。首选方法是与参与调查的每个角色的主要代表一起召开差距评估研讨会。在研讨会期间，将对调查结果之间偏差较大的每项练习的分数进行讨论，并在必要时进行调整。

**步骤 7:执行差距分析**

差距分析过程包括提取差距较大的实践，并根据顾问或主题专家确定的解决方案类别对每个实践进行标记和排名。

差距分析的结果指出了解决方案策略和实施路线图需要关注的地方，以缩小最重要的差距。

**这意味着什么**

显而易见，DevOps 为软件部署提供了巨大的价值，但坚持 DevSecOps 的最佳实践对于降低风险和确保安全性至关重要。每个组织都是不同的，并且具有不同的安全态势。根据对每个组织的风险评估，适用的具体实践、安全工具和指标可能会有所不同。这篇博客解释了一种差距评估方法，它使用了我的 DevSecOps 最佳实践的九个支柱。我的免费 DevSecOps 评估工具有助于对 DevSecOps 和优先级进行差距分析，以改进组织的 DevSecOps 实践。
