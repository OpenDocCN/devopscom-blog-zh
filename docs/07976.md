# 网络安全在网络管理中的重要性

> 原文:[https://devo PS . com/the-importance-of-netsecops-in-network-management/](https://devops.com/the-importance-of-netsecops-in-network-management/)

在过去十年左右的时间里，IT 领域已经发生了翻天覆地的变化。绝大多数企业现在主要在云中运营，这对他们的敏捷性和生产力产生了显著的影响。最近对 1，900 名 IT 和安全专业人员的调查发现，41%的组织在公共云中运行更多的工作负载，而 2019 年只有四分之一的受访者。在新冠肺炎疫情期间，即使是在数字化方面不够成熟、无法充分利用云的企业也将大幅改变其战略，以支持大规模远程工作。

然而，随着云创新在董事会议程中占据如此重要的位置，安全性往往会被抛在后面，从而产生一个漏洞缺口，这是企业在当前风险加剧的形势下难以承受的。同一调查发现，对采用云的首要担忧是网络安全(58%)。

管理组织的网络及其安全性应该齐头并进，但是，正如调查中所反映的那样，公共云安全没有明确的归属。职责分散在 SecOps、NOC 和 DevOps 之间，并且这些团队没有以与业务利益一致的方式进行协作。经验告诉我们，这种孤立的方法会损害安全性，那么企业应该如何应对呢？他们如何弥合网络运营和安全运营之间的差距，以确保网络资产的安全并防止失误？

## 为 NetSecOps 构建案例

当今的数字基础设施需要网络运营和安全运营部门的协作，甚至是融合，以实现最高的安全性和生产效率。虽然大多数企业在这两个部门之间有开放的沟通渠道，但仍有很大一部分网络和安全团队在孤立工作。这造成了不必要的摩擦，对于试图提供最佳最终用户体验的基于服务的企业来说，这可能是个问题。

现实情况是，网络操作系统和安全操作系统有几个共同点。他们都负责业务的关键方面，并且必须适应不断变化的环境，通常是在极其严格的条件下。敏捷性对于安全团队跟上新兴技术的步伐尤为重要，但部署往往会由于配置错误或执行不力而在实施阶段被搁置或放弃。随着企业继续部署[软件定义的网络](https://devops.com/?s=software+defined+networks)和公共云架构，安全性对于网络团队来说变得更加重要，这也是这种融合宜早不宜迟的原因。

我们需要以某种方式将网络安全元素插入网络运营管道，并无缝地使其成为流程中的另一个步骤。如果我们能够在交付前测试阶段自动检查网络连接是否已经启用，这至少可以让我们避免部署无法工作的东西的痛苦。然后，如果我们确实需要打开新的端口，也可以提出更改请求的自动化流程将节省更多的时间。更好的是，如果是那种“这样做那样”的更改，这种更改可以在防火墙上自动实现——只需几分钟，而不是几周。

有了 NetSecOps，应用程序开发人员可以发挥创造力，而不必担心安全性会降低他们的工作效率，网络安全团队也可以高枕无忧，因为他们知道风险和合规性检查内置于持续交付流程中，并有完整的审计跟踪，因此他们甚至可以腾出时间来做一些有趣的事情。

如果我们能够做到这一切，只在极少数新连接需要安全部门批准的情况下，NetSecOps 的梦想就可能成为现实，实现快速、高质量的交付。通过 NetOps 和 SecOps 的合作，组织可以建立一个网络管理和安全的集体方法，为两个团队提供平等的机会来提出和优先处理问题，并从一开始就同意基本规则；天作之合。
