# 研究:实时数据具有变革效应

> 原文：<https://devops.com/study-real-time-data-has-a-transformative-effect/>

我们都知道数据是现代数字业务的核心。[实时数据](https://devops.com/how-stream-processing-goes-beyond-real-time/)可以说是下一次发展——拥有最新数据可以实现节能和[优化以降低成本](https://devops.com/the-rise-of-the-cloud-economist-the-other-cfo/)。实时数据处理还有助于欺诈检测和提高医疗响应能力。

特别是在 DevOps 中，访问实时数据对于减少[事件响应时间](https://devops.com/5-mean-time-reliability-metrics-to-follow/)和通知正在进行的 [SRE 目标](https://devops.com/whats-the-difference-between-sli-sla-and-slo/)是必要的。利用绩效数据还可以[增强未来决策的能力](https://devops.com/leverage-empirical-data-to-avoid-devops-burnout/)并开启新的收入机会。本质上，实时数据可以对数字企业的运营产生变革性影响。

DataStax 和 ClearPath Technologies 最近发布了 [2022 年数据竞赛状态报告](https://www.datastax.com/blog/500-tech-leaders-show-that-real-time-data-has-real-impact)。研究发现，大多数数据领导者正在使用实时数据来创造附加价值。该报告还建议，与前沿的实时数据技术合作增加了[开发者的体验](https://devops.com/understanding-the-importance-of-developer-experience/)，这在[持续的人才短缺](https://www.idc.com/getdoc.jsp?containerId=US48223621)中是很好的保持。

下面，我将回顾该研究的主要发现，以推断组织如何有效地使用这些技术，以及如何减少整个过程中出现的障碍。

## 数据竞赛的状态

简而言之，客户期望实时体验。不仅仅是终端消费者，开发人员也越来越习惯于以这种方式进行构建。最近异步的、事件驱动的架构的兴起就证明了这一点。该调查描述了实时数据对现代软件的重要性，发现 78%的受访者同意实时数据是“必须的”，而不仅仅是“最好的”

这项投资似乎正在获得回报。整整 71%的受访者表示，他们可以直接将收入增长与实时数据联系起来。它不仅似乎对收入增长产生了变革性的影响，而且还提高了生产率——66%的实时数据公司认为开发人员的生产率有所提高。自然，该报告发现，开发人员是最有可能使用实时数据的人。

尽管如今现代数据库和实时数据处理工具更加容易获得(并且[开源](https://containerjournal.com/features/3-cloud-native-database-tools-from-cncf/)，但是数据处理的成熟度并不一致。根据该报告的分类，“数据领导者”仅占受访者的 10%。

## 价值的障碍

通往数据成熟的道路有一些障碍，可能会限制数据的采用。首先是数据复杂性，33%的人表示这是利用实时数据的最大障碍之一。紧随其后的是控制数据成本(32%)、数据可访问性(30%)和可用技能组合(28%)。

最值得注意的是，[技能差距](https://devops.com/4-tips-to-plug-the-cloud-it-skills-gap/)仍然是整个行业报告和各种技术领域中经常提到的障碍。有趣的是，在实时数据成熟度较高的组织中，缺乏内部人才的比例会略高。在这一类别中接受调查的人说，他们的最大障碍是在他们的业务部门找到合适的技能。

网络安全也是一个首要问题——51%的人表示，当涉及到实时数据的使用时，安全挑战是一个大问题。正如该报告推测的那样，实时数据的业务关键性质提高了其作为商品和攻击领域的价值。此外，在新的[数据共享法规](https://nordicapis.com/top-3-global-standards-that-impact-personal-data-sharing-with-apis/)中，IT 部门现在面临着比以往任何时候都更大的压力来维护数据安全并避免敏感个人信息的泄露。

## 解锁实时数据

这个领域有很大的潜力；报告发现，86%的战略性关注实时数据的组织的开发人员表示，“技术比以往任何时候都更令人兴奋”。实时数据开始在边缘应用，人工智能/人工智能可以被训练为现代应用提供更具响应性的优势。然而，组织必须优先采用它，并减少上述障碍，以释放实时数据的优势。

对实时数据进行优先排序不仅仅需要工具，还需要远见。根据该研究，那些通过实时数据计划创造更多价值的人被发现:

*   有明确的产品所有者
*   让业务部门对数据负责
*   并让部门代表在跨职能团队中一起工作。

如你所见，这些基本目标与技术关系不大，更多的是清晰的领导和协作。也就是说，要获得实时数据，无疑需要对适当的工具进行投资。

例如，这个工具可能包含针对实时优化的 NoSQL 数据库或针对高吞吐量优化的 Apache Cassandra。向实时的过渡可能也需要[流媒体和信息技术](https://containerjournal.com/topics/container-management/5-cncf-projects-for-streaming-and-messaging/)。然而，不太明显的是位于应用程序和数据库之间的 API 层。这个元素可以极大地帮助以标准化的方式开放数据可访问性。

## 数据竞赛的现状:最后的思考

处理实时数据可以成为一项优势，并产生变革性的影响。因此，大多数开发人员和领导者将实时数据视为一个令人兴奋的领域。然而，组织必须减少数据孤岛，开放数据访问以实现其优势。这也需要数据科学家和业务部门之间的合作来构建有意义的结果。

[数据竞赛的现状](https://www.datastax.com/blog/500-tech-leaders-show-that-real-time-data-has-real-impact)报告调查了各行业的 500 名领导者。上面，我们总结了调查的要点。鼓励读者拿起一份副本，以获得更多细微差别的数据点和进一步的评论。
