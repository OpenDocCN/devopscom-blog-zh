2019 年冬季发布的主题是智能协作和生产力。此外，这一新版本还配备了 **VersionOne** 、该公司屡获殊荣的企业敏捷扩展解决方案、 **TeamForge** 、安全开发和 CI 平台以及 **Continuum、**该公司的智能发布管理产品，同时还强化了 CollabNet VersionOne 的企业价值流管理产品。

从战略上来说，该版本为 IT 和运营部门的业务领导提供了额外的功能，使他们能够查看其开发管道的整个范围，不仅在开发阶段优化价值，而且通过战略规划、预算、路线图、发布和迭代规划，一直到发布自动化和部署，在映射价值方面走得更远。

**此版本中围绕智能特性和工作可见性的新增强包括:**

*   重复出现的投资组合计划的模板创建，或功能开发中的共同工作；这减少了在生成公文包项和相关联的工作项时的错误和失误。
*   Epic/Feature awareness 增加了对工作项的智能跟踪级别，例如 VersionOne、吉拉和 Visual Studio Team Services 中的故事或缺陷，并自动将该工作项链接到父 epic/feature trace，该父 epic/Feature trace 可通过交付在整个 DevOps 工具链的下游进行跟踪。
*   VersionOne 和 Continuum 之间更紧密的集成允许更高级的智能价值流场景，从而提高跨团队的生产力。例如，阻止成员升级或部署代码的能力，如果用户故事没有准备好，或者没有通过合规性检查，则停止构建的能力，或者将自动化活动配置为在未来某个时间执行，而不是立即执行的能力。
*   TeamForge 基线化，它为用户从一个被批准的项目基线中创建一个项目增加了巨大的生产力增强，这个项目基线可以包含跟踪工件、文档版本、文件发布、源代码文件和/或作为被选择基线的一部分的二进制文件。此外，新的基线关联功能提供了将源代码提交文件与吉拉工作项关联的能力。
*   受管副本服务器，允许用户在一个视图中管理项目的所有副本服务器/存储库关联；这节省了时间，因为不需要为每个存储库单独配置副本。
*   增强的 Git 代码审查，跟踪已经审查过的文件；当审查中的变更包含对许多文件和接触点的变更时，这有助于提高 CI/CD 的生产率和安全性。
