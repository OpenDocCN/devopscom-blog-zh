# 当开发运维即服务(DaaS)遇到安全问题时

> 原文:[https://devops.com/when-devops-as-a-service-meets-security/](https://devops.com/when-devops-as-a-service-meets-security/)

作为服务提供的最新 IT 方法之一是 DevOps，这是一种文化和实用的方法，将开发和运营团队聚集在一个工作保护伞下。[开发运维即服务](https://www.techtarget.com/searchitoperations/definition/DevOps-as-a-service-DaaS#:~:text=DevOps%20as%20a%20Service%20is,team%20and%20the%20operations%20team.) (DaaS)确保与开发运维工具和基础设施的选择、管理和维护相关的一切，包括所有政策和程序，都通过一个能够为整个组织的开发团队提供此服务的集中团队来解决。

从 30，000 英尺的高度来看，DaaS 代表了开发者通过[门户](https://devops.com/?s=developer+portal)和 API 可以访问的 DevOps 能力。因为开发运维能力因组织而异，所以这些服务没有标准化。对企业来说，DaaS 增强了竞争。应用程序用户和开发人员发现 DaaS 改进了整体软件开发，而专业的 DaaS 团队说它改进了 DevOps 功能的管理。

## DaaS 的其他好处是什么？

DaaS 提供了许多好处，尤其是对于那些不再想把所有时间都花在配置和部署必要的安全工具来运行 DevOps 管道的开发团队来说。这意味着不再需要理解 DevOps 工具和底层基础设施之间的所有差异，因为适当的工具和技术会自动为开发团队部署。因此，开发人员能够集中精力以业务的速度开发创新软件。

此外，Daas 以更加简化的流程形式提供了清晰的业务和运营优势，以促进整个企业的开发运维。Daas 支持治理，同时还支持对开发工作负载和生产力的集中监督和可见性。正如我们所知，这种转变导致团队之间责任的增加和业务流程的整体优化。再加上通过工具选择的资源效率和成本节约，很容易理解为什么 DaaS 在软件开发领域找到了一个家。

尽管有这些优势，Daas 也带来了传统管理解决方案无法完全解决的新的安全挑战。这些方法通常缓慢、昂贵，并且/或者管理起来极其复杂。

## 安全部门和开发部门呢？

由于 DevSecOps 的原则能够主动应对与 DevOps 流程相关的风险和威胁，因此近年来得到了快速采用。通过在 DevOps 管道中集成 AppSec 工具和策略，开发人员能够在软件生命周期的早期发现并修复安全漏洞，而不是在事后。这种能力允许开发人员从一开始就生成安全、高质量的软件，同时也防止了由于生产中最后一分钟的安全检查而导致的任何部署延迟。有了 DevSecOps 过程，AppSec 测试对开发人员来说变得无缝和透明，他们需要一种无摩擦的体验来促进稳定和可靠的软件开发和部署。

DevSecOps 非常适合“即服务”交付模式。与 DevOps 类似，AppSec 涉及各种工具、策略和流程，需要一定程度的专业知识，更不用说选择、维护和部署它们以推动 DevSecOps 实践所需的时间和资源了。希望实施和管理 DevSecOps 的安全和开发团队可以从集中定义和管理的流程中获得重大收益，该流程将由一个专家团队管理，并作为“服务”提供给负责 DaaS 的人员。

通过 DevSecOps-as-a-Service 产品实现 AppSec 的标准化将保证在所有 DevOps 活动中均匀地应用安全性，从而允许企业一致地生产安全和高质量的软件，同时还支持 AppSec 合规性需求。这种转变还为我们提供了一个基础，使我们能够全面、一致地了解 AppSec，这对于应用程序和业务风险的评估和管理至关重要。

## 开发 DaaS 指标

首先，不要试图通过同时测量所有应用程序来煮沸海洋。相反，选择一个受 DaaS 流程支持的应用程序，并指定一个专门的团队来开发该应用程序的指标。这个团队可以有多种方式——但可能包括利益相关者、开发人员和其他支持 DaaS 的角色。服务级别指标(sli)和服务级别目标(SLO)也融入了这一模型，它们被选择来最大限度地满足单个应用程序的要求。

一旦确定了 DaaS 最有价值的服务级别协议，就可以定制服务来识别最重要的服务级别协议。有了这些知识，就可以确定仪器仪表、遥测和分析要求，并用于设计警报、仪表盘、分析仪等。下一步，围绕责任建立政策和程序就建立了一个完整的解决方案。一旦针对所选应用的解决方案得到测试，信心就会产生，并创建一个更容易的过程来适应和扩展围绕其他应用的过程。

这意味着 DaaS 可以为组织和团队提供许多关键能力。然后，一个专业的 DaaS 团队可以计算出在任何应用程序上进行大规模实施的最佳指标。因此，SLO 可以为将来的其他应用程序发展，用作将来改进的基准。
