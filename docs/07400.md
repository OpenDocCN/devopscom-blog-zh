# Dynatrace 在其软件智能平台中增加了云自动化模块

> 原文:[https://devo PS . com/dyna trace-adds-cloud-automation-module-to-its-software-intelligence-platform/](https://devops.com/dynatrace-adds-cloud-automation-module-to-its-software-intelligence-platform/)

*支持开发、开发运维及 SRE 团队更快、更低风险地交付更高质量的云原生应用*

<time datetime="2021-02-10T13:00:00Z">February 10, 2021 08:00 AM Eastern Standard Time</time>

马萨诸塞州沃尔瑟姆。–([商业资讯](https://www.businesswire.com/))–软件智能公司 [Dynatrace](https://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fwww.dynatrace.com%2F%3Futm_medium%3Dpr%26utm_source%3Ddynatrace-website%26utm_campaign%3Ddynatrace%26utm_content%3Dperform_cloud%26utm_term%3Dnone&esheet=52376851&newsitemid=20210210005222&lan=en-US&anchor=Dynatrace&index=1&md5=9aac17b3248607d79f70a439bd075263) (纽约证券交易所代码:DT)今天宣布在其软件智能平台中增加一个云自动化模块。这一新模块将利用 Dynatrace 平台核心的自动化和智能来协调应用程序开发生命周期流程，包括根据组织的服务水平目标(SLO)自动进行代码测试和质量检查。这种编排是由一个完全支持版本的 [Keptn](https://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fnam02.safelinks.protection.outlook.com%2F%3Furl%3Dhttps%253A%252F%252Fkeptn.sh%252F%26data%3D04%257C01%257Cjerome.stewart%2540dynatrace.com%257Cc88c58c1b1e44912aee308d8c49d0292%257C70ebe3a35b30435d9d677716d74ca190%257C1%257C0%257C637475525948638209%257CUnknown%257CTWFpbGZsb3d8eyJWIjoiMC4wLjAwMDAiLCJQIjoiV2luMzIiLCJBTiI6Ik1haWwiLCJXVCI6Mn0%253D%257C1000%26sdata%3DvqFfb6DVyLVr3R8EOaHO3UFxlSeslPUUq5Xp0S9pZgg%253D%26reserved%3D0%2F&esheet=52376851&newsitemid=20210210005222&lan=en-US&anchor=Keptn&index=2&md5=89b140e0d444ba9e1b0ce4ea7753afe4) 驱动的，这是一个开源的 CNCF 项目。Dynatrace 平台的这一新增功能将改变开发、开发运维及现场可靠性工程(SRE)团队开发和管理云原生应用的方式，使他们能够以更低的风险更快地实现创新。

RedMonk 的分析师兼联合创始人 James Governor 表示:“自动化运营、持续交付和发布管理的需求变得至关重要。Dynatrace Cloud Automation 完全支持嵌入式 Keptn，旨在提供一个端到端的平台，将管道和可观测性结合在一起，以支持现代 DevOps 和 SRE 团队。”

ERT IT 运营总监 Simon Pilar 表示:“Dynatrace 云自动化正在改变我们开发和管理云原生应用的方式。“它通过自动确定应用程序是否准备好进入生命周期的下一阶段，确保我们交付的应用程序符合我们的最高标准。它还能让我们的开发人员花更少的时间进行故障排除，将更多的时间用于推动企业和客户更快的创新。”

Dynatrace 云自动化利用 Dynatrace 软件智能平台的人工智能和自动化，通过以下方式增强开发、开发运维及 SRE 团队的流程:

*   **根据 SLO 的质量关口对预生产应用进行自动质量检查**，确保无错代码通过交付管道的每个阶段，无需人工检查或干预。
*   **对生产中失败的版本进行自动化的闭环补救**,包括回滚金丝雀或蓝/绿部署、管理功能标志、协调补救操作手册、触发事件管理工作流，以及通过人工智能支持的根本原因分析为开发人员提供精确的上下文。
*   **自动发布清单和版本比较**持续评估单个发布版本的性能，如果发现问题，自动恢复最稳定的版本，帮助团队始终如一地交付最高质量的发布。

“在动态云原生架构中，确保高质量、安全和快速的发布周期非常耗时，整个应用生命周期都需要手动流程和孤立的可见性，”Dynatrace 产品管理 said Steve Tack 说。“Dynatrace Cloud Automation 与组织现有的 DevOps 工具链无缝集成，并自动执行日常任务，为团队提供更多创新时间。通过这一版本，我们正在推动我们的客户和行业向高度自动化、人工智能驱动的开发操作系统发展。"

Dynatrace 云自动化模块将在未来 90 天内推出。它将加入基础设施监控、应用和微服务监控、数字体验监控、业务分析和应用安全模块，作为 Dynatrace 软件智能平台的一部分。
