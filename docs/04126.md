# 5 不能错过詹金斯世界的 DevOps 会议

> 原文：<https://devops.com/5-cant-miss-devops-sessions-jenkins-world/>

在不到一个月的时间里，许多 Jenkins 用户和 DevOps 专业人员将聚集在旧金山参加 8 月 28 日至 31 日举行的 [Jenkins World](https://www.cloudbees.com/jenkinsworld) 。有 60 多节课可供选择，涵盖广泛的主题，很难缩小您的日程。这就是为什么带着一个目标去表演是很重要的。

### [**DevOps:兑现创新、敏捷和速度的承诺**](https://jenkinsworld20162017.sched.com/event/BbTz/devops-delivering-on-the-promise-of-innovation-agility-and-velocity)

Forrester 分析师 Rob Stroud 将讨论最近的 Forrester 研究，该研究证实组织正在利用敏捷开发技术、自动化 CI/CD 管道、测试和发布自动化来交付差异化产品。互动讨论将允许与会者回顾他们的 DevOps 成熟度与市场成熟度的对比。斯特劳德还将主持讨论 DevOps 如何在企业内增加价值，以及容器和无服务器计算的采用将如何进一步推动 DevOps 的采用。

### [**DevOps 领导小组**](https://jenkinsworld20162017.sched.com/event/BcIk/devops-leadership-panel)

参加这个小组，听取各种专家关于软件交付挑战、开发运维实现的商业价值、建立内部支持等主题的意见。小组成员将包括:

*   Rob Stroud，Forrester 首席分析师，主持人
*   Kolby Allen，AWS 架构师，WatchGuard Technologies
*   Andrey Falko，Salesforce 首席工程师
*   乔治·斯旺，Autodesk 工程解决方案总监
*   Stas Zvinyatskovsky，埃森哲工程主管和建筑师

### [**用 DevOps 象限成熟度模型绘制您的 DevOps 之旅**](https://jenkinsworld20162017.sched.com/event/Bfsh/mapping-your-devops-journey-with-the-devops-quadrant-maturity-model)

在本次会议中，CloudBees 的 DevOps 宣传员 Brian Dawson 将把 DevOps 之旅映射到 DevOps 象限成熟度模型。他将详细介绍每个象限中组织的特征，并提出迈向 DevOps 成熟度下一阶段的策略。DevOps 成熟度模型是评估您当前 DevOps 状态的好方法。

### [**依靠开发运维:领先的金融服务机构如何推动开发运维转型**](https://jenkinsworld20162017.sched.com/event/AMCa/banking-on-devops-how-leading-financial-services-organizations-are-driving-the-devops-transformation)

聆听金融服务高管讲述他们的组织如何采用 DevOps 实践来加速创新并为客户提供更多价值。金融服务行业面临着特定的安全、治理和监管障碍，在改变流程和技术时，这些障碍可能极具挑战性。本次会议将由 CloudBees 的营销副总裁 Andre Pino 主持。演讲者将包括:

*   布洛克·贝蒂，第一资本公司软件工程总监
*   ESQM 商业技术总监 Karthrik Loganathan
*   Stefan Simenon，ABN AMRO 银行 IT 工具和软件开发主管
*   Moied Wahid，PayPal 软件开发总监

### [**领导一个 DevOps 改造**](https://jenkinsworld20162017.sched.com/event/Agqu/leading-a-devops-transformation)

在本次会议中，DevOps Research & Assessment LLC 首席技术官 Jez Humble(也是 Jenkins World 的主题演讲人)将讨论如何实施 DevOps，而不仅仅是实施工具。他将介绍 DevOps 转型如何要求从预算到管理再到合规性的不同思考方式。他还将具体分析 DevOps 是什么，不是什么。
