# BMC 使组织能够更快地开发和交付代码

> 原文：<https://devops.com/bmc-enables-organizations-to-develop-and-deliver-code-faster/>

*新的增强功能使采用企业开发运维实践、保护运营弹性和保护大型机变得更加容易*

**休斯顿–2021 年 4 月 1 日**–[BMC，](https://www.bmc.com/?cid=pr-ZSO-ag-03-wp-04012021&cc=pr)自主数字企业软件解决方案的全球领导者，宣布了 BMC 自动化大型机智能(AMI)和 BMC Compuware 产品组合中的几个[新产品和集成](https://www.bmc.com/it-solutions/bmc-ami-latest-release.html?vu=bmc-ami-innovations&cid=pr-ZSO-ag-03-wp-04012021&cc=pr)，旨在现代化和简化大型机应用程序开发和交付，同时增加正常运行时间和加快威胁检测。

这一最新版本改变了 BMC Compuware ISPW、BMC Compuware Topaz、BMC Compuware File-AID、BMC AMI Ops Insight、BMC AMI SQL Performance 和 BMC AMI Security 解决方案的大型机管理，使用户能够:

*   通过扩展的跨平台 DevOps 工具链，快速、高效、精确地构建、分析、测试、部署和管理大型机应用程序；
*   通过智能、自动化的可能原因分析，减少并可能消除诊断问题所需的时间；和
*   通过将大型机安全和 IT 运营工作流结合起来，使威胁事件可见、可分配，并更快地检测和响应安全威胁

采用企业开发运维方法的主流大型机开发

BMC 已经完成了对 BMC 公司 Compuware 的整合，创建了基于现有 BMC AMI DevOps 产品的 BMC Compuware 工具套件。BMC Compuware ISPW 解决方案现在与流行的开发工具 Visual Studio Code 集成在一起，开发人员只需单击鼠标右键，就可以编辑源代码并在大型机上进行编译。BMC Compuware Topaz 解决方案带有一个新的命令行界面和多个应用程序编程界面，使开发人员可以灵活地连接到大型机，减少对专业技能的依赖，并扩展 DevOps 工具链。最后，随着新的 BMC Compuware File-AID 21 解决方案的发布，开发人员在查看数十亿个事务和日志文件时，还可以体验到搜索时间的动态缩短。

“BMC 的这一举动发出了一个明确的信号，即 DevOps 已经从广泛采用阶段转向大规模采用阶段。DevOps 已经成为每个企业的桌面赌注；无论他们是在云中还是在大型机上运行，他们都需要采用 DevOps 来保持相关性，更不用说保持竞争力了。然而，对于许多组织来说，采用 DevOps 一直是一个挑战，尤其是那些拥有大型机的组织，”弗雷斯特研究公司的克里斯托弗·康多、克里斯·加德纳和迭戈·洛·朱迪切写道。¹“…如果 DevOps 不能在大型机上扩展，核心的现代化将成为一个障碍，核心将全部迁移到云。BMC 和 Compuware 的赌注是像任何其他平台一样在大型机上扩展 DevOps。”

为了支持 DevOps 流程和提高代码质量，BMC AMI SQL Performance for Db2 解决方案通过一个 Jenkins 插件提供了将 Db2 SQL 的质量控制转移的能力，该插件有助于防止 SQL 违反公司标准和影响生产环境。该解决方案在其当前环境和目标生产环境中查询静态和动态 SQL。这确保了在进入上线流程的下一步之前，报告并解决 SQL 编码标准和性能问题。

从被动解决问题转向实时主动解决问题

BMC AMI Ops Insight 解决方案通过自动智能诊断功能将系统管理从被动模式转变为主动模式，现在可以提供可能的原因。用户尽可能早地收到系统降级的通知，减少误报，并能够查明问题。通过更快地找到可能的原因，BMC AMI Ops Insight 提供了更多的时间来利用可行的见解进行补救。

统一安全和运营工作流

BMC AMI 安全产品可防范漏洞、恶意行为和数据窃取，现在集成了 [BMC Helix ITSM](https://www.bmc.com/it-solutions/bmc-helix-itsm.html?cid=pr-ZSO-ag-03-wp-04012021&cc=pr) 解决方案。如果 BMC AMI 安全解决方案检测到威胁，这种新的集成将在 BMC Helix ITSM 自动生成安全事件，通过单一的实时数据馈送和明确定义的流程为 IT 运营团队提供更大的可见性，以便他们能够更快地行动、跟踪和响应威胁和事件。

除了与 BMC Helix ITSM 的集成之外，新的 BMC AMI Security 版本继续与大量企业安全信息和事件管理解决方案集成，从而提供了更多分配和快速调查事件的方法，缩短了平均修复时间，并使大型机更加安全。

“大型机是我们客户数字化转型战略的重要组成部分。BMC 智能 Z 优化和转型部门 SVP 和总经理 John McKenny 表示:“借助改变大型机管理的工具，我们可以让客户更好地采用 DevOps 实践，并从他们的投资中获得最大价值，从而加速创新我们继续帮助我们的客户实现运营弹性，同时自动化和最大限度地减少手动任务，使他们能够成为自主的数字企业。"
