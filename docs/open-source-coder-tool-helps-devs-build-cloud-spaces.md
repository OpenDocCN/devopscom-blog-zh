# 开源编码工具帮助开发者构建云空间

> 原文：<https://devops.com/open-source-coder-tool-helps-devs-build-cloud-spaces/>

[Coder](https://coder.com/) 今天发布了一个开源项目，为开发人员和数据科学家提供了一种在几分钟内创建云工作空间的方法，从而更容易更快地构建应用程序和模型。

目前，Coder 提供了一个云应用程序开发平台，使开发人员能够使用任何编辑器或集成开发环境(IDE)作为本地桌面的替代方案来构建应用程序。这种能力是通过使用开源的 Terraform 基础设施供应工具来提升开发人员工作空间而实现的。这种方法为开发人员提供了对云资源的访问，这些云资源可以根据需要扩展和缩减，而不是局限于他们可以在笔记本电脑或台式机系统上访问的内存和存储资源。

Coder 的工程副总裁 Ketan Gangatirkar 表示，虽然个别云服务提供商提供了实现这一目标的方法，但这些产品将组织锁定在特定的云服务上。他补充说，编码器方法允许开发人员在多个云服务中使用他们喜欢的工具。

大多数开发人员已经在构建运行在多个云平台上的应用程序。Gangatirkar 说，因此，他们需要访问一套工具，以消除开发人员需要部署和掌握各种工具来构建这些应用程序时产生的开销。

他指出，相比之下，使用编码器工具，开发人员可以打开网络浏览器，选择一个工作空间模板，然后开始编码。Gangatirkar 说，这种能力消除了设置开发人员工作空间通常需要的所有手动、易出错的过程。

他补充说，与保护开发团队使用的每个本地系统所需的时间和精力相比，这种方法还为组织提供了一种更安全的构建应用程序的方法。

一般来说，随着组织增加他们同时构建和部署的应用程序的数量，开发人员的工作效率得到了更多的关注。随着在加速数字化转型的竞赛中对人才的竞争加剧，雇佣和留住开发人员仍是一项重大挑战。总的来说，许多开发人员更愿意为那些允许他们使用他们已经知道和喜欢的工具的组织工作。Gangatirkar 指出，这种方法也使得开发人员参与开发项目变得更加容易。

不清楚有多少应用程序开发计划从未开始，仅仅是因为每当灵感袭来时，就要花费太多的时间和精力来构建开发环境。然而，很明显，开发人员在维护多个本地开发环境上浪费了大量时间。大部分时间可以更好地用于编写实际的应用程序代码。

不管怎样，开源工具——如 Coder 提供的工具——应该可以消除当今应用程序开发中存在的许多摩擦。唯一剩下的问题是这些工具是否会被开发人员自己采用，而不是专门为他们服务的 DevOps 团队。
