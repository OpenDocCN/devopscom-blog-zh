# Gitpod 希望将应用程序开发推进到云中

> 原文：<https://devops.com/gitpod-looks-to-drive-application-development-into-the-cloud/>

Gitpod 今天透露，它将添加一个应用编程接口(API)，使云开发环境(CDEs)更具可扩展性。Gitpod 还表示，Johannes Landgraf 担任 Gitpod 的单一首席执行官，因为前任联合首席执行官 Sven Efftinge 成为该公司的第一位技术人员。

Gitpod 的 Landgraf 刚刚筹集了 2500 万美元的额外资金，他说 API 将使构建 CDE 插件变得更加容易。兰德格拉夫说，随着时间的推移，CDEs 将变得像在本地桌面上运行的任何开发环境一样容易访问。

Landgraf 指出，软件工程师通常每周浪费五个小时来维护应用程序环境。他补充说，通过开源开发平台，Gitpod 自动提供云中的安全工作区，这提供了使多个开发人员更容易合作的额外好处。

GitPod 产品路线图中增加的功能包括显著加快的启动时间、改进的弹性和面向大型公司的 Gitpod 云产品，以及促进协作的预览环境和工作流。Gitpod 还宣布通过了 SOC2 Type II 安全审计。

Landgraf 指出，如今，Gitpod 平台正被 Shopify、Stripe 和 Slack 等组织的 75 万多名开发人员使用。

在构建软件时，一个长期存在的问题是，在本地桌面上运行的代码在部署到生产环境中时并不总是有效。每当开发人员访问 CDE 时，Gitpod API 将使构建各种平台和工具变得更加简单。Landgraf 指出，目标是使更接近地复制开发人员最终部署其应用程序的生产环境成为可能。

此外，他补充说，CDE 应该减少开发人员在构建更复杂的应用程序时，在笔记本电脑和云计算环境之间例行移动的数据量。

随着经济衰退的到来，如今人们更加关注开发人员的工作效率，尤其是在组织寻求加速数字业务转型的时候。Landgraf 说，当许多开发人员继续在家工作时，DevOps 团队也渴望找到集中管理[开发人员体验](https://devops.com/?s=developer+experience)的方法。

目前还不清楚 CDE 将在多大程度上取代运行在本地机器上的集成开发环境，但是所有主要的云服务提供商都鼓励开发者在云中构建应用程序。兰德格拉夫指出，面临的挑战是，这些产品往往会将开发者锁定在一个特定的云平台上。他补充说，今天大多数开发人员正在构建需要跨多个云部署的应用程序。

无论如何，软件开发正越来越多地转向云。大多数组织可能会试图在本地机器上开发的代码量和云中开发的代码量之间找到一个平衡，开发者可以在心情激动时启动本地机器。最直接的挑战是确保 CDE 的体验和现有的本地 IDE 一样好。
