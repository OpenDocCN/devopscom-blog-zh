# 使 DevOps 更加平滑

> 原文:[https://devops.com/making-devops-smoother/](https://devops.com/making-devops-smoother/)

作为一名 DevOps 开发人员，不仅仅需要构建产品所需的专业技术知识。它也是关于学习促进整个过程的操作。这意味着使用敏捷方法，与一个快节奏、高绩效、跨职能的团队合作。但是由于这些有时相互冲突的需求，有时会遇到困难。

在这里，我们将为 DevOps 开发人员提供六点建议，以确保开发过程更加顺利。

## 设定可衡量的目标

当朝着一个特定的目标努力时，精确性是最重要的。准确地定义问题会导致更快的解决方案。例如，与“为北美配置服务 X 的所有组件的部署”相比，“为新区域配置部署”太模糊了后者的目标更加具体，使有关各方更容易发现问题，制定计划并迅速解决问题。

大多数 DevOps 开发人员在 sprint 中设定目标时会使用 SMART 这个缩写。

1.  具体的:确切地定义需要做什么。
2.  **M** 可测量:突出显示指示完成所必需的参数和值。
3.  一个 可实现的:定义任务将如何完成，并决定谁将负责完成它。
4.  ealistic:基于上述细节的目标有多现实？
5.  imely:设置任务必须完成的日期和时间。

在为特定的冲刺设定目标时，记住以上五点，以确保任务被精确地定义。

## 记录一切

在整个开发过程中，适当的文档对每个人来说都是有价值的。在许多情况下，DevOps 开发人员在项目开始后加入项目，让他们审阅由其他开发人员创建的文档是他们跟上进度的最简单方法。不仅如此，文档还有助于 [调试](https://devops.com/how-real-time-debugging-improves-reliability/) 、故障排除和指导客户。

记录流程通常被视为无聊和浪费时间。不过，多亏了 [在线文档工具](https://document360.com/blog/software-documentation-tools/) ，记录信息变得简单了许多。使用格式良好的文档模板会使事情变得更容易。

## 交流和协作

从本质上来说，DevOps 开发团队的效率归结于他们在团队内部以及与外部利益相关者的沟通和协作。沟通上的差距会减缓并可能破坏整个开发过程。这一点尤其重要，因为越来越多的团队变得越来越遥远，突然闯入你同事的实际工作场所问一个简短的问题不再可能。

在远程工作环境中，通信可能会变慢。发送文本、等待回复、安排呼叫等。，增加了许多步骤，而不是简单地指着屏幕问一个问题。虽然现在有很多可视化协作工具，视频会议工具也有更多的集成，但还是不一样。

因此，制定严格的沟通和协作规则，并确保从一开始就设定期望，以便团队成员和利益相关者知道期望什么，这一点很重要。最简单的方法之一是为特定的沟通渠道配备特定的工具。通常，有三种类型的沟通工具是必要的:

*   团队沟通: 像 Slack 和微软团队这样的消息应用。
*   **利益相关方沟通:** 大本营、工作区或普通邮件。
*   **视频会议:** 可用于内部和外部沟通。比如 Zoom，Google Meet 等。

## 自动化重复性任务

如果某些周期性任务发生在本地级别，您可以使用 Bash 或 PowerShell。Bash 比较老，被很多 IT 和 DevOps 团队使用。微软在 2006 年开发的 PowerShell 是较新的版本，已经取代了 Windows 环境中的命令 Shell。

虽然 Bash 和 PowerShell 都是跨平台的，但一般来说，Bash 更受 Linux 和 macOS 用户的青睐，而 PowerShell 则更受 Windows 用户的青睐。归根结底，这取决于个人喜好。无论你选择 Bash、PowerShell 还是更新的自动化工具，自动化都将节省大量关键的开发时间，并从开发人员的工作量中移除[辛劳](https://devops.com/?s=toil)。

## 选择正确的技术组合

每个团队都有不同的需求和偏好的工具。“DevOps 工具”是一个涵盖从服务器编排工具、云平台、代码管理到许多其他类别的术语。简单来说，DevOps 工具是使编写、测试、托管和部署代码变得更容易的工具。有许多 DevOps 工具可供开发团队选择。这里有三个值得考虑。

1.  **Git:** Git 是一个版本控制和分布式源代码管理。它使得开发者更容易在主分支上协作、试验和合并代码。
2.  **云 ide:**完全托管的云集成开发环境(ide)在您的本地系统出现故障时保护您的进度。GitHub Codespace，Cloud9 和 Glitch 是最受欢迎的。
3.  **容器:** 容器简化了管道的构建、测试、登台和部署。它们可以是本地托管的，也可以完全托管在云上。容器有助于在部署之前识别和修复问题。

在选择您的技术组合时，牢记用户体验和用户界面、可用性、可扩展性、可用集成和定价非常重要。您的工具应该适合您的过程，而不是相反。

## 重视 DevOps 文化

工作文化对于整个 DevOps 团队的累积成功至关重要。团队相互交流的方式决定了他们的工作方式。当一个团队被提供了足够的自由来做出自己的决定时，它会想出创新的方法来使整个过程更有效率。以自己的方式克服“开发”和“运营”之间的鸿沟将使团队能够做好工作。

话虽如此，请记住，文化是随着时间的推移有机发展起来的，而不是强加于人的。
