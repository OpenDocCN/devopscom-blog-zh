# 企业区块链的采用取决于 DevOps

> 原文：<https://devops.com/enterprise-blockchain-adoption-hinges-on-devops/>

现在区块链到处都很热闹，这是有原因的。[生产用例](https://www.forbes.com/sites/michaeldelcastillo/2021/02/02/blockchain-50/?sh=49d6519f231c)继续出现在制造业、能源、航空旅行、保险和金融等领域。 [Gartner](https://www.gartner.com/en/newsroom/press-releases/2020-01-30-gartner-predicts-that-organizations-using-blockchain-) 预测，到 2023 年，使用区块链智能合同的组织的数据质量将提高 50%。

然而，Gartner 还预测，区块链将降低 30%的数据可用性。在它的使用普及之前，基于区块链的应用程序开发还有一些障碍需要克服。如果企业要参与进来，高管们需要看到其价值的切实证据，即与采用区块链直接相关的支出减少和利润增加。

“企业不会为了采用新技术而改变他们的商业模式，”CasperLabs 首席执行官 Mrinal Manohar 说。正如很难戏剧性地改变已有产品的 UX 一样，改变已有软件生产例程的开发人员体验也很难。

我最近会见了 Manohar，讨论了加速企业区块链采用所需的限制和变化。根据 Manohar 的说法，这在很大程度上可以归结为采用 DevOps 方法——减少能源使用，支持 CI/CD，确保稳定的开发环境，并为变更和持续测试提供空间——对于刺激采用至关重要。这也意味着从开发者和最终用户那里抽象出复杂性。然而，一旦有了更多的生产证据，向区块链的转移将是迅速而巨大的。

## 是什么阻碍了企业的采用？

Manohar 说，企业采用归结为用户界面和 UX。他预测区块链将会走上与互联网相似的道路。在命令行时代，互联网只被顽固的技术人员采用。但是，一旦基于视觉的行动通过浏览器成为可能，消费者的兴趣就爆发了。

同样，一旦复杂性被推到后台，区块链可能会看到更多的生产使用。马诺哈尔说:“当一项技术 99%不可见的时候，它是令人敬畏的。然而，他补充说，目前区块链有 99%是可见的。例如，启动一个节点并测试一个区块链应用程序并不容易。通过更多的抽象、模拟和与云提供商的集成，将需求降低到 50%会有很大帮助。

总体而言，Manohar 认为有四个关键因素阻碍了企业的采用:

1.  **企业需要更低的能源足迹**。在公共网络上获得高吞吐量而不牺牲分散性和安全性是很棘手的。如果区块链在扩展时耗尽了运营，组织可能会简单地选择集中式数据存储。因此，企业将需要具有最低能源需求的区块链。
2.  **大公司希望开发环境与现有环境相似**。在许多方面，区块链抛弃了旧剧本。但如此重大的转变在多个 IT 部门之间很难实现。大公司将需要插入现有开发人员堆栈的区块链能力。使用熟悉的编程语言的区块链和流行的 ide 插件，如 Visual Studio 和 IntelliJ，将减少入门工作。
3.  他们需要更多的控制。区块链通常使用不可变合同。这对于维持共享的分布式历史是很好的，但是 Manohar 认为，合同需要更具延展性，以便能够持续升级。此外，在收养区块链时还有隐私问题。一些实现者不希望所有 100%的数据都在公共链上。能够维护智能合约的私有和公共节点有助于实现更大的灵活性。
4.  **企业希望获得持续支持**。Manohar 说，一些公司将自主使用开源软件，但大多数企业不会采用没有支持电子邮件的新技术或新兴技术。因此，可能需要专业服务和外包的区块链专家。

## 应用 DevOps 视角

加速区块链采用的另一大好处是考虑 DevOps 的观点。对 Manohar 来说，将 DevOps 引入区块链需要更多的持续更新、缓解以避免中断变更和模拟以实现更好的测试环境。

一成不变的区块链基础设施似乎与 DevOps 的快速部署信条不一致。因此，Manohar 说，选择一个支持更多可变性的区块链将有助于升级合同，并允许功能随时间变化。对于影响整个公共链的核心基础设施变更，大多数验证者当然必须在发布之前接受升级。

简单的功能，如推送通知，可以简化这一过程，允许运营商快速回答是或否。

另一个潜在的问题是向后兼容性。如果聚合引入了重大的突破性变化怎么办？马诺哈尔说，虽然这在技术上是可能的，但不太可能。“这将是一个有争议的升级，”他说，验证者有动机保持一个坚固的系统完好无损。“只有当他们认为这对网络的长期健康更好时，他们才会这么做。”但是这个过程中的护栏将有助于避免边缘情况。令人欣慰的是，如果一个变化确实扰乱了少数用户，区块链可以分叉。例如，我们已经看到比特币现金和比特币，或者以太坊经典和以太坊发生了这种情况。

帮助企业在使用区块链时采用 DevOps 的另一个转变是更好的测试和部署流程。众所周知，当区块链应用程序接入更广泛的网络时，对其进行测试是一项挑战。因此，工程师可以从完全脱离主网络的模拟运行时间中受益，以用于测试目的。

## 区块链发展的未来

那么，区块链企业采用的未来是什么样子的呢？嗯，看来我们将继续看到新的新颖用例出现，以创新的方式使用区块链。例如，Manohar 指出，区块链被用来为 NFT 数字孪生兄弟、专利市场、供应链标签系统和核心银行系统的抵押贷款流程提供动力。在这些情景中，他注意到区块链是如何以多种方式驱魔的；智能合同正被用于避免法律费用、消除人工处理和改进过错归因。

然而，在区块链成为主流之前，必须将其他企业纳入进来。也就是说，它需要对基础设施层进行更多的抽象。在这项工作中，低代码集成能力有助于使 API 与 ERP 系统或云服务提供商的连接更加容易。

为了鼓励采用，经济也必须改变它对区块链成功要素的看法。到目前为止，常见的 KPI 都是表层的，比如:有多少钱包？交易量如何？相反，Manohar 认为我们需要专注于证明采用区块链带来的实际成本节约。他说，一旦会计师公开报告收入增长，股东将要求每家公司都这样做。

“一旦这种催化剂出现，采用将会非常非常快，企业对区块链的需求将会非常大。”到那时，几年后，区块链可能会真正成为新经济的动力。
