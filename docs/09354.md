# Sysdig 开源扩展到安全云服务

> 原文:[https://devo PS . com/sys dig-open-source-is-extended-to-secure-cloud-services/](https://devops.com/sysdig-open-source-is-extended-to-secure-cloud-services/)

*新的集成支持任何 Falco 插件用于 Sysdig 操作系统*

**西班牙巴伦西亚，(KubeCon + CloudNativeCon Europe)，2022 年 5 月 16 日—** 统一容器和云安全领导者 Sysdig 宣布，Sysdig 开源(容器的事件响应标准)已扩展到云。使用系统调用，Sysdig 开源(Sysdig OSS)传统上提供对正在运行的应用程序以及文件系统访问和网络活动的深入观察，从而加快事件响应和故障排除。团队可以快速过滤来自 Sysdig OSS 的信息并采取行动。随着这一新集成的发布，这些功能已经从容器扩展到任何云环境。

云原生应用程序的复杂性(包含无数组件和变量)使得安全分析师和系统管理员很难快速筛选警报和调试问题。Sysdig OSS 实时捕获进程、文件系统和网络活动，并具有很高的粒度。该工具拥有近 200 万次下载和 6850 个 GitHub stars(T1)，可以显示从执行的命令和文件系统活动到网络活动的所有内容。Sysdig OSS 随后提供高级过滤和故障排除功能，支持安全和性能问题的根本原因分析。

使用一个[新插件框架](https://sysdig.com/press-releases/falco-open-source-adds-aws-cloud-security-monitoring/)——最初由开源社区为 CNCF 项目 Falco 开发——Sysdig 扩展了 Sysdig 操作系统可以连接到任何生成日志或事件的源的数量，包括 Azure、Google 和 AWS CloudTrail 日志。展望未来，每一个为 Falco 开发的插件也可以被 Sysdig 操作系统所利用。使用 Sysdig OSS 等工具来观察整个云原生环境中的事件可以简化调查。为每个环境使用不同的工具增加了复杂性，这使得故障排除变得非常困难。

***在 [Sysdig OSS 0.29 新发布博客](https://sysdig.com/blog/sysdig-open-source-uses-falco-plugins)*** 了解更多关于这个框架的信息

**Sysdig 对开源的承诺**

Sysdig 成立时是一家开源公司，Sysdig Secure 和 Sysdig Monitor 都建立在开源基础上，以应对现代云应用程序的安全挑战。这两个项目都是由 Sysdig 创建的，旨在利用深度可见性作为安全性的基础，它们已经成为容器和云威胁检测和事件响应的标准。Falco 于 2018 年捐赠给 CNCF，现在是一个孵化级托管项目，下载量超过 4500 万次。

Sysdig OSS 和 Falco 可以作为一个强大的开源解决方案一起使用，以降低运行时的风险。Sysdig OSS 充当飞行记录器，捕捉详细记录以供检查。Falco 充当安全摄像头，持续实时检测意外行为、配置更改、入侵和数据窃取。团队可以一起使用 Sysdig OSS 和 Falco 来检测和响应威胁。

Sysdig 的创始人兼首席技术官洛里斯·德吉奥安尼说:“如果你想了解应用程序内部的情况，Sysdig OSS 可以为你提供这份记录。“Sysdig 开源是 Falco 的灵感来源。虽然 Falco 会根据您的策略进行监控和警告，但 Sysdig 开源会告诉您在特定时间发生了什么，在事件发生之前和之后。能够在云中使用这两种开源工具是非常强大的。”
