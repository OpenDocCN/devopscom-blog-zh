# BMC 调查表面 DevOps 主机进展

> 原文:[https://devo PS . com/BMC-survey-surfaces-devo PS-mainframe-progress/](https://devops.com/bmc-survey-surfaces-devops-mainframe-progress/)

一项针对依赖大型机的 1，000 名高管和 IT 专业人员的[全球调查](https://www.bmc.com/newsroom/releases/bmc-survey-confirms-mainframe-platform-innovation.html)发现，70%的受访者现在采用 DevOps 流程在这一古老的平台上构建和部署应用，其中 60%的受访者现在部署应用的速度更快了。根据调查，整整 80%的受访者还希望更快地部署大型机应用程序。

该调查还发现，跨[大型机](https://devops.com/?s=mainframe)和分布式计算环境的 IT 流程的集成已经增加，现在超过一半(57%)的人共享平台运营角色。共有 44%的受访者表示，现场可靠性工程(SRE)是跨两个平台的共同角色。

内部开发的应用程序交付速度比一年前更快，增强了大型机的业务价值，76%的应用程序在 12 个月内实现了价值。

超过一半的受访者(53%)表示，他们的组织也在以跨越所有这些平台的方式实施 DevSecOps 实践。调查还发现，拥有大型机的组织最优先考虑的是安全性和合规性(67%)。

近三分之一还在大型机和分布式计算平台上采用了人工智能 IT 操作(AIOps)。调查显示，采用大型机 AIOps 和 DevOps 的受访者表示，他们对服务问题的了解速度提高了 70%,正常运行时间和可用性提高了 41%。

最后，调查发现，95%的受访者仍然对大型机作为运行应用程序的平台抱有积极的看法，62%的受访者表示大型机的交易量正在增加。超过一半的人(55%)还指出工作负载的波动性和不可预测性正在增加。类似比例(55%)的受访者也表示，他们一半以上的数据都在大型机上。

BMC 的智能 Z 大型机优化和转型高级副总裁兼总经理约翰·麦肯尼(John McKenny)表示，处于低迷时期的 IT 团队像往常一样，试图从现有投资中获得最大回报。变化最大的是，随着 IT 团队努力实现这一目标，大型机现在越来越多地与分布式计算环境集成在一起，他补充道。

McKenny 指出，总的来说，在大型机环境中进一步采用 DevOps 最佳实践还有许多工作要做。每个组织很可能会根据运行在大型机上的应用程序需要更新的频率来决定在什么级别上应用这些实践。McKenny 指出，仍然有大量运行在大型机上的应用程序可能多年都不需要更新，但随着组织接受[数字业务转型](https://digitalcxo.com)计划，更频繁地更新大型机应用程序的需求已经增加。

不管部署的应用程序是什么类型，很明显，大型机与分布式计算环境的其余部分隔离管理的日子正在慢慢结束。可以说，大型机只是部署应用程序的另一个节点。McKenny 说，一个主要的区别是，考虑到平台，大型机要安全得多。

当然，现在的挑战和机遇不仅在于改变对大型机的态度，还在于为负责管理异构 it 环境的 IT 团队提供正确的 DevOps 工具。
