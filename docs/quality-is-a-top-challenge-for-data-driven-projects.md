# 质量是数据驱动项目的最大挑战

> 原文：<https://devops.com/quality-is-a-top-challenge-for-data-driven-projects/>

软件系统不断产生越来越多的数据。而且利用它已经证明了好处——以至于多年来许多分析师都把[数据称为新的石油](https://www.forbes.com/sites/theyec/2021/07/15/data-isnt-the-new-oil--time-is/?sh=2de333d35bbc)。因此，大多数组织都在努力完善他们的数据，事实上，最近的一项研究发现，84%的组织已经部署了数据驱动型项目，或者在他们的路线图中有数据驱动型项目。

像脸书和谷歌这样的公司是收集和利用最终用户数据的商业模式的典型代表。然而，这只是一个方面；内部系统也在产生有价值的数据，如果正确利用，这些数据可以提供对[软件可观察性](https://devops.com/increasing-use-of-slos-to-enable-observability/)的洞察，并提高 DevOps 团队和开发人员的流程自动化。例如，有时间戳的应用程序日志对于[通知 SLO](https://devops.com/whats-the-difference-between-sli-sla-and-slo/)维护可靠性标准是必要的。正在对部署了云原生工具的[人工智能/人工智能进行平行投资，以根据生产数据采取行动，推动进一步的业务增长。因此，63%的 IT 决策者表示，由于数据和分析，新的收入机会已经出现。](https://containerjournal.com/features/why-uniting-cloud-native-and-ai-ml-is-a-win-win/)

由 Foundry(前 IDG 通信公司)开展的 [2022 年数据&分析研究](https://foundryco.com/tools-for-marketers/research-data-and-analytics/)探索了数据驱动的计划如何继续成为行政领导的重要投资领域。下面，我将回顾调查的主要结果，以考虑组织如何在现在和不久的将来继续做出明智的决策。

## 数据驱动型投资的现状

毫无疑问，我们生活在一个由数据驱动的世界里，相关项目的投资也在持续增长。超过一半(55%)的 IT 决策者计划增加以数据为中心的投资，该报告发现未来一年的平均支出为 1230 万美元。这个数字上升到平均 2300 万美元的金融服务，考虑到现代银行业的本质[这是合理的。](https://devops.com/why-open-banking-should-consider-devops/)

自然，分析平台的使用随着产生的数据量的增加而增加。在具体的分析工具方面，50%的受访者表示他们使用商业智能平台，47%的受访者使用关系数据库，19%的受访者计划在未来一到两年内投资。

我们还注意到基于云的解决方案的使用越来越多。例如，组织中 27%的数据分析工作负载现在运行在云中。该报告还发现，基于云的企业级数据仓库技术有所增长，这一领域在未来一年有增长的迹象。

## 共同目标

那么，这些类型的项目背后最常见的驱动因素是什么？嗯，自动化内部业务流程是数据驱动项目的最高目标——一半的 IT 领导者将此描述为主要目标。紧随其后的是其他目标，如提高客户洞察力(46%)、帮助客户支持(43%)和自动化 IT 运营(43%)。

就类型而言，事务性数据往往是最有用的——54%的公司在其数据驱动的项目中使用事务性数据。这包括消费者购买行为，如销售、退货和信用。下一个最常见的类型是机器生成的数据，包括来自日志、传感器、遥测、网络、安全系统和/或物联网设备的信息。第三种最常见的类型是客户档案信息。

作为改进现有数字产品的一种手段，数据在[业务环境](https://devops.com/a-data-driven-approach-to-devops-tips-for-improving-speed-and-scale/)中具有强大的影响力。其他受访者补充说，[数据驱动的开发运维方法](https://devops.com/visibility-drives-data-driven-decision-making/)有助于提高可见性并推动持续改进。

## 数据质量:最高级别的挑战

毫无疑问，特殊的挑战依然存在。最大的障碍是保持数据质量——41%的组织报告在处理这个问题。质量可能很差，因为数据是[非结构化的](https://accelerationeconomy.com/ai-ml/innovating-the-future-of-finance-with-no-code-ai-automation/)或者与其他来源不兼容。其他普遍的挑战包括治理问题和集成来自多个来源的数据。

对于那些承担数据驱动项目的人，44%的人表示他们缺乏适当的技能，如分析培训、数据管理、安全、商业智能和集成专业知识。在开始数据驱动的项目时，公司还经常面临资金和人才方面的困境。例如，根据这项研究，26%的中小型企业缺乏必要的资金来开展数据驱动的项目。

另一个具有挑战性的领域([，我之前已经介绍过](https://containerjournal.com/features/how-to-avoid-drowning-in-cloud-native-observability-data/))是优化数据收集和存储的方式。随着云存储费用的上升，组织可能会希望改进保留和解析，以避免创建不必要的大型、昂贵的数据湖。

## 人工智能/人工智能的发展

AI/ML 作为一种推进数据使用的方式已经得到了很好的确立。超过一半(54%)的公司使用了预测分析，或计划在未来 12 个月内将其整合到他们的系统中。不到三分之一(31%)的企业也使用或计划在未来使用异常检测。这些领域以及其他功能(如自然语言处理和预测分析)可用于辅助应用，如流程自动化、决策支持、客户分析、虚拟代理等。

在开发运维中利用数据，行业正处于一个激动人心的时刻。数据变得越来越容易访问，使 DevOps 能够对应用生命周期有一个[完整的了解。为了指导未来的工程投资，数据驱动的决策将依赖于诸如](https://devops.com/could-full-stack-observability-help-lessen-its-burden/)[性能和使用习惯](https://devops.com/seeing-the-bigger-picture-with-bizdevsecops/)之类的东西。因此，现在是考虑如何利用数据实现更大的过程自动化和软件开发流畅性的时候了。
