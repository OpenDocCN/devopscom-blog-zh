# strongDM 提供基础设施访问，DevOps 可以通过 Terraform Provider 进行编码

> 原文：<https://devops.com/strongdm-delivers-infrastructure-access-that-devops-can-code-with-terraform-provider/>

*企业确保在部署基础设施时锁定访问，而不会中断工作流程*

**Burlingame，CA ，2021 年 5 月 20 日 —** [基础设施接入网 strongDM](https://c212.net/c/link/?t=0&l=en&o=3169941-1&h=518049821&u=https%3A%2F%2Fwww.strongdm.com%2F&a=strongDM) ，今日宣布 strongDM Terraform 提供商。这使得开发运维团队能够在部署资源的同时以代码形式提供安全的基础设施访问。strongDM 客户能够安全地定义和部署对其 100%基础架构的访问，并从一开始就充满信心。

DevOps 团队已经采纳了基础设施可以被视为代码的理念，但是为员工提供对这些系统的访问仍然需要单独的手动步骤。

这个过程远远不是无缝的，这会扰乱和减缓开发周期。此外，安全流程被搁置，因为它们产生了太多的摩擦。当安全被绕过时，它就不起作用了。

strongDM 完成了“基础设施即代码”生命周期，包括“访问即代码”,同时改善了企业的安全状况。开发运维团队能够接受安全实践的左移，而无需额外的步骤，也不会对他们或最终用户的工作流造成任何中断。安全团队确信基础架构访问从一开始就是安全的、受控的和记录的。

“借助 strongDM，我的团队能够完成“as code”生命周期，这允许快速扩展，同时在整个部署过程中遵循我们的安全最佳实践。”说道 [Vivek Desai ，Olive](https://c212.net/c/link/?t=0&l=en&o=3169941-1&h=2063846605&u=https%3A%2F%2Fwww.linkedin.com%2Fin%2Fvivekdesai%2F&a=Vivek+Desai%2C+VP%2C+Cloud+Infrastructure+at+Olive) 云基础设施副总裁。“没有了典型的访问混乱和复杂性，我的团队有时间处理更重要的工作和顺畅的工作流。每个人都可以在需要时准确地访问他们需要的内容，而不会牺牲我们的安全态势。”

" strongDM 从第一天起就被设计成与现代开发人员的工作流程无缝集成."说 [贾斯汀·麦卡锡，strongDM 的联合创始人&CTO](https://c212.net/c/link/?t=0&l=en&o=3169941-1&h=3803903356&u=https%3A%2F%2Fwww.linkedin.com%2Fin%2Fjustinmccarthy%2F&a=Justin+McCarthy%2C+strongDM%27s+Co-founder+%26+CTO)T5。“通过将基础架构即代码的 DevOps 理念扩展到访问即代码，我们可以在几秒钟内管理访问和审计控制。管理基础设施访问的方式首次变得更加安全，也更加方便。”

strongDM 的 Terraform 提供商的优势包括:

*   在部署或销毁资源时，立即授予或撤销对基础架构的访问权限
*   立即在 strongDM 的控制平面内注册数据库、服务器和集群
*   随着基础架构的变化和增长，轻松部署和扩展 strongDM
*   使用 Terraform 还可以配置 strongDM + HashiCorp Vault 集成
