# 机器学习，人工智能推动 DevOps 进化

> 原文：<https://devops.com/machine-learning-ai-driving-devops-evolution/>

自动化是驱动 DevOps 的燃料。自动化常规的、可重复的任务是 DevOps 文化的定义特征之一。随着人工智能(AI)和机器学习的改善，可以自动化的任务的范围和复杂性都在增加，这提高了所有 DevOps 的门槛。DevOps 背后的人类将从更加世俗的任务中解放出来，并能够专注于更具创新性和创造性的努力。

如果你仔细想想，这是有道理的。从内部数据中心的物理服务到虚拟化、云服务、再到开发运维，这一趋势稳步提升了自动化的标准。通过程序和自动化开发、部署、监控和管理的应用程序和整个基础设施越多，一个组织的运作效率和效力就越高。

随着任务被卸载到自动化，人类可以解决其他挑战。这种关注不可避免地导致识别其他可以自动化的新任务。然而，人工智能和机器学习代表了这一演变中的一个巨大飞跃——有了人工智能和机器学习，自动化可以开始自我识别和改进。当自动化可以识别瓶颈和低效并自动适应以克服这些障碍时，人工输入和交互可能会变得几乎过时。

在最近的一次采访中， [Sumo Logic](https://www.sumologic.com/) 的首席技术官 Christian Beedgen 分享了一些关于机器学习对 DevOps 的影响的想法。“通过机器学习和自动化可以提供的进步，机器将提高团队和企业的生产率，”他说。“作为增加人类贡献的必要组成部分，机器学习最终将赋予人类审查数百万比特数据的能力。”

自动化日常任务至关重要，但还有另一个因素影响着人工智能和机器学习对 DevOps 未来的作用。现实是，有些事情人类根本无法像机器一样做得那么好或那么快——尤其是在大规模的情况下。

史蒂夫·伯顿在博客中强调，人类一次只能观察、消化和解释这么多信息。有些人认为圣杯解决方案是某种应用程序性能管理(APM)仪表板，它可以聚合和关联实时数据，以便 IT 人员可以手动监控指标和识别问题。“这在理论上听起来很好，但在现实中，当你的应用程序每天有数千个组件、数十亿个指标和数百个变化时，这是完全无法管理的，”他写道。“人类再也无法应对这种复杂性，他们现在需要机器来做跑腿的工作，处理这些大数据，并提供对正在发生的事情的运营洞察力。”

比根表示同意。他建议，“越来越多的企业将采用机器学习技术，因为它有助于通过揭示未知的未知来增加人类对复杂交互和大型数据集的理解。”

在某种程度上，这一切听起来有点可怕。似乎我们在 DevOps 上越成功，DevOps 进化得越多，人类就越不是等式中必要的一部分。可以想象，你可能非常擅长 DevOps，以至于你可以在工作之余开发自己的方式，或者无意中启动天网，或者两者兼而有之。

这种情况既可行又牵强。我建议你不要害怕那个未来，或者把它看做是机器偷走了你的工作或者接管了人类。拥抱人工智能和机器学习驱动的 DevOps 的未来，找到一线希望。寻找新的前沿和视野，因为总会有没有我们机器根本无法完成的任务——这就是创新发生的地方。
