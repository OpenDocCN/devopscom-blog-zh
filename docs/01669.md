# 网络研讨会:开发运维安全自动化

> 原文:[https://devops.com/webinar-security-automation-for-devops/](https://devops.com/webinar-security-automation-for-devops/)

随着组织更深入地进入持续开发和部署模式，实现持续安全行为的重要性变得不容置疑。云环境在部署、自动扩展事件和自然增长过程中会经历巨大的变化。虽然静态数据中心环境足够简单，可以由人来评估，但是云环境的动态特性太复杂，如果没有自动化，就无法充分保护。您的运营工具可以提供持续的监控和警报，为什么您的安全套件不能呢？

加入《DevOps.com》主编 Alan Shimel、New Context 安全服务副总裁 Andrew Storms 和 Evident.io 联合创始人兼首席执行官 Tim Prendergast 的讨论，他们讨论了为什么 DevOps 商店需要自动化关键功能，以确保其云基础架构的安全性。
