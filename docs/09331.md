# 它支持数字化转型的规模化

> 原文：<https://devops.com/it-enables-scale-in-digital-transformation/>

在过去的几年里，很少有公司职能部门的角色像它一样被打乱。求助服务台的日子已经一去不复返了。随着数字化转型的加速，它越来越不仅仅是单纯的技术，而是创造支持总体业务战略的能力。因此，它是任何公司数字化转型背后的引擎。

以下是 IT 部门从咨询、战略的角度 增强业务能力，推进当今数字化进程的三种方式。

## **数字化的同时走向数字化**

首先，让我们就数字化转型的基本术语达成一致，这可以理解地引发了一些持续的混淆:“数字化”和“数字”，麻省理工学院信息系统研究中心[](https://cisr.mit.edu/blog/documents/2017/10/19/2017_1001_digitizednotdigital_rossbeathsebastian.pdf/)将它们描述为两个独立但互补的事物。简而言之，数字化就是利用技术来改善或增强业务流程、运营和产品开发。相比之下,“走向数字化”是为数字化赋予客户价值，目标是彻底改变客户体验并创造新的数字化商业模式。如果用户体验很差，数字化一个流程或产品有什么意义？

平衡这些互补的努力至关重要。It 的作用是确保数字化和数字化可以同时发生。

## **重新思考经济模式**

为了跟上时代的步伐，IT 必须摆脱传统的基于需求的模式，以产品思维而不是受时间限制的项目思维来实现能力。能力的概念(即，由一组一致的 IT 产品支持的业务领域)对于避免创建一次性或单点解决方案非常重要，这些解决方案只是满足特定需求的手段。相反，必须将 IT 视为不断发展和改进的企业实践(例如网络安全)的推动者。拥抱敏捷方法将使这种方法开花结果。

转向这种基于能力的模型需要一个现代的基于 API 的架构，这是构建和/或替换单一系统的根本出路。由于当今世界需要速度和节俭之间的良好平衡、对资源分配的掌握和优先顺序的严格性，我们必须卸下旨在替换运行良好的系统的资源。

大意如此，亚马逊网络服务有一个伟大的架构原则被称为“ [二比零](https://chiefmartec.com/2019/07/want-innovate-like-amazon-heres-formula/) ” 换句话说，在快节奏的并行开发中出现两个相似的 it 组件比由于项目瓶颈和未处理的相互依赖而什么都没有要好。即使我们想尽可能地驱动重用(并有相应的机制)，我们也不应该因为一个重要的项目而使整体转换停滞不前。因此，不仅要考虑容忍多样性，还要考虑支持多样性，这是构建能够增强灵活性和弹性的架构的好方法。

## **持续解决技术债务**

技术债务可能是每个现有公司在数字化过程中面临的最大挑战。维护遗留和过时系统的成本很高，沉重的技术债务是实现全面数字化转型的另一个障碍。埃森哲的顾问每天都看到这种情况，他们将这种负担描述为 [“阻碍(公司)在数字时代创新和适应能力的代价。”](https://sloanreview.mit.edu/article/technical-debt-might-be-hindering-your-digital-transformation/)

这正是 IT 部门可以伸出援手的地方，他们可以采用结构化的方法和参与模式来简化系统和流程，从而释放资源进行重新分配。首先，根据众所周知和共享的标准(考虑一个评分系统)，重点消除遗留的定制应用程序和本地化工具。在这方面，资源分配可以发挥作用，确保本组织认识到这些遗产的全部成本。

持续努力消除技术债务的巨大附加优势(除了成本、减少过时系统和简化 IT 环境)显然与安全性有关。尤其是目前全球单次数据泄露的平均成本为【390 万美元 。事实上，潜在的网络安全和物联网相关的物理安全问题“将迫使 G2000 公司的首席信息官增加高达 25%的物联网安全支出，到 2020 年暂时抵消企业生产力的提高”(IDC)。转移技术债务可以抵消一部分安全方面的必要支出。

[埃森哲指出](https://www.accenture.com/us-en/insights/cybersecurity/reinventing-the-internet-digital-economy)79%的首席执行官表示，他们的组织采用新兴技术的速度超过了解决相关安全问题的速度。因此，IT 团队必须高举安全火炬，在保护端到端数字生态系统的同时，确定并实施让员工更容易获得安全性的方法。持续努力减少技术债务是改善现状的一种方式，也是众多方式之一。

## **数字转换表的位置**

的确，这对每个企业来说都是激动人心的时刻。数字化是关于创造客户价值的，以更好地理解企业中更深层次的客户需求为指导。由于提供出色的体验不仅与用户界面有关，还与移动内部系统的能力有关，以便大规模交付，因此 IT 专家现在在大规模变革客户体验方面占据了重要位置。

全面数字化转型的基本要素显而易见:数字化和走向数字化，能够利用一个在需要时使多样性高效的架构来拥抱新兴技术，以及创造网络公民来不断减少企业的技术债务。在幕后，现在是前台和中心，它正在推动公司的转型战略，在当今的数字经济中赢得竞争。
