# 22 位工程领袖对 API 发表意见

> 原文：<https://devops.com/22-engineering-leaders-weigh-in-on-apis/>

今天的每一个企业都是数字化企业…如果不是，那也很快了。

德勤最近的 [调查](https://www2.deloitte.com/us/en/insights/topics/digital-transformation/digital-acceleration-in-a-changing-world.html) 发现，数字化成熟的公司能够更好地适应快速变化，因此在财务上表现得更好。  近三分之二的调查受访者断言，未来五年内不进行数字化的组织将“难逃一死”

应用编程接口(API)是这些数字化转型努力的基石。API 不仅仅是让应用程序相互通信的技术管道。在一个软件驱动的世界里，它们允许开发者更快地构建服务，实现创新，打开新的市场并推动竞争优势。

来自 Postman 的一份新的[API 状态报告](https://www.postman.com/state-of-api/) 调查了 28，000 名开发人员和其他技术专家，生动地展示了 API 在现代组织中的核心作用。它表明，API 推动的创新帮助企业对疫情做出了回应；全球 API 生态系统正在增长；开发人员在 API 上花费的时间比以往任何时候都多，几乎所有人都认为 在 API 上的时间和资源投入在未来 12 个月内将会增加或保持不变。

为了更多地了解 API 在企业中的地位，我花了 2021 年的一部分时间为我的播客[](https://podcasts.apple.com/us/podcast/breaking-changes/id1574147652)采访了来自不同公司的 22 位工程领导者。我想深入探究这些领导者如何以及为什么认为 API 对商业战略至关重要，并听听他们会与他人分享哪些经验。

这里，在 20 个一口大小的块中，是重复出现的主题:

****经商。**API 运营产生的最大价值很少直接来自任何单一的 API，而是围绕合作伙伴支持 API 实现的。一次又一次，公司看到 b 建立一个 [API 生态系统](https://devops.com/?s=API+ecosystem)有助于扩展他们的业务范围，加深与合作伙伴和客户的联系。**

****策略。**如果没有适当的规划，当涉及到 API 操作支持的应用程序时，组织将总是停留在被动模式。最好有一个全面的 API 策略。**

****第三方关系。**对第三方系统——组织通过基于云的技术与其共享数据的合作伙伴和供应商的庞大供应链——的依赖呈指数级增长，推动组织开发更正式的外部 API 策略。**

****资源。**围绕 API 操作的资源需要与开发人员交流，支持他们，并帮助他们在使用公司的 API 时生活得更轻松。**

****治理。**在开发跨团队的关系之前，在管道级别的自动化治理中保持谨慎是很重要的，集中式 API 治理方法也是至关重要的。 API 治理是对 API 应用通用规则和安全策略的实践。**

****管理**。API 的管理是一个持续的过程，而不是一个终点，需要大量的计划和关注才能达到预期的目标。**

****全局**。我们都越来越多地接触到 API 的全球受众，每个组织的信息和资源都需要反映日益多样化的受众。**

****多协议。**API 的未来是多协议的，需要一个多样化的 API 协议和模式的工具箱，以在未来十年中适当地发展生态系统。**

****启用**。公司需要授权他们的 API 开发团队，而不是管理他们；确保他们有做好 API 所需要的东西，同时也能安全地失败。**

****规定**。API 法规已经出台，并正在推动主流企业成为技术公司，这要求他们拥有面向未来的 API 战略。**

****移动**。世界在 2019 年达到了一个里程碑:现在手机比人口还多。API 是在领先市场和新兴市场的这些设备上提供数字资源的方式。**

****固执己见**。对 API 生命周期的明智而有主见的方法将比基础设施堆栈中使用的任何单一技术产生更大的影响。**

****智力**。在我们每天跨核心业务应用程序发出的数百万个 API 请求中，有一个巨大的机会来获得总体智能。**

****质量**。组织不仅应该优先测试 API，还应该优先测试 API 测试本身。这确保了所有 API 的质量和可靠性水平，而不管使用它们的团队是什么。**

****教育**。确保团队围绕 API 生命周期进行适当的持续培训和教育必须是 API 操作的首要任务。**

****认证**。API 基础设施的身份验证层拥有丰富的智能。让人类研究这个维度对安全至关重要。**

****数据**。API 数据层的挑战和机遇是巨大的，机器学习正在给团队一个外骨骼，以支持开发期间更快的失败和迭代。**

****隐私**。确保隐私是 API 生命周期的基本组成部分有助于开发人员专注于他们最擅长的领域，从而开发出尊重隐私的高质量应用程序。**

****可观性**。API 分析在过去几年已经成熟；变得更加关注 API 基础设施的可观察性，并帮助组织了解和响应重要的事情。**

****开源**。开源技术是发展和支持 API 生态系统的重要组成部分，提供了健康发展所需的技术和人力资源。**

**通过记住这 20 点，组织可以最大限度地发挥 API 的作用，使其成为利用数据和应用程序功能实现业务成功的极其强大的机制。API 将在未来许多年支撑数字化转型，因此涵盖所有基础非常重要。**
