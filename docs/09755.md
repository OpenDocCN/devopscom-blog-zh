# DevOps World 2022:使用 SBOMs 保护软件供应链

> 原文:[https://devo PS . com/devo PS-world-2022-using-s boms-to-secure-the-software-supply-chain/](https://devops.com/devops-world-2022-using-sboms-to-secure-the-software-supply-chain/)

随着供应链攻击变得越来越普遍，可见性正成为网络安全的一种需要。提高可见性和保护软件供应链的一个方法是使用[软件材料清单(SBOM)](https://devops.com/sboms-101-what-you-need-to-know/) 。

SBOM 是代码库中所有组件的列表。通过提供对开源和第三方组件的深入了解，SBOMs 提高了整体可见性，并且是识别安全或许可证威胁不可或缺的一部分。为了适当地保护软件供应链，组织必须了解其应用程序中的所有组件；对于容器化的应用程序和那些使用云原生的、基于微服务的架构构建的应用程序，这样做尤其重要。这就是 SBOMs 成为全面应用安全战略基石的原因。

## 对能见度的需求

Gartner 最近的报告[sbom 的创新洞察](https://www.gartner.com/en/documents/4011501)强调了持续了解软件开发生命周期的必要性，以及 sbom 如何帮助组织和开发人员实现这一目标。虽然 Gartner 承认开源软件是现代软件开发的基础组件，但它也指出它可能是软件漏洞的主要促成因素。

报告称，虽然可重用组件和开源软件简化了软件开发，但这种简单性暴露了一个关键的可见性差距:组织无法准确记录和总结他们生产、消费和运营的大量软件。如果没有这种可见性，软件供应链很容易受到与软件组件相关的安全和许可合规性风险的影响

为了降低相关风险，Gartner 的报告不仅强调了 SBOM 的重要性，还强调了自动化 SBOM 创建和验证流程的必要性。为了实现大规模的供应链安全，组织应该自动生成和验证 SBOMs，并使用该数据不断评估即将到来的风险。

这与持续集成和持续部署(CI/CD)流程直接相关，该流程使用 SBOM 数据和评估来确保威胁不会被发现并造成严重破坏。您可以更轻松地在管道中的任何阶段识别安全漏洞并消除它们，因为 SBOM 为您提供了更精细的可见性。

美国联邦政府甚至注意到了 SBOMs 的重要性。正如拜登总统的网络安全行政命令 14028 所述，任何向联邦政府出售软件的公司都必须提供完整的 SBOM。

该计划还扩展到了政府的其他部门，网络安全和基础设施安全局也强调了 SBOMs 的重要性。

CISA 在其网站上的一份声明中表示，CISA 将通过促进社区参与、发展和进步来推进 SBOM 工作，重点是规模扩大和可操作化，以及工具、新技术和新用例。

## SBOM 要求

美国商务部和国家电信和信息管理局(NTIA)已经列出了所需的最低要素。首先，需要包含每个组件基线信息的数据字段。此外，还需要自动化支持，因为 SBOMs 需要机器可读性来支持规模化。最后，概述了对实践和过程的需要，要求频繁的 SBOM 生成，包括传递依赖性和向需要的人广泛提供该信息。

在许多方面，SBOMs 在整个软件行业已经变得不可协商。为了有效地检测和响应威胁，组织必须采取先发制人的措施，以确保最大限度地提高可见性，而 SBOMs 正好可以做到这一点。
