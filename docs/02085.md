# CLOUDBEES 和 GOOGLE CLOUD 合作加速 ANTHOS 上的应用程序开发

> 原文:[https://devo PS . com/cloud bees-and-Google-cloud-partner-to-accelerate-application-development-on-anthos/](https://devops.com/cloudbees-and-google-cloud-partner-to-accelerate-application-development-on-anthos/)

*devo PS 和云计算领域各自的领导者合作提供从源代码到产品的端到端应用开发自动化，使企业能够在混合环境中构建和更新应用*

**DEVOPS WORLD | JENKINS WORLD，SAN FRANCISCO****–2019 年 8 月 14 日—**[Cloud bees](https://www.cloudbees.com)^(，推动持续经济发展的企业 DEVOPS 领导者和 [Google Cloud](https://cloud.google.com/) ，今天宣布两家公司正在合作提供一个基于开源技术的现代 DevOps 平台，该平台由 [Google Cloud 的 Anthos](https://cloud.google.com/anthos/) 提供支持。)

CloudBees 为各种规模的公司提供基于 Jenkins 的连续交付解决方案，这些解决方案安全、支持开放工具链，并且可扩展，以在混合计算环境中转变软件交付流程。

谷歌云正在为企业提供一个安全、开放、智能和变革性的企业云平台。Anthos 是一个混合和多云平台，构建于 Google Cloud 开创的开源技术之上，支持内部和云环境之间的一致性。

Google Cloud 和 CloudBees 一起帮助加速应用程序开发，并通过连续交付自动化、服务网格、容器和微服务等转型技术战略性地支持企业。

CloudBees 和 Google Cloud 对开源有很深的承诺。CloudBees 是支持 Jenkins 和 Jenkins X 项目的领先公司之一，Google Cloud 是 Kubernetes 和 Tekton 等项目的主要领导者。两家公司都认为开源是应用开发的未来，也是公共云的长期成功。

此外，CloudBees 和 Google Cloud 是新成立的[持续交付基金会](https://cd.foundation/) (CDF)的创始成员，这是一个开源基金会，旨在促进行业顶级开发人员、最终用户和供应商之间的供应商中立协作，以进一步推动持续集成和持续交付(CI/CD)最佳实践和行业规范。

“开发人员需要以最适合其业务的方式构建和运行应用程序，无论应用程序是在本地运行还是在云中运行。谷歌云混合云合作伙伴主管 Rayn Veerubhotla 表示:“这越来越意味着采用 Anthos 的混合方法，允许开发人员在云中运行关键应用，而无需完全消除他们的内部基础设施。“我们很高兴扩大与 CloudBees 的合作伙伴关系，以加速混合云和公共云环境中的应用开发。”

这些公司正在使用 Jenkins X、Kubernetes 和 Tekton 等转型技术来创建一个统一的端到端软件交付系统。

今年早些时候，CloudBees 宣布在 GCP 市场推出 [CloudBees Jenkins 发行版](https://console.cloud.google.com/marketplace/details/cloudbees/cloudbees-jenkins-distribution?q=cloudbees&id=eccee7b8-70a5-4ac5-9322-6a6d0b751675)，这是一个经过认证的开源 Jenkins 发行版。

8 月 15 日星期四，在 DevOps World | Jenkins World 大会上，Google 云平台的开发者倡导者 Dave Stanke 将发表主题演讲，题目是: [*再见，牧羊犬:迈向云原生 CI/CD*](https://sched.co/S6fo) *，*，他将回顾健康的 DevOps 实践的标志，并描述如何通过一系列战略实现关键能力，从现有系统的增量现代化到全新的云原生 CI/CD 平台。

CloudBees 首席战略官 Francois Dechery 表示:“我们与 Google Cloud 的合作为希望在现代混合环境中加速应用开发的应用开发人员带来了令人惊叹的新功能。“与此同时，企业将能够利用 Anthos，利用现有技能和一致的平台，在内部或云中实现应用程序的现代化，从而更快地将产品推向市场，并最大限度地提高资源利用率。”

**关于 CloudBees**

CloudBees 通过提供世界上第一个端到端连续软件交付管理系统(SDM)为连续经济提供动力。对于数百万推动各种规模企业创新的开发人员和产品团队来说，SDM 建立在持续集成(CI)和持续交付(CD)的基础上，使软件交付组织内部和周围的所有功能和团队能够最好地合作，以扩大价值创造。

CloudBees 是 CI、CD 和应用程序发布流程编排(ARO)的核心，建立在其产品的商业成功及其开源领先地位之上。CloudBees 是[詹金斯](https://jenkins.io/)和[詹金斯 X](https://jenkins-x.io/) 的最大贡献者，也是[持续交付基金会](https://cd.foundation/) (CDF)的创始成员。从全栈开发人员实践 NoOps 的创业公司到大型财富 100 强公司，CloudBees 使所有软件驱动的组织能够在正确的时间智能地部署正确的功能。

超过 3，500 个世界知名品牌和超过 50%的财富 500 强企业投资 CloudBees，因为它能够跨任何云、在任何开发环境中工作，并平衡公司治理和控制与开发人员的灵活性和自由度。CloudBees 是世界领先的 DevOps 专家的家园，帮助数千家公司利用“持续一切”的力量，将他们置于从伟大的想法到伟大的软件，再到伟大的商业价值的最快路径上。
