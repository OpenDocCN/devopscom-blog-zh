# 借助多云服务，从云混乱到云智能

> 原文：<https://devops.com/from-cloud-chaos-to-cloud-smart-with-multi-cloud-services/>

采用多云服务的组织将体验到实实在在的成果和大量优势，包括更强的弹性、敏捷性和数据主权。这就是为什么今天大多数企业使用两个或更多公共云的原因，81%的企业在最近的调查中报告[他们计划到 2024 年采用多云战略。](https://blogs.vmware.com/learning/2022/05/27/what-to-expect-from-multi-cloud-academy/)

但是，尽管有这些好处，事实也证明，多重云可能会很复杂，甚至令人不知所措。它需要一种全新的方法来优化工作负载、简化开发并应对日益复杂的运营模式。这被描述为云混乱，更多的云选择导致了复杂性的大幅增加。大多数人表示，构建新的应用程序既缓慢又繁琐，而且跨不同的云管理他们的整个应用程序组合既困难又昂贵。难怪每个云都要求其团队使用彼此孤立且不兼容的专有工具。同时，对于普通员工来说，从任何地方快速、安全地访问关键应用程序是必不可少的，但这通常是一个挑战。所有这些复杂性对企业有着广泛的影响，从管理成本到吸引和留住顶尖人才。

解决这种多云复杂性需要从创建一个抽象层开始，这个抽象层跨越各种不同的云环境，并且不妨碍对每个云提供商的独特服务的访问。然而，组织必须更进一步，解决如何在多云计算环境中以最佳方式构建、管理、治理和优化应用和工作负载的挑战。

### 多云服务

多云服务是一种快速兴起的 IT 服务类别，是帮助组织实施多云战略的关键，该战略可简化和精简跨云的开发、运营、联网和安全性。多云服务跨云提供一致的 API、对象模型、身份管理和其他核心功能。并且，它具有以下一个或多个特征:

1.运行于单个云上，但支持与至少两个不同云的交互
2。在多个云上运行，支持与至少两个不同云的交互
3。在任何云或边缘上运行，甚至在离线模式下运行，并且基本操作完全自动化

多云服务可用于将云基础设施和运营标准化到一个平台中，以降低从多个云中单独构建或使用等效原生服务的复杂性。在这种模式中，公共云、数据中心、私有云以及边缘位置都是垂直的，而多云服务是水平的，可以在这些环境中提供功能。

让我们仔细看看五个最常见的多云服务类别，以及何时使用它们。

### 1.应用服务程序

应用服务允许团队跨云以标准化的方式使用核心服务。这些核心服务包括可观察性、复制、备份和恢复。通过实现应用服务，企业可以加快创新速度，并使其技术更易于使用。应用服务的例子包括开发者工具、数据库、模块化开发平台、人工智能、机器学习以及无服务器和 CI/CD 能力。

### 2.基础设施服务

如果您的组织选择将其应用程序作为虚拟机、容器或两者来运行，使用基础架构服务是跨云实现管理和运营标准化的正确举措。不同类型的多云基础设施服务包括利用虚拟机或容器管理和观察计算、存储和网络服务的能力、基础设施自动化和 [Kubernetes 管理](https://devops.com/platform9s-managed-kubernetes-solution-adds-arbitrage-capabilities-to-dynamically-run-nodes-in-a-cluster-on-amazon-ec2-spot-instances/)解决方案。应用时，基础设施服务可以提高弹性、安全性、性能、互操作性和可移植性。

### 3.安全服务

当使用多个云时，企业遇到的一个问题是，每个单独的云都有自己的安全工具和方法，因此很难确保您的企业免受网络威胁。这就是多云安全服务的用武之地——集中安全运营和安全软件供应链。例如，网络检测和响应、端点检测和响应、新一代防病毒和安全访问服务 edge 等多种云安全服务。这些服务通过提供单一界面，使组织能够实现一致的可见性、上下文和控制。

### 4.最终用户服务:获取数据和服务

无论您的企业有多大，或者您使用多少应用程序，都有必要为应用程序和数据提供适当的访问级别。但是，当您拥有多云基础架构时，这可能会很困难。由虚拟桌面、设备管理和终端用户应用交付组成的多云终端用户服务将为您的组织提供一种统一的方式，让所有员工能够快速、安全地访问您的应用，同时简化设备管理。

### 5.数据平面

过去，工作负载被限制在单个公共或私有云或边缘位置中包含的单个数据平面上。现在，随着多云服务的兴起，企业可以在多云数据平面中运行工作负载。在多云数据平面中运行的工作负载的一个主要示例是在多个云提供商环境中运行的分布式应用程序。应用程序的一部分可能在私有云中运行，而其他服务部署在公共云中。多云数据平面是容器、服务和数据存在的地方，也是用户交互发生的地方。

### 从云混沌到云智能

每一次云计算之旅的理想目的地都应该是从混乱的云计算环境转向智能的云计算环境。智能云意味着采用结构化和有计划的方法来实现多云和数字化转型。随着越来越多的企业采用多种云，多种云服务将成为云智能的关键，因为它有助于降低复杂性和风险，并证明可以根据技术和业务要求自由选择适合所有情况的云。
