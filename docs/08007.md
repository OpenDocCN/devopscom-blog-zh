# 学习 DevOps 的“第一种方法”——持续流动

> 原文：<https://devops.com/learning-the-first-way-of-devops-continuous-flow/>

许多组织都在努力实现精心设计的开发运维，甚至是开发运维的“第一种方式”——持续流。

DevOps 是一个不断发展的知识体系。没有涵盖开发运维所需的所有角色和全部技能的一站式商店。具备开发运维能力的员工队伍需要持续致力于开发运维技能的培养和掌握。企业必须战略性地开展 DevOps 培训，以最大限度地利用培训资源。

## **学习连续流**

DevOps 的第一种方式，持续流，是 DevOps 的基础。

### 为什么学习连续流程对工程开发很重要？

为了实现战略目标，需要一支知识丰富、技术熟练的员工队伍。如果没有对 DevOps 概念和术语的坚实理解，人和团队之间的交流将是低效的，并且是瓶颈和误解的来源。为了理解和实施 DevOps 管道，需要开发工具和工作流的实践技能。整个行业都缺乏熟练的 DevOps 候选人。新技能促进创新。知识型员工降低生产成本，减少错误。有熟练的同事有助于营造令人满意的工作环境。员工保留更好。竞争优势来源于知识。

### **学习连续流是如何完成的？**

以下是[学习](https://devops.com/?s=learning)连续流的推荐工程实践。

我们鼓励员工不断学习，致力于发展 DevOps 技能。DevOps 培训是战略性的，以确保 DevOps 培训与业务目标保持一致。进行 DevOps 技能差距评估，并用于定义和完善培训策略。鼓励领导和跨职能团队参加 DevOps 基础培训。领导者定义最适合组织需求和优先事项的培训策略，以弥补关键的技能差距。

培训策略考虑哪些技能和角色可以由廉价的自学资源涵盖。确定哪些技能和角色需要更昂贵的培训。DevOps 培训有明确的预算。跟踪 DevOps 培训计划的合规性。DevOps 领导力培训面向与 DevOps 相关的管理层和员工。DevOps 工具培训包括使用服务目录中每个 DevOps 工具的推荐工程实践。我们鼓励并支持员工参加与其指定角色相关的 DevOps 认证。

定义了开发运维培训计划，其中包括针对每个角色的推荐课程，包括领导者、开发人员、QA、Ops 以及在开发运维环境中运行的基础架构和安全人员。推荐的 DevOps 培训资源已编目，所有利益相关方均可查看该目录。奖励和认可计划鼓励员工掌握 DevOps 技能。

所有员工都有时间参加 DevOps 培训。DevOps 指导计划是正式培训的补充。鼓励并计划工作交换和沉浸式培训。鼓励人们参加外部会议和聚会，并与其他团队成员分享他们所学到的东西。

### **实施持续流学习需要什么？**

以下是实施连续流学习计划的推荐工程实践。
为 DevOps 培训领导分配角色和职责(例如，DevOps Sensei、培训 PM、传道者)。进行 DevOps 技能差距评估。定义一个最能弥补差距、满足组织需求和优先事项的培训策略。定义培训预算，包括每个角色和技能的费用和时间分配。

确定哪些技能和角色可以被廉价的自学资源所涵盖。确定哪些技能和角色需要更昂贵的商业培训。确定每个角色的培训目标。确保招聘档案符合与组织战略目标最相关的开发运维技能的技能要求。将使培训资源可见和可用的方法落实到位。为团队和个人设定培训目标。为完成培训制定激励和认可计划。
