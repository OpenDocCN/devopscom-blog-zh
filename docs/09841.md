# Checkly 宣布业务里程碑、SOC 2 合规性、新的剧作家测试运行程序、GitHub Sync、增强的警报通知等等

> 原文:[https://devo PS . com/checkly-announcements-business-milestones-SOC-2-compliance-new-剧作家-测试-跑步者-github-sync-enhanced-alert-notifications-and-more/](https://devops.com/checkly-announces-business-milestones-soc-2-compliance-new-playwright-test-runner-github-sync-enhanced-alert-notifications-and-more/)

**马萨诸塞州波士顿，2022 年 10 月 11 日—**Checkly Inc .是唯一一家拥有专为开发人员打造的合成应用和 API 监控平台的公司，该平台在开发、运营和质量保证之间架起了一座桥梁，该公司宣布已有 600 多家客户和数千名开发人员采用了该平台，与他们现有的开发工作流程无缝集成。该公司已经超过 35 亿次由用户运行的检查，在速度和规模上实现了 Checkly 的端到端监控。该公司还宣布，它正在用一系列新产品功能更新 Checkly 平台，包括新的剧作家测试运行器集成，移动警报改进，以及新的 GitHub Sync 功能的普遍可用性，以更新和控制浏览器校验码。

该公司展示了其对客户隐私、安全性、数据可用性需求的一贯承诺，并响应了企业客户不断增长的要求，现在还完全符合 SOC(系统和组织控制)2 Type 1。SOC 2 被广泛认为是与管理客户数据相关的最重要、最严格的合规标准之一。

Checkly 首席执行官兼联合创始人汉尼斯·伦克表示:“空客、Vercel、1Password、Deno 和 Schwarz IT 等公司的开发团队选择 Checkly，因为我们的可编程、代码监控方法让开发人员相信问题会很快被发现。”。“我们已经达到了许多业务里程碑，并加倍支持领先的开源自动化框架剧作家。我们还实现了 SOC 2 Type 1 合规性，并推出了许多新产品功能，所有这些都支持我们为开发人员创建最佳合成应用和 API 监控平台的单一愿景。”

Checkly 业务亮点:

*   Checkly synthetic 应用和 API 监控平台已被全球顶级品牌和组织的开发团队采用，包括 1Password、Airbus、commercetools、Deno、Hays、Joyn、Prisma、Schwarz IT、Vercel、Yext 等。
*   Checkly 向开发人员展示了该平台的深层价值，它已经运行了超过 35 亿次 API 和浏览器检查，每月进行 3.43 亿次检查和 40 万次警告，并且还在增长。
*   在 Digital Enterprise Journal 进行的一项名为“[2022 年管理 IT 性能的 20 大新兴供应商](https://blog.checklyhq.com/dej-names-checkly-leader-in-cloud-native-monitoring/ "null")”的研究中，分析师 Bojan Simic 发现 Checkly 在与关键用户需求一致的八个类别中处于领先地位。
*   Checkly 从 50 多家领先的远程优先和远程友好型全球公司中脱颖而出，荣获“[领先的远程优先公司](https://runningremote.com/session/winners-of-the-flexible-work-awards-doist-checkly-livit-cimpress-vista/ "null")”类别的灵活工作奖。
*   Cinthia Portugal 加入 Checkly 之前曾在 CrowdStrike 担任 Humio 营销副总裁兼全球负责人。作为 Checkly 的 CMO，Cinthia 将领导全球营销战略，并有助于扩大公司在新的和现有市场的增长，并扩大其全球行业范围。

Checkly 产品更新:

*   对剧作家测试的本机支持将现有的客户 E2E 脚本直接引入 Checkly，并支持 expect、跟踪文件、视频记录等。
*   增强了与领先提供商的本机集成，这些提供商支持 Checkly 的 Monitoring-as-Code (MaC)工作流，包括 Terraform、Pulumi 和新发布的 GitHub Sync。
*   新的警报增强功能增加了 Checkly 警报的透明度，而新的移动改进功能优化了平板电脑和智能手机的 Checkly 体验。GitHub Sync——用于更新和控制浏览器校验码——现在也普遍可用。
*   为了更好地支持企业客户的需求，Checkly 平台完成了从 Heroku 到 Amazon Web Services (AWS)的迁移，以受益于更好的数据库可扩展性、更好的可观察性以及对跨区域备份的完全控制，从而实现灾难恢复。
*   为了定期发布新的平台特性和功能，Checkly 将举办季度发布周，从 [Checkly 发布周 1](https://www.checklyhq.com/launch-week "null") 开始，于今天，10 月 11 日，太平洋时间上午 9:00/美国东部时间中午/欧洲中部时间晚上 9:00 开始。

“我们一直在使用 Checkly 的最新功能，GitHub Sync 和与剧作家测试的集成，我们看到了这两者的巨大价值，”Plex，Inc .的首席软件工程师 Eric Matthys 说，“GitHub Sync 使我们消除了与 CI 管理 Terraform 相关的复杂性，这在以前需要额外的服务。有了 Checkly，现在简单多了，贡献者需要理解的部分也更少了。Checkly 的剧作家测试集成允许我们编写端到端测试，并以(几乎)与其他测试相同的方式在本地运行它们。我们发现异步断言对我们的工作特别有帮助和价值。”
