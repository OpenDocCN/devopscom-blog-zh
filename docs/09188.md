# 实现更可持续发展的 5 个步骤

> 原文:[https://devops.com/5-steps-to-more-sustainable-devops/](https://devops.com/5-steps-to-more-sustainable-devops/)

远程劳动力、加密货币和超大规模数据中心的兴起，将技术发展中一个没有得到足够重视的问题推到了前沿:计算和先进技术对环境的影响。

进入疫情大约六个月后，人们开始注意到更干净的空气、更清澈的水，以及在工业减少的情况下对环境的整体平静效果。迷因被创造出来并广为流传，暗示着也许我们终究是问题所在。

这种情况持续了大约六个星期，然后幽居病就出现了，我们变得不耐烦了，准备像往常一样重新开始工作。

在 2020 年最后一个季度和 NFTs 的突然兴衰之间的时间里，一些胆怯的声音再次引发了对超级计算和其他技术对环境影响的担忧。这些担忧最终在社会经济和政治动荡、对通胀的担忧、员工短缺和围绕供应链中断的物流问题中变得模糊不清。

当我们努力在后大流行时代重新找到自己的立足点时，应用程序开发试图填补这些空白，新的数据中心出现了。

几十年来，环境问题一直是人们越来越关注的问题，但在过去几年中，人们对这一问题的关注尤为强烈，从 [气候友好型投资](https://www.carboncollective.co/interactive-climate-index) 到 [碳抵消](https://www.offsetguide.org/understanding-carbon-offsets/what-is-a-carbon-offset/) 等等，整个行业都在围绕着这一问题发展。尽管人们越来越关注可持续性，但技术对环境的影响已经成为一个尚未解决的问题。

## 在你的集结地向大象致辞

典型的 DevOps 之旅主要关注应用开发将如何影响组织。它很少涉及发展如何影响环境。

除了缩短上市时间、减少人为错误、调试和提高财务资源管理效率等因素外，您的 DevOps 战略还应包括减少碳足迹和对地球影响的方法。

首先，它有助于准确理解技术如何影响气候和耗尽自然资源，这样你就可以以一种有意义的方式将可持续发展 [纳入你的计划](https://devops.com/no-matter-how-things-change-have-a-plan/) 。

*   **碳排放:** 据估计，到 2022 年，全球已经有超过 165 亿部移动设备在使用。仅仅一部 iPhone 在它的生命周期中就排放高达 86 公斤的二氧化碳，其中 80%发生在它到达消费者手中之前。这只是碳排放的冰山一角，而且只会越来越糟。
*   **功耗:** 所有的碳都来自某个地方，从一项技术的能耗就可以看出计算对这个问题的影响有多大。截至 2020 年，数据中心每年使用约 416 太瓦的能源。仅比特币网络消耗的电力就超过了全球所有数据中心的总和。一笔 [单笔 NFT 交易](https://memoakten.medium.com/the-unreasonable-ecological-cost-of-cryptoart-2221d3eb2053) 消耗 82 度电，排放 48 公斤 CO2。

即使是相对被动的技术使用，比如视频流和远程协作，也有可能比整个国家消耗更多的电力。把网飞、放大会议和其他远程活动的所有时间加起来，思考其中的含义。

*   **废物:** 尽管有很多设备在使用，但很少硬件被回收。电子垃圾占全球有毒垃圾的 70%，预计每年 [总量](https://www.theworldcounts.com/stories/electronic-waste-facts)4000 万吨。
*   **水消耗:** 你不经常考虑混合水电，但所有这些数据中心都需要以某种方式冷却。仅互联网使用一项，每人每年就消耗 3000 升水，排放 2000 公斤二氧化碳。

## 为了气候的好处，清理你的 DevOps 过程

要成为解决方案的一部分，除了投资绿色企业之外，更重要的是要有所作为。这里有一些你可以采取的步骤来清理你的 DevOps 过程，并成为解决方案的一部分，而不是造成问题。

### **确定易受攻击的领域**

**软件工程和 DevOps 越来越多地拥抱新的领域，强调可持续工程。正确方向的第一步是意识到在开发周期中能量和资源是如何消耗的。**

**考虑:**

*   **你的力量是如何产生的。例如，你是依赖化石燃料还是更环保的方法？**
*   **您多久更换或回收一次硬件和其他物理组件？这是减少浪费的一个目标。**
*   **您如何冷却数据中心和计算设备？你保持最佳温度的努力是在浪费资源吗？**

### **平衡可持续发展的三个领域**

**在可持续工程领域，流程可分为三类:技术、运营和环境。目标是平衡这些方面，以便 [监控和优化](https://devblogs.microsoft.com/sustainable-software/how-to-measure-the-power-consumption-of-your-frontend-application/) 开发过程，尽可能减少损失。**

### ****实施可持续发展实践****

****一旦您确定了风险领域并设定了可持续软件工程的目标，使用结果来创建并实施一套开发最佳实践，以更好地支持对环境负责的设计和生产。如果你需要一些帮助或想法，你可以在 这些 [新兴的标准化原则](https://principles.green/) 之后对自己的倡议进行建模。****

### ******成为绿色技术社区的一员******

******2021 年，200 多个国家签署了[](https://cloud.google.com/blog/topics/sustainability/thoughts-on-cloud-and-climate-change-from-cop26)Glasgow 气候公约，巩固了他们减少碳足迹和技术开发和使用对环境造成的破坏的承诺。 大会聚集了来自企业、政府机构和其他机构的利益相关者，分享信息并为可持续计算和技术发展制定标准和最佳实践。******

### ******整合支持可持续发展的工具******

******谷歌、亚马逊和微软等科技巨头正在开发工具，帮助开发人员和 IT 部门监控和测量从云计算到应用程序开发的一切事物对环境的影响。找出这些工具并将其整合到您自己的 [最佳实践](https://cloud.google.com/blog/topics/sustainability/new-tools-to-measure-and-reduce-your-environmental-impact) 中，或者创建您自己的解决方案。******

## ********最终想法********

****科技发展旨在提高我们的生活质量，重塑我们的经营方式。没有理由不把意图延伸到我们生活和从事商业的世界。****

****在不对发展产生负面影响的情况下应对气候变化的第一步是理解技术如何影响我们的环境。然后，我们可以做出明智的决策，支持负责任的可持续发展。****
