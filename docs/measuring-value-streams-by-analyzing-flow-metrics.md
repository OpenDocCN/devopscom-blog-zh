# 通过分析流程指标来衡量价值流

> 原文：<https://devops.com/measuring-value-streams-by-analyzing-flow-metrics/>

IT 管理员面临越来越大的压力和需求，要求他们快速、经济地将软件推向市场。为了实现这个目标，IT 部门经常汇编各种度量标准来测量特定的开发过程。

数据和指标是重要的工具，但管理员有时不会考虑通过更广泛的视角来看待项目——通过测量总的生产工作流程。这可能会抑制生产，并对业务成果产生负面影响，因为整个价值流中的低效率未被发现。

换句话说，需要整个[价值链](https://www.vsmconsortium.org/)的总体可见性来完成精益项目管理和优化开发运维[工作流](https://devops.com/?s=workflows)。

让我们来谈谈不同种类的流量指标，为什么它们很重要，以及如何开始度量它们。

## 流量度量:它们是什么意思？

流程度量指示关于期望的业务成果的价值交付率。这使得组织能够在宏观层面上了解生产情况，从而做出更有效的决策。流量度量还可以生成一段时间内的历史性能分析，这对于关注持续增长的企业来说是很有价值的。

IT 领导者现在广泛使用流度量分析来弥合业务团队和技术之间的差距，因为它可以揭示影响关键目标的各种问题。例如，团队可以识别优先化挑战，为什么项目比预期时间长，查明技术债务等等。

当评估流程度量时，IT 部门测量客户或产品团队移交给他们同事的单个工作单元——也称为流程项目。这可能是对新软件功能、安全缺陷或需要修复的东西的请求。

## 有哪些不同种类的流量指标？

随着对流程度量的更好理解，我们现在可以更深入地研究组织在整个开发过程中应该评估的五个流程度量。

### 1.吞吐量

吞吐量衡量流项目总数完成的速率。通过测量这一点，吞吐量可以确定在特定项目中获得和分配的价值总量。

吞吐量通常是按周计算的。在历史基础上测量和评估吞吐量也有助于找出优先级冲突如何影响整体价值。

优化吞吐量可以带来更好的最终用户体验和客户满意度，这自然会推动收入和增长。

### 2.周期

周期时间是 IT 团队交付价值的速度。这意味着考虑到活动时间和等待时间，对流程项目从工作开始到工作完成所需的时间进行计时。

在推动收入和增长方面，周期时间与产量同样重要。测量周期时间可以简化确定项目评估和分配资源的过程，从而增强工作流程并提高效率。

### 3.流动效率

流动效率是活动时间与等待时间的比例。测量流程效率可以发现哪里出现了障碍，以及流程浪费是上升还是下降。期望的流动效率应该高于 15%。

大部分流动时间来自等待，导致路障和生产成本增加。测量流程效率可以让团队查明工作流何时花费了太多时间，然后他们可以采取积极的措施来加速单个流程。

### 4.正在进行的工作

进行中的工作涉及需求测量。团队通过测量价值流中悬而未决或正在开发的流程项目的数量来实现这一点。衡量进展中的工作通常使 IT 领导能够识别价值流是被过度使用还是未被充分使用。

没有人是完美的。这就是为什么领导者必须意识到每个员工的时间是如何被利用的。通过这样做，他们可以系统地分配工作负载，并根据需要提高质量。这反过来降低了倦怠和流动，同时也减少了需要返工的错误。

### 5.工作简介

在确定优先顺序时，衡量工作概况很重要。它被分析为在特定时间段内完成的其他四个流程项目的比率。

评估工作概况有助于优化质量和生产率。这提高了客户体验(CX)，从而导致更好的保留和审查。

## 为什么我应该跟踪流量指标？

在过去，DevOps 专业人员很大程度上没有必要的工具来跟踪和增强流度量，但是最近在跟踪这些度量方面的改进使得该过程更加高效和容易。

流量跟踪有三个主要原因:

### 1.促进交流

缺乏沟通可能是业务和软件团队之间的常见问题。流程跟踪使 IT 管理员能够深入研究流程指标并洞察价值流。这有助于团队提供准确的项目评估以及实时更新，这使得与涉众的交流更加容易。

### 2.确定期望的业务成果

拥有价值流的总体可见性有助于 IT 和业务领导在确定关键项目和确定其优先级以及建立目标方面进行合作。

例如，提高软件质量可能是一个目标。团队可以检查各种流程度量，以确定是什么导致了软件质量问题，如何修复这些问题，以及对软件质量进行优先调整会如何影响其他工作流。

### 3.根据情报采取行动

软件组织花费大量的时间和金钱试图加速软件开发。他们召开一个又一个会议，雇佣顾问，甚至引入新的团队成员，但是他们简化工作流程所需的解决方案可能很简单。他们只是需要被发现。

借助流程跟踪，组织可以全面、实时地了解价值流。这使得根据智能采取行动和增强开发变得快速而简单，从而导致不需要猜测的准确的、数据驱动的决策。

## 通过流量指标提升您的价值流

在工作流中实现最佳效率需要端到端的可见性，通过分析流程指标可以轻松实现这一点。

使用流程度量来最大化价值流提高了可见性，加速了开发并降低了软件生产成本。
