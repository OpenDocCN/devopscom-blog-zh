# 实施云原生开发运维的 5 项最佳实践

> 原文：<https://devops.com/5-best-practices-for-implementing-cloud-native-devops/>

随着世界向云原生产品的过渡成为行业规范，DevOps 在支持更高效的 IT 基础架构方面的关键作用越来越受到重视。DevOps 旨在通过简化自动化流程来加快应用程序的创建和部署，从而促进协作和沟通。实施[云原生](https://containerjournal.com/?s=cloud-native) DevOps 需要组织进行大规模的文化转变。如果没有这种改变，企业就不太可能具有竞争力或者充分利用云所提供的创新。

采用新技术总是有风险的。以下是加速贵公司实施流程和正面应对潜在挑战的五个最佳实践。

### 1.想云，采纳云。

利用各种专为云定制的工具优化技术。了解技术如何在云原生 DevOps 所依赖的开源环境中工作，对于成功管理组件和选择合适的工具同样重要。不要局限于使用一种工具。相反，为每项任务选择合适的工具，同时尽可能保持与云无关。

### 2.实施动态安全策略。

不考虑安全性是组织在使用新兴技术时遇到的最常见陷阱之一。安全性必须构建到云原生开发工作流和思维过程中，并受到严格监控。如果资金允许，任命一名首席安全官负责审查云中的 DevOps 安全性，并为每个触发的警报创建行动手册。由于对手的不可预测性和每个解决方案中涉及的大量技术，网络安全正变得越来越复杂。一个开始的方法是启动一个 bug 赏金计划，让黑客站到你这边来。

### 3.致力于持续的培训。

执行自我审计和/或渗透测试，以揭示公司技术、程序和人员方面的弱点。投资培训以有效解决云技能方面的任何差距。提供发展机会也将提高员工的参与度和忠诚度，这对许多雇主来说是一个挑战。确保您的员工对培训的内容、维护和发展给予持续的反馈。

虽然只强调新员工培训很有诱惑力，但快速变化的技术环境要求培训努力面向整个团队的持续学习。这项投资将会带来回报，DevOps 和云计算将会带来大量的长期节约。为了提高培训效率，培训可以远程进行。根据员工的工作职能整合并有针对性地开展培训，  但不要忘记非 IT 员工，他们也必须对云和云原生有基本的了解。

### 4.一次，两次，自动化！

自动化对于最小化软件开发生命周期的摩擦和消除对手工劳动的依赖是至关重要的。借助基础设施即代码(IaC)工具(如 Terraform ),可以自动配置网络、基础设施、应用程序部署和设置并进行版本控制。容器、云计算、安全、测试、监控等领域也有无限的自动化机会。例如，容器允许开发人员划分应用程序来处理组件，而不用担心它们在其他领域的潜在影响——这使得构建一致、无混乱的部署变得更加容易。然而，管理多个版本可能是一项挑战，需要基础设施和测试投资。

### 5.性能-测试您的测试。

自动化测试将节省开发人员的时间，减少人为错误。它还允许开发人员利用双倍测试的优势，确保每个变化都是稳定的和基准的。在开发阶段进行测试，为有效的 CI 流程奠定基础，尤其是如果您定期推出补丁。自动化测试在基于云的系统中更容易，并且不依赖于足够的硬件来使其正常运行，从而允许云主机轻松地扩展或缩减。

### 最终外卖

云原生 DevOps 正在彻底改变应用程序的构建和管理方式，它始于我们如何看待我们面临的挑战。接受 DevOps 文化使组织能够利用云和云原生解决方案来创建易于扩展的可靠服务。虽然它看起来可能是资源密集型的，但云原生开发运维正在成为企业的必需品。
