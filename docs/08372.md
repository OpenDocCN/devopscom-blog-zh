# 如何改进云管理

> 原文：<https://devops.com/how-to-improve-cloud-management/>

云技术支撑着人工智能(AI)、物联网、边缘计算和集装箱化等新兴技术。组织必须优先实施有效的云管理实践，以避免螺旋上升的成本和潜在的安全性和合规性违规。

新冠肺炎疫情大大加快了云的采用。为了支持远程工作，许多企业已经将工作负载从内部服务器转移到公共云。与此同时，由于人工智能和机器学习等技术的激增，组织正在产生比以往更多的数据，其中大部分也存储在云中。因此，对云服务需求的爆炸式增长丝毫没有放缓的迹象也就不足为奇了。 [Gartner 预测](https://www.gartner.com/en/newsroom/press-releases/2021-04-21-gartner-forecasts-worldwide-public-cloud-end-user-spending-to-grow-23-percent-in-2021)2021 年，全球最终用户在公共云服务上的支出预计将增长 23.1%，达到 3323 亿美元。

但是，这并非一帆风顺。随着一般组织的云基础架构在复杂性和规模上的增长，团队有效管理它变得越来越困难。这具有重大的安全性、合规性和成本影响。[根据 Accenture](https://www.accenture.com/nl-en/blogs/insights/cloud-trends) 的调查，目前有 30%的组织云支出被浪费。

那么，您能做些什么来确保避免最常见的云管理陷阱呢？

## 查看大图

令人惊讶的是，许多企业，甚至那些拥有大型云架构师团队的企业，对其云基础设施的了解是如此之少。例如，最近的一项调查发现，超过 [25%的 IT 部门](https://www.computerweekly.com/news/450432629/IT-departments-have-no-idea-how-much-they-spend-on-AWS-and-Azure)不知道他们每月的云支出实际是多少。

在采取措施优化您的[云管理](https://devops.com/?s=cloud+management)实践之前，充分了解您的现状至关重要。您需要了解您的成本历史、审计跟踪和 CPU 利用率模式，以确定您超支的地方。

此外，确保你一直使用[标记](https://devops.com/best-practices-for-aws-tagging-with-yor/)。一个定义良好的组织范围的标记策略是必不可少的，它使您能够按类别准确地可视化支出。设置当用户启动没有标签的实例时触发的警报也是一个很好的做法。这可以防止任何潜在的疏忽。

## 制定变革管理流程

由 IT 团队单独处理僵化的变更管理流程的日子已经一去不复返了。如今，业务部门甚至不需要 IT 部门的参与就可以获得云服务。

在云中，变更管理是一系列持续的集成、交付和部署。考虑到这种快速变化的环境，实施强大的变更管理流程至关重要，否则将面临成本急剧上升和潜在违规的风险。

重要的是，您的变更管理流程让您能够在不影响交付速度的情况下了解和控制每个基础架构变更。为了在建立强有力的策略和规则的同时实现这一点，值得考虑至少部分应用交付过程的自动化。通过自动批准所有常规和低风险的变更，您可以为开发人员腾出时间来关注更复杂的项目。

此外，将您的变更管理流程与您的 DevOps 工具堆栈相集成也是值得的。您可以使用协作工具，如 Slack、Teams 和/或吉拉，确保每个人都能看到变化并快速解决任何问题。

## 持续优化您的云基础架构

不幸的是，说到有效的云管理，您的工作永远不会真正完成。对您现有的云基础架构进行一次性审查只会让您到此为止。

许多企业犯了这个错误，几个月后发现自己又回到了旧的坏习惯。为了检查您的云基础架构，最好持续跟踪一系列因素，包括成本、变更、用户、权限和合规违规。毕竟，当你能够在问题失去控制之前发现它时，解决问题就容易多了。

现在，当谈到改进您的云管理实践时，您已经知道从哪里开始了。然而，手动处理上述所有考虑事项是一项艰巨的任务。对于许多开发运维团队已经不堪重负的组织来说，这非常耗时。这就是为什么近年来大量基于 SaaS 的云管理工具如雨后春笋般涌现。这些工具提供了对变更、资源利用、成本、安全性、合规性、性能等无与伦比的可见性。是的，它们是有代价的。但是，大多数企业发现，这种工具使他们能够识别的储蓄大大超过任何支出。
