# 反思:德文普斯圣诞颂歌

> 原文：<https://devops.com/reflections-a-devops-christmas-carol/>

“首先，马利已经死了。”我的 DevOps 圣诞颂歌就这样开始了。在这个倒影中，我被德沃普斯的过去、现在和未来的鬼魂拜访。我希望，在听完我的故事后，你会更好地理解 DevOps 的发展方向，它来自哪里，现在在哪里。

## DevOps 过去的幽灵

DevOps 过去的幽灵把我带回了 2012 年的拉斯维加斯。我和吉恩·金分享了一瓶(或两瓶)葡萄酒，吉恩正在谈论他正在写的一本书的手稿[，这本书是根据埃利雅胡·戈德拉特的《目标》写的。这是一个关于资源限制和基于 DevOps 的前进方向的数字故事。我被这种思考开发和运营的新方式迷住了，同时思考这对安全来说是一件多么伟大的事情，这是我当时的事情。一个 全新的世界展现在我面前，我认识并了解了 Patrick Dubois、John Willis、Jez Humble、Damon Edwards 等人的工作。](https://itrevolution.com/the-phoenix-project/)

我被带到科罗拉多州的博尔德，在那里 Foundry Group 和 Techstars 正在举办一个研讨会，以确保他们的投资组合公司在 DevOps 方面是明智的。Rally software 的 CTO 做了一个关于敏捷和 DevOps 的演示，这就像是从一个全新的角度看待 it 世界。在与我的一些朋友 Rajat Bhargava 和 Brad Feld 交谈后，DevOps.com 的想法诞生了。

那是多么美好的时光啊。DevOps 是一个难以捉摸的东西。没有宣言，没有官方定义。有人说这是虚构的，不是真的。还有人说，这只是针对诞生在云端的创业公司。还有人说，这只是为大型企业。DevOps 似乎可以同时出现在任何地方和任何地方。

回顾过去，那是一个令人振奋的时代，哲学争论激烈。在接下来的几周、几个月和几年里，DevOps 的想法变成了现实，它所提供的好处显现出来，并且出现了一套最佳实践。我和我在 DevOps Institute 的合作伙伴试图捕捉这些新兴的最佳实践，并围绕它们制作课件和认证。哦，天哪——这造成了多大的风暴啊。在 DevOps 认证的想法被嘲笑，被嘲讽，被吐槽，被嘲讽。但那也最终过去了。回顾我旅行的这一部分，我意识到，尽管这一切看起来很理想，很激动人心，尽管道路坎坷不平，但我们总是在前进。

## 德沃普斯的鬼魂出现了

接下来，在场的 DevOps 的鬼魂拜访了我。在这个梦里，DevOps 已经开始融入木制品。在很大程度上，这是一种被接受的开发和操作软件的方式。许多原则如 [CI/CD](https://devops.com/ci-cd-is-still-all-about-open-source/) 、持续测试、 [SRE](https://devops.com/?s=SRE) 和 [DevSecOps](https://devops.com/?s=DevSecOps) 是主流。虽然不是 100%完全采用，但是大多数组织在他们的业务中使用了一些 DevOps 模式。

事实上，这已经变得如此平凡，以至于人们开始问:“DevOps 碰壁了吗？”“DevOps 过了巅峰了吗？”以及“DevOps 的下一步是什么？”虽然这听起来有些不祥，但实际上这是新技术采用周期中的正常现象。就像 Gartner 炒作周期一样确定的是，已经进入堆栈的技术似乎是事情的方式。因此，尽管在场的德沃普斯的鬼魂试图吓唬我，但我并不害怕。收养正如预期的那样进行。作为数字化转型计划的一部分，它被牢牢地固定下来。云原生和 DevSecOps 也越来越被接受。

## DevOps 未来的幽灵

接下来，DevOps future 的鬼魂拜访了我。她让我看到了未来的一瞥。在这个未来版本中，SREs 与 DevOps 团队并肩工作。在部署时进行了平稳的移交，确保了应用程序按照预期的方式运行。自动反馈循环允许开发人员将学到的经验整合到未来的版本中。

为开发人员设计的低代码/无代码解决方案允许更快的自动化部署。支持人工智能的应用程序让测试与开发保持同步，更高的自动化支持更快的部署。

人工智能还让可观察性不仅仅是一项观赏性运动。明智地采取行动的可操作情报是未来的标准。当然，安全仍然是一个问题。还有更多法规和合规性要求。坏人和以前一样臭名昭著。似乎即使是一个安全人人有责的 DevOps 未来也不能完全解决我们的安全问题。但是，总的来说，已经取得了进展。

DevOps 团队直接从 Git 部署到云原生堆栈，从 Kubernetes 开始，但并未就此结束。有连接一切的服务网格和 API。大多数供应商都提供无服务器平台。总的来说，没有人希望局限于一家云提供商，为每个项目选择合适的云。事实上，未来的 DevOps 似乎即将认识到并实现它的许多愿望。

我醒来后对发生在 DevOps 身上的一切感觉良好。用小蒂姆在另一首圣诞颂歌中的话说，“上帝保佑我们，每一个人！”
