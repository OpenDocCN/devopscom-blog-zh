# Instana 发布了针对 PHP 和 Python 应用程序的始终在线的连续生产分析

> 原文：<https://devops.com/instana-releases-always-on-continuous-production-profiling-for-php-and-python-applications/>

*公司为唯一的 APM 解决方案增加了新的语言支持，包括自动性能分析*

针对云原生和微服务应用的 APM 和可观测性解决方案的领先提供商 Instana 今天宣布，可为生产 PHP 和 Python 应用提供始终在线分析。由于增加了这种生产分析支持，Instana 成为第一个为 Java、GoLang、Python 和 PHP 提供自动连续生产分析的 APM 解决方案，具有通用接口，并与生产应用程序监控和跟踪相关联，以便进行更好的分析。

这种最新的语言支持建立在 Instana 对该公司去年收购 StackImpact 所获得的技术的集成之上。它标志着 2020 年两种解决方案的第三次集成产品发布。Instana APM 客户可以无限制地使用生产分析，无需额外付费。启用这些功能只需要更改 Instana 配置文件。打开或关闭分析不需要重启应用程序。

Instana 的 APM 和可观察性策略师 Chris Farrell 说:“今天的应用程序的复杂性与使用的应用程序语言的多语言性直接相关。“Instana 能够将配置文件的自动捕获扩展到 Python 和 PHP，这为开发团队提供了优化性能和快速解决代码问题所需的数据。”

Instana 的生产分析解决方案与其他分析工具的不同之处在于，它始终在线，以如此低的开销设计和构建，可以始终捕获每个生产流程的分析。这些配置文件与其他 Instana 指标相关联，包括应用程序服务级别、所有单个请求的跟踪和全栈基础架构指标。该公司的 Unbounded Analytics 允许任何人深入任何层，并在出现问题时快速隔离和修复问题。

Instana 的企业可观察性平台由自动化的[应用程序性能监控](https://instana.com/application-monitoring)提供支持，可自动发现和映射所有服务、基础架构及其相互依赖关系。Instana 吸收所有可观察性指标，跟踪每个请求，分析每个流程，并实时更新应用依赖关系图，以提供开发和运营所需的上下文和可操作的反馈，从而优化应用性能，实现创新并降低风险，帮助他们提高管道的价值和效率。

Instana 为 PHP 和 Python 以及 Java 和 Go 提供的持续概要分析现在在所有 Instana 解决方案中都可以使用，包括它们的 SaaS 版本和本地版本。在 https://instana.com/profiling 了解更多信息。
