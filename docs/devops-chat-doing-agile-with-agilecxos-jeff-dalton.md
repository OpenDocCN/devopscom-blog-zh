敏捷比 DevOps 有更长的历史和更成熟的发展。尽管如此，一个组织怎么能说它事实上是一个“敏捷组织”呢？这到底是什么意思？

我们这次 DevOps 聊天的嘉宾是 Jeff Dalton，他是 Broadsword consulting 的创始人，《伟大的大敏捷》的作者，也是 AgileCXO 的创始人。Jeff 与我们分享了他最近的冒险，即将 AgileCXO 建设成一个全球性的运动，以证明组织实际上是在用最佳实践实践敏捷。听一听。

**道尔顿:**当然。嗯，最大的原因是敏捷和自组织是不可避免的——它们正在到来——这与敏捷毫无关系。有趣的是，敏捷和开发运维是对全球趋势的反应。如果你看一看盖洛普每年进行的关于人们对领导力和机构的信心的研究，他们的研究中明确定义的是——这是一项全球性的研究——人们正在失去对传统领导力的信心。他们对哈佛和斯坦福每天都在培养的那种领导力失去了信心；他们对自我决定和自我组织更感兴趣。

这种情况正在银行、教育、软件开发领域发生——甚至军队也遭受了这种情况，或者说“痛苦”这个词是否恰当。各种各样的行业，普遍地，越来越多，越来越快地朝着自我决定和自我组织的方向发展。这是一个世界性的趋势。你会看到世界上有很多混乱，这才是真正的原因——这是关于传统领导规范的大规模剧变。

因此，不可避免的是，如果你想留住高质量的年轻员工，你想拥有一个长期成功的组织，你需要在你的组织中自上而下地采用敏捷文化。所以你必须完全重新设计你经营公司的方式，否则你就无法吸引有才华的员工。所以这是不可避免的。数据告诉我们，这种情况正在发生，所以这是我们关注这个问题的首要原因。

第二个原因是，领导者真的很难理解敏捷意味着什么，自组织意味着什么，以及如何实际实现它。他们受到了来自组织底层的很多阻力，要求他们变得更加敏捷，但是，如果你曾经在敏捷或 DevOps 团队中工作过，你就会知道，你越往上游，游泳就越困难。这是有原因的，我们称之为“组织类型不匹配”，在一个组织中，我有一个指挥和控制的基础架构，从首席执行官到基本的主管级别，然后所有这些人都说，“我想自我组织，更加敏捷。”

因此，这种文化冲突是非常依赖能源的——换句话说，在你的公司里消耗了大量的能源——以至于你的成本超过了你的利润。因此，这将是这样做的第二个原因，这是如何避免文化、组织类型不匹配的指南。

最后，我认为第三个原因是，它有助于员工更快乐，快乐的员工更有效率，快乐的员工会呆得更久，快乐的员工，团队成员，会更高兴在那里工作，产生更少的摩擦，更少的阻力。所以这对每个人都有好处。因此，所有的好理由去看看这本书，去看看 AgileCxO。如果你是一家咨询公司或公司，真的想帮助公司实现这三个目标，成为我们的合作伙伴，加入我们，加入我们的使命。我们很希望你能来。
