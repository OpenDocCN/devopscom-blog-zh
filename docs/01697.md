*   混合基础设施
*   DevOps 文化

## 网上研讨会后问答:

问:如果一个 IT 组织正在支持软件开发，但它并不敏捷，那么 DevOps 方法会有所不同吗？

*答:是的，正如我们之前提到的，您需要调整您的开发运维方法，以符合您的具体情况。一些组织使用看板代替 DevOps。然而，这里的关键是建立一个迭代的过程，并在增量的基础上驱动价值。–*RB

**问:我知道 web 服务、spa 和 API 可以很容易地用这种方式实现，但是你也用这种方式实现“遗留风格”的有状态 Web 应用吗？如果是这样，你如何处理部署 mid 进程炸毁用户的应用程序状态？**

*答:是的，当然，DevOps 可以用于“传统应用程序”。您将改变您的测试/发布过程，以确保您不会“破坏用户的应用程序状态”。开发运维是一种方法，而不是一个规定的过程。您将调整该过程以匹配您的特定情况，当然，这可以包括如何部署更多“胖”应用程序。*–RB
