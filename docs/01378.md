# VictorOps 企业版通过事件自动化引擎进行了扩展

> 原文：<https://devops.com/victorops-enterprise-edition-expanded-incident-automation-engine/>

*VictorOps 帮助最具创新精神的公司绕过传统的网络运营中心，使用端到端自动化主动接收、转换、路由和解决问题。*

科罗拉多州博尔德市–2016 年 6 月 22 日–[针对 DevOps 团队的实时事件管理公司 VictorOps](https://victorops.com/) 宣布推出事件自动化引擎，这是一套先进的功能，可供待命团队通过使用智能警报、复杂的路由和更智能的系统输出来自动实现更快的 MTTR。

与其他侧重于分类警报和通知的事件管理工具不同，VictorOps 事件自动化引擎提供了真正创新的组合，使系统输入智能化，相关输出完全可操作。该引擎自动执行人类行为，以消除不必要的警报噪音，提供实时补救解决方案，并提供事件期间发生的所有通信和操作的可见性和文档记录。此外，通过使用出站 webhooks，整个 VictorOps 解决方案现在可以配置为向其他工具发送信息，采取自动操作，并与其他警报源和已有的报告系统同步。这种组合在整个事件生命周期中提供了无与伦比的控制和洞察力。

VictorOps 首席执行官兼联合创始人 Todd Vernon 表示:“借助事件自动化引擎，我们的客户现在能够更好地解决甚至预防未来的事件，从而保护收入损失、提高客户满意度并维护品牌声誉。“越来越多的加速组织创新的最佳方式是让开发人员快速行动。当涉及到公司的正常运行时间时，同样的概念也适用。如今，陈旧的警报系统和手动流程浪费了宝贵的时间。该引擎现在成为自动化解决这些低效问题、更清晰地访问可操作信息以及持续评估和改进推动随叫随到流程的输入的基础。”

事件自动化引擎的主要功能包括:

*   **警报转换**–消除无法处理的警报，并自动将特定警报发送给正确的团队成员
*   **警报注释**–自动向警报添加运行手册、监控图表和其他相关信息，为警报接收者提供即时补救措施
*   **出站 web hooks**–将 VictorOps 平台数据扩展到其他系统中，并将 VictorOps 事件集成到其他服务仪表板中
*   **API**–利用 VictorOps API 将 VictorOps 扩展到现有的遗留工具和报告系统
*   **事后报告生成器**–能够自动获取与停机相关的所有活动(警报、对话和补救措施)的快照。这些信息很容易转化为 SLA 报告和团队回顾的便利。

各种规模的组织越来越希望采用事件管理解决方案，以减少 IT 停机的影响。根据 Gartner 的调查，组织的潜在停机成本可能接近每小时 54 万美元。如果缩短，平均问题解决时间(MTTR)可以在一年内避免 300 分钟的宕机，企业可以节省 250 万美元以上
