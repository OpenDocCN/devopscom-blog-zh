# 调查发现应用程序测试分布更加广泛

> 原文:[https://devo PS . com/survey-sees-app-testing-more-wide-distributed/](https://devops.com/survey-sees-app-testing-more-widely-distributed/)

管理测试工具的平台提供商 PractiTest 对 900 名应用程序测试专业人员进行的调查表明，拥有专门测试团队的组织数量正在稳步下降。

调查发现，拥有六个或更多成员的专门测试团队的组织减少了 15%。然而，与此同时，调查发现只有 18%的受访者报告说，所有测试都是由不属于专门测试团队的个人完成的。

PractiTest 的测试专家兼首席解决方案架构师 Joel Montvelisky 表示，这种转变表明更多的组织正在稳步地将[测试](https://devops.com/?s=continuous+testing)纳入 DevOps 工作流。事实上，超过 60%的受访者表示开发运维及敏捷流程使团队更有组织性和稳定性，三分之二(66%)的人表示开发团队更频繁地发布更多功能的特性。

然而，只有 38%的受访者表示他们是持续集成/持续交付(CI/CD)工作流的积极参与者。整整 90%的人说他们需要指导或培训团队中的其他成员，超过一半(54%)的人报告说对开发和测试所需时间的估计是准确的。

三分之二的人还表示，在应用 DevOps 流程时，与开发人员的协作也得到了改善，超过一半的人报告说，生产中出现的严重错误更少了。超过一半的人还说他们在生产和部署环境中监控软件。将自己定义为工程师的测试人员数量比去年增加了 14%。然而，只有 44%的人报告自动化已经取代了他们 50%或更多的手工测试工作。

多达 95%的受访者认为应用程序编程接口(API)测试、敏捷方法、功能测试和脚本是他们最优先考虑的事情。相比之下，只有 60%的受访者认为大数据测试、物联网(IoT)测试和机器学习/人工智能(AI)测试很重要或非常重要。最后，大约三分之一的调查受访者指出，非测试人员现在也参与到测试活动中。

Montvelisky 说，PractiTest 调查清楚地表明，虽然开发人员比几年前更多地参与了测试，但组织仍然依赖于越来越多地集成到 DevOps 工作流中的测试专家。这一点尤其重要，因为越来越多的组织启动了数字业务转型计划，要求他们使用独立软件供应商(ISV)采用的相同类型的 DevOps 流程来构建软件。

现在说测试会在多大程度上转向开发者还为时过早，但随着测试变得更加自动化——部分原因是对人工智能的依赖增加——由专门测试团队进行的测试类型将会发展。事实上，自动化不太可能在短期内消除对这些团队的需求。相反，测试人员将通过人工智能技术得到增强，这将帮助他们跟上应用程序开发的加速速度，这种加速速度是通过在大型和小型组织中更广泛地采用 DevOps 最佳实践来实现的。
