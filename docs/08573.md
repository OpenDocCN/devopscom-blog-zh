# BlazeMeter 收购后的性能图表测试过程

> 原文:[https://devo PS . com/perforce-charts-testing-course-after-blaze meter-acquisition/](https://devops.com/perforce-charts-testing-course-after-blazemeter-acquisition/)

Perforce Software 已经[从 Broadcom 收购了 BlazeMeter](https://www.prnewswire.com/news-releases/perforce-to-acquire-blazemeter-continuous-testing-platform-from-broadcom-301388818.html) ，作为其努力的一部分，以扩展其测试工具的范围，超越其在 2018 年收购的用于测试移动和网络应用程序的 Perfecto 工具。

博通在 2018 年收购 CA Technologies 后获得了对 BlazeMeter 的控制权。CA Technologies 在 2016 年收购了 BlazeMeter。涉及 BlazeMeter 的最新收购条款没有披露。

Perforce 的首席产品官蒂姆·拉塞尔(Tim Russell)表示，BlazeMeter 增加了一个完整的平台，涵盖了从负载和性能测试到应用程序编程接口(API)测试的所有内容。Russell 补充说，目标是使 DevOps 团队能够进一步向左推动持续测试，作为加速应用程序开发和部署的努力的一部分。

Russell 指出，随着越来越多的组织启动多项数字业务转型计划，对持续测试的需求变得更加明显。Russell 补充说，真正的挑战将是找到以足够大的规模将持续测试嵌入 DevOps 流程的方法。

事实上，这些应用程序提供的数字体验的质量最终决定了这些努力可能成功的程度，Russell 指出这就是为什么如此多的企业 IT 组织正在重新设计他们的应用程序测试流程。

![](../Images/1eea4f02ce75661ea252dd36687ae9bc.png)

总的来说，组织正在采取两种方式将测试进一步左移。一些人向开发人员提供应用程序性能测试工具，以便在 DevOps 冲刺阶段使用，而另一些人则试图让应用程序测试人员更早地进入应用程序开发过程。在某些情况下，组织将这两者作为在生产环境中部署应用程序之前发现问题的全面努力的一部分。在部署应用程序之后，修复这些问题成本自然会更高。

Russell 说，Perforce 目前希望继续将 BlazeMeter 和 Perfecto 作为一套互补的基于云的服务，随着时间的推移，这些服务的功能会相互融合。

从长远来看，很明显，更多基于人工智能(AI)的分析将嵌入基于云的平台，这些平台可以收集足够的数据来建立有意义的开发流程模型。IT 团队也应该期待看到测试工具更多地使用计算机视觉算法来解决问题。

随着越来越多的组织同时构建多个应用程序，整个应用程序测试过程将变得更加自动化。这不一定会减少开发人员或测试人员创建和管理该过程的需求，但这确实意味着每个应用程序所需的测试时间在未来的几个月或几年中应该会持续下降。

希望所有这些努力能为最终用户带来更好的应用体验。对于任何应用程序开发团队来说，最令人沮丧的事情莫过于发现一个应用程序 90%的功能都没有被使用，因为最终用户无法轻松地使用它。当然，更糟糕的是，当一个应用程序被部署后，它变得太慢了。在这两种情况下，最终用户对任何新的应用程序体验的最初热情都很难恢复，原因很简单，因为没有人考虑过应该在应用程序开发生命周期的更早阶段测试的所有用例。
