# 你的未来在 SaaS 吗？是的，除了…

> 原文:[https://devops.com/is-your-future-in-saas-yes-except/](https://devops.com/is-your-future-in-saas-yes-except/)

SaaS(包括 IaaS)的发展是惊人的，它已经变得有些无处不在。今天，几乎每个企业，无论大小，都在使用 SaaS 工具。从销售到市场营销到客户管理，有一个 SaaS 工具可以负担得起而且做得很好。暂时如此。云厂商也是如此。云价格通过两种机制保持低价——数量和利润。体积；就像公共交通更便宜的理论一样，通过将许多客户放在一台服务器上降低了云供应商的运营费用，从而降低了云供应商为盈利而必须收取的费用。

但是，随着当前环境中成本的上升和增长的放缓，作为客户，您的 it 部门关注所有 SaaS 产品的成本/收益等式是非常重要的。有些人仍然受益，因为他们卸载的工作，即使价格上涨为您的组织。你会看到，随着成本上升，其他人将很快超越他们带来的价值，还有第三类人不值得这样做——也许从来就不值得，你也没有注意到这一点。

[寻找那些物品](https://www.forbes.com/sites/peterbendorsamuel/2021/08/10/why-is-cloud-migration-reversing-from-public-to-on-premises-private-clouds/)。为了让这一点不那么模糊，这里有一个例子:我参与的一家公司一直在为一项 SaaS 服务付费，该服务只是模拟现场功能。该组织可以(以前也这样做过)放几个盒子进去，获得大致相同的结果。最近，我们更仔细地研究了类似的项目，发现在这种特殊情况下(虽然该组织在 SaaS 使用的大多数情况下不是这样)，投入两个盒子并让他们完成工作的成本(假设这些盒子每隔一年升级/更换一次)比以当前的月价格继续使用 SaaS 解决方案更便宜。事实上，这要便宜得多，以至于可以雇佣一个承包商来处理物理机箱出现的任何问题，唯一增加的工作就是维护更新以保持系统安全。如果解决方案不是由设备组成，而是由两个 Windows 或 Linux 机器组成，那么维护成本也会降到最低。

服务将保持不变或更好——如果需要改变，设备就在那里——并且攻击者的工具/供应商链中将少一个环节(物理解决方案将在私有子网中，SaaS 不能)。假设解决方案是一个高质量的产品，这里真的没有负面影响。然而，几年来，这家公司一直在使用 SaaS 的解决方案。这些都是唾手可得的果实，人们会指着它们称之为“紧缩”或其他什么。别理他们。做对你的组织最有利的事情——无论是在 SaaS 还是在当地——并根据每个工具/每个项目做出决定。

继续踢它。您正在构建让公司持续运营的应用和基础设施。不要犹豫，为你的团队所做的一切感到骄傲，不要犹豫，在无关紧要的事情上削减成本，这样你就有更多的资金去做重要的事情。再次感谢你——来自你组织中所有没有意识到他们应该说谢谢的员工。
