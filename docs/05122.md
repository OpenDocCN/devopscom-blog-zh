# Jenkins X 继续经历着强劲的需求，加速了云原生应用的开发

> 原文:[https://devo PS . com/Jenkins-x-continues-to-experience-strong-demand-speeding-development-of-cloud-native-applications/](https://devops.com/jenkins-x-continues-to-experience-strong-demand-speeding-development-of-cloud-native-applications/)

*随着开源项目的下载量、贡献者和公司支持的增长，CloudBees 为 Jenkins X 推出了第一个 GUI*

加利福尼亚，圣地亚哥。和加州圣何塞。–2019 年 11 月 20 日—推动持续经济发展的企业开发运维领导者 CloudBees 今天在 KubeCon San Diego 为 Jenkins X 推出了一款图形用户界面(GUI)。CloudBees 的宣布正值 Jenkins X 开源项目和 CloudBees 对该项目的支持都在快速增长，贡献者和支持公司都创下了历史新高。

在不到两年的时间里，大约有 885 名开发人员为 Jenkins X 项目贡献了代码，有 176 家贡献公司(来源:持续交付基金会 DevStats，2019 年 10 月)。

Jenkins X 为 Kubernetes 上的云原生应用提供自动化 CI/CD。使用 Jenkins X，开发人员可以利用 Kubernetes 快速轻松地构建和部署云原生应用程序，而不必了解 Kubernetes 及其生态系统的复杂性。由 CloudBees 和其他贡献者于 2018 年 2 月发起的 Jenkins X 项目现在是持续交付基金会的一部分。

一家工作流和决策自动化平台公司的首席软件开发人员、Learnk8s 的讲师 Mauricio Salatino 表示:“作为一名负责将 monoliths 迁移到部署到 Kubernetes 的微服务的软件开发人员，Jenkins X 帮助我和我的团队高效地管理了需要将 100 多个源代码库部署到不同 Kubernetes 集群的复杂项目。“对于一个只有四个人的团队来说，这是一个最初看起来不可能完成的任务，但我们用 Jenkins X 轻松完成了。”

詹金斯 X 继续快速增强。CloudBees 在其 CloudBees Jenkins X 发行版中引入了新的 GUI，为开发人员提供了更大的灵活性和更丰富的用户体验。许多开发人员喜爱的现有命令行界面(CLI)仍然可用，并将继续得到增强，但现在用户可以选择使用 CLI 或新的图形环境，这取决于哪个更适合他们的工作流。CloudBees Jenkins X 发行版是 CloudBees 免费提供的。

CloudBees 的云副总裁 Mo Plassnig 表示:“我们的目标是让 Jenkins X 成为希望以自以为是的自动化方式使用 Kubernetes 的开发人员最无摩擦的环境。“提供更丰富、更直观的 GUI 只是一个开始。整个 CloudBees 和持续交付基金会生态系统的贡献者都在努力创造更多的改进——从日志处理到环境提升再到工作流的加入。”

“这些数字表明，我们对 Jenkins X 的愿景是正确的，”CloudBees 的杰出工程师、Jenkins X 的创始人之一詹姆斯·斯特拉坎(James Strachan)说，“随着越来越多的组织采用 DevOps，对利用 Kubernetes 的强大功能和 Jenkins X 的丰富功能和直接便捷性的 CI/CD 解决方案的需求也在持续增长。”
