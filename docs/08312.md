# 你应该雇佣一名 Clojure 工程师吗？

> 原文:[https://devops.com/should-you-hire-a-clojure-engineer/](https://devops.com/should-you-hire-a-clojure-engineer/)

开发应用程序时，经常需要在执行其他任务的同时执行长时间运行的代码。最近，为了在 Java 虚拟机( [JVM](https://devops.com/?s=JVM) )上运行程序，有必要用 Java 编写程序。由于 JVM 有各种各样的编程语言，那些日子已经一去不复返了。Clojure 是 Rich Hickey 为 JVM 开发的通用动态语言。使用这种语言的工程师的主要目标之一是简化用同时访问数据的多线程编写程序的过程。确切地说， [Clojure](https://mobilunity.com/blog/hire-clojure-developers/) 软件工程师的角色是什么，他们如何改进其产品的功能？

## Clojure 软件工程师可以修复哪些问题？

一门好的编程语言应该帮助我们编写透明和简单的程序。人类创造的软件系统本质上是可变的。在开发中，不可能预先预见所有的情况，尤其是当系统的复杂性随着时间的推移而增长时。即使是最轻微的代码更改也可能导致整个系统行为的重大变化。作为一名开发人员，有必要仔细研究、分析代码并预测每个转变的副作用。随着时间的推移，随着复杂性的增加，彻底分析我们变化的所有可能结果变得几乎不可能。

Clojure 语言就是为了克服这些挑战而创建的。Clojure 有一个更方便、更现代的语法，尽可能地将用户导向函数范式。尽管语言是动态类型的，但 Clojure 程序被编译成 JVM 字节码，并与现有的 Java 程序无缝集成。Clojure 现在已经在几个平台上实现，其中最著名的是 JVM。Net 和 JavaScript。因为它运行在 JVM 上，Clojure 代码是可移植的、稳定的、可靠的和安全的。Clojure 中的每个操作都以三种形式实现:函数、宏或特殊形式。

通常，计算机程序的复杂性会产生副作用。这种语言有助于将它们本地化，并使书写功能更加简洁。这也使得开发人员的测试更加容易，因为他们不必尝试不同的队列并找到输入的正确状态。

## 招聘时要注意什么

大公司尤其看重高级全栈 Clojure 软件工程师，他们可以完成从分析到编码、测试和开发的多项任务。

Clojure 软件工程师可以为企业解决的问题包括:

*   前端和后端系统的开发
*   根据能力/用户工作量创建负载分配算法
*   设计用于预测用户对系统造成的负载的模块
*   Clojure 中的机器学习。

Clojure 也表达了开发人员关于程序员的工具应该有多简单的想法。Clojure 通常只使用四种数据结构:list、vector、display 和 set。主要的 API 是围绕这四个结构构建的。Clojure 中的主要抽象是一个没有副作用的纯函数，因此可重用性更高。

迄今为止，已经为 Clojure 编写了许多库，包括那些用于开发简单和模块化 web 应用程序的库。Clojure 程序是一个开放的架构构造器，这意味着开发人员可以自己选择使用哪些库，而不用考虑他们之前可能已经选择了哪些库。

## 对企业的价值

企业软件性能需求决定了需要使用并行计算来相互交互并同时工作。现有的语言不太适合并行化任务。它们包含许多对象和状态依赖，这转化为并行交互中具有不可预测行为的系统。基于锁和交通灯的解决方案并没有大大简化这些任务的实现；系统状态依赖于对象交互序列的基本问题仍然存在。

通过在 web 开发中使用 Clojure，程序员可以使用其他次要的、面向任务的语言进行开发。例如，ClojureScript 可以快速编译成 JavaScript 并在浏览器中运行。所有这些为构建客户端应用程序提供了一种全新的方法。

由于 Clojure 可以与任何 Java 代码无缝集成，并且可以部署在标准的 Java 应用服务器上，因此它也非常适合 web 应用程序开发。这种语言的灵活性及其令人难以置信的表达能力允许开发人员非常快速地创建全功能的 web 应用程序。

开发人员使用 Clojure 所需要的只是包含在一个 JAR 文件中的 JDK 和 Clojure 库。开发和运行 Clojure 程序有两种标准方法:最常见的方法是使用简单的交互式 REPL，即读取-计算-打印循环。

由于 Clojure 是一种通用语言，程序员不难找到它的用途。因为它是基于 JVM 的，所以已经有大量的库可供选择，这大大降低了精通其他语言的开发人员的入门门槛，并消除了开发人员从头编写代码的需要。

Clojure 可以用在任何需要现代、高效、稳定的数据结构的地方。当然，没有一种通用的工具可以处理所有类型的开发任务。然而，Clojure 开发人员可以使用该语言进行客户机-服务器通信的事实是另一个优势。
