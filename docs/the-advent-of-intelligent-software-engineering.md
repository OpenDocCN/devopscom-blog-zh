# 智能软件工程的出现

> 原文：<https://devops.com/the-advent-of-intelligent-software-engineering/>

今天，为了保持竞争力，每个公司都需要像软件公司一样行事，并以高速度、高质量和高可靠性提供新的解决方案。新功能需要快速推出，但又有高质量的期望。与此同时，部署版本和操作变得越来越复杂，生产中的调试和可追溯性越来越困难，安全性更加突出，正常运行时间和可靠性需求急剧增加……不胜枚举。

简而言之，传统的软件工程正在彻底改变。  导致这种混乱的一些趋势包括:

*   **云正在吃软件。** 这不仅改变了托管软件的基础设施，也改变了应用程序的架构、开发、打包和部署方式。云正在推动云原生技术的采用；无服务器组件和容器化改变了我们构建应用程序的方式。开发团队和基础设施团队之间的界限越来越模糊。
*   网络安全现在是董事会关心的问题。 不断上升的网络攻击和恶意软件威胁让每个人都变得脆弱。传统上，软件每隔几个月发布一次，留给专家足够的时间进行安全测试。今天，随着非常快速的发布周期的到来，安全检查需要在开发过程的早期开始。
*   **AI/ML 是新电但正在引起震荡。** AI/ML 驱动的用例正在跨平台、跨行业寻找出路。传统的软件工程师不熟悉 AI/ML，ML 开发人员不熟悉工程学科和最佳实践，在工程成熟度方面留下了很大的鸿沟。
*   当一家公司提高标准时，其他公司也会跟着提高。 消费者的期望是不断变化的。他们期望从亚马逊、优步和谷歌的每个应用程序中获得同样的体验。人们期望你提供一个无缝的、无摩擦的体验，你的平台在任何时间、任何设备上都是可用的，并且是快速和安全的。
*   **API 正在推动创新:** 速度和规模的创新并不容易，但 API 可以通过利用人群的集体力量来实现这一点。今天，任何开发人员都可以使用平台提供的 API 来构建一个新的、创新的解决方案。API 优先的思维模式正在形成，应用商店模式(例如，Apple App Store、Salesforce AppExchange、AWS Marketplace)已经改变了我们对创新和创建新平台扩展的看法。
*   **低代码/无代码** 平台正在快速发展，无需编写任何代码就能轻松开发小型情景应用。公民开发者正在出现。
*   现代建筑范例推动新的工作方式。 现代平台架构在最近几年有了显著的发展，从单体应用到 SOA，再到基于微服务和 API 的分布式平台。事件驱动的架构越来越突出。流式架构越来越受欢迎，以支持巨大的可扩展性。  这对[工程团队如何组织](https://devops.com/why-you-should-rip-up-your-org-chart-and-reorganize-around-value-streams/)以及他们如何构建、测试、部署和管理新平台有着重大影响。
*   **传统平台增长放缓。**20 多年前构建的遗留应用程序需要根据今天的环境进行现代化改造。现代化是一项巨大的工作，取决于应用的年龄、技术和规模。但是现代化说起来容易做起来难。传统和新世界需要共存一段时间，从技术和文化的角度来看都需要一个桥梁战略。

为了满足这些不断变化的需求，工程需要变得更加智能。

## **智能工程的到来**

我们可以通过使用更多数据来管理项目，利用 AI/ML 分析来提高工程生产率和可预测性，利用更多自动化，并将正确的流程应用于正确的问题，从而为工程带来智能。

### 工程中的数据驱动决策

工程团队在开发环境中使用许多工具来进行代码存储库、错误跟踪、代码扫描、构建、测试、部署等。流行的工具包括 Azure DevOps、GitHub、Atlassian 和 Jenkins 等。这些工具会生成大量粒度数据。所有这些数据孤岛都可以聚合起来，并用于跟踪以下方面的有趣指标:

*   **开发者生产力:**相对于其他活动，花多少时间写多少代码；交付的故事点与提交的故事点
*   **软件质量:**产生的 bug 数量；修复错误的速度；进入生产的 bug 数量

此外，在这些数据的基础上，人们可以使用机器学习建立一个智能层，以得出隐藏的见解，突出相互关系，并进行更好的根本原因分析。这些见解可以为工程结果带来更大的可预测性。例如，开发人员是否能满足发布时间表、质量水平等。

数据提供了团队表现的证据。它有助于消除潜在的障碍，并与所有利益相关者进行更好的沟通。它可用于更好地管理项目绩效，并用于客观决策。  尽管有了这些数据，工程领导们仍在黑暗中操作。决策是基于直觉和直觉，而不是硬数据。 是时候通过在项目绩效跟踪和决策中使用数据变得更加智能了。

### 利用人工智能提高工程服务的生产率和可预测性

AI/ML 正在颠覆每一个行业，现在正在软件工程中寻找出路。支持 AI/ML 的开发工具正在出现，它们可以在整个软件工程生命周期中得到利用。这些工具不仅可以节省重复活动的时间，还可以提高大型复杂任务的软件质量。

支持 AI/ML 的工具可以改善开发人员从编码到测试的日常生活。今天，人工智能工具可以扫描和分析代码，以提供智能代码完成建议，标记任何偏离编码最佳实践的情况(命名约定合规性、变量误用等)。)，执行同行评审，将代码从一种语言转换到另一种语言，查找安全漏洞等。动态应用安全测试(DAST)解决方案使用人工智能在几毫秒内发现潜在的攻击媒介，否则需要几天时间。 AI/ML 可以帮助从手绘的 UI 草图自动生成 HTML 代码。AI/ML 还可以帮助开发人员从现有代码中自动生成单元测试，并提供关于改进测试的建议。

事实上，测试是可以从基于 AI/ML 的智能中受益的最大类别，特别是在大型复杂的应用程序中。依赖手工测试的 QA 团队无法跟上当今代码变更和发布的速度。支持 AI/ML 的工具可以在许多特定的方面帮助测试团队。

*   **变更影响分析:** AI/ML 可以帮助进行覆盖率分析，并根据应用程序中的变更确定需要运行哪些测试。
*   **测试创建:** AI/ML 可以帮助测试团队从简单的英语描述中创建测试用例，也可以学习如何随着时间的推移改进测试；AI/M 还可以自动修复被破坏的测试。 AI 驱动的工具可以自动将手动 UI 测试转换为 API 测试。
*   **视觉测试:** AI 驱动的图像比较技术可以使视觉测试能够分析跨测试检测到的 UI 屏幕差异。
*   **测试分析:** AI 可以分析测试用例以及缺陷度量来增加测试覆盖率，同时减少测试数量。

是时候通过使用人工智能工具来增强工程团队，提高他们的生产力并带来可预测性，从而变得更加智能了。

### 智能自动化:超越开发过程

今天，在软件开发生命周期中有很多自动化选项。DevOps 实现了软件新版本的构建、测试和部署的自动化。您还可以自动化其他活动，例如测试自动化、代码扫描、性能测试等。DevSecOps 支持将安全流程(例如，漏洞代码扫描)自动化到工程生命周期中。AI/ML 驱动的工具可以进一步推动自动化改进(例如，改进 CI/CD 管道中的流程。)总的来说，自动化支持更高的团队生产率，并导致业务敏捷性所需的快速或按需发布周期。

自动化不仅限于开发活动，它也适用于工程生命周期的其他部分，例如:

*   基础架构即代码(IaC)工具可以自动调配和配置云基础架构环境(存储、网络等)。).
*   监控工具可以自动监控可用性、生产负载指标和安全问题，并生成警报。
*   日志管理:当今生态系统中生成的日志数量巨大，手动收集它们是不切实际的。日志管理工具可以自动汇总和分析日志信息。

积极地利用自动化，并在整个工程生命周期中使用它，可以使软件工程更加智能。您应该将重复性任务和低价值流程自动化，并将节省下来的时间用于更高价值的活动。

### 让敏捷更聪明

有几种流行的软件工程方法，如敏捷、扩展敏捷、Spotify 模型等等。  不管你使用的是什么方法或流程，智能地将正确的方法和工具应用到正确的问题上可以使它们更加有效、高效和智能。例如，传统的敏捷流程最初是在制造领域设计的，然后被用于开发成熟的软件。但是在当今快速变化的市场中，开发团队需要快速并持续地尝试新的想法，以展示新的业务特性或对新兴技术进行技术验证。不仅在新产品开发的早期阶段如此，在现有平台上也是如此，在现有平台上，您需要跟上不断变化的市场动态。做好实验的准备。

我们需要更明智地选择正确的流程并根据您的需求进行定制，而不是让流程决定您的结果。

### 如果软件正在吞噬世界，那么复杂性正在吞噬软件

虽然产品工程已经存在很长时间了，但是现代软件工程比仅仅编写代码和遵循敏捷过程要复杂得多。智能工程实践能够以更好的质量和可靠性更快地将解决方案推向市场，为创新赢得更多时间，以跟上当今的业务发展速度。
