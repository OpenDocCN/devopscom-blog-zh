# 调查显示转向 SRE 原则以自动化 IT 管理

> 原文:[https://devo PS . com/survey-shows-shift-to-sre-principles-to-automated-it-management/](https://devops.com/survey-shows-shift-to-sre-principles-to-automate-it-management/)

F5 开展的一项针对 1，500 名 IT 决策者的全球[调查](https://www.f5.com/company/news/press-releases/f5-research-tradeoffs-of-accelerating-digital-transformation)发现，平均而言，组织管理着部署在扩展企业中的 200 多种应用以及 20 多种不同的安全和交付技术。

F5 首席技术官办公室的首席技术宣传员 Lori MacVittie 表示，这种复杂程度表明，it 团队显然需要采用更高水平的自动化来管理 IT 环境，这些环境现在是一系列数字化业务转型计划的核心。调查显示，超过四分之三的受访者(77%)接受站点可靠性工程(SRE)原则来自动化 IT 管理。

整整 95%的受访者还计划挖掘运营数据，以获得可用于改善客户体验和推动业务增长的见解。然而，98%的受访者也表示，他们目前无法从现有系统中提取所需的见解。共有 90%的人表示，他们计划实施某种形式的人工智能(AI)来实现这一目标。超过三分之二的受访者表示，他们的组织意识到为客户创建卓越的数字交互将需要现代化的业务流程和后台职能。

作为这些努力的一部分，调查表明应用程序工作负载正在高度分布式计算环境中转移。除了 84%的受访者计划将工作负载迁移到边缘之外，调查发现，三分之二的组织(67%)目前正在或计划在未来 12 个月内将应用程序从云迁移到数据中心环境。目前，共有 93%的受访者表示使用通过云交付的某种类型的“即服务”产品。

当然，保护这么多平台也是一项重大挑战。然而，尽管存在固有风险，超过四分之三(76%)的受访者表示，如果可以选择，他们会关闭安全措施来提高性能。MacVittie 说，缺乏牺牲性能的意愿表明，尽管现在经常发生违规事件，但人们对安全的态度可能没有太大变化。

不清楚管理企业 It 环境已经变得不可持续到什么程度；然而，很明显，传统的手动流程需要实现自动化。组织无法继续雇佣 IT 人员来管理从网络边缘到公共云的各种 IT 平台。事实上，调查清楚地表明，DevOps 实践——以谷歌首先定义的 [SRE](https://devops.com/?s=SRE) 原则的形式——正在获得关注。这并不一定意味着管理它的每个人都成为了 SRE 或工程师，但这确实表明自动化框架正在被更广泛地使用。

这些自动化框架的普及可能还需要一段时间。然而，不管怎样，许多历史上使 IT 工作变得乏味的任务将会消失。当然，问题是不要以引入更多复杂性的互不相连的自动化孤岛而告终。
