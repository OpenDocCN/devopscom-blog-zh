# Vercel 收购 Turborepo 以获得建筑系统

> 原文:[https://devo PS . com/ver cel-acquisites-turbo repo-to-gain-build-system/](https://devops.com/vercel-acquires-turborepo-to-gain-build-system/)

Vercel 今天宣布已经收购了 turbo repo T1，这是一家为 T2 JavaScript 和 TypeScript T3 应用程序提供构建系统的供应商，它为开发者提供了一个简单易用的代码库。

在那次收购之后，用于访问 Turborepo 的命令行界面(CLI)现在将在开源许可下提供。

作为 Turborepo 和包括 Formik 和 TSDX 在内的其他流行开源项目的创造者，贾里德·帕尔默将作为 Vercel 构建性能团队的新领导，继续致力于加速 Turborepo 的能力。

目标是将 Turborepo 云服务的现有用户迁移到 Vercel 平台，以便使用 Next.js 框架构建应用程序，这使得基于 React 库构建 web 应用程序成为可能，从而创建支持静态站点生成(SSG)和服务器端呈现(SSR)的用户界面。

Palmer 说，像 Turborepo 这样的 monrepositories 不仅使开发人员更容易管理构建，而且还减少了使用持续集成/持续交付( [CI/CD](https://devops.com/?s=CI/CD) )平台处理这些构建所需的时间。

![](../Images/ec9773022031f6ff969f305f65be8f50.png)

竞争对手的构建系统在配置和管理上也要复杂得多，Palmer 说这是许多开发者尚未建立自己的 mononrepositories 的主要原因。

随着组织希望在数字业务转型时代构建和部署的应用程序数量的增加，对开发人员工作效率的关注也在加强。Turborepo 的设计旨在简化配置，而不是维护复杂的构建系统。结果，更多的开发人员可以减少多达 50%的构建时间，帕尔默指出。

他补充说，实际上，通过将该构建系统与 Next.js 框架相结合，前端开发人员能够更容易地成为全栈开发人员。

DevOps 团队将需要适应能够提供更丰富的应用程序体验而不太依赖后端基础架构的应用程序。这种转变对从网络带宽消耗到网络应用性能的方方面面都有影响。Vercel 平台采用缓存、路由和 React 框架来优化应用程序性能，从而减少对后端基础架构的依赖来优化应用程序性能。

随着 web 应用程序开发的不断发展，构建 web 应用程序的整体速度应该会显著提高。当然，DevOps 团队需要考虑更多应用程序通过 DevOps 管道的影响，以及在部署后持续更新这些应用程序的需要。

目前尚不清楚 DevOps 团队是否会或在多大程度上推动开发人员转向 Next.js 等框架，以开发可能不会像前几代应用程序那样对后端基础设施造成太大负担的 web 应用程序。实际上，开发人员可以利用框架，只要有可能，他们就可以使用具有熟悉的 JavaScript 结构的工具，将底层 IT 基础设施视为无服务器的。Vercel 声称，在 Airbnb、Hulu、Nike、Ticketmaster 和优步等组织中，已经有超过 3 万个网站在运行 Next.js。

不管怎样，很明显，构建 web 应用程序的方式正在发生根本性的变化。现在唯一的问题是确定 IT 组织的其余部分准备在多大程度上吸收这种级别的变化。
