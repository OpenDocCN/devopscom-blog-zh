# 新遗迹调查显示对可观测性的更高评价

> 原文：<https://devops.com/new-relic-survey-shows-greater-appreciation-for-observability/>

一项针对 1，300 名软件工程师、开发人员、IT 领导者和高管的全球调查发现，一半的受访者(50%)正在实施可观察性平台。另外 23%的受访者表示，他们根本无法获得端到端的可观察性，而 26%的受访者表示，他们认为他们已经有了成熟的可观察性实践。今天公布的调查是由 CITE Research 代表 New Relic 进行的。

大多数(94%)IT 专业人员表示[可观察性](https://devops.com/?s=observability)对他们的工作角色很重要，而近四分之三(74%)的人表示他们的可观察性实践仍有发展空间。事实上，参与调查的 450 名高管中有 81%表示，他们预计明年将增加可观察性预算，20%预计这些预算将大幅增加。

总体而言，调查发现，缺乏资源(38%)、技能(29%)、对利益的理解(27%)和战略(26%)是实现可观察性的最大障碍。

这种或那种形式的可观察性一直是 DevOps 最佳实践的核心原则。大多数 DevOps 团队依赖于监控工具来观察特定的过程或功能。挑战在于这些监控工具跟踪一组预定义的指标，如资源利用率。理想情况下，可观察性将指标、日志和跟踪(一种专门的日志记录形式)结合起来，以某种方式检测应用程序，从而简化问题的故障诊断，而不必仅仅依靠一组有限的预定义指标。

近四分之三的受访者(72%)指出，如今的 IT 团队需要在至少两种不同的工具之间切换来管理 IT，13%的受访者不得不使用十种以上不同的工具来监控其系统的运行状况。

相比之下，可观察性平台使得跨应用和基础设施关联事件成为可能。这一功能使分析工具更容易识别指示 it 问题根本原因的异常行为。有了这些见解，it 团队更快地解决问题就变得简单多了。

新遗迹调查指出，60%的受访者仍然在应用层面监控遥测数据，这表明大量有价值的遥测数据在很大程度上未被使用。参与调查的 91%的 it 专业人员指出，可观察性在软件生命周期的每个阶段都至关重要，尤其是在规划和 IT 运营方面。

列举的主要优势包括更快的部署(27%)、更具成本效益的 IT 资源使用(25%)、更好的数字体验(23%)和更高的工作效率(22%)。引用的主要使用案例涉及数字化转型计划(42%)，其次是云优化(37%)、开发运维流程(36%)和加快产品上市速度(36%)。

New Relic 的集团副总裁兼总经理 Buddy Brewer 表示，由于 Kubernetes 上运行的基于微服务的应用程序的兴起，可观测性将成为 IT 环境中一个更紧迫的问题。接受调查的 88%的 IT 决策者 New Relic 表示，他们目前正在某种程度上探索 Kubernetes 和容器，但只有 10%的人表示他们已经在生产环境中部署了它们。总共有 40%的汽车预计将在三年内投产。

总的来说，it 团队能够实时查询数据以解决问题变得至关重要，这些问题在组织前所未有地依赖 IT 的时候变得越来越难以解决。布鲁尔指出，希望通过对数据进行采样来发现问题已经不够了。

当然，挑战在于大多数 IT 团队已经使用遗留监控工具很长时间了，很难想象还有更好的方法。这并不一定意味着所有这些遗留工具将被立即取代，但它确实意味着它们中的许多可能会随着时间的推移而逐渐消失。

不管怎样，大多数 IT 团队将很快能够更深入地观察 IT 环境。开发人员越来越多地利用像 [OpenTelemetry](https://devops.com/cncf-advances-opentelemetry-initiative/) 这样的开源工具，以组织更容易负担的方式来装备应用程序，而不管他们是否完全理解现代可观测性与传统监控的细微差别。
