# 云架构师成功所需的 8 项技能

> 原文:[https://devo PS . com/8-skills-cloud-architects-need-to-success/](https://devops.com/8-skills-cloud-architects-need-to-succeed/)

对于云架构师来说，深入了解云工具和技术只是需要掌握的技能的起点。作为云架构师，享受成功的职业生涯需要更多的东西，包括某些通常与技术行业无关的技能。

## 建筑和设计

云架构师需要是端到端系统设计方面的专家，因为他们的主要职责是通过使用技术解决客户的问题。架构师开发解决业务问题的设计的能力是他们真正的价值所在。因为设计就是架构，[云架构师](https://devops.com/?s=cloud+architect)必须具备网络和数据中心中现有的所有现有和遗留系统的专业知识，还必须了解如何将它们迁移到云，并将它们与适当的基于云的系统相匹配。

## 网络和数据中心技术

网络和数据中心技能对于云架构师来说至关重要，因为他们的主要职责之一是设计一个计划，将网络上和数据中心托管的系统迁移到云中。要成功做到这一点，云架构师必须是网络和数据中心技术方面的专家。

深入挖掘，他们必须是路由器、负载平衡器、交换机、路由协议、IP 寻址和 WAN 连接方面的专家，无论这些是基于专线、基于 VPN 还是软件定义的网络。他们必须是服务器和服务器虚拟化以及容器和存储方面的专家:块存储、对象存储和文件存储。云架构师还需要精通文件共享协议，例如 Windows 的服务器消息块和 Linux 的 NFS。安全性也是一项重要的技能；他们必须了解防火墙、IDS/IPS 系统以及 VPN 集中器的工作原理。云架构师需要全面了解数据库，了解 web 应用程序的工作方式，并了解 ERP 应用程序和供应链应用程序。良好、扎实的网络和数据中心知识是监督技术从数据中心向云迁移的架构师的基础。

## 商业头脑

云架构师是数字化转型专家。他们必须为客户精心设计端到端的业务解决方案，目标是改善他们的业务。如果他们对客户的业务没有透彻的了解，他们就不能有效地设计解决方案来解决他们的业务问题。

业务技能包括根据机会的预期价值计算 ROI 模型的能力，阅读财务报表和资产负债表的能力。它包括了解和理解首席执行官、首席财务官、首席信息官和首席技术官所关心的事情，并能够以他们理解的方式与他们沟通。如果不了解业务，云架构师会将自己局限于简单地添加或删除技术，这可能会使业务成果更好，也可能会使业务成果更差。

## 领导力

完成任何伟大的事情都需要团队，这意味着成为一名伟大的云架构师需要领导技能。为客户工作时，云架构师通常需要设定当前系统的基线。有效地实现这一点可能需要一个由 50 名云工程师组成的团队共同评估客户的系统。领导技能可以让你有效地管理工程师团队。

领导技能不仅仅是在人们进行概念验证和部署时有效地管理他们。领导技能使架构师能够确保团队受到激励，充分参与项目，并为成功做出贡献。

## 销售

在职时，云架构师会不断地进行销售。他们设计的解决方案需要卖给客户。在获得必要的资源之前，需要向管理层推销实施解决方案的计划。当需要让人们和我们一起做一个项目时，我们需要向他们推销它的价值。销售技巧有多种应用方式。云架构师向客户、内部利益相关者和管理层销售。

## 介绍会；展示会

云架构师还需要出色的演示技能，因为他们将向客户进行演示。这些客户可能是技术专业人士，比如工程师；他们可以向管理层和/或首席执行官进行演示。此外，云架构师将在管理层简报会和会议上展示信息。一般来说，演示是普通云架构师日常生活的一部分。

## 沟通

确定任何云架构项目的业务需求都需要问很多问题。他们绝不会只是从一个客户手里接过一张纸，说，“造这个。”没有出色的沟通技巧，很难(如果不是不可能的话)确定业务需求，这意味着设计的解决方案可能不正确。

即使已经很好地理解了业务需求，下一步也是将这些需求带给技术专家团队，这意味着与技术人员和业务人员进行有效的沟通。沟通技巧非常重要，包括与客户、团队成员和组织中其他各种利益相关者的沟通。

## 情商

对于那些希望提高自己的职业价值并推动自己的云架构师职业发展的人来说，情商是一项关键技能。情商提高领导能力、理解能力和沟通能力。当在工作中经常使用时，它是职业成功的最佳预测，通常会导致平均个人工资增加约 3 万美元。对于一个想要事业成功的云架构师来说，构建上述技能至关重要。
