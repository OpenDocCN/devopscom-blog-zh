# 如何避免蹩脚的文化，让工程师开心

> 原文：<https://devops.com/how-to-avoid-crappy-culture-and-keep-engineers-happy/>

软件工程师知道他们值多少钱——每个人都想要他们，但没有足够的人来满足需求。虽然对技术人才的需求比以往任何时候都大，但疫情已经转移了许多工程师的优先考虑事项。他们并不孤单。这是世界各地各种领域数百万人共有的现实。

如果所谓的[伟大的辞职](https://devops.com/?s=Great+Resignation)教会了我们什么，那就是每个人都厌倦了蹩脚的职场文化。人们开始质疑为什么他们会呆在一个他们感觉不到重视和支持的地方。对于那些没有在他们的文化上投入时间和精力的公司，祝他们能留住软件工程师。他们知道自己的价值！

在这个竞争异常激烈的环境中，让软件工程师开心并让公司成为理想的加盟地点的最佳方式可能看起来违反直觉，但请听我说:退后。相信工程师会做好他们的工作。

为一个支持性的环境奠定基础，在这样的环境中，人们愿意冒险并以积极的方式表达自己的想法，即使是矛盾的想法。然后，让开。领导者有责任坚持这些价值观，雇佣符合这些价值观的人，即使这意味着不总是挑选最合格的人。

## 不要害怕“愚蠢的想法”

我会第一个说出来——我有很多愚蠢的想法。我很感激曾在鼓励我分享这些想法的地方工作过。它教会我专注于建立一种重视对新思维过程开放的公司文化。因为你永远不知道什么时候一些看起来愚蠢的想法会变成下一件大事。

没有人想因为说出自己的想法而感到羞愧。鼓励人们分享他们的想法，即使是那些还没有充分发展的想法，可以培养团队的信任感，并可以带来更大的创造力和创新。它还通过减少员工的压力和避免负面结果 如精疲力竭和其他可能最终导致离职的健康问题，使公司变得更加强大。

在工作场所达到心理安全的这一点并不是一个容易的过程。鉴于职业世界中经常使用的标准，这必须是高度有意的。长期以来，人们一直被认为是组织结构中可替换的齿轮，而不是具有独特而有价值的技能、天赋和缺点的人。

这是关于认识到人们也是人，并且经常想要在他们的工作中感受真实。这就是为什么谈论改善公司及其产品的新想法以及谈论你如何交谈是有价值的；为团队讨论的重要话题设定一个明确的标准。

培养支持性文化并不意味着没有分歧。但是有一些方法可以提供相反的意见，而不显得像个混蛋。工程师之间的这种相互尊重，让人们可以用批判的眼光看待事物，而不会伤害任何人的感情。

在 CodeSee，我们每周都会做一次产品回顾。这通常会导致这样的陈述:“产品是这样做的，但我希望它是这样做的。”这不是对任何人的批评，每个人都知道。这是每个人在追求让事情变得更好的过程中互相帮助。

## 不要雇用混蛋

向公司灌输这些价值观是一种有益的体验，可以让工程师蓬勃发展。但是如果他们不出现在招聘过程中，这些都不重要。我们在日常工作中同样重视的东西在招聘中也是重中之重:谦逊、信任和自我反省。

我们宁愿有一个符合我们文化并体现这些价值观的体面工程师，也不愿有一个不支持他人的伟大工程师。我们不同意仅仅因为你是天才就可以视他人如粪土的观点。

这些特质在向未来员工提出的问题中表现出来。我们的两个行动相当简单。我们经常要求受访者定义三个优点和缺点。这个数字需要一些真正的自省，并帮助我们了解一个人是否知道他们可以在哪里改进，并确定了个人发展战略。我们还会问，“当我们给前任经理打电话时，他们会对你说些什么？”

这些问题自然是为了引出一些自我反思。如果他们能设身处地为前任经理着想，这表明他们有高度的自我意识和同理心。

令人惊讶的是，这些问题让不少申请人感到恼火。实际上，有人对我大喊大叫，采访的人，因为他们感到不舒服。显然，那种反应告诉了我们所有我们需要知道的事情。

## 成功的工具

机构知识是任何团队的资产。然而，这给已经捉襟见肘的高级工程师带来了额外的压力，他们要把这些信息记在脑子里，然后教给/解释给经验不足的同事。

这是需要改变的事情，以解放高级工程师，让他们更多地关注大局。我们可以用可视化代码的工具来解决这个问题，并投资于模型，为工程师提供这些模型来改善培训。

今天，许多工程师花 60%的时间阅读代码。如果我们能减少到 40%呢？还是 20%甚至 10%？如果我们所有的工程人才花 40%的时间做其他事情，想想我们会在哪里？

然而，生产力工具可能是一把双刃剑。一些公司没有给软件工程师更多的自由和信任，而是利用技术榨取员工的最后一点生产力，这些努力注定会失败，因为它们鼓励有才华的工程师开始计划他们的退出。

底线是每个人都需要工程师。失去至少一些是不可避免的。但是，如果你认真考虑创造一种支持性的创新文化，并保持这种文化，那么竞争就容易多了。

但这不是一项简单的任务。你做的每一个决定——从你雇佣的人到你制造的产品，再到你在会议中相互交谈的方式——都对文化的形成起着作用。工程师无论去哪里都会得到补偿，所以获得成功的最好机会是尽可能建立最好的、最少垃圾的文化。**
