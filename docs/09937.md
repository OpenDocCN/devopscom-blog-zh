# OutSystems 收购 Ionic 以进一步自动化应用程序开发

> 原文:[https://devo PS . com/out systems-acquisites-ionic-to-further-automate-app-development/](https://devops.com/outsystems-acquires-ionic-to-further-automate-app-development/)

OutSystems 本周透露，它已经[收购了 Ionic](https://www.businesswire.com/news/home/20221107005221/en/OutSystems-Acquires-Ionic-to-Unite-the-Power-of-Open-Source-Mobile-Development-with-High-Performance-Low-Code) ，这是一家开源框架提供商，为开发人员构建和部署移动和桌面应用程序提供更高层次的抽象。

OutSystems 首席技术官帕特里克·简表示，Ionic 将加速该公司正在进行的努力，扩大其平台的覆盖范围，以构建低代码应用程序，而开发人员不必担心，例如，应用程序部署在其上的设备的屏幕大小，因为框架会自动调整大小。事实上，Ionic 将为 OutSystems 已经使用的平台提供一个扩展，以自动化低代码应用程序的构建和部署，他补充道。

除了用于构建可以在多种平台上运行的移动应用程序的同名用户界面(UI)框架之外，Ionic 产品组合还包括一个用于本地部署 web 应用程序的工具，称为 Capacitor，以及一个用于构建基于 web 组件的设计系统的工具链，称为 Stencil。总的来说，今天有超过五百万的开发人员在使用这些工具。Jean 说，OutSystems 计划扩展这些工具的功能，使在移动设备和桌面系统上部署应用程序变得更加简单。他补充说，总的目标是减少开发人员目前在需要构建和部署运行在多个平台上的应用程序时所承担的认知负荷。

Jean 指出，随着专业开发人员之外的更多个人开始参与构建、部署和更新应用程序，这种能力将被证明是至关重要的。他补充说，事实上，这些开发者中有许多可能是只兼职开发应用程序的商业人士。

使用[低代码](https://devops.com/?s=low-code+application+development)工具构建的应用程序的百分比急剧增加。专业开发人员已经接受了这些工具和平台，所谓的公民开发人员也使用它们来构建应用程序。随着需要构建的应用程序不断增加，低代码平台在帮助开发人员跟上新应用程序需求的步伐方面发挥了至关重要的作用。

不太清楚的是这些低代码应用程序对 DevOps 工作流的影响。一方面，显然需要部署的应用程序比以往任何时候都多。与此同时，许多曾经需要的 DevOps 和数据建模过程，例如专用的持续集成/持续交付(CI/CD)平台，正在被嵌入到由创建低代码工具的供应商提供的平台中。

不管低代码应用程序是如何部署的，有一点是肯定的，这些工具和平台正在极大地加快数字业务转型的步伐。DevOps 团队将很快需要安装护栏，使具有广泛经验和技能的开发人员能够安全地部署和更新应用程序。

从好的方面来看，DevOps 团队也应该期望以前所未有的规模自动化低代码应用程序的交付变得更加容易。事实上，未来的问题可能不是 IT 是否能跟上业务需求，而是业务是否能跟上现在的创新速度。
