# 海岸线宣布多云事故自动化

> 原文：<https://devops.com/shoreline-announces-multi-cloud-incident-automation/>

*增加了对 GCP 和 Azure 的支持，使 Shoreline 客户能够跨多个云环境进行调试、修复和自动化，从而降低运营复杂性*

**加利福尼亚州雷德伍德城，2022 年 2 月 16 日—** [事故自动化公司 Shoreline.io](https://shoreline.io/) 今天宣布其平台在微软 Azure 和谷歌云平台(GCP)上的全面可用(GA)，建立在现有的对 AWS 的 GA 支持之上。有了今天的新闻，客户可以无缝地修复他们的多云环境中的问题，就像他们在一台机器上工作一样。

如今的 DevOps 团队难以手动管理不同的资源层次结构、标记、凭据和跨多个云设备群的帐户。他们必须了解每个云提供商的工具以及配置和维护其设备群的细微差别。跨云、客户、集群或区域查询资源和执行操作以调试或修复基础架构问题可能是一项挑战。

Shoreline 通过在所有三个云中的虚拟机和容器之间并行执行命令，管理特定于云的发现资源、收集指标和构建自动化的复杂性。无论车队规模大小，交互式命令在几秒钟内返回。这使得 SREs 能够实时调试，并在三个最大的云提供商之间快速构建自动化，从而提高可靠性。

Shoreline 首席执行官兼创始人阿努拉格·古普塔(Anurag Gupta)表示:“多云环境正变得越来越普遍，操作也越来越复杂。“海岸线提供并行全船队调试和自动化补救的能力已经通过 AWS 得到了验证。随着今天的发布，我们现在也在 Azure 和 GCP 提供相同的无缝体验。这使得客户可以更轻松地调试事件和自动修复，而无需为每个云环境进行复杂的手动编码。”

所有的海岸线功能，包括基于角色的访问控制和审计，都一致地跨云运行，减少了管理多云生产环境的挑战。Shoreline 会自动发现资源和标签，提供跨云环境的统一资源层次结构。客户可以跨云、区域、帐户和集群透明地查询和执行操作，而无需登录和注销不同的 VPN，从而减少了人工劳动和使用许多云专用工具的需要。

TigerGraph 产品和创新副总裁兼通用圣地亚哥创新实验室的 Jay Yu 博士表示:“如果您需要管理一个多云车队，Shoreline 是您的最佳选择。“Shoreline 的方法简化了我们生产云中事件的补救，并消除了跨多个云提供商运营的固有复杂性。这意味着我们可以更快地了解和解决问题，从而提高我们的系统可用性。

无论您是在多云环境中处理实例维护事件，将文件移动或复制到云存储，还是自动扩展，Shoreline 都允许用户充分利用每个云，而不必担心云之间的差异。
