# 邮递员调查显示了对构建和部署 API 的关注

> 原文:[https://devo PS . com/postman-survey-shows-focus-on-building-and-deployment-APIs/](https://devops.com/postman-survey-shows-focus-on-building-and-deploying-apis/)

Postman 今天公布的一项针对 28，000 多名开发人员和应用编程接口(API)专业人员的调查发现，近一半的受访者(49%)现在将大部分时间花在开发 API 上。超过四分之一(28%)的人现在每周花 20 个小时或更多的时间使用 API。

构建 API 的平台提供商 Postman 的首席传道者金·莱恩(Kin Lane)表示，这种转变反映了从新冠肺炎疫情开始的软件开发和集成活动的增加。近三分之二的受访者(63%)表示疫情对他们的工作产生了一定或显著的影响，其中近一半的受访者指出他们的组织所做的改变是由 API 促成和支持的。

此外，整整 94%的受访者表示，在未来 12 个月内，对 API 的时间和资源投入将会增加或保持不变。创建 API 的最大障碍是缺少时间(45%)，而使用 API 的最大障碍是缺少文档(55%)。在 API 目标方面，调查发现，质量(80%)、敏捷性(66%)和可靠性(65%)是调查受访者的三大目标。

然而，调查还指出，只有三分之一的受访者表示，他们可以在一天到一周的时间内在任何地方构建和部署 API，大多数人需要一周到一个月的时间。大约三分之一的人还表示，他们每周一次到每月一次将 API 部署到生产环境中。超过 80%的受访者指出，不到 25%的更改失败，而类似比例的受访者表示他们可以在不到一天的时间内恢复，三分之一的受访者表示他们在不到一小时的时间内完全恢复。

Postman 调查还发现，超过四分之三的受访者(76%)拥有不到五年的 API 开发经验。只有一半的受访者表示，他们同意或非常同意他们的 API 团队的领导对他们的 API 实践在五年内应该在哪里有一个清晰的认识。然而，微服务(58%)和 Kubernetes (50%)、容器(46%)、无服务器架构(44%)和 GraphQL (35%)是受访者最期待进入 2022 年的技术。

从积极的一面来看，超过 60%的受访者认为他们的 API 领导和治理团队表现出了“变革的意愿”，并适当考虑了“员工的利益和意见”在定期提供反馈和流程方面，超过一半的领导和治理团队(53%)表现良好。

根据 Postman 的调查，API 团队合作的两种方式是通过专用的合作平台(50%)或通过将 API 工件发布到 GitLab、GitHub 和 Bitbucket 等存储库(45%)。使用最广泛的 DevOps 工具是 Jenkins (37%)、Azure DevOps (26%)、GitHub Actions (25%)、AWS DevOps (23%)和 GitLab Pipelines (22%)。

在部署 API 方面，持续集成/持续交付(CI/CD)平台是最广泛使用的(57%)，其次是在云中部署 API(38%)、框架(34%)和定制部署方法(27%)。

超过 70%的受访者将安全性和性能列为与 API 集成时考虑的首要因素。然而，在生产 API 时，首要因素是与内部系统的集成、与外部系统的集成、为内部系统添加功能以及为最终客户增强功能。类似地，在使用 API 时，首要因素是与外部系统的集成、为最终客户添加功能、为内部系统添加功能以及内部系统之间的集成。

总的来说，Lane 指出，随着越来越多的 API 被开发和部署来推动各种数字业务转型计划，组织将需要更加关注 API 发现和管理。组织内的不同团队有一种趋势，不仅创建冗余的 API，而且忘记所谓的[僵尸 API](https://devops.com/?s=zombie+APIs)，这些 API 不再受支持，但仍可能被网络罪犯利用。

总体而言，Lane 表示，该调查清楚地表明，越来越多的组织正在寻求 API 优先的数字化转型方法。现在的挑战是管理所有的 API，这些 API 会很快变得太好了。
