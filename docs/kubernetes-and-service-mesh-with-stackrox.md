# Kubernetes 和服务网格与 StackRox

> 原文：<https://devops.com/kubernetes-and-service-mesh-with-stackrox/>

DevOps 和 Kubernetes 使组织能够“保持安全性”，将安全性添加到代码中，而不是添加到代码之上。换句话说，安全性是内置的，而不是外加的。

这是 DevOps 世界中 Kubernetes 系统安全性的一个方面，在与媒体运营首席执行官兼主编 Alan Shimel 的独家采访中，StackRox 的联合创始人兼首席技术官 Ali Golshan 解释了这一点。另一个方面是安全性已经变得连续而不是快照，这提供了“对各个阶段、各种部署以及运行环境的更多可见性，”他说。

在采访中，他探讨了容器、编排、Kubernetes 和服务网格。

“利用这种[云原生基础设施](https://devops.com/devops-chat-holistic-kubernetes-and-cloud-native-app-security-with-stackrox/)，Kube 和服务网格以正确的方式减少了您对第三方工具的依赖，允许您将通常交给供应商的事情交给您的基础设施，并在您的基础设施内完成，这实际上使它更加安全和高度可扩展，”Golshan 说。“你可以实现自动化，减少束缚和依赖。”

查看下面的采访，了解更多关于 Kubernetes、服务网格和安全反馈循环的最佳和新兴实践。
