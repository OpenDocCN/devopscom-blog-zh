# 开源比闭源更安全吗？

> 原文：<https://devops.com/is-open-source-more-secure-than-closed-source/>

不可回避的事实是，对于大多数软件开发人员来说，安全性是日常操作中最重要的因素之一。从专有代码到敏感的客户数据以及介于两者之间的一切，安全漏洞可能导致数百种甚至数千种危机情景。

数据安全是每个公司都面临的问题，无论公司规模大小。开发人员寻求遵循当今数据和应用程序安全最佳实践的安全解决方案，以降低风险并为最终用户提供最安全的体验。

用户和开发者之间的一个普遍争论是[开源软件](https://devops.com/?s=open+source)比[闭源软件](https://securityboulevard.com/?s=closed+source+software)更安全还是更不安全。虽然对于什么是真正安全的环境有各种不同的思想流派，但有充分的理由相信，对于那些希望优先考虑安全性的人来说，开源软件是更好的选择。

虽然开源代码可能不会比它的闭源兄弟更安全，但这里有几个几乎可以肯定它是安全的原因——只要你采取正确的方法。

## 开源提供了真正的透明性

成熟的科技巨头和新兴的创业公司都与开发社区中一些最聪明的人合作，将有价值的专有软件产品和服务推向市场。然而，到了最后，用户别无选择，只能相信供应商所说的某个软件实际上是如何“工作”的。因为你看不到代码本身，所以没有办法真正验证你被告知的特性、功能和安全性是否准确。

由于许多原因，这可能是有问题的——尤其是潜在安全缺陷的威胁。一家公司可能会吹嘘其通过精明的公共关系和产品营销努力对用户安全和隐私的承诺。但是，如果代码不符合标准，这决不会给用户提供比口头服务更多的东西；即使它坚如磐石，问题是“它到底有多安全？”永远无法理直气壮的回答。

开源软件为使用它的团队提供了更大的透明度；对代码本身及其维护方式的可见性。给予组织访问源代码的权限使他们有机会自己评估代码的安全性。此外，用户可以更清楚地了解如何对代码库进行更改以及更改了哪些内容，包括预发布审查流程、依赖项的更新频率以及开发人员和组织如何应对安全漏洞。因此，开源软件用户对他们正在使用的软件的整体安全性有了更全面的了解。

## 开源鼓励社区投入

另一个主要的好处是在推动开源软件增长和发展的社区中发现的。绝大多数开源软件是由具有前瞻性思维的开发人员社区支持的，他们中的许多人使用他们构建和维护的相同软件作为与团队成员交流的主要手段。开源开发者和围绕软件的社区在很大程度上重视用户的输入，许多用户的建议最终被合并到新版本中。社区的参与和投资最终会导致比软件封闭源代码更负责任的漏洞披露。

当漏洞报告确实出现时，它们往往更详细，因为代码是开源的；通常，他们甚至会提供如何解决给定问题的代码建议。这是非常有用的，可以大大加快修补漏洞的速度，因为它允许开发人员更快地移动，而不会陷入诊断过程中。

## 更快的安全更新

在闭源软件的框架内，当涉及到接收软件更新时，用户受到那些产品或服务背后的公司的支配――开源往往是一个不同的故事。完全开源项目甚至允许用户和开发人员自己动手，提供反馈和修复，如果他们选择这样做的话。

通常情况下，对备受瞩目的闭源应用程序的更新和修复涉及大量复杂的规划。例如，如果不在预算之内，用户可能会在几周、几个月甚至更长时间内看不到新的更新——无论是否存在安全缺陷。

在大多数情况下，当迭代和发布新版本时，开源往往比闭源移动得更快。这是真的，有几个原因，包括开源软件在任何给定的时间倾向于对源代码有更多的关注，以及社区驱动的尽可能改进产品的兴趣。如果发现了一个重大的安全缺陷，用户甚至可能选择自己修复代码，从而达到在封闭的源代码环境中永远无法达到的控制水平。

## 开源提供了灵活性、协作性和增强的安全性

开源方法本身并不能保证安全性，但是它确实为团队提供了对他们所依赖的软件的更好的洞察力和控制。随着开放核心和源代码软件被更广泛地采用，它们提供了大量的机会来构建令人难以置信的安全环境，而不牺牲最终用户的可用性或官方维护者的支持。
