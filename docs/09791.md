# 通过迁移到云减轻开发人员的工作流程难题

> 原文:[https://devo PS . com/momenting-developer-workflow-feathers-by-move-to-cloud/](https://devops.com/alleviating-developer-workflow-headaches-by-moving-to-the-cloud/)

当谈到[无服务器](https://containerjournal.com/?s=serverless)和云计算时，技术领域的领导者犯了一个常见的错误:他们希望将当前的环境完全迁移到云中。这就像沿着道路网络规划飞行路线和根据街道交通状况决定路线一样有局限性。你可以做到，但每个人都会看着你，好像你疯了一样。

保持这种“搬来搬去”的心态会让开发人员看不到云计算和无服务器技术的真正投资回报。相反，我们必须挑战我们的传统思维方式。

## 重新定义我们对软件开发的看法

软件的传统定义是让硬件做我们想让它做的任何事情。我们使用操作系统来控制内部功能和外围设备。我们使用应用软件来指导我们的计算机执行命令。所以我们倾向于认为软件是可编程的，需要专门的编程语言来开发。但是最流行的函数式编程语言不是 Erlang、Haskell 或 Lisp。如果你相信的话，这是微软的 Excel，拥有超过 7.5 亿用户。

每个 Excel 电子表格都代表了某人的软件；这使得他们的硬件以对他们有价值的方式处理数据。同样，我们在信息技术方面构建的“系统”只是更多的软件。云计算为我们提供了一系列新的强大工具，因此我们可以比以前更快地构建更大的系统。

## 什么是云计算和无服务器计算？

云计算很大程度上参考了数据中心和数据存储。无服务器计算是云计算的一部分。它让开发人员无需管理基础设施就能创建软件。最终，它提高了生产率，加快了产品交付，优化了资源，并使开发人员专注于最终产品而不是基础设施维护。

一旦部署，无服务器应用程序响应需求，根据需要扩大或缩小。因此，当一个无服务器功能闲置时，它不会产生任何成本。

## 云计算和无服务器计算的主要优势

云计算和无服务器框架有很多好处。到目前为止，领导们最兴奋的是[成本控制](https://devops.com/?s=cloud+cost+management)和增加投资回报率。这也是领导者最常忽略的一点。

## **通过云计算和无服务器提高投资回报率**

使用无服务器框架和云计算提高投资回报率的首要方法是避免从头构建任何东西。投资于思想开放的研究。有时你需要自己构建一些东西，但是通常有办法从你的团队转移一些负担。

一个简单的例子是函数。大多数开发人员听到“无服务器”时首先想到的是函数。但是运行代码的函数只是拼图的一小部分。无服务器技术允许开发团队将尽可能多的工作交给云平台，如 Azure、亚马逊网络服务(AWS)和谷歌云平台(GCP)。

就像有人不会买一块地，耕种它，种植小麦，浇水，将小麦脱粒成面粉，烘烤面包并烘烤它以制作一片烤面包一样，当构建块已经存在于他们的云平台中时，开发人员不应该编写自己的代码。以 Azure 的持久功能特性为例。此功能管理有状态的工作流，因此团队不需要在其他地方通过您的数据工作流显式创建、存储和检索进度。这个过程在团队构建的系统中非常普遍，以至于云提供商不需要我们写代码就可以实现它。就像送上门的速食烤面包一样！

一个团队越能利用已经完成的代码，他们就能越多的时间回到他们的计划中，越多的钱回到团队的预算中，这是我们都可以欣赏的！
