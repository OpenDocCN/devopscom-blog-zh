随着行业的发展，Moogsoft 继续发布重要的产品更新，将创新推向新的水平，以简化指标和事件数据的提炼，形成有意义的见解。

新的产品功能允许 Moogsoft 用户优化生产力，将时间集中在创新上。这些功能和更新包括:

**扩展集成能力，轻松公开地连接不同的工具和数据**

*   “创建您自己的集成”更新和增强:“创建您自己的集成”是行业首创。这是连接所有不同的可观察性和监控工具的最简单和最透明的集成方法。它允许用户生成自定义的 API 端点，用户可以向该端点发送任何原始事件和/或指标数据。用户能够查看收到的有效载荷，并轻松地将他们选择的数据映射到 Moogsoft 字段。然后，Moogsoft 会自动将事件正常化并关联到用户可以采取行动的事件中。通过最新的更新，用户能够集成包含多个事件的有效负载。这节省了大量的时间、资源和资金，并且完全透明，可以快速连接和运行。

*   Microsoft Azure App Insights integration(beta):Azure App Insights 是一个云到云的集成，它为管理员提供了摄取在 Microsoft Azure 中配置的指标数据的能力。然后将分析度量数据，以建立具有上限和下限的正常操作行为。如果指标偏离正常行为，将会发生异常事件。这使用户能够理解正常行为和他们需要解决的潜在问题之间的区别，并且消除了管理员为每个指标手动创建阈值的需要。通过了解应用程序的正常行为并根据 Moogsoft 提供的上下文采取行动，用户能够快速解决问题并从中学习，以增强弹性，提高稳定性并最终改善他们的客户体验。

**推动事故工作流程自动化以减少工作量**

*   **通过自动分类自动丰富事件(测试版):**传统上，需要大量规则和查询来将警报分类为已知的基础架构元素和相关类型的故障(例如，网络|可用性)，这是事件管理产品的所有丰富功能的工作方式。直到现在。Moogsoft Classification 使用 ML 来分析您的数据，并自动确定基础架构元素和故障类型，因此您不必。这样可以腾出时间专注于更重要的任务，比如增加可靠性、创新或者多睡一会儿。

*   **具有自动关闭功能的扩展工作流自动化:**自动关闭是一个可配置的设置，它扩展了工作流自动化，以便在任何警报或事件满足用户设置的时间和标准时自动关闭它们。当用户修复了那个烦人的问题，清除事件出现，或者指标回落到正常行为，警报严重性将自动变为“清除”，状态变为“已解决”一旦所有警报都已解决并且时间达到用户配置的时间，警报和事件将自动关闭。这又节省了一个步骤，因此用户可以回到一个创新的氛围，并改善产品和服务。

*   **带标签的自动化事件上下文:**有了 Moogsoft，在 SLA 和 SLOs 中保持良好状态所需的上下文就在您面前。事件将自动聚合和删除警报中的所有标记，包括在此过程中创建的任何自定义标记。这为用户提供了通过自动化事件上下文进行补救的最快途径，因此他们可以缩短平均解决时间，并保持在 SLA 和 SLO 范围内。

**让配置和用户生活更加轻松**

*   导航更新:moog soft 的导航非常流畅，易于使用，用户可以随意悬停或扩展。每个区域都经过精心设计，让用户只需点击一下鼠标就能到达他们需要去的地方，同时还能让用户深入了解产品中的数据流以及事物之间的联系。简单流畅的导航节省了宝贵的时间，帮助 SRE 团队了解更多信息，因此他们不必浪费点击和时间来寻找他们需要的信息。毕竟，他们想解决问题，回到最重要的事情上来。

*   **数据流和配置概述:**为了充分发挥 observability 和 AIOps 产品的价值，用户不仅需要了解概念，还需要了解如何配置数据流和产品。这就是为什么当用户登录时，第一页就准确地显示了数据如何流经产品以及所有可以配置的不同项目。用户能够很快启动并运行，这样他们就可以花更多的时间开发，而花更少的时间来解决如何操作一切。

*   **安全凭证存储:** Moogsoft 通过安全可靠地存储已输入的凭证，在配置任何集成或发现新的监控源时节省时间。用户可以在特定的集成或所有的集成中重用它们。这是他们的选择。但底线是 Moogsoft 省去了每次都需要记住和输入凭证的麻烦。这使得监控发现可以运行，并找到要监控的新源，这样用户就不会错过任何事件而措手不及。用户能够专注于改进和自动化，而不是不断配置和试图了解需要监控的内容。

*   **API 密钥&基于角色的访问控制:**作为管理员，您可以看到每个人的访问级别，包括谁可以通过与用户 ID 直接关联的 API 密钥访问 API。如果用户被挂起或删除，API 密钥也会被自动删除。管理员还可以提升用户权限来共享管理活动。这节省了宝贵的时间，并确保团队自动遵守访问法规。

*   **个性化和交互:**新的个性化功能允许用户更改列的顺序、显示哪些列，甚至更改其 UI 的刷新率，以便以首选速率显示新信息。这将允许用户更好地利用不同的交互，如“分配给我”快捷方式，并查看最相关的信息，以便他们可以更快地做出决策，快速有效地解决问题。公司日复一日依赖 SRE 团队，分秒必争。个性化用户界面将有助于保持组织平稳运行。
