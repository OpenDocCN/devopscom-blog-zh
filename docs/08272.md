有趣的是，为了做到这一点，我们只有一种可行的方法——分析二进制文件。为什么？因为在嵌入式领域，一切都围绕着二进制文件。你有实时操作系统、第三方编译的二进制文件、文件和软件包的映像，但你看不到它们到底发生了什么。你可以从高通或 AUTOSAR，无论什么已知的库，以驱动程序的形式获得它们，制造商需要研究这些，他们没有源代码。他们必须能够分析它们。

因此，我们专注于二进制文件，同样，我们也有了平台，很快，在我们不知情的情况下，我们的客户开始将我们用于嵌入式安全分析之外的目的。他们开始分析 web 应用程序、移动应用程序和容器。因为在处理嵌入式这种被认为是最复杂、最先进或最难自动分析的东西时，做剩下的事情通常更多的是获取更多的数据，而不是开发更多的技术。

因此，我们进行了调整，并允许我们的客户分析与设备通信的应用程序和与设备通信的 web 服务。我们添加了越来越多的功能，到那时，我们说，“等等，为什么要专注于嵌入式系统，如果通过二进制分析方法，我们可以服务于整个行业，我们可以为任何软件产品提供安全性。”
