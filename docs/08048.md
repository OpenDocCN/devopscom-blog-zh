# Docker 推进协作应用开发平台

> 原文:[https://devo PS . com/docker-advances-collaborative-application-development-platform/](https://devops.com/docker-advances-collaborative-application-development-platform/)

*引入 Docker 开发环境，更新 Docker Compose，并通过限定范围的个人访问令牌增强安全性*

**加利福尼亚州帕洛阿尔托，2021 年 5 月 27 日**–[Docker，Inc .](https://www.docker.com/)今天宣布对 Docker 协作应用开发平台进行更新和增强，帮助开发团队通过新功能和改进的工作流程提高速度。在 [DockerCon Live 2021](https://docker.events.cube365.net/dockercon-live/2021) 上宣布，Docker 推出了三项关键增强功能，包括引入 Docker 开发环境、新版本的 Docker Compose，以及用于应用管道自动化的限定范围的个人访问令牌。在另一份新闻稿中，Docker 还宣布为超过 800 万注册 Docker 开发者扩展公司的可信内容产品。

Docker 产品副总裁 Donnie Berkholz 表示:“今天的开发人员面临着各种语言、框架和架构，以及每个管道阶段的工具之间不连续的接口，导致应用开发从源代码到云运行时极其复杂。"今天的公告使开发者能够通过 Docker 将他们的想法变为现实，从而加快发货速度."

Docker 是为开发团队提供协作应用开发平台的领先提供商。Docker 为数百万想要构建、共享和运行从代码到云的优秀应用的开发者提供了简单性、速度和选择。今天发布的详细信息包括:

**Docker 开发环境**

Docker 开发环境构成了 Docker 新的协作团队开发体验的基础。它使团队能够更好地合作，通过花费更少的时间来设置和调试环境，将容器的易用性和可再现性带到交互式开发环境中，从而为客户创造价值。开发人员现在可以轻松地共享不仅仅是他们的代码，还可以获取依赖性和更广泛的应用程序上下文，并与队友共享，就像做 docker 推送一样简单。Docker 开发环境将于下个月推出。

**复合码头 v2**

Docker 继续投资 Docker Compose，这是开发人员最喜欢的运行多个容器应用程序的工具。JetBrains 最近的一项调查“2020 年开发者生态系统的状态”显示，76%的开发者在开发期间使用 Docker Compose 来运行容器化的应用程序。版本 2 将 docker compose 命令合并到 Docker CLI 中。最近发布的 Compose 现在提供了对深度学习和多环境配置的改进支持。对 Windows 和 Linux 的 GPU 支持使开发人员和数据科学家更容易运行深度学习等 GPU 密集型工作负载。服务配置文件使管理和配置切换变得容易，例如开发、测试和生产。Docker Compose v2 目前在 Docker 桌面上可用，并将在今年晚些时候包含在 Docker Linux 包中。

**限定范围的个人访问令牌**

随着对软件供应链的访问成为更高的安全问题，Docker 正在更新个人访问令牌(pat)以支持细粒度的许可。新的 Docker 令牌权限计划于今年夏天晚些时候发布，包括定义只读、读写或只读公共回购的选项。这一安全特性使开发人员和他们的经理能够更严格地控制哪些团队和哪些自动化应用交付管道可以访问他们的容器映像和应用，进一步扩大了 Docker 平台在为开发人员提供构建安全供应链的工具方面的地位。除了保护对应用程序的访问之外，令牌还简化了跨团队协作，让开发团队决定他们希望哪些团队和系统能够不受限制地访问他们的内容
