# 使用 DevSecOps 避免安全冷漠

> 原文：<https://devops.com/avoid-security-apathy-with-devsecops/>

在疫情加速快速数字化转型的背景下，每个行业都出现了高水平网络安全违规事件的增加。随着组织继续支持分布式和[远程工作](https://devops.com/?s=remote+work)，组织必须解决不完整的安全策略和程序导致暴露于各种 IT 风险的风险。网络安全管理软件产品最近公布了其[网络安全管理软件产品 IT 趋势报告 2021:构建安全的未来](https://it-trends.solarwinds.com/)的调查结果，该报告分析了组织在未来管理、缓解和预防风险方面的准备程度。

## 衡量风险

调查发现，46%的 tech pro 受访者承认在过去 12 个月中面临中等程度的企业 IT 风险。感知到的风险水平因组织的规模而异，与小型企业(11%)和中型企业(7%)相比，企业组织(19%)感知到高风险或极高风险。随着过去一年高级别的网络安全漏洞的增加，这是企业内部安全的一个令人担忧的现实。

然而，调查结果强调了一种可能的脱节——尽管企业内部存在企业 IT 风险，但 tech pro 受访者对其风险管理和缓解准备战略充满信心。事实上，73%的 tech pro 受访者“同意”或“非常同意”他们的 IT 组织已准备好管理、减轻和解决风险因素相关的问题，因为他们已经制定了政策和/或程序。

## 如何避免安全冷漠

作为一名技术专家，很容易将安全性视为事后的想法，或者期望所有权完全落在专门的安全团队肩上。对于那些已经在一家公司工作了很长时间并且抗拒改变的技术专业人士来说尤其如此。

当我们从疫情走出时，这些持续的看法反映了在安全问题上的冷漠和自满，只会增加他们业务的风险水平。他们更倾向于限制接触新技术，忽视更好的工作方式，或者更糟糕的是继续忽视组织内其他不太明显的风险领域。有些事情需要改变。

技术专家必须转变观念，接受安全是他们的责任之一。要开始这个过程，IT 团队必须从外到内检查当前的过程，并在评估解决方案时应用严格性。由于经常被营销信息轰炸，这意味着花时间验证工具或解决方案的功能和能力，以帮助区分事实和虚构。

接下来，技术专家必须专注于提供对所有系统的完全可见性，以识别风险和机会领域。即使是很小的调整，如更快的升级和补丁，采用密码管理器和多因素身份认证(MFA)解决方案，也有助于改善组织的整体安全状况。

最重要的是，技术专家应该不断评估他们的风险管理、缓解和协议，以避免自满和对风险“视而不见”。

在疫情引发的一年“危机模式”之后，人们很容易自满，回避风险评估和规避的责任。随着安全形势的不断发展，技术专家必须保持警惕，部署能够更好地洞察系统以发现风险和机会的解决方案。今年的研究揭示了技术专家和 IT 领导在优先事项和政策上保持一致的机会，这些优先事项和政策不仅可以使单个组织，而且可以使整个行业在未来成功管理、减轻和预防风险。
