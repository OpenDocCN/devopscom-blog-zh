# CodeLogic Toolkit 提高了应用依赖关系的可见性

> 原文:[https://devo PS . com/codelogic-toolkit-increases-visibility-into-app-dependencies/](https://devops.com/codelogic-toolkit-increases-visibility-into-app-dependencies/)

CodeLogic 今天发布了一个[工具包](https://www.prnewswire.com/in/news-releases/codelogic-launches-first-of-its-kind-continuous-software-intelligence-platform-848148512.html)，它使开发人员能够扫描二进制文件、运行时应用程序行为和数据库连接，然后利用图形技术实时识别连接和依赖关系。

CodeLogic 首席执行官布莱恩·皮尔斯(Brian Pierce)表示，目标是让开发者更容易识别应用元素之间的关系，以提高整体生产力，并简化向应用开发项目添加新成员的过程。

一般来说，大多数开发人员在向应用程序添加代码时，并没有真正了解这些代码会如何影响其他组件。CodeLogic 创建的工具包是该公司现有连续软件智能(CSI)平台的扩展，该平台采用服务器软件来存储和处理通过其代理软件收集的数据。皮尔斯说，目标是让开发人员更容易通过他们可以直接从内部访问的工具包直接使用这些见解，例如，集成开发环境。

![](../Images/3106f0c3cf8f0eb0d2f89fcfc5eea6c7.png)

在微服务时代，随着应用程序变得越来越复杂，理解应用程序组件之间的所有依赖关系也变得越来越困难。开发人员需要更好地了解应用程序，以便更好地管理在任何环境中添加新代码时不可避免会出现的中断/修复过程。CodeLogic CSI 平台为方法、类和应用程序的整体健康状况提供自动化的影响评分，因为开发代码是为了提供这种可见性。

Pierce 补充说，这些信息在帮助 IT 团队了解任何新发现的漏洞可能对应用程序环境产生的影响方面也发挥着重要作用。

他指出，总的来说，随着时间的推移，组织积累的技术债务水平削弱了开发人员的生产力。Pierce 说，向应用程序环境添加代码变得越困难，开发人员就越不可能热情地付出努力。他指出，用不了多久，应用环境的更新数量就会开始下降。

开发人员的工作效率已经成为一个主要问题，这仅仅是因为组织现在想要构建和部署的应用程序的数量超过了他们可用于雇佣或签约开发人员的财务资源。开发人员花在研究如何构建应用程序上的时间越多，用于编写代码和推动创新的时间就越少。皮尔斯说，事实上，添加代码越令人沮丧，开发人员就越有可能跳槽到一家在构建和部署应用程序方面摩擦少得多的公司。

许多高级 IT 领导在批准项目之前也需要了解应用程序环境。许多 IT 团队发现有多个项目或其他团队编写重复的代码或复制现有的功能。不管什么原因，当更多的开发人员觉得他们的时间和努力没有得到充分的认可时，他们会用脚投票。
