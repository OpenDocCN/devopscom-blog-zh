# Splunk 向观察平台添加数据管理工具

> 原文：<https://devops.com/splunk-adds-data-management-tools-to-observability-platform/>

在其 [.conf22](https://conf.splunk.com/) 活动上，Splunk 今天宣布其正在[通过 Splunk 云平台](https://www.businesswire.com/news/home/20220614005337/en/Splunk-Delivers-Security-and-Observability-Solutions-for-the-Hybrid-Cloud-World)使装载数据和跨混合 it 环境管理数据变得更加容易。

此外，Splunk Enterprise 正在进行扩展，以添加对 Microsoft Azure 的支持，SmartStore for Azure 可存储冷数据以及对亚马逊网络服务(AWS)和谷歌云平台(GCP)的现有支持。Splunk Cloud 还有一个开发人员版本，开发人员可以创建和测试自己的应用程序，目前正在预览中。

Splunk 高级副总裁兼首席产品官加思·福特(Garth Fort)表示，总体目标是更容易自动调整用于在 Splunk [observability](https://devops.com/?s=observability) 平台中存储数据的参数。随着越来越多的组织将可观察性作为 DevOps 的最佳实践，他们中的许多人正在面临管理大量数据的挑战，这些数据现在以日志、指标和跟踪的形式收集。

Splunk 云平台的数据管理器最初将在 AWS 和 Azure 上提供，对 GCP 的支持将在今年夏天晚些时候提供。实际上，它为 it 团队提供了一个跨多个云的数据管理控制平台。

同时，摄取操作使应用粒度控制成为可能，使 it 团队能够过滤、屏蔽和路由 Splunk 平台内或外部 AWS S3 云存储服务中的移动数据。

Splunk 在会议上推出的其他功能包括联合搜索工具的扩展，以实现整个 Splunk 生态系统的单一视图，以及针对 Splunk Enterprise 9.0 的完全托管的 Splunk Assist 服务，为客户提供对其安全环境的深入洞察。

最后，该公司增强了 Splunkbase，这是一个用于访问由 Splunk 及其第三方合作伙伴提供的超过 2，500 个应用程序和集成的精选集合的框架。

作为 IT 运营分析的先驱，Splunk 在企业 IT 环境中占有很大的份额。它现在正在寻求扩大这个基础，成为一个可观测性平台提供商。就向可观察性的过渡而言，现在还为时尚早，但目标是让 It 团队更简单地查询平台和可能导致 it 中断的表面问题。

目前还不清楚有多少组织拥有制定这些查询所需的技能和专业知识，但随着时间的推移，用能够发现这些问题的机器学习算法来增加 It 人员应该变得更加可行。

与此同时，IT 组织正在收集比以往更多的关于其 IT 环境的数据。理论上，这些数据将提供优化日益复杂的应用程序环境所需的环境。当然，接下来的挑战是找到一种方法来高效地存储和管理所有这些数据，无论是在分析之前还是之后。

无论采用何种方法来实现可观察性，超越简单的 IT 环境监控的需求从未像现在这样明显。
