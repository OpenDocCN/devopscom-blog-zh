# Neo4j 将利用资金扩展托管云数据库

> 原文：<https://devops.com/neo4j-will-use-funds-to-expand-managed-cloud-database/>

***确定无疑。让我从第二个开始，把它结合起来。当我九年前加入这家公司时，真正吸引我的是，在我看来，数据中最有价值的东西一直是模型以及模型中包含的内容。这是事物之间的联系，这家公司所做的是在数据空间更倾向于这一点，你可以看到在 NoSQL 发生的一切都是为了真正简化数据模型，这样你就可以做到非常非常高的数量和规模，但非常简单的数据问题，存储和检索。为了做到这一点，您可以减轻某些技术上更复杂的需求，例如，如果您不处理复杂的关系以及大规模存储和检索购物车数据，您就不需要 acid。***

***因此，Neo4j 十多年来一直致力于通过观察事物之间的联系来解决如何理解数据中的上下文和因果关系的问题。因此，如果你看一下我与我的汽车、驾驶我的汽车的其他人、我可能拥有的其他车辆、我可能拥有的其他东西(如房子)的关系，以及与业务和业务的关系，然后将它们结合在一起，突然之间，你就进入了这些复杂的企业数据场景，这些场景一直是一个挑战。作为一名顾问，我经历了很多这样的问题，试图一次性解决这些问题。***

***因此，Neo4j 在当时的数据库领域做了一件有点疯狂的事情，那就是构建一个假设你可以很好地进行随机 I/O 的引擎。因为如果我要了解关系，查询可以通过数据的任何方式进行，如果我在 20 层深度供应链中，可能会非常非常深，例如，跟踪一个项目或一个大型支付网络，并寻找欺诈检测，欺诈模式或洗钱，以便我可以抓住这些。这些都是关于数据中的模式，所以 Neo4j 是一家从头开始建立数据库的公司，希望帮助用户理解关系和数据，以便他们可以更深入地理解上下文、因果关系、二阶、三阶、四阶效应。***
