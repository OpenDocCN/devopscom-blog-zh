# BMC 从大型机扩展了 DevOps 的覆盖范围

> 原文:[https://devo PS . com/BMC-extends-devo PS-reach-from-the-mainframe/](https://devops.com/bmc-extends-devops-reach-from-the-mainframe/)

BMC 今天宣布，它已经[扩展了其面向大型机环境的 DevOps 平台](https://newsroom.bmc.com/news-releases/news-release-details/bmc-hardens-mainframe-security-new-capabilities-protect-against),以提供与 GitHub Actions(管理工作流的框架)和 HCL Launch(HCL 软件的持续交付平台)的集成。BMC 在 2020 年 6 月通过[收购 Compuware](https://www.prnewswire.com/news-releases/bmc-completes-acquisition-of-compuware-to-mainstream-mainframe-devops-301068082.html) 获得了 DevOps 平台。

BMC 智能 Z 优化和转型高级副总裁兼总经理 John McKenny 表示，这些集成以及对 Jenkins、GitLab 和 VS Code 等 DevOps 平台的现有支持，使得加速将大型机作为更大分布式计算环境的一部分的应用程序的开发成为可能。例如，DevOps 团队现在可以为 BMC AMI DevOps for Db2 工具使用一个 HCL 启动插件来自动捕获和传播跨 IBM Db2 数据库的数据库更改。

同时，BMC 还增强了一个用于 Java 环境的 BMC AMI Ops Monitor，它可以自动发现在大型机上 z/OS 环境中运行的所有 Java 虚拟机(JVM)。BMC AMI Ops Monitor for Java Environments 现在包括了发现由这些 JVM 调用的应用程序编程接口(API)和服务的能力。

BMC 今天还透露，它的工作流向导工具已经被接受为由开放主机项目发起的开源 Zowe 倡议的一部分。工作流向导使得安装、配置和维护安装在运行 z/OS 的大型机上的软件变得更加容易。

最后，BMC 透露，它已经通过增强 BMC AMI 安全软件的主管呼叫筛选程序和 Unix 系统服务(USS)数据丰富功能，扩展了其特权访问管理(PAM)工具。McKenny 说，基于 BMC 通过收购 RSM Partners 获得的技术，这种增强提高了 BMC AMI 安全软件提供的保护级别，通过暴露可疑活动来阻止攻击，例如来自可能已经泄露管理员凭据的恶意内部人员的攻击。

McKenny 指出，这些天来，一切都在受到攻击，因为网络犯罪分子一旦发现危及一个平台的方法，他们几乎会立即尝试对其他平台应用相同的技术。McKenny 补充说，大型机环境倾向于处理组织最有价值的数据，自然是高价值的目标。

总的来说，McKenny 说，依赖大型机的 IT 组织继续朝着采用 DevOps 最佳实践的方向稳步前进。Forrester 最近代表 BMC 进行的一项调查发现，80%的受访者表示他们的大型机开发工具需要显著改进才能提供更多价值。

在许多情况下，加速数字化转型计划(通常需要访问驻留在大型机中的数据)的需求迫使许多组织内部出现 DevOps 问题。构建大型机应用程序的软件开发团队几乎总是需要创建 API，使其他应用程序更容易使用这些数据。

当然，并不是每个组织都有大型机。然而，随着数字业务转型的不断发展，需要使用大型机数据的应用程序数量将稳步增长。大型机应用程序管理团队面临的挑战是找到一种方法来提供这种访问，而不使用传统的瀑布流程来构建所需的 API。
