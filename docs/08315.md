# 需要一个新的 DevOps 测试环境平台

> 原文:[https://devo PS . com/a-new-devo PS-test-environment-platform-is-need/](https://devops.com/a-new-devops-test-environment-platform-is-needed/)

测试环境管理是 DevOps 转型的瓶颈，并推动了对新测试环境管理平台解决方案的需求。迫切需要开发人员和测试人员不必等待共享环境的按需测试环境。

证据很明显。在我的书 [Engineering DevOps](http://mybook.to/engineeringdevops) 中，我指出寻找和设置测试环境是 DevOps 价值流中瓶颈的主要来源。

在我 2021 年 2 月与 Plutora 的网络研讨会上，我展示了开发测试管理环境的[最佳实践。](https://www.youtube.com/watch?v=Lhpe1_Kl38o)我指出了编排和管理测试环境通常是价值流中延迟和浪费的主要来源。

PractiTest 的 2020 年测试报告的[状态表明 DevOps 实践增加了 57%；从 2017 年的 26%到 2020 年的 41%。报告显示，46%的测试已经转向使用 CI/CD 流程，55%的测试正在积极参与测试左移，36%的测试正在生产中进行。这是 DevOps 测试呈上升趋势的明显证据。](https://www.practitest.com/resource/state-of-testing-report-2020/)

Wolfgang Platz 2019 年 12 月的文章[为什么软件测试仍然是瓶颈](https://thenewstack.io/why-software-testing-remains-a-bottleneck/#:~:text=In%20most%20organizations%2C%20testing%20delays,it's%20optimal%20to%20do%20so.)表明，测试比开发过程的任何其他部分都要对更多的延迟负责——占开发过程中瓶颈的 52%。在 2020-21 年[世界质量报告](https://techbeacon.com/app-dev-testing/world-quality-report-2020-21-qa-integral-digital-transformation)中，53%的受访者表示测试环境在需要时不可用，67%的人缺乏预订和管理测试环境的能力，71%的人缺乏对可用测试环境的了解。

由 [Velocity](https://docs.velocity.tech/intro-and-overview/why-should-i-use-velocity) 撰写的一篇文章回应了预生产环境的问题。“随着微服务和云原生应用的兴起，环境变得越来越复杂。通常，您需要等待或竞争才能获得在特定微服务上进行开发的权限，或者可能会花时间等待部署到共享的试运行环境，结果却发现有些东西已经发生了变化，于是您又回到了起点。”

凯捷的[持续测试报告–2020](https://www.capgemini.com/research/continuous-testing-report-2020/)将测试环境管理(TEM)定义为管理基础设施、测试数据和应用，以支持按需测试。这些环境是持续测试的最大障碍之一。在这份报告中，36%的受访者表示，他们正在花费超过一半的时间来构建和管理测试环境；44%的受访者表示，他们花费了数周的时间来搜索、管理和生成测试数据。

测试环境应被视为可随意增加或减少的功能，包括虚拟化、基础设施即代码(IaC)实践、服务虚拟化、容器化和改进的测试数据管理。

鉴于以上所有证据，显然需要一种新的 DevOps 测试环境管理方法。

以下是 DevOps 测试环境管理平台需要提供的优势示例:

*   与当前工具兼容:
    *   自动映射和配置应用程序基础架构，以便在运行环境中开发和测试多项服务，就像在生产环境中一样。这需要完全独立于云和技术堆栈，如 Kubernetes 和其他环境。
    *   该平台需要在不中断其他应用程序的情况下并行处理应用程序，并且能够完全访问数据和第三方依赖性。
*   自动配置和维护:
    *   该平台需要实时自动配置和更新生产基础架构映射中的任何更改，以便短暂的环境将始终与生产中的环境保持同步。该平台将确保复制生产的正确环境，而不必花费时间来确保配置任何更改，而不必等待。
*   数据和第三方依赖性:
    *   该平台需要加载相关数据和第三方资源，这些是测试、工作和调试环境的重要组成部分，因此测试环境将尽可能模拟生产环境。
*   团队协作:
    *   该平台需要能够与其他利益相关者共享短暂的测试环境，这样他们就可以获得可见性，并可以合作以获得更快的反馈周期。这对于协作调试也很重要。

### **这意味着什么**

[DevOps 测试](https://devops.com/?s=continuous+testing)正在兴起，测试环境是 DevOps 测试的一个主要瓶颈。需要一个新的 DevOps 测试环境管理平台来减少瓶颈。需要开发人员和测试人员不必等待共享测试环境的按需测试环境。这种解决方案将提高敏捷性，减少测试中浪费的编排瓶颈，通过全面的自动化测试提高质量和风险管理，并通过提供更智能的见解和反馈实现持续改进。
