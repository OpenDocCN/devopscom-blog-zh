# Octopus Deploy 收购 Dist 以支持云本地软件部署

> 原文:[https://devo PS . com/octopus-deploy-acquisites-dist-to-power-cloud-native-software-deployments/](https://devops.com/octopus-deploy-acquires-dist-to-power-cloud-native-software-deployments/)

*   *持续部署流程编排软件公司 Octopus Deploy 宣布收购云原生容器注册解决方案 Dist。*
*   *Dist 的技术将为 Octopus Deploy 客户提供无缝的集装箱登记体验*
*   *这家总部位于布里斯班的公司从 Insight Partners 获得 1.725 亿美元融资后的首次收购*

**澳大利亚布里斯班——2022 年 2 月 1 日(2022 年 1 月 31 日)—**持续部署流程编排软件的领先提供商 Octopus Deploy 今天宣布收购了总部位于悉尼的云原生容器注册和工件存储库服务 Dist。

在过去的十年中，将软件无缝部署到任何计算环境中的容器技术越来越受欢迎。这些容器打包了运行软件所需的一切，从可执行程序、系统工具、库和设置，简化了部署过程，因此应用程序可以在任何操作系统上运行，而无需进行大量修改。

在 Flexera 2021 年云状态报告中，49%的受访组织表示他们目前正在公共云中使用容器化应用，另有 39%的组织正在试验或计划使用容器化应用。

对 Dist 的收购和整合将使使用 Octopus 部署云原生应用变得更加无缝。客户还将能够更快地部署应用程序，满足保留策略的合规性目标，并最大限度地降低云原生应用程序的存储成本。

Octopus Deploy 首席执行官兼联合创始人 Paul Stovell 表示，Dist 的技术和专业知识将成为该公司平台的核心，因为它将继续构建其公共和内部云功能。

Octopus Deploy 在去年 10 月与 Dist 会面，并立即注意到两家公司之间的许多相似之处。就像 Octopus Deploy 一样，Dist 完全是由其两位悉尼的创始人黄云·杨和斯蒂芬·海恩斯启动的，他们都将加入 Octopus Deploy，担任其云部门的云架构师。Dist 品牌也将并入 Octopus Deploy。

“在过去，如果我们发现 Octopus Deploy 中缺少的功能，我们的第一反应会是组建一支工程师团队来开发该功能，但随着我们的成长和成熟，我们意识到，从零开始的内部开发并不总是服务客户的最佳方式，”Stovell 说。

“通过与 Dist 团队合作，我们将使软件团队能够更轻松地提供更多功能，减少停机时间，并减少管理工作量。我很高兴欢迎 Dist 团队加入 Octopus Deploy，并迫不及待地想看看我们共同构建的东西！”

继 2021 年 4 月宣布 [Insight Partners 投资 1.725 亿美元](https://octopus.com/blog/insight-investment)之后，Octopus Deploy 也不断壮大，2021 年收入增长 60%，员工人数增加了一倍多，达到 179 人。

**关于章鱼部署**

Octopus Deploy 成立于 2012 年，帮助超过 25，000 家公司的 DevOps 团队加速跨云和本地基础架构的可靠、可重复和可跟踪的部署。Octopus 提供了 API 优先的方法、内置的多租户、安全性、发布管理、部署以及跨所有团队、项目和技术堆栈的 runbook 自动化。我们的客户通过访问 500 多个自动化步骤模板获得最佳实践。Octopus 还集成了数百种技术，包括 Azure、AWS、GCP 和 Kubernetes。公司使用 Octopus Deploy 来最大限度地减少停机时间，提供更多功能并满足其合规性需求。
