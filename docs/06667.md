# Instana 成为第一家自动检测流程崩溃根本原因的 APM 提供商

> 原文:[https://devo PS . com/instana-becoms-first-APM-provider-to-automatically-detect-the-root-cause-of-process-crashes/](https://devops.com/instana-becomes-first-apm-provider-to-automatically-detect-the-root-cause-of-process-crashes/)

*使用 eBPF 构建的新异常终止检测，适用于 Linux 上的任何受监控技术*

**Chicago–2020 年 8 月 17 日**–面向微服务应用的自动化[应用性能管理](https://instana.com/application-manage)解决方案的领先提供商 Instana 今天宣布，该公司监控的整套技术均可实现自动流程崩溃检测和根本原因分析。这一新功能使 Instana 成为首个 APM 解决方案，能够自动检测和确定由该解决方案管理的所有流程崩溃的根本原因。

Instana 的技术总监兼 APM 策略师 Chris Farrell 表示:“尽管今天的微服务应用旨在动态处理单个服务崩溃，但事实是，异常的进程终止确实会导致应用性能下降和其他严重的运营问题。“Instana 新的自动崩溃检测和根本原因分析允许应用程序相关的任何人查看崩溃发生的时间和方式，以便修复崩溃，甚至是难以排除的问题，从而提高应用程序的稳定性并改善最终用户体验。”

这项新功能的官方名称是“异常进程终止检测”，目前已在 Instana 的 SaaS 和自托管(内部)解决方案的生产版本中提供。Instana 的云原生和微服务应用程序性能监控解决方案免费提供这一新功能，该功能可在 Instana 支持的所有技术和平台上开箱即用，运行于 Linux 机器或容器上，运行于长达四年的内核上。

将异常流程终止故障排除与 Instana 的实时变化检测、即时管道反馈、每个请求的端到端分布式跟踪和自动根本原因分析相结合，在单个应用程序监控解决方案中提供了最全面的故障排除功能。Instana 的应用程序性能管理解决方案免费提供所有功能。

与传统的 APM 工具不同，Instana 的自动化[应用性能监控](https://www.instana.com/application-montioring/) (APM)解决方案可发现所有应用服务组件和应用基础设施，包括 AWS Lambda、Kubernetes 和 Docker 等基础设施。Instana 自动为应用技术堆栈的每个部分部署监控传感器，跟踪所有应用请求并分析每个流程，无需任何人工配置，甚至无需重启应用。该解决方案实时检测应用程序和基础架构的变化，调整自己的模型，并在几秒钟内可视化这些变化和任何性能影响。
