# 您的基础架构为边缘计算做好准备了吗？

> 原文：<https://devops.com/is-your-infrastructure-ready-for-edge-computing/>

与许多 IT 创新一样，边缘计算始于工程师，是技术的自然延伸，旨在满足不断增长的需求。这个概念并不新鲜；分布式计算已经存在了几十年。但是，与此同时，标准开始趋同，边缘硬件开始在贸易展会上出现，宣传机器看到了机会。它扩大了 edge 在减少延迟、提供软件定义的部署、降低云网络成本等方面的巨大承诺。但通常情况下，粗体功能项目符号忽略了企业必须解决的生产问题，包括边缘计算的粗糙点和它增加的额外操作复杂性。

当然，[边缘计算](https://orangematter.solarwinds.com/2019/10/22/the-dos-and-donts-of-edge-computing/)会在一点点过度兴奋的推广中幸存下来，就像它之前许多曾经不可思议的技术一样。人们常说，“什么？将我的所有数据中心应用程序从硬件中抽象出来作为虚拟服务器？不可能！”十年后，我们无法想象如果没有这些，我们将如何提供传统的企业服务、云计算、在线零售、媒体流和其他一切服务。[虚拟化](https://www.dnsstuff.com/virtualization-technology-cloud-computing-types)挺过了尴尬的宣传期，边缘计算也将如此。边缘计算解决方案的需求只会不断增长。

感谢工程师和运营团队，边缘分布式模型正在走向实用。它证明了自己能够通过减少延迟、可扩展性以及更重要的可管理性来满足新级别网络性能的要求。对于一些企业来说，从长远来看，这甚至可以降低成本。

## **并非所有数据都是相同的**

随着联网设备的激增和对 5G 技术的日益关注，技术专业人士应该抛开他们对边缘宣传的自然不愿，考虑这是一种真正的可能性。它的采用是随着新兴技术和最大限度利用它的应用的兴起而出现的:5G、增强现实、自动驾驶汽车、物联网和智能制造。这些环境不仅需要低上游延迟，还需要高性能计算和及时的结果数据。光的传播速度非常快，推动基础设施越来越接近消费者，以品牌差异化用户体验的形式实现更快、更无缝的处理。
