# 低代码如何让 DevOps 更强大

> 原文：<https://devops.com/how-low-code-makes-devops-stronger/>

虽然它的根源可以追溯到快速应用程序开发(RAD)，[低代码](https://devops.com/category/blogs/low-code-no-code/)应用程序开发大约在三年前开始获得强劲的势头。最初，DevOps 社区中的一些人对这种趋势不屑一顾，尽管无数种方法——从面向专业开发人员的无代码到低代码——开始进入市场。

当时，各种规模的企业都采用了敏捷软件开发流程，将一些关键工作负载转移到云基础设施，并广泛使用测试自动化工具。DevOps 作为一门重要的学科出现，人们认为它的优势会盖过低代码。

快进到 2021 年， [Gartner 预测，低代码开发市场将增长 22.6%](https://www.gartner.com/en/newsroom/press-releases/2021-02-15-gartner-forecasts-worldwide-low-code-development-technologies-market-to-grow-23-percent-in-2021) ，一年内总额达到 138 亿美元，因此很明显，低代码的讣告还为时过早。低代码市场的增长也没有限制 DevOps 的增长。数以千计的大中型企业已经采用或计划采用低代码进行专业开发，DevOps 的发展还在继续。用于专业开发的低代码平台还没有取代 DevOps 模型；相反，它将重点放在与业务和 it 合作的健壮的软件开发生命周期上，以提高转换数字体验的速度。

## 对速度的需求

在新冠肺炎疫情的早期，许多组织被迫加快数字创新，以使其业务活动适应新常态，或者寻找新的方式与更灵活的竞争对手竞争，以保持生存。为了能够向突然分散的客户和员工交付服务，他们开始转向低代码开发平台，以便更快地交付解决方案。

低代码软件易于定制，可以适应任何组织需求，不需要深入的编码技能，并加快开发速度。低代码平台具有大量预制工具，可以在几个小时内开发一个应用程序，而不是几周或几个月。

一些低代码开发平台具有集成的 DevOps 服务和自动化测试功能。前沿平台使用人工智能来生成将在云上执行的测试用例，作为持续集成/持续交付(CI/CD)管道的一部分。虽然这些平台不会取代开发人员，但它们可以帮助自动化他们的工作，以便他们可以将时间集中在新功能和伟大的想法上，而不是编码重复但重要的测试，或解决后端复杂性。

## 信心增强剂

借助低代码平台，DevOps 团队获得了用于测试、部署、安全性、数据管理的自动化工具(所有后端开发必备工具)以及前端和设计能力。这是一套非常不同的技能和专业知识，但使用低代码平台来简化开发过程，几乎允许任何开发者在任何应用的整个生命周期中工作。

即使来自组织营销团队的有抱负的程序员构建了一个内部应用程序，DevOps 团队也不必担心构建验证或版本控制，因为低代码实现了一个连续的过程。

一个小型 DevOps 团队可以高效、自信地管理来自任何开发者的应用的构建和部署。它还可以在多个设备上部署这些应用程序，只需要很少的不同编程语言的升级。低代码平台包括用于智能扬声器、可穿戴设备、AR、PWA、原生移动和其他真实世界用例的预连线组件、模板和示例应用。

## 战略重点

一旦低代码将组织的 DevOps 团队从重复任务中解放出来并减轻工作负担，团队就可以专注于更复杂的问题，如升级和维护现有的企业级应用程序(我想到了企业资源规划)，或致力于其他高价值的数字和运营战略计划。

## DevOps 和低代码万岁！

面向专业开发人员的低代码最适合开发消费级移动和多体验应用。无论一个组织是试图数字化一个旧的流程，还是创造新的方式来与消费者、业务伙伴、客户或员工打交道，低代码开发都有助于组织更加响应和敏捷——就像一个良好的 DevOps 文化一样。随着数字经济不断增长的需求和随之而来的开发人才的短缺，通过采用低代码开发方法的速度，我们可以增强开发人员(和 DevOps)模型的能力，以迎接下一个业务挑战。
