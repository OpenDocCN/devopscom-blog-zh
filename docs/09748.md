# 专注于你能做的

> 原文:[https://devops.com/focus-on-what-you-can-do/](https://devops.com/focus-on-what-you-can-do/)

我们现在是在 2022 年下半年，这是板上钉钉的事。无论你在过去几年中有多少预算，无论是大还是小，都将会缩减。这是周期性的，不应该感到意外。预算缩减的幅度可能更大，但在我们出生之前，预算就一直在波动。我认为，他们在 IT 方面的波动比业务的任何其他部分都要大，原因很简单，因为大部分 IT 仍然被视为管理费用，老实说，我们的成本很高。我们的工资、设备、软件和 SaaS 成本高昂，当企业不得不勒紧裤腰带的时候，它们就成了诱人的目标。

所以，让我们来谈谈在事情结束下沉，然后开始复苏的时候，接下来的一年(或五年)该做些什么。新项目将会很少，但是改进一个组织已经部署的项目花费很少。敏捷和 DevOps 的“现在就做”思想在很多方面都有帮助，但是老实说，留下了“可能更好”的实现的痕迹。可以使用迭代来改进的实现。我的“看着我！我也是一名学者！”建议如下:

1.  回顾
2.  巩固
3.  评价
4.  重复

**回顾**你在整个组织中拥有的东西。我以前提出过这个建议——甚至可能就在大约六个月前——但是它[值得重复](https://devops.com/the-great-market-convergence-is-coming/)。我们有一个阶段，团队甚至个人开发人员为产品或产品线做出选择。这创造了一个*地段*的重叠。列出 DevOps 链每一步使用的工具；对于每个项目/开发运维工具链。

**巩固。**从短期来看，成本需要下降，并且一旦下降就要保持可持续性。最好消除软件或 SaaS 冗余，而不是那种*其他*类型的冗余。这一步还意味着简化企业开发运维基础设施。所以比较做同样工作的产品，选择一个。这并不像专家们说的那么简单；每种产品都有独特之处。但是您必须做出艰难的选择，减少维护成本和时间。

**评估。**对于在多家机构工作过的人来说，这并不难相信，但如果有人忽略了这一点，没有一个环境是完美的。他们*所有*都有长处和短处。世界上最好的员工，如果有无限的时间和预算，至少会做一件蠢事。一年后，他们做的许多聪明的事情看起来也很愚蠢——因为市场和/或技术已经发展了。查看您的环境的每个步骤—从包含哪些库/模块到部署步骤和运行时监控。找出哪些地方可以做得更好，然后根据成本进行组织，并把它们计划出来。

**迭代。**从复习阶段开始，再做一遍，因为如果你认为第一次就全懂了，那你就错了。或者你比我们其他人更聪明，更彻底，两者之一。

这将使团队不断改善公司的 IT 环境，同时外部因素会降低预算。这个周期与其他周期不同，因为不清楚 it 人员配备将受到怎样的影响。虽然它看起来越来越像是会有和往常一样的影响，但在这次衰退开始时，这一点一点也不清楚。不管怎样，在经济困难时期，没有百万美元的项目来改善环境是有用的。例如，我的公司目前正在重新考虑我们对框架的选择，以及我们如何仍然使用测试工具作为 DevOps 管理工具。

让灯一直亮着，让运行组织的服务器和应用不断摇摆，把改进的重点放在你无法改变的因素上。但是不管你周围发生了什么，都要不断地让它变得更好。
