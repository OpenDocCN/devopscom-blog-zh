# 2022 年最佳:微软增加大量工具和 Azure 云扩展

> 原文:[https://devo PS . com/Microsoft-adds-raft-of-tools-and-azure-cloud-extensions/](https://devops.com/microsoft-adds-raft-of-tools-and-azure-cloud-extensions/)

随着 2022 年的临近，我们 DevOps.com 想要突出今年最受欢迎的文章。以下是我们 2022 年最佳系列的最新内容。

在本周的 [Build 2022](https://news.microsoft.com/build2022/) 大会上，微软增加了一系列工具和平台更新，作为其提高开发人员和支持他们的 DevOps 团队生产力的努力的一部分。

例如，[微软的 Dev Box](https://techcommunity.microsoft.com/t5/azure-developer-community-blog/introducing-microsoft-dev-box/ba-p/3412063) 提供了对 Azure 云上一组预配置环境的访问，使开发人员在几秒钟内就可以上线，而之前宣布的 [GitHub Copilot](https://blogs.microsoft.com/ai/how-ai-makes-developers-lives-easier-and-helps-everybody-learn-to-develop-software/) 工具利用人工智能来加快代码编写速度，将于今年夏天全面上市。

Azure CTO 孵化器团队的合作伙伴项目经理多诺万·布朗(Donovan Brown)告诉与会者，首要目标是让开发人员在新冠肺炎疫情之后继续远程工作的同时保持敏捷。

对于 DevOps 团队，Azure Monitor 的应用洞察模块中添加了智能地图功能。智能地图利用机器学习算法来发现性能瓶颈或故障热点，也可以作为技术预览版提供。

还有一个工具可以发现并分组相关的 Hyper-V 虚拟机(VM)和物理服务器，以确保在 Azure 迁移期间识别并包含所有必需的组件。

其他更新包括一个[应用服务登陆区加速器](https://techcommunity.microsoft.com/t5/apps-on-azure-blog/what-s-new-in-azure-app-service-at-build-2022/ba-p/3407584)，它提供了一个工件和文档的组合，使用 Azure App Service 3.0 和预览工具来自动化应用部署，以便将 ASP.NET web 应用迁移到平台。此外，微软正在微软 Azure 应用服务上预览对谷歌远程过程调用(gRPC)的支持。. NET 多平台应用程序 UI 框架允许在单个代码库中使用 C#和 XAML 为 iOS、Android、macOS 和 Windows 构建原生编译的应用程序。

微软还扩大了其管理应用编程接口(API)的云平台的覆盖范围。Azure API 管理服务现在支持 Azure Private Link 虚拟专用网络(VPN)以及自动更新证书的功能。此外，微软正在预览对 API 的 GraphQL 查询语言的支持。

此外，还添加了具有 Azure Active Directory(Azure AD)工作负载身份联合功能的 [GitHub OpenID Connect (OIDC ),以消除在 Azure 云平台外部存储和访问机密的需要。](https://techcommunity.microsoft.com/t5/azure-developer-community-blog/build-secure-apps-on-hardened-dev-environments-with-secure/ba-p/2893917)

微软与 VMware 合作，正在添加 Spring framework 的托管 Azure 实例[，用于构建 Java 应用程序，并在 Azure 云上提供作为软件即服务(SaaS)应用程序的](https://techcommunity.microsoft.com/t5/apps-on-azure-blog/azure-spring-apps-enterprise-tier-is-now-generally-available/ba-p/3418245)[Dynatrace observability platform](https://techcommunity.microsoft.com/t5/azure-marketplace-blog/announcing-dynatrace-for-azure-natively-integrated-observability/ba-p/3392551)和 NGINX 代理软件的预览。

最后，微软增加了大量工具，使得在 Kubernetes 集群上构建云原生应用变得更加简单。

尚不清楚开发人员采用这些工具的速度有多快，但 DevOps 团队可以预计在未来几个月内看到在 Azure 云上开发应用程序的速度会增加。当然，挑战将是管理所有这些应用程序——无论是在构建时还是在 Azure 或其他兼容的 IT 环境中部署后。
