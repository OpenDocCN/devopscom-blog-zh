# 蜂巢矩阵使调试云原生分布式系统的矩阵的使用现代化

> 原文：<https://devops.com/honeycomb-metrics-modernizes-the-use-of-metrics-for-debugging-cloud-native-distributed-systems/>

新的度量特性帮助客户在传统调试实践和现代系统之间架起了一座桥梁，同时又不影响对可观察性的深刻见解。

**三藩市、加利福尼亚州2021 年 6 月 10 日 —** [蜂巢](https://c212.net/c/link/?t=0&l=en&o=3190500-1&h=1816891428&u=https%3A%2F%2Fwww.honeycomb.io%2F&a=Honeycomb) ，该公司提供业界领先的可观察性软件，用于理解、故障排除和优化云原生分布式系统，今天宣布推出蜂巢指标。这一新功能允许已经使用可观察性的客户从同一界面关联基于指标的系统健康指标，从而快速识别和解决应用程序问题。这种简化的视图弥合了传统监控和现代可观察性实践之间的差距，为工程团队的调查工作流程提供了简单性和速度。

许多组织使用多个第三方传统监控和调试工具来识别和诊断影响其应用程序级性能的系统级约束。多种工具的复杂性增加了软件开发人员在使用云原生和分布式系统构建和维护生产环境时已经面临的挑战，这使得使用传统的监控工具(在应用程序级别使用指标)来调试应用程序故障几乎是不可能的。

借助 Honeycomb Metrics，客户可以轻松地让工程团队使用一个界面来识别和诊断复杂的应用性能问题，无论这些问题源自应用代码还是系统问题，从而实现技术投资的最大化。它使他们不再需要使用几个遗留的监控工具来诊断应用程序级别的问题或系统级别的问题，也不再需要自己将这些问题联系起来。蜂巢度量让工程团队了解应用程序级的可观察性问题和系统级的度量问题。

蜂巢度量还通过允许团队将传统的监控数据与新的可观察性数据结合使用，帮助团队更容易地开始可观察性。新客户可以使用蜂巢度量作为从传统方法(使用应用级度量进行调试)到并行采用事件驱动的可观察性实践的垫脚石，而无需首先识别和重写现有的工具。

Honeycomb 联合创始人兼首席执行官 Christine Yen 表示:“Honeycomb Metrics 帮助不熟悉可观测性的客户快速了解他们的传统监控世界如何融入现代可观测性世界。“增加对系统级指标的本机支持降低了进入可观测性的门槛，同时仍然在应用程序级别提供基本的、事件驱动的分析功能，这使 Honeycomb 成为可观测性领域的市场领导者。”

Honeycomb Metrics 的工作原理是让客户使用 OpenTelemetry 或其他几种度量格式(包括 StatsD、Prometheus 和 Amazon CloudWatch)之一向 Honeycomb 发送度量数据。Honeycomb Metrics 包括可用性增强，在有用的逻辑可视化中用事件数据覆盖时间序列度量数据。客户可以看到他们的基础架构级指标如何与他们的应用程序级事件相关联，从而获得整体系统性能的一致表示。Honeycomb Metrics 还引入了建议查询，引导客户自动查看他们需要的表示。

Heroku 的平台可观测性首席工程师 Sarah Sherbondy 说:“有了蜂巢指标，我们能够在一个地方非常容易地在搜索、可视化和查看原始指标数据之间切换。“这有助于我们比以前更快地验证和迭代仪器改进。蜂巢式查询接口对于度量标准来说非常棒，就像对于事件一样。只需点击几下鼠标，从我们的指标中获得洞察力和清晰度变得前所未有的简单。此外，由于 Honeycomb 使用 OpenTelemetry 协议，因此设置起来非常容易。这种易用性意味着不费吹灰之力就能提高可靠性！对我们来说是双赢的局面！”
