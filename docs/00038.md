# Red Hat 宣布容器化应用的认证，将客户的信心和信任延伸到云

> 原文：<https://devops.com/red-hat-announces-certification-for-containerized-applications-extends-customer-confidence-and-trust-to-the-cloud/>

### Red Hat Container Certification 为 ISV 提供了安全、一致和简化的平台，以利用 Docker 格式的容器

RALEIGH, N.C., Mar 11, 2014 (BUSINESS WIRE) — Red Hat, Inc. [RHT](https://www.marketwatch.com/investing/stock/RHT) -0.84% , the world’s leading provider of open source solutions, today announced the extension of its application certification program to include containerized applications. The Red Hat Container Certification ensures that application containers built using Red Hat Enterprise Linux will operate seamlessly across certified container hosts. Designed with the needs of independent software vendors (ISVs), service providers and their enterprise customers in mind, the certification extends the confidence customers have with Red Hat Enterprise Linux, which currently supports thousands of certified applications, to certified containers running on certified container hosts. The pending release of Red Hat Enterprise Linux 7 and Red Hat’s OpenShift Platform-as-a-Service (PaaS) offering will both be certified container hosts, with Docker as a primary supported container format.

Linux 容器

Linux 容器已经成为一种关键的开源应用程序打包和交付技术，它结合了轻量级应用程序隔离和基于映像的部署方法的灵活性。开发人员迅速接受了 Linux 容器，因为它们简化并加速了应用程序部署，而许多 PaaS 平台都是围绕 Linux 容器技术构建的，包括 Red Hat 的 OpenShift。Red Hat Enterprise Linux 使用控制组(cGroups)、资源管理、se Linux 和网络名称空间等核心技术来实现 Linux 容器，从而实现安全多租户并降低安全漏洞的可能性。

基于 Red Hat Enterprise Linux 的功能，应用程序容器被部署为软件包，其中包括应用程序及其所有必需的运行时组件。好处包括:

*   **即时可移植性**，在任何认证的容器主机上部署认证的应用程序容器
*   **最小占用空间**，避免包含完整操作系统的虚拟机映像的开销
*   **简化维护**，通过修补应用程序及其所有依赖项来减少工作量和风险
*   **降低开发成本**，针对单一运行时环境开发、测试和认证应用

红帽集装箱认证

Red Hat Container 认证是 Red Hat 行业领先的认证组合的最新成员，该认证组合包括 Red Hat Enterprise Linux 和 Red Hat Enterprise Linux OpenStack 平台。该认证也是一个更大的红帽技术计划的一部分，反过来支持红帽全球生态系统中数以千计的第三方技术合作伙伴

为了支持这一新宣布的集装箱认证，红帽还宣布了集装箱认证合作伙伴早期访问计划(PEAP)。PEAP 将持续到晚春，在认证和合作伙伴计划(目前计划于 2014 年年中)正式启动之前，为生态系统合作伙伴提供参与集装箱化所需工具和资源的早期测试、集成和反馈的能力。从今天开始，有兴趣加入 PEAP 的公司可以通过访问[https://engage . red hat . com/content/enterprise-early-access-se-201402101640](http://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fengage.redhat.com%2Fcontent%2Fenterprise-early-access-se-201402101640&esheet=50821592&newsitemid=20140311005350&lan=en-US&anchor=https%3A%2F%2Fengage.redhat.com%2Fcontent%2Fenterprise-early-access-se-201402101640&index=1&md5=a391104310b54f697eeca17a8e9956e2)了解更多信息。
