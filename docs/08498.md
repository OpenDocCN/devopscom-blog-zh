# 最终用户分析:SLA 的现实检验

> 原文:[https://devo PS . com/end-user-analytics-a-reality-check-to-SLA/](https://devops.com/end-user-analytics-a-reality-check-to-slas/)

每个人都知道良好的用户体验对客户来说至关重要，但很少有组织对自己的员工给予同等程度的考虑。缺乏对内部最终用户体验的洞察会导致沮丧、低效和浪费。详细的分析可以揭开帷幕，揭示您的服务级别协议(SLA)与您当前系统和软件的现实之间的差距。

您的一些员工可能拥有比他们真正需要的更多的应用程序，而其他人可能缺乏有效完成工作的工具。也有可能你的一些员工正在使用过时的软件，这些软件已经不再适用。在你着手改变这种情况之前，你必须对整体情况有所了解。

你的用户体验是什么样的？
很多组织会很难回答这个问题。他们可能不知道自己有多少应用程序，哪些正在使用，以及它们的性能如何。

根据 Gartner 的“[如何通过成本和价值优化明智地管理 IT 预算”](https://www.gartner.com/doc/2727719/manage-it-budget-wisely-cost)，影子 IT 的兴起以及 40%的支出不再由 IT 部门控制的事实进一步搅浑了水。

要确定您公司的用户体验，请问自己以下问题:

*   您有多少用户，他们的角色是如何定义的？
*   您每天都有哪些设备访问您的系统？
*   所有的接入点都在哪里？
*   正在访问哪些内容？
*   正在使用哪些应用程序？

通过在虚拟化 [XenApp](https://www.citrix.com/products/xenapp/overview.html) 或类似环境中削减未使用的应用程序并整合常用应用程序，您可以将资源重新定向到可以提供最大收益的地方。

**服务级别协议失败**
采用服务级别协议模型并在内部应用可能很有诱惑力，但服务级别协议过于简单和单一，无法提供这里需要的洞察力。云服务提供商可能关注可用性和正常运行时间，但是这些指标能告诉你什么呢？像页面加载缓慢这样的问题通常不包括在指标中，但如果你想评估用户体验，它们确实应该包括在内。

这里还有一个信任问题。您能证明您的服务提供商达到了预期吗？安全呢？如果您真的想了解正在发生的事情并采取行动加以改进，依靠提供商的自我报告和一组不能准确反映平台或应用的性能或用户满意度的指标是不够的。

**最终用户分析**
您需要洞察包括传统、移动和虚拟在内的多种环境中所有硬件、用户和应用的实时和历史数据。良好的终端用户分析将标记安全风险和可疑行为，记录应用程序何时挂起或崩溃，并确定特定问题是普遍存在还是局部存在。记录应该足够全面，以便您可以审计关注的领域，并运行根本原因分析以找到改进的方法。将您的最终用户及其设备转变为漫游传感器，描绘出可用性和性能的真实视图。

了解你的员工真正是如何工作的是无价的洞察力，可以简化许多情感对话。了解他们正在使用什么应用程序和设备，以及他们如何使用这些应用程序和设备，可以为您的不同业务领域带来许多潜在优势。您可以确保降低成本，维护法规遵从性，同时最大限度地提高灵活性。对风险和生产力的全新认识将帮助你做出明智的商业决策。

最终用户最了解
软件质量的唯一真正衡量标准是最终用户的满意度。忘记应用程序被设计用来做什么；忘记开发者和服务提供商不包括最终用户分析策略的承诺。切入正题，看看到底发生了什么；分析每一个用户会话，并利用商业智能来提取将产生切实影响的可行见解。

为用户提供他们想要的灵活性和尽可能做好工作所需的工具。快乐和高效的员工是一个好企业的基础。寻找提供透明最终用户分析的服务提供商；这将建立一个信任的基础，你们可以根据实际表现共同提供指导。专注于启动时间、应用程序性能或网络速度:您决定对您的业务重要的指标和对您的客户有影响的指标。

不经过分析就采取行动，并相信 SLA，这是将公司的未来及其未来道路建立在纯粹的猜测之上。借助终端用户分析，您可以确切了解实际需要什么，并确保您走在正确的轨道上。
