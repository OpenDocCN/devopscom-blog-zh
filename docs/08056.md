# 学习 DevOps 的“第二种方式”——持续反馈

> 原文：<https://devops.com/learning-the-second-way-of-devops-continuous-feedback/>

正如我在之前的博客[学习开发运维的“第一种方式”——连续流](https://devops.com/learning-the-first-way-of-devops-continuous-flow/)中所指出的，许多组织都在努力实现设计良好的开发运维，甚至是开发运维的“第一种方式”——连续流。

DevOps 的第一种方式，持续流，是 DevOps 的基础。一旦掌握了，一个组织自然会希望掌握 DevOps 的第二种方式，以获得持续反馈在设计良好时提供的更大好处。DevOps 是一个不断发展的知识体系。没有涵盖开发运维所需的所有角色和全部技能的一站式商店。具备开发运维能力的员工队伍需要持续致力于开发运维技能的培养和掌握。企业必须战略性地开展 DevOps 培训，以最大限度地利用培训资源。

### 学习持续反馈

没有专门针对开发运维第二种方式(持续反馈)的培训资源。不过，许多用于学习 DevOps 第一种方法的资源都有一些相关信息。DevOps 监控、发布自动化和安全工具出版物、文章和工具供应商是很好的来源。拥有成熟开发运维经验的开发运维专家可以选择与企业使用的特定持续反馈工具和反馈指标相关的培训信息。

### 为什么这对工程开发很重要

持续的反馈需要对 DevOps 概念和实践有成熟的理解。实施和操作应用程序性能、数据库性能、高级连续测试、高级发布管理、基础设施性能和连续安全性的高级监控工具和实践需要特定的技能。

### 学习持续反馈是如何完成的？

以下是学习持续反馈的推荐工程实践。

角色和职责被分配给专家，以定义 DevOps 持续反馈的培训计划(例如，DevOps sensei、培训 PM、布道者)。定期进行持续反馈 DevOps 技能差距评估。开发和运营培训战略旨在解决组织的差距和优先事项，以获得开发运维的持续反馈。培训预算包括每个角色的费用分配和时间分配，以支持持续的反馈。为每个角色定义了持续反馈目标和 SLA。

招聘职位描述符合与持续反馈相关的 DevOps 技能的技能要求。培训资源可供需要持续反馈培训的人员使用。为团队和个人设定的培训目标包括持续反馈的目标。激励和认可计划支持持续的反馈培训。持续反馈培训进度被跟踪且可见。

### 实施学习需要什么？

以下是实施持续反馈学习计划的推荐工程实践。

为专家分配角色和职责，以定义 DevOps 持续反馈的培训计划(例如，DevOps sensei、培训 PM、布道者)。进行持续反馈 DevOps 技能差距评估。定义一个最能弥补差距、满足组织需求和优先事项的培训策略。定义培训预算，包括每个角色和技能的费用和时间分配。

确定哪些技能和角色可以被廉价的自学资源所涵盖。确定哪些技能和角色需要更昂贵的商业培训，确定每个角色的培训目标。确保招聘职位符合与组织战略目标最相关的开发运维技能的技能要求。

将使培训资源可见和可用的方法落实到位。为团队和个人设定培训目标。为培训制定激励和认可计划。跟踪培训进度。

### 这意味着什么

要实现开发运维第二种方式的优势——持续反馈需要致力于持续学习和掌握超出开发运维第一种方式——持续流动所需的技能。实施和操作应用程序性能、数据库性能、高级连续测试、高级发布管理、基础设施性能和连续安全性的高级监控工具和实践需要特定的技能。组织需要一支训练有素的员工队伍来成功实施开发运维。组织必须战略性地持续开展开发运维培训，以最大限度地利用培训资源。要了解更多关于我学习所有 DevOps 的蓝图，请参考我的书 [Engineering DevOps。](http://mybook.to/engineeringdevops)
