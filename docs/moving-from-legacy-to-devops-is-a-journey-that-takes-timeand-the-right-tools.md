# 从传统迁移到开发运维是一个需要时间和合适工具的旅程

> 原文：<https://devops.com/moving-from-legacy-to-devops-is-a-journey-that-takes-timeand-the-right-tools/>

如果你明天想开一家新公司，那么很容易看到技术领域和流行趋势，并决定从一开始就围绕 DevOps 文化构建一切。然而，世界上大多数公司已经存在，他们已经有了流程和工具。从 legacy 迁移到 DevOps 不仅仅是一个瞬间的决定，而是一个长期的艰难旅程。

这就是 CloudBolt 的用武之地。上周，我有机会与 CloudBolt 首席执行官 Jon Mittelhauser 和 CloudBolt 首席技术官 Bernard Sanders 谈论该公司、其愿景和 CloudBolt 平台的最新版本。Mittelhauser 解释说，客户欣赏 CloudBolt 的地方在于它能够按照自己的时间表从过去过渡到未来。

改变的一个问题是它需要时间。虽然你要采用的新流程或工具可能在各方面都更好，但实现它可能是一个挑战。与此同时，您不得不管理和维护不同的系统或平台，无论是旧的还是新的。CloudBolt 希望通过提供一个单一平台来帮助组织减少混乱，使客户能够管理上述所有内容。

特别是大型企业不仅仅是集体采用一项新技术。一个站点或部门可能出于需要开始使用 DevOps 工具和流程。另一个部门也可能采用 DevOps，但是选择不同的平台和工具。Mittenhauser 解释说，即使整个组织在使用哪些 DevOps 工具和流程上意见一致，过渡也会随着时间的推移而缓慢进行，通常是由于许可证到期或平台达到其支持生命周期的尽头而造成的损耗。在缓慢的发展过程中，这些客户仍然需要一种方式来查看和管理他们的整个生态系统。

CloudBolt 还分享了其最新客户之一——伦敦卡姆登区——已经采用了 CloudBolt，以更好地了解和控制其庞大的虚拟环境。“在政府紧缩时期，为我们的客户运行低成本、灵活的混合云服务至关重要。伦敦卡姆登区助理总监(ICT)约翰·杰克逊表示:“CloudBolt 提供的服务使我们的客户能够按需获得所需的计算能力，从而提高他们的工作效率并降低我们的管理费用。

从许多方面来说，在整个组织中改变方式并接受 DevOps 文化会更容易，但是对于大型企业，甚至一些中型或小型组织，这是不现实的。对于在过渡期间维护多种技术和环境的组织来说，像 CloudBolt 这样的工具可以帮助 IT 有效地监控和管理一切。
