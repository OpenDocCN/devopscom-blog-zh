# 低代码自动化软件测试驱动开发运维

> 原文:[https://devo PS . com/low-code-automated-software-testing-drives-devo PS/](https://devops.com/low-code-automated-software-testing-drives-devops/)

DevOps 世界已经悄悄地将焦点转向解决一个基本的挑战:使软件团队能够快速创新，同时满足客户对质量的高期望。换句话说，以更少的错误更快地构建有用的东西。这种紧张局势多年来一直在加剧，但 2020 年为云、SaaS 和 DevOps 带来的持续中断增加了远程工作和加速数字化转型的挑战。2021 年，我们需要一个路线图，通过质量工程将[测试自动化](https://devops.com/?s=test%20automation)提升到下一个级别，质量工程是一个将最佳实践和数据驱动测试融入端到端客户体验并推动组织增长的学科。

## 重新思考 DevOps 时代的质量流程

质量工程(QE)一直缺乏跟上数字化转型步伐所需的工具。许多 QE 团队与复杂、脆弱的自动化框架和缓慢的手动测试作斗争，这对 DevOps 的采用造成了严重的瓶颈。随着开发的加速，QE 留下的是不同的或过时的工作流和工具集。

迎接 DevOps 和数字化转型带来的挑战和机遇需要我们在质量方法上进行创新。我们比以往任何时候都更需要将我们的关注点从作为软件开发生命周期(SDLC)后期执行的任务的质量保证转移到贯穿整个生命周期的质量工程实践。它从广泛、有效的测试覆盖开始，从您的开发人员在本地分支工作开始，一直持续到在应用的整个表面区域(web、API、移动、电子邮件、PDF 或其他)的生产中确认高质量的客户体验。也深入；您评估质量的所有方面，确保您满足与功能正确性、可访问性、性能、用户体验等相关的需求。

## 低代码测试自动化开启了质量文化

有了低代码，真正有效的测试在每个角色的指尖，从手工测试人员到开发人员，自动化工程师，产品所有者和客户支持。要求所有这些角色开发专门的编程技能是不切实际的，因此低代码工具通过使每个人都能够参与创作测试、更新测试以及从测试结果中获得洞察力来实现质量工程的民主化。

低代码测试给质量工程带来了几个好处，特别是在从手工测试到自动化测试的过渡中。虽然传统上这个过程会造成严重的破坏，但低代码平台正在铺平道路。在他们对 2021 年 IT 自动化的预测中，Gartner 预计到 2024 年，自动化能力的提高将使 30%的 IT 运营工作从支持转向持续工程。对于手工测试团队来说，这种转变可能会引发人才的变化，以及一个漫长的试点项目、再培训和逐步推广的过程。低代码是管理这种转变、提高工人技能和确保质量的关键。

## 现代自动化解决方案拥抱快速变化

传统测试自动化工具的一个关键挑战是，它们是为不经常改变的应用程序设计的，并且只打算验证代码。他们依靠静态选择器来识别页面元素。如果选择器(比如一个 ID)发生了变化或者对象(XPath)在网页上移动了，那么您的测试就失败了。随着越来越多的企业通过 DevOps、敏捷和 CI/CD 加速开发，测试维护的负担导致了与手工测试相同的瓶颈。

2020 年是低代码测试自动化的突破年。易于使用的新解决方案超越了主导该领域的传统框架的能力。这些解决方案结合了机器学习、数据分析和其他先进技术，大大减轻了测试维护的负担，克服了传统框架的测试刚性挑战。

通过人工智能和机器学习，测试自动化解决方案由测试人员训练，只需导航他们的产品。随着测试人员在共同的客户旅程中的移动，解决方案建立了对每个元素及其上下文的理解，这样，如果一个元素移动或者一个属性改变，包含该元素的测试将被更新，而无需人工干预。这大大减少了确保您的测试是最新的所需的维护量。当测试确实需要更新时，低代码模型使得任何人都可以很容易地参与进来并进行更新。展望未来，自动修复和低代码编辑对于支持高速团队中的质量工程是至关重要的。

## 测试整个客户体验

传统的通过/失败验证无法提供对用户旅程的真正洞察，而这是有效质量工程的关键步骤。下一代测试自动化平台将重点从验证代码扩展到验证整个用户体验。这从扩展测试开始，以覆盖非功能性需求，例如应用程序性能、可访问性和用户体验。例如，识别关键页面的加载时间趋势和监控视觉变化是自动完成的，对于管理客户体验至关重要。从那里，您可以将质量工作的优先级放在应用程序中使用最频繁或风险最高的组件上。这使您能够对测试覆盖范围做出数据驱动的决策——将您的测试工作集中在高优先级页面上，如电子商务网站的结帐页面，或者高风险页面，如具有从其他页面链接的多个交互元素的页面，如注册表单。

但是，即使是只关注应用程序或网站 UI 的完全数据驱动的测试策略也不太可能覆盖完整的端到端用户旅程，从而导致测试覆盖缺口和客户的潜在痛点。例如，一个常见的用户旅程可能包括打开电子邮件、点击链接、查看网站和下载 PDF。传统工具的局限性使测试只能在 web 部分进行，而现代解决方案允许你验证整个用户或客户体验(CX):接收和打开电子邮件，点击链接，登录网站，下载 PDF 并验证其内容。甚至可以通过 API 在后端确认值，而无需编写任何代码。开发无缝监控、修复和管理扩展 CX 的工具是我们创新和客户成功的核心。

## 质量工程与您一样以用户为中心

质量不再是事后的想法，或者只是部署前检查的一个盒子；这是用户体验的重要部分，因此也是在每个使用软件为客户服务的行业中保持竞争优势的重要部分。测试自动化对于高速团队来说是至关重要的，他们竞相更快地创新，转向 DevOps，并在远程工作的同时构建高质量的产品。令人欣慰的是，随着 DevOps 的实施，越来越多的质量保证团队获得了一席之地，提升了质量作为一种战略的形象，并增加了对测试自动化解决方案的需求，以跟上 DevOps 的需求。

为了应对这种需求，新一代智能测试自动化解决方案正在使团队能够通过采用低代码和机器学习来保持质量，从而迎接挑战。为了在这种环境中建立质量工程文化，您的团队需要像您一样以用户为中心的测试自动化解决方案:覆盖整个用户旅程的解决方案。解决方案不仅能验证功能的正确性，还能对质量的各个方面提供丰富的见解，并实现持续改进。也许最重要的是，您需要让团队中的每个人都参与到质量中来的解决方案。这确保了整个 SDLC 积极参与构建用户友好的产品，继续以 DevOps 的速度设定质量标准。
