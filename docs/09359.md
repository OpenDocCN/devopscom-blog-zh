# OCI 在 KubeCon + CloudNativeCon 欧洲

> 原文:[https://devops.com/oci-at-kubecon-cloudnativecon-europe/](https://devops.com/oci-at-kubecon-cloudnativecon-europe/)

我很高兴甲骨文云基础设施(OCI)赞助并参加了今年在西班牙瓦伦西亚举办的 Kubecon + CloudNativeCon EU。在那里，OCI 团队可以回答您关于应用程序现代化的任何问题，并演示客户如何使用云服务以最具成本效益的方式构建和管理现代应用程序。

Oracle 是云原生计算基金会(CNCF)的白金会员，致力于开源社区和简化云原生应用开发。OCI 云原生服务基于 CNCF 开源标准构建，包括 100%上游兼容的 Kubernetes、CloudEvents 等。

Oracle 继续创新并提供其他功能，使客户能够更轻松地大规模构建和运行任务关键型现代应用。我们最近发布了以下产品:

**OCI 服务网**

OCI 服务网格是一项免费的托管服务，为您的微服务提供安全性、可观察性和网络流量管理，而无需对应用程序进行任何更改。这种完全托管的服务使任何规模的组织都可以轻松实施这些功能。OCI 服务网格提供了一个高弹性的中央管理层和一个代理组件，支持微服务和管理层之间的通信。

该代理基于广泛使用的开源代理组件 Envoy。当您向应用程序添加网格时，它会自动在每个微服务前注入代理来管理流量。使用 OCI 服务网格的客户受益于开发人员生产力、运营效率、端到端可见性、安全工作负载以及微服务的动态连接的提高。

**OCI Kubernetes 集群 API 提供商**

Cluster API 是 Kubernetes 的子项目，主要提供声明性 API 和工具来简化多个 Kubernetes 集群的供应、升级和操作。对集群 API 的 OCI 支持使集群管理员能够更好地控制集群中安装的配置和软件，并标准化跨多个云和本地管理集群的方法。集群 API 与 OCI 服务集成，提供广泛的计算形态支持以及与网络和安全服务的集成。它还为第三方 Kubernetes 管理工具提供了一个集成点。

【Kubernetes 增强的安全特性

OCI 提供托管的 Kubernetes 服务，即用于 Kubernetes 的 Oracle 容器引擎(OKE)。OKE 的构建考虑了安全性、灵活性和高性能。OKE 使您的基础设施和应用程序更容易实现端到端安全，同时符合安全技术实施指南(STIGs)和法规遵从性。

客户可以在完全私有的集群中使用 OKE，将 Kubernetes API 端点限制在本地网络或堡垒主机上。他们还可以使用网络安全规则强化网络访问，并使用与 OCI 身份和访问管理(IAM)服务集成的基于角色的访问控制(RBAC)来控制对集群的访问。

OKE 上的工作负载受益于持续的安全评估和问题补救、容器映像漏洞扫描、映像签名和验证以确保应用程序映像的完整性，以及审计日志以检测异常活动。默认情况下，OKE 工作负载和连接的存储卷始终是加密的。OKE 增加了对使用客户管理的密钥来加密静态和传输中的启动卷和持久卷的支持。

**自定义 Kubernetes 工作节点启动**

OKE 通过定制 cloud-init 脚本，对运行 Kubernetes 工作负载的工作节点主机的配置提供了更好的控制。这种控制使客户能够为安全性和合规性配置安全性增强的 Linux 策略，配置公司代理，安装强制的防病毒软件和安全工具，或者配置定制的 yum 代理。客户还可以通过 kubelet-extra-args 来配置污点，以控制 pod 调度、分配 IP 地址或运行 OKE 节点医生来确认工作节点的行为符合预期。

**为函数**提供并发

OCI 客户可以使用 Oracle 功能创建、运行和扩展应用程序，而无需管理任何基础架构。Functions 为您的应用程序提供了一个安全的执行环境，包括代码隔离、安全认证、细粒度访问控制以及图像扫描和签名验证。客户直接调用功能，使用 API Gateway 将它们公开为 API，或者基于 CloudEvents、通知、日志或流服务来触发它们。

构建需要即时响应的应用程序的客户现在可以通过配置供应并发来避免函数的初始启动时间。此功能使您能够通过提前预热功能和配置基础架构来获得一致的亚秒级延迟。
