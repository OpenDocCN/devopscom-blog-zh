# DevOps Chat:与 Diamanti 的 Mark Balch 一起为容器化应用专门构建的设备

> 原文：<https://devops.com/devops-chat-purpose-built-appliances-containers-diamantis-mark-balch/>

在我还安全的时候，构建专用的安全设备风靡一时。然而，随着时间的推移，安全即服务占据了大部分市场(尽管安全设备仍然是一个大卖家)。对于容器化的应用程序，您可能认为云是一条出路。但是迪亚曼蒂公司的人说，为集装箱化应用专门制造的设备是一个更好的选择。

我与他们的产品和营销副总裁马克·鲍尔奇就此进行了交谈，他给了我们迪亚曼蒂的观点，坦率地说，这是一个很好的例子。它还处于容器化应用程序市场的早期，胜利者尚未加冕。马克讲得比我好。听一听，让我们知道你的想法。

简单介绍一下背景或者我们为什么在这里。我们目睹了容器革命的发生。我们相信，容器将成为新的云原生应用的下一个主流应用平台。我们已经看到，用户正在花费大量的时间和金钱，试图定制他们自己的基础架构，并找出如何让云原生应用程序大规模运行，同时保持可靠性和性能。我们说，“你知道，我们希望确保用户可以开箱即用，非常简单地做到这一点，控制他们的成本，并能够将他们的时间集中在构建独特的业务应用程序上，而不是重新发明轮子和基础架构。”这就是我们在这里的原因。

**希梅尔:**优秀。Mark，你知道，有趣的是，在我们看到软件定义一切的时候，Diamanti 推出了一款专为容器化应用程序打造的设备。我可以想象有很多人会说，“哦，不，我们不想卖硬件。”但是，真的，如果你想控制你的环境，你没有太多的选择。

**Balch:** 对许多用户来说，这是一个非常棘手的情况，因为只要去那里，花一点时间在网上搜索，去参加聚会，看看用户的评价，就可以知道在传统模式中以合理的成本获得可靠性能的挑战。真的很难。人们花费大量时间进行定制工程，而这些时间是他们无法用来构建自己独特的服务的。因此，对我们来说，这真的是一个非常简单的价值主张。

我们不是说你必须定制你的应用程序或者做任何专有的事情；我们要说的是，“继续使用您的开源方法—标准 Docker 映像、标准 Kubernetes 流程编排、标准 Linux—但是在一个平台上运行，该平台将为您提供正确部署整个应用程序的自动化，为您提供可预测的性能和结果，并让您以合理的成本运行。”我的意思是，无论是公共云，还是为不同类型的应用程序构建的虚拟化传统基础架构，人们都有巨额账单。人们花费大量的时间和金钱来尝试让云原生应用程序工作，我们的目标是确保人们可以在几分钟内，而不是几周或几个月内让这些应用程序工作。

从治理的角度来看，Kubernetes 是一个很好的生态系统。显然，谷歌去年向 Linux 基金会贡献了这些代码，你已经得到了来自整个生态系统的大大小小的贡献者，包括 Diamanti，为使 Kubernetes 变得更大、更好和更具可扩展性做出了贡献。所以早期吸引我们与 Kubernetes 合作的是治理，在治理中，任何人都可以参与进来。坦率地说，我们欢迎每个人都参与进来并承担领导角色。例如，Diamanti 向 Kubernetes 贡献了一个名为“FlexVolume”的东西。它在上游；那是一年前的事了。我们受到了热烈的欢迎，现在，实际上，有 20 多个存储 SIG 成员与我们合作。Kubernetes 的日程安排也是如此——我们在那里做上游贡献。与 CNI 合作，集装箱网络接口，是另一个例子。

因此，我们刚刚看到了许多优秀的治理和可扩展性，Kubernetes 社区似乎非常关注可扩展性的概念，这并不是说我的解决方案比你的好；我们有 API，可以用来扩展不同的选项，然后用户可以挑选适合他们的选项，但它受一个通用框架的支持。这就是我们喜欢 Kubernetes 的地方。
