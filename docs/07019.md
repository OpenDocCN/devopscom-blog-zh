# 思科 CEO:开发者需要更多的基础设施控制

> 原文:[https://devo PS . com/Cisco-CEO-developers-need-more-infra structure-control/](https://devops.com/cisco-ceo-developers-need-more-infrastructure-control/)

思科系统公司首席执行官罗卓克本周表示，该公司未来的首要任务是让 IT 组织通过抽象层更好地管理应用程序体验，使开发人员能够以编程方式管理基础设施。

罗宾斯在网上举行的年度思科合作伙伴峰会上表示，思科正在努力使开发者不仅能够以编程方式提供 it 基础设施，而且能够影响使用公司目前正在开发的应用网络服务交付服务的方式。

他说，随着时间的推移，随着思科越来越多地提供其技术即服务，整个思科产品组合将要么是云交付的，要么是云管理的。作为这一努力的一部分，思科现在在其产品组合中公开应用编程接口(API)。

该计划的核心是思科对 AppDynamics、ThousandEyes 及其 Intersight IT 运营管理平台的投资。Robbins 说，这些工具将被用来提供对多样化应用环境的洞察力和可观察性，这种环境越来越多地包括在 Kubernetes 集群上运行的云原生应用。

实际上，思科正在发出一个更积极的信号，将基础设施作为云内外的代码进行管理。本周，思科宣布[增加了 Intersight 工作负载优化器](https://newsroom.cisco.com/press-release-content?type=webcontent&articleId=2105978)来优化基础设施消耗，并通过 Cisco Intersight 增加了对 Kubernetes 集群动态配置的支持。该公司还增加了一个思科 Nexus 仪表板，以提供对跨多个云、内部 IT 环境和边缘计算平台的网络的可见性，并扩展了思科身份服务引擎，以保护更广泛的物联网(IoT)终端。

思科和其他所有 IT 基础设施提供商一样，正在向云运营模式转变，该模式承诺让 IT 团队能够以与 IT 团队消耗公共云资源相同的方式管理内部 IT 基础设施。作为这一努力的一部分，Robbins 指出，思科一直在尽可能地将其提供的所有软件功能从硬件中分离出来，这将为 it 团队提供更多的灵活性和对高度分布式 IT 环境的控制。

他补充说，新冠肺炎疫情只是加速了向管理 IT 即服务的过渡，因为组织更加依赖应用程序来吸引他们的最终客户。

当然，思科并不是唯一一家有类似雄心的 IT 基础设施提供商。不仅网络软件变得可以通过 API 访问，而且现在几乎可以在任何地方的商用处理器上部署软件定义的网络。这种转变也是网络运营越来越多地被纳入开发运维团队的主要原因，开发运维团队寻求将计算、存储和网络作为代码在整个扩展企业中进行管理。

网络和存储操作完全融入开发运维流程可能还需要一段时间。然而，现在的问题不再是大多数 it 组织是否以及需要多长时间才能实现这一目标。
