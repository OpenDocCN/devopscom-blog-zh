# 报告:云的状态-本地应用程序安全性

> 原文:[https://devo PS . com/report-the-state-of-cloud-native-application-security/](https://devops.com/report-the-state-of-cloud-native-application-security/)

云在增加部署流动性和自动化方面带来了巨大的能力。随着云的采用，专门为该领域开发应用程序而构建的云原生工具得到了使用。然而，云原生工具带来了一些微妙的安全问题，例如错误配置、已知的漏洞和泄露的秘密。因此，根据 Snyk 最近的一份报告，83%的组织认为安全性对其云原生战略非常重要。

Snyk 的[云原生应用安全状态](https://snyk.io/state-of-cloud-native-application-security/)报告调查了数百名 IT 专业人员对云原生安全的担忧。下面，我们将从报告中挑选出最重要的要点，确定云原生应用程序安全性最常见的问题领域，并了解部署自动化和更高程度的应用程序安全性之间是否存在关联。

## 向云原生迁移

团队正在转向云原生技术，通过软件驱动的架构和基础设施即代码(IaC)来支持开发运维。在这种新的云原生模式中，58%的生产工作负载被部署为容器，21%现在是无服务器的，50%在部署过程中使用某种形式的 IaC。

出于各种原因，组织正在转向云原生技术。首先，容器化的软件提供了更快的速度——研究发现，68%的组织转向容器以提高部署速度。其次是易于管理(67%)和降低成本(43%)。还有采用云原生工具的安全性要求；36%的受访者认为安全性是将生产应用程序迁移到容器的主要原因。

## 云原生安全问题

虽然诸如 containers、Kubernetes、serverless 和 IaC 等云原生技术能够实现更快速的开发，但它们也带来了独特的安全问题。报告发现，[错误配置](https://containerjournal.com/features/insecure-defaults-remain-a-threat-for-kubernetes/)和已知的[未修补漏洞](https://containerjournal.com/features/k8s-and-cloud-security-compliance-just-aint-enough/)是云原生环境中最常见的事件类型。事实上，45%的组织都经历过因配置不当而引发的事故，其次是 38%的组织因已知的未修补漏洞而引发的事故。

其他常见的云原生安全事件包括机密泄露、审计失败和恶意软件。有趣的是，该研究还发现，在云原生采用程度较高的组织中，内部人员的数据泄露更为常见。根据该报告，38%的云采用率较高的组织遭受过内部人员的数据泄露，而在云采用率较低的组织中，这一数字减少了一半，为 17%。像 API 密匙这样的秘密必须小心管理，特别是当[云原生工具](https://devops.com/11-open-source-devops-tools-we-love-for-2021/)强制更多的依赖性时。该报告指出，“对这些类型的工件进行有效管理的需求是与更集中的前云时代的一个关键区别”。

## 自动化安全测试

端到端的部署自动化显示出了希望，但是对于大多数开发团队来说，仍然处于成熟的早期阶段。尽管 95%的组织使用某种部署自动化，但只有大约三分之一的组织拥有完全自动化的部署管道。

随着部署自动化程度的提高，组织也倾向于接受更高程度的安全测试。该报告发现，高度自动化的管道在其整个开发生命周期中使用安全测试的可能性是普通管道的两倍。由于事件通常是由错误配置和已知漏洞引起的，自动扫描可以帮助识别许多云原生问题，将生产代码与已知漏洞的数据库进行比较。

那么，什么时候进行安全测试呢？这通常发生在 CI/CD 管道中。超过 60%的组织在 CI 系统中执行安全测试。这与在开发过程的早期测试源代码库或本地 ide 和 CLI 工具相反。就测试频率而言，执行范围很广。对于部署自动化程度较高的组，70%的人每天或更频繁地测试安全性。自动化安全测试似乎运行良好，因为 72%的全自动化团队在不到一周的时间内发现并修复了关键漏洞。

该报告还揭露了一个有趣的关于安全测试所有权的脱节。只有一小部分安全工程师(不到 10%)认为开发人员应对云原生环境和应用的安全性负责，而 36%的开发人员表示他们对安全性负责。这些数字可能表明安全责任向发展方面左移。或者，随着[全周期开发](https://containerjournal.com/features/can-great-dx-enable-full-cycle-development/)变得更加现实，它可能会强调态度的改变。无论如何，团队必须明确角色，以避免一些困难的对话！

## 云原生安全性

随着向云原生战略的转变，安全标准正在上升，以应对诸如错误配置等新问题。为了应对不断增加的应用威胁，提高自动化程度有助于将安全性向左转移——这份报告和其他报告清楚地表明了全自动部署管道和提高安全性测试制度之间的相关性。

由于采用了云原生工具，58%的组织在采用云原生工具后增加了安全顾虑。其中一部分包括在所有基础架构中采用零信任方法，无论是面向内部还是外部。该报告显示，58%的组织越来越担心配置错误，52%的组织越来越担心不安全的 API，43%的组织越来越担心已知的未打补丁的漏洞，而 41%的组织担心机密泄露。希望这些基准能帮助您了解您的组织与其他组织相比如何。
