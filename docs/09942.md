# 开放银行不应该是你的 API 的开放季节

> 原文：<https://devops.com/open-banking-shouldnt-be-open-season-on-your-apis/>

开放式银行已经成为金融服务应用的最新技术进步，推动了行业的现代化。例如， [万事达卡](https://www.mastercard.com/news/press/2022/june/mastercard-empowers-open-banking-startups-to-scale-and-expand-consumer-choice/) 最近开始了其 Start Path 开放式银行项目，该项目为精选的初创企业提供了利用万事达卡专业知识的机会，同时学习如何将开放式银行应用到自己的业务中。开放银行业务为金融机构及其合作伙伴之间的收入共享和降低数据许可和交易费用创造了新的机会。

利用开放式银行服务的客户还能对自己的财务状况有一个更全面、更全面的了解，这样他们就能更好地管理自己的财富。同样重要的是，开放式银行将提供更多的机会，在其组织提供的所有产品和忠诚度计划中使用数据。像 [万事达卡](https://b2b.mastercard.com/open-banking-solutions/) 这样的金融服务公司也相信开放的银行项目可以提高包容性和金融素养。有了更多的金融知识，客户可以努力减少他们账户中 [欺诈](https://thefinancialbrand.com/news/open-banking/how-open-banking-increases-data-value-for-banks-and-consumers-148755/) 的机会。

在通过新的创新金融技术将金融机构与消费者联系起来方面，这遵循了打破壁垒的总体趋势。事实上，[](https://www.ey.com/en_us/news/2019/06/ey-global-fintech-adoption-index-finds-over-half-64-of-global-consumers-use-fintech)全球 96%的消费者都知道 fintech 服务，包括至少一次转账和支付。此外，美国消费者正在稳步采用新的金融科技，包括点对点支付。

虽然开放式银行有可能动摇银行业的基础，但也有可能成为威胁者和网络罪犯的目标。原因是:支持开放银行业务的应用编程接口([API](https://devops.com/?s=APIs))正成为攻击目标。

## 与开放银行业务相关的危险

开放式银行使用 API 将金融机构与第三方应用程序连接起来。 [开放银行项目](https://www.openbankproject.com/) 是为银行业提供开源 API 的领导者之一，并依靠开源系统为金融服务提供商访问消费者银行和金融数据。

然而，金融服务组织越依赖 API，它们的攻击面就越大。API 作为当今现代 web 应用程序的[](https://www.forbes.com/sites/forbestechcouncil/2022/03/28/apis-are-a-building-block-dont-make-them-a-stumbling-block/?sh=51d46b265ac8)，已经成为威胁者的一大目标和主要攻击媒介。银行和整个金融服务业也是网络罪犯的目标。

IBM 的 [X-Force 威胁情报指数 2022](https://www.ibm.com/security/services?utm_content=SRCWW&p1=Search&p4=43700050370516405&p5=e&gclid=Cj0KCQjwxIOXBhCrARIsAL1QFCbZGPkCRV8dNbHqVWVPVmiArhEBSRYT7sejGw2mS_A3BVYkYBX5VFgaAuyJEALw_wcB&gclsrc=aw.ds) 指出，金融和保险机构是 [第二大](https://www.ibm.com/downloads/cas/ADLMYLAZ) 目标行业。此外，IBM 还发现，在这些攻击中，70%是针对银行的，14%是针对其他金融机构的。这告诉我们，虽然其他组织可能会成为数据泄露的头条新闻，但银行和金融机构最常受到威胁者和网络罪犯的攻击。虽然其中一个主要驱动因素是财务收益，但银行业也有大量数据，包括个人身份信息(PII)、支付卡行业(PCI)数据、账户和支付信息。这其中的大部分可以在暗网上被利用来获取经济利益。

不幸的是，对金融机构和驱动我们参与的 API 的攻击不会很快改变。与 API 相关的问题是，如果不加以保护，它们会为威胁者提供一个可以利用敏感数据的立足点。此外，许多组织并不真正了解他们有多少 API(许多是互连的)或者这些易受攻击的端点是如何受到保护的。保护你甚至不知道存在的东西是非常困难的！第三方 API 常常被依赖，这给 API 难题又增加了一层复杂性。使用这些来源虽然可能是一种节省时间的策略，但可能会使您的网络容易受到攻击。

## 安全开放银行意味着受保护的 API

如果您是一家计划开始利用开放银行业务的公司，那么通过建立完善的 API 保护策略来减轻这些威胁是非常重要的。首先，获得对整个攻击面的可见性。毕竟，了解你自己，包括你的 API 和 web 应用，以及了解你的敌人，将会在战场上取得成功(套用孙子的话)。一旦您对您的终端有了全面的了解，包括那些较旧的和取消供应的 API，您就可以更好地防范它们所面临的众多威胁。理想情况下，您将能够获得到达端点的流量的实时分析，因此您可以获得一个不断变化的表面的更全面的视图。

还应该强调的是，复杂的攻击由生成 API 流量和攻击的 bot 军队组成。恶意僵尸程序正越来越多地被用于攻击，包括 DDoS 攻击和账户接管。虽然许多 bot 攻击是自动进行的，但如果能够实时检测并阻止它们，再加上持续监控，就有助于减轻 bot 带来的威胁。

开放银行业的未来是一个令人兴奋的前景，因为它将把金融控制权交还给企业和消费者。虽然在许多以 API 为中心的开放式银行解决方案中应该有一定的谨慎，但是有一些方法可以减轻这些担忧。强大的 API 安全性，以及 bot 阻止和检测功能，对于防范现代威胁大有帮助。
