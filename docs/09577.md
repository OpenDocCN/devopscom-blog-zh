# Scribe Security 推出一对工具来保护软件供应链

> 原文:[https://devo PS . com/scribe-security-unveils-pair-of-tools-to-secure-software-supply-chains/](https://devops.com/scribe-security-unveils-pair-of-tools-to-secure-software-supply-chains/)

Scribe Security 今天发布了一个 Scribe Integrity 工具，该工具可以扫描软件工件，以确保它们在集成到应用程序之前符合 IT 组织的安全策略。

Scribe Integrity 工具在开放源代码和专有源代码上传到构建之前对其进行身份验证。Scribe Security 的首席执行官露比·阿贝尔说，它假设所有的文物都是“有罪的”，直到他们能够证明自己的清白。他补充说，这种方法可以确保整个软件供应链的完整性，而不会对开发人员的生产力产生负面影响。

此外，该公司推出了 GitGat，这是一款基于开放策略代理(OPA)代理软件的开源策略代码工具，使 DevOps 团队能够定期运行报告，深入了解驻留在 GitHub 存储库中的代码的安全状况。

Arbel 说，随着时间的推移，GitGat 的覆盖范围将会扩大，以增加对其他持续集成/持续交付(CI/CD)平台的支持。

![](../Images/b0731b8cb1b540072df32e1aac17943f.png)

Scribe Integrity 的第一个版本解决了 Node.js 代码和 npm 包管理器的问题，并计划支持其他类型的代码。

他指出，Scribe Integrity 工具还可以识别所有依赖关系，使 DevOps 团队能够在应用程序中包含每个软件工件时生成准确的软件材料清单(SBOM)。Arbel 指出，这一点至关重要，因为它使开发人员、it 运营和网络安全团队能够同时看到哪些构件(包括容器)构成了一个应用程序。他补充说，未来，该公司计划推出一个 Scribe Hub，这将使分享对这些软件工件的见解变得更加容易。

一系列引人注目的安全漏洞清楚地表明了网络犯罪分子在将恶意软件注入软件工件中并破坏将该工件整合到应用程序中的任何应用程序的技能。该恶意软件随后可能会被激活，从而潜在地危害任意数量的下游应用程序。

这些事件使 DevSecOps 保持软件供应链完整性的最佳实践得到了更多的重视。DevOps 团队试图解决的问题是如何构建更安全的应用程序，而不降低这些应用程序的构建和部署速度。因此，DevOps 团队正在向应用程序开发流程添加工具，使开发人员能够更轻松地在代码被包含到应用程序中之前扫描代码，并验证成为 DevOps 工作流一部分的任何软件组件的完整性。

还不知道采用 DevSecOps 最佳实践对应用程序安全性产生有意义的影响需要多长时间。然而，等到应用程序部署完成后再关注安全性就太晚了。如今，网络犯罪分子可以在几分钟内发现应用程序中的缺陷和错误配置。随着更多应用程序的部署，开发人员会发现他们花在修复漏洞上的时间比写新代码的时间还要多。显然需要一种新的方法来构建从一开始就是安全的应用程序。
