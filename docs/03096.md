# 微软将升级计划放入 DevOps Motion

> 原文:[https://devo PS . com/Microsoft-puts-upgrade-plan-in-devo PS-motion/](https://devops.com/microsoft-puts-upgrade-plan-in-devops-motion/)

微软本周在其 [Ignite 2018](https://www.microsoft.com/en-us/ignite) 大会上明确表示，今年秋天，DevOps 团队将忙于升级数据库和操作系统。

Windows Server 公司副总裁 Erin Chapple 宣布 [Windows Server 2019](https://cloudblogs.microsoft.com/windowsserver/2018/09/24/windows-server-2019-announcing-general-availability-in-october/) 将于下个月正式上市。与 Windows Server 2016 一样，有三个版本:标准版、数据中心版和基础版。后一种服务面向小型企业。

然而，该公司为其操作系统提供的管理工具套件要到明年初才能上市。

微软正在对[Windows Server Software Defined](https://www.microsoft.com/en-us/cloud-platform/software-defined-datacenter)(WSSD)2019 进行收尾工作，该软件提供了另一种在超融合基础设施平台上结合计算、存储和网络的方法。该产品预计将于明年年初捆绑在各种硬件平台上。希望从传统的基于机架的数据中心管理方法转变为更现代的 HCI 方法的组织必须等待 WSSD 2019 的到来。

该公司同时还在推进微软 SQL Server 2019 的[公开预览。除了增加对更快查询和内存中持久存储的支持，微软 SQL Server 2019 还将 Hadoop 分布式文件系统和 Apache Spark 内存计算框架与关系数据库打包在一起。微软还使在本地或云中部署的 Linux 或 Windows 上运行的 Kubernetes 集群上运行 SQL Server 2019 成为可能。虽然](https://cloudblogs.microsoft.com/sqlserver/2018/09/24/sql-server-2019-preview-combines-sql-server-and-apache-spark-to-create-a-unified-data-platform/)[公司已经广泛采用了 Kubernetes](https://containerjournal.com/2018/09/27/microsoft-further-embraces-kubernetes-and-containers/) ，但其在 Windows Server 上对 Kubernetes 的支持预计要到 2019 年才能普遍提供。

微软还宣布，Azure SQL 数据库管理实例的[通用服务层将于下周推出。该功能为开发运维团队提供了在公共云中或内部部署 SQL Server 数据库的选项。](https://azure.microsoft.com/en-us/blog/azure-sql-database-managed-instance-general-purpose-tier-general-availability/)

微软鼓吹在内部和云中运行 Windows Server 的能力是其混合云计算战略的核心要素。然而，目前在 Azure 云上运行的最广泛使用的平台是 Linux。该公司还在推动另一种基于微软 Azure Stack 的混合云计算方法，这是该公司用来驱动其公共云的云框架的内部版本。Kubernetes 的一个测试版也可以在微软 Azure Stack 上获得。

本月早些时候，微软[改进了其 AzureOps 工具](https://devops.com/microsoft-revamps-approach-to-devops-on-azure/),使其更容易管理其公共云上的所有变化。但是这些工具尚未扩展到内部环境。

当然，没有要求说任何使用微软技术的混合云方法都需要 Azure。Windows Server 和 SQL Server 都已经广泛部署在多个公共云中。DevOps 团队将面临的挑战是管理所有不同版本的 Microsoft 操作系统和数据库，这些操作系统和数据库可能会分散在整个扩展企业中。任何组织都不可能一次性升级操作系统和数据库。
