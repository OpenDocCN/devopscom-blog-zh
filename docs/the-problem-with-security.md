# 安全问题

> 原文：<https://devops.com/the-problem-with-security/>

作为一名在各个技术和组织级别(从手机原型到银行企业架构，从入门级到 CTO)都有数十年经验的 IT 专业人员，我可以肯定地说，我已经花了太多的时间来思考和处理安全问题。我知道我并不孤单。这是一个常见的主题。你几乎可以把它拍成一部情景喜剧。“你们开发商在干嘛？”嗯……写代码？“那就是安全！”我可以继续下去，从用户到云实例，你可以填写“这就是安全！”每次都妙语连珠。

这是个问题。但不是你想的那样。很多年来，我一直认为自己有些与众不同，因为我总能随机应变，直面安全问题。我工作过的大多数组织都没有专门的安全部门(我可能以前写过这方面的文章)，所以我们中的一些人挺身而出。离全部还差得很远，但足以让我们觉得受到了保护。

我现在意识到我并不那么独特；我们中有更多的人在做“为了安全而学习安全”的事情。我一直在现代组织中看到它。我认为真正的问题是:这些人中的大多数，像我一样，并不认为自己是安全人员——但他们是。大多数组织不承认他们是安全人员，但他们是。

我们长期以来一直在从这个人才库中培养安全人员，现在我建议你更进一步。让这些人成为你计划的一部分。如果你知道 ReallyCoolApp 团队中的某个人正在搜寻安全资料，请他们每周发一封电子邮件来更新 InfoSec。不要让它成为一个巨大的负担，如果他们没有表现出资质和兴趣(你真的不希望不愿意的安全人员)而是利用他们的才能和动力，不要试图强迫他们成为一个纯粹的安全角色。当然，如果这个人做得很好，确保 CoolApp 基本上被锁定，并且 DevSecOps 工具正在对其进行二次验证，那么您有限的 InfoSec 资源可以被引导到其他地方。

[在整个组织中复制这一点](https://securityboulevard.com/?s=security+culture),“这就是安全！”成为一种优势，而不是试图锁定的麻烦。您已经做了一些工作—由于密码自动化，大部分身份管理由运营人员处理，安全性仅提供策略，并在需要时可供运营人员使用。我只是建议您将这种方法扩展到包括每个 IT 部门(以及分布式环境中的一些非 IT 部门)。不要招募每个人都成为信息安全专家，而是招募他们在自己的专业领域做一点信息安全工作。

人们总是说“安全是每个人的问题”，这是真的，但这太模糊了，没有用。这是一个可行的中间体。"[安全是每个团队的问题](https://devops.com/?s=DevSecOps)"招募每个团队中对 InfoSec 最感兴趣的人来正式确定他们的参与是一个更好的方法。

继续摇摆。我们在任何地方都需要安全，因为“不做井”使任何地方都成为信息安全战场的一部分。因此，到处招聘员工，但不要让他们离开当前的角色/职责。不要让你多年来精心培育的甜蜜系统被攻击者搞得焦头烂额，这时一点点招募和培养就能填补缺口。
