# Oracle 支持合作伙伴重用云平台

> 原文:[https://devo PS . com/Oracle-enables-partners-to-reuse-cloud-platform/](https://devops.com/oracle-enables-partners-to-reuse-cloud-platform/)

在其 [CloudWorld](https://www.oracle.com/cloudworld/) 活动上，甲骨文今天宣布，它将使 it 团队能够部署相同的平台，用于在他们认为合适的任何地方独立提供[云服务](https://devops.com/?s=cloud+services)。

甲骨文产品和战略副总裁 Leo Leung 表示，Oracle Alloy 将使组织能够提供一整套他们可以扩展并作为自己品牌的云服务。他说，该公司在甲骨文云上提供的 100 多种基础设施和平台服务现在可以部署在任何地方。

此外，[甲骨文推出了 MySQL HeatWave 服务](https://www.prnewswire.com/news-releases/oracle-distributed-cloud-delivers-more-choices-to-customers-worldwide-301651823.html)，IT 团队可以通过它访问运行在亚马逊网络服务(AWS)云上的开源 MySQL 数据库实例。微软 Azure 云的类似产品也在计划之中。

作为与竞争对手云服务提供商争夺市场份额的持续努力的一部分，甲骨文一直在以更低的成本提供广泛的服务。Oracle Alloy 现在扩展了这些努力，使包括传统企业 it 组织和互联网服务提供商在内的众多组织能够从公司的研发工作中受益。

与 Oracle 结盟的组织可以设置自己的定价、费率卡、账户类型和折扣表，并定义支持结构和服务级别。还提供了一个预配置的 Oracle Fusion Financials 实例，使 Oracle Alloy 合作伙伴能够管理客户发票和账单。

目前还不清楚这些组织在多大程度上想要与亚马逊网络服务(AWS)、微软和谷歌等云服务提供商竞争，但 Oracle Alloy 确实提供了一种替代云计算的方法，使组织能够对他们提供给其他人的 It 服务保持更多的控制。当然，挑战在于获得基础设施只是等式的一部分。组织还需要雇佣和保留持续管理云平台所需的 IT 专家。

随着云计算环境中部署的应用数量稳步增长，Oracle 显然将 Oracle Alloy 视为一个弥补细分市场失地的机会，十多年前云计算首次出现时，Oracle 几乎忽略了这一市场。现在，甲骨文在 22 个国家的 40 个商业和政府区域提供云服务。在过去的一年中，它增加了 10 个公共云区域，并计划再增加 6 个商业区域。

甲骨文认为，在某些情况下，合作伙伴会选择自己部署一些云服务，而依靠甲骨文来部署其他服务，这取决于特定服务类别的需求水平。实际上，Oracle 正在支持一组联合云服务的出现，这些服务不再专门绑定到单一提供商。组织对这种方法的接受程度还有待观察，但可以肯定的是，有能力提供云服务的组织数量可能会大幅增加。至少从理论上讲，竞争的加剧也应该会降低使用这些服务的组织的云成本。
