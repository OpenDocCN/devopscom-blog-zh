## **什么是 CI/CD？**

每当开发人员提交变更并将其推送到存储库时，CI 系统将重建分支并运行所有相关的测试用例，以验证新的变更不会破坏现有的应用程序和功能。

持续交付是持续集成的一个扩展，帮助您通过点击一个按钮来部署您的应用程序。这意味着每次通过 CI 测试时都要部署变更。有了它，您可以轻松地自动化您的构建和测试阶段，以及您的大部分发布过程。

通过持续部署，开发人员将能够在生产环境中自动发布或部署软件。在这个阶段，没有手动代码签入到应用程序中。通过 CI 流程自动测试代码是否有问题，如果没有发现问题，则部署代码。

## **为什么选择 CI/CD？**

CI/CD 是测试和部署应用程序的最佳实践。出于质量保证的目的，使 CI/CD 成为部署代码的单一渠道是最佳实践。DevOps 团队应该确保一切都通过 CI/CD 管道，以便测试和生产中的代码是相同的。这也将减少人为错误，避免部署其他没有被错误测试的版本。

如今，公司实施 CI/CD 的最大原因之一是更快的软件交付，这导致了对客户更频繁的更新。这让他们感到高兴，频繁的发布带来的持续改进将确保提高客户满意度。这是一个竞争优势，因为它允许公司更频繁地提供新功能。它可以帮助公司领先于竞争对手。

通过 CI/CD 管道传递代码可以确保在进入交付管道的下一阶段之前，所做的任何更改都符合既定的标准。任何失败或差异都会阻止此类更改进入下一步，直到得到纠正。这仅仅意味着 CI/CD 的这种方法确保了坏代码不会进入产品。CI/CD 提倡快速失败、快速学习和快速交付的方法。当您有了自动化的 CI/CD 后，代码会随着开发人员的每次提交而不断地在 CI/CD 过程中运行，这样就可以快速地捕捉到 bug——⁠———隔离 bug 变得更加容易，因为每个小的变化都是单独测试的。

## **什么是 CI/CD 管道？**

CI/CD 管道是与自动化集成的部署管道工作流。当操作良好时，它将最小化手工移交和错误，以增强整个软件交付生命周期的反馈循环，允许开发团队在更短的时间内交付更小的发布块。

CI/CD 管道现在是软件公司的主流，它已经成为现代 DevOps 环境的支柱。DevOps 制定了与文化方面相关的原则和标准，CI/CD 则侧重于自动化、流程、过程和工具。随着容器化和微服务的引入，CI/CD 场景发生了变化。现在，每个云供应商都有 Docker 和 Kubernetes 等工具的支持和集成，帮助 CI/CD 管道扩展。

## 持续交付管道的各个阶段

*   测试自动化和 CI: 这是准备新特性和原型的重要阶段，编码完成，然后代码与基础集成。通过使用 CI 服务器，单元测试和打包就完成了。

*   **构建自动化:**这是开发人员将工件(即依赖项、Docker 映像)推送到注册表(即 Amazon ECR 或 Docker hub 或任何容器注册表)的阶段。

*   **Alpha 部署:**在这个阶段，开发人员签入变更。在这里，开发人员可以仔细查看他们所做的更改，以及这些更改是如何相互作用并影响其他系统的。此外，在这里开发人员可以看到他们是否破坏了主分支中的任何东西。

*   测试部署:这是测试人员执行手工测试的阶段。还是要手工测试才能看到 CI 工具的完善。这为 DevOps 团队提供了应用程序状况良好的保证，并且在部署时肯定会按预期工作。

*   **生产部署:**这是应用上线的最后阶段。

## **云和 DevOps 齐头并进**

没有云，你就无法进行开发运维。DevOps 没有它不会有太大的价值，云没有 DevOps 也不会有太大的作用。

云计算有许多定义，对云的基本解释可以定义为启用和支持基础设施供应的系统。因此，它允许将其描述为代码或模板，从而促进可重复过程的创建。 这也是 DevOps 的基本原则，最大程度地自动化软件开发生命周期中的过程/任务。

云对 CI/CD 的最大好处之一是云基础设施的动态性。这非常适合 CI/CD 工作负载的短暂性和突发性。云资源可以根据 CI/CD 工作负载自动扩展和缩减。这为企业带来了巨大的管理和成本效益。企业公司不需要有自己的服务器来维护，他们需要随着 CI/CD 工作负载的增加来增加这些服务器，或者在空闲时浪费服务器资源。

无论您的组织如何定义云技术，也无论您通过哪种 DevOps 途径来实现云集成的数字化转型，云技术和服务的美学都可以轻松融入 DevOps 字典，因为它们从根本上补充了 DevOps 的原则和流程。

对于 DevOps 的实施，敏捷原则是一个驱动因素，因为有更短的发布周期和自动化来快速交付和实现最后期限。为了实现持续的实践，云计算是一个很好的推动者，因为资源可以快速地用于开发和应用程序生命周期管理中涉及的所有活动。敏捷原则，加上云计算和 DevOps 实践，可以提高开发人员的工作效率，加快组织的发展。

借助安全的云网关，用户能够从任何地方使用任何设备轻松访问企业资源，而不会影响网络的安全性。让用户从他们的设备访问您的网络有时可能看起来很可怕，但现代云平台可以提供更多，它们带有内置的安全堆栈，可以帮助管理员提供他们管理和支持安全所需的工具。

DevOps 与云技术的结合为采用它们的企业带来了巨大的好处:

1.  提高开发人员的工作效率和协作。
2.  通过快速方便地访问开发环境，加快上市时间。
3.  帮助自动化不同的重复性任务，以便开发人员可以专注于重要的任务。
4.  随着代码的接近，自动化和基础设施可以极大地降低云复杂性和系统维护。
5.  借助现代云平台和消除手动错误的原则提高安全性。
6.  通过基于云的持续业务运营帮助消除停机时间。开发人员可以通过自动化构建无状态应用程序，这将提高可用性和故障转移能力，同时提高业务可靠性和客户满意度。
7.  扩展到新的高度—企业采用云和开发运维的主要原因之一是根据需要扩展其基础架构和应用。

## **云部署模式**

当云提供商完全托管组织的应用和数据时，它就是一个公共云。只要有互联网连接，采用公共云的公司员工就可以访问应用程序。

私有云是指存储、计算和网络资源等组织资源存在于提供商的数据中心。公司更喜欢私有云，主要是因为安全性以及在运行应用程序时存储高度敏感的数据。

一种更现代的方法是混合云；它是私有和公共云服务的结合。根据流量和需求考虑，组织可以进行更改。一些客户可能有本地 CI/CD 系统。其中一些 CI/CD 系统可以通过混合设置将其工作负载扩展到云中。这可以让他们立即认识到云的优势，而不必进行完整的迁移。

主要是，当有高需求时，工作负载可以在公共云中运行，然后当事情恢复正常时，返回到私有云。这种方法减少了花在云资源上的开销成本。此外，关键的一点是，敏感信息、数据和关键应用程序可以保存在私有云中。相比之下，不太敏感的数据和应用程序可以在公共云中存储和运行。
