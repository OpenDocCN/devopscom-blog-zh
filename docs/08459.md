# 开发者称数字领导力的缺乏令人吃惊

> 原文:[https://devops.com/developers-lack-of-digital-leadership/](https://devops.com/developers-lack-of-digital-leadership/)

*尽管应用程序积压，开发者称他们的努力资金不足，训练不足，并且缺乏对安全的重视*

我们已经写了很多关于企业在开发他们需要的应用程序时所面临的挑战，挖掘出 T2 积压的应用程序和 T4 的技术债务，以及运行高效开发管道的重要性。如何跟上，企业看到越来越多的公民开发者试图弥补他们需要开发的软件应用之间的差距。

这就是为什么最近发布的一份报告“投资于企业开发人员是最好的风险管理系统”令人沮丧，该报告基于应用程序开发人员联盟对 100 多名开发人员的调查，其中 40%的受访者经历过因缺乏资金而导致的开发失败，三分之一的人担心当前的项目可能会因预算原因而被取消。此外，尽管[调查](http://www.gocertify.com/articles/who-goes-there-bcs-survey-says-security-a-top-priority-for-many-it-executives.html)后[调查](http://www.aicpa.org/press/pressreleases/2015/pages/security-regains-place-as-top-technology-priority-for-cpas-north-american-survey-finds.aspx)高管称安全性为“重中之重”，但只有 45%的受访企业开发人员认为他们的组织将安全性设为“高”优先级。

不足为奇的是，根据调查，五分之四接受调查的企业开发人员使用 Windows 和/或 OS X，24%使用 Linux。然而，只有 20%的人专门在移动平台上工作。有趣的是，三分之二的企业开发人员专门开发 HTML5 应用程序，混合或原生，其中原生是最常见的。而且，毫不奇怪，在今天，云是开发、测试或生产期间部署应用程序的最常见方式。

不幸的是，许多开发人员反映在技术、流程和培训方面资金不足。三分之一的受访者表示他们组织中的技术和流程已经过时，超过一半(51%)的受访者表示他们的组织拥有超过 100 名员工。

## 许多领域缺乏领导力

在培育流程和技术指导方面，很多企业也有所欠缺。只有 37%的被调查者说他们的组织有一个卓越中心。不足为奇的是，较大的企业更有可能有一个卓越中心；然而，缺乏这样的中心在北美和欧洲、中东和非洲更为普遍。然而，几乎所有接受调查的亚洲组织都有一个专门的卓越中心。此外，令人惊讶的是，只有 29%的被调查开发人员表示他们需要参加培训、认证项目或其他继续教育。

考虑到越来越多的企业是由他们的数字化努力驱动的，这令人震惊。引用这一点的调查不在少数。在最近由 CA Technologies 委托进行的一项名为“应用经济中的竞争优势之战”的研究中，43%的受访者表示，成为软件驱动型是当今竞争力的核心，近 80%的受访者表示这将在三年内实现。总的来说，为了在未来几年保持竞争力，企业除了现有的核心产品和服务之外，还必须成为优秀的软件公司。然而，一项又一项调查发现，企业在应用程序开发目标方面落后了，并且很难找到他们需要的人才。

此外，应用程序开发人员联盟调查发现，只有 50%的企业讨论他们取得成功所需的能力和技能，而只有 26%的企业将这些能力整合到他们的评估流程中。而且，就像建立卓越中心一样，亚洲公司脱颖而出，78%的亚洲开发人员表示讨论了必要的技能发展。

应用开发者联盟调查发现，卓越中心也是企业是否致力于构建安全软件的优秀指标。虽然 64%的开发人员认为他们在构建安全软件方面受到了“充分”的培训，但在拥有卓越中心的组织中，超过 70%的人认为安全性是他们工作的重中之重。如果他们在一个没有卓越中心的组织中工作，这个数字会下降到 28 %, 17%的人将安全性列为这类企业中较低的优先级。

显然，如果企业希望在未来几年参与全球竞争，他们就需要在应用程序开发预算、培养所需技能和建立数字化企业成功所需的内部组织方面提高优先级。
