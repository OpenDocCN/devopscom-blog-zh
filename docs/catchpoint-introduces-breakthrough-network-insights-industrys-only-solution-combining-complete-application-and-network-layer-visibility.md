# Catchpoint 推出突破性网络洞察:业内唯一结合完整应用和网络层可见性的解决方案

> 原文：<https://devops.com/catchpoint-introduces-breakthrough-network-insights-industrys-only-solution-combining-complete-application-and-network-layer-visibility/>

新的高级边界网关协议监控与域名系统、Traceroute、企业和端点监控相结合，使公司能够最大限度地提高其数字性能

纽约州纽约市—(2019 年 10 月 17 日) [Catchpoint，](http://www.catchpoint.com/)下一代数字体验监控(DEM)的领导者，今天宣布推出 Network Insights，这是一款开箱即用的网络可见性解决方案，旨在解决负责向客户和员工提供高性能数字服务的 IT 团队的主要可见性差距。

Network Insights 将新的边界网关协议(BGP)监控与现有的域名系统(DNS)、跟踪路由、企业和端点监控相结合，以提供对所有网络层的全面了解。这种主动网络监控解决方案允许企业消除由于应用和服务不可阻挡地向云迁移以及越来越多地使用互联网向客户和员工提供服务而产生的盲点。

针对传统数据中心部署的传统网络监控产品与现代数字交付链完全不兼容。他们依靠昂贵的物理探测器和设备来收集和分析网络流量。相比之下，Catchpoint 的 Network Insights 为企业提供即时网络可见性，无需部署探头、设备、软件控制台或复杂的代理。投资回报立竿见影。

由于 Catchpoint 独特的监控方法，这种即时客户投资回报和即时可见性成为可能。Network Insights 由 Catchpoint 的全球监控网络提供支持，该网络是业内最大、最多样化的监控基础设施，部署在全球超过 825 个主干网、宽带、无线、云和最后一英里位置。这使团队能够主动检测网络相关问题，并从最终用户所在的任何地理位置快速确定其根本原因，从而缩短平均修复时间(MTTR)和平均检测时间(MTTD)，从而保护最终用户体验。

“云、SaaS 和互联网是令人难以置信的数字化转型推动者，但它们造成了巨大的可见性差距，因为企业正在失去对基础设施和应用程序的直接控制，”Catchpoint 首席执行官兼联合创始人 Mehdi Daoudi 说。

“网络通常是降低数字性能的罪魁祸首，网络工程团队需要深入了解网络如何影响客户和员工体验，同样重要的是，需要获得可操作的数据来提高性能。跨最终用户、应用和所有网络组件的集成可见性现在终于可以提供我们的客户所需的全面网络级视图。”

Catchpoint 的网络洞察由以下四个部分组成:

*   BGP 监控: BGP 是一种协议，它通过边缘路由器之间的信息交换来管理数据包在互联网上的路由方式。BGP 路由的变化——无论是恶意攻击还是人为错误——都会对服务可用性产生负面影响，直接影响最终用户。Catchpoint 的高级 BGP 监控可快速检测路由劫持、策略配置问题、路由襟翼和对等问题等问题，并调查路由健康状况和公告或撤销等事件。
*   DNS 监控: Catchpoint 是唯一提供两种类型 DNS 监控的供应商:DNS Direct 和 DNS Experience。DNS Direct 测试直接查询域名服务器，以提供可用性数据并确保提供的响应是准确的。DNS 体验测试模拟递归名称服务器，以测量 DNS 解析链中服务器的延迟和可用性。
*   Traceroute Monitoring: 该功能由业内分布最广的测试基础设施提供支持，通过自动化系统(AS)和路由器粒度提供全球网络路径(包括多云和混合云环境)的可视化。这使得团队可以从每一跳收集数据，以便他们可以快速查明根本原因。了解导致延迟和/或数据包丢失的原因有助于网络团队快速解决路由器问题或对等问题，从而减少对最终用户的影响。
*   企业和端点监控: Catchpoint 企业节点部署在企业网络中，提供对本地托管应用程序或降级员工体验的性能问题的双向检测。除此之外，Catchpoint 的端点监控功能可一直延伸至员工设备，从员工的浏览器和工作站进行收集和分析，消除了通常的最终可见性差距。两者都提供了运行连续跟踪路由来收集和分析网络数据的能力。

Daoudi 继续说道:“Network Insights 提供的主动式方法不仅可以帮助客户避免终端用户遭遇基于网络的服务中断，还可以在时间紧迫的三角测量阶段快速排除网络问题。“替代的监控方法不能解决所有网络层的问题，只能从有限的地理位置或有利位置捕获终端用户性能数据，这会导致太多的性能盲点。”
