# 简化云性能监控的 4 种方法

> 原文：<https://devops.com/4-ways-to-simplify-cloud-performance-monitoring/>

在过去的 18 个月里，全球各地的技术专家通过数字战略在品牌和组织的生存中发挥了至关重要的作用。在所有行业中，他们的技能和奉献精神使企业能够快速推出创新的新数字服务，以满足客户需求的巨大波动，并允许所有员工在疫情期间在家工作。

这并不容易，尽管最终用户对无缝数字体验的期望正在上升。事实上，最近的 [2021 应用关注指数](https://www.appdynamics.com/c/dam/r/appdynamics/Gated-Assets/analyst-reports/AppDynamics-App-Attention-Index-2021.pdf)再次确认发现，76%的消费者对数字服务的期望在去年有所增加。85%的受访者表示，应用程序和数字服务已经成为他们日常生活的重要组成部分。这给负责保持应用和云体验平稳运行的技术人员带来了更大的压力。

IT 团队通常面临着监控公司数字服务的挑战，但他们缺乏监控日益扩张的 IT 产业的运行状况和性能所需的可见性，而过去 18 个月向云架构的巨大转变更加剧了这种情况。IT 部门还没有合适的工具和数据来优化其云环境的运行状况和性能。

结果，技术专家不断地解决相同的问题，因为他们不能容易地隔离和识别它们，并且不知道在哪里优先考虑他们的行动。打嗝会导致品牌失去客户——同一份报告显示，超过一半的消费者表示，品牌只有一次机会来打动他们，如果他们的数字服务表现不佳，他们就不会再使用它们。

在疫情开始一年半后，随着云技术的采用在未来几年将进一步加速，技术专家是时候走在前面了。技术人员可以做以下四件事来减轻管理云计算性能的压力。

### 实现自下而上的可视性

如果没有合适的工具，了解应用程序和支持它们的底层云基础架构可能会非常困难，这也给技术人员带来了一些压力。传统的监控允许技术专家识别和捕捉 IT 堆栈中某些领域的问题，但这种方法在动态、分布式和软件定义的环境中用处不大，因为在这种环境中，组织会不断扩大对 IT 和云的使用。

因此，数字服务的性能可能会受到影响，可能会损害最终用户的体验，削弱对品牌的信任。

为了跟上数字世界中消费者不断增长的需求，技术人员需要能够监控整个 IT 产业，从云基础设施到面向客户的应用程序，再到计算、存储和网络等核心基础设施；包括开放网络和安全性。通过使用像可观察性平台这样的工具，技术专家可以减轻单独监控支持数字服务的每个项目的性能的负担。相反，他们可以从云中获得警报、根本原因以及相关指标、事件、日志记录和跟踪(融化)数据分析的整体视图，以便尽早轻松地进行故障排除。

考虑将基础架构(计算、存储和网络)与其执行的服务和应用程序联系起来的工具。当出现应用程序级别的问题时，很容易仔细观察并确定应用程序运行或应用程序本身的问题区域。

### 使 IT 绩效与业务成果保持一致

在聚合应用及其背后的云基础架构的全栈视图后，技术人员可以将可观察性平台识别的性能数据与实时业务指标相链接。

将两者联系起来，使技术人员能够识别数字服务在哪里以及如何影响企业和最终用户。[全堆栈可观察性](https://devops.com/?s=full-stack+observability)自动为技术专家提供这些关联的见解，使他们能够解决最重要的领域。

### 教育团队

除了拥有合适的工具来帮助利用云之外，it 团队精通技术本身也很重要。管理云的性能是复杂的，企业可用的人才库是有限的。因此，企业领导人应该提供适当的资源，以确保团队得到成功的教育和装备。

为了支持无缝实施云基础设施以支持企业的数字服务，企业领导必须投资提升他们团队的技能或招募其他了解云监控的人员。

这将减轻现有团队的压力，并给予技术人员在职业生涯中取得进步的技能和信心。

### 利用自动化

即使拥有最佳的可观察性解决方案和具备必要技能的团队，人们仍然担心未来几年加速的数字化转型会产生大量数据，特别是那些涉及大规模公共云环境的数据。对于技术人员来说，投入所需的资源和技能水平来管理和优化整个 It 产业的性能可能是一个挑战。

为了在未来避免这些潜在的情况，值得考虑如何通过减少 it 部门对手动干预的依赖来监控运行状况和性能并解决问题，从而最大限度地减少数据泛滥的影响。

自动化和人工智能将在跟踪整个 IT 产业的健康和性能方面发挥越来越重要的作用。在大规模公共云环境中尤其如此，这很快成为一些 IT 团队最头疼的问题。

通过人工智能部署来实时识别和修复问题，技术人员可以花更少的时间来解决问题，这要归功于人工智能/人工智能的指导。这将提高工作满意度和成就感，更好地平衡工作和生活。
