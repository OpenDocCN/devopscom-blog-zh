# 建设健康团队的开发人员指南

> 原文：<https://devops.com/a-developers-guide-to-building-healthy-teams/>

为了构建一个伟大的软件产品，团队合作必须延伸到开发者、管理者和执行者。每个人都需要理解每个角色的价值——没有一个好的教练，你就不可能有一个高效的团队，反之亦然。要实现梦想，需要一种建立在同理心、信任和共同责任基础上的文化，以及适量的雄心和人际关系。

实现这种平衡并不容易，但这里有一些帮助你的团队变得更健康、更有效的建议。

### 警惕知识孤岛

当一个人的大脑中存储了太多独特的知识时，你完成事情的能力取决于他们的能力，这可能会危及团队和整个组织。有些筒仓可能是好的，但只有当团队中的每个人都知道每个筒仓中有什么以及如何访问信息时。

知识筒仓可以采取几种形式:

*   **技术技能**，像是团队里唯一一个懂围棋或者 Python 的人。
*   **某个领域的辅助知识**，比如统计学背景。
*   **历史知识**，就像一个开发者从一开始就在那里，知道整个代码库。
*   **机构知识**，比如了解更大的业务和项目战略以及公司的历史。

有一些克服知识孤岛的好方法。简单地了解谁擅长什么——或者每个人都有什么背景——是一个很好的起点。然后，领导者可以创造性地思考团队协作，如结对编程或根据每个人的知识分配拉动请求。推动每个开发人员评估他们自己的筒仓可以帮助团队在即兴会议或预定会议期间更好地共享知识。区分文档的优先级是确保开发人员共享他们所知道的东西的好方法。此外，使用棕色袋子和演示可以帮助你的开发者磨练他们的教学技能。这些策略是投资团队未来发展速度的好方法。

### 关注组成团队的个人

《快公司》发表了一篇伟大的 [文章](https://www.fastcompany.com/90563793/4-personality-types-to-build-an-unstoppable-dream-team) 讲述了组成一个伟大团队的不同性格类型:

*   **大脑**推动他人采取行动追求目标。
*   **声音**了解全局并激励团队。
*   敏锐的目光关注细节，以正确的方式完成工作。
*   **心脏**促进社区、共识和协作。

思考这四个非常不同的特质，并确定团队中每个角色的角色，可以帮助你建立一个和谐的团队，同时确保每个人的优势都得到强调和欣赏。

### 培养同理心，建立信任

对你的团队有同理心通常意味着在赢得胜利或达到某些目标的愿望和理解每个队友的不同需求之间找到平衡。作为一名经理，你应该站在开发人员的角度，鼓励自己问一些深思熟虑的问题。

同理心建立信任。也是如此:

*   支持开发人员的独特水平。
*   培养健康短跑和个人健康的文化。
*   使用数据——而不仅仅是直觉——作为改进的起点
*   清除生产力的障碍，让团队不会觉得超负荷。
*   给予个人贡献者自主权。
*   作为一个团队持续完成目标。
*   对团队面临的挑战保持透明。
*   在整个组织内共享数据。

容易侵蚀信任的因素包括:

*   基于代码行对开发人员进行堆栈排序。
*   以精疲力竭为代价，在雄心勃勃的最后期限前重整旗鼓。
*   用数据贬低你的团队成员。
*   强制实行生产率配额。
*   从自上而下的角度进行管理。
*   将数据限制在领导层。

这些建议帮助团队避免[精疲力尽](https://devops.com/?s=burnout)，精疲力尽肯定会毁掉一个团队。世界卫生组织(世卫组织)将职业倦怠定义为“一种由长期工作压力导致且未能成功控制的综合征”所以这不仅仅是一个人的个人问题——倦怠是你的组织出了问题的迹象。

理解知识筒仓，强调个人优势，培养同理心和建立信任是一个强大的开发团队的基础。一旦你完成了这些，你就能走向更健康、更有效的团队。
