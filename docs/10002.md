# 三是神奇的数字:规划和执行一个成功的 DevOps 项目

> 原文:[https://devo PS . com/three-is-the-magic-number-planning-and-executing-a-successful-devo PS-project/](https://devops.com/three-is-the-magic-number-planning-and-executing-a-successful-devops-project/)

勤奋的准备、明确的工作流程和密切的监控是交付高性能 DevOps 项目的关键。DevOps 是编程中最热门的词汇之一，但当涉及到构建和部署 DevOps 团队时，每个人似乎都有自己的想法。

从根本上说，DevOps 是将开发人员和 IT 运营人员集成到一个组中，目标是提高软件部署的速度和质量。它的创建是对团队分离的关注的回应，这导致了沟通障碍和缺乏凝聚力。

自开发以来，DevOps 越来越受欢迎，83%的 IT 决策者表示他们已经以某种形式实现了这种工作方式。

然而，以这种方式转移业务重点并不能保证立即改变命运——这种动态代表了一种 [IT 文化](https://devops.com/crossing-culture-divide/)的转变，更加强调协作和人际交往技能。为了让这种模式产生回报，工程师应该对新的信息、技术和工作方式持开放态度。

那么，DevOps 团队必须采取哪些重要步骤来确保项目的成功交付呢？

### 了解客户的需求

初始阶段可以成就或破坏一个 DevOps 项目。如果团队没有清晰地了解客户想要什么，以及如何实现这一点，这个项目注定会失败。

至关重要的是，团队和客户都愿意花时间去理解对方的目标，确保他们的愿景与接下来的步骤一致。这可以通过一系列的会议和研讨会来完成，在这些会议和研讨会上，确定了目标，并对最终产品的外观有了很好的理解。

如果执行正确，DevOps 团队将带着定义明确的摘要和对客户希望看到交付的清晰理解退出项目的第一阶段。当这一步匆忙进行时，工程师会因缺乏方向而受到阻碍，增加了成品不能反映客户希望的可能性。

### 配置环境

这个下一阶段是应用程序开发过程的开始，通常使用基于云的解决方案来实现。该团队首先准备环境的美学，找出它应该包含的组件，并探索如何配置它们以最大限度地提高效率。

正是在这个时期，增加了网络安全措施，以便该应用程序的最终版本尽可能接近不可渗透。这说明了为什么 DevOps 团队需要多种多样的技能组合:网络安全专家很少，但至少应该有一名成员的知识可以用来确保保护措施得到正确实施。

一个成功的开发阶段是由结构清晰的工作流程支撑的，项目是一步一步规划出来的，参与的每个人都了解他们在计划中的位置以及他们如何做出贡献。

然而，DevOps 项目从开始到结束很少是一帆风顺的——开发经常被错过的截止日期、错误和冲突的职责所中断，这将工程师从他们的岗位上拉走。由于这个原因，敏捷和多技能的团队成员，他们能够在不同的领域中很好地参与和协助，对于在工作流偏离轨道时确保连续性是极其重要的。

### 避免在项目的最后阶段失败

仅仅因为一个产品的开发已经完成，应用程序已经上线，这并不意味着 DevOps 团队应该满足于现有的成就，开始下一个项目。重要的是，他们要在应用程序的早期阶段继续监控它，这样任何错误都可以被发现并立即修复。

工程师应该到位，以了解环境的负担有多大，以及环境如何根据新用户的速度和数量做出响应。获得的任何数据都是珍贵的商品，应该如此对待，团队在收集到数据后立即订购并标记结果。

一旦收集了基线数据，就可以对结果进行分析，以确定应用程序的哪些部分可能需要重新校准。在进行这些改进后，最终结果应该是产品性能达到最佳水平，同时也满足客户的规格要求。

### 不要催促项目的进程

在所有三个阶段结束时，DevOps 团队应该绝对相信项目正朝着正确的方向前进，并且他们完全有能力执行接下来的程序步骤。

当一个团队由来自一系列领域的专家组成，他们渴望相互合作和交流时，一个设计良好的应用程序要容易得多。集合合适的人员将有助于 DevOps 项目各阶段之间的平稳过渡。
