# Plutora 深入研究价值流分析数据

> 原文:[https://devo PS . com/plu tora-dives-deep-into-value-stream-analytics-data/](https://devops.com/plutora-dives-deeper-into-value-stream-analytics-data/)

Plutora 今天宣布，它已经扩展了其价值流管理(VSM)平台的功能，包括[一个通用对象数据模型](https://www.businesswire.com/news/home/20210622005267/en/Plutora-Introduces-Enhanced-Data-Centric-Platform-to-Guide-Strategic-Decision-Making-and-Achieve-Successful-Software-Development)，该模型可以更有效地规范化从多个 DevOps 工具和平台收集的数据。

Plutora 产品营销和战略副总裁 Jeff Keyes 表示，目标是扩大 Plutora 平台中嵌入的 Plutora 分析引擎分析流量指标的规模和深度。

[VSM](https://devops.com/?s=value+stream+management) 已经成为一个独特的开发运维规程，它提供了一个框架，使组织能够跨开发运维环境优化工作流，从而加快应用开发。核心概念追溯到精益制造方法，持续测量流程的每一步以提高产量。

随着组织对软件依赖程度的加深，理解软件如何构造的兴趣也在增加。例如，组织希望更好地了解错过软件开发最后期限对收入的影响，企业可能无法像预期的那样快速产生收入。

Plutora 正在为一个从多个平台提取数据的专用分析平台做准备。或者，更多的价值流分析功能现在被嵌入到从项目管理软件到持续集成的所有东西中。然而，Plutora 认为，在组成 DevOps 工作流的不同工具所呈现的多个价值流仪表板中导航，并不像依赖于从所采用的每个工具收集数据的单一平台那样简单或有效。

Keyes 指出，跟踪项目的提前期和周期时间以及总体吞吐量的指标变得更加容易展现和理解。

![](../Images/5a54eebb3274280e004887b6d83f62cf.png)

无论如何，DevOps 工作流会变得更加透明。从历史上看，收集这些数据一直很有挑战性，因为它需要开发运维团队从每个正在使用的工具中手动收集数据。实际上，DevOps 团队浪费时间将数据从一个工具复制并粘贴到一个分析应用程序中。

随着聚合所有这些数据变得越来越简单，Plutora 提供的数据模型使得多种分析工具分析相同数据变得更加简单。例如，业务用户可能更喜欢使用现由 Salesforce 拥有的 Tableau analytics 平台来分析数据。

一般来说，分析在软件开发过程中的应用越来越广泛，因为组织需要在降低成本的同时提高生产率。将由每个组织来决定他们在多大程度上想要使用该数据来更好地培训 DevOps 团队，而不是简单地使用它来提醒那些团队保持专注于一组特定的关键绩效指标(KPI)。

与此同时，随着收集指标的过程变得更加自动化，向开发运维团队提出的报告请求的数量应该会下降。现在的挑战将是确保阅读这些报告的每个人不仅对使用手头的资源实际可能实现的目标有相同的坚定理解，而且对任何额外的 DevOps 专业知识如何积极影响这一等式有相同的坚定理解。
