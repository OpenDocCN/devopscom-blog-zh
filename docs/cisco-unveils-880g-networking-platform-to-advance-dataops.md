# 思科推出 800G 网络平台推进数据运营

> 原文：<https://devops.com/cisco-unveils-880g-networking-platform-to-advance-dataops/>

在[开放计算项目(OCP)全球峰会](https://www.opencompute.org/summit/global-summit)会议上，思科宣布其开发了[一种 800 千兆位交换机](https://newsroom.cisco.com/c/r/newsroom/en/us/a/y2022/m10/cisco-delivers-trailblazing-innovations-for-cloud-operators.html)，这种交换机比上一代网络设备消耗的电能要少得多。

思科 Nexus 和 ACI 产品线云网络产品管理副总裁 Thomas Scheibe 表示，思科 One ASIC 处理器的最新 7 纳米迭代所提供的吞吐量主要是处理大量数据的组织所需要的，以在本地数据中心或云中训练人工智能(AI)模型。他指出，挑战在于，许多组织正在寻求建立人工智能模型，以处理大量数据，同时减少其 IT 基础设施的碳足迹。

思科已经能够通过继续投资专有 ASIC 处理器来实现这一目标；Scheibe 说，它们是其网络产品组合的核心，以改善[数据操作(DataOps)](https://devops.com/?s=dataops) 。Scheibe 补充说，许多 IT 团队也在寻求更换传统的网络路由器和交换机，这些路由器和交换机消耗更多的电力，并减少高达 77%的能源成本。在气候影响方面，思科声称其 8111-32EH 交换机现在可以以 15%的功率需求提供 25.6Tbps。思科预计，与 12.8Tbps 的交换机相比，该交换机每年将节省约 10，000 千克二氧化碳当量。根据思科的数据，这相当于一辆普通汽油动力乘用车行驶 26，155 英里所产生的温室气体排放量。

IT 团队还可以选择部署思科的网络操作系统或 SONiC——在云网络操作系统(NOS)中实现开放网络的软件。

思科还为 IT 组织提供了配置其最新路由器和交换机的选项，以运行在 800、400 或 100 千兆位，着眼于在未来某个时候升级吞吐量。

总的来说，随着组织意识到他们需要实施最佳实践来优化整个组织的数据流，作为 IT 学科的数据操作正在走向成熟。思科为 IT 团队提供了采用标准以太网结构或替代结构的选择，通过根据数据包的属性和之前网络流量的行为预测哪些数据包需要被传送到特定位置来提高吞吐量。随着低延迟应用程序需要访问的数据量持续增长，数据操作将成为一项更加重要的 IT 规范。过去，存储管理员倾向于负责数据管理，但随着这些应用程序的不断增加，一类新的数据操作工程师正在涌现，以优化分布式计算环境中的数据流。

从长远来看，目前还不清楚有多少二氧化碳排放会影响 it 决策，但有一些组织已经开始跟踪它，作为降低碳排放的努力的一部分。大多数云服务提供商也承诺实现碳中和。思科确信，在需要处理的数据量只会继续呈指数级增长的时候，实现这一目标将需要升级 IT 基础设施。
