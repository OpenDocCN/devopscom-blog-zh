# Harness 为开发人员提供与云无关的端到端软件交付平台

> 原文：<https://devops.com/harness-empowers-developers-with-cloud-agnostic-end-to-end-software-delivery-platform/>

AWS GovCloud、Azure 和 GCP 集成为开发运维及财务管理团队提供了一个 CI/CD 平台，可用于多种云部署，并实现所有公共云资源的全面成本可见性

旧金山——2021 年 4 月 15 日——[领先的软件交付平台 Harness](https://harness.io/) ，今天宣布与亚马逊(AWS) GovCloud、微软 Azure 和谷歌云平台(GCP)的端到端集成，以帮助开发人员和工程团队执行多云软件部署。Harness 最初是为了部署管道和为 Kubernetes 环境中托管的应用程序提供成本管理而开发的；他们现在正在扩展这一愿景，在三个主要的公共云平台上全面提供第一个 CI/CD 和云成本管理平台。

这对于像 [GoSpotCheck](https://harness.io/customers/case-studies/microservices-ci-cd/) 这样的公司来说极其重要。GoSpotCheck 将其应用程序托管在多个云中，包括 AWS 和 GCP。因此，GoSpotCheck 必须为每个云构建和维护单独的管道，这是一个耗时的过程，需要双方供应商的 API 专业知识。这些管道不是不可知的，也不能跨云、栈和平台移植。

“我们的部署渠道非常复杂，并且不是云不可知的。”–尼克·威尔逊| DevOps 领导

如今，借助 Harness，GoSpotCheck 能够轻松地在云之间迁移微服务，而无需停机。Harness 抽象了与每个云提供商的交互，因此部署可以在云提供商之间轻松切换，而无需了解底层 API。软件交付管道是不可知的、可移植的和动态的——需要工程师或 DevOps 团队进行最少的维护。

根据 Gartner 的调查，[超过 81%](https://www.gartner.com/en/conferences/apac/infrastructure-operations-cloud-india/featured-topics/cloud#:~:text=multiple%20cloud%20providers.-,A%20recent%20Gartner%20survey%20of%20public%20cloud%20users%20finds%2081,best%2Dof%2Dbreed%20solutions.) 在云中运行的组织使用两家或更多的公共云提供商。雪上加霜的是，每个云提供商都需要专门的工程专业知识和工具来管理特定云提供商的细微差别。这减慢了已经背负着太多工具的工程团队的速度。今天的企业需要一个全栈 CI/CD 平台，允许他们在三大公共云提供商(AWS、Azure 和 GCP)之间无缝移动，同时允许他们降低云成本。Harness 使客户能够将其构建、部署、验证和成本管理流程整合到一个简化的平台中，而无需担心多云环境的复杂性。

Harness 首席执行官兼联合创始人 Jyoti Bansal 表示:“我们注意到，容器化软件部署与向云原生架构的迁移之间存在直接关联。“随着越来越多的企业希望利用他们的软件做更多的事情并降低基础架构成本，他们开始转向云计算架构来提高正常运行时间，避免供应商锁定并获得价格优势。通过这些集成，Harness 满足了这一需求，在云部署复杂性和开发人员之间提供了一个抽象层，因此每个公司都可以比以往更快地交付下一代软件。”

通过授权客户直接访问所有主要的公共云，Harness 提供了管理 Kubernetes 上托管的应用程序时所期望的同样深刻的部署和成本管理体验。Harness 与各大公共云提供商的最新集成包括:

亚马逊 AWS GovCloud

在公共部门工作的开发人员需要使用适合他们的监管环境的工具。这通常意味着构建他们自己的软件交付工具或者使用市场上可用的少数工具。Harness 已经支持 AWS ECS、AWS AMI、AWS CodeDeploy、AWS Lambda 等等。现在，开发人员可以使用 Harness 将新软件部署到安全的政府环境中托管的应用程序，让客户及其合作伙伴能够灵活地构建符合政府法规和合规制度的安全云解决方案，例如:

*   FedRAMP 高基线

*   司法部刑事司法信息系统(CJIS)安全政策

*   美国国际武器贩运条例(ITAR)

*   出口管理条例

*   针对影响级别 2、4 和 5 的美国国防部(DoD)云计算安全要求指南(SRG)

这种集成还允许云运营和财务管理团队全面了解其所有公共云资源。

微软 Azure

Harness 已经支持 Azure 虚拟机规模集(VMSS)、Azure Web App 和 Azure Gov 云类型工作负载。根据今天的消息，他们还支持 Azure 原生部署和云账单分析，为在 Azure 云上运行 Harness CI/CD 的客户提供了关键的附加功能。Harness 现在通过 Azure 无服务器技术和 ARM 支持蓝图、管理和资源组来提供帮助。客户还可以通过利用云成本管理深入了解他们的 Azure 成本。

谷歌云平台

Harness 已经支持一些 GCP 部署，如 Google Kubernetes 引擎、Google Cloud Run 工作负载和部署模板功能中的 Google Cloud 功能，但缺乏关键的集成，无法使其功能与其 AWS 产品相媲美。现在，Harness 支持部署原生 GCP 应用程序，并与 Google Secrets Manager、Google Key Management Service、Google Cloud Storage、Google Container Registry、Stackdriver、GCP 工作负载身份和 Google Cloud Build 集成。
