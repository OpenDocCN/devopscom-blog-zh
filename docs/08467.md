# MariaDB 会取代 LAMP 中的“M”吗

> 原文:[https://devops.com/does-mariadb-replace-the-m-in-lamp/](https://devops.com/does-mariadb-replace-the-m-in-lamp/)

如果你明天开始一个新的应用程序，你会使用哪个后端？你可能会说 MySQL。然而，收购甲骨文后，MySQL 发生了变化。甲骨文购买代码后，他们伤透了许多人的心。MySQL 有希望以 MariaDB 的形式重新发明自己

LAMP 栈中的 M (MySQL)一般不被质疑。它起作用了，而且无处不在。但是，缺乏 Oracle 的深度企业支持，加上预期的许可证变更，以及他们缓慢的新功能可能会为 MariaDB 窃取 m 大开方便之门

马里亚布:一个强有力的竞争者

7 月 15 日，MariaDB 宣布了对 MariaDB Enterprise 及其 MaxScale 产品的重大改变，这让我重新审视了从活跃的 MySQL 开发时期到 2010 年后 Oracle 收购时期的转变。

该公告包括几个引起我兴趣的花絮:

*   **预构建的带有 MariaDB 实例的 Docker 容器。** 公共图书馆中的 Docker 容器通常不是我认为的特性或公告。然而，在这种情况下，我认为它们值得一提。要意识到的一件重要的事情是，如果一个商业实体发布了一个工具，他们有责任使它保持最新。
*   **预置厨师食谱**
*   ****Azure market place 中的 Azure 图片****

**所有这些都意味着更快的入职，这是一个好消息，因为入职通常是对应用程序组件进行任何更改的最糟糕的部分。**

*   ****伸缩性** 。在我看来，这是最大的新闻:围绕可伸缩性的功能，以及基于服务器状态变化启动脚本的能力。特别是启动脚本的能力，对于那些不能跟踪所有活动部分的 IT 人员来说，是一个巨大的时间节省器(也是一个屁股节省器)。**

**虽然还不是一个家喻户晓的名字，但 MariaDB 确实有一个令人印象深刻的用户名单，其中包括许多旅游行业的用户，如 Booking.com[](https://mariadb.com/blog/mysql-slave-scaling-and-more)、迪士尼以及热门网站 CraigsList 和维基百科。惊讶吗？我是。因为 MySQL 仍然拥有应用程序后端声音的最大份额。**

**SmartFile ，一个基于云的应用和本地设备，帮助企业安全地连接、管理和共享文件，最近从 MySQL 后端跳到了 MariaDB 开源。SmartFile 的高级软件工程师瑞安·约翰斯顿(Ryan Johnston)表示:“这是一种替代升级。“我们关闭了 MySQL，将 MariaDB 指向我们现有的数据库”。这种快速升级对他的开发团队来说意味着很少甚至没有编码。最大的技术优势是即时的性能提升和轻松测量的能力。“MariaDB 增加了强大的调试功能和分析。我们可以看到哪些查询运行得最慢。SmartFile 的软件工程师特拉维斯·坎宁安(Travis Cunningham)说。现在他们的团队有很大的提升空间。**

**Travis 说，迁移到 MariaDB 的另一个重要因素是“我们希望原作者支持产品向前发展”。不仅是技术特性的有力证明，也是与社区关系的有力证明。**

****但是 MySQL 还活着****

**从 MySQL 社区的发展和 Oracle 的直接贡献来看，这些改进并不是微不足道的，但它们大部分是表面的，并且集中在管理工具而不是核心引擎上。**

**我以前在另一个技术领域见过这种情况，例如 OCR。ABBYY 和 Nuance 这两个主要 OCR 引擎的核心被锁在某个永远不会被改进的黑暗洞穴中，但在表面上有很多事情正在进行。在更广泛的生态系统中也有很多正在发生的事情，像亚马逊 AWS 的 [【极光】](https://aws.amazon.com/rds/aurora/) 和 MySQL 的 RDS。**

****面向成年人的 SQL****

**MariaDB 清楚地展示了改进数据库引擎核心的更大潜力。它还可以将数据库与更广泛的开发过程结合起来，试图将后端开发人员纳入其他人的行列。这可能最终会将后端引入 DevOps 流程。**

**然而，很快就清楚了，你需要他们的商业工具来实现这一切。这是一个商业产品 MaxScale，一个数据库代理，一切似乎都在这里，秘方就在这里。**

**MariaDB 展现出了超越商业努力的可信度。他们坚定地致力于管理 [开源](https://mariadb.org/) 项目。他们通过实施从专业服务和大客户部署中学到的一些功能来回馈社会。而 [社区](https://mariadb.com/kb/en/mariadb/show-contributors/) 本身也是活跃的，包括 Google 那些添加了静态数据加密的发布[10 . 1 . 3](https://mariadb.com/kb/en/mariadb/data-at-rest-encryption/)，这是当今安全气候下急需的功能。**

**在蒙蒂·维德纽斯(Monty Widenius)看到 MySQL 即将被收购并预计会陷入停滞后，他立即行动起来，首先发起了“ [”拯救 MySQL](https://monty-says.blogspot.com/2009/12/help-saving-mysql.html) ”运动，然后冒险成立了 MariaDB Corporation(前身为 SkySQL )。他不仅复活了他的孩子，他还继续改进任何应用程序的关键组件的核心。**

**因为社区多年来一直是 MySQL 背后的主要驱动力，而且它无处不在，所以 MariaDB 不太可能成为新的“M ”,或者与开源 MySQL 竞争。但是现代开发中的 LAMP 也不是过去的样子，也不是通用开发栈的完整画面，所以它可能是一个无声的问题。**

**当你比较 MariaDB 与 Oracle 商业 MySQL 产品时，会发现明显的优势。他们有一个可靠的产品，可以很好地适应后端的发展。但可能不会是旧金山的初创公司或一次性的大量开发者。**
