# DevOps 十二大奖 2021 提名现已开始

> 原文：<https://devops.com/devops-dozen%c2%b2-awards-2021-nominations-now-open/>

DevOps 十几个奖项又回来了，并开放提名。过去的一年是一段不平凡的旅程，我们为 DevOps 社区在这些充满挑战的时代中的支点、创新、适应和超前思考能力感到骄傲。一些杰出的领导者和供应商为了达到他们的目标并取得最高水平的成功而不停地工作。我们很高兴庆祝他们的辛勤工作和成就。

devo PS 十几项大奖旨在表彰对社区做出重大贡献的最具奉献精神、最鼓舞人心的思想领袖和创新者。第二年，我们扩展了我们的奖励计划，纳入了两个不同的 DevOps 十几个部分:DevOps 十几个工具和服务奖以及 DevOps 十几个社区奖。

每个板块有 12 个奖项:

*   **DevOps 十几种工具和服务奖** —表彰那些开发和交付卓越解决方案以增强开发人员、DevOps 和 IT 运营团队能力的公司。
*   **DevOps 十几个社区奖** —表彰在 DevOps 社区中产生积极影响的领导者。

提名是由观众做出的。任何人都可以提名一个工具或服务，尽管工具和服务奖提名需要 250 美元的报名费。我们去年实施了这项收费，以减少垃圾邮件和多余的提名。但是，社区奖提名不收费。请注意，商业工具和服务是 **而不是** 有资格获得社区奖，并且不会进入决赛。

将选出决赛选手，并从决赛选手中选出获胜者。最终入围者将由公众投票(占最终总分的 40%)和我们的评委评选(占最终总分的 60%)决定。

提交提名的最后一天是**10 月 22 日** 。要投票，请按照 [DevOps 打网站](https://devopsdozen.com/) 上概述的步骤进行。

## **2021 年 DevOps 十几个类别**

**DevOps 十几个工具和服务奖项**

*   ****最佳端到端开发运维工具/服务** 开发运维领域的整合为提供端到端解决方案的平台带来了更多选择。该奖项旨在表彰为开发运维团队提供最多样化、最强大解决方案的平台。**
*   ****最佳 DevOps Repo/GitOps 工具/服务** Git 带来了 GitOps，它已经成为 DevOps 工具箱不可或缺的一部分。该奖项授予最杰出的 Repo/Git/GitOps 工具。**
*   ****最佳 CI/CD 工具** 该奖项旨在表彰帮助团队更好地交付高质量代码的最佳 CI/CD 工具。**
*   ****最佳价值流管理工具** 价值流管理已经成为衡量开发运维及应用开发业务价值的重要工具。该奖项旨在表彰最有效地帮助可视化价值流、跟踪指标和衡量改进的价值流管理工具。**
*   ****最佳可观察性解决方案** 该奖项旨在表彰最佳可观察性解决方案，用于观察和监控单个组件和系统以及整个应用的性能。**
*   ****最佳 DevSecOps 解决方案** 该奖项旨在表彰最佳 DevSecOps 解决方案，该解决方案能够在整个 SDLC 中更早、更持续地提供安全性。**
*   **最佳测试服务/工具
    测试是 DevOps 工具链中的支柱。这个奖项是对确保代码质量的最佳测试工具的认可。**
*   ****最佳 Kubernetes 平台/服务** 容器的采用为支持和增强容器技术的整个解决方案生态系统铺平了道路。Kubernetes 可能是这个领域中得到最广泛认可和使用的工具。有许多平台和服务提供基于 Kubernetes 的解决方案。该奖项旨在表彰最佳 Kubernetes 平台/服务。**
*   ****最佳云原生安全解决方案/服务** 由于可用解决方案的深度和广度以及专为云原生环境构建应用和服务的优势，云原生运动席卷了 IT 界。该奖项旨在表彰被视为云原生的最佳工具、解决方案和服务。**

*   ****大型机最佳开发运维解决方案**大型机远非 IT 界的老古董，有多种解决方案可用于在大型机环境和混合环境中实现开发运维实践。这个奖项是为了表彰他们中的佼佼者。** 

*   ****针对数据运营/数据库解决方案的最佳开发运维**  数据库经常被排除在开发运维对话之外，但这种情况正在改变。数据行动的整个概念正在获得更大的吸引力。该奖项旨在表彰开发运维类和数据运维类数据库领域的最佳解决方案。**
*   ****最佳新 DevOps 工具/服务提供商**  该奖项旨在表彰过去一年中为 DevOps 领域带来创新的最佳新服务提供商或工具(必须成立不到 24 个月)。**

****DevOps 十几个社区奖项****

*   ******最佳 DevOps 行业实施**该奖项旨在表彰成功采纳并实施 DevOps 原则和实践的组织。**** 

*   ******最佳 DevOps 调查/分析/研究**该奖项旨在表彰对 DevOps 社区产生重大积极影响的研究。**** 

*   ******最佳 DevOps 相关视频系列(视频)**此类别突出了最具娱乐性、教育性和励志性的 DevOps 视频系列——那些最具吸引力和最高质量的内容。**** 

*   ******最佳 DevOps 相关音频播客系列**今年你最喜欢的 DevOps 播客是哪个？这个类别庆祝了许多伟大的 DevOps 播客引发了许多有趣的讨论，从 CI/CD 和测试到 DevSecOps、Kubernetes、containers 等等。**** 

*   ******年度最佳 DevOps 书籍/电子书**每年都会有大量制作精良的 DevOps 相关书籍和电子书问世。这个类别突出了今年最受欢迎和最好的读物。**** 

*   ******顶级 DevOps 布道者**  这一类是我们的最爱之一。有这么多伟大的福音传播者，我们想知道谁做得最好。被提名者可以在 DevOps 内部或外部工作。**** 

*   ******最佳 DevOps 转型(非供应商)**在数字化转型成为生存必需品的这一年，我们想知道哪些组织在通过 DevOps 实现业务转型的努力中脱颖而出。**** 

*   ******年度最佳虚拟赛事**新冠肺炎改变了举办赛事的游戏规则。2021 年，虚拟活动仍将是常态，当每个人都呆在家里时，虚拟活动将覆盖全球观众。哪个 DevOps 虚拟活动从其他活动中脱颖而出？**** 

*   ******最具创新性的 DevOps 开源项目**开源项目旨在促进创新，提高效率、速度和质量。虽然有许多伟大的开源项目，但最好的是那些创建一个社区的人，他们使用该技术并相互支持。**** 

*   ******DevOps 年度高管(非供应商)** 该奖项旨在表彰高管(或同等职位)级别的 DevOps 领导者的卓越表现。**** 

*   ******年度最佳 DevOps 演示**  该类别从年度众多(虚拟)会议中评选出最佳 DevOps 主题演示。****
*   ******年度最佳 DevOps.com 文章**【DevOps.com】每天发布多篇新文章。2021 年最好的文章是哪一篇？请告诉我们，以便我们能认出作者。****

****所有获得工具和服务奖提名的解决方案将自动注册到由 Techstrong Group 编写的 DevOps 工具指南中。该计划旨在创建一个全面的工具和工具供应商列表，以帮助最终用户更好地了解 DevOps 市场格局。****
