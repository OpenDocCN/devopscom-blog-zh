# PagerDuty 选择威胁堆栈来实现云中的高速安全性和合规性

> 原文：<https://devops.com/pagerduty-selects-threat-stack-enable-high-velocity-security-compliance-cloud/>

*威胁堆栈的持续安全监控有助于确保安全性和可扩展性*

**BOSTON–2016 年 3 月 16 日**–[云安全管理和合规的领导者](http://www.threatstack.com/)今天宣布，敏捷事件管理的先驱[page duty](https://www.pagerduty.com/)选择了 [Threat Stack 云安全平台](https://www.threatstack.com/platform) ^(TM) 来保护其亚马逊网络服务(AWS)环境中快速增长的基础设施。PagerDuty 决定采用威胁堆栈是因为它需要持续的安全监控和对其用户、数据的完全可见性，以及不断扩展的 AWS 和 Microsoft Azure 基础架构。

PagerDuty 基础设施工程总监 Arup Chakrabarti 表示:“作为一家致力于提供可靠性和正常运行时间的快速发展的组织，我们利用云的可扩展性和灵活性，同时确保我们的资源安全至关重要。“威胁堆栈不仅易于部署，还易于与新的和现有的工具集成。现在，我们可以全面了解我们的流程和活动，以便对整个环境进行持续监控和安全管理。”

PagerDuty 平台使 IT 运营和开发团队能够快速高效地管理和响应事件，从而减少任何类型的中断或挫折的可能性。威胁堆栈的持续实时安全监控有助于 PagerDuty 团队快速识别顶级威胁，简化运营工作流程中的流程，并提高整个组织的安全性。

“我们是 PagerDuty 的长期粉丝，很高兴与他们合作，为用户活动、流程、文件和网络连接提供精细、集成的可见性，”Threat Stack 产品和客户宣传副总裁 Venkat Pothamsetty 表示。“我们很自豪能够通过帮助 PagerDuty 扩展其基于云的业务，同时保持其安全性和合规性来加深这种关系。”

PagerDuty 正在使用威胁堆栈云安全平台的全套功能和跨其 [AWS 环境](https://www.threatstack.com/solutions/integration#aws)的特殊集成来提供主机级可见性和其他关键功能，包括:

*   **自动扩展:**威胁堆栈的云原生架构可根据 PagerDuty 的 AWS 和微软 Azure 基础设施的容量自动扩展或缩减，以确保其动态环境的持续可见性。

*   **内部威胁识别:**持续监控未授权行为，如用户权限提升、未授权安装、添加/删除新用户、可疑命令、异常行为、安全组变更、用户会话信息和流程停止。

*   **外部攻击预警:**自动检测异常登录尝试/失败、完全开放的安全组、新进程或内核模块的启动、用户会话信息、进程停止、用于命令和控制的外部连接。

*   **数据安全验证:**自动访问**、**编辑或复制客户或个人数据知识产权，以及内部配置、密码、证书和密钥。

*   **合规性报告**:实时可见性、详细的审计跟踪和内置合规性报告提供了满足合规性法规并确保数据和基础架构受到保护所需的历史记录。
