# MariaDB 和 Node.js 连接增强，增加了功能和更快的性能

> 原文：<https://devops.com/mariadb-and-node-js-connection-strengthens-adding-features-and-faster-performance/>

*与 Node.js 开发人员的替代连接器相比，新的 MariaDB 连接器提供了 1.7 倍的查询性能和 35 倍的批处理速度*

**加利福尼亚州门洛帕克和赫尔辛基——2018 年 12 月 13 日——**[MariaDB Corporation](https://mariadb.com/)今天宣布推出适用于 Node.js 的 MariaDB 连接器，为开发人员提供了一种快速、简化的方法来在 Maria db 成熟的企业关系数据库基础上构建 Node.js 应用程序。Node.js 越来越受欢迎，根据 [Stack Overflow](https://insights.stackoverflow.com/survey/2018/) 和 [HackerRank](https://research.hackerrank.com/developer-skills/2018/) 今年的调查，node . js 最近被评为开发者的首选框架。使用 Node.js 的 MariaDB 连接器，开发人员可以获得比其他连接器更好的性能。当[与其他流行的 Node.js 驱动程序进行基准测试时，MariaDB 连接器在查询模式下的性能提高了 1.7 倍，在批处理模式下提高了 35 倍。](https://mariadb.com/resources/blog/mariadb-connector-node-js-first-beta-now-available/)

MariaDB Corporation 的技术营销总监 Thomas Boyd 说:“开发人员厌倦了在 NoSQL 上构建 Node.js 应用程序的局限性。“我们看到开发人员越来越倾向于使用 MariaDB 来替换 MEAN stack 中的“M ”,以便使用标准 SQL 更广泛地查看整个企业的数据。在最受信任的企业开源数据库 MariaDB 上构建 Node.js 应用程序时，MariaDB Connector for Node.js 为开发人员提供了更快的性能体验。”

使用 MariaDB Connector for Node.js，开发人员可以通过连接器的插入流和管道功能享受更好的整体应用程序性能。这些功能减少了网络流量，并支持来自 Node.js 应用程序的数据库查询的异步处理，从而大大提高了多个数据库查询的处理效率。该连接器与 MariaDB 的标准 SQL 层配合使用，提供对最新 MariaDB 数据库功能和创新的访问，如时态处理、高级数据保护等，以及对用于报告和分析的数据的轻松访问。

Node.js 的 MariaDB 连接器现在可以作为候选版本从[下载](https://mariadb.com/downloads/#connectors),并将于下个月正式发布。参加我们 12 月 18 日在[举办的网络研讨会，开始使用 MariaDB Connector/Node.js](https://go.mariadb.com/GLBL-WBN_2018-12-18Node.jsconnector_Registration-LP-New.html) 构建 Node.js 应用程序，并了解如何利用带有 JSON 函数、时态表等的现代 SQL。

**关于 MariaDB 公司**

MariaDB 将公司从专有数据库的成本、约束和复杂性中解放出来，使他们能够在最重要的方面进行再投资——快速开发创新的、面向客户的应用程序。MariaDB 使用可插拔的专用存储引擎来支持以前需要各种专用数据库的工作负载。随着复杂性和限制的消除，企业现在可以依靠一个完整的数据库来满足其所有需求，无论是基于商用硬件还是他们选择的云。对于事务性或分析性用例，MariaDB 可在几分钟内完成部署，提供无与伦比的运营灵活性，而不会牺牲关键的企业功能，包括真正的 ACID 合规性和完整的 SQL。受到德意志银行、DBS 银行、纳斯达克、Red Hat、Home Depot、ServiceNow 和威瑞森等组织的信任——Maria db 以极低的成本满足了与专有数据库相同的核心要求。难怪它是增长最快的开源数据库。真正的业务依赖于 MariaDB。
