# 大多数组织现在都采用了云计算

> 原文:[https://devo PS . com/major-of-organizations-now-adopt-multi-cloud/](https://devops.com/majority-of-organizations-now-adopt-multi-cloud/)

随着其数字足迹的增长，其云足迹的规模和复杂性都在增加。因此，大多数组织已经成熟为多云运营商。[多云方法](https://devops.com/multi-cloud-is-not-about-lift-and-shift/)使不同部门能够自主选择适合其需求的计算环境。它还支持在需要时在云服务提供商(CSP)之间转移工作负载的能力。采用多云模型可以带来性能优势、成本节约或更好的可扩展性，同时鼓励更加中立的供应商立场。

我们处于一个多云的世界。HashiCorp 最近的一项研究发现，四分之三的组织已经采用了云计算。与几年前相比，这是一个惊人的增长，当时单一云更受欢迎，运营商对 AWS 有着僵尸般的忠诚。多重云的兴起标志着其他电信运营商产品的成熟，如 Azure 和 GCP。它还标志着云之间的差异，因为每个云都有其独特的价值主张。

HashiCorp 的首次[云战略状态调查](https://www.hashicorp.com/state-of-the-cloud)提供了一些非常清晰的关于多云采用状态的结果。下面，我将回顾报告中的主要观点，以更好地了解多云计算的采用情况。综上所述，大型组织往往从采用云计算中受益最大，因为要真正从云计算中获得可观的回报，仍然有很大的障碍。然而，更多的基础设施自动化可能有助于各种规模的公司应对多云复杂性。

## 多重云时代

DevOps 真正在多云时代运营。该报告显示，76%的组织已经采用了云计算；这使得只有四分之一的公司利用单一云。而且，这一趋势不会停止，在两年内，86%的人预测他们将成为云计算运营商。该研究还证实，公司规模越大，就越有可能采用云计算，94%拥有 5，000 名或更多员工的大型企业都采用了云计算。

那么，是什么推动了云的分裂？34%的受访者认为[数字化转型](https://devops.com/the-role-of-apis-in-digital-transformation/)是采用云计算的首要驱动因素。在满足新的[数字创新](https://devops.com/the-state-of-digital-innovation-one-year-into-the-pandemic/)需求的竞赛中，多重云已经成为一股战略力量。为了实现这一快速变化，许多组织正在投资云卓越中心(CCoE)来指导云转型工作。CCoE 在云预算在 500 万美元或以上的大公司中最为常见。

说到预算，[云支出](https://devops.com/how-to-respond-to-rising-cloud-costs/)正在飙升。2020 年，只有一半的公司达到了他们的预计云支出，39%的公司超出了分配的云预算。正如我之前提到的，[云原生成本](https://containerjournal.com/features/identifying-and-trimming-hidden-container-costs/)是出了名的难以识别和削减。如果没有适当的护栏来管理支出，意外的数据出口费用或隐藏的服务器成本可能会使云支票簿变得紧张。目前，40%的公司每年在 100，000 美元到 200 万美元之间，6%的企业每年在云计算上投资 5，000 万美元或更多。随着对云的依赖的扩大，这一支出在未来几年可能会增加。

## 企业受益最大

那么，企业在使用哪些云呢？如果我们比较云的采用，AWS 是一个明显的市场领导者；88%的公司使用 AWS，74%使用 Azure，61%使用 GCP。AWS 的霸主地位预计不会在未来两年内消退。然而，虽然 AWS 的使用将保持不变，但竞争的 CSP 将会有适度的增长。这可能是因为 Azure 和 GCP 在北美以外的地理区域(如亚太和非洲)取得了进展。

在云环境中使用的技术方面，[开源软件](https://devops.com/how-open-source-software-powers-digital-innovation/) (OSS)是一个受欢迎的选择。云原生开源项目，如 Kubernetes、OpenStack 和 Terraform，已经有了自己的生命，有了[充满活力的社区支持](https://containerjournal.com/features/five-values-that-define-kubernetes-culture/)。然而，并不是所有的商店都建立在操作系统*和*之上，自己运行它。有点讽刺的是，我们正看到更多的托管操作系统出现。一个突出的领域是供应，其中 42%的组织使用开源构建并自己运行。

总体而言，多数企业都采用了云计算，53%的受访者表示，云计算帮助他们实现了业务目标。多重云的最大优势是数字化转型，占 34%。其次是避免锁定(30%)、降低成本(28%)和可扩展性(25%)。但多云并不是对所有公司一视同仁；随着公司规模的增加，多云成功的可能性也会增加。不到一半(42%)的小公司表示他们已经成功实现了多云，而对于大企业来说，这一比例跃升至 64%。由于支持资源较少，对于初创公司和中小型企业来说，多云可能不太有利。事实上，41%的专业人士表示，缺乏内部技能阻碍了他们在云领域的成功。

## 混合云世界中的挑战

持续的人才缺口只是云成功的一个潜在挑战。超过一半(51%)的受访者表示，成本问题是他们云计划成功的最大阻碍。[混合云的复杂性](https://devops.com/3-key-issues-with-hybrid-cloud-transformation/)是另一个突出的挑战，35%的组织将其列为主要的云障碍。许多企业仍在努力解决遗留的内部房地产问题。但是[技术债务](https://devops.com/enterprises-face-growing-technical-debt/)并不是混合模式的唯一原因。令人惊讶的是，大型平台甚至建造了自己的数据中心，或者使用[主机托管](https://devops.com/old-man-yells-at-cloud/)作为有意的策略。

47%的 IT 专业人员认为，安全性是首要的云抑制因素。主要的云安全问题包括数据保护(40%)、数据窃取(33%)和法规遵从性(31%)。无数的网络安全威胁每天都在演变，云技术也不能幸免；例如，web APIs 现在是一个重要的威胁载体。另一份报告发现，在过去的六个月里，恶意 API 流量增长了 300%。其他问题，如暴露的密钥、[不安全的默认设置](https://containerjournal.com/features/insecure-defaults-remain-a-threat-for-kubernetes/)和[错误配置](https://containerjournal.com/topics/container-security/misconfigurations-tops-list-of-security-issues-hampering-kubernetes/)都被认为会阻碍云原生基础设施。

与普遍看法相反，近一半(46%)的公司报告称，新冠肺炎没有加快他们的云采用。这一现实可能表明云迁移已经计划了多年，最近的采用只是一个持续趋势的一部分。该研究发现，疫情确实加速了基础设施即代码(IaC)、容器编排、合规性和治理以及其他运营因素。

## 基础设施自动化支持多种云

在一个更加多元化的市场中，多云似乎是加速采用云的必然结果。上升的多云使用也可以说符合[康威定律](https://nordicapis.com/conways-law-what-does-it-mean-for-your-api-strategy/)。随着组织在新的垂直领域开发越来越多的数字体验，支持这些计划的平台和后端流程将不可避免地变得越来越复杂。

为了缓解由云计算带来的一些问题，许多组织正转向提高基础架构自动化。事实上，70%的受访者表示基础设施自动化工具对他们的组织极其重要。在这些工具中，供应自动化(75%)是最常用的一种。其他领域的自动化，如网络(58%)、安全(50%)和应用程序开发(69%)紧随其后。

Hashicorp 的云战略状态调查大胆地宣布，“欢迎来到多云时代”，其调查结果支持了这一观点。如需进一步了解，您可以在这个[网站](https://www.hashicorp.com/state-of-the-cloud)上免费查看结果。
