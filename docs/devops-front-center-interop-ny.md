**专题小组:DevOps 为网络做什么？**
DevOps 之于应用部署，如同 SDN 之于网络。它们最终都是为了实现卓越运营，以缩短上市时间、降低风险和运营开销相关的成本。DevOps 倾向于将脚本和 API 作为实现卓越运营的手段，那么这如何适应将自动化和流程编排引入网络的趋势呢？我们将探讨每个人心中的问题，例如:您是否必须拥有一个软网络才能利用 DevOps？网络工程师需要成为开发人员吗？

**这是另一个讨论**的小组:网络行业认为 CCIE 认证是最难获得也是最理想的头衔之一。许多 CCIEs 享受非凡的职业机会，以及通过臭名昭著的实验室考试带来的直接经济收益。最近，那些看到网络不同未来的人对 CCIE 认证的相关性提出了质疑。主要想法是这样的:如果网络朝着自动化、软件定义的方向发展，CCIE 还适用吗？或者，希望站在行业顶端的网络人员应该通过学习 SDN 革命的新兴工具和技术来区分自己吗？我们的全 CCIE 小组(包括前 CCIE 项目经理)& CCIE 版主将进行讨论。

这个小组将讨论在 SDN 时代 CCIE 认证的未来。随着软件蚕食世界和网络，硬件还有意义吗？应该是大讨论和承诺一些烟火！
