SwaggerHub 中新的开源命令行界面(CLI)工具也使组织能够利用其 OpenAPI 定义来自动化 CI/CD 构建管道。对 SwaggerHub 的 Git 集成的增强包括支持 API 定义与现有 SCM 存储库的同步，以及保留与新 API 版本相关的集成。这些增加还允许开发人员为新的 API 版本自动创建 Git 分支，为新的 API 更改启用自定义提交消息，并将 API 更改从 Git 拉回到 SwaggerHub。

借助新的 CLI 工具，团队现在可以围绕 SwaggerHub 构建自动化和工作流。团队可以在 CI/CD 管道中使用它来创建新的 API，创建和更新 API 版本，并将 API 版本标记为已发布或默认。
