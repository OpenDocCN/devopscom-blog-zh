# Sonatype 发布 Nexus Repository Pro，内置高可用性和软件组件分析

> 原文:[https://devo PS . com/sonatype-announces-nexus-repository-pro-high-avail ability-software-component-analysis-build/](https://devops.com/sonatype-announces-nexus-repository-pro-high-availability-software-component-analysis-built/)

*devo PS native repository manager 为现有客户群提供轻松升级*

**马里兰州富尔顿**–2016 年 9 月 14 日—[软件供应链自动化领域的领导者 Sonatype](http://www.sonatype.com/) 今天推出了最新版本的 Nexus Repository Pro。Nexus Repository Pro 的 3.2 版本将包括对主动-主动高可用性配置的内置支持，以及对开源组件的增强安全性和许可证分析。Sonatype 还扩展了 Nexus Repository 的免费、通用组件支持，以包括 PyPI 和 RubyGems。

Nexus Repository Pro 的新功能将在 Jenkins World 大会(9 月 13 日至 15 日)和 JavaOne 大会(9 月 18 日至 22 日)上展示。该产品目前正在进行私人测试，将于 2016 年第四季度上市。

Sonatype 的 Nexus Repository Pro 已安装超过 100，000 次，现在提供了更多功能，包括:

*   高可用性，价格独特，配置快速，易于管理
*   普遍支持所有流行的组件格式
*   精密元件智能

Sonatype 首席执行官 Wayne Jackson 表示:“我们很高兴能够通过最新版本的 Nexus Repository Pro 提供高可用性，以确保公司能够实现持续交付目标。“我们的客户将能够利用这一先进且经济高效的产品将高可用性无缝集成到他们现有的生产环境中，从而最大限度地减少停机时间并提高可靠性。结合我们精确的组件智能和不断扩展的组件支持，Nexus Repository Pro 可以轻松构建和发布最佳软件。”

**通过高可用性满足持续交付需求**

主动-主动高可用性是寻求持续交付和开发运维工具链 99.999%正常运行时间的组织的一项要求。最新版本的 Nexus Repository Pro 支持主动-主动[高可用性](http://www.sonatype.com/HA)，确保如果一台服务器出现故障，另一台服务器将继续运行而不会中断。高可用性功能内置于 Nexus Repository Pro 3.2 中，非常易于配置。定价从每年 1200 美元起。

**将组件支持扩展到 PyPI 和 RubyGems**

Nexus Repository 是唯一一个为世界上最流行的基于组件的开发语言提供免费支持的存储库管理器，这些语言包括 Java、npm、Bower、NuGet 和 Docker，现在还有 PyPI 和 RubyGems。

**提供精确的组件智能**

每天，Nexus Repository 客户使用[Repository Health Check](https://books.sonatype.com/nexus-book/3.0/reference/rhc.html)(RHC)评估 70，000 个存储库中数百万开源和第三方组件的质量和安全性。RHC 提供有关组件的情报，包括许可证类型、已知的安全漏洞和其他数据，如相对使用流行程度和年龄。Nexus Repository 2.x 的这一功能现已免费提供给 Nexus Repository 3.2 用户。

**将 Nexus Repository 2.x 升级到 3.x**

从发布 Nexus Repository 3.1 开始，Sonatype 为当前使用 Nexus Repository 2.14 的客户提供了无缝升级途径。内置的升级工具和文档为当前的安装基础提供了一种简单的升级方式，以利用最新的功能，包括支持私有 Docker 注册表、高可用性、存储库健康检查和通用组件支持。
