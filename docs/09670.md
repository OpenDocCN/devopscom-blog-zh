# Cycode 扩展 AppDev 安全平台的范围

> 原文:[https://devo PS . com/cy code-expands-scope-of-app dev-security-platform/](https://devops.com/cycode-expands-scope-of-appdev-security-platform/)

在 [Black Hat USA 2022](https://www.blackhat.com/us-22/) 大会上，Cycode 本周宣布[在其基于图形数据库的软件组合分析(SCA)平台](https://www.globenewswire.com/news-release/2022/08/09/2495009/0/en/Cycode-Launches-Next-gen-SCA-Featuring-Full-Pipeline-Composition-Analysis-to-Deliver-Complete-Software-Supply-Chain-Security.html)中增加了静态应用安全测试(SAST)和容器扫描功能。

Cycode 的产品管理总监 Amnon Even-Zohar 表示，这些工具的添加使现在与其知识图数据库集成的模块总数达到了 8 个。他补充说，这些模块解决了从基础设施代码错误配置和 DevOps 管道到二进制和源代码分析的应用程序安全要求，减少了出现的误报数量。

他指出，这种方法可以通过从整个软件开发生命周期中提取数据，而不是只关注源代码中发现的漏洞，为软件修复工作提供更多背景信息。Even-Zohar 说，例如，Cycode 可以精确定位测试和生产环境中易受攻击的依赖项。

他补充说，Cycode 管道组成分析工具还提供了对哪些组件导致漏洞或安全问题的见解，管道组件如何相互关联，以及它们是否存在于运行时环境中。

虽然在一系列高调的 AppSec 违规事件之后，人们对实现 DevSecOps 最佳实践有了很大的兴趣，但许多组织仍然缺乏在端到端的基础上保护应用程序开发环境所需的工具。Even-Zohar 说，Cycode 平台利用 DevOps 工作流中已有的数据，使组织能够更好地优先考虑他们的补救工作，并降低这样做的总成本。

他指出，目标不仅是通过更紧密地整合网络安全和 DevOps 团队，使组织能够将网络安全进一步向左转向开发者，而且进一步向右。

大多数组织试图应对的挑战是，从历史上看，网络安全研究人员发现的漏洞列表是在没有太多背景的情况下与 DevOps 团队共享的。开发人员不知道哪些漏洞是最关键的，也不知道所识别的漏洞是否实际存在于他们的应用程序环境中。从理论上讲，SCA 工具可以使识别变得更加容易，例如，在哪里可以找到 Log4j 或 shell 漏洞。基于图形的工具将这种能力提升到了一个新的水平，它使得可视化任何漏洞实例的位置变得更加容易。如果没有这样的工具，IT 组织将花费数周时间寻找漏洞的每个实例。

DevSecOps 工作流变得普通可能还需要一段时间，但至少现在对软件供应链安全问题有了更多的认识。拜登政府还发布了一项行政命令，要求联邦机构审查软件供应链的安全性。许多企业 IT 组织现在都在效仿。不管怎样，应用程序的安全性将会得到改善。唯一有待确定的是，这些改进将对软件的构建和部署速度产生什么影响。
