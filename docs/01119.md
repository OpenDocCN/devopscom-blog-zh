# Vapor IO 推出 OpenDCRE 1.2，这是第一个真正面向物理基础设施的 DevOps 平台

> 原文:[https://devo PS . com/vapor-io-introduces-opendcre-1-2-first-true-devo PS-platform-physical-infra structure/](https://devops.com/vapor-io-introduces-opendcre-1-2-first-true-devops-platform-physical-infrastructure/)

*OCP 将 Vapor IO 应用于 Micro 和 BASELAYER 团队，通过开源项目进一步努力实现当今数据中心的现代化，带来开箱即用的管理功能，消除复杂性并显著降低资本支出和运营支出*

<time datetime="2016-03-10T22:35:00Z">March 10, 2016 05:35 PM Eastern Standard Time</time>

得克萨斯州奥斯汀—([商业资讯](http://www.businesswire.com/))—[全球首个智能、超模块化数据中心解决方案提供商 Vapor IO](http://cts.businesswire.com/ct/CT?id=smartlink&url=http%3A%2F%2Fwww.vapor.io%2F&esheet=51298470&newsitemid=20160310006621&lan=en-US&anchor=Vapor+IO&index=1&md5=3fa770f4eeb4106861de040533fe5d88) 今天宣布了 Vapor 开放式数据中心运行时环境(OpenDCRE)的新版本。凭借广泛的社区支持，包括技术合作伙伴 Open Compute Project (OCP)、applied micro Circuits Corporation(纳斯达克股票代码:AMCC)和 BASELAYER，OpenDCRE 是第一个连接 IT 和物理基础设施的真正的基础设施自动化和 DevOps 平台。这些合作伙伴关系突出了 Vapor IO 在开源社区内进一步开发超融合数据中心架构的目标。

OpenDCRE 是由 Vapor IO 创建的开源解决方案，用于简化数据中心基础设施通信。新版本现在支持 AppliedMicro X-Gene 服务器系列和 BASELAYER Anywhere 模块化数据中心(MDC)产品。OpenDCRE 是竞争对手专有机架架构/基础设施的卓越替代方案，消除了供应商锁定。

“作为开放计算项目的创始成员，我们是开源解决方案的支持者和早期采用者，”高盛公司技术总经理 Grant Richard 说。“OpenDCRE 提供了简化数据中心基础设施通信的机会，并推进了 OCP 构建最高效的服务器架构的使命。我们很高兴能与 Vapor 和我们在开源社区的合作伙伴一起朝着这个目标努力。”

“开放计算项目致力于确保社区主导的标准通过开放接口进入市场，”开放计算项目首席执行官 Corey Bell 表示。借助 Vapor IO 的 OpenDCRE 硬件(已提交投稿)，最终用户现在能够通过完全免费和完全开放的 API 远程管理 OCP 兼容的基础设施。"

**创建智能的超模块化数据中心**

OpenDCRE 提供了一个安全的 RESTful API，用于监控和完全控制您的整个数据中心和 IT 设备。它可以用作监控和管理数据中心硬件、软件和环境特征的开放式平台。OpenDCRE API 支持与第三方监控、管理和编排提供商轻松集成。作为专有接口和定制芯片的替代产品，它为通用和定制 DevOps 任务提供了一个简单、可卷曲的接口。Vapor IO 致力于促进社区支持的应用、部署、物理安装策略和网络连接选项种类繁多。

OpenDCRE 1.2 为企业提供了以下功能:

*   开放计算项目(OCP)社区的开箱即用管理，同时消除架顶式带外管理交换机和复杂网络架构的复杂性、资本支出和运营支出。
*   能够直接与数据中心关键环境系统对话，从建筑的 M&E 成本中去除额外的资本支出
*   与 GNU 通用公共许可证版本 2 (GPLv2)下的许可的类似 Linux 的兼容性

Applied Micro 软件和平台工程副总裁 Kumar Sankaran 表示:“Applied Micro ARMv8 是基于 64 位四节点 X 基因的 OCP 平台，是同类产品中的第一款，标志着 ARM 服务器进入 OCP 该平台特别适合高密度计算市场。通过将 OpenDCRE 集成到该平台中，它为数据中心内非常高效的管理基础设施铺平了道路。"

BASELAYER 创始人兼首席执行官 William D. Slessman 表示:“我们很高兴与 Vapor IO 合作。Vapor IO 的 OpenDCRE 与 BASELAYER 的模块和 RunSmart 数据中心操作软件的结合扩展了我们的平台范围，并使我们的客户能够实现更高的操作性能。”

“数据中心环境变得越来越复杂，充斥着来自不同供应商的各种硬件和软件解决方案和工具。为了让 IT 组织取得成功，他们需要运行高效和敏捷的环境。Vapor IO 创始人兼首席执行官 Cole Crawford 表示:“构成当今数据中心的各种资源必须无缝协作，同时作为独立组件存在，以便在必要时进行替换、修复或升级。“我们的任务是将分散的数据中心转变为一个高效、连接良好但有意去耦合的 it 系统。”

Vapor IO 由 Cole Crawford 创建，他是一位在开源硬件和软件技术方面拥有深厚专业知识的行业资深人士。Crawford 在数据中心行业拥有超过 20 年的经验，他曾与他人共同创立了致力于 IT 转型的公司和非营利组织，包括 OpenStack 和[开放计算项目](http://cts.businesswire.com/ct/CT?id=smartlink&url=http%3A%2F%2Ft.sidekickopen15.com%2Fe1t%2Fc%2F5%2Ff18dQhb0S7lC8dDMPbW2n0x6l2B9nMJW7t5XZs4XrfJRW5vfVKg2zWN1RW5vw5gT56dC3Jdj28Hs02%3Fsi%3D5115394242641920%26pi%3DCC4549F1-D4E1-45A5-B9CD-E74D9AE2C564%26t%3Dhttp%253A%252F%252Fwww%252Eopencompute%252Eorg%252F&esheet=51298470&newsitemid=20160310006621&lan=en-US&anchor=Open+Compute+Project&index=2&md5=f51ca9f309c903a80545b86d824538a2)，在 Open stack 中，他曾担任基金会成立委员会成员，在中，他曾担任该项目的创始执行董事。
