# Kyndryl 在云计算领域与微软结盟

> 原文：<https://devops.com/kyndryl-aligns-with-microsoft-in-the-cloud/>

从 IBM 剥离出来后，Kyndryl 与微软结盟，以扩大其在 Azure 公共云上提供的 IT 服务范围。

Kyndryl 全球联盟和合作伙伴关系负责人 Stephen Leonard 表示，作为该计划的一部分，Kyndryl 将扩大其服务范围，为构建和部署应用程序的组织以及那些寻求将现有应用程序迁移到微软 Azure 云的组织提供服务。

随着时间的推移，Kyndryl 将扩展其云服务，以包括更多的[公共云](https://devops.com/?s=public+cloud)，作为更大努力的一部分，使组织能够采用混合云方法来实现跨多个内部和云计算环境的企业 IT。

与此同时，作为 Kyndryl 建立的第一个合作伙伴关系的一部分，两家公司致力于提供服务，以解决数据现代化和治理、安全以及注入人工智能(AI)功能的应用程序工作负载。两家公司还计划建立一个联合创新实验室。微软还承诺为微软建立一所 Kyndryl 大学。

Leonard 表示，鉴于 IT 专业知识的整体短缺，越来越多的组织依赖外部 IT 服务来加速应用程序开发等。他指出，Kyndryl 计划与这些组织合作，以最好地评估特定类别的应用程序工作负载可能最适合部署在哪里。

越来越明显的是，越来越多的组织正在尝试将第三方 It 服务集成到开发运维工作流中。在某些情况下，这意味着标准化由 IT 服务提供商定义的固执己见的 DevOps 工作流。在其他情况下，IT 服务提供商专注于特定的任务，例如代表开发人员安全地配置基础架构。

然而，无论采用何种方法，开发运维工作流都正在演变成一项团队运动，由内部和外部的 IT 专业人员组成，他们试图弥合不同组织之间始终存在的文化鸿沟。IT 服务提供商组织最终选择哪一家，不仅取决于他们的技术专长，还取决于他们在文化方面的灵活性。

就其本身而言，微软正依靠 Kyndryl 等 IT 服务提供商来增加在其云上运行的工作负载总数。现在 Kyndryl 独立于 IBM，它可以自由地建立合作伙伴关系，这在以前会更加复杂。

当然，由每个组织来决定他们更愿意与较大的 it 服务提供商合作的程度，而不是与较小的竞争对手合作，较小的竞争对手可能专注于某个特定领域，或者更适合较小的公司。可以肯定的一点是，随着 it 变得越来越复杂，组织不可能全职雇佣所有需要的 IT 人才。组织需要仔细评估何时将某些 it 服务作为他人提供的功能来消费更有意义，而不是试图雇佣和保留人才来维护服务。
