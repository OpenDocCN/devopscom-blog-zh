# 市场大融合即将到来

> 原文：<https://devops.com/the-great-market-convergence-is-coming/>

当你环顾科技市场——几乎任何科技市场——你都可以看到明显的重叠。大量的市场是互补的，它们采取不同的方法来解决同一个问题，或者说，随着时间的推移，这些市场相互融合。

这发生在高度活跃的市场。人们有很好的想法并实施它们，但其他人认为不同的方法是合理的，两者不到万不得已不会直接竞争。 a 市场内有竞争*，然后*市场有竞争*。我认为我们将会看到* IT 市场*的竞争将会大幅增加，特别是 DevOps 市场。我承认我试图在不偏离当前话题的情况下写这篇文章，所以请原谅我。*

我们以动态 app 保护为题。你可以通过你的晶片供应商做到这一点，你可以通过你的晶片供应商做到这一点，如果你的晶片供应商不是 AAP 供应商，你可以通过为网络托管软件提供的各种插件和工具做到这一点，如 T2 Apache T3 或 NGINX …事实上，IDP 实际上只是试图动态地保护你的应用程序。我敢肯定，我错过了其他几个市场，这些市场急于填补实时应用被黑客锁定的空白。我选了一个市场，但是在 DevOps 选一个，开始挖。你会发现同样惊人的重叠。

问题是，我们不需要一个问题有 15 个解决方案；我们需要一两个——最多几个。结合大多数这些市场早就应该整合的事实(考虑到全球经济给贸易带来的压力)，我认为我们将在相对较短的时间内看到相当数量的整合。

出于几个原因，这对您很重要。首先，如果你认为解决问题的一种方法明显优于目前可用的其他方法，你的组织应该通过购买产品和服务来投资。“我们中的更多人应该买 Betamax”不是一个很好的外观。其次，需要有人关注这些变化对 IT 环境的影响。如果你的 DevOps 工具的一个供应商被合并了，不可避免的一个被合并的组织接管了，每个人都以他们的方式做事情。这并不一定是坏事，但是如果你突然收到一封电子邮件，说你部门的核心系统发生了变化，那就太可怕了。意识到外面正在发生什么，至少要想好如果应用——或者整个市场——消失或者发生重大变化，你会如何应对。

老实说，因为我们一直在谈论的应用程序(主要是 DevOps)最有可能与附近的市场融合，所以它看起来很大。但请记住，核心系统很久以前就经历过这种情况，虽然它们可能在未来经历另一次“合并这个与那个”的迭代，但不会像当前非常活跃和年轻的市场似乎即将遭受的那样巨大。

所以不要太担心你使用的是哪种语言，或者是哪种语言，它们有着惊人的重叠，但都倾向于利基市场。关注仍在增长和扩大的市场空间。了解可能存在的风险，并意识到你今天是否需要关注，或者在相关市场崩溃或与附近市场合并之前，你还有时间。

并拥有它。我们都知道，到时候，不管你是否准备好了，你都会想出一个解决方案——因为这就是你要做的，而且没有人在背后支持你。因此，继续让你的组织看起来很棒，并保持对更广泛的 IT 市场趋势的了解，这样你就不会完全措手不及。
