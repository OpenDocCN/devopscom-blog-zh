# Anchore 继续向软件供应链安全市场扩张

> 原文:[https://devo PS . com/ancore-continues-expansion-into-software-supply-chain-security-market/](https://devops.com/anchore-continues-expansion-into-software-supply-chain-security-market/)

*新客户采用和开源社区参与燃料扩展*

**加利福尼亚州圣巴巴拉，2022 年 1 月 28 日—** [Anchore](https://www.anchore.com/) 今天宣布了过去一年在软件供应链安全市场的强劲业绩。随着对软件供应链安全性的担忧推动了对自动化工具的需求，以及 SBOM 在整个行业的采用率上升，Anchore 提供了新的产品功能，见证了其开源工具的爆炸式采用，并继续主动为其客户和组织做好准备，以应对未来不可避免的违规和黑客攻击。

**软件供应链安全焦点**

2021 始于网络安全管理软件产品旭日攻击的余波，终于针对 Log4j 零日漏洞的多次利用，凸显了保护软件供应链的至关重要性。根据 Anchore 2022 年软件供应链安全报告，去年有 62%的组织受到软件供应链安全攻击的影响。软件供应商面临更大的风险，73%的供应商受到攻击的影响。

该报告还强调，组织正在应对这些风险，54%的组织将重点放在保护他们构建和使用的软件上。虽然美国关于改善国家网络安全的行政命令强调软件材料清单(SBOM)是供应链安全的重要基础，但 76%的组织计划明年增加 sbom 的使用。SBOMs 的重要性，加上开发过程中对自动化工具和持续安全检查的需求，推动了 Anchore 软件供应链管理解决方案的显著增长。

“最近的安全漏洞已经将软件安全的话题推到了各地商业对话的前沿。Anchore 首席执行官 Ziouani 表示:“软件供应链安全不仅仅影响软件行业，如今每个组织都需要加强其安全实践，以降低云原生应用程序中的风险。“上个月的 Log4j 零日漏洞强调了组织需要使用 SBOMs 和自动化工具来降低成功攻击的风险，并加快下一个零日漏洞的补救。”

**客户增长**

2021 年，Anchore 的 ARR 比上一年增长了 2.5 倍，因为组织希望主动保护其软件供应链免受不断增长的漏洞攻击。Anchore 的客户包括全球最大的企业以及政府机构。2021 年，Anchore 迎来了领先的财富 100 强组织加入客户名册，加入了数十家使用 Anchore 技术保护其软件供应链的全球 500 强组织和主要软件公司。新客户 [NVIDIA 使用 Anchore 来保护 NVIDIA NGC 上用于人工智能、机器学习和高性能计算的容器](https://anchore.com/press/anchore-secures-containers-for-ai-machine-learning-and-hpc-on-nvidia-ngc/)。

Anchore 在 2021 年将其政府客户增加了两倍多，增加了美国航天部队的小林丸计划，以及美国空军、美国国防部、美国总务管理局、美国海军、美国海军陆战队和国防信息系统局(DISA)的众多计划。Anchore 还扩大了与美国空军平台一号项目的关系，签订了一份价值 460 万美元的合同，以加强其软件供应链，重点是集装箱扫描技术和服务。

**产品扩展**

在过去的 12 个月里，该公司提高了其软件供应链管理能力，发布了多个 Anchore 企业平台。新功能包括:

*   通过使运行映像的安全状态对开发人员和安全团队可见，扩大了软件供应链的覆盖范围，降低了生产应用程序中包含不安全代码的风险。

*   通过补救建议和自动化工作流扩展了补救能力。

*   新的 [FedRAMP 政策包](https://anchore.com/press/anchore-delivers-new-automated-policies-that-accelerate-fedramp-compliance-for-containerized-applications/)使软件供应商和云服务提供商能够识别和解决容器化应用的合规性问题，并缩短获得 FedRAMP 操作授权(ATO)认证的时间。

*   能够自动对云原生应用进行 STIG 检查，并提供美国国防部应用所需的漏洞和 STIG 合规性漏洞的统一视图。

*   一个新的策略包，针对在 CISA 已知被利用漏洞目录中发现的漏洞发出警报。

**快速采用开源工具**

Anchore 见证了其两个开源软件供应链安全工具的加速采用，这两个工具可以轻松集成到开发流程和工具链中。Syft ，一个对容器映像和文件系统进行深度检查以生成 SBOM 的工具，现在已经有超过 400，000 次下载，在过去 5 个月中增长了 150%。 [Grype](https://anchore.com/opensource/) ，一个漏洞扫描器，现在有超过 500，000 次下载，在同一时期增长了 80%。Syft 和 Grype 在 GitHub 上总共获得了超过 4500 颗星，比 2021 年初增加了 10 倍。

Syft 和 Grype 在 2021 年的更新包括能够通过流行的 CI/CD 系统(包括 GitHub、GitLab 和 Jenkins)的插件生成准确的 SBOMs 和漏洞报告。Syft 还增加了对软件包数据交换(SPDX)标准的支持，这使得跨系统和组织共享数据变得容易。SPDX 是国际公认的 SBOMs 标准，由 Linux 基金会发起。

Anchore 首席技术官兼联合创始人 Daniel Nurmi 表示:“由于能够从各种软件工件中产生准确的 SBOM 和漏洞结果，开源社区和最终用户越来越多地采用 Syft 和 Grype，尤其是越来越多的项目维护人员希望将安全工具直接纳入他们的开发环境。“在对 Log4j 零日漏洞的最初和持续响应期间，从业者能够使用 Anchore 的 Syft 和 Grype 工具快速轻松地确定 Log4j 在他们的软件环境中的存在，即使该库嵌套在 Java 归档中的几个层次，这导致了下载和使用的大幅增长。”

“随着企业寻求加快运营速度，越来越多的企业开始专注于改善开发人员体验。RedMonk 首席分析师 Stephen O'Grady 表示:“这些组织面临的挑战是，如何为他们的开发人员提供顺畅的体验，同时改善他们的整体安全状况。“一种越来越受欢迎的方法是将 API 驱动的安全资源集成到他们现有的开发工具链中。这正是创建 Syft 和 Grype 的机会。”
