# WhiteSource 报告发现 NPM 漏洞得到快速修复

> 原文:[https://devo PS . com/whitesource-report-finds-NPM-vulnerabilities-fixed-fast/](https://devops.com/whitesource-report-finds-npm-vulnerabilities-fixed-fast/)

WhiteSource 今天发布了一份报告，该报告发现，影响广泛用于部署 JavaScript 应用程序的节点包管理器(NPM)的大多数漏洞，在国家漏洞数据库(NVD)中被指定为常见漏洞和暴露(CVE)之前很久就被解决了。

该报告基于对 WhiteSource 在其自身漏洞数据库中跟踪的漏洞的分析，并在 [AWS re:Invent](https://reinvent.awsevents.com/) 会议上披露，发现 NPM 软件包中超过 90%的漏洞在漏洞在 NVD 发布之前就已修复。

WhiteSource 的产品管理总监 Susan St. Clair 说，事实表明，总的来说，开源社区在漏洞被披露时就掌握着这些漏洞，而不是等待它们被分配一个 CVE 编号。

在 WhiteSource 报告发布之前，UA-parser.js 遭到破坏，这是一种广泛使用的 NPM，基于 JavaScript 库，可以从用户代理数据中检测浏览器、引擎、操作系统、CPU 和设备类型/型号。未知的恶意行为者发现了一种将恶意软件注入修改库的 NPM 的方法。该漏洞已被修复。

事实上，WhiteSource 报告指出，分配给最广泛使用的易受攻击国家预防机制版本的 83%的 CVE 可以在 CVE 发布之前得到补救。

在一系列引人注目的软件供应链违规事件之后，人们更加关注开源软件实际上是如何得到保护的。商业软件产品有一个分发列表，它们通常向该列表通知安全补丁。另一方面，开源更新需要由使用该软件的开发者来跟踪。个体开发者对任意数量的[开源](https://devops.com/?s=open+source)更新的关注程度可能会有很大差异。

圣克莱尔说，许多开发人员也不愿意立即更新他们的应用程序，因为他们不确定额外的代码会对他们的应用程序产生什么影响。她指出，例如，他们的应用程序可能具有不再向后兼容更新代码的依赖关系。

开发人员还需要确定该漏洞对他们的应用程序的影响程度，因为他们通常不会使用开源库的每个组件，所以可能不需要提供修补程序。

开源项目的维护者通常依靠他们的社区来帮助发现漏洞，并尽快创建补丁。当在开源库中检测到漏洞时，更常见的做法是向 NVD 报告，即使该问题的修补程序已经可用。

总的来说，圣克莱尔指出，开源社区要记住的最重要的事情是在跟踪漏洞时保持主动，然后实施 DevSecOps 最佳实践。有一点可以肯定的是，大量的恶意行为者也在密切跟踪这些漏洞。一旦漏洞通过 NVD 被披露，就会出现一波攻击浪潮，这并不罕见。

当然，挑战在于使用的开源软件越多，单个开发人员就越难控制应用程序组合中的每一个漏洞，而这些漏洞只会继续增长。从积极的一面来看，[供应商已经承诺投入超过 1000 万美元来帮助更好地保护开源软件](https://securityboulevard.com/2021/10/new-openssf-gm-sets-open-source-security-course/)。希望这项投资的回报会很快显现出来。
