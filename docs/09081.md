# CircleCI 为 macOS 开发增加 CI/CD 服务

> 原文：<https://devops.com/circleci-adds-ci-cd-service-for-macos-development/>

[CircleCI 今天为在 macOS 平台上构建应用的开发者增加了](https://circleci.com/docs/2.0/hello-world-macos/)一个专用的持续集成/持续交付( [CI/CD](https://devops.com/?=CI/CD) )版本的同名平台。

CircleCI 首席执行官吉姆·罗斯(Jim Rose)表示，该产品面向那些希望获得专用资源来加快构建过程，或者必须解决合规性和安全性要求的开发人员，这些要求阻止他们通过现有的 CircleCI 云服务使用共享基础设施。

Rose 指出，大多数在 macOS 平台上开发应用程序的开发者正在开发最终将部署在苹果 iOS 设备上的应用程序。Rose 补充说，鉴于这些设备的流行，使用 CI/CD 平台来加速这些应用程序的创建和更新的组织数量已经增加。CircleCI 估计，现在有超过 3000 万开发人员在 macOS 平台上构建应用程序，他们将受益于通过亚马逊网络服务(AWS)云访问的专用裸机基础设施上运行的 CI/CD 平台。CircleCI 的专用主机可安全访问比运行在虚拟机上的 CI/CD 平台多 50%的内核和两倍以上的存储容量。

CircleCI 提供的其他功能包括监控为同一管道中的多个平台开发的应用程序的性能。Rose 指出，许多部署在 iOS 上的应用程序也部署在其他平台上，如 Android、Windows 或 Linux。

CircleCI 还提供对该公司最近添加到其 DevOps 工具组合中的测试工具的访问。Rose 说，考虑到开发人员在试图通过苹果商店获得 iOS 应用程序的批准时面临的严格要求，这些测试工具尤为重要。

当然，对于试图在自动化软件开发任务和构建应用程序的艺术之间取得平衡的组织来说，开发人员的生产力是一个主要问题。虽然开发人员欣赏自动化，但他们不一定想感觉自己是在工厂里工作。考虑到对他们专业知识的需求，当他们觉得软件开发过程变得过于繁重时，许多开发人员会直接跳槽。虽然许多经理倾向于相信他们能够衡量开发人员的生产力，但是他们往往过于关注，例如，相对于实现的业务成果，编写了多少行代码。

最近的 CircleCI [调查](https://devops.com/circleci-survey-shows-the-need-to-prioritize-devops/)也发现超过一半(52%)的受访者不允许他们的软件开发者选择自己的工具。虽然构建和部署应用程序显然需要一致的方法，但如果组织不允许他们尝试不同的工具，他们会发现吸引和留住最好的开发人员是一项挑战。

然而，无论如何，随着 DevOps 最佳实践的采用，开发人员构建和部署应用程序的速度正在提高。尚不清楚组织是否能够吸收所有这些软件，但在新冠肺炎疫情之后，对推动数字业务转型计划的移动应用的需求从未如此之大。
