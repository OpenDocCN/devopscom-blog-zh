# 为什么现在是站点可靠性工程向左转移的时候了

> 原文:[https://devo PS . com/why-its-time-for-site-reliability-engineering-to-shift-left/](https://devops.com/why-its-time-for-site-reliability-engineering-to-shift-left/)

***通过采用多级方法进行站点可靠性工程并为您的团队配备合适的工具，您可以释放出影响整个服务交付连续体的优势***

在当今应用驱动的经济中，支持业务关键型应用的基础架构从未像现在这样重要。作为回应，许多公司正在招募[站点可靠性工程](https://devops.com/?s=Site%20reliability%20engineering) (SRE)专家来帮助他们提高可靠性、可用性、延迟和一系列其他影响客户满意度的指标。这些重要的新人力资源致力于保持网站和服务以最佳状态运行。

不过，这种业务重点有明显的局限性。服务由许多会影响关键业务服务的应用程序组件组成。如果您的工程师只关注后端性能，您如何确保可靠性融入到每个应用程序和每个代码更改中？

一些开拓者开始正面解决这个问题，让他们的 sre 承担支持 DevOps 生命周期关键阶段的新角色。我们的目标是尽早采取主动措施，确保从一开始就建立质量和可靠性。

## 测试团队开创的道路

为了理解为什么这个“[左移](https://devops.com/devops-shift-left-avoid-failure/#:~:text=The%20term%20%E2%80%9Cshift%20left%E2%80%9D%20refers,testing%20earlier%20than%20ever%20before.&text=Shifting%20left%20requires%20two%20key,continuous%20testing%20and%20continuous%20deployment.)如此重要，考虑一下软件测试功能中最近的革命性变化。不久以前，测试被认为是软件生命周期中的一个独特阶段，在应用程序开发人员的工作完成之后很久才开始。卓越测试中心发现了缺陷，并将代码返回给开发团队进行返工。

然而，随着开发人员转向敏捷冲刺，这种测试方法很快成为一个瓶颈。当卓越中心发现问题时，开发人员已经转移到其他项目上了。作为回应，新的[持续测试](https://en.wikipedia.org/wiki/Continuous_testing)项目出现了，由磨练了测试技能的新一代软件开发工程师组成。

这些人现在被嵌入到开发团队中，以确保从一开始就更好地构建应用程序。开源测试自动化工具帮助他们尽早地、经常地进行测试，而不会妨碍敏捷的交付进度。

值得注意的是，卓越测试中心并没有消失。相反，他们承担了更广泛的领导角色，现在负责协调整个企业的测试，以支持持续的软件集成和交付。

## SRE 的三级模型

SRE 团队可以从他们的测试同事的经验中学到很多东西。为了获得最佳结果，将可靠性专家整合到您组织的总体敏捷框架中是非常重要的。考虑采用以下三级模型，将职能转移到左边，并在 DevOps 生命周期的关键点整合熟练的工程师:

1.  **应用级别。**将 sre 嵌入开发和测试团队，以确保应用组件和新版本不会破坏可靠性。这些人专注于应用级服务目标、错误预算和集成到 DevOps 管道中。
2.  **系统级。**指派 sre 专注于贵组织发布系列中的应用程序。这些人支持发布和启动协调，评估系统架构准备情况，并确保您的组织满足系统范围的服务级别目标。
3.  **企业层面。**建立一个卓越的站点可靠性工程中心，由监督企业架构治理的工程师组成。这些人建立最佳实践，并选择所需的工具和资源，以支持您在公司范围内的站点可靠性功能。

## 扩展的工具集

为了支持这一扩大的角色，您将需要能够在整个 DevOps 生命周期中支持您的工程师的工具。您将需要能够看到和跟踪工程师发现的问题，并根据您的服务级别目标评估系统级就绪性和性能。您还需要一个跨越整个企业活动的全局视图。

幸运的是，新一代的解决方案现在可以提供帮助，由人工智能、机器学习和智能自动化提供动力。这些资源可以帮助您的团队处理来自不同工具集的大量数据，并将这些数据转化为行动见解。因此，您将拥有在新的、更广泛的角色中高效工作所需的所有生命周期分析。

## SRE 和左移:广泛的新利益

通过采用多级方法来加强可靠性工程，并用正确的工具武装您的团队，您可以释放影响整个服务交付连续体的益处。

您可以构建一个持续的反馈、协作和治理循环，涵盖新服务的设计、开发、发布和运营。您可以更好地管理配置更改、服务级别和错误预算。您可以更清楚地了解应用程序的可靠性和生产就绪性，以指导规划计划并帮助您确定投资的优先顺序。您还可以构建更加明智的业务案例来推动您的组织向前发展。向左移动显然是一个双赢的举措。
