# Fastly 为边缘计算环境添加了可观察性工具

> 原文:[https://devo PS . com/fastly-adds-observability-tools-for-edge-computing-environments/](https://devops.com/fastly-adds-observability-tools-for-edge-computing-environments/)

Fastly，Inc .本周在其产品组合中增加了一组可观察性工具，这些工具针对连接到其内容交付网络(CDN)的边缘计算平台进行了优化。

Fastly 工程高级副总裁劳拉·汤姆森(Laura Thomson)表示，该公司现在正在提供一套工具，使 DevOps 团队能够通过单一控制台实时捕捉日志数据和指标。

前两个可用工具是 Origin Inspector(观察网络流量，包括 HTTPS 响应代码)和 Domain Inspector(基于历史和实时数据访问指标)。总之，Fastly 现在提供对超过 200 个指标的访问，这些指标涵盖客户端、源、缓存、web 应用防火墙(WAF)、[【电子邮件保护】](/cdn-cgi/l/email-protection)、图像优化服务和其他，涵盖了自该服务启动以来生成的所有数据。

汤姆森指出，没有数据采样，该公司还提供与 Fastly 或第三方提供的 33 种存储和分析服务的集成。IT 团队现在可以通过多种日志格式记录 HTTP 请求和响应的任何方面，包括通用日志格式、JSON 或作为键/值存储。IT 团队还可以存储他们自己的日志，以确保他们保持对访问的完全控制。

Thomson 补充说，在接下来的几个月里，Fastly 将继续扩展这一可观察性工具组合，包括它为在边缘部署应用程序的客户提供的边缘观察者仪表板。

总的来说，[可观察性](https://devops.com/?s=observability)工具不仅用于改善流程和整体客户体验，还用于识别和阻止可疑活动。Fastly 正在为可观察性工具提供案例，这些工具使 DevOps 团队能够实现这些目标，而不必部署单独的可观察性平台。

当然，大多数 DevOps 团队多年来一直在持续跟踪各种预定义的指标。相比之下，可观察性工具旨在使查询日志、指标和跟踪变得可行，以便 it 团队可以在服务中断之前发现问题。当然，挑战在于，并非每个 IT 团队都知道如何形成发现潜在问题的根本原因所需的查询。

无论如何，随着部署在这些平台上的工作负载数量的增加，DevOps 团队将需要能够远程诊断边缘计算问题。IT 团队无法在每次出现问题时都前往每个边缘计算平台的物理位置。在许多方面，边缘计算的兴起将迫使向可观察性工具过渡，这将使 IT 团队能够更加主动地管理 IT 环境，而不是在指标已经超过阈值后才做出反应。

目前还不清楚这种方法将在多大程度上防止 It 问题的发生，但随着对 IT 环境(包括边缘计算平台)的了解越来越多，检测和修复问题所需的时间应该会大幅减少。
