# ACCELQ 进一步自动化了低代码、无代码应用的测试

> 原文:[https://devo PS . com/Accel 8-further-automates-testing-of-low-code-no-code-apps/](https://devops.com/accel8-further-automates-testing-of-low-code-no-code-apps/)

ACCELQ 今天发布了基于云的 [ACCELQ Live](https://www.businesswire.com/news/home/20220405005176/en/ACCELQ-to-Unveil-ACCELQ-Live-A-Revolutionary-Approach-to-Quality-Engineering-for-Cloud-Enterprise-Apps) 测试自动化平台，该平台持续集成了广泛的低代码和无代码应用程序开发和部署平台。

ACCELQ 首席执行官摩哂陀·阿拉迪表示，目标是简化在多云计算环境中运行复杂测试的过程。

自从新冠肺炎疫情开始以来，[低代码和无代码工具](https://devops.com/?s=low-code+no-code)的使用已经大大扩展。许多组织现在都依靠这些工具来加速广泛的数字业务转型计划。Alladi 指出，他们面临的挑战是，这些流程中有许多依赖于多个应用程序，这些应用程序之间有许多相互依赖性，很难进行测试。

ACCELQ Live platform 通过在 Salesforce 或 ServiceNow 平台更新时自动更新自身来降低测试复杂性。它还通过一个实时链接提供对基于业务流程建模的预构建无代码自动化资产的访问。该公司还在努力创建一个市场，软件供应商和服务提供商也可以在其中构建和部署预构建的测试资产。

Gartner 预测，到 2025 年，低代码和无代码技术的采用将增加近两倍。随着应用程序开发速度的加快，测试这些应用程序成为一项重大挑战。Alladi 说，组织能够跟上步伐的唯一方式是采用注入人工智能(AI)功能的测试自动化平台。他补充说，实际上，ACCELQ 的 ACCELQ Live 代表了云测试和企业应用民主化的努力。

随着组织越来越意识到他们对软件的依赖，进行测试的方式也在迅速发展。除了专门的测试团队，开发人员和应用程序所有者也越来越多地参与进来，因为测试应用程序的责任继续向左转移。组织面临的挑战是同时构建的应用程序数量显著增加。自动化许多低级测试功能的平台作为一种使人类能够将更多的时间和精力集中在测试更复杂的功能上的手段，正在获得越来越多的关注。Alladi 说，现在的目标应该是首先尽可能依赖测试自动化平台，然后在必要时雇佣人类来填补空白。

大多数组织要达到测试自动化的水平可能还需要一段时间。然而，随着每一天的过去，很明显，人工智能将在应用程序测试中发挥更大的作用。许多 DevOps 团队现在正在解决的问题是如何将测试自动化平台整合到一个更大的工作流程的上下文中。

与此同时，随着更多的例行错误被机器发现，应用程序测试人员精疲力竭的比率应该会下降。人工智能不太可能很快消除对专门测试团队的需求，但随着更多机械任务变得自动化，在应用程序环境不仅变得更加分散，而且更加复杂的时候，将有更多时间专注于提高整体质量。
