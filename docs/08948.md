# Quali 向 DevOps 自动化平台添加免费层

> 原文:[https://devo PS . com/quali-adds-free-tier-to-devo PS-automation-platform/](https://devops.com/quali-adds-free-tier-to-devops-automation-platform/)

Quali 已经为其 Torque 平台添加了一个免费服务层，用于自动化基础设施供应和应用工作负载部署。

此外，该公司还更新了软件即服务(SaaS)平台，以提供蓝图，通过这些蓝图，它与开源基础设施即代码(IaC)工具 [Terraform](https://devops.com/?s=Terraform) 进行了更紧密的集成，并支持定制标记，从而更容易生成报告和仪表板，从而更好地了解基础设施成本。

Quali 首席执行官 Lior Koriat 表示，免费层旨在使 Torque 平台更容易被 DevOps 专业人员的小团队和个人用户使用。除了提供与各种 DevOps 工具和平台的集成以及对样本蓝图库的访问，它还包括在 Amazon Web Services (AWS)或 Microsoft Azure 云上构建环境的能力。对免费层的支持是通过平台的现有社区版本提供的。

Koriat 表示，Torque 使 IaC 更进了一步，它还使得在基础设施上自动部署完整的应用环境成为可能，这些基础设施是使用蓝图而不是低级 Terraform 脚本作为代码提供的。他补充说，目标是通过扩展 Terraform 来提高开发人员的生产力，以便开发人员可以通过自助服务在多个云计算平台上旋转环境。

这种方法也大大降低了云平台被错误配置的可能性，Koriat 指出这是云安全专业人员存在的祸根。大多数开发人员缺乏确保正确配置云计算环境所需的网络安全专业知识。Koriat 补充说，当开发人员采用 Kubernetes 等复杂平台时，简化基础设施的供应尤为关键。

与此同时，Quali 的 Koriat 表示，随着部署在这些环境中的工作负载数量稳步增加，IT 领导者希望更多地了解云资源是如何被消耗的。当不再需要云资源时，开发人员往往不会记得取消供应，Koriat 指出，这导致了不必要的云成本增加。

目前还不清楚 DevOps 团队会以多快的速度从依赖低级工具转向通过 SaaS 平台访问的自动化框架。然而，随着应用程序环境变得越来越复杂，调配和管理 it 环境变得更加困难，尤其是因为许多开发运维团队仍在家中工作，以帮助限制新冠肺炎疫情的传播。

无论如何，需要减少管理 IT 基础架构所花费的时间和精力。毕竟，尽管应用程序环境的规模和复杂性都在增加，但是组成 DevOps 团队的人数却基本保持不变。更多地依赖自动化框架的需求已经达到了临界点，这一点对所有相关人员来说都变得越来越明显。

随着越来越多的组织开始在数字业务转型的[时代以软件公司的形式运营，](https://digitalcxo.com/welcome-to-digital-cxo/)很明显，过去的 it 管理方式将会发生根本性的变化。现在的问题是确定哪种自动化框架最有助于 DevOps 团队实现这一目标。
