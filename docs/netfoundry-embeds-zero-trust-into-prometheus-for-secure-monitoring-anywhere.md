# NetFoundry 在 Prometheus 中嵌入了零信任，以便随时随地进行安全监控

> 原文：<https://devops.com/netfoundry-embeds-zero-trust-into-prometheus-for-secure-monitoring-anywhere/>

**北卡罗来纳州夏洛特，2022 年 5 月 17 日**——NetFoundry 以领先的开源应用监控工具的本地安全网络连接庆祝普罗米修斯日。

该公司已经将 OpenZiti 直接嵌入到 Prometheus 中，这是在第一天和第二天操作中监控应用性能的事实标准。根据云本地计算论坛的数据，86%的云项目使用了 [普罗米修斯](https://openziti.github.io/articles/zitification/prometheus/part1.html?source=PromPR) 。

使用 OpenZiti 的 Prometheus 可以从任何地方抓取任何东西，并对外部网络攻击提供完全零信任保护。访问是通过仅出站的互联网连接进行的，因此不需要 VPN、复杂的防火墙规则、DNS 修改或 PushGateway。

OpenZiti 加强了 Prometheus 和其他资源之间的安全性和连接性，为网络和安全团队节省了时间和精力，尤其是在复杂的分布式部署中。

NetFoundry 首席执行官 Galeal Zino 表示:“开发者和运营商希望获得价值，但安全很难。他们不需要担心网络管道、VPN 或 DNS 的问题，也不需要在防火墙上打开入站端口。他们希望部署到任何零复杂性的云环境中。

“通过 Prometheus 的智能化，网络和安全会得到自动处理，因此您可以在几分钟内完成部署，消除数天或数周的工作量，同时显著降低攻击面和操作复杂性。”

作为 Linux 基金会的成员之一，NetFoundry 还开发了其他开源工具，包括用于管理 Kubernetes 应用程序的 Helm 和用于与 Kubernetes 集群通信的命令行工具 Kubectl。 [其他许多都在酝酿中](https://openziti.github.io/articles/zitification/index.html?source=PromPR) 。

Zino 说:“应用程序正大规模分布在云、物联网、边缘等连续体中。让网络负责安全已不再有意义。它需要内置到每个应用程序中。有了 OpenZiti，你需要的只是商品出境互联网。没有人能做到这一点。没有人能让工业强度的安全变得简单和免费。”

去年，甲骨文支持 OpenZiti 供甲骨文云基础设施的开发者使用。Oracle 还推荐使用 OpenZiti 来降低互联网上 Java 应用程序中的 Log4J(或任何 CVEs 和零日)风险。

CI/CD 自动化提供商 Ozone 使其平台和 Prometheus 无需入站端口或复杂的防火墙规则即可连接到客户网络。

臭氧技术主管 Moteesh Reddy 说:“Ziti 让我们能够创新并推动具有竞争优势的新业务。如今，Ozone 是唯一一款跨任何平台自动部署私有集群的 CI/CD 工具，为客户提供零信任、安全的私有 Kubernetes 集群应用交付。”

OpenZiti 可以下载，在本地运行，由 ISV 托管。它也可以作为一项由 NetFoundry 托管的服务，提供免费增值服务选项。
