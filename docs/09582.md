# 案例研究:利用 DevOps 改变医疗保健

> 原文：<https://devops.com/case-study-transforming-health-care-with-devops/>

今天，人们对医疗保健体验有了新的、越来越高的期望。从预约提醒和通信到远程医疗访问和药房体验，患者需要各种数字选项。事实上， [70%](https://www.accenture.com/us-en/insights/health/todays-consumers-reveal-future-healthcare%23:~:text=For%2520instance,%2520in%25202019,%252070,to%252039%2520percent%2520in%25202016) 的消费者更有可能使用提供数字提醒和后续护理的提供商。

成功实现数字化转型的医疗保健机构从中获益——53%的消费者更有可能使用提供远程监护设备的提供商。致力于开发运维的组织，如 [希望之城](https://www.cityofhope.org/) ，不仅经历了数字化战略的加速实施，还实现了工作效率的大幅提升，同时在整个 IT 转型过程中保持了安全性和法规要求。

疫情在 2020 年强调了数字化医疗保健服务。前所未有的变化推动组织通过基于云的计算获得一系列全新的优势和挑战。 [截至 2021 年 8 月，73%](https://www.google.com/url?q=https://www.helpnetsecurity.com/2021/08/04/healthcare-devops/&sa=D&source=docs&ust=1654812499326357&usg=AOvVaw2j4UHowtVx56sHKsY8xTLU) 的医疗机构采用了 DevOps。

也就是说，不幸的是，DevOps 的成功可能[难以实现](https://devops.com/why-your-devsecops-initiative-will-fail/) 。根据 Gartner 的调查，“由于组织学习和变革方面的问题， [75%的开发运维计划将无法达到预期](https://www.gartner.com/smarterwithgartner/the-secret-to-devops-success/)。”然而，好处远远大于挑战，通过专门的组织转变，DevOps 使所有人受益，从医疗保健组织到消费者。

### DevOps 在医疗保健方面的优势

快速在线登记系统、健康相关通知、易于访问的患者门户网站、电子药房选项和预约可用性监控等现代化便利设施使患者参与度大幅提升。通过自动化软件交付管理(通过 CI/CD 管道)，医疗保健组织以更高的质量和安全性更快地交付更好的功能和解决方案，从而在市场中保持竞争力。

DevOps 加快了开发周期，解决了 bug 并快速部署了更新。CI/CD 管道自动化减少了瓶颈，减少了不必要的开支。有了 DevOps 工具链，组织可以优化性能，并推动开发人员、运营、安全和质量团队之间的协作文化。随着组织变得更加数据驱动，他们可以通过增强的洞察力和分析来超越竞争对手。虽然好处显而易见，但一些组织将对监管和安全性的担忧列为尚未采用 DevOps 的原因。

### 法规遵从性和安全性 devo PS

法规遵从性和安全性对于高度管控的行业(包括医疗保健行业)至关重要。虽然组织必须维护行业标准，但推迟采用 DevOps 可能更不利。

监管机构经常向组织施压，要求他们尽快弥补安全漏洞。解决这些问题的新软件版本可能需要几个月才能完成。然而，DevOps(和 DevSecOps)自动化了最耗时的流程，并允许医疗保健组织快速有效地弥补安全漏洞。此外，使用左移的安全方法(静态代码分析、动态代码分析、威胁漏洞管理、 [DevSecOps](https://devops.com/?s=DevSecOps) 等)。)，安全性内置于 CI/CD 管道中。

当组织利用他们的 DevOps 方法来自动化配置更改时，他们可以建立一个可重复的标准化流程。DevOps 还提供了整个管道的可见性，允许组织扩展和监控合规性，并有效地识别风险。

### 用 DevOps 加速释放速度

有了 DevOps，团队变得更有生产力，并且在不牺牲安全性的情况下提高了发布速度。更快的上市速度提供了许多优势，尤其是如果一个组织想要率先引入一个新概念或产品。它还让消费者对及时的发布和更新感到满意。

然而，更好的发布速度几乎总是需要组织变革。当以前孤立的团队之间的过程、工具和协作被 DevOps 转换彻底检查时，发布速度有机会飞速增长。像希望之城这样的医疗保健组织通过实施 DevOps，在发布速度方面有了显著的提高。

### 希望之城如何成功领养 DevOps

希望之城是一家顶级癌症研究和治疗中心，从 2022 年开始进行 DevOps 改造。像许多医疗保健组织一样，希望之城本质上并不是一个软件组织。然而，作为一个研究机构，它拥有大量关于治疗和结果的数据，需要进行分析。

为了跟上医疗保健行业的数字化步伐，希望之城的工程团队从头开始创建了一个新的软件开发流程。它需要定制的软件应用程序，以便在精准医疗数据和系统部门进行持续和及时的数据分析。

与许多正在经历 DevOps 转型的医疗保健组织一样，希望之城在转型过程中遇到了必须克服的共同挑战，包括孤立的开发、运营、安全和质量团队；缓慢的发布和批准周期，以及对速度、性能和安全性的极少甚至没有可见性。

转型还必须在不雇佣 DevOps 工程师或顾问的情况下完成。展望不认为自己是“传统”软件组织的医疗保健组织的数字化未来，希望之城通过[无代码](https://devops.com/?s=no-code) DevOps 编排实现了其产品的现代化和升级，而没有了手动编码流程，该流程会显著延迟组织进入数字空间的时间。

希望之城的 DevOps 转型有助于加快软件交付，无代码方法降低了不熟悉 DevOps 工程的人的准入门槛。无代码开发运维流程编排可快速提高组织实现开发运维成熟度和跨所有应用和业务部门扩展的速度。它消除了 CI/CD 管道中耗时的编码过程，同时允许组织跟踪治理并转向更安全的软件交付。

希望之城成功开发了 DevOps 管道，可由希望之城的 IT 人员进行建设和管理。

结果是实质性的，包括:

*   开发人员的工作效率提高了 25%
*   生产力提高 80%，能够在 30 分钟内建立一个全面的集成 DevOps 管道，而不是 2.5 小时
*   最初的管道在短短两个月内建成、运行并提供价值

现在，希望之城可以获得实时洞察，使团队能够快速解决任何问题。虽然它记录了 it 组织内的显著改进和软件开发的速度，但 DevOps 明显给了团队时间来做希望之城最重要的事情——提供创新，帮助癌症成为过去。

随着医疗保健组织转向更多基于云的产品，他们需要正确组合工具、流程、技能和以数字化为重点的文化转变。有了这些，希望之城成功实现了 DevOps 转型，整个渠道的发布速度和可见性提高了 80%。
