艾伦·希梅尔:嗨，大家好，艾伦·希梅尔，DevOps.com 又来参加 DevOps 聊天了。本期 DevOps Chat 的嘉宾是 AppDynamics 的贾斯汀·沃恩·布朗。贾斯汀欢迎来到 DevOps 聊天室。

贾斯汀·沃恩-布朗:谢谢艾伦，很高兴和你在一起。

艾伦·希梅尔:很高兴你能来。贾斯汀，你是 DevOps 行业的资深人士，如果有这样的事情，在相对较短的时间内。但你现在在 AppDynamics 挂帽子了。你在 AppDynamics 工作多久了？

*Justin Vaughan-Brown:* 我一直在 AppDynamics 工作，那是十月的最后一天，大概有两三个月了。

艾伦·希梅尔:太好了，刚好够我猜测那里的地形。正如我们之前所说的，Justin，我想我们的大多数观众，这是忙碌的两到三个月，你知道，在年底和市场上发生的一切之间，你有一个很好的机会，我想你不仅可以了解 AppDynamics 在做什么，还可以了解市场在 DevOps APM 等方面的发展趋势。正确吗？

贾斯汀·沃恩-布朗:绝对是艾伦。我认为非常有趣的是——实际上在加入几周后——11 月在拉斯维加斯的 App Sphere 与客户、合作伙伴和同事一起度过了一段时间:这绝对令人着迷，所以这真的让 APM 客户大开眼界——你知道，AppDynamics 的客户实际上是如何使用和采用 DevOps 的，以及他们如何利用我们的技术来帮助他们采用 DevOps。

*Alan Shimel:* 是的，正如你所知，我实际上是在 App Sphere，我自己也在 App Sphere，这是一个很棒的会议，是一个很棒的以客户为中心的会议，因为那里有很多用户，真正的实时用户，对吗？–这些会议上罕见的、濒临灭绝的现场用户–首先不是业务开发人员，也不是销售人员，而是真正的用户，我自己也有过一些精彩的对话。

Justin，我想花一点时间来谈谈 APM，因为有时当我们看软件开发生命周期以及开发运维活动时，你知道很多人都对配置管理、木偶、厨师、ansible 和 salts 之类的东西感到兴奋和困扰。然后，你知道，他们从配置管理转移到 CICD，当然这是一个独立的世界。

但是，当我们看一下部署后的情况时，你知道，好吧，现在我们已经交付了，而且还在持续交付，但是这会产生什么影响呢？事情进展如何？我们的客户满意吗？我们是否交付了更好的产品，对吗？它们的表现是否符合预期？

你知道所有这些你认为会让 APM 成为城堡之王的东西。但很多时候并不是这样:它有点被包含在整个 DevOps 对话中，或者可能有点在外面。

我感兴趣的是你对此有什么看法？

贾斯汀·沃恩-布朗:这是一个有趣的观点，我明白你的想法。我认为这可能是因为，你知道，从历史上看，许多供应商越来越多地成为你所谓的传统开发方，并产生了很多兴趣和采用。但最终我们看到了公司的不断壮大，我认为真正令人大开眼界的是——在应用领域——看到应用动态在企业开发运维采用中的重要性。

我认为最特别的是——你说到点子上了——如果你看看这些应用程序在生产中的影响，它们实际上是如何执行的，我们的问题在哪里，应用程序本身可以进行微调，并确保你有最好的客户体验。

我认为总的来说——这是一个概括，当然也有例外——我认为我们已经超越了应用程序的状态——除了某些更困难的情况，比如说， 你知道黑色星期五等等，有时一些组织可能会面临挑战，但我认为主要是我们看到的是大多数应用程序在我所说的基础水平上执行，我们现在真正要做的是调整它们:确定绝对最小的一点延迟或停滞或问题，无论它们可能位于何处，以便这些应用程序的性能，无论它们在哪里交付，你知道， 无论是在平板电脑或移动设备上，还是在 PC 或其他设备上，都具有极高的性能，同样，无论平台、操作系统等如何，我认为这已经成为压倒一切的需求之一。

我认为它现在发展得更快了——你知道，我们最近推出了 Business IQ，它实际上对它进行了更深入的研究，并实际上查看了你发布的应用程序的业务优势，消费者在哪里使用该应用程序，他们在哪里，他们在用户旅程中处于什么位置，他们在购买周期中处于什么位置。你会发现，许多消费者正经历着即将购买的过程，但随着交易的进行或不进行，实际购买过程会有所延迟。

因此，我认为它肯定在发展，我认为我们现在看到了 APM 的绝对相关性。

*艾伦·希梅尔:*明白了。绝对的。Justin，让我来——正如我之前所说，你是 DevOps 行业的资深人士，而 AppDynamics 并不是你在 DevOps 列车上的第一站。

如果您能与我们的观众分享一点点，我们希望听到您对的看法。这就是 APM，它的与众不同之处是什么？还有什么？让我们看看 DevOps 领域的趋势，以及它与 APM 的关系。

你还看到了什么样的趋势？

*Justin Vaughan-Brown:* 我认为我们看到的一个有趣现象是——你知道，你已经看到了 Docker 在过去 18 个月到 24 个月中的发展，我认为这和集装箱化的全面兴起产生了巨大的影响。

我最近看到一家银行的一篇文章——他们打算报告他们最近发表的关于——我要说的是集装箱化和采用的显著增长，这几乎是一个没有回头路的例子。

所以我认为现在真的取决于你在哪里采用容器，以及采用的程度，比如说。这就是我们看到我们有能力帮助他们的地方；例如，他们可以轻松地集成和监控 docker 容器的性能，以确保您优化并最大限度地发挥潜力，而不会成为一个黑箱区，在那里您无法获得您所寻求的洞察力的粒度。所以这绝对是一个领域。

我还认为，随着云的采用，能够跨多种云环境进行监控，正如我所说的多种云，无论是 Amazon AWS、Microsoft Azure 还是其他。

因此，我认为有必要保持平台和云不可知论。这是我最近在分析师会议上看到的情况，他们从运营的角度谈论了如此多的企业成为云代理，然后不仅仅关注一家供应商，而是能够根据他们正在开发的特定应用程序或计划等进行切换和更改。所以我认为这是两个明确的领域。

我想我可能要强调的最后一个是微服务，例如，APM 能够自动监控部署在弹性基础架构中的微服务，因此您可以查看威胁争用的相关问题，然后放大并确定和解决其根本原因。

因此，看到所有这些围绕 APM 运行的技术绝对有助于采用。如果你恰当地采用这一点，显然它们是非常有利的。如果你与他们斗争，那么他们实际上会让你倒退，你会比没有踏上那段旅程时更糟糕。这是艾伦第一时间想到的三个问题。

艾伦·希梅尔:贾斯汀，我不反对他们中的任何一个，我认为他们都是对的。

让我再抛一个——这可能对你来说有点难以理解——但作为 DevOps 空间的一名观察员，贾斯汀，我觉得有趣的是，当我们第一次开始谈论 DevOps 时，每个人都有点说，“哦，是的，DevOps 对创业公司来说很棒，因为这是他们的 DNA，他们是通过——你知道，不是选择来做的:他们必须这样做。这是——devo PS 就是为这些人设计的。”

你知道，这一期的“企业发展计划”几乎就像一种不同的动物或一套不同的条纹。你知道，我一直是主张开发运维就是开发运维的人之一，你必须认识到谁是你的客户，谁是你的组织，但是开发运维的原则适用于各种规模的组织。

但是现在，你知道，新的一年开始了，我们在 DevOps 这件事上又花了几年时间，感觉 DevOps 几乎是一个大组织开展业务的方式，甚至可能比初创公司更是如此。你对此有何感想？

贾斯汀·沃恩-布朗:我觉得你绝对是艾伦的偶像。我的意思是，我认为我们都经历过这一过程，我们观察到了某些趋势以及市场是如何演变的。

举例来说，如果你看看去年在旧金山和伦敦举行的 DevOps 企业峰会——这两个地方都有很多人参加——然后如果你看看这两个地方的议程，从我的角度来看，终端用户方面的大多数演讲者都代表着企业组织——家喻户晓——事实上，如果有什么不同的话，那些较小的组织只是少数，甚至是——你知道， 我还认为，对于较大的组织来说，它们不仅仅是纯粹的网上商店，它们没有必须处理的遗留基础架构:许多这样的公司都有 50 到 100 年的历史，已经存在了很长时间，但仍在进行这一勇敢的旅程。

我想引用 Gene Kim 去年的一段话，他说真正让他印象深刻的是——这让他真正想到了这些大企业、这些组织中的英雄，他们，你知道，可能我会说，在他们的职业生涯中，他们在年龄和职业生涯中处于中级水平，他们做出了这些勇敢的决定，他们以不同的方式做事:他们可以像正常人一样生活。

在这样做的过程中，他们在尝试不同的事情，他们开始了新的旅程，他们带着自己的团队，他们更加紧密地合作。

我认为这是——这是那种——我认为 Gene Kim 绝对是那种英雄，是变革的催化剂——坦率地说，我认为 DevOps 绝对适合企业，并将继续如此。

*艾伦·希梅尔:*有道理。贾斯汀我们时间不多了。如果你不介意的话，让我把这个话题转回到 App Dynamics。我们已经讨论了这些趋势，我们看到市场正在形成，这对 AppDynamics 意味着什么？

贾斯汀·沃恩-布朗:我认为这意味着继续帮助企业客户的大量机会。最重要的是，我认为——你知道，从个人层面来说，我对最近发布的 Business IQ 感到非常兴奋，因为我认为这使 DevOps 团队现在能够获取有关该应用程序如何被使用、在何处以及以何种方式使用这些可视化用户旅程的信息——然后实际获取这些信息并将其呈现给业务部门——这使它从纯粹的 DevOps 对话转变为业务对话，只是说，“看，这是——你知道，我们都在过去几个月中为该应用程序而辛苦工作。这是对您业务的实际影响；这就是正在发生的事情，这就是价值。”

然后，也许企业会回过头来说，“你知道吗？有趣的是，这个地区的销售非常好，但不是在这个地区或这个客户群，你知道，我们的金卡会员运营商绝对喜欢这个促销活动，但我们的银卡持卡人不喜欢。也许我们需要以某种方式激励他们，“你知道，等等。

因此，正如我所说，DevOps 团队正在进入一个更具业务性的对话。我们知道这一主题，如公民开发者和明显的影子 iTeam——越来越多的企业参与到这些决策中，这是 DevOps 帮助推动对话的方式，而不是作为接收端接受企业的请求，他们实际上是推动讨论，这使他们处于非常有利的位置。

*艾伦·希梅尔:*优秀。Justin，我们快没时间了，但我想问你最后一个问题，这也是我在 DevOps Chat 上问很多客人的问题。

对于我们的听众来说，如果你必须推荐一本书，而不是通常你会从 DevOps 播客中期待的 Gene Kim Phoenix 项目，那会是什么书？

贾斯汀·沃恩-布朗:我想我会回到我大约 20 年前上过的一本书和一门课，史蒂文·科维写的《t 2:成功人士的七个习惯》,它几乎打破了我所知道的大部分东西，并以另一种方式重新构建，给了我一个全新的视角，所以我强烈推荐它。

如果你时间紧迫，我认为 YouTube 上有一些视频，你可以在 15 分钟内获得这些视频，所以喝点咖啡休息一下是非常值得的。

艾伦·希梅尔:当然。但那是经典。对于没有读过这本书的听众来说，这本书真的应该排在你的清单的首位。

嘿，AppDynamics 的贾斯汀·沃恩·布朗，感谢你在 DevOps Chat 上成为这一集的嘉宾。期待您的再次光临，分享更多的市场见解。

贾斯汀·沃恩-布朗:我的荣幸。非常感谢，艾伦。

艾伦·希梅尔:好的。这是艾伦·希梅尔为 DevOps.com 和德沃普斯带来的访谈。我们将在下一次 DevOps 聊天中再见。
