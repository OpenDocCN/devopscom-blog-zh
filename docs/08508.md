# 企业的下一个前沿:转向产品 IT 运营模式

> 原文:[https://devo PS . com/the-next-frontier-for-enterprises-move-to-a-product-it-operating-model/](https://devops.com/the-next-frontier-for-enterprises-moving-to-a-product-it-operating-model/)

在技术领域，很少有什么事情像企业 IT 最近从推动者上升为战略业务驱动者这样具有戏剧性和影响力。肩负着推动下一代数字业务模式的任务，如今的 IT 团队很快意识到一个残酷的事实，即他们必须从传统的基于项目的运营模式向以产品为中心的运营模式进行全面的战略转变。

传统的基于项目的运营模式将资源集中在特定的应用程序或项目上，导致体验分散，与 IT 组织或公司的其他部分脱节。例如，在许多情况下，一个团队可能在后端系统上工作，而完全不同的团队在前端应用程序或更新数据库上工作。基于产品的运营模式采用更全面的方法来确保项目是集成的，团队同时朝着统一的目标工作。

随着数字化迫使 IT 领导者快速行动，跳出常规思维，拥抱数字世界和物理世界之间模糊的界限，规划/构建/运行的日子屈指可数。以产品为中心的运营模式，有时也称为 DevOps 运营模式，可加速整个组织的数字化转型，并巩固企业的竞争地位。公司必须过渡到以产品为中心的模式，扩大敏捷项目的规模，以支持端到端创新，减少孤立的运营和项目。

已经采用以产品为中心的运营模式的公司正在迅速实现收益，但对于许多 IT 部门来说，他们还没有体验到或完全理解为什么这种转变势在必行。

## **是什么推动了向产品 IT 运营模式的转变？**

随着数字化的深入发展，首席信息官们意识到将 IT 和业务目标结合起来是快速应对市场变化和保持竞争力的关键。自 2018 年以来，波动性大幅回归全球股市。根据瑞士瑞信银行的数据，由于经济处于不断变化的状态，加上快速的全球化和技术带来的前所未有的破坏，公司的平均寿命为 20 年，而 20 世纪 50 年代为 60 年。总的来说，不断变化的业务环境让这一点变得再清楚不过了——敏捷性、对新市场、新产品的渴望、对行动的偏好和快速失败的方法将是 IT 领导者未来的新使命。

为此，大多数首席信息官正在用支持特定业务能力的产品线来取代分散、孤立的项目组合。推动向产品 IT 运营模式转变的关键因素包括:

*   **需要整体转型** :孤立的功能在数字商业世界中没有立足之地，数字商业世界依赖于跨结构、文化、流程和技术的数据和信息的集成。为了实现整体转型，组织必须在组织指导方针与团队在过程、架构、技术、工具等方面的自主性之间取得平衡并建立交叉协作。

*   **传统 IT 运营模式缺乏一致性** :大型团队和多个组织团队(如 IT、财务和业务)之间缺乏一致性，阻碍了组织执行端到端的数字化战略。转向产品 IT 运营模式有助于引导组织单位朝着共同的目标前进。它可以促进支持不同产品和领域的不同 DevOps 团队之间的协作。认识到现有 IT 治理的失败和对共享数字化转型愿景的迫切需求， [IDC](https://www.idc.com/research/viewtoc.jsp?containerId=US41789117) 预测 40%的首席信息官将采用新的数字化治理模式。

*   **对速度的需求** :大多数公司表示，他们的数字化努力受到传统 IT 运营模式的不灵活或缓慢流程的阻碍。采用面向产品的 IT 管理方法可以帮助组织定义并同意符合其准备程度和偏好的变更速度。据 IDC 称，到 2021 年，65%的首席信息官将把敏捷/开发运维实践扩展到更广泛的业务领域，以实现创新、执行和变革所需的速度。

*   **新旧技术之间缺乏集成:新旧技术之间缺乏集成是阻碍组织实现数字化全部潜力的一个关键因素。现代公司需要平衡创新、实验、新技术和工具的采用与企业对业务成果、合作、风险管理和合规性的需求。因为以产品为中心的模型采用整体的、端到端的方法，所以它是大规模集成技术、流程和团队的理想选择。T3】**

## **拥抱新一代运营模式**

为了与脸书、优步、Spotify 等更新、更灵活的公司竞争，认真对待持续增长的传统公司正在不遗余力地拥抱产品 IT 运营模式。首席信息官们已经从可靠的 IT 服务提供商转变为推动业务转型的战略合作伙伴。当今的首席信息官们正在推动业务创新，寻找竞争优势的机会，而不是提供临时的、孤立的服务。随着云优先和人工智能优先成为组织的新使命，重新设计 IT 运营模式正迅速成为创造价值、竞争优势和增长的核心。
