随着组织试图满足更快的交付计划并改善开发和运营之间的协作，DevOps 已经成为一个热门话题。随着苹果、谷歌和微软让开发人员和运营人员构建和部署跨平台应用变得更加容易，在更短的时间内可以测试更多的应用。问题随之而来，测试在这样的 DevOps 策略中处于什么位置？Nikhil Kaul 概述了 DevOps 的最新趋势，特别是过去一年测试行业发生的变化。他探讨了为什么测试变得比以往更加重要，以及测试如何促进向 DevOps 的转变。最后，Nikhil 介绍了构建可扩展测试框架的方法和重要的可重用性实践，这些实践可用于从开发和测试到运营，以减少部署后的调试时间。

在加入 SmartBear 之前，Nikhil 曾在应用程序生命周期管理解决方案的领先提供商 Digité担任各种职务，包括高级软件执行官。在那里，他深入了解了软件开发和测试市场。Nikhil 获得了乔治敦大学的工商管理硕士学位。他积极参与开发和测试社区的讨论。他最近的想法可以在《福布斯》杂志和《CIO.com》杂志中找到。在推特上关注他 [@kaulnikhil](http://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Ftwitter.com%2Fkaulnikhil&esheet=51354426&newsitemid=20160602005446&lan=en-US&anchor=%40kaulnikhil&index=11&md5=f325a050911ffa393f62e95fdf83e546) 或者阅读他的[博客](http://cts.businesswire.com/ct/CT?id=smartlink&url=http%3A%2F%2Fblog.smartbear.com%2Fauthor%2Fnikhil-kaul%2F&esheet=51354426&newsitemid=20160602005446&lan=en-US&anchor=blog&index=12&md5=67f744eacebea71303d7baf23d51c1ea)。
