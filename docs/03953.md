## 副本

艾伦·希梅尔:嘿，大家好，我是艾伦·希梅尔，是时候开始另一个 DevOps 聊天了*。*今天的 DevOps 聊天嘉宾是 Jay Chapel，ParkMyCloud 的首席执行官兼联合创始人。嘿，杰，欢迎来到 DevOps 聊天。

嘿，艾伦，你好吗？谢谢邀请我们。

Shimel: 哦，谢谢——谢谢你能来，Jay。Jay，我邀请你来是因为我想具体谈谈我第一次遇到的一个问题，我猜是七、八、九年前或者更久以前，我称之为云蔓延。你知道，亚马逊和 AWS 的好处在于，它让构建新实例和添加新存储变得如此容易。AWS 和诸如此类的东西不太好的一点是，它让建立新的实例和存储和诸如此类的东西变得如此容易。

Jay，说到点子上了，我知道很多很多公司真的很难控制，甚至更不了解他们实际上在花什么钱，以及他们在云中实际拥有什么资产。你知道，对于任何收到 AWS 账单或试图查看 AWS 账单的人来说，我的美国电话电报公司无线账单可能更容易理解，如果你能相信的话。

所以，这是个现实问题。Jay，我想这是你们帮助公司的一件大事，这就是问题的症结所在。

**礼拜堂:**对，没错。我认为，我们主要看到了三个与亚马逊、Azure、谷歌和/或甚至多云相关的客户问题。其中之一显然就是你刚才提到的成本可见性。第二，一旦你获得了可见性，它就在你的掌控之中。

因此，我很好地了解了我所拥有的东西，我如何使用保留实例或开关之类的东西来控制这些成本。第三件事是，作为开发运维流程的一部分，真正实现自动化。那么，我该如何实现自动化，作为我如何在云中旋转、旋转并真正将其集成到我的基础架构中的一部分。

**Shimel:** 因此，随着自动化的发展，我们倾向于认为一旦东西实现了自动化，我们就可以降低成本，但我们真的可以通过自动化和断断续续的东西来实现。你知道，杰伊，我们可以稍后再谈，也许在另一次 DevOps 聊天中，我们可以进入这一切是如何工作的技术细节。

但是，我今天想将我们的讨论重点真正放在业务所有者或业务利益相关方身上，如果你愿意，他们的任务是将云预算保持在预算内或预算外，他们的任务是确保有足够的资源来完成需要完成的工作，但他们没有多余的资源，我的意思是，这是他们的一个主要问题。

Jay，在不指名道姓的情况下，你能给我讲一些战争故事吗，如果你愿意的话，讲讲你见过的一些公司在加入 ParkMyCloud 之前陷入的最糟糕的困境？

**Chapel:** 是的，所以，我的意思是，就你对企业主的看法而言，我们昨天刚刚与一家迁移到 AWS 的大公司进行了交谈，他们实际上有一个团队在与我们通话，我想他们有五六个人，他们都有成本管理或云报告或某种财务控制方面的头衔，尽管他们都是技术人员。所以，人们——公司让人们负责这个问题，并花钱帮助管理和减少资金。

所以，这是一家拥有大量资源的大型公司，但是，我要说的是，诞生在云环境中的公司，比如过去五年或十年中建立的所有基础设施都在云环境中的金融科技、医疗保健、社交媒体或广告公司，实际上是将这种责任更多地放在了开发人员身上。

这有点有趣，尽管他们是技术人员，但他们也负责控制云成本，因为有人敲他们的肩膀说，“嘿，这个账单上涨了。你需要处理好这件事。这里有一个可见性—现在您需要将控制和机制放在适当的位置，这样人们就不能随意编造事情，并且有一些适当的流程。”

当然，这是一种基于公司规模的混合，但这肯定是我们看到的一个问题，这些人是我们每天都要交谈的人。

绝对的。那么，Jay，我不知道你们是否积累了任何指标或任何东西，但请给我一个公司的规模，以及他们的云账单是多少。本机云和刚刚开始转型的公司之间可能存在差异。但是，你知道，我们有一个计算的领域，我们谈论的是什么样的预算？

教堂:嗯，这个很有意思。所以，很多你实际上在电话中交谈的人并不真正知道他们每个月在云上总共花费了多少。*【笑声】*

哦，我知道了。

**Chapel:** 但是他们确实知道基础设施的规模，所以他们能够告诉你，“嘿，我们有 4000 个实例，我在存储方面有这么多，我在数据库支出方面也有这么多，”诸如此类的事情。但是他们真的不知道他们的总账单。

但是，我们与—我们刚刚与一家有 4，000 个实例的公司谈过，另一家有 7，000 个实例，另一家有 12，000 个实例。所以，它真的无处不在，但如果你有 12，000 个实例，你每个月在亚马逊、Azure 或谷歌上花费数百万美元——在这种情况下，它恰巧是亚马逊。

但更有趣的是，我们看到的是这样一个非常好的例子，我们正在看一家总部位于加拿大的公司，它使用我们的平台，他们在亚马逊有大约 430 个实例，他们停放了其中的 26 个，但他们每月仅停放 26 个就节省了 18，000 美元。

因此，有趣的是实例有多大，然后您可以在它上面节省多少钱。因此，如果它是小型或微型的，您不会节省很多钱，但如果它是数据库和/或大型、超大的，您知道，庞大的实例，您会在这些方面节省很多钱。

因此，您的支出并不总是基于实例数量，而是基于您在云中运行的那些实例或数据库的大小。

**Shimel:** 明白了，明白了。这太疯狂了，我的意思是，当你想一想，他们不知道他们花了多少钱，Jay，而且他们有——你知道，你在和一个人说话，当我过去不得不安装新服务器时，他会拿起螺丝刀进行机架安装，对吗？*【笑声】*所以，400 是很多螺丝钉。

**礼拜堂:**是啊。*【笑声】*

Shimel: 但事实上，他们甚至不知道他们在花什么，我的意思是，在某些时候，这几乎是疏忽，对不对？但是，我明白了。我明白，因为很难把这些联系起来。

嗯，很难把这些联系起来，但是，你知道，这也取决于你在和谁说话。如果你与首席财务官或首席信息官或首席技术官交谈，他们可能知道他们在云上花费了多少。但是，当您在组织中向下移动到开发运维工程师或 IT 运营或总监级别时，他们可能知道也可能不知道总账单，但他们绝对知道他们在基础架构方面负责什么。更重要的是，他们如何对基础架构进行计费？

因此，我们与一些采用纯按存储容量使用计费模式的公司进行了交谈，他们甚至提高了成本。假设一些组织每月花费 100，000 美元在云上，他们实际上将花费提升成本，并说，“好吧，我将在这上面投入 10%或 20%来为您管理它。”我们与一些根本没有退款的公司谈过，他们只是支付了全部费用。

所以，从这个意义上来说，它真的非常类似于传统的数据中心；所以，这仅仅取决于人们把他们的成本模型放在内部的方式，以及他们是否把它收回到项目和其他东西上。当你与人交谈时，肯定会有非常有趣的对话。

绝对的。Jay，既然我们都在这里了，你为什么不介绍一下 ParkMyCloud 的背景呢？我意识到我们以前并没有真正做到这一点，所以到底——你知道，给我们一点背景。

教堂:是的，当然。我们是一家成立两年的公司，总部设在北弗吉尼亚。我们就在杜勒斯机场附近，如果人们知道在哪里的话。基本上，我们所做的是，通过关闭不使用的东西，帮助人们优化他们的云成本。

因此，您可以举一个简单的例子，客户在云中有大约 100 个实例，其中 55 个是生产实例，45 个是非生产实例。嗯，这些非生产实例通常不需要在晚上和周末运行，然后我们帮助他们识别这些实例，根据计划自动执行某种关闭过程，然后在需要时让它们恢复运行。这一切都与单点登录技术(如 Slack 和 Jenkins 审计日志和报告)相集成，因此它实际上是一项企业技术，它甚至有一个 RBAC 模型，允许人们建立用户团队，因此您可以围绕它进行基于角色的访问控制。

但是当您停放一个实例时，每个实例每月的平均节省通常在 65%到 70%之间。因此，根据之前的示例，该公司停放了 26 个实例，每月节省 18，000.00 美元。你真的可以获得可观的投资回报。我们有很多大企业在使用这个平台——Capital One、联合利华、麦当劳等。他们在这个平台上已经有很长一段时间了，我们已经在像 ParkMyCloud 这样的云初创公司中诞生了，他们也在使用这个平台。我们的客户现在遍布全球约 15 个国家。

Shimel: 这太惊人了。非常惊人的故事。杰，你呢？你的背景是什么？

是的，所以我天生就是个销售人员。所以，我从 2000 年开始就进入了 IT 行业。我曾在一家名为 Micromuse 的公司工作。他们有一款名为 Netcool 的产品，所以如果人们熟悉传统的数据中心服务保证、网络保证等。然后我们在 2006 年被 IBM Tivoli 收购，我在 IBM 呆了很长时间。

有趣的是，大约在 2010 年、2011 年、2012 年，我们在 IBM 的 Tivoli，销售基础设施软件，如服务器监控、应用监控、存储等。我们开始把东西输给云公司——像 ServiceNow 这样的公司，像 Splunk 这样的公司，像 New Relic 和 AppDynamics 这样的公司，甚至是太阳能公司之类的公司。我们——你知道，还有 SaaS 科技。

所以，我真的开始看到，好吧，人们将如何使用云，以及随着时间的推移，云将走向何方？显然现在，由于它是一个实用程序，我们真的把 ParkMyCloud 变成了专注于成本控制和优化的东西。有点像“云之巢”——我们就是这样走到今天的。

很酷，伙计。所以，正如我提到的，Jay，我们将再次邀请你和你的伙伴们，因为我想更深入地了解所有这一切的技术方面。但我认为我们今天给人们的是一个很好的商业案例概述，我的意思是，如果你不知道你在云上的支出是多少？打电话，对——给那边打个电话。

如果你不知道你有多少个实例，或者，你知道，你会自动关闭一些东西，或者你只是想自动化，对吗？因为对我来说，这是真正需要自动化的东西——真的，ParkMyCloud 似乎是一个很好的解决方案，你应该考虑一下。

是的，这是一个单一用途的平台，仅仅是为了帮助人们在 AWS、Azure 和 Google 内部进行优化而构建的。我们提供免费试用，而且大部分——我想说至少 75%的客户是 100%自助服务的。所以，我们从来没有和他们谈过，他们付费订阅，他们省钱，他们喜欢使用这项技术。

Shimel: 我的朋友，这是个不错的生意。好吧。
