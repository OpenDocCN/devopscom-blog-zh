# Dynatrace 将应用程序环境提升为代码

> 原文：<https://devops.com/dynatrace-advances-application-environments-as-code/>

Dynatrace 在其在线 [Perform 2022](https://www.dynatrace.com/perform-2022/) 活动上发布了一系列对其产品组合的更新，这些更新使 DevOps 团队更容易以编程方式将可观察性和其他[软件智能](https://devops.com/?s=software+intelligence)功能嵌入到应用程序中。

与此同时，Dynatrace 还透露，它已经将其可观测性平台的范围扩展到现在包括所有主要的无服务器计算平台。

此外，该公司[扩展了其现有的应用安全模块](https://www.dynatrace.com/news/press-release/automatic-detection-and-blocking-of-attacks/),使用开放 Web 应用安全项目(OWASP)定义的框架，提供实时检测漏洞和阻止针对应用运行时的攻击的能力。该模块基于 Dynatrace Smartscape 拓扑软件和 Davis 人工智能(AI)引擎，提供对 IT 环境的实时可见性，使 DevSecOps 团队能够根据漏洞的严重性更好地确定首先修复哪些漏洞的优先级。

Dynatrace 首席执行官 Rick McConnell 告诉与会者，所有这些功能都是由代理软件实现的，即[可以使用应用程序编程接口(API)和模板](https://www.dynatrace.com/news/press-release/dynatrace-delivers-software-intelligence-as-code/)以编程方式嵌入到应用程序中，以自动显示软件智能。麦康奈尔说，IT 团队可以部署单个代理来驱动广泛的分析，而不是必须部署多个代理。

他补充说，在应用程序管理的更多责任转向开发人员的时候，这种能力使 it 组织更容易实现性能、质量和安全性的服务水平目标(SLO)。

目前还不清楚应用程序部署后的管理责任在多大程度上转移给了开发人员。它往往因组织而异。可以肯定的是，越来越多的应用程序正在被检测。应用性能管理(APM)平台的采用以前受到成本考虑和与在应用中嵌入代理软件相关的挑战的限制。随着检测应用程序变得越来越容易，现在可以从日志、指标和跟踪中收集和分析的数据大大增加了。因此，APM 平台现在被定位为可观察性平台，可以将从应用程序及其所依赖的底层基础架构收集的分析关联起来。

麦康奈尔指出，对更大仪器的需求主要是由数字业务转型倡议推动的。这些应用中有许多是使用相互依赖的微服务构建的。他指出，单个代理可以让 it 团队更容易地为云原生应用环境提供工具。

在检测和观察应用程序时，并不缺少选项。现在的问题是将收集的所有数据转化为可操作的情报。因此，可观测性平台被注入了人工智能引擎，使分析海量数据成为可能。有了这些见解，现有的 it 团队就有可能大规模管理更复杂的 IT 环境。事实上，现在已经不再是 AI 引擎是否会像现在一样注入 DevOps 工作流到什么程度的问题了。
