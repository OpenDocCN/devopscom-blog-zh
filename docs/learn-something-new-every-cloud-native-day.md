# 每天学习新的东西

> 原文：<https://devops.com/learn-something-new-every-cloud-native-day/>

作为 Techstrong Group 的新人，前几周我都在努力寻找东西。就像我在博卡拉顿总部时的浴室，或者登录 SurveyMonkey 账户，以便我可以查看我们各种飞行研究项目的状态。

我也意识到我必须学习方言，尤其是关于 DevOps 和云原生的东西。需要说明的是，我研究 DevOps 以及如何将安全性集成到流程中已经有一段时间了(是的， [DevSecOps](https://devops.com/why-your-devsecops-initiative-will-fail/) )。但是这个云原生术语是我需要学习更多的东西。

你可能有点困惑，为什么我对“云原生”这个术语感到困惑。过去 10 年我在云安全领域的工作让我对白话有了一些看法。当我说“云本地”时，我通常指的是您在云提供商中使用的本地服务。比如 AWS Fargate 或者 Azure 容器注册表。这些服务是“云(提供商)固有的”

但我意识到原生云服务只是更大的云原生革命的一部分。与 DevOps 一样，云原生是一种心态，而不是您从云提供商那里购买的服务。这是关于开放。是关于便携性的。这是关于敏捷性。当我意识到我还有很多东西要学的时候，我感到很惭愧。

这就是为什么我期待着 2022 年 8 月 10 日的 [CloudNativeDay 活动](https://www.techstrongevents.com/cloud-native-days-2022)。我有机会深入了解与我的新报道领域密切相关的一些话题。例如，关于传统企业中的云原生挑战的会议引起了我的注意。我曾经参与过诞生在云端的公司，所以我们没有任何客户群需要考虑。但是，当您的公司拥有大量传统基础设施时，您将如何采用新的颠覆性技术呢？迫不及待想听听那些比较传统的企业的看法和经验。

可观察性也是我需要了解更多的事情。几十年来，我一直在与客户合作开发高级*安全*监控策略，但观察应用层和相关基础设施是另一回事。我怀疑会有一些技术上的相似之处(收集数据，分析数据，对数据进行预警等。)，但是用例是根本不同的。

我也为今天的最后一个主题演讲感到兴奋。然而，我有一点偏见，因为它涉及到 Mitch Ashley 和我与 Taos(IBM 公司)的 Donald Lutz 的一次精彩对话，我们深入探讨了他在该领域看到的一些企业迁移挑战。

他们说当你年轻的时候，你什么都知道。但是随着年龄的增长，你意识到自己什么都不懂。我肯定属于后一种阵营，这就是为什么我不会浪费一个很好的学习机会——你也不应该。[注册](https://www.techstrongevents.com/cloud-native-days-2022/begin)并查看活动。我在那里等你。
