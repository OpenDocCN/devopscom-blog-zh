# Mendix 为低代码平台增加了 Git 存储库支持

> 原文:[https://devo PS . com/mendix-adds-git-repository-support-to-low-code-platform/](https://devops.com/mendix-adds-git-repository-support-to-low-code-platform/)

Mendix 今天更新了其低代码开发平台，添加了对 Git 库的[支持，作为使代码共享和重用更容易的努力的一部分，以及可视化冲突解决功能，使开发人员能够轻松看到同一代码的多个版本之间的差异。](https://www.prnewswire.com/news-releases/mendix-is-first-to-market-with-expanded-all-in-one-low-code-platform-launches-pre-packaged-industry-apps-for-even-faster-adoption-of-low-code-in-the-enterprise-301121639.html)

Mendix 9 在其在线 [Mendix World 2.0](https://www.mendixworld.com/entrance/) 会议上宣布，现在还提供对基于可搜索 Mendix 数据中心目录的数据中心的访问，这是一个开放的、基于标准的元数据存储库，旨在帮助开发人员更容易地发现数据。

数据中心连接工具使组织能够填充该数据中心。第一批可用的连接器是开源的 oData 连接器和西门子 Teamcenter 和 SAP 应用程序的连接器。Mendix 是西门子的子公司。Mendix 还包括一个外部实体功能，可以将来自远程系统的数据呈现为本地资源。

Mendix 全球营销总监 Sheryl Koenigsberg 表示，数据中心目录旨在使开发人员更容易将数据拖放到应用程序中，作为简化流程的一部分。

Mendix 今天还发布了一个工作流编辑器，开发团队可以通过它为机器执行和人工执行的流程调用工作流模板。可以通过中央任务管理器访问这些工作流，并且可以使用预配置的仪表盘来跟踪和管理工作流。

![](../Images/5bddc20a9aa8776419122e6ed889d264.png)此外，Mendix 还增强了平台内的人工智能 (AI)工具 Mendix Assist，以包括一个专门关注性能问题的机器人。Mendix Performance Assist 根据已知的最佳实践分析应用程序，以建议或自动实施更改，确保解决方案不会实施降低应用程序性能的反模式。

随着对渐进式 Web 应用程序(PWAs)的支持，移动应用程序开发正在得到改进。开发人员现在可以部署不同类型的移动应用程序，例如，在使用熟悉的 Mendix 建模工具的同一解决方案中，使用高度交互的 React 本地框架以及传统的基于浏览器的框架。

最后，Mendix 正在提供增强现实(AR)小部件，以便更容易地构建下一代令人兴奋的商业应用程序。开发人员现在可以轻松创建用户界面，包括 3D 对象、图像和标记检测。

Koenigsberg 表示，Mendix 专注于加速关键数字商业应用的开发。在新冠肺炎疫情之后，许多组织转向低代码工具，以允许专业和公民开发者更快地构建应用程序。从 DevOps 的角度来看，Mendix 9 等低代码工具加快了应用程序的构建速度，并最终使用低代码工具进行部署，这些工具抽象出了与构建这些应用程序相关的许多复杂性。

Koenigsberg 补充说，Mendix 9 有效地代表了在更广泛的应用程序开发生命周期中扩展抽象层的深度和范围的努力。

现在说新冠肺炎疫情在多大程度上推动组织更广泛地采用低代码工具还为时过早。然而，当许多 DevOps 团队面临着以前所未有的速度交付更多代码的压力时，很有可能良好的低代码平台正在构建许多数字业务应用程序，组织现在需要这些应用程序来生存并最终茁壮成长。
