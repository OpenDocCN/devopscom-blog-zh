# Vectra 为 AWS 推出检测

> 原文:[https://devops.com/vectra-launches-detect-for-aws/](https://devops.com/vectra-launches-detect-for-aws/)

*Vectra 推出首款面向全球 AWS 环境的云本地统一实时威胁检测和响应 SaaS 解决方案*

**加利福尼亚州圣何塞，2021 年 6 月 16 日**——威胁检测和响应领域的领导者 Vectra AI ，今天宣布为亚马逊网络服务公司(AWS) 推出 Vectra Detect。Detect for AWS 是一款云原生 SaaS 解决方案，可提供持续、可扩展的无代理威胁检测、优先级排序、调查以及对针对 AWS 上运行的应用以及用户、计算和存储实例的攻击的响应，包括在 AWS 上使用 控制平面 本身。针对 AWS 的检测在运行时和跨所有 AWS 区域整体工作，并且不需要包镜像。

当今数字环境变化的速度和灵活性促使许多 IT 团队将其传统的内部基础架构转换为云原生架构，从而导致平台即服务(PaaS)和基础架构即服务(IaaS)应用程序的使用增加。随着云采用速度的提高，在云环境 中引入 安全问题的风险也在增加。

Sephora 信息安全高级架构师 John Byun 表示:“安全是 Sephora 的首要任务，这需要安全工具来帮助我的 IT 团队获得深入的可见性，而不仅仅是最初的危害和渗透检测。“为了更安全地扩展我们的云运营，需要深入了解身份泄露、特权升级和威胁在资源之间横向移动的详细信息和背景，而无需管理额外的代理。”

Vectra Detect for AWS 使组织能够:

*   **通过对应用、用户、角色、无服务器计算和存储进行无代理运行时监控，降低云服务被利用的风险** ，实现应用的快速和可扩展部署。
*   **在 AWS 上快速检测针对您的系统和数据的威胁** 使用首批行为人工智能之一来检测威胁并确定其优先级，而不依赖于签名、代理或静态策略，同时防范试图利用错误配置的服务的攻击。
*   **使用 AWS 中的原生功能或与其他安全解决方案的深度集成，自动响应对运行在 AWS 上的应用程序的攻击** ,使团队无需依赖代理就能缓解威胁。

通过观察和了解攻击者行为、云账户、角色、权限、云工作负载的部署配置(容器和无服务器)；Vectra 的行为模型发现并阻止对 AWS 应用程序的攻击，而不会中断操作。作为 Vectra 这使得组织可以放心地大规模迁移、开发和部署更多的 AWS 应用程序，同时降低引入安全问题的风险。

*“*我们很高兴 Vectra AI 继续与 AWS 合作，并将他们的解决方案集成到 AWS 安全服务和 AWS 市场中，”AWS 外部安全服务副总裁 Dan Plastina 表示。“我们的客户希望解决方案能够协同工作，在不显著增加运营复杂性的情况下提高可见性和安全性。”

Vectra partnership 副总裁 Randy Schirman 表示:“超过 70%使用 AWS 的客户表示，他们至少使用了四种 AWS 服务和超过 21 种不同的服务。“由于 AWS 是日常运营的重要组成部分，因此保护其应用程序免受云中横向移动的影响至关重要，我们认为没有比检测 AWS 更好的解决方案来实现这一点。”

**关于 Vectra**

Vectra 是威胁检测和响应领域的领导者-从云和数据中心工作负载到用户和物联网设备。其 Cognito 平台使用人工智能加速威胁检测和调查，以丰富其收集和存储的网络元数据以及正确的上下文，从而实时检测、搜寻和调查已知和未知的威胁。Vectra 在 Cognito 平台上提供了四个应用程序来处理高优先级的用例。cogni to Stream向数据湖和 SIEMs 发送安全性增强的元数据。cogni to Recall是一个基于云的应用程序，用于存储和调查丰富的元数据中的威胁。cogni to Detect使用人工智能快速揭示隐藏和未知的攻击者并对其进行优先排序。Cognito Detect for Office365 和 Azure AD 发现并阻止企业 SaaS 应用程序和微软 365 生态系统中的攻击。更多信息请访问 vectra.ai 。
