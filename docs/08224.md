# 使用 ML/AI 支持基础设施监控

> 原文:[https://devo PS . com/using-ml-ai-to-support-infra structure-monitoring/](https://devops.com/using-ml-ai-to-support-infrastructure-monitoring/)

成功的基础设施监控使 IT 团队能够确保公司系统的持续正常运行时间和性能。机器学习(ML)和人工智能(AI)等技术通过更快地收集和分析构成 it 堆栈的所有硬件和软件组件的数据，使[基础设施监控](https://www.gartner.com/reviews/market/it-infrastructure-monitoring-tools)受益。基础设施的变化比以往任何时候都要快，但是复杂的系统、应用程序的独特性质以及 IT 技能的缺乏在与这些更新的技术集成时会带来挑战。然而，比以往任何时候都更重要的是，系统管理员和 DevOps 团队了解 ML 和 AI 如何能够减轻这些障碍，支持他们保持基础设施性能的领先地位，并快速解决出现的问题。

## 复杂系统的智能监控支持

智能基础设施监控工具和流程最切实的成果是近乎即时地发出性能和正常运行时间问题警报，然后以高效的方式解决这些问题，从而避免业务中断。然而，如果不使用 ML 和 AI 并且手动监控协议仍然存在，复杂的系统会阻碍这些好处。

使用 ML 或 AI 的工具极大地减轻了 IT 人员的工作，释放了关键的业务资源，并有助于整体生产力。这两种技术都可以自动识别和更新构成企业基础架构的所有 IT 堆栈，以使系统保持最新，并与既定的关键绩效指标(KPI)保持一致。此外，智能产品可以根据设定的标准检测和分析这些指标，以便即使在 IT 体系不断变化的情况下，也能发现基础架构中“不健康”部分的早期警报。这大大加快了故障排除的速度。

## 应用中的差异

由各种 IT 堆栈支持的不同应用程序通常都有针对其性能和正常运行时间的独特服务级别协议(SLA ),以及未达到这些服务级别时的补救措施或惩罚措施。此外，给底层基础设施带来压力的系统负载经常变化。由于这些原因，确定什么构成了“健康的”it 堆栈是很重要的，这样基础架构的这些微小部分就不会因为涉及的变化而被忽略。

ML 和 AI 可以通过编程来跟踪支持“健康”IT 堆栈的系统基线。这些技术尤其擅长在数据中发现新颖和不寻常的模式。随着时间的推移，监控和[可观察性](https://devops.com/?s=observability)前景变得越来越复杂，这是由开发人员构建应用程序和系统的方式的实际变化所驱动的，发现和检测数据中的这种模式的能力对于帮助理解数据、进一步减少手动搜索、检测工作和我们都曾经历过的“仪表板死亡”来说至关重要。

## 用智能技术支持 IT 团队技能

在过去几年中，系统管理员(更大程度上是开发人员)的角色发生了变化，变得几乎与他们监管的基础设施一样复杂。如今，似乎要求开发人员具备基础设施各方面的专业知识，从监控到 Kubernetes 到机器学习。这对拥有这些技能的开发人员来说是一个很大的损失，但是从更现实的意义上来说，能够做所有这些事情的开发人员是很难找到的。这些技能的缺乏在行业中普遍存在，这就是为什么 ML 和 AI 可以被视为支持技术——它们可以在一定程度上填补这些空白。

凭借内置的智能和自动化，ML/AI 甚至可以让最缺乏经验的 sysadmin 或 DevOps 专业人员像专业人员一样监控复杂的基础设施，承担大部分收集和分析数据以及确定故障排除位置的时间密集型工作。主要目标是将人类放在驾驶员的位置上，利用 ML 和 AI 来精细地发现系统问题，在 IT 人员对他们的系统进行故障诊断时提供可能最相关的指标或图表，并减少开发人员的认知负荷。

凭借智能技术所拥有的巨大优势，将它们集成到您的 it 体系中有助于缓解复杂系统、应用差异化和 IT 团队技能不足带来的挑战。使 ML 和 AI 在基础设施监控中有效的重要因素是使用整合了正确公式、算法和自动化的工具，这些工具可以最大限度地帮助确定您想要的结果是否成功。
