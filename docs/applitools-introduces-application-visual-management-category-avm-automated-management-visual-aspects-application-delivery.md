# Applitools 引入了应用可视化管理类别(AVM ),用于自动化管理应用交付的所有可视化方面

> 原文：<https://devops.com/applitools-introduces-application-visual-management-category-avm-automated-management-visual-aspects-application-delivery/>

*新品类聚焦数字化转型时代视觉上完美的用户体验*

加州圣马特奥，2018 年 2 月 13 日——人工智能视觉测试和监控领域的领导者 appli tools([【https://applitools.com】](https://applitools.com/?utm_source=PressRelease&utm_medium=Catapult&utm_campaign=body&utm_content=180213-AVM-Category-Launch))今天推出了应用视觉管理(AVM)，这是一个新的类别框架，专注于自动化应用交付的所有视觉方面。AVM 代表了一种新的技术，正在将视觉人工智能扩展到用户界面(UI)和最终用户体验。它概述了支撑数字化转型、CI-CD 和左移 QA 融合的行业大趋势。它还概述了人工智能(AI)如何被用来确保应用程序在发布时在视觉上正确无误——在所有设备、平台和浏览器上实时显示。新兴的应用可视化管理(AVM)类别将自动化可视化测试和监控扩展到其他可视化交付组件，如可访问性、安全性、协作、错误跟踪、源代码控制等。

今天，只有应用程序交付的功能方面是自动化的:测试、监控、CI、安全、错误跟踪、协作、源代码控制等等。同时，应用程序交付的所有可视化方面都是通过检查面向客户的应用程序来手动完成的，这导致了缓慢且容易出错的流程，无法支持持续集成和持续交付(CI-CD)的速度。

AVM 代表了一种新的基于视觉人工智能的方法，可以自动化视觉生产、测试、交付和监控。它提供了一个概念框架，说明一类新兴的工具现在如何集成和支持功能性软件开发工具，包括 UI 开发工具、CI、bug 跟踪、协作、版本控制、监控和安全性。结果是缩短了应用交付周期并提高了软件质量，因为“客户所见”会全天候自动监控，以防止浏览器和微服务更新、新移动设备的引入、操作系统更新和动态内容的影响经常导致的视觉缺陷。

**Applitools 首席执行官 Gil Sever 表示:“作为自动化测试社区的一员，我们勾勒出了一个新的行业类别，它将自动化测试和监控的范围和深度扩展到了数字化转型的最前沿——当新的或修改后的软件发布时，客户首先会看到什么。“应用可视化管理代表了可视化人工智能的发展，并建立在已建立的应用性能管理市场的基础上，以自动化监控客户在其所有设备上每天看到和交互的视觉外观的大量任务。”**

**应用可视化管理的核心特征是:**

1.  **解决使用可视化人工智能来自动化组织管理应用程序可视化方面的新需求，包括测试、监控、CI、错误跟踪、协作、版本控制、安全性等。**
2.  **在数字化转型时代，持续交付、左移、微服务架构以及对应用交付和更新至关重要的浏览器和设备激增等全球趋势的推动下。**
3.  **旨在实现软件应用程序视觉和功能方面的自动化管理，消除阻碍涵盖应用程序所有方面的高质量交付流程的瓶颈。**
4.  **面对不断的变化、快速的技术进步和新的破坏性业务威胁，AVM 还支持生产前检测和更快的视觉缺陷生产解决方案。它旨在组织和自动化视觉测试和监控，帮助组织以最大的信心为所有客户提供卓越的数字体验。**

**Cloud Grey 创始人兼 Appium 项目负责人乔纳森·立普斯评论道:“当今和未来的尖端应用程序都包含越来越难以自动化方式管理的组件。从包含新的 UI 组件到跨各种外形和平台进行完美呈现的需求，自动化应用交付的需求与难度同步增长。业内非常需要对所有应用交付元素(功能和视觉)的准确和快速反馈，以支持对快速和全面 CI/CD 周期日益高涨的期望。”**
