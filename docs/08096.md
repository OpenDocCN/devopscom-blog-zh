# 没有自动化就没有创新

> 原文：<https://devops.com/theres-no-innovation-without-automation/>

如今，几乎所有行业的各种规模的公司都可以随时访问[超大规模](https://devops.com/?s=hyperscale)计算、存储和无数其他技术，而不需要像 AWS 这样的超大规模云提供商的大量预算或技术专业知识。

这是创新的福音。云为成本优化、以前无法实现的云原生工具、安全性改进和其他切实的业务优势打开了大门。这也在行业中创造了一个神话:人们通常认为云是“简单的”是的，将现有应用程序迁移到云中可以创造巨大的商业价值。但是说“云很容易”可能会产生危险的误导。

这种看法是有原因的。例如，拥有信用卡的开发人员可以创建一个帐户，并在几分钟内建立一个新的环境。这与同一个 IT 专业人员可能需要提前几天(或几周)向内部运营团队提交请求来调配新开发环境的日子相去甚远。

然而，在任何企业规模的云中运行都不一定容易。事实上，云可能比内部基础架构复杂得多。想想像 AWS 这样的平台在一个月内推出的新工具和服务的数量——跟上变化和创新的步伐本身就像是一项全职工作。

当团队迁移到云(或开始追求全新的云原生项目)，同时试图以更高的速度和敏捷性移动时，复杂性会成为一个更大的问题，这通常是由公司新的或正在进行的数字化转型项目驱动的。想象一个团队正在经历向开发运维的转变，同时也在将一个主要的遗留系统迁移到云——这是一次要处理的大量工作。

幸运的是，云复杂性是一个非常容易解决的挑战——只要你接受自动化。自动化，包括使用超大规模计算本身的嵌入式或原生工具，对于云的成功至关重要。

自动化使当今的 IT 团队能够实现云迁移的优势，而不会给自己带来难以承受的运营负担。此外，自动化使当今商业和 IT 领域的许多流行口号——如“快速失败”和类似的想法——成为可能。

自动化是创新的基础。没有自动化就没有有意义的创新。这是因为坚持大量手动操作会使其成为一个瓶颈。技术阻碍了整体业务，而不是推动其最紧迫的战略重点。不适应这一现实的组织会面临巨大的阻力。云本身并不能解决问题；云与自动化相结合是我们所需要的。

考虑任何 IT 环境的各种操作要求:供应、配置、安全性、扩展、资源管理和优化，以及变更控制/变更管理。如果没有有效的自动化，其中每一项都需要大量的手动工作才能在云中进行管理，并且很快会变得比普通内部环境中的对手更加复杂。例如，如果您不能随着应用程序需求的变化而快速扩展，无论是由于需求激增还是其他因素，那么您就无法实现云所承诺的速度和敏捷性。同样，如果您不能缩减不再需要的机器，您几乎肯定是在闲置资源上浪费金钱。这就是云平台的自动扩展功能发挥作用的地方。

类似地，考虑您组织的现有变更控制或变更管理过程。这是许多企业的必要功能，尤其是那些必须管理法规遵从性的企业。当在云中手动应用时，它会很快变成一种拜占庭式的工作，严重限制敏捷性。对于大多数现代企业来说，这是不可能的。这是另一个可以通过云中的自动化来解决的挑战。

同样重要的是，要理解自动化推动质量的程度不亚于它推动敏捷性的程度。它使您能够更快更频繁地[交付](https://lemongrasscloud.com/)新产品和服务，同时培养一流的用户体验。例如，上面的自动扩展场景可以减少损害客户体验和利润的与容量相关的停机。通过这种方式，自动化和云共同构成了真正的卓越运营及其容量、弹性、性能和安全性等标志性特征的基础。

这也不仅仅是面向客户的应用程序的范围。有竞争力的企业必须将相同的理念引入其整个 IT 组合。创新依赖于内部系统变得像网络前端一样灵活。

想想像 SAP 这样的 ERP 系统；传统上，企业在 SAP 上的变革速度像大象一样快，而今天的企业运营速度却像羚羊一样快。您还必须将云和自动化的流程、工具和文化应用于后端系统。您可以在云中自动执行操作任务，例如系统刷新和其他系统级操作。这为零停机维护铺平了道路，在操作系统级别修补数据库，同时消除了以前需要的停机时间。或者想想备份和恢复的必要性:通过基于云的自动化，您可以提供点击式恢复，从而大大减少恢复时间。

同样，质量和敏捷性同样重要。将自动化适当地应用于基于云的 ERP 系统也可以减少或消除人为错误，从而进一步提高系统性能和可靠性。

简而言之，云为企业提供了实现转型创新的绝佳机会。  但是，没有敏捷就不能创新，没有自动化就不能敏捷。
