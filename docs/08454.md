# 你会把你的 IDE 装在盒子里吗？在火车上？用火狐？

> 原文：<https://devops.com/will-ide-box-train-firefox/>

正如预测的那样，云 IDE 竞赛正在加速。现在我确定了。不管有没有云 IDE，这都不是一个选择。但这是一个巨大的机会。

如果你看过我以前在 Cloud IDEs 上的帖子，比如这个、[这个](http://thoughtsoncloud.com/2014/03/the-rise-of-cloud-based-ides-q-a-with-ibms-ken-walker/)，甚至这个，你就会知道我对他们的潜力有多着迷。现在它们到处都在出现。

就在最近，Mozilla 发布了他们的 WebIDE，这是一个内置在浏览器中的 HTML5 应用开发环境。目前的功能很酷。但那不是我感兴趣的。

更有趣的是，Chrome 必须是 next，一个基于 Chrome 的 IDE，可以连接到你的谷歌账户，集成到谷歌应用引擎中。那会很酷，但同时也有点令人毛骨悚然。

在我看来，Mozilla 的新 IDE 无法与我目前最喜欢的云 IDE [Cloud9](http://c9.io) 相提并论。但它确实表明了发展世界中一个非常酷的方向。它今天的传递也很弱。你不仅仅是浏览器的一部分，你必须从 GitHub 获取一些信息，然后你所有的应用程序运行都会出现在 Firefox 的一个单独的窗口中。我真的希望它只是未来浏览器版本的一部分，你只需要打开或关闭它。

说到云。他们最近进行了重大更新。这包括巨大的性能增强、 [Docker](https://www.docker.com/) 支持(IDE 渗入 DevOps，我们来了)和更多的语言选项。更不用说新的用户界面，我不确定它还能变得更好。

哦，那么那些不是专门为 IDE 设计的产品中的 IDE 呢？就像 [CodeAcademy](https://www.codecademy.com/) 里嵌入的那个整齐的。随着激动人心的新 [Wolfram 语言](https://www.wolfram.com/programming-cloud/?src=google&1101&gclid=CJmBguna3r8CFQmraQodKUcAlg)的推出，新的版本也随之而来，顺便说一下，如果你是一个“数据呆子”(对不起，新遗物，我偷了它)，那么你必须检查一下。

总之。我们生活在自己的思想中，因为我们这样做了，有时我们没有意识到它在我们生活中有多重要。比如呼吸。但最终，如果开发人员从根本上转向在云中开发，他们就简化了开发过程中的一切。

云 ide 更容易与生产和暂存云环境集成。它们使得持续集成和交付更加容易。它们使得引入新的框架、API 和工具来进行监控和代码管理变得轻而易举。

然而，搬迁并不容易。我知道离开我完美配置的 Visual Studio 2013 有多难，因为我想要它的所有插件和主题。我仍然不时地访问 VS 来创建我的基于控制台的自动化应用程序。但是对于新的事物，我不会考虑转向基于客户端的 IDE。如果互联网断了，哦，好吧，今年夏天我的电已经断了两次，当那发生的时候，我无论如何也不能完成工作。这只意味着是时候探索他们称之为太阳的东西了。但如果我真的疯了，因为它是一个云 IDE，我可以通过手机网络在我的手机或桌子上编码。

你怎么想呢?您准备好转向基于云的开发环境了吗？如果没有，为什么？
