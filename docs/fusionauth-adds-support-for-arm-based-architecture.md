# FusionAuth 增加了对基于 ARM 架构的支持

> 原文：<https://devops.com/fusionauth-adds-support-for-arm-based-architecture/>

大多数开发人员在生活中需要三样东西:控制、有趣的项目和在他们选择的技术上构建的能力。现在，一项新的开发承诺将这三者都交到开发者手中——fusion auth 宣布支持 T2 基于 ARM 的架构 T3。

FusionAuth 解决方案提供多因素身份验证、无密码身份验证和单点登录等功能。基于 ARM 的架构简单易用，能耗更低。转化为 DevSecOps 的好处，这意味着 FusionAuth 只是让开发人员在构建 ARM 架构时更容易满足身份和授权要求，并增强他们的安全性。

我们与 FusionAuth 的联合首席技术官丹尼尔·德格罗夫(Daniel DeGroff)进行了交谈，以了解这种支持如何使[开发人员](https://devops.com/?s=DevSecOps)受益的细节。

## 向基于 ARM 的架构转移

您可能已经注意到许多具有前瞻性思维的开发人员正在转向基于 ARM 的技术。开发人员可以依靠跨不同实现和不同设备的互操作性，ARM 处理器提供更低的工作温度和更低的功耗。它们的生产成本也很低，这使得它们的价格很有吸引力。这为他们赢得了从服务器到智能手机到世界上最快的超级计算机的一席之地。

随着许多开发人员发现了更高性能和成本节约的潜力，像 FusionAuth 这样的供应商也会支持基于 ARM 的基础设施也就不足为奇了。通过与 ARM 处理器合作，该公司可以支持更多的用例，如基于 ARM 架构的物联网和高性能云平台。德格罗夫将其描述为一种务实的——不可避免的——举措，并表示，“如果你想使用这些新机器，你必须以某种方式支持这种架构。”

“我们在苹果 MacBook 上开发，所有新的工作站都将使用苹果基于 ARM 架构的定制芯片，”他指出。“ARM 通常比同等的 x86 服务器平台更快、更便宜，许多低功耗物联网设备也将利用 ARM 架构。”

## DevSecOps 好处:更大的灵活性，更简单的安全性

FusionAuth 的 ARM 支持可以在几个方面让一般有安全意识的开发人员的生活更轻松。FusionAuth 运行在任何平台，任何地方；通过支持基于 ARM 的技术，该公司正在扩展开发人员的能力，以他们想要的方式和地点部署其认证和授权平台。“我们必须确保 FusionAuth 运行在我们的用户想要开发的地方，”德格罗夫说。

在帮助开发人员构建基于 ARM 的架构时，FusionAuth 还使他们更容易使用 API 在应用程序或软件开发的早期阶段引入 FusionAuth。DeGroff 说，该平台的一个卖点是，它使开发者更容易解决典型的认证障碍，如薄弱的密码策略或特权访问规则的困难。通过支持基于 ARM 的技术，FusionAuth 旨在帮助开发人员通过灵活、复杂的身份认证和访问工具成功满足甚至更严格的安全要求，并在开发周期的早期做到这一点。

## 无缝过渡到更智能的身份认证

如果你是一个对探索 FusionAuth 和 ARM 技术的结合感兴趣的开发人员，DeGroff 说任何过渡都会感觉无缝。“将工作负载转移到这种架构有很好的商业案例，”他指出。想要利用改进的性能和更低的成本的开发人员现在可以开始将工作负载迁移到基于 ARM 的平台。"
