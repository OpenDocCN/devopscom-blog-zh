所以今年我们稍微改变了一下关注点。因此，我们真正关注的一件有趣的事情是基础架构模式在 DevOps 环境中的作用。我不只是说，你知道，“你在采用容器吗，你在做宏服务吗，面向服务的架构吗？”但是我们真的在寻找关于架构方面的信息，就你的应用程序设置的方式而言，你的团队是自给自足和自治的吗？你能独立快速地做出改变，而不依赖于整个组织吗？你能真正快速地进行实验吗？

因为我认为我们已经从过去几年的敏捷和所有 DevOps 企业运动中看到，进行小规模测试并根据这些测试的结果快速迭代以设定更大方向的能力对于推进您的服务非常关键，而且要更快、更可靠地做到这一点。因为我认为对于那些不太熟悉这些实践的高级管理人员来说，这是一个真正巨大的新事物，他们意识到，采用 DevOps 实践实际上并不意味着为了速度而牺牲质量，反之亦然。这实际上意味着您正在更快地生产更好、高质量的软件。
