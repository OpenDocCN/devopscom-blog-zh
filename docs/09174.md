# 管理端点法规遵从性

> 原文:[https://devops.com/managing-endpoint-compliance-as-code/](https://devops.com/managing-endpoint-compliance-as-code/)

现代软件交付的快节奏和不断增长的规模已经改变了组织看待和处理安全策略的方式。法规遵从性是当今需要将安全性作为业务流程基本部分的组织所必需的。不再可能由专门的安全团队手动管理合规性。

平衡安全性与不断增长的基础设施需求意味着 IT 安全性和合规性是不容置疑的。在风险和组织快速高效地交付市场现成解决方案的能力之间，没有更令人不安的权衡余地了。

在当今快速创新的世界中，遵守日益增长的安全和合规性法规是一项持续的挑战，影响着所有大小组织。组织正在实施自动化解决方案，以消除对传统、缓慢、易出错的手动流程的依赖，因为行业法规越来越严格，与安全攻击和违规相关的风险也越来越大。

### 什么是管理端点合规性？

法规遵从性是对[法规遵从性](https://devops.com/?s=compliance)控制的编纂，以实现其遵守、应用和补救的自动化。它包括工具和实践，使开发运维团队和开发团队能够整合三个关键的合规性活动:

1.  **检测:** 通过自动资产扫描发现违规行为，并在发现违规基础架构时通知利益相关方。
2.  **补救:** 通过实施即时基础架构变更来纠正不合规情况，以确保大规模的最高合规性。
3.  **自动化:** 通过自动验证计划的变更符合来避免不符合。

管理端点状态 as-code 的合规性有很多重要的使用 案例，包括 c 身份数据保护、检测影子 IT 资源、网络安全、数据公开访问和代码许可合规性 。

### 合规的重要性

大多数组织仍在努力保持安全性并遵守监管标准。许多组织缺乏跨异构基础架构和应用程序的可见性。他们在开发、安全和运营(开发、安全和运营)团队之间交流要求时使用的语言也不一致，并且无法补救发现的问题。

法规遵从性 e 实施全面的法规遵从性战略，包括一套稳健的控制措施；例如，管理数据存储位置和访问控制管理。

确保大规模遵守这些规则对于数字业务的成功至关重要。随着越来越多的组织转向云，不遵守规则的可能性也越来越大。报告表明，一次法规遵从性违规平均会造成业务ses 1500 万美元 [](https://securityintelligence.com/news/failure-to-comply-with-data-protection-regulations-can-cost-firms-nearly-15-million/)。

如果 DevOps、开发和 DevSecOps 团队能够通过采用不同的思维方式和编写可理解的代码来自动化合规性，这种违规是可以避免的。“代码化”策略的采用导致了基础设施代码化( [IaC](https://devops.com/updating-and-managing-infrastructure-as-code-iac/) )，这是 DevOps 团队的成功因素之一。审计是精简的，允许个人专注于更高价值的活动。

### 通过合规管理车队的挑战

通过合规性管理车队并非没有挑战。首先，团队必须弥合 人才缺口，因为端点合规性通常是由 IT 管理员管理的，他们天生就不习惯代码方法。

通常，合规性要求非常密集，其他利益相关者很难理解。理解组织的法规遵从性需求并与其他利益相关者合作将它们转换成 as-code 是开始的最大障碍。

开发人员 t eams 必须找到合适的工具集，并将它们集成到 DevOps 和 DevSecOps 工作流中。这不仅仅是流程的改变，也是文化的改变。煮沸海洋的方法不仅痛苦，而且几乎不可能成功。采取小步骤，不断改进是一个人必须采取的心态。

## 整个生态系统的可见性

无论是台式机、笔记本电脑或移动设备等终端用户设备，还是销售点(POS)或物联网设备等非服务器边缘设备，终端设备都极易受到不良分子的利用。因此，组织有必要像对待服务器端设备一样，将它们置于组织的安全策略和治理要求之下。

采用端点法规遵从性方法有许多优势:

*   用户可以随时轻松了解整个组织的合规状况。
*   组织可以保留整个开发生命周期中发生的所有事情的审计日志，并计算适当的风险认知评级。
*   组织可以评估任何偏离基本法规遵从性要求的情况。

### 继续优先考虑 合规

通过法规遵从性管理多样化的 IT 基础设施群，确保终端设备符合各种安全标准。IT 团队有权创建可在组织内强制实施合规性和建立安全基准标准的规则。在软件开发生命周期的每个阶段，合规性检查都是有效的，并且更加明显。

这保证了整个 IT 生态系统的详细可见性和彻底的合规性标准。
