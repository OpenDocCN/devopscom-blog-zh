# AWS re:Invent–新版本推动云安全创新

> 原文:[https://devo PS . com/AWS-reinvent-new-releases-drive-cloud-security-innovation/](https://devops.com/aws-reinvent-new-releases-drive-cloud-security-innovation/)

最近在拉斯维加斯举行的 AWS re:Invent 会议上，发布了几个新的 AWS 安全服务。其中一些公告，如 AWS 网络应用防火墙和 T2 AWS 检查员似乎让许多安全媒体和博客圈感到惊讶。

然而，像 Evident.io 这样的云原生安全公司对这些声明并不感到意外。事实上，AWS 一直在积极与他们的安全生态系统合作，为合作伙伴创造在这些新的基础服务上创新的机会。

现有的和新兴的合作伙伴都需要记住，平台不是产品，它是一个不断增长和发展的实体，将随着云生态系统的不断变化而不断适应。

如果你想要永恒的稳定，考虑投资一块宠物石。如果你想成为不断改进的技术生态系统的一部分，那就系好安全带，因为 AWS 是这个领域平台的火箭船。

AWS，无论他们创新什么，都将侵占其生态系统中类似的价值位置作为存在的问题。然而，对于平台的客户来说，这可能是最好的情况。

客户胜出是因为他们在平台体验中不断获得技术创新和改进，这是数据中心生态系统中从未有过的。

让我来描述一下这些服务对生态系统的意义:

**AWS 网络应用防火墙(WAF)**

AWS WAF 使客户能够创建过滤规则，将来自互联网的应用程序呼叫列入白名单或黑名单，以访问客户部署的 CloudFront CDN 资源。

例如，客户可以编写一个 WAF 规则，根据源 IP 地址匹配或与请求字符串的模式匹配来过滤 web 应用程序的请求。

如果我经营一家只向美国销售产品的电子商务商店，我会确保我的 WAF 规则包含来自中国、叙利亚、苏丹、朝鲜和其他禁运国家(以及大量恶意流量来源)的 IP 地址范围的阻止请求模式。

虽然这似乎是对一些云安全提供商的直接攻击，但它确实为他们创造了一条新的市场路径和机会，以向客户提供他们的 WAF 智能。

老实说，构建一个大型 x86 设备并让它为您的基础架构运行规则智能并没有什么不同。

AWS 消除了虚拟化设备的障碍，并为这些公司部署其核心价值主张开辟了一条直接的道路，这些核心价值主张是精心策划的强大规则集，可保护客户免受历史、现有和未来的攻击。

期待通过某种市场或其他交付机制看到品牌规则集，因为这是客户需要的。

**AWS 检查员**

在我们看来，检查员是最近 AWS 历史上最重要的公告。我们终于看到了基于云的安全的真正融合开始出现:持续的以 API 为中心的安全智能可访问性，它普遍内置于平台的多样化功能集中。

随着 Inspector 的出现，安全提供商可以直接连接到主机和应用程序，为客户提供深入、全面的安全可见性，而无需部署额外的第三方代理。

让我们面对现实吧，客户在使用他们的安全堆栈之前就已经被代理压得喘不过气来了，而且安全代理提供的价值在云环境中也是值得怀疑的。

期望安全平台集成工作负载范围安全情报的单一视图，以便为客户提供其安全态势、风险状况和攻击面的完整画面。

**配置规则**

配置规则可能是 AWS 在安全领域发布的公告中最不令人兴奋的。不幸的是，对于基于配置的服务交互来说，服务的局限性在最初推出一年后仍然很明显。

配置规则也不例外，在这种情况下，客户定义自定义规则以帮助他们了解影响其环境中安全性的更改的有趣功能受到了有限服务支持的阻碍，这种成本模型对于小客户来说太不可预测，并且除了规则触发时发生的“ping”之外，安全价值极小。

除了这些挑战之外，由非安全专家组成的广大受众编写结构良好且经过深思熟虑的安全规则也存在障碍，这提供了一种近乎虚假的保护感。

与 CloudTrail 相比，cloud trail 提供了在运营层面有意义的有价值的审计信息，配置规则预计客户将拥有丰富的安全经验和运营知识来利用该服务。

但每个用户群中的一些客户都对我们在行业中称之为“足够好”的安全性感到满意，这定义了一个勉强被称为安全技术的产品范围。

提供“足够好”安全性的平台和供应商往往在国际自助餐厅，在那里你可以得到低于平均水平的披萨、普通的墨西哥玉米沙拉和糟糕的汉堡。

你得到的任何东西都不会让你有满足感，因为没有足够的努力投入到最终产品中来真正改变你对体验的看法。

但是成熟、专注的产品将始终提供卓越的客户体验，并且在安全性方面永远不应被视为可有可无。

将注意力分散在成本管理、安全性、优化等方面的供应商。在保护您的基础设施方面，永远不会像该领域的专注创新者那样提供同样的质量或关注。

用户可以期待对新 AWS 服务的服务支持的等待期，这意味着您的安全控制取决于 Config 的路线图。

但是精通安全的客户并不热衷于等待安全在他们周围成长，他们现在就需要它，他们需要在他们 100%的云基础架构上使用它，并且他们需要它是可操作的。

这就是为什么许多人会继续选择每个安全层的领导者，而不是平台本身的基本安全功能。

预计配置规则会随着时间的推移而增长，但如果配置是一个指标，那么配置规则为平台上的客户提供真正的价值还需要很长时间。

这里的真正价值是，安全领域的创新者将继续填补平台服务留下的空白，并在确实成功的技术之上创新客户价值。
