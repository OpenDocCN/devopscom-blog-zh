# 调查显示开发人员的挫折感和成本越来越高

> 原文：<https://devops.com/survey-shows-mounting-devops-frustration-and-costs/>

对美国和欧洲的 400 名开发人员和 DevOps 团队成员的调查发现，平均每周有 15 个小时花在管理任务上，如调试管道和等待测试和构建完成。

该调查由市场研究公司 [Vanson Bourne](https://www.vansonbourne.com/) 代表 Garden 进行，Garden 是一家在 Kubernetes 环境中自动化应用程序开发和部署任务的平台提供商，调查发现超过四分之三(76%)的受访者表示，花在这些和其他手动开发运维任务上的时间普遍被浪费了。

Garden 首席执行官乔恩·埃德瓦尔德(Jon Edvald)说，总的来说，一周 40 个小时的工作时间中有 39%是用来做这些事情的。他指出，如果将今天雇佣的开发人员和 DevOps 专家的数量以及他们的工资考虑在内，仅在美国就有多达 610 亿美元被浪费在这些任务上。

调查受访者列举的开发运维挫折的主要来源是等待管道运行(76%)、等待构建和测试(74%)以及设置、维护和调试管道/自动化(71%)。事实上，只有 11%的受访者对他们的开发设置和工作流程完全满意。

随着组织引入额外的开发运维专业知识，这些隐性成本甚至可能会增加。整整 83%的受访者表示，他们的组织计划在 2021 年雇佣更多员工或更多使用自由职业者/外包到 DevOps 角色。四分之三(75%)的受访者表示，与 2020 年相比，DevOps 预算已经增加或将增加。

此外，Edvald 指出，采用更复杂的平台，如 Kubernetes，也可能导致成本增加。尚未使用 Kubernetes 的组织中的受访者每周平均花费 14.3 小时在低级开发运维任务上，而已经使用 Kubernetes 的组织每周花费 16.5 小时。近三分之二的受访者(62%)表示他们的组织已经在使用 Kubernetes，而 31%的受访者目前正在不同程度上对其进行评估。使用 Kubernetes 的受访者中有整整 95%的人在推出 Kubernetes 后面临挑战，包括培训和入职(37%)或设置/配置的复杂性(35%)。

当然，并不是所有节省下来的时间都会被用于提高整体生产力的任务上；近一半的受访者(49%)表示，如果他们有，他们会花更多的时间开发新产品和服务。

具体来说，46%的受访者表示他们将提高现有产品和服务的速度和交付，而 44%的受访者表示他们将专注于提高现有产品和服务的安全性。

埃德瓦尔德说，这项调查清楚地表明，仍然有很多机会自动化重复性的开发工作。Edvald 指出，组织在开发运维平台上花费了太多时间，占用了构建和部署应用程序的时间。

不太清楚的是，组织何时会达到某种类型的开发运维临界点。Garden 打赌，基于微服务的云原生应用的过渡最终将迫使 DevOps 自动化问题。然而，在只知道他们使用了多年的 DevOps 流程的组织中，改变可能需要很长时间。
