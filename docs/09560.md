# Planview 概述收购 Tasktop 后的 VSM 战略

> 原文：<https://devops.com/planview-outlines-vsm-strategy-after-acquiring-tasktop/>

Planview 本周宣布其[完成了对 Tasktop](https://www.businesswire.com/news/home/20220704005348/en/Planview-Completes-Acquisition-of-Tasktop) 的收购，task top 是一家价值流管理(VSM)平台提供商，该平台使开发运维团队能够将其工作与战略业务目标更紧密地结合起来。

Planview 首席产品官 Louise Allen 表示，目标是加强 Tasktop VSM 平台与 Planview 已经向软件开发团队和业务高管提供的项目管理应用程序之间的集成。

Allen 说，就在 DevOps 工作流中采用 VSM 而言，现在仍处于早期阶段，但随着越来越多的组织意识到他们对软件的依赖程度，他们需要更好地将应用程序开发项目与其战略业务目标相结合。VSM 平台使得分析从各种 DevOps 工具收集的数据成为可能，并使业务和 it 领导能够，例如，可视化一个项目的延迟如何影响下一季度的收入目标。

她补充说，鉴于当前整体经济的疲软，各组织正在寻求更准确地理解软件开发决策将对业务产生的影响。她解释说，作为解决这一需求的努力的一部分，Planview 还将使其应用程序更容易使用最初在 Tasktop 中创建的数据。

VSM 使更多的组织能够像软件公司一样运作，将应用程序更新与收入目标相结合。随着数字业务转型计划数量的增加，组织发现他们需要更像一家制造产品或提供服务的软件公司那样运营。

[价值流管理](https://devops.com/?s=value+stream+management)的概念可以追溯到精益制造方法，该方法要求持续测量制造过程的每一步。随着软件开发从手工发展到使用 DevOps 最佳实践的自动化过程，越来越多的人认识到监控事物的价值，例如错过软件开发最后期限对业务的影响。

在其核心，VSM 平台将自动转换从 DevOps 工作流收集的指标，以跟踪业务定义的一组关键绩效指标(KPI)。一旦收集了这些指标并建立了 KPI，组织就能够应用高级分析来调整实现战略目标所需的资源，例如，将更多开发人员投入到特定项目中。通过 VSM 平台收集的数据还可以与商业智能(BI)应用程序共享，商业领袖通常使用这些应用程序来管理业务。

目前还不清楚商业领导者有多努力地寻求对软件开发计划如何进展的更深入的了解，但至少，DevOps 领导者需要一种更简单的方法来理解计划之间存在的所有依赖关系。否则，他们冒着错过最后期限的风险，仅仅因为没有人意识到一个项目中发生的延迟会对多个其他计划产生连锁影响。无论谁使用 VSM 平台生成的数据，提高软件开发生命周期的可见性的需求从未像现在这样迫切。
