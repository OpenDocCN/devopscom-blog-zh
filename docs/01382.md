# VictorOps 宣布推出事故自动化引擎，进一步推进实时事故管理的创新

> 原文:[https://devo PS . com/victor ops-宣布-事件-自动化-引擎-推进-创新-实时-事件-管理/](https://devops.com/victorops-announces-incident-automation-engine-advancing-innovation-real-time-incident-management/)

*VictorOps 通过使用端到端自动化主动路由警报、转换数据和解决问题，帮助公司绕过传统的网络运营中心*

科罗拉多州博尔德市–2016 年 6 月 22 日–[针对 DevOps 团队的实时事件管理公司 VictorOps](http://try.victorops.com/l/44432/2016-03-17/9p67ff) 宣布推出事件自动化引擎，这是一套先进的功能，可帮助随叫随到团队通过使用智能警报、复杂的路由和更智能的系统输出，自动缩短平均问题解决时间(MTTR)。

与主要关注警报和通知分类的其他事件管理工具不同，VictorOps 事件自动化引擎通过智能系统输入和可操作输出的组合，在整个事件生命周期中提供无与伦比的控制和洞察力。事件自动化引擎自动消除不必要的警报噪音，在发出警报的同时提供补救解决方案，并提供对事件期间发生的所有通信和操作的可见性和记录。此外，通过使用出站 webhooks，整个 VictorOps 解决方案现在可以配置为实时向其他工具发送数据，使用户能够完成从自动修复标准问题到定制仪表板的所有工作。

“越来越多的情况下，加快组织创新的最佳方式是让开发人员快速行动，”VictorOps 首席执行官兼联合创始人托德·弗农说。“当涉及到公司的正常运行时间时，同样的概念也适用。如今，陈旧的警报系统和手动流程浪费了宝贵的时间。[事件自动化]引擎现在成为自动化解决这些低效问题、更清晰地访问可操作信息以及持续评估和改进推动随叫随到流程的输入的基础。”

事件自动化引擎的主要功能包括:

*   **警报转换**–消除不可响应的警报，并自动将特定警报发送给正确的团队成员。
*   **警报注释**–自动向警报添加运行手册、监控图表和其他相关信息，为警报接收者提供即时补救措施。
*   **出站 web hooks**–将 VictorOps 平台数据扩展到其他系统中，并将 VictorOps 事件集成到其他服务仪表板中。
*   **API**–利用 VictorOps API 将 VictorOps 扩展到现有的遗留工具和报告系统。
*   **事后报告生成器**–自动获取与停机相关的所有活动(警报、对话和补救措施)的快照。轻松使用这些数据来报告 SLA 并促进团队回顾。
