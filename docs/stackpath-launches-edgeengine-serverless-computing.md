# StackPath 推出 EdgeEngine 无服务器计算

> 原文：<https://devops.com/stackpath-launches-edgeengine-serverless-computing/>

*利用边缘计算创建自己的服务*

**DALLAS**—2018 年 11 月 27 日— [安全边缘服务的领先平台 StackPath](https://www.stackpath.com/?utm_campaign=Security&utm_source=Free%20Edge%20SSL%20Cert&utm_medium=Press%20Release&utm_content=Homepage) 今天宣布[stack path edge engine](https://www.stackpath.com/services/edgeengine)，这是一项简单而强大的无服务器计算服务，允许开发人员在云的边缘运行代码，无需服务器、虚拟机或容器。

StackPath 联合创始人兼首席执行官 Lance Crosby 表示:“边缘是云计算的新前沿。“EdgeEngine 使开发人员能够在边缘定制工作负载，而没有管理基础架构的负担或开销。这只是我们正在开发的许多边缘计算解决方案中的第一个，对于企业来说，这是一个前所未有的机会，可以控制数据和流量的处理方式，直到它们到达最终用户或影响工作负载。”

StackPath EdgeEngine 建立在 Chrome V8 JavaScript 引擎上，让 StackPath 客户只需将脚本编写或粘贴到 StackPath 客户门户中，并将其分配到其交付站点上的路线。该脚本会立即部署到 StackPath 在全球的所有高级 edge PoPs 中，随时准备在调用指定的脚本 URL 时在离最终用户最近的 PoP 中执行。该服务目前可作为 StackPath [CDN](https://www.stackpath.com/services/content-delivery-network-cdn/?utm_campaign=EdgeEngine&utm_source=EdgeEngine&utm_medium=Press%20Release) 和 [Edge Delivery](https://www.stackpath.com/pricing/?utm_campaign=EdgeEngine&utm_source=EdgeEngine&utm_medium=Press%20Release) 包的附加服务，但很快将可用于任何 StackPath 服务，或作为独立服务，并将支持多种语言。

StackPath 产品高级副总裁 Ben Gabler 表示:“在边缘创造动态用户体验的能力完全改变了游戏规则。“在 StackPath，我们不能总是预测我们将支持的每个功能或用例，但今天这不再是一个问题。凭借 EdgeEngine 的强大功能，客户的可能性是无限的。”

EdgeEngine 是各种用例的理想选择，包括:

*   **根据请求数据交付个性化、动态和独特的内容:**直接从边缘修改最终用户体验，而不是依赖客户端执行
*   定制安全措施:部署在请求到达内容之前执行的脚本，比如阻止或允许 WordPress 管理
*   **构建 API 入口点或整个 API 网关:**创建脚本，指定到交付站点的特定路径(根据需要任意多)
*   **降低总运营成本:**在边缘处理计算任务，而不是将其发送回原点，从而减少原点计算工作负载和数据传输
*   **改善总体最终用户体验:**处理来自最近的 edge PoP 的数据并回复给最终用户，降低响应时间并实现近实时网络应用

现有的 StackPath 客户现在可以使用 EdgeEngine。要开始使用它，StackPath 客户只需通过 StackPath 客户门户将其添加到 CDN 或边缘交付堆栈中。与竞争对手的无服务器边缘计算服务不同，EdgeEngine 的价格极具竞争力，每 1500 万次请求每月每堆栈 10 美元(每增加 100 万次请求 0.60 美元)，每堆栈的脚本或交付域数量没有限制。
