# Flowroute 推出通信应用程序，以加速将通话和信息整合到客户体验中

> 原文：<https://devops.com/flowroute-launches-communication-application-program-accelerate-integration-calling-messaging-customer-experience/>

华盛顿州西雅图市和伊利诺伊州芝加哥市–2016 年 8 月 8 日–[为 SaaS 公司提供高级呼叫和消息服务的领先供应商 Flowroute Inc.](https://www.flowroute.com/) 今天推出了其 Flowroute 通信应用程序。该计划旨在帮助开发人员加快上市时间，降低开发成本，并消除在其应用和服务中提供电信级消息传递功能的障碍。用 Python 编写的前三个应用程序模块包括电话号码屏蔽、双因素身份验证和约会提醒。

创建 Flowroute 的目的是为组织提供对其客户通信的无与伦比的控制。该计划旨在加速 SMS 消息在应用程序中的实施，使开发人员能够专注于产生引人注目的用户体验。初始应用程序容器包括:

*   [**电话号码屏蔽**](https://developer.flowroute.com/docs/sms-proxy)**–**通过短信代理，开发人员可以从一个支持文本的电话号码池中购买和提供电话号码，以匿名方式桥接并发的买方和卖方交互。这在需要通过文本消息匿名通信的市场应用用例中是必不可少的。
*   [**双因素认证**](https://developer.flowroute.com/docs/two-factor-authentication)**–**这通过在 web 和移动应用程序中应用双因素认证，为用户提供了额外的安全层。通过为开发人员提供高度的可配置性，通过 SMS 发送和验证 PIN 来保护交互。
*   [**预约提醒**](https://developer.flowroute.com/docs/appointment-reminder)——开发者可以让应用通过短信向用户发送预约提醒和确认消息。

Flowroute 将于本周在芝加哥举行的 2016 年 ClueCon】上介绍应用容器。Casey MacPhee，该项目的首席开发人员，将于 8 月 11 日(T2)周四(T3)美国东部时间(T4)下午 4:30-5:00(T5)展示利用这些模块的应用程序。

“作为一家公司，我们的目标是让高级通信对开发人员社区来说更容易、更容易。Flowroute 通信应用程序的成功是我们帮助客户解决这些有价值的使用案例的结果，”Flowroute 创始人兼首席产品官 Sean Hsieh 说。“我们很高兴能够继续向该计划添加新的应用，开发人员可以利用这些应用来区分他们的用户体验。”

此外，Hsieh 还将发表题为“让 Flowroute 的 SMS 为您服务”的演讲，届时他将演示如何将公司的 SMS API 集成到 FreeSWITCH 等通信平台中。

开发者可以在 [Flowroute 开发者门户访问应用容器和文档。](http://developer.flowroute.com/)
