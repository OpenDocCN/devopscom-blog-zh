# Cloudflare 将数字体验监控添加到 CDN 产品组合中

> 原文：<https://devops.com/cloudflare-adds-digital-experience-monitoring-to-cdn-portfolio/>

在在线 [CIO Week](https://www.cloudflare.com/cio-week/) 活动中，Cloudflare 今天在其核心[内容交付网络](https://devops.com/?s=content+delivery+network) (CDN)之上的服务组合中增加了数字体验监控功能。

Cloudflare 产品管理总监 Corey Mahan 表示， [Cloudflare 数字体验监控](https://www.businesswire.com/news/home/20230109005143/en/Cloudflare-Announces-New-Solutions-to-Help-CIOs-Maximize-Employee-Collaboration-and-Productivity)服务扩展了基于代理软件 Cloudflare WARP 的 Cloudflare 零信任平台。该解决方案提供了广泛的软件即服务(SaaS)产品，涵盖从数据丢失防护(DLP)到电子邮件安全的方方面面。

他补充说，总体目标是通过使用单个代理而不是为每项服务部署多个代理来更高效地交付 SaaS 应用服务，从而降低 IT 总成本。

cloud flare Digital Experience Monitoring service 作为私有测试版提供，将使 IT 团队能够利用 Cloudflare CDN，通过使用合成数据模拟流量来测量和分析流向公共或私有资源的受控数据流。Mahan 指出，通过与 Cloudflare Zero Trust platform 集成，IT 团队还可以访问实时数据可视化，突出显示连接或性能方面的异常，例如，这些异常可能表明存在安全漏洞。

尽管毫无疑问，IT 组织正在寻求精简他们需要管理的 IT 供应商的数量，但尚不清楚实现这一目标的重心是什么。Cloudflare 正在为使用许多组织已经依赖于网络服务的相同全球 CDN 提供理由。

Mahan 表示，Cloudflare 数字体验监控服务利用公司的 CDN 来简化理解，例如，为什么一个地区的用户与另一个地区的用户相比体验不佳。他补充说，当越来越多的组织拥有跨地域在家工作的员工时，这一点尤为重要。

从长远来看，更多的网络和安全功能将作为服务被消费。接下来的挑战是将这些服务集成到开发运维工作流中，这些工作流用于在高度分布式的计算环境中部署应用。在许多情况下，最初看起来可能是应用程序性能问题，结果却是更容易解决的网络连接问题。

无论如何，在未来的岁月里，IT 服务的管理将继续融合。各种规模的组织都需要确定购买他们需要管理的平台与消费代表他们管理的服务是否有经济意义。大多数情况下，服务更容易消费，并释放出可用于其他 IT 功能的资源，如应用程序开发。

当然，组织还需要注意避免将自己局限于特定的服务提供商。然而，最终的问题是管理一个特定的 IT 功能是否提供了足够的差异化价值，值得在有这么多事情要做的时候花费时间和精力。
