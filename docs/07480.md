# Checkmarx 推出基础设施作为代码扫描解决方案，以保护云原生应用

> 原文：<https://devops.com/checkmarx-launches-infrastructure-as-code-scanning-solution-to-secure-cloud-native-applications/>

*开源 IaC 扫描引擎是业界最全面的解决方案，支持开发人员检测和修复配置问题*

<time datetime="2021-02-25T14:05:00Z">February 25, 2021 09:05 AM Eastern Standard Time</time>

以色列拉马特甘—([商业资讯](https://www.businesswire.com/))—[devo PS 软件安全解决方案的全球领导者 Checkmarx](https://cts.businesswire.com/ct/CT?id=smartlink&url=http%3A%2F%2Fwww.checkmarx.com%2F&esheet=52384482&newsitemid=20210225005052&lan=en-US&anchor=Checkmarx&index=1&md5=55cf131113e0c1173518574e18aeace9) 今天宣布推出 [KICS](https://cts.businesswire.com/ct/CT?id=smartlink&url=https%3A%2F%2Fkics.io%2F&esheet=52384482&newsitemid=20210225005052&lan=en-US&anchor=KICS&index=2&md5=2642ca9f001ac588581d9889eee3cccf) (保持基础设施代码安全)，这是一款开源静态分析解决方案，支持开发人员编写更安全的基础设施代码(IaC)。通过 KICS，Checkmarx 扩展了其 AST 产品线，为保护传统和云原生应用的专有代码、开源组件和关键基础设施提供了单一平台。

近年来，随着组织向云的过渡，并寻求使基础设施的供应更快、更具可扩展性的方法，IaC 的采用率大幅上升。然而，伴随着 IaC 的所有好处而来的是大量的安全性、遵从性和配置风险，开发人员正在努力解决这些风险。当考虑到与错误相关的问题(例如，错误配置和错误交付)现在是数据泄露的第二大原因时，我们就意识到了这一点。

KICS 从 IaC 构建周期的一开始就自动检测漏洞、硬编码密钥和密码、合规性问题和错误配置，使开发人员能够在投入生产之前轻松修复这些缺陷。作为最全面的 IaC 扫描引擎，KICS 支持顶级的 IaC 技术，包括 Terraform、Kubernetes、Docker、AWS CloudFormation 和 Ansible。此外，KICS 提供 1200 多种完全可定制和可调整的查询，涵盖从加密和密钥管理到网络端口安全的 12 个类别。

“随着开发流程的发展和组织加速采用云技术，开发人员在以前所未有的速度交付软件的同时，也承担了更多的安全责任。Checkmarx 首席技术官兼创始人 Maty Siman 表示:“仅仅依靠耗时的手动代码审查是不可能取得平衡的。“KICS 正是基于这一点而构建的，它使开发团队能够在最快、最便宜、最容易修复的时候自动识别 IaC 问题。作为 Checkmarx 产品组合的最新成员，开发人员现在有一个单一的目的地来保护构成当今复杂应用程序的所有组件。”

KICS 的其他主要功能和优势包括:

*   **内置可扩展性:** KICS 提供了任何 IaC 扫描解决方案中最大的查询“库”，所有这些都是完全可定制和可调整的。此外，KICS 稳健而简单的架构允许快速添加对新 IaC 工具的支持。
*   **社区来源:**作为一个开源项目，KICS 的扫描引擎和查询都是清晰的，并对成千上万的安全和 DevOps 专家和软件开发人员开放。加上 Checkmarx 的专门团队不断增加新功能和审查贡献，KICS 能够快速扩展。
*   **无缝 CI/CD 集成:** KICS 可以轻松地与任何 CI/CD 管道集成，包括 GitHub Actions 和 GitLab CI，将漏洞和错误配置检查应用到 IaC，同时让开发人员使用他们首选的工具。

Siman 继续说道:“Checkmarx 是开源项目的大力提倡者，以这种方式创建 KICS 为社区提供了引导方向和促进整个行业创新的机会。我们很高兴看到这个充满热情的社区接受 KICS 并为其做出贡献，因为它已成为每个开发人员的云原生安全工具包的重要补充。”

“随着 KICS 的发布，我很自豪地欢迎 Checkmarx 加入开源生态系统，因为该公司将其丰富的 ast 经验带到了社区，”开源顾问兼布道者 Lior Kaplan 说。“KICS 已经看到参与开源的开发人员和安全专家的浓厚兴趣，随着项目规模的扩大和扩展到代码平台等更多基础设施，这种兴趣将继续增长。”
