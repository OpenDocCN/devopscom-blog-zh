# Infostretch 推出插件来简化向 Jenkins 2.0 的迁移并加速开发运维

> 原文：<https://devops.com/infostretch-launches-plugin-simplify-migration-jenkins-2-0-accelerate-devops/>

***首款将自由式 Jenkins 作业自动转换为编码 DevOps 管道的开源工具***

加利福尼亚州圣克拉拉——2018 年 2 月 14 日——企业数字解决方案提供商 Infostretch 今天为 Jenkins 推出了免费开源插件和服务，以帮助企业简化其 DevOps 管道开发。该插件大大减少了与编写 DevOps 管道和迁移到 Jenkins 2.x 相关的时间和工作量。该插件自动转换约 90%的自由式 Jenkins 作业，同时自动生成编码管道脚本，确保符合适当的编码标准和管道的可管理性。Jenkins 是世界上使用最广泛的开源连续交付自动化服务器，拥有大约 160 万用户。

使用 Jenkins 1.0 编排应用程序版本的组织目前面临着一项耗时的任务，即手动将作业迁移到他们的持续集成和持续交付管道中。当迁移到 Jenkins 2.x 时，转换自由式作业需要 DevOps 工程师深入到数百或数千个作业中的每一个，以了解工具、配置、URL、参数等，然后再用管道语法重写它们。Infostretch 面向 DevOps 团队的 Jenkins 插件自动生成脚本，将自由式 Jenkins 作业转换为编码管道。通过 Jenkins 2.x，用户获得了对连续交付转换的支持，对连续交付(CD)管道的一流支持，交付阶段的清晰可见性和其他可用性增强，以帮助用户更快地开始。

除了 Jenkins 插件，Infostretch 还可以通过定制插件来进一步加速 DevOps 管道，以应对 freestyle Jenkins 作业具有额外依赖性的情况。一旦工作进入 DevOps 管道，Infostretch 成熟的测试和 QA 支持可以确保转换后的管道正确高效地运行。Infostretch 还提供了关于管道即代码的 Jenkins 2.0 培训，以帮助加速 Jenkins 迁移，从而大大减少支持和维护问题。

现在，借助 Infostretch 插件和专家服务将 freestyle Jenkins 作业转换为编码管道，组织可以:
将 DevOps 工程师的手动工作减少多达 60%
强制执行编码标准
支持将任何新应用无缝、系统地过渡到 DevOps 管道
将应用维护成本降低 40%。
