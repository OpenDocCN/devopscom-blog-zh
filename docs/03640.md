# Compuware:调查显示，关键大型机工作负载不断增加，而大型机员工流失仍未填补

> 原文:[https://devo PS . com/compuware-survey-shows-critical-mainframe-workloads-increasing-while-mainframe-staff-loss-remaining-unfilled/](https://devops.com/compuware-survey-shows-critical-mainframe-workloads-increasing-while-mainframe-staff-losses-remain-unfilled/)

许多组织没有衡量、优先考虑重要的关键绩效指标

*   57%拥有大型机的企业目前在该平台上运行超过一半的业务关键型应用程序，这一数字预计到明年将增加到 64%。
*   72%面向客户的应用程序完全或非常依赖于大型机处理。
*   在过去五年中，企业仅补充了 37%的大型机劳动力。
*   许多接受调查的组织目前没有衡量速度或效率的关键绩效指标，这使他们在数字时代处于劣势。

底特律，2018 年 3 月 22 日(环球新闻网)-大型机对大型企业越来越重要，预计明年在该平台上运行至少一半业务关键型应用程序的组织比例将会增加。然而，熟练的大型机员工的流失，以及随后无法填补这些职位，对应用程序的质量、速度和效率构成了重大威胁。

这些是 Forrester Consulting 代表 [Compuware](http://www.compuware.com/) 进行的研究和分析的结果。根据该研究，“随着大型机工作负载的增加——受现代分析、区块链和更多移动活动冲击平台的驱动——以客户为中心的公司应该寻求实现应用交付的现代化，并消除创新的障碍。”

对美国和欧洲大型机决策者和开发人员的调查还显示:

*   大型机越来越重要

o 64 的受访企业将在明年在该平台上运行超过一半的关键应用程序，高于今年的 57%。

o 这些企业中 72%的面向客户的应用程序完全或非常依赖于大型机处理。

*   失去重要的主机工作人员

o 在过去五年中，企业平均损失了 23%的专业大型机员工。

o 这些职位中有 63%尚未填补。

*   未能优先考虑速度和效率

o 虽然 55%的受访者认为应用程序质量“极其重要”，但只有 42%的人同样重视开发速度，46%的人重视效率。

*   未能衡量关键 KPI

o 尽管企业认识到质量、速度和效率的重要性，但仍有很大一部分企业没有衡量它们(质量:27%，速度:38%，效率:39%)。

*   客户体验脱节

o 59 的人认为质量、速度和效率的提高意味着更好的客户体验；然而，只有 31%的人认为这种关注实际上减少了客户发现的缺陷。

这些发现和其他发现表明，大型机投资、文化和管理实践需要根据不断变化的市场现实进行重大改进。例如，对速度的重视不够是早于当今快节奏数字经济需求的早期时代遗留下来的问题。对效率的不充分关注同样反映了一个时代，在那个时代，大型机开发和质量保证人员非常多而且非常稳定。

根据 Forrester 的研究，“大型机是许多企业的核心，并且还在不断发展。IT 决策者不能将他们的大型机应用程序视为受长发布周期约束的静态环境，他们也不能忽视对退休员工的严重依赖。相反，企业必须实施必要的现代工具，不仅要提高大型机的质量，还要提高速度和效率，还要吸引有技能的人在平台上工作。”
